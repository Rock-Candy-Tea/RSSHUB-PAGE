h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h2 data-id="heading-0"><strong>一、 概述</strong></h2>
<p>为了统一前端开发的代码风格，以便后续代码的管理和维护，在使用前端开发框架时请遵循本文档制定的规范编写代码。 本文档涉及的规范主要包含日常代码书写最佳实践，以及项目开发中的一些注意事项。</p>
<h2 data-id="heading-1"><strong>二、前期准备</strong></h2>
<p>项目开始前，请务必配置好 eslint，安装相关依赖。eslint 规则列表，参照文档<a href="https://juejin.cn/post/6942365988042047496/" target="_blank">juejin.cn/post/694236…</a></p>
<p>项目目录结构：</p>
<p>├── build // 构建相关<br>
├── config // 配置相关<br>
├── src // 开发环境代码<br>
│   ├── api // 定义请求接口方法<br>
│   ├── assets // 存放字体文件、图片、icons<br>
│   ├── components // 通用组件<br>
│   ├── directive // 全局指令<br>
│   ├── filters // 全局filter<br>
│   ├── views // 路由<br>
│   ├── router // 路由<br>
│   ├── store // 全局store管理<br>
│   ├── styles // 全局样式<br>
│   ├── utils // 工具方法<br>
│   ├── App.vue // 入口页面<br>
│   ├── main.js // 入口 加载组件 初始化等<br>
├── static // 第三方不打包资源<br>
├── .babelrc.js // babel 相关配置<br>
├── .editorconfig // 编辑器统一配置<br>
├── .eslintrc.js // eslint 配置规则<br>
├── .gitignore // git 忽略配置<br>
├── favicon.ico // favicon图标<br>
├── index.html // html模板<br>
└── package.json</p>
<h2 data-id="heading-2">三、最佳实践</h2>
<p>1、变量命名，避免使用 var 声明变量：</p>
<pre><code class="copyable">- 全局常量，使用 const 声明，全部大写字母命名，多个单词直接用下划线分隔；

- 局部常量，使用 const 声明，小写字母开头，多个单词直接使用驼峰命名；

- 变量，使用 let 声明，小写字母开头，多个单词直接使用驼峰命名。
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img alt="11.png" class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bbcc6051155349c8815065a3f21a4f0a~tplv-k3u1fbpfcp-watermark.image" data-width="800" data-height="600" referrerpolicy="no-referrer"></p>
<p>2、声明函数，使用 () => 代替 function () &#123;&#125;;</p>
<p>3、完整语句的结尾，必须加分号；</p>
<p>4、注意空格，操作符（+ - * / = %，+=，-=， /=， *=，%=）前后需要保留一个空格，参数列表逗号与后面参数保留一个空格（如：(arg1, arg2, arg3, ...)）；</p>
<p>5、生产环境避免打印console.log, console.info ，debugger 等，在开发阶段，尽量使用浏览器打断点的形式调试，避免将打印日志信息提交到 git 仓库;</p>
<p>6、避免请求回调地狱，灵活使用 ES7 的 async 和 await ;</p>
<p><img alt="12.png" class="lazyload" src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f998ea1b461445728d0815512abc91e5~tplv-k3u1fbpfcp-watermark.image" data-width="800" data-height="600" referrerpolicy="no-referrer"></p>
<p>7、巧妙使用 map 数据解构，避免不必要的循环：</p>
<p>比如：有一个用户列表，要根据id找到某个用户，我们可以采用map的数据结构，用id做为key，避免循环遍历。</p>
