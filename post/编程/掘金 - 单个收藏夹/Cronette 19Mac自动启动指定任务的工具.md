<h2 data-id="heading-0">应用介绍</h2>
<p>Cronette是一款Mac自动启动指定任务的软件。它可以在无人参与的Mac服务器上使用，也可以在常规Mac工作站上使用，以便在夜间执行批量无人值守任务或定期执行定期任务。</p>
<p>Cronette提供了不同类型的启动任务，并且可以指定任何任务来启动不同类型的执行。</p>
<h3 data-id="heading-1">工作方法</h3>
<ul>
<li>
<p>Cronette内的所有任务都在文档表中表示。该任务在主界面和详细界面中可视化（众多Mac软件中使用的众所周知的布局，用户在主表格中选择任务，他可以在表格下查看和编辑可视化的详细信息）</p>
</li>
<li>
<p>用户可以插入记录他喜欢的任务</p>
</li>
<li>
<p>用户可以为任务分配一个名称和一个注释以识别它</p>
</li>
</ul>
<h3 data-id="heading-2">分配任务</h3>
<p>用户可以选择要运行的任务，不同的时间计划类型：</p>
<ul>
<li>
<p>每日：在任何一天运行</p>
</li>
<li>
<p>每周：指定要运行任何一周的哪一天</p>
</li>
<li>
<p>每月：指定要运行月份的哪一天</p>
</li>
<li>
<p>每年：指定要运行的月份和日期</p>
</li>
</ul>
<p>并指定：</p>
<ul>
<li>
<p>在每日/每周/每月/每年计划的几天内最多运行十个时间</p>
</li>
<li>
<p>如果每日/每周/每月/每年计划与当前日期匹配并且未达到指定的每日最大运行时间，就要在文档打开时运行。</p>
</li>
</ul>
<h3 data-id="heading-3">定时启动</h3>
<p>最后一项功能非常有用，可以根据时间执行备份，创建报告，电子邮件发送和任何其他您想要执行的任务，但如果当您关闭Mac并且仅在您指定的日期时，用户可以选择以下操作以便在任务启动时执行</p>
<ul>
<li>
<p>启动应用程序或文档</p>
</li>
<li>
<p>打开不同类型的URL</p>
</li>
<li>
<p>在日志中插入消息</p>
</li>
<li>
<p>直接从Cronette执行unix命令</p>
</li>
<li>
<p>复制文件或文件夹</p>
</li>
<li>
<p>移动文件或文件夹</p>
</li>
</ul>
<p>Cronette将按计划在后台启动任务。无需采取进一步行动。</p>
<h3 data-id="heading-4">基本功能的额外特性</h3>
<ul>
<li>
<p>允许使用Mac标准Cocoa控件指定任务（不需要编码）</p>
</li>
<li>
<p>任务插入时即刻诊断，显示错误存在（错误路径，不完整的时间执行输入等等）</p>
</li>
<li>
<p>显示“可运行”任务和“不可运行”任务，以便更好地理解文档中的内容</p>
</li>
<li>
<p>显示任何单独任务的时间安排</p>
</li>
<li>
<p>显示用户选定时间间隔内合并的所有任务的时间安排。用户可以观看未来...</p>
</li>
<li>
<p>在单独的日志窗口中显示任何文档的执行过程</p>
</li>
<li>
<p>它可以创建复制或移动文件的目的，您喜欢的新文件夹，使用日期标签（在执行任务时）指定当前日期依赖的名称，例如：2017-9-26-BACKUP_FILES</p>
</li>
</ul>
<h3 data-id="heading-5">优点</h3>
<ul>
<li>
<p>Cronette它是基于文档的，允许将不同的任务分组到单独的文档中，可以将Cronette作为实用工具运行，只需使用一个文档，并在启动时将其打开，而无需进一步的操作（首选项）</p>
</li>
<li>
<p>使用单独的线程启动进程，并且应用程序永远不会挂起耗时的任务</p>
</li>
</ul>
