                                                                                            <p>GoCD 21.2.0 已发布，GoCD 是一个持续集成和发布管理系统，其前身为 CruiseControl，它是 ThoughtWorks 在做咨询和交付项目时研发的一款开源持续集成工具。后来随着持续集成及持续部署的火热，ThoughtWorks 专门成立了一个项目组，基于 Cruise 开发出这款工具。</p> 
<p>此版本修复了重要的安全问题，官方建议所有用户升级到 21.2.0 版本，以保护 GoCD 服务器。</p> 
<h4>Changes</h4> 
<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgocd%2Fgocd%2Fissues%2F9086" target="_blank">#9086</a> - 升级所依赖的 Java 软件包至 Java 15 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgocd%2Fgocd%2Fissues%2F9086" target="_blank">#9086</a>)</li> 
 <li>从该版本开始，GoCD Agent <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fhub.docker.com%2Fr%2Fgocd%2Fgocd-agent-alpine-3.13" target="_blank">可使用</a>基于 Alpine 3.13 的 Docker 镜像</li> 
</ul> 
<h4>Bug fixes</h4> 
<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgocd%2Fgocd%2Fissues%2F8899" target="_blank">#8899</a> - 修复"Config Changed"差值标记包含破折号 (<code>-</code>)的行被删除的错误</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgocd%2Fgocd%2Fissues%2F9073" target="_blank">#9073</a> - 添加缺失的 Content-Type 检查</li> 
</ul>
