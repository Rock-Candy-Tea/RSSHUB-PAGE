h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h1 data-id="heading-0">基础概念</h1>
<ul>
<li><strong>UnionID</strong> 用于标识用户身份的标识，例如区分在微信小程序和微信h5端登录的两个账号是否为同一个用户</li>
<li><strong>appid</strong> 小程序 AppID 或第三方平台 AppID。如果有提供 project，该选项将忽略</li>
<li><strong>appsecret</strong> 小程序密钥</li>
<li><strong>session_key</strong> 用于保证数据安全完整性，存储在开发者服务器上的会话密钥，调用wx.login会导致上次session_key过期</li>
<li><strong>openid</strong> 支付用户唯一标识</li>
</ul>
<h2 data-id="heading-1">授权</h2>
<ul>
<li>部分接口需要用户授权才能调用</li>
<li>如果用户未接受或拒绝过此权限，会弹窗询问用户，用户点击同意后方可调用接口；</li>
<li>如果用户已授权，可以直接调用接口；</li>
<li>如果用户已拒绝授权，则不会出现弹窗，而是直接进入接口 fail 回调。<strong>请开发者兼容用户拒绝授权的场景。</strong></li>
<li>校开发者可以使用 <a href="https://developers.weixin.qq.com/miniprogram/dev/api/open-api/setting/wx.getSetting.html" target="_blank" rel="nofollow noopener noreferrer">wx.getSetting</a> 获取用户当前的授权状态。</li>
</ul>
<h3 data-id="heading-2">提前发起授权</h3>
<ul>
<li><a href="https://developers.weixin.qq.com/miniprogram/dev/api/open-api/authorize/wx.authorize.html" target="_blank" rel="nofollow noopener noreferrer">wx.authorize</a></li>
<li><code>wx.authorize(&#123;scope: "scope.userInfo"&#125;)</code>，不会弹出授权窗口，请使用 <a href="https://developers.weixin.qq.com/miniprogram/dev/component/button.html" target="_blank" rel="nofollow noopener noreferrer">``</a></li>
<li>需要授权 <code>scope.userLocation</code>、<code>scope.userLocationBackground</code> 时必须<a href="https://developers.weixin.qq.com/miniprogram/dev/reference/configuration/app.html#permission" target="_blank" rel="nofollow noopener noreferrer">配置地理位置用途说明</a>。</li>
<li></li>
</ul>

<h3 data-id="heading-4">授权有效期</h3>
<ul>
<li>一旦用户明确同意或拒绝过授权，其授权关系会记录在后台，直到用户主动删除小程序。</li>
</ul>
<h3 data-id="heading-5">版本兼容</h3>
<ul>
<li>wx.canIUse(string schema)判断小程序的API，回调，参数，组件等是否在当前版本可用</li>
</ul>
<h1 data-id="heading-6">登录</h1>
<ul>
<li>
<p>wx.login() 获取code，发送至后端服务器</p>
</li>
<li>
<p>后段服务器拿到前端的code，将</p>
<pre><code class="copyable">&#123;code, appid, appsecret, ...&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>发送给微信接口服务</p>
</li>
<li>
<p>微信接口服务返回</p>
<pre><code class="copyable">&#123;session_key, openid,...&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>后端把token返回给前端</p>
</li>
<li>
<p>前端将token保存到小程序storage，在下次业务请求时带上token进行请求</p>
</li>
</ul>
