<p style="text-align: center; "><img src="https://img.pintu360.com/editor/563529c5-4cdf-8305-9508-5f16ed211d35.png" alt style="font-size: 12px;" referrerpolicy="no-referrer"><br></p><p><br></p><p>这个一直在坊间流传着的选择题，再一次被拉到了台面上讨论。说起来也颇有意思，国民级社交应用、现象级短视频App是软件提供商，而手机行业巨头苹果则是硬件提供商，在业务不交叉的情况下，两者似乎不应该存在矛盾。</p><p>但苹果一向不按常理出牌，<b>它拿走了互联网广告行业的一块大蛋糕——削弱了开发者搜集用户数据，并以此描绘用户画像以便用于精准广告推送的能力。</b></p><p>但“ 智能相对论”看到，<b>苹果自己没有吃这块蛋糕，而是转身把它丢进了垃圾桶。</b>简而言之，为了保护用户隐私，苹果断人财路。</p><p>据《金融时报》报道，苹果近日正式对字节跳动和腾讯下达了最后通牒——<b>不要尝试绕过隐私功能，否则应用将被下架。</b></p><p>我们发现你的应用正在收集用户和设备信息以便为用户设备创建一个唯一识别符。”苹果在警告的邮件中称。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/046612a6-ec84-029a-48db-87dc8617eb4e.png" alt class referrerpolicy="no-referrer"></p><p>事情的起因在于，苹果在其最新的iOS 14系统中，新引入了一项针对开发者App隐私的内容—— “应用跟踪透明度(App Tracking Transparency，即ATT)”功能。</p><p>在更新iOS 14后，开发商需要向苹果提供他们如何搜集并使用用户数据，详细描述应用对用户各项数据追踪和使用的详情，并展示在App Store的应用下载界面。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/546c67f8-b848-a8d5-d975-e8e4b508d77a.png" alt class referrerpolicy="no-referrer"></p><p>而当即将到来的iOS 14.5版本更新后，应用程序如果要访问用户的信息，<b>就必须先通过弹窗征得用户的同意。</b></p><p>以前，移动互联网广告商可以通过IDFA（iOS设备广告标识符）来精准识别和跟踪用户，并投放定向广告。而在最新的iOS 14.5系统上，这一切都被暴露在用户面前，并且可以被拒绝。</p><p><b>所以，腾讯、字节跳动等中国科技巨头正在测试名为互联网广告标识(CAID)的工具，以规避苹果隐私调整。</b></p><p>得益于iOS系统的封闭性以及App Store的严格审查，苹果在个人隐私安全上，步子迈的远超微软、谷歌等同行。对于这种绕过隐私的模式，苹果表示应用开发者必须遵守新隐私规定，否认他们的应用可能就会被App Store下架。</p><p>其实，早在向中国地区的互联网巨头开战之前，也就是从iOS 14发布的去年六月份开始，扎克伯格和库克关于iOS 14 ATT政策就已经进行了旷日持久的口水仗——而且可预见的还会持续下去。</p><p>这场战斗席卷了与互联网广告业相关的整个行业，Android、Google Play之父谷歌、全球最大的手机厂商三星、Twitter以及许多而小体量玩家也都在观望，等待着这场战斗的结果。</p><p>在欧洲和美国，关于“苹果税”的问题上，App Store最近官司缠身，深陷反垄断调查的舆论之中。对此，苹果也做了很多让步，例如降低中小开发者15%的应用商店抽成等等，虽然控制能力有所减弱，但苹果还是那个苹果，在隐私问题上态度依旧强硬。</p><blockquote><p class="smallHead">苹果做了什么？</p></blockquote><p>回顾苹果2020年6月的发布会，库克着重强调了iOS 14系统对用户隐私的保护：如在摄像头、麦克风等信息敏感硬件被使用时提供明显的标识来告知用户；App读取剪切板内容后进行提示；使用模糊位置替代精确位置，满足不同应用对设备位置信息需求的同时，保护用户位置隐私等。</p><p>但最大的改变，在于iOS 14新引入了一项机制：“应用跟踪透明度(App Tracking Transparency，即ATT)”功能，它可以让用户自由选择是否分享自身的数据给应用开发商。</p><p>“ 智能相对论”了解到，苹果原计划在9月的iOS 14正式版上就推出ATT，但为了给开发者提供更多的准备时间，这项内容将在最近更新的iOS 14.5版本上线。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/bca7da04-6c6e-379c-88ee-5aad8f453eaf.png" alt class referrerpolicy="no-referrer"></p><p>对用户来说，ATT政策可以有效地保护数据和隐私安全，用户可以在详细了解后，自由选择是否提供给应用开发商隐私数据，而不是像<b>“一条砧板上的鱼”</b>一样无可奈何。</p><p>而对应用开发商，特别是广告商来说，这无疑是一个巨大的挑战。根据相关开发人员预计，当用户在应用程序内看到弹窗时，将有超过三分之二的人阻止追踪行为。</p><p>这会导致精准推送广告的失效——因为无法获取推荐算法赖以生存的用户信息。根据Facebook方面的预计，如果苹果推行ATT功能，那么其每年的广告收入可能会下降多达50%。</p><p>这简直是要了老命，Facebook当然坐不住了，在这项政策出炉后就忍不住狂轰苹果。去年12月16日，Facebook声称要“代表”小企业发声，在《纽约时报》、《华尔街日报》和《华盛顿邮报》上刊登整版广告，“我们为世界各地的小企业站出来反对苹果霸权”。</p><p>得益于App Store的严苛要求, “智能相对论”此次以对ATT政策反对声音最大的Facebook为例，看看这些“国民级App”究竟搜集了哪些用户隐私。</p><p><b>至于为什么以Facebook为例，是因为它正是这方面的典范——Facebook搜集的隐私数据基本上涵盖了所有可以搜集的内容。</b></p><p>此处是Facebook在App Store的App隐私主界面。我们可以看到，Facebook收集的信息一共有两种：用于追踪用户的数据；以及与用户关联的数据。</p><p>前者用于其他公司的App和网站中用来识别和追踪用户身份，后者则是本应用开发者搜集与用户相关的数据。从苹果公布的内容来看，后者所搜集的信息量可谓相当惊人——涵盖个人的方方面面。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/e559756f-7d0e-9958-e9dc-e134a955727d.png" alt class referrerpolicy="no-referrer"></p><p>而当“智能相对论”点开右上角的“查看详细信息”时，虽然有所准备，但依旧颇为震惊：<b>如果你是Facebook的用户，那么Facebook可能远比你自己还要了解你。有事情，你可能忘了，但“忠实”记录的Facebook会帮你记起来。</b></p><p>首先是<b>用于追踪的数据</b>，包括用户的实际地址、电子邮件、姓名、电话、用户的ID、设备的ID等等，作为识别网络上一个人的标识符，信息可谓相当详尽。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/835bdd80-d3c6-5efc-f230-26857b60b86c.png" alt class referrerpolicy="no-referrer"></p><p>其次是<b>与用户相关联的数据</b>的使用内容，第一项也是最重要的一项：用于第三方广告的隐私数据。</p><p>这些数据包括购买项目、财务信息、精准和粗略位置、联系信息、联系人、包括照片等在内的用户内容、搜索和浏览历史记录、标识ID、包括产品交互在内的使用数据等等。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/729e5433-fed8-0a53-5e0e-0ad15e6795a3.png" alt class referrerpolicy="no-referrer"></p><p>一 第三方广告</p><p>第二项则是<b>开发者广告或营销内容</b>，顾名思义，是与“第三方广告”相对应的“开发者广告”，所搜集使用的隐私数据与第三方广告内容基本相同，接下来不再赘述。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/1eda1f34-5a9a-b130-2dfb-70f6a1935850.png" alt class referrerpolicy="no-referrer"></p><p>二 开发者广告</p><p>接下来是第三项，用于<b>分析</b>的数据，增加了健康和健身等的身体数据，其他同上。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/e4134ee3-fd3c-a3cd-f4be-9860308c5de6.png" alt class referrerpolicy="no-referrer"></p><p>三 不知道有什么用的分析</p><p>第四项是用于<b>产品个性化</b>的数据，还是老样子。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/e57bc57b-cd8f-f48e-4bfa-19ea30f221f9.png" alt class referrerpolicy="no-referrer"></p><p>四 产品个性化，精准推荐主题？</p><p>第五项，用于<b>App功能</b>的，覆盖内容也很全面。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/7c9c6466-c9e2-7b89-de0d-c90bce708618.png" alt class referrerpolicy="no-referrer"></p><p>五App功能</p><p>最后一项，也就是第六项，是用于<b>其他目的</b>，具体内容不得而知。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/0247f708-6c80-76b5-d1a3-49ef8b2c16a3.png" alt class referrerpolicy="no-referrer"></p><p>六 不知所云的“其他”</p><p>依托对大量用户数据的分析，精准的广告推送撑起了Facebook 8000多亿美元的市值。</p><p>而Facebook能够收集这些数据，都依赖于IDFA。</p><p>IDFA是「Identifier for advertisers」的缩写，是每台iOS设备独有的类似产品序列号的数字与字母的组合，移动广告网络可以用它来跟踪用户，并投放定向广告。</p><p>在引入IDFA之前（也即iOS 6之前），苹果采用的是唯一设备标识符 (UDID)。相对IDFA而言，UDID是设备硬件的永久编号，分享后就无法取消分享，这导致开发者可以跨应用跟踪用户，甚至是违规出售。</p><p>最终，美国国会认定这种做法侵犯了个人的隐私权，苹果推出了可以重置和控制访问权限的IDFA取代了UDID。</p><p>也就是说，从iOS 6到iOS 14.5，苹果在设备隐私上进行了三次升级：从可以硬件追踪，到模糊硬件身份，再到用户可以主动拒绝，对用户隐私数据的保护，越来越严苛。</p><p><b>苹果为何警告字节和腾讯,事件的焦点源于CAID。</b></p><p>在iOS 14更新后不久，腾讯和字节跳动的绝大部分应用已经就苹果的隐私要求，提供了详尽的隐私标签。</p><p>“ 智能相对论”在App Store看到，以微信和抖音为代表，头条系和腾讯系的App都已经详细地描述了所搜集的内容，这一点是符合苹果目前规定的。而且，在即将到来的iOS 14.5版本中，这些App也会增加ATT开关，这是毫无疑问的，否则应用将会被下架。</p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/21d9ed63-87ac-6964-e4be-d4b2e2ad904d.png" alt class referrerpolicy="no-referrer"></p><p>抖音App提供了详细的App隐私内容</p><p>但问题并不仅仅在于此，而在于腾讯和字节跳动正在探索并采用一项新技术：<b>在符合苹果允许关闭IDFA的情况下，依旧能精准描绘用户画像的技术——CAID。</b></p><p align="center" style="text-align: center;"><img src="https://img.pintu360.com/editor/858334b2-28f0-67f4-a6b0-23e9e28007a5.png" alt class referrerpolicy="no-referrer"></p><p>来自中国广告协会的CAID技术</p><p>通过CAID技术，即便用户拒绝了App的IDFA权限，App依然可以对用户的各项数据进行追踪，这种追踪来自许多更加隐秘的方式：设备的软件和硬件特征对比，包括语言、电池电量、互联网连接状态、电子邮件地址，以及同一部设备的重复访问等，实现跨应用的追踪。</p><p>例如，字节跳动建议其开发者使用CAID1和CAID2标识符，CAID1基于设备的IP地址，CAID2则基于手机的IMEI，这显然是违反苹果规定的，因为它们在收集这些数据之前没有征求用户的许可。</p><p>根据《金融时报》的报道，中国“大大小小的公司”都在考虑使用CAID。虽然中国广告协会表示，用户可以选择退出CAID，但根据苹果的要求，这是不被允许的。</p><p>所以，苹果最近针对两大巨头——字节和腾讯的警告邮件或许会让这股热潮暂时熄灭，因为采取这项技术的App大概率会在iOS 14.5更新后被App Store下架处理，没有人敢冒着这个风险。</p><blockquote><p class="smallHead">字节和腾讯会认怂吗？</p></blockquote><p>答案是肯定的，此次事件大概率是以字节和腾讯的放弃CIAD技术并进行整改为收尾。</p><p>首先，如果不进行整改的话，App被苹果下架几乎是板上钉钉的事情——态度强硬的苹果向来说到做到，《堡垒之夜手游》至今无法在App Store上下载就是一个最好的证明。</p><p>而对字节和腾讯来说，目前国内互联网行业竞争激烈，此时与苹果进行“硬碰硬”显然不是一个好的选择，这也意味着“微信或者抖音和iPhone二选一”的情况大概率不会发生。</p><p>其次，CAID技术在很多地方违背了苹果的隐私政策，但在这件事情上，苹果并不是站在立法者的角度，作为iOS系统的开发者和维护者，苹果只是要求开发商提供一个权限，由用户自行判断是否提供隐私数据，这在法理上显然是站得住脚的。</p><p>这里可以参考一下欧洲的相关案例，3月17日，法国竞争管理局驳回了由广告游说机构们提出的终止苹果隐私新功能的请求。</p><p>法国当局认为，苹果的隐私新规没有滥用权力，理由则是：“一家公司，即使处于主导地位，原则上也有制定获取其服务规则的自由，但不得无视法律和适用法规，而苹果的这些规则并不反竞争。”</p><p>此外，随着用户隐私泄露事件的频发，国内消费者的数据隐私保护意识正在不断觉醒。此次事件苹果明显是站在消费者这一边的，即便它可能会从“保护消费者隐私安全”这项美名中获得间接受益。</p><p>无论如何，苹果的隐私政策显然是更利于消费者，在保护数据隐私安全的大背景下，字节和腾讯不会冒着舆论风险站在所有人的对立面。</p><p>即便是换一个角度，ATT政策对每个依赖互联网广告的厂商都有着极大地影响，但当灾难降临到所有人头上的时候，大家的压力会小得多——<b>最差不过是回到同一起跑线上。</b></p><p>此次苹果敢“冒天下之大不讳”也颇令人感慨，无论是在国家还是企业层面上，移动互联网时代的个人隐私保护，依然任重而道远。</p><p>*本文<span
