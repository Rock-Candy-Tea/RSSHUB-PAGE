<p>微软确认，<span class="accentTextColor">将会在 Win10 Build 19043.899（21H1）更新中，彻底从系统删除经典版 Edge</span>。我们知道从去年，微软就开始力推基于 Chromium 内核的全新 Edge 浏览器，现在全新 Edge 浏览器终于成为了系统中的唯一 Edge，经典版 Edge 浏览器就此成为了回忆。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002832_1603.jpg" w="700" h="149" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="149" referrerpolicy="no-referrer"></p><p>▲ 微软将在 Win10 21H1 中，彻底删除 Edge 经典版</p><p>对于很多软粉来说，经典版 Edge 浏览器的消失，令人叹息。作为微软最后一款自研内核的浏览器，为何经典版 Edge 不得不让位给 Chromium 版 Edge？这款曾被微软寄予厚望的浏览器，为何不得不走下历史舞台？一起来聊聊吧。</p><h2>Edge 经典版为何失败：自身表现欠佳</h2><p>微软对于经典版 Edge 浏览器投入过很多心血，早在 Win10 尚未推向市场时，经典版 Edge 浏览器就曾经以 “斯巴达”之名，在微软粉丝圈子中传颂。微软试图通过这款新产品，一改微软浏览器羸弱不堪的印象，“斯巴达”最终取名 Edge、而非传承 IE 的名号，就佐证了微软的决心。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002832_2859.jpg" w="700" h="467" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="467" referrerpolicy="no-referrer"></p><p>▲ 还有多少人记得 Edge 浏览器的起点——斯巴达项目</p><p>经典版 Edge 浏览器的确带来了很多亮点，它的效率非常不错，播放视频硬解能力强且省电，页面滑动顺畅、缩放网页平滑，和 IE 相比的确云泥之别。</p><p>然而，Edge 并不完美。尽管和 IE 相比进步巨大，但它依然存在很多令人无法忽视的缺点，这些缺点将它的用户不断推向以 Chrome 为代表的竞争对手。</p><p>· 更新迭代慢。和 Chrome、Firefox 疯狂刷版本的行径相比，Edge 经典版的更新可谓是慢如蜗牛。Edge 经典版需要和 Win10 大版本更新一起，才能得以升级，这在飞速发展的移动互联网时代，是难以跟上网络新特性和用户痛点的。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002832_8316.jpg" w="700" h="953" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="953" referrerpolicy="no-referrer"></p><p>▲ 5 年间，Edge 经典版就更新了这么几个版本</p><p>· 稳定性欠佳。Edge 经典版更新慢，带来的一大问题就在于很多缺点难以得到迅速修复。长久以来，Edge 经典版的稳定性一直存在问题，闪退、卡死频发，标签页容易出现假死或者重载等情况，当电脑从休眠中恢复时，Edge 经典版更是有不小的概率崩溃，这都让人很难将 Edge 经典版作为自己的首选浏览器。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002832_8002.jpg" w="700" h="441" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="441" referrerpolicy="no-referrer"></p><p>▲ 很长一段时间，用户都为 Edge 经典版的稳定性问题困扰</p><p>· 功能欠缺。这可能是 Edge 经典版的致命伤。直到生命终结，Edge 经典版的功能仍算得上是羸弱，它仍未支持界面定制、自定义字体等常见功能，色彩管理、多媒体控制等专业功能也难以令人满意。特别是对于扩展支持的这一点，令它和 Chrome、Firefox 等浏览器的功能差距愈发巨大。尽管在发布一年后的 2016 年，Edge 经典版可以支持扩展程序，但并不能直接兼容 Chrome 扩展，开发者需要使用转制工具将 Chrome 扩展转为 Edge 经典版的兼容版本，最终 Edge 的扩展还是寥寥无几。</p><p>总体来说，和 Chrome 相比，Edge 经典版的综合产品力是有所不如的，但这并不是 Edge 经典版失败的全部原因。</p><h2>Edge 经典版为何失败？微软互联网话语权旁落</h2><p>实际上，近年来放弃自研内核的浏览器，并不只有 Edge 一家。例如，曾经引领了诸多潮流功能的 Opera，早几年就放弃了 Presto 内核，改用和 Chrome 一样的 Blink 引擎。这次微软放弃 EdgeHTML 拥抱 Chromium，其实也算顺应了业界潮流。</p><p>除了 Firefox 和 Safari，市面上使用自研内核的浏览器，已经几乎不存在。放眼望去，Chromium 系浏览器已经成为了绝对的主流，这和近年来浏览器市场生态变化有很大关系。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002833_8944.jpg" w="700" h="284" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="284" referrerpolicy="no-referrer"></p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002833_9415.jpg" w="700" h="333" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="333" referrerpolicy="no-referrer"></p><p>▲ 无论是桌面平台还是移动平台，Chrome/Chromium 都已经占据了绝对的话语权</p><p>在移动互联网时代，Android 通过移动平台，攻占了大量的份额，而 Android 系统所使用的 WebView 和 Chromium 同源，这令大量网页都倾向于针对 Chromium 优化。尽管谷歌一直自诩推动 Web 标准化，但其实却不断诱导前端开发者去做标识 “-webkit- ”这样的事情，这让其他浏览器内核出现了一些兼容问题。例如，Edge 经典版在 YouTube 上就遇到过兼容问题，自研内核在这个时代，显得吃力而不讨好。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/3/20210322002833_9886.jpg" w="700" h="573" title="微软彻底删除 Edge 经典版，后者为何失败" width="700" height="573" referrerpolicy="no-referrer"></p><p>▲ 谷歌用了一些小手段巩固 Chrome 地位，例如 Firefox、Edge 开启 YouTube 需要额外的加载时间</p><p>换言之，兼容 Chromium 已经成为了事实上的行业标准，相比使用 Chromium 内核的马甲浏览器，自研引擎的浏览器们越来越举步维艰。Chromium 是一个开源项目，多年的发展加上开源社区的维护，令它成熟度极高——特别是和 Edge 这样的新项目相比，不仅成本低而且问题少。对比下来，自研引擎吃力不讨好，费钱还没有好的效果，Opera 和 Edge 放弃自研引擎转用 Chromium，也就可以理解了。</p><p>在此背景下，Edge 经典版的落幕是在情理之中的。微软并未能通过 WP 守住移动市场的立足点，由此丧失了移动互联网时代的流量入口，自然也失去了新时代 Web 标准的话语权。换言之，即使 Edge 经典版自身没有种种缺点、甚至比 Chrome 还更加出色，但在流量霸权已经从桌面转移到移动平台的这个时代，仍免不了失败的最终结局。</p><h2>总结</h2><p>总的来说，Edge 经典版的失败既有自身的原因，但更多是反映了微软在移动互联网时代话语权的旁落。无论如何，Edge 新版的确是综合体验更好的产品，就让我们目送 Edge 经典版的落幕吧。</p>
