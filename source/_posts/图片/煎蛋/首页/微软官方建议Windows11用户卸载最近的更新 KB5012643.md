<blockquote><p>4月25日发布的KB5012643会导致崩溃</p></blockquote><img src="https://cors.zfour.workers.dev/?http://img.jandan.net/news/2019/01/c28bab494748ce8e1d77321ff32dc849.jpg!custom" referrerpolicy="no-referrer"><p>微软建议 Windows 11 用户卸载最近的更新。报告显示更新是可选的，有可能导致各种应用程序崩溃。问题涉及更新与作为 Windows 一部分的 .Net Framework 之间的交互。目前尚不清楚哪些应用程序会受到影响，因此卸载是唯一可行的解​​决方案。</p>
<p>有问题的更新是 4 月 25 日发布的 KB5012643。这是对操作系统的累积更新，有很多小的变化。值得注意的是，更新的注释有一个标有“此更新的已知问题”的部分。在该部分中，它说可能会导致一些 .Net Framework 3.5 应用程序停止工作。</p>
<p>微软表示，“受影响的应用正在使用 .NET Framework 3.5 中的某些可选组件，例如 Windows Communication Foundation (WCF) 和 Windows Workflow (WWF) 组件。” </p>
<p>.Net 框架在 Windows 中无处不在，并被各种应用程序开发人员用作它可以使用的代码基础。安装此更新后，其中一些应用程序现在将崩溃或无法打开。正如 Windows Latest 所指出的，此更新仅适用于使用 Windows 11 21H2 的用户。</p>
<p>Windows 11 中的卸载更新窗口唤起了旧版操作系统的痕迹。</p>
<p>要卸载此更新，请按照下列步骤操作：</p>
<li>从开始菜单中找到 Windows 更新设置。</li>
<li>在该窗口中单击查看更新历史记录。</li>
<li>单击卸载更新。</li>
<li>找到 KB5012643 并单击卸载。</li>
<p>如果失败，您还可以尝试在控制面板中启用 .NET Framework 3.5。如果您是 IT 管理员或高级用户，您还可以通过提升的命令提示符启用它。这是命令：</p>
<h4 class="pullquote">dism /online /enable-feature /featurename:netfx3 /all<br>
dism /online /enable-feature /featurename:WCF-HTTP-Activation<br>
dism /online /enable-feature /featurename:WCF-NonHTTP-Activation</h4>
<p>除了上述外，此更新还破坏了安全模式。微软表示，当用户在没有联网的情况下启动到安全模式时，用户可能会看到屏幕闪烁。根据微软的说法，“依赖 explorer.exe 的组件，例如文件资源管理器、开始菜单和任务栏，可能会受到影响并且显得不稳定。” Microsoft 已经为此发布了一个已知问题回滚 (KiR)，因此应该对其进行修复。如果遇到它，您应该能够通过在安全模式下启用网络支持来解决它。</p>
<p>这只是 Windows 11 更新体验中的另一个日志条目，到目前为止，Windows11一直是情绪过山车。尽管该公司已经推出了许多积极的改变，比如启用一键浏览器默认设置，但<strong>大多数用户在看到“Windows 正在更新”通知时就开始冒汗</strong>。</p>
<p>令人遗憾的是，这让人想起去年发布的少量 Windows 10 更新。当时，著名的 Windows 10 与打印机驱动程序联手导致蓝屏死机。它还在去年四月的更新中破坏了游戏性能。下载可选更新时要小心，包括累积更新和驱动程序更新。另一方面，出于安全考虑，仍应及时更新……</p>
