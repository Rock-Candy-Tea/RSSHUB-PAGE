<blockquote><p>HTTPie 现在是 GitHub 上总体上最受欢迎的 80 个公共存储库之一</p></blockquote><img src="https://cors.zfour.workers.dev/?http://img.jandan.net/news/2019/01/06d5feaa083029f4aab794aa04d5d3c4.jpg!custom" referrerpolicy="no-referrer"><p>“从 2012 年 2 月 25 日在多雨的哥本哈根发布HTTPie 的第一个公开版本以来，我们始终把项目托管在 GitHub 上。”</p>
<p>根据开发者的说法，HTTPie 是一个开源 CLI HTTP 客户端。 HTTPie 的不同之处在于，他们从头开始构建它，以使终端的 API 交互尽可能人性化。如今HTTPie 项目已有十年的历史，积攒了超5.4万个star。</p>
<h4 class="pullquote">GitHub为用户的repo提供一些状态，如watch、star、fork。如果你看到了某个项目，然后点击该项目页上角的<strong>watch</strong>，效果大致相当于特别关注，该项目的每次更新你都能受到提示。类似地，<strong>star</strong>相当于点赞，对项目开发者来说是一种荣誉和支持；对点赞者来说，如果之后你忘了相关项目的信息，记得自己star过，你就可以在自己star历史列表里找到这一项目。</h4>
<p>“我记得 HTTPie 第一次成为 Hacker News 的热门链接并看到 GitHub 社区建立起来的热潮。多年来，随着我们不断改进该项目，它不断吸引着广泛的采用。它成为平台上最受欢迎的 API 工具，GitHub 社区增长到 54k 观星者和 1k+ 观察者。”</p>
<p>HTTPie 现在是 GitHub 上总体上最受欢迎的 80 个公共存储库之一；超过99.99997203%的项目。但是几天前，HTTPie失去了它的全部star……</p>
<p>HTTPie的开发者某次手误，把仓库设置成私有(类似于仅自己可见)，等他恢复公开设置后，意外发现全部star被清零。</p>
<p>原来这是 GitHub 的设定，只要仓库被设置成私有，就会清零star。同时，再开放后也不会恢复。</p>
<p>参考 https://httpie.io/blog/stardust</p>
