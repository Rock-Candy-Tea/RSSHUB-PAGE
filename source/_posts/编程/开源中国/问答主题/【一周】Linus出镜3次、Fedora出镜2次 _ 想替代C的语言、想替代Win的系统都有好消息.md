<h4><strong><a href="https://www.oschina.net/news/117098/make-serpent-a-truly-modern-linux-distribution" target="news" rel="nofollow">Solus Linux 作者正在打造“真正现代化的 Linux 发行版”，不依赖 GNU</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">“这不是‘面向大众的 Linux’，而是一个还原 Linux 真正该有的样子的 Linux”。同时，这一发行版既模块化又针对现代机器进行优化。</span></p> 
<h4><strong><a href="https://www.oschina.net/question/4518215_2317569" target="news" rel="nofollow">不使用 Kubernetes 发行版的 5 个理由</a></strong></h4> 
<p><span>目前有不少公司基于 Kubernetes 封装了自己的商用 Kubernetes 发行版，丰富了开发生态，也提供给开发者更多选择。最近有人提出，选择发行版总体上需要更少的精力和专业知识，但因为种种理由，组织机构其实应该考虑使用原生 Kubernetes。</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117111/linux-new-terminology-bans-master-blacklist" target="news" rel="nofollow">Linus Torvalds 通过了 Linux 中避免 master/slave 等术语的提案</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">Linux 内核维护者 Dan Williams 曾于 7 月初提交一份提案，建议逐步取消 master/slave 和 blacklist/whitelist 术语。</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117118/announcing-zig-software-foundation" target="news" rel="nofollow">想替代 C 的 Zig 语言成立了基金会</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">Zig 是一门通用编程语言，专为稳定性、可维护性和性能而设计，追求替代 C 语言在系统编程上的最佳地位。</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117135/microsoft-not-going-to-officially-support-php" target="news" rel="nofollow">微软宣布 Windows 将不提供 PHP 官方支持</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">目前微软在 Windows 上通过开发和构建 PHP 7.3 和 PHP 7.4 支持 PHP，同时当需要安全修复程序时，微软也帮助在 Windows 上构建 PHP 7.2。但是，“我们不会提供 8.0 及更高版本的任何能力支持 Windows 的 PHP。”</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117139/linus-linux-rust-support-remark" target="news" rel="nofollow">Linus Torvalds 对 Linux 内核支持 Rust 的看法：默认可以有效启用</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">Greg 希望 Linux 中的 Rust 支持是以可选的方式存在，而不能全面默认启用，类似于在一个大的系统配置项里还要单独开启一个针对 Rust 的小选项，但 Linus 则认为应该在</span>默认情况下可以有效地启用支持<span style="background-color:#ffffff; color:#333333">，以确保进行广泛的测试，而不是只有某些开发者孤立地在进行疯狂/错误的使用，因为没有人关注，问题会被掩盖。</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117168/ibm-kubernetes-experience-job-ad" target="news" rel="nofollow">IBM 招聘广告要求应聘者具备至少 12 年 Kubernetes 使用经验</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">Kubernetes 项目在 GitHub 的</span>初次提交<span style="background-color:#ffffff; color:#333333">时间为2014年6月7日，直到2015年5月22日才宣布 1.0 版本进入功能冻结阶段，也就是说诞生至今才 6 年，试问如何在一个 6 年前发布的项目中积累十几年的经验。</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117170/fedora-33-nano-is-the-default" target="news" rel="nofollow">Fedora 33 确认使用 Nano 作为默认文本编辑器</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">与目前的默认编辑器 Vi 相比，Nano 提供的文本编辑图形界面对新手更友好，也在一定程度上降低了使用门槛。当然，Vi 在 Fedora 33 中也会保留，不过不再是默认选项。</span></p> 
<h4><strong><a href="https://www.oschina.net/news/117171/linux-mint-forbid-snapd" target="news" rel="nofollow">未经授权擅自覆盖软件包，Linux Mint 放弃支持 Snap</a></strong></h4> 
<p><span style="background-color:#ffffff; color:#333333">Snap 包含了软件运行需要的所有依赖关系，并将其全部捆绑在一个系统镜像中，这在一定程度上简化了软件管理过程。不仅如此，统一化的 Snap 软件包能够在不同 Linux 发行版上运行，开发者无需再针对不同的发行版分别构建，软件部署的难度有所降低，软件开发和最终部署之间的延迟也得以缩减。</span></p>
