<div class="content">
                                                                                            <p>Visual Studio Code 1.68 现已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68" target="_blank">发布</a><span style="color:#000000">，<span style="background-color:#ffffff">一些主要亮点更新包括：</span></span></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_configure-display-language-improvements" target="_blank">Configure Display Language</a></strong>- 查看已安装和可用的语言包的语言。</li> 
</ul> 
<p><img alt height="245" src="https://oscimg.oschina.net/oscnet/up-5e80a322d682fd9c9d66465b4ce95c85ea8.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_problems-panel-table-view" target="_blank">问题面板表格视图</a></strong>- 以表格形式查看错误和警告，以快速过滤其来源。</li> 
</ul> 
<p>在此版本中为用户添加了一项新功能，可以在 tree 和 table 之间切换问题面板的视图模式。与树状视图相比，表格显示了每个问题的来源（语言服务或扩展），允许用户按来源过滤问题。</p> 
<p><img alt height="65" src="https://oscimg.oschina.net/oscnet/up-befdb70657dff3b021b96911dd1ad0c731d.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="140" src="https://oscimg.oschina.net/oscnet/up-a644206c6b2c7ef04a9834431a524744613.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_deprecated-extensions" target="_blank">不推荐使用的扩展</a></strong>- 了解扩展是否已弃用或应该被替换。</li> 
</ul> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>添加了对 VS Code 中已弃用的扩展的支持。一个扩展可以被简单地弃用或弃用以支持另一个扩展，或者当它的功能内置到 VS Code 中时。VS Code 将在 Extensions 视图中呈现已弃用的扩展，如下所示。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>不再维护的已弃用扩展。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><img alt height="76" src="https://oscimg.oschina.net/oscnet/up-01fb03c2474020778ba6e89f59a6508e964.png" width="500" referrerpolicy="no-referrer"></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>一个扩展被弃用，取而代之的是另一个扩展。在这种情况下，VS Code 不允许用户安装这个扩展。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><img alt height="86" src="https://oscimg.oschina.net/oscnet/up-20fe8934061828b144179bc9069eacd2b18.png" width="500" referrerpolicy="no-referrer"></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>一个已弃用的扩展，其功能内置于 VS Code，可通过配置设置启用。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><img alt height="77" src="https://oscimg.oschina.net/oscnet/up-66f5c9e99cbb7a69f7c7880a6c1f155f1fe.png" width="500" referrerpolicy="no-referrer"></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>VS Code 不会自动迁移或卸载已弃用的扩展。将有一个<strong>迁移</strong>按钮来指导用户切换到推荐的扩展。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><img alt height="116" src="https://oscimg.oschina.net/oscnet/up-9fbc007273fb85bfe497b80a9ca6a66b02a.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_sponsoring-extensions" target="_blank">扩展赞助</a></strong>- 支持构建你喜欢的扩展的开发人员。</li> 
</ul> 
<p>VS Code 现在允许用户赞助他们最喜欢的扩展。当一个扩展可以被赞助时，VS Code 将在 Extensions view Details 页面中呈现一个 <strong style="color:#444444">Sponsor 按钮，如下所示：</strong></p> 
<p><img alt height="297" src="https://oscimg.oschina.net/oscnet/up-161f6b534e740628d5a3a60b3136177cde1.png" width="500" referrerpolicy="no-referrer"></p> 
<p><strong style="color:#444444">Sponsor </strong>按钮会将用户定向到扩展程序的赞助 URL，用户可以在其中提供支持。</p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_hide-files-in-explorer-based-on-gitignore" target="_blank">使用 .gitignore 隐藏资源管理器文件</a></strong>- 重用现有<code>.gitignore</code>资源以隐藏资源管理器中的文件。</li> 
</ul> 
<p>文件资源管理器现在支持解析和隐藏被你的 .gitignore 文件排除的文件。这可以通过 <strong style="color:#444444">Explorer: Exclude Git Ignore</strong><span style="background-color:#ffffff; color:#444444"><span> </span>(</span><code>explorer.excludeGitIgnore</code><span style="background-color:#ffffff; color:#444444">)<span> </span></span>设置启用。这个设置与 files.exclude 一起工作，以从资源管理器中隐藏不需要的文件。</p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_terminal" target="_blank">终端颜色和对比度增强</a></strong>- 查找匹配的背景颜色、最小对比度。</li> 
</ul> 
<p><img alt height="181" src="https://oscimg.oschina.net/oscnet/up-6df4f2f8da3560fc77d66061ef9ea473d5a.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="131" src="https://oscimg.oschina.net/oscnet/up-57d48b88406e0eb1027b63864396943d26d.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="126" src="https://oscimg.oschina.net/oscnet/up-bfea649eeaaf696e444bb8010044aaeadca.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="66" src="https://oscimg.oschina.net/oscnet/up-52ad47234abac7103bf868a316a64c4653a.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="171" src="https://oscimg.oschina.net/oscnet/up-daa3401d68824f15002e7e21a88e59d1b22.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_git-branch-protection" target="_blank">Git 分支保护</a></strong>- 在 VS Code 中提供分支保护。</li> 
</ul> 
<p>通过新的 git.branchProtection 设置，你可以配置特定的分支进行保护。VS Code 将避免在受保护的分支上直接 committing，而是提供机会让你创建一个新的分支来代替 commit。你可以通过 git.branchProtectionPrompt 设置微调此行为。</p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_go-to-source-definition" target="_blank">TypeScript Go to Source Definition</a></strong> - 直接跳转到符号的 JavaScript 实现。</li> 
</ul> 
<p>当你从编辑器上下文菜单或命令面板运行 <strong style="color:#444444">Go to Source Definition </strong>命令时，TypeScript 将尝试跟踪符号的 JavaScript 实现并导航到它。这可能需要花费几秒钟的时间，且可能并不总是得到正确的结果，但它在许多情况下应该很有用。</p> 
<p><img alt height="307" src="https://oscimg.oschina.net/oscnet/up-387e3220cc735aaa18f724f2420d58d6964.gif" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_core-localization-support" target="_blank">VS Code for the Web 本地化</a></strong> - vscode.dev 现在与你选择的浏览器语言相匹配。</li> 
</ul> 
<p><span style="background-color:#ffffff; color:#444444">VS Code for Web 的初始本地化支持。</span><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>对于将浏览器设置为支持的核心语言之一的用户，<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fvscode.dev%2F" target="_blank">vscode.dev</a> 将自动应用该语言的翻译。支持的语言记录在 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmicrosoft%2Fvscode-loc" target="_blank">vscode-loc</a> 存储库中。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>例如，要在 Microsoft Edge 中配置显示语言，可以使用 <strong>Settings</strong> > <strong>Languages</strong>：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p>

<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_development-container-specification" target="_blank">开发容器规范</a></strong>- 了解有关不断发展的开发容器规范的更多信息。</li> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_68%23_markdown-link-validation" target="_blank">预览：Markdown 链接验证</a></strong>- 检测到 <span style="background-color:#ffffff; color:#444444">headers、images 和 files </span>的损坏链接。</li> 
</ul> 
<p>通过链接验证，VS Code 将分析指向 <span style="background-color:#ffffff; color:#444444">headers、images</span>和其他本地文件的 Markdown 链接。无效链接将报告为警告或错误。<span style="background-color:#ffffff; color:#444444">VS Code 甚至可以捕捉到其他 Markdown 文件中特定 headers 的无效链接。</span></p>
