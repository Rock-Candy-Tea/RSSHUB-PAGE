                                                        <p>Node.js 15.13.0 正式发布。Node.js 是能够在服务器端运行 JavaScript 的开放源代码、跨平台 JavaScript 运行环境。Node.js 由 Node.js Foundation（已与 JS Foundation 合并为 OpenJS Foundation）持有和维护，亦为 Linux 基金会的项目。Node.js 采用 Google 开发的 V8 运行代码，使用<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fzh.wikipedia.org%2Fwiki%2F%25E4%25BA%258B%25E4%25BB%25B6%25E9%25A9%2585%25E5%258B%2595" target="_blank">事件驱动</a>、<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fzh.wikipedia.org%2Fw%2Findex.php%3Ftitle%3D%25E9%259D%259E%25E9%2598%25BB%25E5%25A1%259E%26action%3Dedit%26redlink%3D1" target="_blank">非阻塞</a>和<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fzh.wikipedia.org%2Fwiki%2F%25E5%25BC%2582%25E6%25AD%25A5IO" target="_blank">异步输入输出</a>模型等技术来提高性能，可优化应用程序的传输量和规模。这些技术通常用于资料密集的即时应用程序。</p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li>实现 btoa 和 atob</li> 
 <li>npm 升级为 7.7.6</li> 
 <li>在稳定性指数中添加遗留状态</li> 
 <li>添加 http.ClientRequest.getRawHeaderNames() 方法</li> 
 <li>删去多余的参数</li> 
 <li>清理 AbortSignal 重复</li> 
 <li>清除 SignTraits :: DeriveBits 中的错误</li> 
 <li>修复 DiffieHellman 参数验证</li> 
 <li>对 NodeCryptoError 使用宏映射</li> 
 <li>修复 crypto.verify 回调调用私钥对象的问题</li> 
 <li>修复 libinternalbootstrappre_execution.js 中的错别字</li> 
</ul> 
<p>详情请查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fnodejs.org%2Fen%2Fblog%2Frelease%2Fv15.13.0%2F" target="_blank">更新公告</a>。</p>
