<p><img height="281" src="https://oscimg.oschina.net/oscnet/eb635001-a295-45eb-96ad-7cd52e7aaf23.png" width="500" referrerpolicy="no-referrer"></p> 
<p><strong>Curiefense v1.4.0</strong>[1]现已可用。这是一个里程碑式的版本；随着一系列的更新和增加，Curiefense 现在已经与 NGINX 集成。</p> 
<p>NGINX 是世界上最流行的网络服务器，它还有其他用途。现在用户可以在他们的环境中添加内置的网络安全和流量过滤。</p> 
<p><strong>Curiefense</strong>[2]是 NGINX 中第一个完全集成的、全面的 web 安全解决方案。此次发布的重点内容包括：</p> 
<ul> 
 <li> <p>一个一体化的 web 安全套件，包括 WAF、L7 DDoS 缓解、bot 管理、会话流控制、速率限制、API 保护等等。</p> </li> 
 <li> <p>对所有用户免费，不需要额外的附加组件或订阅。</p> </li> 
 <li> <p>所有处理都在用户的范围内执行。（大多数解决方案在外部基础设施上解密和过滤流量，这增加了延迟，并在数据隐私方面造成潜在的危害。Curiefense 驻留在用户的环境中。）</p> </li> 
 <li> <p>开放源代码的可用性和可扩展性，而不是封闭源代码/黑盒解决方案。</p> </li> 
 <li> <p>GitOps 和开发者友好的平台，由 web 控制台 UI 和 REST API 驱动。</p> </li> 
 <li> <p>全流量可见性和实时指标，具有开放和可扩展的仪表板和流量日志。</p> </li> 
</ul> 
<p><strong>Reblaze</strong>[3]于去年 11 月在玛丽·居里的生日当天推出了 Curiefense。GA 发布于 3 月份，并提供了对 Envoy Proxy 的支持，包括 Kubernetes 之类的容器化部署和 Istio 之类的服务网格。</p> 
<p>现在有了 Curiefense 1.4 版，NGINX 也得到了完全支持，目前有超过 4 亿个网站在使用 NGINX。用户现在可以将 Curiefense 的综合安全技术套件作为其环境的固有部分，在保持高性能和完全隐私的同时阻断恶意流量。</p>
