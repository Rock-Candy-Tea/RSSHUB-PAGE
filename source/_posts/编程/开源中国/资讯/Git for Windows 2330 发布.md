                                                                                            <p>Git for Windows 2.33.0 现已发布，主要更新内容如下：</p> 
<p><strong>New Features</strong></p> 
<ul> 
 <li>随附 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit%2Fgit%2Fblob%2Fv2.33.0%2FDocumentation%2FRelNotes%2F2.33.0.txt" target="_blank">Git v2.33.0</a>。</li> 
 <li>随附 <a href="https://www.oschina.net/action/GoToLink?url=http%3A%2F%2Fsearch.cpan.org%2Fdist%2Fperl-5.34.0%2Fpod%2Fperldelta.pod" target="_blank">Perl v5.34.0</a>（和一些更新的 Perl 模块）。</li> 
 <li>现在可以要求 Git for Windows 使用 PATH 上的 SSH，而不是其捆绑的 OpenSSH 执行程序。</li> 
 <li>随附 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmicrosoft%2Fgit-credential-manager-core%2Freleases%2Ftag%2Fv2.0.498" target="_blank">Git Credential Manager Core v2.0.498.54650</a>。</li> 
 <li>实验性的 FSMonitor 补丁被<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit-for-windows%2Fgit%2Fpull%2F3350" target="_blank">更新的版本</a>所取代。</li> 
 <li>随附 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flists.gnupg.org%2Fpipermail%2Fgnupg-announce%2F2021q3%2F000461.html" target="_blank">GNU Privacy Guard v2.2.29</a>。</li> 
</ul> 
<p><strong>Bug 修复</strong></p> 
<ul> 
 <li> <p>在选中 Windows 终端配置文件选项时，安装程序在升级时不再<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit-for-windows%2Fgit%2Fissues%2F3312" target="_blank">显示错误对话框</a>。</p> </li> 
 <li>与<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgerrit.googlesource.com%2Fgit-repo%2F" target="_blank"><code>git repo</code></a>工具的互动得到了改进。</li> 
 <li>捆绑在 Git for Windows 中的 GNU Privacy Guard (GPG)版本在 64 位设置中不起作用，这一点已经被修复。</li> 
</ul>
