                                                                                            <p style="text-align:start">Spring Web Flow 3.0 M1 现已发布，此版本主要关注与 Spring Framework 6 和 Jakarta EE 的兼容性。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fspring-projects%2Fspring-webflow-samples" target="_blank">spring-projects/spring-webflow-samples</a> 上的 Travel booking-mvc 示例已更新，commit history 提供了示例更改。其中一项重大变化是需要删除尚未迁移到 Jakarta EE 的 Tiles。在示例中，改为使用 Thymeleaf Layouts。</p> 
<p style="text-align:start"><span><span style="color:#333333"><span><span><span><span><span><span><span><span><span><span><span><span><span><span><span>公告称：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<blockquote> 
 <p style="text-align:start"><span><span style="color:#333333"><span><span><span><span><span><span><span><span><span><span><span><span><span><span><span>距离上一组 Spring Web Flow 发布已经快 4 年了。尽管如此，该项目仍然很好地满足了特定需求，可以说比其他替代方案更好，并且仍然在积极使用。虽然新版本没有强大的驱动力，但即将推出的 Spring Framework 6 带来了 Java 17 基线并转移到 Jakarta EE，这就产生了对这样一个版本的需求，以便使应用程序能够迁移到这个新基线。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
</blockquote> 
<p>值得注意的是，由于 Spring Faces 与 JSF 的深度集成以及迁移所需的额外时间和精力，因此该版本未包含 Spring Faces。如果你有 Web Flow 和 JSF 方面的经验并有兴趣帮助将 Spring Faces 迁移到 Spring Framework 6 和 Jak<span style="color:#000000">arta EE，可与开发团队进行联系。 </span></p> 
<p>如果你有一个 Web Flow 应用程序并计划迁移到 Spring Framework 6，可尝试这个里程碑并通过项目 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjira.spring.io%2Fbrowse%2FSWF" target="_blank">issue tracker</a> 提供反馈。</p> 
<p>更多详情可<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fspring.io%2Fblog%2F2022%2F08%2F10%2Fspring-web-flow-3-0-m1-released" target="_blank">查看官方博客</a>。</p>
