<div class="content">
                                                                                            <p>PHP 8.1.5 现已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.php.net%2Farchive%2F2022.php%232022-04-14-2" target="_blank">发布</a>，这是一个错误修复版本；官方鼓励所有 PHP 8.1 用户升级到此版本。具体更新内容如下：</p> 
<ul> 
 <li>Core 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8176" target="_blank">#8176</a>（属性初始化程序中的枚举值泄漏）。</li> 
   <li>修复了内部属性参数的释放问题。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8070" target="_blank">#8070</a>（内部函数属性哈希的内存泄漏）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8160" target="_blank">#8160</a>（Alpine 上的 ZTS 支持已损坏）。</li> 
  </ul> </li> 
 <li>Filter： 
  <ul> 
   <li>修复了 php_filter_validate_domain() 中的 signedness confusion。</li> 
  </ul> </li> 
 <li>Intl： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8115" target="_blank">#8115</a>（实例化 Intl 类时无法捕获 arg 类型弃用）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8142" target="_blank">#8142</a>（cygwin 上的编译错误）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F7734" target="_blank">#7734</a>（修复 IntlPartsIterator 键 off-by-one error 和 first key）。</li> 
  </ul> </li> 
 <li>MBString： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8208" target="_blank">#8208</a>（mb_encode_mimeheader: $indent  功能损坏）。</li> 
  </ul> </li> 
 <li>MySQLi： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8068" target="_blank">#8068</a>（mysqli_fetch_object 创建了不可访问的属性）。</li> 
  </ul> </li> 
 <li>Pcntl： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8142" target="_blank">#8142</a>（cygwin 上的编译错误）。</li> 
  </ul> </li> 
 <li>PgSQL： 
  <ul> 
   <li>修复了 LLP64 架构上与 result_type 相关的堆栈损坏。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8253" target="_blank">#8253</a>（pg_insert() 引用失败）。</li> 
  </ul> </li> 
 <li>Sockets： 
  <ul> 
   <li>修复了 Solaris 构建。</li> 
  </ul> </li> 
 <li>SPL： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8121" target="_blank">#8121</a>（SplFileObject - seek 和 key 与 csv 文件不一致）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8192" target="_blank">#8192</a>（在 8.1 中无法在没有返回 typehint 的情况下覆盖 DirectoryIterator::current()）。</li> 
  </ul> </li> 
 <li>Standard： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8048" target="_blank">#8048</a>（强制 macOS 使用 statfs）。</li> 
  </ul> </li> 
</ul> 
<p>更新说明：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.php.net%2FChangeLog-8.php%238.1.5" target="_blank">https://www.php.net/ChangeLog-8.php#8.1.5</a></p>
