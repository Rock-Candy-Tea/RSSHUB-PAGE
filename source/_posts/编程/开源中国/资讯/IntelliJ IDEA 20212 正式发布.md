                                                        <p>IntelliJ IDEA 2021.2 正式发布，更新内容如下：</p> 
<h3>关键更新：</h3> 
<ul> 
 <li>新的项目范围的分析允许开发者在编译前跟踪整个小型或中型项目的错误，这个功能只能够在 IntelliJ IDEA Ultimate 中使用；</li> 
 <li>IntelliJ IDEA 增加了几个操作，当你保存项目时，IDE 将启动这些操作，包括重新格式化代码和优化导入等。所有这些操作都能在偏好/设置|工具|中找到；</li> 
 <li>现在，通过新的软件包搜索用户界面，可以轻松管理 Maven 和 Gradle 的依赖关系。你还可以查看是否有任何依赖的版本更新，并从编辑器中直接升级。</li> 
</ul> 
<h3>编辑器：</h3> 
<ul> 
 <li>检查和快速修复的描述更加全面，其中一些还具有使用实例；</li> 
 <li>图表具有更有信息量，为你提供一个带有图谱的结构视图和所选块及其相邻块的预览；</li> 
 <li>你的项目的版权声明默认包括项目创建年份和当前版本年份；</li> 
 <li>对 Markdown 的支持得到了改进，能够将 .md 文件转换为各种格式，配置图像大小，并使用拖放来插入图像。还修复了列表格式问题，并增加了新的浮动工具栏，它只在你需要的时候才会激活。</li> 
</ul> 
<h3>Java：</h3> 
<ul> 
 <li>你可以用不同的方式标记公共、保护和私有的 Java 成员（方法、字段、类）。要配置这个，请到 首选项/设置/编辑器/颜色方案，并展开可见性节点；</li> 
 <li>引入了一个新的两步式 JavaFX 项目向导；</li> 
</ul> 
<h3>Kotlin：</h3> 
<ul> 
 <li>Kotlin 代码补全现在默认是基于机器学习机制工作的；</li> 
 <li>你不必等待代码分析完成后再运行测试，因为现在你可以在打开文件后立即通过点击运行测试图标启动测试；</li> 
 <li>Coroutines 代理现在可以通过调试工具窗口中的 Coroutines 标签使用。它适用于依赖 kotlinx.coroutines 的 Java 运行配置以及 Spring 和 Maven 运行配置；</li> 
 <li>运行/调试配置相关的按钮在索引期间处于激活状态，这意味着即使在 IDE 对项目进行索引时，你也可以运行和调试你的应用程序；</li> 
 <li>为 Kotlin 引入了 WSL 2 支持和 Run Targets 功能；</li> 
</ul> 
<h3>JavaScript（仅限 IntelliJ IDEA Ultimate）：</h3> 
<ul> 
 <li>IntelliJ IDEA 现在可以同时重命名 useState 值和函数；</li> 
</ul> 
<h3>用户体验：</h3> 
<ul> 
 <li>可以从欢迎屏幕上的专用文件夹中直接打开存储在设备上的 Eclipse 项目，因为 IntelliJ IDEA 现在可以自动检测到此类项目；</li> 
 <li>特定用途的选项现在可以在 首选项/设置中新的高级设置节点中使用；</li> 
 <li>IntelliJ IDEA 自动清理任何最后更新超过 180 天的缓存和日志目录；</li> 
 <li>如果你的项目缺少一个用于特定框架的插件，IDE 会提醒你并提供启用它的机会；</li> 
 <li>IDE 会在 Toolbox App 1.20.8804 或更高版本中提供产品更新通知；</li> 
 <li>在这个版本中，我们在使用上下文菜单、弹出式窗口和工具栏时消除了 UI 块，并将某些需要索引的操作移出了 UI 线程；</li> 
</ul>
