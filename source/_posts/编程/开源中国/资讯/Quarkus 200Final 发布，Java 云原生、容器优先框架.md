                                                        <p>Quarkus  2.0.0.Final 现已发布。Quarkus 是 Kubernetes 原生的 Java 技术栈，它由同类中最佳的 Java 库和标准精制而成，并针对容器和云部署量身定制。Quarkus 的总体目标是将 Java 带入云原生的应用开发，并使它成为 Serverless、云和 Kubernetes 环境的领先平台。同时统一了响应式和命令式编程模型，开发人员可以选择最适合其用例的模型，并更轻松地将其代码与其它组件集成到响应式系统中，例如带有 Vert.x 和 Kafka 的响应式流、响应式数据库 API 等。 </p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li>不再支持 JDK 8，JDK 11 是使用 Quarkus 2.0 的最小版本</li> 
 <li>推荐使用 GraalVM 21.1</li> 
 <li>支持 Vert.x 4</li> 
 <li> <p>支持 Eclipse MicroProfile 4。MicroProfile 4 是 MicroProfile 规范的最新版本。此外，Quarkus 中的所有 SmallRye 组件都已更新至最新，除了实现 MicroProfile 4 外，还有其它新功能</p> </li> 
 <li> <p>支持持续测试，这允许测试在开发模式下自动运行，并且可以通过在控制台（或通过 Dev UI）按 r 来启用。如果开发者启用测试，其测试套件将在开始时运行一次，然后持续测试将在开发者编码时运行其测试，并告诉开发者新的更改是否会破坏任何测试。</p> </li> 
 <li> <p>全新的 Quarkus CLI，目前开发者已经可以使用 JBang 来运行它</p> </li> 
 <li> <p>Quarkus 现在以 SmallRye GraphQL 客户端扩展的形式提供了一个 GraphQL 客户端</p> </li> 
 <li>Hibernate ORM 已升级到 5.5</li> 
 <li>CDI 实现 ArC 支持 CDI 装饰器</li> 
 <li>带有 Panache 的 MongoDB 现在支持事务</li> 
</ul>
