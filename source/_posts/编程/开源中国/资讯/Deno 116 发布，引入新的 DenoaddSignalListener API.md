                                                                                            <p>Deno 1.16 已发布，包括以下新特性和变更：</p> 
<ul> 
 <li><code>fetch</code> 现在支持获取文件 URL；</li> 
 <li>V8 更新至 9.7 版；</li> 
 <li>支持 React 17 中的新 JSX 转换；</li> 
 <li>新的<code>Deno.addSignalListener</code> API，用于侦听操作系统信号；</li> 
 <li><code>Error.cause</code> 现在显示在控制台中，这对于调试发生在应用程序内部深处的错误非常有用；</li> 
 <li>现在可以明确地完成握手 TLS 连接，此版本向<code>Deno.TlsConn</code>中增加了<code>handshake()</code>方法；</li> 
 <li>Web Streams API 的改进</li> 
 <li>Deno 1.16 稳定了<code>Deno.startTls</code> API，这使得为稳定的 Deno 编写 SMTP 驱动程序成为可能；</li> 
 <li>Per-test 权限现在是稳定的，这使得测试程序在不同权限集下的行为变得非常容易；</li> 
 <li><code>localStorage</code>不再需要附带<code>-location</code></li> 
 <li>支持在中止<code>AbortSignal</code>时指定原因</li> 
 <li>Deno to npm 包构建工具：发布了一个名为 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdnt" target="_blank">dnt</a> 的新系统，用于把 Deno 编写的模块发布为 npm 包；</li> 
 <li>WebAssembly Reference 类型现在稳定可用</li> 
 <li><code>findLast</code>和<code>findLastIndex</code>数组方法</li> 
</ul> 
<p>更多详情可查看：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdeno.com%2Fblog%2Fv1.16" target="_blank">https://deno.com/blog/v1.16</a></p>
