<div class="content">
                                                                                            <p><span style="background-color:#ffffff; color:#333333">Visual Studio Code 1.69 现已</span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69" target="_blank">发布</a><span style="background-color:#ffffff; color:#000000">，<span style="background-color:#ffffff">一些主要亮点更新包括：</span></span></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_3-way-merge-editor" target="_blank">3-way merge editor</a></strong>- 在 VS Code 内解决合并冲突。</li> 
</ul> 
<p>合并编辑器允许你快速解决 Git 合并冲突。启用后，可以通过单击源代码控制视图中的冲突文件来打开合并编辑器。<span style="background-color:#ffffff; color:#444444">Checkboxes<span> </span></span>可用于接受和合并"Theirs"或"Yours "的更改：</p> 
<p><img alt height="381" src="https://oscimg.oschina.net/oscnet/up-1ccda3818f8921ddeb5198e73dab613340d.gif" width="500" referrerpolicy="no-referrer"></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>合并编辑器中提供了所有语言功能（包括诊断、断点和测试），因此你可以立即获得有关合并结果中任何问题的反馈。结果也可以直接编辑。注意 checkbox 是如何按预期更新的：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p><img alt height="185" src="https://oscimg.oschina.net/oscnet/up-8ff8d3f85b732916270e9a1513a6805f1c6.gif" width="500" referrerpolicy="no-referrer"></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>关闭合并编辑器或接受合并时，如果未解决所有冲突，则会显示警告。合并编辑器支持 word-level merging —— 只要更改不相交，双方都可以应用。如果插入顺序很重要，可以调换一下。在任何时候，也可以手动解决冲突。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p><img alt height="185" src="https://oscimg.oschina.net/oscnet/up-8703e46cbceca4fc686db56a0e389b29cdb.gif" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong style="color:#444444"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_command-center" target="_blank">Command Center</a></strong>- 用于搜索文件、运行命令和浏览光标历史记录的新 UI。</li> 
</ul> 
<p>取代了正常的标题栏，让你可以快速搜索项目中的文件。单击 main section 以显示带有你最近的文件和搜索框的快速打开下拉菜单。</p> 
<p><img alt height="386" src="https://oscimg.oschina.net/oscnet/up-ab0d50267f5d800a29bf9027f7673c7fb8d.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_do-not-disturb-mode" target="_blank">请勿打扰模式</a></strong>- 使非关键通知弹出窗口静音。</li> 
</ul> 
<p>该模式在启用时会隐藏所有非错误通知弹出窗口。进度通知将自动显示在状态栏中。隐藏的通知仍然可以在通知中心查看。</p> 
<p><img alt height="80" src="https://oscimg.oschina.net/oscnet/up-e7bff82189c4ee6f0847f250e7b1029c1ec.jpg" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_toggle-between-light-and-dark-themes" target="_blank">切换浅色/深色主题</a></strong>- 在首选的浅色和深色主题之间快速切换。</li> 
</ul> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>一个新命令允许你在喜欢的浅色和深色主题之间进行切换，<strong>Preferences: Toggle between Light/Dark Themes</strong>。当处于浅色主题时，该命令切换到深色主题，反之亦然。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>在高对比度主题中时，该命令保持高对比度并切换到相反颜色主题的首选高对比度主题。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>首选主题由以下设置定义：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li><code>workbench.preferredDarkColorTheme</code></li> 
 <li><code>workbench.preferredLightColorTheme</code></li> 
 <li><code>workbench.preferredHighContrastColorTheme</code></li> 
 <li><code>workbench.preferredHighContrastLightColorTheme</code></li> 
</ul> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_shell-integration" target="_blank">Terminal shell integratio</a></strong>- 显示命令状态、运行最近的命令等。</li> 
</ul> 
<p>自 1 月发布以来一直处于预览状态的 PowerShell、bash 和 zsh 的 Shell 集成现已停止预览，官方表示其计划在 1.70 版本中默认启用。Shell 集成允许 VS Code 的终端更多地了解 Shell 内部发生的事情，以启用更多功能。shell 集成的目标之一是使其能够在<strong style="color:#444444">零配置</strong>的情况下工作。</p> 
<p><img alt height="180" src="https://oscimg.oschina.net/oscnet/up-e3d5d6fd98fcb0328509900fbb47e696602.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="120" src="https://oscimg.oschina.net/oscnet/up-6dc7d06671879421abdf0249a0d2fbf37cc.png" width="300" referrerpolicy="no-referrer"></p> 
<p><img alt height="352" src="https://oscimg.oschina.net/oscnet/up-dce3bd2614e8a84ac4534deec6d25114960.gif" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_decorations" target="_blank">Task output decorations</a></strong>- 突出显示任务成功或失败的退出代码。</li> 
</ul> 
<p><img alt height="170" src="https://oscimg.oschina.net/oscnet/up-644ab16b20d2c8a4e1f047fd8f71c4c2349.png" width="500" referrerpolicy="no-referrer"></p> 
<p><img alt height="155" src="https://oscimg.oschina.net/oscnet/up-6e8e6a3d76a17cc7fc98842fa69d63d7afe.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_commit-action-button-for-git-repositories" target="_blank">Git Commit 操作按钮</a></strong>- 配置你的默认 Git Commit 操作。</li> 
</ul> 
<p>添加了一个 <strong style="color:#444444">Commit </strong>按钮，该按钮具有主要操作和一组辅助操作。可以使用<code>git.postCommitCommand</code>设置控制辅助操作，并允许你在提交后进行推送或同步。</p> 
<p><img alt height="313" src="https://oscimg.oschina.net/oscnet/up-30b536dc4c1a488052400f38904aae5b2e2.gif" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_improved-step-into-target-ui" target="_blank">Debug Step Into Target 支持</a></strong>- 允许你在暂停时直接进入函数。</li> 
</ul> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span style="color:#444444"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>一些调试器允许在某行暂停时直接进入特定的函数调用。在这次迭代中，为此改进了 UI：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li>右键单击源行上的目标区域并选择 <strong>Step Into Target </strong>将自动进入目标区域（如果有的话）。</li> 
 <li>Command Palette 中有一个新命令 <strong>Debug: Step Into Target </strong>可用，键盘快捷键是 <span><span><span><span style="color:#0072be"><span style="background-color:#f4f2f9">Ctrl+F11 。</span></span></span></span></span></li> 
</ul> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_javascript-debugging" target="_blank">JavaScript sourcemap 切换</a></strong>- 切换到调试编译而不是源代码。</li> 
</ul> 
<p>现在可以通过单击调用堆栈视图标题中的<img height="45" src="https://oscimg.oschina.net/oscnet/up-d9dfc118c5c9635c1ebaee9effd92da6776.png" width="45" referrerpolicy="no-referrer">罗盘图标轻松地打开和关闭 sourcemaps。当 sourcemaps 关闭时，源代码中设置的断点仍然有效，但会移动到编译代码中的等效位置，并且调试器将逐步执行编译代码而不是源代码。</p> 
<p><img alt height="356" src="https://oscimg.oschina.net/oscnet/up-0df53eedf4670812ca0a860fe81e7cc23c8.gif" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong style="color:#444444"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_theme-tester-supports-built-in-themes" target="_blank">Color theme tester</a></strong>- 使用 vscode.dev 预览 <span style="background-color:#ffffff; color:#444444">color themes</span>。</li> 
</ul> 
<p><img alt height="279" src="https://oscimg.oschina.net/oscnet/up-de6b7c925fb24453e69db7b93e383129e0f.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcode.visualstudio.com%2Fupdates%2Fv1_69%23_vs-code-server-private-preview" target="_blank">VS Code Server 预览</a></strong>- 运行用于远程开发的同一服务器。</li> 
</ul> 
<p>现在提供独立的<strong style="color:#444444">“VS Code Server”的私人预览版，</strong>它是基于远程扩展使用的同一底层服务器构建的服务，以及一些额外的功能，如交互式 CLI 和促进与 vscode.dev 的安全连接，无需 SSH 连接。</p> 
<p><img alt height="223" src="https://oscimg.oschina.net/oscnet/up-da90ff51e9b0849341c8550659431025ba4.png" width="500" referrerpolicy="no-referrer"></p>
