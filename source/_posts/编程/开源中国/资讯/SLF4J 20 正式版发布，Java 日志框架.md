                                                                                            <p style="color:#000000; text-align:start">SLF4J 2.0.x 正式版发布了，2.0 系列要求至少 Java 8 的支持，同时该版本增加了向后兼容的<span> </span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.slf4j.org%2Fmanual.html%23fluent" target="_blank">流式日志 API</a>，但是要求已有框架用户必须移植到<span> </span><code>ServiceLoader</code><span> </span>机制，内部详细的改进详情请看<span> </span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.slf4j.org%2Ffaq.html%23changesInVersion200" target="_blank">detailed</a><span> </span>。</p> 
<p style="color:#000000; text-align:start">新版本详细信息请看 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.slf4j.org%2Fnews.html" target="_blank">https://www.slf4j.org/news.htm</a></p> 
<p style="color:#000000; text-align:start"><span style="background-color:#ffffff; color:#333333">SLF4J 是为各种 loging APIs 提供一个简单统一的接口，从而使得最终用户能够在部署的时候配置自己希望的 loging APIs 实现。 Logging API 实现既可以选择直接实现 SLF4J 接的 loging APIs 如： NLOG4J、SimpleLogger。也可以通过 SLF4J 提供的 API 实现来开发相应的适配器如 Log4jLoggerAdapter、JDK14LoggerAdapter。</span></p>
