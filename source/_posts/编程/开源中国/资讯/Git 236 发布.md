<div class="content">
                                                                                            <p><span style="background-color:#ffffff; color:#333333">Git 2.36 现已</span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flore.kernel.org%2Fgit%2Fxmqqh76qz791.fsf%40gitster.g%2FT%2F%23u" target="_blank">发布</a><span style="background-color:#ffffff; color:#333333">，</span><span style="color:#000000"><span style="background-color:#ffffff">本次更新</span><span style="background-color:#ffffff">包含来自超过 96 个贡献者的功能和错误修复，其中 26 个是新贡献者。该版本包含与最近的维护版本相同的修复，以解决 CVE-2022-24765 问题。</span></span></p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li> <p style="text-align:start"><span><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>增加了一个由</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><code>ort</code><span><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>驱动的新功能，即 --remerge-diff 选项，作为审查合并冲突解决方案的改进体验。Git 的 --remerge-diff 会显示有合并冲突的文件和解决方案之间的差异。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> </li> 
</ul> 
<p style="text-align:start"><span><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><img alt height="230" src="https://oscimg.oschina.net/oscnet/up-40d96e9ad1815b9f20f0d29e8647535115d.webp" width="500" referrerpolicy="no-referrer"></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="text-align:start"><span><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><img alt height="230" src="https://oscimg.oschina.net/oscnet/up-ff0682555cd57039618ac89beaba4d2c75c.webp" width="500" referrerpolicy="no-referrer"></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#121212">更灵活的 fsync 配置。Git 2.36 引入了一套明显更灵活的配置选项，以调整 Git 如何以及何时明确地</span><code>fsync</code><span style="background-color:#ffffff; color:#121212">大量不同种类的文件，而不仅仅是</span><code>fsync</code><span style="background-color:#ffffff; color:#121212">松散的对象。这个新变化的核心是两个新的配置变量：core.fsync 和 core.fsyncMethod。</span></li> 
 <li><span style="background-color:#ffffff; color:#121212">更严格的存储库所有权检查。从 Git 2.35.2 开始，Git 改变了它的 default behavior，防止你在一个与当前用户不同的用户拥有的仓库中执行 git 命令。这是为了防止 git 调用无意中执行了仓库所有者配置的命令。你可以通过设置新的 safe.directory 配置来绕过这个检查，包括其他用户拥有的</span><span style="background-color:#ffffff; color:#24292f">受信任存储库</span><span style="background-color:#ffffff; color:#121212">。safe.directory 选项现在解释为将所有 Git 仓库视为安全的，无论其所有者是谁。你可以在你的 --global 配置中设置这个选项，以便在有意义的情况下选择不使用新的行为。</span></li> 
 <li>git name-rev --stdin 已被弃用，使用时发出警告；使用 git name-rev --annotate-stdin 进行代替。</li> 
 <li>git clone --filter=... --recurse-submodules 只使 top-level  部分克隆，而子模块则完全克隆。这一行为被更改为将相同的 filter 向下传递给子模块。</li> 
 <li><span style="background-color:#ffffff; color:#121212">以及各种其他较小的更改和修复。</span></li> 
</ul>
