                                                                                            <p>HHVM 4.136 现已发布，HHVM <span style="background-color:#f9f9f9; color:#393939">4.131–4.135</span>、以及 4.102 和 4.128 LTS 版本还会继续得到支持。</p> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start"><span style="color:#333333">更新内容如下：</span></p> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start"><strong>Highlights</strong></p> 
<ul> 
 <li>增加 IDE 快速修复缺失的接口方法。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdocs.hhvm.com%2Fhack%2Freadonly%2Fintroduction" target="_blank"><code class="language-plaintext">readonly</code>功能</a>不再被视为不稳定。 
  <ul> 
   <li><code class="language-plaintext">readonly</code>modifier 现在也可以通过<code class="language-plaintext">isReadonly()</code>或<code class="language-plaintext">returnsReadonly()</code>函数在 reflection API 可见。</li> 
  </ul> </li> 
 <li>[WriteHandle::truncate] 方法被添加到 HSL 以截断文件。</li> 
 <li>HSL 中的各种 API 现在被视为纯 API 或<code class="language-plaintext">controlled</code>不再需要<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdocs.hhvm.com%2Fhack%2Fcontexts-and-capabilities%2Fintroduction" target="_blank">默认功能</a>。</li> 
 <li><code class="language-plaintext">noreturn</code>类型现在可以向上转换为<code class="language-plaintext">dynamic</code>. <!-</li> 
 <li>一个实验性选项<code class="language-plaintext">hhvm.jit_attempt_super_inlining</code>被添加到<code class="language-plaintext">.hhconfig</code>，它在 HHVM 中启用了一种新的策略，试图对近似常量值进行常量折叠。–></li> 
</ul> 
<p><strong>Breaking Changes</strong></p> 
<ul> 
 <li>不再支持带有<code class="language-plaintext">private final</code>修饰符的方法。使用<code class="language-plaintext">private</code>来代替。</li> 
 <li><code class="language-plaintext">partial</code>typing mode 被删除。这包括对<code class="language-plaintext">.php</code>文件中<code class="language-plaintext"><?hh // partial</code>的支持，以及对<code class="language-plaintext">.hackpartial</code>文件的支持。</li> 
 <li>当特性是<code class="language-plaintext"><<__ConsistentConstruct>></code>但未声明显式构造函数时引发错误。</li> 
</ul>
