 <p style="text-align:left">Jboot 一个更简单的分布式、微服务框架。</p> 
 <p style="text-align:left">Jboot是一个基于 JFinal、JFinal-Undertow、Dubbo、Seata、Sentinel、ShardingSphere、Nacos 等开发的微服务框架，帮助开发者降低微服务、分布式开发门槛。爽爽开发，快乐生活。</p> 
</blockquote> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">到目前为止，Jboot 已经开源超过了 5 年的时间，迭代了 190+ 个版本，已经被超过 1000+ 公司在使用，其中包含了多个知名的上市公司。</span></p> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">Jboot v3.9.15  主要是针对 Junit4 和 junit5 带来了强大的单元测试能力。</span></p> 
<p style="text-align:left">在 junit4 如下代码所示：</p> 
<pre><code class="language-java">@RunWith(JbootRunner.class)
public class MyAppTester &#123;

    private static MockMvc mvc = new MockMvc();

    @Inject
    private MyService myService;

    @Test
    public void test_url_aaa() &#123;
        MockMvcResult mvcResult = mvc.get("/aaa");

        mvcResult.printResult()
                .assertThat(result -> Assert.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getHttpCode() == 200);
    &#125;

    @Test
    public void test_url_bbb() &#123;
        MockMvcResult mvcResult = mvc.get("/bbb");

        mvcResult.printResult()
                .assertThat(result -> Assert.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getHttpCode() == 200);
    &#125;

    @Test
    public void test_my_service() &#123;
        Ret ret = myService.doSomeThing();
        Assert.assertNotNull(ret);
        //.....
    &#125;
&#125;</code></pre> 
<p>我们可以通过 MockMvc 可以对 JFinal（Jboot） 中的 Controller、Interceptor、Handler 等进行测试，无需启动 JFinal 服务。</p> 
<p>同时可以直接通过 @Inject 或者 @RPCInject 把 Service 注入到 MyAppTester 里，直接对 service 进行测试。</p> 
<p>在 junit5 中，代码如下：</p> 
<pre><code class="language-java">@ExtendWith(JbootExtension.class)
public class MyAppTester &#123;

    private static MockMvc mvc = new MockMvc();

    @Inject
    private MyService myService;

    @Test
    public void test_url_aaa() &#123;
        MockMvcResult mvcResult = mvc.get("/aaa");

        mvcResult.printResult()
                .assertThat(result -> Assertions.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getHttpCode() == 200);
    &#125;

    @Test
    public void test_url_bbb() &#123;
        MockMvcResult mvcResult = mvc.get("/bbb");

        mvcResult.printResult()
                .assertThat(result -> Assertions.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getHttpCode() == 200);
    &#125;

    @Test
    public void test_my_service() &#123;
        Ret ret = myService.doSomeThing();
        Assertions.assertNotNull(ret);
        //.....
    &#125;
&#125;</code></pre> 
<p>主要的不同点是 MyApTester 类的配置不同而已。Junit4 使用 @RunWith(JbootRunner.class) 配置，而 Junit5 使用 @ExtendWith(JbootExtension.class) 配置。</p> 
<p> </p> 
<p style="text-align:left"><strong>Jboot v3.9.15 更新内容如下：</strong></p> 
<ul> 
 <li>新增：新增单元测试的辅助类的支持</li> 
 <li>新增：ActionReporter 新增 render 信息的输出功能</li> 
 <li>新增：工具类 ReflectUtil.java</li> 
 <li>新增：为 undertow 新增默认的 content-type，解决 mp4 等视频不能播放的问题</li> 
 <li>新增：ValidErrorRender，方便用户自定义 "数据验证" 错误的渲染器</li> 
 <li>新增：新增配置 "jboot.app.listener"，用于配置可以执行的 appListener</li> 
 <li>新增：新增配置 "jboot.json.skipModelAttrs" 和 "jboot.json.skipBeanGetters" 配置</li> 
 <li>新增：devModel 可以动态配置，方便在某些场景下切换 devMode</li> 
 <li>优化：调整默认的JbootShiroInvokeListener实现，保存被拦截的请求便于后续跳转使用，感谢 @没牙的小朋友</li> 
 <li>优化：JbootGatewayHandler 默认添加在系统里，方便进行动态路由</li> 
 <li>优化：JbootGatewayHealthChecker 的代码</li> 
 <li>优化：升级 JFinal 等依赖到最新版本</li> 
 <li>优化：对 ClickHouse 高级版本 驱动 进行适配</li> 
 <li>修复：jboot.properties配置文件中 jboot.shiro.ini 配置未生效的问题，感谢 @没牙的小朋友</li> 
 <li>修复：在某些情况下 PathKit.getWebRootPath 得到错误结果的问题</li> 
 <li>修复：当 Interceptor 被 cglib 代理时，无法正确输出其日志的问题</li> 
 <li>修复：在某些极端场景下，PathKit.getWebRootPath() 可能出错的问题</li> 
 <li>修复：当查询的 Page 为 null 时，分页的总页数数据错误的问题</li> 
</ul>
