                                                                                            <p>HttpComponents Core 5.1 已正式 GA。</p> 
<p>值得关注的变更和特性：</p> 
<ul> 
 <li>与 RFC 3986（统一资源标识符 (URI): 通用语法）保持条件一致性</li> 
 <li>改进对经典（阻塞）HTTP 传输的序列外响应信息处理的支持</li> 
 <li>改进消息生成器 (message builders)</li> 
 <li>Bug fix: HTTP negotiator 工厂接收空 TlsStrategy</li> 
 <li>RequestHandlerRegistry 将 127.0.0.1 解析为首要主机</li> 
 <li>HTTPCORE-667, HTTPCORE-668, HTTPCORE-670：增加 content 类型和 HTTP header 常量</li> 
 <li>……</li> 
</ul> 
<p><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdownloads.apache.org%2Fhttpcomponents%2Fhttpcore%2FRELEASE_NOTES-5.1.x.txt" target="_blank">详情查看 Release Notes</a>。</p> 
<p><a href="https://www.oschina.net/action/GoToLink?url=http%3A%2F%2Fhc.apache.org%2Fdownloads.cgi" target="_blank">下载地址</a></p> 
<p>另外值得注意的是，应用程序协议升级支持非阻塞 HTTP/1.1 连接的特性已移至 5.2 版本系列。</p> 
<p>HttpCore 是 HTTP/1.1 和 HTTP/2 传输组件，使用者可通过它以最小的空间构建自定义的客户端和服务器端 HTTP 服务。</p>
