                                                                                            <p><span style="background-color:#ffffff; color:#333333">Dapr 1.5 正式发布。Dapr 是一个开源、可移植的、事件驱动的运行时，可以帮助开发人员构建在云和边缘上运行的弹性的、微服务的、无状态和有状态应用程序，并且关注于业务逻辑而不用考虑分布式相关的问题。 </span></p> 
<p><strong><span style="background-color:#ffffff; color:#333333">主要更新内容</span></strong></p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#333333">组件升级到稳定版</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">增加用于状态管理的查询 API</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">配置 API 构建块</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">虚拟 actor 现在可以在 Dapr Go SDK 中使用</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">发布了对 actor 运行时的增强功能以​​提高可靠性</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">支持 ARM64 Mac（预览版）</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">提高actor分区表传播的可靠性</span></li> 
 <li>修复 Http opa 中间件崩溃的问题</li> 
 <li>将 ReadBufferSize 参数传递给 fasthttp 服务器和 grpc 服务器</li> 
 <li>在 pod 级别添加自定义标签的新参数</li> 
 <li>修复 Dapr 运行时将 InputBinding 事件发送到未指定的位置的问题</li> 
 <li>向 Helm Chart 值添加容忍度支持</li> 
 <li>修复元数据标头未通过 dapr gRPC 代理发送到服务器的问题</li> 
 <li>可重试的 pubsub 错误应包括日志中的主题</li> 
 <li>支持服务调用中的 PATCH 请求</li> 
</ul>
