                                                        <p>在1.3.6版本中，我们发布了一个重大Future，另外对于K8s以及Docker做了较大的改进，具体如下</p> 
<p><strong>1:支持接入SkyWalking</strong></p> 
<p>我们同Apache SkyWalking社区一起协作完成了DolphinScheduler的链路监控设计以及开发：</p> 
<p>主要如下：</p> 
<p>-通信协议改造为支持链路跟踪header传递<br> -任务调度执行核心链路的跟踪<br> -jvm 指标自动采集，观测vm 状态<br> -日志输出到skywalking ，可在skywalking web在线查看日志，并与链路跟踪关联起来</p> 
<p>感兴趣的同学可以根据[文档](https://dolphinscheduler.apache.org/en-us/docs/latest/user_doc/skywalking-agent-deployment.html)接入SkyWalking来实现DolphinScheduler的链路监控。</p> 
<p><br> <strong>2:Docker & K8S</strong></p> 
<p><br> Docker 改进：</p> 
<p>- 修复 WORKER_GROUP 不生效问题，更改为 WORKER_GROUPS ([#4843](https://github.com/apache/dolphinscheduler/issues/4843))<br> - 修复 Windows 的 dolphinscheduler_env.sh 挂载出错问题 ([#4804](https://github.com/apache/dolphinscheduler/issues/4804)) ([#4886](https://github.com/apache/dolphinscheduler/issues/4886))<br> - 容器内进程优化为 supervisor 管理 ([#2687](https://github.com/apache/dolphinscheduler/issues/2687))<br> - 优化 supervisor 进程日志输出 ([#4897](https://github.com/apache/dolphinscheduler/issues/4897))<br> - 适配 docker compose file 3.1+ 版本<br> - 同步最新 1.3.6 版本所有配置项，包括 common、master、worker<br> - 新增 config.env.sh 环境变量配置文件，用于配置 docker compose 和 swarm<br> - 新增 MASTER_SERVER_OPTS,  WORKER_SERVER_OPTS, ALERT_SERVER_OPTS, API_SERVER_OPTS, LOGGER_SERVER_OPTS 环境变量<br> - 支持 SkyWalking 配置和集成<br> - 更新基础镜像为 debian:slim，提升镜像扩展性，如安装 Python 库、Python 3 等 ([#5155](https://github.com/apache/dolphinscheduler/issues/5155))<br> - 镜像移除冗余的 postgresql-client<br> - 支持 Hadoop、Spark、Flink、DataX 等任务扩展 ([#5100](https://github.com/apache/dolphinscheduler/issues/5100))<br> - 添加支持矩阵 ([#5068](https://github.com/apache/dolphinscheduler/issues/5068))<br> - 添加 [Docker FAQ](https://dolphinscheduler.apache.org/zh-cn/docs/1.3.6/user_doc/docker-deployment.html)：如 MySQL、Hadoop、Spark、Oracle 数据源集成等常见问题 ([#4732](https://github.com/apache/dolphinscheduler/issues/4732) [#4871](https://github.com/apache/dolphinscheduler/issues/4871))<br> - 镜像增加 WORKDIR，移除冗余的 container_name 和 dolphinscheduler-postgresql-initdb</p> 
<p>Kubernetes 改进：</p> 
<p>- 修复 WORKER_GROUP 不生效问题，更改为 WORKER_GROUPS ([#4843](https://github.com/apache/dolphinscheduler/issues/4843))<br> - 修复 Minikube 上不正确 host 注册问题 ([#4905](https://github.com/apache/dolphinscheduler/issues/4905))<br> - 修复所有 Kubernetes 1.12+ 的 Ingress 兼容性问题<br> - values.yaml<br>   - 合并 image 的 registry 和 repository 为 repository 字段，支持本地镜像<br>   - 修改 imagePullSecrets 为 image 的 pullSecret 字段<br>   - 新增 externalDatabase 的 type、driver、params 字段，可以支持非 PostgreSQL 的外部数据库<br>   - 新增 common 字段，用于配置 common.properties、dolphinscheduler_env.sh 等公共环境变量配置<br>   - 新增 common.sharedStoragePersistence 字段，用户 api、master、worker 共享存储，如 Hadoop、Spark 二进制包<br>   - 修改 resource.persistentVolumeClaim 为 common.fsFileResourcePersistence 字段<br>   - 移除 resource 字段<br>   - master、worker、api、alert 新增字段：annotations、resources，后者用于配置 cpu、memory 资源<br>   - 新增 api.service 字段，支持 ExternalName、ClusterIP、ClusterIP、LoadBalancer 等多种暴露方式<br>   - 优化 ingress.path 为 /dolphinscheduler<br>   - 移除冗余的 ingress.hosts<br> - 同步最新 1.3.6 版本所有配置项，包括 common、master、worker<br> - 新增 MASTER_SERVER_OPTS,  WORKER_SERVER_OPTS, ALERT_SERVER_OPTS, API_SERVER_OPTS, LOGGER_SERVER_OPTS 环境变量<br> - 支持 SkyWalking 配置和集成<br> - 支持 Hadoop、Spark、Flink、DataX 等任务扩展 ([#5100](https://github.com/apache/dolphinscheduler/issues/5100))<br> - 添加支持矩阵 ([#5068](https://github.com/apache/dolphinscheduler/issues/5068))<br> - 添加 [Kubernetes FAQ](https://dolphinscheduler.apache.org/zh-cn/docs/1.3.6/user_doc/kubernetes-deployment.html)：如 MySQL、Hadoop、Spark、Oracle 数据源集成等常见问题 ([#2584](https://github.com/apache/dolphinscheduler/issues/2584))<br> - 优化 master 和 worker 的长 FQDN 地址为短 FQDN 地址，以及地址在 UI 中的显示<br> - 修复定时任务无法生效的问题 ([#5351](https://github.com/apache/dolphinscheduler/issues/5351))<br> - 修复 DATA_BASEDIR_PATH, RESOURCE_STORAGE_TYPE 或 RESOURCE_UPLOAD_PATH 未设置导致无法部署的问题<br> - 修复容器中始终 Runtime.getRuntime().availableProcessors()=1 导致默认 max.cpuload.avg=2，从而任务无法调度执行的问题<br> - 优化 Helm 的 templates 文件，删除重复的模板和配置<br> <strong>3:Worker Group</strong></p> 
<p>- 支持在 Web UI 创建、编辑、删除等操作</p>
