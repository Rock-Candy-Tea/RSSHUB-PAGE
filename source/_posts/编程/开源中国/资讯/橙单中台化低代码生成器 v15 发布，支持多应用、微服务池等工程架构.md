                                                        <p style="text-align:left">根据用户的需求反馈，经过历时两月的研发，目前已支持多租户、多应用等典型工程结构。在后续版本中，我们仍会持续升级，以支持更多的工程类型。我们的目标是可以生成最为贴近架构师需求的工程架构，可统一的代码结构规范，以及更多可复用的业务代码。更多细节详见下面的新功能列表，及文章末尾的新功能截图（<strong>强烈推荐放大看图</strong>）。</p> 
<h4 style="text-align:left">多应用功能</h4> 
<div style="text-align:left"> 
 <div style="text-align:left"> 
  <ul> 
   <li style="text-align:left">生成器支持多应用功能，应用和服务之间保持多对多关系，服务池功能已基本支持，后续版本会持续优化。</li> 
   <li style="text-align:left">单体工程可同时创建 WebAdmin 后台应用和面向前端 App 的 WebApi 应用。</li> 
   <li style="text-align:left">微服务工程可创建非常典型的单体后台 WebAdmin 应用，及面向前端 App 的 WebApi 微服务应用。补充说明，WebAdmin 后台应用也可以配置为微服务应用，并可与 WebApi 应用共享服务池中的通用业务服务。</li> 
   <li style="text-align:left">多租户工程可同时创建三个应用，分别为租户运营管理后台应用 TenantAdmin，租户运营后台应用，以及面向租户前端 App 的 WebApi 应用。</li> 
   <li style="text-align:left">对于多租户运营管理后台应用 TenantAdmin，不仅内置了自身的权限管理和租户运营管理等功能，同时也支持配置自定义的业务表单和租户统计表单。</li> 
   <li style="text-align:left">定时任务高水标记表新增 datasource_name 字段，升级后我们可以配置 Job 服务，将不同租户数据库中的业务行为数据，分组统计后刷新到租户运营管理数据库中，再由配置的统计表单进行显示。</li> 
   <li style="text-align:left">对于上述介绍的 WebAdmin 后台服务，与之前版本一致，仍然提供表单和权限功能。而 WebApi 作为面向前端的接口应用，为了降低架构师们的裁剪工作量，该类应用将不提供表单和权限功能。</li> 
   <li style="text-align:left">多租户工程，支持全局公用字典和租户字典，前者由租户运营管理后台统一管理，租户字典数据可由租户管理员自行维护。为了保证整体运行时效率，两者均支持缓存 Redis。</li> 
  </ul> 
  <h4 style="text-align:left">修改说明</h4> 
  <div style="text-align:left"> 
   <div style="text-align:left"> 
    <ul> 
     <li style="text-align:left">修改生成器的工程配置页面，以支持多应用。</li> 
     <li style="text-align:left">用户在生成器中配置多个表单相互引用，当表单循环依赖时，生成器权限推演存在循环引用的bug，已修复。</li> 
     <li style="text-align:left">在生成器中刷新导入数据表字段名称时，没有正确同步到服务表中，已修复。</li> 
     <li style="text-align:left">配置一对多从表过滤时，从表的数据字段没有同步到表单的过滤选项中，因此配置表单时无法选择从表字段作为过滤选项，已修复。</li> 
     <li style="text-align:left">导入的数据表字段为 Blob 时，导入报错，目前已支持Blob字段。</li> 
     <li style="text-align:left">对于微服务工程，原有的接口包名为 xxxx-interface，现改为 xxxx-api。</li> 
     <li style="text-align:left">修复在配置全表刷新类型的 Job 处理器时，存在的数据表验证错误的 Bug。</li> 
     <li style="text-align:left">在生成器中，去掉 Api 生成模式，新版本的 WebApi 应用将更好的支持这一特性。</li> 
    </ul> 
    <h4 style="text-align:left">显著优势</h4> 
    <div style="text-align:left"> 
     <div style="text-align:left"> 
      <ul> 
       <li style="text-align:left">有助于中台化平台架构梳理，支持微服务池共享，可生成 70% 以上相对复杂的中台化业务代码。</li> 
       <li style="text-align:left">纯手工打造、合理驾驭模式、教科书般的 5A 级代码质量，顺利通过 SonarQube 和 Alibaba Code Guide 的最严格扫描。</li> 
       <li style="text-align:left">优质的代码、前沿高效的设计、成熟的套路和策略，在提升前期开发效率的同时，也将极大的降低后期维护成本。</li> 
       <li style="text-align:left">支持业内全部主流微服务架构技术栈，并可根据需求进行正交化配置组合。</li> 
       <li style="text-align:left">通过橙单可以配置，应用结构、服务划分、数据库链接、数据表结构、字段约束、数据表关联、数据字典、数据同步规则、Job 分组计算规则、菜单和表单等，随即便可生成开箱即用的完整工程代码。</li> 
       <li style="text-align:left">专业、完善、详尽、暗黑护眼、完全免费开放且同步更新的近 18 万字在线文档。</li> 
       <li style="text-align:left">技术问题的反馈、咨询和交流，我们都会及时回复，经常在开车等灯时答疑。</li> 
       <li style="text-align:left"><strong>生成后工程无任何 License 限制，可进行任意修改，并用于任意数量的商用项目</strong>。</li> 
      </ul>
