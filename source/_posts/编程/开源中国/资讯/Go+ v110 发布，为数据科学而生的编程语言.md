                                                                                            <p>Go+ v1.1.0 已发布，这是一门适用于工程、STEM 教育和数据科学的编程语言。主要特性包括：</p> 
<ul style="list-style-type:disc; margin-left:0; margin-right:0"> 
 <li>静态类型语言。</li> 
 <li>与<span> </span>Go 完全兼容。</li> 
 <li>脚本化的风格，以及比 Go 更易于阅读的数据科学代码。</li> 
 <li> <p style="margin-left:0; margin-right:0"><span style="background-color:#ffffff; color:#24292f">支持字节码后端和 Go 代码生成。在字节码模式下，Go+ 不支持<span> </span></span><code>cgo</code><span style="background-color:#ffffff; color:#24292f">。然而，在 Go 代码生成模式下，Go+ 完全支持<span> </span></span><code>cgo</code><span style="background-color:#ffffff; color:#24292f">。</span></p> </li> 
</ul> 
<p><strong>新版本亮点</strong></p> 
<p style="text-align:start"><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>1. 引入 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgoplus%2Fmod" target="_blank">https://github.com/goplus/mod</a> 模块。此模块提供对 </span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span>Go/Go+ module <span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>相关功能的支持。可以看作是对</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><span style="background-color:#ffffff; color:#24292f"><span> </span></span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fpkg.go.dev%2Fgolang.org%2Fx%2Fmod" target="_blank">golang.org/x/mod</a><span style="background-color:#ffffff; color:#24292f"><span> </span></span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>的增强，其中包括：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fpkg.go.dev%2Fgithub.com%2Fgoplus%2Fmod" target="_blank">github.com/goplus/mod</a>：提供查找 go.mod/gop.mod 文件的能力。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fpkg.go.dev%2Fgithub.com%2Fgoplus%2Fmod%2Fmodfile" target="_blank">github.com/goplus/mod/modfile</a>：为 gop.mod 文件提供读写支持。</li> 
 <li>github.com/goplus/mod/&#123;modload, gopmod&#125;：提供 Go+ Module 类。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fpkg.go.dev%2Fgithub.com%2Fgoplus%2Fmod%2Fmodcache" target="_blank">github.com/goplus/mod/modcache</a>：提供对模块缓存目录的支持。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fpkg.go.dev%2Fgithub.com%2Fgoplus%2Fmod%2Fmodfetch" target="_blank">github.com/goplus/mod/modfetch</a>：提供模块下载功能。</li> 
</ul> 
<p>2. <span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>改进<code>gop mod</code>（模块管理）相关命令，尤其是<code>gop mod tidy</code>很多情况下需要考虑的命令。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p style="text-align:start"><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>3. 增强<code>gop</code>等命令功能的完整性，例如<code>gop build/install/run/test</code>，主要体现在：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li>支持远程包。例如，在 Go+ v1.0 中<code>gop run</code>只能操作本地目录或文件，现在<code>gop run</code>支持运行远程包。</li> 
 <li>保证<code>gop build/install/run/test</code>一致性。以前，<code>gop run</code>和<code>gop build/install</code>的代码是分开编写的，缺乏统一的支持框架，最终行为会有所不同。现在它们被统一到同一个实现框架中。</li> 
 <li>修复 gop 缓存错误，在 Go+ v1.0 中，由于 import 包的实现性能较差，引入了 gop cache 模块缓存，导致模块更新检测出现一些 bug，以及导入一些复杂包的死循环。</li> 
</ul> 
<p style="text-align:start"><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>4. 增加 v1.0 缺少的语法特性。主要包括：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li style="text-align:start">支持重载<code>++</code>,<code>--</code>运算符</li> 
 <li>表达式<code>type()</code>支持用于表示类型<code>type</code>的零值。例如，<code>string()</code>等价于<code>""</code>，<code>int32()</code>等价于<code>int32(0)</code>，等等。</li> 
 <li>支持<code>uint128</code>,<code>int128</code>类型。</li> 
 <li>支持<code>any</code>类型，相当于空接口。</li> 
</ul> 
<p style="text-align:start"><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>5. 支持混写 </span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span>Go/Go+ 代码</p> 
<p style="text-align:start"><img alt src="https://oscimg.oschina.net/oscnet/up-2c554a09053ec4f699c726c6c26760eef1b.png" referrerpolicy="no-referrer"></p> 
<p style="text-align:start">详情参考 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgoplus%2Fgop%2Ftree%2Fv1.1%2Ftestdata%2Fmixgo" target="_blank">gop/testdata/mixgo</a> 示例。</p> 
<p>6. 支持<span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>从 Go+ 调用 C，包括：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li><code>gop c</code>命令（相当于独立<code>c2go</code>命令）可用于将 C 项目转换为 Go 项目。</li> 
 <li><code>import "C"</code><code>import "C/xxx"</code>用于导入 c2go 转换的 C 工程。<code>import "C"</code>是<code>import "C/github.com/goplus/libc"</code>的缩写。</li> 
 <li><code>C"xxx"</code>语法表示 C 风格的字符串常量。</li> 
</ul> 
<p>以下代码是<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgoplus%2Fgop%2Ftree%2Fv1.1%2Ftestdata%2Fhelloc2go" target="_blank"> Go+ 与 C 交互的示例</a>。</p> 
<pre><code>import "C"

C.printf C"Hello, c2go!\n"
C.fprintf C.stderr, C"Hi, %7.1f\n", 3.14</code></pre> 
<p style="text-align:start"><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>目前 Go+ 对 C 的支持只是一个预览版，还没有达到可用于工程项目的程度。据介绍，就 libc 而言，目前的迁移进度只有 5% 左右，而这仅仅是个开始。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>在即将到来的 Go+ v1.2 版本规划中，对 C 的完全支持被列为重中之重。当然，对 cgo 和 Go Module 的支持也在规划中，这是混写 Go/Go+ 代码的关键能力增强。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p>
