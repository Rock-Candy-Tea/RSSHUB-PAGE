                                                        <p><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromereleases.googleblog.com%2F2022%2F03%2Fstable-channel-update-for-desktop_29.html" target="_blank">Chrome 100</a> 现已升级为稳定版，适用于 Linux / macOS / Windows / Chrome OS。“Chrome  团队很高兴地宣布，Chrome 100 已推广到 Windows、Mac 和 Linux 的稳定频道。Chrome 100 也被推广到我们新的 Windows 和 Mac 的扩展稳定频道。这将在未来几天/几周内推出。”</p> 
<p>新版本的众多变化<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.phoronix.com%2Fscan.php%3Fpage%3Dnews_item%26px%3DGoogle-Chrome-100" target="_blank">包括有</a>：</p> 
<ul> 
 <li>更新 cookie 字符串的解析以允许将 cookie 的域属性设置为空字符串。此更改还将纠正与空字符串域相关的失败 Web 平台测试。此外，它使 Chrome 与 RFC6265bis 规范草案保持一致，并将通过匹配对空 cookie 域属性的处理来提高与 Safari 和 Firefox 的互操作性。</li> 
 <li>Capability Delegation 支持，框架可以放弃调用受限 API 的能力并将能力转移到它信任的另一个子框架。</li> 
 <li>引入了数字商品 API，用于查询/管理来自 Web 应用程序的应用内购买的数字产品。Google Play 商店中的网络应用程序现在可以轻松接受数字商品的购买。Digital Goods API 最初被视为 Origin Trial。</li> 
 <li>多屏幕窗口放置 API 能够更好地处理需要准确定位某些窗口和其他类似用例的多窗口应用程序。多屏幕窗口放置 API 的示例用例包括跨屏幕幻灯片放映、跨显示器窗口仪表板的财务应用程序、医学成像应用程序、多屏幕上的创意应用程序等等。</li> 
 <li>mix-blend-mode CSS 选项的“plus-lighter”值，允许两个元素通过将另一个元素的不透明度从 0 更改为 1 和从 1 更改为 0 来交叉淡入淡出，同时保持 common pixels 不变。</li> 
</ul> 
<p>更多详细信息可请访问 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeatures%23milestone%253D100" target="_blank">ChromeStatus.com</a>。与此同时，谷歌方面还发布了一个更新的 Chrome 101 开发版本。</p> 
<p>谷歌<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fweb.dev%2Fchrome-firefox-100%2F" target="_blank">曾警告</a>称，Chrome 和 Firefox 达到 100 版本可能会对那些依靠识别浏览器版本来执行业务逻辑的网站造成破坏，并为此提供了一些<span style="background-color:#ffffff; color:#191919">减轻影响而采取的策略。</span></p>
