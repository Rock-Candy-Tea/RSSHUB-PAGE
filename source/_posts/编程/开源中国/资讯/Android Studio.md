                                                                                            <p>Android Studio "Arctic Fox" 2020.3.1 Patch 1 现已发布。从 2020.3.1 起，Android Studio 开始启用新的版本号命名方案。事实上，2020.3.1 就是原计划的 4.3。<a href="https://www.oschina.net/news/142143/android-studio-arctic-fox-beta" target="_blank">谷歌称</a>采用新的方案主要是为了与 IntelliJ 的版本对齐。此次更新是该系列的第一个补丁，主要用于修复 bug。</p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li>Android Gradle 插件 
  <ul> 
   <li>TypedefRemover 使用 ASM5，并且与需要 ASM7 的 JDK 11 源不兼容</li> 
   <li>AGP 7.0.0 中 的 Groovy DSL 无法使用某些新的 DSL 块</li> 
   <li>AGP 7.0.0 稳定版在 libraryVariants.all&#123;applicationId&#125; 上抛出 ExternalApiUsageException</li> 
  </ul> </li> 
 <li>C++ 调试器 
  <ul> 
   <li>在 Mac M1 上启动本机调试会话时出现 AS Arctic Fox 错误，“发现损坏的 LLDB 配置”</li> 
  </ul> </li> 
 <li>资源管理器 
  <ul> 
   <li>(Windows) 新建 > 矢量资源 > 图片.svg：生成的 <vector> xml 中的 “减号” 字符无效</li> 
  </ul> </li> 
 <li>收缩器 (R8) 
  <ul> 
   <li>修复 BridgeHoisting 中的 NPE 问题</li> 
   <li>升级到 7.0.0 后，R8 崩溃并显示 “内联后方法中遗留的意外使用” 错误</li> 
  </ul> </li> 
</ul> 
<p>详情请查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fandroidstudio.googleblog.com%2F2021%2F08%2Fandroid-studio-arctic-fox-202031-patch.html" target="_blank">更新公告</a>。</p>
