                                                                                            <p>Chrome 101 已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromereleases.googleblog.com%2F2022%2F04%2Fstable-channel-update-for-desktop_26.html" target="_blank">正式发布</a>！该版本引入试用版的联合凭据管理——Federated Credential Management API (FedCM) ，该 API 可创建保护隐私的联合身份，而无需使用第三方 cookie 等跨站点跟踪机制追踪每个具体用户的信息。</p> 
<h3 style="margin-left:0px"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeature%2F5273474901737472" target="_blank">优先提示</a>（Priority Hints）</h3> 
<p><span style="color:#333333">优先级提示功能可指示各类资源对浏览器的相对重要性，从而更好地控制加载资源的顺序。</span></p> 
<h3 style="margin-left:0px"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeature%2F5769668454252544" target="_blank">Web USB sameObject 行为</a></h3> 
<p style="margin-left:0px">使 USBDevice === 上的访问器返回的 USBConfiguration、USBInterface、USBAlternateInterface 和 USBEndpoint 实例具有可比性。</p> 
<h3 style="margin-left:0px"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeature%2F5640127995969536" target="_blank">USBDevice forget()</a></h3> 
<p>新的 USBDevice forget() 方法允许 Web 开发人员自愿撤销用户授予的 USB 权限。</p> 
<h3 style="margin-left:0px"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeature%2F5684870116278272" target="_blank">在第三方上下文中弃用和删除 WebSQL</a></h3> 
<p style="margin-left:0px"><span style="color:rgba(0,0,0,0.87)">Web SQL 数据库标准于 2009 年 4 月首次提出并于 2010 年 11 月放弃。Gecko 从未实现此功能，WebKit 在 2019 年弃用了此功能。因此 </span>Chrome 101<span style="color:rgba(0,0,0,0.87)"> 决定</span>在第三方上下文中弃用和删除 WebSQL 的使用。</p> 
<h3 style="margin-left:0px"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeature%2F5674031696052224" target="_blank">字体调色板</a></h3> 
<ul> 
 <li>新的<code>font-palette</code> CSS 属性，允许开发人员从颜色字体中选择调色板。</li> 
 <li>新的 <code><em>hwb()</em> </code>CSS 函数，用于为色相白度和黑度 (HWB) 值指定 sRGB 颜色。</li> 
</ul> 
<h3 style="margin-left:0px"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeature%2F6242376685191168" target="_blank">WebRTC 的 MediaCapabilities API</a> </h3> 
<p style="margin-left:0px">扩展 MediaCapabilities API 以支持 WebRTC 流。MediaCapabilities API 通过提供“配置是否受支持”以及“播放是否流畅”等信息，帮助网站调整用于视频播放的编解码器、分辨率等设置。</p>
