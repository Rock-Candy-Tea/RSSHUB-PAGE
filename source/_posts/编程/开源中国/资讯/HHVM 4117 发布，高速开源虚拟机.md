                                                        <p>HHVM 4.117 现已发布。这个版本的发布标志着对 4.111 版本的支持已经结束，HHVM 4.112-4.116，以及 4.80 和 4.102 LTS 版本还会继续得到支持。</p> 
<p>HHVM 4.117 更新内容如下：</p> 
<p><strong>Highlights</strong></p> 
<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdocs.hhvm.com%2Fhack%2Fsilencing-errors%2Fintroduction%23silencing-errors-with-hhfixmeunsafe_cast" target="_blank"><code>HH\FIXME\UNSAFE_CAST</code></a> 现在可以用作<code>HH_FIXME</code>comments 的更安全的替代品。</li> 
</ul> 
<p><strong>Breaking Changes</strong></p> 
<ul> 
 <li><code>Stringish</code>接口不再声明一个<code>__toString()</code>方法。此方法现在仅在<code>StringishObject</code>接口上声明（在 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fhhvm.com%2Fblog%2F2021%2F06%2F21%2Fhhvm-4.115.html" target="_blank">HHVM 4.115 中</a>添加 ）。 
  <ul> 
   <li><code>Stringish</code>上<code>__toString()</code>的声明具有误导性，因为 <code>string</code>是<code>Stringish</code>的一个子类型，但是尝试对<code>string</code>值调用任何方法都会导致运行时错误。</li> 
  </ul> </li> 
 <li>删除了各种传统阵列兼容性 INI 选项（自 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fhhvm.com%2Fblog%2F2021%2F03%2F31%2Fhhvm-4.103.html" target="_blank">HHVM 4.103</a> 以来，它们没有效果或没有按预期工作 ）： 
  <ul> 
   <li><code>hhvm.raise_array_serialization_notices</code></li> 
   <li><code>hhvm.hack_arr_compat_intish_cast_notices</code></li> 
   <li><code>hhvm.hack_arr_compat_notices</code></li> 
   <li><code>hhvm.array_provenance</code></li> 
  </ul> </li> 
</ul> 
<p><strong>Future Changes</strong> </p> 
<ul> 
 <li>正如<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fhhvm.com%2Fblog%2F2021%2F06%2F28%2Fhhvm-4.116.html%23future-changes" target="_blank">此前</a>宣布的那样 ，在未来，“fuzzy”比较运算符<code>==</code>和<code>!=</code>以及 <code>switch</code>语句将不再将不同类型的值视为相等。 
  <ul> 
   <li>INI 选项<code>hhvm.notice_on_coerce_for_eq</code>现在支持新值<code>3</code>。这启用了新行为，即比较不同类型的值将返回<code>false</code>。官方建议首先将此选项设置为<code>1</code>（引发通知）或 <code>2</code>（抛出异常）以捕获任何可能受到影响的比较。</li> 
  </ul> </li> 
</ul>
