                                                        <p>VirtualEnv 20.5.0 发布了，VirtualEnv 用于在一台机器上创建多个独立的 Python 运行环境，可隔离项目之间的第三方包依赖，为部署应用提供方便，把开发环境的虚拟环境打包到生产环境即可，不需要在服务器上再折腾一翻。 </p> 
<p>新版更新内容如下：</p> 
<p><strong>Features</strong></p> 
<ul> 
 <li>插件现在使用"selectable"入口点（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fpypa%2Fvirtualenv%2Fissues%2F2093" target="_blank">#2093</a>）</li> 
 <li> <p>将 libffi-7.dll 添加到 PyPy 的 dll 硬编码列表（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fpypa%2Fvirtualenv%2Fissues%2F2141" target="_blank">#2141</a>） </p> </li> 
 <li> <p>使用维护得更好的<code>platformdirs</code>而不是<code>appdirs</code>。（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fpypa%2Fvirtualenv%2Fissues%2F2142" target="_blank">#2142</a>）</p> </li> 
</ul> 
<p><strong>Bugfixes</strong></p> 
<ul> 
 <li> <p>将 pip 升级到 21.1.3，setuptools 升级到 57.1.0。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fpypa%2Fvirtualenv%2Fissues%2F2135" target="_blank">#2135</a>）</p> </li> 
</ul> 
<p><strong>弃用和删除</strong> </p> 
<ul> 
 <li> <p>放弃对 python<code>3.4</code>的支持，因为它自 EOL 起已经超过 2 年时间了。（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fpypa%2Fvirtualenv%2Fissues%2F2141" target="_blank">#2141</a>）</p> </li> 
</ul> 
<p>更新说明：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fvirtualenv.pypa.io%2Fen%2Flatest%2Fchangelog.html" target="_blank">https://virtualenv.pypa.io/en/latest/changelog.html</a></p>
