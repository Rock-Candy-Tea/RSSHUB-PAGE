                                                                                            <p>Parcel 是用于 Web 的零配置构建工具。它将出色的开箱即用开发体验与可扩展的体系结构相结合，可将你的项目从零发展为大规模生产应用程序。</p> 
<p>Parcel 2.5 发布，更新内容如下：</p> 
<h3>新增：</h3> 
<ul> 
 <li>增加对 Web Extension manifest v3 的支持</li> 
 <li>在为 Node.js 目标构建时，重写 <code>__dirname</code> 和 <code>__filename</code> 以参考原始路径</li> 
 <li>为 JSON5 生成 codeframe 位置</li> 
 <li>在 Web Extension manifest 中增加 <code>$schema</code> 支持</li> 
 <li>增加对带有 <code>process.env</code> 的 <code>in</code> 表达式的支持，例如 <code>'foo' in process.env</code></li> 
</ul> 
<h3>修复</h3> 
<ul> 
 <li>更新了 SWC</li> 
 <li>更新 Parcel CSS 到 v1.8.1</li> 
 <li>修复诊断信息</li> 
 <li>修复了 "Asset was skipped or not found" 的错误</li> 
 <li>不把内联 <code><style></code> 元素作为 CSS 模块处理</li> 
 <li>修复 <code>srcset</code> 属性中多个图片的问题</li> 
 <li>修复对等的依赖关系</li> 
 <li>修复解析符号的问题</li> 
 <li>修正在 package.json 中使用 <code>"source"</code> 字段指定条目时加载 <code>.env</code> 文件的问题</li> 
 <li>……</li> 
</ul>
