                                                                                            <p style="color:#000000; margin-left:0; margin-right:0; text-align:start">Apache Traffic Control 6.0.1 现已发布。其可以用于建立一个大规模的内容交付网络。围绕 Apache Traffic Server 作为缓存软件，Traffic Control 实现了现代 CDN 的所有核心功能。</p> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start"><strong>主要更新内容</strong></p> 
<ul> 
 <li style="text-align:start">在 Traffic Ops 中添加了对 httpBypassFqdn 作为主机名的验证</li> 
 <li style="text-align:start">修复 /cdns/&#123;name&#125;/federations?id=# 以搜索 CDN</li> 
 <li style="text-align:start">即使安装了 Python 3，Traffic Ops Postinstall 脚本也可以在 CentOS 7 中运行</li> 
 <li style="text-align:start">修复流量监视器日志不一致的问题</li> 
 <li style="text-align:start">TO /deliveryservices/:id/routing 向所有 TR 而不是 CDN 发出请求</li> 
 <li style="text-align:start">流量操作：在执行 LDAP 查询之前清理用户名</li> 
 <li style="text-align:start">更新了 CDN-in-a-Box，默认情况下不运行 Riak 容器，而是仅在提供可选标志时才运行它</li> 
 <li style="text-align:start">更改了 DNSSEC 刷新 Traffic Ops API，仅在实际刷新任何键或发生错误时才创建新的更改日志条目（以减少更改日志噪音）</li> 
</ul> 
<p style="color:#000000; margin-left:0px; margin-right:0px; text-align:start">详情请查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Ftrafficcontrol.apache.org%2Freleases%2F" target="_blank">更新公告</a>。</p>
