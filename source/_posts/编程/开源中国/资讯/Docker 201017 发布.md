                                                                                            <p><span style="background-color:#ffffff; color:#333333">Docker 20.10.17 现已发布。<span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>此版本的 Docker Engine<span> 附带了 compose、containerd 和 runc 组件的更新版本，以及一些小的错误修复。具体更新内容如下：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p><strong><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Client</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></strong></p> 
<ul> 
 <li>删除 zsh <span style="background-color:#ffffff; color:#24292f">completion script </span>中 docker 命令中的 asterisk <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fcli%2Fpull%2F3648" target="_blank">docker/cli#3648</a>。</li> 
</ul> 
<p style="text-align:start"><strong><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Networking</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></strong></p> 
<ul> 
 <li>修复 Windows 端口与主机模式下发布的端口的冲突，用于 overlay <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43644" target="_blank">moby/moby#43644</a>。</li> 
 <li>确保性能调整始终适用于 libnetwork 沙箱 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43683" target="_blank">moby/moby#43683</a>。</li> 
</ul> 
<p style="text-align:start"><strong><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Packaging</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></strong></p> 
<ul> 
 <li>将 Docker Compose 更新到 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fcompose%2Freleases%2Ftag%2Fv2.6.0" target="_blank">v2.6.0</a>。</li> 
 <li>将 containerd（<code>containerd.io</code>包）更新到 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fcontainerd%2Fcontainerd%2Freleases%2Ftag%2Fv1.6.6" target="_blank">v1.6.6</a>，其中包含针对 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcve.mitre.org%2Fcgi-bin%2Fcvename.cgi%3Fname%3DCVE-2022-31030" target="_blank">CVE-2022-31030 的修复</a>。</li> 
 <li>将 runc 版本更新为v1.1.2 ，其中包含针对 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcve.mitre.org%2Fcgi-bin%2Fcvename.cgi%3Fname%3DCVE-2022-29162" target="_blank">CVE-2022-29162</a> 的修复。</li> 
 <li>将 Go 运行时更新至 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fdoc%2Fdevel%2Frelease%23go1.17.minor" target="_blank">1.17.11</a>，其中包含针对 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcve.mitre.org%2Fcgi-bin%2Fcvename.cgi%3Fname%3DCVE-2022-30634" target="_blank">CVE-2022-30634</a>、<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcve.mitre.org%2Fcgi-bin%2Fcvename.cgi%3Fname%3DCVE-2022-30629" target="_blank">CVE-2022-30629</a>、<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcve.mitre.org%2Fcgi-bin%2Fcvename.cgi%3Fname%3DCVE-2022-30580" target="_blank">CVE-2022-30580</a> 和 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fcve.mitre.org%2Fcgi-bin%2Fcvename.cgi%3Fname%3DCVE-2022-29804" target="_blank">CVE-2022-29804 的修复</a>。</li> 
</ul>
