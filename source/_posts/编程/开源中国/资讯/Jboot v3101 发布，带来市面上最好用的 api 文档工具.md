                                                        <blockquote> 
 <p style="text-align:left">Jboot 一个更简单的分布式、微服务框架。</p> 
 <p style="text-align:left">Jboot是一个基于 JFinal、JFinal-Undertow、Dubbo、Seata、Sentinel、ShardingSphere、Nacos 等开发的微服务框架，帮助开发者降低微服务、分布式开发门槛。爽爽开发，快乐生活。</p> 
</blockquote> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">到目前为止，Jboot 已经开源超过了 5 年的时间，迭代了 200+ 个版本，已经被超过 1000+ 公司在使用，其中包含了多个知名的上市公司，我们了解到的多个使用 Jboot 开发的产品，用户量超过 1亿 以上。</span></p> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">从 Jboot v3.10.0 的开始，带来了自研的 API 文档生成工具。目前市面上的 API 文档工具生成，有非常多的弊端，就拿大名鼎鼎的 Swagger 对比 Jboot 而言。Swagger 有以下几个缺点：</span></p> 
<p style="text-align:left"> </p> 
<p style="text-align:left"><strong>缺点1：大量的繁杂注解，而 Jboot 只有 3 个，比如：</strong></p> 
<p style="text-align:left">Swagger：</p> 
<pre style="text-align:left"><code class="language-java"><span style="color:#6a737d">@ApiOperation(value="更新用户详细信息", notes="...")</span>
<span style="color:#6a737d">@ApiImplicitParams(&#123;
            @ApiImplicitParam(name = "id", value = "用户ID", required = true, dataType = "Long")</span>,
            <span style="color:#6a737d">@ApiImplicitParam(name = "user", value = "用户详细实体user", required = true, dataType = "User")</span>
    &#125;)
<span style="color:#6a737d">@RequestMapping(value="/&#123;id&#125;", method=RequestMethod.PUT)</span>
<span style="color:#d73a49">public</span> String putUser(<span style="color:#6a737d">@PathVariable</span> Long id, <span style="color:#6a737d">@RequestBody</span> User user) &#123;
        User u = users.<span style="color:#d73a49">get</span>(id);
        u.setName(user.getName());
        u.setAge(user.getAge());
        users.put(id, u);
        <span style="color:#d73a49">return</span> <span style="color:#032f62">"success"</span>;
&#125;    
</code></pre> 
<p style="text-align:left">Jboot:</p> 
<pre style="text-align:left"><code class="language-java"><span style="color:#6a737d">@ApiOper(value="更新用户详细信息", notes="...")</span>
<span style="color:#d73a49">public</span> String putUser(<span style="color:#6a737d">@ApiPara('用户ID')</span>Long id, <span style="color:#6a737d">@ApiPara('用户详细实体use')</span><span style="color:#6a737d">@JsonBody</span> User user) &#123;
        User u = users.<span style="color:#d73a49">get</span>(id);
        u.setName(user.getName());
        u.setAge(user.getAge());
        users.put(id, u);
        <span style="color:#d73a49">return</span> <span style="color:#032f62">"success"</span>;
&#125;    
</code></pre> 
<p style="text-align:left"><strong>缺点2：Swagger 无法自动推导参数类型，而 Jboot 可以自动推导更加智能。</strong></p> 
<p style="text-align:left">Jboot：</p> 
<pre style="text-align:left"><code><span style="color:#032f62">@ApiOper</span>(value=<span style="color:#032f62">"更新用户详细信息"</span>, notes=<span style="color:#032f62">"..."</span>)
public String putUser(<span style="color:#032f62">@Min</span>(1) Long id, <span style="color:#032f62">@NotNull</span> <span style="color:#032f62">@JsonBody</span> User user) &#123;
  ...
&#125;    
</code></pre> 
<p style="text-align:left">在 Jboot 中，有很多的验证注解，比如 @Email、@NotNull、@Min、@NotEmpty... 等等注解会自动推导并放到文档（或者测试前端）进行显示或自动验证。</p> 
<p style="text-align:left">除了这些验证注解以外，Java 类型，返回参数等等都可以智能的进行自动推导。</p> 
<p style="text-align:left"><strong>缺点3：Swagger 只能通过其前端进行查看文档或进行测试验证。而 Jboot 可以扩展适配任何的测试框架，比如  Postman、OpenAPI 等等... 又可以生成 Markdown 文档、Html、Word 等等类型文档。</strong></p> 
<p style="text-align:left"><strong>缺点4：Swagger 对 Model 生成的数据需要一大堆的 ApiModel 和 ApiModelProperty 注解，而 Jboot 可以直接通过数据库表的字段注释直接生成，无需手动注解维护。</strong></p> 
<p style="text-align:left"> </p> 
<p style="text-align:left">比如：如下的截图是 Jboot 自动生成的文档，包含了 <strong>请求参数信息</strong>、<strong>是否进行非空验证、请求参数相关含义、返回值、返回值相关字段信息、 Json 示例 </strong>等等<strong>。</strong></p> 
<p style="text-align:left"> </p> 
<p style="text-align:left"><img height="959" src="https://oscimg.oschina.net/oscnet/up-cc86548553fa8dfc9c4c0f88bf3029fc690.JPEG" width="1372" referrerpolicy="no-referrer"></p> 
<p style="text-align:left">查看更多的示例，大家可以参考网址：<a href="https://www.oschina.net/action/GoToLink?url=http%3A%2F%2Fdoc.jpress.io%2Fdevelopment%2Fapi%2Fapi_user.html" target="_blank">http://doc.jpress.io/development/api/api_user.html</a> ，其内容全部都是 Jboot 自动生成的。</p> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#212529">接下来，除了文档以外，Jboot 会推出一些企业的功能，放在 JbootAdmin 里给到企业用户，比如  API 自动测试、 API Mock，一键导入到 Postman 、自动根据 Controller 接口、自动生成  微信小程序的 SDK 、Java SDK 等等功能... 敬请期待。</span></p> 
<p style="text-align:left"> </p> 
<blockquote> 
 <p>总而言之：<strong>Swagger 需要繁杂的注解，无法自动推导，而且，相对市面上的其他文档框架而言，Jboot 的代码是生成工具又是最为灵活和轻量的，除了 Model 定义，5个类就实现了以上全部的功能。</strong></p> 
</blockquote> 
<p style="text-align:left"><strong>Jboot v3.10.1 更新内容如下：</strong></p> 
<ul> 
 <li>新增：ApiDoc 新增 allInOne 模式，方便把所有 api 生成到一个文档里</li> 
 <li>新增：@Api() 注解新增 orderNo 的配置</li> 
 <li>新增：可以通过 api-remarks.json 和 api-mock.json 为文档配置 json 输出</li> 
 <li>新增 "ApiJsonGenerator"，用于生成通过数据库生成 api-remarks.json 和 api-mock.json 文件</li> 
 <li>新增：ApiDoc 文档新增自定义排序 Comparator 配置的支持</li> 
 <li>新增：ApiDoc 新增自定义 ApiMockBuilder 的支持，用于构建任意 Model 的 Mock 数据</li> 
 <li>新增："@ApiResp" 注解，用于对 JFinal 通过 render 而非返回值的形式的支持</li> 
 <li>新增：ApiDocument 等信息添加序列化的支持</li> 
</ul>
