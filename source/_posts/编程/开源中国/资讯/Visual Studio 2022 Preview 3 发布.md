                                                                                            <p>Visual Studio 2022 Preview 3 现已推出，该版本提供了更多有关于个人和团队生产力、现代开发和不断创新为主题的新功能。</p> 
<p>下载地址：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Faka.ms%2Fvs2022preview" target="_blank">https://aka.ms/vs2022preview</a></p> 
<h4><strong>个人和团队生产力</strong></h4> 
<p>在 Preview 3 中，开发团队为一些不太常用但很有用的功能添加了新功能，例如附加到进程。</p> 
<ul> 
 <li><strong>附加到进程改进</strong></li> 
</ul> 
<p>一个例子是对附加到进程的对话框的改进。该对话框现在是异步的、显示进程的命令行参数、w3wp.exe 进程的 IIS 信息，而且该对话框还有一个可选的树状视图模式，用于显示 parent-child 进程关系。这些功能减少了在高级场景中决定调试哪个进程的很多摩擦。</p> 
<ul> 
 <li><strong>新项目设计器</strong></li> 
</ul> 
<p>在 Preview 3 中，为 .NET SDK 项目提供了一个全新的项目属性设计器。新的设计器更容易使用和浏览，单列的选项有清晰的描述。最重要的是，新的设计器有内置的搜索功能，所以现在很容易找到你需要的属性。</p> 
<p><img alt height="394" src="https://oscimg.oschina.net/oscnet/up-41c6e243ea499dd1cd66eaaffe68189566e.gif" width="700" referrerpolicy="no-referrer"></p> 
<ul> 
 <li><strong>Dark 主题改进</strong></li> 
</ul> 
<p>在 Preview 3 中，你会看到 dark 主题的巨大变化，以提高 Visual Studio 的可用性。新的 dark 主题有一个新的强调色，旨在减少分心和眼疲劳。该颜色现在与最新的产品视觉识别相匹配，可帮助你在多个工具之间导航时快速找到正确的窗口。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-83432ca6dd994dc74d993057391f7346436.gif" referrerpolicy="no-referrer"></p> 
<h4><strong>开发现代应用</strong></h4> 
<p>在 Preview 3 中，添加了在 Linux 环境中运行测试的新功能，以及用于使用 TypeScript 或 JavaScript 的 React 和 Vue.js 应用程序的前端开发的新项目类型。</p> 
<ul> 
 <li><strong>远程测试</strong></li> 
</ul> 
<p>通过远程测试，你现在可以从你的跨平台测试中获得反馈，甚至可以在 Visual Studio 中对它们进行调试。该功能适用于一系列的远程环境，如 Linux 容器、WSL 和 SSH 连接 — 使你能够测试现代跨平台的 .NET 应用程序。</p> 
<p><strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdevblogs.microsoft.com%2Fvisualstudio%2Fvisual-studio-2022-preview-3-now-available%2F%23remote-testing" target="_blank"><img alt src="https://oscimg.oschina.net/oscnet/up-8cdbca8d8679bbdab5f3812b37e536046b9.gif" referrerpolicy="no-referrer"> </a></strong></p> 
<ul> 
 <li><strong>新的 JavaScript 和 TypeScript 项目类型</strong></li> 
</ul> 
<p>用于 Angular、Vue 和 React 的新 JavaScript 和 TypeScript 项目类型使你可以轻松地将前端应用程序合并到你的 Visual Studio 解决方案中 – 使你能够从 Visual Studio 启动和调试整个应用程序。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-25f27624de4df4d7db76acbe1f5bd025beb.gif" referrerpolicy="no-referrer"></p> 
<h4>创新触手可及</h4> 
<p>微软方面表示，其<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdevblogs.microsoft.com%2Fvisualstudio%2Fvisual-studio-2022%2F" target="_blank">对 Visual Studio 2022 愿景的</a>另一个关键部分是创新功能，可释放用户作为开发人员的潜力。</p> 
<ul> 
 <li><strong>诊断分析</strong></li> 
</ul> 
<p>Preview 3 中的诊断分析器可用于内存转储，以识别 .NET 应用程序中的常见问题；使你能在短时间内找到复杂问题的根源。</p> 
<p><strong><img alt src="https://oscimg.oschina.net/oscnet/up-09e45a04d0f904117221d53f3118f839539.gif" referrerpolicy="no-referrer"></strong></p> 
<ul> 
 <li><strong>同时使用多个 Git 存储库</strong></li> 
</ul> 
<p>Preview 3 还获得了新的预览功能，例如多个 Git 存储库，它使你能够使用具有多个存储库中的项目的单个解决方案，并从 Visual Studio 的单个实例为它们做出贡献。</p> 
<ul> 
 <li><strong>一键发布到 Azure DevOps</strong></li> 
</ul> 
<p>在 Preview 3 中，通过更新的创建 Git 存储库体验，可以更轻松地将代码发布到 Azure DevOps。只需单击一下，Visual Studio 即可初始化一个新的 Git 存储库并将其推送到 Azure DevOps。</p>
