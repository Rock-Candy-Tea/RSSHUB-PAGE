<h2>一</h2> 
<p>TLog v1.3.5 正式发布！相关文档已经做了更新。</p> 
<p>TLog是一款日志工具，十分钟即可接入，支持众多的框架和主流RPC，你几乎不需要做什么，TLog能自动你的日志进行增强，让你的日志马上升级，微服务之间的日志变得可追溯！</p> 
<p>如果你想了解更多，请点击以下地址进行了解。</p>

<h2>二</h2> 
<p>TLog开源1年以来，作为一款日志增强框架。因为便捷使用和支持众多的其他框架，获得了很多人的喜爱。拥有了一个非常好的社区。</p> 
<p>本次更新主要围绕着继续支持其他框架为主，更新日志如下：</p> 
<ul> 
 <li> <p><strong>feature: #I4H26M 多项目http交互打印上下游项目信息支持HuTool的HttpUtil工具发起的请求</strong> <a href="https://gitee.com/dromara/TLog/issues/I4H26M">https://gitee.com/dromara/TLog/issues/I4H26M</a></p> </li> 
 <li> <p><strong>feature: #I4L239 Tlog支持servlet</strong> <a href="https://gitee.com/dromara/TLog/issues/I4L239">https://gitee.com/dromara/TLog/issues/I4L239</a></p> </li> 
 <li> <p><strong>feature: #I4L23L TLog支持forest框架的调用传递标签</strong> <a href="https://gitee.com/dromara/TLog/issues/I4L23L">https://gitee.com/dromara/TLog/issues/I4L23L</a></p> </li> 
 <li> <p><strong>bug: #I4HM98 修复TLog整合xxl-job时，traceId一样的问题</strong> <a href="https://gitee.com/dromara/TLog/issues/I4HM98">https://gitee.com/dromara/TLog/issues/I4HM98</a></p> </li> 
 <li> <p><strong>bug: #I4KIPK 关于 SpanIdGenerator 在异步调用时 generateNextSpanId 存在重复 spandId 的问题</strong> <a href="https://gitee.com/dromara/TLog/issues/I4KIPK">https://gitee.com/dromara/TLog/issues/I4KIPK</a></p> </li>
