                                                        <p>IntelliJ IDEA 2021.2 EAP 2 现已发布。此版本带来了对 Eclipse projects 的快速访问、为 Windows 用户提供了 Async Profiler、一个捆绑的包搜索插件，对 pre-commit checks 的改变、有用的 UI 变化等等。</p> 
<p><strong>快速访问 Eclipse 项目</strong></p> 
<p>IntelliJ IDEA 现在可以自动检测 Eclipse projects。要在首次启动 IDE 时尝试这一功能，可选择打开现有的 Eclipse projects 选项。</p> 
<p><img alt height="280" src="https://oscimg.oschina.net/oscnet/up-8b75f6cae084cdc1953c1a4e731c3a39f11.png" width="700" referrerpolicy="no-referrer"></p> 
<p>如果这不是你第一次启动 IntelliJ IDEA，则 IDE 会自动检测现有的 Eclipse projects，并将它们放在欢迎屏幕上的专用节点中。这个节点将在最近的项目中可用，直到你决定删除它。当 Eclipse projects 在该节点中时，只要点击它就可以在 ID E中快速打开它。</p> 
<p><img alt height="280" src="https://oscimg.oschina.net/oscnet/up-cc89e01174e10045c1b8f53873a2b67e069.png" width="700" referrerpolicy="no-referrer"></p> 
<p><strong>Package Search integration</strong></p> 
<p>Package Search 现在有了一个全新的界面，并捆绑在 IntelliJ IDEA 社区版和 IntelliJ IDEA Ultimate 中。你可以使用 Package Search 来寻找新的依赖，并自动添加它们。除此之外，如果你的构建脚本中缺少所需的存储库，Package Search 也会将其添加到你的构建脚本中。</p> 
<p>Package Search 还可以让你升级、降级和删除现有的依赖关系，更改其范围并导航到其声明：</p> 
<p><img alt height="280" src="https://oscimg.oschina.net/oscnet/up-4c0f7b08daa2e84642f5347c7cfba04780b.gif" width="700" referrerpolicy="no-referrer"></p> 
<p>开发团队还改进了现有的 Maven 和 Gradle Groovy 集成、加了 Gradle Kotlin DSL 支持；并且其还正在为插件开发许多新功能，包括支持解析 Gradle 项目中的引用、与 Kotlin/Multiplatform 项目的集成以及对网站的许多改进。</p> 
<p><strong>Windows 上的 Async Profiler</strong></p> 
<p>开发团队已经在 Profiler 工具窗口中实现了对 Async Profiler 的支持；因此现在开始 Windows 用户可以将 Async Profiler 应用于正在运行的进程。</p> 
<p><img alt height="280" src="https://oscimg.oschina.net/oscnet/up-b7ce51ae21bdd42ca22575101ae81e37c47.png" width="700" referrerpolicy="no-referrer"></p> 
<p><strong>对 pre-commit checks 的更改</strong></p> 
<p>在此版本中，你只要点击 Commit 视图中的齿轮图标，选择 Run Tests 选项，并选择必要的运行配置。你的 IDE 就会测试你的文件，并在出现问题时通过警告来通知你。</p> 
<p><img alt height="279" src="https://oscimg.oschina.net/oscnet/up-25ac27d9a09bd65302ed28ecbf66405bccf.png" width="700" referrerpolicy="no-referrer"></p> 
<p>你还可以通过点击旁边的"Choose profile"来定制分析代码和清理选项。</p> 
<p><img alt height="280" src="https://oscimg.oschina.net/oscnet/up-1e5b4a8d7ecfcd166af684fe10a7b518174.png" width="700" referrerpolicy="no-referrer"></p> 
<p>值得一提的是，pre-commit checks 的进度和结果现在显示在 Commit 区，而不会用额外的模态窗口来干扰你。</p>
