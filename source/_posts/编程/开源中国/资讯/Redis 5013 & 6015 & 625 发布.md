                                                                                            <p>Redis 5.0.13、6.0.15 与 6.2.5 现已发布，主要包含 32 位版本上 BITFIELD 中的整数溢出问题 CVE-2021-32761 以及一些 bug 修复。</p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li>5.0.13 & 6.0.15 & 6.2.5 
  <ul> 
   <li>修复 32 位版本的 GETBIT、SETBIT、BITCOUNT、BITPOS 和 BITFIELD 溢出</li> 
   <li>修复大端模式（big-endian）平台上的 ziplist 长度更新</li> 
  </ul> </li> 
 <li>6.0.15 & 6.2.5 
  <ul> 
   <li>修复在监视的密钥过期的情况下 EXEC 命令失败的问题</li> 
   <li>修复当成员已经存在时，SMOVE 不会使目标密钥失效的问题</li> 
   <li>修复 SINTERSTORE 出现错误类型错误时不删除 dest 键的问题</li> 
   <li>在入站集群总线连接上设置 TCP keepalive</li> 
   <li>修复无盘副本加载以从模块 AUX 数据的 RDB 短读中恢复</li> 
   <li>修复客户端跟踪中的竞争</li> 
  </ul> </li> 
 <li>6.0.15 
  <ul> 
   <li>如果无盘 repl child 被杀死，请确保获得 child pid</li> 
   <li>为卡在完全同步中的副本添加超时机制</li> 
  </ul> </li> 
 <li>6.2.5 
  <ul> 
   <li>修复密钥丢失时 HRANDFIELD 和 ZRANDMEMBER 的返回类型</li> 
   <li>修复缺少源密钥时 ZRANGESTORE 的返回类型</li> 
   <li>带有负 COUNT 的 ZRANDMEMBER WITHSCORES 可能会返回分数</li> 
   <li>修复启用线程 I/O 配置时 CLIENT UNPAUSE 后的崩溃</li> 
   <li>使用 LIMIT 时 XTRIM 或 XADD 可能会删除比限制更多的条目</li> 
   <li>修复 OpenSSL 1.1.0 的构建问题</li> 
   <li>改进流密钥的内存使用</li> 
  </ul> </li> 
</ul>
