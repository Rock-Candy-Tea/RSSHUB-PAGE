                                                        <p>Apache Kafka 是一个分布式流平台，具有四个核心 API。借助这些 API，Kafka 可以用于以下两大类应用：建立实时流数据管道，可靠地进行数据传输，在系统或应用程序之间获取数据；构建实时流媒体应用程序，以改变系统或应用程序之间的数据或对数据流做出反应。</p> 
<p>近日 Apache Kafka 2.8.0 正式发布，本次更新的显著变化如下：</p> 
<ul> 
 <li>增加描述群组 API；</li> 
 <li>支持 SASL_SSL 监听器上的相互 TLS 认证；</li> 
 <li>JSON 请求/响应调试日志。</li> 
 <li>限制 broker 连接创建率；</li> 
 <li>主题标识符；</li> 
 <li>在 Connect REST API 中公开任务配置；</li> 
 <li>早期访问用一个自我管理的 Quorum 取代 ZooKeeper；</li> 
 <li>更新 Streams FSM，以表明错误状态的含义；</li> 
 <li>扩展 StreamJoined 以允许更多的 store 配置；</li> 
 <li>更加方便的 TopologyTestDriver construtors；</li> 
 <li>引入Kafka-Stream特有的未捕获异常 handler；</li> 
 <li>启动和关闭 Streams 线程的 API；</li> 
 <li>改进 TimeWindowedDeserializer 和 TimeWindowedSerde，以处理窗口尺寸；</li> 
 <li>改进 Kafka stream 中的超时和重试；</li> 
</ul>
