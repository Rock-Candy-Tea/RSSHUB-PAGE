                                                                                            <p style="margin-left:0">Eclipse 基金会已发布 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjakarta.ee%2Frelease%2F10%2F" target="_blank">Jakarta EE 10 平台</a>、Web Profile 和新的 Core Profile 规范。Jakarta EE 10 引入了用于构建现代化、简化和轻量级<strong>云原生</strong> Java 应用程序的功能。据 Eclipse 基金会执行董事 Mike Milinkovich 称</p> 
<blockquote> 
 <p style="margin-left:0">这个版本是在现代微服务和容器时代牢牢植入 Jakarta EE 的“大版本“，Jakarta EE 10 的发布反映了全球贡献者社区的工作，随着这个版本的发布，它现在已经为云原生时代带来了关键的创新，这对我们行业的未来至关重要。<br>  </p> 
</blockquote> 
<p style="margin-left:0">Jakarta EE 10 通过指定 API 中的版本更新，在 20 多个组件规范中提供了新功能。例如：</p> 
<ul> 
 <li>Jakarta Contexts and Dependency Injection (CDI) 4.0，包括支持构建时间扩展的 CDI-Lite </li> 
 <li>Jakarta RESTful Web Services 3.1 标准化了 Java SE Bootstrap API ，并支持多部分/表单数据 </li> 
 <li>Jakarta Security 3.0 支持 OpenID Connect 和 Jakarta Persistence 查询中的新功能，并使用纯 Java 创建 Jakarta Faces 视图 </li> 
 <li>Jakarta EE 10 版本使用 Jakarta EE Core Profile 10 定义了一个新的配置文件规范，该规范定义了一个用于现代化和轻量级 Java 应用程序和微服务的多供应商平台。</li> 
 <li>新的核心配置文件提供了 Jakarta EE 规范的子集，这些规范针对适合具有轻量级运行时的微服务开发的较小运行时，包括用于构建轻量级 Jakarta EE 应用程序的新 CDI-Lite 规范。</li> 
 <li>Jakarta Servlet 6.0 用于简化编程并提高安全性</li> 
 <li>带有使用 CDI 的现代化 API 的 Jakarta Faces (JSF) 4.0</li> 
 <li>Jakarta JSON Binding (JSON-B) 3.0 新增对多态类型的支持</li> 
 <li>Jakarta Persistence 将 UUID 标准化为基本类型并扩展查询语言和查询 API</li> 
 <li>Jakarta Concurrency 3.0 移至 Web Profile ，增强了应用程序可用的并行和反应式编程模型</li> 
</ul> 
<p style="margin-left:0">Jakarta EE 10 版本使构建现代化云原生 Java 应用程序的企业能够：</p> 
<ul> 
 <li>在 Java SE 11 和 Java SE 17（Java SE 的最新 LTS 版本）上开发和部署 Jakarta EE 10 应用程序</li> 
 <li>利用 Java SE 9 中引入并在 Java SE 11 中支持的模块系统等新功能</li> 
 <li>此版本通过支持更广泛地使用附加注释来简化应用程序开发，使构建模块化应用程序变得更加容易，并提供了跨组件 API 的改进集成。</li> 
 <li>此版本可帮助开发人员更轻松地实现具有现代安全要求的应用程序，并删除已弃用的 API，同时还保持与先前 Jakarta EE 版本的高度兼容性。</li> 
</ul> 
<p style="margin-left:0">除了这些增强功能之外，还有越来越多的兼容产品来支持已经丰富的生态系统。例如，Eclipse GlassFish 7、Open Liberty、22.0.0.10-beta 和 Wild Fly 27.0.0 目前正在提供兼容版本。</p>
