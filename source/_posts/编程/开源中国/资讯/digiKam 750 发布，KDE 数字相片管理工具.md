                                                                                            <p>digiKam 是 KDE 桌面环境的影像管理和编辑程序，支持所有主要图像格式，并可以组织目录为基础的照片收藏，或按日期、时间、或标签的动态相册。用户还可以对图像添加标题和注释，搜索他们和透过智能文件夹保存搜索。添加插件还可以输出到 Flickr 的相册、Gallery2、谷歌地球的 KML 文件、Simpleviewer、刻录成光盘或创建 Web 画廊。</p> 
<p>近日，digiKam 7.5.0 正式发布，本次更新的部分内容如下：</p> 
<p><img alt height="436" src="https://static.oschina.net/uploads/space/2022/0118/071713_MRHN_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<h3>翻译和 RTL 支持</h3> 
<p>digiKam 从项目开始就被 KDE 的翻译团队国际化了，目前支持的语言已超过 50 种。</p> 
<p><img alt height="436" src="https://static.oschina.net/uploads/space/2022/0118/071738_DQdu_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<p>在这个版本中，新增了从右到左的语言支持。还进行了一次大规模的翻译更新，测试并修复了所有关于 RTL 支持的主要问题。</p> 
<p>Fcitx aka Platform Input Context 是一个输入法框架，支持 Linux 系统的扩展。Fcitx 支持世界上许多语言的输入法，有大量的引擎。Fcitx 现在已经包含在 digiKam Linux AppImage 捆绑包中。</p> 
<h3>macOS 菜单中的 Open-With 选项</h3> 
<p>与 Linux 和 Windows 版本一样，macOS 用户可以在 digiKam 的上下文菜单中使用 Open-With 选项来打开喜欢的应用程序。随着 digiKam 7.5.0 的推出，这已成为现实。</p> 
<p><img alt height="436" src="https://static.oschina.net/uploads/space/2022/0118/071754_UBvz_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<h3>Bugzilla 统计数据</h3> 
<p>Bugzilla 包括开发者和用户之间所有的建设性交流。由于在这个月里处理了大量的错误，现在是时候对数据库中按平台分类的报告统计数据进行总体反馈了。</p> 
<p>开发 20 年以来，digiKam 团队已经关闭了：</p> 
<ul> 
 <li>9432 个在 Linux 下创建的文件</li> 
 <li>1031 个在 Windows 下创建的文件</li> 
 <li>248 个在 macOS 下创建的文件</li> 
 <li>410 个在其他操作系统下创建的文件</li> 
</ul> 
<p>1097 个文件仍然在待处理列表中开放，这使得在数据库中注册的文件总数达到 12218 个。对于仍在待处理列表中的文件：</p> 
<ul> 
 <li>775 个文件是在 Linux 下创建的</li> 
 <li>203 个文件是在 Windows 下创建的</li> 
 <li>43 个文件是在 macOS 下创建的</li> 
 <li>29 个文件是在其他操作系统下创建的</li> 
</ul> 
<p>虽然主要的平台是 Linux，但自从 7.0.0 版本发布后，由于应用程序具备更好的集成，Windows 和 macOS 的报告在 bugzilla 中也变得越来越重要。</p>
