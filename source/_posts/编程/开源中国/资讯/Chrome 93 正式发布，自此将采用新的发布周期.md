                                                                                            <p>Google 日前开始向用户陆续推送 Chrome 93 稳定版，除了为用户带来了一系列功能改进和错误修复以外，该版本也是 Google 常规发布节奏中的最后一个版本。从下个月的 Chrome 94 开始，该公司将转向 4 周的发布周期。</p> 
<h3>WEBOTP API：跨设备支持</h3> 
<p>Chrome 93 为桌面端用户带来了 WebOTP API 跨设备支持，如果用户在 Android 和 Chrome 桌面端登陆了同一个 Google 帐户，则可以在桌面端 Chrome 无缝处理发送到移动设备的一次性密码。</p> 
<h3>多屏窗口放置</h3> 
<p>增加了新的屏幕信息 API，并对现有的窗口放置 API 进行了逐步改进，使 Web 应用能够提供更好的多屏幕体验。API 实现的用例包括：</p> 
<ul> 
 <li>幻灯片应用在投影仪上展示，而在笔记本屏幕上显示演讲者的笔记</li> 
 <li>创意应用在一个单独的屏幕上显示辅助窗口</li> 
 <li>会议应用在触摸屏设备上显示控制菜单，在电视上显示视频</li> 
 <li>当一个窗口跨越多个屏幕时，优化了内容和布局</li> 
 <li>……</li> 
</ul> 
<h3>剪贴板 API：SVG</h3> 
<p>鉴于 SVG 图像越来越流行，Google 在 Async Clipboard API 中增加了 image/svg+xml 支持。未来支持 SVG 的网站可以在网站间通过剪切板复制/粘贴 SVG 图像。</p> 
<h3>删除 TLS 中的 3DES</h3> 
<p>TLS_RSA_WITH_3DES_EDE_CBC_SHA 是 SSL 2.0 和 SSL 3.0 时代的遗留物。TLS 中的 3DES 容易受到 Sweet32 的攻击，而且速度极慢、对于电量的消耗也更大。作为一个 CBC 密码套件，它也容易受到 "Lucky Thirteen" 的攻击。</p>
