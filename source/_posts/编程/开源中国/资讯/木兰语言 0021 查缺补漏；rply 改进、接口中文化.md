                                                        <p>今后也许会更身不由己，这段时间尽可能将 <a href="https://gitee.com/MulanRevive/mulan-rework">木兰重现项目 </a>和其依赖的 rply 库中的一些纰漏解决、夯实基础。</p> 
<h2>木兰</h2> 
<ul> 
 <li>支持 # 语法，等价于 len()，如 <code>#[3,33]</code> 输出 2。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fcommit%2Fa5f12497465352fb2c0db7cd7de45282a1b208af" target="_blank">见此提交</a></li> 
 <li>输出数据结构时内部字符串略去引号：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fcommit%2F1f20783a49cb66afa68bef628a6640aa16fcdfdf" target="_blank">集合、多项</a>； <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fcommit%2F8ed7d5e58a77eada9efdea0740f1c5f9ecabc357" target="_blank">字典</a>。比如 <code>d=&#123;1:'a'&#125;</code>，python 中 print 输出 <code>&#123;1: 'a'&#125;</code>，而木兰输出 <code>&#123;1: a&#125;</code></li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fissues%2FI37WKB%253Ffrom%253Dproject-issue" target="_blank">报错中模块名为空问题</a> 比较独特，逆向中看不出头绪，硬 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fcommit%2F70a86e16d6f0b2aa13d724d868d844918b3e352e" target="_blank">加了个补丁</a> 实现了与原始可执行文件行为一致</li> 
 <li>之前发现几处 re.DOTALL 不解何意，现 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fcommit%2Fcd68beec2e51e6528e8c9990dfee6e60f2a997b3" target="_blank">基本确定并无效果</a></li> 
 <li>【已实现，未发布】改进报错信息: 形参默认值不全。如下面函数：<code>func f(x=2, y) &#123; return x &#125;</code>，python 并不定位具体参数：<code>non-default argument follows default argument</code>，现在木兰可定位：</li> 
</ul> 
<div> 
 <pre><code class="language-text">   语法错误：文件 "【标准输入】", 第1行, 第13列, 参数“y”之前的“x”有默认值，它也需要一个表达式指定默认值
func f(x=2, ✋y) &#123; return x &#125; </code></pre> 
</div> 
<ul> 
 <li>【待复现】发现木兰支持如此定义多项（tuple）：<code>a = 3; b = 4; c = (a,b)</code>，但并不支持 <code>c=(3,4)</code>。因有依赖打算在 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fissues%2FI3TIKJ%253Ffrom%253Dproject-issue" target="_blank">此悬赏任务</a> 之后完成。</li> 
</ul> 
<p>当前有共三个悬赏任务，欢迎参与。</p> 
<h2>依赖库：rply 木兰定制版</h2> 
<p>自 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fzhuanlan.zhihu.com%2Fp%2F372021884" target="_blank">前文</a> 开始，将 rply 作了分支后剪裁、定制了 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fpypi.org%2Fproject%2Frply-ulang%2F" target="_blank">rply-ulang 库发布在 PyPI</a>。</p> 
<h3>1. API 中文化</h3> 
<p>将木兰中使用到的 rply 的 API 进行了中文化，演示效果如下：</p> 
<p><img alt height="604" src="https://oscimg.oschina.net/oscnet/up-a549e2d711d53f005f707e31e26a9fa2a60.png" width="974" referrerpolicy="no-referrer"></p> 
<p>本来直接把英文 API 改为了中文，后发现为了同时支持 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgithub.com%2FMulanRevive%2Fmulan" target="_blank">逆向工程</a> 正常运行，仍需保留原英文 API。</p> 
<h3>2. 清理木兰重复代码</h3> 
<p>之前发现木兰对 rply 的 parser <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgitee.com%2FMulanRevive%2Fmulan-rework%2Fcommit%2Ff6dadf103508880e42ed054152aadee131e20ee7" target="_blank">作了些许修改</a> 以支持某些语法， 现决定 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgithub.com%2Fnobodxbodon%2Frply%2Fcommit%2Fdd0ef0319556d0d6d76b44d562669e708784bd32" target="_blank">并入 rply 定制版</a>，以减少综合代码维护量。</p> 
<h3>3. 修复报错的列号不准问题</h3> 
<p>半年多前提了 rply 一个 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgithub.com%2Falex%2Frply%2Fpull%2F95%2523issuecomment-729513800" target="_blank">报错列号不准问题</a>，一直没看到回应，研究相关代码部分时顺便作了中文化，加了个 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgithub.com%2Fnobodxbodon%2Frply%2Fcommit%2F4afbd7f12f200790d0466c23f1e0660b770e5ef6" target="_blank">补丁</a></p> 
<p>其他定制内容见 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flink.zhihu.com%2F%3Ftarget%3Dhttps%253A%2F%2Fgithub.com%2Fnobodxbodon%2Frply" target="_blank">项目源码库</a>。</p>
