                                                        <p>jQuery UI 1.13.0-rc.3 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fblog.jqueryui.com%2F2021%2F09%2Fjquery-ui-1-13-0-rc-3-released%2F" target="_blank">已发布</a>，与<a href="https://www.oschina.net/news/158726/jquery-ui-1-13-0-rc-2-released" target="_blank">几周前发布的 rc2</a> 相比，此版本的唯一变化是针对主题的修复。</p> 
<p>由于 CSS 的特殊性问题，1.13.0-rc.2 中的空白图标实际上并不是空白的；rc3 增加了一个相关选择器的权重 (specificity) 来解决这个问题。</p> 
<p>另一组问题也是由对主题的<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fjquery%2Fjquery-ui%2Fcommit%2Fdde9b83df61d1d676e66cb2a2f7970dd44a05137" target="_blank">改动</a>而引起的，这些改动本意是修复其他问题以及改进各种主题的外观。不幸的是，因为这些改动并不完整，因此导致了更大的问题。鉴于 jQuery UI 的维护状态，开发团队决定回滚这些改动。虽然这样操作把 jQuery UI 1.12 中的一些问题带了回来，但这些问题已经存在多年，因此现有项目也需要进行调整。毕竟旧的和已知的错误比新的和未知的错误更容易处理，尤其是在维护项目的资源有限的情况下。此外，开发者也可以更轻松地从 jQuery UI 1.12 升级到 1.13。</p>
