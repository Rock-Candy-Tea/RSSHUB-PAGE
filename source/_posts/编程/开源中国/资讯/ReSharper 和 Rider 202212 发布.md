                                                                                            <p><span style="background-color:#ffffff; color:#27282c">ReSharper 和 Rider 2022.1.2 现已发布，这是一个 bug-fix 更新。具体更新内容如下：</span></p> 
<p><strong><span style="background-color:#ffffff; color:#27282c">ReSharper</span></strong></p> 
<ul> 
 <li>修复了 Visual Studio 的 Show smart tag action 的<code>Ctrl+.</code>热键，该热键曾受到 ReSharper 先前版本的影响 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FRSRP-488222%2FHotkey-Ctrl-not-working-in-EAP6" target="_blank">RSRP-488222</a>)。</li> 
 <li>在 Visual Studio 2022 中更新了 ReSharper 的初始事件缓冲区大小，这有助于消除取消构建过程时的一些小挂起 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FRSRP-488334%2FPerformance-Repeated-mini-hangs-during-cancel-build" target="_blank">RSRP-488334</a> )。</li> 
</ul> 
<p style="margin-left:0; margin-right:0; text-align:start"><span><span><span><span><span style="color:#27282c"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>有关已修复问题的完整列表，可参阅 </span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissues%2FRSRP%3Fq%3DAvailable%2520in%3A%25202022.1.2%2520" target="_blank">issue tracker</a><span><span><span><span><span style="color:#27282c"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p><strong><span style="background-color:#ffffff; color:#27282c">Rider</span></strong></p> 
<ul> 
 <li>修复了 SQL Server 的 Windows 认证 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FDBE-15456" target="_blank">DBE-15456</a> )。</li> 
 <li>修复了 Unity 插件的元数据。插件的 DLL 不再返回错误 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FRIDER-76619" target="_blank">RIDER-76619</a> )。</li> 
</ul>
