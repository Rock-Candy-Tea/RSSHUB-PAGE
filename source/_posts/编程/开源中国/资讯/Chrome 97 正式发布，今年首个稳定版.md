                                                        <p>Chrome 97 已于今天正式发布，这个版本带来了更好的删除存储网站数据的工具、更一致的 Web 应用，以及移动版浏览器更精细的缩放控制，当然也带来了具有争议的键盘映射 API。</p> 
<h3>删除一个网站存储的所有数据</h3> 
<p><img alt height="445" src="https://static.oschina.net/uploads/space/2022/0105/073729_eNxg_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<p>Chrome 97 对隐私和安全设置做了一些改变。你现在可以删除一个网站存储的所有数据。此前，你只能选择删除所有网站数据或删除个别的 cookies。这个新的设置可以在「设置」>「安全和隐私」>「网站设置」>「查看权限和跨网站存储的数据」中找到。</p> 
<h3>Web 应用更像本地应用</h3> 
<p><img alt height="274" src="https://static.oschina.net/uploads/space/2022/0105/073744_nZaq_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<p>之前（顶部）/之后（底部）</p> 
<p>此前让 Web 应用程序看起来<strong>不像</strong>本地应用的一个区域是顶部的应用栏。Chrome 97 允许 Web 应用程序利用这一空间，用于放置搜索栏、导航按钮和颜色等元素。</p> 
<h3>记住每个网站在移动端的缩放设置</h3> 
<p><img alt height="324" src="https://static.oschina.net/uploads/space/2022/0105/073757_nFqZ_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<p>桌面端的 Chrome 浏览器已经能够记住你对特定网站的缩放设置，Chrome 会在每次访问该网站时保持这一设置。如今在 Chrome 97 中加入了一个功能标志，可以在移动端做到这一点。</p> 
<p>该功能可以在 <code>chrome:flags#enable-accessibility-page-zoom</code> 中找到。一旦启用，你可以通过点击地址栏中的锁定图标来调整缩放。下次访问这个网站时，Chrome 会记住这个设置。</p> 
<h3>为 CSS 提供更好的 HDR 支持</h3> 
<p>Google 在 Chrome 94 开始测试一项 CSS 功能，以检测设备屏幕是否支持 HDR 内容。该功能现在已经在 Chrome 97 中上线。这使网页开发人员能够基于用户的显示设备来决定是否启用 HDR 内容，而此前只能全开或全不开，从而影响用户体验。</p> 
<h3>其他功能</h3> 
<ul> 
 <li>Keyboard Map API：Chrome 97 有一个新的 Keyboard Map API，有助于识别不同键盘布局下的按键（<a href="https://www.oschina.net/news/177029/chrome-keyboard-api">可查看综合新闻解释</a>）。</li> 
</ul>
