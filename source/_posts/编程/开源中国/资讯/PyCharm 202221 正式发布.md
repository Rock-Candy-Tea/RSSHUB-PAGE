                                                                                            <p>JetBrains 日前发布了 PyCharm 2022.2 系列的第一个小幅更新版本，该版本主要修复了之前存在的各种错误，以下是这次更新中一些重要的内容：</p> 
<ul> 
 <li>启用了通过 Python 解释器（Python Interpreter）弹出窗口中的显示所有（Show all）弹出菜单来设置解释器的新 UI</li> 
 <li>Docker：如果在 Docker 解释器配置期间设置了其他运行选项，那么通过运行/调试配置设置的运行选项不再被忽略</li> 
 <li>Docker：在 Linux 上使用 Docker 解释器时，控制台和调试器现在可以连接了</li> 
 <li>Docker Compose：用以前的 PyCharm 版本配置的项目解释器现在可以按预期启动</li> 
 <li>Docker Compose：端口配置现在可用于 Docker Compose 解释器</li> 
 <li>Docker Compose：使用 Docker Compose 解释器运行 Django 不再导致 HTTP 错误</li> 
 <li>PyCharm 现在能够识别非默认的 Flask 应用程序名称，并相应地设置 Flask 运行配置</li> 
 <li>Testing：Behave 运行配置可以运行单个场景，而不是运行目标目录中的所有可用文件</li> 
 <li>输出控制台中的 <em>Emulate terminal</em> 选项不再导致控制台或调试器中出现意外缩进</li> 
 <li>Debugger：Python 控制台现在可以正确显示包含 ANSI 颜色序列的文本</li> 
 <li>Debugger：PyCharm 在调试包含非 ASCII 编码的代码时不再产生错误</li> 
 <li>Debugger：调试多进程脚本不再导致 IDE 异常错误</li> 
</ul>
