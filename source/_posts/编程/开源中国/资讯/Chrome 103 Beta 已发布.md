                                                                                            <p><a href="https://www.oschina.net/news/197204/chrome-102-released">Chrome 102 发布</a>之后，Google 已将 Chrome 103 升级为测试版。</p> 
<p><span style="background-color:#ffffff; color:#121212">Chrome 103 beta 引入了 </span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.chromestatus.com%2Ffeature%2F5207422375297024" target="_blank">103 Early Hints HTTP 响应代码</a>进行导航的<span style="background-color:#ffffff; color:#121212">支持，</span>当 103 响应包含<code><link rel=preload></code>或其他链接标头时，Chrome 会在收到最终响应之前尝试预加载（和/或预连接、预取）指定资源。这为 Web 开发人员提供了一种优化<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fweb.dev%2Fvitals%2F" target="_blank">核心 Web</a> Vitals 的方法，例如最大内容绘制 (LCP)。</p> 
<p>此外，Chrome 103 的本地字体访问功能允许枚举本地字体和所述字体的元数据，以便在 Web 应用程序中用于自定义的文本堆栈。</p> 
<p>Chrome 103 Beta 还添加了“deflate-raw”压缩格式支持，允许 Web 开发人员在没有任何页眉/页脚的情况下访问原始 deflate 流。这种 deflate-raw 支持可用于 Web 应用程序读取和写入 zip 文件。</p> 
<p>有关 Chrome 103 Beta 的更多信息，可在 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fblog.chromium.org%2F2022%2F05%2Fchrome-103-beta-early-navigation-hints.html" target="_blank">Chromium 博客</a>中进一步阅读。</p>
