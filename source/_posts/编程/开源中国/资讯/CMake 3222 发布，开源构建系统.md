                                                                                            <p style="color:#000000; margin-left:0; margin-right:0; text-align:start">CMake 是一个跨平台的自动化构建系统，它使用一个名为 CMakeLists.txt 的文件来描述构建过程，可以产生标准的构建文件，如 Unix 的 Makefile 或 Windows Visual C++ 的 projects/workspaces 。文件 CMakeLists.txt 需要手工编写，也可以通过编写脚本进行半自动的生成。</p> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start">CMake 3.22.2 发布，更新内容如下：</p> 
<ul style="margin-left:0; margin-right:0"> 
 <li>增加对 Python 3.11 的支持</li> 
 <li>VS：从 v143 链接标志表中删除<span> </span><code>/guard:cf</code><span> </span>标志</li> 
 <li>FortranCInterface：修复与 GCC gfortran 12 LTO 的兼容性</li> 
 <li>增加对 Boost 1.78 的支持</li> 
 <li>ci：在 Windows 上明确地禁用 Java 测试</li> 
 <li>FindGLUT：在 pkg-config 代码路径中提供传统的 GLUT_INCLUDE_DIR 结果</li> 
 <li>FindBoost：不对现在支持的 1.78 版本发出警告</li> 
 <li>ci：在 VS 构建中明确禁用 Java 测试</li> 
 <li>帮助：在发布说明中添加缺失的 3.22.1 部分</li> 
 <li>gitlab-ci：更新 macOS 作业以使用 Xcode 13.2</li> 
 <li>ASM：修复对 armasm 6.17 以上版本的识别</li> 
 <li>XL：检测传统编译器的默认扩展模式</li> 
 <li>CMP0128：避免测试代码的重复</li> 
 <li>Android：为 pre-21 system STL 修复 android_support 的链接</li> 
 <li>……</li> 
</ul>
