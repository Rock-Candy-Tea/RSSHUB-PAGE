<div class="content">
                                                                                            <p><span style="background-color:#ffffff; color:#333333">Git for Windows 2.34.0-rc0 现已发布，主要更新内容如下：</span></p> 
<p><strong>New Features</strong></p> 
<ul> 
 <li>随附 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit%2Fgit%2Fblob%2Fv2.34.0-rc0%2FDocumentation%2FRelNotes%2F2.34.0.txt" target="_blank">Git v2.34.0-rc0</a></li> 
 <li>引用相对于 Git 安装位置的路径的配置设置<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit-for-windows%2Fgit%2Fpull%2F3472" target="_blank">现在必须通过 <code>%(prefix)/</code>来标记，而不是现在已弃用的前导斜杠</a></li> 
 <li>随附 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit-lfs%2Fgit-lfs%2Freleases%2Ftag%2Fv3.0.2" target="_blank">Git LFS v3.0.2</a></li> 
 <li>包含<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit-for-windows%2Fgit%2Fpull%2F3492" target="_blank">对<code>core.fsyncObjectFiles=batch</code>的新的、实验性的支持</a></li> 
</ul> 
<p><strong>Bug Fixes</strong></p> 
<ul> 
 <li>将 system-wide VS Code 配置为 Git 的编辑器<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fgit-for-windows%2Fgit%2Fissues%2F3471" target="_blank">被损坏</a>，现已修复。</li> 
 <li>现在可以克隆大于 4GB 的文件了，只要它们是通过 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgit-lfs.github.io%2F" target="_blank">Git LFS</a> 传输的。</li> 
</ul>
