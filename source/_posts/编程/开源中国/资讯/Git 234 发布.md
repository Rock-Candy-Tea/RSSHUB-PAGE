<div class="content">
                                                                                            <p>Git 2.34 现已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.blog%2F2021-11-15-highlights-from-git-2-34%2F" target="_blank">发布</a>，本次更新<span style="background-color:#ffffff; color:#2f363d">包含来自超过 109 个贡献者的功能和错误修复，其中 29 个是新贡献者。</span></p> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left"><strong style="color:#333333">主要更新内容</strong></p> 
<ul> 
 <li style="text-align:left">增加了一个支持稀疏索引的功能，以帮助处理类似"monorepo"设置的非常大的Git存储库。该索引格式能够理解标记的目录，在稀疏检出时表明内容之间的关系。</li> 
</ul> 
<p><img height="186" src="https://oscimg.oschina.net/oscnet/up-c80e4b90c815826bdbb6dda2f529bf1da20.png" width="500" referrerpolicy="no-referrer"> </p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#121212">添加了多包可达性位图。</span>Git 2.34 引入了一种新的位图格式，该格式与<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgit-scm.com%2Fdocs%2Fmulti-pack-index" target="_blank">多包索引</a>文件的内容相关联。这意味着位图现在可以灵活地表示多个包中的对象，并且服务器运营商不再需要将他们最大的存储库重新打包到一个包中以充分利用可达位图的优势。</li> 
 <li><span style="background-color:#ffffff; color:#121212">默认使用“ort”合并策略。</span></li> 
 <li><span style="background-color:#ffffff; color:#121212">支持使用 OpenSSH 8.0+ SSH 密钥作为 Git tags 的签名密钥</span></li> 
 <li>在 Git 2.34 中，现在可以配置 Git 以交互方式询问你是否要使用建议的命令重新运行上次操作，方法是将<code>help.autoCorrect</code>设置为<code>prompt</code>。</li> 
 <li>改善了 git fetch 和 git push 的性能。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgit-scm.com%2Fbook%2Fen%2Fv2%2FGit-Tools-Submodules" target="_blank">子模块</a>功能改进。</li> 
 <li><span style="background-color:#ffffff; color:#121212">以及其他各种性能改进...</span></li> 
</ul> 
<p><span style="background-color:#ffffff; color:#333333">更多详情请可查看</span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.blog%2F2021-11-15-highlights-from-git-2-34%2F" target="_blank">更新公告</a><span style="background-color:#ffffff; color:#333333">。</span></p>
