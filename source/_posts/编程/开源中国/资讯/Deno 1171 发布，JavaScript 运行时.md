                                                                                            <p>Deno 是一个简单、现代和安全的 JavaScript 和 TypeScript 的运行时，它使用 V8 并以 Rust 构建。</p> 
<p>Deno 1.17.1 已发布，包括以下变更：</p> 
<ul> 
 <li>功能：为调试测试添加 Code lens (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13138" target="_blank">#13138</a>)</li> 
 <li>功能：在获取注册表配置时提供接受 header 信息(<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13159" target="_blank">#13159</a>)</li> 
 <li>修复了在使用 <code>--inspect</code> 或 <code>--inspect-brk</code> 标志运行时打印到终端的提示 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13123" target="_blank">#13123</a>)</li> 
 <li>修复：按字符索引拆分源代码 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13114" target="_blank">#13114</a>)</li> 
 <li>修复：读取字符串时使用 <code>c_char</code> 而不是 <code>i8</code> (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13118" target="_blank">#13118</a>)</li> 
 <li>修复：WebSocketStream 关闭时不会出现 "sending after closing" 的错误 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13134" target="_blank">#13134</a>)</li> 
 <li>修复：支持导入和导出声明的断言 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F13121" target="_blank">#13121</a>)</li> 
</ul>
