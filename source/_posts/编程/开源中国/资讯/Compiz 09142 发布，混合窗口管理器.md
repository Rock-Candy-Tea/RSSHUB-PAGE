                                                                                            <p>时隔两年多，Compiz 发布了一个小的点版本更新：<strong><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Flists.freedesktop.org%2Farchives%2Fcompiz%2F2022-August%2F003539.html" target="_blank">0.9.14.2</a></strong>，修复了多个错误，以及优化功能。</p> 
<ul> 
 <li>添加对 _GTK_WORKAREAS_Dn 和 _GNOME_WM_STRUT_AREA 的支持</li> 
 <li>修复使用新版本 GCC 时出现的构建错误</li> 
 <li>修复了 OpenGL ES 上 blur 和 opengl 插件中的部分错误</li> 
</ul> 
<p>Compiz 0.9.14.2 还修复了关于 pkg-config 搜索路径的错误，修复了 Compiz CCSM 设置管理器在深色主题中的外观，支持 CMake Unity 构建，以及在决定是否应该出现焦点时使用正确的窗口，还有多项翻译更新。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-39ca5a1fdad9e9c018c96935f97a6af6c9d.png" referrerpolicy="no-referrer"></p> 
<p>Compiz 是第一个由 OpenGL 驱动的运行于 X Window System 上的混合窗口管理器，对 Linux 桌面产生了巨大的影响。Compiz 的混合渲染能力使其可以在窗口管理过程中实现多种视觉效果，比如在矩形虚拟桌面上的窗口最小化。</p> 
<p>Compiz 符合 ICCCM 标准，可以取代 GNOME 环境的 Metacity 视窗管理器，或是 KDE Plasma 环境下的 KWin。Compiz 拥有类似苹果电脑的 Mac OS X 操作系统中所提供的许多桌面特效，例如 Exposé。 Compiz 使用 OpenGL 绘图层 Glitz 架构在 Xgl 这个新的 X 环境上。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-ca9d415f084a536c20f3479a34e19dc72fc.png" referrerpolicy="no-referrer"></p> 
<p>Compiz 最早是由 Novell 在2006年1月时公布新版 Xgl 时所发表的。在开发初期 Compiz 只能正常运行在有限的几种 Xgl 所支援的硬件 3D 加速卡上。尽管 Xgl 和 Compiz 这两个全新构造的系统还存在一些 bug 并且仍处于积极的开发阶段，大多数 NVIDIA 和 ATI 图形处理卡可以在 Xgl 上正常运行 Compiz。Compiz 在其他一些 X 服务器和硬件上也可以正常运行。据报道，Compiz 在现时仍处于开发阶段的 AIGLX 上运行良好。</p> 
<p>2007年7月30日，Compiz 已经与其分支套件 Beryl 合并，并且更名为 Compiz Fusion。2009年2月2日，Compiz 和 Compiz Fusion 合并，新名字叫 Compiz。</p>
