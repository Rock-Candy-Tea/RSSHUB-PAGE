                                                                                            <p>经过长时间的停更，jQuery UI 1.13.0-rc.2 终于<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fblog.jqueryui.com%2F2021%2F09%2Fjquery-ui-1-13-0-rc-2-released%2F" target="_blank">发布</a>了。发布公告写道，此版本功能完整，在 RC 阶段主要是收集开发者遇到的严重问题，如果没有使用者反馈问题，并且在内部测试期间也没有发现任何此类问题，预计会在几周之内发布 1.13.0 正式版。</p> 
<p>据介绍，jQuery UI 1.13.0 的主要更新重点是提升与最新 jQuery 版本的兼容性，因此开发团队将大多数破坏性变更（例如删除已弃用的 API 和删除旧版浏览器支持）推迟到了未来版本。</p> 
<p>其他更新内容包括：</p> 
<ul> 
 <li>删除被废弃的 jQuery API 的使用。<span style="color:#333333"><span style="background-color:#ffffff">当使用 jQuery Migrate 3.3.2 对 jQuery 3.6.0 运行其测试套件时，jQuery UI 1.13 不会触发 jQuery Migrate 警告，即此版本发布时的最新版本。</span></span></li> 
 <li><span style="color:#333333"><span style="background-color:#ffffff">放弃对 jQuery 1.7 的支持；不过仍然支持 jQuery 1.8 和更高版本。</span></span></li> 
 <li><span style="color:#333333"><span style="background-color:#ffffff">在此版本中，下载生成器生成的所有单独模块文件以及捆绑的 jQuery UI 副本的所有代码都在严格模式下运行。这对大多数用户来说并不重要，因为自 2016 年发布 3.0 以来，jQuery 一直在严格模式下运行。</span></span></li> 
</ul> 
<p style="text-align:start"><span style="color:#333333"><span style="background-color:#ffffff">除此之外，还添加了两个小功能：</span></span></p> 
<ul> 
 <li>Accordion的 header 选项现在不仅可以接受匹配 header 元素的选择器，还可以接受一个以 accordion 元素为参数并返回 header 元素的函数；<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fapi.jqueryui.com%2Faccordion%2F%23option-header" target="_blank">详情查看文档</a>。</li> 
 <li>日期选择器选项现在包括可选的 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fapi.jqueryui.com%2Fdatepicker%2F%23option-onUpdateDatepicker" target="_blank">onUpdateDatepicker 回调</a>，当日期选择器小部件的 DOM 更新时会调用。</li> 
</ul> 
<p>开发团队还提到，为了简化 jQuery UI 的维护，他们正在淘汰 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fbugs.jqueryui.com%2F" target="_blank">https://bugs.jqueryui.com</a> 上的旧错误跟踪器（我们将其保持为只读模式），未来将通过 GitHub issue 处理和跟踪 bug。</p> 
<p>在过去的几年里，jQuery UI 一直在努力寻找贡献者；他们的目标是将其更多地转移到维护状态：确保该库与新的 jQuery 版本兼容，并且安全问题得到修复，但没有计划开发新的重要功能。开发团队还将尝试修复 jQuery UI 1.12.1 的重要回归错误；不过较旧的长期错误可能无法修复。</p>
