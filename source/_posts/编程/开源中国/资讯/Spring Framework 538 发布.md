                                                        <p>Spring Framework 5.3.8 现已发布，该版本包括 35 个 bug 修复和优化。</p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li>HttpComponentsClientHttpConnector 应该关闭底层资源</li> 
 <li>StandaloneMockMvcBuilder.useSuffixPatternMatch 的默认值与 RequestMappingHandlerMapping 中的相同属性不同</li> 
 <li>查找方法自动装配忽略方法的通用信息</li> 
 <li>使用 WebMvc.fn 对请求设置 BEST_MATCHING_PATTERN_ATTRIBUTE</li> 
 <li>删除 jackson-module-kotlin 警告</li> 
 <li>生成 WebSession id 后切换回并行线程</li> 
 <li>引入 ResponseEntity.internalServerError()</li> 
 <li>弃用/删除 ScriptUtils 实现中的内部 API</li> 
 <li>当 SockJS 顶部 URL 是 WebSocket 升级时，Tomcat 中出现异常</li> 
 <li>改进对 CorsConfiguration 的 allowedOriginPattern 中端口号的支持</li> 
 <li>添加用于指定时间戳的 ApplicationEvent 构造函数</li> 
 <li>将 awaitExchangeOrNull 扩展功能添加到响应式 Web 客户端</li> 
 <li>自 5.3.3 起，WebClient 中的 MultipartHttpMessageWriter 不再使用自定义 Jackson 编码器</li> 
 <li>修复构建 MultiPart 时 PartFile 名称丢失的问题</li> 
 <li>不替换已弃用的 CronSequenceGenerator.isValidExpression</li> 
 <li>CronExpression 在 spring-context-5.3.6 上被破坏</li> 
</ul>
