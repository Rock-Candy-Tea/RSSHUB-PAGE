                                                                                            <p style="margin-left:0px">Metabase 是一个开源的团队数据分析工具，提供简单快捷的商业智能和分析功能，无论是配置还是使用都非常轻松，让团队的每位成员都能轻松看懂数据细节并参与其中。</p> 
<p style="margin-left:0px">目前 Metabase 更新到 0.44.2 版本，带来如下改动：</p> 
<p><strong>0.44.2 中的增强功能</strong></p> 
<ul> 
 <li>始终在当前集合中创建新项目 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fpull%2F24986" target="_blank">#24986</a> )</li> 
 <li>在编辑模型时，编辑元数据始终显示原始字段名称 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24824" target="_blank">#24824</a> )</li> 
</ul> 
<p><strong>0.44.2 中的错误修复</strong></p> 
<ul> 
 <li>如果沙盒规则使用本机查询，高级字段值不会更改 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24966" target="_blank">#24966</a> )</li> 
 <li>无法在自定义列值的情况下使用细分 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24922" target="_blank">#24922</a> )</li> 
 <li>如果卡片未接触左墙，则仪表板中的单击和拖动过滤会产生偏移（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24912" target="_blank">#24912</a>）</li> 
 <li>单击行为删除 FullApp 覆盖 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24756" target="_blank">#24756</a> )</li> 
 <li>如果您选择要排除的值，则仪表板上的“排除”过滤器会中断查询，然后将其添加回来 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24236" target="_blank">#24236</a> )</li> 
 <li>当没有排除任何内容时，仪表板上的“排除”过滤器不会在过滤器小部件中显示任何值 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F24235" target="_blank">#24235</a> )</li> 
 <li>表单选项卡在悬停时与白色背景融为一体（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F23975" target="_blank">#23975</a>）</li> 
 <li>对具有多个维度的链接表列进行沙箱处理并重新映射，然后“脏”查询失败（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmetabase%2Fmetabase%2Fissues%2F15106" target="_blank">#15106</a>）</li> 
</ul>
