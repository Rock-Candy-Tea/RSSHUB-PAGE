<div class="content">
                                                                                            <p>Go 1.19 现已发布，可以通过访问<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fdl%2F" target="_blank">下载页面</a>获得该版本。"它的大部分变化是在工具链、运行时和库的实现方面。一如既往，该版本保持了 Go 1 的兼容性承诺。我们期望几乎所有的 Go 程序都能像以前一样继续编译和运行"。该版本包括一些内存模型的调整，LoongArch port，文档评论机制的改进等。</p> 
<p>具体更新内容如下：</p> 
<ul> 
 <li style="text-align:start"><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Go 1.19 完善并改进了 Go 1.18 。开发团队将 Go 1.19 的泛型开发集中在解决社区报告的一些细微问题和极端情况，以及重要的性能改进（某些泛型程序性能改进高达 20%）。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></li> 
 <li style="text-align:start"><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>文档注释现在支持<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fdoc%2Fcomment" target="_blank">链接、列表和更清晰的标题语法</a>。此更改有助于用户编写更清晰、更容易浏览的文档注释，尤其是在具有大型 API 的包中。作为此更改的一部分，<code>gofmt</code>现在重新格式化文档注释以将标准格式应用于这些功能的使用。有关所有详细信息，可参阅“ <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fdoc%2Fcomment" target="_blank">Go Doc Comments</a>”。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></li> 
 <li style="text-align:start"><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fref%2Fmem" target="_blank">Go 的内存模型</a>现在明确定义了 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fpkg%2Fsync%2Fatomic%2F" target="_blank">sync/atomic 包</a>的行为。happens-before 关系的正式定义已被修改，以与 C、C++、Java、JavaScript、Rust 和 Swift 使用的内存模型保持一致。现有程序不受影响。随着内存模型的更新， <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fdoc%2Fgo1.19%23atomic_types" target="_blank">sync/atomic 包中还有新的类型</a>，例如 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fpkg%2Fsync%2Fatomic%2F%23Int64" target="_blank">atomic.Int64</a> 和 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fpkg%2Fsync%2Fatomic%2F%23Pointer" target="_blank">atomic.Pointer[T]</a>，以便更容易使用 atomic values。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></li> 
 <li style="text-align:start"><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>出于<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fblog%2Fpath-security" target="_blank">安全原因</a>， os/exec 包不再遵循 PATH 查找中的相对路径。有关详细信息，可参阅 </span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fpkg%2Fos%2Fexec%2F%23hdr-Executables_in_the_current_directory" target="_blank">package documentation</a><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fpkg.go.dev%2Fgolang.org%2Fx%2Fsys%2Fexecabs" target="_blank">golang.org/x/sys/execabs</a> 的现有用途可以在仅使用 Go 1.19 或更高版本构建的程序中移回 os/exec。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></li> 
 <li style="text-align:start"><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>垃圾收集器增加了对软内存限制的支持，在<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgo.dev%2Fdoc%2Fgc-guide%23Memory_limit" target="_blank">新的垃圾收集指南</a>中有详细讨论。该限制对于优化 Go 程序以在具有专用内存量的容器中尽可能高效地运行特别有用。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></li> 
 <li style="text-align:start"><span><span><span style="color:#202224"><span><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>当目标操作系统（GOOS）是任何类 Unix 系统时，新的构建约束 unix 就得到满足。如今，类 Unix 是指所有 Go 的目标操作系统，除了 js、plan9、windows 和 zos。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></li> 
 <li style="text-align:start"><span style="color:#000000">此外，Go 1.19 还包括了各种性能和实现改进，包括动态调整初始 goroutine 堆栈以减少堆栈复制、在大多数 Unix 系统上自动使用附加文件描述符、在 x86-64 和 ARM64 上用于大型 switch 语句的跳转表，支持 ARM64 上的调试器注入函数调用，RISC-V 上的 register ABI 支持，以及在龙芯 64 位架构 LoongArch ( <code>GOARCH=loong64</code>) 上运行的 Linux 的实验性支持。</span></li> 
</ul>
