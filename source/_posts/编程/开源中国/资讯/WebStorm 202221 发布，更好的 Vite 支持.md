                                                                                            <p>WebStorm 2022.2.1 是 WebStorm 2022.2 的第一个错误修复更新，它包含了许多改进，包括对 Vite 别名的支持和对 Vue 3 和 Angular 14 的修复。</p> 
<h3>更好的 Vite 支持</h3> 
<p>WebStorm 现在支持 Vite 别名，并有一个新的部分用于调整你希望 Vite 配置文件的检测方式。你可以在「设置 —— 语言和框架 —— JavaScript —— Vite」中找到这个部分。就像 webpack 一样，你可以选择自动和手动选项，或者选择不检测 Vite 配置文件。</p> 
<p><img alt height="350" src="https://oscimg.oschina.net/oscnet/up-b9ab29ad5546cba0ca05b1051e9b91301cd.png" width="700" referrerpolicy="no-referrer"></p> 
<h3>其他值得注意的改进</h3> 
<ul> 
 <li>WebStorm 将在使用 Vue 和 TypeScript 的项目中手动添加的导入语句中包含 .vue 扩展名</li> 
 <li>Angular Schematics 和 Extract Component 重构现在应该可以在 Angular 14 中按预期工作</li> 
 <li>修正了 Terminal 标签名称的回归错误</li> 
 <li>修正了导致包含非 ASCII 字符的 Mermaid 图表的不正确预览渲染的问题</li> 
 <li>修正了导致 Pinia 的 <code>storeToRefs</code> 方法的类型推断不正确的问题</li> 
</ul>
