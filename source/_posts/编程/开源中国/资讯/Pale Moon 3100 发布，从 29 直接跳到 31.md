                                                        <p><span style="color:#000000"><span style="background-color:#ffffff">Pale Moon 31.0.0 现已发布，</span><span style="background-color:#ffffff">这是一个新的里程碑版本。官方公告指出，</span>在无奈的召回 v30.0.0 和 30.0.1 版本之后，该开发团队的一名核心开发人员离开了；从而导致他们不得不倒带并重做几个月的工作，以排除不需要的代码更改以及可能是召回版本的大量稳定性和运行时间问题的根源。</span></p> 
<p><span style="color:#000000">“我们重回正轨，在 UXP 和 Goanna (v5.1) 上构建了一个新的里程碑，其中包含许多改进和用户要求的附加功能。为了防止用户混淆，我们从 29 跳到 31。”</span></p> 
<p>此版本的重要变化如下：</p> 
<ul> 
 <li>我们再次接受在我们自己的 Pale Moon 独家扩展的同时安装旧版 Firefox 扩展。</li> 
 <li>实现了“optional chaining”。</li> 
 <li>为文本选择实现<code>setBaseAndExtent</code>。</li> 
 <li>实现了<code>queueMicroTask()</code>“pseudo-promise”回调。</li> 
 <li>在 Intersection observers 中实现了接受无单位的 rootMargin 值，以实现 Web 兼容性，使其更像人们所期望的 CSS margin。</li> 
 <li>在规范更改后改进了 CSS 网格和 flexbox 渲染和显示，并提高了 Web 兼容性。</li> 
 <li>改进了 JavaScript 中并行 Web 工作者的性能。</li> 
 <li>改进了<span>草书</span>字体的显示（在 Windows 上）。</li> 
 <li>更新了各种 in-tree libraries。</li> 
 <li>在通过 MSE (RFC-6381) 的媒体交付中添加了对扩展 VPx 编解码器字符串的支持。</li> 
 <li>修复了一个长期回归，即当直接在 HTML tags 而不是 CSS 中表示时，浏览器将不再支持旧样式的正文和 iframe 正文边距。这提高了与特别旧和/或存档网站的兼容性。</li> 
 <li>修复了几个崩溃和稳定性问题。</li> 
 <li>在 Windows 安装程序中添加了一个许可屏幕，以阐明浏览器的许可。在其他安装中，你可能会在浏览器安装位置中添加的 license.txt 文件中找到此许可声明。</li> 
 <li>删除了所有 Google SafeBrowsing/URLClassifier 服务代码。</li> 
 <li>恢复了平台中的 Mac OS X 代码和可构建性。</li> 
 <li>删除了非标准的<code>ArchiveReader</code>DOM API，曾经只是原型实现。</li> 
 <li>从平台上删除了大部分侵入性 Mozilla 遥测代码的最后痕迹。这可能会提高某些系统的性能。</li> 
 <li>删除了剩余的 Electrolysis 控件。</li> 
 <li>删除了更多 Android/Fennec 代码（正在努力清理我们的代码）。</li> 
 <li>删除了 Marionette 自动化测试框架。</li> 
 <li>解决的安全问题：CVE-2022-29915、CVE-2022-29911 和几个没有 CVE 编号的问题。</li> 
 <li><span style="color:#333333">Unified XUL 平台 Mozilla 安全</span>补丁摘要：4 fixed，1 DiD，19 not applicable。</li> 
</ul>
