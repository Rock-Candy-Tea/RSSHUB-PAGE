<div class="content">
                                                                                            <p style="margin-left:0; margin-right:0; text-align:start"><span style="color:#000000"><span style="background-color:#ffffff">ReactOS 0.4.14 现已发布。此版本的改进范围包括有 FreeLoader 修复、Shell 功能、内核修复、NetKVM VirtIO bringup、Xbox 端口的进一步工作以及对 NEC PC-9800 的支持。公告指出，开发团队花了一年多的时间来完善此版本并修复回归问题。</span></span></p> 
<p style="margin-left:0; margin-right:0; text-align:start"><img alt height="282" src="https://oscimg.oschina.net/oscnet/up-4ee8f8b7baff74618b58d094bbbd7d1214c.png" width="500" referrerpolicy="no-referrer"></p> 
<p style="margin-left:0; margin-right:0; text-align:start">此版本的一些更新<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.phoronix.com%2Fscan.php%3Fpage%3Dnews_item%26px%3DReactOS-0.4.14-Released" target="_blank">亮点</a>如下：</p> 
<ul> 
 <li><span style="color:#000000"><span style="background-color:#ffffff">此版本的主要亮点之一是对 Shell 组件所做的大量改进，它构成了 ReactOS 用户体验的重要组成部分。</span><span style="background-color:#ffffff">各种 user-interface/shell 改进，包括 context menus 中的“Send To”支持、“Open Command Prompt”支持和其他附加功能。</span></span></li> 
</ul> 
<p style="margin-left:0px; margin-right:0px; text-align:start"><span style="background-color:#ffffff; color:#121212"><img alt height="281" src="https://oscimg.oschina.net/oscnet/up-3f7b5f7adedb29e91f79acd1db897b7f353.png" width="500" referrerpolicy="no-referrer"></span></p> 
<ul> 
 <li>致力于能够在 NEC PC-9800 (PC-98) 计算机上启动 ReactOS。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fen.wikipedia.org%2Fwiki%2FPC-9800_series" target="_blank">NEC PC-9800</a> <span style="color:#000000">是 NEC 开发制造的 16 位和 32 位日本计算机系列。由于这种类型的硬件基于 x86 处理器，所以它是一个相对容易移植的目标。</span></li> 
</ul> 
<p style="margin-left:0px; margin-right:0px; text-align:start"><img alt height="281" src="https://oscimg.oschina.net/oscnet/up-0136d11a224cb158c064ea1501726ff0517.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li>更好地支持 Internet 控制消息协议 (ICMP)。ICMP 协议的实现了很多改进，包括从 IOCTL_ICMP_ECHO_REQUEST I/O 控制代码的实现到 Icmp** 例程的完全重写。</li> 
</ul> 
<p style="margin-left:0px; margin-right:0px; text-align:start"><img alt height="281" src="https://oscimg.oschina.net/oscnet/up-3aae2b1edfee3356d1bea8f31a6e56f0122.png" width="500" referrerpolicy="no-referrer"></p> 
<ul> 
 <li>ReactOS 内核具有更好的稳定性和其他各种改进。内核已经在改进内存管理、物理地址扩展 (PAE) 处理、CRT 异常处理、更好的即插即用和驱动程序工作方面进行了初步工作。</li> 
 <li>支持用于虚拟机联网的 NetKVM VirtIO 接口。</li> 
 <li>从系统中删除过时的字体以减少新 ReactOS 版本的二进制大小<span style="color:#000000">，<span style="background-color:#ffffff">也减少了 RAM 消耗。</span></span></li> 
</ul> 
<p style="margin-left:0px; margin-right:0px; text-align:start"><span style="color:#000000"><span style="background-color:#ffffff"><img alt height="281" src="https://oscimg.oschina.net/oscnet/up-70b228498aa70808a82d5698fd1636231ad.png" width="500" referrerpolicy="no-referrer"></span></span></p> 
<ul> 
 <li>针对各种开源组件进行更新。</li> 
</ul>
