<h2><code>简而轻的低侵入式在线构建、自动部署、日常运维、项目监控软件</code></h2> 
<h2>📚 您为什么需要 <a href="https://gitee.com/dromara/Jpom">Jpom</a></h2> 
<blockquote> 
 <p>项目部署运维时，通用的方法是登录服务器，上传新的项目包，备份现有版本，执行相应命令。如果管理多个项目则重复操作上述步骤。不同项目亦存在不同的命令处理。项目运行状态无法掌握等.</p> 
 <p>在中小团队中或者没有专业运维的公司（开发兼运维工作），暂时不便引入复杂的 CI/CD 工具时使用 Jpom 轻量化、低侵入的过度</p> 
</blockquote> 
<h3>👴 使用Jpom前</h3> 
<ol> 
 <li>登录服务器（专业软件：RDP、SSH等）</li> 
 <li>不同项目有不同管理命令（不易记、易混淆）</li> 
 <li>查看项目运行状态需要再次使用命令</li> 
 <li>服务器密码分发混乱（安全性低）</li> 
 <li>集群项目不断重复以上操作项目步骤</li> 
</ol> 
<h3>🎁 使用Jpom后</h3> 
<ol> 
 <li>使用浏览器登录方便快捷管理项目</li> 
 <li>方便的用户管理 
  <ol> 
   <li>用户操作监控，监控指定用户指定操作以邮件形式通知</li> 
   <li>多用户管理，用户项目权限独立(上传、删除权限可控制),完善的操作日志,使用工作空间隔离权限</li> 
   <li>账号可以开启 MFA 两步验证提高账号安全性</li> 
  </ol> </li> 
 <li>界面形式实时查看项目运行状态、控制台日志、管理项目文件 
  <ol> 
   <li>在线修改项目文本文件</li> 
  </ol> </li> 
 <li>docker 容器管理、docker swarm 集群管理</li> 
 <li>在线 SSH 终端,让您在没有 Xshell、FianlShell 等软件也能轻松管理服务器 
  <ol> 
   <li>运维登录 Jpom 系统后不需要知道服务器密码</li> 
   <li>能指定 ssh 禁止执行命令，避免执行高风险命令, 并且能自动执行命令日志</li> 
   <li>设置文件目录，在线查看管理对应项目文件及配置文件</li> 
   <li>SSH 命令模版在线执行脚本还能定时执行</li> 
   <li>在线修改文本文件</li> 
   <li>轻量的实现了简单的"堡垒机"功能</li> 
  </ol> </li> 
 <li>使用项目分发一键搞定集群项目多机部署</li> 
 <li>在线构建不用手动更新升级项目 
  <ol> 
   <li>支持拉取 GIT、SVN 仓库</li> 
   <li>支持容器构建（docker）</li> 
   <li>支持 ssh 方式发布</li> 
   <li>支持定时构建</li> 
   <li>支持 webhook 形式触发构建</li> 
  </ol> </li> 
 <li>支持在线编辑 nginx 配置文件并自动 reload等操作 
  <ol> 
   <li>管理 nginx 状态,管理 ssl 证书</li> 
  </ol> </li> 
 <li>项目状态监控异常自动报警、自动尝试重启 
  <ol> 
   <li>支持邮件+钉钉群+微信群通知，主动感知项目运行状况</li> 
  </ol> </li> 
 <li>节点脚本模版+定时执行，拓展更多可能</li> 
 <li>重要路径白名单模式，杜绝用户误操作系统文件</li> 
</ol>

<blockquote> 
 <p>使用项目文件备份说明：</p> 
 <ol> 
  <li>默认未开启文件备份功能</li> 
  <li>可以配置全局开启，插件端配置（ <code>extConfig.yml</code> ）文件中配置<code>project.fileBackupCount</code>属性</li> 
  <li>DSL 项目可以在配置内容新增 <code>file.backupCount</code> 来开启（DSL 配置优先级最高）</li> 
  <li>如果配置值小于等于 0 则不开启备份功能</li> 
  <li>备份文件保留规则为，只保留有差异的文件</li> 
  <li>同时支持配置仅备份指定文件后缀的文件（详情查看配置文件说明）</li> 
 </ol> 
</blockquote> 
<h3>注意：⚠️</h3> 
<p>本版本调整了上传文件大小配置参数位置，在插件端升级后可能出现上传文件失败。</p> 
<p>可能提示如下:</p> 
<ul> 
 <li>节点上传失败,请优先检查限制上传大小配置是否合理</li> 
 <li>上传文件太大了,请重新选择一个较小的文件上传吧</li> 
</ul> 
<p>出现如上提示信息需要对插件端或者服务端进行配置合理对上传文件大小限制。<br> 配置方式：在对应端的 <code>extConfig.yml</code> 配置文件中配置如下代码</p> 
<pre><code class="language-yaml">spring:  
 servlet: multipart: # 上传文件大小限制  
 max-request-size: 2GB max-file-size: 1GB  
</code></pre>
