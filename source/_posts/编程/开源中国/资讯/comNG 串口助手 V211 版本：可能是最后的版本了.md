                                                        <p>comNG 已经开发迭代了大概一年半了吧。目前基本已经实现了预期的绝大多数的功能，所以开发工作也该告一段落了。 我会发布全平台的 v2.1.1 版本，之后如果没有大的功能升级或者 bug 修复，短时间内应该不会有更新了。</p> 
<p><img src="https://oscimg.oschina.net/oscnet/up-2f637b383e3af7023454da5244085985811.JPEG" referrerpolicy="no-referrer"></p> 
<p><strong>更新内容</strong></p> 
<ul> 
 <li>添加多标签支持</li> 
 <li>优化多实例启动速度</li> 
 <li>优化中文字体的显示效果</li> 
 <li>版本更新到2.0版本</li> 
</ul> 
<p><strong>关于 comNG</strong></p> 
<p>comNG 区别于其他串口助手的地方在于其强大的 “现场数据分析“ 能力。简单来说就是 comNG 提供的多种功能以帮助用户更方便的分析打印输出文本。这些功能包括：</p> 
<p>- 专业的 UI 和交互设计<br> - 支持多标签，多实例<br> - 专业的 Hex 模式<br> - 内建的 comNGLang 高亮语法<br> - 手动文本高亮：类似于 notepad++ 的 Style Token<br> - 搜索文本高亮：类似于 vscode 的搜索文本高亮<br> - 选择文本高亮：类似于 vscode 的选择文本高亮<br> - 支持 minimap 功能：类似于 vscode 的 minimap<br> - 基于文本内容的中断功能，这是一个全新设计的功能<br> - 日志文档的签名：时间和姓名<br> - 常用命令支持快捷键操作，比如打开/关闭串口、打开/保存文件、清除文本、高亮文本等<br> - 跨平台：Windows， Mac OS 以及 基于 Debian 的 Linux 系统，比如 Ubuntu 和 Deepin<br> - 自定义主题（只支持软件下方 Bar 的背景色。）<br> - 一个简单的模拟 Terminal 模式</p> 
<p>另外还包含一些串口助手通用的功能：</p> 
<p>- Modem 信号指示和控制<br> - 自定义波特率<br> - 十六进制接收/发送<br> - 接收时间戳<br> - 发送文本<br> - 流控<br> - 文件保存和打开，支持拖动<br> - 抓取至文件</p>
