<p>HarmonyOS 目前提供了 16000 多个 API，而在 API 之外 HarmonyOS 还提供一系列<strong>组件库</strong>供开发者使用，这些库明显降低了应用开发者的开发难度，从而提升开发效率，让应用开发更简单高效。</p> 
<p>HarmonyOS 组件库在 OpenHarmony 开源社区上可以直接获取，华为也还在不断扩增当中。如此好用的 HarmonyOS 组件库到底是什么？我们可以从以下五个方面来理解。</p> 
<p>一、 <strong>HarmonyOS 组件库介绍</strong></p> 
<p>2021 年，HarmonyOS 将陆续推出 1000+组件，包含 UI、动画图形、框架、安全、工具、网络、文件数据、多媒体、图片缓存和基础功能，共 10 类（具体类别可参见下图），具有多设备形态可用、多端部署、性能优化三大特点。</p> 
<p><img height="1074" src="https://static.oschina.net/uploads/space/2021/0428/103203_Kfx3_3820517.png" width="668" referrerpolicy="no-referrer"></p> 
<p>二、 <strong>HarmonyOS 组件 Project 目录结构</strong></p> 
<p>HarmonyOS 组件的项目工程目录结构与 Java 工程类似，分为 build 编译目录、libs 依赖库目录、src 源码目录和构建脚本，结构清晰易懂。</p> 
<p><img height="695" src="https://static.oschina.net/uploads/space/2021/0428/103220_IiFm_3820517.png" width="989" referrerpolicy="no-referrer"></p> 
<p>开发者可以通过 HarmonyOS 的 IDE 工具 DevEco Studio 直接使用组件来进行项目开发，只需打开指定模板，相关目录结构就会自动导入到工程文件中。</p> 
<ul> 
 <li>DevEco Studio 下载链接：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fxie.infoq.cn%2Farticle%2Fbb96bd928dd523f482c782b7b%23download" target="_blank"><u>https://developer.harmonyos.com/cn/develop/deveco-studio#download</u></a></li> 
</ul> 
<p><img height="641" src="https://static.oschina.net/uploads/space/2021/0428/103244_78D0_3820517.png" width="1004" referrerpolicy="no-referrer"></p> 
<p><strong>三、组件的引用方式</strong></p> 
<p>常用组件的引用方式有以下三种：</p> 
<p>1.Har 包引用(build.gradle)</p> 
<p>二进制目录的引用简单直接，开发者只需把组件库都放进 libs 目录即可。做原型时非常方便，是初学者或者测试时常用的方式。</p> 
<pre><code>dependencies &#123;

implementation project(":mylibrary")

&#125;

</code></pre> 
<p>2. 源文件引用(build.gradle)</p> 
<p>开发者对使用的组件有优化需求的时候，可以进行源文件引用，把源码下载下来后同步修改。优化之后，还可以把源码贡献到开源仓中。</p> 
<pre><code>dependencies &#123;

implementation fileTree(dir: 'libs', include: ['*.jar', '*.har'])

&#125;

</code></pre> 
<p>3. Maven 仓引用(build.gradle)</p> 
<p>Maven 仓引用适合大规模团队开发或者商业应用开发，开发者直接在中央仓下载即可。</p> 
<p>步骤一：</p> 
<pre><code>allprojects&#123;

repositories &#123;

maven &#123;

url ‘file://D:/01.localMaven/’  //本地或远程 Maven 仓

&#125;

 &#125;

 &#125;</code></pre> 
<p>步骤二：</p> 
<pre><code>dependencies &#123;

implementation 'com.huawei.har:mylibrary:1.0.1'

 &#125;</code></pre>
