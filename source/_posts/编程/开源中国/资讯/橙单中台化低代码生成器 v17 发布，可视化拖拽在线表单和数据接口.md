                                                        <h3 style="text-align:start">重要更新</h3> 
<p>前端拖拉拽即可完成表单组件、数据接口、多表关联和数据字典的配置，再将表单组件和接口数据进行绑定，最后将配置后的在线表单分配给指定菜单，整个过程无需任何代码修改，即配即用。</p> 
<h3>在线表单</h3> 
<ul> 
 <li>前后端代码完全交付，甚至可配置为自己的项目包名。</li> 
 <li>前后端代码无一丝混淆，且代码质量超高，极易学习和二次开发，同时支持单体和微服务架构。</li> 
 <li>可视化的拖拉拽编辑器，目前已经支持大部分常用组件，后续版本会支持图表。</li> 
 <li>支持主从表联动。</li> 
 <li>支持富文本、多图、多附件的上传和下载。</li> 
 <li>可视化接口配置编辑器，支持多数据库、数据表、一对一表关联、一对多表关联和多种数据字典等。</li> 
 <li>支持字典过滤、主表字段过滤、一对一从表字段过滤，同时支持范围、模糊和等于查询方式。</li> 
 <li>支持主表字段排序、一对一从表字段排序。</li> 
 <li>完美支持数据权限过滤和操作权限控制。</li> 
 <li>接口参数均为数据源 ID，不暴露任何后台数据表细节，从机制上彻底消除了 SQL 注入的风险。</li> 
 <li>运行时效率极高，动态表单关键数据结构均缓存 Redis，从而有效提升数据接口的应答效率。</li> 
</ul> 
<h3>修改说明</h3> 
<ul> 
 <li>升级依赖版本到 Spring Cloud Alibaba 2.2.5。</li> 
 <li>升级依赖版本到 Spring Cloud Dependency Hoxton SR9。</li> 
 <li>升级依赖版本到 Spring Boot 2.3.10.RELEASE。</li> 
 <li>升级 Knife4j 到 2.0.8，同时修改了 common-swagger 中与 swagger 插件相关的部分类名。</li> 
 <li>升级其他第三方依赖库。</li> 
 <li>新增在线表单的通用模块 common-online，仅当用户工程选择支持在线表单时生成。</li> 
</ul>
