<div class="content">
                                                                                            <p><span style="background-color:#ffffff; color:#333333">GNU 调试器 GDB 11.1 现已发布。GDB 是 Ada、C、C++、Go 与 Rust 等源代码级调试器语言，它可以在最流行的 GNU/Linux、Unix 和 Windows 变体上运行，并且调试十几种不同的处理器架构。</span></p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#333333">已删除对 ARM Symbian (arm*-*-symbianelf*) 的支持</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">构建 GDB 现在需要 GMP（GNU 多精度算术库）</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">新的命令行选项 "--early-init-command"（或 "-eix"）和 "--early-init-eval-command"（或 "-eiex"）</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">GDB/MI 更改：</span> 
  <ul> 
   <li><span style="background-color:#ffffff; color:#333333">"-break-insert" 和 "-dprintf-insert" 命令的新 --qualified 选项</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"-break-insert" 和 "-dprintf-insert" 命令的新 --force-condition 选项</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"-break-condition" 命令的新 --force 选项</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"-file-list-exec-source-files" 现在接受一个可选的正则表达式来过滤结果中包含的源文件</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"-file-list-exec-source-files" 的结果现在包含一个 "debug-fully-read" 字段，以指示相应源的调试信息是否已部分读取 (false) 或已完全读取 (true)</span></li> 
  </ul> </li> 
 <li><span style="background-color:#ffffff; color:#333333">TUI 改进：</span> 
  <ul> 
   <li><span style="background-color:#ffffff; color:#333333">现在支持鼠标操作。鼠标滚轮滚动相应的窗口</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">在聚焦窗口上没有特定操作的组合键现在传递给 GDB</span></li> 
  </ul> </li> 
 <li><span style="background-color:#ffffff; color:#333333">Python 增强功能：</span> 
  <ul> 
   <li><span style="background-color:#ffffff; color:#333333">Inferior objects 现在包含一个只读的 'connection_num' 属性，它给出了在 "info connections" 和 "info secondarys" 中看到的连接号</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">新方法 gdb.Frame.level() 返回框架对象的堆栈级别</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">新方法 gdb.PendingFrame.level() 返回框架对象的堆栈级别</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">当命中一个捕获点时，Python API 现在将发出 gdb.BreakpointEvent 而不是 gdb.StopEvent。附加到事件的 gdb.Breakpoint 类型为 BP_CATCHPOINT</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">Python TUI 窗口现在可以接收鼠标单击事件。如果 Window 对象实现了 click 方法，则此窗口中的每个鼠标单击事件都会调用它</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">新设置 "python ignore-environment on|off"，如果为 "on"，则导致 GDB 的内置 Python 忽略任何会影响 Python 行为的环境变量（需要在 "早期初始化" 期间设置（见上文）</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">新设置 "python dont-write-bytecode auto|on|off"</span></li> 
  </ul> </li> 
 <li><span style="background-color:#ffffff; color:#333333">Guile API 增强功能：</span> 
  <ul> 
   <li><span style="background-color:#ffffff; color:#333333">改进了对右值引用的支持</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">获取值变量的新程序：value-reference-value、value-rvalue-reference-value 和 value-const-value</span></li> 
  </ul> </li> 
 <li><span style="background-color:#ffffff; color:#333333">新的 "qMemTags" 和 "QMemTags" 远程协议包（与内存标记相关）</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">当条件在一个或多个位置无效时，"break [...] if CONDITION" 命令不再返回错误。相反，如果条件在一个或多个位置有效，则条件无效的位置将被禁用</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">"condition" 命令的行为已更改以匹配 "break" 命令的新行为</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">支持通用内存标记功能（目前仅限于 AArch64 MTE）</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">现在支持 x86_64 Cygwin 程序的核心文件调试</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">RISC-V 目标的新 "org.gnu.gdb.riscv.vector" 功能</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">GDB 现在支持定点类型，这些类型在 DWARF 中被描述为具有定点编码的基本类型。此外，还添加了对 DW_AT_GNU_numerator 和 DW_AT_GNU_denominator 的支持</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">其它更新：</span> 
  <ul> 
   <li><span style="background-color:#ffffff; color:#333333">新的 "</span><span style="background-color:#ffffff; color:#000000">startup-quietly on|off</span><span style="background-color:#ffffff; color:#333333">" 设置；当 "on" 时，其行为与在命令行上传递 "-silent" 选项相同</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">新的 "</span><span style="background-color:#ffffff; color:#000000">print type hex on|off</span><span style="background-color:#ffffff; color:#333333">" 设置； "on" 时，"ptype" 命令使用十六进制表示法打印结构成员的大小和偏移量。"off" 时，使用十进制表示法</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"inferior" 命令在不带参数的情况下运行时，会打印有关当前次等设备的信息</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"ptype" 命令现在支持 "/x" 和 "/d"，影响用于打印尺寸和偏移量的基数</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">"</span><span style="background-color:#ffffff; color:#000000">info source</span><span style="background-color:#ffffff; color:#333333">" 的输出已被重组</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">新的 "</span><span style="background-color:#ffffff; color:#000000">style version foreground | background | intensity</span><span style="background-color:#ffffff; color:#333333">" 命令来控制 GDB 版本号的样式</span></li> 
   <li><span style="background-color:#ffffff; color:#333333">各种调试和维护命令（主要供 GDB 开发人员使用）</span></li> 
  </ul> </li> 
</ul> 
<p><span style="background-color:#ffffff; color:#333333">详情请查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.gnu.org%2Fsoftware%2Fgdb%2F" target="_blank">更新公告</a>。</span></p>
