                                                                                            <p>PyCharm 2021.3.3 正式发布了，<span style="color:#27282c">此版本改进了代码洞察，修复 Docker 的一些主要 Bug ，带来更流畅的编码体验。</span></p> 
<p style="margin-left:0px"><strong>社区贡献：增强对泛型类型别名的支持</strong></p> 
<p style="margin-left:0px">现在 PyCharm 完全支持泛型类型别名，IDE 可以识别 PEP 484 类型别名中的部分专用泛型类型。</p> 
<p style="margin-left:0px"><strong>改进和修复：</strong></p> 
<ul> 
 <li>代码洞察：空格检查以“match”和“case”开头的变量不再引发误报 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FPY-51141" target="_blank">[PY-51141]</a></li> 
 <li>代码洞察：捆绑的 pycodestyle 已更新到版本 2.8.0  <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FPY-51269" target="_blank">[PY-51269]</a></li> 
 <li>Docker：现在已成功创建 Docker Compose 解释器 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FPY-50932" target="_blank">[PY-50932]</a></li> 
 <li>macOS：现在可以通过 PyCharm 为 M1 处理器创建基于 Python 2 的 venv <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FPY-46430" target="_blank">[PY-46430]</a></li> 
</ul> 
<p style="margin-left:0px">有关 PyCharm 2021.3.2 中解决的问题的完整列表，请查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Farticles%2FPY-A-233537929%2FPyCharm-2021.3.3-%28213.7172.26-build%29-Release-Notes" target="_blank">发行说明</a>。</p>
