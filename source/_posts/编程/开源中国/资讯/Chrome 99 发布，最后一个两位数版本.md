                                                                                            <p>最后一个两位数的 Chrome 版本正式发布，新版本中包括工具栏下载按钮、更好的日期选择器等。</p> 
<h3>Chrome 工具栏下载按钮</h3> 
<p><img alt height="185" src="https://oscimg.oschina.net/oscnet/up-b42b5a02b7e47cddf0820713c19d4b0493f.png" width="700" referrerpolicy="no-referrer"></p> 
<p>Chrome 99 的下载栏不再出现在底部，而是会在顶部工具栏生成一个下载图标。</p> 
<p>当你下载东西时，顶部工具栏上会出现一个蓝色的下载小图标。当下载完成后，它变成灰色并最终消失。</p> 
<h3>Web 应用可以使用系统日期选择器</h3> 
<p><img alt height="190" src="https://oscimg.oschina.net/oscnet/up-cf274759ffe3a4c8ea506987c7a1ab5744f.png" width="700" referrerpolicy="no-referrer"></p> 
<p>这一直是开发者强烈要求的功能，Chrome 99 终于可以为表单使用系统日期选择器了，使 Web 应用更有原生应用的感觉。</p> 
<h3>内置手写识别</h3> 
<p>Google 在 Chrome 91 中开始试验内置手写识别 API。这使开发者能够轻松地在他们的 Web 应用中实现手写功能。Chrome 99 中最终确定了这个 API，无需使用第三方集成来实现这个功能。</p> 
<h3>弃用 Manifest V2</h3> 
<p>2020 年底，Google 推出了 Manifest V3 API，这是一个用于扩展的 API。Manifest V3 的主要内容是用 chrome.declarativeNetRequest API 取代 webRequest API（后者是每个广告屏蔽扩展都在使用的 API）。</p> 
<h3>其他</h3> 
<ul> 
 <li>Web 应用程序现在支持 color_scheme_dark 字段，用于为黑暗模式指定不同的主题色和背景色</li> 
 <li>Chrome 99 引入了新的文本强调 CSS 属性</li> 
 <li>CSS Cascade Layers</li> 
 <li>Battery Status API 不再支持不安全的来源，如 HTTP 页面或嵌入 HTTP 页面的 HTTPS iframes</li> 
</ul>
