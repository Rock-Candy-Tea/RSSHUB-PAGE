                                                        <p><span style="background-color:#ffffff; color:#0f1419"><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Feslint.org%2Fblog%2F2021%2F10%2Feslint-v8.0.0-released" target="_blank">ESLint v8.0.0 已正式发布</a>，此版本属于重大更新升级——增加了新功能，如支持 ES2022 语法，以及修复在旧版本中发现的错误，不过也引入了部分破坏性的变更。</span></p> 
<h3>更新要点</h3> 
<ul> 
 <li>ESLint 的默认解析器和所有核心规则完全支持以下 ES2022 语法： 
  <ul> 
   <li>Class public instance fields and private instance fields</li> 
   <li>Class public static fields and private static fields</li> 
   <li>Class private instance methods and accessors</li> 
   <li>Class private static methods and accessors</li> 
   <li>RegExp match indices</li> 
   <li>Top-level<span> </span><code>await</code></li> 
  </ul> </li> 
 <li>当<code>--fix</code>和<code>--report-unused-disable-directives</code><span>同时使用时</span>，Autofix 可移除未使用的禁用指令</li> 
 <li>新版本已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Feslint%2Feslint%2Fissues%2F15017" target="_blank">不允许</a>在 ES3 中使用保留字作为标识符，这一变化只影响到在解析器选项中将默认解析器配置为<code>&#123; "ecmaVersion": 3 &#125;</code>的用户</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Feslint.org%2Fdocs%2F8.0.0%2Frules%2Fclass-methods-use-this" target="_blank"><code>class-methods-use-this</code></a>规则包含新的<code>enforceForClassFields</code>选项</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Feslint.org%2Fdocs%2F8.0.0%2Frules%2Fmax-classes-per-file" target="_blank"><code>max-classes-per-file</code></a>规则包含新的<code>ignoreExpressions</code><span>选项</span></li> 
 <li><code>meta.docs.category</code><span> </span>属性已从所有核心规则中删除</li> 
</ul>
