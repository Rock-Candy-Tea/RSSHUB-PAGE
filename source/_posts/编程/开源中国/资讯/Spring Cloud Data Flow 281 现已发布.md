                                                        <p>Spring Cloud Data Flow 2.8.1 发布了。Spring Cloud Data Flow 是构建数据集成和实时数据处理流水线的工具包。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-422a9f64c03477f2810e41166fdb6ccedfc.png" referrerpolicy="no-referrer"></p> 
<p><strong>主要更新内容</strong></p> 
<ul> 
 <li>Spring Boot 更新到 2.4.7</li> 
 <li>Spring Cloud 更新到 2020.0.3</li> 
 <li>Java CFEnv 更新到 2.3.0</li> 
 <li>spring-cloud-services-config-client-autoconfigure 更新到 3.2.1</li> 
 <li>现在会使用 postfixed 标签为 jdk8 和 jdk11 创建图片，同时将普通图片默认为 jdk11，以在未来更轻松地添加额外的图像</li> 
 <li>迁移到 junit5</li> 
 <li>在 Skipper 中运行的流应用继承 JAVA_TOOL_OPTIONS</li> 
 <li>删除关键服务 dep 处理</li> 
 <li>在前次启动没有外部执行程序 ID 的情况下启动任务的 NPE</li> 
 <li>NPE 任务从 shell 启动并且没有选择平台</li> 
 <li>任务清理应该清理 TAS 中的所有痕迹</li> 
 <li>如果源/接收器名称相同，应用列表会显示错误的版本计数</li> 
 <li>从任务执行页面上的可用 CTR 属性列表中删除图形属性</li> 
</ul> 
<p>详情请查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fspring-cloud%2Fspring-cloud-dataflow%2Freleases%2Ftag%2Fv2.8.1" target="_blank">更新公告</a>。</p>
