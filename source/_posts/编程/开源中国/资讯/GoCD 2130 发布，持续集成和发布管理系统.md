                                                                                            <p>GoCD 21.3.0 已发布，GoCD 是一个持续集成和发布管理系统，其前身为 CruiseControl，它是 ThoughtWorks 在做咨询和交付项目时研发的一款开源持续集成工具。后来随着持续集成及持续部署的火热，ThoughtWorks 专门成立了一个项目组，基于 Cruise 开发出这款工具。</p> 
<p>此版本对许多内部组件进行了重要的安全修复和升级，官方建议所有用户升级到 21.3.0，以保护 GoCD 服务器。</p> 
<ul> 
 <li>Git、Mercurial、Subversion 和 TFS 材料的所有材料 URL 现在都需要以字母数字字符或<code>/</code>作为开头。如果此项变化对你造成影响，请提交一个 GitHub issue，官方提供了一个临时 flag 用于此项变化</li> 
 <li>删除 Job Details 页面中的"Failures"选项卡</li> 
 <li>支持在 Alpine 3.13 上构建默认 GoCD 服务器镜像</li> 
 <li>自此发行版开始，面向 GoCD 代理的基于 Alpine 3.14 的 docker 镜像已经可以使用</li> 
</ul> 
<p>此外，官方称 GoCD 尚不支持 Java 16+，而升级到 Java 16 需要进行相当多的更改，他们会在不久的将来进行尝试。鉴于 Java 生态的变化（例如 Adoptium 不再提供 JRE），他们可能需要重新考虑坚持使用最新的 JVM，并考虑使用 LTS 版本。</p>
