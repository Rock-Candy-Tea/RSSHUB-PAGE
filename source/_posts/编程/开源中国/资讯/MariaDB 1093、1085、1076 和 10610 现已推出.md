                                                                                            <p><span><span><span><span><span><span style="color:#111111"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>MariaDB 基金会宣布了 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmariadb.com%2Fkb%2Fen%2Fmariadb-1093-release-notes%2F" target="_blank">MariaDB 10.9.3</a>、<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmariadb.com%2Fkb%2Fen%2Fmariadb-1085-release-notes%2F" target="_blank">MariaDB 10.8.5</a> 和 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmariadb.com%2Fkb%2Fen%2Fmariadb-1076-release-notes%2F" target="_blank">MariaDB 10.7.6</a> 的可用性，它们各自的短期支持系列中最新的普遍可用版本（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmariadb.org%2Fnew-innovation-release-model%2F" target="_blank">从第一个 GA 发布日期起维护一年</a>）。以及 10.6 长期支持系列（维护五年）中最新的稳定版本 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmariadb.com%2Fkb%2Fen%2Fmariadb-10610-release-notes%2F" target="_blank">MariaDB 10.6.10</a></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span><span style="color:#000000">，该版本维护至 2026 年 7 月</span>。<span><span><span><span><span><span style="color:#111111"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>这些版本修复<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmariadb.org%2Fregressions-in-recent-mariadb-server-releases%2F" target="_blank">了以前版本中发现的严重回归</a>问题。</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<p><span><span><span><span><span><span style="color:#111111"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>具体更新内容包括：</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li>修复 mtr_t::commit() 中 mysql_mutex_assert_owner(&log_sys.flush_order_mutex) 的断言失败 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjira.mariadb.org%2Fbrowse%2FMDEV-29383" target="_blank">MDEV-29383</a> )</li> 
 <li>修复 OOM 后 InnoDB 中频繁出现的“数据结构损坏” ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjira.mariadb.org%2Fbrowse%2FMDEV-29374" target="_blank">MDEV-29374</a> )</li> 
 <li>修复即时 ALTER TABLE 的恢复或备份不正确 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjira.mariadb.org%2Fbrowse%2FMDEV-29438" target="_blank">MDEV-29438</a> )</li> 
 <li>修复 InnoDB Temporary Tablespace (ibtmp1) 持续增长 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjira.mariadb.org%2Fbrowse%2FMDEV-28240" target="_blank">MDEV-28240</a> )</li> 
 <li>如果在更改完全刷新之前关闭，则修复全文索引损坏 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fjira.mariadb.org%2Fbrowse%2FMDEV-29342" target="_blank">MDEV-29342</a> )</li> 
</ul>
