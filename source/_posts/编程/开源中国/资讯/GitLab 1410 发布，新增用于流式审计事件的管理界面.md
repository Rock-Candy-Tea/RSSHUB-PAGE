                                                                                            <p>GitLab 14.10 已正式发布。新版本亮点包括：引入用于流式审计事件的管理界面、支持 Kubernetes 的 GitLab Runner、从 GitHub 导入默认为当前组路径。</p> 
<p><strong>用于流式审计事件的管理界面</strong></p> 
<p>从该版本开始，用户可以使用 GitLab UI 在组中设置流式审计事件，并在组审计事件页面的新 Streams 选项卡下进行访问。通过该界面可以实现：</p> 
<ul> 
 <li>添加和删除流式审计事件目标</li> 
 <li>查看流审计事件发送到的位置列表</li> 
</ul> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-4dcea61794bc52b77f1fcfa0966ca8efddc.png" referrerpolicy="no-referrer"></p> 
<p><strong>支持 Kubernetes 的 GitLab Runner</strong></p> 
<p>GitLab 13.10 发布了适用于 Kubernetes 的 Red Hat OpenShift 容器平台的 GitLab Runner Operator。该版本为 OpenShift 用户提供了 Operator Framework 的自动化和管理功能，并简化了 OpenShift Kubernetes 集群中运行器的持续管理。从 14.10 开始则提供了 GitLab Runner Operator v1.7.0，用户可以在非 OpenShift Kubernetes 集群中使用。</p> 
<p><img src="https://static.oschina.net/uploads/space/2022/0426/231442_xLE4_2720166.png" referrerpolicy="no-referrer"></p> 
<p><strong>从 GitHub 导入项目默认为当前组路径</strong></p> 
<p>从 GitHub 导入项目时，此里程碑将导入项目的默认目标更改为开始导入的 GitLab 组的上下文命名空间。此前 GitHub 项目则是导入到个人命名空间。这种可用性增强可帮助用户以更直观的方式导入项目，并且不会将导入的项目置于混乱的位置。</p> 
<p><img src="https://static.oschina.net/uploads/space/2022/0426/232027_EVR5_2720166.png" referrerpolicy="no-referrer"></p> 
<p><strong>查看项目的导入历史</strong></p> 
<p>新的项目导入历史页面列出了所有项目导入信息，无论它们的导入路径来自何处。</p>
