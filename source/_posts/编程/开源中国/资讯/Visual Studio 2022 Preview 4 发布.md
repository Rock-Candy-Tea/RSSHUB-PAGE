 <p>Visual Studio 2022 Preview 4 现已发布，该版本中有更多关于个人和团队生产力、现代开发和不断创新等主题的新功能。</p> 
 <h4>个人和团队生产力</h4> 
 <p>Visual Studio 2022 的一个重点领域是提高 Visual Studio 的可伸缩性和性能，而预览版 4 专注于提高几个关键功能的性能。例如，在搜索 Orchard Core 等大型解决方案时，文件中的查找速度现在提高了 3 倍。</p> 
 <p><img alt height="540" src="https://oscimg.oschina.net/oscnet/up-6e981ee950a17ed9d3f810de44f672d77cc.gif" width="968" referrerpolicy="no-referrer"></p> 
 <p>此外，改进性能的其他场景包括 C++ IntelliSense 性能改进，语义着色速度提高了约 12%，符号数据库处理的优化以及解决方案资源管理器中扩展 C++ 项的速度提高了近 2 倍。<br> Preview 4 版本还对调试进行了许多改进。从更新到附加到进程对话框，用户现在可以使用窗口选择器选择进程。从更新到新的外部源功能，可以更轻松地为项目外部的库加载符号，到依赖断点等新功能，用于在第一次命中另一个断点后配置额外的断点。</p> 
 <p><img alt height="423" src="https://oscimg.oschina.net/oscnet/up-014e4f3ca26394888b11acd544930e200c1.png" width="534" referrerpolicy="no-referrer"></p> 
 <h4>开发现代应用</h4> 
 <p>在预览版 4 中，Blazor 和 Razor 编辑器进行了重大更新，解决了预览版 3 中的问题，并增加了 ASP.NET Core 中热重载的新功能——包括文件保存时的热重载和对 CSS 文件实时应用更改。</p> 
 <p><img alt src="https://oscimg.oschina.net/oscnet/up-a0dce5f45d2e848fd5e4f17b145acccae9b.gif" referrerpolicy="no-referrer"></p> 
 <h4>更多创新功能</h4> 
 <p>预览版 4 现在支持将 libFuzzer 与 MSVC 编译器一起使用，因此开发者使用 MSVC 编译的任何二进制文件现在都可以由 libFuzzer 进行模糊测试。并且现在可以根据更新的创建 Git 存储库体验创建 Azure DevOps 存储库。</p> 
 <p><img alt height="567" src="https://oscimg.oschina.net/oscnet/up-c6bb82dcab1e7da86e647a98059f543276c.png" width="768" referrerpolicy="no-referrer"></p>
