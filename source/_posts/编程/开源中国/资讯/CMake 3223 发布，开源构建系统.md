                                                                                            <p>CMake 是一个跨平台的自动化构建系统，它使用一个名为 CMakeLists.txt 的文件来描述构建过程，可以产生标准的构建文件，如 Unix 的 Makefile 或 Windows Visual C++ 的 projects/workspaces 。文件 CMakeLists.txt 需要手工编写，也可以通过编写脚本进行半自动的生成。</p> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start">CMake 3.22.3 发布，更新内容如下：​​</p> 
<ul> 
 <li style="text-align:start"><span style="color:#000000">while：恢复条件错误的容差</span></li> 
 <li style="text-align:start">FindMPI：在 mpi 测试源中先放置静态</li> 
 <li style="text-align:start">NMake：在注释中记录响应文件的编码启发式方法</li> 
 <li style="text-align:start">NMake: 仅在 MSVC 工具中的响应文件中使用 UTF-8 BOM</li> 
 <li style="text-align:start">message：恢复在 stderr 上明确刷新消息的功能</li> 
 <li style="text-align:start">gitlab-ci：在 fedora34-ninja 工作上启用断言</li> 
 <li style="text-align:start">CompilerId：修复针对 MSVC 的 Clang 的默认扩展检查 </li> 
 <li style="text-align:start"> Intel：在 Windows 上添加对 system header files 的依赖性</li> 
 <li style="text-align:start">Tests：在 Qt4 和 Qt5 测试中添加缺失的防护措施</li> 
 <li style="text-align:start">BinUtils：避免苹果平台上的 llvm-ar</li> 
 <li style="text-align:start">FindMatlab：修复对 MCR >= 9.10 的版本检测</li> 
 <li style="text-align:start">......</li> 
</ul>
