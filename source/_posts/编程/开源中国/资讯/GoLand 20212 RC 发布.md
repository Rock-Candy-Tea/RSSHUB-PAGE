<div class="content">
                                                                                            <p>GoLand 2021.2 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fblog.jetbrains.com%2Fgo%2F2021%2F07%2F22%2Fthe-goland-2021-2-release-candidate-is-out%2F" target="_blank">发布</a>了首个 RC 版本。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-d8c174bdaaf75c78fceef0379104858cc5b.png" referrerpolicy="no-referrer"></p> 
<p>此版本主要是修复两个次要问题：</p> 
<ul> 
 <li>修复错误行为导致的 reference resolution 问题，这些问题会导致在带有 replace 指令的项目中出现错误</li> 
 <li>修复格式问题</li> 
</ul> 
<p><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fconfluence.jetbrains.com%2Fdisplay%2FGOLAND%2FGoLand%252B212.4746.57%252BRelease%252BNotes%3F_ga%3D2.70081040.1430227311.1626937558-47346531.1613657090%26_gl%3D1*pdtz7m*_ga*NDczNDY1MzEuMTYxMzY1NzA5MA..*_ga_0WQ2ZF5VGT*MTYyNjk5NDU1OS41OS4wLjE2MjY5OTQ1NTkuMA.." target="_blank">完整修复列表点此查看</a>。</p> 
<p>GoLand 2021.2 的主要变化包括为 Go modules 添加新功能。还有一个新的格式化选项，用于支持 Go 1.17 特性，以及支持 .go 文件中的泛型。此外还增加了新的快速修复功能，包括可以帮助正确使用新的<code>//go:build</code>语法。</p> 
<p>其他方面的更新包括版本控制方面的更新，例如支持用 GPG 密钥签署 Git 提交。还有一些 UI 方面的改进，比如更好地查看附加项目。对于 Web 开发者来说，当代码被保存时，页面将在浏览器中重新加载，而且 MongoDB 字段和操作符也有了自动补全功能。</p> 
<p>下载地址：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.jetbrains.com%2Fgo%2Fnextversion%2F" target="_blank">https://www.jetbrains.com/go/nextversion/</a></p> 
<p><strong>更好地支持不同的 Go 版本</strong></p> 
<p>如果所使用的 Go 语言版本中的功能比 go.mod 文件中指定的版本更加新，那么将会收到错误消息。此时 GoLand 会提示出问题的地方。</p> 
<p><img src="https://static.oschina.net/uploads/space/2021/0710/075519_1SJS_2720166.gif" referrerpolicy="no-referrer"></p> 
<p><strong>格式化工具</strong></p> 
<p>Go 语言本身就包含用于格式化代码的 gofmt 工具，其实 GoLand 也有自己的格式化程序。 </p>
