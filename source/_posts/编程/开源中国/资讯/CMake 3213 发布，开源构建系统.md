                                                                                            <p>CMake 3.21.3 现已发布。CMake 是一个跨平台的自动化构建系统，它使用一个名为 CMakeLists.txt 的文件来描述构建过程，可以产生标准的构建文件，如 Unix 的 Makefile 或 Windows Visual C++ 的 projects/workspaces 。文件 CMakeLists.txt 需要手工编写，也可以通过编写脚本进行半自动的生成。</p> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left">CMake 3.21.3 <span style="background-color:#ffffff; color:#333333">中的一些重要变更有：</span></p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#333333">FindBoost：增加对 Boost 1.77 的支持</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">MSVC：为 VS 2022 v143 工具集修复MSVC_TOOLSET_VERSION</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">IRSL：增加对 VS 2022 v143 toolset redistributables 的发现。</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">gitlab-ci：更新 Windows 构建至 MSVC 19.29-16.11 工具集</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">VS：为 VS 2022 修复托管 C++ 项目的生成</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">VS：为 Preview 4 更新 Visual Studio 17 2022 生成器</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">HIP：提前检测 ROCm 路径</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">HIP：删除 ROMClang compiler ID，直接使用 Clang</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">HIP：简化对 HIP 运行时 CMake 包的检测</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Android：恢复搜索可执行文件的 PATH</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">FindPython：确保使用 Mac M1 上的 homebrew</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Ninja：修复 Windows 下的 ARMClang 路径</span></li> 
 <li><span style="background-color:#ffffff; color:#333333"> HIP：使用 'rocm_agent_enumerator' 来确定 CMAKE_HIP_ARCHITECTURES</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">KWSys：SystemTools：在使用前确保 Windows Vista APIs 是可用的</span></li> 
</ul>
