<div class="content">
                                                                                            <p>WebStorm 2022.1 EAP 4<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fblog.jetbrains.com%2Fwebstorm%2F2022%2F03%2Fwebstorm-2022-1-eap-4%2F" target="_blank"> 现已可用</a>，该版本带来更好的 Next.js 支持、支持 Volta、对 Yarn 和 pnpm 的 Corepack 支持，以及新增一个“运行当前文件”功能。</p> 
<h2>更好的 Next.js 支持</h2> 
<p style="margin-left:0px">此版本改进了对 Next.js 框架的支持。WebStorm 将正确解析组件 <code>href</code> 属性中使用的路径 <code>Link</code>。代码补全、导航和重构（如 <em>重命名 </em>）也正常运行。</p> 
<p style="margin-left:0px"><img alt height="350" src="https://oscimg.oschina.net/oscnet/up-f7caa984338818daf28d8a41095e5691dea.png" width="700" referrerpolicy="no-referrer"></p> 
<h2>支持 Volta</h2> 
<p style="margin-left:0px">WebStorm 2022.1 EAP 4 还与JavaScript 工具管理器<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fvolta.sh%2F" target="_blank">Volta进行适当的集成</a>，它会自动识别使用 Volta 安装的 Yarn 和 npm。</p> 
<p style="margin-left:0px"><img alt height="350" src="https://oscimg.oschina.net/oscnet/up-9c5b6d8146211cea0978725f19494ec1b22.png" width="700" referrerpolicy="no-referrer"></p> 
<h2>对 Yarn 和 pnpm 的 Corepack 支持</h2> 
<p style="margin-left:0px">新增对 Corepack 的支持。Corepack 是一个实验性的 Node.js 工具，无需安装即可使用 Yarn 和 pnpm。该功能默认情况下未打开，更多信息请参见<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fnodejs.org%2Fdist%2Flatest-v16.x%2Fdocs%2Fapi%2Fcorepack.html" target="_blank">官方文档</a>。</p> 
<h2>运行当前文件的新方法</h2> 
<ul> 
 <li>工具栏上现在有一个新的<em><strong>当前文件 </strong></em>选项：可以使用右侧的相应按钮运行和调试当前打开的文件。</li> 
 <li>IDE 将自动使用最适合此文件的运行配置类型，与从上下文菜单运行文件时的方式相同。</li> 
 <li>要运行当前文件，请确保编辑器已聚焦，否则图标将被禁用。</li> 
</ul> 
<p style="margin-left:0px"><img alt height="350" src="https://oscimg.oschina.net/oscnet/up-ebf35ed15aa70c0e3068c93943974b02773.png" width="700" referrerpolicy="no-referrer"></p> 
<ul> 
 <li>仅当项目中没有现成的运行配置时，才会显示新选项。</li> 
 <li>即使确实存在一些运行配置，仍然可以从工具栏上的组合框中选择此 <em><strong>当前文件</strong> </em>选项。</li> 
</ul> 
<p><img alt height="350" src="https://oscimg.oschina.net/oscnet/up-d4c492ebce1e6c7d7a34bef0582221f5915.png" width="700" referrerpolicy="no-referrer"></p> 
<h2>其他显着变化</h2> 
<ul> 
 <li>可以在编辑器中折叠具有多个参数的列表 ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FWEB-29242" target="_blank">WEB-29242</a> )。</li> 
 <li>为 Hermes 调试 添加了一些初始支持( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FWEB-53699" target="_blank">WEB-53699</a> ) 。</li> 
 <li>Tailwind CSS 补全现在应该可以在 FreeMarker 模板中使用  ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fyoutrack.jetbrains.com%2Fissue%2FWEB-54820" target="_blank">WEB-54820</a> ) 。</li> 
</ul>
