                                                                                            <p><span style="background-color:#ffffff; color:#333333">PHP 8.1.4 现已</span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.php.net%2Farchive%2F2022.php%232022-03-17-2" target="_blank">发布</a><span style="background-color:#ffffff; color:#333333">，</span><span style="color:#000000">这是一个错误修复版本</span><span style="background-color:#ffffff; color:#333333">；官方鼓励所有 PHP 8.1 用户升级到此版本。具体更新内容如下：</span></p> 
<ul> 
 <li>Core： 
  <ul> 
   <li>修复 Haiku ZTS 构建。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8059" target="_blank">#8059</a> 未为扩展重新生成 arginfo。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8083" target="_blank">#8083</a> 使用静态变量转储未调用的假闭包时的 Segfault<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8083" target="_blank">。</a></li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F7958" target="_blank">#7958</a>（嵌套的 CallbackFilterIterator 泄漏内存）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8074" target="_blank">#8074</a>（range() result 的错误类型推断）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8140" target="_blank">#8140</a>（Wrong first class 可按名称优化调用）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8082" target="_blank">#8082</a>（观察到带有临时 run_time_cache 泄漏内存的 op_arrays）。</li> 
  </ul> </li> 
 <li>GD： 
  <ul> 
   <li>修复了加载隔行扫描图像时出现的 libpng 警告。</li> 
  </ul> </li> 
 <li>FPM： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=http%3A%2F%2Fbugs.php.net%2F76109" target="_blank">#76109</a>（对 fpm 记分板的不安全访问）。</li> 
  </ul> </li> 
 <li>Iconv： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F7953" target="_blank">#7953</a>（ob_clean() 仅不设置 Content-Encoding）。</li> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F7980" target="_blank">#7980</a>（iconv_mime_decode 的意外结果）。</li> 
  </ul> </li> 
 <li>MBString： 
  <ul> 
   <li>修复了错误<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8128" target="_blank">#8128</a>（7bit 的 mb_check_encoding 错误结果）。</li> 
  </ul> </li> 
 <li>MySQLnd: 
  <ul> 
   <li>修复了错误 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8058" target="_blank">#8058</a>（mysqlnd 包中的 NULL 指针取消引用）。</li> 
  </ul> </li> 
 <li>Reflection： 
  <ul> 
   <li>修复了错误 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F8080" target="_blank">#8080</a>（ReflectionClass::getConstants() 取决于默认顺序）。</li> 
  </ul> </li> 
 <li>Zlib： 
  <ul> 
   <li>修复了错误 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fphp%2Fphp-src%2Fissues%2F7953" target="_blank">#7953</a>（ob_clean() 仅不设置 Content-Encoding）。</li> 
  </ul> </li> 
</ul>
