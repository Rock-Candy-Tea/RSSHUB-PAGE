                                                                                            <p>SystemRescue（以前被称为 SystemRescueCd）Linux 驱动的实时系统救援工具包已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.system-rescue.org%2FChanges-x86%2F" target="_blank">更新</a>至 9.00 版，这是一个带来新功能和技术的主要版本。</p> 
<p>SystemRescue 9.00 版本由最新的长期支持 (LTS) <strong>Linux 5.15 </strong>内核系列提供支持，不仅带来了更好的硬件支持，而且引入了全新的 NTFS 文件系统实现等新功能，提供全功能（读/写）NTFS 支持。完整更新内容如下：</p> 
<ul> 
 <li>将 Linux 内核更新为 LTS 版本的 linux-5.15.14</li> 
 <li>新内核带来了 ntfs3 模块，提供更好的 NTFS 支持</li> 
 <li><span style="color:#000000">该系统附带一份网站副本作为离线手册</span></li> 
 <li>添加<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.system-rescue.org%2Fscripts%2Fsysrescue-customize%2F" target="_blank">了 sysrescue-customize 脚本</a>以创建自定义 ISO 映像</li> 
 <li>初步支持<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.system-rescue.org%2Fmanual%2FConfiguring_SystemRescue%2F" target="_blank">使用 YAML 文件配置系统</a></li> 
 <li>支持使用新的 YAML 配置文件配置自动运行</li> 
 <li>用最小的替代品替换了 mesa 包，节省 52MB 空间(#241)</li> 
 <li>将 bash 系统初始化脚本转换为 python (#170)</li> 
 <li>删除了 XOrg 的驱动程序“xf86-video-qxl”，因为它目前不太稳定 (#235)</li> 
 <li>将意外删除的 inetutils 软件包移回来（<span style="color:#000000">telnet, ftp, hostname,…</span>）</li> 
 <li>新添加一些软件包：aq、libisoburn、patch、python-llfuse、python-yaml、rdiff-backup</li> 
</ul> 
<p>更新公告：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.system-rescue.org%2FChanges-x86%2F" target="_blank">https://www.system-rescue.org/Changes-x86/</a></p>
