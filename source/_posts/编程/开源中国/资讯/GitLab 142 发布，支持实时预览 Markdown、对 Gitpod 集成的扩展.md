<div class="content">
                                                                                            <p>GitLab 14.2 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fabout.gitlab.com%2Freleases%2F2021%2F08%2F22%2Fgitlab-14-2-released%2F" target="_blank">已于上周发布</a>，主要更新内容包括：支持实时预览 Markdown、引入 Build Cloud for macOS beta、对 Gitpod 集成的扩展以及增加新的 DevOps 采集指标。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-5470a4a141fb36e7464809cf69f87c54b0d.png" referrerpolicy="no-referrer"></p> 
<h3>实时预览 Markdown</h3> 
<p>此版本增加了实时预览 Markdown 文本的功能。</p> 
<p>Web IDE 和单文件编辑器中添加了新的“实时预览 (live preview)”选项。在编辑器界面点击右键然后选择"Preview Markdown"，或使用<code>Command/Control + Shift + P</code>组合键即可切换至实时预览 Markdown 内容的分屏模式。预览内容会实时刷新，确保 Markdown 内容会按预期进行渲染。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-4752553dd3d70b331df8985a3fcd0b2a1f5.png" referrerpolicy="no-referrer"></p> 
<h3>GitLab Build Cloud for macOS beta</h3> 
<p>GitLab SaaS 上的 Apple 生态系统开发者需要在自己的 macOS 系统上安装、管理和运行 GitLab Runner 来执行 CI/CD 工作流。</p> 
<p>新版本中，他们可以在适用 macOS 的全新 Build Cloud 测试版上构建应用程序，这是一个由 GitLab Runner 驱动的构建平台，与 GitLab SaaS CI/CD 集成。</p> 
<p>测试版最初仅限内测用户和开源社区计划成员。通过该功能用户可以在任何本地 Apple 环境、MacStadium 或 AWS 上安装 macOS Runner。</p> 
<p><img alt src="https://oscimg.oschina.net/oscnet/up-f25f6bb47c7de63501ebcabd5acb3b5826a.png" referrerpolicy="no-referrer"></p> 
<h3>从 PR 启动预配置的 Gitpod 工作区</h3> 
<p>GitLab 13.5 引入的 Gitpod 集成可帮助用户管理复杂的开发环境。在代码中定义项目配置后，只需单击一下即可启动预构建的基于云的开发环境。这种便捷的工作流程使得生成新更改的速度比以往任何时候都快，但是启动 Gitpod 环境来审查现有 PR 意味着在切换到目标分支并再次构建之前，需要针对主分支构建一个环境。</p> 
<p>在 GitLab 14.2 中，可以直接从 PR 页面启动 Gitpod，预先配置为使用目标分支，以加快审查速度并减少上下文切换的需要。启用 Gitpod 集成后，PR 会显示一个分组的 <strong>Open in </strong>按钮，这样就可以在 Web IDE 或 Gitpod 中打开 PR。</p>
