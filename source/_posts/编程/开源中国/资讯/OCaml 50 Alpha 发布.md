                                                                                            <p>在 <a href="https://www.oschina.net/news/178441/ocaml-multicore-merged-upstream" target="news">Multicore OCaml 被合并进主线</a>五个月后，以及 OCaml 4.14.0 发布三个月后，<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdiscuss.ocaml.org%2Ft%2Focaml-5-0-zeroth-alpha-release%2F10026" target="_blank">OCaml 5.0.0 发布了首个 Alpha 版本</a>。</p> 
<p>由于 OCaml 5.0 完全重写了 OCaml 运行时，这个 alpha 版本预计比常见的 alpha 版本更不稳定。特别是，字节码调试器 (bytecode debugger) 仅在下一个 alpha 版本中可用。同样，在下一个 alpha 版本中，内部 C runtime API 和编译器包安装的文件也会有一些变化。</p> 
<p>官方还提到，OCaml 5.0 本身预计将是一个比平时更具实验性的版本。值得注意的是，原生编译器将仅在此 5.0 版本的 ARM64 和 x86-64 架构上可用。</p> 
<p>虽然 alpha 版本存在不少问题，但对于早期用户和希望尝鲜的用户来说，OCaml 5.0 Alpha 已足够稳定。</p> 
<p>简而言之，dune, merlin, ppxlib, utop, ocamlfind 和 ocamlbuild 可正常运行。此外，5.0 兼容 4.0 系列，<span style="background-color:#ffffff; color:#222222">任何顺序的 OCaml 4 库或程序在 OCaml 5 中应该都支持正常运行（不推荐使用的模块和函数除外）。</span></p>
