                                                        <blockquote> 
 <p style="text-align:left">Jboot 一个更简单的分布式、微服务框架。</p> 
 <p style="text-align:left">Jboot是一个基于 JFinal、JFinal-Undertow、Dubbo、Seata、Sentinel、ShardingSphere、Nacos 等开发的微服务框架，帮助开发者降低微服务、分布式开发门槛。爽爽开发，快乐生活。</p> 
</blockquote> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">到目前为止，Jboot 已经开源超过了 5 年的时间，迭代了 200+ 个版本，已经被超过 1000+ 公司在使用，其中包含了多个知名的上市公司，我们了解到的多个使用 Jboot 开发的产品，用户量超过 1亿 以上。</span></p> 
<p style="text-align:left">在 Jboot 之前的版本中，Jboot 的门户网关功能，只能通过配置文件的方式在网关端添加配置，虽然可以通过插件的方式对网关功能扩展，但不同的程序员实现的性能、易用性都是千差万别的。</p> 
<p style="text-align:left">因此，Jboot 官方在现有的基础上，新增了基于 Nacos 的门户网关自动发现服务的功能，同时提供了 SPI 扩展，用户可以轻易扩展出基于 Zookeeper 等其他的方式。</p> 
<p style="text-align:left">使用方法如下：</p> 
<p style="text-align:left">门户网关在原有的配置基础上，添加如下的配置，开启 Nacos 服务发现功能：</p> 
<pre><code class="language-ini">jboot.gateway.discovery.enable = true
jboot.gateway.discovery.type = nacos
jboot.gateway.discovery.nacos.serverAddr = 127.0.0.1:8848</code></pre> 
<p>而在 web 实例端，添加如下配置即可：</p> 
<pre><code>jboot.gateway.discovery.enable = true
jboot.gateway.discovery.type = nacos
jboot.gateway.discovery.nacos.serverAddr = 127.0.0.1:8848


## 注意：这个配置的 myName 必须和 Gateway 里的 'jboot.gateway.name = myName' 中的 myName 一样
jboot.gateway.instance.name = myName</code></pre> 
<p>更多的配置参考：<a href="https://www.oschina.net/action/GoToLink?url=http%3A%2F%2Fwww.jboot.io%2Fdocs%2Fgateway.html" target="_blank">http://www.jboot.io/docs/gateway.html</a></p> 
<p>相关示例参考：<a href="https://gitee.com/JbootProjects/jboot/tree/master/simples/gateway">https://gitee.com/JbootProjects/jboot/tree/master/simples/gateway</a></p> 
<p> </p> 
<p>同时，在此版本中，修复了文档生成工具的许些 bug，基于 JbootAdmin 扩展了更多的高级功能，保姆式帮助程序员进行快速开发。</p> 
<p><img height="1036" src="https://oscimg.oschina.net/oscnet/up-2ddfaa54b36abf2afc1339f7a465938c0d6.png" width="1366" referrerpolicy="no-referrer"></p> 
<p> </p> 
<p style="text-align:left"><strong>Jboot v3.10.3 更新内容如下：</strong></p> 
<ul> 
 <li>新增：门户网关 Gateway 新增基于 Nacos 的自动服务发现的功能</li> 
 <li>新增：JbootHttpImpl 新增默认的 Content-Type 配置</li> 
 <li>新增：JbootHttpRequest 新增 "instanceFollowRedirects" 配置</li> 
 <li>优化：允许 void 的 Controller 方法生成 retRemarks</li> 
 <li>优化：添加 ApiOper.containerClass 配置，用于对 void 类型的 Controller 设置返回值</li> 
 <li>优化：添加 ApiDocConfig.defaultContainerClass 配置，用于配置默认的 void Controller 返回值</li> 
 <li>优化：对 HttpRequest 进行优化，保证 headers 和 paras 的顺序</li> 
 <li>优化：删除 JbootGatewayHandler 默认添加的必要，修改为默认不添加此 Handler</li> 
 <li>修复：@ApiPara.require() 在方法上不起作用的问题</li> 
 <li>修复：Jboot 升级 JFinal 到最新版本后，代码生成器的路径错误的问题</li> 
 <li>示例：新增 Gateway 通过 Nacos 自动发现的 Demo 示例</li> 
 <li>示例：新增 WebSocket 的 Demo 示例</li> 
</ul>
