                                                                                            <p>Mozilla 于今天正式发布了 Firefox 104。Firefox 104 在 7 月底进入公开测试阶段，在测试期间提供了一项用户期待已久的功能，就是无需按住 Alt 键就可以使用双指水平滑动手势在网页上前后导航的功能。</p> 
<p>但 Firefox 104 在最终版本中并没有实装这一功能，因此该功能有可能会推迟到 Firefox 105 版本。Firefox 104 更新的内容包括：</p> 
<h3>新功能</h3> 
<ul> 
 <li> <p>在画中画（PiP）功能中带来了对 Disney+ 字幕的支持</p> </li> 
 <li> <p>增加了对<code>scroll-snap-stop</code>和<code>re-snapping</code>属性的支持</p> </li> 
 <li> <p>Firefox 浏览器的分析器可以分析网站的电量使用情况（仅限苹果 M1 和 Windows 11）。</p> <p><img alt height="153" src="https://static.oschina.net/uploads/space/2022/0824/072407_edFA_4937141.png" width="700" referrerpolicy="no-referrer"></p> </li> 
 <li> <p>当窗口最小化或被遮挡时，Firefox UI 将受到性能和电池使用的限制，以提高系统的整体性能和改善电池寿命。</p> </li> 
 <li> <p>Linux 用户在最小化或恢复窗口时得到了稳定性和性能的提高。</p> </li> 
 <li> <p>在内置的 PDF 浏览器中增加了对文字书写、绘图和签名的支持</p> </li> 
</ul> 
<h3>修复</h3> 
<ul> 
 <li>在雅虎邮箱和 Outlook 的邮件编辑器中输入 <code>Enter</code> 后，高亮的颜色被正确保留</li> 
 <li>绕过只有 https 的错误页面后，返回导航会把你带到之前被忽略的错误页面。现在返回会带你到之前访问过的网站。</li> 
 <li>粘贴未格式化的快捷键（shift+ctrl/cmd+v）现在可以在纯文本环境下工作，如输入和文本区。</li> 
 <li>各种安全修复</li> 
</ul>
