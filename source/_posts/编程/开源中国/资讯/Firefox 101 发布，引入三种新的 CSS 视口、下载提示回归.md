                                                                                            <p>Mozilla 于今天正式发布 Firefox 101 版本，相比 Firefox 100 版本，该版本仅仅带来小幅更新，引入了少量变化和一些改进。</p> 
<p>Firefox 101 中最有趣的变化是 Mozilla 重新实现了下载文件时保存或打开文件的对话提示，该提示此前在 Firefox 98 中被移除。现在，用户将能够通过设置中的常规 > 应用程序部分之后的一个新选项重新启用该对话提示。那里实际上增加了两个新选项，即 "保存文件"（默认）和 "询问是否打开或保存文件"，用户可以根据自己需要调整设置。</p> 
<h3>新功能</h3> 
<ul> 
 <li>有了 <code>prefers-contrast</code> 媒体查询，它允许网站检测用户是否要求以更高（或更低）的对比度来展示网页内容，给予用户更好的阅读体验。</li> 
 <li>所有未配置的 MIME 类型现在可以在下载完成后分配一个自定义操作。</li> 
 <li>Firefox 浏览器现在允许用户在视频会议中同时使用多个麦克风（音频输入），用户也可以在任何时候轻松地切换麦克风</li> 
</ul> 
<h3>修复</h3> 
<ul> 
 <li>修复多个安全问题</li> 
</ul> 
<h3>开发者</h3> 
<ul> 
 <li> <p>检查器面板：当向/从一个现有的 HTML 元素添加/删除一个类名时，一个自动补全的下拉菜单会自动提供页面上的所有现有类名。在 Firefox 101 中，当用户改变自动补全列表的选择时（使用上/下箭头键），自动补全下拉列表中选择的类名称会立即被应用。这对于快速测试各种样式特别有用。</p> </li> 
 <li> <p>检查器面板：这个新选项可以用来禁用 Rule View 中的 "拖动更新" 功能</p> <p><img alt height="266" src="https://static.oschina.net/uploads/space/2022/0601/072116_akTI_4937141.png" width="700" referrerpolicy="no-referrer"></p> </li> 
 <li> <p>WebDriver BiDi：该协议在稳定版上启用，以支持外部工具。WebDriver-BiDi 旨在为浏览器自动化提供一个跨浏览器协议，以满足现代 Web 应用测试工具的要求。这使得客户端和服务器都可以发送和接收请求和响应。</p> </li> 
</ul> 
<h3>Web 平台</h3> 
<ul> 
 <li>Firefox 新增加了对大、小、动态 Viewport 单元的支持。这使用户可以灵活地选择页面元素的大小是否符合 "最小"、“最大” 或 “动态” Viewport 尺寸。</li> 
</ul>
