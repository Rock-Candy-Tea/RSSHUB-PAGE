                                                                                            <p>Kodi 是由 XBMC 基金会开发的开源媒体播放器，原名 XBMC（最后一个以 XBMC 命名的版本是 13.2 Gotham，14.0 Helix 是第一个以 Kodi 命名的版本），Kodi 可以运行在多种操作系统和硬件平台。 它可以让用户播放本地或网络存储设备中的大多数视频、音乐、播客及各种常见数位媒体文件。</p> 
<p>该软件最初是为计划运行在 Xbox 上的，名称也由此而来。随后有了 Android、Linux、BSD、macOS、iOS 和 Windows 操作系统的原生版。</p> 
<p>时隔两个月，Kodi 团队日前发布了即将推出的 Kodi 20 "Nexus" 的另一个 Alpha 版本 —— Kodi "Nexus" Alpha 3，以下是本次更新值得关注的地方：</p> 
<h3>附加组件</h3> 
<ul> 
 <li> <p>在附加组件安装和存储库检查中的一些崩溃问题已经得到纠正</p> </li> 
 <li> <p>AlwinEsch 已经实现了一个大型功能，允许同一二进制插件的多个实例。</p> <p>例如，这允许一个 PVR 插件的两个实例使用不同的后端实例来运行，需要各种二进制插件的更新。</p> </li> 
</ul> 
<h3>数据库</h3> 
<ul> 
 <li>优化数据库封装器中的各种东西，以获得更好的性能</li> 
</ul> 
<h3>DVD</h3> 
<ul> 
 <li>修复 DRMPrime 的 DVD 菜单指标</li> 
</ul> 
<h3>文件系统</h3> 
<ul> 
 <li>在最近的一些 NFS 更新后，修复了 Windows 平台的 NFS</li> 
 <li>一些 NFS 的改进/清理</li> 
 <li>初步的 NFS4 实现</li> 
</ul> 
<h3>字体</h3> 
<ul> 
 <li>字体/字形处理代码进行了一些改进，尤其有助于低功率机器，但对所有平台都有帮助。</li> 
 <li>Harbuzz 字形缓存的改进</li> 
 <li>修复一些与 RTL 语言不一致的地方</li> 
</ul> 
<h3>游戏</h3> 
<ul> 
 <li>关于控制器的改进/清理</li> 
 <li>修复控制器信息文件没有正确信息时的崩溃</li> 
</ul> 
<h3>常规</h3> 
<ul> 
 <li>Rechi 做了大量的工作来实现静态分析工具的改进，如 CPPCheck 和 ClangTidy</li> 
 <li>从 Core Kodi 安装中删除过时的附加组件</li> 
 <li>修复 STRM 文件的播放器播放列表类型错误的问题</li> 
</ul> 
<h3>输入</h3> 
<ul> 
 <li>修复使用鼠标拖放某些控件的问题</li> 
</ul> 
<h3>特定平台</h3> 
<ul> 
 <li>Linux GBM 
  <ul> 
   <li>可以设置 HDR 输出</li> 
  </ul> </li> 
 <li>macOS 
  <ul> 
   <li>允许 arm64 (Apple Silicon)构建，以正确地从软件库中找到适当的二进制附加组件</li> 
  </ul> </li> 
 <li>Windows 
  <ul> 
   <li>修正窗口过渡到全屏和返回时不正确的窗口定位</li> 
  </ul> </li> 
</ul> 
<p>再次强调，这只是一个 Alpha 版本，在测试前请备份你的配置。</p>
