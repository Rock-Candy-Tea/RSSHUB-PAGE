<h3>弃用的 API</h3> 
<p>在核心 PMD 类中进行了一些 API 的废弃，以提高与 PMD 7 的兼容性。</p> 
<ul> 
 <li><code>report</code> ：构造函数和其他构造方法，如 addViolation 或 createReport</li> 
 <li><code>RuleContext</code> ：所有构造函数、getters 和 setters。在 PMD 7 中， <code>RuleContext</code> 将 是报告违规行为的 API</li> 
 <li>字段 <code>configuration</code> 是未使用的，将被删除</li> 
</ul> 
<h3>内部 API</h3> 
<p>这些 API 将在 PMD 7.0 中被隐藏或删除。</p> 
<ul> 
 <li><code>RuleSet</code> ：用于应用规则的方法，包括 <code>apply</code>、 <code>start</code>、 <code>end</code>、 <code>removeDysfunctionalRule</code></li> 
 <li><code>AbstractAccumulatingRenderer#renderFileReport</code> 是内部 API 而不应该在自己的渲染器中被重写</li> 
</ul>
