<div class="content">
                                                                                            <p>Chrome 98 现已发布，该版本的变化主要集中在开发者方面，但也有一些影响用户的差异。具体更新内容<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.phoronix.com%2Fscan.php%3Fpage%3Dnews_item%26px%3DChrome-98-Released" target="_blank">包括</a>：</p> 
<ul> 
 <li>COLRv1 彩色渐变矢量字体的支持是出于对网络上更小的表情符号文件的需求，同时提供更好的质量。COLRv1 字体压缩效果好，是基于矢量的，<span style="background-color:#ffffff; color:#121212">并且可以很好地处理渐变。</span>Google Chrome 和 Google Fonts 团队认为 COLRv1 规范是 Google Noto 表情符号字体的后续格式。表情符号字体的大小约为以前的 20%，同时还具有更高的渲染保真度。关于 COLRv1 的更多细节可见 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdeveloper.chrome.com%2Fblog%2Fcolrv1-fonts%2F" target="_blank">developer.chrome.com</a>。</li> 
</ul> 
<p><img alt height="185" src="https://oscimg.oschina.net/oscnet/up-1f3840cea0812c423158aa40fc49a3ecab5.jpg" width="300" referrerpolicy="no-referrer"> </p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#121212">Chrome 98 内置了网页截图功能。</span></li> 
 <li>一个新的 CSS 媒体查询，用于检测设备的 display HDR（High Dynamic Range）功能。新查询是“dynamic-range”。</li> 
 <li>使用新参数改进了 windows.open() 处理，以控制所需行为是打开新弹出窗口还是打开新选项卡/窗口。</li> 
 <li>承诺支持 ClipboardItem 对象中的 blob。</li> 
 <li>各种其他开发人员更改。</li> 
</ul> 
<p><span style="background-color:#ffffff; color:#121212">更多详细信息可查看 </span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromereleases.googleblog.com%2F" target="_blank">Chrome Release Blog</a><span style="background-color:#ffffff; color:#121212"><span> </span>和<span> </span></span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fchromestatus.com%2Ffeatures%23milestone%253D98" target="_blank">ChromeStatus.com</a>。</p>
