<div class="content">
                                                                    
                                                        <p><span style="background-color:#ffffff; color:#333333">Apache Storm 2.1.1 现已发布。Apache Storm 是一个分布式的、容错的、高性能的实时计算系统，为数据的处理提供了有力的保障。</span></p> 
<p><span style="background-color:#ffffff; color:#333333"><strong>主要更新内容</strong></span></p> 
<ul> 
 <li><span style="background-color:#ffffff; color:#333333">将日志级别更改为 info 以显示用于验证的配置类</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">修复错误的参数顺序</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Nimbus 日志充斥着 TTransportException 错误消息</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Storm CLI set_log_level 不再更新日志级别</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">如果未安排，拓扑 UI 页面将显示每个组件的总资源</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">拓扑 UI 页面 "更改日志级别" 不应允许空记录器名称</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">上传凭证破坏安全集群中的拓扑</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">在 storm.py 中使用 -c 提供的配置覆盖未传递给所有命令</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Storm UI 可视化抛出 NullPointerException</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">loadaware shuffle 可以使本地工人超载</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">AutoTGT 不应调用 TGT 续订线程（来自 UserGroupInformation.loginUserFromSubject）</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">为 ICredentialsListener spout/bolt 实例更新凭据时出现 ClassCastException</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">在提交拓扑时，storm.py 应该在类路径中包含 lib-worker 而不是 lib 目录</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Storm-kafka-migration 应该将 Storm-client 作为“提供的”依赖项引入</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">Logviewer 应始终允许管理员访问日志</span></li> 
 <li><span style="background-color:#ffffff; color:#333333">getComponentPendingProfileActions 上的 NPE</span></li> 
</ul> 
<p>详情请查看更新公告。</p>
