                                                                                            <p>Docker 20.10.13 现已发布。此版本的 Docker Engine 包含一些 bug-fixes 和 packaging change、对<code>docker scan</code>和<code>docker buildx</code>命令的更新、Go 运行时的更新版本以及<code>containerd.io</code>运行时的新版本。与此版本一起，现在还提供 Docker Compose V2 的<code>.deb</code>和<code>.rpm</code>软件包，可以使用（可选）<code>docker-compose-plugin</code>包进行安装。</p> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left">具体更新内容如下：</p> 
<h3><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Builder</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></h3> 
<ul> 
 <li>将 buildx 的捆绑版本更新为 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fbuildx%2Freleases%2Ftag%2Fv0.8.0" target="_blank">v0.8.0</a>。</li> 
</ul> 
<h3><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Daemon</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></h3> 
<ul> 
 <li>修复更新容器状态时的 race condition。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43166" target="_blank">moby/moby#43166</a></li> 
 <li>更新 etcd 依赖项以防止守护进程错误地持有文件锁。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43259" target="_blank">moby/moby#43259</a></li> 
 <li>修复配置默认的<code>net.ipv4.ping_group_range</code>sysctl 时对用户命名空间的检测。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43084" target="_blank">moby/moby#43084</a></li> 
</ul> 
<h3><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Distribution</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></h3> 
<ul> 
 <li>如果在下载图像的过程中发生连接失败，重试下载 image-manifests。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43333" target="_blank">moby/moby#43333</a></li> 
</ul> 
<h3><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Documentation</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></h3> 
<ul> 
 <li>命令行参考和 API 文档中的各种修复。</li> 
</ul> 
<h3><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Logging</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></h3> 
<ul> 
 <li>将“local”日志驱动程序与产生大量日志消息的容器一起使用时，防止出现 OOM 。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43165" target="_blank">moby/moby#43165</a></li> 
 <li>更新 fluentd 日志驱动程序以防止潜在的守护进程崩溃，并防止容器在使用<code>fluentd-async-connect=true</code>和远程服务器时不可达时挂起。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Fpull%2F43147" target="_blank">moby/moby#43147</a></li> 
</ul> 
<h3><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span>Packaging</span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></h3> 
<ul> 
 <li>为 Docker Compose V2 提供 .deb 和 .rpm 包。<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fcompose%2Freleases%2Ftag%2Fv2.3.3" target="_blank">Docker Compose v2.3.3</a> 现在可以使用 docker-compose-plugin 软件包在 Linux 上安装，它在 Docker CLI 上提供了 docker compose 子命令。Docker Compose 插件也可以安装并独立运行，以作为 docker-compose（Docker Compose V1）的直接替代品  <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fdocker-ce-packaging%2Fpull%2F638" target="_blank">ocker/docker-ce-packaging#638</a>。compos-cli-plugin 包也可以用在支持 CLI 插件的旧版 Docker CLI 上（Docker CLI 18.09 及以上）。</li> 
 <li>为即将推出的 Ubuntu 22.04 “Jammy Jellyfish” LTS 版本提供软件包 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fdocker-ce-packaging%2Fpull%2F645" target="_blank">docker/docker-ce-packaging#645</a>、<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fcontainerd-packaging%2Fpull%2F271" target="_blank">docker/containerd-packaging#271</a>。</li> 
 <li>更新<code>docker buildx</code>到 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fbuildx%2Freleases%2Ftag%2Fv0.8.0" target="_blank">v0.8.0</a>。</li> 
 <li>将<code>docker scan</code><span style="background-color:#ffffff; color:#24292f"><span> </span>(</span><code>docker-scan-plugin</code><span style="background-color:#ffffff; color:#24292f">)</span>更新为 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdocker%2Fscan-cli-plugin%2Freleases%2Ftag%2Fv0.17.0" target="_blank">v0.17.0</a>。</li> 
 <li>将 containerd（<code>containerd.io</code>包）更新到 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fcontainerd%2Fcontainerd%2Freleases%2Ftag%2Fv1.5.10" target="_blank">v1.5.10</a>。</li> 
 <li>将捆绑的 runc 版本更新为 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fopencontainers%2Frunc%2Freleases%2Ftag%2Fv1.0.3" target="_blank">v1.0.3</a>。</li> 
 <li>将 Golang 运行时更新为 Go 1.16.15。</li> 
</ul> 
<p>更新说明：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmoby%2Fmoby%2Freleases%2Ftag%2Fv20.10.13" target="_blank">https://github.com/moby/moby/releases/tag/v20.10.13</a></p>
