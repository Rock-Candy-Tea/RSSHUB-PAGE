                                                                                            <p>Sublime Text 4 Build 4113 正式发布，更新内容如下：</p> 
<ul> 
 <li>改进了编辑大文件时的性能；</li> 
 <li>改进了 OpenGL 的渲染性能；</li> 
 <li>改进了对已删除文件的处理；</li> 
 <li>各种语法高亮的改进；</li> 
 <li>subl 现在可以用来编辑 stdin，例如：echo test | subl | cat；</li> 
 <li>在编辑 stdin 时，现在可以进行语法和缩进检测；</li> 
 <li>增加了 syntax_detection_size_limit 设置，用于控制何时跳过语法检测；</li> 
 <li>主题：改进了 scroll puck 的可见性；</li> 
 <li>主题：修正了自适应主题在浅色方案中不遵循 themed_title_bar 设置的问题；</li> 
 <li>保留大小写现在适用于 unicode 字符；</li> 
 <li>增加了 reveal_menu 设置，用于在 Linux 和 Windows 上按下 alt 时禁止显示菜单；</li> 
 <li>可以通过在数据目录中创建一个名为 .Disable Safe Mode Shortcut 的文件来禁用安全模式键绑定；</li> 
 <li>修复了 Monokai 配色方案中的 Ruby 语法高亮显示的问题；</li> 
 <li>修复了未监视文件夹更改的情况；</li> 
 <li>修正了在高亮线后面画下划线的问题；</li> 
 <li>修复了在语法高亮过程中可能出现的无限循环；</li> 
 <li>修复了 append 命令的 scroll_to_end 参数有时不工作的问题；</li> 
 <li>修复了 Goto 符号有时会被错误地滚动；</li> 
 <li>修复了适用于侧边栏的多选文件限制的问题；</li> 
 <li>修复了一些大文件中与自动完成相关的挂起；</li> 
 <li>Linux：修复打印有时不起作用的问题；</li> 
 <li>Linux：修复了 GTK 对话框中 Yes/No 按钮的错误顺序；</li> 
 <li>Linux：修复字母有时会被切断的问题；</li> 
 <li>Windows：在 Windows 上启动主可执行文件时总是建立一个新窗口；</li> 
 <li>Windows：修正了在 Windows 上窗口图标不能正常缩放的问题；</li> 
 <li>Windows：修正了在 Windows 上某些情况下不能扩展的 globs；</li> 
</ul> 
<p>更多详情可查看：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.sublimetext.com%2Fdownload" target="_blank">https://www.sublimetext.com/download</a></p>
