                                                        <blockquote> 
 <p style="text-align:left">Jboot 一个更简单的分布式、微服务框架。</p> 
 <p style="text-align:left">Jboot是一个基于 JFinal、JFinal-Undertow、Dubbo、Seata、Sentinel、ShardingSphere、Nacos 等开发的微服务框架，帮助开发者降低微服务、分布式开发门槛。爽爽开发，快乐生活。</p> 
</blockquote> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">到目前为止，Jboot 已经开源超过了 5 年的时间，迭代了 200+ 个版本，已经被超过 1000+ 公司在使用，其中包含了多个知名的上市公司，我们了解到的多个使用 Jboot 开发的产品，用户量超过 1亿 以上。</span></p> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">Jboot 从 v3.9.15 开始带来了基于 junit4 和 junit5 的单元测试能力。到目前 v3.9.19 提供的单元测试能力如下，而这些功能，并未依赖其他任何第三方单元测试框架：</span></p> 
<ul> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">1、功能强大的 MockMvc 功能，可以用来测试 Controller、Interceptor、Handler 等</span></li> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">2、在测试类里进行 @Inject ，可以用来测试 Service、或者 RPC 等。</span></li> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">3、@MockMethod 注解的支持，可以用来 Mock 任何 AOP 方法。</span></li> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">4、@MockClass 注解的支持，可以用来 Mock 任何的 AOP 类。</span></li> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">5、自动 Mock 所以依赖接口等等...</span></li> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">6、当 Controller 发生 404 错误，或者 500 错误，自动让 Junit 测试失败</span></li> 
 <li style="text-align:left"><span style="background-color:#ffffff; color:#333333">7、等等等等</span></li> 
</ul> 
<p>凡是用过 Jboot 提供的单元测试功能都说，真的太好用了。 </p> 
<p>以下是一个简单的 Jboot 单元测试示例：</p> 
<pre><code>@RunWith(JbootRunner.class)
@TestConfig(autoMockInterface = true)
public class OptionApiControllerTest &#123;

    private static final MockMvc mvc = new MockMvc();

    @Test
    public void query() &#123;
        mvc.get("/api/option/query?key=myKey").printResult();
    &#125;



    @MockMethod(targetClass = UtmService.class)
    public void doRecord(Utm utm)&#123;
        System.out.println(">>>>>>>>>doRecord: " + utm);
    &#125;
    

&#125;</code></pre> 
<p>非常简单的几行代码、支持请求任何 URL 进行测试，自动对 UtmService.doRecord 方法进行 Mock，凡是遇到接口而没有实现类的进行自动 Mock... 总之：使用 Jboot 的单元测试，要比 Spring Stack 等其他任何框架都更加简单好用。</p> 
<p> </p> 
<p style="text-align:left"><strong>Jboot v3.9.19 更新内容如下（全是新增或优化）：</strong></p> 
<ul> 
 <li>新增：@MockClass() 注解，用于多 AOP 的 Class 进行 Mock</li> 
 <li>新增：MockMvc 新增 requestStartListener 和 requestFinishedListener 的支持</li> 
 <li>新增：@TestConfig.devMode 和 @TestConfig.launchArgs 配置的支持</li> 
 <li>新增：MockMvcResult.getContentAsJSONObject() 和 assertJson() 方法</li> 
 <li>新增：MockHttpServletRequest 新增 addQueryParameter() 方法</li> 
 <li>新增：@DefaultValue() 注解，用与对参数配置默认值</li> 
 <li>新增："jboot.sentinel.reqeustTargetPrefix" 配置的支持，用于配置 sentinel 对某些 url 进行拦截</li> 
 <li>优化：优化控制台 action 的日志输出的性能</li> 
 <li>优化：升级 JFinal 到最新版本</li> 
 <li>优化：通过 MockMvc 进行单元测试的时候，若 Controller 发生 404 或者 500 等错误的时候，则不通过 junit 测试</li> 
 <li>文档：添加验证器错误自定义渲染的相关文档</li> 
 <li>文档：同步 junit 测试文档</li> 
 <li>文档：修改 Sentinel 的里错误的配置文档</li> 
</ul>
