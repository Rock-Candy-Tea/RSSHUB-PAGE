 <p style="text-align:left">Jboot 一个更简单的分布式、微服务框架。</p> 
 <p style="text-align:left">Jboot是一个基于 JFinal、JFinal-Undertow、Dubbo、Seata、Sentinel、ShardingSphere、Nacos 等开发的微服务框架，帮助开发者降低微服务、分布式开发门槛。爽爽开发，快乐生活。</p> 
</blockquote> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">到目前为止，Jboot 已经开源超过了 5 年的时间，迭代了 190+ 个版本，已经被超过 1000+ 公司在使用，其中包含了多个知名的上市公司。</span></p> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">Jboot 在 v3.9.15  版本发布的时候，带来了针对 Junit4 和 junit5 的强大的单元测试能力。</span></p> 
<p style="text-align:left"><span style="background-color:#ffffff; color:#333333">junit4 单元测试代码如下：</span></p> 
<pre><code class="language-java">@RunWith(JbootRunner.class)
public class MyAppTester &#123;

    private static MockMvc mvc = new MockMvc();

    @Inject
    private MyService myService;

    @Test
    public void test_url_aaa() &#123;
        MockMvcResult mvcResult = mvc.get("/aaa");

        mvcResult.printResult()
                .assertThat(result -> Assert.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getStatus() == 200);
    &#125;

    @Test
    public void test_url_bbb() &#123;
        MockMvcResult mvcResult = mvc.get("/bbb");

        mvcResult.printResult()
                .assertThat(result -> Assert.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getStatus() == 200);
    &#125;

    @Test
    public void test_my_service() &#123;
        Ret ret = myService.doSomeThing();
        Assert.assertNotNull(ret);
        //.....
    &#125;
&#125;</code></pre> 
<p><span style="background-color:#ffffff; color:#333333">我们可以通过 MockMvc 可以对 JFinal（Jboot） 中的 Controller、Interceptor、Handler 等进行测试，无需启动 JFinal 服务。</span>同时可以直接通过 @Inject 或者 @RPCInject 把 Service 注入到 MyAppTester 里，直接对 service 进行测试。</p> 
<p style="text-align:left">在 junit5 中，代码如下：</p> 
<pre><code class="language-java">@ExtendWith(JbootExtension.class)
public class MyAppTester &#123;

    private static MockMvc mvc = new MockMvc();

    @Inject
    private MyService myService;

    @Test
    public void test_url_aaa() &#123;
        MockMvcResult mvcResult = mvc.get("/aaa");

        mvcResult.printResult()
                .assertThat(result -> Assertions.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getStatus() == 200);
    &#125;

    @Test
    public void test_url_bbb() &#123;
        MockMvcResult mvcResult = mvc.get("/bbb");

        mvcResult.printResult()
                .assertThat(result -> Assertions.assertNotNull(result.getContent()))
                .assertTrue(result -> result.getStatus() == 200);
    &#125;

    @Test
    public void test_my_service() &#123;
        Ret ret = myService.doSomeThing();
        Assertions.assertNotNull(ret);
        //.....
    &#125;
&#125;</code></pre> 
<p><span style="background-color:#ffffff; color:#333333">Junit4 和 junit5 主要的不同点是 MyApTester 类的注解配置不同而已。Junit4 使用 @RunWith(JbootRunner.class) 配置，而 Junit5 使用 @ExtendWith(JbootExtension.class) 配置。</span></p> 
<p>在 v3.9.15 中，虽然提供了 <span style="background-color:#ffffff; color:#333333">@RunWith(JbootRunner.class) 和 @ExtendWith(JbootExtension.class)，同时 MockMvc 测试类，对 Controller 进行测试。但是还有一些不足的地方，比如 在 Controller 中调用了某个 Service 的方法，而该 Service 可能会依赖很多外部资源，比如 redis、mysql、磁盘文件等等。又或者在多 Maven 模块的情况，我们引用了 Service 接口，却没有引用实现类。此时可能会导致测试异常。</span></p> 
<p><span style="background-color:#ffffff; color:#333333">因为 Jboot 在不赖其他任何第三方测试框架上，提供了注解 @MockMethod 对 Service 方法进行 Mock 的能力。例如：</span></p> 
<pre><code class="language-java">@RunWith(JbootRunner.class)
@TestConfig(autoMockInterface = true)
public class OptionApiControllerTest &#123;

    private static final MockMvc mvc = new MockMvc();

    @Test
    public void query() &#123;
        mvc.get("/api/option/query?key=myKey").printResult();
    &#125;


    @MockMethod(targetClass = JPressCoreInitializer.class)
    public void onHandlerConfig(JfinalHandlers handlers) &#123;
        handlers.add(new JPressHandler());
    &#125;
    

    @MockMethod(targetClass = UtmService.class)
    public void doRecord(Utm utm)&#123;
        System.out.println(">>>>>>>>>doRecord: " + utm);
    &#125;
    

    @MockMethod(targetClass = WebInitializer.class,targetMethod = "onEngineConfig")
    public void mock_on_engine_config(Engine engine)&#123;
        System.out.println(">>>>>>>>>onEngineConfig: " + engine);
    &#125;
&#125;</code></pre> 
<p style="text-align:start"><span style="color:#000000">在以上的代码中，有几个关键的地方：</span></p> 
<ul> 
 <li><code>@TestConfig(autoMockInterface = true)</code>，表示当我们测试 <code>query()</code> 方法的时候，可能会遇到一些注入进来的接口，但是可能没有实现类（或者说实现类在别的 Maven Module，并没有依赖进来），但是保证不出错。 当调用接口方法时，等同于什么都不做，若有返回值，则返回 <code>null</code>。</li> 
 <li><code>@MockMethod(targetClass = JPressCoreInitializer.class)</code> 表示复写 <code>JPressCoreInitializer</code> 类的 <code>onHandlerConfig</code> 方法。</li> 
 <li><code>@MockMethod(targetClass = UtmService.class)</code> 表示复写 <code>UtmService</code> 类的 <code>doRecord</code> 方法。</li> 
 <li><code>@MockMethod(targetClass = WebInitializer.class,targetMethod = "onEngineConfig")</code> 表示复写 <code>WebInitializer</code> 类的 <code>onEngineConfig</code> 方法。</li> 
</ul> 
<p>只能说，对于使用 Jboot 的用户真的是太幸福了~~~</p> 
<p style="text-align:left"><strong>Jboot v3.9.18 更新内容如下：</strong></p> 
<ul> 
 <li>新增：新增 @MockMethod 注解，方便对 AOP 方法进行 Mock</li> 
 <li>新增：@TestConfig(autoMockInterface=false) 配置，方便对接口进行 Mock 操作</li> 
 <li>修复：Motan RPC 框架的 protocol 配置不生效的问题</li> 
 <li>修复：ide 配置错误时给出的 JFinal 配置帮助文档网址错误</li> 
 <li>修复：JbootActionReporter 可能出现 NotFoundException 的问题</li> 
 <li>修复：JbootActionReporter 可能出现 NPE 的问题</li> 
 <li>修复：阿里云商业 MQ Aliyunmq 配置错误的问题，感谢 @不器</li> 
 <li>修复：Junit 测试对于个别 ServletRequest 方法没有 mock 到而出错的问题</li> 
 <li>修复：当 Jboot 有上层 session 时（比如使用 shiro），修改 Controller session 无法同步上层 session 的问题</li> 
 <li>文档：修改某些描述错误的文档</li> 
</ul>
