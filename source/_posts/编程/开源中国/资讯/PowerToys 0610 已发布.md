                                                                                            <p>Powertoys 0.61.0 已发布，本期主要带来 Always on Top、FancyZones 和 PowerToys Run 的质量改进。</p> 
<h3><strong>已知的问题</strong></h3> 
<ul> 
 <li>安装 PowerToys 后，<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmicrosoft%2FPowerToys%2Fissues%2F19124" target="_blank">PowerRename 和 Image Resizer 的新 Windows 11 上下文菜单条目不会在系统重新启动之前出现</a>。</li> 
 <li>有用户<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fmicrosoft%2FPowerToys%2Fissues%2F18015" target="_blank">无法打开“设置”窗口</a>的报告。这是由于与某些应用程序不兼容造成的（已知示例：RTSS RivaTuner Statistics Server 和 MSI AfterBurner）。</li> 
</ul> 
<h3><strong>一般更新</strong></h3> 
<ul> 
 <li>将 Windows 应用 SDK 运行时升级到 1.1.2。</li> 
 <li>新的 Windows 11 上下文菜单条目现在已正确添加到 Windows 11 开发通道内部版本中。</li> 
 <li>旧的上下文菜单条目与新的 Windows 11 上下文菜单条目一起显示，以与覆盖 Windows 11 上下文菜单行为的软件兼容。</li> 
 <li>跨解决方案的整合 C# 语言版本。</li> 
 <li>删除了已弃用的 Segoe 图标字形代码，并将其替换为正确的代码。</li> 
 <li>修复了在启用某些模块时导致在某些键盘布局上按下随机重音键的问题。</li> 
</ul> 
<h3>Always on Top</h3> 
<ul> 
 <li>修复了激活时边框闪烁的问题。</li> 
 <li>修复了退出 PowerToys 时导致 Always on Top 激活并挂起的错误。</li> 
 <li>修复了出现在圆角上的黑色边缘。</li> 
 <li>修复了导致 100% CPU 消耗的错误。</li> 
</ul> 
<h3>FancyZones</h3> 
<ul> 
 <li>修复了当多个显示器报告具有相同序列号时，导致无法正确应用布局的错误。</li> 
 <li>修复了导致布局无法在虚拟监视器设置上正确应用的错误。</li> 
 <li>“行”取代了“列”布局，成为垂直监视器的默认布局。</li> 
</ul>
