                                                                                            <p>GNOME 42 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fmail.gnome.org%2Farchives%2Fdevel-announce-list%2F2022-January%2Fmsg00004.html" target="_blank">发布</a>了首个 Alpha 版本。据称此版本的不少组件已从 GTK 3 移植到 GTK 4，并且用上了 libadwaita，此外还开始支持暗黑模式/暗黑偏好选项，以及其他多项改进。</p> 
<p>主要变化：</p> 
<ul> 
 <li>Baobab 磁盘使用情况分析器已被移植到 GTK4 和 libadwaita</li> 
 <li>GNOME Calls 现支持在呼叫历史和呼叫界面中显示头像</li> 
 <li>Epiphany 网络浏览器现在支持系统的黑暗偏好设置，更新了其捆绑的 PDF.js，修复了内存泄漏问题，现在默认启用其加速合成模式，以及其他改进。此外 Epiphany 还为迁移到 GTK4 做了准备</li> 
 <li>GDM 登录管理器有针对 NVIDIA Wayland 支持的 udev 更新</li> 
 <li>GNOME 蓝牙应用已被移植到 GTK4</li> 
 <li>用于虚拟化的 GNOME Boxes 引入了一个新的首选项对话框，将一些部件移植到 libhandy，以及其他改进</li> 
 <li>GNOME Builder 为这个集成开发环境增加了一个 BuildStream 插件、Rubucop 支持、实验性的 Clangd 插件、Clang-Format 集成、语言服务器协议处理的改进、对 Intelphense 和 Blueprint 语言服务器的支持，以及其他各种改进</li> 
 <li> <p>GNOME 计算器增加了对 Flatpak 开发构建的支持，迁移到 GTK4，以及其他改进</p> </li> 
 <li> <p>GNOME 时钟已移植到 GTK4 和 libadwaita</p> </li> 
 <li> <p>GNOME 桌面将 libgnome-desktop 拆分为 gnome-desktop、gnome-rr 和 gnome-bg 组件</p> </li> 
 <li> <p>GNOME 磁盘工具现在支持系统的暗黑主题偏好设置，为 GTK4 做好准备，以及其他更新</p> </li> 
 <li> <p>对 GNOME 地图的各种改进</p> </li> 
 <li> <p>GNOME Software 更新了各种用户界面元素，提供了更多信息的系统更新进度，允许搜索扩展，添加了新的 GNOME Circle 应用程序，改进了对 Snaps 的元数据支持，以及其他改进</p> </li> 
 <li> <p>Nautilus 文件管理器改进了文件重命名用户界面的设计，改进了文件冲突用户界面的视觉效果，支持按文件创建时间搜索，为最近的文件添加压缩档案，并移植到 GTK4</p> </li> 
 <li> <p>Totem 媒体播放器增加了对 MPL 字幕的支持</p> </li> 
 <li> <p>有关 GNOME Shell 和 Mutter 的改进</p> </li> 
</ul> 
<p>GNOME 42 将在今年春天发布的 Fedora Workstation 36 中使用。Ubuntu 22.04 LTS 也计划采用不依赖于 GTK4 的 GNOME 42 组件。</p>
