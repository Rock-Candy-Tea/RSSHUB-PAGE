                                                                                            <p>Deno 是一个简单、现代和安全的 JavaScript 和 TypeScript 的运行时，它使用 V8 并以 Rust 构建。</p> 
<p>Deno 1.24.3 已发布，值得关注的更新包括：</p> 
<ul> 
 <li>fix(ext/fetch): 添加 socks 代理支持 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15372" target="_blank">#15372</a>)</li> 
 <li>feat(unstable/ext/ffi): 为 Deno.UnsafePointerView 添加静态方法变体 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15146" target="_blank">#15146</a>)</li> 
 <li>fix(cli): 允许配置文件也成为 json 模块 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15444" target="_blank">#15444</a>)</li> 
 <li>fix(ext/ffi): 读取时检查 CStr 的 UTF-8 有效性 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15318" target="_blank">#15318</a>)</li> 
 <li>fix(ext/ffi): 不稳定的 op_ffi_unsafe_callback_ref (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15439" target="_blank">#15439</a>)</li> 
 <li>fix(permissions): 忽略空值 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15447" target="_blank">#15447</a>)</li> 
 <li>fix(task): 子命令解析器跳过全局参数 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15297" target="_blank">#15297</a>)</li> 
 <li>fix: 当没有提供 <code>-location</code> 时，允许设置 <code>globalThis.location</code> (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15448" target="_blank">#15448</a>)</li> 
 <li>fix: 更新 <code>deno_graph</code>，以修复导入配置为 JSON 模块的问题 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15388" target="_blank">#15388</a>)</li> 
 <li>fix: 各种格式化的修复 (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fdenoland%2Fdeno%2Fpull%2F15412" target="_blank">#15412</a>)</li> 
</ul>
