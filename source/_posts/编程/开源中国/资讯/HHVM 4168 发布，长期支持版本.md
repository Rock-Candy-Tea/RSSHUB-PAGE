                                                        <p><span style="color:#393939">HHVM 4.168 (LTS) 现已发布，此版本具有长期支持，因此：</span></p> 
<ul> 
 <li>此版本将支持大约 48 周</li> 
 <li>下一个 LTS 将在大约 24 周内发布</li> 
 <li>HHVM 4.153 (LTS) 和 HHVM 4.165–167 仍然受支持</li> 
 <li>对 4.128 (LTS) 和 4.164 的支持已结束</li> 
</ul> 
<p style="margin-left:0px; margin-right:0px; text-align:left"><span style="color:#000000"><strong>Ubuntu 22.04 支持状态</strong></span></p> 
<p style="margin-left:0; margin-right:0; text-align:left"><span style="color:#000000">此版本原计划提供 Ubuntu 22.04 支持，但最终并没有实现。公告指出，“由于库的版本兼容性问题，我们需要更多的时间来获得这种支持，并且我们不想推迟其他平台的发布。一旦我们在 22.04 上运行构建，就立即发布支持 22.04 的 4.168.1 版本。”</span></p> 
<p style="margin-left:0; margin-right:0; text-align:left"><span style="color:#000000">值得注意的是，其实验性通用包（也可用于版本 4.168）不受此问题的影响，应该可以在任何版本的 Ubuntu 上使用。</span></p> 
<p style="margin-left:0; margin-right:0; text-align:left"><span style="color:#000000"><strong>Highlights</strong></span></p> 
<ul> 
 <li>修复了类型检查器错误，其中递归包含自身的枚举没有被正确拒绝。（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2F6c95bdbe3b" target="_blank">commit</a>）</li> 
 <li>修复了一个错误，当由于 lambda 主体中发生的潜在 shadowing 写入而导致该变量实际上没有被隐藏时，lint 可能会虚假地报告该变量被隐藏。（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2F21afafcb3f" target="_blank">commit</a>）</li> 
 <li>在查找无效枚举类标签的“did you mean”建议时，忽略大小写并且不考虑类名。(<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2F3a3a5a5bb5" target="_blank">commit 1</a> ) (<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2Fbebe707323" target="_blank">commit 2</a> )</li> 
 <li>改进类型检查器错误中未解决类型的 printing；例如 <code class="language-plaintext">Expected Awaitable<[unresolved]> But got int</code>变成<code class="language-plaintext">Expected Awaitable<_> But got int</code> （<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2Fafac86e51e" target="_blank">commit</a>）</li> 
 <li>在访问具有不是文字或类常量的值的 shape 时改进错误消息。（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2Fc2b2433f90" target="_blank">commit</a>）</li> 
 <li>改进来自 linter 的错误消息，指出函数中的所有 return 语句何时返回相同的值。( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2Fd606fb0198" target="_blank">commit</a> ) 还要让这个 linter 停止触发可能返回 unix 退出代码的函数。（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Ffacebook%2Fhhvm%2Fcommit%2Fb10078ba6c" target="_blank">commit</a>）</li> 
</ul>
