                                                                                            <p>Apollo 1.9.0 已发布，此版本包含以下功能提升和问题修复：</p> 
<ul> 
 <li> <p>增加 spring configuration metadata</p> </li> 
 <li> <p>支持 consul 作为服务发现组件</p> </li> 
 <li> <p>支持通过 ApolloInjectorCustomizer 扩展 apollo-client 中的实现类</p> </li> 
 <li> <p>支持 Spring Boot config data loader</p> </li> 
 <li> <p>支持基于 db/redis 实现 apollo-portal 多实例维持登录 session</p> </li> 
 <li> <p>非 properties 格式 namespace 展示更改历史</p> </li> 
 <li> <p>公共 namespace 支持非 properties 格式</p> </li> 
 <li> <p>支持基于配置项的 key 查询更改历史</p> </li> 
 <li> <p>...</p> </li> 
</ul> 
<p>更多内容查看<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fapolloconfig%2Fapollo%2Freleases%2Ftag%2Fv1.9.0" target="_blank"> release note</a>。</p> 
<p><strong>关于 Apollo</strong></p> 
<p>Apollo 项目于 2016 年在携程框架研发部诞生，初衷是为了解决公司内部配置管理尤其是中间件公共配置的管理难题，秉持着开源开放的精神，项目从第一行代码开始就在 GitHub 上开源，可以说是一个完全开放的项目。</p> 
<p>经过多年的发展，Apollo 以其功能丰富、简单易用等特性，得到了社区开发者的欢迎，目前在 Github 上有 25k star，9k fork 以及 80+ contributor，也已在数百家公司中得到广泛使用。</p>
