 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.ruby-lang.org%2Fen%2Fnews%2F2021%2F04%2F05%2Ftempfile-path-traversal-on-windows-cve-2021-28966%2F" target="_blank">CVE-2021-28966：Windows 上的 Tempfile 存在 Path traversal 问题</a></li> 
</ul>

<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.ruby-lang.org%2Fen%2Fnews%2F2021%2F04%2F05%2Fxml-round-trip-vulnerability-in-rexml-cve-2021-28965%2F" target="_blank">CVE-2021-28965：REXML 中的 XML 往返漏洞 (round-trip vulnerability)</a></li>

<h3>Ruby 2.6.7</h3> 
<p>此版本包括以下安全修复：</p> 
<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.ruby-lang.org%2Fen%2Fnews%2F2020%2F09%2F29%2Fhttp-request-smuggling-cve-2020-25613%2F" target="_blank">CVE-2020-25613：WEBrick 中潜在的 HTTP 请求夹带攻击 (HTTP Request Smuggling Vulnerability)</a></li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.ruby-lang.org%2Fen%2Fnews%2F2021%2F04%2F05%2Fxml-round-trip-vulnerability-in-rexml-cve-2021-28965%2F" target="_blank">CVE-2021-28965：REXML 中的 XML 往返漏洞 (round-trip vulnerability)</a></li> 
</ul> 
<p>更多细节查看 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fruby%2Fruby%2Fcompare%2Fv2_6_6...v2_6_7" target="_blank">commit logs</a>。</p> 
<p>此外，Ruby 团队宣布这个版本发布后，他们结束了 Ruby 2.6.x 的正常维护周期，因此 Ruby 2.6 进入了安全维护阶段。这意味着他们将不再将除安全修复程序之外的任何错误修复程序移植到 Ruby 2.6。安全维护阶段的期限计划为一年。在安全维护阶段结束时，Ruby 2.6 将正式 EOL，其官方支持也随之结束。因此，Ruby 团队建议使用 Ruby 2.6 的开发者开始计划升级到 Ruby 2.7 或 3.0。</p> 
<h3>Ruby 2.5.9</h3> 
<p>此版本包括以下安全修复：</p> 
<ul> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.ruby-lang.org%2Fen%2Fnews%2F2020%2F09%2F29%2Fhttp-request-smuggling-cve-2020-25613%2F" target="_blank">CVE-2020-25613：WEBrick 中潜在的 HTTP 请求夹带攻击 (HTTP Request Smuggling Vulnerability)</a></li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.ruby-lang.org%2Fen%2Fnews%2F2021%2F04%2F05%2Fxml-round-trip-vulnerability-in-rexml-cve-2021-28965%2F" target="_blank">CVE-2021-28965：REXML 中的 XML 往返漏洞 (round-trip vulnerability)</a></li> 
</ul> 
<p>更多细节查看 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fruby%2Fruby%2Fcompare%2Fv2_5_8...v2_5_9" target="_blank">commit logs</a>。</p> 
<p>值得注意的是，Ruby 2.5.9 发布后，Ruby 2.5 正式 EOL。换句话说，这是 Ruby 2.5 系列的最后一个版本。因此即使发现安全漏洞，Ruby 团队也不会再发布 Ruby 2.5.10。他们建议所有 Ruby 2.5 用户立即升级到 Ruby 3.0、2.7 或 2.6。</p>
