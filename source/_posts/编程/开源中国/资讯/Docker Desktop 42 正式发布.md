<div class="content">
                                                                                            <p><span style="background-color:#ffffff; color:#333333">Docker Desktop 4.2 现已发布。Docker 是一套平台即服务 (PaaS) 产品，它使用操作系统级别的虚拟化，以称为容器的包的形式交付软件。容器之间相互隔离，并捆绑自己的软件、库和配置文件；它们可以通过定义良好的渠道相互通信，因为所有的容器共享一个操作系统内核的服务，所以它们使用的资源比虚拟机少。</span></p> 
<h4><span style="background-color:#ffffff; color:#333333">使用暂停/恢复节省电池电量</span></h4> 
<p><span style="background-color:#ffffff; color:#333333">暂停/恢复为开发者提供了暂停当前 Docker Desktop 会话并随时恢复工作的能力，在 Docker 暂停时，容器的当前状态被保存在内存中，所有进程被冻结。这降低了CPU的使用率，并将有助于节省笔记本电脑的电池寿命。要恢复 Docker Desktop，可以点击菜单中的 "恢复 "按钮，或者在终端输入任何 Docker CLI 命令。</span></p> 
<p><span style="background-color:#ffffff; color:#333333">不过，暂停/恢复目前在Windows容器模式下不可用。</span></p> 
<p><span style="background-color:#ffffff; color:#333333"><img alt height="913" src="https://oscimg.oschina.net/oscnet/up-7c7bb11818ae4f2927ae4ddf771ab61a74d.png" width="1536" referrerpolicy="no-referrer"></span></p> 
<h4><span style="background-color:#ffffff; color:#333333">告别更新弹出窗口</span></h4> 
<p><span style="background-color:#ffffff; color:#333333">该版本取消了更新弹出窗口，并在 Docker 仪表板中引入了一个新的更新设置部分，在那里可以检查更新并管理更新偏好。</span></p> 
<p><span style="background-color:#ffffff; color:#333333"><img alt height="867" src="https://oscimg.oschina.net/oscnet/up-55883ff8222df6e6916652eacd3b87e39b4.png" width="1536" referrerpolicy="no-referrer"></span></p> 
<h4><span style="background-color:#ffffff; color:#333333">"自动检查更新" 设置现在可用于所有 Docker 订阅层</span></h4> 
<p><span style="background-color:#ffffff; color:#333333">在 Docker Desktop 4.2 中，用户（无论订阅级别如何）可以关闭自动检查更新。当禁用此设置时，鲸鱼菜单和应用程序中的所有通知都将被禁用，必须手动检查更新。</span></p>
