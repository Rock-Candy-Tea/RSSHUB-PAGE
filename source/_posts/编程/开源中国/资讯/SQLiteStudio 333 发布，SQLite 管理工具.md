                                                        <p>SQLiteStudio 3.3.3 版本现已<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fsqlitestudio.pl%2Fnews%2F" target="_blank">发布</a>。SQLiteStudio 是一个跨平台的 SQLite 数据库的管理工具。</p> 
<p>该版本的一些更新亮点内容如下：</p> 
<ul> 
 <li>CHANGE：＃4011 SQLite 更新至 3.35.4、SQLite3MultipleCiphers 已更新为 1.2.4（SQLite 3.35.4）、SQLCipher 更新到 4.4.3。</li> 
 <li>BUGFIX：＃4044 SQLite 链接已还原到动态模型（单独的 so/dll/dylib 文件），因此用户可以随时用最新版本替换 SQLite 库。</li> 
 <li>BUGFIX：＃4017 新的 Table/View 按钮现在始终处于启用状态，因为最近“Table/View”窗口有了自己的数据库下拉列表。</li> 
 <li>BUGFIX：＃4012 修复了使用 Fusion Dark 主题时的导入/导出向导窗口。</li> 
 <li>BUGFIX：＃4010 修复了 EXPLAIN 查询的结果列大小。</li> 
 <li>BUGFIX：＃4059 解决了在 SQL 查询中使用空字符串时崩溃的问题。</li> 
 <li>BUGFIX：＃4034 修复了表格窗口中默认值列的呈现。</li> 
 <li>BUGFIX：＃4035 修复了表达式结果列的完整值加载。还修复了使用别名视图作为数据源的查询的智能执行程序。</li> 
 <li>BUGFIX：＃4043 可移植目录功能在某些先前的 3.3.x 版本中被破坏后已得到修复。</li> 
 <li>BUGFIX：＃4049 修复了在启动（并禁止）辅助实例时将应用程序的主要实例置于最前面的问题。</li> 
 <li>BUGFIX：＃4060 SQL Export 插件现在从 INSERT 语句中排除了 GENERATED 列。</li> 
</ul>
