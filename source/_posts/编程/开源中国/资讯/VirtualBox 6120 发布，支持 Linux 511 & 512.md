                                                        <p>近日，VirtualBox 发布了 6.1.20 版本，也是该软件 6.1 版本系列的第十次更新。该版本增加了对 Linux Kernel 5.11 和 5.12 的支持。而且，对于 Linux Kernel 4.10 以上的纯主机适配器，最大 MTU 大小已经增加到 16110。</p> 
<p>VirtualBox 6.1.20 的其他变化包括：</p> 
<ul> 
 <li>修正了根据各种操作的时间而导致的极差的虚拟机性能；</li> 
 <li>修正了在某些情况下，当 Hyper-V 出现时，客户操作系统挂起的问题；</li> 
 <li>修正了在某些情况下使用嵌套管理程序时的 Guru Meditation 错误；</li> 
 <li>修正了一个与 SMAP 相关的 host panic 问题，影响到使用英特尔 Haswell CPU 或更高版本的 Solaris 11.4 系统；</li> 
 <li>增加对导出到 OCI 和创建 OCI 实例的 cloud-init 支持；</li> 
 <li>修正了 "删除所有文件" 留下的 Logs/VBoxUI.log；</li> 
 <li>多个音频修复和增强；</li> 
 <li>修正了 macOS 上 duplex 音频设备的检测；</li> 
 <li>修正了 "未连接" 适配器的链接状态报告；</li> 
 <li>修正了 OS/2 guests 中 e1000 的连接问题；</li> 
 <li>修正了 VxWorks e1000 驱动的兼容性问题；</li> 
 <li>修正了 GUI 对端口转发规则的检查，即用 "Nat Network" 拒绝 IPv6；</li> 
 <li>修正了在断开模式下使用串行端口时可能出现的虚拟机挂起；</li> 
 <li>修正了与 v4l2loopback 的互操作性，并修正了某些情况下网络摄像头的崩溃；</li> 
 <li>修正了高 CPU 负载下不时出现的 Windows 虚拟机挂起或重启；</li> 
 <li>允许用 "modifyvm" 改变已保存的虚拟机的网络适配器附件；</li> 
 <li>修复了参数处理问题，以尊重"-root"选项；</li> 
</ul> 
<p>更多详情可查看：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.virtualbox.org%2Fwiki%2FChangelog" target="_blank">https://www.virtualbox.org/wiki/Changelog</a></p>
