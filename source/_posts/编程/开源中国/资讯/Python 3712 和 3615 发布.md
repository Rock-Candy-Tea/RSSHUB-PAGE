                                                                                            <p>适用于 Python 3.7 和 Python 3.6 的最新安全修复汇总包 Python <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.python.org%2Fdownloads%2Frelease%2Fpython-3712%2F" target="_blank">3.7.12</a> 和 <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.python.org%2Fdownloads%2Frelease%2Fpython-3615%2F" target="_blank">3.6.15</a> 已发布。</p> 
<p>Python 3.7 现在处于其生命周期的安全修复阶段。在此阶段，仅接受和解决与安全相关的问题。Python 3.7 会提供安全修复程序直至 2023 年中期，即最初发布后的五年。安全修复版本会根据需要定期生成，并且仅提供源代码 —— Python 通常不为安全修复发行版提供二进制安装程序。</p> 
<p>此次更新主要修复的安全问题如下：</p> 
<p><strong>Security</strong></p> 
<ul> 
 <li> <p><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fbugs.python.org%2Fissue44394" target="_blank">bpo-44394</a>：将 libexpat 从 2.2.8 升级至 2.4.1，修复了 CVE-2013-0340 “Billion Laughs” 漏洞，libexpat 常在 Windows 和 macOS 上使用。</p> </li> 
 <li> <p><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fbugs.python.org%2Fissue43124" target="_blank">bpo-43124</a>：使<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fdocs.python.org%2Frelease%2F3.6.15%2Flibrary%2Fsmtplib.html%23module-smtplib" target="_blank"><code>smtplib</code></a>的内部 putcmd 函数在输入时对存在的<code>\r</code> 和<code>\n</code>字符进行安全处理，以避免（不太可能的）命令注入。</p> </li> 
</ul> 
<p><strong>Library</strong></p> 
<ul> 
 <li> <p><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fbugs.python.org%2Fissue45001" target="_blank">bpo-45001</a>：使电子邮件日期解析对格式错误的输入（即仅包含空格的<code>Date:</code>标题）更加稳健。</p> </li> 
</ul> 
<p>值得注意的是，Python 3.9 现在是 Python 3 的最新功能版本系列，官方建议尽快升级，点此获取<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fwww.python.org%2Fdownloads%2F" target="_blank"> 3.9.x </a>的最新版本。</p>
