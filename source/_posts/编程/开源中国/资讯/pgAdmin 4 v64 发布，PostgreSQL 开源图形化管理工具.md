                                                                                            <p style="color:#000000; margin-left:0; margin-right:0; text-align:start"><span style="color:#000000">pgAdmin 是 PostgreSQL 领先的开源图形化管理工具，，提供强大的图形界面，简化了数据库对象的创建、维护和使用。</span></p> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start">pgAdmin 4 v6.4 发布了，此版本的 pgAdmin 4 包括 13 个错误修复和新功能，内容如下：</p> 
<h3 style="margin-left:.6em; margin-right:0; text-align:start">新特性</h3> 
<ul style="margin-left:0; margin-right:0"> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F4803" target="_blank">#4803</a><span> </span>- 添加了对从 GUI 导入/导出服务器组和服务器的支持。</li> 
</ul> 
<p style="color:#000000; margin-left:0; margin-right:0; text-align:start"><span style="color:#0d0a0b">此功能允许用户将服务器定义（及其组）导出到 JSON 文件，然后将它们重新导入相同或不同的系统，以便轻松预配置 pgAdmin。</span></p> 
<h3 style="margin-left:.6em; margin-right:0; text-align:start">优化</h3> 
<ul style="margin-left:0; margin-right:0"> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7018" target="_blank"> #7018</a><span> </span>- React 端口还原对话框。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7019" target="_blank"> #7019</a><span> </span>- React 端口维护对话框。</li> 
</ul> 
<h3 style="margin-left:.6em; margin-right:0; text-align:start">Bug 修复 </h3> 
<ul style="margin-left:0; margin-right:0"> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F6745" target="_blank">#6745</a><span> </span>- 修复了在对话框中显示错误但创建表空间的问题。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7003" target="_blank">#7003</a><span> </span>- 修复了解释分析显示的独占时间为负数的问题。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7034" target="_blank">#7034</a><span> </span>- 修复了添加新行时不显示默认值的列的问题。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7075" target="_blank">#7075</a><span> </span>- 确保程序的帮助应正确可见。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7077" target="_blank">#7077</a><span> </span>- 修复了程序的逆向工程 SQL 中不显示所有者的问题。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7078" target="_blank">#7078</a><span> </span>- 修复了<span style="color:#0d0a0b">弹出的操作错误消息显示的数据库对象名称不正确的问题。</span></li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7081" target="_blank">#7081</a><span> </span>- 修复了 PostgreSQL-14 函数的 SQL 生成问题。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7093" target="_blank">#7093</a><span> </span>- 修复了子计划可能与其他节点重叠，导致其在图形解释视图中无法访问的问题。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7096" target="_blank">#7096</a><span> </span>- 确保截断和重置统计信息应该起作用。</li> 
 <li><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fredmine.postgresql.org%2Fissues%2F7102" target="_blank">#7102</a><span> </span>- 修复了模式差异（<span style="color:#212529">schema diff</span>）问题，生成的脚本为函数/过程/触发函数添加了不需要的行结尾。</li> 
</ul>
