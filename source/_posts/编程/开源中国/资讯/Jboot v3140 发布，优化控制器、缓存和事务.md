 <p style="margin-left:0; margin-right:0; text-align:left">Jboot 一个更简单的分布式、微服务框架。</p> 
 <p style="margin-left:0; margin-right:0; text-align:left">Jboot是一个基于 JFinal、JFinal-Undertow、Dubbo、Seata、Sentinel、ShardingSphere、Nacos 等开发的微服务框架，帮助开发者降低微服务、分布式开发门槛。爽爽开发，快乐生活。</p> 
 <p style="margin-left:0; margin-right:0; text-align:left">支持 JDK8~JDK17。</p> 
</blockquote> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left"><span>Jboot v3.14.0 主要是新增了 Controller 返回值自动匹配渲染器、Controller 缓存（@Cacheable）、@</span>Transactional() 事务注解等的支持，并优化多个细节。</p> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left"><strong>Controller 缓存实例：</strong></p> 
<pre><code class="language-java">@Cacheable(name = "aaa", liveSeconds = 10)
public void json() &#123;
    
    System.out.println("json() invoked!!!!!!!!!");
    
    Map<String, Object> data = new HashMap<>();
    data.put("age", 1);
    data.put("name", "张三");
    data.put("sex", 1);
    
    renderJson(data);
&#125;</code></pre> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left">以上的代码中，在 10 秒钟内多次访问的时候，只有第一次访问到了 json() 方法，其余的都直接使用第一次的结果输出给浏览器。其中，缓存的名称为 "aaa"，key 为<span> </span><code>class.method()</code><span> </span>格式。我们可以通过<span> </span><code>@Cacheable(name = "aaa", key="xxx")</code><span> </span>来指定缓存的key值。</p> 
<p style="color:#242424; text-align:start">其中 key 里的内容还可以使用<span> </span><code>#(方法参数名称)</code><span> </span>和<span> </span><code>@para('http参数名称')</code><span> </span>的方式来定义 key。</p> 
<p style="color:#242424; text-align:start">例如：</p> 
<pre><code class="language-java">@Cacheable(name = "aaa", key="#para('mykey')", liveSeconds = 10)
public void json() &#123;
    
    System.out.println("json() invoked!!!!!!!!!");
    
    Map<String, Object> data = new HashMap<>();
    data.put("age", 1);
    data.put("name", "张三");
    data.put("sex", 1);
    
    renderJson(data);
&#125;</code></pre> 
<p style="color:#242424; text-align:start">当访问<span> </span><code>http://127.0.0.1:8080/json?mykey=thekey</code><span> </span>的时候，该缓存的 key 的内容为："thekey"。然而，在某些极端场景下，我们可能是不需要缓存的，配置如下：</p> 
<pre><code>@Cacheable(name = "aaa", liveSeconds = 10, unless = "para('type')==1")
public void json2() &#123;
     System.out.println("json2() invoked!!!!!!!!!");
     Map<String, Object> data = new HashMap<>();
     data.put("age", 1);
     data.put("name", "张三");
     data.put("sex", 1);
     renderJson(data);
&#125;</code></pre> 
<p style="color:#242424; text-align:start">当访问<span> </span><code>http://127.0.0.1:8080/json2?type=1</code><span> </span>的时候，永远不会命中缓存。</p>

<p style="color:#333333; margin-left:0; margin-right:0; text-align:left"><strong>Jboot v3.14.0  更新内容如下：</strong></p> 
<div style="text-align:start"> 
 <ul> 
  <li>新增：@Cacheable() 等系列缓存注解对 controller 的支持</li> 
  <li>新增：Controller 返回值，自动匹配相应的 render 的支持</li> 
  <li>新增：分布式任务注解 @EnableDistributedRunnable，自定义的 redisKey 和 key 持有时间配置功能。</li> 
  <li>新增：AttachmentManager 新增 getFile(path,localFirst) 方法</li> 
  <li>新增：CookieUtil 添加 "defaultPath" 和 "defaultDomain" 的配置</li> 
  <li>新增：HttpUtil 添加 http 代理的配置支持</li> 
  <li>新增：Columns.addToFirst() 方法</li> 
  <li>新增：JbootDirectiveBase.getParaToString()</li> 
  <li>新增：MQ 新增 stopListening() 方法，可用于定制化关闭 MQ</li> 
  <li>修复：Junit 代码覆盖率测试可能出现多次启动的问题</li> 
  <li>修复：JbootCron4jPlugin 停止后未移除已经停止任务的问题</li> 
  <li>优化：优化 fastjson 序列化的功能，使用 config 而非 features</li> 
  <li>优化：重命名注解 @TxEnable() 修改为 @Transactional()，并为 @Transactional() 添加更多的配置功能</li> 
  <li>优化：为 RabbitMQ 添加更多的自定义配置</li> 
  <li>优化：自动检测是否依赖 jfinal-wexin 并自动配置 JbootAccessTokenCache</li> 
  <li>优化：删除 JacksonSerializer 等无用的代码文件</li> 
  <li>优化：优化配置文件的的自动探测功能，防止在某些新手未编译直接运行找不到配置文件的问题</li> 
  <li>优化：优化 JbootHttpImpl 和 JbootHttpResponse 代码，添加必要的日志输出</li> 
  <li>优化：更新 Copyright</li> 
  <li>优化：升级相关依赖到最新版本</li> 
  <li>文档：完善关于 Controller 和 数据库的相关文档</li> 
 </ul>
