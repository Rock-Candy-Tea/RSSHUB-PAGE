                                                                                            <p style="margin-left:0px"><span style="color:#333333">Krafka 是一个用于简化基于 Apache Kafka 的 Ruby 应用开发的框架，它允许开发者在使用异步 Kafka 消息时使用类似于标准 HTTP 约定（params 和 params_batch）的方法。</span></p> 
<p style="margin-left:0px"><span style="color:#333333">Karafka 不仅可以处理传入的消息，而且还提供了用于构建接收和发送消息的复杂数据流应用程序的工具。</span></p> 
<p style="margin-left:0px"><span style="color:#333333">目前 </span>Karafka v2.0.5 发布了，<span style="color:#333333">此版本更新内容如下：</span></p> 
<ul> 
 <li>修复 Ruby on Rails 的 karafka.rb 模板中不必要的双换行符</li> 
 <li>修复手动暂停的分区在重新平衡后不会被处理的情况 <span style="color:#24292f">(</span><a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fkarafka%2Fkarafka%2Fissues%2F988" target="_blank">#988</a><span style="color:#24292f">)</span></li> 
 <li>提高稳定性</li> 
 <li>在 Github CI 中执行规范的并发性较低</li> 
</ul> 
<p>更新公告：<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fkarafka%2Fkarafka%2Freleases%2Ftag%2Fv2.0.5" target="_blank">https://github.com/karafka/karafka/releases/tag/v2.0.5</a></p>
