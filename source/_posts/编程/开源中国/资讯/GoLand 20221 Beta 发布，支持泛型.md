<div class="content">
                                                                    
                                                        <p>GoLand 2022.1 现已达到 Beta 版，因此该版本已停止添加任何新功能，接下来将专注于修复错误并为 GoLand 的主要版本做准备。</p> 
<p><img alt height="350" src="https://static.oschina.net/uploads/space/2022/0326/071812_c9lT_4937141.png" width="700" referrerpolicy="no-referrer"></p> 
<ul> 
 <li>泛型 
  <ul> 
   <li>改进和错误修复</li> 
   <li>补全</li> 
   <li>实施缺失方法的快速修复</li> 
   <li>支持 Go 工作区</li> 
  </ul> </li> 
 <li>微服务 
  <ul> 
   <li>查找用法和导航</li> 
   <li>生成请求</li> 
   <li>端点工具窗口</li> 
   <li>补全方法和 headers</li> 
   <li>链接和端点带有下划线</li> 
   <li>更好地支持 <em>.proto</em> 文件</li> 
   <li>gRPC 反射支持</li> 
  </ul> </li> 
 <li>编辑器 
  <ul> 
   <li>对 testify 函数的字符串格式支持</li> 
  </ul> </li> 
 <li>调试器 
  <ul> 
   <li>隐藏了选项卡标签</li> 
  </ul> </li> 
 <li>Kubernetes 
  <ul> 
   <li>编辑集群上的资源</li> 
   <li>kubectl 的自定义路径</li> 
   <li>端口转发</li> 
   <li>在 Services 视图中添加 Describe Resource 操作</li> 
   <li>支持集群中的事件</li> 
   <li>支持 Werf</li> 
  </ul> </li> 
 <li>Docker 
  <ul> 
   <li>新的 Services 视图 UI</li> 
   <li>支持 Docker Compose targets</li> 
   <li>支持 Docker Registry V2</li> 
  </ul> </li> 
 <li>Markdown 
  <ul> 
   <li>在 Markdown 块中添加了复制代码的图标，可快速复制代码至剪切板</li> 
   <li>直接从 Markdown 文件运行命令</li> 
   <li>改进的浮动工具栏</li> 
  </ul> </li> 
 <li>用户体验 
  <ul> 
   <li>新的通知工具窗口</li> 
   <li>更新了 Structural Search 和 Replace 对话框</li> 
   <li>在编辑器标签页之间平均分配工作空间</li> 
  </ul> </li> 
 <li>版本控制 
  <ul> 
   <li>提交详细信息窗格包含详细的 GPG 签名信息。</li> 
   <li>改进了使用 Git Blame 进行注释的功能</li> 
  </ul> </li> 
 <li>数据库改进 
  <ul> 
   <li>复制多个对象</li> 
   <li>MongoDB：编辑结果中的字段</li> 
  </ul> </li> 
 <li>Web 开发 
  <ul> 
   <li>改进 Vue 3</li> 
   <li>Next.js 更新</li> 
   <li>ESM 对 webpack 配置的支持</li> 
   <li>与 Volta 集成</li> 
  </ul> </li> 
 <li>可访问性改进</li> 
</ul>
