                                                                                            <p>CopyQ 6.0.0 和 6.0.1 现已发布。CopyQ 是一个剪切板管理工具，可以监控系统剪贴板，并将其内容保存在自定义标签中。保存的剪贴板内容可以在以后直接复制和粘贴到任何应用程序中。</p> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left">v6.0.0 更新内容如下：</p> 
<p style="color:#333333; margin-left:0; margin-right:0; text-align:left"><strong>Added</strong></p> 
<ul> 
 <li> <p><span>如果显示间隔小于10 秒，本机通知现在具有较低的紧迫性。这使得剪贴板更改通知的干扰性降低。</span></p> </li> 
 <li> <p><span>Preview dock 可以使用 Tab 键 focus ( <a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1826" target="_blank">#1826</a> )。Escape、Tab 或 Shift+Tab 将 focus 返回到项目列表。</span></p> </li> 
 <li> <p><span>在使用 copyq config 命令时，所有选项现在都有文件/说明。</span></p> </li> 
 <li> <p><span>命令编辑器现在支持突出显示由反引号括起来的多行字符串（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1845" target="_blank">#1845</a>）。</span></p> </li> 
 <li> <p><span>禁用恢复窗口/对话框几何图形的新选项（更改选项后需要重新启动应用程序）：</span></p> 
  <div> 
   <pre><span><span><span><span><span><span><span><span><span><span style="background-color:var(--color-canvas-subtle)"><code>copyq config restore_geometry false </code></span></span></span></span></span></span></span></span></span></span></pre> 
  </div> </li> 
 <li> <p><span>macOS：启用 </span><span style="background-color:#ffffff; color:#24292f">native tray menu </span><span>的新选项（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1652" target="_blank">#1652</a>）：</span></p> 
  <div> 
   <pre><span><span><span><span><span><span><span><span><span><span style="background-color:var(--color-canvas-subtle)"><code>copyq config native_tray_menu true </code></span></span></span></span></span></span></span></span></span></span></pre> 
  </div> </li> 
 <li> <p><span>支持使用 Qt 6 框架构建源代码。</span></p> </li> 
</ul> 
<p style="color:#333333; margin-left:0px; margin-right:0px; text-align:start"><strong>Changed</strong></p> 
<ul> 
 <li> <p><span>当搜索栏被 focus 时，按 Down 或 PageDown 键现在选择下一个项目而不聚焦项目列表（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1834" target="_blank">#1834</a>）。</span></p> </li> 
 <li> <p><span>内部命令（如“显示/隐藏主窗口”、固定/取消固定、加密/解密）现在将在以下应用程序版本或语言更改时自动更新。副作用是只有图标、快捷方式、启用状态和可以更改这些命令的列表顺序。在应用程序的先前版本（5.0.0 及更低版本）中添加的旧内部命令需要手动删除。</span></p> </li> 
 <li> <p><span>增加存储更改的剪贴板所有者的默认延迟。这有助于在复制操作后立即关闭窗口时为新的剪贴板内容保存正确的窗口标题。可以使用以下方法更改延迟：</span></p> 
  <div> 
   <pre><span><span><span><span><span><span><span><span><span><span style="background-color:var(--color-canvas-subtle)"><code>copyq config change_clipboard_owner_delay_ms 500 </code></span></span></span></span></span></span></span></span></span></span></pre> 
  </div> </li> 
 <li> <p><span>应用程序版本现在不包括 UI 和 CLI 中的“v”前缀。</span></p> </li> 
 <li> <p><span>默认情况下记录 Qt 警告（在警告日志级别消息）。</span></p> </li> 
 <li> <p><span>Linux：现在存储其他数据格式用于主要选择，以便正确支持一些自动命令（例如，当包含特殊格式时忽略选择）。出于性能原因，图像和非纯文本格式仍被忽略。</span></p> </li> 
</ul> 
<h2 style="text-align:start"> </h2> 
<p><span><span><span><span><span style="color:#24292f"><span><span><span><span><span><span><span><span><span><span><span style="background-color:#ffffff"><span><span><span><strong>Fixed</strong></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></span></p> 
<ul> 
 <li> <p><span>拖放操作现在已正确结束（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1809" target="_blank">#1809</a>）。</span></p> </li> 
 <li> <p><span>主窗口现在将仅在可见屏幕区域内打开（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1768" target="_blank">#1768</a>）。</span></p> </li> 
 <li> <p><span>改进了 native tray menu 的初始大小。</span></p> </li> 
 <li> <p><span>修复了删除旧命令配置的备份文件的问题。</span></p> </li> 
 <li> <p><span>修复损坏的项目选择状态（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1828" target="_blank">#1828</a>）。</span></p> </li> 
 <li> <p><span>修复了显示时立即隐藏主窗口的问题。这可能是由窗口管理器中的长动画引起的。</span></p> </li> 
 <li> <p><span>日志记录、应用程序启动和文件同步的进一步性能改进。</span></p> </li> 
 <li> <p><span>Linux：默认使用本机状态图标（使用 D-Bus）而不是传统托盘图标。应用程序启动 delay/sleep hacks 不再需要（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1526" target="_blank">#1526</a>）。</span></p> </li> 
 <li> <p><span>Wayland：改进剪贴板访问。</span></p> </li> 
 <li> <p><span>Wayland：修复选择/剪贴板同步。</span></p> </li> 
 <li> <p><span>Windows：现在任何应用程序实例都会在安装前自动关闭。</span></p> </li> 
</ul> 
<p>v6.0.0 更新内容如下：</p> 
<p><strong>Fixed</strong></p> 
<ul> 
 <li>X11：修复全局/系统范围的快捷方式（<a href="https://www.oschina.net/action/GoToLink?url=https%3A%2F%2Fgithub.com%2Fhluk%2FCopyQ%2Fissues%2F1860" target="_blank">#1860</a>）。</li> 
</ul>
