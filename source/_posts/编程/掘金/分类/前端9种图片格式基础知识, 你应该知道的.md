<h2 data-id="heading-0">彩色深度</h2>
<p>彩色深度标准通常有以下几种：</p>
<ul>
<li>8位色，每个像素所能显示的彩色数为2的8次方，即256种颜色。</li>
<li>16位增强色，16位彩色，每个像素所能显示的彩色数为2的16次方，即65536种颜色。</li>
<li>24位<a href="https://link.juejin.cn/?target=https%3A%2F%2Fbaike.baidu.com%2Fitem%2F%25E7%259C%259F%25E5%25BD%25A9%25E8%2589%25B2" target="_blank" rel="nofollow noopener noreferrer" title="https://baike.baidu.com/item/%E7%9C%9F%E5%BD%A9%E8%89%B2" ref="nofollow noopener noreferrer">真彩色</a>，每个像素所能显示的彩色数为24位，即2的24次方，约1680万种颜色。</li>
<li>32位真彩色，即在24位真彩色图像的基础上再增加一个表示图像透明度信息的Alpha通道。<br>
32位真彩色并非是2的32次方的色数，<strong>它其实也是1677万多色，不过它增加了256阶颜色的灰度,为了方便称呼，就规定它为32位色</strong></li>
</ul>
<h2 data-id="heading-1">图的分类</h2>
<h3 data-id="heading-2">光栅图和矢量图</h3>
<p>对于图片，一般分光栅图和矢量图。</p>
<ul>
<li>光栅图：是基于 pixel像素构成的图像。JPEG、PNG，webp等都属于此类</li>
<li>矢量图：使用点，线和多边形等几何形状来构图，具有高分辨率和缩放功能. SVG就是一种矢量图。</li>
</ul>
<h3 data-id="heading-3">无压缩, 无损压缩, 有损压缩</h3>
<p>另一种分类</p>
<ul>
<li>
<p>无压缩。无压缩的图片格式不对图片数据进行压缩处理，能准确地呈现原图片。BMP格式就是其中之一。</p>
</li>
<li>
<p>无损压缩。压缩算法对图片的所有的数据进行编码压缩，能在保证图片的质量的同时降低图片的尺寸。png是其中的代表。</p>
</li>
<li>
<p>有损压缩。压缩算法不会对图片所有的数据进行编码压缩，而是在压缩的时候，去除了人眼无法识别的图片细节。因此有损压缩可以在同等图片质量的情况下大幅降低图片的尺寸。其中的代表是jpg。</p>
</li>
</ul>
<h2 data-id="heading-4">前端9种图片格式</h2>
<h3 data-id="heading-5">诞生时间</h3>
<p>对于超过30岁的程序员来说，她们都很年轻，真的是遇到好时光！</p>
<p><strong>85年前，人们都在干嘛呢？</strong></p>
<ol>
<li>GIF - 1987</li>
<li>Base64- 1987</li>
<li>JPEG - 1992</li>
<li>PNG - 1996</li>
<li>SVG - 1999</li>
<li>JPEG2000 - 1997 to 2000</li>
<li>APNG - 2004</li>
<li>WebP - 2010</li>
</ol>
<p>ico: 1985年??<br>
查阅文档说ico文件格式是伴随着 Windows 1.0 发行诞生的。</p>
<h3 data-id="heading-6">GIF</h3>
<p>GIF是一种索引色模式图片，所以GIF每帧图所表现的颜色最多为256种。GIF能够支持动画，也能支持背景透明，这点连古老的IE6都支持，所以在以前想要在项目中使用背景透明图片，其中一种方案就是生成GIF图片。</p>
<p><strong>优点</strong>：</p>
<ul>
<li>支持动画和透明背景</li>
<li>兼容性好</li>
<li>灰度图像表现佳</li>
<li>支持交错<br>
部分接收到的文件可以以较低的质量显示。这在网络连接缓慢时特别有用。</li>
</ul>
<p><strong>缺点</strong>：</p>
<ul>
<li>最多支持 8 位 256 色，色阶过渡糟糕，图片具有颗粒感</li>
<li>支持透明，但不支持半透明，边缘有杂边</li>
</ul>
<p><strong>适用场景</strong></p>
<ul>
<li>色彩简单的logo、icon、线框图适合采用gif格</li>
<li>动画</li>
</ul>
<h3 data-id="heading-7">JPG/JPEG</h3>
<p>这里提个问题： <strong>jpg和jpeg有啥区别</strong></p>
<p>平常我们大部分见到的静态图基本都是这种图片格式。这种格式的图片能比较好的表现各种色彩，主要在压缩的时候会有所失真，也正因为如此，造就了这种图片格式体积的轻量。</p>
<p><strong>优点</strong></p>
<ul>
<li>压缩率高</li>
<li>兼容性好</li>
<li>色彩丰富</li>
</ul>
<p><strong>缺点</strong></p>
<ul>
<li>JPEG不适合用来存储企业Logo、线框类的这种高清图</li>
<li>不支持动画、背景透明</li>
</ul>
<h3 data-id="heading-8">JPEG 2000 （了解即可）</h3>
<p>JPEG 2000是基于小波变换的图像压缩标准，由Joint Photographic Experts Group组织创建和维护。JPEG 2000通常被认为是未来取代JPEG（基于离散余弦变换）的下一代图像压缩标准。JPEG 2000文件的副档名通常为.jp2，MIME类型是image/jp2。</p>
<p><strong>JPEG2000的压缩比更高，而且不会产生原先的基于离散余弦变换的JPEG标准产生的块状模糊瑕疵</strong>。JPEG2000同时支持<a href="https://link.juejin.cn/?target=https%3A%2F%2Fbaike.baidu.com%2Fitem%2F%25E6%259C%2589%25E6%258D%259F%25E5%258E%258B%25E7%25BC%25A9" target="_blank" rel="nofollow noopener noreferrer" title="https://baike.baidu.com/item/%E6%9C%89%E6%8D%9F%E5%8E%8B%E7%BC%A9" ref="nofollow noopener noreferrer">有损压缩</a>和<a href="https://link.juejin.cn/?target=https%3A%2F%2Fbaike.baidu.com%2Fitem%2F%25E6%2597%25A0%25E6%258D%259F%25E5%258E%258B%25E7%25BC%25A9" target="_blank" rel="nofollow noopener noreferrer" title="https://baike.baidu.com/item/%E6%97%A0%E6%8D%9F%E5%8E%8B%E7%BC%A9" ref="nofollow noopener noreferrer">无损压缩</a>。</p>
<p>目前就safari支持，<a href="https://link.juejin.cn/?target=https%3A%2F%2Fcaniuse.com%2F%23search%3DJPEG2000" target="_blank" rel="nofollow noopener noreferrer" title="https://caniuse.com/#search=JPEG2000" ref="nofollow noopener noreferrer">can is use-png2000</a>支持18%。</p>
<p><strong>优点</strong></p>
<ul>
<li>支持有损和无损压缩</li>
</ul>
<p><strong>缺点</strong></p>
<ul>
<li>支持率太低了</li>
</ul>
<h3 data-id="heading-9">ICO</h3>
<p>ICO (Microsoft Windows 图标)文件格式是微软为 Windows 系统的桌面图标设计的。网站可以在网站的根目录中提供一个名为 favicon.ICO, 在收藏夹菜单中显示的图标，以及其他一些有用的标志性网站表示形式。<br>
一个 ICO 文件可以包含多个图标，并以列出每个图标详细信息的目录开始。</p>
<p>其主要用来做网站图标，现在png也是可以用来做网站图标的。</p>
<h3 data-id="heading-10">PNG</h3>
<p>PNG格式是有三种版本的，分别为PNG-8，PNG-24，PNG-32，所有这些版本都不支持动画的。PNG-8跟GIF类似的属性是相似的，都是索引色模式，而且都支持背景透明。<strong>相对比GIF格式好的特点在与背景透明时，图像边缘没有什么噪点，颜色表现更优秀。PNG-24其实就是无损压缩的JPEG</strong>。而PNG-32就是在PNG-24的基础上，增加了透明度的支持。</p>
<p>如果没有动画需求推荐使用png-8来替代gif</p>
<p><strong>优点</strong></p>
<ol>
<li>不失真的情况下尽可能压缩图像文件的大小</li>
<li>像素丰富</li>
<li>支持透明（alpha通道）</li>
</ol>
<p><strong>缺点</strong></p>
<ol>
<li>文件大</li>
</ol>
<p>这里额外提一下，gif和jpg有渐进，png有交错，都是在没有完全下载图片的时候，能看到图片全貌。</p>
<p>具体可以看在线示例： <strong><a href="https://link.juejin.cn/?target=https%3A%2F%2Fxiangwenhu.github.io%2FTakeItEasy%2FbgWhithe%2Fprogress.html" title="https://link.juejin.cn/?target=https%3A%2F%2Fxiangwenhu.github.io%2FTakeItEasy%2FbgWhithe%2Fprogress.html" target="_blank">png正常，png交错，jpg渐进</a></strong></p>
<h3 data-id="heading-11">APNG：Animated PNG</h3>
<p>APNG（Animated Portable Network Graphics）顾名思义是基于 PNG 格式扩展的一种动画格式，增加了对动画图像的支持，同时加入了 24 位图像和 8 位 Alpha 透明度的支持，这意味着动画将拥有更好的质量，其诞生的目的是为了替代老旧的 GIF 格式，但它目前并没有获得 PNG 组织官方的认可。</p>
<p>从Can I Use上查看，除了IE系列， chrome, firefox, safari均已支持。2021-08月的时候支持达到94%。</p>
<p>相对GIF来说</p>
<ul>
<li>色彩丰富</li>
<li>支持透明</li>
<li>向下兼容 PNG</li>
<li>支持动画</li>
</ul>
<p><strong>缺点</strong></p>
<ul>
<li>生成比较繁琐</li>
<li>未标准化</li>
</ul>
<h3 data-id="heading-12">webP</h3>
<p>有损 WebP 图像平均比视觉上类似压缩级别的 JPEG 图像小25-35% 。无损耗的 WebP 图像通常比 PNG 格式的相同图像小26% 。WebP 还支持动画: 在有损的 WebP 文件中，图像数据由 VP8位流表示，该位流可能包含多个帧。</p>
<p>包括体积小、色彩表现足够、支持动画。 简直了就是心中的完美女神！！</p>
<p>从<a href="https://link.juejin.cn/?target=https%3A%2F%2Fcaniuse.com%2F%23search%3Dwebp" target="_blank" rel="nofollow noopener noreferrer" title="https://caniuse.com/#search=webp" ref="nofollow noopener noreferrer">can i use - webp</a>上看，支持率95%。 主要是Safari低版本和IE低版本不兼容。</p>
<p><strong>优点</strong></p>
<ul>
<li>同等质量更小</li>
<li>压缩之后质量无明显变化</li>
<li>支持无损图像</li>
<li>支持动画</li>
</ul>
<p><strong>缺点</strong></p>
<ul>
<li>兼容性吧，相对jpg,png,gif来说</li>
</ul>
<h3 data-id="heading-13">SVG</h3>
<p>SVG 是一种基于 xml 的矢量图形格式，它将图像的内容指定为一组绘图命令，这些命令创建形状、线条、应用颜色、过滤器等等。SVG 文件是理想的图表，图标和其他图像，可以准确地绘制在任何大小。因此，SVG 是现代 Web 设计中用户界面元素的流行选择。</p>
<p><strong>优点</strong></p>
<ul>
<li>可伸缩性<br>
你可以随心所欲地把它们做大或者做小，而不用牺牲质量</li>
<li>小<br>
Svg 平均比 GIF、 JPEG、 PNG 小得多，甚至在极高的分辨率下也是如此</li>
<li>支持动画<br>
更灵活，质量无与伦比</li>
<li>与DOM无缝衔接<br>
Svg 可以直接使用 HTML、 CSS 和 JavaScript (例如动画)来操作</li>
</ul>
<p><strong>缺点</strong></p>
<ul>
<li>SVG复杂度高会减慢渲染速度</li>
<li>不适合游戏类等高互动动画</li>
</ul>
<h3 data-id="heading-14">base64</h3>
<p>图片的 base64 编码就是可以将一副图片数据编码成一串字符串，使用该字符串代替图像地址，图片随着 HTML 的下载同时下载到本地，不再单独消耗一个http来请求图片。</p>
<p><strong>优点</strong></p>
<ul>
<li>无额外请求</li>
<li>对于极小或者极简单图片</li>
<li>可像单独图片一样使用，比如背景图片重复使用等</li>
<li>没有跨域问题，无需考虑缓存、文件头或者cookies问题  </li>
</ul>
<p><strong>缺点</strong></p>
<ul>
<li>相比其他格式，体积会至少大1/3</li>
<li>编码解码有额外消耗</li>
</ul>
