<h2 data-id="heading-4">定位</h2>
<p>首先来聊聊定位，我觉得这是做个人主页的最重要的部分。我见过太多人做着做着就偏离目标，最终放弃，俗称 <strong>“需求不明确”</strong>。</p>
<p>比如刚开始做文章博客，在 Markdown 样式各种纠结，然后放弃，又比如有的人喜欢啥都自己造，结果花时间太多在“造轮子”上，反而网页迟迟造不出来，最后放弃。</p>
<p>我以前做个人主页的目的只是为了收集在 <a href="https://www.jianshu.com/u/0340be4082b5" target="_blank" rel="nofollow noopener noreferrer">简书</a> 和 <a href="https://medium.com/@haixiang6123" target="_blank" rel="nofollow noopener noreferrer">Medium</a> 上的文章，顺便写写个人简介，就当个主页了。</p>
<p>当时比较迷恋：<strong>内容为王</strong> 这个思路。所以我的想法很简单：一切以博客文章为主。绝不手写轮子，轮子给我啥我用啥，就用默认的样式。</p>
<p>可以看到无论是 v1.0 和 v2.0 都是一股强烈的 <a href="https://element.eleme.io/#/zh-CN" target="_blank" rel="nofollow noopener noreferrer">Element UI</a> 味，很多组件连改都不改，只在 v2.0 在首页上稍微做了点改进。</p>
<p>而且我还非常的 <em>鸡贼</em>，我偏不踩 Markdown 和编辑器的坑，所以文章链接都是 <strong>外链</strong> 形式，点进去就跳转到 <a href="https://www.jianshu.com/u/0340be4082b5" target="_blank" rel="nofollow noopener noreferrer">简书</a> 和 <a href="https://medium.com/@haixiang6123" target="_blank" rel="nofollow noopener noreferrer">Medium</a> 上。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/70cc7a7230514b7783b3b53f8bb27779~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>一来引流（天真了，有人点进我主页就有鬼了），二来不想浪费时间在 Markdown 的样式了，三来不想自己搞后端服务、博客管理后台。</p>
<p>而对于新版 v3.0，我不再是放各种文章了，因为只要在平台上写好文章，自然就有人关注了。另一方面，别人点进我的主页可能仅仅是因为好奇。</p>
<p><strong>所以，为了满足各位观众老爷的好奇心，我将其定位为个人落地页，也即 Landing Page。主要内容：突出个人，且花里胡哨。</strong></p>
<h2 data-id="heading-5">风格</h2>
<p>落地页最重要的特点就是 <strong>大</strong> 和 <strong>爽</strong>，所以我看了市面上很多个人主页，主要归为几个类：微博类、大佬简约类和欧美巨大类。</p>
<h3 data-id="heading-6">中国微博类</h3>
<p>第一类我称之为“中国微博类”，因为具有非常强烈的微博样式，主要元素有：<em>文章主体、文章目录、文章分类、标签等</em>。代表作：<a href="http://www.ruanyifeng.com/blog/" target="_blank" rel="nofollow noopener noreferrer">阮一峰的网络日志</a>。</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9e67f81013a74f23a451a75ecd4c7242~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>这类主要以文章内容为主，元素非常多和杂。国内使用这类的非常多，导致同质化非常严重，很难搞出新意，而且要做好文章的样式是一件非常麻烦且复杂的事情，与我的定位不符，所以 pass~</p>
<h3 data-id="heading-7">大佬简约类</h3>
<p>这类算是意义上的落地页，但是元素实在太少，只有几行的介绍就完事了。代表作：<a href="http://evanyou.me/" target="_blank" rel="nofollow noopener noreferrer">尤雨溪的主页</a>。</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/41af5bc238ae4d419493a73837e30a28~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>连个图片都没有，属于“爱看看，不看去学习”的风格。然而我不是大佬，pass~</p>
<h3 data-id="heading-8">“奇行种”类</h3>

<h3 data-id="heading-9">欧美巨大类</h3>
<p>国外的人更喜欢 <strong>大图</strong>，<strong>粗线条感</strong> 和 <strong>重口味颜色</strong> 的风格。这类的设计图在 <a href="https://dribbble.com/" target="_blank" rel="nofollow noopener noreferrer">dribbble</a> 和 <a href="https://www.behance.net/" target="_blank" rel="nofollow noopener noreferrer">behance</a> 上面非常多。</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8ffe58eb2546498886b6295fdc3023c7~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>主要特点就是：<strong>线条比较粗犷，色彩比较鲜艳或者淡雅，元素以“大”为主，同时能保持一种简约风格。</strong></p>
<p>“欧美巨大类”是我比较喜欢的类型，跳脱了国内“类微博风”的博客主页，又能有好看的设计。唯一的难度就是自己设计不出来这么好看的，所以我在 dribbble 和 behance 上找了一个模板，再结合别的一些元素，边开发边融合。</p>
<h2 data-id="heading-10">为什么不？</h2>
<p>这时有的老铁就会问了：</p>
<blockquote>
<p>为什么你不去 Hexo 这些网站直接使用免费的模板呢？</p>
</blockquote>
<p>主要还是因为上面的模板大多数都是“类微博风”，很少有 dribbble 网站上的设计风格。但是如果你没有十分特别的需求，我建议最好就用现成的，尽量不要“发明”轮子。</p>
<blockquote>
<p>那 WordPress，Wix 国外的网站生成器也有“欧美巨大类”的呀？</p>
</blockquote>
<p>一个原因是访问速度实在太慢了，不过，主要还是贵 🤣。</p>
<h2 data-id="heading-11">技术栈</h2>
<ul>
<li>React</li>
<li>TypeScript</li>
<li>Sass</li>
<li>Ant Design</li>
</ul>
<p>我相信有很多人都会觉得：</p>
<blockquote>
<p>个人网站就应该自己手写，这样才能吸引到面试官。当面试官问起的时候，才可以滔滔不绝地讲如何攻克某个技术点。</p>
</blockquote>
<p>我的想法是：<strong>不建议这么搞，能用轮子用轮子，除非万不得已，千万别手写！</strong></p>
<p>要时刻记住我们究竟是在 <strong>练习</strong> 还是在 <strong>做产品</strong>。对于前者没什么好说的，手写轮子 + 发技术文章，这在掘金上经常见到。而对于后者，则不应该纠结技术，而是想方设法把 <strong>产品</strong> 做完美的。</p>
<p>自己造轮子的缺点有很多：</p>
<ul>
<li>喜欢 “发明” 轮子，而不是 “造” 轮子</li>
<li>做出来还要维护，而且并不比市面上的轮子好用</li>
<li>最重要的一点，很容易就钻入 “如何解决 XX 问题” 的牛角尖，然后忘了到底是来写主页还是来造轮子的</li>
</ul>
<p>请相信另一句更真实的信条：</p>
<blockquote>
<p>做出来就是 NB，做不出来就是 SB。</p>
</blockquote>
<p>所以，放过自己，站在巨人肩膀上不香么？好了，废话不多说，下面就来说说我是怎么实现的吧。</p>

<p>99% 我都用 fade-in 这个入场效果，过渡更自然点。</p>
<h3 data-id="heading-19">高效</h3>
<p>对于 Banner 页面的文字，刚开始是想用 typed.js 将整段文字输出的：</p>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f8550d9498c64fe0b14e51b322bcea50~tplv-k3u1fbpfcp-watermark.image" alt="complex.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<p>同事看了后，说：“一般面试官是没有耐心看完的”，这让我意识到这么太低效。所以我改成直接展示 90% 的文字，只有最后的词语循环打印。</p>
<p>另一个场景是，我原来是用 <strong>fade in from bottom</strong> 来展示每个项目：</p>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3fd5106ad83b4317916aa0cd95fc685b~tplv-k3u1fbpfcp-watermark.image" alt="fade-in-from-bottom.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<p>每个 ProjectItem 里的文字、图片、图标是比较多的，所以使用了向上动画会让人很难第一时间注意到内容，必须等动画结束了才能“看清楚”内容，而且在手机上尤为明显，同样是 <strong>低效</strong> 的。所以，我后来都改成直接 fade-in，稍快加速动画，同时兼顾了入场动画。</p>
<h3 data-id="heading-20">对比 & 对称</h3>
<p>如果所有入场都用 <strong>fade in from bottom</strong> 就会显得有点重复了，所以加点 <strong>对比</strong> 和 <strong>对称</strong> 可以稍微点缀一下。</p>
<p>首先是在小羊 Lottie 动画和向下箭头 Lottie 动画这里，前者向上渐入，后者向下渐入，形成对比。因为箭头是向下的，所以用 <strong>fade in from top</strong> 会更有 <strong>逻辑性</strong>。</p>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/df77d09ecf0c4576a526227c9e78a665~tplv-k3u1fbpfcp-watermark.image" alt="banner-transition.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<p>另一个地方就是时间轴这里，左边内容使用向右入场，右边内容则向左入场，同时也遵循动画入场的 <strong>逻辑性</strong>。</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/76aa93e7626349fd95982aeb344c00d9~tplv-k3u1fbpfcp-watermark.image" alt="timeline-compare.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-21">强调</h3>
<p>唯一使用了夸张动画的地方，就是 “联系我” 的 <strong>“👍 求点赞，求关注，求转发，一键三连！”</strong>。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bc8786c7cfa84890b8d0e9a6d924637c~tplv-k3u1fbpfcp-watermark.image" alt="contact-transition.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<p>主要还是因为这个 section 真的太单一了，加一个夸张动画增加一点动感，这里用作强调动画应该不过分吧，哈哈 🤣。</p>
<h2 data-id="heading-22">颜色</h2>
<p>颜色方面，国内的审美一般以 <strong>小清新</strong> 为主，而由于走了欧美的 <strong>粗犷</strong> 路线，所以颜色方面使用了重口味的颜色，以 <strong>红、黄、橙、黑</strong> 为主。</p>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/967fb9a53d5648c2bb663af3f5a8e81c~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>这里的颜色值都是比较相近的。千万不要选颜色跨度太大的颜色，不然你的页面就变成东一块，西一块，有很强的割裂感。</p>
<p>如果你对选颜色不是很敏感，可以上 Adobe 的 <a href="https://color.adobe.com/zh/search?q=lol" target="_blank" rel="nofollow noopener noreferrer">主题色推荐</a>，里面有超多的主题色任君选择。</p>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0a3719c8e35341dfbb6bc5defcaf504e~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<h2 data-id="heading-23">背景</h2>
<p>背景真的太难选了！难点有：</p>
<ul>
<li>不能太花里胡哨。不然会喧宾夺主，内容会被背景抢去关注</li>
<li>图片体积要小，很多人只会打开一次你的主页，这时是没有任何 HTTP 缓存的，所以体积大的背景加载时，会出现从头加载到脚的效果</li>
</ul>
<p>虽然只说了两点，但是 90% 的图片都是不合格的：风景图、个人照、文艺照。然而，往往这些照片才能提升主页逼格的关键。那有什么图片，体积又小、又不那么单一的呢？答案是：</p>
<blockquote>
<p>在 background-repeat: repeat 或者 background-size: cover|contain 情况下都不会有太多视觉上变化的图片，才能满足上面的要求。</p>
</blockquote>
<p>这里推荐两个网站，自动生成高级 SVG 背景图：</p>
<ul>
<li><a href="https://www.svgbackgrounds.com/" target="_blank" rel="nofollow noopener noreferrer">svgbackgrounds</a>，有 48 个免费 SVG 背景，而且都可以自定义一些样式的，付费有 200 多种，我觉得 48 个就够用了</li>
<li><a href="https://loading.io/background/" target="_blank" rel="nofollow noopener noreferrer">loading.io</a>，本来是个做 Loading 动画的网站，后面也做 SVG 背景图了，里面更多自定义的模板，免费版只能生成静态的，付费版可生成动态 SVG 背景，同样的，免费版就够用了</li>
</ul>
