<h1 data-id="heading-0">前言</h1>
<h2 data-id="heading-1">大厂案例</h2>
<blockquote>
<p>今年 618 互动的动效可以大致划分为两类，星秀猫主形象和入口引导动效。</p>
</blockquote>
<blockquote>
<p>在项目开始之前，我们就开始沟通整体的技术选型和美术创作工具，也就是创作选型，我们沟通了几种动效选型。一种是帧动画，以逐帧的方式来播放动画；第二种是骨骼动画，基于一套基础的骨架，来播放动画；然后是 Lottie 动画，设计师可以通过 Adobe After Effects（以下简称 AE） 设计动画然后通过插件导出即可渲染出动效。</p>
</blockquote>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/373fec9c24cc44f0a4f52b14667b62d7~tplv-k3u1fbpfcp-zoom-1.image" alt="动效选型" loading="lazy" referrerpolicy="no-referrer"></p>
<blockquote>
<p>由于骨骼动画天然适合这类有一个主体形象的动画，我们选择了2D 骨骼动画  Spine 来制作星秀猫主形象，通过网格自由变形和蒙皮技术在视觉上呈现“3D轴”的偏转。正是在 Spine 强大动画创作的支持下，星秀猫才有了“3D化”的动画化表现力。</p>
</blockquote>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8e11267207554028876b625e7855eed0~tplv-k3u1fbpfcp-zoom-1.image" alt="动效制作 - 星秀猫主形象，Spine" loading="lazy" referrerpolicy="no-referrer"></p>
<blockquote>
<p>而入口引导动效，存在量大且需要频繁调整的诉求，而 AE 是设计师们比较熟悉的软件，使用 AE 软件制作 Lottie 动画设计师们都比较愿意，因此我们采用 Lottie 来制作入口动效，制作简单的同时，时间较长的动画场景下相对 apng 和 gif 所需资源也更小。</p>
</blockquote>

<h2 data-id="heading-2">前置知识</h2>
<p>在看动画方案之前，我们先来了解一下位图和矢量图的概念。</p>
<ul>
<li>矢量图是使用直线和曲线来描述的图形，构成这些图形的元素是一些点、线、矩形、多边形、圆和弧线等，它们都是通过数学公式计算获得的，具有编辑后不失真的特点。</li>
<li>位图是由被称作像素（图片元素）的单个点组成的，放大会失真。</li>
</ul>
<h1 data-id="heading-3">方案介绍</h1>
<p>那了解完矢量图和位图，我们来看看前端要实现一个动画效果，有多少种方案可以选择呢？
常见的动画方案有Gif动画、png序列帧、SVG动画，我们来看看他们有什么区别。</p>
<h2 data-id="heading-4">Gif动画</h2>
<blockquote>
<p>图像互换格式（GIF，Graphics Interchange Format）是一种位图图形文件格式，以8位色（即256种颜色）重现真彩色的图像。它实际上是一种压缩文档，采用LZW压缩算法进行编码，有效地减少了图像文件在网络上传输的时间。适用于小动画如下拉loading、小动态logo、直播小礼物。</p>
</blockquote>
<ul>
<li>优点
<ul>
<li>使用简单，只需要引入一张图片</li>
<li>还原度高</li>
</ul>
</li>
<li>缺点
<ul>
<li>图片容易过大，需要权衡动画帧数和文件大小</li>
<li>Gif图是位图，放大会失真</li>
</ul>
</li>
</ul>
<h2 data-id="heading-5">png序列帧</h2>
<blockquote>
<p>帧动画是一种常见的动画形式，通过系列图片的连续播放来达到动画效果。具体实现是用css keyframes操作每一帧需要展示的图片，当然也可以将图片合并成精灵图（Sprites Map），可参考AlloyTeam的这个方案，<a href="http://www.alloyteam.com/2017/07/gka/" target="_blank" rel="nofollow noopener noreferrer">使用 gka 一键生成帧动画</a>。</p>
</blockquote>
<ul>
<li>优点
<ul>
<li>可插入多帧，从而实现动画效果</li>
</ul>
</li>
<li>缺点
<ul>
<li>每一帧都是一张图片，占比较大的体积</li>
<li>png是位图，放大会失真</li>
</ul>
</li>
</ul>
<h2 data-id="heading-6">SVG动画</h2>
<blockquote>
<p>SVG 意为可缩放矢量图形（Scalable Vector Graphics）。
在 SVG 中，要实现一个动画效果，可以使用 CSS，JS，或者直接使用 SVG 中自带的 animate 元素添加动画。
开发示例可参考<a href="https://juejin.cn/post/6844903476464615437#heading-0" target="_blank">SVG 动画标签，图形渐变，路径动画，线条动画</a>。</p>
</blockquote>
<ul>
<li>优点
<ul>
<li>矢量图，不失真</li>
</ul>
</li>
<li>缺点
<ul>
<li>对初学者不友好，上手成本高，需要先了解SVG的API</li>
</ul>
</li>
</ul>
<h1 data-id="heading-7">更高效方案</h1>
<p>了解了上面的常用方案后，我们可以看到，这些普通方案要么适配会有失真问题，要么上手有难度。我们更希望找到简单、高效、性能好、还原度高的动画方案。
​</p>
<p>那有没有更高效的方案呢？
答案是肯定的，下面有请主角登场。
​</p>
<h2 data-id="heading-8">一号嘉宾：SVGA</h2>
<blockquote>
<p>SVGA 是一种跨平台的开源动画格式，同时兼容 iOS / Android / Web。
SVGA Converter 可以将  After Effects 动画导出成 .SVGA 文件，供 SVGA Player 在各平台播放。
SVGA Player 支持在 iOS / Android / Web / ReactNative / LayaBox 等平台、游戏引擎播放。</p>
</blockquote>
<blockquote>
<p>更多介绍：<a href="https://svga.io/intro.html" target="_blank" rel="nofollow noopener noreferrer">SVGA官网</a></p>
</blockquote>
<blockquote>
<p>预览svga文件的动画效果：<a href="https://www.nangonghan.com/svga/" target="_blank" rel="nofollow noopener noreferrer">SVGA工具</a></p>
</blockquote>
<blockquote>
<p>各端播放器集成指南：<a href="https://svga.io/integrated.html" target="_blank" rel="nofollow noopener noreferrer">SVGA集成</a></p>
</blockquote>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/75b5228ae02e4db3b625be137d43af58~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-9">SVGA 动画库源码思路</h3>
<p>通过阅读源码，我们捋一下他实现的思路</p>
<ul>
<li>一帧一帧</li>
<li>通过设置帧率，来生成一个配置文件，使得每一帧都有一个配置，每一帧都是关键帧，通过帧率去刷每一帧的画面，这个思路跟gif很像，但是通过配置使得动画过程中图片都可以得到复用。性能就得到很大提升。并且不用解析高阶插值（比如二次线性方程，贝塞尔曲线方程）</li>
</ul>
<h3 data-id="heading-10">优缺点</h3>
<ul>
<li>优点
<ul>
<li>使用简单，性能较好</li>
<li>可动态添加或替换动画中的元素</li>
<li>解析、播放器的库比lottie的精简许多(gzip前57KB)，导出的文件也较小</li>
</ul>
</li>
<li>缺点
<ul>
<li>动画是压缩产物，不支持二次编辑</li>
<li>动画帧数低会造成视觉上的卡顿</li>
<li>支持AE特性少</li>
<li>内存占用比lottie稍高</li>
<li>不支持复杂的矢量形状图层、不支持AE自带的渐变、生成、描边、擦除等</li>
</ul>
</li>
</ul>
<h2 data-id="heading-11">二号嘉宾：Lottie</h2>
<blockquote>
<p>Lottie是一个用于Android，iOS，Web和Windows的库。
设计师制作好动画，并且利用Bodymovin插件导出JSON文件。
前端直接引用lottie-web库即可，原理就是用JS操作Svg API，当然也可以指定为canvas。
前端完全不需要关心动画的过程，JSON文件里有每一帧动画的信息，而库会帮我们执行每一帧。</p>
</blockquote>
<blockquote>
<p>更多介绍：<a href="http://airbnb.io/lottie/#/" target="_blank" rel="nofollow noopener noreferrer">lottie官方文档</a></p>
</blockquote>
<blockquote>
<p>动画在线预览：<a href="https://lottiefiles.com/preview" target="_blank" rel="nofollow noopener noreferrer">预览地址</a>、<a href="https://lottiefiles.com/popular?page=1" target="_blank" rel="nofollow noopener noreferrer">社区热门动画</a></p>
</blockquote>
<blockquote>
<p>设计师安装插件：<a href="http://airbnb.io/lottie/#/after-effects?id=installing-bodymovin" target="_blank" rel="nofollow noopener noreferrer">AE安装Bodymovin教程</a></p>
</blockquote>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2290e9c61d954f45b73d4199147bfcf7~tplv-k3u1fbpfcp-zoom-1.image" alt="1.png" loading="lazy" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-12">Lottie 动画库源码思路</h3>
<p>通过阅读源码，我们捋一下他实现的思路</p>
<ul>
<li>一层一层</li>
<li>完全按照设计工具的设计思路来进行还原，将动画脚本导出并解析。动画脚本非常的轻量。</li>
<li>将所有的动画拆成多个层级,每个层级layer都有一个动画配置，播放时解析多个layer的配置，并给每个layer做相应的动画。也达到了图片可以复用。当需要解析高阶插值（比如二次线性方程，贝塞尔曲线方程）时，性能相对而言差一点。</li>
</ul>
<h3 data-id="heading-13">优缺点</h3>
<ul>
<li>优点
<ul>
<li>开发成本低，设计师导出JSON后，开发同学只需引用文件即可</li>
<li>支持服务端URL创建，服务端可以配置JSON文件，随时替换动画</li>
<li>性能提升，替换原使用帧图完成的动画，节省客户端空间和内存</li>
<li>动画比例不会被拉伸，播放较为流畅</li>
</ul>
</li>
<li>缺点
<ul>
<li>对某些AE属性不支持，具体可<a href="http://airbnb.io/lottie/#/supported-features" target="_blank" rel="nofollow noopener noreferrer">对照表</a>​</li>
<li>对缓动曲线的解析会占用非常多的内存</li>
<li>各平台效果的支持都不是很稳定，容易出 Bug</li>
</ul>
</li>
</ul>

<h1 data-id="heading-18">注意</h1>
<p>下面引用字节跳动飞书团队的case，给大家分享下使用时需要注意的细节点</p>
<h2 data-id="heading-19">不需要动画时，及时卸载 Lottie 动画组件</h2>
<p>飞书出现过偶现 CPU 升高的异常案例，经过排查定位到是 Lottie 动画没有卸载引起的 CPU 升高。页面中已经没有动画，但 Lottie 一直在调用 requestAnimationFrame，导致在没有任何操作的情况下，CPU 占用升高至 2%-5% 左右，一般情况在没有任何操作的情况下，cpu 占用 0.1% ~0.2%。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6d818a69e6634745811407848576ce87~tplv-k3u1fbpfcp-zoom-1.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f816b9d61ecb4164b20b45939dbb678c~tplv-k3u1fbpfcp-zoom-1.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer">
Lottie 动画调用 react-lottie 组件，组件在 componentWillUnmount 时，会销毁该动画实例。
​</p>
<p>飞书中 CPU 升高的时候，发现 Lottie 动画中有动画实例尚未销毁，导致会不停的调用 requestAnimationFrame，导致异常的动画是局部加载动画。
​</p>
<p>飞书中用到局部加载的动画的模块有主端（切换会话、联系人页面 、新添加的联系人、机器人、外部联系人、onCall、升级提示弹窗、添加联系搜索、发送云盘文件弹窗、Pin 列表、Docs Webview 加载动画、切换租户）、日历、应用中心等。
​</p>
<p>经过定位发现，应用中心里用到了局部加载动画， 在不需要动画的时候，没有卸载组件，只是通过 CSS 来隐藏了组件，导致没有销毁 Lottie 动画实例，requestAnimationFrame 会一直执行，代码如下：</p>
<pre><code class="hljs language-html copyable" lang="html">// AppHome.js

 <span class="hljs-tag"><<span class="hljs-name">div</span> <span class="hljs-attr">className</span>=<span class="hljs-string">&#123;!isLoaded</span> ? '<span class="hljs-attr">app-home-loadingImg</span>' <span class="hljs-attr">:</span> '<span class="hljs-attr">display-none</span>'&#125;></span><span class="hljs-tag"><<span class="hljs-name">PartialLoading</span> /></span><span class="hljs-tag"></<span class="hljs-name">div</span>></span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/173c013236ef4d0aafa8f77c1c76a36e~tplv-k3u1fbpfcp-zoom-1.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/23ea4df4d4d04f1ea058893411908ee9~tplv-k3u1fbpfcp-zoom-1.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer">
​</p>
<p>解决方案：不需要 Lottie 动画的时候，卸载 Lottie 动画组件。</p>
