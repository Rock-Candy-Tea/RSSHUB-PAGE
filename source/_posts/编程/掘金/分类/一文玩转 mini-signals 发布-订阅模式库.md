h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h3 data-id="heading-0"><span class="prefix"></span><span class="content"><span></span>一、前言</span><span class="suffix"></span></h3>
<p>  mini-signals 是一款轻量、快速的发布-订阅模式库。对于一名合格的前端仔，应该对发布-订阅模式了然于胸。</p>
<blockquote class="multiquote-1"><span>❝</span>
<p>发布-订阅模式定义了对象间一对多的依赖关系，可以用来解决对象间耦合的问题。</p>
<span>❞</span></blockquote>
<p>  从 DOM 的事件系统，到 Node.js 的 events 模块，再到目前流行的 MVVM 架构模式，发布-订阅模式无处不在。</p>
<p>  利用 JavaScript 的一些基础知识，可以快速地实现一个简易的发布-订阅模式：</p>
<pre class="custom"><span></span><code class="hljs copyable">  <span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">EventEmitter</span> (<span class="hljs-params"></span>) </span>&#123;<br>    <span class="hljs-keyword">this</span>.events = &#123;&#125;;<br>  &#125;<br><br>  EventEmitter.prototype.on = <span class="hljs-function"><span class="hljs-keyword">function</span> (<span class="hljs-params">event, fn</span>) </span>&#123;<br>    <span class="hljs-keyword">const</span> &#123; events &#125; = <span class="hljs-keyword">this</span>;  <br>    <span class="hljs-keyword">if</span> (!events[event]) &#123;<br>      events[event] = [];<br>    &#125;<br><br>    events[event].push(fn);<br>  &#125;<br><br>  EventEmitter.prototype.emit = <span class="hljs-function"><span class="hljs-keyword">function</span> (<span class="hljs-params">event, ...args</span>) </span>&#123;<br>    <span class="hljs-keyword">const</span> &#123; events &#125; = <span class="hljs-keyword">this</span>;<br>    <span class="hljs-keyword">const</span> listeners = events[event];<br>    <span class="hljs-keyword">if</span> (!listeners || listeners.length === <span class="hljs-number">0</span>) &#123;<br>      <span class="hljs-keyword">return</span>;<br>    &#125;<br><br>    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">0</span>; i < listeners.length; i++) &#123;<br>      listeners[i].apply(<span class="hljs-keyword">this</span>, args);<br>    &#125;<br>  &#125;<br><span class="copy-code-btn">复制代码</span></code></pre>

<h3 data-id="heading-1"><span class="prefix"></span><span class="content"><span></span>二、基础使用</span><span class="suffix"></span></h3>
<p>  探索源码之前，首先需要了解 mini-signals 的使用方式：</p>
<pre class="custom"><span></span><code class="hljs copyable">  <span class="hljs-keyword">const</span> signal = <span class="hljs-keyword">new</span> MiniSignals();<br><br>  signal.add(task);<br>  signal.dispatch(<span class="hljs-string">'foo'</span>, <span class="hljs-string">'bar'</span>);<br><br>  <span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">task</span>(<span class="hljs-params">foo, bar</span>) </span>&#123;<br>    <span class="hljs-comment">// do something</span><br>  &#125;<br><span class="copy-code-btn">复制代码</span></code></pre>

<pre class="custom"><span></span><code class="hljs copyable">  <span class="hljs-keyword">const</span> events = <span class="hljs-keyword">new</span> EventEitter();<br>  events.on(<span class="hljs-string">'eventname'</span>, task);<br>  events.emit(<span class="hljs-string">'eventname'</span>, <span class="hljs-string">'foo'</span>, <span class="hljs-string">'bar'</span>);<br><span class="copy-code-btn">复制代码</span></code></pre>
<p>  单从使用方式来看，mini-signals 有如下优势：</p>
<ul>
<li>不需要通过额外的常量来管理发布-订阅的事件名。</li><li>事件不再通过一条总线管理，一个实例对应一个独立的事件容器，一定程度上减少了程序的复杂度。</li></ul>
<h3 data-id="heading-2"><span class="prefix"></span><span class="content"><span></span>三、双向链表管理事件监听</span><span class="suffix"></span></h3>
<p>  mini-signals 并没有直接在 MiniSignal 对象上创建一个数组来管理事件监听。</p>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">MiniSignal</span> </span>&#123;<br>  <span class="hljs-keyword">constructor</span> () &#123;<br>    <span class="hljs-comment">// 前驱和后继</span><br>    <span class="hljs-keyword">this</span>._head = <span class="hljs-keyword">this</span>._tail = <span class="hljs-literal">undefined</span>;<br>  &#125;<br>&#125;<br><span class="copy-code-btn">复制代码</span></code></pre>
<p>  而是选择实现一个朴素的<strong><span>「</span>双向链表<span>」</span></strong>来管理事件容器上的事件监听，由于双向链表的节点需要记录其前驱和后继节点，所以需要 MiniSignalBinding 类来创建节点实例：</p>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">MiniSignalBinding</span> </span>&#123;<br>  <span class="hljs-keyword">constructor</span> (fn, once = false, thisArg) &#123;<br>    <span class="hljs-keyword">this</span>._fn = fn;<br>    <span class="hljs-comment">// 回调函数是否只执行一次</span><br>    <span class="hljs-keyword">this</span>._once = once;<br>    <span class="hljs-keyword">this</span>._thisArg = thisArg;<br>    <span class="hljs-comment">// 记录当前节点的前驱和后继</span><br>    <span class="hljs-keyword">this</span>._next = <span class="hljs-keyword">this</span>._prev = <span class="hljs-literal">null</span>;<br>    <span class="hljs-comment">// 归属的事件容器</span><br>    <span class="hljs-keyword">this</span>._owner = <span class="hljs-literal">null</span>;<br>  &#125;<br>  <span class="hljs-comment">// 解除绑定方法</span><br>  detach () &#123;<br>    <span class="hljs-keyword">if</span> (<span class="hljs-keyword">this</span>._owner === <span class="hljs-literal">null</span>) <span class="hljs-keyword">return</span> <span class="hljs-literal">false</span>;<br>    <span class="hljs-keyword">this</span>._owner.detach(<span class="hljs-keyword">this</span>);<br>    <span class="hljs-keyword">return</span> <span class="hljs-literal">true</span>;<br>  &#125;<br><br>&#125;<br><span class="copy-code-btn">复制代码</span></code></pre>
<p>  MiniSignalBinding 内部通过 _prev 和 _next 来记录前驱和后继节点，并且通过 _owner 属性来记录当前节点归属的事件容器。</p>
<p>  这一套操作还是比较专业的。</p>

<h3 data-id="heading-6"><span class="prefix"></span><span class="content"><span></span>七、总结</span><span class="suffix"></span></h3>
<p>  以上便是 mini-signals 库的核心代码，非常的轻量。</p>
<p>  同时也可以看出 MiniSignal 相比较大部分的发布-订阅模式的实现还是有一些优势的：</p>
<ul>
<li>利用双向链表管理事件监听，从而引入了单独的对象来维护每一个事件监听函数，提供了程序的可读性。</li><li>单独的事件容器，相比较事件名的约定，不需要再维护大量的常量事件名，一定程度上减少了程序的复杂度，提高了程序的可维护性。</li></ul>
