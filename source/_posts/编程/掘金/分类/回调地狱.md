<p>异步请求容易构成回调地狱</p>
<h2 data-id="heading-1">怎么优化回调地狱</h2>
<ul>
<li>
<p>拆解function：将各步拆解为单个的function</p>
</li>
<li>
<p>事件发布/监听模式</p>
<ul>
<li>一方面监听某一事件，当事件发生时，进行相应回调操作，另一方面当某些操作完成之后，通过发布事件触发函数，这样就可以将原本捆绑在一起的代码解耦</li>
</ul>
</li>
<li>
<p>Promise</p>

 &#125;).then(res => &#123;

 &#125;).then(data => &#123;
     console.log(data);
 &#125;).catch(err => &#123;

 &#125;);

</li>
<li>
<p>generator</p>
<ul>
<li>generator是es6中的一个新的语法。在function关键字后添加*即可将函数变为generator。</li>
</ul>
<pre><code class="copyable">const generator = function* () &#123;
    yield 11;
    yield 22;
    return 33;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>执行generator将会返回一个遍历器对象，用于遍历generator内部的状态</li>
</ul>
<pre><code class="copyable">    let g = generator();
    g.next(); // &#123; value: 11, done: false &#125;
    g.next(); // &#123; value: 22, done: false &#125;
