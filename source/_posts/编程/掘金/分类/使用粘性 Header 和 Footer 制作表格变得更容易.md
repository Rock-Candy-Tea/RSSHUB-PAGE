<p>不久前，当我在博客文章<a href="https://link.juejin.cn/?target=https%3A%2F%2Fcss-tricks.com%2Fa-table-with-both-a-sticky-header-and-a-sticky-first-column%2F" target="_blank" rel="nofollow noopener noreferrer" title="https://css-tricks.com/a-table-with-both-a-sticky-header-and-a-sticky-first-column/" ref="nofollow noopener noreferrer">一个带有粘性 Header 和粘性第一列内容的表格</a>中写到关于 HTML <code><table></code> 中的粘性 Header 和 Footer 的内容。那时我没有在任何 <code><thead></code>、<code><tfoot></code> 或 <code><tr></code> 元素上使用 <code>position: sticky</code>，因为即使在 Safari 和 Firefox 中我可以这样做，在 Chrome 也做不到。不过它可以应用于像 <code><th></code> 和 <code><td></code> 这样的表格单元格，这是一个不错的解决方法。</p>
<p>嗯，这已经改变了。</p>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3572c14ba0f44efbb17178410ce49621~tplv-k3u1fbpfcp-watermark.image" alt="推特帖子" loading="lazy" referrerpolicy="no-referrer"></p>
<p>听起来像是 Chromium 的渲染引擎中彻底重构了表格的渲染，使得表格的渲染能跟上节奏。这是一项巨大的努力，不仅仅是固定黏在顶上或底下的一行内容这么多，还有<a href="https://link.juejin.cn/?target=https%3A%2F%2Fdocs.google.com%2Fdocument%2Fd%2F16PFD1GtMI9Zgwu0jtPaKZJ75Q2wyZ9EZnVbBacOfiNA%2Fedit" target="_blank" rel="nofollow noopener noreferrer" title="https://docs.google.com/document/d/16PFD1GtMI9Zgwu0jtPaKZJ75Q2wyZ9EZnVbBacOfiNA/edit" ref="nofollow noopener noreferrer">各项其他工作</a>。不过我只关注了粘性 Header 和 Footer，因为那是我所关注的。</p>
<p>我的标题 <code><thead></code> 和 <code><tfoot></code> 是具有粘性的的，这似乎是这里最常见的用例：</p>
<pre><code class="hljs language-css copyable" lang="css"><span class="hljs-selector-tag">table</span> <span class="hljs-selector-tag">thead</span>,
<span class="hljs-selector-tag">table</span> <span class="hljs-selector-tag">tfoot</span> &#123;
  <span class="hljs-attribute">position</span>: sticky;
&#125;
<span class="hljs-selector-tag">table</span> <span class="hljs-selector-tag">thead</span> &#123;
  inset-block-start: <span class="hljs-number">0</span>; <span class="hljs-comment">/* "top" */</span>
&#125;
<span class="hljs-selector-tag">table</span> <span class="hljs-selector-tag">tfoot</span> &#123;
  inset-block-end: <span class="hljs-number">0</span>; <span class="hljs-comment">/* "bottom" */</span>
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fcodepen.io%2Fchriscoyier%2Fpen%2FWNpJewq" target="_blank" rel="nofollow noopener noreferrer" title="https://codepen.io/chriscoyier/pen/WNpJewq" ref="nofollow noopener noreferrer">Codepen chriscoyier/WNpJewq</a></p>
<p>这适用于所有三个主要浏览器。你可能想要变得更机智一些，希望只在特定的最小视口高度或其他地方粘住这几个元素，但关键是它有用能正常工作了。</p>
<p>我也听到了几个关于表格的<strong>列</strong>的问题。我的原文中有一个具有粘性的第一列内容（这就是重点）。虽然其中有一个表格 <code><col></code> 标签，但它……很奇怪。它实际上并没有包装列，它更像是一个指针，可以根据需要调整列的样式。我几乎从未见过它被使用过，但它就在那里。无论如何，你完全<strong>不能</strong>应用 <code>position:sticky;</code> 在一个 <code><col></code> 上，但你<strong>可以</strong>制作粘性列。你只需要选择该列中的所有单元格并将它们粘贴到左侧或右侧。这是使用逻辑属性……</p>
<pre><code class="hljs language-css copyable" lang="css"><span class="hljs-selector-tag">table</span> <span class="hljs-selector-tag">tr</span> <span class="hljs-selector-tag">th</span><span class="hljs-selector-pseudo">:first</span>-child &#123;
  <span class="hljs-attribute">position</span>: sticky;
  inset-inline-start: <span class="hljs-number">0</span>; <span class="hljs-comment">/* "left" */</span>
&#125;
