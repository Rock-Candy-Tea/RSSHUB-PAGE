<h1 data-id="heading-0">8月得进步（二）：浅谈深浅拷贝🐷｜ 8月更文挑战</h1>
<p><strong>前言</strong>：首先在讲述深浅拷贝之前，我想说明两个概念--<strong>值传递</strong>和<strong>引用传递</strong>。</p>
<h2 data-id="heading-1">javascript数据类型</h2>
<p>1，<code>简单数据类型</code>：String，Number，Boolean，undefined，null，Symbol</p>
<p>2，<code>引用数据类型</code>：Object，可细分为 Array 、函数、RegExp等</p>
<p>我们把简单数据存储于<strong>栈内存</strong>中，复杂数据存储于<strong>堆内存</strong>里。</p>
<p>而值传递就是把数据复制一份传递（基础类型），引用传递是把数据地址复制一份传递（引用数据类型）</p>

<h2 data-id="heading-3">深拷贝</h2>
<p>和浅拷贝实现的方法一样，深拷贝不过就是再遍历一次，这里我们封装一个方法，会更方便使用。而再次拷贝引用地址不同，我们改原值是不会影响复制出来的内容的。</p>
<pre><code class="copyable">// 如果遇到object[key]是复杂类型，再次遍历操作

for ( let key in obj ) &#123;
                                // 判断是否为数组
if ( obj[key] instanceof Array ) &#123; 
newObj[key] = [];
deepCopy(newObj[key], obj[key]);
                                // 判断是否为对象
&#125; else if ( obj[key] instanceof Object ) &#123;
newObj[key] = &#123;&#125;;
deepCopy(newObj[key], obj[key])
&#125; else &#123;
newObj[key] = obj[key];

&#125;

<p>实际上就是递归思想，即对属性中引用类型的值进行再遍历，直到是基本数据类型的值为止</p></div>
