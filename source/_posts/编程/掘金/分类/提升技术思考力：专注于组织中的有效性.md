h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><p><strong>引言：</strong> 对技术开发同学来说，我们经常会面对一些在组织中典型的成长性问题，例如：技术思考力是什么？技术到达一定段位，下阶段的成长是什么？如何避免内卷？在组织中如何提升自身影响力？解决这些成长性问题，最重要的是改变自己的思维模式，将专注于<strong>效率</strong>的模式，切换到专注于<strong>有效性</strong>的模式中来。</p>
<h3 data-id="heading-0">工业时代效率先行</h3>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7cbcd027874f4351b64b1d59c3e0d641~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer">
工业社会，无论是操作机器的工人，或是前线战斗的士兵，只需要关注效率，就能对优劣做出评估：每小时生产 100 个标准螺钉的工人就比每小时生产 50 个标准螺钉的工人出色；能够第一时间落实首长命令的士兵，就比拖沓的士兵更加优秀；在工业时代，优劣判断的标准，更关注的是<strong>把事情做对</strong>，而不是做对的事情。</p>
<h3 data-id="heading-1">信息时代的有效性</h3>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d3f7d2afca27429facc926f0d0c1a272~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer">
在信息时代，知识工作者不再依赖于肌肉或者灵活的双手，他们需要更多的使用自己的智慧，这时，评价标准发生了变化：一天写 500 行代码的工程师不一定比一天 200 行代码的工程师水平高。真正评价优劣，此时考察的不是效率，而是工作的<strong>有效性</strong>，只有这些智慧真正的为组织产生了价值和贡献，才有意义。如果一个知识密集型的项目，在开始设计之初就是错误的，那么就算把他的技术架构设计的趋近完美，也没有任何的意义。在信息时代，优劣判断的标准，更关注的是<strong>做对的事情</strong>，而不再是把事情做对，因为只有做对的事情，才能使工作有效。</p>
<p>人们会倾向于做自己擅长且容易的事情，比如说别人告诉我们怎么做的事情，这个时候很容易掉入<strong>效率陷阱，</strong> 而忽略思考事情本身是否是有效的。再做事情的时候，不妨问自己两个问题，这件事是否带来的新的机会，是否解决了老的问题，因为<strong>机会的开发，问题的消失</strong>是有效性衡量的核心指标 <strong>。</strong></p>
<h3 data-id="heading-2">有效性应该如何衡量</h3>
<p>有效性不能用数量来衡量，也不能用投入的成本来衡量，有效性的衡量最终要靠结果衡量。这些结果应该是创造的直接价值、开拓性的创新、或者前沿的技术储备、核心问题的解决。OKR 和 KPI 应该是对核心贡献结果的承诺，也就是对有效性的承诺。</p>
<h3 data-id="heading-3">独立个体的有效性</h3>
<p>组织以外的知识工作者，或者称之为独立个体，基本不会存在有效性的问题。举例来说：对一个个体医生来说，病人求诊便将一切问题都带了进来，医生只需要解决这个病人的病痛，过程中其医学知识便能有效发挥；对一个自由工作的程序员，完成需求按时的交付，便是其专业知识有效性的完美证明。</p>
<h3 data-id="heading-4">在组织中的有效性</h3>
<p>与独立个体相反，组织内的个体在有效性的问题上则面对<strong>自我不可控</strong>的多重现实因素：</p>
<ol start="0">
<li>个体的大量<strong>时间属于别人</strong>，也可能面对大量紧急事项的处理，容易缺少时间进行前瞻性思考。</li>
<li>有效性的<strong>衡量取决于组织</strong>的判断，只有组织认识、利用到你的贡献时，才能体现价值。</li>
<li>在组织内，容易关注内部的信息，而<strong>忽略外部的发展</strong>，做出局限性判断。</li>
</ol>
<p>举例来说，在一个医院组织中的专家医生，大量的时间要和护士、院长、药品机构对接，自己可用的时间被大量压缩，同时，对一个组织来说，真正关注的医生贡献可能是专家医生培养了多少新医生，而非治疗了多少患者。既要处理新人培养，又要处理组织内部的沟通，专家医生很可能又会忽略外部医学技术的发展，逐步丧失自己的专家优势。</p>
<p>因为这些原因，在一个组织中发挥自己的有效性，对个人提出了更大的挑战，这和组织没有关系，只要是在组织中，有效性的挑战就一直存在。</p>
<p>在这些困难背后，组织其实也给其中的个体带来了更大的机会，<strong>组织能够使个人才干有机会增值</strong>。在独立个体重，有效性无法被复制，而在组织中，一件被证明有效的事情可以被推广复用，实现有效性的叠加。所以在组织中，能够证明有效性是件困难的事情，但有效的事情，一定不会被埋没。​</p>
<h3 data-id="heading-5">如何提升自我的有效性</h3>
<ul>
<li>
<p><strong>从多角度看待问题</strong></p>
<ul>
<li>描述这件事情有多困难。</li>
<li>产生这件事情有多困难。</li>
<li>实现这件事情有多困难。</li>
<li>证明事情正确有多困难。</li>
</ul>
</li>
<li>
<p><strong>理解自己的行为本质</strong></p>
<ul>
<li>理解并接受自己的行为，但要意识到这是自己潜意识、以及情绪的自然反应。</li>
<li>训练运用自己大脑的逻辑区。</li>
</ul>
</li>
<li>
<p><strong>合理分配时间，记录、管理、安排时间：</strong></p>
<ul>
<li>建立合理的制度，节约沟通成本。</li>
<li>对根本不必做，做了也是浪费时间，无助于结果的事情，敢于拒绝。</li>
<li>能由别人代为参与，且不影响效果的活动或会议，敢于放权。</li>
<li>做好预判，少走弯路，把重心放到重要而不紧急的事上。</li>
<li>合理的排兵布阵，降低不必要的管理成本。</li>
</ul>
</li>
<li>
<p><strong>结果导向，重视和外界的联系以及对外界的贡献：</strong></p>
<ul>
<li>明确目标，将自己的工作与长远的目标相结合。</li>
<li>关注自己的能做的事情，而非组织能给自己带来什么。</li>
<li>努力了解业务和用户的痛点。</li>
<li>建立好与同事合作方的人际关系，在沟通中寻求启发。</li>
<li>培养技术视野，尝试从更高的角度，从组织内外的维度定义现有的问题。</li>
</ul>
</li>
<li>
<p><strong>善于利用自己和他人的长处：</strong></p>
<ul>
<li>关注别人的长处，而非短处，寻求合作，借力解决问题。</li>
<li>重视绩效考核，结果跟进，KPI，OKR 的制定要具体可衡量。</li>
<li>把控招聘质量，让新鲜血液始终能给团队带来新的机遇。</li>
<li>关注 TL 的长处，寻找有效的沟通交流方式。</li>
<li>给自己制定高目标，别人不让我干一般是惰性和缺乏勇气的借口。</li>
<li>自己的工作能力和习惯自己最清楚，做自己擅长的事情并寻求合作。</li>
<li>善于利用管理手段，通过组织会议或者建立机制，推动结果产出。</li>
</ul>
</li>
<li>
<p><strong>将精力集中放在少数重要的事情上：</strong></p>
<ul>
<li>预留大块时间给重要的事情，重要的事情放在前面做，一次只做好一件事。</li>
<li>把别人设定的重要性当作制约因素，而不是决定因素。</li>
<li>重视机会，有勇气做自己认为优先的事情，而不是对别人优先的事情，敢于做时间和任务的主宰。</li>
<li>不将精力投入到不产生价值的事情，投入到创造未来的新机会上。</li>
</ul>
</li>
<li>
<p><strong>努力做出高质量决策：</strong></p>
<ul>
<li>追求有效，不是追求效率，不能以执行命令为满足。知识工作者必须能对有效性做出决策，并承担起做出贡献的职责。</li>
<li>决策是从见解开始的，见解是尚待认证的假设，要做决策，先要有自己的见解，再听别人的见解，从见解中，寻求事实，在事实中做出决策。</li>
<li>遇到问题，假定问题是一个表面想象，不满足于表面，深钻根本性的问题，重视战略性问题，适当轻视“解决问题”。</li>
<li>研究正确的决策是什么，而不是研究被人接受的决策是什么，行动或者不行动，都要坚持到底，不能半途而废。</li>
<li>决策要有多种预先设置的评价标准，最终使用最合理的标准进行反馈衡量，重视执行过程的反馈，印证正确和有效性。</li>
</ul>
</li>
</ul>
<h3 data-id="heading-6">有效性是一种自我训练</h3>
<p>有效性不是一门课程，而是一种自我训练。在自我训练中，始终专注于价值和贡献的创造，个体才能为组织的创造更广阔的发展，而组织的发展，最终也会创造更大的平台给有效的人创造更多的机会。</p>
<p>回到引言的几个问题中来：技术思考力是什么？技术到达一定段位，下阶段的成长是什么？如何避免内卷？在组织中如何提升自身影响力？我觉得，提升评判现阶段有效性的能力，可能就是这些问题的统一答案。</p>
