<ol>
<li>保留应用程序的状态，例如活动面板、主题、输入选项等。</li>
<li>出于性能考虑，需要预上传、预加载等场景</li>
<li>离线应用，除了初始下载和更新之外没有其他网络请求</li>

<p>客户端存储的数据可以是持久的，或者是临时的。但浏览器在出于安全考虑的情况下，所有的数据都是特定于某个域的。一个域（及子域）存储的值不会被其他域读写或删除。</p>

<h2 data-id="heading-1">常用存储手段</h2>
<h3 data-id="heading-2">变量</h3>
<p>JavaScript变量是最快的存储手段，但用变量存储数据的稳定性堪忧。当用户刷新或导航离开，浏览器会在下一个GC进行回收删除。用变量存储数据一般而言没有存储限制，但浏览器限制不同，当内存使用达到阈值时就会影响使用。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> state = &#123;
  <span class="hljs-attr">bar</span>: <span class="hljs-string">'baz'</span>,
  <span class="hljs-attr">fn</span>: <span class="hljs-function">() =></span> &#123;&#125;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>好处：</p>
<ul>
<li>随处可用，无需特殊支持</li>
<li>不需要序列化和反序列化</li>
</ul>
<p>缺点：</p>
<ul>
<li>数据存储是临时的</li>
<li>第三方脚本可能会覆盖全局值</li>
</ul>
<h3 data-id="heading-3">网络存储</h3>
<p>Web Storage提供了两个API来获取和设置字符串键值对：</p>
<ul>
<li><code>window.localStorage</code>：持久缓存</li>
<li><code>window.sessionStorage</code>：临时会话</li>
</ul>
<p>浏览器将每个域限制为大概5MB的大小，并且读写操作是同步的，因此进行缓存读写会阻塞其他JavaScript进程。更改值会映射到在同一域下所有其他浏览器选项卡，并且触发<code>storage</code>事件，可以在这个事件做一些更新处理。</p>
<p>好处：</p>
<ul>
<li>API简单，浏览器支持完善</li>
<li>可存储的数据量较大</li>
</ul>
<p>缺点：</p>
<ul>
<li>只能处理字符串，需要进行序列化和反序列化</li>
<li>非结构化数据，容易覆盖其他同名数据</li>
<li>保存大型数据会影响页面性能</li>
</ul>
<h3 data-id="heading-4">索引数据库</h3>
<p>IndexedDB 提供了一个类似于MySQL的名称/值数据库，它可以存储至少1GB甚至更多。可以使用异步API应用索引、使用事务和搜索。浏览器支持良好（IE10+），可以在本地存储JavaScript数据，无需进行字符串序列化。非常适合存储大型数据。</p>
<p>但很少有开发者采用IndexedDB。主要原因是因为API不友好，写起来不优美，也很容易进入回调地狱的噩梦。如果要进行现代化异步操作，需要使用<code>Promisify</code>进行封装，以达到<code>async/await</code>的简介。但还是建议在大量数据需要存储，并且性能是第一要素时选用IndexedDB。</p>
<p>好处：</p>
<ul>
<li>具有事务、索引和搜索选项的NoSQL数据库</li>
<li>无需序列化</li>
</ul>
<p>缺点：</p>
<ul>
<li>API不好用，需要开发者进行封装</li>
</ul>
<h3 data-id="heading-5">缓存API</h3>
<p>缓存API也就是HTTP缓存，主要用于缓存网络响应，以便应用程序可以在离线时提供缓存资源。HTTP缓存存储量的大小取决于浏览器。在应用程序需要提供离线文件，以及减少网络请求时可以选用该技术。</p>
<p>好处：</p>
<ul>
<li>良好的基于Promise的API</li>
</ul>
<p>缺点：</p>
<ul>
<li>无法用于存储HTTP响应之外的数据</li>
<li>Safari限制较大</li>
</ul>
<h3 data-id="heading-6">cookie</h3>
<p>cookie不是客户端存储的选项，更推荐用来作为鉴权工具，因为服务端和客户端都可以对其进行修改。但这也是唯一能手动设置过期时间的选项。</p>
<p>一个域最多可以存储20个命名cookie，每个最大字符串为4KB。并且在每次进行网络请求时，都会携带，如果有50KBcookie，请求十个文件，那么就会消耗超过1MB的无用带宽。</p>
<p>好处：</p>
<ul>
<li>登录必不可少</li>
<li>默认情况下当前会话</li>
</ul>
<p>缺点：</p>
<ul>
<li>空间有限</li>
<li>需要序列化</li>
<li>浏览器插件可以阻止cookie</li>
</ul></div>
