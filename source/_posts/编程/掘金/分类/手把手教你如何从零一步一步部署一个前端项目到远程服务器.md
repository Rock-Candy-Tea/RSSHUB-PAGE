h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h3 data-id="heading-0">写这篇文章的背景</h3>
<blockquote>
<p>我们这里用的是CentOS7的服务器，部署的整个流程都比较通畅，没有遇到什么坑，如果没有自己独立部署过前端项目的小伙伴可以先收藏着，方便自己下次部署项目的时候查询哈。</p>
</blockquote>
<p>背景是这样的，我帮朋友做了一个React小项目，然后需要将其部署到服务器上让别人访问。朋友去买了一个Tencent云的服务器，让我赶紧将它部署到上面，但是我现在手头只有一台Windwos10的游戏本，以前没有用Windows系统去部署过项目，这次就索性研究一下看看咋搞。至于没有服务器但又想部署项目的同学嘛，可以直接去到Tencent云或者Ali云去买一个就行，如果是学生就100左右一年，顺带花个十几二十块买个域名备个案，将域名解析到对应的服务器IP地址上就能够通过域名访问服务器了。</p>
<p>PS：怎么买服务器这里就不说了哈，可以去腾讯云/阿里云官网购买，买了之后会得到一个ip地址和密码，然后用本机去连接远程服务器就好了，下面我们就来说说这一步骤。</p>
<h3 data-id="heading-1">XShell面板-更强大的Windows控制台</h3>
<p>这里我们要说的是怎么用Windows系统去做配置，如果是MacOS的话，直接用系统自带的命令行去连接远程服务器就好了。用MacOS的同学可以跳过Xshell这一章节，另外也可以看看我另一篇关于配置的文章<a href="https://juejin.cn/post/6962029339235844127" target="_blank">从0开始配置Mac前端基本开发环境</a></p>
<pre><code class="copyable">ssh root@远程服务器ip地址
# 然后输入密码就ok
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们要部署项目肯定要先能连接到远程服务器上对吧，在Windows系统上我们要链接远程服务器有很多种方法， 其中我用到的工具是<a href="https://www.netsarang.com/zh/xshell-download/" target="_blank" rel="nofollow noopener noreferrer">Xshell</a>，打开官网，我们可以看到下面有免费的版本，商用版是收费的。</p>
<p>点开免费授权页面之后我们只要填写姓名和邮箱就可以了，邮箱就会收到一个下载链接，下载下来就能用了。</p>

<h3 data-id="heading-2">安装环境</h3>
<p>进入到服务器之后我们首先就是要对我们的服务器环境进行搭建</p>
<h4 data-id="heading-3">安装git</h4>
<pre><code class="copyable">yum –y install git
<span class="copy-code-btn">复制代码</span></code></pre>
<p>出现Complete!说明安装成功</p>
<p>查看git版本</p>
<pre><code class="copyable">git --version
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/07ca91a5ccac4498b190d8ec476bc187~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<h4 data-id="heading-4">初始化git</h4>
<p>安装好git之后就要进行初始化操作。第一次使用git的时候我们需要给git配置用户名和邮箱，用户和邮箱可以使用github的，也可以使用gitlab仓库的账号</p>
<p>配置用户名</p>
<pre><code class="hljs language-bash copyable" lang="bash">git config --global user.name <span class="hljs-string">"用户名"</span>
复制代码
<span class="copy-code-btn">复制代码</span></code></pre>
<p>配置邮箱</p>
<pre><code class="hljs language-bash copyable" lang="bash">git config --global user.email <span class="hljs-string">"邮箱地址"</span>
复制代码
<span class="copy-code-btn">复制代码</span></code></pre>
<p>配置好这个以后我们输入便可以看到我们所有的配置信息了，然后可以看到user.name和user.email配置得对不对</p>
<pre><code class="hljs language-bash copyable" lang="bash">git config -l
<span class="copy-code-btn">复制代码</span></code></pre>
<h4 data-id="heading-5">下载Node.js环境</h4>
<p>我们在这里使用<a href="https://github.com/nvm-sh/nvm" target="_blank" rel="nofollow noopener noreferrer">nvm(node版本管理工具)</a>去安装Node，首先我们下载nvm，输入下面的命令</p>
<pre><code class="copyable">curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.38.0/install.sh | bash
<span class="copy-code-btn">复制代码</span></code></pre>
<p>或者</p>
<pre><code class="copyable">wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.38.0/install.sh | bash
<span class="copy-code-btn">复制代码</span></code></pre>
<p>执行命令后可以看到nvm从git clone到/root/.nvm目录下，耐心等待一段时间就好啦</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ac988b0ccfdb4fd184da9a178b7489fd~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>如果下载出现报错的情况，多半是下载速度太慢产生的问题，导致无法从git拉到项目</p>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/15d88ff8afe7470faf1632f00d32f549~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>解决办法就是调整网速限制范围再尝试一下下，命令行输入如下指令</p>
<pre><code class="copyable">git config --global ssh.lowSpeedLimit 0
<span class="copy-code-btn">复制代码</span></code></pre>
<pre><code class="copyable">git config --global ssh.lowSpeedTime 999999
<span class="copy-code-btn">复制代码</span></code></pre>
<p>然后再执行下载nvm的命令，可以看到成功下载了</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ce775792151e4bb8af892a7f7e2ccba5~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>我们刚下载完其实是用不了nvm指令的，看官方文档我们可以看到，如果你执行<code>nvm -v</code>指令之后显示<code>command not found</code>，</p>
<p>那就关了当前的控制台，重新打开一个新的控制台再尝试一遍，或者执行下面三个指令中的一个，我们这里就用的bash，执行下面指令就行，执行完这个指令之后再执行<code>nvm -v</code>显示版本号就可以使用nvm啦</p>
<pre><code class="copyable">source ~/.bashrc
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/11eb4ff1abb54601a1006d50ac65967d~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>接下来我们可以执行下面指令可以看到所有的node版本，选择最新版下载就行，我们当前node最新版是v16.3.0</p>
<pre><code class="copyable">nvm ls-remote
<span class="copy-code-btn">复制代码</span></code></pre>
<p>然后再执行</p>
<pre><code class="copyable">nvm install v16.3.0
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/36dbcfee30fd4446a0afe99437c6ad62~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>到这里我们的node就安装成功咯</p>
<h4 data-id="heading-6">部署前端项目-Nginx</h4>
<h4 data-id="heading-7">安装nginx</h4>
<pre><code class="copyable">yum –y install nginx
<span class="copy-code-btn">复制代码</span></code></pre>
<p>出现Complete!之后就安装成功了</p>
<p>测试nginx配置并查看文件位置</p>
<pre><code class="copyable">nginx -t
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c4eeaa265ecd4f00855fa1bfb376dd3b~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>创建放置静态资源的文件夹，用来存放自己的项目</p>
<pre><code class="copyable">mkdir /www
<span class="copy-code-btn">复制代码</span></code></pre>
<p>然后我们输入下面的指令便可以看到我们的www文件夹了，root文件夹里一般放置一些配置文件，比如nvm或者git这些，而etc里面放着nginx的配置文件</p>
<pre><code class="copyable">cd / 跳转到服务器根目录
ls -a  查看当前路径下的所有文件
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bfe7f323daec47b68488ef1c59953ec1~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>接下来我们到www目录下拉取我们git上面的项目下来，执行下面指令</p>
<pre><code class="copyable">cd www
git clone 项目地址
cd 项目名称
npm install  
# npm install慢的导致安装失败可以使用淘宝镜像源
# npm install -g cnpm --registry=https://registry.npm.taobao.org
使用cnpm install
npm run build

# 在这里我用的是yarn
<span class="copy-code-btn">复制代码</span></code></pre>
<p>项目资源配置好之后，就可以打开nginx进行静态资源服务器的配置了，输入下面的指令打开nginx配置</p>
<pre><code class="copyable">vim /etc/nginx/nginx.conf
<span class="copy-code-btn">复制代码</span></code></pre>
<p>输入vim进入到文件之后，按i键进入INSERT模式，按esc键退出输入模式，然后输入:wq保存退出之后再在命令行输入下面指令重启nginx服务</p>
<pre><code class="copyable">nginx -s reload
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d44d7a68c9e4f04a58a26e308644cda~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>这里贴出基本的配置，端口号我们就用默认的80端口号</p>
<pre><code class="copyable">server &#123;
        listen 80;  # 端口号
        server_name localhost; # 有域名就填域名，没有就填ip地址或者本机

        include /etc/nginx/default.d/*.conf;

    location / &#123;
        root /www/项目目录/build;   # 打包后的文件目录
        index index.html index.htm;
        try_files $uri $uri/ @router;  # 开启了browserRouter模式就写这个
        &#125;

        error_page 404 /404.html;
        location = /404.html &#123;
        &#125;

        error_page 500 502 503 504 /50x.html;
        location = /50x.html &#123;
        &#125;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>配置好之后并重启nginx，然后去浏览器上输入你的IP地址就可以访问到项目啦~~</p>
<h4 data-id="heading-8">端口、防火墙</h4>
<p>CentOS7默认安装了firewalld防火墙，如果系统开启了防火墙，那我们就要手动去开放端口号</p>
<p>查看防火墙是否开启</p>
<pre><code class="copyable">systemctl status firewalld
<span class="copy-code-btn">复制代码</span></code></pre>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ace60f5b421c4bf29c29b62a6d286cb1~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>开启防火墙</p>
<pre><code class="copyable">systemctl start firewalld 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>关闭防火墙</p>
<pre><code class="copyable">systemctl stop firewalld
<span class="copy-code-btn">复制代码</span></code></pre>
<p>防火墙开机自启</p>
<pre><code class="copyable">systemctl enable firewalld
<span class="copy-code-btn">复制代码</span></code></pre>
<p>停止并且禁用防火墙</p>
<pre><code class="copyable">sytemctl disable firewalld
<span class="copy-code-btn">复制代码</span></code></pre>
<p>重启防火墙</p>
<pre><code class="copyable">firewall-cmd --reload
<span class="copy-code-btn">复制代码</span></code></pre>
<p>查看目前已开放端口</p>
<pre><code class="copyable">netstat -anp
<span class="copy-code-btn">复制代码</span></code></pre>
<p>查看指定端口是否已经开启-前提是要先开启防火墙</p>
<pre><code class="copyable">firewall-cmd --query-port=9090/tcp
<span class="copy-code-btn">复制代码</span></code></pre>
<p>对外开放端口号</p>
<pre><code class="copyable">firewall-cmd --add-port=123/tcp --permanent # permanent表示永久开放，没有此参数重启后失效
# 重新加载添加的端口
firewall-cmd --reload
<span class="copy-code-btn">复制代码</span></code></pre>
<p>关闭开放端口号</p>
<pre><code class="copyable">firewall-cmd --permanent --remove-port=9090/tcp
