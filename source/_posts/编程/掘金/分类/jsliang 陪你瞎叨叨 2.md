<h3 data-id="heading-3"><a name="user-content-chapter-three-one" id="user-content-chapter-three-one" href="https://juejin.cn/post/undefined"></a>3.1 如何保持自律？</h3>
<blockquote>
<p><a href="https://juejin.cn/post/6973188801988395045#chapter-one">返回目录</a></p>
</blockquote>
<p>保持自律的方法就是活着、不加班。</p>
<p>在 <strong>jsliang</strong> 2018 年的时候，<strong>jsliang</strong> 也是一个迷途少年，当时刚毕业出来，觉得什么事都可以由着自己信子。</p>
<p>所以晚上 6 点下班，就陪朋友打 LOL、和平精英等，玩到 1-2 点才睡觉，然后有时候再看下说看到 2-3 点，于此之下身体状况一天不如一天，天天颓废如此。</p>
<p>所以后面就调节作息时间：</p>
<ul>
<li>第 1 个月 12:00</li>
<li>第 2 个月 11:30</li>
<li>第 3 个月 11:00</li>
<li>……</li>
</ul>
<p>渐渐养成自己的学习习惯：</p>
<ul>
<li>晚上 23:00 睡觉</li>
<li>早上 06:30 起床</li>
<li>【学习】7:00-8:30</li>
<li>8:30-9:00 吃早餐</li>
<li>12:00-14:00 学习或者吃饭睡午觉</li>
<li>18:00-18:30 跑步</li>
<li>【学习】20:00-23:00</li>
</ul>
<p>因此，人的自律不是天生的，是通过【克制】而实现的。</p>
<h3 data-id="heading-4"><a name="user-content-chapter-three-two" id="user-content-chapter-three-two" href="https://juejin.cn/post/undefined"></a>3.2 如何拥有一个积极的学习观？</h3>
<blockquote>
<p><a href="https://juejin.cn/post/6973188801988395045#chapter-one">返回目录</a></p>
</blockquote>
<p>对于学习观念，最重要的是不要随大流，不因为学习而学习。</p>
<p>尤其是对于毕业后已经出来社会的小伙伴来说更是如此。</p>
<ul>
<li>如果你是转行的，那么可以从培训机构或者私人培训购买全套课程，有问题积极寻找老师，毕竟现在免费午餐是不存在的，交钱了就好好学。</li>
<li>如果你已经度过了新手期，那么应该有一个自己的目标，这样比较容易上手。</li>
</ul>
<p>第 2 点什么意思呢？举例 <strong>jsliang</strong> 来说，在 <strong>jsliang</strong> 的日常工作中，有一项工作内容是多语言翻译，其中要执行的步骤有：</p>
<ol>
<li>切换分支：<code>git checkout <branch></code></li>
<li>拉取代码：<code>git pull</code></li>
<li>前往目录：<code>cd common/xxx/xxx</code></li>
<li>复制文件：<code>Ctrl+C Ctrl+V</code></li>
<li>导入资源：<code>gulp import</code></li>
<li>回退路径：<code>cd ../../../</code></li>
<li>检查多语言：<code>gulp --release</code></li>
<li>Git 新增：<code>git add .</code></li>
<li>Git 信息：<code>git commit -m 'xxx'</code></li>
<li>Git 提交：<code>git push</code></li>
<li>提交 GitLab</li>
<li>审批</li>
<li>合入主干</li>
</ol>
<p>这 13 个步骤，当你每天执行 4/5 次的时候，你就会觉得非常厌恶，恶心，难过……</p>
<p>所以 <strong>jsliang</strong> 就开发一个工具库，将 1-11 的步骤进行简化，大大缩小操作流程。</p>
<p>而有了这个目标，<strong>jsliang</strong> 就会接触一系列 Node 和 Git 相关流程点。</p>
<p>这就是带着目的上手学习。</p>
<h3 data-id="heading-5"><a name="user-content-chapter-three-three" id="user-content-chapter-three-three" href="https://juejin.cn/post/undefined"></a>3.3 初学者如何入手前端和算法？</h3>
<blockquote>
<p><a href="https://juejin.cn/post/6973188801988395045#chapter-one">返回目录</a></p>
</blockquote>
<p>有小伙伴问 <strong>jsliang</strong> 如何快速上手前端，或者快速上手算法，<strong>jsliang</strong> 对此的回复是：</p>
<ul>
<li>学习前端</li>
</ul>
<pre><code class="copyable">a. 学习 HTML/CSS/JS
b. 学习 Vue
c. 学习 Node
d. 学习 MySQL
e. 做好能应聘的内容，就去应聘
f. 应聘成功，开始划水
g. 划水看别人厂子搞什么，然后划水学当前阶段的技术
h. 跳槽
i. 继续划水，划到那个阶段完结
j. 跳槽
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>学习算法</li>
</ul>
<pre><code class="copyable">a. 按板块学习数据结构
b. 按板块学习算法
c. 刷题
d. 面试
e. 继续刷题
<span class="copy-code-btn">复制代码</span></code></pre>
<p>然后小伙伴可能就质疑：啊你这不和市面培训机构一样吗，我是指有没有啥 “快捷” 方式。</p>
<p>有啊，待到晚上，躺在床上，成就于梦上~</p>
<p>开个小玩笑。</p>
<p>事实上，如果你想走快捷方式，那还真没有，如果有也是特殊情况。</p>
<p>所以如果你是初学者，最好<strong>报个班学个系列课程</strong>。</p>
<p>如果你需要脱产学习，最好控制在半年左右（可参考 <strong>jsliang</strong> 去年跳槽），要不然太久会导致你不想上班，甚至颓废掉。</p>
<p>详细的可以找 <strong>jsliang</strong> 吐槽，互相印证。</p>
<h3 data-id="heading-6"><a name="user-content-chapter-three-four" id="user-content-chapter-three-four" href="https://juejin.cn/post/undefined"></a>3.4 如何打造自己的文档库</h3>
<blockquote>
<p><a href="https://juejin.cn/post/6973188801988395045#chapter-one">返回目录</a></p>
</blockquote>
<p>有的小伙伴看 <strong>jsliang</strong> 的文档库能坚持那么久，于是就咨询如何去打造自己的知识体系。</p>
<p>这点 <strong>jsliang</strong> 可以详细跟小伙伴们聊聊。</p>
<p>实际上是这样的：</p>
<ul>
<li>大学期间 <strong>jsliang</strong> 比较迷茫：学了，又忘了，又去学，又忘了，啥都没做成，毕业了啥都不会</li>
<li>于是 2018 年折腾小程序的时候，就提醒自己不能再这样下去了，会一事无成的，所以当时写了篇文章发到 juejin，当时第一遍发的没人看，删了发第二遍，结果火了</li>
<li><strong>jsliang</strong> 在大二、大三的时候开始关注前端，跟着一个大佬学习 React，当时大佬的教学思路就是：将 React 整理成一个系列，然后往复循环给你教学，从而让你形成自己的 React 体系</li>
<li>于是 <strong>jsliang</strong> 就像我的文档库能不能这样子搞呢？所以也慢慢去梳理整理，将自己的文档库搞起来，期间有过 2/3 次的大整改，逐篇梳理自己。</li>
</ul>
<p>这就是 <strong>jsliang</strong> 的文档库形成的起始，小伙伴们可以看看有没有参考。</p>

<h3 data-id="heading-8"><a name="user-content-chapter-four-one" id="user-content-chapter-four-one" href="https://juejin.cn/post/undefined"></a>4.1 求职问题如何解决？</h3>
<blockquote>
<p><a href="https://juejin.cn/post/6973188801988395045#chapter-one">返回目录</a></p>
</blockquote>
<p>有个小伙伴说 <strong>自己专科，然后工资不高</strong>，希望讨论下有没有较好的方式提升自己。</p>
<p>其实这种情况真的真的很常见。</p>
<p><strong>jsliang</strong> 有好几个朋友都是专科的，然后薪酬最高在 20k 左右，并且工作非常累的那种岗位。</p>
<p>在讨论过程中小伙伴提到成人自考，说句实话有些企业不看成人自考，也不承认这玩意，人家就认第一学历。</p>
<p>就算你经验老到、技术牛，但是人家就看你学历，直接否你 offer 或者压你薪酬。</p>
<p>这种情况 <strong>jsliang</strong> 也没得法子，只能说东家不成推西家，总有企业适合你。</p>
<p>（<strong>jsliang</strong> 手头有金山、头条、腾讯、阿里等内推资源，如果对你有帮助的话可以私我）</p>
<p>还有另一个问题就是 <strong>工作经验问题</strong>。</p>
<p>那种毕业 1 年要求 10 年工作经验的咱不去吐槽，就说说常见的一个问题：</p>
<ul>
<li>项目垃圾、技术垃圾，但是公司赚钱，你跳槽难</li>
</ul>
<p>当你初入前端岗位的时候，一定一定要提醒自己：要么公司牛逼，要么项目（技术）牛逼。</p>
<p>这样，当你 2-3 年的时候，你才能因为有见识度和技术深度，去往 <strong>管理岗</strong> 和 <strong>技术大牛</strong> 的路线走，要有自己的灵魂归宿。</p>
<p>只有当你随便面试都可以扯好几个项目的重难点，吹水自己能管理前端甚至小团队的时候，你才能稳住第一阶段。</p>
<p>另外奉上 <strong>jsliang</strong> 的求职系列资料，希望对你有所帮助：</p>
<ul>
<li>系列-面试资料： <a href="https://github.com/LiangJunrong/document-library/tree/master/%E7%B3%BB%E5%88%97-%E9%9D%A2%E8%AF%95%E8%B5%84%E6%96%99" target="_blank" rel="nofollow noopener noreferrer">github.com/LiangJunron…</a></li>
</ul>
