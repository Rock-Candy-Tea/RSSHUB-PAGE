<h3 data-id="heading-0">浏览器输入url到页面呈现出来发生了什么？</h3>
<ol>
<li>
<p>进行地址解析 解析出字符串地址中的主机，域名，端口号，参数等 </p>
</li>
<li>
<p>根据解析出的域名 进行DNS解析 （1）.首先在浏览器中查找DNS缓存中是否有对应的ip地址，如果有就直 接使用，没有 机执行第二步 （2）.在操作系统中查找DNS缓存是否有对应的ip地址，如果有就直接使 用，没有就执 行第三步 （3）.向本地DNS服务商发送请求查找时候有DNS对应的ip地址。如果仍 然没有最后向RootServer服务商查询</p>
</li>
<li>
<p>根据查询到的ip地址寻找目标服务器。（1）与服务器建立连接；（2）进入服务器，寻找对应的请求</p>
</li>
<li>
<p>浏览器收到响应码开始处理</p>
</li>
<li>
<p>浏览器开始渲染dom，下载css，图片等一些资源。直到这次请求完成</p>
</li>
</ol>
<blockquote>
<p>像这种概念类的题目，同学们不用非要按照我这个或者其他网上的答案来背，这主要是考察同学们对这个概念的掌握，只要能把大概步骤都说出来就可以了。</p>
</blockquote>

<h3 data-id="heading-3">将一个整数序列整理为升序，两趟处理后变为10,12,21,9,7,3,4,25，则采用的 排序算法可能是（3）</h3>
<ol>
<li>插入排序</li>
<li>快速排序</li>
<li>选择排序</li>
<li>堆排序</li>
</ol>
<blockquote>
<p>插入排序：基于某序列已经有序排列的情况下，通过一次插入一个元素的方式按照原有排序方式增加元素</p>
</blockquote>
<blockquote>
<p>快速排序：通过一趟排序算法把所需要排序的序列的元素分割成两大块，其中，一部分的元素都要小于 或等于另外一部分的序列元素，然后仍根据该种方法对划分后的这两块序列的元素分别再次实行快速排 序算法，排序实现的整个过程可以是递归的来进行调用，最终能够实现将所需排序的无序序列元素变为 一个有序的序列</p>
</blockquote>
<blockquote>
<p>选择排序：第一次从待排序的数据元素，然后放到已排序的序列的末尾。以此类推，直到全部待排序的 数据元素的个数为零。选择排序是不稳定的排序方法</p>
</blockquote>
<blockquote>
<p>堆排序：指利用堆这种数据结构所设计的一种排序算法。堆是一个近似完全二叉树的结构，并同时满足 堆积的性质索引总是小于（或者大于）它的父节点。堆中的最大值总是位于根节点（在优先队列中使用 堆的话堆中的最小值位于根节点）。</p>
</blockquote>
<h3 data-id="heading-4">rbga()和 opacity的透明效果有什么不同？</h3>
<blockquote>
<p>rgba()和 opacity 都能实现透明效果，但最大的不同是 opacity 作用于元素，以及元素内的所有内容的 透明度；而 rgba()只作用于元素的颜色或其背景色。（设置 rgba 透明的元素的子元素不会继承透明效果） </p>
</blockquote>
<h3 data-id="heading-5">title 与 h1 的区别、b 与 strong 的区别、i 与 em 的区别？ </h3>
<blockquote>
<p>title用于网站信息标题，突出网站标题或关键字，一个网站可以有多个title，seo权重高于H1；H1概括的是文章主题，一个页面最好只用一个H1，seo权重低于 title</p>
</blockquote>
<blockquote>
<p>从网站角度而言，title则重于网站信息标题，突出网站标题或关键字用title，一 篇文章，一个页面最好只用一个H1，H1用得太多，会稀释主题；一个网站可以 有多个title，最好一个单页用一个title以便突出网站页面主题信息。</p>
</blockquote>
<blockquote>
<p>从文章角度而言，H1则概括的是文章主题，突出文章主题，用H1，面对的用 户，要突出其视觉效果。</p>
</blockquote>
<blockquote>
<p><strong>b为了加粗而加粗</strong>，strong为了标明重点而加粗；b这个标签对应 bold，即文本加粗，其目的仅仅是为了加粗显示文本，是一种样式／风格需求；<strong>strong这个标签意思是加强字符的语气</strong>，表示该文本比较重要，提醒读者／终端 注意。为了达到这个目的，浏览器等终端将其加粗显示</p>
</blockquote>
<blockquote>
<p><strong>i为了斜体而斜体，em为了标明重点而斜体</strong>，且对搜索引擎来说strong 和em比b和i要重视的多</p>
</blockquote>

<h3 data-id="heading-7">什么是防抖和节流？有什么区别？</h3>
<blockquote>
<p>**防抖(debounce)：**触发高频事件后 n 秒内函数只会执行一次，如果 n 秒内高频事件 再次被触发，则重新计算时间。</p>
</blockquote>
<blockquote>
<p>举例：就好像在百度搜索时，每次输入之后都有联想词弹出，这个控制联想词的方法 就不可能是输入框内容一改变就触发的，他一定是当你结束输入一段时间之后才会触发。</p>
</blockquote>
<blockquote>
<p>**节流(thorttle)：**高频事件触发，但在 n 秒内只会执行一次，所以节流会稀释函数的 执行频率。</p>
</blockquote>
<blockquote>
<p>举例：预定一个函数只有在大于等于执行周期时才执行，周期内调用不执行。就好像 你在淘宝抢购某一件限量热卖商品时，你不断点刷新点购买，可是总有一段时间你点 上是没有效果，这里就用到了节流，就是怕点的太快导致系统出现bug。</p>
</blockquote>
<blockquote>
<p>**区别：**防抖动是将多次执行变为最后一次执行，节流是将多次执行变成每隔一段时 间执行。 </p>
</blockquote>
