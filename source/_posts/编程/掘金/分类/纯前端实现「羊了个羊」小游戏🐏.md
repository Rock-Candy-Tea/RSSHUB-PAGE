<h1 data-id="heading-2">设计过程</h1>
<h2 data-id="heading-3">分析原型</h2>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/01d9c3d45aa84adfa138babd2c02d7ff~tplv-k3u1fbpfcp-zoom-in-crop-mark:4536:0:0:0.awebp?" alt="image.png" loading="lazy" referrerpolicy="no-referrer">
我们先思考原来的游戏卡片是怎么排布的，可以简单的观察到规律如下</p>
<ol>
<li>同层
<ul>
<li>不会互相覆盖，以 1/4 卡片面积为单位布局</li>
</ul>
</li>
<li>上下层
<ul>
<li>完全覆盖</li>
<li>覆盖 1/4</li>
<li>覆盖 1/2</li>
</ul>
</li>
</ol>
<p>因此我们以 1/4 为单位构造出一个地图即可</p>
<h2 data-id="heading-4">碰撞判定</h2>
<p>我们设定 1 为放置卡牌，因为地图单位是 1/4，实际上 1 的<strong>右一格、下一格、右下一格</strong>都是存在卡片的</p>
<pre><code class="hljs language-csharp copyable" lang="csharp">   <span class="hljs-keyword">const</span> 其中一层 = [
       [<span class="hljs-meta">0,0,0</span>],
       [<span class="hljs-meta">0,1,0</span>],
       [<span class="hljs-meta">0,0,0</span>],
   ]
<span class="copy-code-btn">复制代码</span></code></pre>
<p>所以我们在 <code>?</code> 处判当前是否可以放置卡牌的时候，需要相对它 x 的位置没有卡牌（因为我们是按顺序放，所以不用考虑 ？ 后面的</p>
<pre><code class="hljs language-csharp copyable" lang="csharp">   <span class="hljs-keyword">const</span> 其中一层 = [
       [<span class="hljs-meta">x,x,x</span>],
       [<span class="hljs-meta">x,?,0</span>],
       [<span class="hljs-meta">0,0,0</span>],
   ]
<span class="copy-code-btn">复制代码</span></code></pre>
<h2 data-id="heading-5">遮盖判定</h2>
<p>遮盖可能会发生在第 n 层和 n+m 层，所以不能简单的比较上下，需要构造一个「顶视图」,然后从上到下遍历，每有一张卡片就把顶视图的对应位置填充</p>
<pre><code class="hljs language-csharp copyable" lang="csharp">   <span class="hljs-keyword">const</span> 顶视图 = [
       [<span class="hljs-meta">0,0,0</span>],
       [<span class="hljs-meta">0,0,0</span>],
       [<span class="hljs-meta">0,0,0</span>],
   ]
   
   假设在任意一层 [<span class="hljs-number">1</span>,<span class="hljs-number">1</span>] 的位置有卡牌，顶视图更新为
   
      <span class="hljs-keyword">const</span> 顶视图 = [
       [<span class="hljs-meta">0,0,0</span>],
       [<span class="hljs-meta">0,1,1</span>],
       [<span class="hljs-meta">0,1,1</span>],
   ]
<span class="copy-code-btn">复制代码</span></code></pre>
<p>遍历到每张卡牌的时候只需要判定一下当前位置（包括右一格、下一格、右下一格）的顶视图是否为 0，即可判定卡牌的遮挡情况</p>
<h2 data-id="heading-6">设置值</h2>
<p>要注意卡片的消除是以 3 为倍数，所以我们生成卡片的数量应该也是 3 的倍数，同时每个卡片值也是如此</p></div>
