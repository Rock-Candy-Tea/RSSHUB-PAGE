<h1 data-id="heading-0">同源策略</h1>
<ul>
<li>
<p><strong>同源策略是由浏览器给的</strong></p>
</li>
<li>
<p>浏览器不允许我们向别人发送请求，只能向自己的服务器发送请求</p>
</li>
<li>
<p>当我们想向别人的服务器发送请求的时候，就会被浏览器阻止了</p>
</li>
<li>
<p>什么是 “别人的服务器” 呢？</p>
<ul>
<li><strong>当 请求协议/域名/端口号 有任意一个不同的时候，那么就算是别人的服务器</strong></li>
<li>这个时候就会触发同源策略</li>
</ul>
</li>
<li>
<p>我们管触发了 <strong>同源策略</strong> 的请求叫做跨域请求</p>
</li>
</ul>
<h1 data-id="heading-1"><a href="https://juejin.cn/post/6992459363491250184#%E5%AE%9E%E7%8E%B0%E4%B8%80%E4%B8%AA%E8%B7%A8%E5%9F%9F%E8%AF%B7%E6%B1%82" target="_blank" title="#%E5%AE%9E%E7%8E%B0%E4%B8%80%E4%B8%AA%E8%B7%A8%E5%9F%9F%E8%AF%B7%E6%B1%82"></a>实现一个跨域请求</h1>
<ul>
<li>有的时候我们是需要实现跨域请求的</li>
<li>我们需要多个服务器给一个页面提供数据</li>
<li>那么这个时候我们就要想办法解决跨域问题</li>
</ul>
<h2 data-id="heading-2"><a href="https://juejin.cn/post/6992459363491250184#jsonp" target="_blank" title="#jsonp"></a>JSONP</h2>
<ul>
<li><code>jsonp</code> 是我们实现跨域请求的手段，是把我们之前的东西组合在一起使用的技术手段而已</li>
<li>利用的是 script 标签来实现</li>
</ul>
<h3 data-id="heading-3"><a href="https://juejin.cn/post/6992459363491250184#script-%E6%A0%87%E7%AD%BE%E7%9A%84%E6%9C%AC%E8%B4%A8" target="_blank" title="#script-%E6%A0%87%E7%AD%BE%E7%9A%84%E6%9C%AC%E8%B4%A8"></a>script 标签的本质</h3>
<ul>
<li>
<p>浏览器给我们提供了一个 <code>script</code> 标签</p>
</li>
<li>
<p>它的本质就是请求一个外部资源，是不受到同源策略的影响的</p>
</li>
<li>
<p>同时 <code>script</code> 标签的 <code>src</code> 属性，也是一种请求，也能被服务器接收到</p>
</li>
<li>
<p>并且：</p>
<ul>
<li><strong><code>script</code> 标签的 <code>src</code> 属性请求回来的东西是一个字符串，浏览器会把这个字符串当作 <code>js</code> 代码来执行</strong></li>
</ul>
</li>
<li>
<p>所以我们就可以利用这个 <code>script</code> 标签的 <code>src</code> 属性来进行跨域请求了</p>
</li>
</ul>
<h2 data-id="heading-4"><a href="https://juejin.cn/post/6992459363491250184#cors-%E5%AE%9E%E7%8E%B0%E8%B7%A8%E5%9F%9F%E8%AF%B7%E6%B1%82" target="_blank" title="#cors-%E5%AE%9E%E7%8E%B0%E8%B7%A8%E5%9F%9F%E8%AF%B7%E6%B1%82"></a>CORS 实现跨域请求</h2>
<pre><code class="copyable"><?php
header("Access-Control-Allow-Origin:*");
header("Access-Control-Request-Methods:GET, POST, PUT, DELETE, OPTIONS");
header('Access-Control-Allow-Headers:x-requested-with,content-type,test-token,test-sessid');
?>

<h2 data-id="heading-5"><a href="https://juejin.cn/post/6992459363491250184#%E9%85%8D%E7%BD%AE%E4%BB%A3%E7%90%86%E4%BA%86%E8%A7%A3" target="_blank" title="#%E9%85%8D%E7%BD%AE%E4%BB%A3%E7%90%86%E4%BA%86%E8%A7%A3"></a>配置代理</h2>
<ul>
<li>代理，分成两种，正向代理和反向代理</li>
</ul>
<h3 data-id="heading-6"><a href="https://juejin.cn/post/6992459363491250184#%E6%AD%A3%E5%90%91%E4%BB%A3%E7%90%86" target="_blank" title="#%E6%AD%A3%E5%90%91%E4%BB%A3%E7%90%86"></a>正向代理</h3>
<ul>
<li>有一个客户端需要向一个非同源的服务器B发送请求</li>
<li>我们搭建一个和客户端同源的服务器A</li>
<li>当客户端发送请求的时候，由服务器A来接受</li>
<li>再由服务器A向服务器B发送请求，因为 <strong>同源策略是由浏览器给的</strong>，服务器之间没有</li>
<li>服务器B接受到请求以后，会处理请求，并把响应返回给服务器A</li>
<li>再由服务器A把响应给到客户端就可以了</li>
<li>我们就可以用这个方式来进行跨域请求了</li>
</ul>
<h3 data-id="heading-7"><a href="https://juejin.cn/post/6992459363491250184#%E5%8F%8D%E5%90%91%E4%BB%A3%E7%90%86" target="_blank" title="#%E5%8F%8D%E5%90%91%E4%BB%A3%E7%90%86"></a>反向代理</h3>
<ul>
<li>反向代理一般是用来做负载均衡的</li>
<li>当我请求一个服务器的时候，其实请求的是服务器端设置的代理服务器</li>
<li>由代理服务器把若干大量的请求分发给不同的服务器进行处理</li>
<li>再由服务器把响应给到代理服务器</li>
<li>代理服务器返回给客户端</li>
</ul></div>
