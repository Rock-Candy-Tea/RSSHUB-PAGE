<blockquote>
<p>引入原因：var存在变量提升</p>

<p>由于 JS 存在变量提升，比较容易误解，在 ES6 中引入了块级作用域</p>
<h2 data-id="heading-1">变量提升</h2>
<p>先看下什么是变量提升：</p>
<p>变量的声明会被提升到<strong>函数顶部</strong>，而变量的赋值会在原处执行，所以可以先访问到该变量，其值为 <code>undefined</code></p>
<p><strong>注意是函数顶部</strong></p>

<h2 data-id="heading-2">块级声明（let、const）</h2>
<p>块级作用域（词法作用域），存在于</p>
<ul>
<li>函数内部</li>
<li>块中（<code>&#123; &#125;</code>内）</li>
</ul>
<p>块之外无法访问块内声明的变量</p>
<h3 data-id="heading-3">let</h3>
<p>let 与 var 用法类似，但是有 4 个主要区别</p>
<p><strong>1、不存在变量提升</strong></p>
<p>还是上面两个例子</p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-comment">// 例1</span>
<span class="hljs-built_in">console</span>.log(myName) <span class="hljs-comment">// myName is no defined，报错</span>
<span class="hljs-keyword">let</span> myName = <span class="hljs-string">'Blackn'</span>

<span class="hljs-comment">// 例2，let声明的只在当前作用域有效，即函数体内，或者更小一层，在&#123;&#125;内，这里是在if内</span>
<span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">sayHello</span>(<span class="hljs-params">say</span>)</span>&#123;
    <span class="hljs-built_in">console</span>.log(sayValue + <span class="hljs-string">'1'</span>) <span class="hljs-comment">// sayValue is not defined</span>
    <span class="hljs-keyword">if</span>(say)&#123;
        <span class="hljs-keyword">var</span> sayValue = <span class="hljs-string">'Blackn'</span>
        <span class="hljs-built_in">console</span>.log(sayValue) <span class="hljs-comment">// Blackn</span>
    &#125; <span class="hljs-keyword">else</span> &#123;
        <span class="hljs-built_in">console</span>.log(sayValue + <span class="hljs-string">'3'</span>) <span class="hljs-comment">// sayValue is not defined</span>
    &#125;
&#125;
sayHello(<span class="hljs-literal">true</span>)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>例 2 中第一个 log 直接就报错，函数不会再往下执行，往下的两个打印是为了更好的理解</p>
<p><strong>2、变量只在当前声明的作用域内生效</strong></p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-comment">// 例1</span>
<span class="hljs-keyword">if</span>(<span class="hljs-literal">true</span>)&#123;

<p><strong>3、同一作用域内无法重复声明</strong></p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">get</span>(<span class="hljs-params"></span>) </span>&#123;

<p>同一作用域内无法重复声明，如果在上面的 <code>say</code> 函数外面声明 <code>myName</code> 就没有问题，或者在里面来个 <code>if</code> 判断，在判断内声明也没问题</p>
<p><strong>4、暂时性死区</strong></p>
<p>JS 引擎在扫描代码发现<strong>变量声明</strong>时，只有两种情况</p>
<ul>
<li>遇到 <strong>var</strong> 声明，将声明提升到作用域顶部</li>
<li>遇到 <strong>let、const</strong>，将声明放到 <strong>TDZ</strong> （暂时性死区 temporal dead zone）中，访问 TDZ 中的变量会报错</li>
</ul>
<p>执行过变量声明语句后，变量才会从 TDZ 中移除</p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-keyword">if</span>(<span class="hljs-literal">true</span>) &#123;
    <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">typeof</span> sayValue) <span class="hljs-comment">// 报错，此时sayValue在暂时性死区内，访问会报错</span>
    <span class="hljs-keyword">let</span> sayValue = <span class="hljs-string">'Blackn'</span>
&#125;


<span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">typeof</span> sayValue) <span class="hljs-comment">// undefined，此时sayValue是在声明它的块级作用域外，所以不在暂时性死区内</span>
<span class="hljs-keyword">if</span>(<span class="hljs-literal">true</span>) &#123;
    <span class="hljs-keyword">let</span> sayValue = <span class="hljs-string">'Blackn'</span>
&#125;

<h3 data-id="heading-4">const</h3>
<p>const 用来声明常量，值一旦被设定则无法再被更改，声明时必须赋值</p>
<p>其他的都与 let 相同</p>
<p>需要注意的一个点是，如果 const 声明的是一个对象，可以修改对象的属性</p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-keyword">const</span> age = <span class="hljs-number">21</span>
age = <span class="hljs-number">22</span> <span class="hljs-comment">// Assignment to constant variable，报错</span>

<span class="hljs-keyword">const</span> person = &#123;
    <span class="hljs-attr">age</span>: <span class="hljs-number">21</span>
&#125;
person.age = <span class="hljs-number">22</span>
person.name = <span class="hljs-string">'Blackn'</span>
<span class="hljs-built_in">console</span>.log(person) <span class="hljs-comment">//  &#123;age: 22, name: "Blackn"&#125;</span>
person = &#123;&#125; <span class="hljs-comment">// 报错</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>const声明不允许修改绑定，但允许修改值，在声明对象后，可以修改该对象的属性值</p>
<h2 data-id="heading-5">总结</h2>
<p>对比下 var、let、const的不同</p>
<ul>
<li>var：存在变量提升，无块级作用域，同一变量名可重复声明</li>
<li>let：不存在变量提升，存在暂时性死区，有块级作用域，不可重复声明</li>
<li>const：不存在变量提升，存在暂时性死区，有块级作用域，不可重复声明，声明时值必须赋值，且不能再次赋值（可以修改绑定的值，如对象的属性值）</li>
</ul></div>
