<h1 data-id="heading-0">简述自动生成的社交媒体图片</h1>
<p>我<a href="https://css-tricks.com/tag/social-media-images/" target="_blank" rel="nofollow noopener noreferrer">总是</a>在思考社交媒体图片的问题。当你在 Twitter、Facebook 或 iMessage 等平台分享链接时，这些图片是可以显示出来的。如果没有这些图片，你的链接基本上很难获取用户注意，因为它们能把一个小链接的普通帖子变成一个带有引人注目的图片的帖子，其有一个很大的可点击区域。在网站的所有图片中，社交媒体呈现的图片可能是该网站上浏览量最多、记忆度最高、网络需求量最大、排在首位的图片。</p>
<p>本质上是以下 HTML 使它们起作用：</p>
<pre><code class="hljs language-html copyable" lang="html"><span class="hljs-tag"><<span class="hljs-name">meta</span> <span class="hljs-attr">property</span>=<span class="hljs-string">"og:image"</span> <span class="hljs-attr">content</span>=<span class="hljs-string">"/images/social-media-image.jpg"</span>/></span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>要确保<a href="https://css-tricks.com/essential-meta-tags-social-media/" target="_blank" rel="nofollow noopener noreferrer">读懂它</a>，因为还有一堆其他的 HTML 标签需要弄清楚。</p>
<p>GitHub 似乎有了新的社交媒体卡片，所以我又在思考这个问题了。下面这些是新的卡片：</p>

<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e3987ffd24fc4b99a23ba42fef0eea69~tplv-k3u1fbpfcp-watermark.image" alt="22.png" loading="lazy" referrerpolicy="no-referrer"></p>

<p>那些 GitHub 的社交媒体图片显然是通过程序生成的。请看<a href="https://opengraph.githubassets.com/f55622dadf147f30f9a583a9be18924ac4567e2f8169cab9af601ecb204ec77f/fempire/resources" target="_blank" rel="nofollow noopener noreferrer">一个示例 URL</a>。</p>
<h2 data-id="heading-1">自动生成的方式</h2>
<p>虽然我知道一个纯手工定制设计的社交媒体图片有很多优点，但这对有很多页面的网站来说并不实用：博客网站、电商网站等。对于这样的网站，最好是通过模板来自动创建。我<a href="https://css-tricks.com/social-cards-as-a-service/" target="_blank" rel="nofollow noopener noreferrer">以前</a>提到过其他人在这方面的做法，让我们回顾一下：</p>
<ul>
<li>Drew McLellan：<a href="https://24ways.org/2018/dynamic-social-sharing-images/" target="_blank" rel="nofollow noopener noreferrer">动态的社交分享图片</a></li>
<li>Vercel：<a href="https://og-image.vercel.app/" target="_blank" rel="nofollow noopener noreferrer">开放图谱图片作为服务</a></li>
<li>Phil Hawksworth：<a href="https://github.com/philhawksworth/social-image-generator" target="_blank" rel="nofollow noopener noreferrer">社交图片生成工具</a></li>
<li>Ryan Filler：<a href="https://www.ryanfiller.com/blog/automatic-social-share-images/" target="_blank" rel="nofollow noopener noreferrer">自动分享社交图片</a></li>
</ul>
<p>你知道这些都有什么共同点吗？都使用了 <a href="https://github.com/puppeteer/puppeteer" target="_blank" rel="nofollow noopener noreferrer">Puppeteer</a>。</p>
<p>Puppeteer 可以方便地控制无界面 Chrome（headless Chrome）。它有一个<a href="https://pptr.dev/#?product=Puppeteer&version=v5.2.1&show=api-pagescreenshotoptions" target="_blank" rel="nofollow noopener noreferrer">好用到让人难以置信的功能</a>，那就是对浏览器窗口进行截图：<code>await page.screenshot(&#123;path: 'screenshot.png'&#125;);</code>。这就是<a href="https://github.com/chriscoyier/coding-fonts/blob/master/takeScreenshots.js" target="_blank" rel="nofollow noopener noreferrer">我们的编码字体网站进行截图的方式</a>。这种截图的方式让人们想到了一个不错的做法 —— 为什么不在 HTML 和 CSS 中设计一个社交媒体模板，然后让 Puppeteer 对其进行截图，并将其作为社交媒体的图片呢？</p>
<p>我喜欢这个想法，但这意味着它需要访问一个 Node 服务（Puppeteer 在 Node 上运行），这个服务要么一直在运行，要么可以作为 <a href="https://serverless.css-tricks.com/services/functions" target="_blank" rel="nofollow noopener noreferrer">serverless 功能</a>来使用。因此，也难怪这个想法会引起 Jamstack （译者注：Jamstack 是一种使用 SSG 技术、不依赖 Web Server 的前端架构）人群的共鸣，他们早已习惯于做这类事情，如运行构建过程和利用 serverless 功能。</p>
<p>我认为将 serverless 功能托管在一个 URL 上，并通过 URL 参数将截图中包含的动态值传递给它的想法也很聪明。</p>
<h2 data-id="heading-2">使用 SVG 的方式</h2>
<p>我有点喜欢用 SVG 作为社交媒体图片模板的想法，部分原因是我们可以基于非常固定的坐标进行设计，这与我设计社交媒体图片所需的精确尺寸的心智模型相契合。我喜欢 <a href="https://css-tricks.com/swipey-image-grids/" target="_blank" rel="nofollow noopener noreferrer">SVG 的可组合性</a>。</p>
<p>George Francis 在博客中写道：<a href="https://georgefrancis.dev/writing/generative-svg-social-images/" target="_blank" rel="nofollow noopener noreferrer">“创建你自己的 SVG 生成社交媒体图片”</a>，这是一个很好的例子，所有这些都很好地结合在一起，并带有随机性和奇思妙想。我也喜欢可满足的（contenteditable）技巧，使它成为一次性截图的有用工具。</p>
<p>我也参与了动态 SVG 的创作：请查看我们会议网站上的<a href="https://conferences.css-tricks.com/conferences/2021-magnoliajs/" target="_blank" rel="nofollow noopener noreferrer">这个会议页面</a>。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9bd73624ce941358427c97859c00757~tplv-k3u1fbpfcp-watermark.image" alt="33.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>不幸的是，SVG 并不是社交媒体图片所支持的图片格式。Twitter 明确声明了以下内容：</p>
<blockquote>
<p>社交媒体图片中使用图片的 URL。图片必须小于 5MB。支持 JPG、PNG、WEBP 和 GIF 格式。 GIF 动画只有第一帧会被使用。不支持 SVG。</p>
<p><a href="https://developer.twitter.com/en/docs/twitter-for-websites/cards/overview/markup" target="_blank" rel="nofollow noopener noreferrer">Twitter 文档</a></p>
</blockquote>
<p>不过，用 SVG 进行组合设计也是很不错的。你可以把它转换为另一种格式以便最终使用。一旦你有了 SVG，从 SVG 到 PNG 的转换几乎是轻而易举的。在我的示例中，我使用了 <a href="https://www.npmjs.com/package/svg2png" target="_blank" rel="nofollow noopener noreferrer">svg2png</a> 和在构建过程中运行的<a href="https://github.com/CSS-Tricks/conferences/blob/master/tasks/svg2png.js" target="_blank" rel="nofollow noopener noreferrer">一个很小的 Gulp 任务</a>。</p>
<h2 data-id="heading-3">使用 WordPress 的方式</h2>
<p>我没有为我的 WordPress 网站建立一个程序 —— 至少没有一个在我每次发布或更新文章时运行的程序，但我认为动态的社交媒体图片会让 WordPress <strong>获益最多</strong>（就我个人的情况而言）。</p>
<p>我现在并不是没有这些东西。<a href="https://jetpack.com/support/social/?aff=8638" target="_blank" rel="nofollow noopener noreferrer">Jetpack</a> 在这方面探索了许久。它将帖子的特色图片作为社交媒体图片，允许我预览它，并自动发布到社交网络。<a href="https://www.youtube.com/watch?v=WEKRuohH43A" target="_blank" rel="nofollow noopener noreferrer">下面是我做的一个视频</a>，你可以看到特色图片被加载并很好地显示出来。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9ed67293591c46cdb9965c93e37f9588~tplv-k3u1fbpfcp-watermark.image" alt="44.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>但这并不能使它们的创建自动化。有时，单独定制图片也是一种方式（上面那个可能是一个很好的例子），但也许更多时候，一个很好的模板化图片才是正确的方式。</p>
<p>幸运的是，我从 Daniel Post 那里知道了 WordPress 的<a href="https://socialimagegenerator.com/" target="_blank" rel="nofollow noopener noreferrer">社交图片生成器</a>。</p>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c07e32b4bb5643578a8b7c48d8eb057a~tplv-k3u1fbpfcp-watermark.image" alt="55.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<p>这正是 WordPress 所需要的！</p>
<p>Daniel 亲自帮我创建了一个专门用于 CSS-Tricks 网站的定制模板。一直以来我都有一个很大的梦想，那就是有一堆模板供我选择，其中包括标题、作者、选定的引文、特色图片和其他东西。到目前为止，我们只确定了两个模板，一个是带有标题和作者的模板，另一个是带有特色图片、标题和作者的模板。图片是根据这些元数据即时创建的：</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a2314d22872b4146907365c903683040~tplv-k3u1fbpfcp-watermark.image" alt="66.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>如此巧妙。</p>
<p>这不是 Puppeteer，甚至不是 PhantomJS 驱动的 svgtopng。这是 PHP 生成的图像! 甚至不是 <a href="https://imagemagick.org/index.php" target="_blank" rel="nofollow noopener noreferrer">ImageMagick</a>，而是<a href="https://www.php.net/manual/en/intro.image.php" target="_blank" rel="nofollow noopener noreferrer">直接的 GD</a>，直接内置于 PHP 中的东西。所以这些图片不是用任何一种语法创建的，对前端开发者来说可能感觉很舒服。你最好只使用其中一个模板，但如果你想看看我的自定义模板是如何编码的（由 Daniel 编写），请告诉我，我可以把代码开源。</p>
<p>很酷的结果，对吧？</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3cf348d109584bbcbfacc85b72468ad3~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p><a href="https://twitter.com/css/status/1391758245178511366" target="_blank" rel="nofollow noopener noreferrer">Tweet</a></p>
<p>我明白为什么它必须这样建造：它使用的技术将在 WordPress 可以运行的任何地方工作。这非常符合 WordPress 的风格。但它确实让我希望可以用一种更现代的方式来创建模板。想象一下，如果社交媒体图片的模板就像其他模板文件一样，类似于放在主题根部的 <code>social-image.php</code>，然后用所有正常的 WordPress API 来设计这个页面，就像一个 <a href="https://www.advancedcustomfields.com/resources/blocks/" target="_blank" rel="nofollow noopener noreferrer">ACF 块</a>一样。之后它还会获取截图并使用。这些岂不是很酷吗？是的，这真的很酷。</p>
