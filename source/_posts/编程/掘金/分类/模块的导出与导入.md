<h2 data-id="heading-0">导出的方式</h2>
<p><a href="https://link.juejin.cn/?target=" target="_blank" title ref="nofollow noopener noreferrer"></a><a name="user-content-vsN4y" target="_blank" ref="nofollow noopener noreferrer" href="https://link.juejin.cn/?target=undefined"></a></p>
<h3 data-id="heading-1">在声明前导出</h3>
<p>我们可以通过在声明之前放置 export 来标记任意声明为导出，无论声明的是变量，函数还是类都可以。 例如以下</p>
<pre><code class="copyable">// 导出数组
export let people = ['coolFish', 'teacherGuo', 'pingan8787','ABaoGe'];
​
// 导出 const 声明的变量
export const MODULES_BECAME_STANDARD_YEAR = 2015;
​
// 导出类
export class User &#123;
  constructor(name) &#123;
    this.name = name;
  &#125;
&#125;
​
//导出函数
export function sayHi(user) &#123;
  alert(`Hello, $&#123;user&#125;!`);
&#125;  // 在这里没有分号 ;
<span class="copy-code-btn">复制代码</span></code></pre>
<p><a href="https://link.juejin.cn/?target=" target="_blank" title ref="nofollow noopener noreferrer"></a><a name="user-content-a0CCz" target="_blank" ref="nofollow noopener noreferrer" href="https://link.juejin.cn/?target=undefined"></a></p>
<h3 data-id="heading-2">导出与声明分开</h3>
<p>另外，我们可以将先声明，然后统一导出</p>
<pre><code class="copyable">//  life.js
function play(game) &#123;
  alert(`play, $&#123;game&#125;!`);
&#125;
​
function eat(food) &#123;
  alert(`eat, $&#123;food&#125;!`);
&#125;
​
export &#123;helloWorld, beyWorld&#125;; 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>在上例中，我们依次声明了两个方法，然后用export导出该函数对象集合。 <a href="https://link.juejin.cn/?target=" target="_blank" title ref="nofollow noopener noreferrer"></a><a name="user-content-L60UC" target="_blank" ref="nofollow noopener noreferrer" href="https://link.juejin.cn/?target=undefined"></a></p>
<h3 data-id="heading-3">Export "as"</h3>
<p>我们可以用 as 来改变导出的名称</p>
<pre><code class="copyable">export &#123;helloWorld as H, beyWorld as B&#125;;

<p>现在<code>H</code> 和 <code>B</code>是在外面使用的正式名称</p>
<pre><code class="copyable">import &#123;H,B&#125; from './say.js';
​

B.eat('John'); // Bye, John!

<h3 data-id="heading-4">Export default</h3>
<p>模块提供了一个特殊的默认导出 export default 语法，以使“一个模块只做一件事”的方式看起来更好。 将 export default 放在要导出的实体前</p>
<pre><code class="copyable">export default class User &#123; // 只需要添加 "default" 即可
  constructor(name) &#123;
    this.name = name;
  &#125;
&#125;
//导入
​
import User from './user.js'; // 不需要花括号 &#123;User&#125;，只需要写成 User 即可
​
new User('John');
<span class="copy-code-btn">复制代码</span></code></pre>

<h2 data-id="heading-5">导入的方式</h2>
<p><a href="https://link.juejin.cn/?target=" target="_blank" title ref="nofollow noopener noreferrer"></a><a name="user-content-RnPZf" target="_blank" ref="nofollow noopener noreferrer" href="https://link.juejin.cn/?target=undefined"></a></p>
<h3 data-id="heading-6">import*</h3>
<p>一般我们导入的东西放在 <code>import &#123;...&#125;</code> 中</p>
<pre><code class="copyable">import &#123;play, eat&#125; from './life.js';
​
play('CF'); // play, CF!
eat('apple'); // eat, apple!
<span class="copy-code-btn">复制代码</span></code></pre>
<p>如果想要一次导入一个文件所有内容，<code>import * as<obj></code>将所有内容导入为一个对象</p>
<pre><code class="copyable">import * as life from './life.js';
​
life.play('CF');
life.eat('apple');
<span class="copy-code-btn">复制代码</span></code></pre>
<p>但是使用这种方法并不好，有以下几个原因</p>
<ol start="0">
<li>打包工具的打包逻辑是将所有模块打包到一起，然后进行优化，如果有未使用的模块，那么会将其删除，所以当我们全局引入的话，会影响打包工具的效率。</li>
<li>导入的显式列表，可以更清晰的显式代码的结构和位子，使维护更加简单。</li>
<li>明确导入的内容，会使名称更加整洁，清晰。</li>
</ol>
<pre><code class="copyable">//  life.js
function play(game) &#123;
  alert(`play, $&#123;game&#125;!`);
&#125;
​
function eat(food) &#123;
  alert(`eat, $&#123;food&#125;!`);
&#125;
​
export &#123;helloWorld, beyWorld&#125;; 
​
//index.js
import * as life from './life.js';
​
life.eat('apple');
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这里我们只用到了 eat ，但是我们全局引入了 <code>life.js</code>,所以打包的时候，会先引入<code>play</code>，<code>eat</code>，然后发现<code>play</code>没有用到，又把play删除掉。 <a href="https://link.juejin.cn/?target=" target="_blank" title ref="nofollow noopener noreferrer"></a><a name="user-content-LGjS7" target="_blank" ref="nofollow noopener noreferrer" href="https://link.juejin.cn/?target=undefined"></a></p>
<h3 data-id="heading-7">import "as"</h3>
<p>我们可以使用 as 让导入具有不同的名字。</p>
<pre><code class="copyable">import &#123;play as P, eat as E&#125; from './life.js';
​
P('CF'); // play, CF!
E('apple'); // eat, apple!
