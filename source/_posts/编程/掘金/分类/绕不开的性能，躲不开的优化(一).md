<h3 data-id="heading-1">1.1 浏览器强缓存，协商缓存</h3>
<p>这个是属于<code>HTTP</code>缓存，通过复用以前获取的资源，可以显著提高网站和应用程序的性能。<code>Web</code>缓存减少了等待时间和网络流量，因此减少了显示资源表示形式所需的时间。</p>
<p>如果没有缓存，浏览器就要直接请求服务器</p>
<p><strong><code>HTTP</code>缓存有2大类：</strong></p>
<ul>
<li>私有缓存，指只能用于单独用户，一般是浏览器的缓存</li>
<li>共享缓存，共享缓存可以被多个用户使用。例如，<code>ISP</code> 或你所在的公司可能会架设一个 <code>web</code>代理来作为本地网络基础的一部分提供给用户。这样热门的资源就会被重复使用，减少网络拥堵与延迟。</li>
</ul>
<p>常见的 HTTP 缓存只能存储 GET 响应，对于其他类型的响应则无能为力。</p>
<h4 data-id="heading-2">1.1.1 缓存控制</h4>
<p><strong><code>Cache-control</code> 头</strong>,即控制强缓存</p>
<p>强缓存，当浏览器去请求某个资源文件的时候，服务器就在<code>respone header</code>里面对文件做了缓存配置。缓存时间，缓存类型都由服务端控制。</p>
<p>大概模式就是</p>
<pre><code class="copyable">Response Headers
Cache-control:max-age=3600,private,no-cache
[key]:[value]
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>
<p><strong>重要的缓存请求指令</strong></p>
<ol>
<li><code>no-cache</code>：设置了这个，代表跳过强缓存，但是不妨碍协商缓存，每次请求都询问服务器</li>
<li><code>no-store</code>: 不缓存，服务器和客户端都不缓存</li>
<li><code>max-age</code>: 响应最大值，也就是在请求后多少时间内有效</li>
</ol>
</li>
<li>
<p><strong>重要的缓存响应指令</strong></p>
<ol>
<li><code>no-cache</code>缓存前必须确认其有效性</li>
<li><code>public</code>,向任意方提供响应</li>
<li><code>private</code>,仅向特定的用户返回响应</li>
<li><code>max-age</code>,设置资源过期时间</li>
<li><code>s-maxage</code>,设置公共缓存服务器资源过期时间</li>
</ol>
</li>
</ul>
<p>简单示例：</p>
<ol>
<li><code>cache-control:max-age:3600,public</code>, 强缓存，有效时间为3600，客户端和代理缓存服务器都可以缓存该资源</li>
<li><code>cache-control：max-age:3600,private</code>,强缓存，有效时间3600，只允许客户端缓存</li>
<li><code>cache-control:max-age:3000,immutable</code>，强缓存，有效时间3000，不询问服务器，即时用户做了刷新也不向服务器发起<code>http</code>请求</li>
</ol>
<h4 data-id="heading-3">1.1.2 协商缓存</h4>
<p>上面的强缓存，就是给资源设置一个过期时间。客户端每次请求资源都会看是否过期；只有过期才会询问服务器。</p>
<p>当资源过期了，就去请求服务器，这时候请求服务器的过程就可以设置协商缓存。</p>
<p>协商缓存，其实就是客户端和服务端的交互下的一种缓存</p>
<p><strong>基本过程：</strong></p>
<ul>
<li>一个陈旧的资源（缓存副本）是不会直接被清除或忽略的，当客户端发起一个请求时，缓存检索到已有一个对应的陈旧资源（缓存副本），则缓存会先将此请求附加一个<code>If-None-Match</code>头，然后发给目标服务器，以此来检查该资源副本是否是依然还是算新鲜的，则服务器返回 <code>304 (Not Modified)</code>。若服务器通过 <code>If-None-Match</code>判断后发现已过期，那么会带有该资源的实体内容返回。（该响应不会有带有实体信息），则表示此资源副本是新鲜的，这样一来，可以节省一些带宽。</li>
</ul>
<pre><code class="copyable">GET /index.html
If-None-Match:W/"225-tts73UY9XxnjshrvQS8tolQ7xSw"
...
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面这段意思就是如果<code>ETag</code>和我不相同，说明资源更新，请求服务器，如果相同返回<code>304</code>进行资源缓存重定向，和<code>If-None-Match</code>相反的是<code>If-Match</code></p>
<ul>
<li>当然服务器也能通过 <code>If-Modified-Since</code>判断资源是否过期。</li>
</ul>
<pre><code class="copyable">GET /index.html
If-Modified-Since: Thu, 15 Apr 2021 00:00:00 GMT
<span class="copy-code-btn">复制代码</span></code></pre>
<p>如果该资源在2021年4月15号之前未发生改变则资源重定向<code>304</code>,否则就向服务器获取该资源返回<code>200</code></p>
<p>如何设置呢？毕竟是前端，以<code>node.js</code>为例子</p>
<pre><code class="hljs language-javascript copyable" lang="javascript">res.setHeader(<span class="hljs-string">'max-age'</span>,<span class="hljs-string">'3600 public'</span>);
res.setHeader(etag,<span class="hljs-string">'sci4454885'</span>);
res.setHeader(<span class="hljs-string">'last-modified'</span>:Mon,<span class="hljs-string">'24 Dec 2020 05:04:00 GMT'</span>)

<h3 data-id="heading-4">1.2 使用CDN</h3>
<p>CDN指的是内容分发网络。其基本思路是尽可能的避开互联网上有可能影响数据传输速度和稳定性的瓶颈和环节，使内容传输的更快、更稳定。</p>
<p>可以将图片等静态资源放在CDN,减少了服务器的负载</p>
<h2 data-id="heading-5">2. 浏览器渲染阶段优化</h2>
<p><code>DOM</code>的修改会导致重排和重绘</p>
<h3 data-id="heading-6">2.1 重排</h3>
<p><code>重排</code>是指元素的位置或者大小发生了改变，浏览器需要重新去计算渲染树，新的渲染树建立之后，浏览器会重新绘制所影响的元素</p>
<p><strong>导致页面重排的一些操作（5小点）:</strong></p>
<ul>
<li>
<p><strong>内容改变：</strong> 文本改变或者图片的尺寸改变</p>
</li>
<li>
<p><strong><code>DOM</code>元素的几何属性改变：</strong> 元素的宽高改变时，原来渲染树中的<code>DOM</code>节点会失效，浏览器会根据<code>DOM</code>的变化重新构建渲染树中的相关节点。如果父元素的几何属性发生变化时，会导致子元素以及兄弟元素的位置发生变化，从而引起重排。</p>
</li>
<li>
<p><strong><code>DOM</code>树结构发生变化：</strong> 添加<code>DOM</code>节点，修改<code>DOM</code>节点的位置以及删除某个节点都会对渲染树进行修改，从而导致重排。浏览器的布局是自上而下的，修改当前元素不会都之前的元素造成影响，但是会对之后的元素造成影响，导致重排。</p>
</li>
<li>
<p><strong>获取某些属性时：</strong> 当获取一些属性值时，浏览器为了保证获取到正确的值也会引起重排。如元素的<code>offsetTop</code>，offsetLeft，<code>offsetHeight</code>，<code>offsetWidth</code>，<code>scrollTop</code>，<code>scrollWidth</code>，<code>scrollLeft</code>，<code>scrollHeight</code>、</p>
</li>
<li>
<p><strong>浏览器窗口尺寸发生改变时：</strong> 浏览器窗口的尺寸发生改变时，会导致所有元素的尺寸发生变化，从而导致重排。</p>
</li>
</ul>
<h3 data-id="heading-7">2.2 重绘</h3>
<p>重绘是指元素的样式发生了变化，而大小和尺寸没有发生改变</p>
<p><strong>重绘不一定引起重排，但是重排一定引起重绘</strong></p>
<h3 data-id="heading-8">2.3 基本优化手段</h3>
<ol>
<li>将多次样式改变属性合并成一次操作，样式属性的操作可以使用<code>class</code>代替</li>
<li>批量修改DOM，或者让元素脱离文本流，对其进行多重改变，然后将元素带回文档中</li>
<li>缓存布局信息，比如缓存<code>offsetTop</code>这些属性的信息，减少直接访问</li>
<li>对于动画较多的区域可以使用绝对定位，以此来减少回流带来整体影响</li>
</ol></div>
