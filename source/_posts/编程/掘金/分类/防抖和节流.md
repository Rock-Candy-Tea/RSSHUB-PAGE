h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h1 data-id="heading-0">一. 防抖(debounce)</h1>
<blockquote>
<pre><code class="copyable">在事件被触发n秒后再执行回调，如果在这n秒内又被触发，则重新计时。
<span class="copy-code-btn">复制代码</span></code></pre>
</blockquote>
<p>简单来说防抖就是如何有个点击事件，假设防抖间隔为2s，那点击一次后得2s后点击第二次才能有效，在2s内的点击都会使计时器重新计时。</p>

 *
 * @param &#123;*&#125; func[function] 要执行的方法
 * @param &#123;*&#125; wait[number] 等待执行的时间（用户自己规定多长时间内不算频繁的操作）
 * @param &#123;*&#125; immediate  疯狂点击的时候最终只识别一次，识别第一次还是最后一次，immediate为true，立即执行（第一次）
 * @return 可以被调用执行的函数
 */
    function myDebounce(handle, wait = 300, immediate = false) &#123;
      let timer = null
      return function (...res) &#123;
        let first = immediate && !timer
        clearTimeout(timer)
        timer = setTimeout(() => &#123;
          // 手动让其回归到初始状态(下一轮开始还是从timer为null开始），没加这句，只会执行一次，就没有了
          timer = null
          !immediate && handle.apply(this, res)
        &#125;, wait)

        first && handle.apply(this, res)
      &#125;
    &#125;

    // 定义事件执行函数
    function btnClick(ev) &#123;
      console.log('点击了1111', this, ev)
    &#125;

    // oBtn.onclick = btnClick  // this ev


    oBtn.onclick = myDebounce(btnClick, 2000, true)

  </script>

<h1 data-id="heading-1">二. 节流(throttle)</h1>
<blockquote>
<p>规定在一个单位时间内，只能触发一次函数。如果这个单位时间内触发多次函数，只有一次生效。</p>
</blockquote>
<p>还是以2s间隔为例，节流是点击之后2s后在点击也一定会执行，就算2s内你一直点也会。但防抖不会，防抖是要求你点击之后2s内不能在点击了，否则计时无效。</p>

     // 节流：我们这里的节流指的就是在自定义的一段时间内让事件进行触发
    /**
     * 函数节流：在一段频繁操作中，可以触发多次，但是触发的频率由自己指定
     * @param &#123;*&#125; func[function]：最后要触发执行的函数

     let lastTime = 0 ;
     return function(...res)&#123;

      if(now-lastTime>wait)&#123;

        fn.apply(this,res)

<h1 data-id="heading-2">三. 应用场景</h1>
<ul>
<li>
<p>防抖</p>
<p>search搜索联想，用户在不断输入值时，用防抖来节约请求资源。</p>
<p>登录、发短信等按钮避免用户点击太快，以致于发送了多次请求。</p>
<p>调整浏览器窗口大小时，resize 次数过于频繁，造成计算过多。</p>
<p>文本编辑器实时保存，当无任何更改操作一秒后进行保存。</p>
</li>
<li>
<p>节流</p>
<p>scroll 事件，每隔一秒计算一次位置信息等。</p>
<p>浏览器播放事件，每个一秒计算一次进度信息等。</p>
</li>
</ul></div>
