<p>很多企业的春招马上结束了，2022届毕业的同学如果想在秋招冲击更好的就业，
简历上写实习经验的话，要抓住这个机会。最近身边同学陆续接到面试，但本人水平有限，很是矛盾。但总是要面对。所以先来准备一套‘接化发‘，以供自学和大家参考。</p>
<ul>
<li>接：接受面试</li>
<li>化：化取知识</li>
<li>发：发现问题</li>
</ul>
<h3 data-id="heading-1">2. 接：鸡血注射</h3>
<p>我相信很多人跟我一样，看到网络上的面试题，看到许多不熟悉的却常考的知识点，看到自己一塌糊涂的项目等等都会觉得自己准备不够充分（不要对号入座）。现实可能确实如此，但是需要敢于面试，应为它能带给你在自己学习中得不到的东西。</p>
<ul>
<li>最重要的一点，<code>心里建设</code>：一次次的面试中能提升你的心理强度，抗打击能力，就如同恋爱，初恋时彼此的接触会让你心动不已，紧张十分。但到结婚后会觉得平淡如水。经过一次次的拷问，渐渐适应的心让你<code>更加冷静</code>，能想起更多<code>不应该忘记</code>的答案。如果心里素质实在比较低的话，可以先做一些平常会觉得紧张的事锻炼一下。不要太过盲目。心理建设越早越好</li>
<li>提升<code>表达能力</code>，这种表达能力可以包括你和面试官交流的方式，你回答问题的方式，用词，语气，速度，吐字等。首先要<code>礼貌客气</code>，语速均匀，正常，清晰标准的说话，你可以在自我介绍的第一个阶段后问问面试官自己的表达有没有让他不适的地方。语言适当生活化，营造良好的气氛。回答问题时，首先要<code>诚实肯定</code>，不应支支吾吾。分方面，分<code>熟悉程度和重要程度</code>的先后和分点回答。面试末时，你可以<code>询问面试官</code>关于你的表达方面的建议。在多次的面试中你的表达能力应该会提升。</li>
<li><code>查漏补缺，找到热点</code>：是的，也许你不用面试也知道自己的知识网残破不堪，但面试的经历会让你知道你真正的漏洞，不仅给你提供前所未知的名词，还可能拷问到你对自己颇为自信的某方面失去梦想。面试让你发现新大陆，让你发现自身立足的地基里的'偷工减料'。当你面试后，进行总结。这些经验可能会刻入DNA,映像深刻的知识点才能记得更久。只有总结才会让你亲身体验到一类公司，一类知识点的'热点'</li>
<li>激发学习动力：首先要问自己，<code>真的很努力在学习吗</code>？自己的努力有效果吗？面试可能真的会批到你体无完肤，知耻而后进。鞭策会驱使自己走的更远。（<em><strong>别骂了，人麻了</strong></em> 😭）</li>
</ul>
<p>在此分享一个方法论：<code>麦肯锡学习法：陈述问题(准备是否到达这个阶段？)->分析问题(知识的不足在哪，能力欠缺）->去除所有非关键问题（恋爱，游戏，多余的社交，心理畏惧）->复盘（面试结束后及时总结）**</code></p>
<h3 data-id="heading-2">3. 化：'五'连题</h3>
<p>这套面试题是我偶然得知，简单总结一下。</p>
<h5 data-id="heading-3"><strong>1. 写出几种会话跟踪用户的方法</strong></h5>
<p>一开始看到这个问题我蒙了，好像从未听过，连会话是什么也不知道。基础概念还是要记牢。</p>
<ol>
<li>会话是指客户端连接到服务器并发送请求与服务器响应该请求的<code>全过程</code>。一共<code>四种</code>常用方法。</li>
<li>第一种 <code>URL重写</code>：在URL结尾添加一个附加的会话ID以标识该会话</li>
<li>第二种 <code>隐藏表单域</code>：将会话ID添加到HTML表单元素中提交到服务器</li>
<li>第三种 <code>Cookie</code>：Cookie存储用户信息，每次发送请求的时候包裹cookie，识别用户</li>
<li>第四种 <code>session</code>:用户都有一个不同的session,服务器端创建session对象，产生sessionID来标识session对象，将sessionID放入到Cookie中发送到客户端</li>
</ol>
<ul>
<li>扩展：这里可能会引出cookie 的几种属性和网络安全，浏览器的四种存储的区别，浏览器缓存等问题。可以翻阅之前的文章。</li>
</ul>
<h5 data-id="heading-4"><strong>2.http的method有几种，以及各自含义。</strong></h5>
<p>非常多，可能记不全，分类记，尽量精简取益，根据英文猜中文，有的能对比。</p>
<p><code>获取资源类</code></p>
<ul>
<li>GET:请求页面资源，返回主体</li>
<li>POST：<code>带数据</code>(例如参数)请求，返回<code>处理</code>的资源信息</li>
<li>HEAD：请求页面，返回响应体的<code>首部</code></li>
</ul>
<p><code>操作类</code>：</p>
<ul>
<li>PUT :用客户端传给服务器的数据<code>替换</code>指定的URL里的内容</li>
<li>DELETE:请求服务器删除指定的页面</li>
<li>CONNECT:让服务器作为用户<code>代理</code>，访问其他页面（VPN）。不常考</li>
</ul>
<p><code> 信息类</code>：</p>
<ul>
<li>OPTIONS:允许客户端查看服务器性能</li>
<li>TRACE:协议调试方法,该方法会使服务器原样返回任意客户端请求的任何内容。</li>
</ul>
<p>扩展：这里可能会引出POST和GET的<code>异同</code>，Http协议的<a href="https://blog.csdn.net/aSuncat/article/details/109180153" target="_blank" rel="nofollow noopener noreferrer">相关问题</a>。</p>
<h5 data-id="heading-5"><strong>3. http请求报文中User-Agent和Referer的含义是什么</strong></h5>
<p>比较常考，报文字段的含义和作用，只能说，挑重点记</p>
<ol>
<li>
<p><code> Referer</code>:是HTTP协议中的一个<code>请求报头</code>，用于告知服务器用户资源的<code>来源页面</code>。比如说你从百度搜索结果中点击进入了某个页面，那么该次HTTP请求中的Referer就是搜索结果页面的地址。如果你的博客中引用了其他地方的一张图片，那么对该图片的HTTP请求中的Referer就是你那篇博客的地址。</p>
</li>
<li>
<p><code>User Agent</code>:中文名为<code>用户代理</code>，是Http协议中的一部分，属于头域的组成部分，User Agent也简称UA。它是一个特殊字符串头，是一种向访问网站提供你所使用的<code>浏览器类型及版本、操作系统及版本、浏览器内核、等信息的标识</code></p>
</li>
</ol>
<p>扩展：http请求报头字段<a href="https://blog.csdn.net/wang544831183/article/details/101521608?utm_medium=distribute.pc_relevant.none-task-blog-baidujs_title-0&spm=1001.2101.3001.4242" target="_blank" rel="nofollow noopener noreferrer">解释</a></p>
<h5 data-id="heading-6"><strong>4.如何进行网站性能优化</strong></h5>
<p>多，太多了，个个详细的介绍很难，对于这种问题，要<code>设计包袱</code>，引起面试官注意，加深对你的印象。简单介绍比较简单或不太清楚的，等他进一步询问的时候，仔细讲讲你精通的。记忆时学会分类。在这里只是初步介绍。</p>
<ul>
<li>减少请求和使用缓存类</li>
</ul>
<ol>
<li>减少http请求:   减少<code>http请求和其并发次数</code>，主要手段是<code>合并</code>CSS、合并javascript、合并图片。将浏览器一次访问需要的javascript和CSS合并成一个文件，这样浏览器就只需要一次请求。</li>
<li>使用<code>本地存储</code>进行缓存：Cookie ,LocalStorage,SessionStroage,IndexDB.它们的区别可以看看我的一篇文章</li>
<li>合理设置 HTTP缓存。<code>强缓存</code>：在 web 服务器返回的响应中添加 Expires 和 Cache-Control Header，<code>协商缓存</code>：通过【Last-Modified，If-Modified-Since】和【ETag、If-None-Match】这两对 Header 分别管理。</li>
</ol>
<ul>
<li>压缩类</li>
</ul>
<ol>
<li>启用压缩:在服务器端对文件进行压缩，在浏览器端对文件解压缩，可有效减少通信传输的数据量。如果可以的话，尽可能的将外部的脚本、样式进行合并，多个合为一个。webpack是一个典型的前端工具</li>
</ol>
<ul>
<li>图片类</li>
</ul>
<ol>
<li>css Sprites，base64格式：雪碧图使用一张图片，截取图片的各个部分来用。把图片格式转换成base64格式可以显著减少图片字节数</li>
<li>懒加载(重点，可能还要<code>手写</code>)：对于图片而言，在页面刚加载的时候可以只加载第一屏，当用户继续往后滚屏的时候才加载后续的图。主要思想是不直接使用img的src属性，用<code>data-src存放url</code>,判断图片是<code>否出现在视窗</code>,<code>监听滚动条</code>，出现则让<code>src=data-src</code>.关于详细讲解，掘金搜，找自己理解的即可。<a href="https://juejin.cn/post/6949540539456815140" target="_blank">例如</a>： @云伴风轻舞</li>
</ol>

<h5 data-id="heading-7"><strong>5.display,float,position 的关系</strong></h5>
<p>一开始看到，真想不到它们之间能有什么关系，再想想它们之间可能相近，也可以有排斥的</p>
<ol>
<li>
<p>display属性指定了一个DOM元素说使用的盒子模型（box）。即元素应该生成的框的类型；float 属性定义元素在哪个方向浮动，。浮动元素会生成一个块级框；position属性指出一个元素的定位方法</p>
</li>
<li>
<p>CSS框模型（盒子模型） 一切皆为框，没有display,就没有float和position，<code>display:none,float,position失效</code></p>
</li>
<li>
<p>float和positiond的互斥，<code>display非none,       position:absolute || fixed(绝对定位)，flex失效</code>， display属性的值根据下表自动计算。box的位置取决于<code>属性top，right，bottom，left以及box</code>的容器</p>
</li>
</ol>
<ul>
<li>display,position固定且不影响float时，float的值和元素是否为根元素来决定</li>
</ul>
<ol start="4">
<li>
<p>display非none，并且position非absolute、fixed，<code>float不为none</code>，那么box是浮动的</p>
</li>
<li>
<p>display非none，position非absolute、fixed，<code>float为none</code>,如果元素<code>是根元素</code>，display的值根据下表来计算。</p>
</li>
<li>
<p>display非none,并且position非absolute、fixed,float为none,<code>不是根元素</code>，box按照被指定的display的属性值来显示。</p>
</li>
</ol>

<h5 data-id="heading-8"><strong>6.什么是跨域问题以及如何解决</strong></h5>
<p>前端网络知识关于跨域的问题非常常见，是一个重点。</p>
<p>1.跨越的先决原因：<code>同源策略</code>是指协议，域名，端口相同。跨域含义：受浏览器同源策略的影响，不是同源的脚本不能操作其他源下面的对象。想要操作另一个源下的对象是就需要跨域。</p>
<p>解决方案：</p>
<ol>
<li>
<p><code> JSONP</code> 利用html的某些标签的src属性不受影响的特点，在请求的url后加参数callback=***，=后是一个你程序里接受服务器返回的一段被<code>callback&#123;&#125;包裹</code>的资源文本的处理函数名，该函数负责将文本处理为代码。</p>
</li>
<li>
<p><code>cors</code>：跨域资源共享（Cross-origin resource sharing）；实现CORS通信的关键是<code>服务器</code>。只要服务器实现了CORS接口，就可以跨源通信。</p>
</li>
</ol>
<ul>
<li>简单请求：head，get，post，浏览器发现这次跨源AJAX请求是简单请求，就自动在头信息之中，添加一个<code>Origin字段</code>。Origin字段用来说明本次请求来自哪个源。服务器根据这个值，决定是否同意这次请求。</li>
<li>复杂请求：put和delete 除Origin字段外，"预检"请求的头信息包括两个特殊字段。<code>Access-Control-Request-Method</code>，用来列出浏览器的CORS请求会用到哪些HTTP方法；<code>Access-Control-Request-Headers</code>，指定浏览器CORS请求会额外发送的头信息字段。</li>
</ul>
