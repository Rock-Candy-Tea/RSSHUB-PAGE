h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h2 data-id="heading-0">场景</h2>
<p>当我们请求某一个数据的具体内容的时候，我们可能需要在发起请求的时候传入该数据具体内容的 id 作为请求参数，以此匹配当对应的数据内容。</p>
<p>而这个 id ，也通常都是我们之前发送请求从服务端获取的</p>
<p>当我们的数据较多的时候，服务器返回给我们的 id 可能是一个很大的数字。</p>
<p>这个时候我们携带这个 id 去请求数据，会发现浏览器报错</p>
<pre><code class="copyable">Error: Request failed with status code 404
<span class="copy-code-btn">复制代码</span></code></pre>
<p>404: 表示服务器没有找到对应的资源，是我们的参数错了吗？</p>
<h2 data-id="heading-1">分析</h2>
<p>我们通过 浏览器 调试工具去查看我们之前获取的 id</p>
<p>后端返回给我们的数据，通常都是 <strong>JSON 格式的字符串</strong></p>
<pre><code class="hljs language-json copyable" lang="json">'&#123; <span class="hljs-attr">"id"</span>: <span class="hljs-number">9007199254740995</span>, <span class="hljs-attr">"name"</span>: <span class="hljs-string">"Jack"</span>, <span class="hljs-attr">"age"</span>: <span class="hljs-number">18</span> &#125;'
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们需要将这个字符串，用<code>JSON.parse()</code>转化成对象，才能供我们方便的使用数据</p>
<p>一般 axios 会在内部帮我们处理好数据直接返回给我们</p>
<p>那么现在我们将这个数据转化为 对象</p>
<pre><code class="hljs language-js copyable" lang="js">  <span class="hljs-keyword">var</span> res = <span class="hljs-built_in">JSON</span>.parse(
        <span class="hljs-string">'&#123; "id": 9007199254740995, "name": "Jack", "age": 18 &#125;'</span>
      )
      <span class="hljs-built_in">console</span>.log(res)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们发现输出结果</p>
<pre><code class="hljs language-js copyable" lang="js"> <span class="hljs-built_in">Object</span>: &#123;
        <span class="hljs-attr">age</span>: <span class="hljs-number">18</span>
        <span class="hljs-attr">id</span>: <span class="hljs-number">9007199254740996</span>
        <span class="hljs-attr">name</span>: <span class="hljs-string">"Jack"</span>
        <span class="hljs-attr">__proto__</span>: <span class="hljs-built_in">Object</span>
      &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这里解析出来的 id 比之前 服务器返回给我们的 id 要大1</p>
<p><strong>这是因为 JS 语言自身的原因：</strong></p>
<p>JavaScript 能够准确表示的整数范围在<code>-2^53</code>到<code>2^53</code>之间（不含两个端点），超过这个范围，无法精确表示这个值，这使得 JavaScript 不适合进行科学和金融方面的精确计算。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-built_in">Math</span>.pow(<span class="hljs-number">2</span>, <span class="hljs-number">53</span>) <span class="hljs-comment">// 9007199254740992</span>

<span class="hljs-number">9007199254740992</span>  <span class="hljs-comment">// 9007199254740992</span>
<span class="hljs-number">9007199254740993</span>  <span class="hljs-comment">// 9007199254740992</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>所以当我们再用这个 被解析错误的 id 再去发起请求，服务器自然是找不到对应的资源。</p>
<h2 data-id="heading-2">解决</h2>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fgithub.com%2Fsidorares%2Fjson-bigint" target="_blank" rel="nofollow noopener noreferrer" title="https://github.com/sidorares/json-bigint" ref="nofollow noopener noreferrer">json-bigint</a> 是一个第三方包，它可以帮我们很好的处理这个问题。</p>
<h3 data-id="heading-3">使用步骤</h3>
<ul>
<li>
<p>安装</p>
<pre><code class="copyable">npm i json-bigint
<span class="copy-code-btn">复制代码</span></code></pre>
</li>
<li>
<p>导入</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">import</span> jsonBig <span class="hljs-keyword">from</span> <span class="hljs-string">'json-bigint'</span>
<span class="copy-code-btn">复制代码</span></code></pre>
</li>
<li>
<p>基本使用（具体看官方文档，这里只说怎么解决这个问题的使用）</p>
<pre><code class="hljs language-js copyable" lang="js"> <span class="hljs-keyword">var</span> overDate = <span class="hljs-string">'&#123;"value": 9007199254740993,"name": "booker"&#125;'</span>
      <span class="hljs-built_in">console</span>.log(jsonBig.parse(overDate))
<span class="copy-code-btn">复制代码</span></code></pre>
<p>返回一个对象</p>
<pre><code class="copyable">&#123;value: BigNumber, name: "booker"&#125;
&#123;
 name: "booker"

c: (2) [90, 7199254740993]
e: 15
s: 1

&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>通过这个方法解析的对象，过大的数字会被转化成一个对象<code>BigNumber</code></p>
<p>**注意：**我们可以直接用这个数作为参数，因为字符串拼接的途中会自动将对象转化成字符串类型</p>
<pre><code class="hljs language-js copyable" lang="js"> <span class="hljs-built_in">console</span>.log(<span class="hljs-string">'booker'</span> + res.value)
 <span class="hljs-comment">// booker9007199254740993</span>
<span class="copy-code-btn">复制代码</span></code></pre>
</li>
</ul>
