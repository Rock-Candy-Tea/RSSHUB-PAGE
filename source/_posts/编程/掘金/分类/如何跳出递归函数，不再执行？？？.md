<h3 data-id="heading-0">前言</h3>
<p>前几天看到一个面试题：</p>
<p><strong><code>请实现一个 find 函数，功能等同于 document.getElementById</code></strong></p>
<p>分析：</p>
<p>就是一个查找节点函数，然后节点是可以嵌套节点的，而你是不知道会嵌套几层，正常做法是使用<strong>递归</strong>查找，直到递归到最底层。</p>
<p>三下五除二，刷刷刷，代码写就出来了。</p>

<p>正常的做法应该是如果找到对应的节点后，就应该把跳出递归函数，不应该再循环了，这样子就可以解决以上两个问题。</p>

<h3 data-id="heading-2">跳出递归</h3>
<p><strong>那递归函数如何跳出？</strong></p>
<p><code>break</code>可以吗？？？</p>
<p>大家知道break是在for循环，while循环中使用，恰巧我们函数有用了for循环，但是我们函数不是普通的函数，它是递归函数，<strong>函数</strong>在内部调用自己，所以它会嵌套很多层函数，你break只对那一层函数有用，其他层函数还会照样执行。所以要是要有效果，必须得在最顶层break。这样break才能break掉整个函数。</p>
<p>大家先看看代码：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> findId = (<span class="hljs-function"><span class="hljs-keyword">function</span> (<span class="hljs-params">findNode</span>) </span>&#123;
  <span class="hljs-keyword">return</span> <span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">fn</span> (<span class="hljs-params">id, node = <span class="hljs-built_in">document</span>.body</span>) </span>&#123;
    <span class="hljs-keyword">const</span> children = node.children
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> i = <span class="hljs-number">0</span>; i < children.length; i++) &#123;
      <span class="hljs-keyword">if</span> (children[i].id === id) findNode = children[i]
      <span class="hljs-keyword">if</span> (findNode = fn(id, children[i])) <span class="hljs-keyword">break</span>     
    &#125;
    <span class="hljs-keyword">return</span> findNode
  &#125;
&#125;)(<span class="hljs-literal">null</span>)
findId(<span class="hljs-string">'content-child'</span>)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>针对解法2，我们做了一些改进， 当我们利用递归找到findNode的时候，我们直接<code>break</code>，注意此时我们的break是在最顶层，这样子函数就直接执行return了，不会执行后面的函数了。</p>
<p>效果如下：</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4ac8a9743042438ca391662dd12bef1e~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>当然我们还可以把<code>break</code>改成使用<code>return</code>。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> findId = (<span class="hljs-function"><span class="hljs-keyword">function</span> (<span class="hljs-params">findNode</span>) </span>&#123;

      <span class="hljs-keyword">if</span> (children[i].id === id) <span class="hljs-keyword">return</span> children[i]
      <span class="hljs-keyword">if</span> (findNode = fn(id, children[i])) <span class="hljs-keyword">return</span> findNode
    &#125;
  &#125;

<p>这里原理类似，只要找到findNode，就直接在顶层<code>return</code>。函数就不会执行后面的了。</p>
<p><strong>还有其它解法吗？</strong></p>
<p>有。</p>
<p>当我们找到节点的时候，使用抛出异常，来中止递归函数。</p>
<p>以下是代码部分：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> findNode = <span class="hljs-literal">null</span>
<span class="hljs-keyword">try</span> &#123;

      &#125;
      findId(id, children[i])
    &#125;
  &#125;
  findId(<span class="hljs-string">'content-child'</span>)
&#125; <span class="hljs-keyword">catch</span> (error) &#123;
  <span class="hljs-built_in">console</span>.log(findNode)
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>不过我不是很推荐这种方式，因为需要在外面定义一个变量，然后还要写try-catch。</p>
<h3 data-id="heading-3">总结</h3>
<p>以上就是自己总结的如何跳出递归函数，一般有三种方法可以实现：</p>
<ul>
<li>break</li>
<li>return</li>
<li>抛出异常（不推荐）</li>
</ul>
