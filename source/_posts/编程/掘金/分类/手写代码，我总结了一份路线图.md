<h2 data-id="heading-0">准备工作</h2>
<h3 data-id="heading-1">API</h3>
<p>作为一个工作过三年以上的老前端而言，都会明白一个事情:  <strong>API的设计比实现更为重要</strong>。</p>
<p>何解？</p>
<p>如 <code>compose</code> 函数常用在各种中间件设计中，如 <code>redux</code> 等。<code>redux</code> 函数的实现极为简单，甚至一行就能实现，但是能够第一个想到 <code>compose</code> 的更不容易。</p>
<p>因此前端面试中的许多面试题以 ES API 与 lodash API 的模拟实现为主，因此在手写代码前需对 <code>lodash</code> 与 <code>ES6+</code> 文档较为熟悉。</p>
<h3 data-id="heading-2">代码规范</h3>
<p>在面试过程中考察代码，除了可以考察候选人的逻辑能力，其次，可查看候选人的代码能力，比如</p>
<ol>
<li>是否有一致的代码规范</li>
<li>是否有清晰可读的变量命名</li>
<li>是否有更简介的代码</li>
</ol>
<p>对于优雅代码的养成能力可以查看 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fgithub.com%2Fryanmcdermott%2Fclean-code-javascript" target="_blank" rel="nofollow noopener noreferrer" title="https://github.com/ryanmcdermott/clean-code-javascript" ref="nofollow noopener noreferrer">Clean Code concepts adapted for JavaScript.</a>，在 Github 上拥有 50+ K的星星数。</p>
<p>比如关于命名优化</p>
