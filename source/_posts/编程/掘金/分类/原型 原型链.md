<p>js分为<strong>函数对象</strong>和<strong>普通对象</strong>，每个对象都有__proto__属性，但是只有函数对象才有prototype属性</p>
</li>
<li>
<p>Object、Function都是js内置的<strong>函数</strong>, 类似的还有我们常用到的Array、RegExp、Date、Boolean、Number、String</p>
</li>
</ul>
<h2 data-id="heading-0">prototype是什么</h2>
<p>是函数所独有的，从一个函数指向一个对象。</p>
<p>它的含义是函数的原型对象，也就是 <strong>用这个构造函数创建的实例</strong> 的原型对象</p>
<h2 data-id="heading-1">__proto__是什么</h2>
<h2 data-id="heading-2">几个规则</h2>
<ul>
<li>所有的引用类型，都具有对象特性，即可自由扩展属性（除了<code>null</code>以外）</li>
<li>所有的引用类型，都有一个隐式原型 <code>__proto__</code> 属性，属性值是一个普通的对象</li>
<li>所有的引用类型，隐式原型 <code>__proto__</code> 属性值指向它的构造函数的显式原型 <code>prototype</code> 属性值</li>
<li>当试图得到一个对象的某个属性时，如果这个对象本身没有这个属性，那么它会去它的隐式原型 <code>__proto__</code>（也就是它的构造函数的显式原型 <code>prototype</code> ）中寻找</li>
</ul>
<blockquote>
<p>引用类型：常见的 数组，对象，函数，还包含Date、RegExp。</p>
</blockquote>

<h3 data-id="heading-5">规则三</h3>
<p>所有的引用类型，隐式原型 <code>__proto__</code> 属性值指向它的构造函数的显式原型 <code>prototype</code> 属性值</p>
<pre><code class="copyable">// obj是由Object函数构建出的对象，由此可以证明 obj.__proto__ 指向的是构造它的函数的显式原型
console.log(obj.__proto__ === Object.prototype) // true

// arr是由Array函数构建出的对象
console.log(arr.__proto__ === Array.prototype) // true

// obj是由Function函数构建出的对象
console.log(fn.__proto__ === Function.prototype) // true
<span class="copy-code-btn">复制代码</span></code></pre>
<h3 data-id="heading-6">规则四</h3>
<p>当试图得到一个对象的某个属性时，如果这个对象本身没有这个属性，那么它会去它的隐式原型 <code>__proto__</code>（也就是它的构造函数的显式原型 <code>prototype</code> ）中寻找</p>
<pre><code class="copyable">console.log(obj) // &#123;a: 100&#125;
console.log(Object.toString()) // "function Object() &#123; [native code] &#125;"
console.log(obj.toString()) // "[object Object]"
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>obj</code> 对象并没有 <code>toString</code> 属性，之所以能获取到 <code>toString</code> 属性，是遵循了第四条规则，从它的构造函数 <code>Object</code> 的 <code>prototype</code> 里去获取。</p>
