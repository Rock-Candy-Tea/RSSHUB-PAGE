h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h2 data-id="heading-0">前言</h2>
<p>给定一个矩阵和一个字符串，如何从矩阵中寻找出这个字符串在矩阵中的路径？本文就跟大家分享下如何使用回溯法来解决这个问题，欢迎各位感兴趣的开发者阅读本文。</p>
<h2 data-id="heading-1">实现思路</h2>
<p>我们先从题目给出的条件入手，逐步分析得出思路，矩阵就是一个二维数组，字符串可以切割成一个数组，我们要做的就是按顺序取出字符串中的每个字符，判断其是否在矩阵中，能否组成一条完整的路径出来。</p>
<h3 data-id="heading-2">举例分析</h3>
<p>现有一个矩阵（如下所示），有一个字符串<code>bfce</code>，我们需要从矩阵中找出这个字符串在矩阵中所连接起来的路径。</p>
<pre><code class="hljs language-bash copyable" lang="bash">a  b  t  g
c  f  c  s
j  d  e  h
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们从矩阵的<code>[0][0]</code>位置作为入口开始寻找，要查找的第1个字符为<code>b</code>：</p>
<ul>
<li>0,0 位置的元素是<code>a</code>，与目标值不匹配，继续寻找0,1位置</li>
<li>0,1 位置的元素是是<code>b</code>，与目标值匹配，继续查找第2个字符<code>f</code>
<ul>
<li>更新寻找方向，向下查找</li>
</ul>
</li>
<li>1,1 位置的元素是<code>f</code>，与目标值匹配，继续查找第3个字符<code>c</code>
<ul>
<li>更新寻找方向，向下查找</li>
</ul>
</li>
<li>2,1 位置的元素是<code>d</code>，与目标值不匹配，回到上一步1,1位置
<ul>
<li>更新寻找方向，向上查找，</li>
<li>0,0位置的值已经寻找过了，回到上一步1,1位置</li>
<li>更新寻找方向，向右查找</li>
</ul>
</li>
<li>1,2 位置的元素是<code>c</code>，与目标值匹配，继续查找第4个字符<code>e</code>
<ul>
<li>更新寻找方向，向下查找</li>
</ul>
</li>
<li>2,2 位置的元素是<code>e</code>，与目标值匹配，所有字符寻找完毕，该路径存在与矩阵中</li>
</ul>
<p>保存每一步已找到元素在矩阵中的索引</p>
<ul>
<li><code>[2,2]</code>位置</li>
<li><code>[1,2]</code>位置</li>
<li><code>[1,1]</code>位置</li>
<li><code>[0,1]</code>位置</li>
</ul>
<p>最终路径为：<code>[0][1]、[1][1]、[1][2]、[2][2]</code></p>
<h3 data-id="heading-3">思路分析</h3>
<p>通过上述举例，我们可以总结出下述思路：</p>
<ul>
<li>寻找一个切入点，从第一个字符开始寻找其在矩阵中的位置</li>
<li>进入矩阵后，每一步都会有4个移动方向：下、上、右、左</li>
<li>每移动一个方向，都会判断移动后位置的值是否与当前要查找的字符是否相等
<ul>
<li>如果相等，则标识当前位置的元素为已访问状态，沿着四个移动方向继续寻找下一个字符</li>
<li>如果不相等，则回到上一步的位置点，尝试其他的三个方向是否有匹配的元素</li>
</ul>
</li>
<li>重复步骤3，直至所有匹配字符的四个方向都被移动
<ul>
<li>字符串中的全部字符都被找到后，则取出每一步的正确索引位置将其保存起来</li>
<li>四个方向都被移动后，仍未找到与字符所匹配的元素，则证明该字符串不存在于矩阵中</li>
</ul>
</li>
</ul>
<blockquote>
<p>注意：我们在矩阵中找到与目标字符匹配的元素后，我们需要将这个位置的元素先存起来，然后再改成<code>.</code> 用于标识这个元素已经访问过了，当所有元素找到后再将存储起来的值进行还原。</p>
</blockquote>
<h2 data-id="heading-4">实现代码</h2>
<p>我们分析出思路后，接下来我们来看下实现代码，代码分为2部分：</p>
<ul>
<li>主函数，用于参数规则判断、寻找切入点、返回找到的路径</li>
<li>寻找路径函数，用于在矩阵中寻找每一个字符</li>
</ul>
<h3 data-id="heading-5">主函数</h3>
<p>主函数接受2个参数：路径矩阵、目标字符串</p>
<ul>
<li>我们需要先对参数进行判空</li>
<li>遍历矩阵从<code>0,0</code>位置开始寻找路径</li>
<li>路径找到则返回路径索引，否则返回目标路径不存在</li>
</ul>

<h3 data-id="heading-6">寻找路径函数</h3>
<p>寻找路径函数接受5个参数：路径矩阵、目标字符串、要寻找的行、要寻找的列、要寻找的字符索引</p>
<ul>
<li>首先，我们需要判断下要寻找的行、列是否超越矩阵的界限</li>
<li>矩阵中要寻找的行、列位置的元素与要寻找的字符不相等则直接返回<code>false</code></li>
<li>判断所有字符是否都查找完成
<ul>
<li>完成的话则存储行、列索引，返回true</li>
<li>未完成则保存当前行、列处的值、修改该位置的值为<code>.</code>用于标识为已访问状态</li>
</ul>
</li>
<li>从当前坐标点位置沿着其四个方向：下、上、右、下进行查找</li>
<li>查找完成后保存已找到字符的坐标点，还原当前位置所保存的值</li>
</ul>
<p>代码实现如下：</p>
<pre><code class="hljs language-typescript copyable" lang="typescript">  <span class="hljs-keyword">private</span> findPath(

    <span class="hljs-comment">// 保存当前坐标值</span>
    <span class="hljs-keyword">const</span> tmp = matrix[row][col];

    &#125;
    <span class="hljs-comment">// 递归完成，复原当前坐标</span>
