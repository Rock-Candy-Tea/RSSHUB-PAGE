<h1 data-id="heading-0">2021 年 Angular vs. React vs. Vue 前端框架对比</h1>
<blockquote>
<p>一个是 UI 库（React），另一个是成熟的前端框架（Angular），而其中最年轻的（Vue）则可以称之为渐进式框架。</p>
</blockquote>
<p>你可能是一个 React 开发者，可能是一个 Vue 开发者，也可能是一个只专注于技术的 Angular 开发者。不过，你还是不能忽视开发社区不断进行的框架比较。有充分的理由认为：他们三个框架，一个是 UI 库（React），另一个是成熟的前端框架（Angular），而其中最年轻的（Vue）则可以称之为渐进式框架。每一个框架都拥有一些独特的优势和性能指标。正是这些不可忽略的优势和指标，我们不能不对这三种框架进行对比分析。</p>
<p>这几个框架都是基于组件的框架，都有快速创建 UI 的功能。大部分时间，它们可以相互替代来用于构建前端应用。然而它们并非 100% 相同。</p>
<p>这就是为什么我们决定为你创建一个简短的指南，但最重要的是，为你提供一个参考帮助你在未来进行技术选择。</p>
<p>让我们开始吧！</p>
<h2 data-id="heading-1">许可证</h2>
<p>当然，在使用一个开源框架或库之前，一定要彻底检查许可证。幸运的是，React、Angular 和 Vue 都使用 <a href="https://opensource.org/licenses/MIT" target="_blank" rel="nofollow noopener noreferrer">MIT 许可证</a>。它提供了有限的复用限制，而且我们甚至还可以在专有软件中使用。在使用任何框架或软件之前，一定要留心，注意了解许可证的内容。</p>
<h2 data-id="heading-2">架构设计</h2>
<h3 data-id="heading-3">Angular</h3>
<p>Angular 框架属于 MEAN 框架，是如今创业公司最热门的技术栈。Angular 是一个完整的基于 TypeScript 的 Web 应用开发框架，主要用于构建单页 Web 应用（SPA）。</p>
<p>与 AngularJS 这一早期的框架不同，Angular2 是基于组件的，与 MV* 模式没有什么关联。Angular 的结构方式包括模块、组件和服务。</p>
<p>在 Angular 框架中，每个组件都有一个类或模板，定义了应用逻辑和 MetaData（装饰器）。组件的这些元数据为创建和呈现其视图所需的构件在哪里提供了指引。</p>
<p>Angular 架构的另一个重要因素是，模板是用 HTML 编写的。它们还可以包含 Angular 模板语法，并带有特殊指令以输出响应式数据，并且可以渲染多个元素。</p>
<p>服务 —— Angular 应用中的一个独特元素，被 Components 用于委托业务逻辑任务，如获取数据或验证输入。虽然使用服务并没有严格执行，但是将应用程序结构作为一组可复用的不同服务则是比较明智的。</p>
<h3 data-id="heading-4">React</h3>
<p>React 是一个开源的前端库，主要用于开发用户界面。这种灵活的前端解决方案并不强制执行特定的项目结构。一个 React 开发者可能只需要几行代码就可以开始使用它。</p>
<p>React 是基于 JavaScript 的，但在大多数情况下，它与 JSX（JavaScript XML）相结合。JSX 是一种语法扩展，允许开发人员同时创建包含 HTML 和 JavaScript 的元素。实际上，开发者可以用 JSX 创建的任何东西也可以用 <code>React JavaScript API</code> 创建。React 元素比 DOM 元素更强大，它们是 React 应用的最小组成部分，即组件。</p>
<p>React 组件是一种构建模块，它决定了在整个 Web 应用中使用独立和可重用的组件。</p>
<h3 data-id="heading-5">Vue</h3>
<p>用于开发用户界面和单页 Web 应用，Vue 是一个开源的 <code>Model-View-View-Model</code> (MVVM) 前端 JavaScript 库。它被称为渐进式框架，与其它工具一起被用于前端开发。Vue 的多用途、高性能和它在 Web 应用程序上的最佳用户体验成就了它的流行。</p>
<p>使用 Vue 时，开发者主要在 ViewModel 层上工作，以确保应用数据的处理方式能让框架呈现最新的视图。</p>
<p>Vue 的模板语法将可识别的 HTML 与特殊的指令和功能相结合。该语法允许开发人员创建 View 组件。</p>
<p>现在 Vue 中的组件是小巧、自成一体和可复用的。单文件组件（SFC）使用扩展名 <code>.vue</code> ，包含 HTML、JavaScript 和 CSS，因此所有相关代码都存放在同一个文件中。</p>
<p>在大型的 Vue.js 项目中，我们通常推荐使用 SFC 来组织代码。要将 SFC 移植到工作的 JavaScript 代码中，你需要 Webpack 或 Browserify 这样的构建工具。</p>
<h2 data-id="heading-6">适用目标和范围</h2>
<h3 data-id="heading-7">Angular</h3>
<p>Angular 最适合大型和高级项目。这些可能包括但不限于：</p>
<ul>
<li>用于开发渐进式 Web 应用程序（PWA）。</li>
<li>用于重新设计网站应用程序。</li>
<li>用于建立基于内容的动态网页设计。</li>
<li>用于创建有着复杂基础架构的大型企业应用程序。</li>
</ul>
<h3 data-id="heading-8">React</h3>
<p>React 来自 MERN 架构，一种以构建复杂的业务应用程序而闻名的技术架构。当将它与 Redux、MobX 或其它 flux 模式的状态管理库一起使用时，React 就能够成为强大的工具。React 最适合以下项目：</p>
<ul>
<li>对于涉及包含导航项，折叠或展开的手风琴分节，可用或不可用状态，动态输入，可用或不可用按钮，用户登录，用户访问权限等的许多组件的应用程序。</li>
<li>对于具有扩展和增长可能的项目，因为 React 组件具有声明性，因此它可以轻松处理此类复杂结构。</li>
<li>当 UI 是网络应用程序的中心时。</li>
</ul>
<h3 data-id="heading-9">Vue</h3>
<p>因为 Vue 具有可接受且快速的学习曲线，Vue 最适合解决短期的小型的问题。它可以轻松地与现有代码块集成。在以下情况下可能需要 Vue：</p>
<ul>
<li>你需要带有动画或交互式元素的 Web 应用程序的开发项目。</li>
<li>无需高级技能即可进行原型制作。</li>
<li>需要与多个其他应用程序无缝集成的应用程序。</li>
<li>更早推出 MVP。</li>
</ul>
<h2 data-id="heading-10">性能和开发</h2>
<h3 data-id="heading-11">Angular</h3>
<p>Angular 性能方面的一些亮点包括：</p>
<ul>
<li>有无缝的第三方集成，以增强产品或应用程序的功能。</li>
<li>提供强大的组件集合，从而简化了编写，更改和使用代码的过程。</li>
<li>它的“提前编译器”赋予了应用程序更快的加载时间和安全性。</li>
<li>MVC 模型通过允许视图分离来帮助减少后台查询。</li>
<li>促进使用将依赖项注入的外部元素来让组件解耦，从而为可复用性以及简化管理和测试铺平了道路。</li>
<li>通过将任务分成逻辑块来减少网页的初始加载时间。</li>
<li>可以完全自定义的设计。</li>
<li>便于将 HTML 和 TypeScript 编译为 JavaScript —— 大大加快了代码的编译速度，并将编译提早到远早于浏览器开始加载 Web 应用程序之前。</li>
</ul>
<h3 data-id="heading-12">React</h3>
<p>在性能方面 React 与 Vue 不相上下，因为两者具有相同的架构，即与 DOM 的交互。React 开发 Web 的性能可以评估如下：</p>
<ul>
<li>支持打包和 tree-shaking —— 这对于减少最终用户的资源负载至关重要。</li>
<li>由于提供了单向数据绑定支持，因此可以更好地控制项目。</li>
<li>便于进行测试和监控管理。</li>
<li>最适合需要频繁更改的复杂应用程序。</li>
</ul>
<h3 data-id="heading-13">Vue</h3>
<p>最贴切的形容 Vue 的词组是“令人难以置信的快速”。它的一些性能指标是：</p>
<ul>
<li>更快的学习曲线。</li>
<li>单页应用程序高效精密。</li>
<li>高级功能使它具有多功能性。</li>
</ul>
<h2 data-id="heading-14">各自的优点</h2>
<h3 data-id="heading-15">Angular</h3>
<p>有对模板、表单、引导程序或架构、组件以及组件之间交互的完整的文档：</p>
<ul>
<li>平滑的双向数据绑定。</li>
<li>MVC 架构。</li>
<li>内置模块系统。</li>
<li>大大减少了网页的初始加载时间。</li>
</ul>
<p><strong>使用 Angular 构建的流行应用程序：</strong></p>
<p>Youtube TV | PayPal | Gmail | Forbes | Google Cloud</p>
<h3 data-id="heading-16">React</h3>
<ul>
<li>通过模块化的结构使其拥有灵活的代码，节省时间和成本。</li>
<li>助力复杂应用程序的高性能的实现。</li>
<li>使用 React 前端开发能够更容易去做代码维护。</li>
<li>支持适用于 Android 和 iOS 平台的移动端原生应用程序。</li>
</ul>
<p><strong>使用 React 构建的流行应用程序：</strong></p>
<p>Tesla | AirBnB | CNN | Nike | Udemy | Linked-in</p>
<h3 data-id="heading-17">Vue</h3>
<ul>
<li>它的体积小巧，便于安装和下载。</li>
<li>倘若我们正确利用，我们就可以在多处重用 Vue。</li>
<li>Vue.js 允许我们更新网页中的元素，而无需渲染整个 DOM，因为它是虚拟的 DOM。</li>
<li>需要较少的优化。</li>
<li>加速 Web 应用程序的开发，并允许大佬将模板到虚拟 DOM 与编译器分开。</li>
<li>经过验证的兼容性和灵活性。</li>
<li>不管应用程序的规模如何，代码库都不会变。</li>
</ul>
<p><strong>使用 Vue 构建的流行应用程序：</strong></p>
<p>Gitlab | Spendesk | Behance | 9Gag | Wizzair | Nintendo</p>
<h3 data-id="heading-18">社区支持与学习曲线</h3>
<p>React JS 库是由 Facebook 创建的，拥有大量的贡献者以及一个庞大的开发者社区，为各种问题贡献他们的解决方案。Angular 也有一个庞大的开发者社区，对最具挑战性和最怪异的案例都有解决方案。Vue 具有良好的生态系统，并具有 React 和 Angular 框架的所有特性。说到学习曲线，React 无疑是开发者能够最快学习和适应的，其次是 Vue 和 Angular。</p>
<h2 data-id="heading-19">本文总结</h2>
<p>从前端开发者的角度来看，React 将是 2021 年最快学会使用的框架，Vue 和 React 都是轻量级、直观的并且性能完美良好的框架。</p>
