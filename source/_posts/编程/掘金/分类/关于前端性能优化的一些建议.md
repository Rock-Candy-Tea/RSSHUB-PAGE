<h2 data-id="heading-0">编码</h2>
<h3 data-id="heading-1">避免全局查找</h3>
<p>使用全局变量和函数要比局部的开销更大，可以将全局变量保存在局部的变量中，再去调用.<br>
<strong>将在一个函数中多次调用的全局对象保存到局部变量总是没错的<sup>[1]</sup></strong></p>
<pre><code class="hljs language-javascript copyable" lang="javascript">    <span class="hljs-comment">// 全局查找 document</span>

<h3 data-id="heading-2">避免with语句</h3>
<p><code>with</code>语句相信大家都听说过。它会创建自己的作用域，因此会增加其中执行代码的作用域链长度，造成<code>with</code>语句中执行的代码会比外面执行的代码要慢。</p>
<h3 data-id="heading-3">避免不必要的属性查找</h3>
<p>能减少算法的复杂度要尽量减少，尽可能地多使用局部变量<strong>将属性查找替换为值查找</strong>。</p>
<pre><code class="hljs language-javascript copyable" lang="javascript">    <span class="hljs-comment">// 代码1</span>
    <span class="hljs-keyword">var</span> a = <span class="hljs-number">1</span>;
    <span class="hljs-keyword">var</span> b = <span class="hljs-number">100</span> + a;
    <span class="hljs-built_in">console</span>.log(b);
    
    <span class="hljs-comment">// 代码2</span>
    <span class="hljs-keyword">var</span> arr = [<span class="hljs-number">1</span>, <span class="hljs-number">100</span>];
    <span class="hljs-keyword">var</span> b = arr[<span class="hljs-number">0</span>] + arr[<span class="hljs-number">1</span>];
    <span class="hljs-built_in">console</span>.log(c);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面两端代码执行效率是一样的<br>
代码1它会进行4次常量查找，分别是：a，数字1，数字100，b，时间复杂度为<code>O(1)</code><br>
代码2的时间复杂度也是<code>O(1)</code></p>
<p>若是用对象声明变量查找，他会比在数组上访问变量时间更长，比如</p>
<pre><code class="hljs language-javascript copyable" lang="javascript">    <span class="hljs-keyword">var</span> obj = &#123; <span class="hljs-attr">a</span>: <span class="hljs-number">1</span>, <span class="hljs-attr">b</span>: <span class="hljs-number">100</span> &#125;;
    <span class="hljs-keyword">var</span> result = obj.b - obj.a;
    <span class="hljs-built_in">console</span>.log(result);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面代码访问变量的时间复杂度为<code>O(n)</code>，对象在查找时必须向<strong>原型链</strong>中搜索该属性，所以对象属性越多，查找时间越长。</p>
<h3 data-id="heading-4">简化循环终止条件</h3>

<h2 data-id="heading-8">优化DOM交互</h2>
<h3 data-id="heading-9">最小化现场更新</h3>
<p>浏览器中渲染界面是一个非常消耗性能的事情，所以减少频繁地插入DOM节点显得非常有必要。例如：</p>
<pre><code class="hljs language-javascript copyable" lang="javascript">    <span class="hljs-keyword">var</span> list = <span class="hljs-built_in">document</span>.getElementById(<span class="hljs-string">'list'</span>);
    
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> x = <span class="hljs-number">0</span>, max = <span class="hljs-number">10</span>, x < max; x++) &#123;
        <span class="hljs-keyword">var</span> li = <span class="hljs-built_in">document</span>.createElement(<span class="hljs-string">'li'</span>);
        li.innerHTML = x;
        list.appendChild(li);
    &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面的代码会向<code>list</code>重复插入10个节点，每次插入节点，浏览器都会重新计算一次页面的位置。消耗非常大的性能。解决的办法就是使用<code>createDocumentFragment</code></p>
<pre><code class="hljs language-javascript copyable" lang="javascript">    <span class="hljs-keyword">var</span> list = <span class="hljs-built_in">document</span>.getElementById(<span class="hljs-string">'list'</span>);
    <span class="hljs-keyword">var</span> fragment = <span class="hljs-built_in">document</span>.createDocumentFragment();
    
    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> x = <span class="hljs-number">0</span>, max = <span class="hljs-number">10</span>, x < max; x++) &#123;
        <span class="hljs-keyword">var</span> li = <span class="hljs-built_in">document</span>.createElement(<span class="hljs-string">'li'</span>);
        li.innerHTML = x;
        fragment.appendChild(li);
    &#125;
    list.appendChild(fragment);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>使用文档片段改进后，只有一次插入节点的操作，有效减少节点更新造成的性能消耗。<br>
除此之外，使用<code>innerHTML</code>也可以造成差不多的效果，但是字符串拼接会有一定的性能损失。</p>
<pre><code class="hljs language-javascript copyable" lang="javascript">    <span class="hljs-keyword">var</span> list = <span class="hljs-built_in">document</span>.getElementById(<span class="hljs-string">'list'</span>);

    <span class="hljs-keyword">for</span> (<span class="hljs-keyword">let</span> x = <span class="hljs-number">0</span>, max = <span class="hljs-number">10</span>, x < max; x++) &#123;
        html += <span class="hljs-string">`<li><span class="hljs-subst">$&#123; x &#125;</span></li>`</span>;

<h2 data-id="heading-11">优化启动性能</h2>
<ul>
<li>使用dns预获取文件</li>
<li>在需要异步执行的脚本使用 <code>defer</code> 或 <code>async</code> 属性加载。这可以让HTML解析器更高效地处理文档。</li>
<li>使用 <a href="https://juejin.cn/post/6971451348227194894" target="_blank">Web workers</a> 运行持续时间长的 JavaScript 代码逻辑</li>
<li>所有能并行的数据处理都应该并行化。不要一团接一团地处理数据，如果可能的话，同时处理它们</li>
<li>在你启动的HTML文件中，不要包含不会在关键路径下出现的脚本或样式表。只在需要时加载他们<sup>[2]</sup></li>
</ul>
