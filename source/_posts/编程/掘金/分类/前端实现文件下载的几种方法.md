<h1 data-id="heading-0">一、location.href</h1>
<p>对于一些浏览器无法识别的文件格式，可以直接再浏览器地址栏输入url即可触发浏览器的下载功能。对于单文件下载没有什么问题，但是如果下载多文件，点击过快就会重置掉前面的请求</p>
<p>适用场景：</p>
<ul>
<li>get请求</li>
<li>单文件下载</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-built_in">window</span>.location.href = url;
<span class="copy-code-btn">复制代码</span></code></pre>
<h1 data-id="heading-1">二、window.open</h1>
<p>和location.href类似</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-built_in">window</span>.open(url);
<span class="copy-code-btn">复制代码</span></code></pre>
<h1 data-id="heading-2">三、a标签</h1>
<p>直接下载仅适用于浏览器无法识别的文件。如果是浏览器支持的文件格式，如html、jpg、png、pdf等，则不会触发文件下载，而是直接被浏览器解析并展示，这种情况下，可以使用a标签下载文件，download属性可以设置文件名。适用于单文件下载，如果下载多文件，点击过快就会重置掉前面的请求。</p>
<p>适用场景：</p>
<ul>
<li>get请求</li>
<li>单文件下载</li>
<li>需要自定义文件名</li>
</ul>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">//写法1</span>

<h3 data-id="heading-9">通过# FileReader.readAsDataURL()下载</h3>
<p><strong><code>readAsDataURL()</code></strong> 方法会读取指定的 Blob 或 File 对象。读取操作为异步操作，当读取完成时，可以从onload回调函数中通过实例对象的result属性获取data:URL格式的字符串（base64编码），此字符串即为读取文件的内容，可以放入a标签的href属性中。</p>

<h3 data-id="heading-10">两者的区别</h3>
<ul>
<li>返回值</li>
</ul>
<p>FileReader.readAsDataURL(blob)可以得到一段base64的字符串<br>
URL.createObjectURL(blob)得到的是当前文件的一个内存url</p>
<ul>
<li>内存</li>
</ul>
<p>FileReader.readAsDataURL(blob)依照js垃圾回收机制自动从内存中清理
URL.createObjectURL(blob)存在于当前document内，清除方式通过revokeObjectURL()手动清除</p>
<ul>
<li>执行方式</li>
</ul>
<p>FileReader.readAsDataURL(blob)通过回调的方式f返回，异步执行<br>
URL.createObjectURL(blob) 直接返回，同步执行</p>
<ul>
<li>多个文件</li>
</ul>
<p>FileReader.readAsDataURL(blob)同时处理多个文件时，需要一个文件对应一个FileReader对象<br>
URL.createObjectURL(blob) 依次返回，没有影响</p>
<ul>
<li>优势对比</li>
</ul>
<p>URL.createObjectURL(blob)得到本地内存容器的<code>URL</code>地址，方便预览，需要注意手动释放内存的问题，性能优秀。<br>
FileReader.readAsDataURL(blob)可直接转为<code>base64</code>格式，直接用于业务</p></div>
