<p>逐帧动画是一种常见的动画形式（Frame By Frame），其原理是在“连续的关键帧”中分解动画动作，也就是在时间轴的每帧上逐帧绘制不同的内容，使其连续播放而成动画，下面来分析下 GIF 和雪碧图这两种实现方式优点和不足。</p>
<h2 data-id="heading-0">一、GIF</h2>
<p>GIF 是一种图形交换格式，它的大致原理是，图片由许多像素组成，每一个像素都被指定了一种颜色，而这些像素综合起来就构成一个图片并且支持文本控制，多图像叠加以及多帧图像的动画播放。</p>
<h3 data-id="heading-1">优点：</h3>
<ul>
<li>几乎所有主流浏览器都支持 GIF</li>
<li>实现成本比较低，目前很多工具都可以实现</li>
<li>支持动画，允许某些像素透明，体积小，可以更好的在网络上传播</li>
</ul>
<h3 data-id="heading-2">不足：</h3>
<ul>
<li>不支持半透明，支持颜色少，最多存储256色，不支持24bit彩色模式，图像比较复杂会有锯齿毛边</li>
<li>不支持控制播放、暂停、等交互操作</li>
<li>不停地播放 GIF 会引起页面周期性的绘制，CPU 和 GPU 需要不停地在工作，性能有损耗</li>
</ul>
<h3 data-id="heading-3">实现方式：</h3>
<ul>
<li>设计师提供 GIF</li>
<li>视频转 GIT</li>
</ul>

<h3 data-id="heading-5">GIF 的简单交互：</h3>
<p>如果需求要能实现暂停和播放，可以通过其它插件来支持，比如使用插件 <a href="https://github.com/krasimir/gifffer" target="_blank" rel="nofollow noopener noreferrer">gifffer</a> 来实现暂停和播放，具体原理是将GIF转成了canvas来实现一些交互操作，示例 <a href="http://krasimir.github.io/gifffer/" target="_blank" rel="nofollow noopener noreferrer">Demo</a></p>
<h2 data-id="heading-6">二、雪碧图</h2>
<p>CSS雪碧，即 CSS Sprite，是一种 CSS 图像合并技术，该方法是将多个图片合并到一张图片上，利用 CSS 或者 JS 来控制图片的定位来实现逐帧动画。</p>
<h3 data-id="heading-7">优点：</h3>
<ul>
<li>可减少加载图片对服务器的请求次数，降低服务器压力</li>
<li>可提高页面加载速度</li>
</ul>
<h3 data-id="heading-8">不足：</h3>
<ul>
<li>维护成本较高，后期图片改动频繁的话</li>
<li>开发成本较高，拼图和实现动画计算，图片位置的坐标调整比较麻烦</li>
</ul>
<h3 data-id="heading-9">实现方式：</h3>
<h4 data-id="heading-10">CSS3</h4>
<p>利用 CSS3 的 steps 属性切换雪碧图序列帧的方式形成动画效果，示例参考 <a href="http://201704.baidu.renhe.org.cn/" target="_blank" rel="nofollow noopener noreferrer">Demo</a></p>
<h4 data-id="heading-11">JS</h4>
<p>采用 JS 切换雪碧图序列帧的方式形成动画效果，示例参考 <a href="http://bbs.feidee.com/m/activity/childrenday/index.html" target="_blank" rel="nofollow noopener noreferrer">Demo</a></p>
