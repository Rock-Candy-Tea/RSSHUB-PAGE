h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h1 data-id="heading-0">1. 跨域</h1>
<h2 data-id="heading-1">1.1 基本概念</h2>
<ul>
<li>
<p>跨域请求：在浏览器上当前访问的网站向另一个网站发送请求获取数据的过程就是跨域请求。</p>
</li>
<li>
<p>跨域是<em><strong>浏览器</strong></em>的<a href="https://developer.mozilla.org/zh-CN/docs/Web/Security/Same-origin_policy" target="_blank" rel="nofollow noopener noreferrer">同源策略</a>决定的，是一个重要的浏览器<em><strong>安全策略</strong></em>，可以使用 <a href="https://developer.mozilla.org/zh-CN/docs/Glossary/CORS" target="_blank" rel="nofollow noopener noreferrer">CORS</a> 配置解除这个限制。</p>
</li>
</ul>
<blockquote>
<p>详细可查看<em><strong>MDN</strong></em>的 <a href="https://developer.mozilla.org/zh-CN/docs/Web/Security/Same-origin_policy" target="_blank" rel="nofollow noopener noreferrer">浏览器的同源策略</a></p>
</blockquote>

<h1 data-id="heading-6">3.nginx</h1>
<h2 data-id="heading-7">3.1 基本概念</h2>
<ul>
<li>Nginx (engine x) 是一个高性能的HTTP和反向代理web服务器，同时也提供了IMAP/POP3/SMTP服务。占有内存少，并发能力强。</li>
</ul>
<h2 data-id="heading-8">3.2 example</h2>
<p>页面：<a href="http://example.com/" target="_blank" rel="nofollow noopener noreferrer">example.com</a><br>
接口：<a href="http://api.example.com/" target="_blank" rel="nofollow noopener noreferrer">api.example.com</a><br>
在页面上请求接口不做任何配置的话，浏览器会提示跨域，因为违反了浏览器的同源策略。</p>
<h2 data-id="heading-9">3.3 通过反向代理来解决跨域</h2>
<p>可以通过路径来区分：<br>
<a href="http://example.com/" target="_blank" rel="nofollow noopener noreferrer">example.com</a>       # 请求页面<br>
<a href="http://example.com/api/" target="_blank" rel="nofollow noopener noreferrer">example.com/api/</a>  # 请求接口</p>
<pre><code class="hljs language-js copyable" lang="js">    # config
    server &#123;
      listen <span class="hljs-number">80</span>;
      server_name example.com;
      location / &#123;
        root   /;
        index  index.html index.htm;
      &#125;
      location /api/ &#123;
        proxy_pass http:<span class="hljs-comment">//api.example.com/api;</span>
      &#125;
    &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>把请求 <a href="http://example.com/api/" target="_blank" rel="nofollow noopener noreferrer">example.com/api/</a> 代理到 <a href="http://api.example.com/" target="_blank" rel="nofollow noopener noreferrer">api.example.com</a><br>
这样资源就算在同一域名下，不存在夸域的问题了。</p>
<h2 data-id="heading-10">3.4 通过配置header来解决跨域</h2>
<ul>
<li>主要是设置Access-Control-Allow-等相关属性，来让浏览器知道，网站允许哪些域名、哪种请求方式的请求。</li>
<li>浏览器遇到跨域请求是会先发送OPTIONS请求预检</li>
<li>这种设置header的方式，请求然会是跨域，但是因为服务端的允许，所以跨域请求是通畅的。</li>
</ul>
<blockquote>
<p>在api.example.com在的服务器做配置，允许来自<a href="http://example.xn--com-lb7ey31gf9ce3sqnifoki05ay8ce9b/" target="_blank" rel="nofollow noopener noreferrer">example.com网站的跨域请求资源</a></p>
</blockquote>
<pre><code class="hljs language-js copyable" lang="js">server &#123;
  listen       <span class="hljs-number">80</span>;
  server_name  api.example.com;
  
add_header <span class="hljs-string">'Access-Control-Allow-Origin'</span> ‘http:<span class="hljs-comment">//example.com’;   # 允许的origin， 可以为*，带cookie的请求不支持*</span>
add_header <span class="hljs-string">'Access-Control-Allow-Credentials'</span> <span class="hljs-string">'true'</span>;    # 为 <span class="hljs-literal">true</span> 可带上 cookie
add_header <span class="hljs-string">'Access-Control-Allow-Methods'</span> <span class="hljs-string">'GET, POST, OPTIONS'</span>;  # 允许请求方法
add_header <span class="hljs-string">'Access-Control-Allow-Headers'</span> <span class="hljs-string">'*'</span>;  # 可以具体设置
add_header <span class="hljs-string">'Access-Control-Expose-Headers'</span> <span class="hljs-string">'Content-Length,Content-Range'</span>;

        <span class="hljs-keyword">if</span> ($request_method = <span class="hljs-string">'OPTIONS'</span>) &#123;
            # OPTIONS 请求的有效期，在有效期内不用发出另一条预检请求
            add_header <span class="hljs-string">'Access-Control-Max-Age'</span> <span class="hljs-number">1728000</span>;   
            add_header <span class="hljs-string">'Content-Type'</span> <span class="hljs-string">'text/plain; charset=utf-8'</span>;
            add_header <span class="hljs-string">'Content-Length'</span> <span class="hljs-number">0</span>;
            <span class="hljs-keyword">return</span> <span class="hljs-number">204</span>;
&#125;
  
location / &#123;
    root   /;
            index  index.html index.htm;
&#125;
&#125;
