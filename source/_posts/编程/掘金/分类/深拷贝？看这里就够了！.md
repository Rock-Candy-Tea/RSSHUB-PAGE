<h1 data-id="heading-1">深拷贝、浅拷贝</h1>
<h2 data-id="heading-2">什么是引用、什么是深拷贝、什么是浅拷贝？</h2>
<ul>
<li>引用：内存中存储对象的指向</li>
<li>浅拷贝：内存中存储对象的第一层是新的对象</li>
<li>深拷贝：内存中存储一个新对象</li>
</ul>
<p>一头雾水吗？接着看</p>
<pre><code class="copyable">let a = &#123;x: 1&#125;

b.x = 2
console.log(a) // &#123;x: 2&#125;
console.log(b) // &#123;x: 2&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>是不是看到 b 的值修改了 a 也跟着变？我想我们很多时候并不想这样。比如：</p>
<pre><code class="copyable">let a = &#123;x: 1&#125;
let b = a
let c = b
let d = c
// ....
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们通常会用到多层级的引用，那么一个值的更新所有的变量跟着更新，恐怖如斯。</p>
<h2 data-id="heading-3">什么时候是深拷贝、什么时候是浅拷贝？</h2>
<p>一般情况下，引用类型直接赋值为引用、值类型赋值是深拷贝。</p>
<blockquote>
<p>什么是引用类型？</p>
<ul>
<li>object</li>
</ul>
<p>什么是基本类型（值类型）？</p>
<ul>
<li>string</li>
<li>number</li>
<li>boolean</li>
<li>symbol</li>
</ul>
<p>什么是空类型？</p>
<ul>
<li>undefined</li>
<li>null</li>
</ul>
<p>基本类型、引用类型、空类型 会在单独的文章张开讲。</p>
</blockquote>

<h1 data-id="heading-8">深拷贝</h1>
<h2 data-id="heading-9">JSON.stringify 实现深拷贝</h2>
<p>注意：JSON.stringify 实现深拷贝不是不能用，请在用之前确保自己知道其缺点。</p>
<blockquote>
<p>缺点：部分类型会导致出问题</p>
<ul>
<li>时间类型：时间对象变为字符串</li>
<li>RegExp：结果为空对象</li>
<li>Error：结果为空对象</li>
<li>Function：结果为函数丢失</li>
<li>undefined：结果为undefined丢失</li>
<li>NaN：结果为 null</li>
<li>Infinity：结果为 null</li>
<li>-Infinity：结果为 null</li>
<li>丢失对象的 constructor</li>
<li>对象如果存在循环引用，会爆炸</li>
</ul>
</blockquote>
<p>实现：</p>
<pre><code class="copyable">let a = &#123;x: 1, y: &#123;z: 2&#125;&#125;

console.log(b) // &#123;x: 1, y: &#123;z: 200&#125;&#125;
