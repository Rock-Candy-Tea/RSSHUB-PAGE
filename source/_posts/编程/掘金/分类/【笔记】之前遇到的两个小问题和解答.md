<h2 data-id="heading-3">2.如何使用git对项目进行管理</h2>
<ol>
<li>先安装 Git,</li>
</ol>
<p>Git 安装包下载地址<code>https://git-scm.com/downloads</code></p>
<ol start="2">
<li>基本信息设置</li>
</ol>
<p><code>git config --global user.name "lbj"</code>
<code>git config --global user.email "bingjin.liao@outlook.com"</code></p>
<p>注：该设置后，在github仓库主页显示谁提交了文件</p>
<ol start="3">
<li>在项目目录上，初始化一个新的Git仓库</li>
</ol>
<p><code>git init</code></p>
<ol start="4">
<li>常用git命令</li>
</ol>
<p><code>git add .</code>-----添加到暂存区</p>
<p><code>git commit -m '描述'</code>-----将文件从暂存区提交到仓库</p>
<p><code>git status</code>-----来查看状态</p>
<ol start="5">
<li>GIt管理远程仓库</li>
</ol>
<p>使用远程仓库的目的：备份、实现代码的共享集中化管理</p>
<ul>
<li>首先，在远程创建对应仓库，</li>
</ul>
<p>如在github中创建，创建后出现一个仓库的地址：<code>https://github.com/xxx/xxxx.git</code></p>
<ul>
<li>接着，在本地仓库设置（添加远程仓库），告诉本地仓库你所对应的远程仓库在哪。</li>
</ul>
<p>格式：<code>git remote add 远程名称 远程地址</code></p>
<p>如：<code> git remote add webstoreAtGithub https://github.com/jCodeLife/webstore.git</code></p>
<ul>
<li>然后，可以通过命令查看项目链接的远程仓库：</li>
</ul>
<p><code>git remote</code>   列出当前本地仓库所连接的所以远程仓库</p>
<p><code>git remote  -v</code>  会有详细信息</p>
<ul>
<li>接着，将本地仓库同步到git远程仓库中（上传代码）</li>
</ul>
<p><code>git push -u 远程名 分支名</code></p>
<p>如：git push -u webstoreAtGithub master</p>
<p>master表示默认的分支名</p>
<p>origin 表示默认远程仓库名称</p>
<ul>
<li>如果没有本地仓库，可以将将远程仓库克隆到本地，通过：</li>
</ul>
<p><code>git clone 仓库地址</code></p>
<ul>
<li>有时候需要拉取远程最新的文件，使用</li>
</ul>
<p><code>git pull</code>命令</p>
<h2 data-id="heading-4">3.为什么vue的路由正常跳转，但页面没有变化</h2>
<p>需求：点击注册按钮时，显示出注册页面</p>
<p>问题：地址正常跳转，但对应组件在对应位置没显示、也没报错？</p>
<pre><code class="hljs language-html copyable" lang="html"><span class="hljs-tag"><<span class="hljs-name">router-link</span> <span class="hljs-attr">to</span>=<span class="hljs-string">'./sign_up'</span>></span>
    <span class="hljs-tag"><<span class="hljs-name">el-button</span> <span class="hljs-attr">type</span>=<span class="hljs-string">"danger"</span> <span class="hljs-attr">size</span>=<span class="hljs-string">"small"</span>></span>注册<span class="hljs-tag"></<span class="hljs-name">el-button</span>></span>

<p>解决：通过反复测试，后面发现有一条不显眼的警告消息：</p>
<p>[vue-router] Non-nested routes must include a leading slash character. Fix the following routes:...</p>
<p>大概意思是：非嵌套路由必须包含一个前导斜杠字符。</p>
<p>于是我在在定义路由路径path处修改了一下，解决成功！即：<strong>在填写path时，前面需要将添加一个斜杠字符</strong></p>

     &#125;

<h2 data-id="heading-5">4.v-for中的:key报错了</h2>

<p>报错信息： [Vue warn]: Avoid using non-primitive value as key, use string/number value inst....</p>
<p>大概意思：避免使用非基值作为键，应使用字符串/数字值</p>
<p>原错误代码</p>
<pre><code class="hljs language-html copyable" lang="html"><span class="hljs-tag"><<span class="hljs-name">div</span> <span class="hljs-attr">id</span>=<span class="hljs-string">"main"</span>></span>

<p>在v-for遍历时，如果将遍历出来的值作为:key的值时，并且这个值恰巧是一个对象就会报错。</p>
