<h2 data-id="heading-1">定义</h2>
<p>单例模式，保证一个类仅有一个实例，并提供一个访问它的全局访问点。</p>
<h2 data-id="heading-2">自己的理解</h2>
<p>通常我们可以让一个全局变量使得一个对象被访问，但它不能防止实例化多个对象。一个最好的办法就是，让类自身负责保持它的唯一实例，且这个类保证没有其他实例可以被创建，并且它可以提供一个访问该实例的方法。</p>
<h2 data-id="heading-3">实现单例模式</h2>
<p>要实现一个简单的单例模式并不复杂，无非是用一个变量来缓存一个类实例化生成的对象，然后用这个变量来判断一个类是否已经实例化过。如何变量有值，则在下一次要获取该类的实例化生成的对象时，直接返回这个变量（之前实例化生成的对象）。</p>
<pre><code class="copyable">var Singleton = function(name) &#123;
    this.name = name;
    this.instance = null;
&#125;;
Singleton.prototype.init = function() &#123;
   //...初始化
&#125;;
Singleton.getInstance = function(name) &#123;
    if (!this.instance) &#123;
        this.instance = new Singleton(name);
        this.instance.init();
    &#125;
    return this.instance;
&#125;;
var a = Singleton.getInstance('a');
var b = Singleton.getInstance('b');
console.log(a === b); // true
<span class="copy-code-btn">复制代码</span></code></pre>
<p>正常来说一个类每次实例化生成的对象是不同的，在上述代码中分别使用<code>Singleton.getInstance</code>获取<code>Singleton</code>类实例化生成的对象并赋值给对象a和对象b，执行<code>console.log(a === b)</code>打印出<code>true</code>，说明上述代码实现单例模式成功。</p>
<p>或不把实例化生成的对象挂载到类的属性上也可以实现，代码如下所示：</p>
<pre><code class="copyable">var Sington = function( name )&#123;
 this.name = name;
&#125;;
Singleton.prototype.init = function() &#123;
   //...初始化
&#125;;
Singleton.getInstance = (function()&#123;
 var instance = null;
 return function( name )&#123;
 if ( !instance )&#123;
   instance = new Singleton( name );
   instance.init();
 &#125;
 return instance;
 &#125;
&#125;)(); 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>变量<code>instance</code>用来缓存类实例化生成的对象，用自执行匿名函数创建了一个闭包，把变量<code>instance</code>存储在闭包中，每次执行<code>Singleton.getInstance</code>时就可以用变量<code>instance</code>来判断<code>Singleton</code>类是否已经实例化过。</p>
<h2 data-id="heading-4">透明的单例模式</h2>
<p>上述实现单例模式的方法中有一个问题，就是增加了这个类的“不透明性”，<code>Singleton</code>类的使用者必须知道这是一个单例类，跟以往通过<code>new XXX</code>的方式来实例化类，这里要使用<code>Singleton.getInstance</code>来实例类。故要换一种方法来实现透明的单例模式。</p>
<pre><code class="copyable">const Singleton = (function() &#123;
    let instance;
    const Singleton = function(name) &#123;
        if (instance) &#123;
            return instance;
        &#125;
        this.name = name;
        this.init();
        return instance = this;
    &#125;;
    Singleton.prototype.init = function() &#123;
        //...初始化
    &#125;;
    return Singleton;
&#125;)();
const a = new Singleton('a');
const b = new Singleton('b');
console.log(a === b); // true

<span class="copy-code-btn">复制代码</span></code></pre>
<p>以上代码中，<code>Singleton</code>类的构造函数是一个自执行的匿名函数，会构成一个闭包，可以把判断类是否实例化过的和缓存类实例化生成的对象的变量<code>instance</code>缓存起来，最后在返回真正的<code>Singleton</code>类构造方法。那么就可以使用<code>new Singleton</code>来实例化<code>Singleton</code>类。</p>
<h2 data-id="heading-5">必须遵循设计模式的原则</h2>
<p>上述代码还不够完美，有两点：</p>
<ul>
<li>
<p>把<code>Singleton</code>类的真正的构造函数放在自执行的匿名函数中返回，有点奇怪。</p>
</li>
<li>
<p><code>Singleton</code>类的构造函数中违背了设计模式的单一职责原则，做了二件事情，创建类实例化生成的对象和执行初始化<code>init</code>方法，保证了只能实例化一次。</p>
</li>
</ul>
<p>实现任何设计模式都要遵循设计模式的原则，要把“创建类实例化生成的对象和执行初始化<code>init</code>方法”和“保证了只能实例化一次”这两个职责分隔开来。下面用代理实现单例模式。</p>
<pre><code class="copyable">const Singleton = function (name) &#123;
  this.name = name;
  this.init();
&#125;
Singleton.prototype.init = function () &#123;
  //...初始化
&#125;;
const PropxSingleton = (function () &#123;
  var instance;
  return function (name) &#123;
    if (!instance) &#123;
      instance = new Singleton(name);
    &#125;
    return instance;
  &#125;
&#125;)();
var a = new PropxSingleton('a');
var b = new PropxSingleton('b');
console.log(a === b);//true
<span class="copy-code-btn">复制代码</span></code></pre>
<p>以上用代理类<code>propxSingleton</code>实现了<code>Singleton</code>类的单例模式，而<code>Singleton</code>类仍旧是一个普通的类，不影响在其它地方的使用。</p>
<p>然后以上代码还不够完美，假如要给另外一个类实现单例模式，得去修改代理类<code>propxSingleton</code>的代码，不符合设计模式中的开放-封闭原则，得封装一个获取不同类的代理类<code>propxSingleton</code>的<code>getPropxSingleton</code>方法。</p>
<pre><code class="copyable">const Singleton = function (name) &#123;
  this.name = name;
  this.init();
&#125;
Singleton.prototype.init = function () &#123;
  //...初始化
&#125;;
const getPropxSingleton = function (customClass) &#123;
  return (function () &#123;
    var instance;
    return function (name) &#123;
      if (!instance) &#123;
        instance = new customClass(name);
      &#125;
      return instance;
    &#125;
  &#125;)()
&#125;;
const PropxSingleton = getPropxSingleton(Singleton);
var a = new PropxSingleton('a');
var b = new PropxSingleton('b');
console.log(a === b); //true
