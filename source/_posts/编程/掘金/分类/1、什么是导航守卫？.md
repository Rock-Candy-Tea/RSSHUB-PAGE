<ul>
<li>在进行路由切换的过程就是导航；</li>

<h6 data-id="heading-0">以下结合各路大神的总结，望海涵。</h6>
<h2 data-id="heading-1">一、导航守卫的分类</h2>
<h4 data-id="heading-2">1、有3种导航守卫</h4>
<ul>
<li>全局守卫</li>
<li>单个路由独享守卫</li>
<li>组件内的守卫</li>
</ul>
<h4 data-id="heading-3">2、都有3个参数，除了 afterEach守卫（后面详解）</h4>
<ul>
<li><strong>to：</strong>   要进入的目标路由</li>
<li><strong>from：</strong>   离开当前的路由</li>
<li><strong>next：</strong>  控制路由执行（必须要有），一定要调用该方法来 resolve 这个钩子</li>
</ul>
<h4 data-id="heading-4">3、next 4个行为：</h4>
<pre><code class="copyable">next();  // 钩子执行完了，可以进入下一个路由；导航守卫的状态:confirmed
next(false); //中断当前路由跳转行为
next('/'); next(&#123; path: '/' &#125;) // 根据路由跳转条件来进入相应的路由，进入一个新的导航
next( new Error ) // 中断路由，错误会被传递给 router.onError() 注册过的回调中
<span class="copy-code-btn">复制代码</span></code></pre>
<h2 data-id="heading-5">二、3种导航守卫详解</h2>
<h3 data-id="heading-6">1、全局路由守卫（有3个）</h3>
<ul>
<li>挂载到 router 实例上</li>
<li>用于控制路由的每一次跳转</li>
</ul>
<h4 data-id="heading-7">1.1、beforeEach：全局前置守卫</h4>
<pre><code class="copyable">const router = new VueRouter(&#123; ... &#125;)

router.beforeEach((to, from, next) => &#123;
  // to 代表去的路由 a => b  b就是去的路由
  // from 代表来自于哪个路由 a => b  a就是来自的路由
  // next 控制路由的跳转与取消跳转
      // 不加上 next ，页面跳转不了
      // next() 执行跳转
      // next(false) 取消跳转 a => b
      // next 重新定向到某个页面 next("/login")
&#125;)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>作用： 常用于登录验证</p>
<pre><code class="copyable">router.beforeEach((to, from, next) => &#123;
  if (to.name !== 'Login' && !isAuthenticated) next(&#123; name: 'Login' &#125;)
  else next()
&#125;)
<span class="copy-code-btn">复制代码</span></code></pre>
<h4 data-id="heading-8">1.2、beforeResolve：全局解析守卫</h4>
<p>你可以用 <code>router.beforeResolve</code> 注册一个全局守卫。这和 <code>router.beforeEach</code> 类似，区别是在导航被确认之前，<strong>同时在所有组件内守卫和异步路由组件被解析之后</strong>，解析守卫就被调用。</p>
<h4 data-id="heading-9">1.3、afterEach：全局后置钩子守卫</h4>
<p>这些钩子不会接受 <code>next</code> 函数也不会改变导航本身：</p>
<pre><code class="copyable">router.afterEach((to, from) => &#123;
  // ...
&#125;)
<span class="copy-code-btn">复制代码</span></code></pre>
<h3 data-id="heading-10">2、单个路由独享守卫（1个）</h3>
<h4 data-id="heading-11">2.1、 beforeEnter</h4>
<p>可以在路由配置上直接定义 <code>beforeEnter</code> 守卫；只有进入到当前页面才会执行的守卫函数；</p>
<pre><code class="copyable">const router = new VueRouter(&#123;
  routes: [
    &#123;
      path: '/foo',
      component: Foo,
      beforeEnter: (to, from, next) => &#123;
        // ...
      &#125;
    &#125;
  ]
&#125;)
<span class="copy-code-btn">复制代码</span></code></pre>
<h3 data-id="heading-12">3、组件内的守卫（3个）</h3>
<h4 data-id="heading-13">3.1、beforeRouteEnter</h4>
<ul>
<li>在渲染该组件的对应路由被 confirm 前调用（大概意思：进入之前调用）；</li>
<li><code>重点：</code>不能获取 this 实例。因为当守卫执行前，组件实例还没被创建；</li>
<li><code>顺序：</code>beforeEach 和 独享守卫 beforeEnter 之后，全局 beforeResolve 和 afterEach 之前调用；</li>
</ul>
<pre><code class="copyable">const Foo = &#123;
  template: `...`,
  beforeRouteEnter(to, from, next) &#123;
  &#125;,
 &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li><code>解决： beforeRouteEnter</code> 守卫 <strong>不能</strong> 访问 <code>this</code>，因为守卫在导航确认前被调用，因此即将登场的新组件还没被创建。可以通过传一个回调给 <code>next</code>来访问组件实例。在导航被确认的时候执行回调，并且把组件实例作为回调方法的参数。</li>
</ul>
<pre><code class="copyable">beforeRouteEnter (to, from, next) &#123;
  next(vm => &#123;
    // 通过 `vm` 访问组件实例
  &#125;)
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<h4 data-id="heading-14">3.2、beforeRouteUpdate</h4>
<ul>
<li>在当前路由改变，但是该组件被复用时调用；</li>
<li>由于会渲染同样的 Foo 组件，因此组件实例会被复用。而这个钩子就会在这个情况下被调用；</li>
<li>可以访问组件实例 <code>this</code></li>
</ul>
<pre><code class="copyable">const Foo = &#123;
  template: `...`
  beforeRouteUpdate(to, from, next) &#123;
  &#125;,
 &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<h4 data-id="heading-15">3.3、beforeRouteLeave</h4>
<ul>
<li>导航离开该组件的对应路由时调用</li>
<li>可以访问组件实例 <code>this</code></li>
</ul>
<pre><code class="copyable">beforeRouteLeave(to, from, next) &#123;
    // 导航离开该组件的对应路由时调用
    // 可以访问组件实例 `this`
  &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>beforeRouteLeave 使用场景：</code></p>
<ol>
<li>当组件内有定时器，需要清除时，路由切换以免导致内存占用；</li>
<li>将公用信息保存到 session 或 vuex 中；</li>
<li>当页面有未关闭的窗口，或有未保存的时候，该导航可以通过 <code>next(false)</code> 来取消阻止页面跳转，示例如下：</li>
</ol>
<pre><code class="copyable">beforeRouteLeave (to, from, next) &#123;
  const answer = window.confirm('Do you really want to leave? you have unsaved changes!')
  if (answer) &#123;
    next()
  &#125; else &#123;
    next(false)
  &#125;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<h3 data-id="heading-16">4、完整的导航解析</h3>
<ol>
<li>导航被触发。</li>
<li>在失活的组件里调用 <code>beforeRouteLeave</code> 守卫。</li>
<li>调用全局的 <code>beforeEach</code> 守卫。</li>
<li>在重用的组件里调用 <code>beforeRouteUpdate</code> 守卫 (2.2+)。</li>
<li>在路由配置里调用 <code>beforeEnter</code>。</li>
<li>解析异步路由组件。</li>
<li>在被激活的组件里调用 <code>beforeRouteEnter</code>。</li>
<li>调用全局的 <code>beforeResolve</code> 守卫 (2.5+)。</li>
<li>导航被确认。</li>
<li>调用全局的 <code>afterEach</code> 钩子。</li>
<li>触发 DOM 更新。</li>
<li>调用 <code>beforeRouteEnter</code> 守卫中传给 <code>next</code> 的回调函数，创建好的组件实例会作为回调函数的参数传入。</li>
</ol>
