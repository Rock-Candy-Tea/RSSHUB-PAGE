<p>接下来，阿宝哥将以文字稿的形式，总结一下《7 年技术写作，分享 6 点心得体会》PPT 分享的内容，将从 <strong>突破自己、如何选定写作主题、如何开始写作、写作技巧与写作建议和写作辅助⼯具</strong> 5 个方面展开介绍。</p>
<h3 data-id="heading-0">一、突破自己</h3>
<p>在刚开始写作的时候，阿宝哥没有搭建自己的博客平台，而是直接使用 Github.IO 来写博客。之后也“混”过多个主流的技术社区，但目前最主要的技术写作平台是掘金。我觉得人最大的敌人就是自己，写作的第一步就是要 <strong>突破自己</strong>。</p>
<p>在写作之初，相信一些小伙伴跟我一样，会有以下困惑：</p>
<ul>
<li>写的文章太简单了，觉得对别人没有启发；</li>
<li>写的文章太 “烂” 了，害怕会被别人吐槽；</li>
<li>文笔不好，不知道怎么下手；</li>
<li>想写的话，又不知道写什么内容；</li>
<li>辛辛苦苦写的文章，又没有人看。</li>
</ul>
<p>不知道你们在开始写作的时候是否也有以上的困惑，以上这几个问题曾困扰过我一段时间，之后在不断的写作过程中，我找到了自己的写作初衷 —— <strong>”做技术沉淀“</strong> ，于是写作也就慢慢坚持了下来。</p>
<p>我们公司前端团队中的一些小伙伴此前也有和我聊过一些写作方面的困惑，他们不知道写什么内容，也不太敢尝试写作。后面我就以技术分享的形式，来推动他们写作。我会定期选择一些技术分享专题，比如 <strong>设计模式专题、Webpack 专题和 Vue2 源码专题</strong>，然后让他们每个人都去准备对应的分享内容和对应的技术文章，接着让他们在组内进行分享，参加分享会的同学会对分享的内容进行打分，最终我们会评选出每次分享会的前 3 名，给出一定的奖励并且会在公司内部平台进行宣传。通过这种方式，让他们踏出第一步，慢慢养成写作的习惯。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/90449f66c310452f931be1278960c21f~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>近一两年来，我在技术写作上的一些收获。曾获得 2020 年掘金年度人气作者第 10 名、<strong>掘金首批签约作者</strong>，掘金 <strong>Lv6</strong>，文章百万阅读。2020 年利用业余时间我写了《重学 TS》、《前端进阶篇》和《源码探秘篇》等电子书，下载 <strong>5W</strong> 余次。在掘金等技术社区发布了《跟着阿宝哥学前端》、《Angular 修仙之路》专栏，其中 《Angular 修仙之路》是 2017 年开始写的，那时候国内 Angular 的文章还比较少，我就通过文章的方式记录了自己的学习 Angular 的过程，写了上百篇的文章。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/25e5f80bc2ad43cfb6ae21ccae103aa6~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-1">二、如何选定写作主题</h3>
<p>这里我会给大家分享一些选题的方向：</p>
<p>1、记录某个项目的踩坑经历或项目实战/优化经验；</p>
<p>2、记录你对某些面试题的剖析与思考；</p>
<p>3、分享算法的一些刷题或解题心得；</p>
<p>4、分享阅读某个开源项目的收获；</p>
<p>5、记录阅读经典书籍，比如前端方面的 “红宝书”、“绿皮书” 或 “犀牛书” 的学习笔记；</p>
<p>6、总结学习某些技术或计算机基础知识的相关内容；</p>
<p>7、分享某个项目的技术架构或核心功能的设计与实现；</p>
<p>8、分享个人的成长经历、面试经历等。</p>
<p>而其他的写作方向，大家可以参考技术社区的文章排行榜或分析你所关注的技术大佬，也可以找到一些写作的主题。比如通过掘金首页 —— 推荐栏位，就可以看到 <strong>热门、最新和热榜</strong> 的文章：</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d773f78d333c44469c82e02a0ab7dbee~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>通过分析这些热榜文章，你就可以找到一些写作热点。另外，我的 QQ 邮箱有订阅一些不错的 Weekly。比如：<strong>JavaScript Weekly、Node Weekly、TypeScript Weekly 和 React Status</strong> 等。这些 Weekly 每周会推送一些不错的文章或开源项目。这些 Weekly 提供的信息，也是我选题的一些素材来源。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5c50c3e519964632afafe576b120326c~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-2">三、如何开始写作</h3>
<p>在确定好写作主题之后，我建议大家先不要急着动手写，可以先列出文章的大纲，这样可以让后续的写作过程更加顺畅。其实这跟平时写代码类似，想清楚后再开始写代码，效率会更高。这里我以 <strong>你不知道的 Web API</strong> 系列文章为例，来分享一下我写 Web API 文章时写的大纲。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0c981ce0640a4a83bb156fc3fac916df~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>在写 <strong>你不知道的 Web API</strong> 系列时，我整理了一份通用的大纲。比如：</p>
<ul>
<li>XXX 是什么/什么是 XXX</li>
<li>了解 XXX API</li>
<li>XXX API 应用场景</li>
<li>XXX API 与相关知识点的区别</li>
</ul>
<p>今年有幸成为掘金的首批签约作者，在写首篇签约文章的时候，我也在纠结到底写什么文章。后面突然想起之前看到掘金上 <strong>浪里行舟</strong> 和 <strong>秋风</strong> 大佬写的跨域文章。这两篇文章总结了我们工作中， 跨域问题的多种解决方案和实现原理。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a3fe88814f794124be6b12b82e94db94~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>受到这个的启发，我就想到了日常工作中，文件上传的场景。所以我就先根据以往的经验，梳理了文件上传、多文件上传、拖拽上传和剪贴板上传的场景。之后，我就使用 <strong>文件上传</strong> 关键字，在掘金、谷歌、百度等平台上搜索相关文章，进而不断完善大纲。在初步整理完大纲之后，我就会对大纲的内容进行二次加工，比如进行筛选和排序。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9a564d276f224d639b2db242c59c3149~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>在确认好大纲之后，为了保证文章的一致性。我又对每个场景的结构进行梳理。比如把单文件上传、多文件上传和目录上传场景拆分成 <strong>前端代码</strong> 和 <strong>服务端代码</strong> 两部分内容来介绍。在确认完文章骨架之后，后面写作起来就会清晰很多。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/bdfc441c8beb456fb5379cb894609458~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>掘金社区的掘友们还是很给力的，除了对文章的内容提出建议或反馈问题之外，还会帮你选题，甚至标题都给你想好了。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/893dff8246b44e67ba5fd2fd5451ebf3~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>既然掘友有要求，连标题也帮我想好了，所以后面我又写了一篇 <strong>文件下载，搞懂这9种场景就够了</strong> 的文章。在写作的过程中，同样也是先梳理大纲，然后再对大纲进行二次加工。等文章脉络清晰了之后，才会动手开始完善文章中每个章节的内容。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f2ba8dcfe3a14367a24aecbd5d246bdf~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-3">四、写作技巧与写作建议</h3>
<h4 data-id="heading-4">4.1 写作技巧</h4>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/649aa7400163451eb4815e8ec7d4ee8c~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<h4 data-id="heading-5">4.2 写作建议</h4>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/00178641320f4ee8abf78aa3b77d549d~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>1、多写专题的文章，后期就可以方便地整理成电子书或小册。如果写得好的话，还可以联系出版社出书。比如前端进阶篇电子书，我包含了 <strong>玩转系列</strong>、<strong>你不知道的系列</strong> 和 <strong>一文读懂系列</strong>。而重学 TS 1.0 电子书我包含了设计模式专题的文章。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b1228dd4e14d4742acbec5fb1c252004~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>2、写文章，多画图。特别是源码分析的文章，这样可以帮助读者更好地理解核心的流程。比如之前写 <strong>JavaScript</strong> <strong>中如何实现大文件并行下载</strong> 和 <strong>Axios</strong> <strong>如何取消重复请求</strong> 文章时，我就画了对应技术方案的流程图。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b58bf1a69e1c40da951571a2de86a7bb~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>3、尽量减少读者不必要的操作，为引用的内容添加快速链接。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/18937e9823cc41e5892676ba8542de7e~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>4、如果是自己写的可执行代码，最好能运行一下，保证一下代码的正确性和可用性。更近一步的话，可以搞个专门的项目来管理。比如在写 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fmp.weixin.qq.com%2Fs%2FaRfDFVpDFftn1qFYeLg4Iw" target="_blank" rel="nofollow noopener noreferrer" title="https://mp.weixin.qq.com/s/aRfDFVpDFftn1qFYeLg4Iw" ref="nofollow noopener noreferrer">文件上传，搞懂这8种场景就够了</a> 和 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fmp.weixin.qq.com%2Fs%2FPysSe6MykjYzVrWQCKJXvg" target="_blank" rel="nofollow noopener noreferrer" title="https://mp.weixin.qq.com/s/PysSe6MykjYzVrWQCKJXvg" ref="nofollow noopener noreferrer">文件下载，搞懂这9种场景就够了</a> 这两篇文章时，我就把文章中使用的示例代码统一放到 Github 仓库中。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/471cf492f94e4b13bc8cfeeb22a697b8~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>

<h4 data-id="heading-7">5.1 Markdown 工具</h4>

<h4 data-id="heading-8">5.2 绘图工具</h4>
