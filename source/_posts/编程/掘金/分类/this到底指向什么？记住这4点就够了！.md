class="prefix"></span><span class="content">本文主要在于掌握this在JavaScript的指向，了解this是在什么时候确定的，谁决定了this的指向。</span><span class="suffix"></span><span> </span></h2>
<h1 data-id="heading-1"><span class="prefix"></span><span class="content">4大规则</span><span class="suffix"></span></h1>
<blockquote class="multiquote-1">
<p>优先级权重：依次<strong>从上到下</strong>，权重值<strong>从小到大</strong>。</p>
</blockquote>
<ol>
<li>（优先级最小）默认绑定规则： console.log( this === window ); // true</li><li>（优先级次于后两项）隐式绑定规则：谁调用就指向谁;(隐式丢失)。</li><li>（优先级次于new绑定）显示绑定规则：call,apply,bind。</li><li>（优先级最大）new 绑定规则。</li></ol>
<h2 data-id="heading-2"><span class="prefix"></span><span class="content">this 的默认绑定规则</span><span class="suffix"></span><span> </span></h2>
<blockquote class="multiquote-1">
<p>全局的this默认指向window,如果开启严格模式的话，则this指向undefined。</p>
</blockquote>
<ul>
<li>例子1：</li></ul>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span> === <span class="hljs-built_in">window</span>) <span class="hljs-comment">// true 严格模式下全局的this为undefined</span><br><span class="hljs-built_in">console</span>.log(&#123;&#125; === &#123;&#125;) <span class="hljs-comment">// false</span><br><span class="copy-code-btn">复制代码</span></code></pre>
<ol>
<li>this和window之所以输出的是true,是因为他们指向的是同一个地址；</li><li>两个对象为什么不相等呢，主要是因为，对象对比的是指针地址，每声明一个对象就开辟一个空间生成一个指针地址指向存数据的地方。</li></ol>
<h2 data-id="heading-3"><span class="prefix"></span><span class="content">this 的隐式绑定规则</span><span class="suffix"></span><span> </span></h2>
<blockquote class="multiquote-1">
<p>如果方法是通过某个对象点的方法调用的话，则this指向调用的对象，如果是直接调用（自主调用）的话，则this指向window.</p>
</blockquote>
<ul>
<li>例子1:</li></ul>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">test</span>(<span class="hljs-params"></span>)</span>&#123;<br>    <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span> === <span class="hljs-built_in">window</span>); <span class="hljs-comment">// true</span><br>&#125;<br>test() <span class="hljs-comment">// 可以理解为window.test();</span><br><span class="copy-code-btn">复制代码</span></code></pre>
<ol>
<li>函数的独立调用，其实就是相当于window.xxx()的方式调用，因为全局声明的函数都是默认挂载到window上的，所以this也默认指向window。</li></ol>
<p>• 例子2:</p>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-keyword">var</span> obj = &#123;<br>    <span class="hljs-attr">a</span>: <span class="hljs-number">2</span>,<br>    <span class="hljs-attr">foo</span>: <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>)</span>&#123;<br>        <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span>); <span class="hljs-comment">// obj</span><br>        <span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">test</span>(<span class="hljs-params"></span>)</span>&#123;<br>            <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span>); <span class="hljs-comment">// window</span><br>        &#125;<br>        test();<br>    &#125;<br>&#125;<br>obj.foo();<br><span class="copy-code-btn">复制代码</span></code></pre>
<ol>
<li>因为foo是obj调用，所以foo函数的this应该指向调用他的对象，所以内部打印出来的this为obj；</li><li>而test虽然是在foo的内部调用，但是属于直接调用所以this依旧指向window。</li></ol>
<h2 data-id="heading-4"><span class="prefix"></span><span class="content">this 的显示绑定规则</span><span class="suffix"></span><span> </span></h2>
<blockquote class="multiquote-1">
<p>如果通过call,apply,bind强制更改this指向，则this指向方法传入的第一个参数。</p>
</blockquote>
<p>• 例子1</p>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-keyword">var</span> name = <span class="hljs-string">'test1'</span><br><span class="hljs-keyword">var</span> obj = &#123;<br>    <span class="hljs-attr">name</span>: <span class="hljs-string">'test2'</span><br>&#125;<br><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">fn</span>(<span class="hljs-params"></span>)</span>&#123;<br>    <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span>.name);<br>&#125;<br><span class="hljs-keyword">var</span> bindFn = fn.bind(obj);<br>bindFn() <span class="hljs-comment">// test2</span><br>fn.call(obj) <span class="hljs-comment">// test2</span><br>fn.apply(obj) <span class="hljs-comment">// test2</span><br><span class="copy-code-btn">复制代码</span></code></pre>
<ol>
<li>因为使用了call，apply，bind强行更改了this指向，所以this指向传入的第一个参数（obj）。</li></ol>
<h2 data-id="heading-5"><span class="prefix"></span><span class="content">this 的new绑定</span><span class="suffix"></span><span> </span></h2>
<blockquote class="multiquote-1">
<ol>
<li>如果构造函数内部没有返回引用类型变量，则构造函数内部会创建一个新的对象，this就是这个对象；</li><li>如果构造函数内部返回了引用类型变量，则this为返回的这个对象。</li></ol>
</blockquote>
<p>• 例子1:</p>
<pre class="custom"><span></span><code class="hljs copyable"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">Body</span>(<span class="hljs-params"></span>)</span>&#123;<br>    <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span>); <span class="hljs-comment">// Body &#123;&#125;</span><br>&#125;<br><span class="hljs-keyword">new</span> Body()<br><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">Person</span>(<span class="hljs-params"></span>)</span>&#123;<br>  <span class="hljs-built_in">console</span>.log(<span class="hljs-keyword">this</span>); <span class="hljs-comment">// &#123;name: "test"&#125;</span><br>  <span class="hljs-keyword">return</span> &#123;<br>    <span class="hljs-attr">name</span>: <span class="hljs-string">'test'</span><br>  &#125;<br>&#125;<br><span class="hljs-keyword">new</span> Person()<br><span class="copy-code-btn">复制代码</span></code></pre>
<ol>
<li>因为Body内部没有返回引用类型的数据所以this指向新创建的对象，Person构造函数内部返回了一个对象，所以this则指向返回的对象。</li></ol>
<h1 data-id="heading-6"><span class="prefix"></span><span class="content">总结：</span><span class="suffix"></span></h1>
<p>在js中函数定义时，this的指向是不确定的，只有在执行的时候，才能确定this的指向，所以可以总结成一句话，<strong>“this指向是根据执行调用时的动态上下文来决定的”</strong>。</p>
