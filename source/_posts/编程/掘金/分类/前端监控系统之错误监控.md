<blockquote>

<p>语法错误:</p>
<pre><code class="hljs language-html copyable" lang="html"><span class="hljs-tag"><<span class="hljs-name">script</span>></span><span class="javascript">
  <span class="hljs-keyword">const</span> value = <span class="hljs-number">10</span>
  @!
  <span class="hljs-built_in">console</span>.log(value)  
</span><span class="hljs-tag"></<span class="hljs-name">script</span>></span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>js引擎在解析scrpit代码时，<strong>会先进行词法分析</strong>， 将js转换成[&#123;&#125;] 格式的tokens流， 为什么要这么做呢？ 因为后面要将tokens流转换成抽象语法树， 在生成语法树的过程中，引擎会有语法分析器对语法进行判断， 上述代码语法分析器判断无法生成一颗有效的语法树，抛出语法错误。停止对代码的解析。</p>
<p>抛出错误： <code>Uncaught SyntaxError: Invalid or unexpected token</code></p>
<p>引用错误：</p>
<pre><code class="hljs language-html copyable" lang="html"><span class="hljs-tag"><<span class="hljs-name">script</span>></span><span class="javascript">
  <span class="hljs-keyword">const</span> value = <span class="hljs-number">10</span>
  <span class="hljs-built_in">console</span>.log(test)
  <span class="hljs-built_in">console</span>.log(value)
</span><span class="hljs-tag"></<span class="hljs-name">script</span>></span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>js引擎对当前script中的代码先进行词法分析，语法解析完成后，成功构成一颗ast语法树，此时js引擎会对当前ast树进行<strong>预编译</strong>。即在内存中开辟空间，将变量/函数存放到分配的空间中，声明变量/函数，此时变量/函数提升就发生在这个阶段。</p>
<p>预编译阶段的执行流程</p>
<ol>
<li>创建GO对象</li>
<li>声明的变量赋予GO对象，值为undefined，声明的函数赋予GO对象，值为函数体
<ol>
<li>遇到函数，就会创建AO对象</li>
<li>查找函数形参及函数内变量声明，形参名及变量名作为AO对象的属性，值为undefined</li>
<li>实参形参相统一，实参值赋给形参</li>
<li>查找函数声明，函数名作为AO对象的属性，值为函数引用</li>
</ol>
</li>
<li>一切声明的全局变量，全挂载到window上。</li>
</ol>
<p>当预编译完成后，js引擎会进入<strong>运行阶段</strong>， 代码运行到<code>console.log(test)</code> 时，进行作用域查找，当查找到顶层window时此时还没有这个变量的声明，那么js引擎抛出错误. 代码不再向下执行。</p>
<p>抛出的错误<code>test.html:12 Uncaught ReferenceError: test is not defined</code></p>
<p>范围错误：</p>
<pre><code class="hljs language-js copyable" lang="js"><script>
  <span class="hljs-keyword">const</span> value = []
  value.length = -<span class="hljs-number">1</span>
  <span class="hljs-built_in">console</span>.log(value)
</script>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>当预编译完成后，js引擎会进入运行阶段，代码运行到<code>value.length = -1</code> 时， js引擎发现了value的length被赋值为了<code>-1</code>。此时js引擎抛出 <code>Uncaught RangeError: Invalid array length</code> 。因为<code>-1</code> 值不在数组所允许的范围或者集合中。抛出错误后，js代码终止。</p>
<p>还有其他这几种情况</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">// Number 对象的方法参数超出范围</span>

<p>抛出的错误: <code>Uncaught RangeError: Invalid array length</code></p>
<p>类型错误：</p>
<pre><code class="hljs language-js copyable" lang="js"><script>

<p>promise未catch的错误</p>
<pre><code class="hljs language-js copyable" lang="js"><script>
  <span class="hljs-keyword">const</span> value = []

<p>在promise中出现的错误会被统一放到Promise的catch中进行处理。如果没有catch则向上抛出，因为执行机制的原因，并不会阻塞线程执行，所以value是可以正常打印</p>
<p>抛出错误： <code>Uncaught (in promise) ReferenceError: a is not defined</code></p>
<p>跨域脚本错误</p>
<pre><code class="hljs language-js copyable" lang="js"><script src=<span class="hljs-string">"https://test.bootcdn.net/ajax/libs/test.js"</span>></script>

<p>前端需要在script标签中配置<code>crossorigin</code> 才可以捕获到跨站脚本内部报出的错误，因为浏览器只允许同域下的脚本捕获具体的错误信息。</p>
<p>因为可能有些浏览器对crossorigin 不支持，此时我们用try catch继续向上抛出。</p>
<h3 data-id="heading-2">如何捕获</h3>
<ul>
<li>
<p>window.onerror</p>
<p>当发生 JavaScript 运行时错误(包括处理程序中引发的语法错误和异常)时，会触发window.onerror的回调。</p>
<p>但是window.onerror捕获不到资源加载错误</p>
</li>
<li>
<p>使用window.addEventListener('error')捕获资源错误，但是window.addEventListener('error')也可以捕获到js运行错误，可以通过target?.src || target?.href区分是资源加载错误还是js运行时错误。</p>
</li>
</ul>
<p>既然window.addEventListener('error')也可以捕获到错误，那么我们为什么要用window.onerror呢？</p>
<p>因为window.onerror的事件对象数据更加多，更加清晰。</p>
<ul>
<li>
<p>window.addEventListener('unhandledrejection')</p>
<p>捕获promise未catch的错误</p>
</li>
</ul>
