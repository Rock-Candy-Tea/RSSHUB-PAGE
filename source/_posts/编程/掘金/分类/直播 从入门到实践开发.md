<p>RTMP 协议</p>
<p>RTMP 是 Real Time Messaging Protocol （实时消息传输协议）的首字母缩写。 该协议基于 TCP，是一个协议族，包括 RTMP 基本协议及 RTMPT/RTMPS/RTMPE 等多种变种。RTMP 是一种设计用来进行实时数据通信的网络协议，主要用来在 Flash、AIR 平台和支持 RTMP 协议的流媒体/交互服务器之间进行音视频和数据通信。主播<strong>客户端</strong>使用。flv视频格式。</p>
<p>采集端用 web 来做 协议是 webrtc。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/4a5c572b4df34b59a7f01c0a9b73c895~tplv-k3u1fbpfcp-watermark.image" alt="RTMP 协议" loading="lazy" referrerpolicy="no-referrer"></p>
<p>HTTP-FLV协议</p>
<p>接合二者优点</p>
<ul>
<li>flv 视频格式</li>
<li>建立长连接</li>
<li>建立 HTTP 协议</li>
</ul>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ad3df6434a994d0bb11ccbb8e5e54292~tplv-k3u1fbpfcp-watermark.image" alt="HTTP-FLV协议" loading="lazy" referrerpolicy="no-referrer"></p>
<ol>
<li>可以在一定程度上避免防火墙的干扰（例如：有的机房只允许 80 端口通过）</li>
<li>可以很好的兼容 HTTP 302 跳转，做到灵活调度</li>
<li>可以使用 HTTPS 做加密通道</li>
<li>很好的支持移动端（android,ios）</li>
</ol>
