h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><p>一道著名的坑人题，可能一些同学见过：</p>
<pre><code class="hljs language-js copyable" lang="js">[<span class="hljs-string">'2'</span>, <span class="hljs-string">'3'</span>, <span class="hljs-string">'4'</span>].map(<span class="hljs-built_in">parseInt</span>);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>说出上面代码的执行结果。</p>
<p>如果不过脑子的说是 <code>[2, 3, 4]</code>，那么肯定是错了。实际上，真正的执行结果是 <code>[2, NaN, NaN]</code>。为什么会这样呢？是因为 map 的算子是有两个参数的，第一个参数是被迭代数组的元素，第二个参数是该元素的下标。所以 <code>['2', '3', '4'].map(parseInt)</code> 实际上相当于执行了 <code>[parseInt('2', 0), parseInt('3', 1), parseInt('4', 2)]</code>，结果就变成了 <code>[2, NaN, NaN]</code> 了。</p>
<p>今天我们在这里主要不是讨论为什么 <code>parseInt('2', 0)</code> 是 2，而 <code>parseInt('3', 1)</code> 是 NaN，实际上我们期望的结果应该是 <code>parseInt('2', 10)</code> 和 <code>parseInt('3', 10)</code>，把字符串 '2'、'3' 转成十进制 number。</p>
<p>所以，正确的写法应该是：</p>
<pre><code class="hljs language-js copyable" lang="js">[<span class="hljs-string">'2'</span>, <span class="hljs-string">'3'</span>, <span class="hljs-string">'4'</span>].map(<span class="hljs-function"><span class="hljs-params">num</span> =></span> <span class="hljs-built_in">parseInt</span>(num, <span class="hljs-number">10</span>));
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面这个写法对于这个问题来说是简单的，和函数式编程关系不大。但是，这个问题如果用过程抽象的思路通用化思考，应该是这样的：</p>
<pre><code class="hljs language-js copyable" lang="js">[<span class="hljs-string">'2'</span>, <span class="hljs-string">'3'</span>, <span class="hljs-string">'4'</span>].map(<span class="hljs-built_in">parseInt</span>.bindRight(<span class="hljs-literal">null</span>, <span class="hljs-number">10</span>));
<span class="copy-code-btn">复制代码</span></code></pre>

<p>bindRight 代码并不复杂，如果 bindRight 的参数个数比函数形参多，那么简单将参数次序 reverse 之后传给原来的函数，否则将函数前面的参数补齐。</p>

<p>上面的这段代码里我们通过 reverseArgs 和 fixArgsLength 以及 bind 来组合出 bindRight。reverseArgs 是将函数参数顺序倒序的高阶函数，fixArgsLength 是将函数参数个数固定的高阶函数，我们可以看到 bindRight 就是先 reverseArgs 再 bind，最后 fixArgsLength（大家可以思考下为什么要 fixArgsLength）。这样就可以了。</p>
