data-id="heading-0">dotenv</h3>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fwww.npmjs.com%2Fpackage%2Fdotenv" target="_blank" rel="nofollow noopener noreferrer" title="https://www.npmjs.com/package/dotenv" ref="nofollow noopener noreferrer">dotenv</a> 是一个开源，没有依赖的 npm 包。它的作用是避免把自己的数据库连接方式提交到 github，变成公开的</p>
<p>使用上在开发环境，可以在根目录下新建 .env 的配置文件，里面去写数据库连接方式的环境变量，在 .gitignore 中配置这个文件不被提交到 git 上，在生产环境，通过在打包时注入环境变量的方式，配置数据库的连接方式，详细的配置见上一篇文章</p>
<p>但是像比较基础设施比较完备的公司，有自己的 git 私有化部署，就不需要这个工具了，直接把数据库连接配置提交到 git 上也是没问题的</p>
<h3 data-id="heading-1">React Fast Refresh</h3>
<h4 data-id="heading-2">what is react fast refresh</h4>
<p>react fast refresh 是 react 官方支持的一个热更新方案，之前被用到 react native，因为底层的实现是相同的，于是也可以被用到 web，但是需要和已有的工具比如 webpack 集成（已经有插件）</p>
<p>fash refresh 实时更新，可以保存组件的状态（函数组件），并且可以在程序编译出错时，在屏幕上覆盖展示错误信息</p>

<h4 data-id="heading-3">how to use</h4>
<p>在开发环境配置 webpack 的插件：<a href="https://link.juejin.cn/?target=https%3A%2F%2Fgithub.com%2Fpmmmwh%2Freact-refresh-webpack-plugin" target="_blank" rel="nofollow noopener noreferrer" title="https://github.com/pmmmwh/react-refresh-webpack-plugin" ref="nofollow noopener noreferrer">github.com/pmmmwh/reac…</a></p>
<p>注意因为 react-refresh 包的限制，react 版本需大于 16.9.0</p>
<h4 data-id="heading-4">compare with react-hot-loader</h4>
<p>react fast refresh 是 react 官方支持的热更新工具，react-hot-loader 的 readme 也在推荐迁移到 fast refresh，不再用 hot loader 了</p>
