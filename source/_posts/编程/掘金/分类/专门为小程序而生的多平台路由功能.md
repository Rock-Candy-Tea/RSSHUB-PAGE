<h2 data-id="heading-0">简介</h2>
<p><code>medusa-router</code> 对官方接口进行了封装增强，让你能以更加优雅的方式读取和使用路由。它相较于官方接口 <strong>更加容易记忆</strong> 并且 <strong>路由参数能够保持原有的数据类型</strong>。 <code>medusa-router</code> 被设计为可以在各类小程序当中使用的 SDK，目前已经在微信、百度、支付宝、Taro、uni-app 框架上测试通过。
<a name="user-content-g31HU" href="https://juejin.cn/post/undefined"></a></p>

<h2 data-id="heading-4">$route 对象</h2>
<p><code>ms.$route</code> 对象包含当前页面的信息，该对象包含 <code>getCurrentPages()</code> API 获取的页面栈中最顶层的页面信息和扩充属性 <code>fullPath</code> 和 <code>query</code> 。</p>
<ul>
<li><code>&#123;String&#125; fullPath</code> - 页面完整路径，不包含参数</li>
<li><code>&#123;Object&#125; query</code> - 路由参数，其中的属性保留原有的数据类型</li>
</ul>

<h2 data-id="heading-5">API 说明</h2>

<h3 data-id="heading-6">getPage()</h3>
<ul>
<li>说明：获取当前页面信息，相当于使用 <code>getCurrentPages()</code> API 获取页面栈中最顶层的页面；</li>
</ul>
<p><a name="user-content-icqz2" href="https://juejin.cn/post/undefined"></a></p>
<h3 data-id="heading-7">push(options)</h3>
<ul>
<li>参数
<ul>
<li><code>&#123;Object&#125; options</code></li>
<li><code>&#123;String&#125; options.url</code> - 目标页面路径</li>
<li><code>&#123;Object&#125; [options.query]</code> - 路由参数</li>
<li><code>&#123;Object&#125; [options.events]</code> - 页面通信接口</li>
<li><code>&#123;Function&#125; [options.success]</code> - 成功时回调函数</li>
<li><code>&#123;Function&#125; [options.fail]</code> - 失败时回调函数</li>
<li><code>&#123;Function&#125; [options.complete]</code> - 结束时回调函数</li>
</ul>
</li>
<li>说明：
<ul>
<li>跳转至目标页面，与小程序的 <code>navigateTo(...)</code> API 具有相同的功能；</li>
</ul>
</li>
</ul>
<p><a name="user-content-YIxus" href="https://juejin.cn/post/undefined"></a></p>
<h3 data-id="heading-8">replace(options)</h3>
<ul>
<li>参数
<ul>
<li><code>&#123;Object&#125; options</code></li>
<li><code>&#123;String&#125; options.url</code> - 目标页面路径</li>
<li><code>&#123;Object&#125; [options.query]</code> - 路由参数</li>
<li><code>&#123;Boolean&#125; [options.closeAll = false]</code> - 控制是否关闭当前路由栈中存在的所有页面</li>
<li><code>&#123;Function&#125; [options.success]</code> - 成功时回调函数</li>
<li><code>&#123;Function&#125; [options.fail]</code> - 失败时回调函数</li>
<li><code>&#123;Function&#125; [options.complete]</code> - 结束时回调函数</li>
</ul>
</li>
<li>说明：
<ul>
<li>跳转至目标页面并且关闭当前页面，与小程序的 <code>redirectTo(...)</code> API 具有相同的功能；</li>
<li>当 <code>options.closeAll</code> 参数为 <code>true</code> 时，将跳转至目标页面并且关闭路由栈中所有页面，相当于小程序的 <code>reLaunch(...)</code> API 的功能；</li>
</ul>
</li>
</ul>
<p><a name="user-content-jtXC1" href="https://juejin.cn/post/undefined"></a></p>

</ul>

<h3 data-id="heading-10">goBack(options)</h3>
<ul>
<li>参数
<ul>
<li><code>&#123;Object&#125; [options]</code></li>
<li><code>&#123;String&#125; options.delta = 1</code> - 返回几层页面，默认值为 1</li>
<li><code>&#123;Function&#125; [options.success]</code> - 成功时回调函数</li>
<li><code>&#123;Function&#125; [options.fail]</code> - 失败时回调函数</li>

<ul>

<h3 data-id="heading-12">decoding(options)</h3>
<ul>
<li>参数
<ul>
<li><code>&#123;Object&#125; options</code> - onLoad 生命周期函数的参数 options</li>

</li>

<li><code>&#123;Object&#125; query</code> - 路由参数对象</li>

<h2 data-id="heading-13">增强功能说明</h2>
<p><a name="user-content-WvOnZ" href="https://juejin.cn/post/undefined"></a></p>
<h3 data-id="heading-14">路由参数保真功能</h3>
<ul>
<li><code>medusa-router</code> 中提供的跳转功能均支持原生的传参方式，使用该方式得到的路由参数的值均为字符串类型；</li>
<li>使用 <code>push</code> 和 <code>replace</code> 功能时，运用 <code>query</code> 属性传递参数能够 <strong>保持原有的数据类型</strong> 。在目标页面当中需使用 <code>$route.query</code> 对象或者调用 <code>decoding(options)</code> API 才能获取正确的路由参数；</li>
</ul>
<p><a name="user-content-si9BH" href="https://juejin.cn/post/undefined"></a></p>
<h3 data-id="heading-15">API 传参简化</h3>
<ul>
<li>具有跳转功能的 API 如果不需要传递路由参数和回调函数时，可以 <strong>直接传递目标页面路径字符串</strong> 作为参数；</li>
<li><code>goBack(options)</code> 回退功能不需要使用回调函数时，可以直接将 <strong>回退页面层级</strong> 作为参数；</li>
</ul>
