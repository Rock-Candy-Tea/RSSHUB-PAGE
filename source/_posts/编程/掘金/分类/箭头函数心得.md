h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h1 data-id="heading-0">箭头函数</h1>
<p>箭头函数是匿名函数，语法比函数表达式更简洁，并且没有自己的<code>this</code>，<code>arguments</code>，<code>super</code>或<code>new.target</code>。箭头函数表达式更适用于那些本来需要匿名函数的地方，并且它不能用作构造函数。</p>
<p>引入箭头函数有两个方面作用：更简短的函数并且不绑定<code>this</code>。</p>
<h2 data-id="heading-1">特性</h2>
<ul>
<li>没有自己的<code>this</code></li>
<li>没有自己的<code>argument</code></li>
<li>没有<code>prototype</code>属性</li>
<li>没有<code>super</code>或<code>new.target</code></li>
<li><code>yield</code> 关键字通常不能在箭头函数中使用（除非是嵌套在允许使用的函数内）。</li>
<li>不能当作构造函数,不能进行<code>new</code>操作</li>
</ul>
<h2 data-id="heading-2">语法</h2>
<p>一般语法</p>
<pre><code class="hljs language-js copyable" lang="js">(param1, param2, …, paramN) => &#123; statements &#125; 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>当只有一个参数时，圆括号是可选的，下面两种都对。</p>
<pre><code class="hljs language-js copyable" lang="js">param1 => &#123; statements &#125; 
(param1) => &#123; statements &#125; 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>如果没有参数时，需要保留圆括号。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">// 没有参数的函数应该写成一对圆括号。</span>
() => &#123; statements &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>另外还有一些简写写法</p>
<pre><code class="hljs language-js copyable" lang="js">(param1, param2, …, paramN) => expression
<span class="hljs-comment">//相当于：</span>
(param1, param2, …, paramN) =>&#123; <span class="hljs-keyword">return</span> expression; &#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>建议无论是否只有一个参数，都保留圆括号（方便以后添加参数或删减参数）</p>
<p>高级语法：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">//加括号的函数体返回对象字面量表达式：</span>
params => (&#123;<span class="hljs-attr">foo</span>: bar&#125;)

<span class="hljs-comment">//支持剩余参数和默认参数</span>
(param1, param2, ...rest) => &#123; statements &#125;
(param1 = defaultValue1, param2, …, paramN = defaultValueN) => &#123; 
statements &#125;

<span class="hljs-comment">//同样支持参数列表解构</span>
<span class="hljs-keyword">let</span> f = <span class="hljs-function">(<span class="hljs-params">[a, b] = [<span class="hljs-number">1</span>, <span class="hljs-number">2</span>], &#123;x: c&#125; = &#123;x: a + b&#125;</span>) =></span> a + b + c;
f();  <span class="hljs-comment">// 6</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<h2 data-id="heading-3">箭头函数的this</h2>
<p><strong><code>而箭头函数不会创建自己的this,它的this是在定义函数的时候绑定的,它只会在自己的作用域链向外逐层地查找，找到最近的普通函数的this对象（类似查找变量），然后继承这个this</code></strong></p>
<p><code>被继承的普通函数的this指向改变，箭头函数的this指向会跟着改变.并且在箭头函数this对象的指向是不能直接改变的。</code></p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">fn1</span>(<span class="hljs-params"></span>)</span>&#123; <span class="hljs-comment">// 第一层普通函数</span>
    <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>); <span class="hljs-comment">// window</span>
    <span class="hljs-keyword">let</span> obj1 = &#123; <span class="hljs-comment">// 第二层对象</span>
        <span class="hljs-attr">fn2</span>: <span class="hljs-function">() =></span> &#123; <span class="hljs-comment">// 第三层箭头函数</span>
            <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>); <span class="hljs-comment">// window</span>
        &#125;
    &#125;
    obj1.fn2();
&#125;
fn1()
<span class="copy-code-btn">复制代码</span></code></pre>
<p>fn1是普通函数，它的<code>this</code>指向全局对象<code>window</code>，而fn2是一个箭头函数，它被嵌套在fn1的obj1对象内，它继承了fn1的<code>this</code>，因此箭头函数的<code>this</code>也指向<code>window</code>。</p>
<p><code>当箭头函数的作用域链都没有能继承的this时，它的指向是全局对象。</code>如下</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> arrow = <span class="hljs-function">() =></span> &#123;
     <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>);
  &#125;
arrow() <span class="hljs-comment">// window对象</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<pre><code class="hljs language-js copyable" lang="js">    <span class="hljs-keyword">let</span> o1 = &#123;
        <span class="hljs-attr">o2</span>: &#123;
            <span class="hljs-attr">arrow</span>: <span class="hljs-function">() =></span> &#123;
                <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>);
            &#125;
        &#125;
    &#125;
    o1.o2.arrow() <span class="hljs-comment">// window对象</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面代码中箭头函数都无法在祖先作用域中找到能继承的<code>this</code>，因此无论是否在严格模式下都会指向<code>window</code>，下面把o2改成一个普通函数（拥有自己的<code>this</code>），此时箭头函数能找到继承的<code>this</code>，于是指向。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> o1 = &#123;
    <span class="hljs-function"><span class="hljs-title">o2</span>(<span class="hljs-params"></span>)</span>&#123;
        <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>); <span class="hljs-comment">// o2对象</span>
        <span class="hljs-keyword">let</span> arrow = <span class="hljs-function">() =></span> &#123;
            <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>);
        &#125;
        arrow()
    &#125;
&#125;
 o1.o2() <span class="hljs-comment">// o2对象</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<h2 data-id="heading-4">箭头函数的arguments</h2>
<p>箭头函数没有自己的<code>arguments</code>，直接使用会报错</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> fn1 = <span class="hljs-function">(<span class="hljs-params">a</span>) =></span> &#123;
    <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">arguments</span>);
&#125;
fn1(<span class="hljs-string">"a"</span>)
<span class="hljs-comment">// Uncaught ReferenceError: arguments is not defined</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>但是和<code>this</code>一样，它能继承普通函数的<code>arguments</code></p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">fn1</span>(<span class="hljs-params">a,b</span>)</span>&#123;
    <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">arguments</span>); <span class="hljs-comment">// Arguments(2)&#123;...&#125;</span>
    <span class="hljs-keyword">let</span> arrow = <span class="hljs-function">() =></span> &#123;
        <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">arguments</span>); <span class="hljs-comment">// Arguments(2)&#123;...&#125;</span>
    &#125;
    arrow()
&#125;
fn1(<span class="hljs-string">"a"</span>,<span class="hljs-string">"b"</span>)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>在大多数情况下，使用<strong>剩余参数</strong>是相较使用<code>arguments</code>对象的更好选择。</p>
<h2 data-id="heading-5">使用 new 操作符</h2>
<p>箭头函数不能用作构造器，和 <code>new</code>一起用会抛出错误。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">var</span> Foo = <span class="hljs-function">() =></span> &#123;&#125;;
<span class="hljs-keyword">var</span> foo = <span class="hljs-keyword">new</span> Foo(); <span class="hljs-comment">// TypeError: Foo is not a constructor</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<h2 data-id="heading-6">使用prototype属性</h2>
<p>箭头函数没有<code>prototype</code>属性。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">var</span> Foo = <span class="hljs-function">() =></span> &#123;&#125;;
<span class="hljs-built_in">console</span>.log(Foo.prototype); <span class="hljs-comment">// undefined</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<h2 data-id="heading-7">使用 yield关键字</h2>
<p><code>yield</code> 关键字通常不能在箭头函数中使用（除非是嵌套在允许使用的函数内）。因此，箭头函数不能用作函数生成器。</p>
<h2 data-id="heading-8">修复缺陷</h2>
<p>由于Javascript的设计缺陷，在非严格模式下，在函数内部定义的函数 <code>this</code> 指向<code>window</code>，严格模式下，this为<code>undefined</code>。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> oTest = &#123;
    <span class="hljs-function"><span class="hljs-title">fn</span>(<span class="hljs-params"></span>)</span>&#123;
        <span class="hljs-keyword">let</span> fnTest = <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>)</span>&#123;
            <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>);
        &#125;
        fnTest();
    &#125;
&#125;
oTest.fn(); <span class="hljs-comment">// window对象</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>而箭头函数则完全修复了这个缺陷。只需要把上述代码修改成箭头函数即可。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">let</span> oTest = &#123;
    <span class="hljs-function"><span class="hljs-title">fn</span>(<span class="hljs-params"></span>)</span>&#123;
        <span class="hljs-keyword">let</span> fnTest = <span class="hljs-function"><span class="hljs-keyword">function</span>(<span class="hljs-params"></span>)</span>&#123;
            <span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">this</span>);
        &#125;
        fnTest();
    &#125;
&#125;
oTest.fn();
<span class="copy-code-btn">复制代码</span></code></pre></div>
