<h2 data-id="heading-0">前言</h2>
<p>在面试高级前端时，往往会遇到一些关于设计模式的问题，每次都回答不太理想。恰逢8月更文挑战的活动，准备用一个月时间好好理一下关于设计模式方面的知识点，给自己增加点面试的底气。</p>
<h2 data-id="heading-1">定义</h2>
<p>迭代器模式是指提供一种方法顺序访问一个聚合对象中的各个元素，而又不需要暴露该对象的内部表示。迭代器模式可以把迭代的过程从业务逻辑中分离出来，在使用迭代器模式之后，即使不关心对象的内部构造，也可以按顺序访问其中的每个元素。</p>
<p>迭代器模式是一个非常简单的设计模式。简单到迭到很多时候我们都不认为它是一种设计模式，因为目前的绝大部分语言都内置了迭代器，比如JavaScript中的<code>Array.prototype.forEach</code>。</p>
<h2 data-id="heading-2">实现</h2>
<p>迭代器模式的实现非常简单，如下所示</p>
<pre><code class="copyable">const each = (ary, callback) => &#123;
  for (let i = 0, l = ary.length; i < l; i++) &#123;
    callback.call(ary[i], i, ary[i]);
  &#125;
&#125;;
each([1, 2, 3], function (i, n) &#123;
  console([i, n]);
&#125;);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>通过<code>each</code>方法将<code>ary</code>数组每个元素传入<code>callback</code>中处理，这就是迭代器模式。</p>
<h2 data-id="heading-3">内部迭代器和外部迭代器</h2>
<p>迭代器可以分为内部迭代器和外部迭代器，上述实现的<code>each</code>函数属于内部迭代器，<code>each</code>函数的内部已经定义好了迭代规则，它完全接手整个迭代过程，外部只需要一次初始调用。</p>
<p>那么现在有一个需求，构造一个函数来比较<code>ary1</code>和<code>ary2</code>的内容是否相等。是否可以用<code>each</code>函数实现。当然可以了，如下这样实现：</p>
<pre><code class="copyable">const compare = (ary1, ary2) => &#123;
  if (ary1.length !== ary2.length) &#123;
    throw new Error('ary1 和 ary2 不相等');
  &#125;
  each(ary1, function (i, n) &#123;
    if (n !== ary2[i]) &#123;
      throw new Error('ary1 和 ary2 不相等');
    &#125;
  &#125;);
  console.log('ary1 和 ary2 相等');
&#125;;
compare( [ 1, 2, 3 ], [ 1, 2, 5 ] ); // 'ary1 和 ary2 不相等' ; 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>但是这样实现太不优雅了，<code>each</code>函数中的迭代规则已经在内部定义好了，无法改变，这也是内部迭代器的缺点。</p>
<p>要让<code>compare</code>函数内部实现得优雅点，得借助外部迭代器。外部迭代器必须显式地请求迭代下一个元素。
外部迭代器增加了一些调用的复杂度，但相对也增强了迭代器的灵活性，我们可以手工控制迭代的过程或者顺序。</p>
<p>下面用外部迭代器来实现<code>compare</code>函数。</p>
<pre><code class="copyable">const Iterator = function (obj) &#123;
  let current = 0;
  const next = function () &#123;
    current += 1;
  &#125;;
  const isDone = function () &#123;
    return current >= obj.length;
  &#125;;
  const getCurrItem = function () &#123;
    return obj[current];
  &#125;;
  return &#123;
    next: next,
    isDone: isDone,
    getCurrItem: getCurrItem
  &#125;
&#125;;
const compare = function (iterator1, iterator2) &#123;
  while (!iterator1.isDone() && !iterator2.isDone()) &#123;
    if (iterator1.getCurrItem() !== iterator2.getCurrItem()) &#123;
      throw new Error('iterator1 和 iterator2 不相等');
    &#125;
    iterator1.next();
    iterator2.next();
  &#125;
  console.log('iterator1 和 iterator2 相等');
&#125;
const iterator1 = Iterator([1, 2, 3]);
const iterator2 = Iterator([1, 2, 3]);
compare(iterator1, iterator2); // 输出：iterator1 和 iterator2 相等 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>其中 <code>Iterator</code>函数就是一个外部迭代器，返回一个对象，其中暴露出一个<code>getCurrItem</code>方法来显式地请求迭代下一个元素。</p>
<p>外部迭代器虽然调用方式相对复杂，但它的适用面更广，也能满足更多变的需求。</p>
