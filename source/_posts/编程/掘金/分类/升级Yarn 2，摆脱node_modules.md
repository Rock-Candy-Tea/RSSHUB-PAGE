h3&#123;font-size:18px&#125;&#125;</style><p><code>node</code>项目中最臭名昭著的莫过于<code>node_modules</code>文件夹，这个糟糕的结构动辄使你的文件数目增加几万甚至几十万，无论是安装还是删除，都要消耗大量时间，并且占据大量<code>inode</code>结点，我们随便进入一个<code>react</code>项目文件夹，看一下由于有<code>node_modules</code>会使你的项目中的文件个数变成多少：</p>
<pre><code class="copyable">$ find . -type f | wc -l
223629
<span class="copy-code-btn">复制代码</span></code></pre>
<p>仅仅一个项目下面就有多达22万个文件。</p>
<p>现在我们来看一下目前的<code>yarn</code>版本号是多少：</p>
<pre><code class="copyable">$ yarn --version
1.22.11
<span class="copy-code-btn">复制代码</span></code></pre>
<p>嗯，目前<code>yarn</code>的版本号是<code>1.22.11</code>，那我们如何安装<code>yarn 2</code>呢？答案是不需要安装，只需要设置就可以了。</p>
<pre><code class="copyable">$ yarn set version berry
<span class="copy-code-btn">复制代码</span></code></pre>
<p>设置完了之后，我们再来看一下<code>yarn</code>的版本号：</p>
<pre><code class="copyable">$ yarn --version
3.0.0
<span class="copy-code-btn">复制代码</span></code></pre>
<p>不是说好的升级到<code>yarn 2</code>吗？怎么变成<code>3.0</code>了？不用恐慌，越高越好。</p>
<p>然后我们来看一下项目文件夹下多了这么几个文件，首先就是根目录下多了一个<code>.yarnrc.yml</code>，里面只有一句话：</p>
<pre><code class="hljs language-yml copyable" lang="yml"><span class="hljs-attr">yarnPath:</span> <span class="hljs-string">.yarn/releases/yarn-berry.cjs</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>相应的，还多了一个文件夹<code>.yarn</code>，里面有一个子文件夹<code>releases</code>，里面有一个文件<code>yarn-berry.cjs</code>，这几个文件就是全部<code>yarn 2</code>增加的内容了，这些内容不要在<code>.gitignore</code>里忽略，其它的内容是需要忽略的，现在我们来在<code>.gitignore</code>里增加一些需要忽略的内容：</p>
<pre><code class="copyable">/node_modules
/.pnp
.pnp.js
.pnp.cjs
.yarn/cache
.yarn/unplugged
.yarn/install-state.gz
<span class="copy-code-btn">复制代码</span></code></pre>
<p>接下来，我们准备利用新版的<code>yarn</code>安装我们的依赖文件，在此之前，我们需要先设置一下<code>yarn</code>库的镜像服务器以加快整个下载过程：</p>
<pre><code class="copyable">$ yarn config set npmRegistryServer https://registry.npm.taobao.org
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这时候，你再打开项目根目录下的<code>.yarnrc.yml</code>文件，会发现里面多了一行：</p>
<pre><code class="hljs language-yml copyable" lang="yml"><span class="hljs-attr">npmRegistryServer:</span> <span class="hljs-string">'https://registry.npm.taobao.org'</span>
<span class="hljs-attr">yarnPath:</span> <span class="hljs-string">.yarn/releases/yarn-berry.cjs</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>所以我们知道其实这个<code>yarn config</code>命令也没有什么特别的地方，只是通过它来修改<code>.yarnrc.yml</code>文件而已，你也可以通过直接修改<code>.yarnrc.yml</code>文件来达到同样的效果。</p>
<p>现在，我们开始删除旧的<code>node_modules</code>文件夹和<code>yarn.lock</code>文件，并重建整个项目：</p>
<pre><code class="copyable">$ rm -rf node_modules
$ rm -f yarn.lock
$ yarn
<span class="copy-code-btn">复制代码</span></code></pre>
<p>整个下载过程应该还是比较顺利的，我们来看一下项目文件夹中多了哪些文件：</p>
<pre><code class="copyable">.yarn/cache
.yarn/unplugged
.pnp
<span class="copy-code-btn">复制代码</span></code></pre>
<p>没有了<code>node_modules</code>文件夹，我们来看一下<code>.yarn/cache</code>文件夹下有什么内容，里面有我们之前依赖的<code>node_modules</code>文件夹下的所有依赖包，但不再是以目录的形式存在，而是变成了一个个<code>zip</code>文件，<code>yarn 2</code>就是利用项目根目录下的<code>.pnp.cjs</code>文件定位到这些<code>zip</code>文件以达到取代<code>node_modules</code>的作用，这样极大程度地减少了项目中的文件个数。</p>
<p>下面我们开始启动项目：</p>
<pre><code class="copyable">yarn start
<span class="copy-code-btn">复制代码</span></code></pre>
<p>十有八九你的项目这时候是启动不起来的，不要慌，这篇文章告诉你所有的解决方法。</p>
<p>首先，你遇到错误可能是这样：</p>
<pre><code class="copyable">Error: Your application tried to access terser-webpack-plugin, but it isn't declared in your dependencies; this makes the require call ambiguous and unsound.
<span class="copy-code-btn">复制代码</span></code></pre>
<p>具体内容可能不一样，但你要注意这个关键词<code>Your application</code>，这说明是你的代码当中的某个位置引用了后面的插件，但你没有在<code>package.json</code>文件中显式声明它，那为什么之前用<code>yarn 1</code>或者<code>npm</code>的时候没有这个问题呢？因为以前是有<code>node_modules</code>文件夹的，所有依赖包都被平摊在这个文件夹中，即使是被其它依赖的依赖引入的，它也会被释放在<code>node_modules</code>根目录下，所以<code>node</code>可以很轻松地找到它，而现在这个文件夹没有了，我们必须显式地在<code>package.json</code>文件中引用它，才能引导<code>yarn</code>找到这个依赖项。因此，解决这种<code>Your application</code>缺乏某个依赖项的方法很简单，我们只需要用<code>yarn</code>安装它就可以了：</p>
<pre><code class="copyable">yarn add -D terser-webpack-plugin
<span class="copy-code-btn">复制代码</span></code></pre>
<p>哦，又出错误了：</p>
<pre><code class="copyable">Invalid options object. Terser Plugin has been initialized using an options object that does not match the API schema.
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这是因为我们在安装的时候没有指定版本，导致安装的插件版本过高，我们在<code>package.json</code>里把版本降低一些：</p>
<pre><code class="copyable">"terser-webpack-plugin": "^4.2.3",
<span class="copy-code-btn">复制代码</span></code></pre>
<p>然后重新执行<code>yarn</code>进行安装，运行<code>yarn start</code>再次启动，终于启动起来了！不过，不要高兴得太早，又遇到了这样的问题：</p>
<pre><code class="copyable">Error: Your application tried to access @babel/plugin-transform-async-to-generator, but it isn't declared in your dependencies; this makes the require call ambiguous and unsound.
<span class="copy-code-btn">复制代码</span></code></pre>
<p>不要慌，既然还是<code>Your application</code>缺乏某个依赖包，那就还是我们的问题，停下来再安装它，然后再启动，直到解决完所有<code>Your application</code>引起的问题。</p>
<p>这时候，产生了新的错误：</p>
<pre><code class="copyable">Module not found: rc-bmap tried to access babel-runtime, but it isn't declared in its dependencies; this makes the require call ambiguous and unsound.
<span class="copy-code-btn">复制代码</span></code></pre>
<p>虽然同样是找不到依赖项，但这次的错误不是由于我们自己的应用导致的，而是由于依赖项自身导致的，这种问题该如何解决呢？不要急，我们打开<code>.yarnrc.yml</code>文件，按照错误提示增加以下设置：</p>
<pre><code class="hljs language-yml copyable" lang="yml"><span class="hljs-attr">packageExtensions:</span>
  <span class="hljs-string">'rc-bmap@*'</span><span class="hljs-string">:</span>
    <span class="hljs-attr">dependencies:</span>
      <span class="hljs-attr">'babel-runtime':</span> <span class="hljs-string">'*'</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>缺什么咱们就增加什么，有时候还要注意版本号。同样，这个问题不是由于<code>yarn 2</code>导致，而是因为我们的依赖项该增加的依赖没有增加而已，我们这里只是给它补全依赖，使它得以正常运行。</p>
<p>别忘了，每次修改完<code>.yarnrc.yml</code>之后，都需要重新执行<code>yarn</code>，然后再执行<code>yarn start</code>。</p>
<p>至此为止，我们的项目终于能够成功运行了！我们来看一下目前项目文件夹中的文件个数：</p>
<pre><code class="copyable">$ find . -type f | wc -l
17433
<span class="copy-code-btn">复制代码</span></code></pre>
<p>现在只有<code>17000</code>个文件了，比我们最开始的<code>22</code>万个文件减少了<code>20</code>多万，运行速度也成倍提升。</p>
<p>怎么样，是不是很值得一试呢？</p></div>
