<h1 data-id="heading-0">需求介绍</h1>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7070edc5ba934ad89d55a1da78d981e8~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<ul>
<li>触摸虚线内的不规则区域，触发响应逻辑</li>
</ul>
<h1 data-id="heading-1">问题分析</h1>
<ul>
<li>一般游戏引擎提供给我们的按钮一般都是矩型或者其他的规则形状，无法完成上述需求</li>
<li>透明区域，图片的 alpha 通道值为 0(), 非透明区域 alpha 值不为 0，根据这个特征实现不规则边缘检测，从而判断响应事件是否触发响应逻辑</li>
<li>实时点击扫描运算量较高，在本项目场景中没有运行效率没有问题，但是在实时高频场景性能吃紧，为了做到一次封装万年使用，因此需要在初始化过程存储 alpha 信息备用。</li>
<li>美术工作： 提供不通的不规则按钮，N个按钮拼接成图中的不规则按钮</li>
</ul>
<h1 data-id="heading-2">原理拓展</h1>
<ul>
<li>图片存储原理</li>
</ul>
<p>一个 4x3 像素的图片，以 RGBA8888 色彩模式加载到内存中后，那么会以一大串连续的数据存储在内存中，例如：</p>
<p><img src="https://imgkr2.cn-bj.ufileos.com/086a9a22-eac6-4b29-a19a-2f6488ea2907.png?UCloudPublicKey=TOKEN_8d8b72be-579a-4e83-bfd0-5f6ce1546f13&Signature=2eHOiCRiD09cUESZcJwrybtQKEM%253D&Expires=1606535627" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>序号 1、2、3、4、5 各自表示一个像素点的信息，每一个像素点中的四个值分别代表 R(红色)、G（绿色）、B、（蓝色）、A（透明度），每个值的范围为 0 ～ 255（图中以 16 进制表示即范围 00~FF），每个值可用 8 个 bit 进行表示。</p>

<h3 data-id="heading-5">1、读取像素值并数据判断每个像素点的 alpha 值，采取 bool 类型进行存储。</h3>
<ul>
<li>
<p>内存分析：</p>
<p>首先 RGBA（32bit）我们只取了 Alpha（8bit）通道进行判断，判断结果按 bool 值进行存储，一个 bool 值类型在内存中占 8bit，因此此时内存为图形内存的 1/4。</p>
</li>
<li>
<p>优化</p>
<p>为了进一步减少内存，增加了另一种按位存储的方式。</p>
<p>倘若我们结合位的或运算将每 8 个判断结果存入一个 char 的每一个 Bit 中，该 bit 为 1 则不透明，为 0 则透明，那么内存又为原来 1/8，总的算起来就是图形内存的 1/32 拉。</p>
</li>
<li>
<p>内存示例</p>
<p>一张 1024x1024 的图，内存占用大小为 1024x1024x（32bit/8）= 4194304byte = 4096kb = 4m,那么我们的存储数组内存大小为 4096/32 = 128kb，应该可以应付绝大多数的变态内存需求了。</p>
</li>
<li>
<p>先扫描进行存储的目的是避免用户进行操作时再进行扫描，这样可能在平凡操作中用户会有卡顿感以及手感不舒服。</p>
</li>
<li>
<p>此处实现不一定判断值为 0，可为用户指定临界值。</p>
</li>
</ul>
