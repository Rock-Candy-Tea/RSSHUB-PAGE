<h2 data-id="heading-0">分治法</h2>
<blockquote>
<p>分治法是建基于多项分支递归的一种很重要的算法范型。字面上的解释是“分而治之”，就是把一个复杂的问题分成两个或更多的相同或相似的子问题，直到最后子问题可以简单的直接求解，原问题的解即子问题的解的合并。</p>
</blockquote>
<p>比较典型的有：排序算法（归并排序、快速排序）、傅立叶变换（快速傅立叶变换）等；</p>
<p>其中最重要最常考的是<strong>快排</strong>，应该都很熟悉了，快速过一遍：</p>
<ol>
<li>选择一个元素作为"基准"（pivot）。</li>
<li>于"基准"的元素，都移到"基准"的左边；所有大于"基准"的元素，都移到"基准"的右边。</li>
<li>准"左边和右边的两个子集，不断重复第一步和第二步，直到所有子集只剩下一个元素为止。</li>
</ol>
<pre><code class="copyable">var quickSort = function(arr) &#123;

　　var right = [];

　　　　　　left.push(arr[i]);

<h2 data-id="heading-1">动态规划法</h2>
<p>动态规划太重要了！动态规划于算法，可能相对于杜兰特于篮网吧~ ORZ</p>
<blockquote>
<p>动态规划（简称DP）背后的基本思想非常简单。即：大致上，若要解一个给定问题，我们需要解其不同部分（即子问题），再根据子问题的解以得出原问题的解。</p>
</blockquote>
<p>其中最著名的有<strong>背包问题</strong>、<strong>爬梯子问题（斐波那契数列）</strong>、<strong>寻找最长公共子串</strong> 这三个（每个都是重点常考！）。</p>
<ul>
<li><strong>背包问题</strong></li>
</ul>
<p>背包问题的基础是【0-1背包问题】，先吃透它：</p>
<p>题目：有 N 件物品和一个容量为 C 的背包。第 i 件物品的重量是 w[i]，价值是 v[i]。求解将哪些物品装入背包可使价值总和最大。</p>
<p>解：：每种物品仅有一件，可以选择放或不放。用 <em>子问题定义状态</em>：即 f[i][w] 表示前 i 件物品恰放入一个容量为 c 的背包可以获得的最大价值。则其状态转移方程便是：</p>
<pre><code class="copyable">f[i][c] = max&#123;f[i-1][c], f[i-1][c-w[i]]+v[i]&#125; 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这个方程的释义，即分别对应两种情况：</p>
<ol>
<li>
<p>如果背包不放第 i 件物品，那么问题就转化为 前 i-1 件物品放入容量为 v 的背包中，价值为 f[i-1][c]；</p>
</li>
<li>
<p>如果放第i件物品，那么问题就转化为 前 i-1 件物品放入剩下的容量为 c-w[i] 的背包中，此时能获得的最大价值就是 f[i-1][c-w[i]]再加上通过放入第 i 件物品获得的价值 v[i]，即f[i-1][c-w[i]]+v[i]。</p>
</li>
</ol>
<p>这两者的最大值就是我们最终的解！</p>
<blockquote>
<p>物品一个一个选，容量也一点一点增加去考虑，这一点是「动态规划」的思想。</p>
</blockquote>
<pre><code class="copyable"> function knapsack(weights, values, W)&#123;
    var n = weights.length -1
    var f = [[]]
    for(var j = 0; j <= W; j++)&#123;

                f[i][j] = Math.max(f[i-1][j], f[i-1][j-weights[i]] + values[i]) 
            &#125;
        &#125;
    &#125;
    return f[n][W]

<li><strong>爬梯子问题</strong></li>

<p>爬梯子也是算法高频考点无疑了！</p>
<p>题目：你准备要爬楼梯。你面对的是一个 n 步可以走上去的楼梯。你每次可以走一步或者两步，那么你有几种不同的方式可以爬上去吗？</p>
<p>解：每次只能爬 1 步或 2 步，爬到第 n 层的方法要么是从第 n-1 层 1 步上来的，要不就是从 n-2 层 2 步上来的。采用递归！但要注意避免递归中运算重复的部分：</p>

    &#125;

        return temp[n]
    &#125;

<p>斐波那契数列指的是这样一个数列从第3项开始，每一项都等于前两项之和，比如：1, 2, 3, 5, 8, 13, 21......，用到递归无疑了，但是一定要记得缓存递归项，否则会内存溢出（比如<code>climbStairs(5)</code>分解为<code>climbStairs(3)</code>和<code>climbStairs(4)</code>，<code>climbStairs(4)</code>又分解为<code>climbStairs(2)</code>和<code>climbStairs(3)</code>，这样就有两个重复的<code>climbStairs(3)</code>了）。</p>

<h2 data-id="heading-2">贪心法</h2>
<p>贪心算法，即使你不怎么用，一定也听过它的大名！</p>
<blockquote>
<p>贪心算法（英语：greedy algorithm），又称贪婪算法，是一种在每一步选择中都采取在当前状态下最好或最优（即最有利）的选择，从而希望导致结果是最好或最优的算法。</p>
</blockquote>
<p>最典型的例子有：旅行商问题（最短路径问题）（TSP）</p>
<p>之前有写过一篇关于最短路径问题的文章：<a href="https://juejin.cn/post/6905692756190953480" target="_blank">《会一会改变世界的图算法——Dijkstra（狄克斯特拉）算法》</a>。</p>
<p>与之最大不同的是，旅行商问题是一个 <em>NP 问题</em>，即只是一个近似算法，没有完全准确的解，所以需要尽可能的“贪心”。</p>
<p>题目：给定一系列城市和每对城市之间的距离，求解访问每一座城市一次并回到起始城市的最短回路。</p>
<p><a href="https://leetcode-cn.com/problems/shortest-path-visiting-all-nodes/solution/lu-xing-shang-wen-ti-by-liyongjia-li-lue/" target="_blank" rel="nofollow noopener noreferrer">leetcode#847</a></p>
<h2 data-id="heading-3">回溯法</h2>
<blockquote>
<p>回溯算法实际上一个类似枚举的搜索尝试过程，主要是在搜索尝试过程中寻找问题的解，当发现已不满足求解条件时，就“回溯”返回，尝试别的路径。</p>
</blockquote>
<p>回溯算法和穷举法很像，都是树的深度优先遍历，但回溯法会进行'剪枝'，比如第 5 层某 i 叶子结点时发现该节点已经无意义，会直接跳过该节点下面的遍历，提高了效率。</p>
<p>回溯法通常用最简单的递归方法来实现，在反复重复上述的步骤后可能出现两种情况：</p>
<ul>
<li>找到一个可能存在的正确的答案</li>
<li>在尝试了所有可能的分步方法后宣告该问题没有答案</li>
</ul>
<p>在最坏的情况下，回溯法会导致一次复杂度为指数时间的计算。</p>
<p>著名的是 <a href="https://zh.wikipedia.org/wiki/%E5%85%AB%E7%9A%87%E5%90%8E%E9%97%AE%E9%A2%98" target="_blank" rel="nofollow noopener noreferrer">八皇后问题</a>。像这种<strong>棋盘问题</strong>也是高频考点。（面试腾讯 PCG 遇到过）</p>
<p>题目：在 8×8 的国际象棋棋盘上放置八个皇后，使得任何一个皇后都无法直接吃掉其他的皇后。为了达到此目的，任两个皇后都不能处于同一条横行、纵行或斜线上。问，一共有多少情况可满足？</p>
<p>解：
由于一行只能有一个皇后，所以选择一行一行地填写皇后。在填第n行的皇后时不能与[0, n-1]行已填写的皇后在同一列、同一正对角线与反对角线上。若满足条件则继续递归，否则回溯重新选择下一列。</p>
