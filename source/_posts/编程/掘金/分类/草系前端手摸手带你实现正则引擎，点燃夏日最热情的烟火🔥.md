<h1 data-id="heading-3">有限自动机基础</h1>
<h2 data-id="heading-4">章节引论—有限自动机（FA）</h2>
<blockquote>
<p>tip: 如果看不懂特点和形式定义中的话，请大家移步后面<code>我来总结一下</code>部分，帮助大家对有限自动机的定义有一个粗略的理解。</p>
</blockquote>
<p>首先在开始下面的话题之前，我们有必要去了解一下，什么是有限自动机， 有限自动机也被称为：<code>时序机</code>。</p>
<p><strong>有限自动机有以下特点</strong>：</p>
<ul>
<li>
<p>系统具有有限个状态，不同的状态代表不同的意义。按照实际的需要，系统可以在不同的状态下完成规定的任务。</p>
</li>
<li>
<p>我们可以将输入字符串中出现的字符汇集在一起构成一个字母表。系统处理的所有字符串都是这个字母表上的字符串。</p>
</li>
<li>
<p>系统在任何一个状态下，从输入字符串中读入一个字符，根据当前状态和读入的这个字符转到新的状态。</p>
</li>
<li>
<p>系统中有一个状态，它是系统的开始状态。</p>
</li>
<li>
<p>系统中还有一些状态表示它到目前为止所读入的字符构成的字符串是语言的一个句子。</p>
</li>
</ul>
<p><strong>有限自动机的形式定义</strong>：</p>
<p>有限状态自动机是一个五元组 <code>M=(Q, Σ, δ, q0, F)</code>，其中：</p>
<ul>
<li>
<p>Q——状态的非空有穷集合。∀q∈Q，q称为M的一个状态。</p>
</li>
<li>
<p>Σ——输入字母表。</p>
</li>
<li>
<p>δ —— 状态转移函数，有时又叫作状态转换函数或者移动函数，δ：Q×Σ→Q，δ(q,a)=p。</p>
</li>
<li>
<p>q0 —— M的开始状态，也可叫作初始状态或启动状态。q0∈Q。</p>
</li>
<li>
<p>F —— M的终止状态集合。F被Q包含。任给q∈F，q称为M的终止状态</p>
</li>
</ul>
<p><strong>我来总结一下</strong>：</p>
<p>大家不要被上面列出的一大堆定义和特点绕晕，其实总结起来非常简单：</p>
<ul>
<li>首先有限状态机有一个开始状态，具一个不太恰当的例子：比如说你要提交一个工单，你会经历直属领导审批，人事审批，老总审批几个阶段。那么<code>提交工单</code>就是这个工单系统的有限自动机的开始状态，对应五元组里面的<code>q0</code></li>
<li>当然，除了开始状态我们还有一个结束状态，那么对于我们提交工单这个流程来讲，我们的结束状态就是<code>工单成功</code>或者<code>工单结束</code>，大家可以发现这个有限自动机具有两个终止状态，所以终止状态是一个集合，并不一定只有一个结束状态，这个结束状态就是五元组里面的<code>F</code></li>
<li>还有我们有限自动机有限两个字很重要，有限指的是具有有限个状态，就像我们下图中的<code>提交工单</code>,<code>待上级领导审批</code>，<code>待人事审批</code>，<code>待老总审批</code>， <code>工单成功</code>， <code>工单失败</code>就是我们的状态集，即五元组中的<code>Q</code></li>
<li>那么作为自动机，我们该怎么知道我们的下一个状态是啥呢，所以其实我们需要两个东西，一个是现在的状态，一个是状态转移函数δ，比如：我们现在的阶段是人事审批，我们的输入其实就是人事审批，经过<code>状态转移函数（审批结果判断）</code>之后我们就可以得到下一步的状态，状态可能是<code>工单失败</code>或者<code>待老总审批</code>。</li>
</ul>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/144a5a81fea4462abcd877aafe08c75d~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>不知道经过我的举例之后，大家有没有对定义有了一定的理解，其实总结起来很简单，其实：</p>
<blockquote>
<p>有限自动机 = 有限的内部状态集合 + 一组控制规则</p>
</blockquote>
<h2 data-id="heading-5">DFA — 确定有限自动机</h2>
<h3 data-id="heading-6">定义</h3>
<p>前文我们已经了解了有限自动机，那么确定有限自动机有哪些特别的点呢，下面我们来看一下<code>确定有限自动机</code>的定义：</p>
<ul>
<li>确定有限自动机M为一个五元组 M = ( S, ∑, s0, f, Z)</li>
<li>S:一个有穷状态集，它的每个元素称为一个状态;</li>
<li>∑:一个有穷字母表，它的每个元素称为一个输入字符;</li>
<li>s0∈S:唯一的初始状态(开始状态);</li>
<li>f:状态转换函数:S×∑→ S，且单值函数，f(Si,a)=Sk。<code>当前状态Si，遇输入字符a时，自动机将唯一地转换到状态 Sk，称Sk为 Si的一个后继状态</code>;</li>
<li>Z⊆S:终止状态集(可接受状态集、结束状态集) 其中每个元素称为终止状态(可接受状态、结束状态), Z可空.</li>
</ul>
<p>我来总结一下：</p>
<blockquote>
<p>我们其实可以和上面的形式定义做一个比较，我们会发现一些很重要的点：</p>
<ul>
<li>初始状态唯一</li>
<li>状态转换函数是单值函数</li>
<li>终止状态集Z可以为空</li>
</ul>
</blockquote>

<h3 data-id="heading-8">DFA接受的字符串</h3>
<ul>
<li>
<p>对于 ∑ 中任何字符串t，若存在一条从初始结点到某一终止结点 的路径，且这条路上所有弧上的标记符连接成的字符串等于t， 则称 t 可为DFA M所接受</p>
</li>
<li>
<p>若DFA M的初始状态同时又是终止状态，则空字符串可为DFA M所接受.</p>
</li>
<li>
<p>DFA M 所能接受的字符串的全体记为L(M)</p>
</li>
</ul>

<h3 data-id="heading-9">DFA的确定性</h3>
<p>那么我们为什么说 DFA 是确定有限自动机呢，确定这两个字体现在哪里呢？</p>
<ul>
<li>初始状态唯一</li>
<li>状态转换函数f: S×∑→S是一个单值函数，即对任何状态s∈S，输入 符号a∈∑， f(S, a)唯一确定下一状态.</li>
</ul>

<h2 data-id="heading-11">NFA — 非确定有限自动机</h2>
<h3 data-id="heading-12">定义</h3>
<p>非确定有限自动机M为一个五元组 M = ( S, ∑, S0, f, Z)</p>
<ul>
<li>S: 一个有穷状态集，它的每个元素称为一个状态;</li>
<li>∑: 一个有穷字母表，它的每个元素称为一个输入字符;</li>
<li>S0 ⊆ S: 非空初始状态集;</li>
<li>Z⊆S: 终止状态集;</li>
<li>f : 状态转换函数，是从S×(∑∪&#123;ε&#125;) 到 S 子集的映射，即S×(∑∪&#123;ε&#125;)→ 2^S</li>
</ul>
<blockquote>
<p>注意，这里的后继状态不是单一状态，而是状态集S的子集, 即转换函数不是单值.</p>
</blockquote>
<p>我这里总结一下，大家看区别就是<code>状态转换函数的结果不再是单值</code>了, <code>起始状态也这是一个集合</code>而不是一个确定的值，以及转换函数的的输入是<code>∑∪&#123;ε&#125;</code>就表示<code>有向弧上面的标记可以是空</code>。</p>

f(<span class="hljs-number">0</span>,b)=&#123;<span class="hljs-number">0</span>&#125; f(<span class="hljs-number">1</span>,ε)=&#123;<span class="hljs-number">2</span>&#125; f(<span class="hljs-number">2</span>,b)=&#123;<span class="hljs-number">2</span>&#125;

<h2 data-id="heading-15">DFA 与 NFA 对照</h2>
<p>那在本章结束之前，我们回顾一下～</p>
<p>DFA：</p>
<ul>
<li>开始状态唯一</li>
<li>状态转换函数为单值函数</li>
</ul>
<p>NFA：</p>
<ul>
<li>开始状态是一个状态集合</li>
<li>状态转换函数的结果是一个集合</li>
<li>有向弧上面的标记可以是空</li>
</ul>

<p>前文我们已经讲解过了 DFA 和 NFA，即确定有限自动机和非确定有限自动机，根据前面的铺垫想必各位大佬已经可以将正则引擎与自动机关联起来了，而正则引擎大体也可以分成这样的两大类，即：DFA 正则引擎和 NFA 正则引擎。</p>
<h2 data-id="heading-18">DFA引擎</h2>
<h3 data-id="heading-19">举个例子</h3>
<p>我们直接举一个比较简单的例子：</p>
<p>正则表达式是 <code>a[db]c</code><br>
待匹配的字符串是 <code>abc</code></p>
<blockquote>
<p>此处我们使用‘[]’的原因是第三章<code>手摸手，带你实现简易版正则引擎</code>我们将会去实现‘[]’</p>
</blockquote>
<p>下面我们开始匹配：</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c77a2edd8dfc40b8953e6be4aa42a7c9~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>不知道大家有没有理解，我描述一下对比的过程：</p>
<ol>
<li>第一次是字符 a 和正则表达式 a 比较</li>
<li>匹配成功后，是字符 b 同时比较表达式中的  b 和 d</li>
<li>再次匹配成功，字符 c 和正则表达式 c 比较</li>
<li>匹配成功</li>
</ol>
<p>这里面我们值得注意的点是，第二次匹配是 b 同时和 b， d 进行比较，所以可能会消耗更多的内存。</p>
<h3 data-id="heading-20">特点</h3>
<p>我们从上面的例子可以看出一些DFA正则引擎的特点：</p>
<ul>
<li><code>文本主导</code>：按照文本顺序执行，所以保证了DFA正则引擎的确定性</li>
<li><code>记录当前所有有效可能</code>：正如前文示例中的第二次匹配一样，同时比较了 b 和 d ，所以需要消耗更大的内存</li>
<li><code>每个字符只检查一次</code>：提高了执行效率，因为没有回溯操作重复匹配的过程</li>
<li><code>不能使用反向引用等功能</code>：因为每个字符只检查一次，只记录当前比较值，所以不能使用反向引用、环视等一些功能</li>
</ul>
<h2 data-id="heading-21">NFA引擎</h2>
<h3 data-id="heading-22">举个例子</h3>
<p>例子还是刚才的例子，方便大家对照：</p>
<p>正则表达式是 <code>a[db]c</code><br>
待匹配的字符串是 <code>abc</code></p>
<p>下面我们开始匹配：</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b077cbe427184c95b4b1d8bc1550274a~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>这里和前面的DFA模式做一下对比，我们会发现区别，NFA引擎在匹配之前会记录字符的位置，然后选择其中一个可能状态进行匹配，如果匹配失败，会进行回溯，进入其他分支进行匹配。</p>
<h3 data-id="heading-23">特点</h3>
<p>我们从上面的例子可以看出一些NFA正则引擎的特点：</p>
<ul>
<li><strong>文表达式主导</strong>：按照表达式的一部分执行，如果不匹配换其他部分继续匹配，直到表达式匹配完成。</li>
<li><strong>会记录某个位置</strong>：我们看到当执行到<code>[db]</code>时，NFA引擎会记录字符的位置，然后选择其中一个先匹配。</li>
<li><strong>单个字符可能检查多次</strong>：我们看到当执行到<code>[db]</code>时，比较<code>d</code>后发现不匹配，于是NFA引擎换表达式的另一个分支<code>b</code>，同时文本位置<strong>回溯</strong>，重新匹配字符'b'。这也是NFA引擎是非确定型的原因，同时带来另一个问题效率可能没有DFA引擎高。</li>
<li><strong>可实现反向引用等功能</strong>：因为具有<strong>回溯</strong>这一步，所以可以很容易的实现反向引用等一些功能！</li>
</ul>

<h2 data-id="heading-28">思路解析</h2>

<p>那么，我们首先要思考我们的目标是啥，既然我们本篇文章的主题是<code>自动机</code>，也没必要卖关子， 我们第一步想到的肯定是， <code>把正则表达式转换成自动机</code>， 之后<code>借助这个正则匹配的自动机去匹配我们的字符串</code>。</p>
<p>那么我们<strong>如何把一个正则表达式转换成一个自动机</strong>呢？<br>

<pre><code class="hljs language-mermaid" lang="mermaid">graph TD

<p>我来简单解读一下，我会把这个问题分成两部分，首先我需要解析字符串，之后转换成具有匹配含义的独立单元序列，即 TOKEN 序列。什么叫做具有匹配含义的独立单元序列呢？</p>
<p>我举个例子：</p>

<p>这里其实就是一个递归程序：</p>
<ul>
<li><strong>如果自动机的当前处于起始状态</strong>：不进行匹配，循环匹配 <code>next</code>，只要有一条分支匹配成功，就是合法字符串</li>
<li><strong>如果自动机的当前处于结束状态</strong>：判断方法传入的 <code>str</code> 长度是否是 <code>0</code> ，如果是 <code>0</code> 则表示待匹配字符串也匹配完成了，是合法字符串，反之不合法。</li>
<li><strong>其他情况</strong>：匹配当前字符是否在 <code>pattern</code> 数组中，如果在就表示当前字符匹配，继续循环匹配 <code>next</code>，只要有一条分支匹配成功，就是合法字符串</li>
</ul>
