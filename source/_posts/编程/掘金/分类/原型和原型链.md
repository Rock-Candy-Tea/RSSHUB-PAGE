<h3 data-id="heading-0">构造函数</h3>
<p>是一个专门用来创建对象的函数，当需要创建的对象格式一样时，我们就可以使用构造函数来创建。如下：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">User</span>(<span class="hljs-params">name, age</span>) </span>&#123;

    <span class="hljs-built_in">this</span>.age = age;

<p>这样是不是比我们平时用对象字面量来创建更加方便呢？<br>
我们接着往下看:</p>

<p>这个时候，<code>user1</code> 和 <code>user2</code> 的 <code>sayHellow</code> 函数用的就是同一个了。</p>
<p>而且，当我们直接输出 <code>user1</code> 或者 <code>user2</code> 查看对象的时候，是看不到 <code>sayHello</code> 这个函数的，那我们怎么可以调用呢？这就得说说原型了，也就是我们今天的主角。</p>
<p>使用构造函数的注意事项：</p>
<ol>
<li>为了便于分清楚普通函数和构造函数，我们一般把构造函数的首字母大写，也就是大驼峰命名！</li>
<li>构造函数如果不 <code>new</code> 的话，和普通函数没有任何区别！</li>
<li>当你 <code>new</code> 一个构造函数的时候，构造函数内部会生成一个空对象，并且 <code>this</code> 会自动指向这个空对象，最后会自动返回 <code>this</code> 指向的这个对象。如果手动 <code>return</code> 一个原始值的话，构造函数会把这个 <code>return</code> 忽略，(还是返回 <code>this</code> 指向的那个对象)；如果是 <code>return</code> 一个引用值的话，那么就会把这个引用值当作返回对象，忽略 <code>this</code> 指向的那个对象。</li>
</ol>
<h3 data-id="heading-1">原型</h3>
<p>原型，其实就是一个普通的对象，每一个函数对象，都有一个原型（<code>prototype</code>），现在我们就来看看原型长什么样吧，我们通过 <code>__proto__</code> 来访问原型：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">User</span>(<span class="hljs-params"></span>) </span>&#123;&#125;;
<span class="hljs-keyword">const</span> user = <span class="hljs-keyword">new</span> User();
<span class="hljs-built_in">console</span>.log(user.__proto__); <span class="hljs-comment">// &#123;constructor: f&#125;;</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们可以看到，<code>user</code> 的原型是一个对象，对象中有一个 <code>constructor</code> 的属性，我们把它叫做构造器，它的值一个函数，我们再来看看 <code>constructor</code> 的值是什么吧：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">// 接着上个代码块</span>
<span class="hljs-built_in">console</span>.log(user.__proto__.constructor); <span class="hljs-comment">// function User() &#123;&#125;;</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>从上面的输出，我们可以看到 <code>user.__proto__.constructor</code> 指向的是构造出 <code>user</code> 这个实例对象的构造函数 <code>User</code>。就如下图所表示的原因：</p>

<p>user1 上根本就没有 <code>sayHello</code> 这个方法呀，那么它是怎么找到这个函数并且成功执行的呢？</p>
<p>这个可全要归功于 <code>__proto__</code> ，这个大功臣呀，让我来简单的介绍一下。</p>
<h3 data-id="heading-2"><strong>proto</strong></h3>
<p>每个对象身上都有一个属性，那就是 <code>__proto__</code> 了，它是用来指向构造出它的构造函数的原型对象的，相信刚刚上面的示例代码，大家也可以看出来吧。它还有个功能，让我们来看看下面的代码吧：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">A</span>(<span class="hljs-params"></span>) </span>&#123;&#125;;
A.prototype.name = <span class="hljs-string">'aaaa'</span>;
<span class="hljs-keyword">const</span> a = <span class="hljs-keyword">new</span> A();

<span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">B</span>(<span class="hljs-params"></span>) </span>&#123;&#125;;
B.prototype = a;
<span class="hljs-keyword">const</span> b = <span class="hljs-keyword">new</span> B();
    
<span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">C</span>(<span class="hljs-params"></span>) </span>&#123;&#125;;
C.prototype = b;
<span class="hljs-keyword">const</span> c = <span class="hljs-keyword">new</span> C();
<span class="hljs-built_in">console</span>.log(c.name); <span class="hljs-comment">// aaaa</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p>我们明明没有给 <code>c</code> 添加 <code>name</code> 属性，为什么可以输出呢？</p>
<p>这是因为，当 <code>c</code> 想要访问 <code>name</code> 的值时，它会现在自身查找，自身没有时，它会沿着隐式原型 <code>__proto__</code> 去找 <code>C.prototype</code>，看它的身上有没有，如果没有的话，他就会继续沿着 <code>__proto__</code> 去找 <code>B.prototype</code>,看它的身上有没有，如果没有的话，他就会继续沿着 <code>__proto__</code> 去找 <code>A.prototype</code>,然后看到 <code>A.prototype</code> 上有 <code>name</code> 这个属性，那么就会返回 <code>name</code> 的属性值。如果还没有找到，它就会继续沿着隐式原型 <code>__proto__</code> 去找 <code>Object.prototype</code>，看它的身上有没有，如果没有的话，他就会沿着 <code>__proto__</code> 去找，然后返回 <code>null</code>。这样一层一层靠着 <code>__proto__</code> 形成的链式结构，我们就把它叫做原型链。</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-built_in">console</span>.log(<span class="hljs-built_in">Object</span>.prototype.__proto__ === <span class="hljs-literal">null</span>); <span class="hljs-comment">// true</span>
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>null</code> 是原型链的顶端！</p>
<p>所以有些时候，我们明明没有创建一些方法，却可以使用，就是要归功于原型链！</p>
