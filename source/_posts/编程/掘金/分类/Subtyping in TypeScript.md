<h2 data-id="heading-0">Subtyping 和 Inheritance 的区别</h2>
<p>千里冰封小时候，在知乎上提过一个问题——<a href="https://www.zhihu.com/question/57486254" target="_blank" rel="nofollow noopener noreferrer">subtyping 和 inheritance 的区别是什么？</a>，大家可以看看这里的回答。</p>
<p>用<a href="https://www.cmi.ac.in/~madhavan/courses/pl2009/lecturenotes/lecture-notes/node28.html" target="_blank" rel="nofollow noopener noreferrer">一句话</a>总结一下：</p>
<ul>
<li>Subtyping refers to compatibility of interfaces. A type B is a subtype of A if every function that can be invoked on an object of type A can also be invoked on an object of type B.</li>
<li>Inheritance refers to reuse of implementations. A type B inherits from another type A if some functions for B are written in terms of functions of A.</li>
</ul>
<p>java 的 class extends 同时声明了类型上的关系 —— subtyping 和实现上的重用 —— inheritance，所以容易引起大家的混淆。</p>
<h2 data-id="heading-1">类型构造器</h2>
<p><del>用维基百科的话就是：一个类型构造器就是把若干已知类型组合成一新类型的手段。可以看作是类型的构造函数。</del></p>
<p>我的理解就是，部分内容是类型，它整体还是类型的东西。比如
TS 中 <code>number</code> 是类型，<code>number[]</code> 也是类型，这里面有个固定的方法，把某个类型变成另一个类型，就是 <code>? []</code> 这里的 <code>?</code> 可以换成任意的类型。</p>
<p>我们把这个类型构造器写出来，这是一元类型构造器 <code>T[]</code>,其中</p>
<ul>
<li>T 是一个 meta variable，类似于我们之前学的形式参数。可以被实际参数替换，比如 number。这里我们叫它 meta variable，是为了和编程语言里的 variable 做区分。</li>
<li>这个类型构造器接受一个参数，所以是「一元」</li>
<li>在之前的例子中，它的作用是把 number 类型转变成为 number 数组类型。number 是一个类型，<code>number []</code>是一个类型，<code>number []</code> 是这个构造器接受 <code>number</code> 类型后生成的新类型。</li>
</ul>
<p>构造函数函数是从一个或多个值构成另一个值的抽象手段，比如 <code>const getArray = x => new Array(x)</code>
而类型构造器是从一个或者多个类型构成另一个类型的抽象手段，比如 <code>type Array<A> = A[]</code></p>
<p>有些人也会把类型构造器叫做类型上的构造函数。</p>

<h2 data-id="heading-8">Mutable ref</h2>
<p>注意，我在上面写的例子都加上了 readonly，因为它如果是 mutable 的，他就不安全了。</p>

  <span class="hljs-attr">current</span>: <span class="hljs-number">1</span>,

<span class="hljs-keyword">const</span> ref2: &#123;

<p>但是 TypeScript 为了大家学习简单一些，这里做了协变处理。</p>
<h2 data-id="heading-9">Method 参数双变</h2>
<p>我们下面的讨论都开启 strictFunctionTypes。</p>
<p>我们观察到 <a href="https://www.typescriptlang.org/tsconfig/#strictFunctionTypes" target="_blank" rel="nofollow noopener noreferrer">strictFunctionTypes</a> 的文档给了一个例子</p>
<pre><code class="hljs language-ts copyable" lang="ts"><span class="hljs-keyword">type</span> Methodish = &#123;
  func(x: <span class="hljs-built_in">string</span> | <span class="hljs-built_in">number</span>): <span class="hljs-built_in">void</span>;
&#125;;

<span class="hljs-function"><span class="hljs-keyword">function</span> <span class="hljs-title">fn</span>(<span class="hljs-params">x: <span class="hljs-built_in">string</span></span>) </span>&#123;
  <span class="hljs-built_in">console</span>.log(<span class="hljs-string">'Hello, '</span> + x.toLowerCase());
&#125;

<span class="hljs-comment">// Ultimately an unsafe assignment, but not detected</span>
<span class="hljs-keyword">const</span> m: Methodish = &#123;
  <span class="hljs-attr">func</span>: fn,
&#125;;
m.func(<span class="hljs-number">10</span>);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这个例子说明了，现在大量的人使用了不安全的继承，所以 TS 特别允许了 method 的参数双变，虽然它是不安全的</p>
<p>这个行为影响到了很多地方，举个例子</p>
<pre><code class="hljs language-ts copyable" lang="ts"><span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">List</span><<span class="hljs-title">T</span>> </span>&#123;
  push(x: T): <span class="hljs-built_in">void</span> &#123;
    <span class="hljs-keyword">throw</span> <span class="hljs-keyword">new</span> <span class="hljs-built_in">Error</span>(<span class="hljs-string">'unimplemented'</span>);
  &#125;
  pop(): T &#123;
    <span class="hljs-keyword">throw</span> <span class="hljs-keyword">new</span> <span class="hljs-built_in">Error</span>(<span class="hljs-string">'unimplemented'</span>);
  &#125;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>提问： <code>List<number></code> 和 <code>List<unknown></code> 的子类型关系</p>
<p>我们观察一下这里的泛型参数出现在了 push 的参数和 pop 的返回值。而 method 的参数双变，也就是 T 应该或者逆变或者协变。T 出现在 pop 的返回值，所以 T 应该协变。结合一下两条要求，得到 T 是协变的，所以 <code>List<number></code> 是 <code>List<unknown></code> 的子类型</p>
<pre><code class="hljs language-ts copyable" lang="ts"><span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">List2</span><<span class="hljs-title">T</span>> </span>&#123;
  <span class="hljs-attr">push</span>: <span class="hljs-function">(<span class="hljs-params">x: T</span>) =></span> <span class="hljs-built_in">void</span> &#123;
    <span class="hljs-keyword">throw</span> <span class="hljs-keyword">new</span> <span class="hljs-built_in">Error</span>(<span class="hljs-string">'unimplemented'</span>);
  &#125;
  pop(): T &#123;
    <span class="hljs-keyword">throw</span> <span class="hljs-keyword">new</span> <span class="hljs-built_in">Error</span>(<span class="hljs-string">'unimplemented'</span>);
  &#125;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>提问： <code>List2<number></code> 和 <code>List2<unknown></code> 的子类型关系</p>
<p>这里的 T 出现在了 push 的参数位置，但是 push 不是 method，只是普通的 field，而且 这个 field 是函数类型，函数的参数逆变，而 pop 要求 T 协变，所以 T 不变。也就是 <code>List2<number></code> 和 <code>List2<unknown></code> 无子类型关系。</p>
<p>这个 demo 解释了 为什么 TS 中的 <code>Array<T></code> 的 T 是协变的。你们可以点进 Array 的 <code>.d.ts</code> 看，你会发现 <code>Array<T></code> 的 T 出现在 method 的参数和返回值，还有 mutable field 的值。而 TS 专门做了两个不安全的设定，method 参数双变，mutable 的字段的类型协变，导致了 <code>Array<T></code> 的 T 是协变的。</p>
<p>在这里我建议大家开启一下 eslint 的这条 rule <a href="https://github.com/typescript-eslint/typescript-eslint/blob/master/packages/eslint-plugin/docs/rules/method-signature-style.md" target="_blank" rel="nofollow noopener noreferrer">method-signature-style</a>，并且选择 property 选项。这才能使自己的代码更安全。</p>
<p>如果开启了这条 rule，你需要做类型继承时候，可以这样子做</p>
<pre><code class="hljs language-ts copyable" lang="ts"><span class="hljs-keyword">interface</span> T1 &#123;
  <span class="hljs-keyword">readonly</span> func: <span class="hljs-function">(<span class="hljs-params">arg: unknown</span>) =></span> unknown;
  <span class="hljs-keyword">readonly</span> bar: <span class="hljs-built_in">string</span>;
  <span class="hljs-keyword">readonly</span> baz: <span class="hljs-built_in">string</span>;
&#125;

<span class="hljs-keyword">interface</span> T2 <span class="hljs-keyword">extends</span> Omit<T1, 'func'> &#123;
  <span class="hljs-keyword">readonly</span> func: <span class="hljs-function">(<span class="hljs-params">arg: <span class="hljs-built_in">string</span></span>) =></span> <span class="hljs-built_in">string</span>;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这时候 T1 和 T2 没有任何子类型关系，非常安全。</p>
