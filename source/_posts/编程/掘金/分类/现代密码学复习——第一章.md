<h2 data-id="heading-1"><a href="https://juejin.cn/post/6948666131276267533"></a>1：保密学的基本概念</h2>
<h3 data-id="heading-2"><a href="https://juejin.cn/post/6948666131276267533"></a>什么是保密学？</h3>
<blockquote>
<p>定义：保密学是研究信息系统安全保密的科学，它包括两个分支，一是密码学，对信息进行编码实现隐蔽信息的科学，二是密码分析学，研究分析破译密码的科学。二者相互对立，而又互相促进地向前发展。</p>
</blockquote>
<h3 data-id="heading-3"><a href="https://juejin.cn/post/6948666131276267533"></a>其他相关概念</h3>

































































<table><thead><tr><th>名词</th><th>概念</th></tr></thead><tbody><tr><td>明文</td><td>需要被隐蔽的消息</td></tr><tr><td>密文</td><td>明文经密码变换成的一种隐蔽形式</td></tr><tr><td>加密</td><td>将明文变换为密文的过程</td></tr><tr><td>解密</td><td>加密的逆过程</td></tr><tr><td>加密员</td><td>对密码进行操作的人员</td></tr><tr><td>加密算法</td><td>密码员对明文进行加密时所采用的一种规则</td></tr><tr><td>接收者</td><td>传送消息的预定对象</td></tr><tr><td>解密算法</td><td>接收者对密文进行解密时所采用的一组规则</td></tr><tr><td>密钥</td><td>控制加密和解密算法操作的数据处理，分别称作加密密钥和解密密钥</td></tr><tr><td>截收者</td><td>在信息传输和处理系统中的非授权者，通过搭线窃听、电磁窃听、等来窃取机密信息</td></tr><tr><td>密码分析</td><td>截收者试图通过分析从截获的密文推断出原来的明文或密钥</td></tr><tr><td>密码分析员</td><td>从事密码分析的人</td></tr><tr><td>被动攻击</td><td>对一个保密系统采取截获密文进行分析的攻击</td></tr><tr><td>主动攻击</td><td>攻击者主动向系统窜扰，采用删除，伪造等篡改手段，向系统注入假消息，达到自己的目的</td></tr></tbody></table>

<h3 data-id="heading-6"><a href="https://juejin.cn/post/6948666131276267533"></a>保密系统应当满足的要求</h3>

























<table><thead><tr><th>序号</th><th>要求</th></tr></thead><tbody><tr><td>1</td><td>系统即使理论上可破，也应该为实际上不可破，即从截获的密文或某些已知明文密文对，要决定密钥或任意明文在计算上是不可行的。</td></tr><tr><td>2</td><td>系统的保密性靠的不是加密体制或算法，而是密钥</td></tr><tr><td>3</td><td>加密和解密算法适用于所有密钥空间中的元素</td></tr><tr><td>4</td><td>系统便于实现和使用方便</td></tr></tbody></table>
<h3 data-id="heading-7"><a href="https://juejin.cn/post/6948666131276267533"></a>认证与认证系统</h3>
<ul>
<li>认证系统</li>
</ul>
<blockquote>
<p>为了防止消息被篡改、删除、重放和伪造的一种有效方法，使发送的消息具有被验证的能力，使接收者或第三者能够识别和确认消息的真伪，实现这类功能的密码系统被称作为认证系统。</p>
</blockquote>
<ul>
<li>保密性</li>
</ul>
<blockquote>
<p>保密性是使截获者在不知密钥条件下不能解读密文的内容。</p>
</blockquote>
<ul>
<li>认证性</li>
</ul>
<blockquote>
<p>让任何不知密钥的人不能构造一个密报，让合法接收者明白哪些是合法的消息。认证理论和技术是保密学研究的一个重要领域。如传统的手写签字正在被更迅速、更经济和更安全的数字签字替代。</p>
</blockquote>
<h3 data-id="heading-8"><a href="https://juejin.cn/post/6948666131276267533"></a>安全认证系统应当满足下面四个条件</h3>

























<table><thead><tr><th>序号</th><th>条件</th></tr></thead><tbody><tr><td>1</td><td>合法接收者能够检验消息的合法性和真实性</td></tr><tr><td>2</td><td>消息的发送者对所发送的消息不能依赖</td></tr><tr><td>3</td><td>除了合法的消息发送者之外，其他人不能伪造合法的消息，而且在已经明文和密文的情况下不能推断出密钥和伪造密文</td></tr><tr><td>4</td><td>必要时可由第三方做出仲裁</td></tr></tbody></table>
<h3 data-id="heading-9"><a href="https://juejin.cn/post/6948666131276267533"></a>完整性</h3>
<blockquote>
<p>在有自然和人为干扰条件下，系统保持检测错误和恢复消息和发送消息一致性的能力。实际中常常借助与纠检错技术和杂凑技术来保证消息的完整性。</p>
</blockquote>
<h2 data-id="heading-10"><a href="https://juejin.cn/post/6948666131276267533"></a>2：密码体制分类</h2>
<h3 data-id="heading-11"><a href="https://juejin.cn/post/6948666131276267533"></a>密码体制分类</h3>
<ul>
<li>单钥体制：加密密钥和解密密钥相同。</li>
<li>双钥体制：加密密钥和解密密钥不同。</li>
</ul>
<h3 data-id="heading-12"><a href="https://juejin.cn/post/6948666131276267533"></a>单钥体制的两个主要方面</h3>
<ul>
<li>密钥产生</li>
<li>密钥管理</li>
</ul>

<h3 data-id="heading-15"><a href="https://juejin.cn/post/6948666131276267533"></a>什么是公钥体制（也叫双钥体制）？</h3>
<p>Diffie和Hellman1976年首次提出，每个用户都有一对选定的密钥（公钥K1和私钥K2），公开的密钥K1可以像电话号码一样进行公布。</p>
<ul>
<li>双钥体制是如何进行加解密的？<br>
<img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/514882d2d9864c4397b6449cebabc99c~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"><br>

</ul>
<h3 data-id="heading-16"><a href="https://juejin.cn/post/6948666131276267533"></a>公钥体制的主要特点：加密和解密的能力分开</h3>
<ul>
<li><strong>公钥加密私钥解密</strong>：多个用户加密的消息，只能由一个用户进行解读，可用于公共网络中实现保密通信。</li>
<li><strong>私钥加密公钥解密</strong> ：只能由一个用户加密消息，而使多个用户可以解读，可用于认证系统中对消息进行数字签名。</li>
<li>无需事先分配密钥。</li>
</ul>
<h3 data-id="heading-17"><a href="https://juejin.cn/post/6948666131276267533"></a>双钥保密体制</h3>
<p><img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ec4f2078199842ba9c95b695798571ae~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"><br>

<ul>
<li>安全保障：从公开密钥Kb1和密文c不能推出B的私钥Kb2和明文。</li>
<li>由于任何人都可以使用B的公钥进行加密，所以密文c不具有认证性。</li>
</ul>

<p><img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cd27b65039f14b81aff0144001ef038a~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-19"><a href="https://juejin.cn/post/6948666131276267533"></a>双钥保密和认证体制</h3>
<p>所谓的双钥保密和认证体制就是说，假如A想发送消息给B，先用A的私钥对明文m进行加密，然后再使用B的公钥进行加密，这样就进行了两次加密过程，在解密的时候同样也是需要两次，先是使用B的私钥进行一次解密，然后使用A的公钥进行一次解密，这样B就可以获得需要的明文。<br>

<h2 data-id="heading-20"><a href="https://juejin.cn/post/6948666131276267533"></a>3：古典密码</h2>
<blockquote>
<p>古典密码是密码学的渊源，这些密码大都比较简单，可用手工或机械操作实现加解密，现在已经很少采用，然而研究这些密码的原理，对于理解、构造和分析现代密码都是十分有益的。</p>
</blockquote>
<h3 data-id="heading-21"><a href="https://juejin.cn/post/6948666131276267533"></a>代换密码</h3>
<ul>
<li>明文字母表A</li>
<li>密文字母表A’<br>
<img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/212009b8fd264a969294defd521a8bc0~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"></li>
<li>加密变换<br>
明文空间到密文空间的映射</li>
<li>判断有无数据拓展（明文字母表和密文字母表长度相同的时候）<br>
明文长度 = 密文长度 没有数据拓展<br>
明文长度 < 密文长度 有拓展<br>
明文长度 > 密文长度 明文数据被压缩<br>
再保密通信中需要拓展，认证系统中需要压缩，因为压缩是不可逆的，拓展存在很多种情况，所以拓展可以用来保密，压缩是唯一的，所以压缩可以用来进行认证。</li>
<li>什么是分组密码和流密码？</li>
</ul>
<blockquote>
<p>流密码指的是就一组序列密码，例如0101000010<br>
分组密码指的是密码被分成好几组，例如0101000 1010000</p>
</blockquote>
<p><img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d86ac7f9d6442d2a6b488bded06b269~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"></p>
<ul>
<li>什么是单表代换和多表代换？<br>
加解密使用同一个固定的代换表进行，我们称之为<strong>单表代换</strong>，例如经典的凯撒密码。<br>
<img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/6ef00e0f2bfc40dd8f09286342ee56ac~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"><br>
加解密使用一个以上的代换表进行代换，我们称之为<strong>多表代换</strong>，例如经典的维吉尼亚密码。具体选用哪个代换表进行，是根据密钥来的。<br>
<img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/55e30ea2fb054c3d90fee6b911139fbe~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"><br>

</ul>

<h3 data-id="heading-34"><a href="https://juejin.cn/post/6948666131276267533"></a>密码分析学</h3>
<blockquote>
<p>研究分析解密规律的科学。即截收者在不知道解密密钥及通信者所采用的的加密体制的细节条件下，对密文进行分析，试图获取机密信息。</p>
</blockquote>
<ul>
<li>密码分析在外交、军事、公安、商业等方面都具有重要作用，也是研究历史、考古、古语言学和古乐理论的重要手段之一。</li>
<li>密码分析之所以能够破译密码，最根本的是依赖于明文中的多余度。</li>
</ul>
<h3 data-id="heading-35"><a href="https://juejin.cn/post/6948666131276267533"></a>破译密码的方法</h3>
<ul>
<li>
<p>穷举破译法（也叫强力法）</p>
</li>
<li>
<p>分析法（有确定性和统计性两类）</p>
<ul>
<li>确定分析法：如已知密文或者明文密文对，用数学关系式表示出所求未知量（密钥）</li>
<li>统计分析法：密码破译者对截收的密文进行统计分析，总结出其间的统计规律，并与明文的统计规律进行对照比较，从中提取出明文和密文之间的对应或变换信息。</li>
</ul>
</li>
</ul>
<h3 data-id="heading-36"><a href="https://juejin.cn/post/6948666131276267533"></a>密码可能经受不同水平的攻击</h3>
<p><img alt class="lazyload" src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/82f70777f0394869ad5b186a66ca8dc2~tplv-k3u1fbpfcp-zoom-1.image" data-width="800" data-height="600" referrerpolicy="no-referrer"></p>
<h3 data-id="heading-37"><a href="https://juejin.cn/post/6948666131276267533"></a>无条件安全</h3>
<ul>
<li>对密码体制的任何攻击，都不优于对明文的完全盲目的猜测，这样的密码体制就称为无条件安全的。</li>
<li>一次一密的加密方式容易实现无条件安全性。因为密钥实时更新，所以以往得到的任何明文密文对，对于破译新的密文没有任何的帮助，只能做完全盲目的猜测。</li>
</ul>
<h3 data-id="heading-38"><a href="https://juejin.cn/post/6948666131276267533"></a>计算安全</h3>
<blockquote>
<p>计算安全是一个模糊的概念，我们可以给出以下三个级别的定义。</p>
</blockquote>
<ul>
<li>对密码体制的任何攻击，虽然可能优于完全盲目的猜测，但超出了计算者的计算能力。这是最高级别的计算安全。</li>
<li>对密码体制的任何攻击，虽然可能没有超出攻击者的计算能力，但所付出的代价远远大于破译成功所得到的利益。这是第二级别的计算安全。</li>
<li>可能没有超出攻击者的能力，但是破译成功所需要的时间远远大于明文本身的有效期限。这也是第二级别的计算安全。</li>
</ul>
