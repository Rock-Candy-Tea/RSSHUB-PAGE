<h2 data-id="heading-0">题目</h2>
<p>给你一个 32 位的有符号整数 <code>x</code> ，返回将 <code>x</code> 中的数字部分反转后的结果。</p>
<p>如果反转后整数超过 32 位的有符号整数的范围 <code>[−231,  231 − 1]</code> ，就返回 0。</p>
<p><strong>假设环境不允许存储 64 位整数（有符号或无符号）。</strong> </p>
<p><strong>示例 1：</strong></p>
<p><strong>输入：</strong> x = 123
<strong>输出：</strong> 321</p>
<p><strong>示例 2：</strong></p>
<p><strong>输入：</strong> x = -123
<strong>输出：</strong> -321</p>
<p><strong>示例 3：</strong></p>
<p><strong>输入：</strong> x = 120
<strong>输出：</strong> 21</p>
<p><strong>示例 4：</strong></p>
<p><strong>输入：</strong> x = 0
<strong>输出：</strong> 0</p>
<p> </p>
<p><strong>提示：</strong></p>
<ul>
<li><code>-231 <= x <= 231 - 1</code></li>
</ul>
<h2 data-id="heading-1">解题思路</h2>
<h3 data-id="heading-2">思路1</h3>
<p>反转数字，我们首先想到的就是取余，但是可能会遇到反转后，数字超过了最大限制或最小限制。</p>
<p>所以我们可以一边反转，一边检测，这样做的话，若遇到问题，将会立即停止。</p>

<h3 data-id="heading-4">思路2</h3>

<p>因为分割成字符数组的原因，最后一步处理的String在x<0时会失去结尾的负号，所以x<0时结果加负号。</p>
