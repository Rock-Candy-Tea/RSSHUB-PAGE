<p>在处理复杂的 CSS 动画时，我们总是会倾向于使用大量声明创建扩展的 <code>@keyframes</code>。虽然我想谈谈一些技巧可能有助于使构建复杂动画变得更容易，同时还能保持使用原生 CSS 完成这一切：</p>
<ol>
<li>多种动画；</li>
<li>计时功能；</li>
</ol>
<p>第一个更广泛使用和熟悉，但第二个不太常见。这可能是有充分理由的 —— 用逗号链接动画比了解我们可用的各种计时函数及其作用相对容易。有一个特别简洁的计时功能，让我们可以完全控制创建自定义计时功能。那将是 <code>cubic-bezier()</code>，在这篇文章中，我将向您展示它的强大功能以及如何使用它不会太复杂地创建精美的动画。</p>
<p>让我们从一个基本示例开始，展示我们如何沿有趣的方向移动这个红色小球，例如无穷大（∞）形状：</p>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fcodepen.io%2Ft_afif%2Fpen%2FeYvmOxR" target="_blank" rel="nofollow noopener noreferrer" title="https://codepen.io/t_afif/pen/eYvmOxR" ref="nofollow noopener noreferrer">Codepen t_aifi/eYvmOxR</a></p>
<p>如您所见，没有复杂的代码 —— 只有两个关键帧和一个**“奇怪的”** <code>cubic-bezier()</code> 函数。然而，我们却创建一个看起来非常复杂的无限形状动画。</p>
<p>这个动画说起来还真挺酷，对吧？让我们深入研究一下！</p>

<p>从这些例子中可以看出一些事情：</p>
<ul>
<li>每个关键帧仅使用一个包含增量的声明来定义。</li>
<li>元素的位置和动画是独立的。我们可以轻松地将元素放置在任何地方，而无需调整动画。</li>
<li>我们没有进行任何计算。没有大量的角度或像素值。我们只需要关键帧中的一个小值和 <code>cubic-bezier()</code> 函数中的一个大值。</li>
<li>只需调整持续时间值即可控制整个动画。</li>
</ul>

<h2 data-id="heading-7">就是这样！</h2>
<p>现在你有了一些神奇的 <code>cubic-bezier()</code> 曲线，也了解到了它们背后的数学。当然，好处是像这样的自定义计时功能让我们可以在没有我们通常需要的复杂关键帧的情况下制作精美的动画。</p>
<p>我知道不是每个人都有数学头脑，这没关系。有一些工具可以提供帮助，例如 Matthew Lein 的 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fmatthewlein.com%2Ftools%2Fceaser" target="_blank" rel="nofollow noopener noreferrer" title="https://matthewlein.com/tools/ceaser" ref="nofollow noopener noreferrer">Ceaser</a>，它可以让我们拖动曲线点来获得所需的内容。而且，如果我们还没有为它添加书签，<a href="https://link.juejin.cn/?target=https%3A%2F%2Fcubic-bezier.com%2F" target="_blank" rel="nofollow noopener noreferrer" title="https://cubic-bezier.com/" ref="nofollow noopener noreferrer">cubic-bezier.com</a> 是另一个选择。如果你想在 CSS 世界之外玩三次贝塞尔曲线，我推荐使用一下 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fwww.desmos.com%2Fcalculator%2Febdtbxgbq0%3Flang%3Dfr" target="_blank" rel="nofollow noopener noreferrer" title="https://www.desmos.com/calculator/ebdtbxgbq0?lang=fr" ref="nofollow noopener noreferrer">desmos</a>，你可以在那里看到一些数学公式。</p>
<p>不管你如何获得你的 <code>cubic-bezier()</code> 函数，希望现在你已经了解了它们的力量，以及了解了它们使如何在这个过程中帮助编写更好的代码。</p>
