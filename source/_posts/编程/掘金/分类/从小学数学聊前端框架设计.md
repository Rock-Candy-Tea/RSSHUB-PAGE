<h2 data-id="heading-0">自变量、因变量与响应式更新</h2>
<p>这个小学知识就是：<code>自变量</code>与<code>因变量</code>。</p>
<p>对于如下等式：</p>
<pre><code class="hljs language-js copyable" lang="js">2x + <span class="hljs-number">1</span> = y
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>x</code>是<code>自变量</code>，<code>y</code>的值受<code>x</code>影响，是<code>因变量</code>。</p>
<p>在很多框架与状态管理库中，同样存在自变量与因变量。</p>
<p><code>Vue3</code>中的自变量：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> x = value(<span class="hljs-number">1</span>);
<span class="hljs-comment">// 取值</span>
<span class="hljs-built_in">console</span>.log(x.value);
<span class="hljs-comment">// 赋值</span>
x.value = <span class="hljs-number">2</span>;
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>MobX</code>的自变量：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> x = observable(&#123;<span class="hljs-attr">data</span>: <span class="hljs-number">1</span>&#125;);
<span class="hljs-comment">// 取值</span>
<span class="hljs-built_in">console</span>.log(x.data);
<span class="hljs-comment">// 赋值</span>
x.data = <span class="hljs-number">2</span>;
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>React</code>的自变量：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> [x, setX] = useState(<span class="hljs-number">1</span>);
<span class="hljs-comment">// 取值</span>
<span class="hljs-built_in">console</span>.log(x);
<span class="hljs-comment">// 赋值</span>
setX(<span class="hljs-number">2</span>);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这些框架（或库）的自变量由<code>getter</code>（取值）与<code>setter</code>（赋值）两部分构成。</p>
<p>有了<code>自变量</code>，当然也有<code>因变量</code>。我们可以按有无<strong>副作用</strong>区分。</p>
<p><code>Vue</code>的因变量：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">// 无副作用</span>
<span class="hljs-comment">// 赋值</span>
<span class="hljs-keyword">const</span> y = computed(<span class="hljs-function">() =></span> x.value * <span class="hljs-number">2</span> + <span class="hljs-number">1</span>);
<span class="hljs-comment">// 取值</span>
<span class="hljs-built_in">console</span>.log(y.value);

<span class="hljs-comment">// 有副作用</span>
watch(<span class="hljs-function">() =></span> <span class="hljs-built_in">document</span>.title = x.value);
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>MobX</code>的因变量：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">// 无副作用</span>
<span class="hljs-keyword">const</span> y = computed(<span class="hljs-function">() =></span> x.data * <span class="hljs-number">2</span> + <span class="hljs-number">1</span>);
<span class="hljs-built_in">console</span>.log(y.get());

<span class="hljs-comment">// 有副作用</span>
autorun(<span class="hljs-function">() =></span> <span class="hljs-built_in">document</span>.title = x.data);
<span class="copy-code-btn">复制代码</span></code></pre>
<p><code>React</code>的因变量：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-comment">// 无副作用</span>
<span class="hljs-keyword">const</span> y = useMemo(<span class="hljs-function">() =></span> x * <span class="hljs-number">2</span> + <span class="hljs-number">1</span>, [x]);
<span class="hljs-built_in">console</span>.log(y);

<span class="hljs-comment">// 有副作用</span>
useEffect(<span class="hljs-function">() =></span> <span class="hljs-built_in">document</span>.title = x, [x]);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>有了<code>自变量</code>与<code>因变量</code>，再配合<code>描述视图的方式</code>，就能描述组件<code>UI</code>。</p>
<p>比如在<code>React</code>中，通过<code>JSX</code>描述视图：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> [x, setX] = useState(<span class="hljs-number">21</span>);
<span class="hljs-keyword">const</span> y = useMemo(<span class="hljs-function">() =></span> x * <span class="hljs-number">2</span> + <span class="hljs-number">1</span>, [x]);

<span class="hljs-keyword">return</span> <span class="xml"><span class="hljs-tag"><<span class="hljs-name">p</span>></span>我的战绩是 0/&#123;x&#125;/&#123;y&#125;<span class="hljs-tag"></<span class="hljs-name">p</span>></span></span>;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>再加上各种使用户可以操纵<code>自变量</code>的事件，如给<code>p</code>增加<code>onClick</code>：</p>
<pre><code class="hljs language-js copyable" lang="js"><p onClick=&#123;<span class="hljs-function">() =></span> setX(x + <span class="hljs-number">1</span>)&#125;>我的战绩是 <span class="hljs-number">0</span>/&#123;x&#125;/&#123;y&#125;</p>;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>最后再加上少量辅助的钩子函数，如：<strong>组件发生错误时的钩子函数</strong>。</p>
<p>就组成一个功能完备的组件。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0b70bffa4092441b86984b4b90bb453b~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>这就是所有<strong>细粒度更新</strong>框架的底层共通之处：</p>
<blockquote>
<p>通过事件驱动自变量改变，进而最终驱动视图（或副作用）变化</p>
</blockquote>
<h2 data-id="heading-1">面向对象之痛</h2>
<p>在我们初学编程时，都学过一个概念 —— <strong>面向对象</strong>（下文简称<code>OO</code>），也很容易接受一个设定 —— <code>OO</code>可以提高可读性且易维护。</p>
<p>原因是：<code>OO</code>是对现实世界的模拟。比如：</p>
<blockquote>
<p><strong>人</strong>可以继承<strong>哺乳动物</strong>的属性，这就是个<code>OO</code>模型</p>
</blockquote>
<p>然而实际操作起来却事与愿违。</p>
<p>回想你学习<code>React</code>的<code>Class</code>组件时，在<code>OO</code>简单的表象背后，是复杂的<strong>生命周期</strong>概念，随便问你几个问题：</p>
<ul>
<li>
<p><code>shouldComponentUpdate</code>的原理是？</p>
</li>
<li>
<p><code>componentWillReceiveProps</code>什么时候触发？</p>
</li>
<li>
<p><code>getDerivedStateFromProps</code>中<code>derivedState</code>是什么意思？</p>
</li>
</ul>
<p>好在<code>React</code>团队也意识到这个问题，并着手做出改变。</p>
<p>改变的结果，就是<code>Hooks</code>。</p>
<p>使用<code>Hooks</code>的函数组件与<code>Class</code>组件最大的区别是：</p>
<blockquote>
<p>从拥有<strong>生命周期的实例</strong>向<strong>自变量、因变量与视图的映射关系</strong>转变</p>
</blockquote>
<p>如果接受了这个设定，想想现在主流的学习<code>Hooks</code>的方式（甚至<code>React</code>官网也是如此），居然是：</p>
<blockquote>
<p>用生命周期函数来类比<code>Hooks</code>执行时机</p>
</blockquote>
<p>是不是还蛮搞笑的。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7ce94d1bb11b42c6a12e45b3380407b5~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<h2 data-id="heading-2">戴着脚镣跳舞的React</h2>
<p>理想是美好的，可是<code>React</code>底层并不是一个<strong>细粒度</strong>框架。</p>
<p>这就造成在实现<code>自变量</code>、<code>因变量</code>时会有诸多限制，比如：</p>
<blockquote>
<p><code>Hooks</code>调用顺序不能变（不能写在条件语句中）</p>
</blockquote>
<p>再比如，不知道你发现一个细节没：</p>
<p><code>React</code>实现<code>因变量</code>时需要第二个参数<strong>显式</strong>指出自己的<code>自变量</code>是谁。比如：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> y = useMemo(<span class="hljs-function">() =></span> x * <span class="hljs-number">2</span> + <span class="hljs-number">1</span>, [x]);
useEffect(<span class="hljs-function">() =></span> <span class="hljs-built_in">document</span>.title = x, [x]);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>反观其他框架（或库）就不需要。比如<code>Vue</code>：</p>
<pre><code class="hljs language-js copyable" lang="js"><span class="hljs-keyword">const</span> y = computed(<span class="hljs-function">() =></span> x.value * <span class="hljs-number">2</span> + <span class="hljs-number">1</span>);
watch(<span class="hljs-function">() =></span> <span class="hljs-built_in">document</span>.title = x.value);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>为什么会有这些限制呢？我用两个比喻来解释。</p>
<p>刚才聊到，在<code>细粒度</code>框架中，交互流程可以概括为：</p>
<pre><code class="hljs language-js copyable" lang="js">用户触发事件 -> 自变量改变 -> 因变量改变 -> 映射到视图变化
<span class="copy-code-btn">复制代码</span></code></pre>
<p>就像一个画家在画画，画的每一笔对应一个<code>自变量</code>变化，再最终对应画面变化。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/afe4b4f84f7b4f1ead2aa347289fbbe3~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>而<code>React</code>的更新机制大体概括为：</p>
<pre><code class="hljs language-js copyable" lang="js">用户触发事件 -> 触发更新 -> 虚拟DOM全量对比 -> 将对比结果映射为视图操作
<span class="copy-code-btn">复制代码</span></code></pre>
<p>就像一个人拿相机拍一张照片，再拿这张照片和上次拍的照片找不同，最后把不同的地方更新了。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/5865c8b495ae43d4bbc546c6923a3b68~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>当调用<code>this.setState</code>（或<code>useState</code>的<code>setter</code>），并不是画下一笔，而是按下快门。</p>
<p>怎么能从一张新照片中发现<code>自变量</code>呢？所以<code>React</code>只能拿新老照片对比。</p>
<h2 data-id="heading-3">净整些奇怪的</h2>
<p>社区早有人意识到这个问题，于是<code>Mobx</code>诞生了。他带来纯粹的<strong>细粒度</strong>更新能力。</p>
<p>然而，这个能力是建立在<code>React</code>更新机制之上，就像：</p>
<p>一个画家，拿画笔在画板上一戳，戳到什么呢？戳到相机快门了。</p>
<p>咔嚓拍了张照片，画家再拿照片与老照片对比，将对比结果再画到画布上。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c1263b0e050040d995dbbad2a486016d~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>所以有人吐槽：用<code>React</code>+<code>Mobx</code>为啥不直接用<code>Vue</code>？</p>
<p>然而，<code>Vue</code>本身也依赖<code>虚拟DOM</code>，粒度并不是最细的。</p>
<p>更准确的说法应该是：用<code>React</code>+<code>Mobx</code>为啥不直接用<code>SolidJS</code>？</p>
<p>呐，过几天我们来聊聊纯粹的细粒度更新框架（<code>SolidJS</code>）的实现原理吧。</p></div>
