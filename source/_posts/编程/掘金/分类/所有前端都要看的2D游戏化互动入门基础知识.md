<h1 data-id="heading-0">背景</h1>
<p>现在越来越多的公司和 App 开始使用游戏化的方式去做产品了，所谓游戏化，是指在非游戏环境中将游戏的思维和游戏的机制进行整合运用，以引导用户互动和使用的方法。</p>
<p>支付宝里面的蚂蚁庄园、蚂蚁森林，通过游戏和公益的结合实现用户的留存和活跃。淘宝支付宝的芭芭农场、京东的东东果园、拼多多的多多果园、美团的小美果园...无一不是通过游戏化的方式去提升用户留存的方案。</p>
<p>本篇文章，我们会列出一些游戏化互动类的游戏，然后对一个案例进行拆分，带大家学习一些2D互动最基础的知识，让大家能够快速上手写互动游戏。</p>
<h1 data-id="heading-1">能做什么</h1>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/877ec6a9d92b4bdbb6c5c0cbd7e51e24~tplv-k3u1fbpfcp-zoom-1.image" alt="1.png" title="1.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>我们来看几个2D互动项目，目前，大多数的互动都是以游戏的形式展现，通过游戏的玩法和精致的效果，让用户有更好的互动体验，我们通过养成，采集，塔防，抓娃娃等类似游戏的形式，结合业务属性，达到更好的业务效果。</p>
<h1 data-id="heading-2">基础学习</h1>
<h3 data-id="heading-3">2D互动常用能力</h3>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b47764a86bde4166af454c5143e55bb6~tplv-k3u1fbpfcp-zoom-1.image" alt="2.png" title="2.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>首先，我们看一下2D互动游戏所用到的常用能力，第一部分是前端知识，主要包括渲染所需的绘制工具，游戏循环，资源加载的能力。</p>
<p>然后是基础的绘制和动画能力，也就是前面提到的一些游戏基础元素。在游戏开发中，会涉及到很多数学相关的知识，比如让游戏中的物体模拟真实的物理效果，或者像一些游戏中的人机对战中的机器人，是使用游戏 AI 来实现的，在本文中不会过多讲解数学类知识。</p>
<h3 data-id="heading-4">互动游戏是如何运行起来的</h3>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/39fe00c2216d49369995e530559c7c30~tplv-k3u1fbpfcp-zoom-1.image" alt="3.png" title="3.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>互动游戏是如何运行起来的呢？</p>
<p>首先，我们知道，目前很多前端项目都是通过数据带动视图的，游戏也是这样的，比如说，我们在游戏里面有一个飞机，那么，我们需要定义飞机的尺寸，以及他在游戏中的位置，和他对应的飞机图片，这些属于游戏数据，我们将数据提交给渲染引擎，渲染引擎根据这些数据内容将对应的内容渲染到画布上。</p>
<p>游戏是动态运行的，为了实现一些动画/真实物理效果效果，我们通过动画，AI，物理引擎等工具控制数据的变化，然后通过循环来持续修改数据，并且渲染到画布实现游戏的运行。</p>
<h3 data-id="heading-5">循环</h3>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f0cfec19f10c47bcb24ecac413e8cf03~tplv-k3u1fbpfcp-zoom-1.image" alt="4.png" title="4.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>我们知道，通过循环来实现游戏的运行效果，接下来我们来看一下在前端浏览器环境下，游戏循环是如何实现的。</p>
<p>浏览器提供了 requestAnimationFrame 方法，要求浏览器在下一次绘制之前，调用制定的回调函数，这个方法一般是用于更新动画的。</p>
<p>浏览器在的每一次重绘我们叫做1帧，浏览器默认的绘制频率是60帧，也就是说，正常情况下，浏览器一秒会刷新60次。</p>
<p>通过下面的方法，我们可以保证每一帧渲染之前，我们可以进行数据的计算以及调用渲染方法：</p>
<pre><code class="copyable">const loop = () => &#123;
  requestAnimationFrame(loop)
  // 计算数据
  // 绘制图形
&#125;
requestAnimationFrame(loop)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>因为requestAnimationFrame方法只会在下一次绘制前被调用，所以，我们需要在每次调用方法的时候去调用一次这个方法保证游戏继续运行，所以在loop方法里面会重复调用这个方法。</p>
<p>一般情况下，我们会把这个方法放在函数的最前面，因为，如果在计算数据和绘制图形的过程中报错了，会导致程序无法执行到这个方法，游戏也就停掉了。</p>
<h3 data-id="heading-6">画布</h3>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/aa8f1e590282484aa801b28f32a80a2e~tplv-k3u1fbpfcp-zoom-1.image" alt="5.png" title="5.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>在 html 中，我们一般使用 canvas 标签来绘制图像，它本身没有绘制能力，使用 getContext 获取绘制上下文，调用上下文上面的方法进行绘制。</p>
<p>常用的绘制上下文有 Canvas API 和 WebGL，一般 CanvasAPI 来绘制2D图像，WebGL 可绘制 2D 和 3D 图像，他的性能更高。</p>
<p>canvas 提供了一些比较基础的 API，但是在互动游戏中的元素是比较复杂的，所以一般都会有渲染引擎和游戏引擎来承接这些元素。</p>
<p>本文不会详细讲解 Canvas，可以到 MDN 等平台学习。</p>
<h3 data-id="heading-7">基础渲染</h3>
<p>接下来我会介绍一下，在2D游戏化互动游戏中，我们经常用到几种渲染方案。</p>
<ul>
<li>图片</li>
<li>文本</li>
<li>图形</li>
<li>精灵</li>
<li>九宫格</li>
<li>遮罩</li>
</ul>
<p>图片和文本就不用说了，是视图开发中最常用到的。我们从图形开始说</p>
<h5 data-id="heading-8">图形</h5>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/3a866436a9004d8e8316537912e81b8a~tplv-k3u1fbpfcp-zoom-1.image" alt="6.png" title="6.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>一般在开发中会经常使用一些简单的图形，图片不仅会用在直接展示内容，也会用在对渲染内容的遮罩，例如一张图片只显示图形内的内容，也会用在按钮区域判断、物理引擎碰撞的形状等等地方。</p>
<h5 data-id="heading-9">精灵</h5>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/2d0844e079b442c888ec0df5808cba63~tplv-k3u1fbpfcp-zoom-1.image" alt="7.png" title="7.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>精灵图也是我们在 CSS 中接触的精灵图，就是将多张图片合成在一张大图中，在使用时渲染其中某个位置，通过精灵图的方式，我们可以提高网络加载效率以及渲染效率。一般精灵资源是由两个文件组成，一个是图片文件，另外一个是位置信息文件。一般使用引擎进行渲染时，只需要关心对应小图的名称。</p>

<h1 data-id="heading-16">项目实战</h1>
<p>了解到以上的内容，我们就可以开发互动项目了，工欲善其事，必先利其器，这里我们推荐由淘系技术部开源的 Eva.js，它是专门给前端开发者提供的开发游戏化互动项目所设计的。目前淘宝、天猫、支付宝、优酷、阿里妈妈、AliExpress、Lazada、考拉等很多产品都在使用，2020年双11养猫猫项目也是使用 Eva.js 实现的。</p>

<p>Eva.js 的游戏是由游戏对象和组件构成，游戏对象代表游戏中的一个物体，组件代表物体的能力，在这个例子中，只有一个物体，他的能力有三个：</p>

<h5 data-id="heading-17">Step1 添加资源&创建游戏</h5>
<pre><code class="copyable">import &#123; resource, Game &#125; from '@eva/eva.js'

resource.addResource([
  &#123;
    name: 'imageName',
    type: RESOURCE_TYPE.IMAGE,
    src: &#123;
      image: &#123;
        type: 'png',
        url:
          '//gw.alicdn.com/bao/uploaded/TB1lVHuaET1gK0jSZFhXXaAtVXa-200-200.png',
      &#125;,
    &#125;,
    preload: true,
  &#125;,
]);

const game = new Game(&#123;
  systems: [
    new RendererSystem(&#123;

&#125;);

<p>addResource 传入了一个资源的里面，这里不一定只有图片资源，还可以有帧动画、骨骼动画等等资源，这里以图片资源举例子。更多Demo可以进入 Eva.js 官网 中查看。</p>
<p>在添加资源之后，我们也创建了一个游戏实例，这是运行游戏的主要运行时， 因为 Eva.js 只有一个最核心的游戏运行时，所以我们所有的功能都是要自己安装的哦～所以我们要安装这个游戏所需要的系统，图片、事件、动画。</p>
<ul>
<li>RendererSystem 是用来将游戏渲染出来的系统，所有渲染的能力都依赖这个系统，里面设置了宽高以及所要渲染的canvas对象。</li>
<li>ImgSystem 是用来画图片的系统</li>
<li>EventSystem 是用来触发点击事件的系统</li>
<li>TransitionSystem 是用来做位移动画的系统</li>
</ul>
<h5 data-id="heading-18">Step2 创建对象，并设置定位</h5>
<pre><code class="copyable">import &#123; GameObject &#125; from '@eva/eva.js'

const heart = new GameObject('heart', &#123;
  size: &#123; width: 200, height: 200 &#125;,
  position: &#123;
    x: 0,
    y: 0,
  &#125;,
  origin: &#123; x: 0, y: 0 &#125;,
  anchor: &#123;
    x: 0,

  &#125;,
&#125;);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>GameObject 的第一个参数为对象的名称，第二个参数为对象的位置信息，其中 size 设置对象大小， position 设置位置，其他的可以后续参考文档学习哦～</p>
<h5 data-id="heading-19">Step3 添加所需要的组件</h5>
<p>刚刚我们在 new Game 的时候添加了实现视频功能所需要的系统，这些系统是为了读取组件上面的数值然后实现功能的，所以，我们需要给对象添加组件以后，才能够让对象实现对应的功能。</p>
<p>我们目前所需要的功能是图片渲染、点击事件、位移动画，所以我们要添加三个组件</p>
<p><strong>图片渲染</strong></p>
<pre><code class="copyable">import &#123; Img &#125; from '@eva/plugin-renderer-img'

heart.addComponent(
  new Img(&#123;
    resource: 'imageName',
  &#125;),
);
<span class="copy-code-btn">复制代码</span></code></pre>
<p>调用 heart 的 addComponent 方法既可添加组件，这里我们添加 Img 组件， Img 组件有个 resource 参数，该参数是图片资源的名称，其实对应了 Step1 中添加的图片资源的名称。当然雪碧图、骨骼动画也是同样的原理，需要在 resource 中添加资源，在添加组件的时候使用。</p>
<p><strong>点击事件</strong></p>
<pre><code class="copyable">import &#123; Event &#125; from '@eva/plugin-renderer-event'

const evt = heart.addComponent(new Event())
evt.on('tap', () => &#123;
  alert(1)
&#125;)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>给游戏对象添加一个 Event 组件，并通过 on 方法绑定 tap 事件， on 的第二个参数为 tap 事件所触发的函数，当然，Event 组件还有其他事件，我们可以通过 Eva.js 文档查看。</p>
<p><strong>位移动画</strong></p>
<pre><code class="copyable">import &#123; Transition &#125; from '@eva/plugin-transition'

const transition = heart.addComponent(new Transition())
transition.group = &#123;
  idle: [
    &#123;
      name: 'position.x',
      component: heart.transform,
      values: [
        &#123;
          time: 0,
          value: 0,
          tween: 'ease',
        &#125;,
        &#123;
          time: 1000,
          value: 400,
          tween: 'ease',
        &#125;,
        &#123;
          time: 2000,
          value: 0
        &#125;
      ]
    &#125;
  ]
&#125;
transition.play('idle', Infinity)
<span class="copy-code-btn">复制代码</span></code></pre>
<p>上面的代码中，我们创建了一个动画组，名字叫做 idle 当前动画组里面，我们对 heart.transform 组件的 position.x 属性进行数值变化，0->1000ms，数值从0->400，1000ms->2000ms，数值从400->0，然后使用 Transition 组件的 play 方法，让动画执行 Infinity 次。</p>
