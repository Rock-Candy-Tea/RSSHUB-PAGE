<h2 data-id="heading-0">前言</h2>
<p>在面试高级前端时，往往会遇到一些关于设计模式的问题，每次都回答不太理想。恰逢8月更文挑战的活动，准备用一个月时间好好理一下关于设计模式方面的知识点，给自己增加点面试的底气。</p>
<p>在 <a href="https://juejin.cn/post/6998153008915677198" target="_blank" title="https://juejin.cn/post/6998153008915677198">上一篇</a>简单实现了一个职责链模式，但是其中职责的传递非常僵硬，且违背了开放-封闭原则，故本文来介绍一种一种灵活可拆分的职责链节点的实现。</p>

<p>先将其解耦，怎么解耦呢，要做个约定，如果某个节点不能处理请求，则返回一个特定的字符串 <code>toNextNode</code>来表示该请求需要继续往后面的节点传递。</p>
<pre><code class="copyable">// 预缴1000定金

<h2 data-id="heading-2">组装职责链</h2>
<p>经过解耦，上面的每个函数都变成一个个普通函数，这里要创建一个包装函数来把普通函数包装成职责链节点。</p>
<p>用一个类来作为包装函数，类中有方法，用来设置下个请求节点，还得有一个方法用来执行当前节点，且执行完后，若请求无法被处理，继续请求下个节点。</p>
<pre><code class="copyable">class Chain&#123;
  constructor(fn)&#123;
    this.fn = fn;
    this.nextNode = null;
  &#125;
  setNextNode(fn)&#123;
    return this.nextNode = fn;
  &#125;
  passRequest()&#123;
    const res = this.fn.apply(this,arguments);
    if(res === 'toNextNode')&#123;
      return this.nextNode && this.nextNode.passRequest.apply(this.nextNode, arguments);
    &#125;
    return res;
  &#125;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>其中<code>setNextNode</code>方法设置下个请求节点，<code>passRequest</code>方法用来执行当前节点，且执行完后，若请求无法被处理，继续请求下个节点。</p>
<p>现在把之前的三个函数包装成职责链节点：</p>
<pre><code class="copyable">const chainOrder1000 = new Chain(order1000);
const chainOrder500 = new Chain(order500);
const chainOrderNormal = new Chain( orderNormal); 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>然后组装职责链节点成一条职责链：</p>
<pre><code class="copyable">chainOrder1000.setNextNode( chainOrder500 );
chainOrder500.setNextNode( chainOrderNormal ); 
<span class="copy-code-btn">复制代码</span></code></pre>
<p>最后开始执行职责链：</p>
<pre><code class="copyable">chainOrder1000.passRequest( 1, true, 500 ); // 输出：1000 元定金预购，得到 500 优惠券
chainOrder1000.passRequest( 2, true, 500 ); // 输出：500 元定金预购，得到 200 优惠券
chainOrder1000.passRequest( 3, true, 500 ); // 输出：普通购买，无优惠券
chainOrder1000.passRequest( 1, false, 0 ); // 输出：手机库存不足

<h2 data-id="heading-3">拆卸职责链重新组装</h2>
<p>比如现在要新增一种的营销策略换成预缴300元定金，得50优惠券。可以这么处理。</p>
<pre><code class="copyable">// 预缴300定金
const order300 = (orderType, pay, stock) =>&#123;
  if (orderType === 4 && pay === true) &#123;
    console.log('300 元定金预购, 得到 50 优惠券');
  &#125; else &#123;
    return 'toNextNode'; // 我不知道下一个节点是谁，反正把请求往后面传递
  &#125;
&#125;;
chainOrder300= new Chain(order300);
chainOrder1000.setNextSuccessor(chainOrder500);
chainOrder500.setNextSuccessor(chainOrder300);
chainOrder300.setNextSuccessor(chainOrderNormal); 
chainOrder1000.passRequest( 4, true, 500 );// 输出：300 元定金预购，得到 50 优惠券
<span class="copy-code-btn">复制代码</span></code></pre>
<p>这样实现职责链，其职责链节点可以灵活拆卸。在变更职责链中节点顺序或新增职责链节点时，不会修改职责链节点的内容，遵循开放-封闭原则。</p>
<h2 data-id="heading-4">异步的职责链节点如何实现</h2>
<p>上面介绍的职责链节点中都是同步返回一个特定的字符串 <code>toNextNode</code>来表示该请求需要继续往后面的节点传递。那如果职责链节点的请求是异步的呢，等异步请求返回的结果才能决定是否继续请求下一个职责链节点。那要如何实现？留个思考题，将在下篇文章中介绍。</p></div>
