<p>现代软件架构的复杂性需要协同开发完成，如何高效地协同呢？无规矩不成方圆，无规范难以协同，比如，制订交通法规表面上是要限制行车权，实际上是保障公众的人身安全，试想如果没有限速，没有红绿灯，谁还敢上路行驶。对软件来说，适当的规范和标准绝不是消灭代码内容的创造性、优雅性，而是限制过度个性化，以一种普遍认可的统一方式一起做事，提升协作效率，降低沟通成本。代码的字里行间流淌的是软件系统的血液，质量的提升是尽可能少踩坑，杜绝踩重复的坑，切实提升系统稳定性，码出质量。</p>
</blockquote>
<h2 data-id="heading-0">JS 项目规范</h2>
<h3 data-id="heading-1">命名规范</h3>
<h4 data-id="heading-2">项目命名</h4>
<p>全部采用小写方式， 以中划线分隔。</p>
<pre><code class="copyable">正例：mall-management-system
<span class="copy-code-btn">复制代码</span></code></pre>
<pre><code class="copyable">反例：mall_management-system / mallManagementSystem

<h4 data-id="heading-3">目录命名</h4>
<p>1、全部采用小写方式， 以中划线分隔，有复数结构时，要采用复数命名法， 缩写不用复数</p>
<pre><code class="copyable">正例： scripts / styles / components / images / utils / layouts / demo-styles / demo-scripts / img / doc
<span class="copy-code-btn">复制代码</span></code></pre>
<pre><code class="copyable">反例： script / style / demo_scripts / demoStyles / imgs / docs
<span class="copy-code-btn">复制代码</span></code></pre>
<p>2、<strong><em>VUE 的项目中的 components 中的组件目录，使用 kebab-case 命名</em></strong></p>
<pre><code class="copyable">正例： head-search / page-loading / authorized / notice-icon
<span class="copy-code-btn">复制代码</span></code></pre>
<pre><code class="copyable">反例： HeadSearch / PageLoading

<h4 data-id="heading-4">JS、CSS、SCSS、HTML、PNG 文件命名</h4>

<pre><code class="copyable">正例： render-dom.js / signup.css / index.html / company-logo.png

<h4 data-id="heading-5">命名严谨性</h4>
<p>代码中的命名严禁使用拼音与英文混合的方式，更不允许直接使用中文的方式。 说明：正确的英文拼写和语法可以让阅读者易于理解，避免歧义。注意，即使纯拼音命名方式也要避免采用</p>
<pre><code class="copyable">正例：henan / luoyang / rmb 等国际通用的名称，可视同英文。

<span class="copy-code-btn">复制代码</span></code></pre>
<pre><code class="copyable">反例：DaZhePromotion [打折] / getPingfenByName() [评分] / int 某变量 = 3
<span class="copy-code-btn">复制代码</span></code></pre>
<p>2、杜绝完全不规范的缩写，避免望文不知义：</p>
<pre><code class="copyable">反例：AbstractClass“缩写”命名成 AbsClass；condition“缩写”命名成 condi，此类随意缩写严重降低了代码的可阅读性。
<span class="copy-code-btn">复制代码</span></code></pre>

<p>使用双引号(" ") 而不是单引号('') 。</p>
<p><code>正例：""</code></p>
<p><code>反例：''</code></p>
<h3 data-id="heading-12">CSS 规范</h3>

<h4 data-id="heading-14">选择器</h4>
<p>1、css 选择器中避免使用标签名，从结构、表现、行为分离的原则来看，应该尽量避免 css 中出现 HTML 标签，并且在 css 选择器中出现标签名会存在潜在的问题。</p>
<p>2、很多前端开发人员写选择器链的时候不使用 直接子选择器（注：直接子选择器和后代选择器的区别）。有时，这可能会导致疼痛的设计问题并且有时候可能会很耗性能。然而，在任何情况下，这是一个非常不好的做法。如果你不写很通用的，需要匹配到 DOM 末端的选择器， 你应该总是考虑直接子选择器。</p>
<pre><code class="copyable">不推荐：

  font-size: 2rem;
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>

<span class="copy-code-btn">复制代码</span></code></pre>

<h3 data-id="heading-19">LESS 规范</h3>

<pre><code class="copyable">反例： _name / name_ / name$ 2) 方法名、参数名、成员变量、局部变量都统一使用 lowerCamelCase 风格，必须遵从驼峰形式。

<h4 data-id="heading-24">代码格式</h4>

<h4 data-id="heading-27">使用 ES6,ES7</h4>
<p>必须优先使用 ES6,ES7 中新增的语法糖和函数。这将简化你的程序，并让你的代码更加灵活和可复用。</p>
<pre><code class="copyable">必须强制使用 ES6, ES7 的新语法，比如箭头函数、await/async ， 解构， let ， for…of 等等
<span class="copy-code-btn">复制代码</span></code></pre>
<h4 data-id="heading-28">括号</h4>
<p>下列关键字后必须有大括号（即使代码块的内容只有一行）：if, else, for, while, do, switch, try, catch, finally, with。</p>
<pre><code class="copyable">正例：

if (condition) &#123;
  doSomething();
&#125;
<span class="copy-code-btn">复制代码</span></code></pre>
