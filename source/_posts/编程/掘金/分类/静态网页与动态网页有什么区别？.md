<h2 data-id="heading-0">什么是网页？</h2>
<p>首先，我们需要了解什么是网页和网站，以及它们有何不同。为此，让我们考虑一个百科全书。</p>
<p>百科全书（例如维基百科）由许多页面组成。每个页面都有自己的内容：标题、段落、图像、图表、要点、列表，当然还有您要查找的术语的定义。</p>
<p>在百科全书中，经常会在页面上找到引用（或链接）到另一个页面以获取其他信息的单词。</p>
<p>这与网页和网站非常相似。</p>
<p>网页是单个文档，可以包含文本、图像、超文本或任何其他元素。我们使用超文本标记语言（通常称为 HTML）等标记语言创建网页。</p>
<p>超文本是包含超链接的任何 Web 文档。超链接是网页中的任何元素，当您单击它时，链接到另一个网页。</p>
<p>这些相互连接的网页形成了一个有组织的网页网络，我们称之为网站。</p>
<p>互联网上可访问的每个网页都必须有自己的 URL。这是网页的典型 URL：</p>
<blockquote>
<p><a href="https://juejin.cn/user/2040341402229751" target="_blank" title="https://juejin.cn/user/2040341402229751">juejin.cn/user/204034…</a></p>
</blockquote>
<h3 data-id="heading-1">现在，我为您做一个简单的测试：</h3>
<p>打开您的网络浏览器。导航到地址栏并输入（或选择）您知道的任何随机但有效的 URL。点击进入并等待。网页将呈现在您的浏览器窗口中。拍摄网页当前状态的快照并关闭它。</p>
<p>等待一段时间，然后再次访问该 URL。然后回答以下问题：</p>
<ul>
<li>如果将页面的当前状态与其先前状态进行比较，其内容是否有任何差异？</li>
<li>例如，页面的 URL 是以文档扩展名/.html结尾还是以端点结尾？/profile例如）</li>
<li>假设您更改了浏览器设置（例如清除 cookie），点击相同的 URL 会返回不同的页面吗？</li>
<li>在浏览器上呈现页面之前，系统会提示您提交表单吗？</li>
</ul>
<p>您如何回答这些问题（以及我将在下面提出的更多问题）将决定页面是静态页面还是动态页面。</p>
<h2 data-id="heading-2">什么是静态页面？</h2>
<p>静态页面具有以下特点：</p>
<ul>
<li>该页面甚至在用户请求之前就已经存在。静态页面必须在用户提出请求时已经物理存在并已被水合（即包含内容）。如果它不存在，则它不是静态的。</li>
<li>每次用户请求时，页面通常都会维护相同的内容。如果点击相同的 URL 返回不同的内容，那么该页面根本就不是静态的。这并不是说不能修改静态页面。但更改静态页面的唯一方法是创建者手动编辑内容（如 HTML 文档）</li>
</ul>
<p>下面是一个静态页面的例子：</p>
<blockquote>
<p><a href="https://link.juejin.cn/?target=http%3A%2F%2Fwww.example.com%2Fabout.html" target="_blank" rel="nofollow noopener noreferrer" title="http://www.example.com/about.html" ref="nofollow noopener noreferrer">www.example.com/about.html</a></p>
</blockquote>
<h2 data-id="heading-3">什么是动态页面？</h2>
<p>动态页面具有以下特点：</p>
<ul>
<li>当用户发出请求时，该页面实际上并不存在于服务器上</li>
<li>当用户发出请求时，脚本或程序会运行并最终生成网页。它通过与数据库交互来检索它打包并作为页面发送的数据来实现这一点。</li>
<li>对于每个请求，创建的每个新页面都可能与上一个不同。</li>
</ul>
<p>这是因为创建的页面依赖于用户的信息和程序。创建者不必像静态网页那样手动编辑内容。</p>
<p>因此，例如，如果不同的用户请求相同的页面，则返回不同的内容。</p>
<p>或者当用户更改设置时，会返回一个新页面。</p>
<p>或者当时间改变时，返回不同的内容。</p>
<p>下面是一个动态页面的例子：</p>
<p><a href="https://link.juejin.cn/?target=http%3A%2F%2Fwww.example.com%2Fcourses" target="_blank" rel="nofollow noopener noreferrer" title="http://www.example.com/courses" ref="nofollow noopener noreferrer">www.example.com/courses</a></p>
<p>/course URL 不是文档扩展名，而是一个端点。</p>
<p>向该端点发出请求将触发一个程序，该程序将使用提供的用户数据（例如用户名和密码）和一些外部变量（例如时间和日期）与数据库交互并最终创建并返回新形成的网页.</p>
<p>该页面从未作为服务器上的文件存在，而是由在用户请求时运行的脚本创建的。</p>
<h2 data-id="heading-4">总结</h2>
<p>网页是包含文本、图像、超文本和其他元素的单个文档。</p>
<p>超文本是包含超链接的 Web 文档。超链接将一个网页与另一个网页链接起来。</p>
<p>相互链接的有组织的网页网络称为网站。</p>
<p>对于被视为静态的网站，每次调用相同的 URL 都会返回相同的网页。</p>
<p>另一方面，如果内容变化很大，那么该网页是动态的。动态页面也以端点结束，而不是文件路径。</p>
