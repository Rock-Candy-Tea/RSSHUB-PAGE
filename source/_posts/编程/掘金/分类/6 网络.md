h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h2 data-id="heading-0">OSI 七层模型（是理想化网络模型）</h2>
<ol start="7">
<li>应用层</li>
<li>表示层</li>
<li>会话层 实际中 5.6.7 归成应用层</li>
<li>传输层 TCP UDP 协议</li>
<li>网络层 路由器</li>
<li>数据链路层 交换机 网卡</li>
<li>物理层 物理设备 网线 光纤</li>
</ol>
<h3 data-id="heading-1">实际五层</h3>
<ul>
<li>实际应用中会将会话层、表示层、应用层合并在一起为应用层</li>
</ul>
<ol start="5">
<li>用户要传递的数据 装包 HTTP DNS</li>
<li>传输层 用户要传递很大的数据包 会对大的数据包进行拆分并标明<code>序号</code> <code>端口号</code>用于传输到哪 TCP、UDP <code>段</code></li>
<li>网络层 对数据再次进行包装 ip 地址 如果传输层的数据还是非常大 网络层会再次进行分包 IP 协议 <code>包</code></li>
<li>数据链路层 再次包装成 <code>帧</code></li>
<li>物理层 这两层涉及到的是硬件</li>
</ol>
<h3 data-id="heading-2">五层模型（统称为 TCP/IP 协议 协议簇）</h3>
<ul>
<li>能称为协议的都在数据链路层之上，所以在网络层、传输层、应用层才有协议</li>
<li>协议的功能就是把数据按照某个规范去封装，再把数据进行传输（协议就是对数据的封装+传输）</li>
<li>网络层
<ul>
<li>ip 协议 寻址</li>
<li>ARP 协议 从 ip 地址获取 mac 地址（局域网下）</li>
</ul>
</li>
<li>传输层：TCP、UDP</li>
<li>应用层: http 超文本传输协议、 DNS 域名解析成 ip 地址、 FTP 文件传输协议</li>
</ul>
<h3 data-id="heading-3">DNS 协议</h3>
<ul>
<li>DNS 服务器进行域名和对应 ip 地址转换的服务器</li>
</ul>
<h3 data-id="heading-4">IP 协议</h3>
<ul>
<li>寻址 通过 ip 地址定位到最终设备</li>
</ul>
<h3 data-id="heading-5">ARP协议</h3>
<ul>
<li>怎么通过ip地址找到mac地址的
<ul>
<li>ARP协议：目的就是通过ip地址找到mac地址 （在局域网下）</li>
<li>会广播给每一个，对应到要找的IP地址会做出响应，将mac地址返回</li>
<li>ARP会有缓存记录 有记录就不用广播  缓存的是 ip:mac</li>
<li>交换机缓存的是 mac:端口</li>
</ul>
</li>
</ul>
<h2 data-id="heading-6">TCP 协议</h2>
<ul>
<li>传输控制协议 特点是可靠、传输效率低</li>
<li>TCP 提供全双工服务，即数据可在同一时间双向传播</li>
<li>数据是无序的在网络间传递，接收方需要有一种算法在接收到数据后恢复原有的顺序</li>
<li>三次握手
<ul>
<li>我能主动给你打电话吗</li>
<li>可以啊 那我也能给你打电话吗</li>
<li>可以 建立连接成功</li>
</ul>
</li>
<li>四次挥手
<ul>
<li>我们分手吧</li>
<li>回复收到分手的信息</li>
<li>好吧 分手</li>
<li>行 那就到这</li>
</ul>
</li>
<li>小结
<ul>
<li>TCP 是双工的，所以握手需要三次，保证双方达成一致（建立连接浪费性能）</li>
<li>当断开连接时，发送（FIN）时另一方需要马上回复（ACK），但此时可能不能立即关闭（有未发送完的数据，还有一些准备断开的操作），所以等待确认可以关闭时再发送（FIN）</li>
</ul>
</li>
<li>滑动窗口
<ul>
<li>窗口大小是以字节为单位</li>
<li>建立TCP连接时，接收方会告知自己的窗口可以接收多少字节的数据</li>
<li>滑动窗口的目的是实现丢包重传，并且将数据有效的发送给接收方，而且可以知道每次发送多少，从而做到流量控制</li>
<li>所以滑动窗口的核心是<code>控制流量</code></li>
<li>TCP会做什么：传输的时候会做一个滑动窗口，不停地在协商滑动窗口的大小，如果值满了就停止传输，不停地发探测包，如果有了可用窗口大小，接着去传输数据。按照顺序，成功接收到一个数据之后滑动窗口会向后移动，并根据协商的结果修改窗口大小</li>
</ul>
</li>
<li>慢启动、拥塞避免、快重传、快恢复
<ul>
<li>也是用来做流量控制的</li>
<li>慢开始 超时</li>
<li>快重传 不等超时 三次确认就立马重发</li>
</ul>
</li>
</ul>
<h2 data-id="heading-7">http</h2>
<ul>
<li>发展历程
<ul>
<li>http/0.9 传输过程中没有请求头和请求体 内容采用ASCii字符流传输html</li>
<li>http/1.0 加了请求头和响应头 实现多类型数据传输</li>
<li>http/1.1
<ul>
<li>持久连接 一个TCP连接上可以传输多个http请求</li>
<li>管线化方式 每个域名最多维护6个TCP持久连接 有队头阻塞问题</li>
<li>引入客户端cookie机制，安全机制等</li>
</ul>
</li>
<li>http/2.0
<ul>
<li>采用多路复用机制 一个域名使用一个TCP长链接</li>
<li>头部压缩</li>
<li>服务端推送</li>
</ul>
</li>
<li>http/3.0 基于UDP</li>
</ul>
</li>
<li>当访问网页，发生了哪些事情
<ul>
<li>客户端发送一个请求  发送的是一个域名 域名会发送给dns服务器</li>
<li>dns解析出一个ip给到客户端</li>
<li>http会把数据进行传输，http协议生成针对目标web服务器的http请求报文</li>
<li>交给TCP进行传输：为了方便通信，将http请求报文按序号分割成报文段，把每个报文段可靠的传给对方（丢包或者超时就重发）</li>
<li>（TCP自身没有传输能力，如何传靠的是网络层）ip协议：寻址和路由  找到对方地址，通过mac地址一个个找到下一站，不停中转</li>
<li>从对方接收到TCP报文段， 对报文段按照原来的序号进行重组报文</li>
<li>http解析传递过来的数据，对内容进行处理</li>
<li>处理完成之后再增加一些响应信息，同样利用TCP/IP通信协议向用户进行回传</li>
</ul>
</li>
</ul>
<blockquote>
<p>HTTP是不保存状态的协议， 使用cookie来管理状态</p>
<p>为了防止每次请求都进行tcp链接的建立和断开， 采用保持链接的方式<code>keep-alive</code></p>
<p>以前发送请求后需要等待并收到响应，才能发下一个，现在都是管线化方式</p>
</blockquote>
<h2 data-id="heading-8">http应用</h2>
<ul>
<li>http也是封装了一些信息和传输</li>
<li><code>curl -v www.baidu.com</code></li>
<li>http分为两部分 （发送请求 client）req -> （接收请求 server）res</li>
<li>请求分为三部分（三部分都可以传输数据）
<ul>
<li>请求行  通过方法 路径（传输的数据有限制 url大小限制）
<ul>
<li>如 GET / 版本号</li>
</ul>
</li>
<li>请求头  放一些自定义信息  约定的信息  请求头不要过大</li>
<li>请求体  传输的数据</li>
</ul>
</li>
<li>响应也分为三部分
<ul>
<li>响应行
<ul>
<li>如 版本号 200 ok</li>
</ul>
</li>
<li>响应头</li>
<li>响应体</li>
</ul>
</li>
<li>restFul风格
<ul>
<li>GET/POST/PUT/SELETE/OPTIONS</li>
<li>OPTIONS请求代表的是跨域访问时可能会出现 预检请求 试探请求 如果对方确认后 可以发真实的请求</li>
<li>OPTIONS请求只在复杂请求的状态下才能发送（get、post都是简单请求，如果增加了自定义header那么就是复杂请求）</li>
<li>OPTIONS可以定义发送的时间间隔</li>
</ul>
</li>
<li>常见的状态码
<ul>
<li>可以自己设定 但是浏览器和服务器之间是有一些约定的</li>
<li>1xx 服务器收到了信息 等到浏览器后续要做的事  websocket</li>
<li>2xx 成功</li>
<li>3xx 重定向 缓存</li>
<li>4xx 客户端出错 （浏览器参数、或者服务器无法解析客户端参数）</li>
<li>5xx 服务器错误</li>
<li>200 请求成功</li>
<li>204 请求成功 但是无响应内容</li>
<li>206部分内容 分段请求</li>
<li>301 302 永久重定向和临时重定向</li>
<li>304缓存</li>
<li>400 客户端请求错误</li>
<li>401 权限问题 当前用户没登录 无权限观看</li>
<li>403 登陆了 但是还是没有权限</li>
<li>404 找不到</li>
<li>405 服务器只支持get、post 但是发送了put请求 服务器就会响应找不到此方法</li>
<li>500 请求服务解析出错了 无法完成响应</li>
<li>502 服务期收到的内容无效</li>
<li>503 负载均衡挂了</li>
</ul>
</li>
</ul>
