<h2 data-id="heading-1"><p align="center">长列表数据解决方案之虚拟列表</p></h2>
<h3 data-id="heading-2">长列表问题如何解决</h3>
<ul>
<li>
<p>懒加载->常见的无线滚动的，每次只渲染一部分（比如10条），等剩余部分滚到可见区域，就再渲染一部分。</p>
<ul>
<li>缺点:当翻了很多页之后,页面中有很多dom节点,用户上滑的体验不好</li>
</ul>
</li>
<li>
<p>可视区域加载->只渲染可见部分，不可见部分不渲染。</p>
<ul>
<li>有点类似于js实现无缝轮播的效果,在可视区域显示数据,在非可视区域提取渲染好即将展示的内容,但无缝轮播的dom还会在,虚拟列表的dom会被替换更新掉</li>
</ul>
</li>
</ul>
<h3 data-id="heading-3">虚拟列表的实现</h3>
<ol>
<li>把列表分为三个区域,上缓冲区,可视区,下缓冲区</li>
<li>计算可视区域最多能展示多少项,并记录高度</li>
<li>监听用户滚动事件/触摸事件,记录用户滚动的高度,以此来计算可视区域的起始索引和结束索引(如果不是整数则设置偏移量).</li>
<li>创建一个函数,每当用户滚动时重新渲染可视区域的dom和偏移量,可以缩减到16ms执行一次</li>
<li>函数执行会获取最新的数据生成dom结构去替换之前可视区域的dom列表.</li>
</ol>
<h2 data-id="heading-4"><p align="center">设计模式之观察者模式</p></h2>
<h3 data-id="heading-5">观察者模式</h3>
<blockquote>
<p>通常又被称作为发布-订阅者模式。它定义了一种一对多的依赖关系，即当一个对象的状态发生改变的时候，所有依赖于它的对象都会得到通知并自动更新，解决了主体对象与观察者之间功能的耦合。</p>
</blockquote>
<blockquote>
<p>观察者模式必须包含两个角色：观察者和观察对象，两者之间存在“观察”的逻辑关联，当观察对象状态发生改变时，将通知相应的观察者以更新状态。</p>
</blockquote>
<h2 data-id="heading-6"><p align="center">Vue插件发布流程</p></h2>
<ol>
<li>
<p>封装一个高复用组件(一定要设置name)</p>
</li>
<li>
<p>创建文件夹把组件放进去并创建入口文件index.js</p>
</li>
<li>
<p>创建install方法并把它暴露出去, 方法里面实现环境判断以及组件全局注册.</p>
</li>
<li>
<p>package.json设置打包命令压缩打包,然后设置插件信息(备注,协议,main文件路径,是否私有)</p>
</li>
<li>
<p>npm login 登录 -- npm publish 发布插件</p>
</li>
</ol>
<h2 data-id="heading-7"><p align="center">Vue的性能优化</p></h2>
<ul>
<li>
<p>函数式组件,正常组件有template模板,有生命周期函数等等,在虚拟dom里会被递归,但函数式组件就是正常的真实dom,没有多余的东西,所以提示js渲染性能,在vue3中不推荐使用.</p>
</li>
<li>
<p>缓存组件,被keep-alive包裹的组件不会因为切换路由而销毁实例和数据.</p>
</li>
<li>
<p>非响应式数据,对于不需要响应式的对象或数组,进行手动标识(Object.freeze)不需要响应式,vue3里面已经实现</p>
</li>
<li>
<p>异步组件(component：() => import('./component/helloworld'))如dialog框表单.</p>
</li>
</ul>
