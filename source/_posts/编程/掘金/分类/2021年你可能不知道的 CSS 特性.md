<p>虽然直到目前为止还没有 <code>:parents()</code> 选择器，但庆幸的是，<code>:has()</code> 选择器即将到来，它可以用来选择父级元素。目前 Igalia 公司正在为 Chrome 实现该选择器，其团队成员 Brian Kardell 新发表的《<a href="https://bkardell.com/blog/canihas.html" target="_blank" rel="nofollow noopener noreferrer"><strong>Can I :has()</strong></a>》文章中对 <code>:has()</code> 选择器进行了详细的阐述。 ​</p>
<pre><code class="copyable"><section><!-- section 边框颜色是 blue,margin-bottom是 30px --> 
  <h1>H1 Level Title</h1>
</section>  

<section><!-- section 边框颜色是 #09f,margin-bottom是 30px --> 
  <h1>H2 Level Title</h1>
</section>  

<section><!-- section 边框颜色是 red --> 
  <p>Text Paragraphs</p>
</section>  

/* CSS */

// 将匹配含有h1子元素的 section元素
section:has(h1) &#123;
border-color: blue;
&#125;
// 将匹配含有h2子元素的 section元素
section:has(h2) &#123;
border-color: #09f;
&#125;
// 将匹配含有p子元素的 section元素
section:has(p) &#123;
border-color: red;
&#125;
// 将匹配除了含有p子元素的 section元素

&#125;
复制代码
<span class="copy-code-btn">复制代码</span></code></pre>

<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/fdc090bde68c4929b6b6735fe4d4f941~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>示例中还演示了 <code>:not()</code> 和 <code>:has()</code> 组合在一起使用的，但两者组合在一起所达到的意思却完全不一样。其中 <code>:not(:has(selector))</code> 匹配不含有 <code>selector</code> 元素的父元素，而 <code>:has(:not(selector))</code> 匹配含有的不是 <code>selector</code> 子元素的元素。两者主要区别在于，<code>:has(:not(selector))</code> 写法必须要含有一个子元素，而 <code>:not(:has())</code> 可以不含有元素也会被匹配。 ​</p>
<p>有意思的是，<a href="https://github.com/w3c/csswg-drafts/issues/4903" target="_blank" rel="nofollow noopener noreferrer">Github 中有一个 Issue</a> 在讨论** <strong><code>**:has-child()**</code></strong> 选择器**，或许哪一天，我们在组件中就可以这样使用： ​</p>

<h3 data-id="heading-4">:empty 和 :blank</h3>
<p>在现代 Web 的开发的过程中，总是无法避免数据吐出为空的情况，此时往往会给我们的 UI 带来额外的麻烦。比如说，在同一类的 UI 元件中编写了一定的样式规则，但数据为空，此时在页面上可能会出现这样的场景： ​</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/7ca7b6519b95471bbee6928d4ef2489a~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>​ CSS 的 `:empty` 和 `:blank` 两个伪类选择器可以帮助我们避免这种现象。这两个选择器都很有用： ​</p>
<ul>
<li>给空元素添加样式</li>
<li>创建空的状态</li>
</ul>
<p>​</p>
<p>既然都是可以为空元素添加样式，那何为空元素，他们之间的差异又是什么？先来回答第一个问题，何为空元素？空元素是指元素没有任何子元素或子节点，比如： ​</p>
<pre><code class="copyable"><!-- 空元素 -->
<div class="error"></div>
<div class="error"><!-- 注释 --></div>

<!-- 非空元素 -->
<div class="error"> </div><!-- 中间有一个空格符 -->
<div class="error">
</div><!-- 断行 -->
<div class="error">
<!-- 注释 -->
</div><!-- 注释断行排列 -->
<div class="error"><span></span></div>
复制代码
<span class="copy-code-btn">复制代码</span></code></pre>
<p>​</p>

<p><code>:blank</code> 要比 <code>:empty</code> 灵活地多，只要该元素中无任何子元素都能被识别。不过 W3C 规范对该伪类选择器的定义更趋向于作用到表单控件中，比如用户没有在 <code>input</code> 或 <code>textarea</code> 中输入内容时，提交表单能被识别到。有点类似于表单验证的功能。 ​</p>

<p>到目前为止，<code>:empty</code> 已得到主流浏览器支持，可以用于实际生产中，但 <code>:blank</code> 伪类选择器还是存在一定的争议的，<a href="https://github.com/w3c/csswg-drafts/issues/1967" target="_blank" rel="nofollow noopener noreferrer">有关于这方面的讨论可以点击这里查阅</a>。</p>
<h3 data-id="heading-5">:focus-visible 和 :focus-within</h3>
<p>CSSer 对于 <code>:focus</code> 应该不会感到陌生，早期对于可聚焦元素可以使用 <code>:focus</code> 来给元素设置焦点状态下的 UI 风格，即焦点环样式： ​</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c5e403e13be646dfa1777b350a70bd65~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>Chrome 86 开始，另外引入了 <code>:focus-visible</code> 和 <code>:focus-within</code> 两种伪类选择器来帮助我们更好的控制 UI 焦点环的样式： ​</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/43c52ff3c8e34eefa8166b5c7cc147e8~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>虽然 <code>:focus</code> 、<code>:focus-within</code> 和 <code>:focus-visible</code> 都可以用来设置焦点环的样式，但他们之间还是有一定的差异： ​</p>
<ul>
<li><code>:focus</code> ：当用户使用鼠标点击焦点元素或使用键盘的 <code>Tab</code> 键（或快捷键）触发焦点元素焦点环的样式</li>
<li><code>:focus-visible</code> ：只有使用键盘的 <code>Tab</code> 键（或快捷键）触发焦点元素焦点环的样式。如果仅使用 <code>:focus-visible</code> 设置焦点环样式的话，那么用户使用鼠标点击焦点元素时不会触发焦点环样式</li>
<li><code>:focus-within</code>：表示一个元素获得焦点，或该元素的后代元素获得焦点。这也意味着，它或它的后代获得焦点，都可以触发 <code>:focus-within</code>​</li>
</ul>

<p><code>color-adjust()</code> 函数可用来在给定颜色空间中通过指定的变换函数对该颜色进行调整，除非另有规定，否则调整是在 <code>lch()</code> 色彩空间中进行的。比如 <code>color-adjust(#0af lightness +50%)</code> 是颜色 <code>#0af</code> 在 <code>lch</code> 颜色空间中高度增加 <code>50%</code> 。</p>
<p>也就是说，在不久的未来，这些颜色函数可以很好的帮助我们控制Web中的颜色值。因为这些颜色函数都已成为主流浏览器的实验性属性，特别是 Safari 浏览器，对这方面的支持度要高于 Chrome 和 Firefox 浏览器。 ​</p>

<h3 data-id="heading-9">background-repeat</h3>
<p>在 <code>background-repeat</code> 属性上除了可以使用我们熟悉的 <code>no-repeat</code> 和 <code>repeat</code> (或者 <code>repeat-x</code> 和 <code>repeat-y</code>) 之外还可以使用 <code>round</code> 和 <code>space</code> 。 ​</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9836d1bd9d0b457ba84221f9eca5fb1e~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>

<h2 data-id="heading-12">CSS 自定义属性</h2>
<p>CSS 自定义属性可以说是 CSS 的标配了，正如 W3C 规范所描述的，<a href="https://www.w3.org/TR/css-variables-1/#cycles" target="_blank" rel="nofollow noopener noreferrer"><strong>CSS 自定义属性又被称为 CSS 变量</strong></a>： ​</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b0fc923b6f354d8d941a69e6f2c2a0dc~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>不过在这里要说的不是 CSS 自定义如何使用？要和大家说的是 CSS 自定义属性中的无效变量。CSS 自定义属性中的无效变量是很有用的一个特性，它可以实现 <code>1</code> （真）和 <code>0</code> （假）的开关切换效果。换句话说，可以很好的实现不同状态的 UI 效果。 ​</p>
<p>同样地，在 W3C 规范中对无效变量有详细描述，但不仔细的话，还是会忽略该特性。先来看<a href="https://www.w3.org/TR/css-variables-1/#invalid-variables" target="_blank" rel="nofollow noopener noreferrer"><strong>规范是怎么描述无效变量</strong></a>的： ​</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/e9ddd6f8b95c49babce865b554517934~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>它的意思是： ​</p>
<blockquote>
<p><strong>当一个自定义属性的值是 <strong><code>**initial**</code></strong> 时，</strong><code>**var()**</code>** 函数不能使用它进行替换。除非 指定了一个有效的回退值，否则会使声明在计算值时无效。 **</p>
</blockquote>
<p>​</p>
<p>在 CSS 中注册自定义属性时是使用 <code>--</code> 来注册的，可以给已注册的自定义属性赋值，包括空字符串，但其中有一个细节非常有意思： ​</p>
<pre><code class="copyable">:root &#123;
--invalid:; // 注意冒号和分号之间无空格符，也无任何字符
  --valid: ;  // 注意冒号和分号之间只有一个空格符
&#125;
复制代码
<span class="copy-code-btn">复制代码</span></code></pre>
<p>其中，<code>--invalid</code> 自定义属性被称为无效变量，而 <code>--valid</code> 自定义属性是一个有效变量。使用上面这种方式来区分有效和无效变量对于开发者而言，可读性极差，而且有些文本编辑器可能会对代码按自己配置的规格格式化，有可能会造成 <code>--invalid:;</code> 变成 <code>--invalid: ;</code> （有空格）。为此，一般使用关键词 <code>initial</code>来显式声明一个自定义属性为无效变量：</p>
<pre><code class="copyable">:root &#123;
--invalid: initial; // 无效变量
  --valid: ;// 有效变量
&#125;
复制代码
<span class="copy-code-btn">复制代码</span></code></pre>
<p>如果不使用 <code>var()</code> 函数调用已注册的自定义属性的话，那么对于已注册的自定义属性而言，不会起任何作用。而 <code>var()</code> 函数有两个参数，第一个参数就是自定义属性，第二个参数是备用值。当第一个参数是个无效值时，会采用第二个参数。正因如此，对于已注册的无效自定义属性（即无效变量），比如 上面代码中的 <code>--invalid</code> 。那么 <code>var()</code> 没有提供备用值（第二个参数），则会使 CSS 样式规则（声明）在计算值时无效： ​</p>
<pre><code class="copyable">:root &#123;
