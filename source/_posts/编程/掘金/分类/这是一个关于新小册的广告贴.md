h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h2 data-id="heading-0">前言</h2>
<p>今天上了一本预发的小册《基于 Node 的 DevOps 实战》。</p>
<p>趁着这个首发机会跟大家介绍一下整个小册规划以及适合的人群。</p>
<h2 data-id="heading-1">小册的由来</h2>
<p>之前写过博客连载过一样的系列，但是中途有些事情耽搁后面就没有继续了，小册也是为了弥补之前的一个遗憾。</p>
<p>其次由于之前的博客系列细节部分很少，包括环境、配置、框架使用等等很多的细节都是默认会用就略过了，这样导致有些入门的同学看完之后，似懂非懂也没法下手。</p>
<p>所以现在的小册添加了很多环境配置、教学等章节。</p>
<h2 data-id="heading-2">适用人群</h2>
<p>最开始设计的适用人群其实针对中高级前端的，但根据之前以及写的过程中得到反馈，重新修改了一下用户群体。</p>
<p>中高级开发本身自学能力就非常强，大概率也会处于一个中型以上的团队或公司，对这一套东西比较熟悉，虽然 DevOps 的大体流程都是一样，但是在不同的业务场景下依然有不同的实践与理解。</p>
<p>那么对于初中级前端来说，在小规模大团队与公司的可能性更大，接触到成套体系的机会比较少。感觉小册内容对他们的收益会更大，所以后面大量添加了基础章节过度。</p>
<h2 data-id="heading-3">小册的规划</h2>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/9bd77dfbb3cc4c19ad2f4a786c864fe6~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>整个小册的学习模块将分为上述 4 个：</p>
<ol>
<li>服务端</li>
<li>客服端</li>
<li>CI/CD</li>
<li>监控</li>
</ol>
<p>通过这 4 个模块的串联，可以构建一个简单的 DevOps 体系，这里之所以用的是的<strong>体系</strong>而不是项目是因为不仅仅只介绍了 Node 服务端的开发，也会从 Cli、插件入手，从研发开始就把整个一套体系搭建出来，如果团队已有项目管理的话，也可以尝试通过 Cli 或插件来改进现有的流程的欠缺与不足。</p>
<p>在写的过程中，我也会将对应的 code 放在 github 上，同时会根据反馈、建议修改代码、流程、小册内容等。这也算是一个以我为主的共建项目吧。</p>
