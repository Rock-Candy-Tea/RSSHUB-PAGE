h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h1 data-id="heading-0">前言</h1>
<p>近期经过自己的努力（真的一直在努力呜呜呜），获得了阿里的意向书，很多同学和朋友感到了意外（但我自己并不感到意外），也特地地来询问关于这一方面的事情。想想自己为面试准备的这一年半载，也当是一次总结和一次不辜负自己。当然，也可以给自己身边的朋友或者一样努力的人儿相互学习，来获得一种<strong>自我认知存在感的意义</strong>。这一篇是准备篇，也就是非面试技术相关的。</p>
<h2 data-id="heading-1">背景</h2>
<p>简单说一下本人的背景叭，坐标于广东深圳，学籍于广州一所普通二本的高校 22 届软件工程专业的学生。为什么要强调学籍呢，因为报考期间我选择了特殊的一种培养方式：<strong>就是学籍在广州，就读在广东深圳</strong>。当时经过自己很多深思熟虑的考量，也经受了很多长辈和朋友同学们的质疑（因为当时成绩可以报考北方的重本）。或许自己从小比较独立的原因，也深刻地问过自己，认清了自己，也就不在乎外界是是非非，也就自我内心坚定了许多，懂得什么是自己真正想要。</p>
<blockquote>
<p>所以一定要认知，认清自己。</p>
</blockquote>
<h1 data-id="heading-2">经历</h1>
<p>说一下自己杂七杂八的经历吧，不能说这些经历一定有用或是有参考意义。但对于个人来说，在后边的几轮面试，也确实带给我更多的优势和机会。</p>
<p>而我的经历其实都带有一定的目的性和目标性，也就是我的座右铭：<strong>我需做一位经得起繁华，且耐得住寂寞的人。</strong></p>

<p>反正我有个概念是：<strong>能不待在宿舍就不待在宿舍，逼自己找事做。</strong></p>

<p><strong>这一些收获，我愿意称之为我的繁华，并在一次次地经受着。</strong></p>

<h2 data-id="heading-6">技术经历</h2>
<p>或许是恐惧促使我不断地争取新的机会。大一上册除了熟悉环境以及社团活动等，在结束了课程后，猛然发觉课程的进度实在是太慢了，自己有很多很多的空余时间被遗忘。或许是对时间流逝的恐惧，在期间不断地上网找自己的目标方向和今后的就业方向，也就毅然决然地选择了前端这条道路。因为自己喜爱心理学，因各种原因终究还是选择了计算机行业，对于前端，个人有一种朦胧地觉得<strong>其价值就是以彼此的交互，来影响用户心理效应，达到吸引、满意更或是依赖作用。</strong>（好吧，其实是更对冰冷冷数据的处理感到无感鹅鹅鹅）</p>
<p>所以自己可以简单总结以下三个阶段：</p>
<h3 data-id="heading-7">自我</h3>
<p>自我的学习主要是自己单打独斗的时间，泡在图书馆，有些许盲目地找各种方向和资源。不过找得越多，走得弯路越多，也就越能回到正轨上来。</p>
<ul>
<li>网上视频资源（b站、腾讯课堂、极客等等）</li>
<li>动手敲各种案例</li>
<li>个人方向的不断摸索</li>
<li>熟悉初级前端的概念究竟是什么</li>
</ul>
<h3 data-id="heading-8">实验室</h3>
<p>在大一下册偶然接触到了我的一位专业老师（也就是今后实验室的导师）。当时导师简单地提了一句他那边有对应的学习资源可以培养人员技术，于是自己觉得需要争取这个机会，哪怕再小的资源也好过自己瞎摸索。当时害怕自己失去这个机会，还急燎燎地去找老师交流。没想到是好像就我一位，这么焦虑有危机意识地少年郎鹅鹅鹅鹅。这段经历可以细分为以下阶段：</p>
<ul>
<li>大一下册除了自己之前学过的前端基础，还跟随一位学长的方向首次接触了 ECMAScript、网络，node.js 以及 vue 框架。一步步地学习沉淀后，在大二上册正式长期进入实验室里开始做项目。</li>
<li>边做项目需求，边学习新知识。（一边跟进需求，一边积累经验并记录总结）</li>
<li>慢慢地自己也能独立地负责一整个企业级的项目，当时没日没夜地不断做项目需求。
<ul>
<li>回忆起来确实付出了很多很多，刨去自己课程时间外，其余时间也就所剩无几。加上自己整活了那么多东西，还好自己计划方法得当，一步步解构成小目标，并且坚持了下来，也就经历了过来了。</li>
<li>记得每晚都是晚上 10 点左右独自回宿舍，时常还要赶在点名之前回去。</li>
</ul>
</li>
<li>整自己的项目，尝试接触更多的技术的知识。
<ul>
<li>比如自己的微信小程序。</li>
<li>node 的各种知识，如 koa、koa2、egg 框架等</li>
<li>electron 的远程小应用</li>
<li>智能识别等</li>
</ul>
</li>
</ul>
<blockquote>
<p>除了项目之余，更多的是看各种各样的实战视频来接触其他新鲜技术。</p>
<p>最大的收获就是企业级的项目是如何从零到一的整体流程，一个项目团队的沟通和交流，自我解决各种需求问题的能力，抗压精神，项目的实战经验等等。</p>
</blockquote>
<h3 data-id="heading-9">如今</h3>
<p>往后自身到了一个瓶颈期，首先是项目类型受限，接受的前端项目大多是 toB 类型。虽然也接触了 egg 后端和微信小程序等，但接触地不深，只是能一块块做对应的需求。所遇到的问题越来越特殊具体化，简单查询也无法快速解答，询问学长们也慢慢受限。于是开始转变自己的学习方向和阶段。</p>
<ul>
<li>接触社区（个人最喜欢的就是泡在掘金里）</li>
<li>整自己的博客，开始记录，逼迫自己来输出自己的知识点。</li>
<li>查看官方文档</li>
<li>阅读书籍（很感谢带我的学长，给了我第一本红宝书，也影响我沉下心来阅读书籍，是很重要很重要的沉淀。）</li>
<li>回滚自己的各种基础知识点</li>
</ul>
<blockquote>
<p>总之这一阶段，自己开始不断地摸索如何扎实自己，发现自己基础原理实在是太弱了。起因其实很简单，大二暑期的时候投了一次字节的日常实习，结果被吊打，很多概念都是模糊不清的，懂，但是理不清。于是自己每天抽出时间不断地回滚自己的基础，刷各种前端基础题，数据结构算法等等。</p>
<p>现在想想也不知自己当时的鲁莽不知说好还是不好，好是自己意识到了自己致命的弱点，不好是字节从此无限拒我的简历。（我真的已经努力了超级多了！）</p>
</blockquote>

<h1 data-id="heading-11">面试</h1>
<p>个人一开始投递面试的方向也杂七杂八，不知道该如何下手，也走了很多很多的弯路，下边就是我如今总结的收获。</p>
<h2 data-id="heading-12">时机</h2>
<p>投递的时机非常重要，公司一般都有固定的一段时间来进行招聘。只有当人家公司需要人手的时，而你也恰好逢时投递，这才是双方互利的时机。像我每个时机段都投递过，大多数都没有回应，在近期才有了实际上的效果回应。</p>
<p>像本人投递的实习生职位实际上没有专门的时机段，有的是乘上春、秋招的快车。</p>
<ul>
<li>春招：金三银四</li>
<li>秋招：金九银十/十一</li>
</ul>
<blockquote>
<p>为啥称之为金、银。这个大致搜索一下原因就知道啦</p>
</blockquote>
<h2 data-id="heading-13">机会</h2>
<p>我只说我的经历和感觉作为参考价值，或许有其他更好的方式，也麻烦指教一下呐！以下是个人内心排序的顺序。</p>
<h3 data-id="heading-14">方式</h3>
<ul>
<li>内推</li>
<li>官网</li>
<li>牛客</li>
<li>BOSS</li>
</ul>
<p>其他很多的招聘平台我也尝试过，只不过投递过后并没有回应，所以就不列出来了。</p>
<h3 data-id="heading-15">信息</h3>
<ul>
<li>我做了一件事，把所有我听过的，我认为是我需要争取机会的公司的官方招聘公众号全部关注了。
<ul>
<li>可以及时获得招聘时态和信息进度，但有推进的时候，也就意味着时机开始了。</li>
<li>在对应的留言后台，或推文留言可以拿到自己想要的内推码。</li>
</ul>
</li>
<li>我频繁地叨扰在牛客的内推人
<ul>
<li>我有个概念，这是一个两向有益的事，对方需要人才，我需要机会，所以只有争不争取这件事情罢了。</li>
<li>记得当时每天就叨扰留言一家公司的多个内推人来询问机会。</li>
<li>要是幸运联系到了，还可以不断完善自己的简历和需要做啥。</li>
</ul>
</li>
<li>投递
<ul>
<li>投就对了，若是合适会联系你的。当然重点是！简历是合适的喔！</li>
</ul>
</li>
</ul>
<h3 data-id="heading-16">简历</h3>
<p>关于简历，本人也是在不断地完善当中，自认为并不能给出正确的方向和实例，抛出一个严重的问题吧。在我面试的过程中，我发觉了一件特别严重的问题。我以为是我的优点，没想到是我致命的弱点。</p>
<blockquote>
<p><strong>在我的简历中我的项目过于多和琐碎，知识点和技术过于广和杂。</strong></p>
</blockquote>
<ul>
<li>虽然我写上去的，自己很清楚是曾有过一段时间花时间用心学过，只是掌握程度和熟悉的问题。
<ul>
<li>但是知识点越多，技术越杂显得你整个人"花"，不懂深入原理。
<ul>
<li>是的，现在好好想一下，很多知识点和技术方面确实只是了解，过了一遍，停留在会用的层面。</li>
</ul>
</li>
<li>越多，你的漏洞就越多，逐一深入便逐一击破。</li>
<li>面试官疲倦，面试时间有限，哪些是你的优势，哪些可问，没有引导，面试官很累。</li>
</ul>
</li>
<li>项目多是因为自己确实一直泡在实验室，花自己课余时间堆出来的。
<ul>
<li>但短短的大学时间那么多项目，初次接触下，疑惑和质疑占据大部分。</li>
<li>即便真的花了时间，真的可以细究项目吗？</li>
<li>即便自己在不同项目下有一直尝试着使用新技术和深入能力，但在需求和进度下，实现程度又能完备几何？</li>
</ul>
</li>
</ul>
<blockquote>
<p>因此综上，实际上也是在自我反问。</p>
<p>知识点和技术除特别熟悉和掌握外，不写，除非有针对性公司招聘需求。</p>
<p>项目写两个代表性足以，吃透项目难点，角色定位，业务输出价值，个人贡献价值。</p>
</blockquote>

<h3 data-id="heading-17">联系</h3>
<p>在开始决定投递后，便要做好接通各种莫名电话的准备。<strong>拦截的机制</strong>一定要关掉！hr 电话奇奇怪怪就被拦截了鹅鹅鹅鹅。</p>
<blockquote>
<p>当然除了面试邀请啥的，也就不要听了，被诈骗了我可不背锅</p>
</blockquote>
<p>一定要清楚自己的身体的状态，面试形式我自身主要经历过两种，电话/视频。以我为例，我的面试时间安排大概是这样。</p>
<ul>
<li>电话面试
<ul>
<li>能晚上就晚上，不能就下午4/5点。</li>
<li>个人晚上时间段会较为清醒。白天也不会太杂，就是晚上有点冷别着凉了。</li>
</ul>
</li>
<li>视频面试
<ul>
<li>能下午就不晚上。</li>
<li>不挑选晚上是因为楼梯间那个灯一熄一灭的，无语子。</li>
</ul>
</li>
</ul>
<blockquote>
<p>所以无论这个面试形式是如何，亦或是有多难得，沟通的时间除了自己硬性的时间安排外。一定要注意自己的身体生理上的状态以及自身周围环境的问题，不要硬着头皮上，我个人因为这几个问题，挂了不下3次的面试。</p>
</blockquote>
<h3 data-id="heading-18">准备</h3>
<p>关于约好时间后的准备，我大致如下。</p>
<ul>
<li>不刻意地刷面经，该系统复习还是复习，等到前一晚再刷下对应的面经，熟悉对应公司的面试思路。</li>
<li>了解对应公司的参考重点更应该说是特点。
<ul>
<li>字节：算法。腾讯/网易：原理。阿里：项目。</li>
<li>个人认为比较有代表的吧，也不知道对不对，纯属个人感觉，可以自我总结一下，当作是参考意义。</li>
</ul>
</li>
<li>面试前一小时，一定要趴一下，趴到脑子不沉为止，别睡多，会晕。（问就是晕过）</li>
<li>面试前 15 分钟，不断背诵熟悉个人介绍。
<ul>
<li>个人介绍是整个面试的开端，一个良好的开端为整个面试节奏奠定基础。（除了某些面试太赶时间，直接跳过了个人介绍。）</li>
<li>引导个人擅长部分，擅长 JS 基础，项目经验框架使用，还是网络等基础内容。</li>
</ul>
</li>
<li>面试前 3 分钟，调整亢奋状态，引导面试官的情绪。
<ul>
<li>因为个人平时性子较缓，语速较慢，所以这一点对于我非常重要。</li>
</ul>
</li>
</ul>
<h1 data-id="heading-19">后言</h1>
<p>简单说一下个人面试完后的事儿吧。</p>
<h2 data-id="heading-20">心态</h2>
<p>无论面试完结果如何吧，不要太沮丧，可以有一点生气。生气自己为啥努力了这么久还是没获得对应值得的成果，于是要更努力来争取下一次的机会，一次次来逼迫自己向前进，一次次问自己凭啥就止步于此。</p>
<p>对于面试机会的心态，用心投，随心等的心态是最合适的，遇到问题和完善的意见也要积极地改进和自我检讨。当然要是一直一直没有得到回应，一定要去向别人指教和学习，或者寻求帮助找对应的文章学习，方法和准备缺一不可。</p>
<h2 data-id="heading-21">总结</h2>
<p>总结！非常非常非常重要！我个人对于面试形式更喜欢是电话的形式，因为可以录音，可以完整地记录自己不足的地方。不断地总结，不断地完善是非常非常重要的。而且说不定会有意外的惊喜，像一次腾讯的面试，很温柔和可爱的两位面试官说有查看我做的总结笔记是一个十分好的习惯，这种被人承认努力的感觉真的太好了！希望也有人能看见你的努力，接下来你就要想办法向别人证明和实现你的努力！</p>
<h2 data-id="heading-22">目标</h2>
<p>下一阶段目标：转正。没为啥，反正凭啥我不能转正呢？还有一定要每天警醒自己，优秀的人还有很多很多，你要做更多充足的准备。</p>
<p>反正未来未知的事还有很多很多，把握珍惜当下，时间可贵。</p>
<p>最后衷心感谢每个位不论什么形式我曾接触的人儿，还有小恶魔。</p>

对啦，这只是个人针对自身实习的个人总结，若有不理解或不认同，大胆推翻我即可。</p>
