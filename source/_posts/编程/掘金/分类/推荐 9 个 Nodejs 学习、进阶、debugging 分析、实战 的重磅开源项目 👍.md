<h2 data-id="heading-5">6. Nodejs-Roadmap</h2>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a71fcabcb68541dfb49e52bf06b7af28~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>这个项目是作者从事 Node.js 开发以来的学习历程，旨在为大家提供一个较详细的学习教程，侧重点更倾向于 Node.js 服务端所涉及的技术栈，如果本文能为您得到帮助，请给予支持！</p>
<p>Node.js 由 Libuv、Chrome V8、一些核心 API 构成，如下图所示：</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/ef0cbdf89088437fa8a87936b8974d34~tplv-k3u1fbpfcp-zoom-1.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<p>以上展示了 Node.js 的构成，下面做下简单说明：</p>
<ul>
<li>
<p>Node Standard Library：Node.js 标准库，对外提供的 JavaScript 接口，例如模块 http、buffer、fs、stream 等</p>
</li>
<li>
<p>Node bindings：这里就是 JavaScript 与 C++ 连接的桥梁，对下层模块进行封装，向上层提供基础的 API 接口。</p>
</li>
<li>
<p>V8：Google 开源的高性能 JavaScript 引擎，使用 C++ 开发，并且应用于谷歌浏览器。如果您感兴趣想学习更多的 V8 引擎知识，请访问 <a href="https://v8.dev/" target="_blank" rel="nofollow noopener noreferrer">What is V8?</a></p>
</li>
<li>
<p>Libuv：是一个跨平台的支持事件驱动的 I/O 库。它是使用 C 和 C++ 语言为 Node.js 所开发的，同时也是 I/O 操作的核心部分，例如读取文件和 OS 交互。来自一份 <a href="https://github.com/luohaha/Chinese-uvbook" target="_blank" rel="nofollow noopener noreferrer">Libuv 的中文教程</a></p>
</li>
<li>
<p>C-ares：C-ares 是一个异步 DNS 解析库</p>
</li>
<li>
<p>Low-Level Components：提供了 http 解析、OpenSSL、数据压缩（zlib）等功能。</p>
</li>
</ul>
<p>以上只是做一个初步的认知，如果你想深入了解 Node.js 那么多每个点都是值得你深入研究的。</p>
<p>无论是内容还是阅读效果，都很好，是值得推荐的一个学习 Node 的开源项目。</p>
<blockquote>
<p><a href="https://github.com/qufei1993/Nodejs-Roadmap" target="_blank" rel="nofollow noopener noreferrer">github.com/qufei1993/N…</a></p>
