<h2 data-id="heading-0">青铜洗牌 🤬</h2>
<p><strong>题目：给你一副崭新的扑克牌（54 张），你如何 <strong>“洗乱”</strong> 它？？</strong></p>
<p>咱青铜玩家通常很暴躁！</p>
<p>不就是洗牌嘛！聪明的青铜玩家，先将问题抽象为算法模型！</p>
<p>问题转化为：</p>
<p>已知：一个数组 nums = [1,2,3,4,5,...,51,52,53,54]，求解：一个乱序的新数组 radomNums。（简直不能再 nice 了）</p>
<p>然后采用 <strong>【暴力抽取】</strong></p>
<p>在 1 至 54 之前随机生成一个整数，然后把它放到新数组里，然后再随机生成一个整数，如果和之前生成的没重复，直接放入新数组，如果和之前重复了，那再随机生成一个......</p>
<p>这样直至数组中所有数字都被抽取放到新数组，最终得解！</p>
<p>代码实现：</p>
<pre><code class="copyable">// 生成 nums：
let nums=[]
for(let i=1;i<=54;i++)&#123;
    nums.push(i)
&#125;

// 青铜洗牌算法：
let count = 0 
const shuffle = function(nums) &#123;
    let radomNums = []
    while (radomNums.length < nums.length) &#123;
        count++; // count 计数洗牌次数
        let rand = randOne()
        if(radomNums.includes(rand))&#123; // 随机数重复
            rand = randOne() // 再次生成
        &#125;else&#123;
            radomNums.push(rand)
        &#125;
    &#125;
    return radomNums
&#125;

// 在 1 至 54 之间任意取一整数；
const randOne= function() &#123;
    return Math.floor(Math.random() * 54) + 1;
&#125;

console.log(shuffle(nums))

<h2 data-id="heading-1">白银洗牌 📍</h2>
<p>白银玩家看了青铜玩家的操作，不禁放声大笑！</p>
<p><strong>“痴线~”（sb）</strong></p>
<p>把上述代码拷贝至控制台运行发现，基本上打乱这副扑克牌要洗 200 ~ 300 次！因为越往后，生成随机数重复的概率就越大！</p>
<p>只有痴线才会洗这么多次吧~</p>
<p>于是，白银玩家开始操作起来！遥感</p>
<p>关键词：<strong>【交换位置】</strong>。</p>
<p>将牌随机分成两堆，让它们交换，然后再随机分成两堆，再让它们交换，然后再随机分出两堆......这样重复洗十几、二十次后，完成洗牌。</p>
<p>实际上，在现实中，我们玩牌，大部分玩家也是这样去洗的，它也叫【印度洗牌法】（难道是阿三发明的？）</p>

<h2 data-id="heading-2">黄金洗牌 💡</h2>
<p>这时，黄金玩家又笑了：<strong>“呵呵，你个渣渣白银，根本还没看懂题目！”</strong></p>
<p>“你懂不懂洗牌问题最最关键的 <strong>“洗乱”</strong> 二字意义所在？！”</p>
<p>黄金洗牌来揭晓答案：</p>
<blockquote>
<p>随机的结果要能够覆盖所有的情况，并且随机结果出现的概率相等；</p>
</blockquote>
<p>洗 54 张牌，随机结果需覆盖所有情况就应该是 54 张牌的排列方式，A<sup>54</sup><sub>54</sub>，即 54!（54 的阶层）种随机结果。</p>
<p>两两对换：</p>
<p>洗 1 次，会得到 n 种可能的结果；</p>
<p>洗 2 次，会得到 n*(n-1) 种结果；</p>
<p>洗 3 次，会得到 n*(n-1)*(n-2) 种结果；</p>
<p>......</p>
<p>洗 n 次之后，我们才满足了：<strong>随机结果【覆盖所有情况】，并且所有随机结果【出现概率相等】</strong>。</p>
<p>所以，必须洗 54 次，才能达到目的。</p>
<p>代码实现：</p>
<pre><code class="copyable">// 生成 nums：
let nums=[]
for(let i=1;i<=54;i++)&#123;
    nums.push(i)
&#125;

// 黄金洗牌算法：
const shuffle = function(nums) &#123;

    // 高手都用 slice(0) 复制数组
    const radomNums = nums.slice(0);
    let n = radomNums.length;

    // 产生的结果有 n! 种可能
    for (let i = 0; i < n; i++) &#123;

        // 从 i 到 n-1 随机选一个
        const rand = randOne(i, n - 1); 

        // 交换nums数组i和rand下标的两个元素
        [ radomNums[i], radomNums[rand] ] = [ radomNums[rand], radomNums[i] ];
    &#125;

    return radomNums;
&#125;;

// 获取闭区间 [n, m] 内的一个随机整数
const randOne= function(n, m) &#123;
    return Math.floor(Math.random() * (m - n + 1)) + n;
&#125;;


console.log(shuffle(nums))
// (54) [39, 40, 11, 35, 1, 47, 33, 9, 44, 32, 31, 45, 41, 4, 51, 42, 8, 10, 16, 14, 18, 17, 13, 6, 34, 53, 48, 5, 15, 22, 38, 37, 49, 43, 3, 20, 26, 52, 30, 19, 7, 50, 12, 21, 46, 36, 23, 27, 28, 25, 2, 29, 24, 54]

<h2 data-id="heading-3">铂金洗牌 🤣</h2>
<p>正在网吧的铂金大神看到上面这些弟弟，笑出猪叫~~~</p>
<p>但凡上点网，学点攻略，就不至于在洗牌这个问题上没听说过 <strong>Fisher-Yates</strong> 洗牌算法！</p>
<p>思路：</p>
<p>随机生成 1 至 54 之间的整数，将它和数组的最后一位替换；</p>
<p>然后再在 1 至 53 之间随机生成一位整数，将它和数组的倒数第二位替换；</p>
<p>然后再 1 至 52 之间随机生成一位整数，将它和数组的倒数第三位替换；</p>
<p>......</p>
<p>直至在 1 至 1 之间随机生成一位整数（即 1），将它和数组第 1 位替换（即替换自身）；</p>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fimagelol.com%2Fimage%2FIdW3Yw" target="_blank" rel="nofollow noopener noreferrer" title="https://imagelol.com/image/IdW3Yw" ref="nofollow noopener noreferrer"><img src="https://s3.jpg.cm/2021/07/14/IdW3Yw.md.png" alt="IdW3Yw.md.png" loading="lazy" referrerpolicy="no-referrer"></a></p>
<p>这样做，时间复杂度为 O(n)，且任意一张牌出现的概率都是 1/52，满足：随机结果覆盖所有情况，随机结果出现概率相等！</p>
<p>数学证明：一张牌被放到第 i 个位置的机率为 P，则 P 会等于前 i-1 个位置都未选到这张牌且第 i 个位置选到这张牌。</p>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fimagelol.com%2Fimage%2FIdWw1W" target="_blank" rel="nofollow noopener noreferrer" title="https://imagelol.com/image/IdWw1W" ref="nofollow noopener noreferrer"><img src="https://s3.jpg.cm/2021/07/14/IdWw1W.png" alt="IdWw1W.png" loading="lazy" referrerpolicy="no-referrer"></a></p>
<p>代码实现：</p>
<pre><code class="copyable">// 生成 nums：
let nums=[]
for(let i=1;i<=54;i++)&#123;
    nums.push(i)
&#125;

// 铂金洗牌算法：
const FYShuffle = function (nums) &#123;

    const radomNums = nums.slice(0);
    let len = radomNums.length;
    
    while (len > 1) &#123;
        let rand = Math.floor(Math.random() * len);
        len--;
        let temp = radomNums[len];
        radomNums[len] = radomNums[rand];
        radomNums[rand] = temp;
    &#125;

    return radomNums;
&#125;

console.log(FYShuffle(nums))
// (54) [47, 17, 33, 13, 37, 26, 20, 39, 45, 44, 25, 40, 49, 7, 36, 38, 6, 15, 31, 18, 52, 46, 28, 11, 43, 1, 22, 19, 53, 9, 14, 27, 35, 8, 51, 42, 50, 2, 23, 5, 30, 54, 4, 21, 29, 16, 10, 24, 48, 34, 32, 12, 41, 3]

<h2 data-id="heading-4">钻石洗牌 😎</h2>
<p>钻石大神拍案站起，惊呼道：<strong>鸽尾式洗牌法（Riffle Shuffle），才是永远滴神！</strong></p>
<p>现实中很多扑克高玩都会这样洗吧（一图胜千言）</p>
<p><img src="https://p1-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8ae8e83d53d64887b2279c4a5de21d64~tplv-k3u1fbpfcp-watermark.image" alt="127614630_14271639688491n.gif" loading="lazy" referrerpolicy="no-referrer"></p>
<p>原理：将数组一分为二，再穿插合并，再不断重复这样的操作；</p>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2Fimagelol.com%2Fimage%2FIdnLHG" target="_blank" rel="nofollow noopener noreferrer" title="https://imagelol.com/image/IdnLHG" ref="nofollow noopener noreferrer"><img src="https://s3.jpg.cm/2021/07/14/IdnLHG.md.png" alt="IdnLHG.md.png" loading="lazy" referrerpolicy="no-referrer"></a></p>
<p>研究表明：<strong>用鸽尾式洗牌法【洗七次】是最有效的打乱手法！</strong>（谁研究的？自行 Google）</p>
<p>代码实现：</p>
<pre><code class="copyable">// 生成 nums：

    &#125;

<h2 data-id="heading-6">王者洗牌 👑</h2>
<p>此时，只见身在高处的王者背影。他缓缓转过身来。用手中的权杖指向了洗牌的终极的秘密：<strong>random 函数</strong>！神圣而又庄重！</p>
<p>你可曾想过，前文处处在使用的 random 函数的机制是什么？</p>
<p>如果我们破解了 random 函数的生成规则，是不是意味着破解了随机！</p>
<p>ES5 对 random 函数是这样解释的：</p>
<blockquote>
<p><a href="https://link.juejin.cn/?target=https%3A%2F%2F262.ecma-international.org%2F5.1%2F" target="_blank" rel="nofollow noopener noreferrer" title="https://262.ecma-international.org/5.1/" ref="nofollow noopener noreferrer">15.8.2.14 random ( )</a></p>
<p>Returns a Number value with positive sign, <strong>greater than or equal to 0 but less than 1</strong>, chosen randomly or pseudo randomly <strong>with approximately uniform distribution over that range</strong>, using an implementation-dependent algorithm or strategy. This function takes no arguments.</p>
</blockquote>
<p>简单翻译就是 random() 函数产生的值在 [0,1) 之间，具有<strong>大致均匀</strong>的分布，不用带参数。</p>
<p>进一步了解得知，在 Windows 系统 chrome V8 引擎下，浏览器最终调用一个 <strong>rand_s 函数</strong>，rand_s 是 Windows 系统提供的一个随机性足够强的随机数发生器。在其他系统下，也是基于当前系统获取高精度的随机数。所以，<strong>随机的根本源头是借助了操作系统的能力。</strong></p>
<p>但，MDN 上仍表示 Math.random 是一个<strong>伪随机数</strong>，它是不安全的。从 V8 的源码可以看到 Math.random 的值来源是 /dev/random，取 64 位，值的可能个数为 2 ^ 64，随机算法相对简单，只是保证尽可能的随机分布。</p>
<p>2 的 64 次方还不够大吗？没错，它还不够！要知道 54 张扑克牌的全排列 54!（阶层）的值远远大于它。所以，它还不能覆盖大量的组合情景。</p>
<p>那，这个世界上有 <strong>真随机数</strong> 吗？王者莞尔一笑~</p>
<p>目前认为，真随机数需要从现实世界采集，比如 <a href="https://link.juejin.cn/?target=http%3A%2F%2Frandom.org" target="_blank" rel="nofollow noopener noreferrer" title="http://random.org" ref="nofollow noopener noreferrer">random.org</a> 这个网站是通过采集大气噪音生成随机数。😮</p>
<p>还有：量子随机数，原理是 LED 发出随机数的光子，CMOS 捕获光源散粒噪声产生随机序列，量子芯片通过测量光量子态得到的随机数，再进一步来加密信息。</p>
<p>不得不说，产生真随机数，是一件非常有意义的事情。至少在目前世界还未证实 <strong>P ≠ NP</strong> 的情况下，它是非常有意义的！！</p>
