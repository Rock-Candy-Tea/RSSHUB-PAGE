h2&#123;font-size:20px&#125;.markdown-body h3&#123;font-size:18px&#125;&#125;</style><h3 data-id="heading-0">浏览器向服务器发起请求</h3>
<ul>
<li>浏览器从 DNS 服务器得到域名对应的 IP 地址</li>
<li>浏览器向这个 IP 地址对应的服务器发送请求</li>
<li>服务器接收、处理请求并返回内容（响应）</li>
<li>浏览器下载响应内容</li>
</ul>
<h3 data-id="heading-1">浏览器渲染过程</h3>
<ul>
<li>根据 HTML 构建 HTML 树（DOM）</li>
<li>根据 CSS 构建 CSS 树（CSSOM）</li>
<li>将 DOM 和 CSSOM 合并成一个“渲染树”（Render Tree）</li>
</ul>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/c7acabf117614b3fad5d9b4ef61b6931~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<ul>
<li>根据 Render Tree 来布局（Layout），计算文档流、盒模型、大小和位置等</li>
<li>将各个节点绘制（Paint）到屏幕上；（文本、颜色、图像、边框和阴影等）</li>
<li>根据层叠关系合成（Composite）展示画面</li>
</ul>
<p>浏览网页看起来可能很简单，但实际上浏览器需要完成相当多的工作。如果 DOM 或 CSSOM 被修改，浏览器只能再执行一遍以上所有步骤，以确定哪些像素需要在屏幕上进行重新渲染。</p>

<h3 data-id="heading-2">如何更新样式</h3>
<p>一般采用JS更新，例如：</p>
<pre><code class="hljs language-javascript copyable" lang="javascript">div.style.background =<span class="hljs-string">'red'</span>

<span class="copy-code-btn">复制代码</span></code></pre>
  <br>
  <br>
<p>浏览器在实现视觉变化时，通常有三种方式：
<br>
<br>
<br>
【1】全走</p>
<p>JS / CSS > 样式 > 布局 > 绘制 > 合成</p>
<p><img src="https://p6-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cf54445f7dff43a1afa7ab6a61ee4ee9~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>例：<code>div.remove()</code>会触发当前元素消失，布局发生改变，其他所有元素重新布局（Layout），任何受影响的部分都需要重新绘制，而且最终绘制的元素需进行合成。
<br>
<br>
<br>
【2】跳过Layout</p>
<p>JS / CSS > 样式 > 绘制 > 合成</p>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/1e679348e1474c929cca08a23807f70d~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>例：修改背景颜色、文字颜色或阴影等，不会影响页面的布局，浏览器会跳过布局（Layout），直接绘制（Paint）+合成（Composite）。
<br>
<br>
<br></p>
<p>【3】跳过Layout和Paint</p>
<p>JS / CSS > 样式 > 合成</p>
<p><img src="https://p9-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/f9f155f78db9455e86e201d7084ebc71~tplv-k3u1fbpfcp-watermark.image" alt="image.png" loading="lazy" referrerpolicy="no-referrer"></p>
<p>例：改变transform等，浏览器直接跳到合成（Composite）。
<br>
<br>
<br>
以上三种方式，第三种所消耗的性能最少。
