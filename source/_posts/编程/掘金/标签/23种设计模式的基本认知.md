<li>1.单例模式</li>
</ul>
<p>确保某一个类只有一个实例。而且自行实例化并向整个系统提供整个实例。</p>
<ul>
<li>2.工厂模式</li>
</ul>
<p>定义一个用于创建对象的接口，让子类决定实例化哪一个类。工厂方法使一个类的实例化延迟到其子类。</p>
<ul>
<li>
<ol start="3">
<li>抽象工厂模式</li>
</ol>
</li>
</ul>
<p>为创建一组相关或相互依赖的对象提供一个接口。而且无须指定它们的具体类。</p>
<ul>
<li>
<ol start="4">
<li>模板方法</li>
</ol>
</li>
</ul>
<p>定义一个操作中的算法的框架，而将一些步骤延迟到子类中。使得子类可以不改变一个算法的结构即可重定义该算法的某些特定步骤。</p>
<ul>
<li>
<ol start="5">
<li>建造者模式</li>
</ol>
</li>
</ul>
<p>将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。</p>
<ul>
<li>
<ol start="6">
<li>代理模式</li>
</ol>
</li>
</ul>
<p>为其他对象提供一种代理以控制这个对象的访问。</p>
<ul>
<li>
<ol start="7">
<li>原型模式</li>
</ol>
</li>
</ul>
<p>用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象。</p>
<ul>
<li>8.中介者模式</li>
</ul>
<p>用一个中介对象封装一系列的对象交互，中介者使各对象不需要显示地相互作用，从而使其耦合松散，而且可以独立地改变它们之间的交互。</p>
<ul>
<li>
<ol start="9">
<li>命令模式</li>
</ol>
</li>
</ul>
<p>将一个请求封装成一个对象，从而让你使用不同的请求把客户端参数化，对请求排队或者记录请求日志，可以提供命令的撤销和恢复功能。</p>
<ul>
<li>
<ol start="10">
<li>责任链模式</li>
</ol>
</li>
</ul>
<p>使多个对象都有机会处理请求，从而避免了请求的发送者和接受者之间的耦合关系。将这些对象连成一条链，并沿着这条链传递该请求，直到有对象处理它为止。</p>
<ul>
<li>
<ol start="11">
<li>装饰模式</li>
</ol>
</li>
</ul>
<p>动态地给一个对象添加一些额外的职责。就增加功能来说，装饰模式相比生成子类更为灵活。</p>
<ul>
<li>
<ol start="12">
<li>策略模式</li>
</ol>
</li>
</ul>
<p>定义一组算法，将每个算法都封装起来，并且使它们之间可以互换。</p>
<ul>
<li>
<ol start="13">
<li>适配器模式</li>
</ol>
</li>
</ul>
<p>将一个类的接口变换成客户端所期待的另一种接口，从而使原本因接口不匹配而无法在一起工作的两个类能够在一起工作。</p>
<ul>
<li>
<ol start="14">
<li>迭代模式</li>
</ol>
</li>
</ul>
<p>它提供一种方法访问一个容器对象中各元素，而又不需要暴露该对象的内部细节。</p>
<ul>
<li>
<ol start="15">
<li>组合模式</li>
</ol>
</li>
</ul>
<p>将对象组合成树形结构以表示“部分-整体”的层次结构，使得用户对单个对象和组合对象的使用具有一致性。</p>
<ul>
<li>
<ol start="16">
<li>观察者模式</li>
</ol>
</li>
</ul>
<p>定义对象间一种一对多的依赖关系，使得每当一个对象改变状态，则所有依赖于它的对象都会得到通知并被自动更新。</p>
<ul>
<li>
<ol start="17">
<li>门面模式</li>
</ol>
</li>
</ul>
<p>要求一个子系统的外部与其内部的通信必须通过一个统一的对象进行。门面模式提供一个高层次的接口，使得子系统更易于使用。</p>
<ul>
<li>
<ol start="18">
<li>备忘录模式</li>
</ol>
</li>
</ul>
<p>在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态。这样以后就可以将该对象恢复到原先保存的状态。</p>
<ul>
<li>
<ol start="19">
<li>访问者模式</li>
</ol>
</li>
</ul>
<p>封装一些作用于某种数据结构中的各元素的操作，它可以在不改变数据结构的前提下定义作用于这些元素的新的操作。</p>
<ul>
<li>
<ol start="20">
<li>状态模式</li>
</ol>
</li>
</ul>
<p>当一个对象内在状态改变时允许改变行为，这个对象看起来像改变了其类。</p>
<ul>
<li>
<ol start="21">
<li>解释器模式</li>
</ol>
</li>
</ul>
<p>定义一门语言，定义它的文法的一种表示，并定义一个解释器，该解释器使用该表示来解释语言中的句子。</p>
<ul>
<li>
<ol start="22">
<li>享元模式</li>
</ol>
</li>
</ul>
<p>使用共享对象可有效地支持大量的细粒度的对象。</p>
<ul>
<li>
<ol start="23">
<li>桥梁模式</li>
</ol>
</li>
</ul>
<p>将抽象和实现解耦，使得两者都可以独立地变化。</p>
