<p>今天，遇到了一个静态图片需要单独拿出来做 CDN 加速的需求，之前的方式是通过 API 接口请求图片路径后再进行渲染。</p>
<p>以前也听说过把图片等文件单独上传的 CDN 服务器，但是并没有实践过，正好了解下。</p>
<p>如下图，在一个网络请求开始到完成后，看到各个资源加载的耗时，这里是使用 CDN 后的耗时，非常之快。</p>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/8fc4f3845fc746c8963311bf97557952~tplv-k3u1fbpfcp-watermark.image" alt="网络请求耗时详情" loading="lazy" referrerpolicy="no-referrer">​</p>
<p>网络请求耗时详情，Chrome F12 查看</p>

<p>这里，主要想讲讲以下三点：</p>
<blockquote>
<ol>
<li>什么是动静分离？</li>
<li>动静分离有什么用？</li>
<li>怎么用？</li>
</ol>
</blockquote>
<hr>
<h2 data-id="heading-0">一，什么是动静分离</h2>
<blockquote>
<p>一句话就是：静态不变的资源获取与动态资源分离开。</p>
<p>举例：</p>
<ul>
<li>图片资源无需使用接口，直接访问路径（如存储在七牛云等图片服务器），可以使用 CDN 加速。</li>
<li>HTML，CSS，JS 等文件亦可分离出系统单独访问。</li>
</ul>
</blockquote>
<hr>
<h2 data-id="heading-1"> 二，动静分离有什么用 </h2>
<blockquote>
<ol>
<li>提高服务的访问性能和可维护性</li>
<li>良好的用户体验，说白了，访问就是快，秒开！（咳咳，某些时候太快并不好）</li>
</ol>
</blockquote>
<hr>
<h2 data-id="heading-2"> 三，怎么用</h2>
<blockquote>
<ol>
<li>将静态资源部署在 nginx 上，后台项目部署到应用服务器上，根据一定规则去请求 nginx 上的资源。</li>
<li>直接将静态资源全部存放在CDN服务器上，包括 JS，CSS，HTML，图片 等等。（推荐）</li>
<li>比如 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fmars-assets.qnssl.com%2FFspZJF8xgIz24cG8xWeq8Sq3yIKB" target="_blank" rel="nofollow noopener noreferrer" title="https://mars-assets.qnssl.com/FspZJF8xgIz24cG8xWeq8Sq3yIKB" ref="nofollow noopener noreferrer">mars-assets.qnssl.com/FspZJF8xgIz…</a> 这个图片就使用了 CDN 加速。</li>
</ol>
</blockquote>
<hr>
<p>注意事项：</p>
<ul>
<li>
<p>我们使用的是原生接入 RN 来进行应用的开发，图片地址固定后，后端替换图片后图片不会立即刷新。</p>
<ul>
<li>原因：图片有 Http 缓存，可以参考 【 <a href="https://link.juejin.cn/?target=https%3A%2F%2Fwww.jianshu.com%2Fp%2Ff080181021cb" target="_blank" rel="nofollow noopener noreferrer" title="https://www.jianshu.com/p/f080181021cb" ref="nofollow noopener noreferrer">HTTP 缓存机制详解</a>】 。</li>
<li>解决方案：在请求的图片 url 后面增加时间戳即可解决，如：<a href="https://link.juejin.cn/?target=http%3A%2F%2Fxxxx%2Fxxx%2Fxxx.png%3Ft%3D" target="_blank" rel="nofollow noopener noreferrer" title="http://xxxx/xxx/xxx.png?t=" ref="nofollow noopener noreferrer">http://xxxx/xxx/xxx.png?t=</a> timestrap</li>
</ul>
</li>
<li>
<p>可能会增加项目开发成本，如各个图片的地址需要出一个详细的文档等。 </p>
</li>
</ul>
