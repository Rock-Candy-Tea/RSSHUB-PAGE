<h3 data-id="heading-4">失败之后更改请求体</h3>
<p>不知道大家在使用的过程中，有没有遇见这样的情况：</p>
<ul>
<li><strong>场景一</strong>：通常情况下我们的默认域名都是同一个，但是有的时候也会出现这样的场景，常规操作之下使用默认域名，如果失败则换另外的备用域名</li>
<li><strong>场景二</strong>：网络请求第一次失败之后，更换另外一套网络接口</li>
</ul>
<pre><code class="copyable">KJNetworkThiefPlugin * plugin = [[KJNetworkThiefPlugin alloc] init];
// 该字段必须开启，才会再次调用网络请求

plugin.kChangeRequest = ^(KJNetworkingRequest * _Nonnull request) &#123;
    
    // 场景一，失败之后更换ip
    if (request.opportunity == KJNetworkingRequestOpportunityFailure) &#123;
