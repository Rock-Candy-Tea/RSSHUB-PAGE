h3&#123;font-size:18px&#125;&#125;</style><p>按照国家关于发展大数据、建设数字中国和智慧社会、提高社会治理智能化水平的部署要求，各地分别组建了大数据管理局，负责推进政府数字化转型和大数据资源管理等工作。通过组建这一机构，进一步加强互联网与政务服务的深度融合，统筹管理公共数据资源和电子政务，推进政府信息资源整合利用，打破信息孤岛、实现数据共享，进一步助推“<strong>最多跑一次</strong>”改革和政府数字化转型。</p>
<p>毫无疑问，大数据管理局的设立，为政府数字化转型提供了有力抓手，但要真正转型成功，<strong>大数据管理局面临哪些挑战？又要如何去解决呢？</strong></p>
<p>博云BeyondCMP一体化云管理平台推出至今已帮助众多客户提供了数字化转型解决方案，大幅度提升了企业信息化管理的效率和服务水平。接下来，我们将以博云服务过的某大数据管理局“统一云管理平台”建设案例来做分析，详细介绍“统一云管理平台”如何赋能政府数字化转型。</p>
<p><strong>01</strong></p>
<p><strong>项目挑战</strong></p>
<p>首先我们分析一下当前大数据管理局相关人员的工作职责，主要涉及以下四类人员：</p>
<p><strong>• 大数据管理局管理人员</strong>：政务云网资源的审批、协调、运营决策等管理类工作；</p>
<p><strong>• 大数据管理局运维人员</strong>：政务云网资源数据监控、运维等技术类工作；</p>
<p><strong>• 委办局业务系统开发和运维人员</strong>：以业务维度申请政务云网资源，使用、监控、运维业务相关云网资源；</p>
<p><strong>• 供应商相关技术人员</strong>：政务云网软硬件运维等技术支撑类工作；</p>
<p>通过分析上述四类人员当前工作中的问题，项目挑战可主要分为以下三大类：</p>
<p><strong>• 云网资源服务申请体验差、效率低</strong></p>
<p>各委办局业务系统开发和运维人员申请政务云网资源以邮件方式申请，大数据管理局管理人员通过邮件方式审批，云网资源线下申请审批，流程和资源不能自动联动供给，缺乏自助服务，服务体验差、效率低。</p>
<p><strong>• 运营决策数据、服务考核评价体系不完善</strong></p>
<p>大数据管理局管理人员缺乏多维度运营分析数据（资源、业务、费用等维度），缺乏政务云网整体运行情况统计数据，从而无法有效进行运营决策，另外缺乏服务考核评价体系，无法有效提升服务体验和质量。</p>
<p><strong>• 云网资源运维能力分散或缺失</strong></p>
<p>大数据管理局运维人员、各委办局运维人员、供应商相关技术人员云网资源配置管理、基础监控、运维操作能力分散或缺失，缺乏统一管理，无法有效进行资源管理和数据统计，无法及时定位故障原因并及时处理。</p>
<p><strong>02</strong></p>
<p><strong>解决方案</strong></p>
<p>那么如何建设“统一云管理平台”赋能大数据管理局，解决当前面临的诸多挑战，助力政府数字化转型呢？</p>
<p>此大数据管理局“统一云管理平台”建设按照服务化、移动化，运维和运营两手抓的目标，我们认为首先要<strong>建设夯实运维和运营能力</strong>（包括：资源纳管能力、IT运维服务能力、IT基础监控能力、配置管理能力、服务运营能力、考核评价能力、统计报表能力、大屏展示能力等），然后通过<strong>三个门户</strong>（统一运维运营门户、统一服务门户、统一移动APP）将资源、运维、运营能力以自助服务的方式进行发布，赋能大数据管理局解决当前面临的挑战，有效提升服务质量、体验及效率。</p>

<p>                                                               ▲三个门户</p>
<p><strong>•</strong> 统一服务门户：各委办局统一政务云网服务申请的入口，自主管理服务门户。大数据管理局所有对外提供的服务以服务目录形式发布到统一服务门户，各委办局进行申请、审批，自主查看工单进度、资源使用情况和云网消费统计。</p>
<p><strong>•</strong> 统一移动APP：不限地点约束，大数据管理局管理人员、运维人员，各委办局可随时通过统一移动APP进行政务云网服务申请、审批和处理，充分发挥APP的灵活性，提高大数据管理局服务的体验和效率。</p>
<p><strong>•</strong> 统一运维运营门户：大数据管理局和各委办局用户/应用统一接入、统一授权，业务互通、数据互通、使用分离、互不干扰，解决大数据管理局和各委办局运维运营工作碎片化的痛点，实现运维运营工作统一入口、集中管理。</p>
<p><strong>运维运营能力</strong></p>
<ul>
<li><strong>资源纳管能力</strong></li>
</ul>
<p>跨越政务云不同数据中心和不同异构云环境进行统一资源纳管。同时，针对不同数据中心内部的基础设施资源，包括服务器、网络设备、安全设备、链路等。为异构云资源提供灵活的调度策略，为多云环境多集群环境提供统一资源池管理，包含计算、网络、存储资源池，为大数据管理局和各委办局运维人员提供云网资源管控能力。</p>
<ul>
<li><strong>IT运维服务能力</strong></li>
</ul>
<p>政务云网服务流程的统一管控，涉及云网资源申请、变更、注销，安全服务防火墙策略开通，通用事件申请、日常桌面运维服务等，规范化、标准化大数据管理局线上服务审批流程，可跟踪可溯源。</p>
<ul>
<li><strong>IT基础监控能力</strong></li>
</ul>
<p>统一监控云资源和基础设施，云资源包括云平台、集群、宿主机、虚拟机的使用情况，基础设施包括服务器硬件、操作系统、网络设备、安全设备、链路，全面监控、及时预警，保障业务系统稳定运行。</p>
<ul>
<li><strong>CMDB配置管理</strong></li>
</ul>
<p>作为数据汇聚中心，统一接入云网资源、业务、基础设施中服务器、网络设备、安全设备、链路等信息。确保云网环境中所有设备/系统及其配置信息得到有效完整的记录和维护，同时承担云网服务流程和CMDB的联动、更新，为大屏和统计报表提供数据消费，为大数据管理局和各委办局运维人员提供资源统计数据。</p>
<ul>
<li><strong>考核评价能力</strong></li>
</ul>
<p>包括考核指标设置、考核计划、考核结果展示，分三个维度考核，包括对供应商服务的考核、对各委办局云主机资源使用情况的考核、各委办局对大数据管理局服务的考核，最终系统按月自动得打分，得出绩效等级和绩效分值，督促相关方提高运维效率、服务水平、资源利用率。</p>
<ul>
<li><strong>统计报表能力</strong></li>
</ul>
<p>提供多种维度、多种类型的统计数据，涵盖业务工单、云资源统计、网资源统计、云资源消费、网资源消费、告警统计、资产统计等，支持以饼图、柱状图、列表、环图、TOP等多种展示形式，以报告形式对资源和运维工作全面统计，为大数据管理局管理人员运营决策提供数据支撑。</p>
<ul>
<li><strong>大屏展示能力</strong></li>
</ul>
<p>从数据中心、云资源、网资源、业务、机房、云网资源总览等多个维度进行统计及3D可视化展示，便于大数据管理局管理人员及时、准确了解政务云网整体情况，为运营决策提供数据支撑。</p>
<p><strong>03</strong></p>
<p><strong>价值收益</strong></p>
<p><strong>统一管理</strong></p>
<p>统一管理云规划、标准、制度和技术体系，采用安全可控的软硬件云产品，建立自助化、自动化和智能化的政务云网管理体系，减少分散管理资金投入。</p>
<p><strong>按需使用</strong></p>
<p>提高基础设施资源的利用率，实现按需调配、即需即用、有效共享。通过合理规划、小步快跑的方式，实现建设集约化、信息共享化、效益最大化。</p>
<p><strong>有效节流</strong></p>
<p>通过统筹政务云、政务网基础建设，有效降低分散性信息化投资，提升整体运维运营效率，减少资源、信息及数据孤岛形成，推动形成面向数字政府的强大信息化引擎。</p>
