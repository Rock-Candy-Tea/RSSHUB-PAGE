<h2 data-id="heading-1">0x1、定义与使用场景</h2>
<p>又称 <strong><code>克隆模式</code></strong>，定义如下：</p>
<blockquote>
<p>如果对象的 <strong>创建成本较大</strong>，且同一个类的不同对象间差别不大(大部分字段相同)的情况下，可以利用已有对象(原型)进行复制(或者叫拷贝、克隆)的方式来创建新的对象，以达到节省创建时间的目的。</p>
</blockquote>
<p>简单点说就是以某个对象为原型，克隆出一个一模一样的对象，常见应用场景：</p>
<ul>
<li><strong>资源优化</strong> (对象初始化需使用较多外部资源时，如IO、数据库、CPU，权限等);</li>
<li><strong>复杂依赖</strong> (依赖嵌套，A创建依赖B，B创建依赖C，一连串的对象get和set);</li>
<li><strong>同一对象多个修改者</strong> (多个对象的调用者都需要修改对象的值，克隆多个供使用其使用，保护性拷贝);</li>
<li><strong>需保存原始对象状态</strong> (如记录历史操作的场景，通过原型模式快速保存记录);</li>
<li><strong>结合工厂模式使用</strong> (定义统一的复制接口，如clone、copy，使用一个工厂来统一进行拷贝和新对象创建，然后由工厂方法提供给调用者使用)。</li>
</ul>

<h2 data-id="heading-2">0x2、深浅拷贝</h2>
<p>原型模式分为 <strong><code>浅拷贝</code></strong> 和 <strong><code>深拷贝</code></strong> 两种，<strong>浅拷贝只复制对象中基本数据类型和引用对象的内存地址</strong>，不会递归地复制引用对象，以及引用对象的引用对象...而深拷贝得到的则是一份完完整整独立的对象，所以，深拷贝更耗时和耗内存空间。</p>
<p>注：浅拷贝得到的对象和原始对象会共享部分数据，可能出现数据被修改的风险，故更适用于不可变对象。</p>
<p>很多开源框架和组件中都有原型模式的相关实现，并不一定非得从零去实现浅拷贝和深拷贝。下面以Java为例，讲解其中深浅拷贝相关的姿势~</p>
<h3 data-id="heading-3">① Java中==与equals的区别：</h3>
<ul>
<li><strong><code>==</code></strong>：<strong>基本数据类型(int,long等)</strong> → 比较存储的值是否相等，<strong>引用型变量</strong> → 比较所指向对象地址是否相等；</li>
<li><strong>equals</strong>：不能用于比较基本数据类型，没有对equals()方法进行重写，默认比较指向的对象地址，若想比较对象内容，需自行重写此方法，做相应的判断。</li>
</ul>
<h3 data-id="heading-4">② 克隆必须满足的三个条件</h3>
<ul>
<li>对任何对象x，都有 x.clone() != x，即不是同一对象；</li>
<li>对任何对象x，都有 x.clone().getClass == x.getClass()，即对象类型一致；</li>
<li>如果对象obj的equals()方法定义得当的话，obj.clone().equals(obj)应该是成立的(推荐，不强制);</li>
</ul>

<h2 data-id="heading-7">0x3、加餐：四类创建型设计模式总结</h2>
<ul>
<li>
<p><strong><code>单例模式</code></strong> → 创建全局唯一的对象；</p>
</li>
<li>
<p><strong><code>工厂模式</code></strong> → 创建不同但类型相关的对象(继承同一父类或接口)，由给点参数决定构建那种类型的对象；</p>
</li>
<li>
<p><strong><code>建造者模式</code></strong> → 创建复杂对象，通过设置不同的可选参数，定制化创建不同对象；</p>
</li>
<li>
<p><strong><code>原型模式</code></strong> → 针对创建成本较大的对象，利用已有对象进行复制的方式创建，以此达到节省创建时间的目的；</p>
</li>
</ul></div>
