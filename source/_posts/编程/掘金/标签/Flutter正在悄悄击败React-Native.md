<h2 data-id="heading-8">两者的区别</h2>
<ul>
<li>这篇文章就写得很好，两者的区别，这里搬一些过来，就不重复写了,如果要深度对比，可能又是一个长篇大论</li>
</ul>
<pre><code class="copyable">https://www.jianshu.com/p/da80214720eb
<span class="copy-code-btn">复制代码</span></code></pre>
<ul>
<li>数据的一些对比：</li>
</ul>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/b5c3a2c3258e4d8d89ba37c30fd4f293~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<ul>
<li>原理的对比（最重要）:
<ul>
<li>React Native 是一套 UI 框架，默认情况下 React Native 会在 Activity 下加载 JS 文件，然后运行在 JavaScriptCore 中解析 Bundle 文件布局，最终堆叠出一系列的原生控件进行渲染。</li>
<li>简单来说就是 通过写 JS 代码配置页面布局，然后 React Native 最终会解析渲染成原生控件，</li>
</ul>
</li>
</ul>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/cb48d963b2594853ab2b90d687411928~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<ul>
<li>Flutter 中只需平台提供一个 Surface 和一个 Canvas ，剩下的 Flutter 说：“你可以躺下了，我们来自己动”。</li>
<li>Flutter 中绝大部分的 Widget 都与平台无关， 开发者基于 Framework 开发 App ，而 Framework 运行在 Engine 之上，由 Engine 进行适配和跨平台支持。这个跨平台的支持过程，其实就是将 Flutter UI 中的 Widget “数据化” ，然后通过 Engine 上的 Skia 直接绘制到屏幕</li>
</ul>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/0550521b9ad84134940126b5fca9e2b5~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<ul>
<li>剩下的请看上面推荐的文章</li>
</ul>

<h2 data-id="heading-10">顺便说一点</h2>
<ul>
<li>不要学我，看我整篇文章<code>react-native</code>和<code>flutter</code>一会大些一会驼峰，要规范命名（主要是Mac键盘难用你们懂的～）</li>
<li>如果不规范大小写命名，就会遇到下面这个情况</li>
</ul>
<p><img src="https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/d7f7c8d5062d4c709394968df603b084~tplv-k3u1fbpfcp-watermark.image" alt loading="lazy" referrerpolicy="no-referrer"></p>
<ul>
<li>我已经遇到很多同事和粉丝问我，为什么我在本地构建没问题，一上服务器就失败，找不到文件名啊～ 那是你大小写不规范，不同操作系统的规则不一样，最终构建的结果也不一样，所以推荐使用react-native这种来命令，而不是大小写</li>
</ul>
