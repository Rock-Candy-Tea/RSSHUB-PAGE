<h1 data-id="heading-1">后台访问地址</h1>
<p>前端访问后台地址其实是需要区分本地开发和服务器部署。<br></p>

<h2 data-id="heading-2">本地开发访问：</h2>
<p>本地开发是开发环境，因为本地启动项目，开发环境可以通过接口代理的方式解决跨域相关问题，所以我们配置访问地址的时候会比较简单。<br>
<br>前端为了统一处理接口访问，会设置baseUrl，这里的bms是应用地址前缀。</p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-comment">// 路径地址：src/services/baseUrl.js</span>

<span class="hljs-comment">// eslint-disable-next-line import/no-mutable-exports</span>
<span class="hljs-keyword">let</span> baseUrl = <span class="hljs-string">'/api/bms'</span>;

<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> baseUrl;
<span class="copy-code-btn">复制代码</span></code></pre>
<p>代理配置文件</p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-comment">// 路径地址： config/config.js</span>

<span class="hljs-attr">proxy</span>: &#123;
  <span class="hljs-string">'/api/bms'</span>: &#123;
    <span class="hljs-attr">target</span>: <span class="hljs-string">'http://xxxxx:8000/'</span>, 
    <span class="hljs-attr">changeOrigin</span>: <span class="hljs-literal">true</span>,
    <span class="hljs-attr">pathRewrite</span>: &#123; <span class="hljs-string">'^/api/bms'</span>: <span class="hljs-string">'/'</span> &#125;,
  &#125;,

<h4 data-id="heading-5">本地环境和服务器部署的理解</h4>
<p>因为服务器部署走的打包文件，和本地开发大不相同，没有代理了，访问的都是静态文件。所以这时候我们的代码里只有<strong>baseUrl.js</strong>里的/api/bms了。这个是相对路径，直接访问会有问题。举个简单的例子：目前bms的开发环境部署地址是：<br><code>http://xxxxx:8000</code><br>
<br>那我们服务器部署的接口访问地址就是：<code>http://xxxxx:8000/api/bms/interface</code>。然后开发环境的swagger地址是：<code>http://xxxxx:8001/interface</code>。这个肯定对应不上啊。所以就需要前端在当前这个<strong>baseUrl.js</strong>文件内部进行。<br>
<br>那现在我们的文件是相对路径，所以需要通过处理变成对应环境的绝对路径。<br>
<br><strong>这里会有一个问题？我们现在有三套环境，开发，测试，和生产。写哪一个呢，需要有一个变量来确定在不同的环境设置正确的后台访问地址接口。</strong><br></p>
<p><a name="user-content-AGkao" href="https://juejin.cn/post/undefined"></a></p>
<h4 data-id="heading-6">可以用来判断环境的变量BUILD_TYPE</h4>
<p>这里涉及到一个前端打包环境变量的概念：BUILD_TYPE。至于这个概念可以先不管，当时我们怎么理解呢。理解他就可以当成是一个变量，可以被设置值，可以被访问到。我们在baseUrl.js里都可以访问。我们可以通过判断这个变量的类型来决定baseUrl设置成哪个环境的api访问绝对地址。</p>
<pre><code class="hljs language-javascript copyable" lang="javascript"><span class="hljs-keyword">let</span> baseUrl = <span class="hljs-string">'/api/bms'</span>;

<span class="hljs-comment">// eslint-disable-next-line no-undef</span>
<span class="hljs-keyword">if</span> (BUILD_TYPE === <span class="hljs-string">'build'</span>) &#123;
  baseUrl = <span class="hljs-string">'http://xxxxxx:8001/api/bms'</span>;
&#125;

<span class="hljs-keyword">if</span> (BUILD_TYPE === <span class="hljs-string">'qa'</span>) &#123;
  baseUrl = <span class="hljs-string">'http://yyyyy:8001/api/bms'</span>;
&#125;

<span class="hljs-keyword">if</span> (BUILD_TYPE === <span class="hljs-string">'release'</span>) &#123;
  baseUrl = <span class="hljs-string">'http://zzzzzz:8001/api/bms'</span>;
&#125;

<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> baseUrl;

<span class="copy-code-btn">复制代码</span></code></pre>

<h3 data-id="heading-9">后端需要注意的点：</h3>
<p>以前碰到过的是，前期开发好好的，等到部署环境的时候就会出现跨域的问题，然后找后端的同学，就会问，为啥啊，不是一直用的好好的嘛。</p>
<p><strong>回答</strong>：因为前期开发走的本地环境，本地做了接口代理处理，项目结构自带的，所以不会有问题。一旦走了服务器部署，就需要后端接口做跨域处理了。因为处理跨域有很多种办法，<strong>前端代理</strong>或者<strong>后端做跨域</strong>处理。所以有时间前端还是后端的同学都需要了解一下同源策略和跨域处理。</p>

<h1 data-id="heading-10">nginx代理处理后端服务地址问题</h1>
<p>经过在个别项目当中发现，部署环境存在有的时候打包的接口地址没有按照预期的访问，可能我打的是<strong>qa环境，但是部署出来的地址还是开发环境的地址</strong>，具体原因应该是cicd的环境变量问题。后来我们的项目就是用nginx统一转发 <br></p>
<p><strong>前端build的时候不设置baseUrl</strong>，统一相对路径/api/，通过服务器一次性设置代理转发。比如：/api/统一访问接口，其他的访问前端资源。</p>
