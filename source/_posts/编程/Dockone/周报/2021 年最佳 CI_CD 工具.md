<br>【编者的话】为持续集成（CI）和持续交付（CD）选择最好的工具是一件相当有挑战的事。以下列出了 30 多种工具和平台，可帮助你做出充分了解。<br>
<h3>持续集成（CI）和持续交付（CD）简介</h3>CD 的意思是持续交付，而 CI 的意思是持续集成。<br>
<br>CI 和 CD 都是 DevOps 的一部分，在这种情况中，将开发和运营流程结合到单一的协作工作流中，以确保两个团队在同一页面下。<br>
<br>CI 代表什么？持续集成是指应用程序的新建的部分都集成到现有的部分中，经过仔细测试，然后进入下一阶段。CD 持续交付是开发人员逐步构建项目，每个阶段都可以随时交付给客户。<br>
<h3>2021 年可供选择的 CI/CD 工具</h3>下面列出了 30 多种 CI CD 工具，这些工具可用于 DevOps 流程中的通用和特定任务。<br>
<h4>1. Jenkins</h4>基于 Java 基础，免费开源软件市场上最受欢迎的 CI/CD 工具之一。<br>
<br>Jenkins 将持续交付和集成工具与实时测试和报告相结合。<br>
<h4>2. TeamCity</h4>TeamCity 是 JetBrains 的产品。与 Jenkins 一样，也是 Java 的开源软件。TeamCity 与 Docker 和 Kubernetes 完美集成。<br>
<h4>3. CircleCI</h4>CircleCI 简化了 DevOps 自动化和软件部署流程的最佳 CI/CD 工具。借助CircleCI，工作流可以在多个容器之间拆分、共享和重用。<br>
<h4>4. Travis CI</h4>Travis CI 增强了安全级别，成为企业发展首选。该平台与 Lambda Test 完美集成，简化了跨不同浏览器、平台和环境的 DevOps 测试过程。<br>
<h4>5. Bamboo</h4>Bamboo 被认为是最流行的持续集成工具之一。Bamboo 是由 Atlassian 开发。该平台允许启动和测试 100 个远程构建代理，并基于云或自托管的解决方案。<br>
<h4>6. GOCD</h4>具有突出特点的持续集成平台：GOCD允许 DevOps 专家创建价值流图，以更好地查看和映射DevOps 管道。<br>
<h4>7. CodeShip</h4>CodeShip 的显着特点是可以直接从 GitHub 进行测试、构建和部署。<br>
<h4>8. GitLab CI</h4>如果开发团队在 GitLab 存储库中托管代码，则使用此工具进行 DevOps 工程是一个明智的选择。GitLab CI 允许 DevOps 开发人员从单个仪表板审查代码、部署、集成和交付。<br>
<h4>9. Jenkins X</h4>Jenkins X 是用于 DevOps 部署使用 Kubernetes 和 Docker 构建的应用程序的 CI/CD 工具之一。该平台非常适合在将不同环境合并到存储库之前对其进行智能管理。<br>
<h4>10. Shippable</h4>Shippable 的理念是使软件发布更具可预测性并避免Bug和错误。Shippable 结合了广泛的测试、自动化和部署工具。<br>
<h4>11. Buildkite</h4>如果要在自己的基础架构上开发应用程序，Buildkite 非常适合此特定功能。Buildkite 构建快速、安全且有良好的测试管道，以简化持续集成。<br>
<h4>12. Concourse CI</h4>非常适合在 DevOps 流程中使用 Vagrant 的应用程序的 CI/CD 工具之一。它侧重于非常直观的 UI，以帮助 DevOps 顾问构建可视化的管道。<br>
<h4>13. Codefresh</h4>此工具基于 Kubernetes 构建，可以使用在 Azure 和 Amazon Web Services（AWS）上进行部署。内置库加快了添加和删除开发人员需要或不需要的功能的过程。<br>
<h4>14. Buddy</h4>Buddy 的开发人员表示，这是获得最快 CI/CD 的平台。在Buddy的帮助下，平均部署时间减少到 12 秒，而 DevOps 团队采用 CI/CD 的速度可以提高 87%。<br>
<h4>15. Buildbot</h4>一个用 Python 构建的持续交付测试框架。Buildbot 还具有一组用于发布管理、应用程序部署和自动化的功能。<br>
<h4>16. Semaphore</h4>Semaphore 是特别适用于 iOS 应用程序的 CI/CD 工具。Semaphore 允许为 Windows 和 Linux 构建的解决方案运行 DevOps 流程，此外Semaphore 还支持许多用于创建 Android 应用程序的编程语言。<br>
<h4>17. Wercker</h4>Oracle 创建的基于 Docker 的 CD 工具。Wercker 可以免费使用，也可以在私有云中付费使用。<br>
<h4>18. Integrity</h4>具有令人印象深刻的标准功能集的持续集成服务器。<br>
<h4>19. Weave Flux</h4>该平台是在 CD 管道末端使用的持续交付工具。它帮助开发人员确保配置更改已上传到集群。<br>
<h4>20. NeverCode</h4>一个完全自动化的工具，可在不同环境中部署测试场景，生成报告，并在每次提交后构建代码，无需人工干预。<br>
<h4>21. AutoRabit</h4>如果使用 SalesForce 开发应用程序，可以考虑使用此工具，因为 AutoRabit 与 SalesForce 完美配合。SalesForce 平台允许开发人员改进他们的代码、管理发布和提高数据安全性。<br>
<h4>22. CruiseControl</h4>一个适用于自定义解决方案的开源和基于 Java 的持续集成框架。它带有数 10 个插件，开发人员可以根据自己的需要进行定制。<br>
<h4>23. Bitrise</h4>如果未来的解决方案需要一流的用户体验，除了其他 CI/CD 工具外，可以关注此平台。它主要支持用于创建移动应用程序的编程语言，并提供一种制作具有竞争力的解决方案的方法。<br>
<h4>24. Drone CI</h4>CICD 工具列表中又一个值得关注的平台。Drone CI 的显着特点是可以创建分布式管道。<br>
<h4>25. UrbanCode</h4>此解决方案适用于企业应用程序。持续集成和持续交付的每个功能都是根据企业解决方案的需求量身定制的，增强的数据安全性就是其中之一。<br>
<h4>26. Strider</h4>Strider 是用于简化 CI/CD 任务的完美可定制且有效的解决方案。它与 Heroku 集成（云服务提供商）并支持 Docker。<br>
<h4>27. FinalBuilder</h4>一种持续交付工具，可以最大化的自动化构建。它带有 600 多个可用于测试的预开发脚本和操作。<br>
<h4>28. GitHub Actions</h4>此工具允许直接在 GitHub 存储库中创建 CI/CD 工作流，这样就可以在 GitHub 上管理所有任务。<br>
<h4>29. Azure DevOps</h4>Azure DevOps是微软开发的一体化解决方案。有了它，你可以使用内置的 Azure Pipelines、Visual Studio Team Services（VSTS）和软件交付服务。<br>
<h4>30. Terraform</h4>另一个著名的基础设施管理平台。考虑将 Terraform 添加到 CircleCI 以在同一管道中部署您的基础设施。<br>
<h4>31. Tekton</h4>借助这个直观的工具，您可以使用简单的代码通过并行执行来表达复杂的工作流。Tekton是与 Kubernetes 一起使用的最佳工具之一。<br>
<h4>32. Big Eval</h4>在一个面向数据的项目，Big Eval 可以满足 DevOps 需求的解决方案。该工具在回归测试和基于元数据的测试的帮助下检查项目开发的每个阶段的数据质量，从而创建合格的 DataOps 流程。<br>
