A system is said to be observable if, for any possible evolution of state and control vectors, the current state can be estimated using only the information from outputs (physically, this generally corresponds to information obtained by sensors). In other words, one can determine the behavior of the entire system from the system's outputs. On the other hand, if the system is not observable, there are state trajectories that are not distinguishable by only measuring the outputs.</p><p>简单表述为，可观测性是一种方法，通过系统的外部输出推导出系统内部的状态。</p><p>下图简化了系统的组成和系统间的交互：</p><p><span class="img-wrap"><img class="lazy" src="https://segmentfault.com/img/remote/1460000039762468" alt="1.png" title="1.png" referrerpolicy="no-referrer"></span></p><p>从上述交互图可了解到，系统的交互行为有如下几种形态：</p><ul><li><p>系统内部</p><ul><li>组件功能闭环，不与其他组件或系统交互</li><li>组件之间交互</li></ul></li><li><p>系统之间</p><ul><li>系统和系统之间进行交互</li></ul></li></ul><p>这样，通过如下两种形态的信息，就可以通过系统的外部输出了解到系统的内部状态：</p><ul><li>组件闭环的信息</li><li>组件间或系统间流动的信息</li></ul><h2>可观测性的问题域是什么？</h2><p>可观测性的核心在于 <strong>通过观测数据、满足不同人群、对于系统状态的理解需求</strong>，这里先抽象观测数据的生命周期，有如下图示：</p><p><span class="img-wrap"><img class="lazy" src="https://segmentfault.com/img/remote/1460000039762459" alt="2.png" title="2.png" referrerpolicy="no-referrer"></span></p><p>观测数据通过 App 生成，经过中间处理环节后进行存储，然后提供查询服务。</p><p>观测数据服务于不同类型的人群，如产品的用户、业务、研发、SRE，不同的人群通过不同的形态来使用这些数据，包括 SLA / SLO / SLI / Alert 等。</p><p>根据可观测数据的生命周期，可粗略总结可观测性的问题域：</p><ul><li><p>生成端</p><ul><li>观测数据的数据模型</li><li>观测数据的生成</li><li>观测数据的导出</li></ul></li><li><p>处理端</p><ul><li>观测数据的采集</li><li>观测数据的处理</li><li>观测数据的导出</li></ul></li><li><p>存储端</p><ul><li>观测数据的存储</li><li>观测数据的查询</li><li>观测数据的使用</li></ul></li><li><p>使用端</p><ul><li>观测数据的消费</li></ul></li></ul><h2>软件开发生命周期中，可观测性的服务目标是什么？</h2><p>从项目整体视角来看软件开发的生命周期，有如下的流程：</p><p><span
