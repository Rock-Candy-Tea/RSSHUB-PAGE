<div class="text-center"><iframe frameborder="no" border="0" marginwidth="0" marginheight="0" width="290" height="450" src="https://indienova.com/widget/indie-game-news/devlog-mino-1/"></iframe></div><h4>开发进度 4</h4><p>鉴于之前的开发日志太啰嗦了，这次打算精（tuo）简（lan）一点</p><p><strong>开发进度</strong></p><ul><li>设定了 Demo 共 28 题，做好了 15 题，进度 53%</li><li>做了按键提示</li><li>优化了主菜单界面</li><li>顺便实现了多语言，支持了简中、繁中和英文</li><li>Steam 商店页面完成度 60%</li></ul><p><strong>机制分享</strong></p><p>米诺拼图，米诺拼图，拼图在哪？这次就来看看拼图机制</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2022/03/u-4065508132388P83.gif" referrerpolicy="no-referrer"></p><p>这个机制一目了然，不用过多说明了。有意思的是，这个机制虽然看起来是个全新的机制，但其实从逻辑上，它完全是之前机制的组合，题目格子颜色机制，积木可以由多个颜色组成，主要由这 2 个机制组合而成，在代码上完全不用实现新的东西。</p><p>这种感觉就像《见证者》里早期的点机制 ，最开始玩到的点机制谜题，题面都是零散的几个点，后来出现了题面全是点的谜题，虽然这种谜题本质上也是点机制，但玩家的感受完全不同，就像是完全不同的机制一样。如下图</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2022/03/u-4065508131634jJR.png" referrerpolicy="no-referrer"></p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2022/03/u-4065508131643lJF.png" referrerpolicy="no-referrer"></p><p><strong>说说废话</strong></p><p>最近突然觉得，或许我发这些开发日志，也许不完全是为了分享进度、分享游戏机制，或许只是想满足一下自己无处发泄的表达欲，把自己的一些关于独立游戏的感受写下来，算是给自己内心的躁动不安一些安抚吧。（好家伙，把开发日志当成树洞在写了）</p><p>最近还在业余的业余时间补谜题游戏，已经白金几款稍微简单的了，现在被《Snakebird》卡得死死的，无数次想过放弃，但想通谜题解法的那一刻，那种无比的成就感还在诱惑着我，我突然发现这就和在魂类游戏里，自己开荒击杀 Boss 的感觉是类似的吧。从这点看来，硬核的谜题游戏，也可以说是解谜游戏里的黑魂了。（顺带一提，现在卡在第 33 关，传送门机制真的很卡我思维(lll￢ω￢)）</p><p>如果有时间，我也想分享下这些谜题游戏里的机制，解题思路，以及从设计者角度聊聊它们的谜题设计上的各种优缺点。（醒醒，你没有时间，这篇日志也够长（shui）了）。</p><h4>开发进度 5</h4><p><strong>1. 后处理</strong></p><p>虽然本人美术不行，但美颜还行，借助后处理技术提升了一下画面的质感，看着还不错哦。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2022/04/u-4065500262414Fcu.png" referrerpolicy="no-referrer"></p><p><strong>2. 音乐音效</strong></p><p>添加了背景音乐、操作的音效，资源基本来源于之前购买的音效包，以及<a href="https://www.zapsplat.com/" rel="nofollow" target="_blank">zapsplat</a>上的免费资源。</p><p><strong>3. Steam 页面</strong></p><p>终于做好了 Steam 页面，现在 G 胖审核中，等审核通过了后续就能把链接贴出来收集愿望单以及申请测试了，YEAH！</p><p>做了一个大概 1 分半的预告片，全程 0 基础，通过 Unity 的 Recorder 插件录制游戏内视频，然后通过免费的 Hitfilm Express 来剪辑，比预想的要简单，最终效果我还挺满意的，毕竟是第一次。</p><p>视频等 Steam 页面好了之后再发出来吧。</p><p><strong>4. UI 系统</strong></p><p>完成了游戏的 UI 系统，制作了多语言选择、存档管理、音乐音效控制、全屏/窗口化、分辨率等常见的设置功能。由于 Demo 还不支持手柄，所以打算把按键映射功能延后到正式版了。</p><p>PC 平台游戏的 UI 系统和我工作中接触的手游 UI 有很大区别，主要难点在于 PC 平台需要支持手柄操作以及要运行时切换多语言，这要求 UI 有导航功能，以及控件需要根据多语言自适应大小和排版。</p><p>这里我主要参考了同类游戏《SOKOBOND》，UI 很简单，但扩展性和兼容性都很好。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2022/04/u-40655002108764Su.gif" referrerpolicy="no-referrer"></p><p><strong>5. 通关了 Snakebird（喂，这也算进度吗？）</strong></p><p>这算是这段时间玩过最好的一款解谜游戏了，谜题的质量非常高！打算再找时间总结归纳下它的谜题设计技巧，如果可能的话，也想写成文章发布出来。</p><p><br></p><p><strong>心得</strong></p><p>Steam 页面的耗时比我想得要多，主要有几点：</p><p>因为动态 IP 的问题，挂 VPN 连 Steamwork 经常被踢，后面要考虑换个支持静态 IP 的 VPN 了，当然不挂 VPN 也可以连，就是不稳定而已，特别在上传各种图片的时候。</p><p>制作预告片花了点时间，我对预告片的质量要求不高，但奈何自己是完美主义者，最后还是在一些细节上花了不少时间。最开始还打算用 Win10 自带的视频编辑器的，但功能实在太少了，做下视频相册还行，做预告片就太难为它了。最后选了<a href="https://fxhome.com/product/hitfilm-express" rel="nofollow" target="_blank">Hitfilm Express</a>，功能已经很完善了，做预告片完全够用，唯一的缺陷是我把视频片段加速后，其对应音频的 pitch 变高，导致声音变得很刺耳，我在网上找了一圈没找到方案，最后只能不加速了，所以预告片的节奏有点慢。</p><p>独立游戏开发者都用哪些免费软件？我参考的是这份：<a href="https://www.youtube.com/watch?v=ung_U17kxDI" rel="nofollow" target="_blank">Free Gamedev Tools I Use</a></p><p>Steam 页面真的要早点做，我感觉我太拖延了，导致每次在网上宣传时都无法把链接放出来收集愿望单，下次开发时，我应该会把制作 Steam 页面提前到早期开发中，先做一个初版的，随着游戏基本成型慢慢完善，然后等待时刻时机就发布出来。</p><p><br></p><p><strong>废话</strong></p><p>这段时间做的事情基本都不是直接和游戏玩法相关的，而是游戏周边的事情，这些工作往往是我不擅长的，枯燥的，需要持续投入时间的，这也是我之前做个人项目时，特别容易产生拖延的工作，这次趁着疫情，多了点个人时间，坚持把这些工作给完成了，我给自己点个赞，哈哈。</p><h4>Steam 页面</h4><p>Steam 页面终于上线了，有兴趣的可以去加个愿望单，对我帮助很大，谢谢了。</p><p>地址：<a href="https://store.steampowered.com/app/1798240/_/" rel="nofollow" target="_blank">https://store.steampowered.com/app/1798240/_/</a></p><p class="hidden-xs"><iframe src="https://store.steampowered.com/widget/1798240/" frameborder="0" style="width: 100%;
