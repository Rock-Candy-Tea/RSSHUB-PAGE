<h4>1. AVG 创新的思考</h4><p><strong>H：</strong>为什么时隔一个多月才发布新的开发日志，就是因为这段时间一直在设计和制作 UI。其实这个 topic 可以谈的东西很多，这次还是谈 GUI 方向的事情。</p><p>一般的 GALGAME 或者乙女 game 等视觉小说有既定的一套游戏系统，系统层面的创新性不是很强，也有非常成熟的引擎。但是如果我们要往这个方向做的话，拼不了美术资源量（都是钱）和剧本体量。如果用这种专门的引擎，可能只能在美术风格化或者布局上下文章。</p><p>因此整个赛博失物招领处（以下简称 LOST）的游戏线路也并不会朝着传统视觉小说的方向去做。而是选择了在系统上进行创新。那么随之而生的，便是非常规视觉小说化的 UI。</p><p>并不是说仅仅将布局和风格改变了，而是从功能层级上，根据策划系统需求进行改变。下面会详细地解释目前 UI 的设计过程和一些相关的思考。</p><h4>2. 引擎的选择</h4><p>游戏引擎一开始就选择为并不是那么传统适合 AVG/视觉小说的 Unity，是考虑到系统的可拓展性的。同时，也希望能引入一些其他纯 2D 引擎所不能具备的效果，例如镜头角度的转移（还在尝试中）。同时也希望拥有一套更 RPG 化的任务系统。</p><p><img class="fr-fin fr-dib" alt="unity也有成熟的avg系统插件，可惜DIY程度不符合我们的需求。" src="https://hive.indienova.com/farm/blog/2021/07/u-379628607055081Q.png" title="unity也有成熟的avg系统插件，可惜DIY程度不符合我们的需求。" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>unity 也有成熟的 avg 系统插件，可惜 DIY 程度不符合我们的需求。</small></div><p>LOST 规划中的打牌模式（附加玩法）也是考量中的重要因素，unity 可以更便于将来的 TCG 开发。</p><h4>3. 殊途同归：沉浸式 GUI 的两个极端</h4><p>在视觉美学层面，我们希望提供一种沉浸式的体验。沉浸式 UI 有两种极端，最大化和最小化。</p><p><br></p><p><strong>1. Minimization</strong></p><p>最小化或许是一般意义理解上的“沉浸式”，以育碧为代表的 3D 动作游戏，因场景和人物都精美且写实，UI 在屏幕上占比非常小，确保玩家的注意力都在画面和操作上。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070603RtV.png" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>刺客信条：奥德赛。选项都很小，整个画面除了画面就是画面。</small></div><p>一些拥有精美 CG 背景和人物立绘甚至动态立绘的 AVG 或许可以这样做，但是我们的项目不符合这个要求。</p><p><br></p><p><strong>2. Maximization</strong></p><p>UI 本身也是“一幅画”，重设计，重世界观元素，甚至叙事也可以在上面体现出来。《辐射 4》在动力盔甲里的显示界面让人感觉到自己是在盔甲当中，符合第一人称视角的操作；而当切换到 BB 小子的界面上，又有着浓烈的美式冷战宣传漫画风格，符合国情也符合核战争的题材。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070641L1j.jpg" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>辐射 4。很有特色的交互界面。</small></div><p>虽然有不少批评的声音表示辐射 4 这 UI 并不用户友好——不过或许对于一个真实的世界的操作版面来讲，学习成本确实有点高，做进游戏里可以说是行为艺术吧。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-37962860706674uv.jpg" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>波音 747 的驾驶舱。</small></div><p>因为第一点最小化，是绝对无法适用于 LOST 的（没有大量精致的美术），所以只能往最大化的方向来思考，接下来需要从世界观故事背景、受众和人物风格等方面来推论如何去呈现我们的 UI。</p><h4>4. LOST 世界观介绍，题材与故事对 GUI 设计的影响</h4><p>LOST 是赛博朋克题材的故事。</p><p>主角因为过去的一次黑客事件引发的全球性危机中不幸被植入脑部的芯片击昏成为植物人，苏醒后不得不成为赛博社畜，用仅存的意识工作来换取医疗费治愈自己身体。他在作为失物招领处的业务员工作的日子里，认识了许多有趣的来自各个阶层的人，并且逐渐发现这个世界和他昏睡过去前发生了天翻地覆的改变……</p><p><strong>H：</strong>我因此联想到，首先我们是一个主角第一人称的游戏，所以可以借鉴辐射 4 的思路，我们的意识被困在一个机器里，一个有界面的电脑系统，是我们的操作版面。同时所有的界面都可以当成是某种操作系统。其次是因为题材，这个系统应该是比较科幻电脑风的。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070702cpq.jpeg" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>网络来源参考图。类似监控器风格的界面，色调比较单一。</small></div><p>因此采用了黑白+单色调的电脑风格。</p><p>不过请放心，大部分时间因为都是在阅读角色对话，所以不存在复杂如辐射四的操作。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070794KDe.png" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>LOST。主界面示意。</small></div><h4>5. LOST 界面设计的其他思路，作者画风，成本</h4><p><strong>H：</strong>虽然谈到了赛博朋克，大家想到的一定是酷炫的霓虹灯，还有光污染，Glitch Art。但是由于我的人物画风是比较有个性（残念）的漫画风，所以反而不好契合这样高大上的风格。</p><p>PC-98 和像素风虽然也比较有个性，但是制作起来却一点都不简单，可能比新世代的赛博风还要难做。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070813iEF.png" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>VA-11 HALL-A。著名的赛博题材调酒游戏。</small></div><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070763OZC.gif" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>Virgin Angel。有机械计算机也有美少女。</small></div><p><strong>H：</strong>于是我又思考到，很多优秀的赛博朋克题材作品也是传统黑白漫画，比如大家都熟悉的《攻壳机动队》，最早也是画在黑白稿纸上的。因此漫画风并不是不能表现这个题材，倒不如说“赛博朋克题材=霓虹灯光污染 GlitchArt 的视觉美术”，其实完全是一种由于影视化特效发展后而带来的偏见。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070838I8L.jpg" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>攻壳机动队封面。比起光污染，更强调机械改造人和电脑母版的概念。</small></div><p>因此我们最后决定要找到一种能适应漫画风，但又能体现出科幻、赛博朋克属性的 UI 风格。</p><p>再考虑到制作成本和难度，最终还是选择了目前的“极简黑白计算机科幻界面风”。</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070863obf.png" referrerpolicy="no-referrer"></p><div class="text-center image-caption"><small>LOST 的系统版面一览。</small></div><h4>6. UX Design，易用性与叙事间的取舍</h4><p>在设计过程中还遇到了许多问题，比如什么时候刷新任务，什么时候让主角有机会进行归还物品的操作。</p><p>这其中归还物品作为作品主题、玩法的核心，是我们思考的基准。</p><p><strong>H：</strong>首先我想到的是，作品希望的核心体验是一种“社畜上班还债”的情景。假如还物品的操作相当于一般工作上的业务实操，而和 NPC 对话相当于接待客户，和系统 AI 对话则是和领导开会，那么实际上你能做业务实操的时机在一天当中是相当有限的，大部分时间都用在看客户心情和与领导开会上面，可能只有下班之前少数时间或者领导催你的时候才能进行。</p><p><strong>L：</strong>也是程序写的时候方便程度的问题，还有系统设计易用性的问题。</p><p><strong>H：</strong>这样也方便剧本把控节奏。因为在看对话的中途如果玩家可以随便点击归还物品做任务，而无视角色对话的节奏的话，会破坏很多体验。因为剧情对话是核心体验，不能破坏它。</p><p>因此最后归还物品并完成任务的界面，从一开始的可以随时用界面上的按钮打开，变成了只有特殊的时机才能自动弹出。</p><p>UX 要说起来，优化需要很多次迭代，将来也会有。在大家没有 demo 可以玩的时期，现在就不详细地讲解过程了。</p><p><br></p><p>感兴趣的话关注我们的新浪微博吧！@决斗者俱乐部</p><p><img class="fr-fin fr-dib" src="https://hive.indienova.com/farm/blog/2021/07/u-3796286070892cuE.jpg" width="162" referrerpolicy="no-referrer"></p><p>这次我们会参加 ChinaJoy 的 Indie 展区，摊位号是 IG012，展馆 W5。欢迎来玩~</p>
