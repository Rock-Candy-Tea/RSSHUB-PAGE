<div>   
<p style="text-indent:2em;">
一些 Windows 11 用户可能会需要推迟系统更新，因为最新的预览更新正导致一些使用 .NET Framework 3.5 的应用程序无法运行。值得庆幸的是，有一些方法可以解决这个问题，但目前最好的办法是暂时避免使用 KB5012643 预览版。
</p>
<p align="center">
<img src="https://img.3dmgame.com/uploads/images/news/20220505/1651745186_238831.png" alt="Win11预览更新导致部分微软商店应用无法运行" referrerpolicy="no-referrer">
</p>
<p style="text-indent:2em;">
微软在官方 Windows 11 文档的状态更新中确认了这个问题。它指出该问题仅限于运行安装了 KB5012643 更新的 Windows 11 版本 21H2 的用户，这意味着使用 .NET Framework 3.5 的应用程序可能无法正常工作，甚至直接无法打开。
</p>
<p style="text-indent:2em;">
这似乎会影响框架中的特定组件，因此并非所有应用都会受到影响。看起来在 3.5 框架上使用 Windows Communication 和 Windows Workflow Foundation 等可选功能的应用程序最有可能遇到问题。
</p>
<p style="text-indent:2em;">
这可能包括许多主机应用程序和 Windows 服务等。 Microsoft 尚未提供受影响的特定应用程序列表，因此很可能会影响大量的程序。
</p>
<p align="center">
<img src="https://img.3dmgame.com/uploads/images/news/20220505/1651745195_176307.png" alt="Win11预览更新导致部分微软商店应用无法运行" referrerpolicy="no-referrer">
</p>
<p style="text-indent:2em;">
如果您遇到任何问题，微软建议您卸载更新。您可以通过进入 PC 系统设置中的 Windows 更新设置。您应该能够查看更新历史记录，并回滚到以前的版本。如果这不可行，另一个修复方法是在通过控制面板找到的 Windows 功能中重新启用 .NET Framework 3.5 和 Windows 通信基础。
</p>
<p style="text-indent:2em;">
目前微软正在努力解决此问题。 .NET Framework 3.5 现在将获得两年多的主要支持，随后将提供五年的扩展支持。
</p>
