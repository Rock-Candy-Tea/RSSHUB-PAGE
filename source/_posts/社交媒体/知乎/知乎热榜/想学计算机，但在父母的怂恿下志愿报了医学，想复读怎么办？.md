编程指北的回答<br><br><p><b>和题主差不多！！看到这个问题，必须回答一下了，因为我当年就是以为计算机、编程、软件都是非常高深、极客的事情，差点去复读了！！</b></p><p>只从某个同学那里听说 C 语言非常难学，编程也很难，所以让我产生了复读的想法，现在想象也真是年轻。</p><p><b>父母那一代，基本上只有师医公才是正确的出路！</b></p><p><b>难得你自己喜欢计算机，完全可以放手去学CS，但是我不建议复读，而是建议你还是去川大华西读一下，因为华西临八本来是非常好的专业，你可以试一下，万一能喜欢上呢？</b></p><p>如果不喜欢，也可以自学计算机或者转专业，川大本身转专业是比较方便的，如果你能把<b>计算机和医学</b>结合起来的话，会更加吃香，现在人工智能在疾病诊断方面也有很多团队在探索，包括川大几个实验室。</p><p>但是复读，说实话，一年的机会成本太大了，存在太多的偶然性因素，相信你也清楚，复读能保证考到现在的分数，或者更高吗？</p><p>按照你的分数本来去武大或者北邮，其实比川大就算好一点，也好不了太多，大家都是差不太多了。</p><p>相信我，影响不像你想象那么大，计算机主要靠自学的，在这方面，作为 985，川大绝对不会拖你的后退！</p><p>不管是保研还是出国，名额都是挺多的。</p><p><b>但是就像你说的，确实会一定程度上“浪费”你的分数，不过我倒是觉得「阴差阳错，未必比如愿以偿好」！！</b></p><p><b>到时候转专业会有很多计算机大类专业，比如计算机科学与技术、软件工程、大数据这些，在这说下我的一些计算机自学建议吧：</b></p><p><b>---------以下是自学计算机的建议----</b></p><p>如果你打算自学计算机，也不要怕，因为我发现身边95%以上的同学都是从来没接触过计算机编程的，大家其实都是从零学起的！</p><p>有编程基础的都是中学参加过信息学奥赛的那些人，但是这部分人每年就那么一两千人，而全国两千多所高校，计算机、软件录取的人数保守估计在 200 * 1000 = 20w 左右。</p><p>即便在 985 CS、软院里面，有基础的学生顶多 1%-5% ，大可不必担心。</p><p>希望看到这个问题的学弟学妹们不要再被「<b>不了解</b>」所劝退。</p><p>最近不是高考季嘛，很多高考完的同学还有部分大一下分流的同学问</p><blockquote> 计科，软工，大数据，物联网，网络工程这些该选什么？  </blockquote><h2><b>一、计算机大类专业如何选？</b></h2><p>首先给出我的一个排序：</p><p><b>软工 ≈ 计科 > 网络工程 > 大数据 ≈ 物联网</b></p><p>计科和软件从就业、上的课程来说都是非常接近的。</p><p>非要说差别的话，大概是计科考公或者走选调选择会更多一些，软件工程的学费会贵一倍左右（我们学校软件接近1w，计科就和大部分专业一样的学费 4k-6k 的样子）</p><p>从上的课程来说，软件会多出一些软件设计、软件工程、软件建模这样的课程，计科则会多一些偏硬件设计的课程，比如 微机原理与系统。</p><p>但是无论计科还是软件、物联网这些专业，在课程设置上基本上跳不出这个框架（大同小异）：</p><figure data-size="normal"><img src="https://pic3.zhimg.com/v2-45ba67261d003f57c29e3055eef53a00_1440w.jpg" data-rawwidth="693" data-rawheight="468" data-size="normal" data-caption data-default-watermark-src="https://pic4.zhimg.com/v2-947840b823e078e2ae0add36a752046a_720w.jpg" class="origin_image zh-lightbox-thumb" data-original="https://pic3.zhimg.com/v2-45ba67261d003f57c29e3055eef53a00_r.jpg" referrerpolicy="no-referrer"></figure><p>不管咋说，这些学科出来的归属大概率都是互联网公司码农，我学计科一样可以去做大数据、物联网，学物联网一样也可以去做 Java。</p><p>这里的 ＞ 不是说网络工程就比大数据和物联网好，而是更适合大部分人，除非你有特别爱好，否则不要去选物联网、网络安全、大数据这类，虽然说上的课和计科、软甲都差不多，但是会学得更杂一点。</p><p>计算机科学与技术（Computer Science and Technology）是国家一级学科，下设信息安全、软件工程、计算机软件与理论、计算机系统结构、计算机应用技术、计算机技术等专业。</p><p>大家可能现在还会听到物联网、人工智能、大数据等专业，不管叫什么，这些专业要学的核心主干课程就是那些CS基础课，只是专业方向侧重不同而已！</p><p><b>而且这些专业出来大概率最终就业归属都是：互联网公司（BATTMD等）码农</b></p><p>而且这些细分的二级学科方向不会限制死你的方向，比如你学软件工程，你一样可以研究大数据、人工智能。</p><p>所以如果问大家常说的计算机到底选哪个专业，我的建议是：</p><ol><li><b>计算机科学与技术</b></li><li><b>软件工程</b></li></ol><p>基本上核心课程 90%都是一样的，大概是下面这些：</p><p><b>高级语言程序设计、数据结构与算法、电路与电子技术、集合论与图论、代数与逻辑、数字逻辑、计算机组成原理、操作系统原理、数据库原理、编译原理、软件工程、计算机网络。</b></p><p>学习计算机最核心的方法，也是小北一路走来总结的，按照学长推荐的这个路线走，大学毕业成为offer收割机，去BAT TMD 这些都没啥问题：</p><p><b>入门看视频 + 不断编码 + 深入看源码和经典大黑书 + 做国外计算机硬核Lab</b></p><p><b>大黑书有哪些？</b></p><p>可以看看这篇高赞文章： <a href="http://link.zhihu.com/?target=https%3A//mp.weixin.qq.com/s/qsJ7vvYpZcUpRe8AIAUl_A" class=" wrap external" target="_blank" rel="nofollow noreferrer">计算机必读经典书籍</a></p><p>学习计算机一定不要想着速成，要下苦功夫，去啃那些经典书和源码，然后自己动手写代码。</p><p>看书方法可以参考这篇文章：</p><p><a href="http://link.zhihu.com/?target=https%3A//mp.weixin.qq.com/s/nfnN0TWIsuIPgXeSSKze5A" class=" wrap external" target="_blank" rel="nofollow noreferrer">计算机看书小建议~</a></p><h3><b>二、CS 如何学？</b></h3><p>接下来，从我科班踩的坑出发，给还在上大学的学弟学妹们一点学习建议以及划重点。</p><p><b>这是我其中一个学期的成绩单，可以看到很多课程都是上面 这些课程：</b></p><figure data-size="normal"><img src="https://pic1.zhimg.com/v2-815ba22ab172af92bdf4bdbb36467a9e_720w.jpg" data-size="normal" class="content_image" referrerpolicy="no-referrer"><figcaption>img</figcaption></figure><p><b>在大学，首先树立一个观念，那就是学习靠自己，要培养自学能力。</b></p><p><b>虽然我是985 CS 科班出身，即便是所谓的科班，但任然认为自己是”自学编程“。</b></p><p>为什么这么说呢？</p><p>因为我想了下，很多东西、编程知识都是我自学的，而学校上的CS基础课也仅仅是满足考试而已。</p><p>大一上完C语言课，我的水平几乎停留在做课后习题，自己对于内存、指针都是看似懂了，实际没有领悟到精髓。</p><p>而后我依靠看大量C语言和系统底层方面的书籍，比如《CSAPP》、《汇编语言》《程序员自我修养》、《C和指针》、《深入理解C指针》等，才勉强说对于内存、指针这些彻底搞懂了。</p><p>所以我说认为自己是「自学编程的科班生」，相信这样的同学还不在少数。</p><p><b>过去几个月，我微信、知乎收到很多学弟学妹私信，问通信、电子、机械这些专业的同学转码要怎么学？</b></p><p><b>是不是很难追上科班？</b></p><p><b>如何才能减少和科班的差距？</b></p><p>我们可以看下网易云课堂这个计算机科班的课程体系，虽然不是每个学校都一样，但是大同小异:</p><figure data-size="normal"><img src="https://pic4.zhimg.com/v2-6a48ce54c78f685b08a8be3ce91543a4_720w.jpg" data-size="normal" data-caption class="content_image" referrerpolicy="no-referrer"></figure><p>比如计组、汇编、体系结构等课程会让我们明白计算机的构成、运行机制，以及典型的冯诺依曼结构，也会对于内存和指针这样两块在任何编程语言中都是极其重要的概念理解非常深入（不要扯Java这类语言没指针，你完全可以把引用当做受限指针）。</p><p>另外，像数字逻辑、模电这样的课程主要是让你明白数字：浮点数、整数都是如何在计算机内部表示和存储的，以及一些门电路的知识。</p><p>操作系统则让你明白「一个程序是如何跑起来的」，一个它是如何编译、链接、加载、运行的，在这一整个过程几乎会涉及到操作系统的所有环节，学了这个你就会明白 synchronized 关键字在OS层面到底是如何保证的。</p><p>而计算机网络则是让你明白「一个数据包是如何从一台主机发送到另外一台主机的」，这里面涉及到路由算法、差错检测、寻址、重试策略等等，包括 TCP/IP整个协议栈</p><p>编译原理则是让你明白高级语言都是如何被编译程序识别、转换为另外一种语言的，我认为编译器的本质就是做转换。很多人觉得平常用不到编译原理，其实不然。</p><p><b>我一直觉得有件事是很美妙的，就是：</b></p><p><b>你清楚的知道你写下的每一行代码都会做什么，会调用到哪些系统资源，它会占用、依赖哪些系统资源。</b></p><p>比如一条SQL，哪里会慢、会遇到锁会有并发问题。</p><p>虽然不了解原理，也不妨碍使用，但是往往在设计和排查问题的时候，原理非常重要，因为原理就是理论，理论可以帮助我们排查掉各种明显错误的分支，而留下少数理论验证通过的分支，会大大降低我们的工作量。</p><p>也就是剪枝。</p><p>否则我们排查系统错误或者设计方案可能只能一个个去试。</p><h2><b>三、科班很神秘？</b></h2><p><b>那么计算机科班学生该如何学习呢？</b></p><p><b>把操作系统、组成原理、系统编程这些学了，认认真真的把计算机基础打扎实了，那你其实就是一个「计算机科班学生了」</b>。</p><p><b>接下来说下我认为对一个「计算机知识体系完整的毕业生」最重要的课程，你至少得把下面这些课 都学了：</b></p><p><b>1、 编程语言：C/C++ （学好了这两门，基本其它编程语言一周就上手了）</b></p><p><b>2、《系统级编程》(我们教材是CSAPP，这是我本科上过最值的课！System Programing)</b></p><p><b>3、《数据结构与算法》</b></p><p><b>4、《组成原理和体系结构》</b></p><p><b>5、《操作系统》</b></p><p><b>6、《编译原理》</b></p><p><b>7、《计算机网络》</b></p><p><b>8、《数学课》：线代、离散、高数</b></p><p><b>计算机科班刚毕业一年，我觉得大学如果知道这些，可能会少走很多弯路！</b></p><p>首先，送大家一句话~：</p><blockquote><b>万丈高楼平地起，勿在浮沙筑高台。</b> </blockquote><p>所以一定要在大学期间把基础打牢固，整扎实，这是科班的核心竞争力！</p><p>而且非科班同学学完这些课，也可以说是科班了！</p><p><b>那么怎么学？</b></p><p><b>看书 + 看视频 + 实践</b></p><p><b>接下来推荐一些我觉得特别好的资料，尽量保持精简：</b></p><p><b>一、编程语言：C/C++</b></p><p>没有比C语言更适合用来理解计算机系统了, 科班学生一定要学好 C语言。</p><p>推荐：《C程序设计语言》、《C与指针》、《C++ Primer》、《C++对象模型》、《Effective C++ 》</p><p>具体参考：</p><ul><li><a href="http://link.zhihu.com/?target=https%3A//mp.weixin.qq.com/s/WW_X12bTm94iaCgWBgYtJw" class=" wrap external" target="_blank" rel="nofollow noreferrer">如何系统地学习 C++ 语言?</a> </li><li><a href="http://link.zhihu.com/?target=https%3A//mp.weixin.qq.com/s/SQESC1_zv-EfNzyPrA8HsQ" class=" wrap external" target="_blank" rel="nofollow noreferrer">如何系统学习 C 语言？</a> </li></ul><p><b>二、系统级编程</b></p><p>不用说肯定推荐《深入理解计算机系统》</p><p>其实它的简介和第一章《计算机系统漫游》已经写得非常清楚了：</p><p>从一个简单的hello world程序在计算机上的执行过程：</p><p>预处理->编译->汇编->链接->可执行文件->装载->数据流->屏幕输出显示</p><p>汇总成一句：信息 = noreferrer">如何系统学习操作系统</a></p><p><b>六、计算机网络</b></p><p>推荐《计算机网络自顶向下》、《TCP/IP详解》</p><p>具体参考：</p><ul><li><a href="http://link.zhihu.com/?target=https%3A//mp.weixin.qq.com/s/bc_PoBGfdcCbtaHMo0Jezg" 领域知识 : 技能」 接近 7 : 2 : 1。</p><p>这也是我推荐你在大学期间分配学习时间的比例，至少学习基础知识的时间不少于 50%，当然，这些东西你都学完了那可以去找找感兴趣的方向专研一下。</p><p>不要大一、大二一上来就扎进 Java Web、Python 爬虫这种东西，这些可以学，但不是重点。</p><p>这篇文章也没写到具体该如何去学，有哪些好的资料，由于篇幅限制，我准备把这个单独再写一个 《How 篇》，持续关注我哟~</p><p>那么如何检验学得如何呢？</p><p>想必你一定听说这个计网面试题：</p><p>“从 URL
