崔庆才丨静觅的回答<br><br><p data-pid="o66I0GWN">GitHub 因为一些原因，访问速度过慢或者有时无法访问，国内的很多开发者就选择了 Gitee，在 Gitee 上也有不少优秀的开源项目。所以 Gitee 对国内开发者还是比较友好的。</p><p data-pid="AK4inhVo">当然了 Gitee 相比 GitHub 的最大劣势就是整个代码库的体量比较小，GitHub 全世界的人都在网上放代码，想搜什么代码很大概率能搜到，Gitee 相比起来整个体量就小很多了，基本都是国人在用，所以相比 GitHub 就是小巫见大巫了，这个自然都是大家接受的事实。</p><p data-pid="-td-B_2g">但体量小归体量小，起码这是国内开源界为数不多做的比较不错的代码托管网站了，也在一定程度上推动了国内开源的发展，这一点我是表示肯定的。</p><p data-pid="2XWVEjyX">首先，看到这个消息，我是震惊和不解的。</p><p data-pid="pgVaAQ49">我们知道，国内对互联网内容监管这一块是非常严格的，比如短视频、帖子、微博、博客等等，不行的就要封禁的，比如就拿一些内容网站来说吧，简书、博客园之前不都全站封停整顿过一段时间吗？</p><p data-pid="35YcPihZ">嗯，说上面这些类别的网站监管严格点，我觉得多少还能理解。</p><p data-pid="IsZ9it8y">但让我震惊的是，这监管居然都到了代码份上了？</p><p data-pid="0thuZGVe">看看 Gitee 官方的表述，自己其实也是“实属无奈”，可能也真是没办法的事情了。</p><p data-pid="f8vPsiIq">我随便找了一个 Gitee 的仓库看了看，果然我之前的开源仓库变成私有的了：</p><p><br></p><figure data-size="normal"><img src="https://pic3.zhimg.com/v2-d976e28ed79dea24780e2a8cff17eb85_1440w.jpg?source=b1748391" data-caption data-size="normal" data-rawwidth="2508" data-rawheight="654" data-default-watermark-src="https://pica.zhimg.com/v2-356f23a5e790bb6a5529e07e390b4565_720w.jpg?source=b1748391" class="origin_image zh-lightbox-thumb" data-original="https://pic3.zhimg.com/v2-d976e28ed79dea24780e2a8cff17eb85_r.jpg?source=b1748391" referrerpolicy="no-referrer"></figure><p><br></p><p data-pid="Y6zxsJ6b">如果要转开源，那么就需要到设置里面提交申请，然后我来到设置，看着有如下内容：</p><p><br></p><figure data-size="normal"><img src="https://pic1.zhimg.com/v2-e88edf4b2e276eba2ef592c20e74c29f_1440w.jpg?source=b1748391" data-caption data-size="normal" data-rawwidth="1850" data-rawheight="826" data-default-watermark-src="https://pic1.zhimg.com/v2-5c838d9659d3437d33e75ce4c7fe1427_720w.jpg?source=b1748391" class="origin_image zh-lightbox-thumb" data-original="https://pic1.zhimg.com/v2-e88edf4b2e276eba2ef592c20e74c29f_r.jpg?source=b1748391" referrerpolicy="no-referrer"></figure><p><br></p><p data-pid="4Ae-N1P6">这里让我们勾选几个承诺，然后还需要添加 README 才能提交开源审核。</p><p data-pid="-vlQYxOC">我们看看承诺的内容吧，其实主要就是合规性的问题还有侵权问题，我觉得官方所说的“实属无奈”应该就是第一条导致的吧。</p><p data-pid="5phnGUX4">因为毕竟万一有哪个仓库违反了法律法规，整个 Gitee 可能就会面临关停整顿的风险了。</p><p data-pid="KyWK8UUn">所以，Gitee 权衡下利弊和风险，也只好这么做了。</p><p data-pid="3DYHtYEG">事实就是这样了，那这个有什么影响呢？或者有什么意义呢？</p><p data-pid="kGdUJwNM">其实影响的话，无非就是国内的开源生态会进一步变差了。</p><p data-pid="mUnhtbC-">本来 Gitee 的出现能够让大家在上面放放代码分享下。</p><p data-pid="2DKFN4ah">怎么说，使用 Gitee 的程序员可能主要有两类，一类是把 GitHub 上的仓库同步到 Gitee，毕竟 Gitee 在国内的访问速度快不少；另一类就真的愿意用 Gitee 作为自己的代码仓库来分享代码了。</p><p data-pid="scpQ31QU">结果这下子还给全转私有了。。</p><p data-pid="Lz7DWqlP">尤其是 Gitee 老用户，比如几十个仓库的那种，难不成还得花时间一个个提交申请来公开出来？我想愿意一个个去提审的人少之又少。</p><p data-pid="rtRUkQwD">仓库转私有之后，以后大家还能搜到啥代码呢？搜个代码结果就是 Not Found，谁还愿意再来搜代码、分享代码呢？这生态影响太大了。</p><p data-pid="m1IiR5ez">另外还有就是，我真的怀疑，这个审核真的有意义吗？感觉好像是自欺欺人。</p><p data-pid="NmhsyIfl">比如说，我一开始创建了一个 README，然后接着我就提审了，变成公开仓库了，然后我后面再 commit、push 一些奇怪的内容，这不照样还是有风险吗？做到真正规避了吗？如果不是每个 commit 都审核，那做这个审核到底还有什么意义呢？</p><p data-pid="29cjgI3X">我觉得比较理想的就是 Gitee 做一个全站代码风险和版权检测工具，去自动搜查非法内容，如果真的有，那把非法仓库再转私有也行啊，一下子全转私有，这真的让我感到不能理解。</p><p data-pid="w7iG_-kb">那以后咋办呢？</p><p data-pid="BtmoKpYD">说实话，对我可能真的没多少影响，因为我用 Gitee 其实就是同步一些 GitHub 的仓库而已，我主要还是在 GitHub 上分享和保存代码。</p><p data-pid="G-r65yRs">其实我在 Gitee 上从 GitHub 同步了上百个仓库，但我想说，我真的没这耐心把我一个个仓库去设置、提审公开啊。。以后我估计也就放弃 Gitee 了。</p><p data-pid="-iCJExOY">其实为 Gitee 感到有些惋惜，但这个环境下，真的是 Gitee 所说的“实属无奈”了。</p><p data-pid="_RuuGx8I">嗯，虽然说为 Gitee 感到惋惜，但实话讲，作为一名程序员，处在一个健康、丰富的开源生态里面还是很重要的。所以，从我们个人角度来说，还是坚定去拥抱 GitHub 吧，在 GitHub 我们能够搜索到更多丰富的资源，了解到更多优秀的开源项目，自己学到了，变强了，才是最重要的。</p>
