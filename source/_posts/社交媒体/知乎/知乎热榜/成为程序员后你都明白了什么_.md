referrerpolicy="no-referrer"></figure><p data-pid="ZFv91rvI">我感觉内容很有意思，值得一读，和题主的这个问题也挺贴合，一个 10 年老程序员的肺腑之言。</p><p data-pid="zWlKx-Tw">原文是英文的，阮一峰老师翻译了一下。</p><blockquote data-pid="JI9pkG_e">链接：<a target="_blank" rel="nofollow noreferrer">程序员的酒后真言 - 阮一峰的网络日志</a><br></blockquote><p data-pid="utz67RKq">我选取了一些放在下面：</p><p data-pid="RnLok5Vr">（1）职业发展的最好方法是换公司。</p><p data-pid="z7mbM26B">（2）技术栈不重要。技术领域有大约 10-20 条核心原则，重要的是这些原则，<a href="https://www.zhihu.com/search?q=%E6%8A%80%E6%9C%AF%E6%A0%88&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra=%7B%22sourceType%22%3A%22answer%22%2C%22sourceId%22%3A2141087828%7D" class="internal">技术栈</a>只是落实它们的方法。你如果不熟悉某个技术栈，不需要过度担心。</p><p data-pid="T4Gf_OsR">（3）工作和人际关系是两回事。有一些公司，我交到了好朋友，但是工作得并不开心；另一些公司，我没有与任何同事建立友谊，但是工作得很开心。</p><p data-pid="JgRnwjSe">（4）我总是对经理实话实说。怕什么？他开除我？我会在两周内找到一份新工作。</p><p data-pid="fsQPhMav">（5）如果一家公司的工程师超过 100 人，它的期权可能在未来十年内变得很有价值。对于工程师人数很少的公司，期权一般都是毫无价值。</p><p data-pid="3Bc5nzua">（6）好的代码是初级工程师可以理解的代码。伟大的代码可以被第一年的 CS 专业的新生理解。</p><p data-pid="7PGZQlO1">（7）作为一名工程师，最被低估的技能是记录。说真的，如果有人可以教我怎么写文档，我会付钱，也许是 1000 美元。</p><p data-pid="Ha2ubvkn">（8）网上的口水战，几乎都无关紧要，别去参与。</p><p data-pid="dgauL80V">（9）如果我发现自己是公司里面最厉害的工程师，那就该离开了。</p><p data-pid="1P7cSJ-Z">（10）我们应该雇佣更多的实习生，他们很棒。那些精力充沛的小家伙用他们的想法乱搞。如果他们公开质疑或批评某事，那就更好了。我喜欢实习生。</p><p data-pid="ZGMJKueJ">（11）技术栈很重要。如果你使用 Python 或 C++ 语言，就会忍不住想做一些非常不同的事情。因为某些工具确实擅长某些工作。</p><p data-pid="vbLcrtUz">（12）如果你不确定自己想做什么东西，请使用 Java。这是一种糟糕的编程语言，但几乎无所不能。</p><p data-pid="HufD8CKP">（13）对于初学者来说，最赚钱的编程语言是 SQL，干翻所有其他语言。你只了解 SQL 而不会做其他事情，照样赚钱。人力资源专家的年薪？也许5万美元。懂 SQL 的人力资源专家？9万美元。</p><p data-pid="QXQy1wiv">（14）测试很重要，但 TDD （测试驱动的开发）几乎变成了一个邪教。</p><p data-pid="OA7IEV5V">（15） 政府单位很轻松，但并不像人们说的那样好。对于职业生涯早期到中期的工程师，12 万美元的年薪 + 各种福利 + 养老金听起来不错，但是你将被禁锢在深奥的专用工具里面，离开政府单位以后，这些知识就没用了。我非常尊重政府工作人员，但说真的，这些地方的工程师，年龄中位数在 50 岁以上是有原因的。</p><p data-pid="DaRqaD_B">（17）大多数头衔都无关紧要，随便什么公司都可以有首席工程师。</p><p data-pid="cQ9PhFK-">（18）手腕和背部的健康问题可不是开玩笑的，好的设备值得花钱。</p><p data-pid="TkUs2fRk">（19）当一个软件工程师，最好的事情是什么？你可以结识很多想法相同的人，大家互相交流，不一定有相同的兴趣，但是对方会用跟你相同的方式思考问题，这很酷。</p><p data-pid="XoqF5W_n">（20）有些技术太流行，我不得不用它。我心里就会很讨厌这种技术，但会把它推荐给客户，比如我恨 Jenkins，但把它推荐给新客户，我不觉得做错了。</p><p data-pid="B5f4zXDi">（21）成为一名优秀的工程师意味着了解最佳实践，成为高级工程师意味着知道何时打破最佳实践。</p><p data-pid="Y2xBbnyU">（22）发生事故时，如果周围的人试图将责任归咎于外部错误或底层服务中断，那么是时候离开这家公司，继续前进了。</p><p data-pid="Y2nbKVyc">（23）我遇到的最好的领导，同意我的一部分观点，同时耐心跟我解释，为什么不同意我的另一部分观点。我正在努力成为像他们一样的人。</p><p data-pid="rNXPXSLH">（24）算法和数据结构确实重要，但不应该无限夸大，尤其是面试的时候。我没见过药剂师面试时，还要测试<a href="https://www.zhihu.com/search?q=%E6%9C%89%E6%9C%BA%E5%8C%96%E5%AD%A6&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra=%7B%22sourceType%22%3A%22answer%22%2C%22sourceId%22%3A2141087828%7D" class="internal">有机化学</a>的细节。这个行业的面试过程有时候很糟糕。</p><p data-pid="ZDTMHSFv"><b>补充：</b>但是虽然这么说，面试之前算法和数据结构还是得刷起来，这时候你就需要 <a href="https://zhuanlan.zhihu.com/p/407414826" class="internal">算法小白的 LeetCode 刷题顺序（持续更新）</a></p><p data-pid="k84B8tIk">身为前 ACM 选手，我觉得吧，后面这个也是你需要的：<a href="https://zhuanlan.zhihu.com/p/418890402" class="internal">刷了几千道算法题，ACM学长的一些心得。</a></p><p data-pid="0KnOwqyr">（25）做自己喜欢的事情并不重要，不要让我做讨厌的事情更重要。</p><p data-pid="k91XdJRA">（26）越接近产品，就越接近推动收入增长。无论工作的技术性如何，只要它接近产品，我都感到越有价值。</p><p data-pid="owH7HKc8">（27）即使我平时用 Windows 工作，Linux 也很重要。为什么？因为服务器是 Linux 系统，你最终在 Linux 系统上工作。</p><p data-pid="mzJYDMt4">（28）人死了以后，你想让代码成为你的遗产吗？如果是那样，就花很多时间在代码上面吧，因为那是你的遗产。但是，如果你像我一样，更看重与家人、朋友和生活中其他人相处的时光，而不是写的代码，那就别对它太在意。</p><p
