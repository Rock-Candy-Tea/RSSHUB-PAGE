雨花的回答<br><br><p>曾经实验室花大价钱买了一台自动给RT-PCR上样的机器，可以连接电脑，用它的软件编制上样程序，界面有点像少儿编程，拖放控件，然后下拉菜单选判断、分支、指令。</p><p>写简单的程序很简单，很直观，但逻辑稍微复杂一点，比如样品数量不是4的整倍数，就开始复杂起来，在一屏又一屏的菜单里面挑来挑去，看得眼睛花。最后这台机器就闲置在那里，因为谁也不想在菜单的海洋中遨游了。</p><p>后来另一个实验室买了过去，找了专人写VB脚本，就快捷多了。为什么快捷？如果一个VB新手跟一个图形编程新手，一个碰到新的语法就搜索一下MSDN，一个在菜单的海洋里遨游，速度可能差不多快，图形编程可能还稍微快一点，但关键是写程序的人肯定会变得熟练，VB写熟练以后大部分用法都是不需要现搜的，直接打字就行了，还能调用别人写的库，不需要知道别人怎么写，IDE能补全。但图形界面你熟练了也仍然在菜单的海洋里遨游，一层一层的菜单鼠标指向、点选都花时间， 即使修炼成CS达人也不如打字表达自己的意思来得快捷，下限高，上限低，学它的价值就小。</p>
