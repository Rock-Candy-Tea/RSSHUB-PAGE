pansz的回答<br><br><p data-pid="DIk_ikdW">呃，这究竟是什么奇怪的问题？</p><p data-pid="eELTX9za">整个 Linux 系统几乎就是为了给 C++ 提供开源库而生的，ubuntu包含的所有软件的开源代码库早已经超过 500GB（其实这还是若干年前的数据）。其中绝大多数都是C或者C++语言。</p><p data-pid="ht4AmLLO">在语言排名中，C++的名次虽然比不上Python，但基本上也是紧随其后。</p><p data-pid="zKnpQOb5">要说绝对数量上 C++ 开源库能不能跟 python 比，那肯定是远远超过的。</p><p data-pid="57XwMq5x">但 C++ 的一个重要问题在于，它缺乏一个靠谱的标准的广泛认可的包管理系统，同时，在项目中引入C++开源库的难度远远高于其他语言。以至于大多数开源库散落在世界各地。并且集成进你项目的方法也千奇百怪。因此，「你看不到很多C++开源库」并不等于C++开源库真的会比python开源库少。看不到的原因是因为C++并没有标准的包管理系统，仅此而已。</p><p data-pid="sN3uUbYd">至于什么时候C++标准委员会的那套包管理能上线，也许那个时候才能真正解放C++生产力吧。</p>
