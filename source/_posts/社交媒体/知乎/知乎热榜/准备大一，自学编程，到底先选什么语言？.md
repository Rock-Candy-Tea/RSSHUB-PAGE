编程指北的回答<br><br><p>大一，自学编程。</p><p>如果你是科班计算机软件的话，建议就学C，你说的《CPrime Plus》是 C 语言的书而不是一门语言。</p><p>其实入门 Python、C 都可以，如果是非科班的我觉得先学 Python 比较好，Python上手快、简单一些，先用 Python 养成编程思维、喜欢上编程，再去学其它的也可以。</p><p>当然科班的依然可以先学 Python，并且国外不少高校第一门语言现在比较喜欢教 Python、Schema 这类，但是我觉得用 C 作为科班的第一门语言也不错，虽然很多初学者反映比较难。</p><p>但是我觉得掌握好 C 是一个学计算机的同学必备的技能，如果认真学都觉得 C 语言难，那真的要怀疑一下是否真的适合学计算机了。</p><p>对于大部分初学者，学习C语言的目的是希望做一名合格的程序员，开发出靠谱的软件来。</p><p>但是学了C语言的基本语法后，发现只能开发“黑底白字”的DOS程序，完全没有漂亮的界面和生动的交互。</p><p>于是学数据结构，学算法，学操作系统，越陷越深，越来越难，最后迷茫了，不知道学C语言能做什么，认为学习编程很难，开始怀疑自己，甚至想放弃。</p><p>其实，这是很多初学者都会踩到的一个坑！</p><p>C语言本身是一门很简单的语言，提供的实用功能不多，大部分要借助操作系统、第三方库、单片机来完成。</p><p>也就是说，只学C语言基本什么也做不了，也基本找不到工作。</p><p>C语言是一门通用性的语言，并没有针对某个领域进行优化，在实际项目中，C语言主要用于较底层的开发，例如：</p><ul><li>Linux、Unix 等操作系统的内核90%以上都使用C语言开发；</li><li>单片机和嵌入式属于软硬件的结合，有很多使用C语言的地方；</li><li>开发系统组件或服务，用于支撑上层应用；</li><li>如果对软件某个模块（例如算法和搜索部分）的效率要求较高，也可以使用C语言来开发，不过现在也多用C++，对于开发效率和运行效率都比较好</li></ul><p>既然C语言的应用这么多，为什么很多同学觉得它什么也做不了呢？</p><p>主要还是 C 语言标准库和第三方库都比较少，很多初学者理解不了编译、链接这些，使用第三方库也比较麻烦。</p><p>---update----</p><p>有学弟学妹问我要学习资料</p><p>在这，顺便分享一份大学期间自己整理的电子书库，绝不是在网上那种打包下载的，而是自己需要学到某个方向知识的时候，去网上挨个找的，最后汇总而成。</p><p>汇集了编程语言(Java、C++、C、Python等等)、操作系统、计算机网络、系统架构、设计模式、程序员数学、测试、中间件 、前端开发、后台开发、网络编程、Linux使用及内核、数据库、Redis....等主流的编程学习书籍。</p><p>这部分我是会不断把它完善的，当成自己的小电子书库，不多，但贵在精。</p><figure data-size="normal"><img src="https://pic2.zhimg.com/v2-d7a9537446e8aafeb0c5feb8cce72fc5_720w.jpg" 但是要注意时间成本，软件行业已经不是一般的复杂和巨大，任何一个领域的知识的复杂性都足够耗费掉我们一生的时间，所以一定要抓住主线，对于技术和知识，要学通用的、流行的，可以尝试面向面试学习。 “打破砂锅问到底”式的学习虽然精神可敬，但效率并不划算。 要在适当的层次上抽象出一层，并且认可这一层提供的接口，不去深究内部实现，了解原理即可，不必深究内部实现。</p><p> 比如学习HTTP，那么就先认可 TCP 提供的稳定可靠传输，而不继续深挖TCP的内容。 新手，一定不要一直看书，保持看书的时间不超过 50%。</p><ol><li>看书学习基本的理论</li><li>编程练习</li><li>有了新领悟，继续看书</li></ol><p>反复的循环。</p><p><b>觉得有帮助的话，记得帮我</b> <a class="member_mention"
