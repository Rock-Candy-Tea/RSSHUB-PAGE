李察德的回答<br><br><h3>史诗级进化</h3><h3>屏下指纹去踏马</h3><h3>Apple Watch 恐成最大输家</h3><hr><p data-pid="inuc66Vp">戴口罩面容 ID (或者叫眼神 ID ？)支持通过识别用户眼部特征解锁手机或者安全支付。</p><p data-pid="vFw8CjMZ">目前仅支持 iPhone 12 和 13 系列，不支持 X、XS系列、XR、11系列等搭载面容 ID 的老机型，不知道是算力跟不上还是库克老贼刀法犀利。</p><p data-pid="3Y_Q3K7b">希望正式版能支持更多的面容 ID 机型吧。</p><hr><p data-pid="AS7q2EhX">我很早以前就在知乎说过：苹果不会采用屏下指纹，也不会上 面容 ID+触摸 ID 的方案。</p><p data-pid="y5CBvqmg">手机屏下指纹的问题在于不能活体检测，一块单位打卡常用的指纹膜就能解锁，安全级别很低。</p><figure data-size="normal"><img src="https://pic4.zhimg.com/v2-935a88ae1fea77588c26dd4e835df7e3_720w.jpg" data-rawwidth="280" data-rawheight="499" data-size="normal" class="content_image" referrerpolicy="no-referrer"></figure><p><br></p><p data-pid="yzVynu62">面容 ID+触摸 ID 的问题在于双生物识别技术系统如果以或有形式通过，安全等级最终只会以较低的生物识别——比如触摸 ID 为准，那么面容 ID 的高安全优势就失去意义了。</p><p data-pid="xo3JMYa-">假如手机的解锁方式是通过面容 ID (1/100万误判率)<b>或</b>触摸 ID (1/5万误判率)，那这么一个生物识别系统解锁的最高误判率就是 1/5万 。</p><p data-pid="BvScl8cE">所以疫情之下，苹果有几条路可选：</p><ol><li data-pid="Ua0SiOaG">提供安全级别更高或者持平，方便性也持平且不需要面部特征的生物识别技术，例如<b>解决了应用场景狭隘问题的虹膜识别</b>。</li><li data-pid="ut3-9tb6">提供方便性互补，安全等级持平的双生物识别技术，例如<b>把侧边触摸 ID 的误判率怼到 1/100万</b>。</li><li data-pid="CKNdajrv">退而求其次，砍掉面容 ID ，用相对安全性更低的触摸 ID 顶上。</li></ol><p><br></p><p data-pid="vA9BxGev">现在，果子给出了答案，选择了方案 1 。</p><p data-pid="rJ9zo0ct">在原有的面容 ID 基础上开发出仅识别眼部特征就能解锁的算法，做到了类似虹膜识别的方案，但又不丢失原有的方便，完美解决了兼具安全性和便捷性的难题。</p><p data-pid="PYmK_OdZ">除了戴墨镜的场景还不支持，眼神 ID 能做到面容 ID 的一切。</p><p data-pid="WI1RE9U6">这波属于老树开新花，恭喜 A14 和 A15 的 iPhone 用户。</p><h3>你永远可以相信苹果。</h3>
