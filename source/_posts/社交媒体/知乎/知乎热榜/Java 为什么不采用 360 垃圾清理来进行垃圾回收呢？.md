张小方的回答<br><br><p>谢邀，作为早期 Java 的开发者之一，我们团队当初确实考虑过使用 360 垃圾清理来对 Java 进行垃圾回收。</p><p>我们早期也曾发布了使用 360 垃圾回收的 Java 版本，当然部分用户使用了我们的这个版本之后，又成功地回到单身贵族家庭，我们对他们的付出表示感谢，并邀请这批用户来我们 Java 炸鱼实验室参观。由于省去了不必要的恋爱、婚姻花销，他们有更多的时间加班了。我们收到几百家公司老板的感谢信。这是怎么回事呢？</p><p>众所周知，360 垃圾清理，顾名思义，一年中只能工作 360 天，有 5 天是不能工作的，而我们都知道 Java 虚拟机每天都会产生垃圾，这显然是不能满足要求的。 Java 一般都用于大型服务器后端服务开发，其访问量都是非常巨大的，如果一年 5 天不能工作，那么可靠性只能达到 98.9%，这是远远不够的。而且不算闰年，闰年不能工作的时间将由 5 天变成 6 天，这对于使用 Java 的互联网公司是无法接受的。</p><p>试想，你和你的女友约好年前最后六天去见丈母娘，并在见面后将彩礼转到丈母娘的卡上。</p><p>但是由于年底的最后一天支付宝和相关的银行卡服务器采用了 Java，而 Java 采用了 360 进行垃圾回收，最终由于你无法完成彩礼转账，导致丈母娘拒绝把女儿嫁给你。</p><p>你最终：</p><p>老婆 -1</p><p>这是大多数人无法接受的，由于不少未婚小青年都是在年底回去和丈母娘谈判，所以最终：</p><p>老婆s - 10086</p><p>这样社会上光棍就多了，社会上光棍多了就不利于社会的稳定，所以最终有关部门会出来调查问题，所以排查下来，原因是使用 360 垃圾回收的 Java 无法在年底的 5 天或者 6 天工作。</p><p>当然，我们也尝试和 360 团队进行沟通，360 团队拒绝为一年剩下的五天提供服务，他们解释说，这样会导致他们的同事需要加班，少则 996，多则 007。这不是一种好的社会风气。</p><p>因此，我们最终没有在 Java 中使用 360 的垃圾回收，并忍痛下掉了使用 360 垃圾回收的 Java 版本。</p><p>Java 团队和 360 团队都是伟大的团队，他们在垃圾回收中做的工作不分伯仲，都值得我们尊敬。</p>
