<p>因为项目的原因，最近半年小鱼一直在进行联邦学习方面的调研和研究。</p>
<p>题主关于联邦学习是否是伪需求的疑问，也是我一直以来的问题。因为我在这个领域还是萌新，所以仅分享一下看法，希望能抛砖引玉，和大家一起讨论。</p>
<p>题主提到的“google 将手机上的数据拿来做机器学习”应该指的是联邦学习的开山之作<sup>[1]</sup>，这篇文章我个人认为不仅<strong>首提了联邦学习的一个商业落地应用</strong>（隐私保护前提下，在用户的手机上训练模型，来进行输入法下一词预测的任务<sup>[2]</sup>）。</p>
<figure><img class="content-image" src="https://picx.zhimg.com/v2-753ab5830f633d627f4a16e6943d5ef8_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"><figcaption>图源 Federated Learning for Mobile Keyboard Prediction</figcaption></figure>
<figure><img class="content-image" src="https://pica.zhimg.com/v2-407e3e7595292ccf60bf759912e24664_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"><figcaption>图源 Federated Learning for Mobile Keyboard Prediction</figcaption></figure>
<p>同时<strong>还明确指出了适用联邦学习的场景</strong>（当然后续几千篇的联邦学习论文大大拓展了适用场景，不过在此处先回顾一下谷歌这个工作的观点~）</p>
<p><strong>联邦学习场景的数据应具有以下特性：</strong></p>
<ol>
<li>训练来自移动设备的真实数据比数据中心提供的代理数据具有明显的优势；</li>
<li><strong>数据是隐私敏感的或较大规模的，不需要仅出于训练模型的目的将其记录在数据中心</strong>；</li>
<li>对于监督任务，可以从用户交互中自然地推断出数据的标签。</li>
</ol>
<p>许多移动设备上的智能应用存在满足上述标准的数据</p>
<ul>
<li>图像分类：预测哪些照片最有可能被分享出去</li>
<li>语言模型：用于改善语音识别性能或触摸屏键盘上的文字输入效果</li>
</ul>
<p>它们同时满足上述三个特性：</p>
<ol>
<li>数据是敏感的： 用户的照片或键盘输入的文本；</li>
<li>数据的分布也与代理数据提供的不同， 更有用户特点和优势；</li>
<li>数据的标签也是可以直接获得的：比如用户的照片和输入的文字等本身就是带标签的；照片可以通过用户的交互操作进行打标签（删除、分享、查看）。</li>
</ol>
<p>同时，这两个任务都非常适合学习神经网络。</p>
<figure><img class="content-image" src="https://pic2.zhimg.com/v2-53e4f7c28863bc45d1afb844c6556542_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"><figcaption>图源 Communication-Efficient Learning of Deep Networks from Decentralized Data</figcaption></figure>
<hr>
<p>针对题主的问题逐条进行分析：</p>
<ul>
<li><strong>“实际上将数据做好匿名化不就可以上传到服务器集中处理了吗”</strong></li>
</ul>
<p>我认为匿名化数据然后集中再训练模型确实是一种可行的方案。但现在<strong>因为 GDPR</strong><sup>[3]</sup><strong>等政策性的规定</strong>，<strong>共享数据是一种不被允许的不合规的行为</strong>。从这一点上来看，联邦学习并不是伪需求，而是一种适应隐私保护政策的（<strong>数据不出本地的</strong>）可行方案。</p>
<ul>
<li><strong>金融保险医疗机构自身的数据真的少到满足模型训练的需求（而且未来的增长也不会满足）吗？</strong></li>
</ul>
<p>各机构确实是不缺乏数据的，但肯定是缺乏“<strong>标注数据</strong>”的。对数据进行标注需要投入大量的人力物力，而有些领域需要邀请专家进行标注的代价更是高昂。</p>
<p>从标注数据的稀缺的角度去做事情的话，其实有很多思路，比如半监督学习 / 无监督学习，最近火热的自监督学习 / 小样本学习 / 零样本学习等研究领域，而联邦学习作为一种特殊的分布式机器学习方法，是有助于利用到更多地方的标注数据信息的。（同时联邦学习还提供了满足隐私保护，Non IID 数据需求的一些工具）</p>
<ul>
<li><strong>即使有更多的数据，当准确率达到一定程度后，边际效益也是递减的，且各个公司的数据分布、格式、完整性都千差万别，dirty work 不可避免，联邦学习带来的收益能覆盖这些成本吗？</strong></li>
</ul>
<p>因为小鱼我不在企业中，所以很难回答这个问题。我所知的是在互联网企业中，如果是关键业务，那么提升一两个百分点可能也是需要一个团队投入大半年的工作的。因为互联网平台受众广，哪怕提升很小，但也能影响到非常多的人。</p>
<p>我觉得这和另一个问题有些异曲同工之处：<a class="internal" href="https://www.zhihu.com/question/32022042/answer/910303553">为什么互联网行业工资高？</a>。当然如果提升性能的代价太大，我想企业也不会提出对应的 KPI 的…是不是可以将提升模型性能需要付出的精力和提升性能后带来的经济收益作为一个多目标优化问题去进行优化呢，我想这个可能是一个运筹学的问题了; )</p>
<p>而针对“数据分布、格式、完整性都千差万别”的现状，联邦学习中非常重要的研究领域就是<strong>Non IID 数据场景</strong>（ <a class="member_mention" href="http://www.zhihu.com/people/9ca23a25c24e381bc9ec03a58d107812">@lokinko</a><a class="internal" href="https://zhuanlan.zhihu.com/p/372511964">「联邦学习」— Non-IID 研究简介</a>， <a class="member_mention" href="http://www.zhihu.com/people/935d2c1ab228703e82399c6b9d8a953a">@李新春</a><a class="internal" href="https://zhuanlan.zhihu.com/p/309381344">联邦学习系列：NonIID 数据</a>）。除了数据的异质性以外，甚至还有考虑系统、模型等异质性的研究工作<sup>[4]</sup>。</p>

</ul>

<hr>
<p>另外 <a class="member_mention" href="http://www.zhihu.com/people/691770b5e578459f065fe603cd8e90d0">@一般通过路人</a><a class="internal" href="https://www.zhihu.com/question/497705225/answer/2569977027">联邦学习是一个伪需求吗</a> 提到很有现实价值的一个观点，我认为确实是联邦学习落地过程中的一个痛点，摘录如下：</p>
<blockquote>我觉得目前来说落地最大的痛点，就拿横向联邦学习举例，公司要和用户 share 同一个 model，然后用户一起 train，上传 model parameter/gradient 给公司 aggregate。用户的隐私是被保护了，但公司的 model 呢？<br>换句话说，现在一个训练好的神经网络那么贵，<strong>哪个公司会愿意把自己动辄几千万美金训练的模型 share 给你一个普通用户</strong>？就算法律强制规定不得让用户上传原始的隐私数据，那公司也会更愿意选择其他的能保护住自己模型的方法，而不是我们现在看到的联邦学习。<br>所以现有的联邦学习从定义开始就是有问题的，忽略了对公司模型的保护，完美地体现了学术界和工业界的 gap。<br><br></blockquote>
<p>这是一个很好的问题，我将其归类为<strong>针对模型的知识产权保护问题</strong>。据我所知这个还是一个较为前沿的领域，我也遇到了类似的问题，此前见到了三篇<sup>[7]</sup><sup>[8]</sup><sup>[9]</sup>很有意思的论文（比如 <a class="member_mention" href="http://www.zhihu.com/people/256a19b10d710c5cf3f57fee56cf75a8">@可别当个键盘侠</a><a class="internal" href="https://zhuanlan.zhihu.com/p/470233635">ICLR 2022 Oral: Non-Transferable Learning(反迁移学习)</a>），同大家分享一下。</p>
<p><strong>要是有相关的论文也请分享给我，可以一起研究一下~~</strong></p>
<p>总的来说，我认为联邦学习是一套在特定场景下很有用的解决方案，它不能满足现实场景中模型训练和部署的所有需求，但确实提供了有参考价值的方法。如果将目光放更远一些，为了满足模型的可解释性、隐私保护、公平性、稳健性、泛化和迁移性能等需求可能涉及更大的领域，比如<strong>可信 AI（Trustworthy AI</strong>）。联邦学习是可信 AI 的重要一环~</p>
