<div class="question">
为什么感觉每天晚上 12 ：01 移动 5G 网就会极差？真的是这样吗？

<div class="answer">

<strong>
<img class="avatar" src="https://pica.zhimg.com/v2-7af3fdbdf67525f431e69689917b9f61_l.jpg?source=8673f162" referrerpolicy="no-referrer">
<span class="author">甜草莓，</span><span class="bio">不是专家 | 人人都有不知道的东西，讨厌嘲讽类评论</span>
<a href="https://www.zhihu.com/question/490198891/answer/2328660560" class="originUrl" hidden>查看知乎原文</a>
</strong>

<div class="content">
<p>其实这道题下边各位的回答已经指出了问题，大概率是 5G 基站在 0-8 点间开启了功率控制，但是应该不会影响通话和上网质量，如果有影响（比如试试手机游戏延迟和测一下下载速度），如果低于日常速度建议投诉工信部，非常管用。</p>
<p>我这里之所以说大概率，是因为手机信号变弱的原因通常比较复杂，比如我大学还在宿舍的时候，发现晚上 12:00 手机信号必然从满格变到 0，后来一投诉，才发现原来我们宿舍楼顶基站接入的是学校电路，这样只要晚上宿舍断电基站关停，一栋楼的手机都会退站再重新接入，从晚上 12 点开始信号就会非常差。</p>
<p>刨除这些偶然因素，目前 5G 现网其实基本都实装了节能方案。根据测算，2025 年通信行业将消耗全球 20%左右的电力，而在组成 5G 网络的所有设备中中，<strong>基站设备能耗占总能耗的 85%以上</strong>。这里有一个简单的对比，如果按照单位基站来测算，<strong>5G 基站空载能耗大约 2.2-2.3kW，满载功耗大约是 3.7-3.9kW，从单个站点来看，能耗是 4G 基站的 3-4 倍</strong>；而<strong>按数据吞吐量 / 能耗比的角度，5G 基站其实要比 4G 低不少</strong><sup>[1]</sup><strong>。</strong></p>
<figure><img class="content-image" src="https://pic1.zhimg.com/v2-4447816d06bfc04e92f4e548ca707b49_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"><figcaption>来源中国移动白皮书</figcaption></figure>
<blockquote>可以做个比较形象的比喻，我们假设山上住了 10 个人，每天都需要有 2 个人抬水上山，10 年时间里山上又来了 5 个住户，这时候水桶明显不够了，而山上的人往山上搬迁的意愿又十分强烈，于是山上的住户们凑钱，换了个可以给 100 个人提供日常用水的大水车，这个水车需要 5 个人专门负责运营，短时间内运营成本提高了，但是大家发现靠水费承担不起，于是把水车空转 50%。这大概就是目前 4G 到 5G 网络的运行状况。</blockquote>
<p>因为目前 5G 基站普遍不能满载，设备商们发现射频拉远单元(AAU)约占总体基站功耗的 90%，于是大家想尽办法来降低设备功耗，当然整体的方案其实比各位说的要复杂的多，不仅仅体现在节能方式上，触发方式也十分复杂<sup>[2]</sup>。</p>
<p>基本主流的基站节能方案有两种<sup>[3]</sup>，<strong>硬关断</strong>和<strong>软关断</strong>：</p>
<ul>
<li>硬关断：</li>
<ul>
<li>电池脱钩，</li>
<li>远程继电器关断，</li>
<li>FSU 关断；</li>
</ul>
</ul>
<p><strong>硬关断的问题是往往需要比较长的恢复时间，所以目前使用比较少</strong><sup>[4]</sup>。</p>
<ul>
<li>软关断：</li>
<ul>
<li><strong>符号关断</strong>：就是在把<strong>部分帧没有发送数据作为触发条件</strong>，在这些帧周期关闭功放，这里用户体验最好，但是触发条件往往比较难，因为这种符号关断往往需要增强型数据帧，这不仅仅需要基站支持，手机也需要支持；</li>
</ul>
</ul>
<figure><img class="content-image" src="https://pica.zhimg.com/v2-1a14df50518340c33f2897f43d47b651_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"><figcaption>比如上图的某些符号周期内没有承载数据，可以执行关断节能操作</figcaption></figure>
<ul>
<ul>
<li><strong>通道关断</strong>（就是其他答主说的）：把小区内业务量作为触发条件，如果满足一定阈值，把小区基站从多入多出（MIMO）切换为单入多出（SIMO），通俗意义上理解，就是原本假如有 64 通道，可能会根据业务量变成 32 通道或者 16 通道，但是依然采用 64 个天线发射，这样会降低基站的数据吞吐量，同时降低覆盖，因为关闭了一个或者多个通道的功率放大器，会导致输出信号能量降低。</li>
</ul>
</ul>
<figure><img class="content-image" src="https://pic1.zhimg.com/v2-a6b45cbca467ee98ecb15d8e717c6bbe_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"><figcaption>通道关断，关闭某些通道的功放</figcaption></figure>
<ul>
<ul>
<li><strong>载波关断：</strong>这个很容易理解，就是原本小区内部有 2 个以上载波向用户提供服务，有的手机还能依次完成载波聚合提供高数据吞吐量，但是现在考虑到小区内部业务不足，关掉一个载波同样可以降低功耗。</li>
</ul>
</ul>
<figure><img class="content-image" src="https://pic2.zhimg.com/v2-8e0d401284a3996dac9252ed1f6b4a58_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"></figure>
<ul>
<li><strong>深度休眠</strong><sup>[5]</sup>：如果小区内部是双层网络（宏基站和小基站），小基站可以在负载非常低的时候执行休眠，由宏基站执行覆盖和日常数据处理业务。通俗来讲，就是如果有一个非常强的 WiFi 节点，同时它的覆盖区内有一个比较弱的 WiFI 节点用来分流数据，那么在整体小区负载非常低的情况下，这个弱节点可以关闭，让强 WiFi 节点来处理数据。</li>
</ul>
<p>实际操作里，基本是各种节电方式来汇总累加判断效果，甚至用上的机器学习等算法<sup>[6]</sup>，这个复杂程度我可以用现网的一个判断流程来给你们举例：</p>
<figure><img class="content-image" src="https://pic2.zhimg.com/v2-977cdec93edc441c07f92f0a1b3c4b03_720w.jpg?source=8673f162" alt referrerpolicy="no-referrer"></figure>
<p>题主小区应该是触发了某种判决条件，导致原本接入的基站执行了通道关断或者深度休眠，信号变得比较弱；但是虽然信号比较弱，根据现网的策略，如果业务负载提升（比如题主用手机大流量下载），基站应该会重新恢复工作状态，满足用户需求。</p>
<p>所以，可以测试一下实际速率和通话效果，如果和平日差别比较大，建议直接投诉....会有人来修的。</p>
<p>大概如此。</p>
