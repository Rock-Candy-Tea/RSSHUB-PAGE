【分布式搜索第2次技术研讨会确定QTC的生态蓝海和技术突破方向】8月27日，亚太区的存储服务商通过线上和线下的方式，直通QTC的创始团队。针对QitChain的共识，技术和生态流通等核心三大要点进行了确认和研讨。其CTO Sam Catchpole表示：QTC的基础设施建设过程中会参考Elasticsearch 、 Solr 等老牌的分布式搜索的架构和技术理念，但QTC和它们还是有很大的区别的： 
（1）在 IPFS 的基础上进行搜索，并利用 IPFS 的去重技术解决了数据冗余的问题 
（2）QTC对用户搜索的信息进行加密和存储，需要在隐私计算等技术方向探索 
（3）一般分布式搜索的底层是 java 开发，主要基于 lucene 软件类库框架。而 QTC 则是直接基于 IPFS 网络基础，应用 java 、 c ++、 python 等语言，开发难度大。需要各种API的调用、Defi的接入，跨链和NFT等领域的开发。<br><a href="https://mp.weixin.qq.com/s/RuyqOn5ndwuFEmOOCgDosw" target="_blank">原文链接</a>
