<p style="text-indent: 2em; text-align: left;">继 AWS 和 Azure 之后，谷歌云今天推出了其首款基于 Arm 的虚拟机，紧随 AWS 上的 Graviton 实例和微软 Azure 上新推出的 Arm 虚拟机的脚步。另一方面，谷歌云正在效仿微软 Azure，采用 Ampere 的处理器。 “Tau T2A”名称将用于新的 Tau 虚拟机，这些虚拟机现在在 Google Cloud 上进行预览。 AMD Milan 处理器几乎正好在一年前用于该系列的首次亮相，目的是提高性价比。</p><p style="text-align: center"><img src="https://cors.zfour.workers.dev/?http://images.ofweek.com/Upload/News/2022-7/15/zhijian/20220715174445_1657878285973667970.png" width="500" height="271" title alt referrerpolicy="no-referrer"></p><p style="text-indent: 2em; text-align: left;">既然我们已经进入了我们多年来一直参与的 Arm 生态系统，我们的客户将拥有更多的选择和自由。”据谷歌基础设施副总裁兼总经理 Sachin Gupta 称，谷歌云支持的操作系统、数据库、编程语言和其他工具包括 Linux、Windows、Mac OS X 等。</p><p style="text-indent: 2em; text-align: left;">新芯片的预定义 SKU 将包括多达 48 个 vCPU 和多达 4GB 的 RAM。除了标准的各种存储选择外，这些虚拟机还将具有高达 32 Gbps 的网络容量。除了网络服务器和容器化微服务，谷歌声称这些 CPU 要求将允许这些机器执行各种其他类型的工作负载，例如数据记录应用程序。</p><p style="text-indent: 2em; text-align: left;">谷歌将它们视为性价比优化的选项，就像 AMD 驱动的 Tau 处理器一样。例如，Google Cloud 的 us－central1 区域中的 32 核 Tau T2A 虚拟机每小时收费 1．232 美元。</p><p style="text-indent: 2em; text-align: left;">除了 Google 的 Container－Optimized OS，用户还可以在这些计算机上运行 RHEL、CentOS、Ubuntu 和 Rocky Linux 上的容器化应用程序。 Arm 支持已经成为绝大多数操作系统和软件制造商的标准配置，从而增强了这些虚拟机（以及谷歌竞争对手的虚拟机）的实用性。</p><p style="text-indent: 2em; text-align: left;">新的虚拟机现在只能在少数地区使用，包括 us－central（爱荷华州 － A、B、F 区）、europe－west4（荷兰 － A、B、C 区）和 asia－southeast1（新加坡 － B 区， C），但它们将在未来推广到更多的数据中心。</p><p style="text-indent: 2em; text-align: left;">根据 Ampere Computing 首席产品官 Jeff Wittich 的说法，“Ampere Altra 云原生处理器是自下而上构建的，以满足当前云应用程序的需求。”由于我们与 Google Cloud 的紧密关系，现在有新的性价比优化的 Tau T2A 实例可用，即使是最苛刻的横向扩展应用程序也可以快速可靠地安装。”</p><p style="text-indent: 2em; text-align: left;">除了 Google Cloud 中的 Compute Engine，这些虚拟机现在可以与 Kubernetes Engine、Dataflow 流和批处理以及 Batch（Google 今天宣布的一种新的批处理作业调度程序）一起使用。预计 Gupta 的新容量将推动重要的面向吞吐量的计算用例，如天气预报和电路设计自动化。这项新服务的主要目标是为批处理的云计算容量在时间、位置和成本方面提供无与伦比的灵活性。</p>
