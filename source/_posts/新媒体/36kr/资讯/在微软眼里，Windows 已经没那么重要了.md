</blockquote> 
<p>编者按：Windows 曾是<a class="project-link" data-id="3967413" data-name="微软" data-logo="https://img.36krcdn.com/20200916/v2_811751a081924fa9af8741ce120bd7bf_img_png" data-refer-type="2" href="https://36kr.com/projectDetails/3967413" target="_blank">微软</a>的明星产品，但是今非昔比，微软的未来是微软，不是Windows。科技评论博主 Ben Thompson在Windows 11发布会之后分析了Windows 在微软眼中的定位，以及它衰落的原因。</p> 
<p>上个月， 微软举行了 Windows 11 的发布会，伊恩谢尔（Ian Sherr）在 CNET 上写道：</p> 
<blockquote> 
 <p>当你购买电脑或智能手机时，你可以有多个选择。苹果系列的产品有 Mac、iPhone 和 iPad，你可以用来轻松地共享文件、视频聊天和看电视。谷歌也有一系列基于安卓系统的产品，手机、平板电脑和电脑，等等。但这次微软希望借助 Windows 11，希望打破这种模式。</p> 
 <p>在这次发布会上，微软提到今年秋天将免费升级 Windows的下一个主要版本，提供一系列新功能，与苹果、谷歌及其他互联网企业并驾齐驱。</p> 
 <p>微软正不断加大对安卓应用程序的支持，人们能轻松地在电脑上运行手机应用程序。与苹果FaceTime植入mac系统的做法不同，微软将Teams软件植入Windows系统并不具有排他性。微软已经有了针对Mac、iPhone和安卓系统的Teams 应用程序。(微软 CEO 萨蒂亚·纳德拉(Satya Nadella)甚至告诉一名记者，他很乐意在微软电脑上使用FaceTime。)</p> 
</blockquote> 
<p>我真的很喜欢这个发布会，活动时长44分51秒，有一种活泼和轻盈的感觉，与苹果新冠肺炎疫情时期商业风格的演示形成了鲜明对比。这里面有点角色逆转的味道，20年前，微软在CES上推出自己呕心沥血制作的产品，而苹果在Macworld大会上只推出一些低成本的产品。</p> 
<p>仔细想想，这并不奇怪。在那个时候，Windows的新版本意味着几乎所有使用电脑的人都有了新的体验，开发者有了新的机会，竞争对手也有了新的理由去担心微软产品在该年度将独树一帜。而另一方面，苹果公司没什么可失去的，大可不顾一切往前冲。</p> 
<p>今天的情况正好相反：苹果在计算机领域占据了主导地位。iOS 15将为10亿用户带来新体验，新的API将为开发者提供新的机遇，苹果不只是推出一些对抗竞争对手的功能，还会创造新的功能(如Facebook和应用安装广告)；现在看似Windows没什么可失去的。</p> 
<h2>Windows 的衰落</h2> 
<p>当然，Windows 保留了一些重要的软件，那些拥有超过10亿的用户基础，特别是在企业发挥关键作用的软件(然而，正如该公司在活动中强调的，疫情重新确立了个人电脑对消费者的重要性)。然而，Windows不再是微软业务的核心，这让微软未来有了更多发展空间，这也是微软 CEO 萨蒂亚·纳德拉最大的胜利。我在2018年的《<a target="_blank" rel="noopener noreferrer" href="https://36kr.com/p/1722401423361">微软重组的背后：未来属于“微软”，而不只是 Windows</a>》一文中描述了他是如何在改变微软以Windows为中心的模式的:</p> 
<blockquote> 
 <p>Windows 的衰落是一个相对简单的故事，也是一个经典的颠覆案例：</p> 
 <ul class=" list-paddingleft-2"> 
  <li><p>互联网极大地减少了对应用程序的锁定</p></li> 
  <li><p>个人电脑变得“足够好”，延长了升级周期</p></li> 
  <li><p>智能手机首先解决了个人电脑无法满足的需求，然后随着时间的推移，开始“接管”个人电脑的功能</p></li> 
 </ul> 
 <p>不过，更有趣的是Windows在雷蒙德研究院（Redmond）衰落的故事。上周的重组过后，是1980年以来的第一次，微软没有专门负责个人电脑操作系统的部门（Windows被拆分，核心工程组被安置在 Azure 下，而其他的团队实际上处于 Office 365之下；后续仍然会有Windows版本发布，但它不再是独立业务了）。在五年前，这种情况似乎不太可能实现，在另一次重组时，前首席执行官史蒂夫·鲍尔默（Steve Ballmer）写了一份备忘录，坚持认为Windows是（微软的）未来······</p> 
 <p>因此，我的观点是，微软接受Windows的衰落的现实，比Windows的衰落更有趣。这就是首席执行官萨蒂亚·纳德拉如何说服微软接受这一显而易见的事实的过程。</p> 
</blockquote> 
<p>这篇文章其余部分讲述了纳德拉是如何带领微软走到这一步的，包括在iPad上安装Office软件，他的几份战略备忘录，以及抛弃 Windows Phone；我知道，说什么会成为极好的商业案例研究是陈词滥调了，但这真的是一个伟大的商业案例研究!</p> 
<p>纳德拉的转型策略促成了 Windows 的新“开始”，这个词是乔安娜·斯特恩(Joanna Stern)对纳德拉采访时的表述。</p> 
<p>我这里所引用的“开始”是指新系统将“开始”按钮移动到屏幕中心位置，同时也指Windows 面临的新机会，这个机会之所以存在，是因为过去的一切都已经结束。</p> 
<h2>苹果的自救</h2> 
<p>20年前，对比 Windows 和 Mac，苹果的问题是缺少开发者，但最重要的原因是没有足够多的用户。苹果解决问题的方式正如你对苹果的期望——依靠自己。在史蒂夫·乔布斯时代，最聪明但最被低估的举动之一是 iLife 应用套件的开发——<a class="project-link" data-id="523791" data-name="IMO" data-logo="https://img.36krcdn.com/20200916/v2_811751a081924fa9af8741ce120bd7bf_img_png" data-refer-type="2" href="https://36kr.com/projectDetails/523791" target="_blank">imo</a>vie、iPhoto、iTunes、iDVD 和 GarageBand。我在去年的《<a target="_blank" rel="noopener noreferrer" href="https://36kr.com/p/970913007053057">从 1.0 到 3.0：苹果差异化战略如何演变？</a>》一文中指出:</p> 
<blockquote> 
 <p>OS X 将软件推到了最前沿，它不仅提供了一个技术上完善的操作系统，而且还提供了一个基于 Unix的操作系统，这使得它对开发者特别有吸引力。而在消费者端，苹果发布了iLife，这是一套让Mac对普通用户有吸引力的应用软件。我自己也是在这个时候买了第一台Mac，因为我想用GarageBand；16年过去了，我的音乐梦想不再，但我用Mac的习惯还在。</p> 
 <p>当我我第一次买Mac时，尽管它的硬件已经足够吸引人了：虽然我的iBook（苹果电脑公司出品的一款笔记本电脑）足够吸引人，但它的处理器是摩托罗拉G4，与英特尔的x86处理器相比没有丝毫竞争力；当年晚些时候，乔布斯做出了一个当时令人震惊但回想起来却很明显的决定，将Mac转向英特尔处理器。在这种情况下，拥有和业界其他所有人一样的硬件，对苹果来说是一个很大的胜利，更能让他们蓬勃发展的软件差异化大放异彩。</p> 
</blockquote> 
<p>这篇文章是写在苹果M1 Mac发布之初，主要讨论了苹果在软件方面的差异化战略，尤其是在 Mac 软件产品中的差异化战略，很多年来变化很少。这正好解释了苹果公司的差异化战略转移回到了硬件上。有趣的是，微软的侧重点放在了另一面：苹果的芯片在多大程度上优于x86(或其他ARM芯片)，而Windows在多大程度上就劣于X86。</p> 
<h2>比尔·盖茨线</h2> 
<p>和苹果一样，微软也在用自己最擅长的事情做出回应，但因为它是微软，它的做法与苹果恰恰相反：他们并不是为了吸引消费者而进行更深入的整合和行动，而是为了吸引开发者以及那些不想被束缚在苹果生态系统中的消费者而开放思路并消除限制。</p> 
<p>纳德拉在 The Verge 杂志上与尼莱·<a class="project-link" data-id="429006" data-name="帕特尔" data-logo="https://img.36krcdn.com/20201021/v2_5168a584d1884132ada5a675f6289d53_img_000" data-refer-type="2" href="https://36kr.com/projectDetails/429006" target="_blank">帕特尔</a>(Nilay Patel)的一次精彩采访中阐述了这一点：</p> 
<blockquote> 
 <p>帕特尔：这让我想到Windows的一些重大变化，这些变化从根本上说就是Windows将会采用哪种操作系统，你可以在Windows上开展什么样的业务，以及这一切会给微软带来什么。</p> 
 <p>Windows有一个新的用户界面，“开始”按钮位于屏幕中央，有一些表面的差异，但你也可以安卓应用程序上运行Windows。你还把亚马逊的安卓应用商店整合在了一起，你还对商店做了一些改变，把佣金减到15%，这与苹果饱受争议的30%的收费标准形成了对比。</p> 
 <p>然后你就对开发者说，“如果你想要使用自己的付费模式，你可以进入我们的商店，但却不需要向我们支付任何佣金。”这其中有多少是投机取巧的变化——你看到了争议，觉察到了市场机遇？这在多大程度上是转移业务的正确方式?</p> 
 <p>纳德拉：我认为这是竞争推动的。我的意思是——微软应该做些什么来管理平台和平台规则，让自己能茁壮成长呢?</p> 
 <p>我对平台作用的理解是：它们必须为那些在平台上开发的人创造机会，这是让他们与平台建立联系的方法。如果你给别人创造了一个契机，让他们在你的平台上生根发芽并茁壮成长——我觉得这样的微软是我想看到的，是我所在的Windows 应该做的。无论是 Adobe 用户创建他们的 Creative Cloud、SAP，或者搭建ERP系统，还是在当今世界，在 Windows 上为玩家创建社区还是其他业务，都离不开Windows。</p> 
 <p>对我来说，我所关注的是如何让 Windows 百尺竿头更进一步?</p> 
 <p>我觉得这是一个真正的机会，因为另外两个初显规模的生态系统，由于它们自身内部差不多的原因，已经将平台和聚合层用一套规则合并了——至少在我看来是这样的。其实，没有理由要有一套规则，完全可以分解开来。毕竟，我们都有自己的业务范畴。你可以全用，也可以自己带，这对微软来说是一件很实际的事情。</p> 
 <p>在这里，我并不是想做什么道德评判，说微软在这方面做得好，而其他公司则不然。别人选择它自有选择它的理由，这是设计的选择和商业模式的选择。我想做出我们自己的一套设计和商业模式选择，让创作者找到更多的选择。</p> 
</blockquote> 
<p>我们很清楚的知道一个事实，微软在应用商店方面绝对没有什么可损失的。他们并没有从Windows 应用商店中赚到多少钱，而且他们(仍然)需要为自己的平台开发更多的应用软件。与此同时，纳德拉的历史框架非常符合微软一直以来对Windows的看法。我把它叫做<a target="_blank" rel="noopener noreferrer" href="https://36kr.com/p/1722543996929">比尔·盖茨线</a>：</p> 
<blockquote> 
 <p>在过去的几个星期里，我一直在探索平台和聚合者之间的区别，这让我想起了在塞<a target="_blank" rel="noopener noreferrer" href="https://36kr.com/projectDetails/258858">米尔</a> · 沙阿（Semil Shah）采访查马斯·帕里哈皮蒂亚(Facebook前用户增长副总裁)时的一则轶事:</p> 
 <p>塞米尔 · 沙阿 :你有没有看到你在 Facebook 上的时间和 Facebook 平台的连接有什么相似之处？以及，Uber 是如何掌控它们的平台的？</p> 
 <p>查马斯:它们都不是平台。它们的努力就像把你作为第n个优先事项一样滑稽。我负责Facebook平台。我们大张旗鼓地宣传它，好像这是一个热门的大买卖。我还记得3、4个月后，我们从比尔· 盖茨那里筹集资金的时候，盖茨说了一些话:“那是一堆狗屎。这不是平台。平台是指每个使用它的人的经济价值，超过创建它的公司的价值。然才是一个平台。”</p> 
 <p>按照这一标准衡量，Windows 确实是终极平台——微软过去常常宣传自己只占Windows生态系统总价值的一小部分——而操作系统的明显继任者是亚马逊网络服务和微软自己的Azure云服务。在这三种情况下，都有一些强大而持久的业务建立在上面。</p> 
</blockquote> 
<p>纳德拉在采访中引用了这一点：</p> 
<blockquote> 
 <p>纳德拉：以微软为例，我一直认为平台的定义至少应该是：如果无法诞生比平台更大的东西，那么它就不是一个平台。网络是在 Windows 平台上发展起来的。如果我们说，“所有的商业都是通过我们进行的”，如果我们说，“我们要有自己的商业模式”，亚马逊就不可能存在。</p> 
 <p>因此，我认为每个公司都必须选择并观察什么聚合层次、平台层次以及什么规则适合他们和他们的生态系统。但是，对于我们来说，很明显我们想要解决同样的安全性问题，可发现性问题，因为这是我们强调商店的原因之一。与此同时，使不同的创建者可以使用不同层次的商店。我们希望这种灵活性能成为一个具有竞争优势的差异化战略。</p> 
</blockquote> 
<p>需要说明的是，这里有一些明确的推理；当被问及微软是否会允许<a class="project-link" data-id="3968996" data-name="谷歌" data-logo="https://img.36krcdn.com/20200916/v2_811751a081924fa9af8741ce120bd7bf_img_png" data-refer-type="2" href="https://36kr.com/projectDetails/3968996" target="_blank">谷歌</a>的 Play Store 在Windows上运行时(答案是肯定的)，纳德拉列举了多个游戏市场存在的例子，包括微软商店、XBox Game Pass、Steam和Epic。我可以告诉你的是，我10年前用微软商店的初始版本时，许多人都对Steam创造了微软认为它应该拥有的内容感到震惊；如果该公司首先意识到应用商店的商机会怎样?如<a class="project-link" data-id="228998" data-name="果事" data-logo="https://img.36krcdn.com/20200729/v2_d2c335f0a36e431bb7f07b53fb52a2f2_img_000" data-refer-type="2" href="https://36kr.com/projectDetails/228998" target="_blank">果事</a>关数十亿美元，它在竞争模式上还会如此漫不经心吗?</p> 
<p>答案或许是否定的，但我认为这是因祸得福。除了从经济角度，我们很难从其他角度看出App Store对苹果的正面影响；这不仅仅是一种战略税，而是一种文化税，它不仅破坏了公司与开发者的关系，而且实际上给用户带来了更糟糕的体验，而且还带来了巨大的监管风险。</p> 
<h2>Windows的回归？</h2> 
<p>早在2005年，保罗·格雷厄姆就写过《<a target="_blank" rel="noopener noreferrer" href="http://www.paulgraham.com/mac.html">Mac的回归</a>》:</p> 
<blockquote> 
 <p>我认识的所有最好的黑客都在逐渐转向Mac。我的朋友罗伯特说，他在麻省理工学院的整个研究小组最近都给自己买了Powerbooks。这些人不是90年代中期苹果低谷时购买Mac电脑的平面设计师和老奶奶，他们是核心操作系统的黑客。原因购买Mac 的原因当然是因为OS X。Powerbooks 设计精美，可以运行FreeBSD。你还想知道什么?</p> 
 <p>如果你想吸引黑客来编写软件，帮助你销售软件，那么你必须让他们自己也使用你的软件。仅仅“开放”是不够的，它必须是开放和优秀的。而开放和优秀是Mac的最终特征。</p> 
</blockquote> 
<p>Windows是开放的，你可以在上面写软件，你可以运行软件，你可以打造自己的商业模式；微软已经向所有人敞开大门。这就引出了下一个显而易见的问题：这是好事吗?当然，大家对这种方法的意见不统一；帕特尔问纳德拉，反对Windows上的安卓应用程序的最佳理由是什么:</p> 
<blockquote> 
 <p>纳德拉：我认为争论的焦点总是围绕着，“我们必须要有一个固定的应用程序模式吗?”因为如果你想创新——有没有哪种NUI或AI芯片是我们想使用的?我们以这样的方式调用API 可以运行应用程序？当你有多个子系统和多个应用模型时，你能在自己的平台进行系统创新，调用所有的应用程序吗?</p> 
 <p>这将是这个世界面临的根本性挑战，但我们觉得有路可循。在我看来，其中一种可取的做法是，通过一些新的AI API，你将来或现在可以在 Windows 上使用安卓应用程序或PWA应用程序。</p> 
 <p>在微软，我们为iOS开发、为安卓开发，也为Windows开发，这是(我们的)基本挑战之一。我们努力确保，作为开发者，我们可以利用尽可能多的公共代码基础、尽可能多的云，但同时又保持每个平台的本土化。</p> 
</blockquote> 
<p>纳德拉的最后一点实际上回答了保罗·格雷厄姆的问题：这重要吗?苹果通过硬件实现 Mac 的差异化战略的转变，并仅仅是因为M1功能，而是因为Mac 和其他电脑上绝大多数基本软件都是网络应用程序，或者运行在像Electronic这样的应用程序上；然而，在本土化最重要的高性能游戏领域，Windows仍然占据着主导地位。</p> 
<p>也许这就是为什么这次演讲感觉如此有趣和轻松的原因：它可能没那么重要。Windows 是一项必不可少的业务，但纳德拉降低了它对微软的重要性，不是出于怨恨，而是因为世界在微软改变之前就已经改变了。</p>
