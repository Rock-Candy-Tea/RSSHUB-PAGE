<p>作者｜吴思瑾</p> 
<p>编辑｜王与桐</p> 
<p>36氪获悉， 定位于 MarTech 赛道，专注于为云服务商提供智能获客服务的 FORK AI ，于近日公布了由绿洲资本独家投资的数千万元人民币 Pre - A 轮融资。</p> 
<p>企业或个人开发者在开发软件的过程中，除了按自己意图开发特定的功能外，在一些通用功能上，还会直接使用免费的开源 SDK ，或向云服务商采购相关的 SDK 。</p> 
<p>SDK（Software Development Kit ）全称为软件开发工具包，是由不同类别的云服务商开发的专用于某一类软件的相关文档、范例和工具的集合；企业或个人开发者通过 API 安装第三方 SDK ，避免重复开发通用功能，缩短开发周期。</p> 
<p>以下图为例，企业开发者开发的小红书应用，使用了来自云服务商腾讯旗下的 3 款 SDK 。其中，腾讯 QQ SDK 和腾讯微信 SDK 分别实现了支持使用其账号登陆小红书的功能；腾讯 Bugly SDK 则具有监控应用 bug 的功能。</p> 
<p class="image-wrapper"><img data-img-size-val="864,743" src="https://img.36krcdn.com/20220417/v2_a1c18559ace64929b69d2205e8792fd0_img_png" referrerpolicy="no-referrer"></p> 
<p class="img-desc">图源：Fork Stack智能获客引擎产品截图</p> 
<p>本文报道的<strong> FORK AI 打造了一款基于云原生的通用 SaaS —— Fork Stack 智能获客引擎，旨在解决云服务商的精准获客问题。</strong>之所以选择这个赛道，与 FORK AI 创始人何李石的从业经历有关，其曾在第三方独立云计算及数据服务<a class="project-link" data-id="1678231147951113" data-name="七牛云" data-logo="https://img.36krcdn.com/20200616/v2_b65b7020a706422faa1aa58d21f61c7a_img_000" data-refer-type="1" href="https://36kr.com/project/1678231147951113" target="_blank">七牛云</a>中担任商业化产品经理和客户成功负责人，在支持销售团队的过程中，其发现当下主流的销售漏斗式获客方式和销售与客户技术负责人之间的技术鸿沟成为了制约云服务商获客效率的重要因素。</p> 
<p>主流的获客方式是，云服务商主要通过在 SEM （搜索引擎营销）和 DSP （需求方平台）投放广告来获取销售线索，销售从 CRM 中认领客户并逐一询问采购需求；但往往由于线索不够精准，导致浪费人力资源和销售效率低的情况。</p> 
<p>技术鸿沟是，由于云服务技术复杂，初级销售往往难以精准把握企业开发者的实际需求，形成面对销售提供的服务方案时，技术负责人往往难以抉择的尴尬局面。</p> 
<p>基于上述痛点， <strong>Fork Stack 智能获客引擎通过数据挖掘和逆向拆解应用背后的 SDK 分布和 API 变动情况，打破云服务商与企业开发者技术负责人之间的技术鸿沟，用可视化的方式为云服务商精准定位开发者需求，结合工商信息辅助判断销售机会，提高获客效率。</strong></p> 
<p>具体来说， <a class="project-link" data-id="1702301369602821" data-name="Fork" data-logo="https://img.36krcdn.com/20220417/v2_46d32b5e3826417eb11fedf22204b163_img_000" data-refer-type="1" href="https://36kr.com/project/1702301369602821" target="_blank">Fork</a> Stack 智能获客引擎覆盖了5大主流应用商店，通过数据挖掘和逆向应用拆解技术，分析应用背后的 SDK 、 API ，可视化展示某个公司主体旗下开发了多少款应用、分布于哪些应用商店、每款应用的 SDK 构成统计和 SDK 分别属于哪家云服务商；以及按时间维度呈现该应用的 SDK 安装、卸载变动情况；以此监测应用对于云服务的需求所在。</p> 
<p class="image-wrapper"><img data-img-size-val="3840,2160" src="https://img.36krcdn.com/20220417/v2_7bd332f70b314153b7389f8a9210cff5_img_jpg" referrerpolicy="no-referrer"></p> 
<p class="img-desc">图源：Fork Stack智能获客引擎产品截图</p> 
<p><strong>“该模式的价值在于通过精准获客的逻辑，帮助云服务商的获客从海量线索中聚焦至 5 大主流应用商店的 250 万开发者的 800 多万个应用中。”何李石介绍道。</strong></p> 
<p>对市场部门而言，可通过组合筛选多维度数据进行行业分析和生成开发者客户线索。包括准确统计云服务商企业的 SDK 产品所处领域的市场占有率和竞争对手客户画像、市场分布分析，挖掘企业产品的下一个市场增长点；同时支持构建和导出目标客群列表和竞争对手客户列表和表现分析。</p> 
<p>对销售部门而言，可筛选不同开发者旗下应用分布状况和分析目标开发者的应用排行变化预估下载量变化；查看 SDK 构成和现有 SDK 云服务商分析，以此推断其云服务需求变化和体量，定制销售策略。</p> 
<p>对客户成功部门而言，可实时了解开发者客户的市场表现和 SDK 动态，制定续费、复购、召回等运营策略。</p> 
<p>对比现有专业调研公司通过调研访谈和桌面研究的做法， Fork Stack 智能引擎的优势表现在数据展示的全面性、客观性和实时性上。</p> 
<p>此外，开发者详情页面还可帮助云服务商结合企业工商信息辅助判断销售机会，提高销售效率。</p> 
<p><strong>据何李石介绍，“区别于传统销售漏斗式获客， FORK AI 是市面上目前唯一一家通过标准化产品为云服务商规模化提升获客效率的企业。”</strong>这种方式固然好，但换个角度来看意味着市场空间较小且市场教育将存在一定难度。为此，何李石表达了自己的看法：</p> 
<p>首先，目前确实因为方式太新，在做产品推荐时需要花费较多篇幅介绍，但云服务商一旦理解产品逻辑后就很容易接受。</p> 
<p>其次， FORK AI 团队对云服务厂商的需求有着深刻的理解和洞察，目前产品模式已经基本跑通且通过数家客户的实操验证，在行业 Know - How 、技术和服务经验等方面已经积累了相当的经验。</p> 
<p>再次，未来， FORK AI 还将继续加大对云服务供应链的标准化，打通双边需求，为开发者提供面向云服务商的安全分析服务，最终实现类似 BOSS 直聘供求双方互选的模式，进一步促进云服务商销售转化。</p> 
<p>目前， FORK AI 以 SaaS 订阅费作为主要收入来源，其中包含企业定制服务和个人订阅服务，在企业定制服务方面按数据量进行收费。去年已实现收入近百万，客单价 30 万左右，且均已再次续订。</p> 
<p>团队上，尚处于初创期的 FORK AI 不足 20 人，主要来自于国内外优秀云计算公司及优秀开源社区、网络安全专家等，平均行业经验 10年 以上。</p> 
<p>据介绍，本轮资金用途为升级产品和扩大商业化，除自有销售渠道外， FORK AI 还将以 CRM 厂商为渠道进行拓客。</p> 
<h3><strong>投资人说</strong></h3> 
<p>绿洲资本投资负责人表示：“大家对消费互联网公司利用数据精准获客耳熟能详，企业服务虽百花齐放，但实际大量公司的获客还处于粗放阶段， FORK AI 抓住了运用数据提升对客户需求洞察的关键。何李石出身七牛云早期创业团队的一员，拥有多年国内企业服务公司的经历以及对全球企业服务的发展之洞见，绿洲期待和有全球化视野的  FORK AI 团队一起成长，赋能企业服务。”</p> 
<p>险峰长青副总裁杨阳表示：“长期来看，在中国云服务渗透率快速增长的背景下面，云基础设施企业的数量在未来也将持续增加，行业粗放发展的同时，上下游存在着信息极大不对称。 FORK AI 独创的需求挖掘匹配引擎，为客户呈现完整云服务供应链信息，帮助企业更高效以及低成本开拓市场。李石作为七牛云早期布道师，见证了云服务在国内的崛起，也兼顾国际视野，我们相信不久 FORK AI 会成为全球云服务领域的标配增长工具。”</p>
