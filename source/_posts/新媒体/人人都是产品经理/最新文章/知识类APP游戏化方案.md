<blockquote><p>编辑导语：知识付费时代，知识的获取变得更简单、更便捷。不论是学生党还是上班族，一定都接触过不少知识类APP，这类APP假如形式和内容很枯燥，便不能吸引用户，因此需要进行游戏化，增加趣味性，本文作者为我们分享了他的思考方案。</p></blockquote>
<p><img data-action="zoom" class="aligncenter size-full wp-image-4433394" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/03/CGyRon1hYt1RQCnzxbfy.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<h2 id="toc-1">一、明确产品目标和用户需求</h2>
<ul>
<li>产品目标：传播知识，让人们学会有用的东西进而成为更好的人。</li>
<li>用户需求：通过习得知识来提升自己，进而能够运用到学习、生活和工作中，使其变得更好。</li>
</ul>
<p>商业目标分四种模式：</p>
<h3>1. 广告</h3>
<p>几乎是百搭的模式，主要在于控制投放的力度（投放频率、投放页面、投放范围），哪个平台都可以考虑。需要更高的用户活跃量，更精准的投放及更高的点击率。</p>
<h3>2. 内容付费</h3>
<p>对于能抢到优质内容资源、且能和免费内容资源形成明显的高低档位差的平台，可以考虑。需要付费内容比免费内容更优质、对用户帮助更大。在此前提下，引导用户不断购买付费内容是最终的目的。</p>
<h3>3. 学习模式</h3>
<p>会员付费对于平台内容有一定的保证，且将学习工具打造得非常高效实用的平台，可以考虑。</p>
<p>如果可以保证游戏化的学习模式会让用户学的更多，吸收的更好，那么可以将游戏化的学习模式设置为付费项目。用户注册后赠送一个月会员，引导用户为此模式付费及续费是最终目的。</p>
<p>要求：保证所有内容对所有用户免费开放，传统的学习功能免费（收藏、点赞、评论、笔记等），收费的只有游戏化激励模式涉及到的功能。</p>
<h3>4. 内容订阅付费</h3>
<p>对于有持续的优质内容资源供应的平台，可以考虑。如果对平台内容的价值性和学习工具的高效性有信心，则可以考虑订阅制，届时，引导用户订阅付费和续费就是最终目的。</p>
<p>先看看短期内无法用内容付费模式取得良好收效的产品，比如拿不到优质的付费内容，比如刚起步品牌背书比不过那些行业大哥的的产品，或者品牌定位本身就是传播免费的学习内容的产品，那么，打造一套高效且吸引用户的学习工具可以作为一种使产品存活下去的出路。</p>
<h2 id="toc-2">二、定义业务指标</h2>
<ul>
<li>会员每日、周、月学习频率；</li>
<li>会员每日、周、月学习时长；</li>
<li>日活会员用户量、及对日活的占比；</li>
<li>日活会员用户中新会员用户量，及对日活的占比；</li>
<li>未到期提前续费的用户量，及对日活会员的占比；</li>
<li>会员到期后一天、两天、三天、一周、两周、一月内、一月外续费的用户量、及对日活会员的占比；</li>
<li>会员用户累积开通时长为一月、两月、三月、三月~半年、半年~一年，一年以上的用户量，及对历史会员用户总量的占比；</li>
<li>到期后再未付费的流失会员量，及对历史会员总量的占比。</li>
</ul>
<h2 id="toc-3">三、定义用户类型</h2>
<ul>
<li>想探索这个世界，对世界充满了好奇的人；</li>
<li>想拓宽自己的知识面、或者精进自己的技能、或者加深自己对某个领域的了解，以达成生活中某项目标的人；</li>
<li>想将自己的知识传递给更多的人，对这个世界有所贡献的人；</li>
<li>想和他人交流自己的知识和观点的人。</li>
</ul>
<h2 id="toc-4">四、定义期望行为</h2>
<p>完成期望行为后会来到胜利状态。</p>
<h3>1. 发现阶段</h3>
<p>核心：让用户快速get到使用产品的原因</p>
<ul>
<li>通过广告宣传、朋友推荐、或他人分享的知识内容下载APP；</li>
<li>在App中找到并学习自己感兴趣的知识；</li>
<li>注册/登录。</li>
</ul>
<h3>2. 入门阶段</h3>
<p>核心：让用户开始学习游戏玩法和工具</p>
<ul>
<li>对学完的知识内容进行分类收藏</li>
<li>对学完的知识内容进行表态</li>
<li>对学完的知识内容做笔记</li>
<li>对学完的知识内容发表评论</li>
<li>分享学完的知识内容</li>
<li>查看某知识主题的主页</li>
<li>关注某主题</li>
<li>查看某知识单的主页</li>
<li>关注某知识单</li>
<li>查看某知识提供方的主页</li>
<li>关注某知识提供方</li>
<li>查看关注的内容更新</li>
<li>查看推荐的内容更新</li>
</ul>
<h3>3. 塑造阶段</h3>
<p>核心：促进用户重复进行日常行为以完成他们的目标</p>
<ul>
<li>每天学完一份知识内容并收藏</li>
<li>每天对一份知识内容进行表态</li>
<li>每周做三份知识笔记</li>
<li>每周发表三条评论</li>
<li>每周分享一份学完的知识内容</li>
<li>每周学完任意主题下的两份知识内容</li>
<li>每周学完任意知识单内的两份知识内容</li>
<li>每周学完任意知识提供方的两份知识内容</li>
</ul>
<h3>4. 终局阶段</h3>
<p>核心：加固加深产品体验，留住忠实用户</p>
<ul>
<li>整理现有知识内容，创建知识单发布出来，让其他用户也可以按照此单学习；</li>
<li>根据做的笔记，整理知识长文发布出来，供其他用户学习；</li>
<li>参与知识测试；</li>
<li>创建并执行学习计划；</li>
<li>加入学习小组，完成团队学习任务，参与小组讨论；</li>
<li>参与某知识测试题的创建，提交测试题；</li>
<li>创建学习小队，布置团队学习任务，组织小组讨论；</li>
<li>制作知识内容（视频、音频、图文）并发布；</li>
<li>帮助新手用户更高效的学习。</li>
</ul>
<h2 id="toc-5">五、定义反馈机制</h2>
<p>反馈机制明确了用户胜利状态的进度，让用户感受奖励的不确定性，每个反馈机制都应该带来新的触发器，进而继续触发期望行为。</p>
<h3>1. 哪些位置可以植入反馈机制和触发器？</h3>
<ul>
<li>知识内容详情页</li>
<li>站内信</li>
<li>通知栏消息</li>
<li>首页</li>
<li>我的页</li>
<li>APP内的全局提示bar</li>
</ul>
<h3>2. 可以应用哪些反馈机制？</h3>
<ul>
<li>进度条</li>
<li>任务板</li>
<li>任务完成提示</li>
<li>角色成长图表</li>
<li>不同主题的知识等级</li>
<li>角色称号表</li>
<li>知识图谱</li>
<li>成就象征墙</li>
<li>收藏集</li>
<li>随机神秘宝箱</li>
<li>他人的互动消息提示</li>
<li>影响力分值</li>
<li>贡献力分值</li>
<li>探索力分值</li>
<li>证书册</li>
<li>活跃度系统</li>
<li>待解锁的任务与功能</li>
<li>计划结束的倒计时</li>
</ul>
<h2 id="toc-6">六、定义奖励和激励</h2>
<ul>
<li>任务进度</li>
<li>完成任务后的经验值</li>
<li>连续完成任务后的活跃度</li>
<li>角色等级</li>
<li>知识分支等级</li>
<li>点亮的知识分支</li>
<li>角色称号</li>
<li>新解锁的任务和功能</li>
<li>完成一定任务后获得的成就象征</li>
<li>完成任务后随机出现的收藏品</li>
<li>宝箱中随机出现的奖励</li>
<li>他人的互动</li>
<li>影响力分</li>
<li>贡献力分</li>
<li>探索力分</li>
<li>测试通过的证书</li>
<li>学到的知识</li>
<li>学到知识的成就感</li>
<li>完成计划的成就感</li>
<li>帮助别人得到的成就感</li>
<li>团队荣耀感</li>
<li>排名</li>
<li>某一知识分支领域内的领头羊</li>
<li>免费的会员卡</li>
<li>会员卡优惠券</li>
<li>合作的广告商的商品</li>
</ul>
<h2 id="toc-7">七、安排游戏玩法</h2>
<p>从八大核心驱动力角度安排：</p>
<h3>1. 史诗意义与使命感</h3>
<p>通过广告宣传和启动引导，告诉用户，人类文明之所以会不断发展，正是因为对知识的不断探索、积累、运用和传播，我们每个人都是一环，每个人都参与其中，承担起自己的角色是我们的责任。</p>
<h3>2. 进步与成就感</h3>
<ul>
<li>不要让用户困惑下一步该做什么，适时给予引导；</li>
<li>任务完成时给予完成奖励，并显示总体进度和接下来该完成哪个；</li>
<li>经验值、活跃度、影响力、贡献力、探索力会随着完成的任务而增加；</li>
<li>完成某些任务后，会获得成就卡；</li>
<li>随着经验值的增加，知识分支的会被点亮，知识等级也会增加，人物等级也会增加；</li>
<li>完成某些成就时，会获得特定称号。通过测试时，会得到通关证书。完成学习计划时，给予特殊的成就卡；</li>
<li>用户排行榜（只展示用户当前排名前后的用户得分，分设分时榜、进步榜）。</li>
</ul>
<h3>3. 创意授权与反馈</h3>
<ul>
<li>制定自己的学习计划并执行；</li>
<li>做学习笔记，并根据自己的理解，合并总结成知识长文；</li>
<li>制作知识内容并发布；</li>
<li>组建学习小队，布置小队任务；</li>
<li>贡献测试问题；</li>
<li>创建知识单；</li>
<li>达到特定等级时，新解锁特定的任务和功能。</li>
</ul>
<h3>4. 所有权与拥有感</h3>
<ul>
<li>在用户完成某任务后，有一定几率得到一张名言卡，随机出现一位名人的名言，会将其放入名人名言语录中，玩家需不断收集；</li>
<li>将用户已经累积看了多少知识、已经连续多少天学习知识、累积多少天学习知识、各项知识技能等级、探索力、影响力、贡献力分值、各种成就荣誉称号等等显示在个人主页，让用户方便查看；</li>
<li>给用户推送消息提醒用户查看自己的知识周报、月报、年报；</li>
<li>如果用户超过3天没有学习知识，推送消息提醒用户完成任务及查看个人记录；</li>
<li>通过收集用户的兴趣标签，为用户推荐感兴趣的学习内容。</li>
</ul>
<h3>5. 社交影响及关联性</h3>
<ul>
<li>引导用户创建学习小队并在小组设立团队任务，和团队一起努力完成集体目标，设立小队间的活跃度排名；</li>
<li>对于知识做的笔记可以对其他人可见，从对知识的态度将笔记分为四类：认可、反驳、疑问、思考中，其他人可以对此表态和发布评论，同时还可以将此笔记发布到学习小队和大论坛中，与更多人互动；</li>
<li>在获得成就卡、称号、证书等时，鼓励用户分享出去；</li>
<li>在个人主页里直观的展示成就、等级、称号等信息。让用户自定义除了个人主页外，头像和昵称下展示的内容；</li>
<li>用户对其他用户的内容可以进行点赞、收藏；</li>
<li>测试失败时，可以邀请他人赐予一张“复活卡”；</li>
<li>鼓励用户成为某某方向的新手导师，在新人注册后，为其推荐导师，并组成师徒小组，由师傅推荐给徒弟学习内容。</li>
</ul>
<h3>6. 稀缺性与渴望</h3>
<ul>
<li>周末学习经验翻倍活动；</li>
<li>对于低阶用户隐藏部分高阶功能，待解锁后再显示；</li>
<li>在用户取得阶段性奖励时，提示用户下一阶段奖励内容的大概；</li>
<li>用户每天能完成的任务数量有限制，知识或人物等级越高可完成任务量越多。</li>
</ul>
<h3>7. 未知性与好奇心</h3>
<ul>
<li>在用户第一次进入新功能模块时，给用户引导和指示，不让用户过多思考就能顺利的进入下一步期望行为；</li>
<li>任务奖励中，保留一项随机的奖励，包括经验值、收藏集的卡片等；</li>
<li>设定一系列复杂的动态规则来向用户发放彩蛋宝箱，宝箱内含有特殊奖励；</li>
<li>每周都抽一轮奖品，包括会员卡、会员卡优惠券、经验值、广告商的产品等。</li>
</ul>
<h3>8. 亏损与逃避心</h3>
<ul>
<li>新用户注册后即送一个月会员，用户一个月中积累的成果和养成的习惯粘住用户，使之续费；</li>
<li>在双倍经验活动中设立活动结束倒计时；</li>
<li>在用户连续2天+完成任务后，如果第x天22点前没有完成当日任务，则发消息提示用户，即将中断连续记录。</li>
</ul>
