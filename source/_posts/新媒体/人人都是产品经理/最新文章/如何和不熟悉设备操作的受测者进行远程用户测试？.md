<h2 id="toc-1">一、远程用户体验测试</h2>
<p>自 2008 年以来，随着全球各地的移动设备和网络覆盖率大幅提升，在美国加利福尼亚州的硅谷，一种让UX研究员和测试参与者以远程形式进行的用户测试方法应运而生。参与者可使用移动端 / 电脑端参与测试，并在<strong>录制屏幕 + 音频 + 网络摄像头的同时执行一系列任务</strong>。每个任务之间，通常还会穿插量化指标问题。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/05/jikCE9b5x2g8IQcfmSgp.png" alt width="501" height="429" referrerpolicy="no-referrer"></p>
<h2 id="toc-2">二、当受测者不熟悉电脑或手机操作</h2>
<p>虽然远程用户测试让用研人员能更容易触及各类画像的受测者，但<strong>这些测试参与者未必熟悉操作手机或电脑，或是进行必要的技术设置以利于展开远程测试</strong>。</p>
<p>因此，设计完好的远程测试项目对于提高各类参与者的项目访问和执行能力非常重要。设计测试脚本时将「科技小白」类的参与者考量在内，能让更多参与者成功完成可用性测试，并带来更高质量的测试结果。</p>
<h3>诀窍一：保持测试任务清晰易懂</h3>
<p>在整个可用性测试过程中，<strong>使用清晰的字体且编排一致的文字让用户更舒适阅读</strong>。此外，<strong>使用简单易懂的说明和描述</strong>，以尽可能减少用户对任务和测试活动的误解。这点在自主式测试（unmoderated testing）中—没有访谈主持人指引参与者的情况下—更为重要。</p>
<h3>诀窍二：采用低技术操作的线上用户测试工具</h3>
<p>某些<strong>线上用户测试工具和服务允许不熟悉科技产品的受测者轻松参与远程测试</strong>。這些工具包二个关键类型测试软件：</p>
<ul>
<li>完全线上录制器（No-download ｗeb recorder）</li>
<li>覆盖式线上录制器（Overlay ｗeb recorder）</li>
</ul>
<p>通过这两种录制器软件，参与者只需一个链接和一个常用的网络浏览器，就可以于任意时间在家中或办公室加入测试。下面，我们将介绍一下这两个网络录制器方便使用的原因：</p>
<p><strong>（1）完全线上录制器</strong></p>
<p><strong><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/05/soxsYjIjNKtg50gez916.png" alt width="500" height="363" referrerpolicy="no-referrer"></strong></p>
<ul>
<li>不需要下载任何浏览器扩展或插件</li>
<li>支持 Chrome、Firefox、Safari 和 Microsoft Edge 以及任何主要操作系统，包括 Windows、Mac OS 和 Chrome OS</li>
<li>允许参与者在企业网络环境中（比如防火墙）使用，并增加参与者成功加入测试项目的几率</li>
</ul>
<p><strong>（2）覆盖式网络录制器</strong></p>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/05/YyONTlrVVFEVEjMmZ1hA.png" alt width="501" height="449" referrerpolicy="no-referrer"></p>
<ul>
<li>专门设计用于网站和产品原型的远程用户测试软件</li>
<li>需要下载一个网络浏览器扩展（一键快速处理）</li>
<li>支持 Chrome、Firefox、Safari 和 Microsoft Edge 浏览器</li>
<li>浮动的任务窗口设计</li>
<li>高运行稳定性</li>
</ul>
<h3>诀窍三：明确告知用户远程测试的硬件要求</h3>
<p>以往在传统的真人线下用户测试中，研究人员对测试用的设备、操作系统、互联网连接有绝对的控制，然而在远程测试环境中，需要考量参与者之间的硬件差异。因此，<strong>明确定义远程可用性测试的技术要求，并确保所有参与者在每次测试开始时都满足这些要求至关重要</strong>。这可以大大减少测试过程中用户因为硬件配置无法满足而退出的风险。</p>
<p>某些线上用户测试工具在测试正式展开前会告知参与者明确的硬件和技术要求（如浏览器、互联网连接、视频解析度等），并<strong>要求用户进行「测前计算机 / 手机设备配置测试」</strong>，以确保参与者能够满足配置要求。</p>
<h3>诀窍四：运用额外方法增加用户粘性，减少测试退出率</h3>
<p>参与者中途退出的情形大多发生在测试的开始阶段。影响参与者退出的因素有很多，比如加载时间、软件布局设计、测试物件和硬件要求所造成的问题。下面将介绍如何提高测试参与者的积极性和参与度，以减少测试退出率：</p>
<p><strong>（1）提供清晰的项目概述</strong></p>
<p>在测试一开始就清晰描述可用性测试的目标、目的和过程，好让参与者理解研究项目的背景，并为接下来的任务做好心理准备。阅读项目概述也可以增强受测者的内在动机和参与感。</p>
<p><strong>（2）告知项目估计时间和提示进度</strong></p>
<p>为测试提供总时间估计，并在测试过程中向参与者提示进度。预告估计时间可以让参与者了解完成任务需要多少时间和精力，而提示进度则可以提高参与者的成就感，让他们有更多动力继续当前测试。</p>
<p><strong>（3）奖励或酬劳</strong></p>
<p>奖励或酬劳可以额外激励受测者加入并完成用户测试项目。此外，从心理学角度的来说，针对用户表现做出反馈或赞赏，可以让受测者对整个测试体验更满意。</p>
<h3>诀窍五：试运行研究项目</h3>
<p><strong>在正式启动可用性测试前最好进行一次项目试运行，确保测试内容在不同的计算机、显示器和浏览器上正确显示</strong>，而且用户从加入测试、执行任务到最后上传结果的流程是流畅的。</p>
<p>此外，让多名不同的受测者参与试运行的项目，确保不同行业和背景的用户能轻松理解测试说明。这点在远程自主式测试项目中尤为重要，因为没有测试主持人会协助指引项目的进行。</p>
<h2 id="toc-3">结语</h2>
<p>在每个远程测试项目中，参与者的背景可能跨越不同文化，且拥有程度不一的软硬件操作能力。想要<strong>最大限度增强各类参与者的测试执行能力，选择好的远程用户测试工具以及设身处地考虑参与者的需求是非常重要的</strong>。</p>
