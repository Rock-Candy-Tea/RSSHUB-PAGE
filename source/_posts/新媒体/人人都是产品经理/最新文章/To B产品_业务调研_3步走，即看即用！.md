<p><strong>“</strong>没有调查，没有发言权！不做业务调研，不能随便YY需求。<strong>”</strong></p>
<p><strong>一则新闻：</strong></p>
<p>陈龙博士为了完成论文，加入了中关村的一家外卖骑手团队，花了5个月时间进行田野调查。在今年出版的博士论文中，陈龙有一个总结性的论断：外派平台在压缩配送时间上永不满足，它们总在不断试探人的极限。</p>
<p>“<strong>没有调查，就没有发言权！</strong>”，不论是做科学研究，问题论证，没有前期的调研，就无法发表有说服力的言论。</p>
<p>同样，对于做软件产品，<strong>没有业务调研，就不能做出符合客户需要的产品</strong>。产品人不能YY需求，所有的需求都要能够满足业务场景，具备价值。</p>
<p>To C产品的用户调研方法论已经有很多了，To B产品的业务调研，在方法上，可以有一些共用，但在内容上相差甚远。</p>
<h2 id="toc-1">一、为什么业务调研</h2>
<h3>1. 为什么要业务调研</h3>
<p>To B的产品面向是企业客户，能够深度洞察不同行业不同客户的经营管理模式和用户诉求，是一个成熟产品经理必须拥有的技能。</p>
<p>业务调研可以很好地帮助产品经理快速获取目标客户的想法，从而让产品顺利落地。主要有以下几点：</p>
<ul>
<li><strong>业务调研是源头</strong>：从业务中来，到业务中去，好的产品设计起源于业务调研，避免我们闭门造车，而从一开始就设计出符合企业诉求的产品。</li>
<li><strong>业务调研是洞察企业业务模式和用户心理的有效手段</strong>：可以深入理解业务场景和业务故事，明确用户使用习惯和诉求。</li>
<li><strong>业</strong><strong>务调研是B端产品经理沉淀业务知识有效途径</strong>：懂业务是B端产品经理的一个坎，是必须掌握的。</li>
</ul>
<h3>2. 业务调研整体框架</h3>
<p>“<strong>凡事预则立，不预则废</strong>”。在调研准备开展业务调研时，脑海里先要有一个整体的业务调研框架，这样不至于调研过程中手忙脚乱，毫无章法。</p>
<p>有了框架能够让我们有更好的体系化思路，更好的调研节奏把控，更好的全方位调研，更强的自信心，达到我们最初的调研目的和效果，甚至超出预期。下面我给出一个自己整理过业务调研框架，供大家参考：</p>
<p><img data-action="zoom" class="rich_pages aligncenter" title="《To B产品“业务调研”3步走，即看即用！》" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/08/x6uRRUMblR5GEAtbUP7e.png" alt="《To B产品“业务调研”3步走，即看即用！》" width="707" height="225" referrerpolicy="no-referrer"></p>
<p>主要包括<strong>知己知彼、调研前、调研中和调研后</strong>四个方面，从认知、心理、技能、方法几个方面进行了阐述和说明。接下来，我们一起开启调研之旅！</p>
<h2 id="toc-2">二、知己知彼</h2>
<p>孙子兵法曰“知己知彼，百战不殆”。知己知彼意思就是说对敌我双方的情况了如指掌。用在业务调研上，就是我们既要知道我们本身的优劣势，也需要知道我们服务的目标客户的情况。分开来说，就是“知己”，知道自己，“知彼”，洞悉对方。</p>
<h3>1. 知己</h3>
<p>知道自己的情况，可以从以下几个方面开始，包括不限于：</p>
<ul>
<li><strong>人</strong>：我们拥有的各种什么样的人，是能人吗？</li>
<li><strong>技能</strong>：我们的人具备什么技能，能够开疆扩土吗？</li>
<li><strong>心态</strong>：我们的人具备什么样的心态，具备激情吗？</li>
</ul>
<p><strong>人方面</strong>：盘点我们拥有的能人，他们是否具备独立业务调研和对行业和客户有足够了解的视野和洞察力。</p>
<p><strong>技能方面</strong>：我们是否有成套的技能方法体系供学习，我们的人是否具备这方面的技能，技能掌握的熟练度如何。</p>
<p><strong>心态方面</strong>：调研是一项有节奏的工作，是需要与单个人或一群人进行沟通，调研者需要有大局观，具备足够的激情、开放的心态容纳不同的意见。</p>
<h3>2. 知彼</h3>
<p>能够洞察目标行业和客户，了解他们的运作模式和想法，能够更好的在调研过程中挖掘用户真正的诉求。</p>
<p>这里包括了解和掌握三个方面：<strong>被调研对象的行业情况，被调研对象公司情况，被调研对象的领域情况</strong>。</p>
<p>本文简单地阐述一下三个方面，如何在调研之前快速掌握这些信息，可参考《被业务怼？想改变？你要的To B产品“如何快速搞懂一家公司的业务”！别再错过！》</p>
<ul>
<li><strong>被调研对象行业情况</strong>：行业情况也就是目标客户行业所属、市场规模、所处阶段，以及市场竞争格局。</li>
<li><strong>被调研对象公司情况</strong>：公司情况也就是组织架构、商业模式、运营模式、产品矩阵和护城河等情况。同时也需要了解公司的组织架构、岗位职责、KPI和业务流程等。</li>
<li><strong>被调研对象领域情况</strong>：你的产品聚焦解决客户哪个领域问题，你要提前去了解客户该领域的痛点和问题。</li>
</ul>
<h2 id="toc-3">三、百战不殆</h2>
<p>进入实际的业务调研中，就是如何做到“百战不殆”，每次调研都能够达到调研的目的和效果。这里分为三个阶段完成一次业务调研的全生命周期。</p>
<h3>1. 调研前</h3>
<p>调研之前，先做好准备和计划。计划不是万能，没有计划万万不能。这里主要包括五个方面的准备：<strong>目的、对象、方法、资源，和计划</strong>，下面分别进行说明。</p>
<p><strong>1）调研目的</strong></p>
<p>顾名思义，就是本次调研的目标，解决为什么要做业务调研，期望解决什么问题，达到什么样效果。</p>
<p>产品不同阶段，调研目的侧重点也是不一样，如下表：</p>

<p>常用的<strong>调研思路</strong>：从战略层、战术层和执行层三个方向把握战略定位和目标，管控组织及发展方向现状了解和分析，核心业务关键成功因素分析，人力资源布局分析，流程管理及流程体系现状分析；信息化建设的现状及主要提升点分析等。如下图：</p>

<p><strong>2）调研对象</strong></p>
<p>B端产品的调研对象，涉及到企业运营的多个角色，包括<strong>基层、</strong><strong>中层、高层</strong>多个用户。这里主要从<strong>利益相关者和权力大小</strong>两个方向进行识别。</p>
<ul>
<li><strong>利益相关者</strong>：直接利益人、间接利益人；</li>
<li><strong>权力大小</strong>：高层、中层、基层。</li>
</ul>
<p>直接利益人的高层、中层、基层我们按照业务分析框架，重点进行沟通和调研。</p>
<ul>
<li><strong>对高层</strong>，可以谈战略和战术，包括战略定位，战略目标，经营策略和管控模式；</li>
<li><strong>对中层</strong>，可以谈组织架构，人力资源，流程，绩效和风控；</li>
<li><strong>对基层</strong>，可以谈具体业务操作和详细的作业细节。</li>
</ul>
<p>间接利益人的高层、中层和基层我们也可以从战略、战术、执行三个方面进行了解他们在调研的领域里所发挥的作用，结合他们的视角更好地熟悉和了解所调研的业务情况。</p>
<p><strong>3）调研方法</strong></p>
<p>To B的调研方法基本也是常用的那几种类型，包括定性法和定量法。具体采用哪种方式，需要结合具体的情况选取合适的调研方法。比如，深入客户现场调研，可以采用访谈，而不用设计调查问卷。</p>
<p>这里提供几个常用的调研方法，包括<strong>访谈、轮岗、问卷、数据和行业分析</strong>等。</p>

<p><strong>用户访谈</strong>：准备好访谈大纲，包括战略层，战术层和执行层的大纲，针对大纲列明核心的问题，形成问题清单，如下调研模板：</p>

<p>制定详细的访谈调研计划，提前与被调研方确认好行程、住宿、议题、时间、会议室、双方人员等。如下图的客户现场业务调研日程表，若是问卷调研则明确问卷的收发截止日期。</p>

<h3>2. 调研中</h3>
<p>开始进入调研后，不同的调研方法调研技巧不同，如下简要阐述了方法异同点。</p>
<p><strong>访谈</strong>：需要到现场进行业务调研，一般分三步走，包括<strong>开场白的自我介绍，调研过程的访谈记录和调研结束的表示感谢</strong>。</p>
<p><strong>轮岗</strong>：需要到业务的岗位进行轮岗实习，一般制定和轮岗计划和岗位，轮岗时做到多观察，多思考，多记录。</p>
<p><strong>问卷</strong>：设计好问卷通过网络渠道送达调研对象，一般在规定的时间完成问卷提交，做好奖品发放。问卷是需要统计问卷完成率，以及问卷有效率。</p>
<p><strong>数据</strong>：通过数据分析方法，找到产品迭代的需求点。</p>
<p><strong>行业</strong>：通过行业分析，竞品分析找到用户需求盲点。</p>
<p>这里重点阐述一下<strong>访谈方法的调研技巧：三步走，完成一个调研闭环</strong>：</p>
<ul>
<li>开场白：自我介绍和表明来意；</li>
<li>调研过程：做好记录，控制好时间；</li>
<li>调研结束：表示感谢，建立关系。</li>
</ul>
<p><strong>1）开场白</strong></p>
<ul>
<li><strong>自我介绍</strong>：让大家知道你是谁，建立初步的信任感，如果有名片可以递送给客户名片。穿着需要正式一点，代表公司形象。</li>
<li><strong>表明来意</strong>：表明此次调研的目的和计划安排，让大家有一个心理预期。</li>
</ul>
<p><strong>2）调研过程</strong></p>
<p><strong>做好记录</strong>：提前熟悉调研的问题和调研对象的职位，做到不同职位不同问题，同时现场允许录音，可录下当时访谈过程，回头补上遗漏的点。</p>
<p><strong>控制好时间</strong>：做到计划时间内完成问题的访谈，控制好时间节奏，不要给对方留下拖堂的印象。</p>
<p><strong>调研大纲：</strong>To B产品的调研套路从战略层、战术层和执行层三个方面进行，分步骤完成业务调研。</p>
<ul>
<li>战略层：战略定位、战略目标；</li>
<li>战术层：经营策略，管控模式；</li>
<li>执行层：组织架构、业务流程、人力资源、绩效和风控。</li>
</ul>
<p>三个层面覆盖的组织架构和岗位、业务概况、业务详细说明、管理报表、现有系统和用户诉求等。</p>
<p><strong>注意事项：</strong></p>
<p>① 用户吐槽跑题时，及时拉回访谈正轨，做到访谈信息的高质量。</p>
<p>② 用户不配合时，学会换位思考，拉近距离，让对方打开心扉，遇到实在不配合的人员，可多角度多方位地了解他所负责的工作。比如，通过调研该岗位的上下游岗位，以前负责的人员、制度和岗位职责，以及操作单据等。</p>
<p>③ 信息遗漏或挖掘不深入时，通过现场录音或二次求证的方式挖掘真实情况。</p>
<p>④ 用户说不清时，重复对方观点，明确对方的核心意思。</p>
<p>⑤ 访谈过程不做引导，不做承诺，尽量保持身份中立，可以让信息收集更趋于真实。</p>
<p>⑥ 同频沟通很重要，做业务调研尽量使用业务语言，而不是产品语言，更不是技术语言。</p>
<p>⑦ 调研对象尽量选择利益相关方。</p>
<p><strong>3）调研结束</strong></p>
<ul>
<li><strong>表示感谢：</strong>感谢被调研对象所付出的努力和时间，让对方能够被认同。</li>
<li><strong>建立连接：</strong>访谈的结束，不代表关系的结束，我们需要建立良好的长远关系，可以给对方一些小礼品表示感谢。</li>
</ul>
<h3>3. 调研后</h3>
<p>调研后，不同调研方法获得需求和问题需及时整理成调研结果和问题清单，建议大家当天晚上或第二天记忆最深的时候整理完毕，整理的格式尽量要按大纲的目录进行分类和总结。</p>
<p>调研后，主要会包括<strong>调研结果整理、调研报告输出和知识沉淀</strong>三部分事项处理。</p>
<p><strong>1）整理调研结果</strong></p>
<p>将前期不同调研方法所获的结果整理成书面化。</p>
<ol>
<li>去除干扰项、无效信息、合并同类项；</li>
<li>按人物、角色、大纲将问题或需求整理分类并记录，形成问题或需求清单；</li>
<li>同步问题或需求清单给相关干系人，加深与客户的信任和关系；</li>
</ol>
<p><strong>2）输出调研报告</strong></p>
<p>将调研结果整理成调研报告，作为向领导汇报或项目阶段性产物。那么，<strong>如何制定调研报告呢？</strong>调研报告一般会包括：</p>
<ul>
<li><strong>调研背景&目的</strong>：描述业务调研的背景和目的；</li>
<li><strong>调研计划</strong>：描述本次调研的人员，访谈对象，日程，议题等信息；</li>
<li><strong>调研内容</strong>：描述调研过程的访谈记录，包括大纲问题及相关的业务描述，以及专题清单；</li>
<li><strong>客户期望：</strong>整理和总结用户的期望诉求清单；</li>
<li><strong>后续计划</strong>：需进一步讨论的问题或后续的行动计划。</li>
</ul>
<p>调研报告形式一般包括：WORD、EXCEL、PPT等，如下图给出WORD版的样例。</p>

<p><strong>3）知识沉淀</strong></p>
<p>将调研报告分享输出给相关干系人，做到调研报告反馈，同时将调研报告进行归档，作为组织过程资产进行项目资料沉淀，以便后续同事、新人、其他团队成员能够快速了解背景，进行借鉴和应用。</p>
<p><strong>最后，</strong>产品不同阶段调研目的不同，采取的调研方法也不同。不同目的和不同方法得出结论和行动计划也会不同。调研之前想清楚：<strong>是不是一定得调研</strong>？如果要进行业务调研，<strong>先建立业务调研框架</strong>，有计划有节奏地进行，让业务调研的结果真正达到我们的目的和结果。</p>
