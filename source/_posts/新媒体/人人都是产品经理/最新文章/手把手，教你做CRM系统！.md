<div>   
<blockquote><p>编辑导语：只要是公司就会有销售团队，而销售团队后面就是客户，所以CRM也是每个公司必不可少的一个工具。那么，如何设计完整的CRM架构呢？应该注意哪些问题？一起来看一下吧。</p></blockquote>
<p><img data-action="zoom" class="aligncenter size-full wp-image-5520987" src="https://image.woshipm.com/wp-files/2022/07/7RMJLqQ2SrJsGS3YJehh.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<p>上周有读者朋友询问我，近期甲方委托我们公司开发一套 CRM 软件，需求大概了解清楚，但如何设计完整的 CRM 架构呢？应该注意哪些问题？从我以往经验来看，大大小小的也服务了五六个 CRM 项目，<strong>其中功能的搭建是最为简单的，而企业内部的业务流程和角色权益是一个难点，这需要在项目初期花大量的时间去调研。</strong></p>
<p>毋庸置疑一点，只要是公司就会有销售团队，销售团队后面就是客户，哪怕公司很小，客户十分不稳定，就算平常只是用表格来记录跟进，那这个表格也就是你们公司的CRM。</p>
<h2 id="toc-1"><strong>一、CRM 客户管理的价值是什么？</strong></h2>
<p>维基百科上面说：客户价值管理就是企业根据客户交易的历史数据，对客户生命周期价值进行比较和分析，发现最有价值的当前和潜在客户，通过满足其对服务的个性化需求，提高客户忠诚度和保持率。</p>
<p>其根本目的是使企业的经营理念、能力、过程及组织结构与客户感知的价值因素相适应，来向客户传递最大化的价值。</p>
<p>简单一句话：<strong>CRM 就是企业管理客户的一种工具，这种工具可以是软件，可以是表格，当然还可以是纸质笔记本。</strong></p>
<p>CRM 只是一种形式，这种形式可以是多种多样，只不过 CRM 软件更容易使企业在线上记录跟踪服务客户。</p>
<p>再深度剖析还可以分为：<strong>各个角色之间的权益系统。</strong></p>
<ul>
<li>领导有了它可以更直观的看到客户数据，从而可以更好的确认战略方向</li>
<li>员工有了它可以摆脱纸质文档，查找信息繁琐，客户分类杂乱无章等等</li>
<li>客户有了它可以随时向企业提出问题，并寻找客服人员进行答疑</li>
</ul>
<p>我曾服务于一个传统企业，是做工业机床出售的，他的客户档案全部封存在档案室里，这样就出现一个问题，每次当想要寻找某个客户的信息，需要翻箱倒柜好长时间，非常耗时耗力。</p>
<p>平常都是用 WX沟通，导致每一条记录不能删除，因为一旦删除就恢复不了，还有的客户是销售团队一手带起来了，一旦员工离职，这些信息模块也会随之消失。</p>
<p>看，这就是 CRM 的价值体现，也正是为何近几年 CRM 赛道如此火热，<strong>纷纷入场，在一团乱粥里寻找合适的器皿。</strong></p>
<h2 id="toc-2">二、如何开始搭建</h2>
<p><strong>因为每一家企业业务不同，客户也不尽相同，需要了解企业内部的真实需求，就算这些都相同，体量也不可能高度一致。</strong></p>
<p>那么应该如何从 零开始构建一套 CRM 系统呢？</p>
<p>这里结合以往的项目经验给大家几点建议，<strong>堪称保姆级填空题。</strong></p>
<p>首先确定你的工作阶段：<strong>前期准备——实施过程。</strong></p>
<h3>1. 前期准备</h3>
<p>这个阶段是项目最重要的部分，也直接影响最终的交付成果，这时候你要考虑项目的需求来源，需求来源基本两部分：</p>
<ol>
<li>投招标项目：这类需求其实较为简单，因为他们都会有正式的需求合同，一旦中标成功，你只需要按照要求完成内容就好</li>
<li>作为乙方：这就是开篇读者朋友这种类型，相对来说比较复杂，因为没有正式的需求合同，你需要前期做大量的调研工作，下图梳理了一些项目初期关键填空题</li>
</ol>
<p><img data-action="zoom" class="rich_pages wxw-img aligncenter" title="CRM 从 0 到 1 应该如何搭建？（建议收藏）" src="https://image.woshipm.com/wp-files/2022/07/jFNXc4ddIXTY19iG8K0P.png" alt="CRM 从 0 到 1 应该如何搭建？（建议收藏）" referrerpolicy="no-referrer"></p>
<p><strong>为什么首先要了解企业概况？</strong></p>
<p>目的就是为了在项目的大方向上少出错，比如你不了解企业为什么要构建一套 CRM 系统，企业负责人给你说我就是想做套系统，能帮助我提高销售额就行，这时候你要考虑构建什么业务才能提高企业销售额？<strong>是统一管理客户、客户画像、还是自动化营销。</strong></p>
<p>确定了这些，你的产品才会有思路。</p>
<p><strong>1）组织架构</strong></p>
<p>简单来说就是这套软件给谁用？每个角色想通过该软件获得什么？角色之间的权益是如何分配的？是否存在角色隐患？</p>
<p>说一个我之前踩过的坑，在项目初期没有询问使用者，心想都给钱了，只要按期交付就行了。项目一上线就出现了致命的问题，使用者是销售团队，他们平常都在外地跑客户，之前一直用微信沟通，很多客户信息也都只是记录在笔记本或者表格中。</p>
<p>按常理来说，我们这套软件正好解决他们沟通问题和客户信息问题，只要把客户信息录入到系统中，既可以随时查找，又可以不占用自己的资源。没想到的是他们根本不用。</p>
<p>为何？</p>
<ul>
<li>学习成本高，很难转变自己的工作习惯</li>
<li>信息录入起来太麻烦，他们宁愿去笔记本上找，也不愿抽时间把客户信息录入系统中</li>
<li>最为关键一点：有的客户是没有动用企业任何资源，完全靠自己来收录的，认为这个客户是他自己的，上传到系统中，客户就变成了公用客户了</li>
</ul>
<p>夺命三连，这就是前期调研工作做的不完善，所以才导致这些问题的出现，是系统上线后才出现的吗？<strong>显然不是，问题一直存在，只是我没有发现。</strong></p>
<p><strong>2）客户来源</strong></p>
<p>客户从哪里来，到哪里去，调研他们平常记录客户信息都有哪些条例，还有有没有扩展的地方，客户需要不需要做分类，这些确认好以后，才能更加准确的构建功能。客户是从线上还是线下，还是转介绍，这些分类要搞清楚。</p>
<p>软件系统很受用的一句话，<strong>系统是蓄水池，水从哪里流入，水流的方向中间会经过什么？从哪里流出！</strong></p>
<p>建议把每个问题，对照的解答都记录下来。以上弄清楚以后，就到了真正的实施阶段。</p>
<h3>2. 实施阶段</h3>
<p>我把概况设计分为了四个步骤：<strong>需求确认-方案搭建-模块清单-原型绘制。</strong></p>
<p><strong>1）需求确认</strong></p>
<p>前期所有的工作都是为了把需求确认下来，最好出需求合同让对方签字画押，避免在开发实施过程中出现方向偏离。也能很好的规避风险。然后开始着手搭建方案，这一步要确定该做什么不做什么。</p>
<p><strong>2）方案搭建</strong></p>
<p>可以先用思维导图把基本的功能罗列出来，然后将方案输出成 ppt，以便于甲方沟通。方案的搭建不仅可以让你的目标更明确，更可以代表公司的实力。</p>
<p>很多人说漂亮的 ptt 只是花架子，没什么用。我不这样认可，表面工作都体现不出你们的优势，那如何让别人有兴趣了解企业的内在。</p>
<p><strong>换个纬度来说，代码谁不会写，又有哪个产品不是抄袭和借鉴。那又为什么选择你？</strong></p>
<p><strong>3）模块清单</strong></p>
<p>这一步骤，根据需求来确认功能模块，比如哪些是一级模块，二级模块，三级模块，这些模块之间有没有必要的联系。</p>
<p><strong>一比一复制的产品没有任何意义，可以借鉴，但不能完全不加思考的复制。</strong></p>
<p>好比你的需求清单根本就没有会议这一项功能，你自认为不错，然后加入到了模块清单中，到最后客户询问你这个会议干嘛用的？你说：当时考虑企业在工作当中，应该会使用这项功能，为了方大家实时沟通，能让工作更高效。觉得天衣无缝。</p>
<p>其实客户并不认账，为什么？会议这项功能会不会产生其他费用？如果只是通告大家，我还不如在微信群里@所有人几点开会。在软件上我还需要勾选。</p>
<p>你们开发的这个可以实现会议直播吗？有的同事在其他地方办公，直播的流量要不要付费？发现了问题关键没有？<strong>因为一个想法，导致这些坑永无止境，有可能还会被对方认为你不够专业，缺乏经验。</strong></p>

<h3><strong>3. 原型设计</strong></h3>
<p>以上都确认完以后，就可以开展原型设计了，最简单的方式就是根据你的模块清单来一步一步完善。</p>
<p>因为每个项目需求都不一样，所以我们挑选市场上基本都会用的基本功能来举例。以下我用 Zoho CRM 来举例，它的CRM系统功能齐全、逻辑性较高，操作难度小，更方便大家理解。</p>
<p><strong>1）主页</strong></p>
<p>主页要体现什么？首先如果你是一名销售人员你关心的指标有哪些？</p>
<p>线索、商家、客户进度、销售额、我的任务。这些优先级虽然很高，但难免使用者就是想展示其他内容怎么办？</p>
<p><img data-action="zoom" class="rich_pages wxw-img js_insertlocalimg aligncenter" title="CRM 从 0 到 1 应该如何搭建？（建议收藏）" src="https://image.woshipm.com/wp-files/2022/07/CBtkjrPnLstpFCANMWJI.png" alt="CRM 从 0 到 1 应该如何搭建？（建议收藏）" referrerpolicy="no-referrer"></p>
<p>在这里可以主页可以设置成灵活性高的组件模块，比如我想在首页上显示供应商的列表怎么办？你只在添加组件上选择自己的想要的内容，展示即可。</p>
<p>就这一步操作，也许就可以拉开同行很远的距离。灵活性高，想用就放上，不想用删除即可。</p>
<p><strong>2）线索</strong></p>
<p>什么是线索？<strong>简单来说就是销售部门从各个渠道来记录的信息，为了防止错乱，这个功能可以更好的记录信息来源，进行分组。</strong></p>
<p>要确认好你的线索有哪些分类？新建线索需要准备哪些字段？模块可分为：<strong>新建线索，导入，导出，批量操作。</strong></p>
<p><img data-action="zoom" class="rich_pages wxw-img aligncenter" title="CRM 从 0 到 1 应该如何搭建？（建议收藏）" src="https://image.woshipm.com/wp-files/2022/07/Z8O1SzeODkPhJYzC3jz4.png" alt="CRM 从 0 到 1 应该如何搭建？（建议收藏）" referrerpolicy="no-referrer"></p>
<p><strong>3）联系人和客户</strong></p>
<p>联系人和客户有什么不一样？<strong>简单来说可以看作父子关系，比如你的客户是一家企业，那么联系人就是企业下具体的联系人。</strong></p>
<p>左边导航栏筛选条件，右边展示客户信息。</p>
<p><img data-action="zoom" class="rich_pages wxw-img aligncenter" title="CRM 从 0 到 1 应该如何搭建？（建议收藏）" src="https://image.woshipm.com/wp-files/2022/07/Mm7nn0zhJDQl9h71eYLz.jpeg" alt="CRM 从 0 到 1 应该如何搭建？（建议收藏）" referrerpolicy="no-referrer"></p>
<p>值得一提的是，该软件下方还有一些小功能，比如便签、活动提醒、奖励机制、最近访问都挺方便的。</p>
<p>便签功能：我们来模拟下场景，在工作中我们对单一客户打电话，需要记录重要信息，这时候你就可以打开标签，随时记录，在通话结束后，来进行查阅。</p>
<p><img data-action="zoom" class="rich_pages wxw-img aligncenter" title="CRM 从 0 到 1 应该如何搭建？（建议收藏）" src="https://image.woshipm.com/wp-files/2022/07/NzBT0G5IJtRmxEc7GFzU.png" alt="CRM 从 0 到 1 应该如何搭建？（建议收藏）" referrerpolicy="no-referrer"></p>
<p><strong>4）商机</strong></p>
<p>到底什么是商机？官方解释：所谓商机就是指销售机会，销售人与和客户沟通过程中，了解客户意愿，从而达成交易。其实并不完全是销售机会。</p>
<p>从使用者角度来看，可以很好的理解成：<strong>是对前期客户线索更深一层的检索，因为录入的信息很多是不具备营销价值的。</strong></p>
<p><strong>有了线索才有商机，没有孤立无援的商机！其实从实际情况出发，如果假设成立，那么商机就不会存在什么创建商机，而是线索商机。</strong></p>
<p><strong>5）数据分析</strong></p>
<p>数据分析指使用者可以通过可视化的方式，直观的查看自己的客户情况，收入明细，和任务列表。</p>
<p>基本上以上功能就可以满足百分之80的企业，其他的功能作为延伸，在本期文章就不做过多展示了，感兴趣的可以去体验下。</p>
<p><img data-action="zoom" class="rich_pages wxw-img js_insertlocalimg aligncenter" title="CRM 从 0 到 1 应该如何搭建？（建议收藏）" src="https://image.woshipm.com/wp-files/2022/07/taJubwqGHYTfYOWSaVjP.png" alt="CRM 从 0 到 1 应该如何搭建？（建议收藏）" referrerpolicy="no-referrer"></p>
<h2 id="toc-3">三、写在最后</h2>
<p>总的来说，这些你都了解清楚了，也都把填空题填写了，那么在你实施项目过程中，就不会出现大的偏差。</p>
<p>回顾 CRM 的历史，它的价值被越来越多人关注。由我个人而言：</p>
<p>它的价值在于实现信息共享和管理集成，随后可以通过这些数据指标来判断企业目标方向。</p>
<p>CRM系统具有促进销售流程规范、整合、优化的价值。实施CRM系统能够改善企业的销售流程，加强对潜在客户的机会管理。让销售人员更加有针对性地把握销售业务的进展与销售策略，CRM系统能够简捷地预测销售业绩，评估企业绩效。</p>
<p>CRM系统可以与企业的SCM系统、ERP系统无缝集成，实时数据交换，密切企业与供应商、合作伙伴、客户间的关系，加快客户服务与支持响应速度，增强企业的社会竞争优势。</p>
