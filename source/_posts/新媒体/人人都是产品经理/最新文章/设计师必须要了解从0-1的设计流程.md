<h2 id="toc-1">01 设计师面临的知识困境</h2>
<p>网络上碎片化的知识越来越多，UI设计师正在面临着2大困境，第一大困境就是有太多的知识观点需要学习，第二大困境是不知道如何去将这些知识进行取舍和归纳。</p>
<p>各大设计平台写设计流程的文章也是多如牛毛，而这些文章的共同点都基本上是在Design Thinking的模型之下进行演进和变化的，而如果仔细分析和观察，你会发现他们的共同点和差别，只有了解这2点，才能够帮助我们更好的学习设计流程。</p>
<h2 id="toc-2">02 3种常见的设计流程模型</h2>
<h3>1. 设计思维</h3>
<p>设计是一种基于解决方案的解决问题的设计方法,在“设计思维（Design Thinking）”在被不同的学者提出之后，IDEO是第一家将设计思维应用于商业问题的解决之中。</p>
<p>IDEO的创始人，David Kelley，后来又在美国斯坦福大学创建了著名D.School，斯坦福设计学院。设计思维强调以人为本的设计思想，它需要花费更长的时间做观察、访谈来深入理解用户，对解决未定义或未知的复杂问题极其有用。它主要分为5个步骤：</p>
<ol>
<li><strong>同理心（Empathy）：</strong>收集对象的真实需求</li>
<li><strong>定义（Define）：</strong>分析收集到的各种需求，提炼要解决的问题</li>
<li><strong>头脑风暴（Ideate）：</strong>打开脑洞，创意点子越多越好</li>
<li><strong>原型制作（Phototype）：</strong>把脑子中的想法动手制作出来</li>
<li><strong>测试（Test）：</strong>优化解决方案</li>
</ol>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/03/L6OVqHCYpSDPcWHu12FA.png" referrerpolicy="no-referrer"></p>
<p style="text-align: center;">*图片素材来源于网络，侵权请告知删除</p>
<h3>2. 设计冲刺</h3>
<p>设计冲刺（Design Sprint）最初起源于谷歌，融合了设计思维（Design Thinking）和敏捷开发（Agile）两大方法论，设计冲刺更适用于已知需要解决的且较为明确的问题，它融合了敏捷开发，最大的优势在于 5 天内完成整个创新流程。它主要分为6个步骤：</p>
<ol>
<li><strong>理解（Understand）：</strong>理解目标</li>
<li><strong>定义（Define）：</strong>定义问题</li>
<li><strong>发散（Diverge）：</strong>探索各种解决方案</li>
<li><strong>抉择（Decide）：</strong>找到最优的解决方案</li>
<li><strong>原型（Prototype）：</strong>设计原型</li>
<li><strong>验证（Validate）：</strong>验证方案的可行性</li>
</ol>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/03/cyhcTBsarQmk3Q5MtzgM.png" referrerpolicy="no-referrer"></p>
<p style="text-align: center;">*图片素材来源于网络，侵权请告知删除</p>
<h3>3. 双钻模型</h3>
<p>双钻模型来源于IDEO的以人为本的设计思想，以及@d.school的设计流程，“双钻模型”和设计思维模型一样，需要花费更长的时间来观察和理解用户，对解决未定义或未知的复杂问题极其有用，一般在互联网行业中使用的比较多，它分为4个步骤：</p>
<ol>
<li><strong>发掘/调研（Discover /Research）：</strong>洞察存在的问题 (发散过程)</li>
<li><strong>定义/归集（Define/Synthesis）：</strong>聚焦、界定要解决的问题 (聚焦过程)</li>
<li><strong>前进/构思（Develop/ Ideation）：</strong>探索潜在的各种解决方案(发散过程)</li>
<li><strong>交付/实现（Deliver /Implementation）</strong> <strong>：</strong>找到最优的解决方案(聚焦过程)</li>
</ol>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/03/h7bByEcmxBrcqzpgyUIm.png" width="965" height="543" referrerpolicy="no-referrer"></p>
<p style="text-align: center;">*图片素材来源于网络，侵权请告知删除</p>
<h2 id="toc-3">03 经典设计模型的共性</h2>
<p>我们如果仔细观察就会发现，这3种设计方法论不变的框架都是从发现问题（发散、聚焦）到解决问题（发散、聚焦）的过程，其中每个环节都是从发散到聚焦，围绕这个大的框架，我们接下来要思考的就是：</p>
<p><strong>（1）发现问题</strong></p>
<p>问题有哪些？（发散）</p>
<p>问题这么多，哪些才是真正的问题？（聚焦）</p>
<p><strong>（2）解决问题</strong></p>
<p>如何解决这些问题？（发散）</p>
<p>这些解决方案，哪些是最优可实施的？（聚焦）</p>
<h2 id="toc-4">04 我总结的设计流程</h2>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/03/rWW7xbVPUXwuE8OJq9NB.png" referrerpolicy="no-referrer"></p>
<p>在这张图中，我在双钻模型的基础之上进行演变，让他更加匹配现在互联网设计团队的需求，其中增加了一个钻的模块，分别为交付和验证的环节，主要原因在于现在的公司更加重视设计师的落地和数据分析能力。整个的设计流程分为6个步骤，分别为：</p>
<ol>
<li><strong>探索：</strong>洞察问题（发散）</li>
<li><strong>定义：</strong>定义关键问题、定义业务和设计目标（聚焦）</li>
<li><strong>发散：</strong>找到各种解决方案（发散）</li>
<li><strong>设计：</strong>找到最优的解决方案，进行设计执行（聚焦）</li>
<li><strong>交付：</strong>完备的交付文件（发散）</li>
<li><strong>验证：</strong>验证设计方案的效果（聚焦）</li>
</ol>
<p>同时我也将每个阶段的目标、所需的工具方法、交付物总结了出来，方便大家进行有针对性的学习</p>
<h3>1. 探索（洞察问题）</h3>
<p><strong>目标</strong></p>
<ul>
<li>洞察用户需求（表层需求&深层需求）</li>
<li>了解业务需求（需求的几种来源：高层、用户、竞品、数据、内部走查）</li>
</ul>
<p><strong>工具方法</strong></p>
<p>问卷调研、用户访谈、竞品分析、用户建模、用户体验地图、桌面研究、焦点小组、数据分析、头脑风暴、交互视觉走查</p>
<p><strong>交付物</strong></p>
<p>问卷分析报告、用户调研报告、竞品分析文档、用户体验地图、桌面研究报告、数据分析报告、交互走查报告</p>
<h3>2. 定义（定义关键问题、定义业务和设计目标）</h3>
<p><strong>目标</strong></p>
<ul>
<li>将问题收拢聚焦关键问题</li>
<li>定义业务目标和设计目标</li>
</ul>
<p><strong>工具方法</strong></p>
<ul>
<li>需求分类(卡片分类)</li>
<li>需求如何删减(商业价值&用户价值)</li>
<li>如何制定需求的优先级(KANO模型)</li>
<li>数据指标（GSM模型、五度模型）</li>
<li>其他（AARRR模型、用户体验地图）</li>
</ul>
<p><strong>交付物</strong></p>
<ul>
<li>问题总结文档</li>
</ul>
<h3>3. 发散（找到各种解决方案）</h3>
<p><strong>目标</strong></p>
<ul>
<li>收集尽可能多的有效、优质解决方案</li>
</ul>
<p><strong>工具方法</strong></p>
<ul>
<li>头脑风暴</li>
<li>竞品分析</li>
</ul>
<p><strong>交付物</strong></p>
<ul>
<li>头脑风暴记录文档</li>
<li>竞品分析文档</li>
</ul>
<h3>4. 设计（找到最优的解决方案，进行设计执行）</h3>
<p><strong>目标</strong></p>
<ul>
<li>筛选最佳的解决方案</li>
<li>将解决方案制作成交互原型</li>
<li>产出视觉方案</li>
<li>设计评审，打磨方案</li>
</ul>
<p><strong>工具方法</strong></p>
<ul>
<li>方案筛选（实现成本&用户成本）</li>
<li>交互5要素</li>
<li>情绪版</li>
<li>形、色、质、构、质</li>
<li>栅格</li>
</ul>
<p><strong>交付物</strong></p>
<ul>
<li>UE（信息架构图、交互原型图）</li>
<li>UI（UI设计稿、动效设计、IP形象设计、规范系统、控件系统、图标系统、插画系统）</li>
</ul>
<h3>5. 交付（完备的交付文件）</h3>
<p><strong>目标</strong></p>
<ul>
<li>配合开发交付全面的设计文件</li>
</ul>
<p><strong>工具方法</strong></p>
<p>暂无</p>
<p><strong>交付物</strong></p>
<ul>
<li>交互设计文档</li>
<li>UI设计文档（UI设计稿、切图文件、标注文件）</li>
<li>动效文档（动效标注、json代码）</li>
</ul>
<h3>6. 验证（验证设计方案的效果）</h3>
<p><strong>目标</strong></p>
<ul>
<li>验证设计效果</li>
</ul>
<p><strong>工具方法</strong></p>
<ul>
<li>问卷调研</li>
<li>用户访谈</li>
<li>数据分析</li>
</ul>
<p><strong>交付物</strong></p>
<ul>
<li>问卷分析报告</li>
<li>用户调研报告</li>
<li>数据分析报告</li>
</ul>
<h2 id="toc-5">05 总结</h2>
<p>这是馒头根据自身多年的工作经验总结出来的设计流程，每家公司根据本身的特点可能会有些许差异，欢迎大家一起进行补充和交流。这个设计流程能够帮助我们看到设计全貌，知道自己所处的环节，知道自己需要补全哪些知识。</p>
