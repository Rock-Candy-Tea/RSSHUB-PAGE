<h2 id="toc-1">一、为什么做调研</h2>
<p>在开始做调研之前，必须先明确为什么要做。很多时候，答案可能是因为「我要做一个相似的产品/功能」、「因为他们是行业领先的产品，xxx方面数一数二」等等，但都不全面。在我看来，<strong>做调研目的是为了协助自身增强对业务/领域的认知</strong>。</p>
<p>那么「产品调研」就是帮助我们获取新的认知的其中一种手段。本着精益原则，认知获取成本越小，就越高效。最近在课程上看到了一个<strong>「认知获取成本公式」：行业常识 < 产品调研 << MVP验证 <<< 全量开发。</strong>基于公式，产品调研是帮助我们解决行业常识无法解决的问题、成本较低的一种方式。</p>
<h2 id="toc-2">二、产品调研的思路</h2>
<p>在了解为什么做之后，如何做分为5步。根据自己调研火花思维的思路，复盘调研方法论。</p>
<h3>1. 明确调研目的</h3>
<p>首先是明确调研目的，也就是我想通过这次调研得到什么东西。<strong>以目的结果为导向</strong>，才不至于调研过程中，东看看西看看，降低效率。</p>
<p>「火花思维转介绍调研复盘」：</p>
<p>考虑到火花思维业务线分为直播课和AI课，APP也分为学生端和家长端，所以对标自己的业务线和转介绍现状，最终决定以家长端APP为主，调研火花思维直播课的转介绍是怎么做的。</p>
<h3>2. 缩小调研范围</h3>
<p>缩小调研范围，重在理清调研框架。<strong>清晰的调研框架可以帮我们砍掉90%无需关注的内容</strong>，从而快速准确帮助我们拿到想要的东西。</p>
<p>「火花思维转介绍调研复盘」：</p>
<p>在明确要调研火花思维直播课家长端APP后，纵观火花这么多转介绍功能、活动，应该调研什么？基于现状和策略，我只研究火花思维的转介绍核心功能，是长期持续的、固定核心的那部分，能够组成火花完整的产品运营思路的功能。</p>
<p>基于已有了解，梳理出4个核心功能是要调研的：分享海报功能、上传截图功能、月度活动功能、拼团功能；至此，已经完成了缩小调研范围这一步，调研框架也逐渐清晰：</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/t450JR4vjyNOgq3Uo4TO.png" alt width="1474" height="766" referrerpolicy="no-referrer"></p>
<p>这个框架是可以复用的，从功能分析到产品结构到产品框架，逐级往下渗透；从功能交互、数据流转等体验，分析这些功能是如何串联并贯穿用户生命周期，做到合理布局；最后思考这些内容背后是什么样的底层逻辑支撑着它们，形成完整的产品策略。</p>
<h3>3. 罗列调研list</h3>
<p>在明确调研框架后，需要罗列调研的产品清单。一般这里是根据目的去筛选调研的产品，比如像在线教育行业，我要去了解如何做转介绍，就需要去瞄准哪几个做转介绍做的较好的产品；比如像调研我要去了解瑞幸咖啡数据有没有造假，在实地调研前需要明确调研多少个门店、哪几个门店。</p>
<h3>4. 获取调研内容</h3>
<p>这个步骤就是执行层面的。获取调研内容也分两步，分别是<strong>已有资料调研+实地调研</strong>。</p>
<p>已有资料调研，是从网上搜罗调研产品的资料，多看一些专家访谈、媒体报道、行业报告等；实地调研可以包括去线下门店、APP体验、内部员工了解等。</p>
<p>「火花思维转介绍调研复盘」：</p>
<ul>
<li>第一步，先搜索网上已有资料，看看别人对火花思维转介绍的分析框架及思路；</li>
<li>第二步，去APP实际把需要调研的内容实际操作一遍并全流程截图，包括：每一环节流程体验、异常情况流程体验；</li>
<li>第三步，基于体验的功能，思考背后的底层逻辑。有时候，可以多横向对比其他产品的相同功能，理解相似之处并思考为什么这样做。</li>
</ul>
<h3>5. 合理判断，达成目的</h3>
<p>拿到调研内容后，需要对调研内容对合理判断，给出调研结果。但是面对碎片化的、复杂的，甚至是相互矛盾的调研内容，如何得出高质量的结论，需要提高自身辨别信息的能力，可以从几个角度：</p>
<ul>
<li>区分拿到的信息是一手还是二手信息；</li>
<li>区分拿到的信息是事实还是观点；</li>
<li>基于常识做判断；</li>
<li>归因到深层问题（底层逻辑）。</li>
</ul>
<h2 id="toc-3">三、火花思维转介绍调研报告</h2>
<p>根据上述调研思路，本月输出火花思维转介绍调研报告：</p>
<h3>1. 产品概况</h3>
<p>火花思维是一家K12领域在线儿童教育平台，专注于3-12岁在线儿童教育启蒙和思维训练。公司正处于成长期，正式学员超过12万，续费率80%，转介绍率达到82%，完课率高达99%**。</p>
<p>火花思维的业务线分为直播课和AI课，APP分为学生端和家长端，本次体验的是火花思维直播课的家长端。</p>
<h3>2. 体验环境</h3>
<p>运行设备：移动客户端苹果IOS系统及安卓系统下APP</p>
<p>运行网络：WIFI及4G网络下</p>
<h3>3. 产品结构分析</h3>
<p><strong>转介绍产品功能布局结构图：</strong></p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/HLi500XozfdZiqeObivF.png" alt width="5172" height="2696" referrerpolicy="no-referrer"></p>
<p>家长端APP 功能布局体验总结：</p>
<p><strong>不同页面入口定位清晰：</strong></p>
<ul>
<li>「首页」入口以活动为主：当月主推活动，常见于顶部banner和信息流广告位，提升曝光率。</li>
<li>「我的」入口以基础功能为主：固定功能入口，便于培养用户的转介绍习惯。</li>
</ul>
<p><strong>功能布局广、形式多样：</strong></p>
<ul>
<li>布局广：APP共有5个tab，除了「火花课程」、【我的课程】这两个tab之外，其余页面均布局有转介绍入口。其中【首页】、【我的】每个页面有多个入口。</li>
<li>形式多样：以【首页】的布局为例，有轮播图、小卡片、正方形卡片等形式，动静结合，内容与广告结合，看起来不像广告，减轻家长抵触心理。</li>
</ul>
<p><strong>弹窗触达链路完善、触达位置灵活：</strong></p>
<ul>
<li>完善：【每次打开APP→弹窗→进入某个活动→不喜欢,离开某个活动→弹窗推送别的相同类型的活动】【浏览tab的内容→弹窗推送与位置相关的内容】</li>
<li>灵活：弹窗的触达位置、频率、内容、样式多，背后有较完善的CMS后台支持（触达什么）</li>
</ul>
<p>微信服务号、小程序 功能布局体验总结：</p>
<p><strong>完善的服务号矩阵：</strong></p>
<ul>
<li>覆盖多个用户场景：课程拼团、家长交流分享、线下课堂、数理学习、国文学习等等场景。</li>
<li>增加活动曝光量：同一个活动在不同服务号推送，避免用户因关注的号数量过多而错过活动。</li>
<li>互推导流，形成良性循环：各服务号之间相互导流，增加粉丝数，慢慢培养粘性。</li>
</ul>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/cMY664pKgVkyF1qZBwT9.png" alt width="1280" height="880" referrerpolicy="no-referrer"></p>
<p><strong>小程序，便利快捷：</strong></p>
<ul>
<li>缩短推荐人分享链路：从朋友圈转移到微信群，一键分享。</li>
<li>缩短被推荐人购课链路：小程序内授权登录方便快捷。</li>
</ul>

<p>海报功能对于转介绍效果的提升，火花是怎么做的？</p>
<ul>
<li>分享效率：① 各环节上交互清晰，降低分享难度；② 利益刺激分享动力（火花币，课时）。</li>
<li>分享频次：① 月度活动设置阶梯式奖励、阶段性开启任务，增强持续参与频次。</li>
<li>转化效率：① 打磨海报的设计、文案。</li>
</ul>

<p>虽然如此，但是海报对转介绍的效果仍不明显，因此火花转向更短的分享链路——微信群&小程序。</p>
