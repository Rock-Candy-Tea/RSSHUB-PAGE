<h2 id="toc-1">一、何为大后台？</h2>
<p>本文讲述的大后台是基于作者的实际工作需要而提出的后台整合型的概念，是内部多个后台系统的集合。</p>
<p>具体是指，多个产品后台通过一个域名入口，接入集团通讯中心，实现统一的登录和权限管理，采用类似的UI规范展示；并在这过程中形成统一的产品矩阵，形成技术影响力。</p>
<p>因此，用户可更加便捷访问相关产品，管理员也可以在权限设置上达到事半功倍的效果。</p>
<h2 id="toc-2">二、为什么要进行大后台的整合？</h2>
<p>在两年多的时间内，产品技术部开发了多款不同领域的产品，目前负责开发和运维的产品后台有6个。</p>
<p>早期的产品都是独立域名、独立登录账号系统、独立的UI视觉和交互。后期的几个产品在进化过程中，有部分功能进行了统一，但是并没有达到完全的统一化管理。</p>
<p>不同的后台系统采用不同的登录形式、不同的结构框架。没有统一的入口，没有标准的交互规范，操作日志和权限没有统一管理。</p>
<p>造成了交互体验断层，内部使用效率低、管理混乱的局面。</p>
<p>所以，需要进行大后台的整合，以便达到体验、交互、管理上的一致性。</p>
<h2 id="toc-3">三、大后台整合的目的是什么？</h2>
<p>作为产品开发者和运维者，你需要一个统一的大后台，对所有的产品进行权限管理、角色设置和员工管理，并且统一记录所有的操作日志。</p>
<ol>
<li>统一登录态：一键登录，可访问有权限的所有产品和服务，提升访问和操作效率。</li>
<li>权限管理：将各个产品的后台纳入到大后台进行统一的管理，指定管理员，并设置该产品的角色，统计角色数量，可进行自定义的设置。</li>
<li>角色管理：可灵活地为多个产品配置不同导航和操作权限，一个角色可设置多个员工账号。</li>
<li>员工管理：每个员工账号都通过内部通讯工具实名认证登录，归属于某个（或多个）角色，具有操作某个（或多个）产品相关功能模块的权限。某个员工账号允许绑定多个角色。</li>
<li>日志管理：统一记录所有产品、所有角色、所有员工的所有关键操作记录。</li>
</ol>
<p>除此之外，在产品表现层，还可以尝试做到以下两点，来提升整体的产品调性。</p>
<p>1）打造统一的视觉交互UI，强化产品矩阵</p>
<ul>
<li>设计Logo</li>
<li>创造Slogan</li>
<li>设计“欢迎页”</li>
<li>统一页面框架结构</li>
<li>统一UI视觉</li>
<li>统一交互体验</li>
<li>统一操作逻辑</li>
</ul>
<p>2）传播团队影响力</p>
<ul>
<li>产品介绍展示</li>
<li>操作手册下载</li>
<li>团队风采展示</li>
<li>技术支持通道</li>
<li>需求提报通道</li>
<li>意见反馈通道</li>
</ul>
<p>由此将产研团队的服务更好地传递给用户，同时让用户的心声能更好地反馈到产研团队。积极建立和用户的紧密联系，打通产研团队与用户之间的沟通路径。</p>
<p>我想，这大概就是大后台提供的的超预期的用户体验了。</p>
<h2 id="toc-4">四、何如进行大后台的规划设计？</h2>
<p>本文的权限管理模块，采用RBAC权限模型。</p>
<p>RBAC：即基于角色的访问控制（Role-Based Access Control），主要通过角色和权限建立管理，再赋予用户不同的角色，或者角色下添加不同的用户账号，来实现权限控制的目标。</p>
<p>利用该模型来配置权限，直接优点是角色的数量比用户的数量更少，先把权限赋予角色，即可完成权限的分配。再为用户分配相应的角色，即可直接获得角色拥有的权限。</p>
<p>在本文的大后台产品中，权限颗粒度仅限于菜单维度，只需定义有限的角色拥有哪些菜单权限即可。</p>

<h3>1. 统一登录态</h3>
<p>本文的统一登录态是指登录流程前置，在访问产品前进行登录态的判断。登录后，根据角色定位有权限访问的产品，并给出定制化的产品列表。</p>
<p>触发登录的两种行为：</p>
<p>1）主动触发</p>
<p>用户主动点击【登录】按钮，在未设置角色前提下，所有登录大后台的用户均为“游客”角色，仅有查看首页、相关信息页面的权限，不能进入后台产品。</p>
<p>2）被动触发</p>
<p>用户点击产品入口，先判断登录态，未登录的需跳转到登录页面。</p>
<h3>2. 权限管理</h3>
<p>本文的权限管理，指多个产品是否纳入了权限管理的范畴，并对已有的产品进行管理。被添加的产品或系统，可被划分权限，设置管理员，下设角色和员工账户。</p>
<p>该模块主要的功能点包括：</p>
<ul>
<li>添加系统</li>
<li>配置管理员</li>
<li>系统名称</li>
<li>系统描述</li>
<li>启用/停用状态设置</li>
<li>二次编辑</li>
</ul>
<p>在系统管辖权内统计角色的数量，设置启用和停用的状态，因为目前的产品数量有限，暂时不设置翻页组建和删除功能。</p>

<h3>3. 角色管理</h3>
<p>角色是具有相同权限的账户的集合体。</p>
<p>该模块主要的功能点包括：</p>
<ul>
<li>添加角色</li>
<li>添加员工</li>
<li>角色名称</li>
<li>角色描述</li>
<li>启用/停用状态设置</li>
<li>二次编辑</li>
</ul>

<h3>4. 员工管理</h3>
<p>员工管理指的是登录系统的独立账号，被赋予一个或多个角色，享有一定的权限，可进行相关的操作。</p>
<p>该模块通过内部IM工具登录之后，自动获取其系统账号、员工姓名、工号、注册时间。</p>
<p>初次登录属于“游客”角色，需要管理员或者超级管理员赋予角色才能执行操作。</p>
<p>员工账号正常使用是属于“已启用”的状态，如果员工离职，账号删除、禁用、或者冻结等情况下，账号切换到“已停用”状态。</p>
<p>对于员工账号支持角色的二次编辑，支持各类条件查询等。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/05/gXc5czj0fWGxvF93zpb4.png" alt width="2482" height="620" referrerpolicy="no-referrer"></p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/05/zVgnEMQjUmq3tOtDDfDT.png" alt width="300" height="264" referrerpolicy="no-referrer"></p>
<h3>5. 日志中心</h3>
<p>日志中心是对所有产品敏感性操作的记录，支持汇总查看、按产品查看、按操作员查看、按操作时间查看等模式。</p>
<p>集中的日志管理，有助于系统管理者快速了解各个产品的使用情况，明确操作员的动态。在遇到问题，发生风险的时候，可以回溯日志，定位操作节点，锁定相关人员。</p>
<p>考虑到服务器储存压力的问题，可设置日志最多储存10000条，或者最多储存3个月内的数据。后续每多出一条数据，循环删除最早的一条数据。</p>
<p>如果条件允许，建议保留所有的历史操作日志。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/05/dVArvGlO5YiDBWsROMCP.png" alt width="2490" height="640" referrerpolicy="no-referrer"></p>
<h3>6. 打造统一的视觉交互UI，强化产品矩阵</h3>
<p>大后台的整合不仅仅需要体现在流程和功能上，还需要体现在视觉和交互上。所以，作者提出了打造统一的视觉交互UI，强化产品矩阵的设想。并通过需求方案的输出，帮助设想落地实现。</p>
<p>设计Logo：</p>
<p>为每个产品设计一个独特的、贴合产品调性的Logo，同时还要做到所有产品Logo有机整体的融合，视觉统一。</p>
<p>创造Slogan：</p>
<p>为每个产品创造一条简单易记、体现产品属性和价值的Slogan，助力产品传播和推广。</p>
<p>设计“欢迎页”：</p>
<p>为每个产品设计一款欢“迎页面”，通过具有感染力的画面，提升用户体验，传播产品功能价值。</p>
<p>统一产品各级页面呈现的框架结构、UI视觉、交互体验、操作逻辑等。</p>
<ol>
<li>顶部logo+产品名称+slogan+登录/退出入口</li>
<li>左侧导航栏</li>
<li>右侧大面积操作区域</li>
<li>新建按钮在操作区域的左上角</li>
<li>查询功能在列表上方</li>
<li>列表采用统一的组建</li>
<li>翻页组建统一</li>
<li>弹窗交互和样式统一</li>
<li>删除等操作二次确认</li>
<li>统一toast提示样式和消失时间</li>
<li>同一类型按钮采用相同的默认色值、交互色值</li>
<li>同一类型的文本采用统一的字体/字号/字色</li>
<li>导航栏交互统一</li>
<li>说明性文案布局一致、采用统一的字体/字号/字色</li>
<li>增删改查的操作保持统一的交互体验</li>
</ol>

<h3>7. 传播团队影响力</h3>
<p>产品介绍展示：</p>
<ul>
<li>这个模块是关于产品的介绍，图文结合，主要内容包括：产品的性质、产品的功能、产品的合作伙伴、相关负责人、产品的一些关键数据指标等。</li>
<li>该模块主要是静态图文的展示，需要设计师出方案稿。</li>
</ul>
<p>操作手册下载：</p>
<ul>
<li>提供相关产品的操作手册的查看和下载。</li>
<li>文档格式建议PDF。</li>
</ul>
<p>团队风采展示：</p>
<p>图片、文字、视屏等形式展示团队风采，包括：团队成员、岗位介绍、能力展示、团建活动、团队合影等。</p>

<p>在使用产品的过程中，如遇到技术问题，请通过【XX】联系我们的开发工程师。</p>

<h2 id="toc-5">五、总结</h2>
<p>以上内容是作者搭建大后台的初始版本的产品设计方案，主要聚焦在当前产品需要整合的问题，最主要解决的是产品权限管理的问题。</p>
<p>相信，等整个大后台概念落地，产品上线之后，后续的迭代会让产品的体验更加美好、产品层次更加丰富、也会呈现更加多维度的内容。</p>
