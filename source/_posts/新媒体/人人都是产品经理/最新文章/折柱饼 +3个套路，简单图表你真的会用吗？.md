<blockquote><p>编辑导语：图表作为数据可视化的有效手段之一，在日常业务中经常被使用。不过，你的图表真的能够有效、高效地传递信息吗？本篇文章里，作者总结了数据分析的“套路”，介绍了简单易学的图表制作思路，一起来看一下。</p></blockquote>
<p><img data-action="zoom" class="size-full wp-image-4863808 aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/SDrrBETDqYCFSyBRNPtq.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<p>本文一句话概括：数据可视化中，如何用最简单的图表高效地传递信息。</p>
<h2 id="toc-1">一、人人熟知的 3 类图表</h2>
<p><b>看似简单的 3 类常挂在嘴边的「折柱饼」，你真的知道怎么使用吗？</b></p>
<blockquote><p>@rubyxrli 在纽约的Uber数据可视化大会上所做演讲中提到：根据我的经验（基于原型研究超过10年）大多数可视化问题可以通过一些图表来解决。很少情况下，需要你去想出一个全新的表达方式。<b>像条形图、折线图、散点图、透视表等「图形化主力」真的很难被替代</b>！</p></blockquote>
<p>对于大部分的企业级产品使用者来说，<b>基本的图表类型</b>，<b>通常可以为大多数当前页面的问题作出解释</b>，但是我们要做并不是简单地调用一个图表就结束了，如果想让你的页面更<b>有效地传递结论、原因</b>，可以让使用者<b>探索更多可能性</b>，甚至去发现新的分析思路，那么首先你需要掌握下面的分析套路。</p>
<h2 id="toc-2">二、3 个套路</h2>
<p><b>「对比，细分，溯源」，给简单的图表加点「戏」</b></p>
<p>在数据分析中，分析思路一般可以概括为「对比，细分，溯源」，这也是《数据化管理》中提到六字箴言。首先我们来了解一下这六字箴言的具体含义。</p>
<p>此处引用部分《分析思维第三篇：数据分析的三板斧》内容。</p>
<h3>1. 对比</h3>
<p>对比分为<b>横向对比</b>和<b>纵向对比</b>。</p>
<ul>
<li><b>横向对比</b>是指和“他人”比较，比如，两个网站的用户流失率；</li>
<li><b>纵向对比</b>是指按照某个维度，和“自己”比较，比如，这个网站上个月今天和这个月今天的用户流失率。</li>
</ul>
<h3>2. 细分</h3>
<p>细分是指<b>分维度</b>、<b>降低粒度</b>来分析数据。</p>
<ul>
<li>分维度是指增加维度，比如，离职率按照部门维度来分析；</li>
<li>降低粒度是指降低数据聚合的程度，比如，离职率不按年份，而按照月份来统计。</li>
</ul>
<h3>3. 溯源</h3>
<p>溯源是指在对比、细分锁定到具体维度和粒度之后，依然没有结论，那就需要<b>查看原始数据或相关联数据</b>，洞察数据，从数据中寻找灵感。</p>
<h2 id="toc-3">三、3×3 实践</h2>
<p>第一个 3 代表图表类型；第二个 3 代表分析套路。</p>
<p>了解分析思维的三个套路后，可以尝试在自己的图表中检验打钩。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/SG65wwYficqOHIDGHfeU.png" alt width="677" height="168" referrerpolicy="no-referrer"></p>
<p>下面我用几个例子来说明一下，如何打开图表设计的脑洞。</p>
<h2 id="toc-4">四、折线图</h2>
<p>通常在一个数据概览页面，人们都会使用一个折线图，来表示一个关键指标的发展趋势，当然这么做非常直观，也非常准确。<b>当我们为折线图添加一个对比的维度，就可以使折线图表达的内容更加丰满。</b></p>
<h3>1. 横向对比</h3>
<p>可以是横向对比“和他人比”。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/yoCZQyWZBc14zNdugPOD.png" alt width="677" height="495" referrerpolicy="no-referrer"></p>
<h3>2. 纵向对比</h3>
<p>也可以是纵向对比“和自己比”。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/YYY1DDoabsDbPPbJb5le.png" alt width="676" height="291" referrerpolicy="no-referrer"></p>
<h3>3. 添加辅助元素，强化关键信息进行对比</h3>
<ul>
<li>参考型组件：参考线、趋势线、辅助区域等；</li>
<li>极值型组件：最高/低点、预警线等。</li>
</ul>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/0vgBFokWE2bNR2UxeYV2.png" alt width="678" height="282" referrerpolicy="no-referrer"><br>
<img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/GLV7TVFEpkP6ISVPa7Hh.png" alt width="674" height="251" referrerpolicy="no-referrer"></p>
<h3>4. 细分维度</h3>
<p>通过右上角的 TAB 切换，提升和降低数据聚合的程度。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/0jgAXcC6b7vpAAiSF0Lf.png" alt width="678" height="268" referrerpolicy="no-referrer"></p>
<h2 id="toc-5">五、柱 / 条形图</h2>
<h3>1. 横向对比</h3>
<p>和别人比。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/2RXU7OUPzyvTPeEWsT7l.png" alt width="676" height="461" referrerpolicy="no-referrer"></p>
<h3>2. 细分维度</h3>
<p>将维度拆分进行分析。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/7Z6WK3hzgQBTiBCpKSPv.png" alt width="675" height="240" referrerpolicy="no-referrer"></p>
<h3>3. 在使用柱状图/条形图时，问问自己「对比」的呈现效果好吗？</h3>
<p>这是使用者在使用柱形图做对比时，经常遇到的问题：</p>
<ul>
<li>X 轴类别数量过多，柱子非常密集；</li>
<li>轴文字标签数量过多，文字在 45%旋转后可读性降低。</li>
</ul>
<p>这时，设计者应该试着旋转柱图，把它变成一个条形图。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/gSypzuKs7VGA9ufbyDMR.png" alt width="674" height="278" referrerpolicy="no-referrer"></p>

<h2 id="toc-7">七、拓展：NxN 实践</h2>
<ul>
<li>第一个 N 可以代表图表类型，也可以代表页面，但需要是一个实体；</li>
<li>第二个 N 代表分析思路，上文中提到的 3 个套路只是众多分析思路中被总结出最为常用的，还可以根据不同业务场景的诉求去添加思路。</li>
</ul>
<p>这个矩阵的表格填出来后，除了可以帮助设计者检验是否高度利用了图表区域之外，还可以总结出一份属于自己业务场景的图表指引，传递给下一个设计者使用。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/wULSeWkQaCCqiehTKb4F.png" alt width="677" height="318" referrerpolicy="no-referrer"></p>
<h2 id="toc-8">八、结语</h2>
<p>在企业级产品设计的领域中，用可视化的方式表达信息，让数据不再枯燥难懂，是一种常见的设计手法。</p>
<p>在真实的业务场景中，大部分用户青睐 dashboard（报表页面），来呈现「数据概览」、「数据监控」、「数据明细」等信息，其中的<b>重要组成部分即「统计型图表」</b>。如何恰当使用「统计型图表」来实现用户的分析目的？ 这对没有可视化基础的制作者来说，还是有一定的难度。</p>
<p>通过以下三步，让用户更高效地读取重要信息，<b>驱动业务决策，实现商业价值</b>。</p>
