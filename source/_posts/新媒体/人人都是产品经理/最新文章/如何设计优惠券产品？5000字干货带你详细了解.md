<h2 id="toc-1">一、优惠券基本概念</h2>
<p>对于在电商平台上购物过的小伙伴来说，相信优惠券对大家都不陌生。现在我们打开手机上的电商APP，各种五花八门的优惠券让人眼花缭乱，本文我们将从四个方面展开讲讲优惠券的相关知识。关于优惠券的展现形式相信不用我多介绍大家也都很熟悉了，我们总结一下优惠券的使用场景，我们可以将优惠券的使用场景分为三类：</p>
<ul>
<li>日常促销：用于日常降价的促销活动，一般让利降价幅度较低</li>
<li>大促活动：让利幅度非常大，一般会和满减活动等叠加使用</li>
<li>精准营销：向不同的人群，在合适的场景下，触发优惠券推送</li>
</ul>
<p>而在这些不同的场景下，优惠券的相关用户可大致分为三类：运营、商家、用户。</p>
<ul>
<li>运营：一般的平台运营人员会负责创建整个平台层面使用的优惠券，一般可用于整个平台品类或商家。</li>
<li>商家：对于一些平台型的电商，会有商家进行入驻，平台会给商家管理后台用于创建店铺可用的优惠券。</li>
<li>用户：即优惠券的使用者。</li>
</ul>
<p>除了使用场景，我们总结一下优惠券能为我们提供的价值，主要有五个：</p>
<ul>
<li>提升GMV：以满减券为例，为了凑满减的优惠，用户会增加购买的金额以达到使用优惠券的门槛金额，从而提升了平台的GMV。</li>
<li>用户拉新：以新人券为例，平台经常会为新人设置大额的优惠券，仅限新人使用，以吸引用户使用平台购物，达到拉新的目的。</li>
<li>易于创新：优惠券是可以说是大部分促销活动的基石，我们可以在优惠券的基础上进行包装与开发，如一些社交优惠券的玩法。</li>
<li>认可度高：优惠券在国内电商平台使用频率非常高，加上电商这几年的高速发展，用户已经被教育得非常成熟，优惠券已经成为一种十分受用户认可的促销手段，以至于现在很多用户会在只用优惠券的时候才购物。</li>
<li>易于理解：相较于一些比较复杂的活动，优惠券活动比较简单易懂，比如无门槛券、直降券等，能够简单了解即了解产品规则。</li>
</ul>
<p>最后一个部分我们大致探讨一下优惠券在产品发展不同阶段我们需要的建设程度。我们将电商系统粗略得划分为三个阶段：初期阶段——>中期阶段——>后期阶段。</p>
<p><strong>1）首先是初期阶段，本阶段优惠券产品的主要目标是“保证交易流程通畅”</strong></p>
<p>对应的优惠券产品应该具备的功能有：</p>
<ul>
<li>创建优惠券，基础的券有满减券和现金券</li>
<li>发放优惠券，有入口可以让用户获取优惠券</li>
<li>下单时可以选择使用优惠券</li>
<li>优惠券数据的查看，包括领取/发放/使用等数据</li>
</ul>
<p>需要注意的点：</p>
<ul>
<li>初期阶段我们最主要的是保证整条交易链路的顺畅，保证正向与逆向流程。</li>
<li>初期优惠券不需要开发过多类型，比如最基本的满减券、现金券</li>
<li>不一定需要单独的后台创建，可以根据业务灵活设计</li>
</ul>
<p><strong>2）然后是中期阶段，本阶段优惠券产品的主要目标是“发放方式的创新”</strong></p>
<p>对应的优惠券产品应该具备的功能有：</p>
<ul>
<li>商品购买的核心链路上透出优惠券，如商品列表、商品详情、购物车等</li>
<li>单独的优惠券领取中心，可在此处领取所有可领取的优惠券</li>
<li>优惠券玩法创新，结合社交裂变、抽奖或任务的形式发放优惠券</li>
</ul>
<p>需要注意的点：</p>
<ul>
<li>优惠券创建需要考虑优惠是否超发的问题</li>
<li>大额优惠券创建后，可加入审核流程，保证正确性</li>
<li>此阶段可开发单独的后台配置优惠券</li>
</ul>
<p><strong>3）最后是后期阶段，此阶段优惠券产品的目标是“券场景的变化”</strong></p>
<p>对应的优惠券产品应该具备的功能有：</p>
<ul>
<li>渠道控制：创建单独渠道可用的优惠券，如限制APP，限制PC等</li>
<li>券类型扩充：增加不同场景可用的券，如门店券、外卖券、运费券等</li>
<li>场景限制：对单独某一人群或地域进行券的生成和发放</li>
</ul>
<p>需要注意的点有：</p>
<ul>
<li>需要新增券类型时，优先考虑当前优惠券是否可用，遵循最小成本原则</li>
<li>切勿盲目发放优惠券，导致券的价值变低</li>
<li>我们要遵循现有业务再有产品的原则，不要凭空臆想创建没有业务场景的优惠券</li>
</ul>

<h2 id="toc-3">三、优惠券发放/领取</h2>
<p>本小节我们探讨优惠券的发放/获取方式。我们从用户角度看，根据领取方式的不同可以分为系统直发、日常领取、活动领取三种方式，同时我们也可以将系统直发划分为被动获取的方式、日常领取和活动领取划分为主动获取的方式。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/5NCuyKF2tfenvhj73gna.png" alt width="1890" height="959" referrerpolicy="no-referrer"></p>
<p>下面我们分别介绍这几种优惠券的发放与获取方式。</p>
<p><strong>1）系统直发—人群定向发放</strong></p>
<p>人群定向是通过人群ID导入后，直接向对应人群发放优惠券的方式，以平台拉新人举例说明：</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/F9Qh43Qnrcz0twjZiBWd.png" alt width="1512" height="852" referrerpolicy="no-referrer"></p>
<p>新人定义为有意义注册但未有购买行为的用户，可定义为一年/半年内未购买的用户。</p>
<p><strong>2）系统直发—个性化人群</strong></p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/LQr0RLjyMd520fCKQIJC.png" alt width="1538" height="573" referrerpolicy="no-referrer"></p>
<p>系统根据推荐算法将合适的优惠券推荐给合适的人群，提高流量价值。</p>
<p><strong>3）日常领取</strong></p>
<p>在核心支付链路上，列表页、商详页、购物车、收银台、支付完成页、评价页，均可以展现优惠券领取入口，提高优惠券的曝光量。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/dSsyTw4KsxRJylW2WMps.png" alt width="2122" height="1096" referrerpolicy="no-referrer"></p>
<p><strong>4）活动领取—定时抢券</strong></p>
<p>我们来讲一个案例：双11大促，老板给你定了几个KPI，分别是</p>
<ul>
<li>把图书卖爆，要是日常的10倍以上</li>
<li>要有5倍日常流量引入，吸引用户进入专题页</li>
<li>但是，预算有限，只能给部分补贴，视情况增减</li>
</ul>
<p>遇到这种情况你该怎么办？方法之一是引入定时抢券的玩法，其好处有下：</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/C3hZk0JWwHjv0AxuM3c3.png" alt width="600" height="534" referrerpolicy="no-referrer"></p>
<p><strong>5）活动领取—瓜分优惠券</strong></p>
<p>相信小伙伴们经常会看见饿了么与美团的外卖红包，第X个人领取最大的红包。我们称这种形式为瓜优惠券，其领取流程见下</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/YXcutXbPNnXAfmrPL20c.png" alt width="1006" height="1020" referrerpolicy="no-referrer"></p>
<p>产品逻辑：</p>
<p><strong>1）购买者</strong></p>
<ul>
<li>外卖平台（也可以是其他电商平台）下单，获取红包分享资格</li>
<li>将红包分享到微信群聊</li>
</ul>
<p><strong>2）领取者</strong></p>
<ul>
<li>查看链接</li>
<li>输入手机号和验证码，判断是否注册，已注册直接领取，未注册则默认手机号注册</li>
<li>领取优惠券，需要校验每日的领取次数</li>
<li>优惠券从后台配置读取，需要提前设置优惠券</li>
</ul>
<p>瓜分优惠券方案有两种，一种是按领取顺序排列，第N个领取获得最大优惠。比如800元的优惠额度，按领取顺序优惠额度为200、100、300、100、100。第二种是将整体额度拆成几种面额的优惠，用户领取时按一定的概率领取相应的优惠券。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/ZnrsfqkI4IP854p3SaSk.png" alt width="1790" height="725" referrerpolicy="no-referrer"></p>
<p>使用优惠券这种促销形式时需要注意的点如下：</p>
<ul>
<li>优惠券是最常见的权益，互动玩法无论怎么变化，最后结果都是获取到优惠券2）把优惠券换成其他奖励，玩法同样成立，如获得红包</li>
<li>难度较大的玩法，需要匹配价值感高的券，否则会出现没人玩的场景</li>
<li>提前约束玩法发放的优惠券数量，以保证不在会亏损</li>
</ul>
<h2 id="toc-4">四、优惠券管理</h2>
<p>就像新鲜的苹果不吃就会过期，优惠券不使用也会过期，这一部分，我们来聊一聊优惠券的管理。首先是优惠券的核销，我们分别分析京东、淘宝、拼多多的优惠券核销逻辑</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/CeVtP2trjlbl7MGYSR7O.png" alt width="600" height="604" referrerpolicy="no-referrer"></p>
<p>京东优惠券使用逻辑：</p>
<p>1） 京东优惠券可以下单使用，支付时使用，同时也支持分享赠送，这些操作都称为核销</p>
<p>2）优惠券需要提前领取，只可选择已有的优惠券</p>
<p>3）京东优惠券在结算页时，会自动推荐优惠券，逻辑是：</p>
<ul>
<li>优先查询可用的优惠券列表，包括可以叠加的优惠券</li>
<li>若有多个券可用，优先选择金额较大的优惠券</li>
<li>自动勾选优惠券，并展示在界面上</li>
</ul>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/bgnsr00532G8JjBF2Dzt.png" alt width="600" height="584" referrerpolicy="no-referrer"></p>
<p>淘宝优惠券使用逻辑：</p>
<p>1）淘宝优惠券只能下单使用并核销</p>
<p>2）优惠券不需要提前领取，进入结算页后，系统自动领取符合条件的优惠券，如订单金额是50元，系统会领取50元以内最高额度的优惠券</p>
<p>3）淘宝优惠券在结算页时，不会展示具体的优惠券，是以打包优惠的形式展现</p>
<ul>
<li>若优惠券是可以叠加，打包计算总的优惠金额</li>
<li>自动勾选优惠券并展示最终总的金额</li>
</ul>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/fxj5dYQJFP8WAZjPGkCW.png" alt width="600" height="602" referrerpolicy="no-referrer"></p>
<p>拼多多优惠券使用逻辑：</p>
<ul>
<li>拼多多优惠简单清晰，只有单品促销和单品优惠券</li>
<li>优惠券只能下单使用并核销</li>
<li>优惠券不需要提前领取，既可以在商详页领取，同时也可以在结算页领取，领取后默认使用</li>
<li>优惠券可以重复领取，直至达到限定的次数</li>
<li>拼多多优惠券分为店铺券和平台券，分开展示，分开领取</li>
</ul>
<p>接着我们看一下优惠券的逆向流程，分为售前取消与售后取消。</p>
<p>首先是售前取消的逻辑：</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/u5qh6TzzdWujRs0iDbPu.png" alt width="1946" height="797" referrerpolicy="no-referrer"></p>
<p>售前取消分用户主动取消、被动取消（如商家发货前取消订单等特殊情况），但是处理逻辑是一样的。订单是否拆单也会对券的逆向流程产生影响，未拆单下券全额返还，已拆单的情况下根据是否全部取消情况也不同。注意部分取消的情况，返拆分券的情况不常用，返等值代币和不返的情况比较多，需要注意的是不返的情况需要提前在优惠券的使用规则中说明。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/07/G72kYyXxXcIFvz4h0g0x.png" alt width="1415" height="734" referrerpolicy="no-referrer"></p>
<p>另一种是售后取消，售后只能用户主动取消了，商家或系统不能未经用户同意擅自取消其订单。其余逻辑判断与售前取消一致。在订单使用优惠券后产生售后，其优惠分摊是按商品价格加权按比例分摊的，公式为：（商品金额/订单总金额）*优惠券金额。如订单金额1000元，商品A和B的价格分别为800元和200元，使用一种满100-50的优惠券。那么A分摊40元的优惠，B分摊10元的优惠。</p>
<p>接下来我们根据一个实际案例来看优惠券的逆向流程：</p>
<p>小明购买了一台1499元的手机和一个99元的充电宝，下单时使用一张满500-50的优惠券，两个商品分属不同商家，订单拆单发货。</p>
<p>场景1：小明发现更好的充电宝需将原充电宝退掉，券如何退？</p>
<p>我们首先根据价格计算出充电宝的优惠金额=（99/1548）*50=3.20元</p>
<ul>
<li>方案1：返还等值优惠券，如满10-3或满10-4的优惠券</li>
<li>方案2：返还等值代币，如平台积分与人民币是1比100，则返还320积分</li>
<li>方案3：不返，提前告知用户优惠券使用规则</li>
<li>场景2：小明不想要了，整单退款，则原优惠券全额返还。</li>
</ul>

<h2 id="toc-5">五、总结</h2>
<p>至此，我们总结一下本章内容，分为四个部分：</p>
<p><strong>1）了解优惠券建设程度</strong></p>
<p>分为三个阶段，初期，保证交易流程通畅；中期，建设玩法发放方式的创新；后期，丰富场景，券场景的变化。</p>
<p><strong>2）掌握优惠券创建过程</strong></p>
<p>券主要分为满减券，折扣券和现金券，各种券创建时，必备字段包括使用时间，优惠券金额，优惠券门槛，优惠券库存，优惠券限制等。</p>
<p><strong>3）掌握优惠券发放玩法</strong></p>
<p>主要两种发放方式，直接领取和被动领取，直接领取一般在购物链路上可以领取如商详页或购物车等，再者是通过玩法获取。被动领取一般是系统定向直发优惠券，如发给新人人群优惠券</p>
<p><strong>4）掌握优惠券生命周期</strong></p>
<p>重点学习优惠券的正逆向使用过程，优惠券在整个过程中的状态变化，以及优惠券的数据统计等流程。</p>

<p>在电商系统中，优惠券可以说是所有促销优惠活动的重要基石，我们可以在优惠券的基础上灵活开发更多丰富好玩的活动，所以学习好最基础的优惠券产品非常重要。以上就是优惠券模块所有的内容了，部分内容只是简单介绍，有兴趣的小伙伴可以深入了解。下一期，我将分享电商的用户体系相关知识，尽请期待。</p>
