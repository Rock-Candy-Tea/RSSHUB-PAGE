<blockquote><p>编辑导语：在做新一年的增长策略时，你有尝试过用新的方法拆解吗？这篇文章中作者分享了增长策略的3种拆法，并从「为什么拆、怎么拆、实操案例」来总结，推荐想要了解增长策略的童鞋阅读。</p></blockquote>
<p><img data-action="zoom" class="size-full wp-image-761390 aligncenter" src="https://image.yunyingpai.com/wp/2022/02/WCtYU1WjsIQSmNw1j2qE.png" alt referrerpolicy="no-referrer"></p>
<h2 id="toc-1">一、为什么要拆解</h2>
<p>对于为什么要拆解增长策略这件事情上，首先要理解「增长策略」。增长策略是体系化、有方法论的，能让业务增长有迹可循；也能让业务不增长时快速定位到哪个环节出了问题。当然增长策略不是今天说要做这个，我们思考一下策略，明天说再增加一个，我们再重新整理一下策略…这样来的，如何做到<strong>科学制定增长策略</strong>，就涉及到「拆」。</p>
<p>其次，如何理解「拆解」。它不是朝思暮想：看到市场、竞对在用什么方式就什么都尝试（“不断变化”的拿来主义）；也不是投机取巧：走便宜的渠道或者方法，快速起量；而后又快速下降却不知道为什么。「拆解」是<strong>通过建立增长模型，找到每个阶段的关键假设（最最能影响增长的那个东西），不断快速验证，再根据数据和用户反馈来优化，始终围绕这增长模型来实验、来迭代</strong>。</p>
<p>行动的第一步往往最艰难，拆解也是一样。所以<strong>「拆」与「不拆」区别很大</strong>！拆得好不好倒是其次了。因此先不要想着自己拆得对不对、拆得好不好，先拆起来，多从几个维度来拆，再去自我博弈+小组共创，自然就能得到答案。</p>
<h2 id="toc-2">二、怎么拆解&实操案例</h2>
<h3>拆法1：战略+策略+打法</h3>
<p><strong>1.1 战略选择</strong></p>
<p>战略层最应该先判断的是<strong>「做不做」</strong>，可以从2个方面来评估：<strong>1）ROI评估；2）价值分析</strong></p>
<ul>
<li><strong>ROI评估</strong>：评估成本、收入，以此来衡量做这个事情是否划算。</li>
<li><strong>价值分析</strong>：横向评估，思考可替代项的价值有多少。</li>
</ul>
<p>其中个人感悟的是，ROI评估适用于中小型公司的新业务及新活动，不是很适用于大公司已有业务，原因是大公司中变量太多，同时受其他部门动作影响，较难只是通过单一的ROI指标去衡量，因此通常会产生转化人数「复算」的情况。</p>
<p>价值分析特别适合去评估一个策略、一个活动做不做。可以从2个方面来做价值分析：</p>
<ul>
<li><strong>机会成本</strong>：在此之前，肯定是需要先了解「用户是谁、现状如何、矛盾点是啥」，才能助于判断机会。</li>
<li><strong>窗口期</strong>：主要是看外部环境给到的空间大不大，判断现在是不是做的最佳时机。</li>
</ul>
<p>以滴滴为例，窗口期就特别重要。2012年成立，在往后的几年时间内发展迅速。其中离不开中国整个互联网环境带来的机遇和造势，假设将时间再往前推个10年，那会儿的中国互联网还没发展起来，各项技术、人们的观念、普及程度都不足以支撑滴滴往更精细化的方向发展。</p>
<p>战略层在判断完「做不做」后，需要将<strong>「第一指标锁死」</strong>。锁死的好处是，防止在执行的过程中，团队成员因为各种各样的状况不得不抉择舍弃需求点时，能始终以第一指标作为最高优先级考虑，不会迷失了方向。</p>
<p><strong>1.2 策略制定</strong></p>
<p>策略制定的环节，方法是<strong>「指标拆解→加减法思考→阶段性复盘」。</strong></p>
<ul>
<li><strong>指标拆解</strong>：这个在 拆法2 中详细讲解。</li>
<li><strong>加减法思考</strong>：基于分析用户和场景；然后做大量加法，找出活动玩法所有可能的选项；而后不断筛选，找出最佳选项。</li>
<li><strong>阶段性复盘</strong>：复盘是非常好的反思过程，以前会理解是活动/项目结束后才进行复盘，这样会有一个弊端是拿反馈、拿认知太慢了，因此更需要我们做阶段性复盘，现在频率变为一周/双周一复盘，且在团队里重视复盘是非常好的习惯。</li>
</ul>
<p><strong>1.3 战术打法</strong></p>
<p>从战略到策略到战术，是从宏观到中观到微观、一步步下来的。战术层即微观的技巧和套路。这里注意是「技巧和套路」，这是离不开复盘和总结的，希望每次打法，都能够基于上次经验形成方法、形成套路，多复用便多节省探索成本。</p>
<p><strong>1.4 实操案例</strong></p>
<p>以最近负责的存量转化业务制定增长策略为例：</p>
<p><strong>① 第一步：战略选择</strong></p>
<p>战略的各个方面最好以Q1或H1为单位来更新，多关注用户群、矛盾点、外部环境是否有变化。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/02/Y2PhnfAZnd8IcFC9bGyE.png" alt width="599" height="262" referrerpolicy="no-referrer"></p>
<p><strong>② 第二步：策略制定</strong></p>
<p>以下是针对拆解公式中需要重点关注的指标做的一些策略思考，不全、还在补充和battle中。这里也需要拆分角度来看，比如是短期策略还是长期策略，当前对指标的要求是快速起量还是稳健增量，这些都有助于我们做减法，找到最适合当前达成目标的几条策略。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/02/cC5RyTYVZ4KAHbygS34a.png" alt width="601" height="254" referrerpolicy="no-referrer"></p>
<p><strong>③ 第三步：战术打法</strong></p>
<p>战术打法这里就不赘述了，到比较微观的层面则是根据业务来不断调整了。</p>
<h3>拆法2：业务公式模型</h3>
<p>拆法2是以业务公式模型核心，有4步骤需要完成：<strong>① 建模型&拆解GMV → ② 明确指标 → ③ 梳理方向及制定策略 → ④ 落地执行</strong></p>
<p><strong>2.1 建模型&拆解GMV</strong></p>
<p>建模型是一种初步定量的分析方法，需要找到现有业务各环节之间的联系。常见的几种拆法是：拆收入、拆ROI。</p>
<ul>
<li><strong>拆收入</strong>：可以以倒推的方式来组装模型，如教育行业，收入来自用户购买课程数/包（客单价），那整体收入可以理解为=用户数*客单价。以这个来倒推，用户数等于进入落地页人数*转化率；进入落地页人数又等于A*B….不断倒推，来组建模型。</li>
<li><strong>拆ROI</strong>：ROI=收入/成本。上面是拆收入，那拆成本可以分为显性成本和隐性成本。以一个活动来说，显性成本包括活动推广成本、活动奖励成本、采购成本等；隐性成本包括活动人力成本、维护成本等。另外还有履约成本，如通讯成本、税收成本、仓储成本等。</li>
</ul>
<p>用哪种拆法，取决于公司发展阶段及业务。比如中小型公司及新业务，拆ROI为主；上市公司及业务扩张期，拆收入为主。（个人理解）</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/02/KJBrekqxqvtA8Gl336OP.png" alt width="478" height="448" referrerpolicy="no-referrer"></p>
<p style="text-align: center;">图：核心指标体系，来源书《引爆用户增长》</p>
<p><strong>2.2 明确指标</strong></p>
<p>明确指标也分3步：<strong>① 警惕虚荣指标 → ② 找到第一指标 → ③ 找到当前急需解决的指标</strong></p>
<ul>
<li><strong>警惕虚荣指标</strong>：是那些只能振奋人心，但无法指导我们去做增长决策的指标。没有任何说服力、不能体现真正产品价值的指标。如点击量、访问人数等等（详见下图）</li>
<li><strong>找到第一指标</strong>：具有3个特点，即<strong>真实的，切实可行的，具有先导性。</strong>通常第一指标直接for转化或收入，会根据上层领导及业务发展来决定。（在 <a href="http://www.woshipm.com/operate/5075742.html">之前的文章</a> 有提及，因此不在此赘述）</li>
<li><strong>找到当前急需解决的指标</strong>：<strong>这个对于至少0-2岁的产品经理来说，非常重要；同时也是宝贵的锻炼数据分析及提出假设能力的机会</strong>。如何找到需要分析各环节漏斗情况，清楚认知当前提升哪个指标能最影响第一指标。在此基础上，提出合理假设并不断自我博弈，再去低成本验证；这样一套下来，是比较能培养增长产品经理做好科学增长的方式了。</li>
</ul>

<p style="text-align: center;">图：8种虚荣指标，来源书《精益数据分析》</p>
<p><strong>2.3 梳理方向及制定策略</strong></p>
<p>下图是个人思考策略的模板，依然基于业务公式模型，从【目标→指标→执行方向→落地执行】拆解，这样做的好处是：</p>
<ul>
<li><strong>清楚定位</strong>：能清楚知道自己做的事情是为了哪个指标服务，不是为了做而做。</li>
<li><strong>合理预判</strong>：当他人跟你提需求时，能根据这套策略来判断这个需求是否合理、从哪个方向来打，不至于迷失了方向，预判错误造成大损失。</li>
</ul>

<p><strong>2.4 实操案例</strong></p>
<p>还是拆法1中的案例，换成拆法2来做：</p>
<p><strong>① 第一步：拆解业务公式，建模型</strong></p>
<ul>
<li>已定Q1目标是提升PC (paying client) 人数，PC人数 = 存量用户数 x 用户活跃率 x 活动参与率 x 活动转化率 x 审核通过率。光是知道这个公式很难落地，所以进一步拆解：</li>
<li>PC人数 = 提交申请人数 x 审核通过率</li>
<li>提交申请人数 = 参与活动人数 x 活动转化率</li>
<li>参与活动人数 = 可用用户量 x 活动参与率</li>
<li>可用用户量 = 存量用户数 x 用户活跃率</li>
</ul>
<p>拆解后，就能明确自己可以往哪个指标怎么发力，否则光知道一个「提升PC人数」太泛，很难聚焦。</p>
<p><strong>② 第二步：明确指标</strong></p>
<ul>
<li>首先第一指标是明确的：提升PC人数</li>
<li>其次当前急需解决的指标，根据数据分析来。比如数据显示参与活动的人很多，但是转化的人很少，<strong>且在整个业务公式中是下降(少)得厉害的</strong>，那当务之急就是优化活动流程/体验，提升转化率。明确这个指标后，需要落地指标，比如从25%提升到35%，不能光说「提升转化率」。而后再往下拆解，是哪个环节导致转化差，这个环节用户表现如何，走到哪一步就不继续了，为什么不继续了，有什么办法让他继续….一系列思考，非常锻炼人！</li>
</ul>
<p><strong>③ 第三步：制定增长策略</strong></p>
<ul>
<li>比如我要提升活动参与率，应该如何思考执行方向才比较不MECE，个人习惯从「<strong>渠道、内容、产品</strong>」这三个维度来思考：</li>
<li>渠道：不要狭义，要广义来理解。渠道除了指站内站外各种资源位外，还包括<strong>MOT(关键时刻，moment of truth)，是指用户在使用产品过程中的关键时刻，简单可以理解为「体验好的、他愿意为你买单的时刻」</strong>。这些也是很好、高性价比的「渠道」，比信息流有效多了。</li>
<li>内容：有了渠道把用户引来了，提供什么内容给用户呢？是增加更多活动类型、还是现有类型迭代，还是….，这些都要谨慎思考。</li>
<li>产品：有了好内容，产品的作用就是让用户在这段「体验旅程」中最大程度地享受这些内容。包括产品路径的设计、页面的包装、恰到好处的挽留、有规律的召回等等。</li>
</ul>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/02/JS4jxJv7FJsaCjWQU94W.png" alt width="487" height="112" referrerpolicy="no-referrer"></p>
<p style="text-align: center;">图：从渠道、内容、产品来制定提升活动参与率的执行方向</p>
<p><strong>④ 第四步：落地执行</strong></p>
<ul>
<li>落地执行就不展开赘述了，具体要以需求的优先级来安排。实验的优先级怎么排，个人习惯是按照<strong>「高性价比的策略优先级高」、「影响大的策略优先级高」</strong>这两个方法来排优先级。</li>
</ul>
<h3>拆法3：增长飞轮</h3>
<p>亚马逊的杰夫·贝索斯的增长飞轮，本质上是串联起关键增长要素，让业务的增长形成一个高效方便的闭环，持续运转。一旦飞轮滚起来，就会越滚越大，最终形成增长壁垒。</p>
<p><strong>3.1 要做成一个飞轮，需要以下4步：</strong></p>
<ul>
<li>找出关键增长因素：尤其是企业已经实现的、可复制的成功，如果没有，那就从失败过的经历中尝试总结找出「可能成功」的增长因素。</li>
<li>分析关系：分析各因素之间的因果关系，如A→B→C之间的逻辑关系。</li>
<li>尝试闭环：将几组关系尝试串连，不断快速试错。</li>
<li>跑通拉动：跑通的闭环，尽量多沉淀复用，让轮子越滚越省力。</li>
</ul>
<p>增长飞轮更适用于战略层，特别是在思考增长方向和发力点时。当找到了关键增长因素，才能继续往下拆解。</p>

<li>已经实现的成功：口碑推荐，APP内利益刺激，人工外呼….</li>
<li>有机会成功：KOL计划，直播，讲座，高级内容付费包…</li>
</ul>

<h2 id="toc-3">三、心得总结</h2>
<p>写到最后，附上一张拆法的方法论简图，此外有2点特别说明：</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.yunyingpai.com/wp/2022/02/nbeaReIuyGG8WXE2nWjQ.png" alt width="601" height="310" referrerpolicy="no-referrer"></p>
<p style="text-align: center;">图：3种拆法总结简图</p>
<ol>
<li> <strong>这3种拆法有何联系：</strong>从图中的红色箭头可以看到，我会理解为这3种拆法是从上到下、从宏观判断到微观落地。从拆法1的策略制定开始，便可用拆法2这套模型来建立并细化；拆法2的具体执行，又可从拆法3来挖掘和尝试。</li>
<li><strong>如何判断什么时候该用哪种拆法：</strong>没有明确的范围界定，视公司当前发展阶段、个人负责业务板块及个人熟练程度来定。如果是做新业务，我会习惯用拆法1来评估这个新业务是不是适合做；用拆法2来明确GMV公式及策略方向。当然<strong>不管用什么拆法，先跑起来最重要</strong>。</li>
</ol>
<p>「方法论」总是纸上得来终觉浅，绝知此事要躬行，因此<strong>刻意练习→不断更新→反复迭代</strong>是将方法论内化很好的步骤。</p>
