<blockquote><p>编辑导语：在竞争愈来愈激烈的互联网时代，互联网公司的内卷也十分严重，那么如何在这激烈的竞争提升能力并且得到晋升呢？本篇文章详细讲述了在职场中不断提升所具备的能力等，一起来学习一下。</p></blockquote>
<p><img data-action="zoom" class="size-full wp-image-5347847 aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2022/03/BNkt65DTXmkHlHqwhu29.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<p>职场是个组织，这里的升迁等有一定的规律，从执行到带人，是职场发展的第一关。</p>
<p>但在互联网企业呆了7年以后，不能说完全了解升迁的规律，但我已经知道了。</p>
<p>什么样的人执行这一关算是练满级了，即具备了升职条件。主要是以下几方面：</p>
<ul>
<li>业务能力：做事利落，产出及格分以上；</li>
<li>跨部门沟通：足够的沟通和说服力，能顺利推动工作；</li>
<li>向上管理：做好汇报，上下级关系和谐；</li>
<li>个人影响力：为团队做贡献（制度化，做团队服务，为团队提效做贡献）。</li>
</ul>
<p>这篇讲的东西适合产品、运营和一切其他的职位，因为我比较熟悉产品工作，所以以产品工作来举例，但相信对于别的职位也有同样的参考价值。</p>
<h2 id="toc-1">一、业务能力</h2>
<p>业务力的底层其实是解决问题的能力，今天分享几个比较通用的职场好习惯，做好的话无论是产品、运营还是其他的职位，都可以成为一个基本面很好的职场人士，未来可期。</p>
<p>就产品经理来讲，业务能力主要是以下这些方面，每个方面都是复杂且需要大量努力来精进的。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2022/03/wiu8rMLF20Q8emRnLujC.png" alt width="2294" height="654" referrerpolicy="no-referrer"></p>
<h3>1. 拆解任务的能力</h3>
<p><b>会拆解任务的话，即使没有经验也能通过自己的逻辑把事情做的不错，是很牛的底层大招。解决问题的高手都是拆解问题的高手。</b></p>
<p>今天芋圆就来教大家拿到任务后如何拆解：</p>
<ul>
<li>首先，通过反复问自己目标是什么，要怎么达成，来拆成尽可能多的步骤，每一步都拆到足够简单，容易做到，拆解的过程就是迫使自己思考解决方法的过程了。</li>
<li>通过一步步的拆解，把一整个事情的达成路径拼出来，剩下的就是一步步去做就好了。</li>
<li>做的时候可能会在某个环节遇到问题或者很复杂，那就是之前的设想还是太简单了，就接着针对这个环节进行细拆。</li>
</ul>
<p>就像著名心理咨询师武志红老师说的：</p>
<blockquote><p>“当你把一个大目标拆解成很多小目标后，或者说，把一个完整的过程，分解成一个又一个具体的小步骤时，这个光滑的斜坡，就变成一个又一个台阶了。有台阶之前，做事情真是不进则退，而且可以一退就退到原点；有了台阶之后，可以只进不退了，因为你可以安全、稳固地待在你爬上的一个又一个台阶，而且你还可以在上面休整，准备出更多的动力，让自己继续努力”。</p></blockquote>
<h3>2. 学会站在前人的肩膀上，不要重复造轮子</h3>
<p>有句话说得很好：你遇到的99.9%的问题都有人遇到，并解决过了。所以站在前人的肩膀上去做微创新是性价比最高的。</p>
<p><b>1）利用好搜索引擎。</b></p>
<p>互联网时代分享方法论的人很多，搜索一下，看到好的方法补充道自己的原则里，每天都持续这么做，啥事磕不明白呢。</p>
<p><b>2）</b><b>多利用同事的经验也是很有智慧的方式。</b></p>
<p>同事的经验更有因地制宜性，而且也可以跟同事拉进关系不是。</p>
<h3>3. 积累自己的做事框架</h3>
<p>资深产品经理和新人产品经理的差异就是是经验，经验说白了就是，这些事情他做过，下次再遇到知道怎么做。</p>
<p>那没经验的新人该怎么办呢——通过我们的拆解也好，搜索也好，得到做事的方法。做的过程发现还有很多没想到的坑，一路填坑背锅，最后终于把事情做完了。</p>
<p>马上梳理记录下来下次这类事情怎么做，作为SOP<strong>。</strong></p>
<p>一个新人每次做事都这样复盘记录一遍，和一个老人做事情纯靠感觉和手熟，前者是不是肉眼可见很快就可以超过后者了。</p>
<p>所以我之前总说<b>产品新人2-3年获得别人5-5年经验，完成是有可能的</b>，只不过产品职位的知识体系太荒芜，大家看不到较短的成长路线。</p>
<p>当你积累的框架越多的时候，框架之间也会融汇贯通、举一反三，正确做事情的逻辑就在脑子里自发形成，就会触类旁通了。</p>
<p>关于积累自己做事的框架，有本书专门布道这个方法，就是著名的《原则》。</p>
<h3>4. 工作日志——真的不要小瞧计划和复盘</h3>
<p><b>1）每天列to do，是掌控自己工作最好的方法。</b></p>
<p>产品经理每天处理的事情多且繁杂，不早点养成记录和计划的习惯，很容易丢三落四，让自己的工作进入困难模式。这样基础的问题都没搞定，又如何进入更高层次的成长呢。</p>
<p><b>2）复盘才能成长。</b></p>
<p>我也是这两年才深刻理解这个点。认知的成长来源于大脑中原本孤立的知识点产生连接。</p>
<p>不复盘相当于知识点衍生了，但最重要的连接那一步没做，是非常可惜的。</p>
<p>随着时间的流逝就淡忘了。</p>
<p>复盘的时候具体复盘什么：</p>
<ul>
<li>完成了什么；</li>
<li>还有什么待做；</li>
<li>获得了什么成长；</li>
<li>错事本，错话本。</li>
</ul>
<p>这个是个很厉害的工具，可以用在任何自己不擅长的事情上，它就像上学时候的错题本，熟悉可以练习的人都知道，错题本是刻意练习的精髓呀。</p>
<p>送给大家一句话，<b>万事皆可盘，万事也都经不住盘。</b></p>
<p>哪些可以形成原则，形成SOP。</p>
<h3>5. 随手“美化”自己的产出，会让你的工作更顺利</h3>
<p>人都是视觉动物，产出乱七八糟，谁也不想看。</p>
<p>不是鼓励做表面功夫，跟我们出门也希望把自己打扮的干干净净的是一个道理。</p>
<ul>
<li>形式美化——你不得不承认每个人都是“外貌协会”。</li>
<li>总是为自己的论点、判断寻找支撑，会早早训练你的思维，长期下来自己会是一个做事判断都很靠谱的人（注意是真实的支撑，不是狡辩）。</li>
</ul>
<p>尤其是产品经理、运营，因为身上承担的责任很重，经常会要面对和解答很多别人的疑问，这个习惯就更加重要。</p>
<h2 id="toc-2">二、沟通</h2>
<p>职场中的每一项工作任务都有无数次的信息交换，信息交换过程中会出现损失、错误等。</p>
<p>在职场上久了你会发现，光是大家理解歧义导致的沟通问题都到处可见。</p>
<p>沟通是一门深奥的学问，这也是为什么沟通能力强的人往往在职场中升迁更顺遂。</p>
<p>下面从职场中常见的沟通场景来讲：</p>
<h3>1. 接收任务的正确姿势</h3>
<p><b>接收任务时，一定要多问。</b></p>
<p>在职场上混迹久了的人都知道，接任务时反复问问题的人，比只是点头说好的人做事让人放心得多。</p>
<p>我自己为了保证这一步高效无遗漏，也会给这一步准备一个checklist，也是我慢慢记录迭代起来的。</p>
<p><b>1）养成做任务明确价值的习惯。</b></p>
<p>很多人觉得在职场上就是听吩咐做事。但恕我直言，如果你只是听吩咐，在职场上很难有很大的发展。</p>
<p><b>2）最重要的是培养自己的思考能力，</b>事情都是给你练手的。</p>
<p>不要成为做事的老黄牛。</p>
<h3>2. 会议沟通好习惯</h3>
<p><b>1）开会前了解会议目的，列出所有自己的疑问，会议上逐个解决。</b></p>
<p>不打无准备的仗，尤其是产品经理、运营出去开会经常是去接需求的。</p>
<p>对于要聊的事情肯定事无巨细要了解清楚，这也是一个职场上很好的习惯。</p>
<p>建议大家早点养成，我现在就是这样，其实也比较简单，会前浏览一下材料，随手记录问题，十分钟就搞定了。</p>
<p><b>2）</b><b>一定要有会议后发会议纪要的习惯。</b></p>
<p>很多职场新人不知道会议记录记什么，往往啥都记，累的不行还会遗漏很多重点，会议记录的重点主要是：</p>
<ul>
<li>达成一致的结论；</li>
<li>存在的分歧；</li>
<li>会后的to do；</li>
<li>以及to do对应的交付时间。</li>
</ul>
<h3>3. 跨部门沟通</h3>
<p>跨部门沟通也是需要小心处理的，毕竟出去是代表自己整个部门，所以要注意以下几点：</p>
<p><b>1）项目进度，多同步相关的人。</b></p>
<p>事事有着落、有回应是互联网职场的潜规则啦。</p>
<p><b>2） 遇到分歧的时候，新人多汇报请示。</b></p>
<p>不知道如何决策的话，不要急于下结论，“我回去内部讨论一下，有结论同步”是一句很好用的话。</p>
<h3>4. 以下小习惯让别人喜欢你</h3>
<p><b>1）不要不经思考就随便问别人。</b></p>
<p>大家平时工作都很忙，能自己搜索到答案的简单问题就不要去问别人。太不经过大脑的问题，容易遭到别人的反感。搜索途径可以是组内文件，内网，专业社区网站，或者搜索引擎等。</p>
<p><b>2）多夸奖人，谨慎发表负面评价。</b></p>
<p>职场上口无遮拦是大忌，因为真的，不知道哪天，那个人可能就变成你领导了。还是要多结善缘。</p>
<p><b>3）实事求是，诚恳的沟通态度，永远是对的。</b></p>
<p>这是更高效，在互联网行业中也更受欢迎的为人处事态度。</p>
<h2 id="toc-3">三、向上管理</h2>
<h3>1. 请示上级意见时，要有方案后再请示，甚至不止一套方案。让领导做选择题</h3>
<p>这个逻辑是，任务交给你，这个任务相关费心费时的事情都应该由你来搞定。</p>
<p>领导最多帮你做个决策或review。否则任务没有做完整，属于失职，会让老板产生不如自己做的想法。</p>
<h3>2. 收到任务，一定要给到预期完成时间</h3>
<p>这个也是汇报的潜规则了，别让领导老为你悬着一颗心。</p>
<h3>3. 及时汇报进展</h3>
<p>这个跟上一条一样都是为了让领导放心。</p>
<p>你多主动及时汇报，领导就会对你产生信赖，知道你不需要他的督促时刻在替他推进着这个项目，他就会专心去干别的了，下次依然会把重要的事情交给你。</p>
<p>这就是上下级打配合的体验。</p>
<p><b>职场新人往往不知道该如何把握汇报的节奏，我认为要分情况：</b></p>
<p>1）非常紧急关键的任务每天发日报</p>
<ul>
<li>项目是否有什么问题，进展是否顺利；</li>
<li>有什么需要协助；</li>
<li>今日进展；</li>
<li>预计结束时间，是否如期结束；</li>
<li>如果出现延期，或者延期的预兆要早早汇报。</li>
</ul>
<p>2）非紧急关键，重要进展汇报就可以</p>
<ul>
<li>重要进展是什么；</li>
<li>项目是否进行顺利，是否需要领导提供什么帮助；</li>
<li>预计什么时候完成；</li>
<li>如果出现延期，或者延期的预兆要早早汇报。</li>
</ul>
<p><strong>关于汇报怎么组织语言，这里也有模版供参考：</strong></p>
<p>1）汇报要有逻辑</p>
<ul>
<li>普遍原则——金字塔模型，职场上最适用的，没有之一；</li>
<li>紧急事项处理；</li>
<li>现状，下一步采取的措施，什么时候处理完成。</li>
</ul>
<p>2）汇报进度</p>
<ul>
<li>目前进度；</li>
<li>后续还需要做什么；</li>
<li>后续还需要多少时间。</li>
</ul>
<p><strong>另外，汇报也有一些注意事项：</strong></p>
<ul>
<li>重大的事情，一定要向领导汇报，不要私自做决定；</li>
</ul>
<ul>
<li>跟领导汇报时，要逐级上报，不要越级。</li>
</ul>
<h3><b>4. 跟领导保持密切的联系</b></h3>
<p>这个关系到我们在职场的向上发展。</p>
<p>很多同学是“好学生思维”，觉得自己做好工作（就像做学生时考好试）就行了，其他的不用管。</p>
<p>但职场上大家是战友关系，尤其是你的领导，不夸张地说，你在职场上的一举一动都跟他紧密相关，所以一定要跟领导保持密切的联系。</p>
<p>我自己也是一个社恐，尤其害怕跟权威角色相处。</p>
<p>但领导一般年纪都比我们大一点，跟他们相处，可以给到我们一些人生建议，这对于我们来说也是很宝贵的事情。</p>
<p>这里是一些小抓手：</p>
<ul>
<li>可以季度或者年度找领导主动聊一下自己的成长；</li>
<li>主动做帮助团队提效、服务的工作——当然是得到领导的许可之后；</li>
<li>多发现领导身上的优点，和你可以学习的地方，这样可以自然而然地跟领导是和谐的关系，很多人都喜欢在领导身上找缺点，但这对自己一点好处都没有；</li>
<li>领导的兴趣爱好确实是一个好的话题方向；</li>
<li>多一些私下的相处，偶尔一起吃个饭聊下工作外的。</li>
</ul>
<h2 id="toc-4">四、个人影响力</h2>
<p>所谓个人影响力，说到底就是有多少人认可你。</p>
<p>当有足够的人认可你的时候，你会发现到任何地方都会如履坦途。</p>
<p>如果你只是埋头干活，即使你很厉害，但没有多少人知道你厉害，自然晋升什么的也难以轮得到你。</p>
<p>个人影响力有两个方向，人数越多越好，职级越高越好。</p>
<p>但传播影响力有个小前提，就是你的基本面已经很好了，否则就像体验不好的产品做营销，传播越多失败越快。</p>
<p>那传播个人影响力具体如何做呢：</p>
<h3>1. 契约精神</h3>
<p>做事靠谱，说到做到，在合作者中建立口碑。</p>
<p>这个可以说是基础，否则很容易被打上不靠谱的标签。</p>
<p>不仅个人品牌受影响，还会增加自己的日常工作难度，因平时我们的跨部门协作都很多（产品经理们要尤其注意）。</p>
<h3>2. 有个人特色</h3>
<p>除了工作靠谱以外可以为团队做贡献，例如会写PPT。</p>
<p>我们都知道工具产品，刚性最强。所以当你有一定的特长且在团队内有工具属性的时候，例如擅长做APP（或者是擅长策划团建也可以，虽然频次低一点），是比较容易在团队中担当比较重要角色的，也非常有利于在新团队打开局面。</p>
<p>所以大家可以想一下，有什么可以挖掘发展成特长的东西。</p>
<h3>3. 在团队中做制度化的那个人</h3>
<p>团队的沉淀很重要，但可能团队领导过于忙，没时间做，大家看到有用的方法可以帮忙沉淀，维护团队知识库对个人和团队都是很有价值的事情。</p>
<h3>4. 有团队外的分享活动积极参与</h3>
<p>这个就不用说了，分享是传达影响力的重要形式。</p>
<h3>5. 不要与人交恶</h3>
<p>这个就是要在职场中保持理性和稳定，否则被大家说成情绪不稳定的话，犹如定时炸弹，不容易被交付重要的任务。</p>
<h3>6. 坦诚清晰，错了就认</h3>
<p>看到一些产品经理，尤其评审的时候，当自己的方案被指出问题，仍不愿意承认，编各种理由为自己狡辩，会破坏大家的信任感。</p>
<h2 id="toc-5">五、总结</h2>
<p>大家职场成长不易，今天写下这些，也不是我从一开始就懂得这些，而是经过磕磕绊绊，走了很多弯路以后，才更知道直线应该怎么走。</p>
<p>所以分享出来，希望大家不要像我一样在一片草莽中摸索吧。</p>
<p>最后还有2点我认为很重要的心法分享给大家：</p>
<ol>
<li>第一：我们要树立正确的职场观，即<b>任何时候提升自己的能力都是第一位的</b>，这个能力不光局限在处理事情的能力，还有沟通能力、处理关系的能力等等。</li>
<li>第二：有一种美德是认识到人生的真相仍然保持热爱。职场是一个组织，人在组织团体中难免被影响，就像风吹过，稻穗容易弯腰一样。</li>
</ol>
<p>有些不好的环境会导致大家想走捷径，或者做出以后的自己会觉得遗憾的行为。</p>
<p>所以希望大家早早认识并决定自己要坚持什么。毕竟职场只是很小的一部分，我们还有广阔的人生。</p>
