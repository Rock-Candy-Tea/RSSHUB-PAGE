<p><img data-action="zoom" class="size-full wp-image-5114317 aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/08/eefn7kVn3qXYKG1MRygQ.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<p>假设甲方爸爸想了解当前产品客户体验的现状，你让下属做客户体验研究并输出一份报告，然后下属把调研结果呈报给你，你觉得这个报告写的一团糟，背景不明确，逻辑不清晰，数据不充分，结论没价值，你看了直摇头，这个时候，你会怎么想？</p>
<p>A：他肯定是能力不行，影响我的工作</p>
<p>B：这事是他做的，不关我的事</p>
<p>C：早知道我就把研究计划做好让他执行就好了</p>
<p>D：这事要不要重新做一遍再给客户汇报，我需要思考一下</p>
<p>E：这个报告是我们专业水平的体现，这样交付会影响公司名声</p>
<p>其实以上你的选择就是我们说的思维能力，思维能力的差异性，导致看待问题的角度就会不一样，这也是优秀的项目经理与萌新项目经理的差距。</p>
<h2 id="toc-1">一、常见项目经理思维类型</h2>
<h3>1. 抱怨型思维</h3>
<p>思考模式：所有问题都是你们的错。</p>
<ul>
<li>客户需求变更频繁时，抱怨客户不懂范围，没有按照合同来做事，甚至抱怨公司没有协调客户关系，导致客户处处为难项目。</li>
<li>项目成员冲突时，抱怨员工职业素养太低，抱怨行政人事招聘水平。</li>
<li>项目质量出现问题时，抱怨客户要求太高，抱怨员工工作不认真，不听指挥。</li>
</ul>
<p>总之，当项目中出现问题时，首要思考的不是如何解决问题，而是把出现问题归因为环境的不好，早早的撇开自己的责任，同时抱怨这些问题导致自己的工作受到影响，没有任何动作去积极解决问题。</p>
<p>这类项目经理也会寻找解决办法，顺着“改变身边环境”的的路径去思考，比如打同事小报告、频繁跳槽等。他们对事情的理解被死死的困在“外在环境”这个最底层，从而给自己给公司带来诸多的麻烦。</p>
<h3>2. 员工型思维</h3>
<p>思考模式：我要做好项目工作。</p>
<p>这是典型的缺乏管理意识，项目经理是沟通协调会占据大部分工作时间的工种，而不是把时间放在具体的工作上。</p>
<p>一个员工型项目经理的典型画像就是项目上的“救火队长”。</p>
<p>计划阶段：项目经理一手制定项目计划，不经过项目成员的讨论与预估，统一按照自己的想法制定包括范围、进度、质量、成本等计划。</p>
<p>实施阶段：</p>
<ul>
<li>项目出现质量问题，项目经理自己亲自整改；</li>
<li>项目出现难点，项目经理出具解决方案；</li>
<li>人员请假，项目经理在岗顶上；</li>
<li>只要项目上有事，项目经理就亲力亲为的去解决。</li>
<li>最终形成项目的一套独特的工作模式，项目经理就是那个最忙的“救火”人。</li>
</ul>
<p>这类项目经理在做项目的时候会感觉到十分的疲惫，因为既辛苦但项目又很难取得成效。若以上是一个员工的表现，则无疑这个员工是非常优秀的，但如果是项目经理，则可能连合格都称不上了。</p>
<h3>3. 自勉型思维</h3>
<p>思考模式：我还不够努力。</p>
<p>在职场上有句话：“选择大于努力”，在项目管理上其实也同样适用。</p>
<p>处在这个思维的项目经理是大家眼中的“实干家”、“行动派”，他们认为所有的问题都是自己做的不够多造成的，他们相信，只要持续努力，就一定能够取得转机。</p>
<p>但是努力一定能够解决问题吗？</p>
<ul>
<li>客户对项目质量提高要求时，项目经理强抓狠抓项目质量，以让客户满意为中心，无限满足客户高质量服务的要求，导致项目成本失去控制。</li>
<li>客户需求扩张时，项目经理努力完成客户要求的工作，安排项目加班、赶工，团队干得热火朝天还是完不成客户要求的工作量，公司无法协调更多资源来帮助项目经理时，导致客户不满意。</li>
<li>项目启动后，项目经理带领团队成员努力工作，赶进度想要提前交付，导致项目质量问题频发，最终得不偿失。</li>
</ul>
<p>很多时候，项目追求的是一个“平衡”。追求成本与质量的平衡、需求与资源的平衡、质量与进度的平衡、公司利润与甲方需求的平衡等等。</p>
<p>所以，努力确实是项目成功的一个必要条件，但远远不是一个充分条件，项目经理就像一个“平衡”的艺术家，一旦失去平衡，那么也就会失去对项目的掌控力。</p>
<h3>4. 学习型思维</h3>
<p>思考模式：方法总比问题多。</p>
<p>有这个思维能力的项目经理一般有着强大的学习能力和应用能力，能把遇到的各种问题通过分析与学习，将问题转化为可解决的方法，进而改善效率。</p>
<p>学习型思维的本质就是有“撅根到底”的底层逻辑。因为处在这个思维层次的项目经理，即使有一身的技能，也会在处理各种问题中陷入迷茫和焦虑。如何分析问题，如何搭建解决问题的思维框架，是这个层次的项目经理必备的思维能力。</p>
<p><strong>（1）项目技术要求超出自己的专业能力范围怎么办？</strong></p>
<p>研究项目背景与客户需求，主动分析项目具体的交付成果是什么，拆解达成目标需要涉及到什么技术工具，预判市场上是否有符合要求的技术人员，并在成本范围内聘用专业的人员来解决这个问题，沟通协调公司的资源看能否来解决这个问题。</p>
<p><strong>（2）项目成员不稳定，员工起冲突怎么办？</strong></p>
<p>分析员工为什么起冲突，当前冲突对项目是否有影响，影响层面有多大，有什么方案来把影响降到最低，能够处理好本次冲突，如何有效避免类似的问题再次发生。</p>
<p>这类项目经理明白，项目不是在个人的盲目摸索中推进，在项目管理中通过分析与学习更成熟的经验和方法，来帮助优化项目实施的工作效率，助力项目更好的交付。</p>
<h3>5. 选择型思维</h3>
<p>思考模式：什么才是最重要的。</p>
<p>在项目管理过程中，人、事、物、环境等等因素均会不断的给项目经理传递各种各样的干扰，在有限的精力条件下，如何找到什么可以做，什么不可以做，什么事情先做，什么事情后做，这就是做选择题的能力。</p>
<p>选择什么，首先基于信念，其次基于价值观，回归本质，就是做人做事的原则。这个原则指导更高效的做出选择，不用每次都思考、讨论，权衡利弊。保持独立的人格，拥有独立思考和判断是非的能力，坚持为人处事的原则，不断获取新的知识，站在公司的高度来思考问题。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/08/SwMDFcsK03NtxUqwsh1D.png" alt width="1061" height="115" referrerpolicy="no-referrer"></p>
<p>这类项目经理谦恭有内涵，温和却有力量，在人群中能够凝心聚力形成天然的磁场。他们从见到的人，做过的项目，解决过的问题中提升自己的思想高度、眼界和宽广的胸襟。</p>
<h2 id="toc-2">二、塑造选择型思维能力</h2>
<p>在项目经理在职场中能否晋升或工作愉快，最关键的还是在于：是否获得自我成长，而自我成长的前提是是否具备自我教育的能力。</p>
<p>自我成长的目的就是建立起强大的选择型思维能力，具备了这样的能力，就能够看清真相，即，看清工作中什么是最重要的环节，从而产生重要的，有效的工作成绩，否则一辈子都从事着重复的或不重要的工作内容，在职业上很难有所建树。</p>
<p>选择型思维是在多年工作中沉淀下来的隐形财富，需要时间的沉淀和经验的历练，以及不断的自我总结才能形成的一套自我工作以及为人处事的方法。</p>
<p>能够应对“<strong>公司看结果、项目管工作、发展摸方向”</strong>这众多复杂繁琐的事情，并能够在其中快速做出正确的选择。</p>
<p>那么，项目经理如何在工作中有意培养和练习选择型思维呢？</p>
<h3>1. 由底层到顶层</h3>
<p><strong>【永远要坚持以“顶层战略”为工作指导核心，看清什么结果是最重要的】</strong></p>
<p>想象一下，公司任命你为一个项目的项目经理，你该如何带领项目成功，给公司一个满意的结果？</p>
<ul>
<li>A项目：最终项目盈利20%，客户满意度低，项目经验没有提升。</li>
<li>B项目：最终项目亏损20%，客户满意度高，项目做了新的试点创新。</li>
</ul>
<p>如果以利润率为工作指导核心，A项目显然更成功；但如果以项目持续性发展的眼光来看，显然B项目更成功；</p>
<p>因此没有顶层战略做指向，我们在做选择与抉择的时候，就会在底层工作逻辑中徘徊迷茫。</p>
<p><strong>什么是底层工作逻辑？</strong></p>
<p>从项目的视角出发，为满足项目足额回款、高质量交付、客户满意等关键指标，在项目实施交付过程中的一系列管理计划与监控控制。</p>
<p>比如：项目视角的管理。</p>
<ul>
<li>客户频繁变更需求：我们的关注点是走变更流程，评估项目成本、质量与风险等等；</li>
<li>项目成本超支：我们的关注点是控制成本，调整工作质量计划等一系列措施；</li>
<li>项目任务包完成时间超时：我们的关注点是赶工或缩小范围，跟进整体的工作进度计划；</li>
</ul>
<p><strong>什么是顶层工作逻辑？</strong></p>
<p>从公司战略的视角出发，识别项目的核心目标，是利润、客情、经验、人才、试点还是其他战略目标，要深刻理解本项目对公司的核心价值是什么。</p>
<p>比如：公司核心目标是经验沉淀。</p>
<ul>
<li>客户频繁变更需求：我们的关注点是需求变更的背景，总结这个服务周期的核心价值变动，并积极配合客户做需求调整；</li>
<li>项目成本超支：我们的关注点是分析项目各部分工作板块成本支出情况，重新评估成本计划是否合理；</li>
<li>项目任务包完成时间超时：我们的关注点是重新评估任务包难度与进度计划，攻克任务包的技术难点积累经验。</li>
</ul>
<p>看清楚什么结果是最重要的，在项目管理过程中，不管遇到多少问题，心中都会有一把“天平”，让我们在选择或抉择的时候，都能面向正确的方向。</p>
<h3>2. 由现象到本质</h3>
<p><strong>【事物的本质存在于现象之中，看清什么工作是最重要的】</strong></p>
<p>“蓝色、帆船、海鸥”，你能联想到“大海”，这是一个信息匹配的能力。</p>
<p>由现象看到本质，“看”是人捕捉信息的方法，看到一个事物的信息，有些人会仔细分析，有些则走马观花般拂过脑海。我们往往认为，那些能从现象中看到本质的人，是观察得更仔细，更认真的一群人，他们能够从表像中看出事物的深层次结构。</p>
<p>实际上每个能从现象看到本质的人都是在脑中预留了相似的信息，然后捕捉到类似信息就会去进行匹配。就像有些项目经理在看到一个事件现象后，能快速识别某些风险。</p>
<p>这就是一个信息匹配的过程，如果没有通过学习、实践、复盘、优化并在脑海中预留相应的风险信息，在受到事件触发的时候，就不会有看穿“本质”的洞察力。</p>
<p>那么在项目经理经验并不丰富，没有形成自我信息匹配模型时，有其他方法通过现象看清本质吗？</p>
<p>有，那就是“抓住主要矛盾”<strong>。</strong></p>
<p><strong>所谓“透过现象看本质”就是指在看待问题时，能够抓住问题背后的根本性矛盾，理清构成矛盾的条件和发展方向，而不是被表象、无关因素、环境等影响工作判断。</strong></p>
<p>比如评价一个员工的工作绩效，却扯上他的性格、生活习惯等。评价一个员工的性格时，又论及他的工作能力。其实，很多时候我们并不是抓不住主要矛盾，而是“浮云遮望眼”，根本没有分清主次矛盾，把精力纠缠在次要矛盾上，甚至都谈不上时矛盾的表象上。</p>
<p>举个真实的案例：</p>
<p>背景：项目经理向公司项目管理部求助，客户要求我司建一个满意度监测的系统，来协助其部门工作，但研究合同发现并没有相应的技术要求，关键是我司不具备相应的开发条件，因此项目经理十分为难，导致客户非常不满意。</p>
<p><strong>现场项目经理遇到这个问题后的动作：</strong></p>
<ul>
<li>研究满意度监测系统的技术条件</li>
<li>协调技术部门解决系统所需技术</li>
<li>向公司求助，无法解决</li>
</ul>
<p><strong>公司项目管理部收到信息后的动作：</strong></p>
<ul>
<li>找出问题的主要矛盾：<strong>客户需求与服务能力不匹配</strong></li>
<li>分析客户需求分解：客户想要的并不是系统，而是解决满意度客户识别与预判的一套可行性方法，系统只是其中的一种方式</li>
<li>解决主要矛盾：建立数据分析模型，通过对关键字段的抓取与分析，实现不满意客户预判，省去系统开发环节</li>
</ul>
<p>所以有的事不能仅看表象，那结果往往会出现意识形态的偏差，要透过现象看本质，抓住主要矛盾并重新解决矛盾，这才是我们要选择做的最重要工作。</p>
<h3>3. 由被动到主动</h3>
<p><strong>【项目有周期但事情的发展不会停滞，看清什么方向是最重要的】</strong></p>
<p>项目有三大特性：</p>
<ul>
<li>临时性：有明确的开始时间和结束时间；</li>
<li>独特性：每一个项目的产品和服务都是唯一的、独特的；</li>
<li>渐进明细性：项目的过程和成果性目标是逐步完成的；</li>
</ul>
<p>很多时候，我们被“临时性”蒙蔽了眼睛。</p>
<p>项目虽然有明确的开始时间和结束时间，但项目的招标可能会持续数年，而且项目成果并不一定是临时的，也可能是阶段性发展的。</p>
<p>“<strong>被动</strong>”的项目管理，就是指项目经理只关注当前项目周期内的工作，对项目服务周期完成后的服务规划，缺乏“<strong>主动</strong>”思考和推动的意识。</p>
<p><strong>举个简单的例子，客户的需求是要将100吨水果运输到某市，后续每年运输量可能会按50%的量增加，运输过程中水果损坏率要尽量控制到最低；</strong></p>
<p>A公司的项目经理思考逻辑：就用“货车”这个技术工具，解决客户的问题，只要在服务周期内解决问题就行。</p>
<ul>
<li>第一年的服务方案：提供了2辆货车满足客户的需求；</li>
<li>第二年的服务方案：提供了3辆货车满足客户的需求，后续以此类推；</li>
</ul>
<p>B公司的项目经理思考逻辑：用更前沿的工具和技术解决当前客户运输量递增的需求，并帮助客户更好的实现运输过程中的损耗保障问题。</p>
<ul>
<li>第一年的服务方案：提供了2辆货车满足客户的需求，总结提炼了很多不足之处；</li>
<li>第二年的服务方案：根据第一年的经验，改进了运输路线，车辆运输条件，增加了保鲜技术和水果分类存储运输方案；</li>
</ul>
<p><strong>如果你是客户，你会选择哪一家公司来服务呢？</strong></p>
<p>其实在整个过程中，项目经理应该是总结服务经验，改善服务能力的“<strong>主动</strong>”推动者，而不是围绕项目“<strong>被动</strong>”服务的管理者。</p>
<p>恰恰，我们很多项目经理没有跳出项目圈定的范围，无法站在客户的角度去思考问题。就像我们一直在做的满意度研究相关的项目，团队为解决客户满意度管理的问题，不断的迭代研究方法。</p>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2021/08/FzzrpkM54qkxeh0RzFh3.png" alt width="789" height="461" referrerpolicy="no-referrer"></p>
<h2 id="toc-3">三、总结</h2>
<p>项目经理的工作，既要保障公司的结果要求，又要细化管理项目的各种工作，最后还得在项目持续性发展中指明方向，在面向种种环境影响因素下，如何正确高效的选择与抉择？</p>
<p>塑造选择型思维，无外乎也是帮助项目经理在工作中，认清“什么结果是最重要的、什么工作是最重要的，什么方向是最重要的”，在思维能力的加持下，让我们具备“尽管前路漫漫风险汹涌，我亦目标坚毅扬帆远航”的勇气。</p>
