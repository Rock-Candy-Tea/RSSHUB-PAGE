<p><img data-action="zoom" class="size-full wp-image-5566470 aligncenter" src="https://image.woshipm.com/wp-files/2022/08/NpYmcOdDPjECrMT6MBwa.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<p>随着互联网的发展，人类已经进入了一个“美第奇效应”的时代，新技术层出不穷。市场变化如此剧烈的根源就是技术和商业模式的快速发展，只有紧跟技术和商业模式的发展，才能掌握或预测市场，而很公司都缺乏使他们能够有效应对市场中这些变化的投资组合管理流程。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.woshipm.com/wp-files/2022/08/gGuU9Dg4eiZgmYZlPeGw.png" alt width="940" height="473" referrerpolicy="no-referrer"></p>
<p>即便是许多拥有充分投资组合流程的公司也缺乏做出快速改变方向的决策能力，这就会导致产品失去市场竞争地位，从而无法达到收入目标。</p>
<h2 id="toc-1">一、缓慢的决策=失败的投资组合管理</h2>
<p>假设有一家大型技术设备公司的一个部门提出了一种针对公寓居民的创新移动安全产品的概念，该产品包括一个可穿戴硬件（类似手环）和一个应用程序。</p>
<p>由于这是公司的一项新业务，他们首先编写了详细的战略计划，这些计划包括：市场投资策略、上市策略、工程、服务和支持计划等内容。然而，这家公司并未创建全面的路线图，以便展示产品的技术和平台演变。</p>
<p>此时，一家开发出优秀解决方案（关键技术）的小型初创公司随后出现在市场，因为大公司没有详细的路线图，他们不知道与新进入者竞争的技术是在当前还是下一代平台上。他们不知道什么时候会出现竞争交集，因此，他们无法加快开发进程来应对竞争威胁。</p>
<p>结果公司失去了三个月的反应时间，他们永久地将该技术产品让给了竞争对手，并且不得不专注于不同类型的差异化。</p>
<h2 id="toc-2">二、授权+路线图</h2>
<p>如果你的公司面临类似的挑战，可以通过以下两个方面来解决问题：</p>
<h3>1. 授权</h3>
<p>首先，当需要进行中途修正以应对竞争威胁时，授权项目经理参与决策，项目经理是应对这些竞争威胁的关键。</p>
<p>为什么项目经理应该享有更广泛的决策？这是因为他们接近日常运营，并且了解项目的定义、资源、风险和财务。</p>
<p>虽然这样的中期决策通常是总经理的责任，但有必要获得跨职能团队的投入。</p>
<p>业务发展、企业发展、产品开发和营销通常都在谈判桌上进行，然而，项目经理经常被排除在外。他们需要一个平台，需要有权做出决定。</p>
<p>在以下情况下，项目经理在投资组合决策中才能发挥应有的作用：</p>
<ul>
<li>项目经理是决策小组的一部分；</li>
<li>高管理解并支持项目经理的决策角色；</li>
<li>项目群管理在决策制定中的作用被传达给更广泛的组织，当且仅当项目经理具有决策角色并且每个人都知道时，项目经理才有发挥作用的平台支撑。</li>
</ul>
<p>使你的项目经理能够参与并强烈影响决策，并将决策权写入工作描述和计划。确保所有利益相关者都了解项目经理具有这种权力和责任，并且高层管理人员对此表示认同。</p>
<h3>2. 路线图</h3>
<p>当出现意外的竞争威胁时，路线图也是一种有效的投资组合管理工具。</p>
<p>将市场、产品和技术类别放在纵轴上，将时间放在横轴上。此类图表为团队提供了在发生意外变化或风险时进行决策的可视化指南。</p>
<p><strong>那什么是路线图呢？</strong></p>
<p>路线图（Roadmap）是一张产品发展的规划，通常翻译为“路线图”或“蓝图”，目前并没有一个公认的定义。</p>
<p>Roadmap一般用于产品经理进行产品管理的中长期规划，也称路标规划。</p>
<p>Roadmap由时间周期（横轴）、项目事件（纵轴）和路标（里程碑）组成。</p>
<p>需要注意的是：</p>
<ul>
<li>同一项目与其对应的事件应使用同一种颜色表示；</li>
<li>同一项目下的相关事件，应在同一行表示；</li>
<li>同一项目下的事件排序，仅根据开始时间的先后顺序；</li>
<li>若同一项目下的事件开始时间相同，结束时间较近的优先排序。</li>
</ul>
<p>工具：PPT、Excel 、Visio等。</p>
<p><strong>为什么要做Roadmap？</strong></p>
<p>制定Roadmap需要依据公司战略、产品目标、用户需求和市场环境，其中公司战略和产品目标短时间内不会变化，可以理解为定量，而用户需求和市场环境可以说是瞬息万变的，也是不可控的因素。</p>
<p>因此，Roadmap一般按季度划分，平衡变与不变。</p>
<p><strong>Roadmap内容：</strong></p>
<p>Roadmap主要有时间周期和项目事件（或产品）和路标三部分组成。</p>
<p>① 时间周期</p>
<p>即产品规划的时间区间。</p>
<p>通常，时间周期的长度是产品大版本（如3.0.0→4.0.0）开发周期的3~5倍，如果大版本的开发周期是3个月，那么Roadmap时间周期长度就在9个月至15个月之间。</p>
<p>② 项目事件</p>
<p>是指完成产品总体计划必须要完成的工作项。</p>
<p>③ 路标</p>
<p>是指关键工作项的完成的时间接点，也称里程碑。</p>
<p><strong>1）产品路线图</strong></p>
<p><strong>① 工具简述</strong></p>
<ul>
<li>有时候，营销团队使用产品路线图，将客户的关键需求纳入产品研发计划，以便顺利签下订单；</li>
<li>有时候，产品管理团队使用产品路线图来向企业高层担保，产品处于符合企业愿景的开发计划中。</li>
</ul>
<p>在实际使用中，产品路线图的上述两种常见用法，最终的效果都不太理想，因为它们都背离了该工具的真正意义。</p>
<p>产品路线图旨在：</p>
<ul>
<li>用清楚直观的图表表现企业战略，紧密团结工程、市场、销售、技术支持等部门和企业高层，从而实现产品研发的目标；</li>
<li>同时，通过标明重要的差异化领域，赋予企业创新灵感；</li>
<li>再者，能够激发执行能力，促进产品平台和衍生战略之间的沟通，并表明衍生战略是如何随着时间推移而逐渐展开的。</li>
</ul>
<p>产品路线图工具是按照时间推移（通常是两个产品周期，或者为 24~36个月）对一组相关产品所作出的图示，多数情况下以每月为单位时间（图表上常标示每季度的数据）。</p>
<ul>
<li>纵轴是战略上最重要的部分，表示的是产品之间如何相互关联，以及如何有效影响企业竞争力；</li>
<li>最常用的纵轴是成本，另外也有将速度或其他关键特征参数作为纵轴的情况。</li>
</ul>
<p><strong>② 直观图示</strong></p>
<p>产品路线图体现出，产品的功能和价格点是如何随着时间的推移而发生变化的。纵轴通常是成本，而横轴是时间。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.woshipm.com/wp-files/2022/08/vRi1kO2Y1yprlTAxGmjH.png" alt width="828" height="548" referrerpolicy="no-referrer"></p>
<p><strong>③ 新颖之处</strong></p>
<p>为了获得成功，需要对共同的愿景详细阐述，但是，想要所有投资人和股东真正理解愿景，却是不太容易办到的。</p>
<p>打造与众不同的产品，能有效增强企业竞争优势，当产品创新的地位超过分销战略、金融运作的时候，产品路线图就是非常必要的规划文件。</p>
<p>一份功能强大的产品路线图必须是多功能的，是通过企业高层审查并批准的重要文档。仅供企业内部使用，将企业注意力集中在一段时间内要相继研发的产品序列上。</p>
<p>产品路线图是一份动态文档，是与项目有关的各种观点的集合。企业可以将其融入产品研发流程。它需要定期审查，至少每季度更新一次。</p>
<p><strong>④ 工具优势</strong></p>
<p>工具优势主要包括以下几个方面。</p>
<ul>
<li>更直观清晰地展示产品战略；</li>
<li>激励企业上下支持创新、推动重要参数；</li>
<li>鼓励企业生产平台衍生产品、获取经营优势；</li>
<li>作为预估企业新品上市时间的参考。</li>
</ul>
<p>是企业决定技术需求、资源配置和产品定位的重要参考。</p>
<p><strong>⑤ 适用情形</strong></p>
<p>经常有些经理会说他的企业没有战略，或是战略不为多数人熟知。而一份引用频繁的产品路线图可以帮助企业更好地进行战略调整，让更多的员工参与其中，同时便于管理层和营销团队对新产品研发流程进行成本管理。</p>
<p>产品路线图不应该用来排除竞争，但可以积极指明研发过程中的下一步骤，并至少可以在以下两个层面帮助企业实现创新：</p>
<p>一来，其纵轴表示出对企业至关重要的成本。</p>
<p>二来，它为工程师、研究人员和其他设计人员提供了战略背景，将大家的工作思路及观念导入支持、补充、延伸此规划蓝图所述战略意图的大背景中。</p>
<p>在产品研发全球化的大背景下，产品路线图还可以配合支持企业的国际市场产品发布战略，规划进入不同市场的时间，并对不同国家的监管、标准、语言和本地化等问题进行预期。</p>
<p>产品路线图必须能够预测市场、供货渠道、竞争环境中的种种变数，才能发挥自身的最大用处。在很多行业里，如果你计划在 12个月之后推出新产品，就需要提前预测你的竞争对手会有什么反应。</p>
<p>一份行之有效的产品路线图必须包括X、Y轴，标明各种细节、产品发布的准确时间，以及各产品之间的联系，同时还需说明新产品的特性。</p>
<p><strong>2）技术路线图</strong></p>
<p><strong>① 工具简述</strong></p>
<ul>
<li>产品路线图是从客户的角度出发，对产品的发展进行规划；</li>
<li>技术路线图却是从研发者的角度出发，对技术发展进行预测。</li>
</ul>
<p>技术路线图可以标明一项具有潜力的技术在何时能够发展成熟，并应用于上市产品中。</p>
<p>你可以创建一张横向条形统计图，引入几组要素来说明技术发展规划。</p>
<p>当然，在构建技术路线图的过程中所进行的讨论和思考，远比绘制蓝图本身更重要。</p>
<p>以笔记本电脑为例，需要引入的要素包括中央处理器、显卡、硬盘、电池管理及机械零部件。</p>
<p>你可以对技术的组成部分进行再分解并分组，用条形来代表每部分所需的时间，完成后便可以直观得出大致的交付时间。</p>
<p>想要合理清楚地绘制技术路线图，你需要决定：</p>
<ul>
<li>哪些关键技术是当前产品需要使用的；</li>
<li>哪些技术是若干年后需要使用的。</li>
</ul>
<p>虽然供应商和合作伙伴通常是非常好的关键技术的参照来源，但具体使用哪些技术，必须由你决定。</p>
<p>随着越来越多的竞争优势来源于企业外部，寻找新技术的重要性不言而喻，你很可能需要寻找可以共同完成技术研发工作的合作伙伴。</p>
<p>将企业外部技术和内部能力加以比较、综合，有助于厘清公司内部研发计划。</p>
<p>技术路线图的第二个组成部分是产品时间表，同样使用横向条形统计图，显示出每月或每季度的产品推出计划。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.woshipm.com/wp-files/2022/08/oiJJvSFF5y5OS3ywWyQh.png" alt width="828" height="351" referrerpolicy="no-referrer"></p>
<p>该直观展示某种消费类电子产品中的技术要素的影响，图表最右侧则标明了这些技术要素的重要程度，以及对其市场竞争力的预期。</p>
<p>这张图看起来有点像化学元素周期表，不同的”族”里都绑定有”技术”这一要素。在图中用垂直的线连接产品和技术，这样就可以推断某种特定产品的上市时间。</p>
<p>需要注意的是技术路线图需要定期更新（一般来说每季度更新一次），因为技术和产品会随着时间的推移发生巨大的变化。</p>
<p><strong>② 工具优势</strong></p>
<p>工具优势主要包括以下几个方面：</p>
<ul>
<li>重点标出需要解决的产品关键缺陷；</li>
<li>帮助确定生产某种商品所需的特定技术；</li>
<li>预见未来，预估所存在的风险；</li>
<li>指明可能存在漏洞的方面和有潜力的方面；</li>
<li>连接管理、工程和市场部门，以提高团队协作能力。</li>
</ul>
<p><strong>③ 适用情形</strong></p>
<p>技术路线图中集成的技术战略对企业而言极为重要，所有的行业，甚至包括服务业，都可以通过建立定义良好、能协助提高客户满意度的技术战略来提升竞争力。</p>
<p>技术路线图还可以帮助企业避免因某一新组件工期延误而导致的整个项目的延误，因为企业通过其建立的套监控系统，为各种可能遇见的状况提供了备选方案。</p>
<p>构建一份清楚完善的技术路线图是一项极其繁杂的工作，需要花费大量时间，克服诸多困难。</p>
<p>此外还需要准确把握企业核心竞争力和客户需求，并不是一项可以一次性完成的工作——技术路线图需要定期更新，并且接受管理层每个季度的审查评验。</p>
<p>技术路线图必须与企业管理联系起来，它的真正意义在于影响企业的整体战略。如果可能，最好将技术路线图纳入企业年度发展战略的考虑范围。</p>
<h2 id="toc-3">三、结论</h2>
<p>投资组合流程是敏捷性的杠杆，如果你的公司没有产品和技术的组合流程，则需要从开发流程开始。</p>
<p>然而，投资组合管理是必要的，但还不够。你的公司还需要一张清晰的地图，阐明哪些功能在哪些平台上推向市场，产品和技术路线图是跟踪程序并围绕它们做出决策的最佳工具之一。</p>
<p>当影响单个项目的市场或技术发生有意义的变化时，项目经理参与决策也很重要。他们拥有自己项目的全局视野。</p>
<p>不要把中途修正的问题单独留给总经理或营销主管。项目经理必须在制定竞争性回应方面发挥关键作用。</p>
