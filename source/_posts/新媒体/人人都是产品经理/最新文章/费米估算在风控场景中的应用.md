<blockquote><p>在建设风控，需要用一些工具来帮助业务决策，这里常用的是用费米估算在风控场景中的应用。本文以北京胡同口一天能卖出多少煎饼为例，分析如何应用费米估算，以期帮助大家提升自我数据思维。</p>
</blockquote><p><img data-action="zoom" class="size-full wp-image-5573747 aligncenter" src="https://image.woshipm.com/wp-files/2022/08/4TTzCEBYRijMny31AzgK.jpg" alt width="900" height="420" referrerpolicy="no-referrer"></p>
<p>通常新业务在进入快速发展期，业务同学更关注的是用户拉新、业务增长等直接业务指标。</p>
<p>对于场景中的刷单用户数、订单数等健康度指标关注相对较少。</p>
<p>回过头来看的时候，发现平台已经被薅取了很多的营销资金，便开始着手去建立风控治理能力。</p>
<p>在什么时候开始建设风控，需要一些工具来帮助业务进行决策。</p>
<h2 id="toc-1">一、费米估算</h2>
<blockquote><p>百度上解释：费米估算指的是解决未知结果的估算问题，将复杂的问题拆解成小的、可知结果的部分。</p></blockquote>
<p>直白一点，就是把复杂的大问题，拆分成一个个复杂的小问题。</p>
<p>同时将小问题的求解逻辑，转变为熟悉的业务因子和计算逻辑。</p>
<p>将已知的业务因子进行代入，即可计算出该问题的估算值。</p>
<p>说的很绕口，要入门的同学，可百度搜索：芝加哥有多少个钢琴调音师。</p>
<p>基本看一遍就知道咋用了。</p>
<h2 id="toc-2">二、北京胡同口一天能卖出多少煎饼</h2>
<p>了解基本用法后，用一个习题来进行巩固：北京胡同口一天能卖出多少个煎饼？</p>
<p>解题思路：</p>
<p><strong>从需求侧出发：</strong></p>
<blockquote><p>煎饼数量 = 购买人数 * 购买数量；</p>
<p>购买人数 = 胡同口覆盖人数 * 购买率</p>
<p>因胡同主要集中在北京中心区，胡同口覆盖人数 = 北京中心区可购买人数 * （胡同数量 / 道路数量）/ 胡同数量</p>
<p>可购买人数 = 北京中心区人数 * （1 – 婴幼儿比例）</p>
<p>按照21年高德统计数据，北京总人口为2188W，胡同数量1004条，道路数量14848条，北京中心区人口占比50%，婴幼儿占比20%；</p>
<p>通常一个人每周吃煎饼的次数，最多不会超过7次，最少不会超过1次，平均下来一周7天吃3次，购买率 = 3 / 7 = 43%</p>
<p>煎饼的季节性干扰很低，但时间段干扰较严重，通常集中在早饭，且一个为主。因此购买数量 = 1；</p></blockquote>
<p><strong>以上数据，可得出：</strong></p>
<blockquote><p>可购买人数 = 2188W * 50% *（1 – 20%）= 875W；</p>
<p>胡同口覆盖人数 = 875W * (1004 / 14848) / 1004 = 569；</p>
<p>购买人数 = 569 * 43% = 244个；</p>
<p>煎饼数量 = 244 * 1 = 244个；</p></blockquote>
<p><strong>从供给侧出发：</strong></p>
<blockquote><p>煎饼数量 = 摊主工作时长 / 每个煎饼耗时</p>
<p>煎饼的售卖集中在早上6:00 – 9:00，3个小时 = 10800秒；</p>
<p>每个煎饼的制作时间约为30秒；</p></blockquote>
<p>以上数据，可得出：煎饼数量 = 10800秒 / 30秒 = 360个；</p>
<p>煎饼生意明显是个供大于需的生意，因此主要是需求侧来驱动，整体量级会接近需求侧量级。</p>
<h3>三、业务场景中存在多少的刷单</h3>
<p>到这里，我们可以借用费米估算，来估算交易场景中存在多少的刷单订单，以此来辅助业务进行决策。</p>
<p>解题思路：</p>
<p><strong>从需求侧出发：</strong></p>
<ul>
<li>刷单订单数 = 刷单用户数 * 户均刷单数</li>
<li>刷单用户数 = 新客刷单数 + 老客刷单数</li>
<li>新客刷单数 = 新客注册数 * （1 – 新客正常流失率）</li>
<li>老客刷单数 = 老客活跃数 * （1 – 老客正常流失率）</li>
<li>户均刷单数 = （总订单数 – 正常订单数）/ 完单用户数</li>
<li>正常订单数 = 预估订单数 * （1 + 预估浮动比例）</li>
<li>预估订单数 = 预估完单用户数 * 户均订单数</li>
</ul>
<p><strong>从供给侧出发：</strong></p>
<ul>
<li>刷单订单数 = 总订单数 * 实际优惠补贴差值</li>
<li>实际优惠补贴差值 = 实际优惠率 – 预估优惠率</li>
</ul>
<p>这是一个需求侧驱动场景？还是供给侧驱动场景？</p>
<h3>四、总结</h3>
<p>这里，培养的是数据分析思路。</p>
<p>在实际业务场景中，经常会碰到很多问题：比如常见的“DAU下降问题”、“支付转化率下降问题”等。</p>
<p>其实每个答案都不是唯一，重要的是思考的逻辑和数据的推理。</p>
<p>即便有些事情看上去无法确定，但可以通过分析拆解，逐步逼近结果。面对不确定性，我们不应停滞不前，而是要懂得抓住身边有价值信息，快速决策。</p>
<p>最后，留个习题：中国有多少个出租车司机？</p>
