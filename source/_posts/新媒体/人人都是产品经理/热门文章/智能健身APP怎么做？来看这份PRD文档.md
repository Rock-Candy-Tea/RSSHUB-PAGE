<h2 id="toc-2">二、文档说明</h2>
<h3>产品说明</h3>
<p>单肌电APP是一款针对肌肉训练、采用肌电设备小模块利用电极片吸附在身体上、获取肌肉发力大小和肌肉激活度跟采集心率的产品。此版本为初代版本，文中涉及到的原型图均以iPhone11 pro为基础制作。</p>
<p><b></b><strong><b>1）产品定位</b></strong></p>
<p>针对专业健身人员和国际运动员，反映很难体会到肌肉发力的感觉或是不知道如何去判断动作是否正确标准。长久以来，我们一直将肌肉充血以及随之而来的紧迫感作为训练是否有效的标准，也没有更准确更客观的判断方法。而我们的产品是根据肌肉发力的激活度区间值、形成左右对比，从监测、评估到最后的调整一体化改善。</p>
<p><b></b><strong><b>2）用户人群</b></strong></p>
<p>健身达人、专业的健身职业人员，国家级运动员。</p>
<h2 id="toc-3">三、市场用户需求分析</h2>
<h3>1. 市场需求分析<b></b></h3>
<p><img data-action="zoom" class="aligncenter" src="https://image.woshipm.com/wp-files/2022/09/FaFyoI2jWvNYKOO89BUd.png" alt width="960" height="410" referrerpolicy="no-referrer"></p>
<p>根据艾瑞网2021年的报告，中国智能运动健身行业的发展，根植于我国体育健身行业的快速发展与智能技术的应用。随着行业的孕育与发展，智能技术与健身的融合由起初简单的模式创新演变至场地、硬件设备的不断更迭，由单一的健身房场景进化至健身房与居家健身场景结合的健身服务组合，整体呈现出技术运用面更广、服务健身场景更具针对性、智能健身硬件应用领域划分更细致的趋势。</p>
<p><img data-action="zoom" class="aligncenter" src="https://image.woshipm.com/wp-files/2022/09/ShcWfnUVKAZCukBbpoI5.png" alt width="943" height="479" referrerpolicy="no-referrer"></p>
<p>在每月最少进行一次或以上运动健身的人群中，使用过智能健身APP的人群占比为75%，其中男性占比60%，女性占比40%。</p>
<p>使用智能健身APP健身的人群每年在健身上的开销相比以传统方式健身的人群更高，每年健身开销在5,000元以上的占比总共达到30%。使用智能健身APP健身的人群中年轻群体占比较高，30岁以下占比达50%以上。年轻且愿意为运动健身进行消费的人群是消费品行业争取的目标人群，在智能健身领域该类型群体亦成为消费主力人群。</p>

<h2 id="toc-8">八、核心页面说明</h2>
<h3>1. 启动/引导/登录/注册</h3>
<p><strong><b>功能描述</b></strong>：登录。</p>
<p><strong><b>优先级</b></strong>：p1。</p>
<p><strong><b>输入/前置条件</b></strong>：打开智能肌电APP，跳转至启动页3s跳转至登录页面（首次下载安装跳转至引导页），阅读并确认《隐私政策》《服务条款》，输入手机号，获取验证码登录/第三方授权登录。</p>
<p><strong><b>页面流程：</b></strong></p>
<p><strong><b>① 手机号+验证码</b></strong></p>
<ul>
<li>初始页面下，输入手机号。</li>
<li>点击登录按钮，进入验证码登录页面。</li>
<li>验证码页面输入完成后，点击登录，验证正确完成登录，跳转至进入首页。</li>
</ul>
<p><strong><b>② 第三方授权登录</b></strong></p>
<ul>
<li>初始界面下，在登录按钮下方，呈现”第三方账号登录“按钮。</li>
<li>点击按钮，提供微信、QQ、新浪微博平台按钮，选择后点击，进入此社交平台授权登录界面。</li>
<li>首次授权依旧需要手机号+验证码条件完成登录，跳转至进入首页。</li>
<li>二次授权登录无需手机号+验证码，可直接登录，跳转至进入首页。</li>
</ul>
<p><strong><b>异常流程</b></strong></p>
<ul>
<li>点击输入框，底部弹出数字键盘，手机号输入格式判断，满足11位。</li>
<li>手机号不合法，系统弹出toast提示“手机号不合法”，2s后消失，用户重新输入。</li>
<li>输入正确手机号，点击（发送验证码），手机将收到短信包含6位验证码。（发送验证码）按钮60秒内不能重复发送，60秒倒计时结束后将倒计时变成重新发送验证码按钮，如果收到的短信验证码输入错误将提示“验证码错误，请重新输入”，验证码有效时间为20分钟。</li>
<li>首次下载安装登录之后跳转至信息页面，填写信息可跳过至首页。</li>
</ul>
<p><strong><b>输出/后置条件</b></strong>：登录成功，进入APP已登录状态下的《首页》页面。</p>

<h3>4. 分析报告</h3>
<p><strong><b>功能描述</b></strong>：给自己训练做一个体检报告，报告会指出安全项、风险项和危险项。</p>
<p><strong><b>优先级</b></strong>：p1。</p>
<p><strong><b>输入/前置条件</b></strong>：底部导航栏点击数据icon跳转至《分析报告》。</p>
<p><strong><b>页面流程</b></strong><strong><b>：</b></strong></p>
<p><strong><b>① 评分</b></strong></p>
<ul>
<li>0 -100分机制。</li>
<li>一共4项；每一项25分。</li>
<li>75-100分正常、50-74分风险、0-49分以下危险。</li>
</ul>
<p><strong><b>② 判定</b></strong></p>
<p>正常、风险项、危险项。</p>
<p><strong><b>③ 报告内容</b></strong></p>
<ul>
<li>饮水量、心率值、肌肉区间、卡路里的总平均值。</li>
<li>给出饮水量、心率值、肌肉区间、卡路里的正常范围值。</li>
</ul>
<p><strong><b>④ 日历</b></strong></p>
<p>可翻查24小时之前的分析报告。</p>
<p><strong><b>输出/后置条件</b></strong>：点击开始分析加载生成《报告》，点击报告跳转至《报告详情》。</p>

<h3>7. 训练开始</h3>
<p><strong><b>功能描述</b></strong>：根据制定的训练动作训练的肌肉，从而知道哪些动作对应哪些肌肉部位。在根据我们的设备得到一系列数据，能让自己从数据看到自己训练过程的变化。</p>

<p><strong><b>② 输入值</b></strong></p>
<ul>
<li>输入需要做的组数 （默认1组）</li>
<li>每一个组要做几次 （默认1次）</li>
<li>每一次需要负重多是KG (默认0KG)</li>
<li>每做完一组动作 间歇多少秒（默认30s）</li>
</ul>
<p><strong><b>③ 连接部位</b></strong></p>
<p>提示该动作的肌肉部位，高亮显示。</p>
<p><strong><b>④ 连接说明</b></strong></p>
<p>如何开启设备连接到肌肉部位。</p>
<p><strong><b>⑤ EMG设备</b></strong></p>
<ul>
<li>点击连接跳转至《设备》去开启设备，根据动作系统给出设备数量，需全部开启才可开始训练。</li>

</ul>
<p><strong><b>⑥ 倒计时</b></strong></p>
<p>倒计时2.1 开始，语音播报同时进行。</p>
<p><strong><b>⑦ 开始</b></strong></p>
<ul>
<li>记录该动作总组数和剩余组数。</li>
<li>显示该组动作的总进度条。</li>
<li>开始时间计时00.00。</li>
<li>暂停：点击暂停所有进行的视频、动作、数据都停止。</li>
<li>结束：长按3秒弹出信息框，是否保存数据，点击否的话 则不记录该条数据，是的话，则保存在数据库。</li>
<li>间歇：每做完一组动作 则弹出间歇倒计时30秒，可增加和跳过，没增加一次加30s，跳过则立即开始下一组动作。</li>
<li>语音播报：语音播报需要跟动图/视频同步进行，再次点击为静音；语音包内容：2.1开始，第一组某某动作，第一组动作完成，好了，我们休息一下，充满力量！我们开始第二组动作，第二组动作完成，好了，我们休息一下，最后一组动作啦！再坚持一下已完成全部动作，辛苦主人啦。</li>

<li>心率：根据时长和心率区间值得到一条心率曲线，曲线包括平均心率值、最大心率值和最低心率值；每个心率区间逗留的时长是多少（区间:热身区间（97-115次/分）、燃脂区间（116-134次/分）、有氧耐力区间（135-154次/分）、无氧耐力区间（155-174次/分）、极限区间（174-194次/分））。</li>
<li>卡路里：实时消耗的卡路里值。</li>
</ul>

<h2 id="toc-9">九、补充需求</h2>
<h3>1. 安全需求</h3>
<ul>
<li>所有用户个人信息，比如用户名、性别、年龄、身高、体重等内容，需要在接口层加密传输。</li>
<li>所有前端页面，需要使用HTTPS协议，考虑到加密性的问题。</li>
<li>APP安装包混淆代码，以防破解。</li>
</ul>
<h3>2. 性能需求</h3>
<ul>
<li>APP从启动到显示首页，时间不超过5秒。</li>
<li>页面加载时间不能超过3秒，3秒后无响应，需要给出提示：网络繁忙。</li>
<li>对页面元素做预加载处理。</li>
</ul>
<h3>3. 可用性需求</h3>
<ul>
<li>操作系统支持：所有安卓机，暂不支持IOS。</li>
<li>信息支持缓存机制，只要加载完成，网络中断后仍可显示信息。</li>
<li>提供意见反馈入口 （暂无此功能，考虑在V2.0版本更新迭代）。</li>
</ul>
