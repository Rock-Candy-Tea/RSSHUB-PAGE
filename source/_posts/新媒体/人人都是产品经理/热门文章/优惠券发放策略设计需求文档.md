<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2018/01/DSXInDwPGsVekRn7Ozy3.jpg" referrerpolicy="no-referrer"></p>
<h2 id="toc-1"><strong>一、目的</strong></h2>
<p><strong>背景：</strong>若我是某<strong>餐饮类app</strong>的产品经理，由于公司<strong>推广需求</strong>，现在需要设计一个针对新老用户发送<strong>优惠券的H5界面</strong>，用于<strong>微信</strong>里用户<strong>自发的</strong>分享传播。</p>
<p><strong>要求：</strong></p>
<ul>
<li>以H5为载体，主要用于微信聊天窗和朋友圈的分享「此处为两种分享模式」；</li>
<li>注意分享的流程以及流程里必备的判断；</li>
<li>注意后台逻辑和前端逻辑的配合，不要忽略后端；</li>
<li>重要的页面需要完整原型，其他页面需要简单的页面流程，需要有整体的串联。</li>
</ul>
<p><strong>工作：</strong></p>
<ul>
<li>明确需求背景；</li>
<li>需求分析：通过简单的产品功能调研，提供决策过程和依据；</li>
<li>需求目的：明确需求目的和功能列表；</li>
<li>梳理流程所处的模块关系：业务逻辑、业务流程、页面流程；</li>
<li>功能详细介绍：原型设计；</li>
<li>考核指标和计算方法。</li>
</ul>
<p><strong>学习目的：</strong>锻炼撰写需求文档的能力；锻炼从0到1设计某事件的能力。</p>
<h2 id="toc-2"><strong>二、需求背景</strong></h2>
<ol>
<li>各竞争餐饮平台发布了优惠券策略，且取得了较好成果，导致用户的优惠的需求变成产品选择的重要参考因素。</li>
<li>用户增长点达到瓶颈，需要拉进新鲜血液，选中微信好友或微信群为突破点实现社群传播，扩大推广渠道与力度。</li>
<li>为老客户发放福利，提高其认可度，忠诚度，促进再次消费。</li>
<li>用户与用户之间粘性不够，同样通过微信的社交能力带动我方产品的用户互动。</li>
</ol>
<h2 id="toc-3"><strong>三、需求分析与目的</strong></h2>
<h3><strong>竞品功能点调研</strong></h3>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2018/08/XGTX6Mpr3HRWZkvz08F5.png" alt width="767" height="566" referrerpolicy="no-referrer"></p>
<p><strong>获取方式：</strong></p>
<p><strong>相同点</strong></p>
<p>美团外卖和饿了么有4种共同的获取方式：</p>
<ul>
<li>一是进入App时随机发放商家代金优惠券；</li>
<li>二是进店领取商家代金优惠券；</li>
<li>三是下单支付后获得可分享的红包优惠券；</li>
<li>四是邀请新用户后获得的邀请红包。</li>
</ul>
<p><strong>不同点</strong></p>
<p>美团外卖特有的获取方式为偷红包与购买红包套餐，偷红包增加了用户之间的互动与趣味性，更能获取用户相关社交圈的数据；红包套餐以优惠增加用户粘性，用户首先会被其大幅度的优惠以及可用性、易用性（无门槛红包）所吸引购买，然后抱着自己买的红包，怎么也要用完，不然就亏大了的心理进行多次消费，以保障用户活力。</p>
<p>饿了么则推出了超级会员红包模式以留存，且其会员特权如专享红包、奖励金、专属客服、专属折扣、金币特权等可提高用户忠诚度。且多出了一种邀请老用户下单，自己获得红包的模式来促进转化。并且更有玩游戏领红包的趣味获取方式，可以看出饿了么的合作方式的多样性。</p>
<p><strong>红包效能：</strong></p>
<ul>
<li><strong>共同点：</strong>都有把优惠券区分为红包、商家代金券、互拆券等形式，都会在分享的优惠券上放上平台链接以及商家或合作伙伴链接，分享的优惠券都可多人领取且按顺序发放最大额度红包，并能多次分享。都有每单只能使用1个红包，每天最多使用2个红包，每个红包仅限20个人领取的限制。</li>
<li><strong>不同点：</strong>美团推出了同享券（指可与其他优惠活动一起使用的优惠券），而饿了么则推出了红包提现功能，加大了红包的吸引力，与会员功能相互结合体现了饿了么对于新钱包支付模式的尝试。</li>
</ul>
<p>核心在于H5是否能能否传播开来引导新老用户参与，以及优惠券是否能够打动新用户注册，老用户再次消费。</p>
<h3><strong>用户-场景-需求-产品需求</strong></h3>
<p><img data-action="zoom" class="aligncenter" src="https://cors.zfour.workers.dev/?http://image.woshipm.com/wp-files/2018/08/2qf3ZcbkbSVeiWRisrlw.png" alt width="776" height="591" referrerpolicy="no-referrer"></p>
<p><strong>产品需求小结：</strong></p>
<p>优惠券基础，优惠券发放基础，优惠券可多次发放、多人领取；按领取顺序中的某位发放最大奖励；付款自动生产优惠券；活动发放优惠券；规则诠释；平台链接；店铺链接；根据习惯智能推荐商家优惠；店铺专属优惠券；专属优惠券附带专属评论；每次付款生产独立优惠券；下单时满足条件自动选择优惠券；手动添加优惠券；广告投放；优惠券分类；有条件优惠券（时间）；有条件优惠券（金额）；有条件优惠券（及时性限制）。</p>

<h2 id="toc-6">六、考核指标和计算方式</h2>
<ol>
<li><strong>分享率：</strong>分享优惠券数量 / 支付成功数量 * 100%</li>
<li><strong>分享成功率：</strong>优惠券被打开数量 / 优惠券分享数量 * 100%</li>
<li><strong>领取成功率：</strong>优惠券被领取数量 / 优惠券分享数量 * 100%</li>
<li><strong>新用户注册率：</strong>通过优惠券H5手机验证成功自动注册的新用户 / 分享优惠券数量 * 100%</li>
<li><strong>回流率：</strong>通过优惠券H5链接进入App数量 / 分享优惠券数量 * 100%</li>
<li><strong>下载率：</strong>通过优惠券H5链接进入转跳页面下载App数量 / 分享优惠券数量 * 100%</li>
<li><strong>优惠券使用率：</strong>优惠券被领取数量 / 支付时使用优惠券数量 * 100%</li>
<li><strong>激活率：</strong>发放优惠券功能前用户活跃度 / 发放优惠券功能后用户活跃度 * 100%</li>
<li><strong>转化率：</strong>发放优惠券功能前用户订单成交数量 / 发放优惠券功能后订单成交数量* 100%</li>
</ol>
