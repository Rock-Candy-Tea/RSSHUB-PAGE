lxc，但相比都还不够成熟。</p><p><img class="lazyload aligncenter size-medium" src="https://img.iplaysoft.com/wp-content/uploads/2022/p/freebsd/freebsd_13.jpg" alt="FreeBSD 13" width="680" height="382" 服务器、<a href="https://www.iplaysoft.com/tag/dns" target="_blank" rel="noopener">DNS</a> 服务器 和路由器等，因此应用场景也非常丰富。</p><p>加上 FreeBSD 开箱就带编译器调试器，企业级的文件系统、虚拟化平台，以及能全套能自己构建一个完整镜像的源码，可以基于它轻松进行各种二次开发与魔改，因此也基于 等游戏机系统。</p><p><img class="lazyload aligncenter size-medium" src="https://img.iplaysoft.com/wp-content/uploads/2022/p/freebsd/freebsd_13_console.png" alt="FreeBSD" width="680" height="382" referrerpolicy="no-referrer"></p><h3>FreeBSD 与 Linux 发行版的区别</h3><p>Linux 发行版（distribution）本质上是一个组装品。发行版的制作者将 Linux 内核与其他开源软件整合起来，例如 GRUB、GNU C Library、Bash、Coreutils1、net-tools2 等，再加上自己的包管理系统，最后分发给了普通用户。</p><p>FreeBSD 是一套完整的操作系统。FreeBSD 团队不但开发自家的内核，还开发 libc、POSIX shell、大部分系统命令、<a href="https://www.iplaysoft.com/tag/%E6%96%87%E6%A1%A3" target="_blank" rel="noopener">文档</a>手册等。所有这些都在一棵源码树下，有一套完整的构建脚本（Makefile），高级用户可以通过 svn 和 make 命令更新操作系统或者自制安装镜像。</p><p><img class="lazyload aligncenter size-medium" src="https://img.iplaysoft.com/wp-content/uploads/2022/p/freebsd/freebsd_12.png" alt="FreeBSD 系统" width="680" height="382" referrerpolicy="no-referrer"></p><h4>其他区别如下：</h4><ul><li>FreeBSD root 用户的 shell 是 tcsh ，普通用户的 shell 是 sh。最好别改 root 用户的 shell。</li><li>FreeBSD 的关机命令是 shutdown -p now。</li><li>FreeBSD 的 make/grep/sed/awk 与 GNU 那套不同。</li><li>FreeBSD 有个叫 ee 的文本编辑器，风格与 nano 类似。</li><li>FreeBSD 管理系统用户与组的命令是 pw。</li><li>FreeBSD 使用 gpart 命令管理分区。</li><li>FreeBSD 原生的防火土啬是 ipfw。</li><li>FreeBSD 原生的文件系统只有两种： UFS、ZFS。</li><li>FreeBSD 有自己的引导方式，不一定要用 GRUB。</li><li>FreeBSD 不是 Unix®，然而历史告诉我们 FreeBSD 就是 Unix。</li></ul><h4>开发者需要注意的：</h4><div><ul><li>不要在脚本中写 #!/bin/bash，要写 #!/usr/bin/env bash。perl、python 等同理。</li><li>FreeBSD 第三方库默认安装路径是 /usr/local/。</li><li>FreeBSD 没有 epoll，但是有 kqueue。</li><li>FreeBSD 没有 inotify，若要移植 Linux 相关代码可安装 libinotify。</li><li>FreeBSD 没有 sha256sum 命令，但是有 sha256。md5 等类似。</li><li>FreeBSD 特有的 jot 命令能够生成数列，某些场景下可以与 xargs 搭配使用。</li><li>FreeBSD 的音频系统是 OSS，API 比 ALSA 简单好用。</li><li>FreeBSD 的 filemon 内核模块可以监控子进程的所有文件操作。</li><li>FreeBSD 的 libutil 库也许会派上用场，相关文档：/usr/src/lib/libutil/*.3。</li><li>FreeBSD 安装 Linux 二进制兼容层后，能够直接运行 Linux 可执行程序。</li><li>FreeBSD 项目大部分源代码遵循 BSD License，对商业闭源应用十分友好。</li></ul></div><h3>FreeBSD 13.1 版本更新：</h3><ul><li>FreeBSD 13.1 稳定版新版本最大的亮点是针对 AMD64 的 UEFI 启动进行了改进，特别是可以让更多的系统现在表现正常，以摆脱传统的 BIOS 启动。</li><li>另外还增加了对 freebsd-update 的支持，以 allwo 创建启动环境的自动快照，试图使操作系统的更新万无一失。在 13.1 中还有各种各样的硬件驱动改进，特别是改进了英特尔 WiFi 驱动。</li><li>FreeBSD 13.1 的其他一些变化包括在 64 位架构上默认启用位置独立可执行文件 (PIE) 支持，新的"zfskeys"服务脚本用于自动解密 ZFS 数据集，Bhyve 管理程序的 NVMe 模拟，chroot 现在支持非特权操作，各种 POWER 和 RISC-V 改进，大 endian 支持改进，支持 HiFive Unmatched RISC-V 开发板，更新对 OpenZFS 文件系统的上游支持，以及这个 BSD 开源生态系统的许多其他变化。</li></ul><div id="post_download" class="entry-dl-area"><h3 class="entry-dl-area-head"><span
