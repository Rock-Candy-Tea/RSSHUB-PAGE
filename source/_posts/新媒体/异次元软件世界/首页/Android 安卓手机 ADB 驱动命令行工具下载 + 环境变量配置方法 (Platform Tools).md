rel="noopener">Vysor</a>等投屏控制软件，这个 <strong>adb 命令行工具</strong>「Android SDK Platform Tools」都是必不可少的。你可以根据自己的操作系统，下载安装它即可。</p><h3>配置 adb 的 Windows 环境变量教程</h3><p>如果你需要在命令行里使用 <code>adb</code>，那么还需要在 Windows 上设置系统的环境变量。这样才能让命令行在执行 <code>adb</code> 命令时，能成功找到 <code>adb.exe</code> 所在的正确路径。</p><h4>第一步：下载并解压 adb 文件</h4><p>请务必将 adb 解压的文件<span style="color: #ff6600;">放到一个不容易被删除或移动的目录</span>，在下面的例子中，我们将使用以下的路径：<code>C:\platform-tools</code></p><h4>第二步：添加环境变量</h4><p>右键点击「此电脑」›「属性」›「高级系统设置」›「高级」，如下图。点击「环境变量」然后「新建」，在弹出的对话框里，填入变量名：<code>adb</code>，变量值：<code>C:\platform-tools</code>，之后点确认即可。</p><p><img class="lazyload aligncenter size-medium" src="https://img.iplaysoft.com/wp-content/uploads/2019/p/adb/adb_add_env.png" alt="添加环境变量" width="680" height="412" referrerpolicy="no-referrer"></p><h4>第二步：添加 Path 变量</h4><p>在刚才的“系统变量”列表里找到一个名为 <code>Path</code> 的项目，然后点「编辑」› 「新建」，在新建的项目里输入 <code>%adb%</code> (前后无空格)，然后点确定保存。</p><p><img class="lazyload aligncenter size-medium" src="https://img.iplaysoft.com/wp-content/uploads/2019/p/adb/adb_edit_path.png" alt="adb 添加 path 环境变量" width="680" height="494" referrerpolicy="no-referrer"></p><h4>第三步：测试环境变量是否正确</h4><p>打开 Windows 终端或 cmd 命令行，输入 <code>adb version</code> 命令并回车，如果环境变量配置正确，那么就会显示出 adb 的版本号。</p><p><img class="lazyload aligncenter size-medium" src="https://img.iplaysoft.com/wp-content/uploads/2019/p/adb/adb_version.jpg" alt="adb version 检查环境变量" width="680" height="452" referrerpolicy="no-referrer"></p><p>出现 adb 的版本号就表示环境变量配置成功了。如果有错误，请反复检查环境变量是否填写正确。最后，<strong>adb 命令</strong>的具体用法有很多，大家可以参照网上的各种教程使用。</p><div id="post_download" class="entry-dl-area"><h3 class="entry-dl-area-head"><span class="h3_icon ipsicon
