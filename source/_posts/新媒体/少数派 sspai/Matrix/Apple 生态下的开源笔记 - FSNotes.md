<div class="articleWidth-content" data-v-e1f0100a><div class="content wangEditor-txt minHeight" data-v-e1f0100a><h2>介绍</h2><p>FSNotes是一款支持 macOS 和 iOS 的开源笔记软件。<br>官方列出了以下主要特点</p><blockquote><p>Markdown 优先，支持纯文本和 RTF 文件<br>轻量流畅，10k+ 文件仍然运行良好<br>iCloud Drive 或 Dropbox 同步（任何同步盘）<br>多文件夹存储<br>操作以键盘为中心，快捷键启发自 nvALT<br>自适应双窗格视图，可选垂直或水平排布<br>外部编辑器支持，无缝实时同步<br>支持超过 170 种编程语言的代码高亮<br>支持行内图像<br>支持标签<br>支持笔记跳转 [[双方括号]]<br>支持固定笔记<br>支持笔记快捷复制到剪贴板<br>黑暗模式<br>AES-256 加密<br>支持 Mermaid 与 MathJax<br>可选的 Git backtick</code> - 用 CMD+反单引号 (TAB上面那个) 禁用/启用预览模式。</p><p>替代方案: 您可以也可以通过菜单栏中的<strong>视图-切换预览</strong>来完成以上操作 <code>View - Toggle Preview</code></p><h4>全局</h4><ul><li><code>cmd + option + shift + L</code> - 打开主窗口并聚焦搜索栏</li><li><code>cmd + option + shift + n</code> - 保存剪贴板</li><li><code>enter</code> - 向右移动文件夹层级（向下）; 侧边栏 > 笔记列表 > 笔记</li><li><code>cmd + enter</code> - 向左移动文件夹层级（向上）; 笔记 > 笔记列表 > 侧边栏</li></ul><h4>搜索栏</h4><ul><li><code>down arrow</code> - 用下箭头将焦点移到笔记列表上</li><li><code>enter</code> - 回车新建笔记</li></ul><h4>主窗口</h4><ul><li><code>esc</code> - 将光标移到搜索栏并清屏（不删除编辑内容）</li><li><code>cmd + L</code> - 聚焦搜索栏</li><li><code>tab</code> - TAB转到下一区域</li><li><code>cmd + backtick</code> - CMD+反单引号 预览模式 (仅限 markdown)</li><li><code>up, down arrows</code> - 下箭头选择笔记</li><li><code>cmd + j</code> - 下一篇笔记</li><li><code>cmd + k</code> - 上一篇笔记</li></ul><h4>笔记列表</h4><ul><li><code>cmd + delete</code> - 删除笔记</li><li><code>cmd + r</code> - 重命名笔记</li><li><code>cmd + 8</code> - 固定笔记</li><li><code>cmd + 7</code> - 去除加密（解密之前加密的笔记）</li><li><code>cmd + n</code> - 新建笔记</li><li><code>cmd + shift + n</code> - 新建 RTF</li><li><code>ctrl + cmd + e</code> – 在外部编辑器中打开所选内容</li><li><code>ctrl + cmd + o</code> – 在Finder中显示所选内容</li><li><code>shift + cmd + m</code> – 在存储中移动所选笔记</li><li><code>shift + cmd + b</code> – 隐藏侧边栏</li><li><code>cmd + s</code> – 将修订版本保存在git中</li><li><code>cmd + d</code> – 创建笔记副本</li><li><code>cmd + alt + l</code> - 加密/解密笔记（AES 256）</li></ul><h4>编辑</h4><ul><li><code>fn + f5</code> - 自动完成笔记名称</li><li><code>cmd + ` </code>，或点击 <code>View - Toggle preview</code> - 用于启用或禁用 Markdown 预览</li></ul><h3>4. 侧边栏</h3><p>要切换项目和标签侧边栏按钮 <code>control + cmd + shift + b</code></p><p>您可以选择和过滤无限制的项目和标签。</p><h4>每个子文件夹都是一个项目</h4><p>在您的存储空间中创建无限制的文件夹。右键单击根文目录（Documents）并单击“新建文件夹” <code>⌥ option + ⇧ shift + n</code>.</p><p>每个项目都有自己的设置，右键单击项目 - "显示视图选项" <code>cmd + shift +,</code>.</p><figure class="image ss-img-wrapper"><img src="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/aEpF5U.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="View" data-original="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/aEpF5U.jpg" referrerpolicy="no-referrer"></figure><p>您可以配置排序、方向、可见性设置。</p><figure class="image ss-img-wrapper"><img src="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/8SI0z8.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="Preference" data-original="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/8SI0z8.jpg" referrerpolicy="no-referrer"></figure><h3>5. 标签和子标签</h3><p>FSNotes version 4 带来了一个令人惊叹的行内标签系统。只需在文字前面加上井号 (#)即可为笔记添加标签。像这样：#hello。或者，像这样为它们添加子标签：#hello/world。您能给您的笔记贴多厚的标签？像 #unlimited/sub/tags。标签会自动完成。输入井号 (#)和标签的第一个字符，然后：</p><figure class="image ss-img-wrapper"><img src="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/980h2m.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="Hello" data-original="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/980h2m.jpg" referrerpolicy="no-referrer"></figure><p>很好！还有别的。每个项目都有自己的标签命名空间。例如，选择“存档”时，仅显示“存档”的笔记中的标签。</p><p>在侧边栏选择的同时按住<code>cmd</code>可以多选多个项目和标签。<br> </p><figure class="image ss-img-wrapper"><img src="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/Dw6q1e.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="Tags" data-original="https://gitee.com/marcuspierce/image-hosting-service/raw/master/uPic/Dw6q1e.jpg" referrerpolicy="no-referrer"></figure><h3>6. 容器</h3><p>什么是容器？“容器”是存放您的文件的文件夹。容器保存一个笔记中使用的文本和其他资源。当然，您可以选择不使用容器。打开偏好设置-> 基本-> 容器并选择“无”。笔记将以纯文本、 Markdown 或 RTF 格式存储。</p><p>我建议对敏感数据使用TextBundle容器和加密TextBundle容器。请继续阅读</p><h4>TextBundle容器</h4><p>文件扩展名 – <code>.textbundle</code></p><p>TextBundle – 这种格式旨在在应用程序之间交换纯文本文件(如Markdown或RTF)时提供更无缝的用户体验。 http://textbundle.org</p><p>例如，一个 Markdown 文件可能包含对外部图像的引用。当从 Markdown 编辑器发送这样的文件到预览器时，用户必须明确指定每一个图像文件的位置。这就是 TextBundle 派上用场的地方。通过将 Markdown 文本和所有引用的图像打包在一个文件中，Text Bundle提供更方便的体验。</p><h4>加密TextBundle容器</h4><p>文件扩展名 – <code>.etp</code>.</p><p>加密文本包用于笔记加密。它是加密的文本包（压缩包）并用 RNCryptor 加密。RNCryptor 是一种跨平台的数据格式，有很多实现方案。在软件后台，我们有：</p><p>AES-256 加密<br>CBC 模式<br>PBKDF2 密码延伸<br>密码加盐<br>随机 IV<br>加密后哈希 HMAC<br>开放和跨平台</p><p>您可以使用 Python 或 Ruby、JS 等方式解密任何 FSNotes 笔记(完整列表可在此处找到)<br>解压缩并用普通的文本格式快活吧。</p><h3>9.
