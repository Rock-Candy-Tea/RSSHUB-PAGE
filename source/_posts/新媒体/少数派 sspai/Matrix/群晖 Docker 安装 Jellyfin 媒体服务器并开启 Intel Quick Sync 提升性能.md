不比 J4125 更旧，大概率一样管用。但太新的 Intel CPU 图形架构改了（Xe），我就没有验证了。</p><h2 style="margin-left:0px;"><strong>0. 为什么要用 Quick Sync， VAAPI 又不是不能用？</strong></h2><ul><li>实测 VAAPI 的性能可能连 Quick Sync 的一半都没有，至少在 J4125 上是这样的</li><li>VAAPI 对 HDR 色调映射的支持也不同样好搞定，即使搞定，这个性能也差很多</li><li>Quick Sync 是 Intel 的专有技术，所以开源软件包是不会预装的，只能自己动手丰衣足食</li><li>作为硬件编码三剑客（Quick Sync/NVENC/AMF）中最不拉垮的，钱是你花的，还不用用吗？</li></ul><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/18/ea72de5a1ba7da224c7dc65052b333aa.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/18/ea72de5a1ba7da224c7dc65052b333aa.png" referrerpolicy="no-referrer"></figure><p>（左边是 VAAPI，只有26fps；右边是 Quick Sync，有37fps）</p><h2 style="margin-left:0px;"><strong>1. 使用 LinuxServer.io 出品的 Jellyfin 镜像</strong></h2><p>在群晖的 docker 注册表里搜 <code>linuxserver/jellyfin</code> 即可，详细参数设置可参考<a href="https://docs.linuxserver.io/images/docker-jellyfin"><code>官方说明</code></a>。你只需要参考里面的参数说明，不需要使用命令行，你可以在群晖 Docker 图形界面中直接配置各项参数。</p><p>注意3点：</p><ol><li>如果你曾经用过 <code>jellyfin/jellyfin</code> 的镜像，二者的config目录结构是不通用的</li><li>记得使用 host 网络，不要用 bridge，否则 DLNA 可能不能正常工作</li><li>记得使用高权限运行 docker，否则你需要手动处理 <code>/dev/dri</code> 的设备映射和权限</li></ol><h2 style="margin-left:0px;"><strong>2. 进入容器的 bash 进行进一步操作</strong></h2><p>在 docker 的容器详情里，新增一个终端机。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/18/article/9f4d706e3b4e342d2c1da07a62ea9355?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="截屏2021-07-17 下午7.37.10.png" data-original="https://cdn.sspai.com/2021/07/18/article/9f4d706e3b4e342d2c1da07a62ea9355" referrerpolicy="no-referrer"></figure><p>LinuxServer.io 出品的这个镜像，是基于 Ubuntu 的，因此先运行 <code>apt update</code>。</p><h2 style="margin-left:0px;"><strong>3. 更新 ffmpeg-jellyfin 软件包</strong></h2><p>先查看现有软件包的版本</p><p><code>/usr/lib/jellyfin-ffmpeg/ffmpeg</code></p><p>如果版本号低于 4.3.2，就要升级了。但估计阅读本文的未来人，已经没有这个烦恼了。</p><p>升级方法：</p><ol><li>安装 wget 命令：<code>apt install wget -y</code></li><li>去 <a href="https://repo.jellyfin.org/releases/server/ubuntu/versions/jellyfin-ffmpeg/"><code>Jellyfin 官方软件仓库</code></a> 找到最新的软件包版本</li></ol><ul><li>如何确定 Ubuntu 的版本名？ <code>cat /etc/lsb-release</code></li><li>如何确定 CPU 的架构？ <code>uname -a</code></li></ul><ol><li><code>cd /home && wget https://.....</code> 刚刚找到的软件包地址</li><li><code>dpkg -i xxxx.deb</code> 刚刚下载的软件包文件名</li></ol><h2 style="margin-left:0px;"><strong>4. 安装 Intel 核显的解码驱动（non-free）</strong></h2><p>感谢 LinuxServer.io 已经做了准备工作，因此很简单：</p><p><code>apt install intel-media-va-driver-non-free vainfo -y</code></p><p>装好以后，可以通过 <code>vainfo</code> 检查，能看到 <code>iHD driver</code> 的版本号是 21.1.1 以上几成功了：</p><pre class="language-"><code>libva info: Trying to open /usr/lib/x86_64-linux-gnu/dri/iHD_drv_video.so
