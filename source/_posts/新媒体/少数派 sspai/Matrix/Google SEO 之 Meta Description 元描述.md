<div class="articleWidth-content" data-v-d534f82e><div class="update-wrap" data-v-d534f82e></div><div class="content wangEditor-txt minHeight" data-v-d534f82e><p>上一篇文章<a href="https://sspai.com/post/68905" target="_blank">《7000 字告诉初学者 2022 Google SEO 怎么玩》</a>从整体上说了一下影响谷歌搜索引擎的结果页面的排名影响因素。本文以及随后的几篇文章我们会展开这些影响因素聊聊，具体哪些点对排名结果更重要，以及我们在网站的搭建以及内容创作等方面的技巧和方法。而本文先介绍的是 Meta Description 这个因素，翻译过来可以称之为元描述。</p><h2>Meta Description 是什么</h2><p>Meta Description 其实也不是什么高深的东西，首先这玩意跟元宇宙没什么关系，虽然听着很“元”的样子。Meta Description 本来是网页前端在 <head> 里面的一个标签，由于这个标签可以影响搜索结果页的呈现内容，也有可能会影响到点击率才得到行业内的重视。而在谷歌搜索的结果页出现在标题下面的一段文字被官方称为 Snippet （摘要）才是我们的“考试成绩”，我们的网页内容和 Meta Description 只是来影响这个考试结果而已。所以我们才会想尽办法来撰写更好的 Meta Description 来提高点击率。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/fa59b01f8b289473f55a705fd85e59af?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/fa59b01f8b289473f55a705fd85e59af" referrerpolicy="no-referrer"></figure><h2>Meta Description 元描述与 Snippet 摘要</h2><p>就像谷歌在 Google Search Central 介绍的那样，Snippet 就是搜索结果的说明或者说是摘要。写过论文的朋友可以简单理解为论文的摘要，只是从篇幅上来说要短很多，因为要在相对有限的空间内展示多个链接元描述，所以简明扼要很重。</p><p>根据谷歌最近的博客更新来看，谷歌应该是更少的影响 Snippet 的结果，只有在网页没有写 Meta Description 或者偏差很远的情况下谷歌的算法才会介入修改，否则经过网站所有者编写过得 Meta Description 基本不会改动。这对我们来说是好事，更加可控的 Snippet 对我们影响我们的用户至关重要，至少能否通过简短的文字提高我们链接的点击率这件事我们会起主要作用。</p><p>另外值得一提的是，谷歌提到会根据搜索关键词的不同呈现不同的 Snippet，那也就是说谷歌可能会根据用户的关键词判断到底是想要平底锅的购买链接还是使用方式，从而调整网页在搜索结果页的摘要，听起来好像对网站拥有者是好事，不过仔细一想，其实谷歌只是站在用户的角度为用户考虑，更精确的搜索结果呈现能让用户更加信任谷歌，而对众多网站拥有者来说，谷歌还是一视同仁，只是呈现最适合的给搜索用户的。当然如果能满足谷歌提到的Review，Recipe，Business，or Event这样有标签属性的内容应该是个不错的加分项。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/147750a750101238584c57287d03d57b?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/147750a750101238584c57287d03d57b" referrerpolicy="no-referrer"></figure><h2>控制 Meta Description</h2><p>在<meta>这个标签里如果我们提供了对用户来说更准确的网站描述Google也会直接采纳。当然我们也可以通过在前端代码里添加<a href="https://developers.google.com/search/docs/advanced/robots/robots_meta_tag#nosnippet" target="_blank"><code>nosnippet</code> meta tag</a>来避免Google显示网站的snippet，但绝大多数情况下我们很少用到这个东西，而通过添加<a href="https://developers.google.com/search/docs/advanced/robots/robots_meta_tag#max-snippet" target="_blank"><code>max-snippet:<i>[number]</i></code></a>可以控制精确的snippet字数，对我们来说也是好事。最后我们还可以使用<a href="https://developers.google.com/search/docs/advanced/crawling/special-tags#data-nosnippet" target="_blank"><code>data-nosnippet</code></a>来告诉Google不要引用某一个段落的内容在snippet里，也增加我们进一步控制snippet的一个手段。只不过上述这些方法不懂前端代码的同学可能有些困难了，需要找人协助一下。对于WordPress和 shopify 的同学可能要通过一些插件才能实现这些功能。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/0ced81ebc1da54a47e0ecaac7b1ef7c2?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/0ced81ebc1da54a47e0ecaac7b1ef7c2" referrerpolicy="no-referrer"></figure><h2>官方指导建议</h2><p>在谷歌的官方指导中也有提示如何创建最佳元描述的做法。首先谷歌直接说明元描述是“元描述没有长度限制，但谷歌搜索结果中的摘要会在必要时被截断，通常是为了适应设备宽度。”其实本质上就是有限制，因为搜索结果页有固定的位置和长度，更为精简且有吸引力的元描述是很有必要的；第二个就是，我们要为每个单独的网页创建独特的元描述，也就说最好不要大量复制同一个元描述在很多网页，毕竟每个单独的网页都是有不同的内容。如果真的没有足够的时间和精力为每个网站写元描述，比如电商网站有几千个产品，工作量很大。那么为首页和热门网页写上元描述是非常必要的；第三个指导，可以在元描述中添加一些非重要的内容信息，比如文章的发布日期、作者和署名信息等，而商品的网页也可以把产品价格、生产日期或者制造商等类似这样的信息添加进来。对于文章来说日期关乎于时效性，作者关乎于文章的品质都是对用户很重要的信息，也在某种程度上决定用户会不会打开你的文章链接；商品更是如此，价格这样的重要信息直接出现在搜索结果页会很大程度上提高产品链接的点击率；第四点，谷歌提示在类似电商网站这样的情况下可以程序化生成元描述。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/369168d0d67efe88e7c995303bac9612?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/369168d0d67efe88e7c995303bac9612" referrerpolicy="no-referrer"></figure><p>最后一点是看着像废话一样的，使用高质量的元描述。虽然在实操当中这个提示没什么直接的用处，但其实不然，这句话用到实操中可以落实成在同样的时间内宁愿写几个高质量的元描，述也不会把这个时间用在大量复制出所有的网页低质量的元描述，相信我，肯定有不少人这么做的。因为相对于5条高质量的元描述的工作成就感，100条网站元描述会显得多一些，而且高质量不像数量那样更容易量化，作为考核或者KPI。</p><p>上面就是对谷歌<a href="https://developers.google.com/search/docs/advanced/appearance/snippet" target="_blank">官方文档</a>对于元描述的简单指导的解读和一些实操的方法，下面我会在几个SEO领域关注比较多的网站里摘出一些实用的元描述方法和步骤。</p><h2>SEJ 的建议</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/a7da3cb9f432e96880e065412e23500f?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/a7da3cb9f432e96880e065412e23500f" referrerpolicy="no-referrer"></figure><p>第一个SEJ,也就是search engine journal，一个专注搜索引擎优化和社交营销的内容平台</p><p>第一篇文章How To Create Awesome Meta Descriptions，发布于2021年12月23日，挺新的一篇文章</p><p>首先文章介绍了一下meta description以及SERP snippet的区别，我们上述有谈到，然后也强调了一下重要性，紧接着说了关于字符数的问题，他的建议是160-165字符，然后给出了7条具体的建议：</p><ol><li>研究你的竞争者 第一步，搜索你的关键词；第二步，研究你的竞争者是如何做的；第三步，追踪你的竞争者元描述，当然有一个插件可以给你提供帮助，链接在原文里有。</li><li>描绘出用户在你的网站的浏览路径并根据你的营销漏斗，来写出对应网页的元描述。</li><li>用你的品牌或者产品的独特性来为元描述提供参考</li><li>在元描述中包含的核心关键词</li><li>运用社交平台的热词的高流量（谷歌趋势、BuzzSumo、推特热搜）</li><li>描述中尝试满足用户的具体需求，而不是很泛的词语</li><li>定期更新网站一些旧的元描述</li></ol><p>最后作者也举例了一些好的元描述的实例有兴趣的可以打开链接<a href="https://www.searchenginejournal.com/on-page-seo/optimize-meta-description/?itm_source=site-search" target="_blank">查看原文</a></p><p>第二篇文章，10 Mistakes to Avoid When Writing Your Meta Descriptions 10个可以避免的元描述错误做法，文章时间是2020年10月24日，稍微有点久远了</p><ol><li>太长的元描述。这里给的建议是PC端的长度在160个字符左右，而移动端的长度在120字符左右</li><li>太短的元描述。（怀疑这个作者在凑字数）</li><li>元描述与文章的内容不一致，不用过多解释，解释用引人入胜的元描述吸引了用户的点击，网页的文章内容或者产品与元描述不符也会增加网页的跳失率，从而影响在搜索页的排名</li><li>没有加入关键词</li><li>复制元描述</li><li>关键词堆砌。谷歌的爬虫很智能大量堆砌关键词很容易被识破的，还是别干了</li><li>元描述写得没有吸引力。这个不仅是写个像谷歌爬虫这样的机器看的，还是直接给真实的用户看的，有吸引力会提高点击率。</li><li>没有加引导行动的词。Call to Action可以让用户更容易行动</li><li>混淆元描述和片段的含义</li><li>没写元描述</li></ol><p>以上10条就是作者认为大多数人常犯的错误，具体内容可以<a href="https://www.searchenginejournal.com/meta-description-mistakes/250897/?itm_source=site-search" target="_blank">查看原文</a></p><h2>Ahrefs 的建议</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/c189f0289c98b3fed7a2ef45a7b20e3c?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/c189f0289c98b3fed7a2ef45a7b20e3c" referrerpolicy="no-referrer"></figure><p>接下来我们说一下著名 SEO 厂商 Ahrefs 网站给的建议，这篇 2020 年 9 月 29 日的文章在谷歌的搜索结果中排名也比较靠前，说明文章内容符合 SEO 这个主题，再者就是关联度大访问比较高。</p><p>首先文章说了为何要重视元描述的原因，然后介绍了 5 个写好元描述的角度：</p><ol><li>把元描述当做是标题的一个扩展标签</li><li>匹配搜索的意图 用户是想找信息吗？用户是想买什么产品吗？用户是在找一个具体的网站吗？还是用户想找一个快问快答？</li><li>使用主动语态 Use active voice 翻译过来应该是祈使句的意思。可以以动词作为开头这样用户就会潜意识的成为主语，有行动感。</li><li>简明扼要 这里甚至说出了具体字符数所占的像素单位160字符920px，而移动端680px是120字符</li><li>部署元描述 在CMS管理平台如wix、shopify、WordPress都可以通过插件或者平台本身就可以单独撰写和管理。而独立网站可以在前端代码里编辑meta description</li></ol><p>紧接着介绍了他们出的一个工具可以帮助网站主批量查看网站的元描述，并获得他们的修改建议。最后也举例说明好的元描述是什么样的，<a href="https://ahrefs.com/blog/meta-description/" target="_blank">阅读原文</a>可以看到更详细的信息。</p><h2>Yoast 的建议</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/03/article/0310204a017031b5e2ef9300a6ec505f?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/03/article/0310204a017031b5e2ef9300a6ec505f" referrerpolicy="no-referrer"></figure><p>然后我们看一下另一个研究SEO的公司yoast，他们是这个领域的服务商有不少的行业积累，当然也有一些付费的工具和服务，这里只看看他们的文章给我们提供了什么样的方法</p><p>和其他文章一样先介绍元描述是什么东西，以及为何重视，紧接着列出7个值得学习的方面</p><ol><li>保持字符数在155字符左右 谷歌呈现出来的片段一般在120-156字符左右</li><li>使用主动语态，让他有可操作性</li><li>加入Call to Action 采取行动的词语</li><li>使用与网页内容有紧密关联的关键词</li><li>如果是产品页面可以提供具体的产品规格</li><li>确保和网页的内容紧密相关</li><li>让每个网页的元描述独特一些，不要重复。</li></ol><p>随后文章说明如果是WordPress的网站可以使用yoast的插件帮助网站主来检测网页的元描述撰写情况，并提供可以改进的方案，当然一些进阶的功能还是需要付费的。</p><p>看到这里基本上我们也可以总结出几个常见的影响元描述的因素和一些改进的思路，更多的文章不一一介绍了，进本大差不离。</p><h2>其他参考链接</h2><p>下面我也会列出几个在这个领域比较专业的网站的文章链接，有兴趣的可以继续阅读研究，基本都是英文原文，部分可能需要梯子。</p><ol><li><a
