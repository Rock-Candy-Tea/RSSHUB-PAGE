data-v-e1f0100a><h2>前言</h2><p>让我萌生叛逃 Windows 系统的契机有两个，一个是本学期的操作系统课程，另一个是特别充满活力的 Python 老师。</p><p>当时那个周末感冒了，睡觉睡不好，也没心思看书，就决定折腾一下电脑。</p><figure class="image ss-img-wrapper image_resized" style="width:358px;"><img src="https://cdn.sspai.com/2021/05/08/da916b846f37e3bccfa52edea734733b.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/da916b846f37e3bccfa52edea734733b.png" referrerpolicy="no-referrer"><figcaption>我还为这件事更新了一条朋友圈</figcaption></figure><p>不过，既然我写了这篇文章，显然我被 Linux 桌面版劝退了。我不是一个喜欢折腾的科技宅，对于各种电脑参数只是略知一二，像 WebDAV 等运维软件更是知之甚少。</p><p>而且，对于我来说，GUI 的美观不是 Nice-To-Have，而是 Must-Have，否则我大概率是会拒绝使用的。</p><p>下面，就开始我的叛逃之旅。</p><h2>从 Windows 叛逃到 Linux 桌面版</h2><p>最开始选择的是我在浏览 YouTube 时被安利的 <a href="https://elementary.io/" target="_blank">Elementary OS</a>。</p><p>Elementary OS 是基于 Ubuntu LTS 开发的，这个发行版的 GUI 制作相比于其它的而言更美观，这当然也是我首选它的原因。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/05/08/91091202cf28f812b26b371a988e2f6d.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/91091202cf28f812b26b371a988e2f6d.png" referrerpolicy="no-referrer"><figcaption>Elementary OS</figcaption></figure><p>官网提供了购买和下载方式，因为我没有办理国际信用卡，所以我自定义金额为 0 元，直接下载，并按照官网提示更换操作系统。</p><p>整个安装过程还是蛮顺利的，没有遇到驱动安装的阻挠，顺顺利利地安装了下来。</p><p>在触控板、快捷键、命令行方面，除了小小的不适应，稍微学习一下也都可以接受，系统自带软件方面，很 Geek，但功能确实是够用的。不过使用了一小会儿之后，它就开始不断地崩溃，一崩溃就会退回到系统的登录界面。</p><p>我查了日志文件，也搜过 Google，但因为这毕竟是一个小众的 Linux 发行版，文档、论坛及测试不算特别完善，最后我没有找到解决方案。我安装的时候选择的是双系统，不得已我只能退回 Windows。</p><p>不过我觉得不能就这样结束了。既然 Elementary OS 不够稳定，那么 <a href="https://ubuntu.com/" target="_blank">Ubuntu</a> 作为以桌面应用为主的 Linux 发行版，而且运营了这么久，应当没有这么容易崩溃，所以我选择了 Ubuntu 20.04.2 LTS。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/05/08/5706de20e2f1a6f5f87876b64a62b0ba.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/5706de20e2f1a6f5f87876b64a62b0ba.png" referrerpolicy="no-referrer"><figcaption>Ubuntu 20.04.2 LTS</figcaption></figure><p>相对于 Elementary OS 初始界面，Ubuntu 的 GUI 充满工程师设计感，因此我首先需要对其进行美化，包括 Dock 栏、字体渲染、桌面背景等。因为我特别喜欢 macOS 的工业设计，所以美化的主要朝向就是 macOS，但因为我当时没有截图，所以这里放一张从知乎上扒下来的别人的美化结果，大概跟我的很类似。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/05/08/8d2672793298297b8f477bc6d9378287.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/8d2672793298297b8f477bc6d9378287.jpg" referrerpolicy="no-referrer"><figcaption>Ubuntu 美化后可以达到的效果</figcaption></figure><p>我当时实现的效果其实并没有上图这么好，除了时间问题，还有我的笔记本电脑配置并不支持我做过多的美化工作。在 Gnome Tweaks Tool 中如果安装过多的插件，应用中心会崩溃。所以我只是给 Ubuntu 换了个 Icon 主题、字体以及把一些 GUI 的位置稍微挪了挪窝。</p><p>美化的过程还是非常顺利的，没有遇到什么障碍，于是我就开始尝试将它作为我的主力操作系统使用。</p><p>Linux 桌面版，一个非常劝退用户的事情就是软件生态。</p><p>最开始是中文输入法。Ubuntu 自带的 iBus 使用起来是没有问题的，但因为屏幕缩放和字体大小的问题，我始终没有办法将它调节得特别舒适。后来使用上了基于 fcitx 的搜狗输入法 Linux 版，整体效果依旧没有 Windows 10 的微软拼音好用，比如切换输入法的系统提示响应速度较慢，且不够明显。这整的我不是很舒服，但整体可以接受。</p><p>接着是浏览器。我最开始使用的 Ubuntu 自带的 Firefox，它对于 Linux 桌面版的优化确实是做的非常良心的，作为日常使用是完全足够的。不过因为开发需要，所以我需要一个 Google Chrome。然而 Google Chrome 的官方下载路径却折腾了我好久，因为网络环境的问题，下载变得十分不稳定，全局代理使我打不开网页，不开代理又无法开始下载。虽然最后搞定了，不过这个过程确实削弱了我不少心气儿。</p><p>然后是开发工具 Visual Studio Code。一开始我使用的是 Ubuntu 应用商店 Snap 版本的 VS Code，但后来发现它并不支持中文，于是我又在官网下载了对应的版本。因为 VS Code 的 Settings 是支持使用 GitHub 或 Microsoft 账号同步的，所以使用起来还算方便，各种快捷键用起来也比较顺手。</p><p>再往下是 Office 套件。我下载了 LibreOffice，因为我也不是特别会使用 Microsoft 的 Office，所以使用起来倒还算顺利。不过以上这所有的软件，都有一个无可调和、或者说在我没有耐心之前调和失败的问题：字体大小及字体渲染。</p><p>系统字体的方枘圆凿成为将我从 Ubuntu 劝退的最后一根稻草。尽管我设置了相同的缩放比例和字体大小，但仍然表现得不尽人意。如 Google Chrome 的地址栏使用无缩放的系统字体大小，而标签页、收藏夹和网页本身则受系统缩放和浏览器设置的字体大小的影响，使得整体观感显得参差不齐。无独有偶的还有 LibreOffice。</p><p>而在字体渲染上，尽管可与 macOS 并驾齐驱，然而我的 1080P 屏幕与苹果的 Retina 屏幕相比简直判若云泥，整体效果没有 Windows 的微软雅黑更优雅。</p><p>其实常用软件还有聊天工具。而本身并不支持 Linux 的微信和 Tim，需要用到 <a href="https://github.com/wszqkzqk/deepin-wine-ubuntu" target="_blank">Deepin-Wine-Ubuntu</a>。然而我并没有尝试，我选择直接逃回 Windows 的怀抱。</p><p>当然，上面我吐槽了一轮 Linux 桌面版，但它也不是完全不好用。我记得我在跟朋友聊天的时候谈到，「Linux 桌面版，除了命令行，其它都不好用」。</p><p>Linux 的命令行是我唯一觉得特别舒适的工具，Linux Shell 的命令相对于 PowerShell 更精简，比较符合「小即使美」的 Unix 哲学。不过更主要的是我没有学习过 PowerShell，而对 Linux Shell 更熟悉一些。</p><h2>从 Linux 桌面版叛逃回 Windows</h2><p>因为我安装 Ubuntu 的时候，选择的也是双系统，所以我又回到了 Windows。</p><p>但这显然不够好玩，我居然被劝退了。为了避免这次活动毫无收获，考虑到我所在的大学提供了正版 Windows 10 专业版，以及专业版提供组策略、Hyper-X、Sandbox 等更多的功能，我决定将家庭版升级为专业版。</p><p>Windows 10 Pro 的安装很顺利，因为我格式化了全盘，所以软件都需要重新安装和配置。</p><p>我尝试了 Microsoft 提供的一些新功能，如 Windows 下的包管理 <a href="https://docs.microsoft.com/zh-cn/windows/package-manager/winget/" target="_blank">WinGet</a>。尽管它做的像个安装包管理器，而且现在并不完善，但相较于之前需要在无数下载站中穿过层层广告和流氓软件后才能顺利安装，已经很不错了。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/05/08/b1354990422f2e80b9c4520bd8ad1709.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/b1354990422f2e80b9c4520bd8ad1709.png" referrerpolicy="no-referrer"><figcaption>WinGet</figcaption></figure><p>我还体验了 <a href="https://docs.microsoft.com/zh-cn/windows/powertoys/" target="_blank">PowerToys</a>，一款 Microsoft 官方开发的、能给你带来近乎 macOS 下 Spotlight 的软件，它可帮助用户调整和简化其 Windows 10 体验，从而提高工作效率。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/05/08/b122c03a040164918b2cf44b66d7bf26.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/b122c03a040164918b2cf44b66d7bf26.png" referrerpolicy="no-referrer"><figcaption>PowerToys</figcaption></figure><p>自从 Windows 引入了 <a href="https://docs.microsoft.com/zh-cn/windows/wsl/" target="_blank">Windows Subsystem for Linux</a>，让使用 Linux Shell 管理 Windows 下的文件变得简单，摘掉了对命令行不友好的帽子。加上对 Linux Shell 的好感，两件事直接促使我打算去认真学习 Linux Shell。</p><p>还有我最爱的编辑器 <a href="https://code.visualstudio.com/" target="_blank">Visual Studio Code</a>，强大的文件目录管理和智能提示、高性能、自定义配置主题、集成 Git、良好的插件生态、开源，再加上支持通过命令行直接唤醒，妈妈我再也不想用集成开发环境了 😂。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/05/08/d3f0378c2a1b01be1ceabe766a6a72c1.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/08/d3f0378c2a1b01be1ceabe766a6a72c1.png" referrerpolicy="no-referrer"><figcaption>Visual Studio Code</figcaption></figure><p>除了 Microsoft 为工程师打造的开发环境外，还有 Microsoft Store 里面的 <a href="https://www.microsoft.com/zh-cn/p/quicklook/9nv4bs3l1h4s" target="_blank">QuickLook</a>、<a href="https://zh.snipaste.com/" target="_blank">Snipaste</a> 等软件，能够极大地提高电脑使用效率，而不需要引入大量的流氓软件，对于非软件开发行业的用户友好度也是杠杠的。</p><p>当然，想要把 Windows 配置得无干扰、无弹窗、专注工作，譬如说杀毒软件的选择、浏览器配置、搜索引擎的设置、压缩软件，以及电脑的正确使用方式，对于普通用户而言，并不是易如反掌的。不过也不是全无可能，这是需要你稍加关注一下，动动手的。</p><h2>结语</h2><p>尽管 Windows 经常受到 Geek 们的吐槽，但它确实大幅度降低了普通用户使用电脑的门槛，你甚至无法在世界上找出不是基于 Windows 运行的国家。除了前文提到的，更广阔的包括一些大型游戏、股票、金融、网银、大型专业软件等我平时没有接触的，加上 Windows 近乎疯狂的兼容性，以及巨头们并没有任何意愿去更新几乎已经没有资金流水的旧版软件，也是仅在 Windows 上兼容运行和体验良好。</p><p>反复横跳之前，我几乎对 Windows 没有特别好感，只知道我像很多同龄人一样，在它的帮助下完成了电脑的启蒙。写完这篇文章之后，我对 Windows 的好感上升了好几个数量级。</p><p>同时我也看到了自己内心的想法：我并不喜欢折腾，对科技也并没有到痴迷的地步，只想找一个可靠稳定的系统用下去。这次折腾，只是生活缺少波澜，想找点刺激的事情做做。</p><p>不过，人一世物一世，什么都要试一试。亲身下河才知深浅，不体会一下以后怎么跟别人吹牛逼呢。</p><p>当然，这不会阻挡我想去尝试 macOS 的脚步。除了我所处的行业允许，即没有大型专业软件需要仅在 Windows 上运行外，更多的还有 Apple 的优秀生态环境。不过程序员大概率也不会只有一台电脑，我猜我以后也会自己组装一台 Windows 台式机拿来玩游戏。</p><p>希望等我体验完之后，不要再写一篇猛吹 Windows 的文章吧 🤣。</p></div><!----></div><div
