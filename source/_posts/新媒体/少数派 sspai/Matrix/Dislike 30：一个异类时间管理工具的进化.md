<div class="articleWidth-content" data-v-0b37afcb><div class="update-wrap" data-v-0b37afcb></div><div class="content wangEditor-txt minHeight" data-v-0b37afcb><p style="margin-left:0px;">Dislike 是一个日常生活事务管理工具。</p><p style="margin-left:0px;">之所以说这款 App 另类，一则 Dislike 专门设计来打理生活琐事，而非像多数同类产品一样着眼于效率提升；二则 Dislike 没有遵循任何既定的时间管理理论，完全依照普通人的日常使用「直觉」设计、开发。</p><p style="margin-left:0px;">这意味着使用 Dislike，你不需要为了记录、提醒一些事情去强迫自己学习任何理论，或是改变一种生活方式。</p><p style="margin-left:0px;">关于时间管理的完整思考，我在《<a href="https://sspai.com/post/61776">我们究竟需要怎样的时间管理工具</a>》这篇文章里有较详细的记录，感兴趣的朋友可以翻阅。</p><p style="margin-left:0px;">如果市面上鼎鼎大名的 GTD 类 App 你用了个遍都无法坚持，说明你很可能和我一样不是个「效率达人」，那么我非常推荐你试试 Dislike。</p><p style="margin-left:0px;">Dislike 基础功能介绍参见这篇文章：</p><span class="ss-linkCard" link-card-href="https://sspai.com/post/61390"> </span><p style="margin-left:0px;text-align:justify;">Dislike 3.0 做了 10 余项较大的功能新增及优化，以及一堆没仔细数过的细节提升。</p><p style="margin-left:0px;text-align:justify;">虽然做了很多工作，但 UI 层面仍然保持了精简、克制，这也是从 Dislike 着手设计 1.0 至今一直贯彻的理念。</p><p style="margin-left:0px;text-align:justify;">拣几个 3.0 的重点功能说说。</p><h2 style="margin-left:auto;text-align:center;"><strong>事件创建重构</strong></h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/18/8e5b2e9fe982766f63f9ef9001329fcb.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/18/8e5b2e9fe982766f63f9ef9001329fcb.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">Dislike 3.0 完全重新设计、开发了事件创建功能。</p><p style="margin-left:0px;text-align:justify;">这也是 3.0 版本最耗费精力的一块开发工作。</p><p style="margin-left:0px;text-align:justify;">2.x 版本的事件创建逻辑本身没问题，同类日程 App 大多采用的也是类似的创建逻辑（包括 iOS 系统日历）。优点是创建流程清晰易懂，缺点是步骤稍显繁琐。</p><p style="margin-left:0px;text-align:justify;">尤其对于复杂事件，常常需要多次往返于二级、甚至三级设置页面。</p><p style="margin-left:0px;text-align:justify;">相较同类 App，Dislike 事件的日期、重复、提醒等设置更为灵活，这进一步加剧了 Dislike 2.x 版本创建事件的交互复杂度。</p><p style="margin-left:0px;text-align:justify;">不破不立，<strong>3.0 版本推翻重构后的事件创建模块 ，所有设置项均可在 1 个页面内完成。</strong></p><p style="margin-left:0px;text-align:justify;">定制化的日期和时间控件，左右滑动可快速定位日期选择，双击返回今天。</p><p style="margin-left:0px;text-align:justify;">提醒时间设置，起始、结束日期设置，重复设置都做了相应优化调整，大幅提升了事件创建效率。</p><h2 style="margin-left:auto;text-align:center;"><strong>月日历视图</strong></h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/18/9992fb66f57233b7af83dcb0c384ea45.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/18/9992fb66f57233b7af83dcb0c384ea45.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">从 Dislike 1.0 版本上线开始，月日历视图就一直是 Dislike 用户反馈最多的功能之一。</p><p style="margin-left:0px;text-align:justify;">其实在 Dislike 的 Mac 和 Windows 端，很早就上线了月日历视图功能。移动端迟迟没做，主要是因为对同类 App 目前普遍沿用的 UI 布局不满意，一时半会又没有更好的方案。</p><p style="margin-left:0px;text-align:justify;">同类 App 普遍的月日历 UI 布局问题在于日历格子太窄，能承载的事件标题信息极少。手机屏幕宽度本身比较小，按常规日历视图布局，一周 7 天均分后每个日历格子的空间很有限，顶多能放下 4~5 个汉字。</p><p style="margin-left:0px;text-align:justify;">这导致的结果就是在 UI 舒适度和信息传递效率之间，很难找到一个让人满意的平衡。</p><p style="margin-left:0px;text-align:justify;">Dislike 3.0 的月日历视图通过精巧的 UI 布局解决了这个问题。</p><p style="margin-left:0px;text-align:justify;">手机屏幕宽度紧张，但高度是相对富裕的。通过两个相邻日历格子事件条目交错的方式，Dislike 日历事件可以占据两个格子的宽度，足以展示较长的事件标题信息，也预留出了 UI 布局的「呼吸」空间。</p><p style="margin-left:0px;text-align:justify;">设计和功能实现并不复杂，却在保持 UI 优雅的前提下，很好的提升了月日历视图的实用性。</p><h2 style="margin-left:auto;text-align:center;"><strong>打卡事件</strong></h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/18/fa84088216b62ec1507d98a46ae40205.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/18/fa84088216b62ec1507d98a46ae40205.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">Dislike 3.0 新增了打卡事件类型，主要解决的问题是高频重复事件对时间线的信息干扰。</p><p style="margin-left:0px;text-align:justify;">2.x 版本我们提供了事件「仅当天显示」的选项试图去解决这个问题，但或许是存在一定的理解门槛，实际效果不大明显。很多用户的时间线仍然被大量高频重复事件充斥。</p><p style="margin-left:0px;text-align:justify;">一般而言，高频重复的事件往往不那么重要或容易遗忘——毕竟是需要天天处理的事情。这些事件如果正常显示在个人时间线或日程上，导致的结果就是真正重要的事情被大量高频重复事件「淹没」。</p><p style="margin-left:0px;text-align:justify;">所以 Dislike 3.0 引入了打卡事件类型。</p><p style="margin-left:0px;text-align:justify;">在 3.0 版本创建高频发生的事件（如：每天早起锻炼身体），系统会自动标记为打卡事件。打卡事件默认仅在时间线「今天」节点下显示，以保持个人日程的简洁干净。</p><p style="margin-left:0px;text-align:justify;">UI 形式上与一般事件不同，打卡事件以卡片形式呈现。点击卡片左侧可快速完成打卡，左右滑动查看全部打卡任务。</p><p style="margin-left:0px;text-align:justify;">后续我们也会推出一些常用的打卡订阅项目，方便直接订阅。</p><h2 style="margin-left:auto;text-align:center;"><strong>生活盘点</strong></h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/01/18/216a185a1b691ac9ab816afdb66cd749.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/01/18/216a185a1b691ac9ab816afdb66cd749.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">有一说一，我个人不大喜欢那种流于肤浅的简单数据报表分析。</p><p style="margin-left:0px;text-align:justify;">部分同类 App 的典型数据分析会提供类似「今天完成了 9 条任务，同比提升 20%，环比提升 35%」这种东西，在我看来是毫无意义的。</p><p style="margin-left:0px;text-align:justify;">Dislike 生活盘点提供了 6 个维度的数据。</p><p style="margin-left:0px;text-align:justify;">分别是：活跃事件数量、近 30 天完成事件数量、事件记录表、拖延度趋势、打卡事件每周完成度、事件分类比例。</p><p style="margin-left:0px;text-align:justify;">前两个是基础数据。</p><p style="margin-left:0px;text-align:justify;">事件记录表程序员朋友肯定很眼熟，借鉴了 Github 的提交历史记录。</p><p style="margin-left:0px;text-align:justify;">拖延度趋势计算了近 30 天的事件处理平均耗时，绘制了个人拖延度趋势以及与所有用户平均拖延度的比较。</p><p style="margin-left:0px;text-align:justify;">打卡事件每周完成度统计了成功打卡百分比。</p><p style="margin-left:0px;text-align:justify;">这个指标结合拖延度趋势，方便用户对自己的拖延严重程度有一个直观认识。</p><p style="margin-left:0px;text-align:justify;">事件分类比例指标根据用户创建事件及订阅的事件标签，统计了近 90 天内各个分类的事件数量。一定程度上可以透过这个指标发现自己的日常生活是否顾此失彼。</p><p style="margin-left:0px;text-align:justify;">（生活盘点功能目前仅 Android 端支持，iOS 端后续上线）</p><h2 style="margin-left:auto;text-align:center;"><strong>还有这些</strong></h2><p style="margin-left:0px;text-align:justify;">除上述功能，Dislike 3.0 还新增了事件标签、历史事件记录、事件拖拽等功能，UI 上也做了全新升级，不一一赘述。</p><span class="ss-application" app-id="14920"> </span><span class="ss-application" app-id="15592"> </span><p style="margin-left:0px;text-align:justify;">最后，借 3.0 发布的契机再分享下 Dislike 的产品理念及后续计划。</p><p style="margin-left:0px;text-align:justify;">Dislike 从一开始定位就不是一个典型的效率向 App。</p><p style="margin-left:0px;text-align:justify;">同类产品关心效率的太多，关心生活的太少。比起提高效率，Dislike 更希望立足生活事务，通过减轻生活琐事的处理负担来增进生活幸福感。</p><p style="margin-left:0px;text-align:justify;">正如我们一直挂在官网的一句话：</p><blockquote><p style="margin-left:0px;">把不喜欢的事交给 Dislike，你清空思绪，去关心真正的生活。</p></blockquote><p><br> </p></div><div
