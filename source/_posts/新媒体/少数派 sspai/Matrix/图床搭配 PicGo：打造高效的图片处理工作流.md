<div class="articleWidth-content" data-v-e1f0100a><div class="content wangEditor-txt minHeight" data-v-e1f0100a><p style="margin-left:0px;"><strong>Matrix 首页推荐</strong></p><p style="margin-left:0px;"><a href="https://sspai.com/matrix">Matrix</a> 是少数派的写作社区，我们主张分享真实的产品体验，有实用价值的经验与思考。我们会不定期挑选 Matrix 最优质的文章，展示来自用户的最真实的体验和观点。</p><p style="margin-left:0px;">文章代表作者个人观点，少数派仅对标题和排版略作修改。</p><hr><p>我们在写文章时，通常需要单独存储图片资源，不仅占用磁盘空间，而且不好管理。自从全面使用 Markdown 进行个人文件记录后，深感其在这图片处理的无力。</p><p>但是，用了图床之后，一切都不一样了。</p><h2>写在前面</h2><p>Markdown 语法处理图片返回的是一个 URL 链接。</p><p>如果上传的是本地图片，则会显示文件的本地路径，既然是本地路径，分享文件时就显得极不方便了。它需要将文章中的图片一并发送，接受者还要配置本地环境与发送者图片路径一致才可看到，或者根据自己的目录，修改文章中图片路径。如果使用的是图床工具显示的是外链，可以「随时随地」查看，并不需一并发送图片，也无需自行配置本地环境。</p><p>或许上面说的过于抽象，我们设想这样一个场景：</p><p>你在 Typora 里写好一篇图文文章（为了显示差异，这篇图文采用本地图片和图床链接两种形式），然后发送这个 <code>.md </code> 文件给协作者进行编辑修改。</p><figure class="image ss-img-wrapper"><img src="https://mytuchuang-1303248785.cos.ap-beijing.myqcloud.com/picgo/20210203105504.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://mytuchuang-1303248785.cos.ap-beijing.myqcloud.com/picgo/20210203105504.png" referrerpolicy="no-referrer"></figure><p>必定不久你的协作者就会一脸疑惑，说 Ta 根本就看不到第一张图片，因为在 Ta 那里，是这样的：</p><figure class="image ss-img-wrapper image_resized" style="width:438px;"><img src="https://mytuchuang-1303248785.cos.ap-beijing.myqcloud.com/picgo/20210203114853.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://mytuchuang-1303248785.cos.ap-beijing.myqcloud.com/picgo/20210203114853.png" referrerpolicy="no-referrer"></figure><p>如果你又不小心删除了第一张图片，那么连你也无法在 Typora 看到它了。而你试着删除使用图床外链的第二张图片，你发现仍然能够使用，实际上的效果和上图一样。</p><p>想必好奇的你一定会惊喜「图床」是什么？为什么图床会这么方便？！</p><p>图床，顾名思义就是进行图片存储的服务器，同时允许对外连接网络，所有人可以访问。</p><h2>好用图床推荐</h2><p>图床服务分为免费和付费两种。如果你对图片资源管理和版权要求不是很高，或者仅是偶尔写写文章的话，免费的图床也是足够了。如果你想更有效地掌握自己的图床资源，还是建议采用付费云服务。</p><p>以下列举了几个常见的和相对好用的图床服务。</p><h3>免费图床</h3><p>免费的代价：众多免费图床服务注册条款里均有禁止商用说明，万一哪天你使用的服务挂掉了或者关闭了图片外链，那你所有的链接都无法访问了，对你造成的损失或许不小。比如，前几年的微博图床。</p><p><a href="https://sm.ms/">SMMS</a>：稳定且快捷，是不少人主力图床的选择，同时开放 API，虽然已经推出付费套餐，但免费版本已经足够能用，不过公众号无法识别 SMMS 图床链接。</p><p><a href="https://imgurl.org/">ImgURL</a>：对游客有限制，每日最多上传 10 张，单张图片不能超过 5M，偶尔用用还是挺好的。</p><p><a href="https://github.com/">GitHub</a>、<a href="https://gitee.com/">Gitee</a>：代码托管云服务网站，帮助开发者存储和管理其项目源代码，且能够追踪、记录并控制用户对其代码的修改。不同的是，GitHub 服务器在国外，Gitee 服务器在国内，访问速度更快。甚至你可以更简单粗暴地把它理解为一个巨型网盘，可以存储任何东西。</p><p>如果想要免费使用，极推荐使用这两款服务，毕竟跑路的可能性极小。</p><p>在 <a href="https://www.githubs.cn/post/what-is-github">这里</a> 和 <a href="https://gitee.com/help/articles/4105#article-header0">这里</a> 你可以更详细地了解它们。</p><h3>付费图床</h3><p><a href="https://gitee.com/help/articles/4105#article-header0">腾讯云</a>：下面是个人版定价表。</p><figure class="image ss-img-wrapper"><img src="https://gitee.com/primary_rainbow/images20201215/raw/master/img/20210202155733.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://gitee.com/primary_rainbow/images20201215/raw/master/img/20210202155733.png" referrerpolicy="no-referrer"></figure><p><a href="https://cn.aliyun.com/">阿里云</a>：目前国内最大的云计算服务商，<a href="https://cn.aliyun.com/price/detail/oss">这里</a> 是阿里云 OSS 的定价单。</p><p><a href="https://www.qiniu.com/">七牛云</a>：提供的免费额度还挺高，是不少博主的选择。这里查看 <a href="https://www.qiniu.com/prices/kodo">七牛云定价</a></p><p>虽然比较来看，七牛云价格可能稍便宜些，但作为图床的云存储来说，一年或许也就几十块钱，没太大差距。而且腾讯云对于微信公众号的运营来说会更流畅些。</p><p>更多图床的介绍可以移步：<a href="https://sspai.com/post/55032">无需注册、打开即用，这 8 个免费好用的图床工具值得一试</a></p><h2>图片上传器：PicGo</h2><blockquote><p><strong>PicGo: 一个用于快速上传图片并获取图片 URL 链接的工具</strong> ——来自官网的自述</p></blockquote><p>有了图床，我们可以将图片（或 GIF 等）返回一个 URL 链接，而不需要专门编辑、存放、管理众多杂乱无章的图片，直接复制，粘到 Markdown 编辑器里就能显示出来。嗯挺好，但不舒服。</p><p>一篇文章，尤其是教程文章，势必会用大量图片，一张张上传、复制、粘贴，不断切换应用和浏览器，不仅繁琐的流程让人烦，打断的思路更难以寻回。</p><p>此时就需要一款图片上传工具了，支持全平台的有 <a href="https://github.com/Molunerfinn/PicGo/">PicGo</a>、仅限 macOS 系统的有 <a href="https://github.com/gee1k/uPic">uPic</a>。</p><p>我仅使用过的 PicGo，这里就推荐他了（如果你对uPic对同样感兴趣，可以看看 <a href="https://sspai.com/post/64169">我派的这篇文章</a>）。</p><p>PicGo 不仅是图片上传工具，同时也提供简易的图床相册管理功能。PicGo 开源且免费，跨平台支持 Windows、macOS、Linux 系统，使用极为简单。</p><p>可以在 <a href="https://github.com/Molunerfinn/PicGo/releases">这里</a> 下载软件安装包，或者通过包管理器 <a href="https://scoop.sh/">Scoop</a>、<a href="https://chocolatey.org/install">Chocolatey</a>、<a href="https://brew.sh/">Homebrew</a> 一键安装。</p><p>Chocolatey：<code>choco install picgo</code></p><p>Scoop：<code>scoop install picgo</code></p><p>Homebrew：<code>brew install picgo --cask</code></p><p>下载之后，界面是这样：</p><figure class="image ss-img-wrapper"><img src="https://gitee.com/primary_rainbow/images20201215/raw/master/img/20210202170013.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://gitee.com/primary_rainbow/images20201215/raw/master/img/20210202170013.png" referrerpolicy="no-referrer"></figure><p>初始配置好存储方案，PicGo 支持 SMMS、腾讯云、阿里云、七牛云、GitHub、又拍云、Imgur 等，同时支持 Gitee、青云等第三方图床服务。使用时，拖放图片到主窗口或 mini 窗口（macOS 为顶部状态栏），PicGo 会自动上传至云服务器并返回链接到剪贴板，直接粘贴即可使用。</p><p>PicGo 的 UI 界面不算好看，但也不难看。虽设计普通简洁，却上手容易，小白也能理解。</p><p>选择图片上传，PicGo 会自动返回链接到剪贴板，只需要在相应编辑器里粘贴即可。</p><figure class="image ss-img-wrapper"><img src="https://mytuchuang-1303248785.cos.ap-beijing.myqcloud.com/picgo/20210202200201.gif" alt URL 一键上传。</p><h3>丰富的链接格式</h3><p>PicGo 支持 Markdown、HTML、URL、UBB 和 Custom 五种格式的链接，对于 Markdown 写作者而言，最常用的就是 Markdown 格式，当然也支持自定义 Custom 。</p><h3>多样的的上传方式</h3><p>PicGo 支持的上传方式也比较丰富。除了主窗口上传之外，Windows 和 Linux 用户支持 Mini 小窗上传，macOS 用户则还可选择顶部状态栏上传，使用更加便捷。</p><figure class="image ss-img-wrapper image_resized" style="width:541px;"><img 不仅是图片上传工具，同时也提供简易的图床相册管理功能。</p><p>打开「相册区」，可以看到当前图床中所有的图片集合。支持复制、修改 URL 和删除的操作，分别对应每张图片左下方的三个小按钮。</p><figure class="image ss-img-wrapper"><img 的图片管理摸索了好久。不过一年多折腾下来，也打磨出了一套很好用的免费方案 —— Gitee 作为主力图床，搭配 SMMS。</p><p>为什么选择这组「套餐」呢？</p><p>首先，Gitee 作为国内最大的代码托管平台，经过几年发展，已经成为 GitHub 的国内替代产品，专业性强，相比于其他免费服务，风险性很低。</p><p>之所以选择组合 SMMS 使用，不仅是因为其在 PicGo 中可直接开启服务，还有 Gitee 对于单张图片限制在 1M 以内，所以一些体积较大的 GIF 文件，可能不够用，而 SMM 的限额更高，为 5M。</p><h3>PicGo 与 Gitee</h3><p>既然是免费服务，首先需要建立自己的 Gitee 图床库，详细步骤如下。</p><ul><li>注册个人 Gitee 账号</li><li>新建图床仓库</li></ul><figure PicGo 没有默认提供 Gitee 选项，所以需要通过插件开启第三方服务。首先在插件市场打开搜索 <code>Gitee</code>，点击任意一个，即可完成安装和所有配置。</p><figure class="image ss-img-wrapper"><img
