<div class="articleWidth-content" data-v-e1f0100a><div class="content wangEditor-txt minHeight" data-v-e1f0100a><p style="margin-left:0px;text-align:justify;">无需安装设置，无需代码命令行。上传俩文件，传回 Word 来。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/30/article/1e7593225f0dc4cdf6e8ce748de8ba46?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2021/03/30/article/1e7593225f0dc4cdf6e8ce748de8ba46" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">自从接触到了 Markdown ，我就有一种执念——用它写论文。</p><p style="margin-left:0px;text-align:justify;">这事儿理论上没有问题，只是实践中很麻烦。</p><p style="margin-left:0px;text-align:justify;">交叉引用、图表功能这些就不说了，对于大论文来说，最麻烦的部分其实就是文献引用。</p><p style="margin-left:0px;text-align:justify;">Markdown 基本语法里面，不带文献引用功能。因为人家设计的初衷就是简洁，替代的目标是 HTML，本不是给你写学术论文用的。</p><p style="margin-left:0px;text-align:justify;">所以，我们就需要借助外力。</p><p style="margin-left:0px;text-align:justify;">外力是一款「黑魔法神器」，叫做 Pandoc 。</p><p style="margin-left:0px;text-align:justify;"> </p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/30/article/13783d369b49712a2d9c92b967757393?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="Image.jpeg" data-original="https://cdn.sspai.com/2021/03/30/article/13783d369b49712a2d9c92b967757393" referrerpolicy="no-referrer"><figcaption>（图片来源：ResearchGate）</figcaption></figure><p style="margin-left:0px;text-align:justify;"> </p><p style="margin-left:0px;text-align:justify;">这东西，可以把许许多多种文档格式，转换成其他许许多多种文档格式。</p><p style="margin-left:0px;text-align:justify;">例如我曾经给你介绍过的《<a href="http://mp.weixin.qq.com/s?__biz=MzIyODI1MzYyNA==&mid=2653540521&idx=1&sn=1dc993c4c80c9ef7bf750805efef6031&chksm=f389ba7ec4fe33684c089a57b343d20f3141b93bc344817b09ee9bfc79acef2cbe14c6328b71#rd">如何用 Markdown 做幻灯？</a>》，《<a href="https://sspai.com/post/47110">如何把 Markdown 文件批量转换为 pdf？</a>》，就是这种黑魔法的体现。</p><p style="margin-left:0px;text-align:justify;">当然，更为重要的，是《<a href="https://mp.weixin.qq.com/s/iyaJHLHK94rWWaTtulvQpA">如何用Markdown写论文？</a>》这篇。我在自己跑通了流程之后，把方法分享出来。独乐乐，不如众乐乐。美美与共，才是大好。</p><p style="margin-left:0px;text-align:justify;">只不过，有些小伙伴兴致勃勃打开教程，一看发现居然还得使用命令行，甚至有代码，立即表示「劝退」了。</p><p style="margin-left:0px;text-align:justify;">对此，我非常能够理解。这几年，我一直在进行「低代码」（low code）编程的普及工作，<a href="https://mp.weixin.qq.com/s/vouP_vuemnt_4E8X6pSfiw">写了一系列的 Python 教程</a>，给你介绍如何用几行代码，就做出让人惊艳的分析结果。然而，根据我掌握的数据，真正上手去尝试的人，并不多。</p><p style="margin-left:0px;text-align:justify;">写公众号就是有这好处，可以获得真实的反馈数据。</p><p style="margin-left:0px;text-align:justify;">大部分非技术专业的人，头脑中有一堵围墙，在提醒自己「代码？那是程序员的事儿，我哪里学得会？」然后继续望洋兴叹。</p><p style="margin-left:0px;text-align:justify;">怎么办呢？</p><p style="margin-left:0px;text-align:justify;">我的解决方案，是进一步降低你实践的难度，把「低代码」干脆变成「无代码」（no code），然后尽量用 Web 应用方式呈现。</p><p style="margin-left:0px;text-align:justify;">用 Web 应用，是为了平台通用性。</p><p style="margin-left:0px;text-align:justify;">想想看，你在使用 iPad ，根本装不上 pandoc，也装不上 Zettlr 这样的工具，却希望把手头这一篇论文的 Markdown 修改版导出为 Word ，发给导师或者合作者，怎么办？</p><p style="margin-left:0px;text-align:justify;">这时候，如果有 Web 应用，那么你就可以随便打开一个浏览器，选择好引文样式（例如 APA, GBT7714-2015，或者自己上传你需要的格式），上传引文数据（Better BibTex CSL JSON）以及 Markdown 正文，然后 Word 文件就生成了，点击下载到本地，就可以进行下一步的编辑或者共享，方便不？</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/30/article/ee4609b485d6e4193e59643bcbef603e?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2021/03/30/article/ee4609b485d6e4193e59643bcbef603e" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">它的背后，其实还是 Pandoc。只不过是云服务，而<strong>不需要</strong>你在本地进行安装设置。</p><p style="margin-left:0px;text-align:justify;">我一直想做这个应用，但是面临的最大问题，就是麻烦。</p><p style="margin-left:0px;text-align:justify;">如果我需要租用服务器、绑定域名、进行环境配置、安装软件包、处理数据隔离、操心网络安全……那我就没有动力做了。如果再免费提供这个服务，我就真成了「赔本赚吆喝」了。</p><p style="margin-left:0px;text-align:justify;">好在，新的工具给我赋能了。这些事儿我根本不用操心，只需要把功能做出来就行。开心啊！这真是个对懒惰开发者友好的时代。</p><p style="margin-left:0px;text-align:justify;">我做了一段视频，给你详细介绍这款免费 Web 应用的使用方法。请你先花 4 分钟，看一下视频，了解基本使用方法。</p><p style="margin-left:0px;text-align:justify;">应用的链接，我放在了视频下方的文字介绍区域里。这里就不重复了。</p><p style="margin-left:0px;text-align:justify;"><a href="https://www.bilibili.com/video/BV1wv41187dz/">视频链接在这里</a>。</p><iframe class="ss-videoIframe" src="//player.bilibili.com/player.html?bvid=BV1wv41187dz"> </iframe><p> </p><p style="margin-left:0px;text-align:justify;">希望这款应用，对你写作学术论文能有帮助。至少，可以让你在不同环境下，都能进行论文的修改和生成，降低拖延症的发病概率。</p><p style="margin-left:0px;text-align:justify;">也欢迎你把使用中的问题与建议留言反馈给我。</p><p style="margin-left:0px;text-align:justify;">祝论文写作顺利！</p><p style="margin-left:0px;text-align:justify;">如果你觉得本文有用，请点赞。</p><p style="margin-left:0px;text-align:justify;">如果本文可能对你的朋友有帮助，请转发给他们。</p><p
