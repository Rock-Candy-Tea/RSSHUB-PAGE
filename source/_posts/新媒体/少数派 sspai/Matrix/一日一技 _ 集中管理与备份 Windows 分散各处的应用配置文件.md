<div class="articleWidth-content" data-v-e1f0100a><div class="content wangEditor-txt minHeight" data-v-e1f0100a><p style="margin-left:0px;">在 Windows 中，用户在 GUI 上对应用设置的更改会以文本文件或二进制文件的形式被保存在特定的几个文件夹中。卸载应用后配置文件一般不会消失，用户也可以复制这些配置文件到其他环境中使用。这意味着，只要存留了应用配置文件，只需一次配置，就能在重装系统、更换电脑后让应用们恢复如初。实现这个目标需要解决三个问题：</p><ul><li>Windows 应用分散在多个文件夹中，怎么统一管理和备份？</li><li>使用什么工具备份这些多而小且随时会被占用的文件？</li><li>如何排除文件夹中不需要保存的文件？</li></ul><p style="margin-left:0px;">本文提供的思路如下：将所有需要备份的配置文件搬到同一个文件夹中，再用目录链接 & 硬链接的方式将他们链接到原位置。此配置文件夹使用 Git 做版本管理和备份工具，不需要上传的文件用 .gitignore 来精确控制。下面将详细解释此方法的相关知识和操作步骤。</p><h2>相关知识</h2><h3>配置文件位置</h3><p style="margin-left:0px;">首先，我们需要了解 Windows 的应用配置文件在哪些位置。Windows 是一个多用户的操作系统，从注册表，到文件目录都严格遵循系统 - 用户的两级排列方式，配置文件也不例外，它们被设计在存放以下两个文件夹中：</p><ul><li>系统盘根目录隐藏文件夹：ProgramData。存放以管理员权限安装的各类应用的默认设置，如 Microsoft OneDrive。</li><li>用户目录隐藏文件夹：AppData\Roaming。存放适用于此用户的应用配置。如 Office。</li></ul><p style="margin-left:0px;">此外：</p><ul><li>命令行工具，如 Git，Nodejs 的包，其配置文件一般在用户目录，可能被隐藏。</li></ul><p style="margin-left:0px;">绿色软件配置文件在自己的文件夹内，UWP 软件被严格隔离在沙盒内，用户正常无法访问其目录。也有特殊的应用，配置文件可以自定义位置，我们能在应用内设置中发现其路径。</p><h3>目录链接和硬链接</h3><p style="margin-left:0px;">将要使用的硬链接，我已经在此文简单说明了其原理和特性：<a href="https://sspai.com/post/66834">符号链接、硬链接及其在 Windows 上的应用举例</a>。其中提到，硬链接只能指向同一卷已经存在的文件。这意味着，我们必须把配置文件集中在<strong>系统盘内的一个目录</strong>。</p><p style="margin-left:0px;">之所以使用目录链接，而不是符号链接，是因为目录链接经微软调优过，具有更高的跳转效率以及同源目录更高的相似度，Windows 系统自身的各类目录跳转都使用目录链接，这样能避免一些可能出现的奇怪问题。</p><p style="margin-left:0px;">因此，在处理配置文件时，遇到文件就使用硬链接，遇到目录就使用目录链接，创建的工具建议使用我在文中提到的 Link Shell Extension。</p><h3>Git 与 .gitignore</h3><p style="margin-left:0px;">OneDrive 等云盘并非为了同步多而小的文件设计：像是坚果云有秘而不宣的同步频率限制，同步这类小文本有概率丢失文件。很多配置文件一旦打开相应的应用，就是被占用状态，云盘无法同步。另外，这些云盘也无法做到精确的选择性同步。</p><p style="margin-left:0px;">因此我们使用天生为文本文件而设计的 Git，再使用 .gitignore 忽略掉无需跟踪的二进制文件。.gitignore 创建在 Git 库的根目录时，对所有文件生效，创建在子文件夹时，仅对该文件夹生效。</p><h2>操作步骤</h2><p style="margin-left:0px;">接下来我以用户根目录下 Git 的配置文件 .gitconfig、Roaming 下 Everything 的设置文件夹以及配置目录在文档中的 ShareX 为例，展示具体操作步骤。</p><h3>移动配置文件（夹）</h3><p style="margin-left:0px;">在系统盘的用户目录的任意位置创建一个文件夹 Settings，作为集中管理这些配置文件的地方。用 <code>git init</code> 初始化。</p><p style="margin-left:0px;">然后，打开文件管理器设置中的「显示隐藏文件」。找到 Git、Everything 和 ShareX 的配置文件，在关闭这些应用的情况下，将文件或目录移动至 Settings 文件夹中。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/21/b568be0cf6c3c46afa7ac30cc74b7672.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/21/b568be0cf6c3c46afa7ac30cc74b7672.png" referrerpolicy="no-referrer"></figure><h3>创建链接</h3><p style="margin-left:0px;">因为 .gitconfig 是一个文件，我们使用 Link Shell Extension 的中键拖拽功能，为它在原来的位置创建一个硬链接。</p><p style="margin-left:0px;">同样的道理，给 ShareX 和 Everything 创建目录链接。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/21/7f249ed977fe1c53d68019b2a949b98b.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/21/7f249ed977fe1c53d68019b2a949b98b.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;">此时，可以打开这些应用检验一下功能是不是正常，配置文件应该会继续生效。</p><h3>忽略非关键文件</h3><p style="margin-left:0px;">在这三个例子中，Git 和 Everything 配置文件夹中没有什么好忽略的，我们可以观察 ShareX 的文件夹结构以及文件内容：</p><figure class="image ss-img-wrapper image_resized" style="width:770.833px;"><img src="https://cdn.sspai.com/2021/07/21/article/bfc0bbbc2475412dd361870ad6a9524e?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="202107211424" data-original="https://cdn.sspai.com/2021/07/21/article/bfc0bbbc2475412dd361870ad6a9524e" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;">不难发现，Logs、Backup 文件夹中记录的是运行的日志和定时保存的无用文件，无需备份。因此在 ShareX 目录创建 .gitignore 写入下面内容：</p><pre class="language-"><code>/Logs 
/Backup</code></pre><h3>提交到远程仓库并设置定时任务</h3><p style="margin-left:0px;">一切准备就绪后，便能链接到远程仓库并提交了，在 GitHub 等处新建私人仓库后：</p><pre class="language-shell"><code>git add .
git commit -m "Settings 2021-07-21"
git remote add origin https://github.com/example/example.git
git push -u origin master</code></pre><p style="margin-left:0px;">要设置定时提交，则可通过 Windows 任务计划程序和脚本，示例脚本如下：</p><pre class="language-shell"><code>$date = Get-Date -Format 'yyyy-MM-dd HH:mm'
cd $home/Settings
git add .
git commit -m "Autobackup $date"
git push -q</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/21/1bdd0a2d6163410ac55fba9dba5a1ae4.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/21/1bdd0a2d6163410ac55fba9dba5a1ae4.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;">此方法上源自一位 Linux 开发者，他将 Linux 的关键设置使用 Git 提交到远程仓库，这样不论在去哪儿工作，使用什么设备，一句 <code>git clone</code>，不多时就能让系统变成自己想要的样子。在 Windows 环境中，也有管理员将 Roaming 文件夹共享给员工来统一管理设备的例子。</p><p style="margin-left:0px;">希望本文的思路能帮到你。</p><p style="margin-left:0px;">题图：<a
