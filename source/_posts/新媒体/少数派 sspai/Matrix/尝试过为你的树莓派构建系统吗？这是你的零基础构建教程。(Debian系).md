<div class="articleWidth-content" data-v-e1f0100a><div class="content wangEditor-txt minHeight" data-v-e1f0100a><blockquote><p>之前看到一位<a href="https://blog.csdn.net/nanhantianyi/">大佬的博客</a>，一下子激起了我为自己树莓派构建系统的兴趣，本文教程基于他的博客进行扩充</p></blockquote><p>本文系统基于<code>Ubuntu Server 20.04.1 LTS</code>（基于Windows 10的VMWare虚拟机），<code>x64架构</code>CPU进行交叉构建。</p><p>考虑到有些萌新还没有使用过VMWare虚拟机，本文也将教授虚拟机配置过程。</p><h2>一、前期准备</h2><h3>1 VMWare Workstation Pro 安装以及虚拟机配置</h3><h4>1.1 安装VMWare Workstation</h4><p>伸手党福利！官方链接<a href="https://www.vmware.com/cn/products/workstation-pro/workstation-pro-evaluation.html">点此</a>即可，点击“立即下载”。安装完后输入许可证密钥（如果没钱激活的话可以自行百度，当然不激活影响不大，只要不被判定为商业用途即可）。</p><h4>1.2 配置虚拟机</h4><ol><li>左上角<code>文件</code>-<code>新建虚拟机</code></li><li>不要典型！选择<code>自定义</code>！！</li><li>两次下一步之后选择<code>稍后安装操作系统</code>！不要直接选择映像文件！</li><li>客户机操作系统选择<code>Linux-Ubuntu</code></li><li>名称自定，其他选项根据自身情况配置，下一步即可</li></ol><h4>1.3 安装系统</h4><ol><li><a href="https://cn.ubuntu.com/download">点击此处下载官方Ubuntu映像</a>，建议使用下载工具（IDM等等）</li><li>右键虚拟机-编辑虚拟机设置</li><li>点击<code>CD/DVD</code>，勾选<code>启动时连接</code>，并点击<code>使用ISO映像文件</code>，选择你刚刚下好的映像</li><li>关闭窗口，开启虚拟机</li></ol><p>PS. 由于我这里已经安装好系统，此处不再演示</p><h3>2 安装依赖</h3><pre class="language-shell"><code># 获取root权限，便于后续操作

sudo su
</code></pre><h4>2.1 换源，系统更新</h4><p>个人习惯使用nano编辑文件，不喜欢也可使用vim等等</p><pre class="language-shell"><code>mv /etc/apt/sources.list /etc/apt/sources.list.bak
nano /etc/apt/sources.list
</code></pre><p>这边采用清华大院开源镜像站TUNA的源，文档来自<a href="https://mirrors.tuna.tsinghua.edu.cn/help/ubuntu/">Ubuntu 镜像使用帮助</a></p><pre class="language-shell"><code>deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-updates main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-backports main restricted universe multiverse
deb https://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-security main restricted universe multiverse</code></pre><p><code>Ctrl+O</code>，<code>Ctrl+X</code>保存并退出。</p><p>更新系统</p><pre class="language-shell"><code>apt update
apt upgrade
</code></pre><h4>2.2 安装应用依赖以及下载文件</h4><p>一揽子全部安装，省的后面再弄，麻烦。</p><pre class="language-shell"><code>apt install kpartx build-essential gcc-aarch64-linux-gnu bc bison flex libssl-dev make libc6-dev libncurses5-dev git debootstrap qemu-user-static
</code></pre><p>下载内核文件准备编译</p><pre class="language-shell"><code>wget https://cdn.jsdelivr.net/gh/yuzh0816/tools@0.1/firmware.tar.xz
# 此处可能会非常慢，因为文件来自Github
wget -O rpi-firmware.tar.gz https://github.com/Hexxeh/rpi-firmware/tarball/master</code></pre><h2>二、开始构建</h2><p>如果想要在工作目录中构建，可以创建文件夹</p><pre class="language-shell"><code>mkdir debian
cd debian</code></pre><h3>1 创建镜像</h3><p>注意，此处设置镜像文件大小为3G，实际情况可以根据自身情况调整</p><pre class="language-shell"><code># 创建buster.img空镜像文件
dd if=/dev/zero of=buster.img bs=3G count=0 seek=1</code></pre><p>使用<code>cfdisk</code>命令进行GUI模式镜像分区</p><pre class="language-shell"><code>cfdisk buster.img</code></pre><p>创建两个分区，一个是FAT32的<code>boot</code>分区，一个是<code>rootfs</code>分区。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/02/16/4f83be60f7fcd9e52f36b325a48d1362.gif" alt="创建镜像" data-original="https://cdn.sspai.com/2021/02/16/4f83be60f7fcd9e52f36b325a48d1362.gif" referrerpolicy="no-referrer"></figure><p>挂载镜像文件并查看挂载的设备名称</p><pre class="language-shell"><code>losetup -f --show buster.img</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/02/10/092df0f2de537f2de43a600b1f9d4b88.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="查看挂载的设备名称" data-original="https://cdn.sspai.com/2021/02/10/092df0f2de537f2de43a600b1f9d4b88.png" referrerpolicy="no-referrer"></figure><p>此时loop设备在<code>/dev/loop8</code>下挂载，我们用kpartx把它两个分区挂载到<code>/dev/mapper</code>下</p><pre class="language-shell"><code>kpartx -va /dev/loop8</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/02/16/165d5ceb1569b485430b745ca2d2f430.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="挂载" data-original="https://cdn.sspai.com/2021/02/16/165d5ceb1569b485430b745ca2d2f430.png" referrerpolicy="no-referrer"></figure><p>可以看到挂载成功。</p><p>这时第一个分区在<code>/dev/mapper/loop8p1</code>，第二个是在<code>/dev/mapper/loop8p2</code>下。</p><p>现在格式化这两个分区(记得把下面的目录改成上面显示的loop-p-，警告不用管)。</p><pre class="language-shell"><code>#格式化boot分区

mkfs.fat -F 32 -n "boot" /dev/mapper/loop8p1

#格式化rootfs，设置label

mkfs.ext4 -L rootfs /dev/mapper/loop8p2</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/02/16/f528f2eb0e9249b030af0142aee66ac8.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="格式化分区" data-original="https://cdn.sspai.com/2021/02/16/f528f2eb0e9249b030af0142aee66ac8.png" referrerpolicy="no-referrer"></figure><p>然后，在工作目录下新建两个分区的目录，并挂载。</p><pre class="language-shell"><code>mkdir boot rootfs
mount /dev/mapper/loop8p1 boot/
mount /dev/mapper/loop8p2 rootfs/</code></pre><p>此时，空镜像文件已经制作完成，执行<code>lsblk</code>命令看挂载情况。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/02/21/37228072b925b0c8aa5db1a7c395066f.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="挂载情况" data-original="https://cdn.sspai.com/2021/02/21/37228072b925b0c8aa5db1a7c395066f.png" referrerpolicy="no-referrer"></figure><p>第一步完成。</p><h2>三、RootFS构建</h2><h3>1 初步构建阶段</h3><p>这里可以选择你想要构建的系统，此处选择为图方便选择Debian。</p><p>镜像可以选择的有很多（必须为Debian系统），列举两个清华大学TUNA镜像地址：</p><ol><li><a href="https://mirrors.tuna.tsinghua.edu.cn/debian/">Debian</a></li><li><a href="https://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports/">Ubuntu</a>（Ports版，支持Arm64）</li></ol><p>经过亲自测试可行的有Debian、UOS和Ubuntu。</p><pre class="language-shell"><code># --foreign表示异构系统构建，即在不同CPU架构下进行构建
debootstrap --arch=arm64 --no-merged-usr --foreign buster rootfs/ https://mirrors.tuna.tsinghua.edu.cn/debian/</code></pre><p>等待数分钟，执行后rootfs目录结构如下</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/28/1b6c9f22fad2d30a5e986d97c317e8f4.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/28/1b6c9f22fad2d30a5e986d97c317e8f4.png" referrerpolicy="no-referrer"><figcaption>目录</figcaption></figure><p>解压之前下好的压缩包（驱动文件以及固件）</p><pre class="language-shell"><code>tar -Jxvf firmware.tar.xz -C rootfs/lib/
mkdir rpi-firmware
tar -zxvf rpi-firmware.tar.gz -C rpi-firmware --strip-components 1
cp -r rpi-firmware/vc/hardfp/opt/vc rootfs/opt/
cp -r rpi-firmware/vc/sdk/opt/vc rootfs/opt/</code></pre><p>编辑文件系统信息</p><pre class="language-shell"><code>#此处依然要将loop后的数字进行更改
blkid /dev/mapper/loop8*</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/28/3e0b9c1d677593d6edaee3874cd92909.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/28/3e0b9c1d677593d6edaee3874cd92909.png" referrerpolicy="no-referrer"><figcaption>blkid</figcaption></figure><p>复制两串PARTUUID，粘贴到<code>rootfs/etc/fstab</code>中。</p><pre class="language-shell"><code>nano rootfs/etc/fstab</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/28/548c5a4cf05b669927a014534c2852e4.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/28/548c5a4cf05b669927a014534c2852e4.png" referrerpolicy="no-referrer"><figcaption>fstab</figcaption></figure><p>然后换源</p><pre class="language-shell"><code>nano rootfs/etc/apt/sources.list</code></pre><pre class="language-shell"><code>deb http://mirrors.tuna.tsinghua.edu.cn/debian/ buster main non-free contrib
deb http://mirrors.tuna.tsinghua.edu.cn/debian-security buster/updates main
deb http://mirrors.tuna.tsinghua.edu.cn/debian/ buster-updates main non-free contrib
deb http://mirrors.tuna.tsinghua.edu.cn/debian/ buster-backports main non-free contrib</code></pre><h3>2 第二构建阶段</h3><p>开始构建</p><pre class="language-shell"><code>chroot rootfs/ debootstrap/debootstrap --second-stage
cp /usr/bin/qemu-aarch64-static rootfs/usr/bin/</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/7e44b6361f712c36364b0c368e0589c9.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/7e44b6361f712c36364b0c368e0589c9.png" referrerpolicy="no-referrer"><figcaption>chroot构建</figcaption></figure><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/9a39d64bf8cc92bdd39ff21f0a599ec0.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/9a39d64bf8cc92bdd39ff21f0a599ec0.png" referrerpolicy="no-referrer"><figcaption>完成！</figcaption></figure><hr><p>chroot进入系统根目录</p><pre class="language-shell"><code>chroot rootfs/ /usr/bin/qemu-aarch64-static /bin/bash</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/8352f4d417a7ce67a2075dd1710b01bd.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/8352f4d417a7ce67a2075dd1710b01bd.png" referrerpolicy="no-referrer"><figcaption>chroot</figcaption></figure><p>开始进行系统内配置：</p><p>1.  更新软件</p><pre class="language-shell"><code>dpkg --add-architecture armhf
apt update
apt upgrade</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/8aa8bc7ca43324ecfa07779e76936c16.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/8aa8bc7ca43324ecfa07779e76936c16.png" referrerpolicy="no-referrer"><figcaption>更新</figcaption></figure><p>2.  安装基本软件以及运行库</p><pre class="language-shell"><code>apt install libc6:armhf
apt install locales rng-tools wpasupplicant dhcpcd5 sudo parted
apt install net-tools avahi-daemon dphys-swapfile ssh openssh-server 
apt install htop bash-completion wget curl vim nano</code></pre><p>3. 添加vc-userland支持</p><pre class="language-shell"><code>echo "/opt/vc/lib/"   >/etc/ld.so.conf.d/vc-userland.conf

echo -e "net.core.default_qdisc=fq\nnet.ipv4.tcp_congestion_control=bbr" >> /etc/sysctl.conf</code></pre><p>5. 设置时区（中国大陆选择上海）</p><pre class="language-shell"><code>ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime</code></pre><p>6. 配置wpa_supplicant，并链接到boot目录下的wpa_supplicant.conf</p><pre class="language-shell"><code>ln -sf /usr/share/dhcpcd/hooks/10-wpa_supplicant /lib/dhcpcd/dhcpcd-hooks/10-wpa_supplicant
ln -sf /boot/wpa_supplicant.conf /etc/wpa_supplicant/wpa_supplicant.conf</code></pre><p>7. 配置主机名称（引号内“yuzhpi”内容更改为主机名称，可在局域网内显示、连接）</p><pre class="language-shell"><code>echo "yuzhpi" > /etc/hostname
echo -e "127.0.0.1\yuzhpi" >> /etc/hosts</code></pre><p>8. 重新设置软件包区域、语言（经本人尝试，Ubuntu似乎不可行  ||  英语好的可以跳过，这步其实没啥必要）</p><pre class="language-shell"><code>dpkg-reconfigure locales</code></pre><p>用<code>空格键</code>选择<code>en_US.UTF-8</code>和<code>zh_CN.UTF-8</code>。<code>Tab</code>键选择<code>Ok</code>。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/67d4ef401ff22b671034108c2e325d60.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/67d4ef401ff22b671034108c2e325d60.png" referrerpolicy="no-referrer"></figure><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/9e2d6243c2dd0a10d5a8d0d477b1227e.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/9e2d6243c2dd0a10d5a8d0d477b1227e.png" referrerpolicy="no-referrer"></figure><p>选择<code>zh_CN.UTF-8</code>，<code>Tab</code>选定。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/089575497d5b3bef81fad16d2521a3bd.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/089575497d5b3bef81fad16d2521a3bd.png" referrerpolicy="no-referrer"></figure><p>9. 添加用户（最好为pi）</p><pre class="language-shell"><code>useradd -m -s /bin/bash pi
passwd pi #输入密码x2
echo "pi    ALL=(ALL:ALL)  NOPASSWD: ALL" > /etc/sudoers.d/00_pi</code></pre><p>10. 安装常用软件</p><p>此处自便，根据需求安装即可。</p><hr><p>退出chroot。</p><pre class="language-shell"><code>sync
exit
echo > rootfs/root/.bash_history #清除bash记录
sync</code></pre><h2>四、Boot分区（启动分区）配置</h2><h3>基本启动配置</h3><p>复制Bootloader启动驱动文件</p><pre class="language-shell"><code>cp rpi-firmware/bootcode.bin rpi-firmware/fixup* rpi-firmware/start* boot/</code></pre><p>然后同上文<code>blikd</code>命令，执行下面那句命令，将8改为最早弄的那个数字，保存<code>PARTUUID</code>（其实前面已经执行过了，就是之前让你们保存的那串PARTUUID）</p><pre class="language-shell"><code>blkid /dev/mapper/loop8p2</code></pre><p>编辑<code>cmdline.txt</code>：</p><pre class="language-shell"><code>nano boot/cmdline.txt</code></pre><p>粘贴，把UUID覆盖此处UUID。</p><pre class="language-shell"><code>dwc_otg.lpm_enable=0 console=serial0,115200 console=tty1 root=PARTUUID=ca013eb7-02 rootfstype=ext4 elevator=deadline fsck.repair=yes rootwait</code></pre><p>编辑<code>config.txt</code>：</p><pre class="language-shell"><code>nano boot/config.txt</code></pre><p>文件内容如下</p><pre class="language-shell"><code>arm_64bit=1
dtparam=audio=on
[pi4]
dtoverlay=vc4-fkms-v3d
max_framebuffers=2
[all]</code></pre><h3>无线网络配置</h3><pre class="language-shell"><code>touch boot/wpa_supplicant.conf</code></pre><p>如无需配置无线，仅需执行上面那行命令即可。</p><pre class="language-shell"><code>nano boot/wpa_supplicant.conf</code></pre><p>替换提示文本为网络配置。</p><pre class="language-shell"><code>ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev
update_config=1
country=CN

network=&#123;
    ssid="wifi名称"
    psk="wifi密码"
    key_mgmt=WPA-PSK
    priority=1
&#125;</code></pre><h2>四、内核构建</h2><p>Git Clone官方内核</p><pre class="language-shell"><code> git clone --depth=1 https://github.com.cnpmjs.org/raspberrypi/linux</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/42245583ee54a38a9d7c0825c7cfbb94.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/42245583ee54a38a9d7c0825c7cfbb94.png" referrerpolicy="no-referrer"><figcaption>git clone</figcaption></figure><p>可以通过修改<code>linux/Makefile</code>第五行<code>EXTRAVERSION</code>自定义版本号</p><p>e.g: <code>EXTRAVERSION = -yuzh</code>，则显示<code>[版本号]-yuzh</code></p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/03/29/b7d37ef4fc688e8f7f66e56fe4916c59.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/03/29/b7d37ef4fc688e8f7f66e56fe4916c59.png" referrerpolicy="no-referrer"></figure><h3>开始编译</h3><p>这步需要好久。好久。好久。。</p><pre class="language-shell"><code>cd linux
make ARCH=arm64 CROSS_COMPILE=aarch64-linux-gnu- bcm2711_defconfig
