<div class="articleWidth-content" data-v-e1f0100a><div class="content wangEditor-txt minHeight" data-v-e1f0100a><blockquote><p>2017年6月1日，简悦 1.0 正式在 Chrome Webstore 公开下载</p></blockquote><p>昨天（5月31日）在跟 waychane 讨论简悦的事情，突然跟他说简悦马上在 Chrome Webstore 就四周年了。 事后回想，我应该从未在正式场合说过简悦的故事，正好四周年之际，想跟你聊聊简悦的故事。这里不会讨论技术，也没有购买链接，只是纯粹的跟你唠唠嗑。</p><h2><strong>缘起</strong></h2><p style="margin-left:0px;">简悦存在的时间其实要远早于 2017年6月1日，非要追溯的话，应该是始于 2016 年...那个时候因为家里的一些事情，导致我解散了团队，注销了公司。 曾有段时间特别沮丧，精神无法集中，看什么都看不下去，所以当时的想法就是找一个可以让我沉浸式阅读体验的 Chrome 扩展，找了很多很多都不满意... 所以决定自己开发，另一个决定我开发的原因是：我发现只要沉浸在代码的世界中，我才能集中精神，不再胡思乱想。</p><h2><strong>一个不一样的阅读模式</strong></h2><p style="margin-left:0px;">如何让阅读模式成为真正的阅读模式而不是简单的提取正文，所以我决定采用一些另类的方式，「<strong>为每个不同的网址适配不同的规则</strong>」，以达到「<strong>完美阅读模式</strong>」的目的。 </p><p style="margin-left:0px;">有了 Idea 后，就撸袖子开干了，因我当时的技术栈还是以后端为主，前端其实只是个菜鸟。为了实现我理想中的阅读模式，我自己捣鼓了 React / Webpack / CSS 这些内容。 </p><p style="margin-left:0px;">经过半年的时间，终于上架到了 Chrome Webstore，首批适配了 120+ 个站点，在经过一个月的自用，于 v2ex 上首发了简悦，得到了很多赞誉。 随后我想在一个更大平台上曝光简悦，进而想到了少数派...</p><p style="margin-left:0px;">我记得那个时候可能还没有 Matrix（或者也只是个雏形），发布后我直接找到了时任编辑的文刀漢三，然后就有了下面的对话内容。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/06/02/88e21bb54a47b7bd4a8c29533d8422e2.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/06/02/88e21bb54a47b7bd4a8c29533d8422e2.png" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;">再之后简悦收获了第一批种子用户，非常感谢少数派在当时的帮忙，如果不收获这些种子用户，或许可能也看不到简悦的今天。</p><p style="margin-left:0px;">再再之后的简悦在阅读模式这条路上一骑绝尘：站点适配器，论坛类页面，分页，识别 LateX，识别 txt / md / html...</p><ul><li>1.0.3 推出了导出到本地与各种生产力工具的功能。</li><li>1.0.5 增加了导出功能，加入了更多的生产力工具。</li><li>1.1.0 增强了适配站点的适配规则。</li><li>1.1.2 推出了插件系统。（因为我认为阅读是件很个性的事情，没有一个统一的标准，所以通过插件来满足不同人群的阅读需求）</li><li>1.1.4 接入了 Notion</li></ul><h2><strong>阅读模式的延伸</strong></h2><p style="margin-left:0px;">直到去年简悦 2.0 版本之前简悦一直都是一个阅读模式扩展，但是我在使用简悦的时候，越深入使用，越发现 <strong>阅读模式其实只是一个入口，一个汲取信息的入口</strong>。</p><p style="margin-left:0px;">既然是信息，就会因人而已，所以增加标注（摘录）与稍后读的功能，它们都是对信息整理的功能。</p><p style="margin-left:0px;">在实现这个两个功能前，我一直比较担心两个事情：</p><ul><li>如何能保证收费用户的最大利益</li><li>如何保证用户的最大隐私权</li></ul><p style="margin-left:0px;">其实这两个顾虑可以归结为：</p><blockquote><p style="margin-left:0px;">如果有一天我不在开发了，如何能让它继续运行下去。</p></blockquote><p style="margin-left:0px;">当时的开发思路：<strong>尽量降低后台对简悦的耦合性，将数据保存在用户自己的「地盘」</strong>。</p><h2><strong>简悦 2.0 是个异类</strong></h2><blockquote><p style="margin-left:0px;">没有登录系统，不保存用户数据。</p></blockquote><p style="margin-left:0px;">完成这些之后，简悦拥有了一个我从未对外公布的特点：</p><blockquote><p style="margin-left:0px;">在某种情况下，简悦 2.0 的功能（含高级账户功能）均可以在本地使用，完全不依赖于服务器，只要 Chrome Extensions API 不取消，就可以一直运行下去，只要有适配规则，阅读模式就一直使用下去。</p></blockquote><p style="margin-left:0px;">在首发时，获得极高的关注度，各项数据也在飙涨，但也因没有登录系统，导致产生了很多问题，当然这些问题最终都被解决了，也并不能称为问题，只是缺乏经验造成的。</p><h2><strong>朋友们</strong></h2><p style="margin-left:0px;">随着简悦 2.0 的发布，我认识了很多朋友，如：</p><ul><li>技术大牛 binsee，帮忙我解决了大部分适配规则的疑难杂症</li><li>Bug 小能手 Yuanio，发现了很多我完全无法重现的 Bug</li><li>Quicker 高手 yu jiale ，为简悦制作了很多精良的 Quicker 插件</li><li>lyserenity 催更队大队长，不仅贡献了大部分催更事宜，还是个隐藏的技术高手，Popclipe 插件就是出自于他</li><li>zy Tomorrow 也是一个技术高手，Onenote 的相关问题都是他一己之力完成的。</li><li>Mr.流，动手能力虽然差点，但氛围组组长的 Title 不是吹的，再加上非常好的脾气，帮忙我管理 Telegram 群</li><li>lock heart 隐藏的工作流高手，多次为简悦仗义执言</li><li>TG BON 2.2 定制化 Markdown 的思路就是来自于他</li><li>Shawn 一个即将毕业的设计类院校的小朋友，同步助手的 Logo 就出自于他</li><li>煎饺 新晋的 UE 大神，将会帮助我解决简悦一系列 UE 问题</li><li>又一隐世高手 废柴，只在必要时出手，一出手就节省了我 nnn 多时间</li><li>Peakman Cheung 简悦的狂热级 fans / 深度使用者，未来的简悦文章就靠他了</li><li>给简悦帮忙颇多的 IG WXZ ，很多文案我都是直接发给他原稿，润色都是校长所为</li></ul><p style="margin-left:0px;">当然还有很多大咖的帮助：</p><ul><li>玉树老师 / 少楠老师 / JimmyLv / 天生 / 六老板 / Linmi / 倪爽老师 / 钟大，以及在独立开发者群里每个人；</li><li>少数派这里的各位，老麦 / waychane / 路中南 / 李曦 / sainho 甚至更多我都不认识的默默付出的工作人员；</li><li>很多媒体朋友，如：爱范儿 / 最美应用，正因为你们，简悦在媒体推广上一分钱都没有花；</li><li>还有太多太多在推上 / Email 中帮助过的朋友了。</li></ul><p style="margin-left:0px;">我想如果没有你们，肯定是没有现在的简悦，没有 1.5K+ 好评的简悦。 随着 2.2 的发布，我想简悦的队伍又要壮大了...</p><h2><strong>最后</strong></h2><p style="margin-left:0px;">我还想聊聊 <a href="https://simpread.pro/welcome/version_2.2.0.html">简悦 2.2</a>，我不是一个喜欢重复造轮子的人，所以简悦的稍后读不会走其它产品的老路。</p><p style="margin-left:0px;">所以，你在这里不会看到：</p><ul><li>超级简单的导入方式（我认为稍后读的最大「问题」就是：方便的导入</li><li>传统的稍后读界面以及稍后读的设计思路</li><li>所谓的 Capture 方案</li></ul><p>但，你可以拥有：</p><ul><li>如何解决稍后读不读 / 越积越多的问题</li><li>稍后读的真正使用方式</li><li>真正的 Capture 方案</li><li>你将拥有真正意义 / 完整是数据所有权</li></ul><p>虽然拥有这些你会损失一些所谓的易用性，但「<strong>自由与完整意义的所有权</strong>」不更加难能可贵吗？</p><p>最后，简悦 2.2 才是我心里的 <strong>真正 2.0 </strong>版本，同时也是我在心中 Roadmap 第一步而已，还有很多步没有走，希望你能一如既往的喜欢简悦，信任简悦。</p><blockquote><p style="margin-left:auto;">谢谢与你同行。</p></blockquote></div><!----></div><div style="border:1px solid transparent;"
