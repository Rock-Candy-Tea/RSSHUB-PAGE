style="margin-left:0px;text-align:justify;">前些日子，我在 B 站做了一次直播，讲如何利用 Keyboard Maestro 快速采集输入临时笔记。很多小伙伴观看之后都表示很感兴趣，并且 <a style="margin-left:0px;text-align:justify;">这个<strong>可以有</strong>。</p><h2 style="margin-left:0px;text-align:justify;"><strong>效果</strong></h2><p style="margin-left:0px;text-align:justify;">我们先来看看，应用了 Keyboard Maestro 之后，上传图片的操作可以变得多简单，甚至是有趣。</p><p style="margin-left:0px;text-align:justify;">这个宏操作，可以让你用以下三种方式，上传图片到指定图床。这里我们以微博图床为例。</p><p <code>Option+Cmd+u</code>，这张图就被上传到了微博图床，剪贴板的内容，变成了图床上该图片的 Markdown 链接。</p><p style="margin-left:0px;text-align:justify;">例如我现在写作中，就把编辑器里这段文字截图。然后执行组合键，可以获得如下的返回链接：</p><pre class="language-hljs style="margin-left:0px;text-align:justify;">有了上面这三项图片上传功能，我自己工作流中一般的插图情境，就可以<strong>这一个组合键全覆盖</strong>了。</p><p style="margin-left:0px;text-align:justify;">随时随地，你可以把网上的、剪贴板里的、相机中的，甚至是存储在电脑里面的若干张图片，快速变成可以直接引用的 Markdown 图片链接。</p><p style="margin-left:0px;text-align:justify;">这样做的好处，是你再也无需顾虑自己用哪种编辑器，也不必担心写作完毕之后，还得重新折腾插图导出和导入。因为所有的图片，都能在你<strong>挪动纯文本的同时，跟上你的脚步，一起走</strong>。</p><p style="margin-left:0px;text-align:justify;">是不是很方便呢？</p><p style="margin-left:0px;text-align:justify;">顺便说一句，这个工作流也支持动图（gif），因而不必担心你好不容易录制下来的操作步骤演示，被无端鼓捣成一张静态图片。</p><h2 style="text-align:justify;"><strong>原理</strong></h2><p style="margin-left:0px;text-align:justify;">把图片上传到图床，并非难事儿。在 macOS 上，你有好几款工具可以选择。免费的就包括 iPic 和 uPic 等。但是真正用起来，你会发现这些工具有些不尽如人意。</p><p style="margin-left:0px;text-align:justify;">例如有的工具需要你先把图片弄到剪贴板里，才能上传；有的虽然可以选择本地文件上传，但是会弹出一个对话框，让你每次都手动选择；更要命的是，面对多个图片文件，大部分工具都不支持批量操作 —— 你不得不对每一个文件，都重复一遍上传流程。</p><p style="margin-left:0px;text-align:justify;">我在 Keyboard Maestro 里面，做了个宏命令。然后绑定到 <code>Option+Cmd+u</code> 快捷键上。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/9b256de7eee7f72d20ee886f290d626d?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/9b256de7eee7f72d20ee886f290d626d" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">Keyboard Maestro 可以模拟的，是一系列人工手动操作。你自己做也不是不行。只不过做久了，不少人想用头撞树。</p><p style="margin-left:0px;text-align:justify;">下面，我来给你介绍一下这套快捷插图上传流程用法。让你在几分钟之后，就愉快地用上它。</p><p style="margin-left:0px;text-align:justify;">你需要准备两样东西，分别是 iPic 和 Keyboard Maestro 。</p><p style="margin-left:0px;text-align:justify;">我们先来看看 iPic 的设置。</p><h2 style="text-align:justify;"><strong>iPic</strong></h2><p style="margin-left:0px;text-align:justify;">iPic <a href="https://apps.apple.com/cn/app/id1101244278?mt=12">可以从 App Store 直接下载</a>。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/fb4d04155bf286e0ee0590aa2eaa43e0?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/fb4d04155bf286e0ee0590aa2eaa43e0" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">iPic 的免费版本，功能上是有限制的，那就是只支持一个图床。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/a0b51260835628dfa5efddec852e8ac8?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/a0b51260835628dfa5efddec852e8ac8" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">但是好消息是，这仅有的一个图床，恰恰就是我们需要的微博图床。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/990542cc81b38fefe756361201f52605?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/990542cc81b38fefe756361201f52605" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">更好的消息，开箱即用，无需设置图床。</p><p style="margin-left:0px;text-align:justify;">下面我们来说说 iPic 里，你需要设置的地方。</p><p style="margin-left:0px;text-align:justify;">点击状态栏上面 iPic 的图标，选择「更多」，然后点击「偏好设置」（Preferences...）。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/98b98c8c64134eddc53b4e397e2f5c44?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/98b98c8c64134eddc53b4e397e2f5c44" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">你需要做的改动，是确保以下两个设定：</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/61d54d49e486b15f5787a68c0f4b5193?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/61d54d49e486b15f5787a68c0f4b5193" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">首先，开机启动 iPic 。这样后面用起来更方便。</p><p style="margin-left:0px;text-align:justify;">其次，把上传图片的快捷键设置成 <code>Option + Shift + Command + U</code>。<strong>这个很关键</strong>，不然后面我为你定制的 Keyboard Maestro 宏（Macro）起不到作用。</p><p style="margin-left:0px;text-align:justify;">这一页面还有个「上传前压缩」选项，我觉得没必要勾选。你就是不选择，上传后的图片也会压缩。对于普通文章（包括技术演示类）撰写，清晰度也算足够了。你看看本文的插图，都已是压缩后的结果。</p><p style="margin-left:0px;text-align:justify;">如果你是个摄影师，需要保持原始图片格式…… 这个图床不适合你，还是换付费图床吧。</p><p style="margin-left:0px;text-align:justify;">这几步做完，iPic 就算是设置好了。下面我们来看看 Keyboard Maestro 这边的操作。</p><h2 style="text-align:justify;"><strong>Keyboard Maestro</strong></h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/9ab899f26877771cef703259cfbcd241?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/9ab899f26877771cef703259cfbcd241" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">下载安装 Keyboard Maestro 后，请你 <a href="https://wise-pullover-00f.notion.site/Keyboard-Maestro-Macro-678fa52d1f584a94b709484395ef5d2e">下载我给你做好的宏命令</a>。存放到本地备用。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/af7d4c30e6b74caf69a07d01f9f946dc?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/af7d4c30e6b74caf69a07d01f9f946dc" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">准备好后，进入 Keyboard Maestro ，菜单中选择「导入」-「安全导入宏命令」。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/fa2bfe8a27367c04fc18722145162e88?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/fa2bfe8a27367c04fc18722145162e88" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">在弹出的文件选择框里，选择你下载好的宏命令，按确定按钮后即可导入。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/258cc1a5439472d767ab6af6defa8c49?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/258cc1a5439472d767ab6af6defa8c49" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">注意导入之后，该宏命令默认的状态是「关闭」，即上图左上角图标为灰色。</p><p style="margin-left:0px;text-align:justify;">你需要手动开启一下。方法为，在宏命令名称处（如下图所示）单击右键，选择「开启」（Enable Macro）。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/8f4c0944e80b3db8c556bf1b19140b11?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/8f4c0944e80b3db8c556bf1b19140b11" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">之后，我们需要做一些修改。</p><p style="margin-left:0px;text-align:justify;">首先（可选），是根据你自己的喜好，修改一下宏命令的名称和激活的快捷键（默认是 <code>Option+Cmd+u</code> ）。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/6e969d5e51a6829b2e54e325094b786c?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/6e969d5e51a6829b2e54e325094b786c" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">其次，最重要的部分来了。往下翻，找到这个位置。里面有个变量，叫做 <code>exported_dir</code>，对应的默认内容是<strong>空白的</strong>。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/8ece6dc4417128adf899ded0013934b0?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/8ece6dc4417128adf899ded0013934b0" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">这里需要填写的，是你准备临时存放导出图片的目录名称。</p><p style="margin-left:0px;text-align:justify;">建议你跟我一样，把临时目录的路径，设置在下载（Downloads）目录的子文件夹里。注意一定要使用<strong>绝对路径</strong>。</p><p style="margin-left:0px;text-align:justify;">你不知道啥叫绝对路径？没关系。给你举个例子。好比说你的 macOS 用户名是 tom ，子目录名称叫做 photos_exported ， 那么此处对应填写的内容应该是：</p><pre class="language-hljs awk"><code>/Users/tom/Downloads/photos_exported/</code></pre><p style="margin-left:0px;text-align:justify;">设置结束。</p><p style="margin-left:0px;text-align:justify;">下面你就可以仿照前面的范例，从不同的渠道选择图片，然后尝试用 <code>Option+Cmd+u</code> 快捷键上传图片，并且获取 Markdown 链接了。</p><h2 style="text-align:justify;"><strong>对比</strong></h2><p style="margin-left:0px;text-align:justify;">原先 <a href="https://mp.weixin.qq.com/s/XewWFnn_nB7i3z8_7ExxBg">我的插图流程</a>，需要一种特殊 Markdown 打包格式的辅助，叫做 Textbundle 。</p><p style="margin-left:0px;text-align:justify;">Textbundle 其实就是个文件夹，它包含了 Markdown 文本文件本身，外加一系列的图片（放在指定的子目录），然后 Markdown 文本引用这些图片的相对链接。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/32869cb80f501849fd33c8af35d363e4?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/32869cb80f501849fd33c8af35d363e4" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">这样一来，你挪动 Markdown 文本的时候，图片都跟着走，数据是<strong>完整</strong>的。</p><p style="margin-left:0px;text-align:justify;">这种方式有个弊端——<strong>交换数据太麻烦</strong>。</p><p style="margin-left:0px;text-align:justify;">不是每一款笔记工具和编辑器，都支持 Textbundle 格式。你在笔记的采集、汇总、加工等阶段，可选择的余地就变少了。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/28/article/20817d4881804ac33807d23dc2cae15a?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2022/02/28/article/20817d4881804ac33807d23dc2cae15a" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;text-align:justify;">对我来说最大的坏消息是，Roam Research 就无法支持这种格式。</p><p style="margin-left:0px;text-align:justify;">另外，到了发布内容的时候，你不能使用本地相对路径的图片啊。所以，这些图片<strong>早晚都得上传</strong>。既然如此，为何不早点儿把它直接放在图床上？这样<strong>一劳永逸</strong>，不用在图片链接上，各种来回瞎折腾了。</p><h2 style="text-align:justify;"><strong>疑问</strong></h2><p style="margin-left:0px;text-align:justify;">你可能会有个疑惑：</p><blockquote><p style="margin-left:0px;">图床挂了可咋整？</p></blockquote><p style="margin-left:0px;text-align:justify;">这是个好问题。我们必须随时注意数据的备份。不光是图床，笔记的文字本身也是需要做好备份工作的。</p><p style="margin-left:0px;text-align:justify;">好在如果你把笔记的内容都归拢到了 Roam Research ，那么你就可以利用之前教程里，我给你介绍的 <a href="https://sspai.com/post/60633">自动按时增量备份工具了</a>。立青（JimmyLv）改进之后，这个工具不仅可以备份文本，也可以自动备份其中包含的图片链接。这样一来，你就可以做到有「备」无患了。<a href="https://github.com/JimmyLv/roam2github-actions">这里是该项目的地址</a>。</p><p style="margin-left:0px;text-align:justify;">第二个疑惑是：</p><blockquote><p style="margin-left:0px;">上述方法是不是只适合微博图床？</p></blockquote><p style="margin-left:0px;text-align:justify;">当然不是。不管你用的是 uPic 还是本文中的 iPic ，都可以支持多种图床。例如 uPic ，官方介绍支持的图床包括 <a href="https://sm.ms/">smms</a>、 <a href="https://www.upyun.com/products/file-storage">又拍云 USS</a>、<a href="https://www.qiniu.com/products/kodo">七牛云 KODO</a>、 <a href="https://www.aliyun.com/product/oss/">阿里云 OSS</a>、 <a href="https://cloud.tencent.com/product/cos">腾讯云 COS</a>、<a href="https://weibo.com/">微博</a>、<a href="https://github.com/settings/tokens">Github</a>、 <a href="https://gitee.com/profile/personal_access_tokens">Gitee</a>、 <a href="https://aws.amazon.com/cn/s3/">Amazon S3</a>……，甚至是 <a href="https://blog.svend.cc/upic/tutorials/custom">自定义上传接口</a>。</p><p style="margin-left:0px;text-align:justify;">另外的一个疑惑，可能是：</p><blockquote><p style="margin-left:0px;">如果我用的不是 macOS ，是不是就不能享受这种便利了？</p></blockquote><p style="margin-left:0px;text-align:justify;">不会。Windows 上面也有类似自动化工具和图片上传应用。但是因为我平时用 macOS ，所以不了解这些工具的特点，无法给你分享了。如果你尝试过，并且也能进行图片的批量上传和 Markdown 链接返回，欢迎在留言区分享你的经验心得。</p><h2 style="text-align:justify;"><strong>小结</strong></h2><p style="margin-left:0px;text-align:justify;">本文为你介绍了我的图片快速处理方案。主要用到了 iPic 和 Keyboard Maestro 这两款应用。</p><p
