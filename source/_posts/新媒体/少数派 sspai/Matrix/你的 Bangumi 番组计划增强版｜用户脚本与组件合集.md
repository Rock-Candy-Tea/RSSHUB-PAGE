referrerpolicy="no-referrer"></figure><p>让我们进入正题⬇️</p><h2>全局：番组域名重定向</h2><p>目的是统一域名。Bangumi 网站主要有三个域名：bangumi.tv、bgm.tv 以及 chii.in。在站内页面互相跳转的时候，域名一般是统一的；而从外部点击链接进入网站，会出现域名不统一的情况。（比如你一直在 bgm.tv 上，这时好友发给你一个链接是 bangumi.tv 的，就会导致域名不统一，而且两边都要各自登录。）另外，部分脚本也没有覆盖到 chii.in，虽然也可以自己后期追加，但本着省事（看着也舒服）的原则，还是统一域名比较好。</p><p>要完成这件事，相比脚本，建议直接使用扩展，因为扩展更通用，还能应用于其他重定向任务上。</p><p>以 Redirector 为例（<a href="https://addons.mozilla.org/en-US/firefox/addon/redirector/" target="_blank">Firefox</a> <a href="https://chrome.google.com/webstore/detail/redirector/ocgpenflpmgnfapjedencafcfakcekcd" target="_blank">Chrome</a> <a href="https://microsoftedge.microsoft.com/addons/detail/redirector/jdhdjbcalnfbmfdpfggcogaegfcjdcfp" target="_blank">Edge</a>），新增一个规则，填写：</p><p>Include pattern:</p><p><code>^https?://(doujin\.)?(?:bangumi|chii)\.(?:tv|in)/(.*$)</code></p><p>Redirect to:</p><p><code>https://$1bgm.tv/$2</code></p><p>Pattern type: Regular Expression</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/editor/u_/c6sm58lb34tbesgmdl6g.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c6sm58lb34tbesgmdl6g.png" referrerpolicy="no-referrer"></figure><p>这样，当访问 bangumi.tv 和 chii.in 时，都会被重定向至 bgm.tv。（注：我个人偏好使用 bgm.tv 这个域名，可以自行修改。）</p><h2>用户脚本推荐</h2><p>在正式推荐前，想简单说明一些 Bangumi 番组计划专用名词，以更好理解脚本目的。</p><ul><li>条目：即一个「作品」，比如这是一个 <a href="https://bgm.tv/subject/253" target="_blank">动画条目</a>，这是一个 <a href="https://bgm.tv/subject/62229" target="_blank">游戏条目</a></li><li>目录：条目的合集，相当于豆瓣的「片单」「书单」一类，比如这是一个 <a href="https://bgm.tv/index/15045" target="_blank">目录</a></li><li>条目列表：顾名思义，条目以列表形式展现的地方，比如 <a href="https://bgm.tv/anime/browser?sort=rank" target="_blank">排行榜</a> 和 <a href="https://bgm.tv/index/15045" target="_blank">目录</a> 都是条目列表</li><li>首页、点格子：指的是登录后的主页，因为标记进度是以格子形式展现，所以叫做「点格子」</li></ul><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/12/15/4cf200917d09504f3c1376a25ffb6ad1.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/12/15/4cf200917d09504f3c1376a25ffb6ad1.png" referrerpolicy="no-referrer"><figcaption>首页点格子</figcaption></figure><ul><li>章节：即一部作品的「一话」或「一集」，每一个章节会对应一个 <a href="https://bgm.tv/ep/3354" target="_blank">章节讨论</a></li><li>吐槽：200 字以内的简评，相当于豆瓣的「短评」，比如这是一个 <a href="https://bgm.tv/subject/253/comments" target="_blank">吐槽页</a></li><li>评论：长评文章，相当于豆瓣的「剧评」，比如这是一个 <a href="https://bgm.tv/subject/253/reviews" target="_blank">评论页</a></li><li>讨论版：针对条目的讨论帖，相当于豆瓣的「讨论区」，比如这是一个 <a href="https://bgm.tv/subject/253/board" target="_blank">讨论版</a></li><li>小组：顾名思义，讨论某一主题的小组，比如 <a href="https://bgm.tv/group/download" target="_blank">宽带综合症候群</a> 和 <a href="https://bgm.tv/group/dev" target="_blank">番组开发</a></li></ul><h3>适合搭配其他脚本：自动加载下一页</h3><p><a href="https://github.com/bangumi/scripts/tree/master/liaune#%E8%87%AA%E5%8A%A8%E5%8A%A0%E8%BD%BD%E4%B8%8B%E9%A1%B5" target="_blank">GitHub</a> <a href="https://bgm.tv/dev/app/17" target="_blank">超合金组件</a> <a href="https://bgm.tv/group/topic/344601" target="_blank">站内讨论</a></p><p>Bangumi 的分页采用的是「点击进入下一页」的形式。脚本安装后，在有分页的页面，滚动到最下方，分页栏右侧会出现两个按钮。点击「Auto Load Pages」，将会随着鼠标滚动自动加载下一页，类似 infinit scroll（无限滚动）。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/editor/u_/c6sm58tb34tbesgmdl70.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c6sm58tb34tbesgmdl70.png" referrerpolicy="no-referrer"></figure><p>点击「Load All Pages」，将会一次性自动展开所有页面（建议慎重在几百页的地方使用，比如 <a href="https://bgm.tv/anime/browser?sort=rank" target="_blank">排行榜</a>）。展开过程中点击「Stop Loading All Pages」将会停止加载。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/editor/u_/c6sm595b34tbdihvm2b0.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c6sm595b34tbdihvm2b0.png" referrerpolicy="no-referrer"></figure><p>这个脚本单独使用可能没有什么大感觉，不过在配合其他脚本时，能够发挥很大作用。因为按照原有的分页模式，获取的数据也只有当前页的，假设我们使用一个排序脚本，也只能排序当前页；使用自动加载下一页脚本后，能对所有已加载的页面实行排序动作。</p><h3>首页点格子超进化</h3><h4>Bug Fork</a></p><p>Bangumi 首页进度管理分为列表模式和平铺模式。点击全部、动画、三次元、书籍，将会切换到对应的条目类型。</p><figure class="image ss-img-wrapper"><img data-original="https://cdn.sspai.com/editor/u_/c6sm5a5b34tbdihvm2bg.png" referrerpolicy="no-referrer"></figure><h3>标签管理</h3><h4>标签批量管理&批量设置仅自己可见</h4><p><a href="https://greasyfork.org/zh-CN/scripts/21981-bangumi-%E6%A0%87%E7%AD%BE%E6%89%B9%E9%87%8F%E7%AE%A1%E7%90%86" target="_blank">Greasy Fork</a></p><p>与豆瓣一样，Bangumi 也有标签系统，用于分类标记自己的收藏，比如按播出日期、按系列、按导演等。<strong>但是这个标签系统其实挺难用的，它不能批量修改。</strong>比如下图中「頭文字D」标签标记了6部作品，假如想把标签名称改成简体的「头文字D」，就得在每一个条目上点击「修改」，把标签「頭文字D」改成「头文字D」才行。同样，如果想给多个条目打上新标签，也要一个一个加；删除标签也是如此。也就是说，有几个条目要改，就得操作几次。</p><p>另外，Bangumi（不知何时）引入了「仅自己可见」系统，不想公开的收藏可以设置可见范围，但也要一个一个去标记，比较麻烦。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/editor/u_/c6sm5adb34tbdpmufj40.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c6sm5adb34tbdpmufj40.png" referrerpolicy="no-referrer"></figure><p>安装批量管理脚本后，可以看到标签栏出现了许多新按钮，条目列表也出现了多选框。</p><p><strong>如何批量添加标签、修改可见范围</strong></p><ol><li>选中条目右侧的多选框</li><li>点击标签列表顶部的「+」「p」「c」按钮，分别代表批量加标签、批量设置仅自己可见、批量设置公开可见</li></ol><p><strong>如何修改已有标签</strong></p><ul><li>标签名称右侧的「#」「x」按钮，分别代表修改标签名称、删除该标签</li></ul><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/12/15/5bb34d674845b857e92b280bcb0696fb.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1"
