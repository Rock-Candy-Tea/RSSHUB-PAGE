<div class="articleWidth-content" data-v-3e67f99b><div class="content wangEditor-txt minHeight" data-v-3e67f99b><p style="margin-left:0px;">转眼已经是到了使用iPad Pro <strong>作为生产力工具的第三年</strong>，它和Apple Pencil <strong>曾经</strong>陪我渡过了大部分学习和创作。而如今从灵感的记录，到创作的实现，<strong>iPad的生产力能力却让我用回了熟悉的MacBook，为何？</strong></p><p style="margin-left:0px;">2021年，很多人都由于疫情的原因入手了新的学习和工作设备，从桌面端到移动端，iPad 作为生产力工具总是一个脱离不开的话题。</p><p style="margin-left:0px;">作为一种定位介于桌面端和移动端的产品，iPad 凭借其<strong>贯穿的用户体验，</strong><s><strong>完整</strong></s><strong>残缺的应用生态，出色的硬件配置</strong>等特性，在生产力的设备清单里一枝独秀，<strong>’你的下一台电脑何必是一台电脑’？</strong></p><p style="margin-left:0px;">iPad Pro 能否替代电脑？购入iPad 后是否能解决效率问题？这可能取决你的工作流 -- </p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/26/article/170f84b65e37d2371c117b6b9921a4dd?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="iPad Pro 11" data-original="https://cdn.sspai.com/2021/07/26/article/170f84b65e37d2371c117b6b9921a4dd" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;">结合了MacBook Pro, Mac mini, iPad Pro的深度使用体验后，我归纳了iPad和桌面操作系统在生产力上的本质区别。</p><h2 style="margin-left:0px;">围绕软件的使用逻辑是iPad 的唯一限制，却也是优势</h2><p style="margin-left:0px;">随着iPad OS 15的推出，iPad OS 的多任务管理越来越完善，无论从多窗口还是侧滑调出应用都代表了iPad <strong>结合自身触屏对桌面端多任务的模仿</strong>。 </p><figure class="image ss-img-wrapper image_resized" style="width:372px;"><img src="https://cdn.sspai.com/2021/07/26/66fe4f6a2505fd18db25c972be2b5eb7.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/26/66fe4f6a2505fd18db25c972be2b5eb7.jpg" referrerpolicy="no-referrer"><figcaption>图源Unsplash, iPad OS 14</figcaption></figure><p style="margin-left:0px;">然而受限于iPad 本身的定位，窗口管理仍然是<strong>基于应用</strong>的交互模式。相比较之下，桌面端的交互方式 -- <strong>文件管理模式和窗口的自由度</strong>，导致了iPad 使用者<strong>思维逻辑的不同</strong>。</p><p style="margin-left:0px;">在iPad使用者的眼里，自己当前正在处理的任务是基于两个<strong>应用本身</strong>的；而在Mac使用者的眼里，同一类型的任务是基于<strong>任务本身</strong>的。</p><p style="margin-left:0px;">在我个人的工作流中，如果使用iPad 完成复杂的工作流，不断的切换不同的软件相当繁琐，所以iPad 受限于<strong>简单，线性</strong>的工作，例如上课记笔记，码字，图片浏览，游戏等等，并且能<strong>相对高效地执行</strong>。</p><p style="margin-left:0px;">一旦涉及到<strong>围绕任务本身的工作流</strong>，而<strong>不是iPad所擅长的围绕应用</strong>（通常需要两个以上多方应用参与执行任务）的情况下，例如查询资料和论文，编程（即使使用pythonista 等软件workaround 已经可以在iPad 端实现），日程管理，需要reference 的笔记等等，iPad 线性的<strong>基于应用</strong>的任务管理系统明显式微，繁琐的应用切换根本应付不过来思维周期的短暂，很多时候想完成一个任务由于操作反应太慢<strong>逐渐在脑海中堆积了一堆任务</strong>。</p><p style="margin-left:0px;">因此，在处理复杂等级高的时候我宁愿放弃触屏的体验和120帧高刷新率而切换到Mac mini 上。</p><figure class="image ss-img-wrapper image_resized" style="width:412px;"><img src="https://cdn.sspai.com/2021/07/26/9bc7350848f087a99969456256b2e903.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/26/9bc7350848f087a99969456256b2e903.jpg" referrerpolicy="no-referrer"></figure><h2 style="margin-left:0px;">围绕软件vs围绕任务，两类生产力的不同</h2><p style="margin-left:0px;">具象地来说，在我的工作流中：</p><h3 style="margin-left:0px;"><strong>iPad的使用场景例子</strong></h3><ul><li>Lightroom</li><li>MarginNote + GoodNotes</li><li>Reeder</li><li>Notability</li><li>……</li></ul><h3 style="margin-left:0px;"><strong>Mac的使用场景例子</strong></h3><ul><li>引用文献：Pages/Word + EndNote + Safari</li><li>编程：iTerm + VS Code + Finder</li><li>写作：Ulysses + Safari + Obsidian</li><li>照片编辑：Photos + Lightroom + Finder</li><li>日常管理：Email + Calendar + Reminder + Notes</li><li>笔记查找：Teams + OneDrive + PowerPoint</li><li>Flashcard：Anki + Preview + PowerPoint + Teams + Numbers</li><li>……</li></ul><p>两种设备的工作流中，<strong>最本质的区别</strong>仍然是：在iPad 上，我思维经过App 联想到任务，一个任务往往只和一个特定App 有联系；在Mac 上，任务总是<strong>以一整个工作流</strong>的方式出现，如果一个任务仅仅需要一个App 的线性工作流，<strong>那说明它在iPad 上可能生产力更高</strong>。</p><p>这可能可以帮你判断一个任务究竟该<strong>在哪个平台上执行更有效率</strong>。例如在我的工作流中，抛开平台功能的不一样，Lightroom 如果仅仅是简单的编辑，那在iPad 上的体验得益于触屏和性能，<strong>更让我感到乐趣和交互的直观</strong>。</p><p style="margin-left:0px;">在使用Mac 的思维模式中，这种生产力为<strong>横向生产力</strong><sup class="ss-footnote" href title="非原创词汇，来源不明，欢迎指出" footnote-id="1">1</sup>。与其相反的iPad 生产力因为其基于软件的线性思维，我称之为<strong>纵向生产力</strong><sup class="ss-footnote" href title="同上" footnote-id="2">2</sup>。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/26/c18bf8aaf55cea2eab999c51ebb2413b.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/26/c18bf8aaf55cea2eab999c51ebb2413b.jpg" referrerpolicy="no-referrer"></figure><h2 style="margin-left:0px;"><strong>触屏作为Primary 交互方式, Smart Keyboard 能改变什么</strong></h2><p style="margin-left:0px;">对于没有深挖iPad 生产力的大部分人来说，iPad可能只是一个<strong>爱奇艺设备和玩具</strong>，无法充当生产力工具，根本原因就是<strong>基于触屏的本质</strong>。Smart Keyboard 解决了iPad的一个短板：<strong>文字处理能力和快速切换任务能力</strong>。</p><p style="margin-left:0px;">iPad作为一个缺乏文件管理模式和窗口的自由度的设备，导致了使用者思维逻辑和桌面设备用户的不同。Smart Keyboard 的诞生意味着Apple 填补了桌面端和移动端两个截然不同终端的中间，<strong>且令其更加模糊</strong>，<strong>但还没有完全实现桌面端的交互方式</strong>，所以像上文说的Mac 的使用场景的例子目前还是无法实现，所以在本质上，如果你是<strong>围绕任务的重度生产力用户</strong>，iPad Pro就算有Smart Keyboard 的加持可能也<strong>满足不了你</strong>。</p><p style="margin-left:0px;">另一方面，加上Smart Keyboard，独一无二的iPad OS软件生态仍然能给用户不同的软件体验。而没有Smart Keyboard，我单单用Apple Pencil 也深挖了iPad 的生产力潜能。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/07/26/7910cbc89a702c9b1a7772eb044fbca8.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/07/26/7910cbc89a702c9b1a7772eb044fbca8.jpg" referrerpolicy="no-referrer"></figure><p style="margin-left:0px;"> </p></div><!----></div><div style="border:1px
