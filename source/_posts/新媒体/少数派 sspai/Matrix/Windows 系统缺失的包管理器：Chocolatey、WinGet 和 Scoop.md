style="margin-left:0px;">文章代表作者个人观点，少数派仅对标题和排版略作修改。</p><hr><h2>写在前面</h2><p>如果你也身处 Windows 系统的生态环境之中，或许你身边也有小伙伴正在遭受这样的烦恼：他们永远搞不懂自己辛辛苦苦在搜索引擎找到并下载安装的软件到底在哪里？他们也不清楚为什么电脑中总会出现形形色色、不明不白的垃圾软件和弹窗？即便电脑已经越来越臃肿，他们也不敢碰除个人文件外的任何东西，生怕系统崩坏、数据丢失；他们没有文件管理习惯，快捷方式是他们认为软件是否安装的唯一标志，甚至认为删除即卸载；他们深知自己电脑的内部文件体系已是一团乱糟糟，所以桌面总是满满当当、密不透风。</p><p>搜索、下载、安装，这是几乎任何一个软件在「食用」之前要做的准备工作。每一步都有坑：百度搜索里的「海底捞」，点击下载那一刻的战战兢兢，等待安装时的擦亮双眼、诚惶诚恐。如果算上在使用过程中的升级和使用完之后的卸载，那麻烦就更多了。</p><p>上述或许并不夸张，正是你我之前曾经有过或正在经历的困惑。</p><p>如果这样，那么本文正是为你量身打造、包管理器也非你莫属。</p><p>「包管理器」一行命令，帮你打通软件从搜索、下载、安装到配置、升级、卸载的全流程。</p><h2>什么是包管理器</h2><blockquote><p>包管理器又称<strong>软件包管理系统</strong>，它是在电脑中<strong>自动安装、配置、卸载和升级</strong>软件包的工具组合，在各种系统软件和应用软件的安装管理中均有广泛应用。</p><p>——引自<a href="https://baike.baidu.com/item/%E5%8C%85%E7%AE%A1%E7%90%86%E5%99%A8/22785432?fr=aladdin">百度百科</a></p></blockquote><p>如果你用过 <a href="https://baike.baidu.com/item/Python">Python</a> ，那么对 pip 一定不陌生，Python 对所有第三方库的查找、下载、安装、卸载等都可以通过 pip 来完成的。需要哪个库，<code>pip install</code>帮你安装；不需要了，<code>pip uninstall</code>一键就能卸载。简单省事、方便快捷。不少编程语言均有各自对应的包管理器，比如 Nodejs 的 npm 等。</p><p>其实在操作系统上，包管理器应用就更广泛了，尤其是 Linux 系统和 macOS 系统已经相当成熟了。比如 Ubuntu 的 apt、CentOS 的 yum、 macOS 系统下的 Homebrew 等。Windows 平台的包管理器虽然仍受诟病，但目前来看已经能够完成绝大多数的工作了。</p><p>Windows 系统上常见的包管理器主要有 <strong>Chocolatey</strong>、<strong>winget</strong> 和 <strong>Scoop</strong>。</p><h2>为什么要用包管理器</h2><ul><li>降低安装维护软件的成本</li><li>避免安装大量软件造成的路径污染</li><li>不必查找和安装软件的其他依赖项</li><li>避免捆绑和垃圾软件</li><li>彻底地卸载</li></ul><p>仅第一条理由就足以让我拥抱它~~🥂🥂</p><h2>王牌劲旅：Chocolatey</h2><h3>安装前提</h3><p>在安装之前，必须要保证自己电脑满足以下标准：</p><ul><li>Windows 7+ / Windows Server 2003+</li><li>PowerShell v2+</li><li>.NET Framework 4+</li></ul><p>如何查看 PowerShell 版本号？以下两种方式都可以实现。</p><ul><li>在 PowerShell 里运行<code>$PSVersionTable.PSVersion</code>，Major Chocolatey 的使用</figcaption></figure><h3>软件列表备份</h3><p>想想看，系统重装或迁移之后的软件安装、配置工作，有多麻烦。</p><p><code>choco list -l > chocolatey.txt</code> 可以助你把 Chocolatey 安装的软件导出（导出符 <code>＞</code>后面指的是导出路径，可自行更改，这里表示导出到当前目录的 <code>chocolatey.txt</code> 文件中）。</p><p>拿到导出的软件清单，你就可以在新机上 <code>choco install -y 软件1 软件2</code>。把电脑放一边运行着，该干啥干啥，两不耽误，方便快捷，好不惬意！</p><h3>ChocolateyGUI：「APP 虽然界面确实不好看）。</p><h2>后起之秀：WinGet</h2><p>WinGet（Windows 程序包管理器：Windows Package Manager）是微软为 win10 开发的一款开源的软件包管理器，于 2020年5月的 Microsoft Build 开发者大会上 <a href="https://zh.wikipedia.org/wiki/Windows%E7%A8%8B%E5%BA%8F%E5%8C%85%E7%AE%A1%E7%90%86%E5%99%A8">首宣</a>。</p><p>前提：<strong>Windows 10</strong> 1709 及以上版本</p><p>当前 WinGet 支持的安装程序类型尚不多，除了EXE、MSIX、MSI三种之外，还能够在自定义配置后下载部分微软应用商店的程序。</p><h3>下载、安装和验证</h3><p>你可以在 <a href="https://github.com/microsoft/winget-cli/releases">Github Release</a>下载<code>.appxbundle</code>格式的文件，双击打开并运行（官方 <a href="https://github.com/microsoft/winget-cli">GitHub 主页</a> 和 <a -?</code>（-?可选）；查看特定命令的详细帮助文档：<code>winget [<命令>] -?</code>，如<code>winget install -?</code></li><li>显示软件详细信息：<code>winget show <包名></code></li><li>搜索软件：<code>winget search <包名></code></li><li>安装软件：<code>winget install <包名></code></li></ul><p>是的，如你所见，WinGet 并没有自动开启卸载、升级等功能，这也是其在发布之后很长时间不温不火的重要原因。</p><p>但是不要着急，截止到2021/03/25发布的最新预览版<code>v0.2.10771</code>，WinGet 实验性功能（需要用户自启）的数量已经增加到了八个，其中就包括<code>uninstall</code>、<code>upgrade</code>、<code>list</code>等重要功能，以及支持安装 Microsoft Store 上的应用程序！</p><figure class="image ss-img-wrapper"><img &#125;,

</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/04/07/article/8b5f28d735e4aa808f1fecd372be96d3?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt href="https://scoop.sh/">Scoop</a> 功能更全面、可配置性更高，但由于国内下载速度较慢，所以放在了最后介绍。</p><h3>下载安装</h3><p>前提：</p><ul><li>Windows 7 SP1+ / Windows Server 2008</li><li>PowerShell 5+（include <a href="https://docs.microsoft.com/en-us/powershell/scripting/install/installing-powershell-core-on-windows?view=powershell-6">PowerShell Core</a> and <a href="https://www.microsoft.com/net/download">.NET Framework 4.5</a>）</li></ul><h4>安装</h4><p>管理员身份运行<code>PowerShell</code>，输入以下两条命令，待安装完成之后，输入<code>scoop</code>显示帮助文档即说明安装成功。</p><pre class="language-shell"><code>Set-ExecutionPolicy RemoteSigned -scope CurrentUser
iwr -useb get.scoop.sh | iex</code></pre><p> </p><p>注：Scoop 支持利用<code>aria2</code>进行多线程下载。所以可先<code>scoop install aria2</code>下载 aria2，之后所有的下载任务就均可以调用 aria2 多线程下载来提高速度了。</p><h4>更换下载位置</h4><p>Scoop 默认安装位置<code>C:\User\Your Username\scoop</code>，如果想要更改安装位置，比如<code>D:\Scoop</code>，可以在安装之前，先运行：</p><pre class="language-shell"><code>$env:SCOOP='D:\Scoop'
[Environment]::SetEnvironmentVariable('SCOOP', $env:SCOOP, 'User')

</code></pre><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/04/07/article/2204028a573bcda9ba1e3867ceeeb52b?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2021/04/07/article/2204028a573bcda9ba1e3867ceeeb52b" referrerpolicy="no-referrer"></figure><p>Scoop 会自动将<code>.\Scoop\shims</code>文件添加到系统环境变量，免去 Windows 用户自行添加的麻烦；同时在<code>.\Scoop\apps</code>存放所有已安装的软件，便于统一管理。</p><h3>基本使用</h3><p>刚认识 Scoop就能看出，Scoop 支持的命令更多、可玩性更强~~</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/04/07/article/785a80c707139670f0d78133ae436808?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/2021/04/07/article/785a80c707139670f0d78133ae436808" referrerpolicy="no-referrer"></figure><p>除此之外，与 Chocolatey 和 winget 类似，<code>scoop export > scoop.txt</code>也可导出软件列表，便于备份与换机。</p><h3>添加其他 bucket 软件库</h3><blockquote><p>语法：<code>scoop bucket add [软件源名字] [源地址]</code></p></blockquote><p>bucket 指的是软件仓库，也是Scoop的「内核」了。未进行配置的Scoop默认只会安装<code>main</code>一个bucket，由于官方的严格限制，<code>main</code>里的软件列表不足以涵盖了用户对大多数软件的需求，所以「搜索失败」时常发生，因此自行安装其他 bucket就成了使用 Scoop 的必备。</p><p>bucket 分为官方维护和社区维护两种。</p><p>官方 bucket 软件仓库主要有以下几个（可通过<code>scoop bucket known</code>查看）:</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/04/07/article/a71bc32f55ad3f456caa6409481a0b73?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="已知官方应用软件库 | 图源 Scoop GitHubhttps://github.com/lukesampson/scoop/" data-original="https://cdn.sspai.com/2021/04/07/article/a71bc32f55ad3f456caa6409481a0b73" referrerpolicy="no-referrer"></figure><p>其中最重要、也最推荐的 bucket 是<code>extras</code>（包含了大量的GUI程序）。配合<code>main</code>可满足多数的下载需求了。</p><pre class="language-shell"><code># 官方提供
scoop bucket add extras
scoop bucket add java
</code></pre><p>除了官方 bucket 之外，GitHub 上有众多网友维护的 <a href="https://github.com/rasa/scoop-directory">社区 bucket</a></p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/04/07/article/85c6888f266676023578966fe99b50cc?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt="支持按照不同排序方式来查看" data-original="https://cdn.sspai.com/2021/04/07/article/85c6888f266676023578966fe99b50cc" referrerpolicy="no-referrer"></figure><pre class="language-shell"><code># 社区提供
scoop bucket add echo https://github.com/echoiron/echo-scoop

scoop bucket add dorado https://github.com/chawyehsu/dorado

scoop bucket add dodorz https://github.com/dodorz/scoop
</code></pre><p>一般来说，可谷歌搜索「软件名+Scoop」就可以找到我们想要安装的软件有没有被某个bucket 软件仓库收录。添加上面推荐的三个社区bucket，配合官方的main和extras，应付日常使用已经可以游刃有余了。</p><h3>换源：提高下载速度</h3><p>要改善 Scoop 的下载速度，详细可以参照 <a href="https://gitee.com/squallliu/scoop#install-scoop-to-a-custom-directory-by-changing-scoop">Scoop | Gitee 版</a> 的说明更换下载源。换源之后的Scoop，速度提升不是一星半点儿。</p><ul><li>更换 Scoop 源</li></ul><pre class="language-shell"><code>scoop config SCOOP_REPO https://gitee.com/squallliu/scoop

scoop update
</code></pre><ul><li>更换 bucket 源</li></ul><pre class="language-shell"><code>scoop install git

# 注意：引号里面换成自己的路径，如果是默认路径则为$&#123;Env:USERPROFILE&#125;\scoop\buckets\<bucket_name>

git -C "D:\Scoop\buckets\main" remote set-url origin https://hub.fastgit.org/ScoopInstaller/Main.git

git -C "D:\Scoop\buckets\extras" remote set-url origin https://hub.fastgit.org/lukesampson/scoop-extras.git
</code></pre><h2>结语</h2><p>无需访问搜索引擎苦苦寻觅，无需次次配置系统环境变量，无需点击图形界面、一不小心安装个「全家桶」，一行命令、几个单词，就能满足大多数人对日常软件的所有需求。</p><p>那到底哪个包管理器更好呢？</p><p>软件源丰富、又能做到配置简单的 Chocolatey，支持安装MSStore程序、而又拥有强大卸载功能的winget，用户自定义配置选项灵活多样的Scoop，只能说各有各的优点，配合使用或许效果最佳。</p><h2>关联阅读</h2><ul><li><a href="https://www.cnbeta.com/articles/tech/874537.htm">软件包管理工具选 Scoop 还是
