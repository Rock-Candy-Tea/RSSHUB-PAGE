href="https://www.hostwinds.com/vps/unmanaged-linux" target="_blank">Hostwinds Umanaged Linux VPS</a> 最低配，$4.99/mo，Ubuntu 18.04</li><li>域名：<a href="https://www.namecheap.com/" href="https://yukieyun.net/tech/shared-service-same-server-wordpress/" target="_blank">用 Nginx 反代实现 docker 安装 WordPress 与其他服务并存</a>（by 云五，参考了 90%）、<a href="https://blog.konata.co/?p=1" target="_blank">不只是 Blog——VPS 搭建 WordPress 记录</a>（by 此方）以及 <a href="https://www.digitalocean.com/community/tutorials/how-to-install-wordpress-with-docker-compose" target="_blank">How To Install WordPress With Docker Compose</a>（by DigitalOcean），在这三篇的基础上有少量改动。</p><p style="margin-left:0px;">[对于以上几位作者：如果您认为此文侵犯了您的版权，请联系我，我会立即将此文删除]</p><h4 style="margin-left:0px;"><strong>服务器环境配置</strong></h4><p style="margin-left:0px;">据说<a href="https://security.stackexchange.com/questions/47576/do-simple-linux-servers-really-need-a-non-root-user-for-security-reasons" target="_blank">直接用 root 装网站是一个坏习惯</a>，因此我们要做的第一件事就是用 root 登录，开一个新的有 sudo 权限的用户，之后的所有操作都用这个用户进行：</p><pre class="language-shell"><code>export $UNAME=<username>
useradd $UNAME -m
usermod -s /bin/bash $UNAME
passwd $UNAME
chmod u+w /etc/sudoers
# 在/etc/sudoers的root    ALL=(ALL:ALL) ALL后面添加
# <username> ALL=(ALL:ALL) ALL
chmod u-w /etc/sudoers</code></pre><p style="margin-left:0px;">登出 root，登入新开的这个用户，安装一些部署用的库：</p><pre class="language-shell"><code>sudo apt install docker docker-compose
sudo apt install nginx</code></pre><p style="margin-left:0px;">允许新用户不用 sudo 就能使用 docker：</p><pre class="language-shell"><code>sudo gpasswd -a $USER docker</code></pre><p style="margin-left:0px;">（非必需）<a href="https://www.hostwinds.com/guide/how-to-enable-googles-tcp-bbr-linux-cloud-vps/" target="_blank">开启 Google BBR 拥塞控制算法</a>，以加快网站访问速度：</p><pre class="language-shell"><code>sudo vim /etc/sysctl.conf</code></pre><p style="margin-left:0px;">在文件末尾加上：</p><pre class="language-shell"><code>net.core.default_qdisc=fq
net.ipv4.tcp_congestion_control=bbr</code></pre><p style="margin-left:0px;">然后刷新：</p><pre class="language-shell"><code>sudo sysctl -p
# Test success
sysctl net.ipv4.tcp_congestion_control  
# should output `net.ipv4.tcp_congestion_control = bbr`
</code></pre><h4 style="margin-left:0px;"><strong>在 80 端口安装 Wordpress</strong></h4><p style="margin-left:0px;">开一个目录用于存放 wordpress 配置文件（我放在~/wordpress），并进入它：</p><pre class="language-shell"><code># 我将教程中提到的几个配置文件打包放在一个github仓库里了
git clone https://github.com/maskbot/wordpress.git
cd wordpress</code></pre><p style="margin-left:0px;">1. 将 nginx 配置文件中的域名修改成你自己的：</p><pre class="language-shell"><code>vi nginx-conf/nginx.conf</code></pre><p style="margin-left:0px;">将其中所有的 example.com 都替换成你自己的域名。</p><p style="margin-left:0px;">2. 创建一个环境变量文件（用于设置你的 MySQL 数据库用户名和密码，以及 wordpress 使用的 HTTP 端口）：</p><pre class="language-shell"><code>vim .env</code></pre><p style="margin-left:0px;">.env 的内容如下（自行设置前三项的值，随便设置即可，密码别太弱）：</p><pre class="language-shell"><code>MYSQL_ROOT_PASSWORD=your_mysql_root_password
MYSQL_USER=your_wordpress_database_user
MYSQL_PASSWORD=your_wordpress_database_password
HTTP_PORT=80</code></pre><p style="margin-left:0px;">需要注意的是，.env 文件包含敏感信息，因此一定要避免它被加入 docker container 或上传到 github。在上面 clone 的 github repo 中，我已经将其加入了. dockerignore 和. gitignore。</p><p style="margin-left:0px;">此时你的~/wordpress 目录应该呈现以下结构：</p><pre class="language-"><code>.
├── .dockerignore
├── .env
├── .gitignore
├── docker-compose.yml
└── nginx-conf
    └── nginx.conf</code></pre><p style="margin-left:0px;">最后，启动 wordpress 服务！🚀</p><pre class="language-shell"><code>docker-compose up -d</code></pre><p style="margin-left:0px;">可以用 docker-compose ps 命令检查是否启动成功，如果成功的话，可以看到以下输出（所有 State 都应为 Up）：</p><pre class="language-"><code>  Name                 Command               State          Ports
-------------------------------------------------------------------------

    listen      80;
    listen [::]:80;
    server_name example.com;
    location / &#123;
        proxy_set_header Host $host;
        proxy_pass http://127.0.0.1:8080;
    &#125;
&#125;</code></pre><p style="margin-left:0px;">然后开启这个反向代理 🚀：</p><pre class="language-shell"><code>cd /etc/nginx/sites-enabled/

    listen [::]:80;
    server_name example.com;
    return 301 https://$host$request_uri;
&#125;

    listen 443 ssl;
    server_name example.com;

    ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem; # managed by Certbot
    include /etc/letsencrypt/options-ssl-nginx.conf; # managed by Certbot
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem; # managed by Certbot
    location / &#123;
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    &#125;
&#125;
</code></pre><p style="margin-left:0px;">刷新一下反向代理，现在我们可以用 https://example.com 来访问博客啦！🚀</p><pre class="language-shell"><code>systemctl reload nginx</code></pre><p style="margin-left:0px;">恭喜🎉！ 到这里，wordpress 的安装就结束了。之后的所有操作（改主题、装插件、写博客）都可以通过 wordpress 前端的 WebUI 完成。</p><h3 style="margin-left:0px;"><strong>🎄 博客个性化</strong></h3><h4 style="margin-left:0px;"><strong>数据备份</strong></h4><p style="margin-left:0px;">超好用的免费备份插件：UpdraftPlus。你可以设置它每天自动备份，并将备份文件自动上传到 Google Drive，保留最新的 X 个备份。这个备份文件中包括了你所有的博文、图片、网站主题、插件等等，亲测可以将你的博客一键恢复，哪怕你整个重置了服务器都没问题。</p><h4 style="margin-left:0px;"><strong>外观（主题、颜色、字体等）</strong></h4><p style="margin-left:0px;">博客外观：</p><ul><li>主题：Blocksy，干净好用（外观 > 自定义）</li><li>颜色：我把字体调成了黑的，其他的没动（外观 > 自定义 > 颜色）</li><li>字体：我把代码字体从 monospace 改成了 Roboto Mono，别的没动（外观 > 自定义 > 字体样式）</li><li>摘要：我把博客语言调成了中文（设置 > 常规 > 站点语言），然后把摘要长度调成了 50 字（外观 > 自定义 > 博客文章 > 卡片选项 > 摘要）</li></ul><p style="margin-left:0px;">后台界面：</p><ul><li>隐藏 Wordpress 屏幕上方的那条深蓝色工具栏：Better Admin Bar 插件，在设置里把所有透明度都调成 0%</li><li>后台主题：Kodeo Admin UI 插件，个人认为比较清爽好看，而且首页不拥挤，第一个 widget 默认是「快速草稿」也是一个很大的加分项。</li></ul><h4 style="margin-left:0px;"><strong>侧边工具栏（目录、超链接等）</strong></h4><p style="margin-left:0px;">启用侧边栏：</p><ul><li>首页侧边栏：外观 > 自定义 > 博客文章 > 侧边栏，选为 on</li><li>文章内页侧边栏：外观 > 自定义 > 文章详情页 > 选择有侧边栏的设计</li></ul><p style="margin-left:0px;">侧边栏加入管理菜单：</p><ol><li>进入 widgets 编辑界面：外观 > 小工具</li><li>添加一个自定义 HTML 组件：从左侧选中「自定义 HTML」，将其拖拽到右侧的主边栏中</li><li>在其中添加如下代码（请将 example.com 替换成你自己的域名），会得到一个内容包括 Dashboard、New Post、Draft、Logout 4 项的工具菜单：</li></ol><pre class="language-htmlembedded"><code><div class="ct-widget widget_meta">

对上传图片最大尺寸的限制？</li><li>自动设置文章封面图片：Easy Add Thumbnail 插件，它会检查你的文章中是否至少存在一张图片，如果存在则自动将其设为封面（即 feature image）</li></ul><h4 style="margin-left:0px;"><strong>网站测速和数据统计</strong></h4><ul><li>安装 Site Kit by Google 插件，可以显示你博客的点击量、点击量来源、访问速度等</li><li>测试国内访问你博客的速度：我用过<a href="https://tool.chinaz.com/sitespeed" target="_blank">站长工具</a>和 <a href="https://www.17ce.com/" target="_blank">17ce</a>，都能以地图形式看到你的网站在全国各地的访问速度</li></ul><h3 style="margin-left:0px;"><strong>✍️ 写文章 </strong></h3><p style="margin-left:0px;">最终的多设备编辑 - 更新体验还是比较流畅顺滑的，这也是我在选择建站方案时最看重的一个因素：</p><ul><li>登录后，可以在界面上（文章末尾处）直接点击 “编辑” 按钮进入编辑器修改文章，非常方便。</li><li>草稿箱功能非常实用，可以临时记录灵感，方便日后整理成文。</li><li>每个设备都可以通过网页版编辑器进行编辑和上传，平板和手机可以选择使用 wordpress app</li></ul><p style="margin-left:0px;">毕竟无障碍地写博客才是自己建站的核心目的，如果为了折腾而折腾就显得有些本末倒置了。</p><p style="margin-left:0px;">整个博客搭建好之后只有一个感觉：<strong>wordpress 真香！</strong></p><h2 style="margin-left:0px;">总的来说，上面这堆东西全折腾完，零零碎碎的时间加起来怎么也有两三天吧。中间遇到了不少蛋疼的问题，不过好在最后都解决了 🐶。</p><figure class="image ss-img-wrapper image_resized" style="width:425px;"><img src="https://cdn.sspai.com/2021/05/06/article/0d3045979d9c020be1913cda8bf232e9?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/05/06/article/0d3045979d9c020be1913cda8bf232e9" referrerpolicy="no-referrer"><figcaption>作者此刻的心理状态⬆️</figcaption></figure><h3 style="margin-left:0px;"><strong>内存不足</strong></h3><p style="margin-left:0px;">我的服务器一共 1G 内存，光是 Wordpress 的数据库——MySQL 就占了 700-800G，导致我之后连 zsh 都装不上，一装就炸内存，更别提 RSSHub 之类的了。解决方案是<a href="https://askubuntu.com/questions/1066358/increasing-the-swap-space-on-ubuntu" target="_blank">参考此贴</a>给服务器加了 1G 的 swap 空间（大概就是允许在内存不足的时候把一部分暂时用不到的数据给腾到硬盘上）。使用 free -h（或者 htop）命令可以查看你的服务器是否已经开启了 swap。如果没有（i.e. swap 容量为零），那么你可以通过开启 swap 来获得更多的虚拟内存：</p><pre class="language-shell"><code># 开启1G的swap空间，扩大虚拟内存
sudo fallocate -l 1G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile</code></pre><h3 style="margin-left:0px;"><strong>SSL 超过使用频率限额</strong></h3><p style="margin-left:0px;">是的，免费的 SSL 提供服务 Let’s Encrypt 对每个域名申请 SSL 的频率有限制：<a href="https://letsencrypt.org/docs/rate-limits/" target="_blank">Rate Limits – Let’s Encrypt</a>，每周不能申请超过 50 张证书，每周相同的一组域名（比如 example.com 和 www.example.com 这个<strong>组合</strong>）不能重复申请超过 5 张证书（如果你违背了后面这个 limit，一个简单的 hack 是再加一个二级域名，比如 blog.example.com，然后连带着这个域名一起申请 SSL 证书，这样就不会被视为相同的一组域名了）。如果你在频繁地修改和测试网站，请在 certbot command 里加上<code>--staging</code>选项，测试完没问题了再申请正式 SSL 证书。</p></div><!----></div><div style="border:1px solid transparent;" data-v-e1f0100a></div><div class="article-side sideTop" style="display:none;left:0;" data-v-7be936cf data-v-e1f0100a><div
