<div class="articleWidth-content" data-v-0b37afcb><div class="update-wrap" data-v-0b37afcb></div><div class="content wangEditor-txt minHeight" data-v-0b37afcb><h2 style="margin-left:0px;"><strong>写在前面</strong></h2><p>大家好，我是 Kenshin，简悦的作者，这次跟大家分享我对极简效率工作的一些理解，我接触效率相关的内容，与简悦的诞生有关：</p><blockquote><p>简悦始于 2016 年……那时因为家里的一些事情，我解散了团队，注销了公司。</p><p>曾有段时间特别沮丧，精神无法集中，看什么都看不下去，所以当时的想法就是找一个可以让我产生沉浸式阅读体验的 Chrome 扩展，但找了很多都不满意……</p><p>于是决定自己开发，而另一原因则是：我发现只有沉浸在代码的世界中，我才能集中精神，不再胡思乱想。</p></blockquote><p>故事的最后，我的「毛病」被简悦治愈了。借助简悦，我体会到了极简效率带来的改变，自此开启了我的「极简效率之旅」。</p><h2>极简之「简」</h2><p>我非常不喜欢安装各种 App，再加上越来越强大的电脑性能与大前端的普及，所以一个浏览器几乎可以覆盖全部的工作，因此我的工作环境被压缩为下面几个工具：</p><ul><li>Chrome</li><li>VS Code</li><li>Github + git</li><li>iTerm + oh-my-zsh</li></ul><p>无论是前端、还是后端开发，使用 VS Code 足够了，余下的项目环境相关：Trello、Slack、Notion、Github 等均可使用浏览器替代对应的 App，甚至于大部分的 App 也只是套壳浏览器的 Electron 方案，如：大名鼎鼎的 VS Code，因此更没有安装 App 的理由。</p><p>我的写作工具一直是 <strong>Typora；</strong>聊天方面因为不喜欢使用微信，但有时无法逃离，所以微信也只使用网页版，即时通讯工具仅安装了 <strong>Telegram</strong>，虽然 Telegram 有 Web 版，但无奈视觉效果和性能确实不如原生 Telegram，因此特别安装了 App。</p><p>在相当长的一段时间内，上面几个工具构建了我全部的工作环境。</p><blockquote><p>集中精力、排除杂念和保持「桌面」整洁都很重要，它们能让你专注于当前的工作。</p></blockquote><p>这就是我使用上面这些工具的原因，一直以来，我都沉浸其中，乐此不疲。</p><h2>极简不代表简单</h2><p>大多数对极简效率感兴趣的朋友，大多都看到过下面这幅「教主」的图：</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/2dd7acc86429078ce738ea11dc4f1794?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/2dd7acc86429078ce738ea11dc4f1794" referrerpolicy="no-referrer"></figure><p>不止乔帮主，即便在双链笔记流行的当下，知名效率界名人 Tim Urban 仍旧使用着最简单的记录方式，在接受 <a href="https://www.creatorlab.fm/tim-urban-wait-but-why-podcast/">Creator Lab 播客</a> 采访时的回答时，他说：（节选，原文来自 <a href="https://happyxiao.com/tim-urban/">happy xiao</a> ）</p><ul><li>我用 text editor（文本编辑器），我甚至都不需要 Word 或 Pages</li><li>我使用 Chrome 浏览器，每次复制好引用部分，我就 cmd + tab 找到12个窗口里对应的那个然后扔进去</li><li>我不需要复杂的功能，因为我知道我的东西是什么，它们在哪</li><li>现在我有12个段落，有时我事先就列好了12个段落的提纲，然后把我所有的研究分发进去</li></ul><p>极简的例子比比皆是，让很多喜欢效率的朋友趋之若鹜，但大多数人（包括我）都忽略了一件事情：上面的例子都是各个领域的精英，因此他们做的很多事情并不能被普通人「成功效仿」，这些人不仅使用着最简单且最高效工作方式，甚至很多理论在几十年后仍影响着我们。</p><p>比如现在流行的双链笔记（卡片笔记法）实际发明者 Niklas Luhmann 来自上个世纪，大致的发明时间为 1954 至 1955 年之间，但他的理论却影响了 70 多年后的我们。</p><p>不止如此，金庸前辈在他的《倚天屠龙记》里面也描述过：</p><blockquote><p>四十岁后，不滞于物，草木竹石均可为剑。自此精修，渐进于无剑胜有剑之境。</p></blockquote><p>只有独孤求败可以「草木竹石均可为剑」，如果换到我们呢？拿着「草木竹石仗剑江湖」？</p><h2>极简不是目的，高效才是</h2><p>我们追求极简的目的：更专注，减少外界对我的干扰，最终实现：<strong>高效率</strong>，那什么又是高效：<strong>在相同或更短的时间里完成比其他人更多的任务，而且质量与其他人一样或者更好</strong>（来自百度百科）。</p><p>我没法像 Tim Urban、独孤前辈、Niklas Luhmann 教授一样「草木竹石均可为剑」，所以我的极简并不意味着更高效，而 <strong>提升单位时间的效率，才是高效的必要手段</strong>！</p><p>例如：我每天都要固定做八件事情，每件事情需要四个步骤，每个步骤花费十五分钟，这样每天正好每天八小时。如果将这些步骤简化 / 优化后，每个步骤只要简化为三步，就能节省 15 * 8 共计 120 分钟，也就是在相同条件下，六个小时后，我就可以休息了。</p><p>工欲善其事，必先利其器，选对工具是提升效率的必要手段。</p><h2>放弃纠结，拥抱妥协</h2><p>开发简悦之后，经常能遇到喜欢效率的朋友 / 用户，也会常常听到下面这些讨论：</p><ul><li>Notion、Obsidian、Logseq 我该如何选择？</li><li>印象笔记、Onenote、为知笔记选哪个？</li><li>Typora、Bear、Ulysses 买哪个？</li></ul><p>这些都是典型的 All-in-one 思想导致的，因为极简所以同类产品你「只能使用」一个，孰不知存在即合理，既然同一类别有这么多可选择的方案，则说明每个产品都有它存在的理由。</p><p>如果为了极简而极简则显然丧失了极简的目的：<strong>高效</strong>。</p><ul><li>只选择 Notion 的用户，虽然可以一键登录你全部的数据，但失去了 Obsidian / Logseq 本地应用的高效性。</li><li>只选择印象笔记的用户，虽然得到了强大无比的全平台一键导入的便利性，但失去了「数据所有权」。</li><li>只选择 Typora 的用户，虽然得到了完美的所见即所得，但失去了同步的便利性。</li></ul><p>所以，不应奢望 All-in-one 想法，没有一个工具可以 100% 满足全部用户的需求：</p><blockquote><p>应该允许在你的生活 / 工作中，出现功能相似，但用途不一样的产品；应该接受每一款你使用的产品的瑕疵，只要它能节省你的时间。</p></blockquote><h2>重器轻用</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/00939352c1ff2876f5b569f6eabf046c?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/00939352c1ff2876f5b569f6eabf046c" referrerpolicy="no-referrer"></figure><p>上面这段内容来自少数派对四位效率工具使用者访谈时的一段内容，截图 <a href="https://36kr.com/p/1402159224362632">来源</a>。</p><p>很简单、朴实的一段内容，最终彻底解决了我的困扰：</p><blockquote><p>放弃纠结、拥抱妥协、重器轻用后，你会得到一个真正的高效工作环境。</p></blockquote><h2>增加个浏览器，效率增加不止一倍</h2><p>前文中提到使用一个浏览器「打天下」，在放弃纠结、重器轻用与拒绝 All-in-one 思想趋势下，我获得了更加高效的工作效率。</p><p>利用 Vivaldi 的侧栏功能和标签组功能，将主浏览器常开的网站如： Trello、Slack、Notion、Github、Gmail 全部转移到 Vivaldi 中，并根据 <strong>域名分组</strong>。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/443757a81e3c210d5a13096623f9abe6?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/443757a81e3c210d5a13096623f9abe6" referrerpolicy="no-referrer"></figure><p>实现简洁、高效的标签组效果。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/1d58563948bcd7cfb5e3d9282f72320d?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/19aadb81c7028689ab90eb1311bff8ad?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/19aadb81c7028689ab90eb1311bff8ad" referrerpolicy="no-referrer"></figure><h2>利用侧栏与顶部菜单栏</h2><p>不止增加 Vivaldi 我还使用 Slidepad 将常用的 Slack、Notion 放到侧栏，随时随时记录，尤其是配合 Notion 使用间隔笔记的方式做工作日志。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/c73b727635871ca81605c528ca55873c?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/c73b727635871ca81605c528ca55873c" referrerpolicy="no-referrer"></figure><p>利用 MenubarX 将常用的工具放到顶部菜单栏，包括：</p><ul><li><a href="https://boomb.cn/">Boomb</a> 方便传图到自己的 Github repo 图床</li><li><a href="https://time.is/">TIME.IS</a> 方便查看简悦团队小伙伴各自的本地时间</li><li>简悦的收藏助手，方便从任意 App 拖拽 URL 到此自动生成稍后读和快照</li><li><a href="https://app.simplenote.com/">Simplenote</a> 方便做速记，因为拥有自动保存、Markdown 支持、全平台覆盖的特点</li></ul><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/3c51725a551d8fedbfe90da324c4000b?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/3c51725a551d8fedbfe90da324c4000b" referrerpolicy="no-referrer"></figure><h2>纠结书签工具，不如看看实际使用场景</h2><p>我有跨设备的工作需要，再加上开发遇到问题时，需要查阅和临时使用大量的页面，甚至于晚上在 iPad 上继续查找问题的习惯，但目前暂时没有一个 App 可以完美的符合这个场景，所以我使用了组合方案：OneTab + Slack 的组合方案：</p><ul><li>OneTab 负责一键保存全部标签页</li><li>Slack 负责全设备的跨平台使用</li></ul><p>在有需要的时候，一键将 Tabs 保存到 OneTab 并手动复制给 Slack，而后在 iPad 上继续调查并更新这个列表，有需要的时候导入到 OneTab 中一键全部打开。</p><p>为什么使用 Slack 作为暂存方案而不使用 Raindrop 等书签工具？</p><ul><li>Slack 有着全平台覆盖的优势，免费版足够使用。</li><li>当保存链接后会自动生成链接的 Deeplink。</li><li>同时具有 Thread src="https://cdn.sspai.com/2022/02/21/article/8a3e3f53d34539085a4f8c0228c4cb81?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/8a3e3f53d34539085a4f8c0228c4cb81" referrerpolicy="no-referrer"></figure><h2>笔记的不纠结</h2><p>因为没有 All-in-one 的思维方式，进而可以根据自身需求考虑，所以 Notion、Obsidian、Logseq 我全部都在使用，利用它们自身的优势：</p><ul><li>Notion 多视图功能、协作功能。</li><li>Obsidian 完美的 Markdown 支持，可以作为团队内容 Wiki 使用。</li><li>Logseq 方便的 Publish、以及类似 Roam Research 层级展示方式，非常使用做 Knowledge Garden。</li></ul><h3>不同的场合使用不同的写作工具</h3><p>之前的写作环境是 Typora 中开始。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/aff564c0e34d4cb502418a276344e59a?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/aff564c0e34d4cb502418a276344e59a" referrerpolicy="no-referrer"></figure><p>当文章完成后的校稿阶段，需要可以深度阅读的场合（一般这个场景只发生在咖啡馆）。事先将 Markdown 导入到 Notion，为防止多个版本互相干扰，我一般使用 Comment 来添加备注，在 iPad mini 批阅。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/1000e9c0b0d1cd752614f279e20fa8cf?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/1000e9c0b0d1cd752614f279e20fa8cf" referrerpolicy="no-referrer"></figure><h3>Notion 用来辅助 Trello 做简单的子项目处理</h3><p>我是 Notion 深度使用者，一直在使用 Notion 辅助 Trello 来管理简悦的全部项目，但我从不使用 Notion 做「仓储管理」。</p><p>以下就是利用 Notion 视图、协作优势的一些使用场景。</p><p>使用 Timeline 视图，直观查看简悦每个子项目的时间。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/b41bc5ca257bb55683681549f0e018a3?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/b41bc5ca257bb55683681549f0e018a3" referrerpolicy="no-referrer"></figure><p>使用 Board 视图，作为简悦每个功能发布产品动态一览，从某种程度上也是对简悦团队的激励。</p><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/9994be4f5257697144ca709464cf82cf?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/article/9994be4f5257697144ca709464cf82cf" referrerpolicy="no-referrer"></figure><p>使用协作方式，将需要对外宣传的文章进行校对。</p><p>关于我在 Notion 的更多细节，可以看少数派的 <a href="https://kb.simpread.pro/">简悦教程</a>。</p><ul><li>在 Logseq 客户端编写教程</li><li>提交到 Github，利用 <a href="https://pengx17.github.io/knowledge-garden/#/page/logseq%20publish%20github%20action">Logseq Publish GitHub Action</a> 自动发布它。</li></ul><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/article/631f52fd70a073184bc99e4e969c3518?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" href="https://zhuanlan.zhihu.com/p/467192292">简悦 + Logseq + Github Page 无代码全自动化知识管理发布方案</a>。</p><h2>串联多个产品背后的粘合剂</h2><p>无论是锤子便签、SimpleNote、Notion、Typora 还是 Logseq、Obsidian 能让多个产品中的内容方便流转的原因，它们的背后都有默默工作的简悦：</p><ul><li>锤子便签（富文本）可以方便的导出 Markdown</li><li>Notion、Obsidian、Typora 互相转换</li><li>多个浏览器共享一套稍后读方案</li></ul><p>这些都是简悦的功能，也是 2.0 版的目标：<strong>做各种生产力工具的粘合剂</strong>，但简悦不可能是万能的，所以我在选用这些工具时有个标准：</p><ul><li>支持 Markdown</li><li>支持 Web 端，当然全端更好</li><li>方便的导入、导出</li><li>支持公有文本格式、API</li></ul><p>只要是网页就能使用简悦的阅读模式，如：锤子便签、Notion 均是如此；只要支持 Markdown 与 API，就能方便地导入 / 导出；使用公用格式就可以利用本地检索工具，进行全文检索，如 DEVONthink。</p><p>虽然，看起使用的工具很多，但只是共享了「一份数据」而已。</p><h2>更少的操作就是更高效？</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/bd14ae21ce66ef8f727ca5d298ea03ee.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/bd14ae21ce66ef8f727ca5d298ea03ee.jpg" referrerpolicy="no-referrer"></figure><p>「效率圈」一直在流行这样一个观点：用更少的工具串联你的工作流。这个思路看起没什么问题，工具的选择越少越好，甚至于越简单越好，如：</p><ul><li>选择线上而非 Local first 产品</li><li>全平台方案</li><li>一键登录</li></ul><p>试想一种所谓极简方案：</p><ul><li>使用手机号，甚至于仅扫码微信即可登录（最差也得用 Email 登录）</li><li>在不同的平台安装对应的 App</li><li>登录后会得到你的数据</li><li>方便的一键导入，哪怕是微信公众号也能容易导入</li></ul><p>我也曾这么理解，几年前一度纠结于信息的线上管理，因而辗转在印象笔记、Onenote 中，直到开发了简悦后再到现在流行的「A Text Renaissance」才发现，一切的线上信息保存都是不可取的，原因在于：</p><ul><li>线上存储只不过将信息从 服务器 A → 服务器 B，无论是 A 还是 B 都不是你自己的「地盘」，你只是租用了某个线上存储服务的空间而已。</li><li>在线快照最大的问题是「成本」：用户增多 → 成本指数增加（如带宽成本、容量成本等）→ 涨用户年费，这是唯一的一条路，印象笔记的 VVIP 就是最好的证据。</li></ul><p>基于此，市面上的几乎全部的存储服务都具有：易导入但难导出的特点，原因也很简单：</p><ul><li>只要是线上存储，一键导出全部数据是需要付出带宽成本的， 导入数据虽然也是要成本，但这是它的本职工作，导出却不是。</li><li>越没法离开就越离不开，最终只能将用户绑在同一服务上。</li></ul><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/11/17/365ff29b5ab274691d99b1417912aade.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/11/17/365ff29b5ab274691d99b1417912aade.png" referrerpolicy="no-referrer"></figure><p>我在 <a href="https://sspai.com/post/69972">本地存储 + 线上获取：我的个人数据库建构路径</a> 已经非常明确的阐述了这个「问题」，而这导致了另外一个问题：因为数据是线上的，你没法使用便捷的本地全流程自动化，如：</p><ul><li>在网页标注后，自动导入到 Obsidan / Logseq</li><li>通过 DEVONthink 检索刚刚导入的本地数据内容</li><li>你的标注 / 稍后读数据保存到坚果云 / Dropbox 甚至支持任意网盘</li><li>通过 Alfred / Raycast / uTools 等一切快捷检索工具简单到你刚刚保存的稍后读数据</li></ul><p>而基于 Local first 概念的简悦，可以完全做到这点，正因如此：我才能方便地使用简悦作为 Obsidian / Logseq 等产品的粘合剂，不仅如此，简悦还支持多种线上保存方案，并包含了快照功能：</p><ul><li>利用 Telegram bot 可全设备的保存导入的稍后读</li><li>支持 Pocket / Instapaper / Raindrop / Pinbox / Inoreader 的导入</li><li>支持 IFTTT / Zapier / integromat / integrately 各种自动化方案</li></ul><p>虽然实现上述方式需要配置，但配置只是一时而已，配置完后，就可以拥有一整套的信息归于一处的方案，并且是你「自己的地盘」。</p><h2>少即是多，多即是少</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/f15255560198017ce08e9a59d446c6ae.jpg?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/f15255560198017ce08e9a59d446c6ae.jpg" referrerpolicy="no-referrer"></figure><p>诚然我的工具虽然增加了：</p><ul><li>常用浏览器变成了两个：Chrome + Vivaldi</li><li>书签管理变成了：Slack + OneTab 结合方式</li><li>同时使用 Notion、Obsidian 与 Logseq 作为笔记工具</li><li>需要配置的简悦作为知识管理的粘合剂</li></ul><p>但效率却增加了不止一个量级：</p><ul><li>虽然使用了两个浏览器，但减少了常用 Tab 反复切换 / 开启 / 关闭的时间，解决了常用服务多开 Tab 的问题</li><li>虽然使用了两个工具，但增加了一键保存，全设备同步，方便检索的便利性</li><li>虽然多了配置，但获得了全部设备归于你的地盘的信息收集 / 处理方案</li></ul><p>这就是所谓「退一步，海阔天空」最好的诠释吧。</p><h2>补充</h2><p>因篇幅有限不仅使用上述内容，还包括：</p><ul><li>使用 VS Code - insider 作为增强文本编辑器（辅助 Typora、Logseq 编写 Markdown）</li><li>使用 Typefully 作为写作工具，用于不同的写作场景</li><li>使用 <a href="https://fig.io/">Fig</a> 增强在 iTerm 的自动补全功能</li><li>Typora + <a href="https://github.com/pluveto/upgit">Upgit</a> + Github repo 一键传图到自己的地盘</li></ul><p>等等，我的工具们比原来的多了不止一倍，但效率的提升也不止一倍。</p><h2>最后</h2><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2022/02/21/e205dd8ddc3f6c7458889f6583558431.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2022/02/21/e205dd8ddc3f6c7458889f6583558431.png" referrerpolicy="no-referrer"></figure><p>复杂都是相对的，你认为的复杂可能是他人的利器，反之亦然。只要能解决你的问题、节省你的时间就是简单的，适合自己的就是好的。</p><p>存在即合理，即便是同类型产品，有些是为了解决简单需求的，所以它简单；有些是为了解决复杂需求的，所以它专业；有些需要配置，是为让你更好的使用；有些没有配置，只为让你放弃「自由」。</p><blockquote><p>放弃纠结拥抱妥协，重器轻用才是我的高效率！</p></blockquote><p>本文配图来自：</p><ul><li><a
