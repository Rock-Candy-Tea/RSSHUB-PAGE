<div class="articleWidth-content" data-v-6a669db8><div class="content wangEditor-txt minHeight" data-v-6a669db8><p>自从 Vision 2.0 上线后，这是第四次在少数派写文章了。跟前三篇都是安利文不同，这次我想聊聊我的一些感触，关于 OKR，关于独立开发，也关于我自己。</p><h2>一些不堪回首的历史</h2><p>翻开我的 QQ 空间日志，还能看到初中时想要“打败”腾讯，高中时还想做像乔布斯一样的人这样的字眼。然而怀揣这么多梦想、雄心壮志的我，最终没有能够渡过高三这个难关，选择了退学。因为我没有办法在对除了英语和数学的其他科目产生任何兴趣的情况下坚持下去，可能没有人能理解我当时的心理，那是我人生中第一次崩溃。除了面试过我的公司领导以及几个特别好的朋友知道我的学历，我从来不敢在任何场合聊起我的这段经历。即使到现在，我只要听到别人在聊大学生活、专业之类的话题，我都会赶紧逃走，或者转移话题。那真是一段噩梦。</p><p>退学以后，我沉迷了《星际争霸2》这款游戏。我花了大量的时间去练习这款竞技性非常高的游戏，加入了一个业余战队。由于我在高一的时候折腾了一段时间网络建站，我当时主动跟战队队长提议说，我想给战队建一个论坛。我记得是用了<i>discuz！</i>这个开源的建站工具，它提供了图形化的安装界面，所以在我当时不懂任何编程的情况下，也能很轻松地搞定它。</p><p>有一天，因为星际2的一场关注度非常高的赛事要开打了（我记得是 GSL 决赛 Soulkey vs Innovation），我特别想在论坛上搞一个竞猜的功能。然而并没有找到能用的插件，于是在战队里一个程序员的帮助下，我成功地用 PHP 和 HTML 写了这个插件。这是我第一次接触代码。</p><p>逐渐地，我开始对写代码有了兴趣。后来我买了 PHP 编程语言的书，因为我想用 iPad 看，而只有英文版可以买到正版的 PDF，所以我从一开始学编程都是看的英文教材。那时微信公众号刚开始兴起，而美团外卖、饿了么还没有出现的时候，我在我家当地的大学旁边，做起了微信公众号在线点餐的创业项目，而后台正是我用刚学的 PHP 写的。不过，美团饿了么很快通过补贴抢走了我们的生意。</p><p>我一直以为 JavaScript 比 PHP 要难很多，所以一直不敢去学。但因为我还是想要做网站，所以不得不接触它。我在网上找到了盗版的英文版的犀牛书 <i>JavaScript: The Definitive Guide</i>，但最终能让我熟练起来的还是不断地去写代码。</p><p>直到 2016 年，我才真正打算去外面找找工作，没想到第一份工作很快就找到了，一家刚刚起步的创业公司，我是唯一的 Web 程序员，工作上并没有感觉到任何挑战，但感觉项目并没有什么前景，只做了 3 个月就辞职了。</p><p>后来，我又沉迷打羽毛球，然后又基于公众号做了羽毛球相关的 Web 应用，比如用手机给比赛双方记分，并记入历史战绩。</p><p>2017 年初，我入职了第二家公司，做羽毛球自媒体和互联网产品。那一年微信小程序刚面世，而做一款自己的微信小程序，又成为了我心心念念的事。那年年底，<i>羽易羽毛球</i>诞生了，它是一款用来给羽毛球爱好者报名俱乐部打球用的小程序。虽然它在使用体验上比起当时的竞争对手好很多，但由于我们无法推广出去，更无法盈利，最终并没有坚持下去。</p><p>于是，我又回到了打工人的岗位上。那两年时间，我沉迷写代码中，学习各种技术，主动加班，甚至帮同事改代码。当时就想得很简单了，在 Web 这个领域深挖下去，去做开源，或者做更加复杂的产品。直到去年下半年，我的工作突然变得清闲，让我这种不习惯躺平，有很强危机感的陷入了迷茫。我想在公司里尝试新的技术，但由于公司要求的是稳定、能用就行，逐渐我也失去了兴趣。</p><h2>OKR 让我走出了当时的困境</h2><p>那段时间，公司引入了 OKR，作为没有任何自主性的一线员工，我并没有机会参与公司的 OKR。但 OKR 是对全公司成员公开的，我还是有机会去看他们到底制定了啥。使命、愿景、战略，再到每个周期的 OKR，我也大致了解了公司的方向。对我来说也并不重要，因为我只需要做被布置的任务就行了。但我对 OKR 这个方法论突然产生了兴趣。一个周末，我问一个当时关系非常好的管理层的同事，OKR 对个人有没有帮助，她回答说有。于是我开始疯狂地谷歌有关个人 OKR 的资料，并发现了不少有用的信息。</p><p>因为我发现在制定 OKR 之前，一定要想出自己的<strong>愿景</strong>。我的愿景，当然是未来五年我的职业发展路线。于是我分析了诸如：</p><ul><li>研究算法和编译器</li><li>机器学习</li><li>Swift 与 iOS 编程以及独立开发</li></ul><p>我结合自身的能力和兴趣，最终选择了 iOS 编程。因为</p><ul><li>我一直以来都是做客户端、UI 编程的相关工作</li><li>我受到很多独立开发者独立和工作地点自由的启发和诱惑</li><li>我也是一个从高中时期就开始的果粉，不做 iOS 开发者太遗憾了</li><li>还是想要做自己的产品，那颗想要创业的心又回来了</li></ul><p>而终于，在去年十月一日，在与一个前同事聊完天后，她激发了我制定人生中第一次 OKR 的决心。于是，整个国庆节假期，我都在思考我的 OKR。</p><p>而接下来的故事大家也很容易猜到了，为了转型 iOS 开发的第一个 OKR，然后诞生了 Vision.</p><span class="ss-linkCard" link-card-href="https://sspai.com/post/67497"> </span><p> </p><h2>OKR 的魅力</h2><p>看完上面的故事，大家可能对 OKR 的魅力有一点认识了。它不是一个对具体任务的管理系统，不是大家常见的 To Do 类的 app，它是一种思考问题的方式，是一种精神，是驱动你走上你最想要去的地方的路线图，它可能是抽象的，它是你的人生的指导性的方针，它是你的决策层。</p><p>而 Vision 正是想要帮助用户，像我当时一样，去找到自己通往自己想要去的地方的路线图。而你要做的任何具体的事，都是根据这张路线图来的。而你要想计划好具体做的事，没有路线图，就像是失去了方向。</p><p>关于 OKR，我在 Vision 的<a href="https://okr.vision/zh/docs/okr-guide/okr-introduction" target="_blank">官方文档</a>进行了详细的解释，请有兴趣的读者一定要去看一看。</p><h2>Vision 2.4 后的困境和思考</h2><p>自从 2.3 我们上线专业版后过去了 3 周时间，我们上线了 2.4，同时也上线了官方网站和文档。我们本以为在这个版本中对 OKR 中的关键结果进行了深度的优化，应该有一些好的反应，然而我们才发现，我们很多用户似乎是期待着我们去做“任务”的相关功能，他们想要日程表、日历，对具体的事的规划的更丰富的功能。我们意识到他们并没有把 Vision 当作 OKR 工具来使用，而是又一个 To Do 类的任务管理工具。殊不知，OKR 和任务管理是两个完全独立的功能。</p><p>很显然，应用市场中 To Do 类的工具已经过于饱和，我们没有必要在我们核心功能没有开发完成的情况下进入这个领域。反而意识到：如果我们并不能把<strong> </strong>OKR<strong> </strong>的方法论普及给我们的用户，那么就算 Vision 再多的用户，都不是真正使用 OKR 的用户，在 Vision 始终坚持做 OKR 的工具的前提下，那么这些用户就不是我们真正的用户 —— 从长远来看，他们无法从 Vision 的功能中获得价值 —— 他们因此会抛弃 Vision 转向其他 app.</p><p>这样分析一来，现阶段我们最重要的功能可能就是去做普及 OKR 的工作。可能是通过生产内容，搬运内容，甚至是把我们的网站打造成一个分享 OKR 和它的故事的平台。</p><p>今天我观看了好几个跟 OKR 相关的视频，包括 OKR 界最著名的布道者，John Doerr 的 Ted 演讲和他的采访。无一例外地通过他经历的活生生的例子来讲述 OKR 是如何帮助人们突破自己、创造价值的。这些鲜活的例子让我感觉到 OKR 其实离自己很近，甚至感觉突破自己也没有想象中那么遥远，即使他举的例都是一些行业大佬，如谷歌创始人，U2 乐队的主唱等等。而他的网站 <a href="http://whatmatters.com/" target="_blank">WhatMatters.com</a> 上记录了许多各行各业的普通人是如何用 OKR 改变他们的行业，创造了价值，让我感觉到<strong> </strong>Vision<strong> </strong>是否真正的价值是否应该在传播这种普通人通过<strong> </strong>OKR，通过<strong> </strong>Vision<strong> </strong>来改变他们自己的故事，而不仅仅是工具本身。</p><h2>如果我在高中时就会使用 OKR</h2><p>那么一定会有一个“考上一所计算机科学厉害的大学”作为目标，那么一定是基于分数线量化的关键结果。虽然我会很讨厌那些科目，但一定认为这个目标是正确的，那么我会咬着牙完成这些关键结果。</p><p>由于没有后悔药可以吃，我必须接受这些现实。但自从理解 OKR 以后，我认为即使在现在，我也能通过使用 OKR 的方法，学习我任何想学的东西，做任何想做的事。</p><h2>Vision 的使命是帮助更多人找准并实现自己的目标</h2><p>虽然 Vision 是我多次创业以来第一次有真正收入的项目，但我并不认为赚钱是做 Vision 的最终使命，它承载了太多我想要实现和表达的内容。想到这点，我终于解决了 2.4 上线以来不好看的数据带来的万分焦虑，并找到了真正的方向 —— 通过我们开发的工具，以及生产的内容去传播和普及 OKR，帮助那些有梦想，但又被现实世界迷失方向的人们找到属于自己的路线图。</p><p>我们会继续开发和优化 OKR 相关功能，我们会去认真研究用户使用 Vision 的工作流，并优化这个工作流。工具应该给用户带来价值，而不是负担。</p><p>同时，我们会通过生产内容，分享不同场景的 OKR 周期，制作易懂的教程。</p><span class="ss-application" app-id="16122"> </span><p>我的邮箱：grepug@icloud.com</p><p>这是我们的网站：<a href="https://okr.vision/zh" target="_blank">https://okr.vision</a></p><p>更多联系方式：<a
