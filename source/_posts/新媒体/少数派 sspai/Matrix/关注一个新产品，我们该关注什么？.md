data-v-d534f82e><p>过去的两年里，由于国内外的疫情，优秀的产品层出不穷。眼花缭乱的软件中，我们如何找到真正能帮到自己的软件呢？</p><figure class="image ss-img-wrapper image_resized" style="width:460px;"><img src="https://cdn.sspai.com/editor/u_/c7cel4db34t8pteg8hj0.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c7cel4db34t8pteg8hj0.png" referrerpolicy="no-referrer"><figcaption>一口气随便写的软件表列</figcaption></figure><h2>发现</h2><h3>Product Hunt</h3><p><a href="https://www.producthunt.com/" target="_blank">Product Hunt</a> 是一个供用户分享和发现产品的网站，有类似 <a href="https://zh.wikipedia.org/wiki/Hacker_News" target="_blank">Hacker News</a> 或 <a href="https://zh.wikipedia.org/wiki/Reddit" target="_blank">Reddit</a> 的留言系统。在一天中取得最高票的产品在当天的列表中排序最高，便于我们发现热门且高质量的最新产品。</p><p>如果你想要快人一步发现最新的软件、工具的话，Product Hunt 是肯定不会让你失望。</p><figure class="image ss-img-wrapper image_resized" style="width:467px;"><img src="https://cdn.sspai.com/editor/u_/c7cel4lb34t8pteg8hjg.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c7cel4lb34t8pteg8hjg.png" referrerpolicy="no-referrer"></figure><p>我们的时间是宝贵的，不必每天浏览网站，你可以订阅 <a href="https://www.producthunt.com/newsletter" target="_blank">Product Hunt Daily</a> 来获取每周的最热门产品和产品精选集的资讯。</p><h3>GitHub Trending</h3><p>GitHub 想来大家都不陌生，其实有些好用的工具并没有官网，而是开源在 GitHub ，或者在 GitHub 上有不少 Star，我们就可以 GitHub 的 Explore 里面找到 Trending 面板。</p><figure class="image ss-img-wrapper image_resized" style="width:475px;"><img src="https://cdn.sspai.com/editor/u_/c7cel4tb34t8pteg8hk0.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c7cel4tb34t8pteg8hk0.png" referrerpolicy="no-referrer"></figure><p><a href="https://github.com/trending" target="_blank">Github Trending</a> 显示了项目每日、每周和每月被用户加星的排名，如果你经常逛 Github，不妨找找有什么令人兴奋的开源工具吧！</p><h3>NoteApps.info</h3><p>如果你对新的笔记软件有兴趣，却不知道从哪里下手，可以去浏览一下 <a href="https://www.noteapps.info/" target="_blank">NoteApps.info</a> 这个网站，它就像一个笔记应用的维基百科，把市面上热门的笔记软件和他们的功能全部囊括其中，进入软件的词条，它的功能、支持系统一目了然。</p><figure class="image ss-img-wrapper image_resized" style="width:495px;"><img src="https://cdn.sspai.com/editor/u_/c7cel55b34t8ob5d73o0.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c7cel55b34t8ob5d73o0.png" referrerpolicy="no-referrer"></figure><p>更厉害的是，NoteApps.info 也统计了笔记软件支持插入的格式，例如是否可以插入文件、PDF、表格或者 Twitter，还统计了笔记的反馈渠道和社群等等。</p><p>可以说这个网站囊括笔记软件的方方面面，刚开始选软件的朋友值得参考，选中了之后就可以仔细研究，短时间可以不用换了。</p><h3>Newsletter</h3><p>这里推荐一个我个人很喜欢的推荐产品的 Newsletter，它的名字叫 <a href="https://itsprobablynothing.substack.com/" target="_blank">Probably Nothing</a>。</p><figure class="image ss-img-wrapper image_resized" style="width:499px;"><img src="https://cdn.sspai.com/editor/u_/c7cel5db34t8ob5d73og.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c7cel5db34t8ob5d73og.png" referrerpolicy="no-referrer"><figcaption>Probably Nothing 的介绍页面</figcaption></figure><p>推荐产品的质量都很不错，而且每期 Newsletter 都会有产品的演示和亮点，从这个 Newsletter，我已经发现了好几个我现在都在深度使用的产品了。</p><h2>入门</h2><h3>官方文档</h3><p>大部分成熟的软件都有自己的官网，我们可以直接从软件的官网通过视频、功能介绍、博客等获取基本信息。</p><p>对于一个比较复杂的软件，很多官网上会有一个官方文档，会教你如何快速上手一个工具，也会让你对软件的功能一目了然。比如说 Obsidian 专门创建了一个库来让你直接在软件里尝试操作，很快我们就会快速熟悉软件的功能 (Obsidian 其实是一个相当复杂、难上手的笔记软件了)。</p><figure class="image ss-img-wrapper image_resized" style="width:519px;"><img src="https://cdn.sspai.com/editor/u_/c7cel5lb34t8pnofe0mg.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/editor/u_/c7cel5lb34t8pnofe0mg.png" referrerpolicy="no-referrer"><figcaption>Obsidian 的官方帮助库</figcaption></figure><p>通过官网，我们还可以进入软件的官方社区和一些讨论组来进一步给官方提出功能、BUG，也可以和软件的用户一起交流软件的深度用法。</p><h3>Discord、Slack</h3><p>关于国外软件官方的社群，大部分聚集在 Discord 上，也有少数在 Slack 上。</p><p>得益于 Discord 和 Slack 给每个聊天分主题的特性，软件开发者大多把主题分成这几个部分：</p><ul><li>Bug-report (幺蛾子报告)</li><li>Feature-request (新整活需求)</li><li>Use-cases (大家有什么新活)</li></ul><p>除开这些基本的，开发者也会设置与软件无关的一些主题：</p><ul><li>Introduce-yourself (介绍你自己)</li><li>General (普通讨论组)</li><li>Tool-for-thought (软件、工具推荐）</li><li>Article-sharing (文章推荐)</li><li>Workflows (工作流推荐)</li></ul><p>以上这些主题，尤其是后面三个，真的给了我很多灵感和想法，也让我发现了不少好用的工具。</p><p><strong>如何进入类似的官方社群呢？</strong></p><p>大部分官方社群都会在软件的官网展示出来，只要点击链接就可以直接进入社群。不要忘了自我介绍哦 —— 这是快速融入社群的好方法。</p><h3>微信群</h3><p>国内软件的社群大多聚集在微信，例如 flomo、logseq 的官方微信群，可以直接和开发者提建议和需求。</p><p>一个很大的优势就是和国内优秀的使用者的交流也更加便捷，交流群里的朋友们也会分享不少很有启发的使用心得。甚至有些群友也开始了线下的合作，我认为这是一个很好的趋势。通过社群来创造更紧密地连接。</p><h3>视频平台和论坛</h3><p>随着国内更多优秀软件的出现，哔哩哔哩上优秀的推荐文章、视频教程也日益增多，虽然这些教程时效性不高，但是创作者们还是有很多本地化的有趣想法，也会根据自己的例子，在中文语境的情况下给出更合适的解决方案。</p><p>若是有些教程 B 站找不到，Youtube 是另一个很好的解决方案，而且教程较新，配合中文字幕食用也可以解决语言问题。</p><p>知乎和 Reddit 也都有各个软件的讨论组，里面不乏优秀的入门文章和如何把工具发挥完美的讨论，创作者通过生动的图文教程和深刻的思考，来让一个软件的可能性变得很大。</p><h2>追踪新功能</h2><h3>官方社区</h3><p>在下面的图片里，展示的就是双链笔记软件 Obsidian 和 Logseq 的官方社区，较为活跃。从官方的社区里，刚开始使用的初学者就可以看看分享来上手一些较为复杂的软件，如果使用中遇到问题，也可以提需求、反馈 Bug。</p><figure class="ss-imgRows" figcaption><img src="https://cdn.sspai.com/editor/u_/c7cel5tb34t8phluhki0.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/editor/u_/c7cel5tb34t8phluhki0.png" referrerpolicy="no-referrer"><img src="https://cdn.sspai.com/editor/u_/c7cel65b34t8phluhkig.png?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" alt data-original="https://cdn.sspai.com/editor/u_/c7cel65b34t8phluhkig.png" referrerpolicy="no-referrer"></figure><p>官方也会把版本更新的公告，同步到论坛里，如果没有关注官方的媒体或者社群，从这里可以很快了解更新内容，并展开讨论。</p><h3>官方社交媒体</h3><p>很多软件开发者或者软件官方都有自己的社交媒体，他们毫无疑问会在社交媒体上发布自己的更新进展。类似开发者常驻的平台就有 Twitter、即刻、微信公众号当然还有少数派。</p><p>少数派上开发者有：简悦开发者 <a href="https://sspai.com/u/kenshin/updates" target="_blank">@Kenshin</a>、白描开发者 <a href="https://sspai.com/u/wf1e9oo4/updates" target="_blank">@Tolecen</a>、Flomo 开发者 <a href="https://sspai.com/u/plidezus/updates" target="_blank">@少楠Plidezus</a> 等等，大家可能都比较熟悉。</p><h2>后记</h2><p>介绍了那么多关注、上手新产品的渠道，其实我心里是不安的，我害怕读者陷入一种唯工具论的陷阱里面无法自拔。</p><p>我确信工具有好的，也有不那么好的，但是其实只要能帮助你达成目标的就是好工具。我写这篇文章的初衷就是想帮你找到最能帮你解决问题的产品，并且告诉你怎么更好的使用它。</p><p>有些朋友沉迷于把很多工具组合起来，形成一个特别复杂的工作流，其实这样会增加系统的摩擦力，反而适得其反。我认为我们需要把工作流控制在一个适当的范围里，避免成为工具的奴隶。这也是我在写完上一篇文章后得出的反思。</p><p> </p><p>延伸阅读：<a href="https://sspai.com/post/70641" target="_blank">2021 —— 我的工作流定型过程</a></p><p> </p></div><div class="update-details-wrap" data-v-d534f82e></div><!----></div><div style="border:1px solid
