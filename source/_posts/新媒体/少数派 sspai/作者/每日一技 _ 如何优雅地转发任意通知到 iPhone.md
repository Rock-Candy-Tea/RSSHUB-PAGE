是少数派的写作社区，我们主张分享真实的产品体验，有实用价值的经验与思考。我们会不定期挑选 Matrix 最优质的文章，展示来自用户的最真实的体验和观点。</p><p>文章代表作者个人观点，少数派仅对标题和排版略作修改。</p><hr><blockquote><p>优雅的实现自动化，是每个 Geeker 应有的能力。----我编的</p></blockquote><p>工作需求，目前双持 iPhone 和 Android (工作用)设备。有需求将 Android 设备的某些通知转发到 iPhone上。</p><p>短信里的验证码，微信群聊中的 @xxx，来电通知，我全都想要转发。一方面是想在玩 iPhone 的时候，不错过一些工作通知，另一方面想让 Android 的通知走 Apple Watch 推送，这样就能更及时的看到消息了。</p><p>尝试过《小转发》等优秀的转发软件，但觉得设置过于复杂，不直观。偶然之间，发现了可以优雅实现转发的方法，下面请允许我娓娓道来~</p><h2>前情提要</h2><h3>设备</h3><p>iPhone + Android 设备</p><h3>应用服务</h3><ol><li><strong>Bark</strong> => iOS端自定义接收通知的工具</li><li><strong>通知滤盒</strong> => 版本号 2.x.x ，付费应用，可能需要高级版</li></ol><span class="ss-linkCard" link-card-href="https://sspai.com/post/53090"> </span><span class="ss-linkCard" link-card-href="https://sspai.com/post/59502"> </span><span class="ss-linkCard" link-card-href="https://sspai.com/post/60536"> </span><h2>实现思路</h2><blockquote><p>通知滤盒根据自定义规则，拦截指定通知；</p><p>将标题内容提取，再拼接到 Bark 的 URL 中，发送 Webhook 请求；</p><p>即可优雅的实现自定消息转发。</p></blockquote><p><strong>以下，以转发微信中所有 「@我」的通知举例。</strong></p><h3>打开新增 Webhook 规则页面</h3><ol><li>在「规则」页面中，找到「增强」区块。</li><li>轻点「Webhook」卡片。</li><li>轻点「+」按钮以新增规则。</li></ol><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/08/09/bee28f13158047a275c783d79ca9d25b.JPEG?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/08/09/bee28f13158047a275c783d79ca9d25b.JPEG" referrerpolicy="no-referrer"><figcaption>新增「增强」规则</figcaption></figure><h3>设置转发规则</h3><ol><li>命名规则 => Hi,有人找你（这个自行定义即可）</li><li>选择 App => 选择「微信」</li><li>选择内容 => 选择「包含任一文本」</li><li>填写匹配规则 => 「@我」、「@所有人」</li><li>根据需求选择时间段 => 此处我选择所有时间（如果在晚上转发，会被 iPhone 上的勿扰拦截掉）</li><li>根据需求选择手机状态 => 我个人保持初始值</li><li>填写 URL 值 => 请打开 Bark 获取请求 URL</li><li>选择请求方式 => Bark 支持 POST 和 GET 方法</li><li>保存规则</li></ol><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/08/09/b58300cfcee50c22b9a4e43e844d1df3.JPG?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/08/09/b58300cfcee50c22b9a4e43e844d1df3.JPG" referrerpolicy="no-referrer"><figcaption>规则详情</figcaption></figure><h3>测试</h3><ol><li>拖动页面到底部，轻点「模拟测试」；</li><li>展开区块中，轻点「测试」按钮；</li><li>通知滤盒将会列举出符合条件的所有通知，和所预期的做对比；</li><li>如果符合条件~ 恭喜你，已经优雅的设置好了转发；</li><li>现在找个号，@自己一下试试。</li><li>如果一切正常，现在您可以在 iPhone 上收到这条消息啦！</li></ol><figure class="image ss-img-wrapper"><img src="https://cdn.sspai.com/2021/08/09/742bc8c9663a7fe0803af4ea636ab5cd.JPEG?imageView2/2/w/1120/q/40/interlace/1/ignore-error/1" data-original="https://cdn.sspai.com/2021/08/09/742bc8c9663a7fe0803af4ea636ab5cd.JPEG" referrerpolicy="no-referrer"><figcaption>验收成果～</figcaption></figure><h2>扩展</h2><h3>转发消息到另一个微信</h3><ul><li>通过「Server 酱」转发；</li><li>通过「WxPusher」转发；</li></ul><h3>添加到 Flomo 笔记</h3><ul><li>替换为 Flomo 的新增笔记 URL 即可；</li></ul><h3>添加到 GTD 中</h3><p>例如 Todoist，转发 URL 为 https://api.todoist.com/rest/v1/projects/xxxxx 即可，接口详情请参考 <a href="https://developer.todoist.com/sync/v8/#overview" target="_blank">此页面</a>。</p><h3>转发信息到其他 Webhook</h3><ul><li>可以自己起一个后台服务，做更硬核的事情；</li><li>也可以转发到 IFTTT ，联动更多设备。</li></ul><h2>尾巴</h2><p>预祝各位 Geeker 们玩的愉快😉</p><span class="ss-application"
