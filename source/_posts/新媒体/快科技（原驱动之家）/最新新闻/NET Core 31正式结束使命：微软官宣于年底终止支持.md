<p>作为微软推出，且免费开源的软件框架，.NET Core有着不少的受众用户。</p>
<p>近日，<span style="color:#ff0000;"><strong>微软在.NET官方博客平台发布文章，宣布将在今年12月13日，正式终止对.NET Core 3.1版本的支持。</strong></span></p>
<p align="center"><a href="https://img1.mydrivers.com/img/20220731/50a4aa1c829d417ea7f8a47767f96502.png" target="_blank"><img alt=".NET Core 3.1正式结束使命：微软官宣于年底终止支持" h="257" src="https://img1.mydrivers.com/img/20220731/s_50a4aa1c829d417ea7f8a47767f96502.png" style="border: black 1px solid;" w="600" referrerpolicy="no-referrer"></a></p>
<p>根据文章描述，在12月13日之后，虽然基于.NET Core 3.1的应用程序仍可正常运行，但将不再获得任何漏洞补丁。</p>
<p>同时，从Visual Studio 2019 16.11、Visual Studio 2019 17.0和Visual Studio 2022 17.2的2022年12月服务更新开始，Visual Studio中的.NET Core 3.1组件将更改为不支持状态，且默认处于可选（非必需）状态。</p>
<p>因此，<strong>为了程序的稳定性与安全性，微软建议目前仍在使用.NET Core 3.1的用户尽快迁移到.NET 6或更高版本。</strong></p>
