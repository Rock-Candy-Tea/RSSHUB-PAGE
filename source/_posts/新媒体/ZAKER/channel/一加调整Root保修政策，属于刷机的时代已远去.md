<p>不知道大家还记得上一次给手机 Root 是在什么时候？而这个问题，或许也是许多老 Android 用户的共同感受。但无论是否承认，都不妨碍曾经那个充斥着 Root、刷机、变砖，拿一台手机使劲折腾的时代，已经彻底远去。甚至不仅仅是用户们已经不再 " 折腾 "，手机厂商也同样亦如此，日前就连一加也已宣布 "Root 保修 " 成为了历史名词。</p><p></p><div data-original="http://zkres1.myzaker.com/202204/626a7bd18e9f0939dc630621_1024.jpg" data-height="258" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202204/626a7bd18e9f0939dc630621_1024.jpg" referrerpolicy="no-referrer"></div></div>日前，@一加蓝帅 在社交平台发布 " 更正 Root 保修政策的说明 "。其中显示，一加新的保修政策中明确 Root 可恢复才能享有正常的退换货和保修政策，而不可恢复主要指的是 Root 后导致主板硬件故障、无法刷机恢复，将不享受退换，仅享受保修。同时他还特意说明，一加的售后中线上线下均不提供 Root 服务，并且官方也不鼓励 Root 行为。<p></p><p>一加不再为 Root 后不可恢复的手机提供相关售后服务，无疑也正应了那 "Root 有风险，刷机需谨慎 "。</p><p>在目前的法律规定下，由于未按照使用说明书要求使用、维护、保养而造成的产品损坏都不在三包范围内，所以一加方面确实有做出更正 Root 保修政策的理由。并且对于几乎所有的手机厂商而言，售后都是一个纯粹的成本部门，甚至就连苹果方面都曾表示，" 自 2009 年以来，每年提供维修服务的成本都超过了维修产生的收入。"</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630622_1024.jpg" data-height="258" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630622_1024.jpg" referrerpolicy="no-referrer"></div></div>而 Root 于 Android 手机之所以是一项非常危险的权限，是因为 Root 后手机变砖是常有的事情，坚持 Root 也保修将会提升售后部门的成本。如今，一加的做法就是 " 勿谓言之不预也 "，未来用户如果艺高人胆大继续 Root 也无妨，但 Root 后变砖就别找官方保修了。也就是说 Root 与否是由用户自己决定，所以代价也需要用户来承担，几乎就等于是在劝退单纯想要尝鲜 Root 的这一批用户胡了。<p></p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630623_1024.jpg" data-height="450" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630623_1024.jpg" referrerpolicy="no-referrer"></div></div>所谓 Root，是指基于 Linux 开发的 Android 系统的超级管理员账户，可对系统内的所有文件、包括根目录文件进行增删改查而得名。用户在通过在卡刷、线刷等方式获得 Root 权限后，就可以实现诸如卸载不需要的系统自带应用、通过启动项管理提高续航、获得一些功能强大的插件等操作，甚至于还可以给自己的手机换个系统，可谓是海阔凭鱼跃、天高任鸟飞。<p></p><p>但问题是，Root 同时也是一把双刃剑，由于可以对系统文件进行删改，所以一旦删除了核心文件、手机变砖就成为了必然。</p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_3" data-original="http://zkres1.myzaker.com/202204/626a7bd18e9f0939dc630624_1024.jpg" data-height="600" data-width="471" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202204/626a7bd18e9f0939dc630624_1024.jpg" referrerpolicy="no-referrer"></div></div>早期，Root 是 Android 系统魅力的一种体现方式，只要有一定的基础、且时间充裕的情况下，在硬件允许的范围内几乎什么样的功能都可以实现，这对于玩家而言显然是一件很有吸引力的事情。况且从 Android 1.0 到 Android 4.4，Android 本身也还处于成长期，谷歌为其所提供的功能都并不全面，而 Root 后为系统装上 magisk、xposed 框架、绿色守护、Kernel Auditor 等，在当时也可以让一台 Android 手机从 " 能用 " 变为 " 好用 "。<p></p><p>然而 Root 的时代最终还是远去了，近年来的 Android 用户更是几乎对 Root 可谓是闻所未闻。毕竟由于 Root 所带来的结果，是会让用户几乎彻底脱离了手机厂商提前准备好的软件生态，互联网式的后向收费也要与 Root 后的手机无缘，所以手机厂商一直以来往往都是不鼓励、不支持 Root。</p><p>同时在隐私和安全这面大旗下，谷歌一步步收紧了用户权限，并加快了安全补丁的更新速度，Root 可以使用的漏洞也越来越少。没错，Root 的难度不断加大同样也是越来越多用户不愿 " 折腾 " 的一个核心因素，并且不是这届 Android 用户 " 折腾不动 "、而是大家没法 " 折腾 " 了。</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_4" data-original="http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630625_1024.jpg" data-height="404" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630625_1024.jpg" referrerpolicy="no-referrer"></div></div>众所周知，Root 的实现有两种路径，主流的是通过解除 BootLoader、也就是俗称的 "BL 锁 "，进入 Recovery 模式或 Fastboot 来获取 Root 权限。但遗憾的是，出于自身商业利益或者说是为了保护用户体验，提供解除 BL 锁的手机厂商如今已然寥寥无几了。<p></p><p>而非主流的 Root 路径则是通过 " 提权漏洞 " 来实现，由于 Android 系统的代码来源较为复杂，其中既有 Linux 内核的代码、有高通等 SoC 厂商的代码，还有小米等手机厂商的代码，也导致 Android 直到如今在安全性上与苹果的 iOS 还有着不小的差距。</p><p>例如在曾经 Root 辉煌的时代，Linux 内核的 CVE-2011-3874 漏洞就促成了 ZergRush 的诞生，而 CVE-2014-3144 则产生了 Towelroot。然而随着智能手机接受安全补丁速度的加快，使用漏洞来 Root 的难度已经变得越来越大。</p><p></p><div class="img_box" id="id_imagebox_5" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_5" 360 一键 Root 等诸多工具的出现，让许多完全不具备相关知识的小白，也能通过简单地点击鼠标能完成 Root。但当下不是不能 Root，而是分享 Root 方法的人几乎消失了。而具备这些知识的人之所以越来越不愿意将此事分享，是因为分享 Root 的性价比在不断下降。<p></p><p>在曾经 Android 系统 " 四处漏风 "，手机厂商不仅不锁 BL、反而重视 Root 的时代，Root 本身其实是难以变现的。既然不指望此事可以获利，那么这些 " 大佬 " 在社区通过分享来提升声望就是一个不错的选择。要知道在互联网社区里，声望、影响力或名气在某种意义上也是资源、同样是可以变现的，要不为什么会有那么多人投身开源呢。</p><p></p><div class="img_box" id="id_imagebox_6" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_6" data-original="http://zkres2.myzaker.com/202204/626a7bd18e9f0939dc630627_1024.jpg" data-height="542" data-width="600" 工具不再是一条快速积累声望的途径。更为重要的是，针对 Android 系统的漏洞，谷歌的态度也发生了 180 度的大转弯，现在是直接选择用金钱来悬赏，也导致 Root 可用的漏洞变得极为罕见。而有能力的 " 大佬 " 要么转到了其他领域，要么就是直接当起了安全漏洞的 " 赏金猎人 "。<p></p><p>所以用一句话来说，就是现在 Root 已经成为了少数极客玩家的小众爱好，曾经 " 无 Root 不 Android" 的时代过去了。既然如此，一加方面关于 Root 保修政策的变化就是在顺应潮流了，毕竟昨日之日不可留。</p><p>【本文图片来自网络】</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
