<p>这 Arc 显卡糟糕的表现，让 Intel 引来了诸多用户的吐槽，而官方也是再次回应，确保在 Arc GPU 上提高 DirectX 11 和传统 API 游戏性能。<strong>最初在 LinusTechTips 的评论中报道，该技术机构在利用 Arc A770 显卡的系统上对《古墓丽影》进行基准测试时，目睹了 DirectX 11 和 12 版本之间 50% 的 GPU 性能差异。</strong></p><p>DirectX 11 和旧版 API 的功能与更新的 DirectX 12、Vulkan 和其他当前 API 不同。较早的 API 技术需要图形驱动的大部分处理，从增强功能到为性能较低的图形卡进行定制。需要 GPU 处理更多的游戏工作是为了减轻一些来自游戏开发者的负担，他们希望优化游戏的外观。</p><p>有了 Vulkan 和当前的 DX12 API，提升不再取决于图形驱动，而是取决于游戏的图形引擎。现在，游戏开发者需要处理图形优化的责任，特别是在较弱的系统中，并在游戏的代码中放置任务来承担这一负担。这方面的一个例子是视频内存分配。</p><p>Intel 长期没有关注图形 API，因为他们多年来没有开发 GPU。现在，随着该公司的 Arc 系列图形卡，他们不得不追赶多年来专注于此类技术的公司，即该公司的竞争对手 AMD 和 NVIDIA。</p><p>该公司对 DirectX 11 和旧 API 的无所作为，导致 Intel 承认需要相当长的时间来了解并找到困扰其目前 iGPU 和 dGPU 的问题的解决方案。</p><p>" 我们在独立显卡上发布的软件显然表现不佳，"Gelsinger 说。" 我们认为我们将能够利用集成图形软件栈，但它完全不能满足我们需要的性能水平和游戏兼容性等。因此，我们在独立显卡领域没有达到 400 万台的目标，即使我们现在正在追赶并获得更好的软件发布。"</p><p><strong>" 虽然我们不会达到我们的 GPU 单位目标，但我们仍然有望在今年实现超过 10 亿美元的收入。在第二季度，我们开始为笔记本电脑的 Intel Arc 显卡加量，OEM 厂商包括三星、联想、宏基、惠普和华硕等。"Intel 方面仍自信的表示。</strong></p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres2.myzaker.com/202208/62f0846a8e9f096a0234bd96_1024.jpg" data-height="394" data-width="700" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202208/62f0846a8e9f096a0234bd96_1024.jpg" referrerpolicy="no-referrer"></div></div><p></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
