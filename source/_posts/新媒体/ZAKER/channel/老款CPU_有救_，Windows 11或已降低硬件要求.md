<p>提到即将推出正式版的 Windows 11，许多朋友或许都有着相当复杂的心情。一方面，在看过微软的 Windows 11 发布会，或是此前我们三易生活数篇分析内容的朋友都知道，Windows 11 此次的确有着不少的 " 新玩意 "。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div HDR 屏幕更为完善的支持，还有更加便利的窗口分屏、文件云同步功能。当然众所周知的是，Windows 11 未来还将具备窗口化原生运行 Android 应用的能力，这与现在那些不带 3D 加速、没法玩大型游戏的模拟器可有着截然不同的体验。<p></p><p>然而从另一方面来说，Windows 11 此次对 PC 硬件要求的大幅提升，也确实让不少朋友感到头大。且不说强制电脑必须要有 TPM 2.0 安全模块，让不少其实主板自带该模块，但却不知道该如何打开的用户烦恼了一阵子。单就 Windows 11 对于 CPU 规格的要求，就让许多配置其实并不算太低的用户感到无所适从。</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img 系或以上的 AMD 锐龙 CPU，或是 8 代及以上的 Intel 酷睿 CPU。后来微软方面曾更新过一次 Windows 11 的硬件要求，新增了对少数几款高端 7 代酷睿移动版 CPU（7820HK、7920HQ）以及 7 代酷睿 X 系列 CPU（7800X、7920X、7980XE 等）的支持。<p></p><p>这也就意味着，诸如 6 代、7 代酷睿的主流型号（比如 6700K、7700K），以及 AMD 初代锐龙桌面版（1700X、1800X）、初代锐龙线程撕裂者（1950X）等一大批性能尚可的 CPU，被 Windows 11" 拒之门外 " 了。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres1.myzaker.com/202109/6144926c8e9f09459149c8a0_1024.jpg" data-height="618" data-width="750" 代酷睿，还是初代锐龙和锐龙线程撕裂者，它们对应的主板（Z170、Z270、X370 和 X399）都能完美支持 TPM 2.0 模块，可以满足 Windows 11 对于安全性的要求。<p></p><p>另一方面，在 Windows 11 原生支持的处理器中，不乏许多性能还不如那些被 " 拒绝 " 了的老型号。比如说，酷睿 i3-8100、锐龙 5 3400G 等中低端 CPU，它们怎么看也不可能比酷睿 i7-7700K、锐龙 7-1800X 更快。所以说老款处理器是因为性能不够而运行不了新系统，显然也是站不住脚的。</p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy 适配范围，让更多老电脑也能用上新系统呢？<p></p><p>虽然表面上，微软对此并未做出回应，甚至宣称 " 不支持的硬件平台可能会导致更多蓝屏 " 云云。但从我们三易生活的亲身经历来看，他们很有可能已经偷偷 " 知错就改 " 了。</p><p>为什么我们会这么说？其实问题的关键出在微软日前推送的两个 Windows 11 测试版本上。</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_4" data-original="http://zkres1.myzaker.com/202109/6144926c8e9f09459149c8a2_1024.jpg" Build 22000.194。这是一个比较接近最终正式版的版本，并且根据官方公布的信息显示，其主要会被推送给两类用户，其一是具备 Windows Insider 测试资格，且选择了 "Beta" 版本渠道，电脑配置也符合 Windows 11 要求的用户。<p></p><p>另一类，则是具备 Windows Insider 测试资格，选择了更激进 "Dev" 版本渠道，但本身电脑硬件并不符合 Windows 11 最低需求的用户。这类用户在收到 Windows 11 Build 22000.194 版更新的同时，更新渠道会被自动改为 Beta 版，并在 Windows 11 正式发布后停止得到自动更新。</p><p>乍看之下，微软对于硬件 " 不达标 " 的用户，使用了一种相当残酷的 " 逐客 " 方式，阻止他们使用 Windows 11。但问题在于，官方公告虽然说是这样说的，可实际情况又是怎样的呢？</p><p></p><div class="img_box" id="id_imagebox_5" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_5" PC。而其早在数天前，就已经收到了 Windows 11 Build 22454.1000 的系统更新推送。<p></p><p>没错，微软向不符合 Windows 11 现行硬件要求的电脑，推送了原本应该得不到的开发版系统。</p><p>不仅如此，我们也询问了一位使用酷睿 i7-6700K 老款机型的朋友，得到了这样的一张截图。</p><p></p><div class="img_box" 的初代锐龙，Intel 本该不支持 Windows 11 的 6 代酷睿，同样也被推送了最新的 Dev 版系统。而微软此前声称 " 不符合标准的电脑，系统测试渠道会被降级，最终被停止更新 " 的状况，压根就没有发生。<p></p><p>要知道，Windows 11 的正式版版本号，目前基本已经可以确认是 Build 22000，所以这种情况基本就只可能有两种解释。</p><p>要么，是微软 " 刀子嘴豆腐心 "。虽然已经数次宣布不再支持老款处理器，但实际上依然还在对一些处理器进行兼容性测试，未来或许会在 Windows 11 正式版 RTM 的时候开放支持，并且至少将允许 6 代以上的酷睿以及初代锐龙处理器升级 Windows 11。</p><p></p><div class="img_box" id="id_imagebox_7" onclick><div 发布后的后续升级版本中完成。也就是说，Windows 11 的首个正式版本可能确实还不兼容老款处理器，但是未来新版的 Windows 11（例如 21H2 或 22H1）则可能会增加处理器的兼容性。<p></p><p>毕竟，Windows 11 Build 22454 理论上已经是比正式版更加领先的版本，而其能够支持老款处理器，多少也说明了微软到底还是听进去了这些意见的。</p><p>【本文图片来自网络】</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
