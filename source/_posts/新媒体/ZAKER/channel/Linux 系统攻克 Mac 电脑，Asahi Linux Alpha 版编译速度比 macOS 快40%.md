<p>整理 | 彭慧中 责编 | 屠敏</p><p>2020 年末，Asahi Linux 项目由程序员 Hector Martin 众筹发起，是一个<strong>专门为 Apple Silicon 系列 Mac 电脑做 Linux 系统移植的项目</strong>，还受到过 Linux 之父 Linus Torvalds 的关注。Hector Martin 是一位网安高手，也是资深的操作系统移植专家，曾为各种设备提供非官方的开源支持来移植 Linux 系统，包括任天堂 Wii、索尼 PS 系列游戏主机等。</p><p>2021 年 1 月份，Asahi Linux 项目众筹完毕，正式启动。今年 3 月份上线了测试版。据其官网称，Asahi 这个名字也是有一定来历的：它在日语中的意思是<strong> " 朝日 "</strong>，也是<strong>日语对 Mcintosh 这个苹果品种的称呼</strong>。当年苹果公司注册商标时候，因为 Mcintosh 已经被一家音响品牌先占用了，所以加了个 a 变成 Macintosh，也就是今天 Mac 的全称。</p><p>现在，M1 Mac 可以顺利运行原生 Linux 了。Asahi Linux 项目的 Alpha 版本已释出（版本地址：https://github.com/AsahiLinux）。有些网友早已经迫不及待地上手，纷纷晒出自己的成功界面，这其中还不乏 " 体验良好 " 的声音。</p><p>那么，在苹果电脑上装一个原生 Linux 系统，究竟是一种怎样的体验？首先，磨刀不误砍柴工，你需要具备官方指定的安装条件：</p><p>一台 M1、M1Pro 或 M1Max 的 Mac（Mac Studio 除外）</p><p>MacOS12.3 或更高版本，以管理员用户身份登录</p><p>至少有 53GB 的可用磁盘空间（桌面安装）</p><p>有效的互联网连接</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy 官网<p></p><p>准备好试一试了吗？确保将以上安装条件都满足后，你只需在终端输入一行代码（curl https://alx.sh | sh）即可启动安装。安装好之后，我们就可以一起来看看，究竟有多 " 香 "！</p><p>一位有 30 万 + 粉丝的 YouTuber 在他的 M1 Mac 上亲测了一下。他在完全相同的硬件上运行某些程序，<strong>Linux 竟然比 macOS 还快！</strong>且这个速度<strong>与在有两倍性能核心的 Mac Studio 上跑 macOS 的速度不相上下</strong>。</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy 视频播放性能时发现，在没有使用 GPU 加速器的情况下<strong>可以播放 0 掉帧的全屏 HD 视频</strong>，且 UI 反应亦很快，不像其他基于 ARM 的模拟器那样迟钝。Mac 电脑也不像 iPhone 手机和 iPad 系统那样封闭，因为苹果官方会<strong>积极为用户修复 Bootloader 启动器上发现的 Bug</strong>。不仅如此，Asahi Linux 还能够<strong>降低 CPU 负载</strong>，Asahi Linux 的<strong>编译速度比 macOS 足足快了 40%。</strong></p><p>当做一些日常的基准测试时，如 Geekbench，M1 Mac 单核 1749 多核 7703，而 M1 Asahi Linux 单核 1633 多核 6764，明显性能上亦存在差距，虽然不算很多。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" GPU 加速，一切图像的工作都由 CPU 一力承担，很多功能亦无法使用。<p></p><p>不过 Asahi Linux 官方也表明：" 请记住，这仍然是一个非常早期的 alpha 版本。它适用于开发人员和高级用户；如果您决定安装它，我们希望您能通过提交详细的错误报告和调试问题来帮助我们。" 这样看来，Asahi Linux 走向成熟还要一段路途需要走。毕竟连 Linux 之父都吐槽过在 Mac 上运行 Linux 困难，主要是因为 M1 芯片的 GPU 以及相关的设备没有任何 Linux 支持。不过现在这个突破也让 Linux 的版图再拓一步。或许，Linux 之父期待的愿景将于不日之后完美呈现，让我们拭目以待。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
