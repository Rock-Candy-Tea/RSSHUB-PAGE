<p>IT 之家 9 月 26 日消息 谷歌在 Chrome 88 版本时推出了 Manifest V3 扩展程序规范，为将来 Chromium 内核的浏览器奠定了扩展开发的新方向。</p><p>近日，谷歌在官网公开了 Manifest V2 规范的 " 退休 " 计划，有两个关键时间点：</p><p><strong>2022 年 1 月 17 日：</strong>Chrome 网络商店将不再接受新的 Manifest V2 扩展，开发人员仍然可以将更新推送到现有的 Manifest V2 扩展，但不能提交新的 Manifest V2 扩展。</p><p><strong>2023 年 1 月：</strong>Chrome 浏览器将不再支持运行 Manifest V2 扩展，开发人员也不可向现有 Manifest V2 扩展推送更新。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202109/614fdf9bb15ec0686a737ddb_1024.jpg" data-height="275" data-width="842" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202109/614fdf9bb15ec0686a737ddb_1024.jpg" referrerpolicy="no-referrer"></div></div>IT 之家了解到，Manifest V3 比其前身更安全、性能更高、隐私保护更好。在过去的几个月里，谷歌在新的脚本 API 中引入了其他机制，并扩展了声明性 Net Request API，支持多个静态规则集、基于选项卡 ID 的过滤和会话范围的规则。<p></p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres1.myzaker.com/202109/614fdf9bb15ec0686a737ddc_1024.jpg" data-height="720" data-width="1440" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202109/614fdf9bb15ec0686a737ddc_1024.jpg" referrerpolicy="no-referrer"></div></div>谷歌表示，在接下来的几个月里，还将推出对动态可配置内容脚本和内存存储选项的支持，以及其他新功能。<p></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
