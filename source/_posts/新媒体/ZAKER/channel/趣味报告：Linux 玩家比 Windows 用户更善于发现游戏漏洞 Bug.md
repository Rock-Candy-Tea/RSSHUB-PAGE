<p>IT 之家 10 月 26 日消息，据 Neowin 报道，太空探索游戏《ΔV: Rings of Saturn》（ΔV: 土星环）的独立游戏开发商 Kodera Software ，近期发布了一些关于游戏中的错误 Bug 报告的有趣发现。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202110/6177b79d8e9f092b89281806_1024.jpg" data-height="338" data-width="338" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202110/6177b79d8e9f092b89281806_1024.jpg" referrerpolicy="no-referrer"></div></div>IT 之家获悉，这款游戏已经在早期访问中运行了几年，开发商指出，在其游戏中发现的所有错误 Bug 中，约有 38% 来自 Linux 社区。尽管在总共售出的 12000 份拷贝中，只有 700 份是基于 Linux 的，这仅占《ΔV: 土星环》整个用户群的 5.8%。因此，Kodera 赞扬了典型的 Linux 玩家，说他们报告的 Bug 多了 650%。<p></p><p>" 截止到今天，我总共卖出了 12000 多套《ΔV: 土星环》。其中 700 份是由 Linux 玩家购买的。这相当于 5.8%。我总共收到了 1040 份错误 Bug 报告，其中大约有 400 份是由 Linux 玩家提出的。这意味着平均每 11.5 个用户有一个报告，每 1.75 个 Linux 玩家有一个报告。这就对了，一个普通的 Linux 玩家会给你带来 650% 的错误 Bug 报告。"</p><p>不过，Linux 版本上的 Bug 报告多是否只是因为该平台更容易出错呢？Kodera 很快就指出，情况并非如此。它指出，在 Linux 的 400 个错误 Bug 报告中，只有 3 个是针对平台的。</p><p>" 你知道在这 400 个 Bug 报告中，有多少是真正针对平台的？3 个，从字面上看，只有 3 个 Bug 是只在 Linux 平台上出现的问题。"</p><p>Kodera 认为，这是因为像 Linux 这样的开源软件及其周围社区的运作方式，也是 Linux 平台上的游戏玩家似乎更善于发现并向开发者报告 Bug 的原因。开发者将游戏者的努力与免费的质量保证（QA）检查进行了比较。</p><p>" 问题是，Linux 社区在报告 Bug 方面训练有素。这只是开源的方式。这 5.8% 的玩家发现了所有影响大家的 Bug 的 38%。就像拥有自己 700 人的强大 QA 团队一样。这对我来说不是 38% 的额外工作，这只是免费的 QA!"</p><p>不仅如此，报告的质量和细节在 Linux 玩家中也是特别好的，Kodera 说，一些用户甚至直接参与进来。</p><p>" 你会得到所有的软件 / 操作系统版本，所有的日志，你会得到核心转储，你会得到复制的步骤。有时我和玩家通过 discord 联系，我们很快就迭代了几个版本，逐步修复，以隔离问题。"</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
