扩展，方便我们为网页相关内容打上「高亮」并直接分享给他人，让他人可以一键跳转、查看到我们想分享的内容。</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_4" 中，这个原本交给扩展来实现的小功能正式「转正」，当你希望为分享的网页打上标注，那么只需要选中网页中需要突出显示的文本，然后右键选择「复制指向突出显示的内容的链接」，Chrome 就会生成指向高亮文本的链接用于分享了。<p></p><p>目前该功能尚未全量上线，不想等的朋友可以通过 chrome://flags/#copy-link-to-text 来手动开启。</p><p><b><strong>优先请求 HTTPS，网页加载更快</strong></b></p><p>在 Chrome 89 以及之前的版本中，当我们在地址栏中输入未指定协议的域名时，浏览器会首先定向到不那么安全的 HTTP 链接，如果 HTTP 不可用或网站仅提供 HTTPS，浏览器才会进一步请求 HTTPS 链接。</p><p>在 HTTPS 几乎已经「标配」的 2021 年，上面的这种访问逻辑机制不仅显得过时，更直接影响了网站的加载效率。因此从 Chrome 90 开始，浏览器将会先请求更安全的 HTTPS 链接，如果对应的网站未提供 HTTPS 链接才会再尝试通过 HTTP 来访问加载，最为直观的好处自然是大家常用的网站加载速度更快了。</p><p></p><div class="img_box" id="id_imagebox_5" onclick><div 的基础上能够进一步节省约 20% 的传输带宽并且免收版权费用。</p><p>正因为如此，AV1 近年也受到 Netflix、YouTube 等流媒体平台的青睐，Android 12 还在最近的更新中加入了对由此而来的 AVIF 图像编码格式的支持。</p><p></p><div class="img_box" id="id_imagebox_6" onclick><div 版本更新中为桌面端浏览器加入了对 AV1 的支持，对那些依赖计费数据网络进行移动办公的人来说，在外使用 Chrome 观看流媒体内容时流量或许可以少跑一点了。</p><p><b><strong>▍</strong><strong>Edge</strong></b></p><p><b><strong>更优雅的浮窗式下载管理器</strong></b></p><p>以往通过 Chrome 和 Edge 进行文件下载时，很多朋友的第一反应都是先看浏览器底部，毕竟除了单独的下载管理标签页面外，那里就是我们查看下载进度、管理下载文件的直接入口了。</p><p>但这种在网页底部「拉横幅」的下载任务展示方式其实是非常具有侵入性的，在小屏幕尺寸的设备上更是直接影响网页浏览体验。因此紧随收藏、扩展、集锦、截图等功能之后，下载功能在 Edge 90 中也用上了工具栏弹出浮窗这种新样式。</p><p></p><div class="img_box" id="id_imagebox_7" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_7"
