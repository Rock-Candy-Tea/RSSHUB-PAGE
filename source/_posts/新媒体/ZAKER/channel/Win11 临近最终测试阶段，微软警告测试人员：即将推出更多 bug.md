<p>IT 之家 8 月 22 日消息 Windows 11 目前处于测试的最后阶段，预计将于 10 月开始向满足升级条件的测试 PC 推送正式版更新。当然，现在你还可以通过加入 Dev 和 Beta 频道获取 Windows 11 预览版。</p><p>众所周知，虽然 Windows 11 目前仍处于测试阶段，但如果你能自行处理一些小 bug，它在日常使用中是还是相当稳定的。不过，现在看来，微软似乎决定要干点大事。</p><p>微软现在已经向用户发出了警告，表示他们正计划在 21H2 版本 ( 或称为 2021 年 10 月更新 ) 上线后专注于 Windows 11 的下一个版本设计。</p><p>微软曾表示，Windows 11 每年只会进行一次大版本更新，今年下半年只会带来一些功能更新，例如安卓 App 支持。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres2.myzaker.com/202108/61219b748e9f090af0050e24_1024.jpg" data-height="365" data-width="696" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202108/61219b748e9f090af0050e24_1024.jpg" referrerpolicy="no-referrer"></div></div>目前正在测试的 Windows 11 21H2 版本将于 2021 年 10 月推送给用户。在 Dev 频道中，微软表示他们将要针对明年发布的版本进行测试，而新版本构建除了新特性和升级外可能会有很多 bug。<p></p><p>Windows 11 尚未命名的第一个版本更新已经在内部启动测试，而 Windows 11 的下一个版本在内部代号为 22H2 版本 ( 2022 年下半年 ) ，目前它似乎并没有什么特殊的称谓。</p><p>IT 之家了解到，在接下来的几周中，微软将逐步向测试人员发布 Win11 的不稳定版本，而 Dev Channel 测试人员将最先尝到这些拥有新特性和 bug 的新蛋糕。</p><p><b>微软警告 Windows 11 测试人员 : 切换到 Beta 频道</b></p><p>在一封发给测试者的电子邮件中，微软强调了运行 Dev Channel 版本的风险，并警告用户，他们将会迎来 Windows 11 测试版中 bug 更多的版本。如果用户只是想测试 Windows 11 的第一版和原始版本，建议退出。</p><p>最初的时候，用户很难在 Dev Channel 中看到太多显著变化或新功能，但在某些设备上可能会遇到 bug（毕竟测试版没有硬件限制）。</p><p>如果你还没有准备好接受下个版本可能会到来的 bug 或新特性，你应该切换到 Beta 频道。</p><p>" 如果你想体验更可靠的 Windows 11 内部预览版，请通过 Windows Insider 程序将您的设备迁移到 Beta 通道 " 微软在一封向 Windows Insider 用户发出的电子邮件中指出。</p><p>微软将使用 Dev 通道测试 Win11 的实验性功能和未公布的新特性，而 Beta 通道将与即将发布的稳定版本 "Windows 11 version 21H2" 绑定，不会存在如 Dev 通道里那种极其影响体验的 bug。</p><p>值得一提的是，目前 Dev 和 Beta 两个通道的用户目前都在测试 Windows 11 Build 22000.160，拥有完全相同的功能。</p><p>所以说，如果你更需要稳定的版本，那么 Beta 频道才是更好的选择，以下是切换方法 :</p><p>打开 Windows 设置。</p><p>进入 Windows Update > Windows Insider 程序。</p><p>在 " 选择您的设置 " 部分，通过点击下拉菜单选择 Beta 频道。</p><p>当然，如果你非常想测试这些 2022 年版本的实验特性，则无需操作，之后耐心等待即可。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
