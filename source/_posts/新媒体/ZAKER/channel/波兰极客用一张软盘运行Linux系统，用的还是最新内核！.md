<p><strong>用软盘启动 Linux 系统</strong>曾经很 " 家常便饭 "，当然那都是 90-00 年代的事了。</p><p>有年纪（bushi）的同学可能熟悉。</p><p>但现在，即使你还有一张能用的<strong>3.5 英寸</strong>软盘，可<strong>1.44MB</strong>的容量远远装不下一个现代 linux 内核，更不用说还得加上所有支持软件了。</p><p>但奇人有招，波兰一位游戏开发小哥只用<strong>一张软盘</strong>就把现代 Linux 操作系统嵌进去了！</p><p>盘上还有<strong>几百 KiB</strong>的剩余空间！而且用的都是最新 " 组件 "，包括今年 5.16 号才发布的 5.13.0-rc2 版本的 Linux 内核。</p><p></p><div 系统<p></p><p>小哥把这个系统命名为<strong>Floppinux</strong>，在它的官网上带大家走了整个过程，包括从下拉（pull down）、编译源代码到创建最终的磁盘映像的所有命令。</p><p>而之所以做这么一件事情，是因为小哥觉得自己用了好多年 Linux，也用过很多 Live-CD（能够在不安装到硬盘的前提下，体验 Linux 操作系统的东西）。</p><p>但他对其背后的基本原理知之甚少，所以决定动手研究一下。</p><p>小哥的第一个目标是运行 Nomad Diskmag 程序。</p><p><strong>Diskmag</strong>这个远古东西不知道有人了解吗？</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres2.myzaker.com/202107/60e3ebb18e9f090c3e2a0510_1024.jpg" data-height="944" data-width="776" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202107/60e3ebb18e9f090c3e2a0510_1024.jpg" referrerpolicy="no-referrer"></div></div>它的全称叫 disk magazine，也就是磁盘杂志，是一种在上世纪 80-90 年代，以软盘形式发行的电子杂志。90 年代后就被在线出版物所取代了。<p></p><p>小哥已经用 bash 脚本搞定了前端界面，就差封面、目录和 cat 每个文件的正文了。</p><p>为了运行他写的脚本，需要一个可用的 Linux 发行版，也就是一个可以在软盘上运行的系统。</p><p>动手！</p><p>因为在 64 位系统上编译 32 位代码有点棘手。为了更简单，小哥用他的 32 位 CPU 的旧笔记本来做这一切。</p><p>可以使用 32 位系统的 VirtualBox，如果要用 64 位，添加命令 "ARCH=x86"，例如：make ARCH=x86 tinyconfig。</p><p>下面就是把现代 Linux 操作系统装进一张 1.44MB 软盘的大概过程：</p><p><strong>1、创建并进入你想要保存文件的目录</strong></p><p><strong>2、配置和构建定制内核</strong></p><p>使用最新 Linux 内核（版本 5.13.0-rc2）：</p><p>git clone --depth=1 https://git.kernel.org/pub/scm/linux/kernel/git/stable/linux.git</p><p>进行最小配置：make tinyconfig</p><p>添加额外配置：make menuconfig</p><p>从菜单中选择以下选项：</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres2.myzaker.com/202107/60e3ebb18e9f090c3e2a0511_1024.jpg" data-height="598" arch/x86/boot/bzImage 中构建，把它移到主目录。<p></p><p><strong>3、 添加工具</strong></p><p>wget https://busybox.net/downloads/busybox-1.33.1.tar.bz2</p><p>进入目录，进行启动配置：make allnoconfig</p><p>然后选择你想要的工具：make menuconfig</p><p>每个菜单项都显示各工具需占用多少 KB，合理选择哦。</p><p>小哥的选择：</p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_3" referrerpolicy="no-referrer"></div></div>保存配置并退出，编译完成后<strong>_install</strong>目录下会创建一个包含所有文件的文件系统，把它移到主目录。<p></p><p><strong>4、添加目录结构</strong></p><p>有了内核和基本工具，仍然需要一些额外的目录结构：</p><p>cd ../filesystem</p><p>mkdir -pv &#123;dev,proc,etc/init.d,sys,tmp&#125;</p><p>sudo mknod dev/console c 5 1</p><p>sudo mknod dev/null c 1 3</p><p>接下来创建几个配置文件，启动后显示欢迎消息：</p><p>cat >> welcome << EOF</p><p>Some welcome text...</p><p>EOF</p><p>然后配置处理启动、退出和重启的<strong>Inittab 文件</strong>& 实际的<strong>初始化脚本</strong>，并使初始化脚本可执行，并将所有文件的所有者设置为 root。（限于篇幅命令已省略，具体可查看文末链接 [ 1 ] ）</p><p>最后，将此目录压缩为一个文件。</p><p>可通过从主目录运行 QEMU（在 GNU/Linux 平台上广泛使用的模拟处理器）对以上所有内容进行测试。</p><p><strong>5、下面就是把这一切放进软盘了</strong></p><p>创建指向内核和文件系统的 Syslinux 引导文件（boot file）：</p><p>cat >> syslinux.cfg << EOF</p><p>DEFAULT linux</p><p>LABEL linux</p><p>SAY [ BOOTING FLOPPINUX VERSION 0.1.0 ] </p><p>KERNEL bzImage</p><p>APPEND initrd=rootfs.cpio.gz</p><p>EOF</p><p>chmod +x syslinux.cfg</p><p>创建空软盘映像：</p><p>dd if=/dev/zero of=floppinux.img bs=1k count=1440</p><p>mkdosfs floppinux.img</p><p>syslinux --install floppinux.img</p><p>Mount it ! 并将 syslinux、内核和文件系统复制到软盘映像：</p><p>sudo mount -o loop floppinux.img /mnt</p><p>sudo cp bzImage /mnt</p><p>sudo cp rootfs.cpio.gz /mnt</p><p>sudo cp syslinux.cfg /mnt</p><p>sudo umount /mnt</p><p>完成！</p><p>现在你就有了自己的发行版映像 floppinux.img，你可以烧录到软盘，然后在真正的硬件上启动它了！</p><p>启动耗时 1 分多</p><p>小哥花了不到 3 分钟烧录成功，然后开始了首次启动。</p><p>成功！大概只花了 1 分多钟。</p><p>啊，从屏幕上看了小哥似乎不年轻，头发也秃得让人落泪。</p><p>小哥（老哥）表示，在这种裸机的现代硬件上，唯一能阻止启动速度的就是软驱的实际速度。它们最大原始速度为 125KB/s。实际上可能会更慢。</p><p>下面是软盘占有空间总结，可以看到还剩 272KiB。</p><p></p><div class="img_box" id="id_imagebox_4"
