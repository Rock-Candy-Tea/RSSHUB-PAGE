月：我和伙伴伊芙被困在机场国内转机楼的落地签证处。其余的转机乘客，都是埃塞俄比亚人，正我们身边走过，在护照检查处排成一个长队。海关人员一给我盖了章，我就立即冲刺，在更多的乘客溜到我们前面之前守住我们的位置。<p></p><p>十分钟后，伊芙找到了我，我们乘坐的是深夜航班，她昏昏沉沉的，对自己在埃塞俄比亚的签证台前跑来跑去感到不解。" 你为什么要这样跑？" 不知何故，" 这样我们就可以在登机口而不是在护照检查处排队 "，似乎不是一个很好的理由。</p><p>正是在这个时候，我意识到我是一个没有耐心的人。</p><p> ( 现在回想起来，我应该有一些强迫症，但我认为这很正常，直到我开始和别人约会，发现他们经常以优先级的顺序做事。 ) </p><p>既然我已经承认了自己不耐心的毛病，我将用这篇文章试图说服你，不耐心实际上是一件好事。</p><p> ( 不耐心的一个表现是经常问自己 " 这件事怎么能少花点时间 " 的习惯。我不建议处处都这样，比如在机场的时候甩掉你的伙伴 ) </p><p>不耐心是让事情变得更快的最好方法。在许多领域中，真正的 " 快 " 与 " 有效 " 是紧密相关的。</p><p>很明显，其中很多都是非因果关系。不过，不同例子都体现了这一点：</p><p><b> 电子邮件</b></p><p>这似乎微不足道，但很多不同的人都观察到，回复电子邮件太慢是一个不好的迹象，许多名人回邮件出奇的快。</p><p>Sriram Krishnan（前 Facebook、Twitter 和 Snap 产品负责人，现任 A16z 投资人）</p><p>最著名、最有趣、最强大的人都会自己读电子邮件</p><p>他们几乎都善于快速回复邮件</p><p>他们总是非常、非常好奇。</p><p>他们的时间非常少。任何有摩擦的事情都会 " 晚点 " 完成</p><p>Sam Altman（Y Combinator 总裁）：</p><p>人们往往不是慢性子就是急性子，这似乎更难改变。做一个快速行动者是一件大事；一个有点微不足道的例子是，我几乎没有在慢吞吞回复重要电子邮件的创始人身上投资过一份钱。</p><p><b> 创业公司</b></p><p>迅速启动（和迭代）是初创企业能否成功的主要因素，这已经成为一种共识。</p><p>Sam Altman：</p><p>你必须要果断，优柔寡断是创业公司的杀手。平庸的创始人花了很多时间谈论宏伟的计划，但他们从未做出决定。他们在谈论我可以做这件事，或者我可以做另一件事，他们犹豫不决，从不行动。而你实际上需要的是对行动的偏爱。</p><p>最好的创始人做的事情看起来很小，但他们的行动非常迅速，他们真的很快就把事情做完了。每次你和最好的创始人交谈，他们都能完成新的事情。事实上，这是我们在 YC 学到的最能预测创始人成功率的一件事。如果我们每次与一个团队交谈时，他们都能完成新的事情，这就是我们对一个公司是否会成功的最佳预测因素。</p><p>这甚至会传递到你部署软件变化的速度层面。Nick Schrock 在 Facebook/Instagram 上快速迭代并创新：</p><p>Instagram 甚至比 Facebook 更快，他们有持续部署。任何工程师都可以在任何时候进行部署（他们会运行一个叫做 "Yolout " 的命令）。Facebook 当时在网络上是每周部署一次，在移动端则慢得多。</p><p>IG 的故事无疑为 " 谁交付谁赢 " 的说法提供了可信度。</p><p>值得注意的是，Facebook 从中吸取了教训。随着公司的发展，新版本发布节奏越来越快。首先是每天，然后是每天两次，现在是连续的。这是发布工程团队的一个了不起的成就。</p><p>不仅仅是 Facebook；Accelerate 发现部署频率和绩效之间有很强的关联性：</p><p>总结一下，在 2017 年，我们发现，与低绩效者相比，高绩效者有：</p><p><b> APP/ 工具</b></p><p>" 速度非常重要 " 的原则也适用于那些已经非常快的东西。谷歌已经发现，用户极其喜欢响应更快的网页：</p><p>英国广播公司（BBC）发现，他们的网站每多花一秒钟的时间加载，就会多损失 10% 的用户。</p><p>谷歌的 DoubleClick 发现，如果一个页面的加载时间超过 3 秒，53% 的来自移动端的用户都会放弃访问。</p><p>······</p><p>AutoAnything 将页面加载时间减少一半时，他们看到了 12-13% 的销售增长。</p><p>James Somers：</p><p>谷歌以优先考虑速度的特点而闻名。他们意识到，如果搜索速度快，你就更有可能去搜索。这鼓励你去尝试东西，能快速得到反馈，然后再尝试。当你产生一个想法时，你知道谷歌很可靠。在思想和行动之间没有延迟，没有机会失去搜索东西的冲动。用谷歌的预计成本是零，它感觉就像你自己思想的延伸。</p><p>Nelson Elhage 说得最好：</p><p>也许不太明显，拥有更快的工具会改变用户使用工具或执行任务的方式。用户几乎总是有多种策略来追求一个目标：包括决定去做别的事情，他们会越来越频繁地选择使用更快的工具。快速工具不只是让用户更快地完成任务；它们还让用户以全新的方式完成全新类型的任务。我在从事 Sorbet 和 Livegrep 的工作时清楚地看到了这种现象 ······</p><p>在 Stripe，许多单独的测试需要 10-20 秒或更长时间 ······ 由于我们成功地构建了 Sorbet，使其能够在该时间窗口内对整个代码库进行类型检查，因此它成为许多开发人员能够对其代码获得适当反馈的最快方式，以检查基本的错别字、误用的 API 和其他错误。由于这是他们最快的选择，我们看到用户在开发和推广的早期就将 Sorbet 作为他们检查代码的第一选择。迅速获得哪怕是平庸的反馈和一些信心，比任何需要花费几分钟的事情都要重要得多。</p><p>看着用户使用 livegrep，我从它的性能中看到了一个相关的好处：由于 livegrep 的速度非常快，所以用户可以通过罕见的人与其他搜索引擎互动的方式来使用它：输入一个初始查询，如果得到的结果太多或者太少，他们就根据结果列表来编辑它，从而得到一组新的结果，他们继续细化或扩展这些结果，直到找到所寻求的结果。</p><p><b> 个人工作流程</b></p><p>Sam Altman：</p><p>我还努力提高打字的速度，学习有助于我工作流程的键盘快捷键。</p><p>Steve Yegge（从业二十余年的资深程序员，Google 员工）：</p><p>我一直想知道哪一门是一个 CS 学生可以学习的最重要的计算机科学课程，最后发现是打字。</p><p>我认识的那些真正杰出的工程师，那些创造伟大事物的人，他们打字都很快。</p><p>这与我在 Wave 的经验相吻合，在那里，最好的工程师很可能是打字最快的，知道键盘快捷键，并投入大量的时间使他们的普通任务变得更高效。</p><p>显然，这些并不是使他们成为优秀工程师的主要因素，但我认为这些对他们的帮助是很明显的。</p><p><b> 谈判</b></p><p>在 Wave 招聘时，我们已经注意到，在招聘过程中加快某人的步伐，使他们更有可能接受我们的职位。如果我们花了很长时间来回复或安排下一个流程，这既给了他们更多的时间失去兴趣，也让人觉得我们对他们并不感兴趣，这可不是一个好现象。</p><p>这显然是销售中常见的套路，" 时间杀死了所有的交易 "。</p><p>保罗 · 格雷厄姆谈 Stripe 的创始人是如何让时间站在他们这边的：</p><p>在 YC，我们用 "Collison installation "（以著名的 Stripe 公司的 Patrick Collison 命名）这个词来形容他们发明的技术。更加谨慎的创始人会问：" 你会尝试我们的测试版吗？" 如果答案是肯定的，他们会说：" 太好了，我给你一个链接。" 但是 Collison 不打算等待。当有人同意尝试 Stripe 时，他们会说：" 那好，把你的笔记本电脑给我 "，并当场为他们设置好软件。</p><p>Fred Wilson（风险投资公司 Union Square 的共同创始人）：</p><p>那天晚上，当我回到家时，我告诉 Gotham Gal：" 我今天遇到了 Danny Meyer（纽约市餐厅老板），他给了我名片，说只要我需要预约，就可以给他打电话。" 她回答说 " 明天去那里吃午饭 "。而我告诉她 " 我明天不吃午餐 "。她说 " 就明天去。他明天还会记得你是谁，但下个月就不会了。"</p><p><b> 战斗</b></p><p>最有影响力的军事战略作家之一 John Boyd 因 " 包以德循环 ( OODA Loop ) " 而闻名：观察（Oberve）、调整（Orient）、决策（Decide）以及行动（Act）。Boyd 的主张是，更快地通过包以德循环是一个决定性的优势：</p><p>为了取胜，我们应该以比对手更快的速度或节奏运作，或者，最好是进入对手的观察 - 调整 - 决策 - 行动的时间周期或循环 ······ 这将使我们显得模棱两可（不可预测），从而在对手中产生混乱和无序，因为我们的对手将无法产生与他们正在竞争的威胁性以及更快的瞬时节奏或模式相一致的心理图像。</p><p>例如，当新冠疫情发生时，许多州和国家等待了太长时间才推出隔离政策，因为病毒进入了他们的包以德循环：</p><p>现在，各级机构的认知度都在下降。个人、公司、政府、国家元首、股票交易商、联合国，每个人都觉得他们正在失去主导权。据我们所知，一种病毒已经进入人类的包以德循环，并夺取了主动权，而我们却在努力想出该如何称呼它。它的传播速度比我们最快的真理、谎言和废话还要快。</p><p><b> 生活</b></p><p>Patrick Collison：</p><p>如果你十几岁：这是黄金岁月 !</p><p>那些做了伟大事情的人往往在非常年轻时就做到了。 ( 他们成名时已经白发苍苍 ...... ) 所以，快速行动！你可以做大事。</p><p>Sam Altman：</p><p>一旦你想好了要做什么，就要势不可挡地迅速完成你那一小撮优先事项。我还没有见过一个行动缓慢的人非常成功。</p><p><b> 为什么要不耐心？</b></p><p>很明显，速度是很重要的：如果你的生产力提高 10%，你完成工作的时间会减少 10%，所以你总共可以多做 10% 的工作。但我不认为这是速度之所以重要的主要原因。</p><p>在此值得指出的是，上面所有的引文都不仅仅是关于工作，它们是关于更快速地处理信息。你处理信息的速度越快，你就能越快地将结果纳入你接下来的工作中。</p><p>换句话说，快速的主要好处是，你最终会做不同的事情。Nelson Elhage 的观点：" 拥有更快的工具会改变用户使用工具的方式 "，这几乎适用于每个领域。</p><p>如果你快速地回复电子邮件，你将获得更多的新机会，并最终优先于你本来要做的任何事情。</p><p>如果你测试代码的速度提高 10 倍，你不仅仅是节省了等待测试的时间，你可以开始做测试驱动的开发，更早地发现你的错误，并使你自己免于走歪路。</p><p>如果你现在而不是下周部署你的新应用，你将提前一周了解用户对新功能的喜爱程度，你将能够把这些知识反馈到未来的产品决策中。</p><p>这意味着，快速移动是一种复合的优势。两倍的速度不仅使你的产出增加，而且使你的产出增长率增加。随着时间的推移，这将产生巨大的差异。</p><p>译者：蒂克伟</p><div
