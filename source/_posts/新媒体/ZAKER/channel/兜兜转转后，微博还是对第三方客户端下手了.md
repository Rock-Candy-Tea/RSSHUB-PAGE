<p>如今作为中文互联网的舆论中心，微博几乎成为了每一位网民手机中必装的 APP，" 刷微博 " 也是大家足不出户就能知晓天下大事的重要方式之一。不过，大家都是用什么 APP 来 " 刷微博 " 呢？没错，这确实并不是明知故问，而是微博与当下的许多 APP 不同，它其实有着诸多第三方版本。但随着时间的推移，第三方微博客户端可能将成为历史名词了。</p><p></p><div "，而微博起诉的理由则是不正当竞争。<p></p><p>据悉，VVebo 是一款使用 iOS 扁平化风格设计的第三方微博客户端，主要卖点就是简洁的 UI 和易用性出色的交互方式。但其实在 VVebo 方面被微博告上法庭前，微博第三方客户端 See 的开发者就已宣布，" 我们意识到 See 始终不是一个合规的产品，从今天开始 See 将不再维护和支持使用，应用市场也做了下线处理。"</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" "，微博如今绞杀第三方客户端的行为，也只是在为当年的选择买单。有些朋友可能会很疑惑，如今国内的一众网络平台里为什么只有微博存在 " 第三方客户端 "，但诸如抖音、微信、快手、知乎等平台就没有。<p></p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202209/6323197c8e9f0950b735234a_1024.jpg" referrerpolicy="no-referrer"></div></div>事实上，第三方客户端这一概念是 PC 互联网时代的产物，许多老网民最先接触的无疑是腾讯 QQ 所谓的第三方客户端，比如 QQ 珊瑚虫、QQ 快乐无极版、QQ 飘云版等。特别是当年的 QQ 珊瑚虫，查看好友 IP 的功能在本世纪初可谓是风靡全网，甚至于在某段时间的知名度更是压倒了 QQ 本身。当然，QQ 珊瑚虫与 VVebo 并不能一视同仁，因为 QQ 和微博方面对于第三方客户端的态度截然不同。<p></p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_3" data-original="http://zkres2.myzaker.com/202209/6323197c8e9f0950b735234b_1024.jpg" data-height="410" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202209/6323197c8e9f0950b735234b_1024.jpg" referrerpolicy="no-referrer"></div></div>从本质上来说，QQ 珊瑚虫其实并非第三方客户端，而是从插件行为衍生到黑盒破解。最初珊瑚虫、飘云等就没有得到腾讯方面的许可，这也是为什么 QQ 珊瑚虫的作者最终会被腾讯起诉，并以侵犯著作权罪被判刑的原因。然而微博的情况却完全不同，其诞生之初对于第三方客户端是举双手赞成的，因此后者的出现也是微博一手促成的。<p></p><p>新浪微博诞生于 2009 年，彼时微博这个词还不是专指新浪微博，而是分享简短实时信息的社交平台统称。2006 年被称为 " 互联网短信服务 " 的 Twitter 出现，作为传统博客碰撞社交平台的产物，Twitter 这类 " 微博客 " 很快就成为了新的潮流。在 "Copy-to-China" 这一模式下，微博这个概念也进入了国内市场，彼时除了新浪外，腾讯、搜狐、网易这三大门户也陆续上线了自己的这类产品。</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_4" data-original="http://zkres1.myzaker.com/202209/6323197c8e9f0950b735234c_1024.jpg" data-height="512" data-width="600" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202209/6323197c8e9f0950b735234c_1024.jpg" referrerpolicy="no-referrer"></div></div>模仿 Twitter 的新浪微博不仅模仿了前者的设计，更吸纳当时 Twitter 等硅谷互联网产品的开放平台理念。而所谓的开放平台，就是类似微信小程序这样产品，通过开放 API 接口的方式引入第三方开发者。只不过以今天的视角来看，微博的开发平台相比于微信、抖音、快手、支付宝的小程序，显得有些 " 开放过头 " 了。<p></p><p>开发平台的运营模式，是基于互联网厂商自身业务来做延伸，并将他们掌握的能力从业务里抽象出来，以快速实现服务交付，赋能第三方的开发者。当时，腾讯、阿里的开发平台所提供的 API 接口功能比较有限，但新浪微博毕竟是诞生在 PC 时代的产品，并且在创设之初全盘学习了 Twitter，所以彼时其所提供的 API 功能也极为完善。</p><p></p><div 年前的第三方微博客户端，可以说除了比微博自己的 APP 更好用之外，在功能上几乎一模一样，该有的转评赞、私信、分享、话题等功能都有。从某种意义上来说，在 PC 互联网向移动互联网转型的那几年里，新浪微博压倒腾讯微博、搜狐微博，也离不开 VVebo 等更好用的第三方客户端。<p></p><p>诚然，VVebo、See 这类第三方客户端好用归好用，但在新浪微博拥有 " 微博 " 这一专有名词后，它们的存在就显得有些不合时宜了，并从新浪微博当初击败腾讯、搜狐时的小帮手，变成了与其抢用户的竞争对手。</p><p>早在 2017 年，微博方面就开始对第三方客户端申请 API 接口进行了严格的限制，大量的高级功能被禁用，所以如今的第三方微博客户端除了刷微博和发微博外，几乎已经没有其他的花样了。</p><p></p><div class="img_box" "，其二则是基本不含广告。在第三方客户端上除了信息流中插入的广告外，微博其他几乎所有用于盈利的产品都无法在第三方客户端里展现，更无法统一展示效果。毫无疑问，这对于微博来说就很要命了，无法统计展示效果就无法归因，更不要提让广告主来买单了。<p></p><p></p><div class="img_box" id="id_imagebox_7" onclick><div 的时候，顺带把这项政策也搬了过来，毕竟第三方客户端的用户几乎都可以算作是毫无贡献值的用户。</p><p>所以只能说此一时彼一时，在微博本身还在为 " 吃肉 " 发愁时，又怎么会给第三方客户端 " 喝汤 " 的机会呢。</p><p>【本文图片来自网络】</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
