<p>随着 Windows 11 正式上线的时间越来越近，关于微软这款全新操作系统的信息最近也开始爆炸性地多了起来。</p><p>例如在 CPU 限制方面，最早微软的说法是 Windows 11 只支持 8 代酷睿或锐龙 2000 系以上，但后来官方增加了对 7 代酷睿 X 系列 CPU 的官方支持。而在最新一版的 Windows 11 开发版（Bulld 22454）发布后，原本不被支持的 6 代、7 代普通酷睿 CPU，以及 1000 系的锐龙 CPU，也收到了更新推送，暗示未来 Windows 11 在 CPU 方面的限制可能会进一步放宽。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy 更新向开发版测试用户推送了他们的 WDDM 3.0 驱动。而根据 Intel 最近公布的消息显示，AMD 最快可能会在 10 月正式向所有用户提供针对 Windows 11 的正式版驱动下载。虽然这比 Intel 要足足晚了半年多（Intel 是第一家发布 Windows 11 显卡驱动的厂商，早在今年二月就已经将驱动放了出来），但也意味着三大芯片厂商都赶上了 Windows 11 的正式版推送日期。<p></p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres2.myzaker.com/202109/613f36c9b15ec01c831bf600_1024.jpg" data-height="468" data-width="750" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202109/613f36c9b15ec01c831bf600_1024.jpg" referrerpolicy="no-referrer"></div></div>除此之外，根据近日微软企业管理副总裁 Steve Dispensa 公布的一份数据显示，与 Windows 10 相比，微软在 Windows 11 上针对超多应用程序并行处理的使用情况（比如说，开着 QQ、玩着游戏，还正在直播和放歌什么的）进行了大幅度的优化。这使得 Windows 11 能够比前代平均减少 32% 的内存和 37% 的 CPU 占用率，同时将更多的资源分配给最重要的前台程序，从而有效提高计算机的反应速度，同时降低功耗。<p></p><p>不过，以上这些都不是今天的重点。事实上我们三易生活今天要讲的，反而是 Windows 11 此前被很多人诟病的一项功能，那就是强制必须支持且开启 TPM 2.0 模组。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres1.myzaker.com/202109/613f36c9b15ec01c831bf601_1024.jpg" data-height="393" data-width="750" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202109/613f36c9b15ec01c831bf601_1024.jpg" referrerpolicy="no-referrer"></div></div>什么是 TPM 2.0？简单来说，它是电脑上的一个 " 安全处理器 "，在一些古老的（比如 5 代以前的酷睿平台）电脑上，TPM 2.0 模组必须由用户自行购买并安装到主板上的一个小芯片。而在 6 代以后的酷睿平台上则是直接被集成在主板里的，只不过大多时候默认都没有开启而已。<p></p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_3" data-original="http://zkres1.myzaker.com/202109/613f36c9b15ec01c831bf602_1024.jpg" data-height="403" data-width="750" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202109/613f36c9b15ec01c831bf602_1024.jpg" referrerpolicy="no-referrer"></div></div>正因如此，当微软在 Windows 11 上要求计算机硬件必须支持且开启 TPM 2.0 时，就出现了很多消费者因为不知道如何开启这个功能，导致无法安装 / 更新 Windows 11 的问题。一时间，关于 TPM 2.0 的批评声四起。即便那些知道该如何打开这项功能的用户，有些也会质疑 TPM 2.0 对于 Windows 11 的必要性。毕竟在现有的 Windows 中，基本上也只有磁盘加密会用到 TPM 模块，而磁盘加密自然不是所有用户都需要的功能。<p></p><p>然而随着时间的推移，一个我们此前并未考虑过的可能性逐渐浮出水面。如果微软在 Windows 11 上强制要求 PC 具备 TPM 2.0 安全模块，并不是为了系统本身的功能服务，而是给第三方软件提升安全系统打基础呢？</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_4" data-original="http://zkres2.myzaker.com/202109/613f36c9b15ec01c831bf603_1024.jpg" data-height="295" referrerpolicy="no-referrer"></div></div>就在近日有游戏玩家发现，他们的射击游戏《Valorant》突然无法打开。而游戏给出的报错信息中，赫然写着 " 你必须开启 TPM2.0 和安全启动，才能玩到当前版本的游戏 "。<p></p><p>作为知名游戏公司拳头开发，当前月活用户达 1400 万的一款大型网络 FPS 游戏，《Valorant》为何会出现这样的报错？其实原因很简单，因为在最新的版本里，这款游戏会默认在 Windows 11 上启动一个名为 "Vanguard" 的反作弊软件。而该软件则会调用 PC 上的 TPM 2.0 模块对游戏里的关键数据进行加密，以达到令外挂 " 失效 " 的作用。</p><p></p><div class="img_box" id="id_imagebox_5" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_5" data-original="http://zkres2.myzaker.com/202109/613f36c9b15ec01c831bf604_1024.jpg" data-height="394" data-width="750" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202109/613f36c9b15ec01c831bf604_1024.jpg" referrerpolicy="no-referrer"></div></div>不仅如此，对于在游戏中作弊、采用不正当竞争手段严重的玩家，《Valorant》甚至还会借助 PC 内置的 TPM2.0 模块，对玩家的 PC 进行 " 硬件锁机 "。由于这种封号记录的是 PC 主板里的安全识别信息，因此除非更换主板，否则即便重新注册账户也无济于事。<p></p><p>当然，更为重要的一点在于，通过《Valorant》的反作弊机制，拳头相当于向其他厂商展示了借助 Windows 11 强制 TPM 2.0 特性来反作弊的案例，而这有可能会激发更多的公司开发出基于 Windows 11 的反作弊、反盗版、反隐私窃取机制。</p><p></p><div 的做法得到了业内人士大力赞赏<p></p><p>如此一来，一方面对于第三方软件来说，他们本身的安全性确实可以得到大幅提升；而另一方面来说，微软也由此有了更多的理由，在新版系统中推行各种先进的硬件级安全机制。毕竟，如果这些安全技术确实能够带来用户也可以察觉到的体验改善，那么要让大家接受起来自然也就容易得多了。</p><p>【本文图片来自网络】</p><div
