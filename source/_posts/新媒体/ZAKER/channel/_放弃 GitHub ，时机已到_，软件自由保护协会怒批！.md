<p>整理 | 彭慧中 责编 | 屠敏</p><p>诞生 14 年来，凭借完整协议支持、Git 格式的版本库托管等优势，GitHub 成为了最受开发者欢迎的代码托管平台。</p><p>不过，近几年来， 随着 GitHub 的一些转变，引发了行业内一些组织的不满。6 月 30 日，软件自由保护协会（以下简称 SFC）在官网发表了一篇题为<strong>《放弃 GitHub，时机已到》</strong>的长文，怒称：由于 GitHub 的不当行为，<strong>SFC 将结束自己对 GitHub 的所有使用，并将协助其他自由软件项目从 GitHub 迁移</strong>。虽然 SFC 现在不会强制要求现有会员迁移项目，但他们将不再接受不打算从 GitHub 迁移的新会员项目。</p><p></p><div class="img_box" 的服务是自由软件开发者向 GitHub 和微软抗议其不良行为而发出的强烈信号。<p></p><p>那么 GitHub 究竟是如何 " 惹恼 " 了 SFC 呢？</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202207/62bea299b15ec0406d4bae9a_1024.jpg" referrerpolicy="no-referrer"></div></div>图源 SFC 官网<p></p><p>在过去的十年里，GitHub 改变了人们的编程方式。不仅让编程变得更简单，还改变了软件开发者对编程的看法。它找到了全世界数百万人正在努力解决的一个大问题——如何在代码上协作，并设计出了市场急需的的解决方案，实现了令人难以置信的增长和成功。</p><p>通过围绕开源项目 Git 构建 SaaS 服务，GitHub 为开源生态系统提供价值并从中获利。<strong>具体来说，GitHub 的利润主要来自那些希望使用 GitHub 工具进行内部专有软件开发的人。</strong></p><p>然而，SFC 认为，尽管 GitHub 一直标榜自己为这么多自由软件提供了大量的服务，但从大科技公司的许多免费服务中可以看到：<strong>如果你不是 GitHub 的客户，那么你就是其产品。</strong>科技公司用自由软件的开发方法开发成了 GitHub 的产品，而 GitHub 常常在对方不知情的情况下将其私有化并重新包装。</p><p>FOSS 的开发者长期以来对于 GitHub 的这类行为一直以<strong>" 温水煮青蛙 " 的心态</strong>。软件自由保护协会也意识到自己的包容造就了这个问题的一部分，当 GitHub 的行为变得越来越糟糕，而他们一直在原谅、忽视、默许 ...... 直到最近，<strong>终于该 " 秋后算账 " 了</strong>。</p><p><strong>SFC 和 GitHub 长达一年的持久战</strong></p><p><strong></strong></p><p>具体 " 引爆 "SFC 此次大动作的事件，则是 GitHub 在长达一年的时间内拒绝回应 SFC 对于 Copilot（AI 自动编程工具）在公开代码上进行训练的合法性的质疑。而就在上周，GitHub 宣布<a href="http://iphone.myzaker.com/zaker/link.php?pk=62bebc5b8e9f0960914098a1&b=aHR0cDovL21wLndlaXhpbi5xcS5jb20vcz9fX2Jpej1Nak01TWpBd09ETTRNQT09Jm1pZD0yNjUwOTI1OTA3JmlkeD0xJnNuPTdlNGE1NDRhNGRjNDY3NDJjM2I3YjU4NWY3YTBhZDE0JmNoa3NtPWJkNTk5YTgwOGEyZTEzOTZlMDQ0NjgzNDVkNWJkZDQwMjljYTY2MGM3Njg3ZDAxNDIzYjRhZWZhODYxNDM5ZDFiNTY5MzNkZTM3ZWYmc2NlbmU9MjEjd2VjaGF0X3JlZGlyZWN0&bcode=3ab73c99&target=_new" target="_blank">Copilot 成为一个商业化的盈利性产品</a>，彻底激怒 SFC。</p><p><strong>事件经过如下：</strong></p><p>2021 年 7 月，SFC 就曾与微软和 GitHub 的代表进行了视频通话，提出了几个问题，他们说当时无法回答，但会 " 很快回答 "。</p><p>在六个月没有回应之后，SFC 在其文章《<strong>如果软件是我的副驾驶，谁给我的软件编程？</strong>》（https://sfconservancy.org/blog/2022/feb/03/github-copilot-copyleft-gpl/）一文中公开提出了这些问题，但 GitHub 仍然没回应。</p><p>三周后，SFC 成立了一个专家委员会，讨论 " 人工智能辅助软件的产生的伦理道德问题 "（https://sfconservancy.org/news/2022/feb/23/committee-ai-assisted-software-github-copilot/），并同时进行公开讨论（https://lists.copyleft.org/mailman/listinfo/ai-assist）。他们邀请微软和 GitHub 的代表参加公开讨论，但微软和 GitHub 无视 SFC 的邀请。</p><p>2022 年 6 月下旬，在 SFC 提醒 GitHub 关于以下 2 点："<strong>1. 这几个悬而未决的问题我们</strong><strong>已经等了一年</strong><strong>；2. 你们为何拒绝加入关于这个话题的公开讨论？</strong>"</p><p>然而 GitHub 在一周后回应：<strong>他们不会加入关于这个问题的任何公开或私下讨论</strong>，因为 " 关于人工智能辅助软件的伦理 " 的讨论似乎不太可能改变 SFC 的立场，这就是为什么 GitHub 没有回应 SFC 的原因。</p><p>2022 年 6 月 22 日，不仅拒绝回应 SFC 的 GitHub 还宣布将 Copilot 商业化，SFC 的底线终于被突破。</p><p>由此可见，GitHub 对 Copilot 的最终立场是：<strong>如果 SFC 不同意 GitHub 对 Copilot 相关政策事项的看法，那么 SFC 就不值得微软或 GitHub 回复。</strong>只有当 SFC 认为可以立即将其立场改为 GitHub 的政策立场时，GitHub 才会费心回复。即便如此，就连这个答案微软和 GitHub 会时隔一年才告知！在这期间，GitHub 还在一直努力推进 Copilot 的商业化！</p><p>而 SFC 一直希望微软 /GitHub 回应的关于 Copilot 的三个主要问题是：</p><p>1. <strong>在微软和 GitHub 的公开声明中，依据的是什么判例法（如果有的话）</strong>。GitHub 当时的 CEO 说："（1）在公共数据上训练 ML 系统是合理使用（2）输出的代码版权属于操作者，就像编译器一样。" 为了透明和尊重自由和开放源码软件社区，也请向社区提供你们完整的法律分析来证明你们这些说法是有理有据的。</p><p>SFC 认为，微软和 GitHub 的拒绝回答，表示他们仍然坚持他们的前 CEO 的声明（这也是他们在这个问题上的唯一声明），但事实上他们没有相应的法律理论来支撑。</p><p>2. 如果像微软 /GitHub 所说的那样，允许在任何代码上训练模型（并允许用户基于该模型生成代码）而不受任何许可条款的约束，<strong>你们为什么只选择在自由软件上训练 Copilot 的模型？例如，为什么微软 Windows 和 Office 代码库不在你的训练集中？</strong></p><p>SFC 认为，微软和 GitHub 的拒绝回答也暗示了这个问题的真正答案。虽然 GitHub 很乐意利用 FOSS，但他们对自己的知识产权的重视程度远远超过 FOSS，并且满足于忽视和侵蚀 FOSS 用户的权利，而不是自己的。</p><p>3.<strong> 微软 /GitHub 能否提供一份用于 Copilot 的训练集的许可证的清单，包括版权人的名字和 / 或 Git 存储库的名字？</strong>如果不能，那么为什么不向社区提供这些信息呢？</p><p>SFC 认为，GitHub 拒绝回答的原因是他们没有仔细复制他们的模型，所以他们实际上不知道他们侵犯了谁的版权，以及何时和如何侵犯。</p><p>综上，这些不仅问题被挡了回来，最终，GitHub 的做法，导致 SFC 愤而 " 暴走 "。</p><p><strong>Copilot 的合法性是值得商榷的吗？</strong></p><p>在宣布 " 离家出走 " 后，SFC 的最新声明也在 Hacker News 上引来了 400 多位开发者的热评，针对 SFC 铿锵有力的指责，GitHub 推出商业化的 Copilot，其合法性是否真的有待商榷呢？</p><p>一名开发者指出，SFC 提出的关于 "<strong>为什么微软 Windows 和 Office 代码库不在你的训练集里？</strong>" 是他最喜欢的问题。</p><p></p><div class="img_box"
