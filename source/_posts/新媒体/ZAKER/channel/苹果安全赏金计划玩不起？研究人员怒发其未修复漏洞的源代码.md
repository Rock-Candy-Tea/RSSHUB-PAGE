上这些漏洞进行攻击的源代码。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" </strong></p><p>通过这个漏洞可以允许访问用户以下数据：</p><p><strong>苹果 ID 帐户的电子邮件， 苹果 ID 所有者的全名，以及授权认证，允许以苹果 ID 的名义向苹果服务器发送请求</strong>。</p><p>访问下面文件的内容 :</p><p>/var/mobile/Library/CoreDuet People/ interactionC.db ——包含<strong>邮件、短信和其他通讯 APP（例如 WhatsApp 等）的联系人列表 , 以及有关与这些联系人的借用的元数据，包括每次与每个联系人交互的统计信息和确切时间</strong>。</p><p>/var/mobile/Library/Preferences/com.apple.mobilephone.speeddial.plist ——<strong>包含手机应用程序中的快速拨号联系人列表。</strong></p><p>/var/mobile/Library/AddressBook/AddressBook.sqlitedb ——<strong>包含一个完整的通讯录，上面有设备上所有的联系人。</strong></p><p>/var/mobile/Library/AddressBook/AddressBookImages.sqlitedb ——<strong>包含通讯录中联系人照片。</strong></p><p><strong>2、Nehelper installed apps 0-day ( iOS 15.0 ) </strong></p><p>允许通过捆绑 ID<strong>检查设备上是否安装了应用程序</strong>。</p><p><strong>3、Nehelper wifi info 0-day ( iOS 15.0 ) </strong></p><p>一个轻微的漏洞，<strong>允许访问有关设备当前连接的 Wi-Fi 热点的信息</strong>。</p><p><strong>4、Analyticsd（在 iOS 14.7 中修复）</strong></p><p>允许访问设备上的分析数据，数据包括：</p><p><strong>有关使用设备的信息</strong>（用户拿起设备的次数、推送通知的到达次数以及用户对它们的反应）</p><p><strong>屏幕使用时间信息</strong>（用户在每个应用程序中花费的时间以及他们打开这些应用程序的次数）</p><p><strong>医疗信息</strong></p><p><strong>来自 " 健康 " 应用的元数据</strong></p><p><strong>有关所有与设备（如 AirPod）相关的配件信息</strong></p><p><strong>有关应用程序崩溃信息，包括其捆绑 ID 和导致崩溃的异常代码</strong></p><p><strong>用户在 Safari 中查看的页面的语言</strong></p><p>据其他分析过这些漏洞的研究人员说，这篇博客文章和源代码让其他安全研究人员以及恶意黑客有能力复制这些未修补的漏洞并加以利用。</p><p><strong>令人不满的苹果安全赏金计划</strong></p><p>此外，托卡列夫也在博客中透露了关于苹果漏洞赏金计划的一些细节，他在博客中写道， <strong>参与苹果安全赏金计划是 " 一次令人沮丧的经历 "</strong>。</p><p>" 今年 3 月 10 日至 5 月 4 日，我报告了 4 个零日漏洞<strong>（编者注：" 零日漏洞 " ( zero-day ) 又叫零时差攻击，是指被发现后立即被恶意利用的安全漏洞。通俗地讲，即安全补丁与瑕疵曝光的同一日内，相关的恶意程序就出现。这种攻击往往具有很大的突发性与破坏性）</strong>，到目前为止，还有三个漏洞仍然出现在最新 iOS 版本 ( 15.0 ) ，但是苹果决定掩盖它们，而不是将这些问题列在安全内容列表页面。当我质问他们时，他们向我道歉，并向我解释这是由于处理过程不当导致的问题，并承诺将其列在下次更新的安全内容页面上。从那以后，他们发布了三次安全内容列表页面，每次都食言。十天前，我要求苹果给我一个解释，并警告苹果说，如果我得不到一个合理的解释，我将公开我的发现。我的请求被忽略了，所以我决定公开这件事。"</p><p><strong>托卡列夫称，在苹果安全赏金页面上，他所报告的漏洞的价值为 100,000 美元。</strong>（Broad app access to sensitive data normally protected by a TCC prompt or the platform sandbox）</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div
