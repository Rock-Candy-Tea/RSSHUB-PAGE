<p>硬件门槛让不少人对 Windows 11 吐槽，可当下市场上也出现了各种硬装 Windows 11 的方法，真的要不管不顾的尝试吗？</p><p><strong>硬上 Windows 11 会受到蓝屏死机的困扰</strong></p><p>微软已经确认只有部分计算机能平滑升级到 Windows 11，虽然这一决定颇具争议，但该公司此后一直在夜以继日地工作，试图弄清更多采用旧处理器的 PC 是否能够处理新的操作系统。而且，虽然支持的处理器名单的一些小规模添加已经发布，但微软表示可以确定，旧的 CPU 在 Windows 11 上出现崩溃，也就是说蓝屏死机的可能性会增加。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202108/612dfa6e8e9f093ec044ff02_1024.jpg" data-height="540" data-width="641" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202108/612dfa6e8e9f093ec044ff02_1024.jpg" referrerpolicy="no-referrer"></div></div>图片来源：微软官方网站<p></p><p>这并不一定令人惊讶，微软说其中一个原因是旧的驱动程序支持，在 Windows 11 这样的现代操作系统上特别麻烦。</p><p>" 随着时间的推移，保持可靠性与 OEM 和 IHV 驱动支持高度相关。Windows 11 上支持的处理器在 OEM 和 IHV 的支持范围内，并使用现代（DCH）驱动程序。该公司解释说：" 转向现代驱动使驱动和相关软件能够通过 Windows Update 以协调的方式进行安装和服务，并提供更好的机制来跟踪驱动健康状况。从运行 Windows Insider 计划中的构建的 PC 上收集的数据使其非常明显，如果他们运行新的操作系统，会有 BSOD 的重大风险。"</p><p>" 这种协调的结果是，系统驱动程序在更新后正确安装并发挥作用，在升级到 Windows 11 时提供可靠的体验。从 Windows Insider 机器来看，那些不符合最低系统要求的机器，其内核模式崩溃（蓝屏）比符合要求的机器多 52%。符合要求的机器提供了 99.8% 的无崩溃体验，OEM 和 IHV 通过现代驱动更新管理进行有效管理。此外，在不支持的硬件上，应用程序出现卡死的可能性增加了 17%，对于第一方应用程序，我们看到崩溃的可能性增加了 43%。"</p><p>Windows 11 已经处于最后的开发阶段，该操作系统预计在 9 月正式签署 RTM 版本，然后从 10 月开始向第一波生产设备发货。对符合条件的计算机的推广也将持续到 2022 年。</p><p><strong>愈发混乱的系统需求</strong></p><p>目前看 Windows 11 系统的升级情况一片混乱。即使很多用户能安装新系统，也会被告知自己的电脑配置不够新不够好。今年 6 月份，有报道称由于数百万台 PC 的处理器并没有所要求那么多功能，因此可能无法更新到微软全新的 Windows 11 系统。</p><p>不过，虽然听起来微软似乎是要让 Windows 11 兼容更多设备，但系统升级的现实并没有发生太大变化。虽然 PC 游戏玩家、IT 管理员和其他高级用户能够在现有硬件设备上安装 Windows 11，但微软主旨仍是鼓励数百万人更换他们手中的 PC。</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres1.myzaker.com/202108/612dfa6e8e9f093ec044ff03_1024.jpg" data-height="427" data-width="641" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202108/612dfa6e8e9f093ec044ff03_1024.jpg" referrerpolicy="no-referrer"></div></div>想象一下，普通用户从未听说过 ISO，只是希望在今年秋天将老旧电脑升级到最新的 Windows 11 操作系统，首要想法就是搞清楚自己的机器能否运行 Windows 11。这样一来，用户就会启动微软所宣传的 PC 健康检查应用程序，但这个工具很大程度上会误导用户。微软表示，PC 健康检查应用程序仍然会对照微软支持的处理器列表检查用户设备的 CPU，如果用户设备配置中没有搭载微软推荐的芯片，应用程序就会引导用户阅读 " 包括潜在补救措施的相关支持文章 "。<p></p><p>当然，大多数 Windows PC 用户可能不会花心思使用 PC 健康检查应用程序。他们只会等待 Windows 自动推送免费更新，就像微软 2015 年发布 Windows 10 操作系统时提供的免费更新一样。但在 Windows 11 发布后，微软只会更新那些 CPU 能兼容 Windows 11 的电脑。</p><p>微软的做法也有一定道理：它不想因为 Windows 操作系统的用户体验变差而受到指责，微软也无法在测试中遍历世界上每一种 PC 硬件配置。</p><p>微软表示，当硬件配置无法满足 Windows 11 系统的最低要求时，电脑有 52% 的概率会出现蓝屏。如果能满足 Windows 11 系统硬件配置要求，机器不会出现蓝屏的概率为 99.8%。对于管理数千台旧电脑的系统管理员来说，多一点停机时间就不可接受；但对于每 4 到 6 年才买一台新电脑的普通用户来说，情况完全不同。</p><p><strong>重新发布 Windows 11 兼容性检查工具</strong></p><p>为照顾想要升级 Windows 11，但又不确定当前硬件和固件是否支持的客户，微软已经重新发布了 Windows 11 兼容性检查工具。据悉，微软于 2021 年 6 月份正式宣布了 Windows 11 操作系统，并计划于今年晚些时候正式向广大用户提供。至于新版 Windows 11 兼容性检查工具，它现在能够更加轻松地判断您的设备是否与之兼容。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy 系列及更高版本硬件的平台，都可无缝支持 Windows 11 操作系统。<p></p><p>此外根据来自 Windows Insider 测试项目的遥测数据，微软最近又添加了对英特尔 7 代芯片的支持，具体包括如下三类：</p><p>● 酷睿 X 系列 HEDT 处理器；</p><p>● 至强 W 系列处理器；</p><p>● 以及酷睿 i7-7820HQ 处理器（用于微软 Surface Studio 2 等设备）。</p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_3" Surface 产品线，也被拦在了 Windows 11 的升级大门之外。<p></p><p>当然，就算微软坚持为新一代操作系统强制指定 TPM 2.0 支持，后续仍不排除有奇妙的漏洞允许用户在任何设备上安装体验。</p><p>早些时候，该公司因为由部分用户抱怨无法正常工作、而下架了旧版的 "PC 健康检查工具 "。</p><p></p><div class="img_box"
