X，我们让荣耀手机的性能体验位列翘楚。如今，荣耀将把手机端的成功经验运用于 PC，基于笔记本底层调校的 Magic OS for Windows 即将与大家见面！让科技理想主义照进 PC，打造属于自己的 Magic OS 新生态！无限可能，敬请期待！</p><div class="img_box" id="id_imagebox_1" onclick><div src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202205/627634b3622768299b0008ad_1024.jpg" referrerpolicy="no-referrer"></div></div><p></p><p>从赵明的微博中不难看出，荣耀想在笔记本电脑上复制手机的成功。必须承认，荣耀手机是非常成功的。而这种成功的背后也离不开软硬件的深度调教优化。经典案例便是荣耀在 2018 年推出的 GPU Turbo 技术。</p><p>彼时的荣耀还未脱离华为，由于华为的麒麟芯片在 GPU 性能上略弱。于是，荣耀就通过底层优化让 CPU 和 GPU 之间的指令传输效率大幅度提升，最终让荣耀手机在几乎不损失游戏画质的前提下，游戏帧数大幅提高，GPU Turbo 技术在荣耀 Play 手机上进行了首发。当时 GPU Turbo 以吓人的技术而闻名。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres2.myzaker.com/202205/627637258e9f09494570337d_1024.jpg" data-height="733" data-width="1200" 科技却认为，这一次，荣耀要想复制手机上 GPU Turbo 的成功却没那么容易。<div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_3" 的内核代码是不开源的。这也就意味着荣耀可能无法对 Window 系统的底层进行修改和调教，</strong>所以对赵明表示的基于笔记本底层调校，ZAKER 科技持保留态度。</p><p>如果荣耀无法修改 Windows 的内核代码，那么 Magic OS for Windows 的实际意义可能会非常有限，因为内核已经在很大程度上决定了操作系统的运行效率，想要在不修改内核的情况下大幅提高系统运行效率，这非常困难。</p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " 这个表述本身也存在争议，因为 Windows 本身已经是一个独立的操作系统，但荣耀又在这前面加多了 Magic OS，OS 本身也是操作系统的意思（Operating System）这多少让人觉得摸不着头脑。或许叫做 Magic UI 更合适。</p><p>如果荣耀选择在 Windows 系统上做二次开发，那么多少有点画蛇添足的感觉，因为这必将占用额外的硬件资源，反而有可能降低系统运行效率。</p><p>社群里的不少朋友都向 ZAKER 科技表示，他们觉得官方纯净版的 Window 系统是最好用且最稳定的，对于第三方 OEM 定制系统并不喜欢。更有人表示，买了品牌 PC 回来第一件事就是重装纯净版系统。</p><p>这种情况下，荣耀所谓的 Magic OS for Windows 还有多大的价值就是一个值得商榷的问题。</p><div class="img_box" id="id_imagebox_5" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_5" Windows，实现多屏协同、一碰连、隔空传等高级功能。</p><p>在这方面做得最好的显然是苹果的 MacBook 和 iPhone。但苹果的芯片和 MacOS 以及 iOS 都是自己研发的，这种优势荣耀显然不具备。回过头来想想，荣耀当年的 GPU Turbo 之所以能够大获成功很大程度上也得益于华为自家的麒麟芯片。</p><div class="img_box"
