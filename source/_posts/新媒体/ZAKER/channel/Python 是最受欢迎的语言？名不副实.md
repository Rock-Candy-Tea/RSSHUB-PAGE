<p>【CSDN 编者按】如果说有哪种语言是初学者的最爱，那么 Python 肯定榜上有名。C++ 之父 Bjarne Stroustrup 说过：" 世界上只有两种编程语言：一种是总是被人骂的，一种是没人用的。" 这不，虽然 Python 成为了世界上最受欢迎的编程语言，但是技术分析师 Christopher Tozzi 却认为，Python 名不副实。这是为什么呢？</p><p>译者 | 章雨铭 责编 | 屠敏</p><p>对于 Python 来说，时代从未如此美好。TIOBE 指数的数据显示，<strong>Python 是目前世界上最受欢迎的编程语言，</strong>过去二十年也一直处于增长趋势。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div ，而现在已经发展成最通用的语言之一了。从 Web 开发到物联网编程，再到 AI 等各个方面都能用到它。另外，Python 代码非常容易编写和执行，这使 Python 成为入门编程语言的首选。基本上每个在大学学习编码或者参加编程训练营的人，都会学习 Python。<p></p><p>然而，我想知道 Python 的鼎盛时期会持续多久。虽然语言本身没有什么问题，但是从某些方面来说，我对它能成为 " 万人迷 " 感到很惊讶。</p><p>我认为它受到的欢迎程度远超出它所应得的。为什么呢？主要有以下四个原因。</p><p><strong>Python 速度不快</strong></p><p>可以说，Python 最大的缺陷就是<strong>用它编写的应用程序运行速度不快。</strong>至少，这些应用程序远不及用 C 或者 Java（本身不是特别快的语言）等语言编码的应用程序快。</p><p>因此，我担心用 Python 编写这么多代码可能会像搬起石头砸自己的脚。代码可能易于编写和部署，但是我们却牺牲了速度、效率和性能。在这个分秒必争的世界里，Python 显然不是一个很好的选择。</p><p><strong>Python 的语法过于僵化</strong></p><p>对于编码小白来说，Python 如此受欢迎的部分原因是它的语法非常具体，所以写出来的代码非常整洁而且可读性高。</p><p>对于那些不介意花时间研究 Python 所有语法规则的人来说，Python 很不错。但是，如果你只是想快速生成代码，那 Python 并不是最好的选择。</p><p>因此，<strong>如果优先考虑灵活性或者动态性</strong>，而不是让代码看起来美观且一致，那么 Python 可能就不是未来最好的语言。</p><p><strong>Python 提供的编程语言有限</strong></p><p>易于使用是 Python 受新手小白喜爱的另一大原因。</p><p>但是<strong>实际上 Python 没有提供全面的开发体验</strong><strong>。</strong>学习 Python 编码的人并没有了解传统软件开发系统的各个方面。</p><p>虽然这听起来可能有些苛刻，但是我认为知道如何用 Python 写代码只不过比知道如何在 Bash shell 中运行程序更高级一点，而 Bash shell 根本不算编码。</p><p>从这个角度来说，Python 正在创造一代不知道 " 完整 " 编程的程序员，他们只知道如何编写解释型代码。</p><p><strong>没有什么大型的产品是用 Python 写的</strong></p><p>当环顾全球各种项目时，<strong>我发现没有任何真正大型、复杂且重要的应用程序或者平台是用 Python 写的。</strong></p><p>当然，很多网站使用 Python，主要是在后端，可能有数百万小型应用程序是用 Python 开发的。但是没有基于 Python 的类似 WordPress 的东西（尽管有一些基于 Python 的 CMS，其中大部分你可能都没有听说过），没有非常依赖 Python 的办公生产力套件，当然也没有用 Python 编码的操作系统，甚至是操作系统的主要部分。</p><p>这并不是在谴责 Python，只是在强调这样一个事实。如果说 Python 在未来几十年仍然保持顶级编程语言的地位，那么你会认为现在有人会用它做一些比编写脚本和简单应用程序更有趣的事。然而他们没有，可能也不会。</p><p><strong>结论</strong></p><p>诚然，以上我对 Python 的观点都是主观的，很多人比我更看好 Python。我并不是说永远不应该使用 Python，它当然有很多用处，但主要用于编写相对简单的脚本和应用程序。所以它并不是编写所有内容的首选语言。</p><p>毕竟，Python 最初设计为一种 shell 脚本语言。但现在已经变成一种非常受欢迎的语言，我认为这种趋势是一种反常现象。十年后，我怀疑，当我们回顾过去的时候，会思考我们让 Python 成为世上最流行的编程语言时，我们在想些什么。</p><p>你对此有什么看法，欢迎留言告诉我们。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
