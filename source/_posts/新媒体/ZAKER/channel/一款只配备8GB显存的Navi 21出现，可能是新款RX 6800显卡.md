<p>目前 AMD 使用 Navi 21 GPU 的显卡包括 RX 6900XT、RX 680XT、RX 6800 均配备 16GB 的 GDDR6 显存，没有其他显存容量的产品，而配备 12GB 显存的 RX 6700XT 用的是 Navi 22 GPU，发布的时候看起来 AMD 并没有给 Navi 21 配 8GB 显存的意思，但现在事情好像又有了点转机。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres2.myzaker.com/202108/610d0ae48e9f0960bc215f45_1024.jpg" data-height="608" data-width="1080" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202108/610d0ae48e9f0960bc215f45_1024.jpg" referrerpolicy="no-referrer"></div></div>推特用户 @Komachi_Ensaka 在 UserBenchmark 发现了一款未发布的 AMD Radeon 系列显卡，它的设备 ID 是 1002 73BF，这明显是 Navi 21 的衍生物，而子系统 ID 则是 1002 0E3A，与 AMD Radeon RX 6800 匹配，但它只有 8GB 的显存。<p></p><p>其实 AMD 要出 8GB 的 Radeon RX 6800 显卡传闻已久，但这东西官方从来没有证实过，而大容量显存一直是 AMD RX 6000 系列显卡的卖点之一，对手 NVIDIA 除了最顶级的 RTX 3090 之外，没一款产品显存容量是超过 16GB 的，但至于 AMD 现在为啥要弄个 RX 6800 8GB 出来，可能是想在 RX 6800 与 RX 6700XT 之间再弄一档产品出来。</p><p>想把 RX 6800 16GB 变成 RX 6800 8GB 有两种方法，一个是只开启 Navi 21 GPU 的一半显存接口，这样用四颗 16Gb GDDR6 显存就能实现 8GB 的容量，代价自然是显存带宽直接减半，性能损失非常大，我不认为 AMD 会在高端 GPU 上这么干。第二个方法自然是把限制的 16Gb GDDR6 显存换成 8Gb GDDR6 的显存，这样显存带宽不变，只是容量减半。</p><p>个人认为这款 RX 6800 8GB 在零售市场出现的可能性不大，主要是没啥必要在 RX 6800 与 RX 6700XT 之间再插一款产品出来，但 AMD 把它投向 OEM 市场的可能性非常大。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
