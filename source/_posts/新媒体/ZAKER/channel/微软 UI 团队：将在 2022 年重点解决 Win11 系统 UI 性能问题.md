<p>IT 之家 11 月 19 日消息，在近期的微软反馈中心上，有用户抱怨在 Windows 11 中使用含有 WinUI 或者 XAML UI 元素的应用程序时自己的电脑会出现一些画面方面的性能问题，比如在 Windows 11 Dev 预览版上使用 Windows 文件资源管理器时会随机出现界面崩溃或冻结等情况。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202111/61974c048e9f09423b4dc3f6_1024.jpg" data-height="107" data-width="838" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202111/61974c048e9f09423b4dc3f6_1024.jpg" referrerpolicy="no-referrer"></div></div>IT 之家通过微软反馈中心了解到，此外还有一位用户表示在使用 OneDrive 同步文件夹时，软件界面整体的流畅度下降的很明显，且应用的反应相当滞后，使用 F11 按键全屏显示虽然能缓解这个问题，但这并不是长久之计，微软需要改善 Windows 画面组件的性能。"<p></p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres2.myzaker.com/202111/61974c048e9f09423b4dc3f7_1024.jpg" data-height="90" data-width="826" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202111/61974c048e9f09423b4dc3f7_1024.jpg" referrerpolicy="no-referrer"></div></div>对此情况微软官方表示，目前微软在一直努力增强 Dev 预览通道 Windows 11 的性能表现，并将在 Build 22478 版本中对此做出改进，以解决系统出现非必要计算的情况（该情况会出现在用户选择文件的时候）。<p></p><p>此外，微软 UI 团队也计划在 2022 年将重点改进系统 UI 的性能表现。在微软官方的 Reddit 帖子下方有用户抱怨在使用 Windows 11 中含有 WinUI 或 XAML 元素的程序会出现画面不流畅的情况，微软随即表示该该问题将在明年的 Windows App SDK 1.1 版本中进行改进。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" data-original="http://zkres1.myzaker.com/202111/61974c048e9f09423b4dc3f8_1024.jpg" data-height="305" data-width="731" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202111/61974c048e9f09423b4dc3f8_1024.jpg" referrerpolicy="no-referrer"></div></div><p></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
