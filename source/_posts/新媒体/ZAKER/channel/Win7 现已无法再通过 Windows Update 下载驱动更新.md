<p>IT 之家 6 月 18 日消息 众所周知，微软已于 2020 年 1 月停止了对 Windows 7 的支持。</p><p>微软现（2021 年 6 月 17 日）已停止发布适用于 Windows 7 SP1、Windows Server 2008 和 Windows Server 2008 R2 的 Windows Update 驱动程序。</p><p>IT 之家提醒： 此次是因为 SHA-1 证书到期，无法再提供 Windows 更新的发布选项。</p><p>也就是说，这有可能导致功能降级或导致设备启动时间更长，主要还是是因为未打补丁的系统在使用 SHA-2 签名驱动程序时会出现代码完整性问题。</p><p>为了避免以及降低不兼容的潜在影响，微软决定直接停止提供更新。这包括：</p><p>（开发者）为当前支持的 Windows 和 Windows Server 版本的多目标提交的任何驱动程序包</p><p>任何针对已终止支持的 Windows 或 Windows Server 版本的驱动程序包。</p><p>从此之后，驱动程序提交者将需要在删除不受支持的系统驱动版本后向微软重新提交以重新发布。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202106/60cbf2588e9f09401145fa47_1024.jpg" data-height="163" data-width="289" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202106/60cbf2588e9f09401145fa47_1024.jpg" referrerpolicy="no-referrer"></div></div>值得一提的是，用于 Windows 硬件兼容性计划 ( WHCP ) 的 Windows 7、Windows Server 2008 和 Windows Server 2008 R2 驱动程序提交将继续提供到 2023 年 1 月。<p></p><p>这些版本的系统将继续提供支持，以确保为批量许可的企业客户提供可靠性已及扩展安全更新 ( ESU ) 计划。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
