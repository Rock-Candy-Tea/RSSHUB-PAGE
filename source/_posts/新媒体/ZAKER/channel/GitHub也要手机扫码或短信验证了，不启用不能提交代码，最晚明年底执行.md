<p>GitHub 现在很焦虑，因为针对开源软件的黑客攻击越来越多了。</p><p>他们统计了一圈所有账号的安全设置后，发现了一个情况：只有<strong>16.5%</strong>的用户启用了<strong>双重身份认证</strong>功能。</p><p>现在 GitHub 正式宣布：</p><p>要求所有代码贡献者在 2023 年底之前启用双重身份认证。</p><p>换句话说，<strong>要是不启用这个功能，以后就不能往 GitHub 仓库里提交代码了</strong>。</p><p>所谓双重身份认证（Two-Factor Authentication），就是在账号密码以外还额外需要一种方式来确认用户身份。</p><p>国内这种做法已经很常见，比如手机 App 扫码，或者接收短信验证码。</p><p>具体到 GitHub 的做法由于没有自己的手机客户端，而是支持使用第三方验证工具如 1Password 或微软的 Microsoft Authenticator。</p><p>至于短信验证码也不是所有手机号都能收，比如<strong>我们的区号 +86 就不支持</strong>……</p><p></p><div class="img_box" id="id_imagebox_0"
