协议强迫用户使用 Edge 浏览器打开链接。微软的做法很实在，所以用户也很实在，于是开发出一个名为了 Edgedeflector 的第三方应用，它可以更改捆绑的 microsoft-edge: URI 协议，让用户可以自己设置默认浏览器，而不是只能使用 Edge，例如你可以用 Chrome、火狐等浏览器应用打开内置软件的链接。<p></p><p></p><div class="img_box" 更新之后会让这个第三方应用以及类似的应用程序失效，在 Win11 的 Build 22494 中，微软修改了 microsoft-edge:// 协议，使得该协议无法被挟持，从根源上杜绝了这些工具的正常运作。<p></p><p>这一行为导致了大量用户的不满，对此微软则认为他们仅仅是在 " 修复漏洞 "，所做的一切都是为了完善用户体验。微软的发言人在回应中称，Windows 10 和 Windows 11 中提供了大量的 " 端到端的客户体验 "，任务栏的搜索和 Win 11 的新闻小组件都是这样的端到端体验，这些都不是为了重定向至第三方浏览器而设计的。</p><p>" 当我们意识到不正确的重定向时，会发布修复程序来修复它 "，微软的发言人如是说。</p><p>这里发言人讲到的 " 修复 " 是即将到来的 Windows 11 更新的一部分，这项更新将在本周五晚间向 Beta 和 Release Preview 用户推送。至于修复之后，用户能否修改浏览器的相关设置还不得而知。</p><p>值得注意的是，火狐方面已经察觉到微软浏览器政策的改变，于是宣布 Firefox 浏览器即将上架 Microsoft Store。目前，Firefox 浏览器需要前往 Mozilla 官方网站进行下载安装，Opera 浏览器也于近日上架 Microsoft Store，值得注意的是，虽然该应用被列在商店中，但它的管理和更新是独立于微软的。</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" 中推出。</p><p>从目前来看，似乎还没到 " 微软封杀第三方浏览器 " 的地步，但微软确实在加大 Edge 浏览器的推广力度，例如打开链接时，默认启用的是 Edge 浏览器。</p><p>所以，这合理吗？</p><p><strong></strong></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
