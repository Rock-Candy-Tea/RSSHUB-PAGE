<p>自从 Microsoft Edge 转以 Chromium 内核为基础进行产品开发后，Edge 更新的脚步可以说是越来越快，在功能和用户体验上甚至已经有了赶超 Google Chrome 的迹象。</p><p>上周 Edge 和 Chrome 又迎来了一次稳定版更新并带来了一系列的改进和优化，在二者即将从 6 周一次的平稳更新节奏跃入 4 周一次的「快车道」之前，让我们一起来看看本次更新的主要内容吧。</p><p><b><strong> Edge 93 </strong></b></p><p><b><strong>▍</strong><strong>更沉浸的垂直选项卡模式</strong></b></p><p>垂直选项卡模式是 Edge 特有而 Chrome 所不具有的功能，可以一定程度上解决选项卡过多时无法完整查看网页标题名称的问题，不过这样也会造成长长的标题栏仅显示当前网页的标题和三大金刚按钮，对于屏幕显示面积本就有些捉襟见肘的笔记本电脑用户有些浪费显示空间。</p><p>针对这一点，本次 Edge 更新推出了垂直选项卡模式下隐藏窗口标题栏的选项。</p><p></p><div class="img_box" id="id_imagebox_0" 在本次更新中重新上线了标签页分组功能，对早期基于 Chromium 自带的标签页分组功能的交互与视觉设计做了改进优化，其中主要变化为增加了圆角矩形的半径、加入了可以提示折叠或展开状态的小三角形等。</p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div 键并选中多个标签页而后单击右键来将它们快速创建、加入新的标签组。对于标签组，我们可以为其命名、指定颜色来与其他标签组进行区分，同一组的标签页将在最左侧以一个圆角矩形的标识（横向状态）或是最顶部以带有小三角形的圆角矩形（纵向状态）来显示该组的名字与颜色。<p></p><p></p><div class="img_box" id="id_imagebox_4" onclick><div 的视觉设计则是在同一分组的标签页底部显示颜色线条，像丝带一样包围着当前活动的标签页，更能突出显示正在看的标签页，而 Edge 的分组中当前正在看的标签页在视觉上与组内其他标签页的对比度不够高，有些不够明显。</p><p>在实际的应用场景中，我们可以根据手中所做的事情来对标签页们进行分类，比如上班时将与工作相关的页面分成一个组，休息时想看的页面分成另一个组，从而实现高效工作，开心摸鱼。</p><p><b><strong>▍</strong><strong>一键开启视频画中画模式</strong></b></p><p>在某些休闲场景下，我们可能希望能够一边刷着社群平台的信息流一边追新剧，画中画模式就能很好地解决这个问题。老版本的 Edge 中，如果需要启用网页视频画中画功能，我们往往需要在视频上进行两次单击右键后才能在右键菜单中看到开启画中画模式的选项。</p><p>在新版 Edge 中，这个操作变得更加快捷便利：当光标移动视频窗口底部触发播放控制 UI 时，视频顶部会出现一个 PiP 按钮，点击即可启动画中画模式。</p><p></p><div class="img_box" id="id_imagebox_6" 中推出，如果你在更新后没有看到相关选项，则需要手动开启 edge://flags/#global-media-controls-picture-in-picture 和 edge://flags/#global-media-controls 两个功能标签。另外正如上面所说，这个画中画按钮只有在鼠标移动到视频底部时才会出现，相比之前的方案减少了触发画中画界面所需的点击次数，却增加了鼠标移动轨迹，后续显然还需要优化一下。<p></p><p><b><strong>▍</strong><strong>更加丰富的初始化首选项定制</strong></b></p><p>针对政企商用户群，新版的 Edge 上线了 Initial Preference 功能，即支持首次启动时批量定制 Edge 中一定数量的首选项。在初次部署的时候，政企商单位的 IT 管理员可以为其管理的设备上所安装的 Edge 浏览器批量设定默认设置，其中包括设定浏览器主页、工具栏是否显示一系列图标、是否启用暗黑模式等等，这为 IT 管理员带来了便利，提高了工作效率。</p><p>在具体进行操作时，他们需要前往针对商业用户的 Edge for Business 页面下载名为「initial_preferences」的示例配置文件，然后针对所需的定制项改动该配置文件中相应的 JSON 代码，确认无误并保存后将配置文件部署至所有用户设备的特定文件目录下，完成以上步骤后在下一次用户启动 Edge 之时，相应的定制选项便会即刻生效。</p><p><b><strong>▍</strong><strong>IE 模式下的网页将采用「不合并」策略进行加载</strong></b></p><p>相比于 Chrome 而言，Edge 的一大特色便是它的 IE 模式了，这也是为了照顾那些不得不使用古老的、仅针对 IE 浏览器开发的网站的用户。而从 Edge 93 版本开始，其中的 IE 模式将默认以「no-merge（不合并）」模式启动，这意味着当一个新的 Edge 窗口以 IE 模式启动时，它将开启一个新的、独立于原有 Edge 窗口之外的会话进程。</p><p></p><div class="img_box" id="id_imagebox_7" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_7" data-original="http://zkres1.myzaker.com/202109/613c35f18e9f090a2633ac80_1024.jpg" data-height="514" data-width="1080" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202109/613c35f18e9f090a2633ac80_1024.jpg" referrerpolicy="no-referrer"></div></div>IE 模式选项卡拥有独立的任务进程<p></p><p>之后在上述的新窗口打开其他网页，将会同样以 no-merge 策略下的 IE 模式打开，且同样独立于原有的 Edge 进程；若要结束这种机制，则需要确保该窗口内所有以 IE 模式打开的网页都已被关闭。</p><p><b><strong> Chrome 93 </strong></b></p><p><b><strong>▍</strong><strong>PWA 程序变得更像本地应用</strong></b></p><p>PWA（渐进式 Web 应用）由于其轻量的特性受到不少用户的欢迎，在新版 Chrome 中，它们与本地安装的应用程序之间的差距被进一步弥合。</p><p>Chrome 93 允许我们通过特定的 URL 链接直接打开 Chrome 中已经安装的 PWA 程序，例如若我们在 Chrome 中安装了 YouTube PWA，那么点击所有相关域名链接都可以直接通过该 PWA 打开，而不是作为新的标签页打开，就如同跳转使用本地应用那样。</p><p>作为网页版 Gmail 用户，相信这将极大地改善平时使用 Gmail 的体验，比如直接点击网页上的 email 地址便能直接启动 Gmail PWA 应用开始撰写邮件。</p><p><b><strong>▍</strong><strong>更好的应用程序多屏与多窗口支持</strong></b></p><p>随着许多人为自己的计算机使用环境配备了多台显示器或是开始使用多屏设备，多屏状态下的软件用户体验也变得越来越重要。Chrome 93 为开发者加入了数个新的多屏幕信息 API，并对现有的窗口放置 API 进行了改进，从而使得 Web 应用的开发者能够利用多屏幕所带来的更大的视觉空间，为 Web 应用程序带来更高的效率以及更佳的用户体验。</p><p>在支持了这些 API 后，一系列网页端的应用都能更充分地利用多屏显示空间来进行多窗口显示，例如：</p><p>幻灯片放映程序在放映时能够在投影仪画面显示幻灯片内容，副屏幕显示演讲者备注提示</p><p>创意插画类程序在主屏幕显示画布，在副屏幕显示调色板及绘画工具</p><p>开会展示内容时在电视机上播放开会内容，在触摸屏上显示播放控制按钮</p><p>交易系统通过多个窗口平铺来展示股市的实时走势</p><p><b><strong>▍</strong><strong>SVG 文件也能自如地复制粘贴</strong></b></p><p>SVG 格式的图像文件由于其能够在空间中对图像进行有效地编码并且在放大时依然能够保持图像的显示质量而正被越来越广泛地应用，更新后的 Chrome 剪贴板自然也增加了对这种图像格式的支持。此后，我们便能够在矢量图设计软件与一些设计工具类的网页间对 SVG 格式的图像文件进行复制与粘贴操作。</p><p>对于设计师来说，当在某个网站上发现了自己喜欢的 svg 图像素材，他们便能直接通过简单的复制与粘贴操作将其加入诸如 Adobe Illustrator 一类的本地应用中，抑或是将其粘贴加入诸如 Figma 的网页端应用程序中。</p><p><b><strong>▍</strong><strong>更好的验证码自动填写体验</strong></b></p><p>出于安全性因素的考量，许多网站对账户登录行为都默认开启了两步验证，这也让用户在登录自己的帐号时多了一步手动输入一次性验证码进行身份验证的操作，有些麻烦。</p><p>在当前的 Android 手机上，系统已经能够自动识别短讯息中的一次性验证码并进行自动填写以减少用户的操作成本。Google 在 Chrome 93 中为其加入了桌面端的 WebOTP API，这在未来将使得一次性验证码能够在不同的设备间同步（主要是从移动端自动同步至桌面端），并于桌面端 Chrome 中进行自动填写，为我们省去一个比较烦人的步骤。</p><p>当然了，若要实现这一功能，我们需要在桌面端 Chrome 和 Android 端 Chrome 上登入同一个 Google 账号，并且也需要开发者配合添加支持。</p><p><b><strong>▍</strong><strong>允许网页应用申明为笔记应用</strong></b></p><p>在 Chrome 93 中，Web 应用可以将其本身识别为笔记记录应用，这类应用可以直接通过 URL 链接进行打开，体验和 PWA 非常相似，从特性和使用场景来说更加适合 Chrome OS。</p><p>不过目前还没有适配了这一特性的笔记应用出现。</p><p><b><strong>▍</strong><strong>小结</strong></b></p><p>除了上述的一些功能优化，两个浏览器的本次稳定版更新还带来了一些安全性上的改进，本文就不详细展开了。</p><p>Edge 和 Chrome 在功能改进与开发节奏上「你追我赶」，受益的自然还是我们这些用户。两个优秀的浏览器你选择哪一个作为主力使用？欢迎在评论区分享你的使用体验和看法。</p><p><strong><strong><strong></strong></strong></strong></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
