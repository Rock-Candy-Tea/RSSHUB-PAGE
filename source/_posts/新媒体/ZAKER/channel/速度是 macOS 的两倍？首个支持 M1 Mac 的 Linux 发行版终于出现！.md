<p>整理 | 郑丽媛</p><p>出品 | CSDN</p><p>2020 年 11 月，苹果自研芯片 M1 的推出令业界轰动。彼时，M1 芯片强大的性能令许多 Linux 爱好者期待：未来 M1 Mac 可否顺利运行 Linux？</p><p>在当时 Linux 之父 Linus Torvalds 认为因 "M1 芯片的 GPU 及其周围相关设备没有任何 Linux 支持 " 而 " 不太可能 " 实现让 M1 Mac 运行 Linux 的情况下，<strong>知名开发者 Hector Martin（即 Marcan）" 迎难而上 "，启动了一项向 M1 Mac 移植 Linux 的项目</strong>。</p><p>经过一年多的摸索，如今<strong>该项目终于取得了重大进展，我们也似乎迎来了第一个支持苹果 M1 系列 Mac 的 Linux 发行版</strong>：</p><p>3 月 18 日，由 Hector Martin 领导的 Asahi Linux 团队宣布<strong>其发行版已成功在 M1 MacBook Air 笔记本电脑上运行，并由此发布了 Asahi Linux 的 Alpha 版本</strong>。</p><p></p><div class="zaker_div"><div class="zk_decsimg zk_alignleft"><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres2.myzaker.com/202203/623965028e9f0904803a9ef2_1024.jpg" data-height="800" data-width="800" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202203/623965028e9f0904803a9ef2_1024.jpg" referrerpolicy="no-referrer"></div></div><p></p><p>（图片来自维基百科）</p></div><div class="edi_oper"></div></div><p></p><p><strong>" 适用于开发人员和高级用户 "</strong></p><p>据团队官方介绍，Asahi Linux 主要基于 Arch Linux ARM，不过该 Alpha 版本较为早期，因此 "<strong>它适用于开发人员和高级用户</strong>"，团队也希望<strong>使用者可以提交详细的错误报告和调试问题帮助团队进一步完善该发行版</strong>。</p><p>想尝试 Asahi Linux 的开发者需具备以下安装要求：</p><p>除了 Mac Studio，<strong>M1、M1 Pro 和 M1 Max 设备皆可</strong>。</p><p>电脑系统需为 <strong>macOS 12.3 及以上版本</strong>，且以管理员用户身份登录。</p><p><strong>确保至少有 53GB 的可用磁盘空间</strong>：首先 Asahi Linux Desktop 需要 15GB，同时安装程序会在 macOS 中预留额外的 38GB 磁盘空间以避免破坏 macOS 更新。</p><p>以上要求均满足的前提下，开发者即可根据提示开始安装 Asahi Linux Alpha 版（可参见 https://asahilinux.org/2022/03/asahi-linux-alpha-release/ 中的 " 安装 " 部分讲解）。</p><p>但正如 Asahi Linux 官方承认道，该早期版本还存在一定缺陷，因此开发者在决定试用前，需了解具体支持 / 不支持功能：</p><p>支持的功能包括：<strong>Wi-Fi</strong>、USB2（雷电接口）、USB3（Mac Mini Type A 接口）、屏幕（无 GPU）、NVMe、笔记本屏幕开合、电源键、内置显示器（仅限帧缓冲区）、内置键盘 / 触摸板、显示屏背光开 / 关、电池信息 / 充电控制、实时时钟、以太网（台式机）、CPU 频率切换。</p><p>除了以上的共性功能，还有部分芯片额外附带了一些其他功能：耳机插孔（仅限 M1 机型）、HDMI 输出（仅限 Mac Mini）、SD 读卡器 ( 仅限 M1 Pro/Max ) 。Asahi Linux 还透露道，USB3、扬声器、显示控制器等功能也即将支持。</p><p>其中，用户如无法连接 Wi-Fi，可尝试在网络管理菜单中将其关闭再打开；如果耳机插孔不无效或只有一边有声音，可尝试重新启动。</p><p>不支持的功能包括：显示接口、蓝牙、<strong>GPU 加速</strong>、视频编解码加速、神经引擎、睡眠模式、相机、触控栏等。</p><p>除了以上功能尚未实现，还有一些应用程序目前也无法在 Asahi Linux 运行良好：Chromium（需自愿修复）、Emacs（修复已提交，未发布）、所有使用 jemalloc 的应用（如 Arch Linux ARM 中的 Rust）及所有使用 libunwind 的应用（修复已提交，未发布）。</p><p><strong>Asahi Linux 的 " 强烈安利 "</strong></p><p>为了证明项目成果，Asahi Linux 团队在推特上发布了其<strong>在 M1 MacBook Air 上成功运行 Asahi Linux 的视频</strong>及截图：</p><p>同时，Asahi Linux 还表示："Asahi Linux 集成了 M1 Mac 上的原生多引导支持，安装 Linux 也根本不会影响 macOS 的安全级别。你可以继续使用 FileVault、运行 iOS 应用程序甚至以 4K 画质观看 Netflix！"</p><p>在这条推特下方，许多不甚了解的网友对 Asahi Linux 提出了不少疑问，官方也及时进行了统一回答。</p><p><strong>苹果允许这样做吗？这合法吗？</strong></p><p>正如起初 Linus Torvalds 认为 M1 缺乏对 Linux 的支持而难以实现 Linux 移植，此番 M1 MacBook Air 运行 Asahi Linux 的事实引起部分网友担忧：" 苹果允许这样做吗？"</p><p>Asahi Linux 给出的回答是：" 苹果允许在 M1 系列 Mac 上启动未签名 / 自定义内核而无需越狱，所以我们这并不是黑客或钻漏洞行为，而是苹果内置在这些设备中的实际功能。此外，<strong>只要不从 macOS 获取代码来构建 Linux 支持，开发结果可以完全合法地分发且供最终用户使用</strong>，因为它不是 macOS 的衍生作品。"</p><p><strong>可以启动 macOS 和 Linux 双系统吗？</strong></p><p>Asahi Linux 的发布给了部分钟爱双系统的用户一种可能性：" 可以实现 macOS 和 Linux 的双系统了？"</p><p>对此，Asahi Linux 表示：" 可以，实际上<strong>我们也支持你开启双系统</strong>。因为目前 Asahi Linux 安装程序还不支持替换 macOS，所以我们希望你保留 macOS，况且 macOS 和 Linux 之间也彼此独立且互不妨碍。" 切换系统的方法也很简单：<strong>按住电源键启动，然后选择要启动的操作系统即可</strong>。</p><p><strong>它安全吗？会不会破坏 Mac？</strong></p><p>对于这个常见的用户担忧，Asahi Linux 表示 " 已努力使此安装程序尽可能安全。"</p><p>团队解释道，所有磁盘管理操作都是使用本地 macOS 工具在幕后执行，Asahi Linux 安装程序并不会做任何真正引发安全问题的操作，甚至<strong>在 Asahi Linux 的整个开发测试过程中，也没有发现任何 Mac" 变砖 " 的现象</strong>。</p><p>不过，其团队还是谨慎道：" 与所有开源软件一样，尤其是像这样的 Alpha 版本，我们不能做出任何承诺。"</p><p>从 Asahi Linux 官网介绍可以看出，这次的 Alpha 版本仅是一个开始："<strong>我们的目标不仅仅是让 Linux 在这些设备上运行，而是将它打磨到可以用作日常操作系统的程度</strong>。"</p><p><strong>不仅成功运行，速度还是 macOS 的两倍？</strong></p><p>在 Asahi Linux 的 " 强烈安利 " 下，许多开发者开始尝试在他们的 M1 系列 Mac 上安装定制化 Asahi Linux（安装 GNOME、LibreOffice、Docker/k3s、Bluefish、dotfiles 等）。</p><p>在这之中，知名 IT 专业人士 Jason Eckert 发现了一丝不寻常："<strong>我注意到应用启动异常快，甚至 LibreOffice 应用都可以立即打开</strong>。" 感到神奇的 Jason Eckert 立刻在 Asahi Linux 中安装了 Hugo（由 Go 语言实现的静态网站生成器），以此确定这种 " 快 " 是否源于操作系统的不同。</p><p>上个月，Jason Eckert 正好曾对同一 macOS 版本下英特尔芯片和 M1 芯片的 Hugo 表现做了对比："Intel Core i9 用了 6126 毫秒（6 秒）来生成网站的所有 273 个页面，而 M1 只用了 549 毫秒（半秒）——<strong>苹果 M1 的速度提高了 11 倍以上！</strong>"</p><p></p><div class="zaker_div"><div class="zk_decsimg Linux）</p></div><div class="edi_oper"></div></div><p></p><p>对于这个对比结果，许多网友<strong>直呼不理解</strong>：" 为什么呢？在我看来，这可能是由于 Rust 的 libc 没有在 macOS 上实现 statx 调用而导致的 2 倍性能差异，但这不应该被认为是操作系统的固有属性吧。对我来说，2 倍的性能差异听起来像是想要邀请进行更多基准测试，毕竟很难理解这是为什么。"</p><p>也有网友指出，<strong>275 个页面太少</strong>：" 这个时间（210 毫秒）对于有意义的基准测试来说太短了，因为还有初始化和其他类似工作会影响测试结果。你需要一个页面多 100 倍的测试。"</p><p>但也有人指出，<strong>Linux 的性能的确优于 macOS</strong>：" 我也有这种感觉，在 Linux 上编译 C 程序的速度几乎是 macOS 上的两倍。"</p><p>那么在了解过后，你是否期待 Asahi Linux 稳定版，又是否想要尝试一番呢？</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
