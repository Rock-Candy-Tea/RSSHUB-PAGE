<p>IT 之家 11 月 21 日消息，近期，微软发布了 Windows 11 Dev 预览版 Build 22504，随后，微软面向 Dev 频道内部人员发布了新的 Windows 11 Build 22504.1010（KB5008697）。新的构建版本并没有带来任何新的东西，因为它是一个累积更新，该公司正在测试其服务管道，正如它经常做的那样。事实上，上一次类似这样的更新是在一周前。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202111/6199f9f38e9f09525c4d7bd1_1024.jpg" data-height="692" data-width="1440" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202111/6199f9f38e9f09525c4d7bd1_1024.jpg" referrerpolicy="no-referrer"></div></div>这些服务管道的更新是必要的，以确保在没有任何值得注意的问题或故障的情况下向用户提供构建版本。因此，在新闻稿中，微软公司只指出：<p></p><p>" 这次更新不包括任何新的东西，旨在测试我们在开发渠道中构建的服务管道。"</p><p>因此，早先发布的 Windows 11 Build 22504 版本的所有错误修复和改进都被带到了 Dev 频道。IT 之家获悉，该版本带来了改版的 "Your Phone" 应用、输入法自定义主题、个性化的表情符号等。</p><p>有趣的是，虽然在这个构建版本中没有新的变化，但像这样的服务测试构建可能会破坏一些功能。例如，此前的 Windows 10 Build 21292.1010 对 ARM PC 上的 x64 模拟器造成了问题。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
