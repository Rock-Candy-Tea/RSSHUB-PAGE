<p>IT 之家 5 月 29 日消息，在最近举行的 Computex 2022 电脑展上，AMD 推出了 SmartAccess Storage（SAS）存储智取技术。</p><p>AMD 表示，它是将现有 SAM 显存智取技术与微软的 DirectStorage 技术结合起来，通过直接将存储器中的数据解压到 GPU 上，而不是通过 CPU 进行中转。与传统的 Win32 存储 API 相比，这些将极大地缩短游戏加载时间。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres2.myzaker.com/202205/6292cb0c8e9f09492d12f181_1024.jpg" data-height="497" data-width="898" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202205/6292cb0c8e9f09492d12f181_1024.jpg" referrerpolicy="no-referrer"></div></div>Computex 主题演讲中没有提供有关 AMD 如何合并这两种技术的详细信息。然而，AMD 的游戏解决方案和营销首席架构师 Frank Azor 最近接受 PCWorld 采访时向大家详细介绍了关于 AMD Smart Access Memory 如何与 DirectStorage 一起发挥作用以催生 SmartAccess Storage 的故事。<p></p><p>当被问及 SAS 的最低规格要求时，Azor 透露，并非所有 NVMe PCIe 驱动器都将获得该技术的认证，合格的驱动器必须满足 AMD 设定的某些性能要求，即使是最新的 PCIe 5.0 SSD。</p><p>因此，我们有一份经过授权的供应商清单，列出了我们目前正在开发的与 NVMe 驱动器和控制器有关的部件，以满足我们对 SmartAccess 存储的性能要求。如果刚好你有一个 PCIe Gen4 或即将到来的 PCIe 5 硬盘 … 并不意味着这些硬盘将能够跟上 SmartAccess 存储的性能要求和能力。</p><p>SAS 的基础技术，比如 DirectStorage 就没有这样的要求。根据微软的要求，任何 NVMe 驱动器 ( PCIe Gen3 或更新版本 ) 和 DirectX12-ready 配置的着色器 6.0 GPU 都是有必要的。</p><p>根据 Azor 的说法，Smart Access Storage 基本上是 PCIe Resizable Bar 的变型，它可将显卡的整个 VRAM 显存缓冲区用于 DirectStorage 和数据的解压缩。如果没有 Smart Access Storage，则代表显卡只有 256MB 的 VRAM 来进行寻址并转让。</p><p>理论上，这应该使 SmartAccess Storage 比 DirectStorage 快得多，但也意味着并非所有 Radeon GPU 都支持 SmartAccess Storage 技术。</p><p>IT 之家了解到，英伟达此前已经应用了类似的 RTX IO 技术，但是目前还没有确定 RTX IO 是否使用 PCIe Resizable BAR 方面的成果。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
