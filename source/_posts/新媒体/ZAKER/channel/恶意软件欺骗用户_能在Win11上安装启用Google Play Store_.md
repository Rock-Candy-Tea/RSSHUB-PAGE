商店的第三方工具被发现是恶意软件。<p></p><p>这款名为 "Powershell Windows Toolbox" 的工具托管在 GitHub 上，用户 LinuxUserGD 注意到底层代码很神秘，并且包含了恶意代码。随后，用户 SuchByte 为该工具提出了 issue 。Powershell Windows Toolbox 已经从 GitHub 上被删除。</p><p>以下是该工具声称要做的所有事情：</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" 部分，开发人员已指示用户在 CLI 中运行以下命令：<p></p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" 代码，它们从 Cloudflare workers 加载恶意脚本，并从用户 alexrybak0444 的 GitHub 存储库中加载文件，该用户可能是威胁参与者或其中之一。这些也被报告和删除。<p></p><p></p><div class="img_box" id="id_imagebox_3" onclick><div class="content_img_div ExperienceMaintenance</p><p>MicrosoftWindowsServicesCertPathCheck</p><p>MicrosoftWindowsServicesCertPathw</p><p>MicrosoftWindowsServicingComponentCleanup</p><p>MicrosoftWindowsServicingServiceCleanup</p><p>MicrosoftWindowsShellObjectTask</p><p>MicrosoftWindowsClipServiceCleanup</p><p>同时删除恶意脚本在感染期间创建的 "C:systemfile" 隐藏文件夹。如果你正在执行系统还原，请确保使用不包括 Powershell Windows Toolbox 的还原点，因为它不会从系统中删除恶意软件。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
