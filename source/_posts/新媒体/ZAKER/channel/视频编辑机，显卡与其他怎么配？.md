referrerpolicy="no-referrer"></div></div><strong>多核一定强到没边吗？</strong><p></p><p>看了昨天的分析文章，肯定有朋友认为，如此说来，核心越多、单核频率越高，做视频编辑就越强了。</p><p>其实这个是在典型工作流程下，一定范围内成立。但是对于一般学生和创业者的视频来说，不设计过多的 CG 特效的时候，工作负担没有这么重。盲目追求编辑环节的过剩核心数并不是最佳的选择。</p><p>根据国外 CG 行业网站的统计，当执行相关的视频编辑类软件跑分时，单核和多核成绩存在一个有趣的曲线（纵轴为测试分数）：</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres2.myzaker.com/202209/632995a5b15ec052317bc5a8_1024.jpg" data-height="606" data-width="701" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202209/632995a5b15ec052317bc5a8_1024.jpg" referrerpolicy="no-referrer"></div></div>可见在一般的桌面编辑系统重，如果要达到单核和多核性能最佳调和，16 核左右的配置是比较好的性能交叉点，往后核心越多，单核（单帧编辑渲染能力）和多核（解码能力）就会出现较大的性能分离，直到接近 64 核心才会再次进行性能曲线的贴合。<p></p><p>因此单从编辑性能来说，16 核心的桌面处理器以 i9-12900k 和 Ryzen5950X 为性能上的优选选择，而在价格方面 Ryzen5950X 更具有性价比。</p><p><strong>缓存片段：内存条多大多快合适？</strong></p><p>当你从你的高速 SSD 读取你的素材视频，用 CPU 进行解码，编辑处理特效时，软件将：</p><p>1. 读取和解码镜头</p><p>2. 计算画面上的效果</p><p>3. 将结果存储在系统内存（RAM）中。</p><p>这种工作状态通常被称为缓存工作。</p><p>所有缓存的帧都保存在内存条中，软件不必再计算所有的东西，而是可以从 RAM 中读取已经计算好的结果，这要比反复用 CPU 计算快得多。</p><p>而包括播放编辑好的片段时的提前 " 缓冲 "，也是为了将软件的计算结果暂时存储到缓存中，以避免中断你的实时播放体验。</p><p>当内存条缓存容量不够时，系统会在硬盘上创建缓冲文件，但此时，工作结果的预览读取就会慢很多。这也是为什么昨天小狮子建议购买 2TB 的 NVMe SSD 做系统和软件盘，因为 PR 等工作缓存区一般是放在系统盘上的，系统盘越快，其读取瓶颈与内存条比差距就越小。</p><p>现在，从高频 DDR4 到 DDR5，内存的速度是如此之快，视频编辑已经不需要过分关心其速度（除非你超频了 CPU）。</p><p>唯一重要的是要有足够大的内存容量，以存储编辑缓存的结果，如果它不够大，软件编辑结果缓存到你的磁盘上，放慢工作效率。</p><p>至于内存的容量，同样 CG 行业也有一个经过考验的曲线表格供大家配机参考：</p><p></p><div class="img_box" referrerpolicy="no-referrer"></div></div><strong>四通道比双通道好吗？</strong><p></p><p>在一些论坛里，小狮子看见过视频配机方案推荐四通道内存方案的。但对于大多数普通学生朋友接触的片源素材，和工作流程来说，并没有这个必要。</p><p>尽管理论上内存带宽随着通道搭建增加会大幅度增加，但软件（例如 Premiere Pro）不会利用它，因为不需要更高的带宽，特别是现在 DDR5 的传输速度已经超过 5GB/s。一般 25FPS-60FPS 的视频项目不会使这个带宽饱和。</p><p>让我们看一个例子。</p><p>在一个 25FPS 的项目中，这意味着你有最大 " 5GB/s 内存速度 /25 帧 "= 每帧 200MB。</p><p>也就是说，当你的素材，一帧画面（图像、画面及包括其特效数据）的容量低于 200MB 时，你的内存就能以 25FPS 的速度快速编辑和回放它。除了一些 8K 的 EXR 素材，现在所有器材的素材通常不会达到这个每帧大小，所以内存带宽速度的重要性就降低了。</p><p><strong>显卡呢？显卡不重要吗？</strong></p><p>到这里小狮子还没有谈及显卡对视频编辑性能的影响。是的，现在除了 3D CG 细分领域外，显卡对视频编辑的影响仍然没有 CPU 那么大。</p><p>下面是美国著名的系统定制公司 Puget 基于每年数万台定制视频编辑 PC 出厂时进行的测试的结果。在编辑（解码）环节，关闭 CPU 编解码后，使用显卡后的整体视频编辑得分。</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div 的视频编辑中，多显卡配置并不能使你受益。</p><p>但如果你是在 3D 系统中（如 Blender）进行 GPU 渲染，那么多显卡配置带来的性能提升是明显的。</p><p>因此，根据你的预算，小狮子推荐 RTX 3060Ti 或 RTX 3070 就行了，它们性价比都很高，在许多视频编辑相关的基准测试中得分很高，而任何跑分高于 RTX 3070 的产品都会贵不少，性能却只增加了一点。</p><p>考虑到目前显卡市场每天 " 你我 200" 的节奏，小狮子这里就不推荐具体型号产品了，反正等要配机时候选择含矿率低的 3060Ti 或者 3070 新卡就行啦。</p><p><strong>项目渲染出片：CPU 依然重要</strong></p><p>现在，还有一件事很重要，即渲染编码出片阶段。这是你的项目变成成片的最后一步。</p><p><strong>什么是视频编码</strong></p><p>除了原始的 RAW 拍摄素材，通常项目出片，不是把编辑好的帧未经压缩地转储到你的磁盘上，而是把这些帧打包和编码 / 压缩后，封装成一种文件格式（例如 AVI,MP4），使得用户可以用播放器以更小资源占用播放。</p><p>目前大家在后期出片时，常用的编码器 / 规范主要有 H.264、H.265、WMV9、ProRes、DNxHD、Sorenson、Cinepak 等等。</p><p>大多数流行的编解码器使用的逻辑是只保存两帧之间的差异，而不是完整地保存每一帧，因为两帧之间的差异进行存储，通常在文件大小上比存储单帧要小得多。到用户播放器端用算法还原就行了。还有一些编解码器将像素阻隔在一起，或减少颜色和对比度信息以节省空间。</p><p>因此，它是一种算法引擎，那么在出片时，依赖有相应计算器（编码单元）的 CPU 也就很正常了。</p><p>在编码渲染输出方面，由于 intel 的 CPU 的编码器的丰富程度，因此在这个环节 intel 可以说是扳回一程。同样基于 puget 公司的定制视频机出厂测试成绩，我们可以看出 12 代酷睿在最后环节还是有一定性能优势的。</p><p></p><div class="img_box" id="id_imagebox_6" onclick><div
