<p>IT 之家 5 月 17 日消息，据 Neowin 报道，由于驱动程序错误，微软 Windows 11 最近一直在崩溃，其中一些驱动程序对于防病毒平台的平稳可靠运行至关重要。网络安全公司 Sophos 最近发布了两个临时解决方法来修复随机 BSOD（蓝屏死机）。该公司现在已经发布了一个修复程序，可以完全解决这个问题。</p><p>最近的 Windows 11" 星期二补丁 " 包括 KB5013943，这引起了相当大的麻烦。似乎错误的更新导致许多应用程序出现故障、产生黑屏问题、蓝屏死机等等。</p><p>网络安全公司 Sophos 声称 Windows 11 随机崩溃，导致短暂的蓝屏死机屏幕，是由于驱动程序有问题。有趣的是，一些引起麻烦的驱动程序是防病毒解决方案所必需的。</p><p>这家防病毒公司最近发表了一篇博客文章，重点介绍了两种临时解决方法。但是，Sophos 已经更新了页面，以确认它正在推出完整的修复程序，将完全解决问题。该补丁旨在仅为 Sophos 客户修复 Windows 11 蓝屏问题。</p><p>一些从微软收到令人不安的更新的 Windows 11 用户声称，臭名昭著的蓝屏死机出现的时间非常短暂。这使得几乎不可能读取到错误代码。但是，一些用户报告说看到错误 "APC_INDEX_MISMATCH"。神秘的 " 错误 " 性质使得解决问题变得困难。</p><p>由于 KB5013943 更新是强制性的，因此许多 Windows 11 用户很可能会继续遇到随机崩溃。然而，Sophos 声称其客户将不会面临同样的麻烦。</p><p>安装 KB5013943 更新后，其他防病毒公司尚未针对困扰 Windows 11 的蓝屏死机问题提供修复或解决方案。然而，既然 Sophos 声称它已经发现了根本原因，那么其他公司可能会跟进更新相关补丁。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
