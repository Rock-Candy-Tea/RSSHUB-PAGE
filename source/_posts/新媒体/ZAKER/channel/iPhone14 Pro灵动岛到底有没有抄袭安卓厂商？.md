发布之前，关于该手机的各种配置与升级都被扒的七七八八，但是最让人眼前一亮的却不是所谓的卫星通信以及 A16 芯片，而是不起眼的小创新—— " 灵动岛 "，这个看似不起眼的小功能却引起了很多人的关注与讨论。</p><p>刘海屏本身是一个屏幕的设计缺陷，但苹果用设计和交互，将其调整成一个药丸的区域，将通知、交互、卡片、弹窗等一系列交互功能集中于此，灵动变化形态，这就是 " 灵动岛 " 的交互模式。</p><p>这个交互设计将药丸屏的缺陷变成了亮点，给用户留下了深刻的印象。以至于发布会之后，灵动岛上了热搜，朋友圈也被刷屏。灵动岛也吸引了不少消费者想为之买单与尝鲜。</p><p>9 月 9 日当晚 iPhone 14 系列预售太火爆而导致苹果官网和其他渠道几乎全线崩溃，从不少消费者购买 iPhone14 系列的理由来看，许多消费者认为‘灵动岛’的创新感觉好玩有趣，想体验一下。</p><p>在近期，灵动岛更引发了抄袭争议。androidcentra 网站认为，iPhone 14 系列上至少有 5 个功能都是抄袭了安卓，至少也是借鉴了安卓厂商的灵感，包括灵动岛、打孔前置镜头、常亮显示、动态刷新率、车祸检测在内的所谓新功能都是华为、LG 等安卓厂商玩过的。</p><p><strong>问题来了，灵动岛到底有没有抄袭安卓厂商？</strong></p><p>要回答这个问题，肯定需要对比两者有何不同。我们知道，灵动岛是苹果为了弥补药丸屏的缺陷而设计出来的一个 UI 交互，谈不上大的创新，可以认为是一种趣味性的微创新。</p><p>我们知道，无论是刘海屏或者打孔屏，都是屏幕本身带来的缺陷，无论是过去安卓厂商还是灵动岛设计，其初衷都是为了遮盖这个瑕疵，类似给伤疤纹身。</p><p>但从业内的观察来看，包括三星、荣耀、华为、LG、魅族都曾经为了遮盖屏幕瑕疵做了类似的设计。</p><p>典型的是 LG V20 手机，如下图，其正面采用 LCD 屏幕，最上方还有一块 " 第二屏 "，分辨率为 1040x 160，不仅可以开启 Wi-Fi、蓝牙等功能，还能完成音乐控制，开启最近应用等操作。</p><p>但最终，这个设计并没有获得媒体及用户们的青睐，因为本质上，LG 是加了一块副屏，利用这块副屏可以发展出更多的功能——切换应用、音乐播放器、手电筒等等，简单来说，是多做了一个功能任务栏，在视觉观感上非常粗糙，本质是一种硬件的延伸。</p><p></p><div 系列，是围绕在摄像头周围的解锁动效，通知呼吸灯和环形电量等。荣耀 V20 基于左上角的单挖孔，通过前摄药丸和 UI 联动成一个实时通话时间的胶囊，在自拍时胶囊会显示提醒 " 看这里 "。<p></p><p>安卓阵营在这方面均侧重于轻提醒功能，没有用户的人机互动，仅仅是做了美化，补了漏洞，试图掩盖摄像模组的挖孔给人视觉层面带来的观感不适。</p><p>苹果灵动岛其实是一种打破常规的操作，它把系统里原有的通知体系，卡片，弹窗控件的公共规范全部刷新重制，灵动岛将这个区域变成了一个伸缩性、弹性、动态、可变的动态图标交互，它可以搭配不同 App 支持各种动效设计，支持第三方 APP，是一种软硬件融合模式。</p><p>恰如有人说到，牛的不是灵动岛的设计，牛的是苹果能让第三方软件开发商适配灵动岛的交互，这本身就构成了一个完整的软硬件交互生态的闭环。而是通过动效进行反馈互动，强化了用户和摄像模组周边的交互操作，" 灵动岛 " 给人有创意的感觉就在于它不再基于屏幕瑕疵遮遮掩掩，而是喧宾夺主，反着来。</p><p>事实上，国产厂商一直在改刘海，本质都是一根筋的为了美观而已，没有交互逻辑层面的深入思考。但从苹果的交互设计来看，它超脱了美观修饰范畴，让其变成了一个趣味化的创新亮点，放大了这个设计的交互乐趣——包括显示专辑封面、放大 Face ID 认证图标，勿扰模式消息、音乐卡片，倒计时等在内的诸多通知等，让其喧宾夺主，在特定的场景中，成为了主角。</p><p>关于灵动岛的设计，前两天老罗还在吐槽，他表示，灵动岛的功能确实非常聪明，但不得不说这个灵动岛的功能也是建立在苹果硬件太烂的基础上开发出来的，倒逼 UI 设计师不得已憋出来的邪招，果粉嗨得莫名其妙。</p><p></p><div class="img_box" referrerpolicy="no-referrer"></div></div>罗永浩还表示，安卓在屏幕上已经搞过小黑点前摄、升降式摄像头、屏下摄像头等更好的解决方案，没必要多此一举。<p></p><p>但事实上，罗永浩并不清楚的是，iPhone 的 FaceTime 摄像头和传感器需要支持 Face ID 解锁，需要支持 Center Stage 的人物居中，简言之对 FaceTime 摄像头的成像素质，以及对传感器的 Face ID 安全性都有要求，屏下镜头，不一定能满足这一点。</p><p>一块黑色的遮罩以比镜头模组大的大小随意变动，实现不同的交互，让镜头的突兀感降低，这是其一，其二满足了面容解锁；其三，超越了给屏幕缺陷美容的核心目的，变成了一个创新亮点。</p><p>从这个意义来看，" 灵动岛 " 是个突破人们惯性思维，不仅仅是将通知功能换面，而是系统级的交互设计方案。可以看出，苹果公司对软件交互设计的层面的创新与探索能力是安卓厂商所不具备的。</p><p><strong>安卓厂商可能模仿不了 " 灵动岛 " 了</strong></p><p>当然，很多用户认为是苹果本身品牌的加持效应与放大效应以及果粉用户对苹果的盲目崇拜，但事实上，即便如今把它们重新放在一起对比，就能发现它们是完全不同的东西——毕竟，人都是有眼睛，有没有抄袭，是骗不过人们的眼睛与直觉。承认别人优秀，补齐自己的短板，才能进步。</p><p>如果厂商们真觉得自己有基于该功能做了精心设计，为避免被抄袭，完全可以申请专利保护，并大肆宣扬，一旦有其他厂商抄袭，都可以通过诉诸法律、对簿公堂来解决。</p><p>但从目前来看，似乎没有厂商认为他们过去的此类设计是独创性的功能亮点，也似乎并没有厂商在该设计层面申请了相关专利，并将其列为重点推广方向，反而是苹果对 " 灵动岛 " 的做了非常严密的保密与专利设计。比如说，苹果为了保密 " 灵动岛 " 只能在发布会上揭晓，商标的注册地选择在非洲国家牙买加——原因是牙买加规定注册商标必须采用线下方式，且不会开放线上查询库，确保不会有人在发布会之前抢先泄密或提前申请专利保护，可见苹果对该功能的重视程度。</p><p>灵动岛适配大量 APP 动效设计其实是一种软件生态层面的加持。安卓厂商过去的静态图标美化在这种动态交互面前——显得非常无力。当然，你也可以说，安卓对打孔屏以及刘海屏的美化尝试对苹果有一定的启发作用，苹果在此基础上，更深入的做出了体验与观感与目的不一样的东西。</p><p>如前所述，在灵动岛出来之后，很多业内人士都已在预测，该功能将被安卓阵营大规模复制，一如当年的刘海屏。但事实上，该功能即便要复制，其难度非常之大，可以说几乎很难呈现一模一样的东西。这一方面的苹果在牙买加提交了其 " 灵动岛 " ( Dynamic Island ) 全球专利申请。</p><p>另一方面，不同于此前刘海屏只是硬件层面的设计， 灵动岛是软件动画修饰硬件，相应 APP 能随时应你所需，无缝呈现重要信息。这看起来简单，但其实并不好好学——从目前来看，灵动岛适配大量 APP 动效设计其实是一种软件生态层面的加持，这是安卓厂商过去以及现在也不具备的。这是一种软硬件设计方案，是一套标准化的系统交互规范体系，它与整个系统融为一体，需要 APP 开发者配合，也要给开发者足够的接口做适配，这可不是厂商学的来的。</p><p>甚至我们可以畅想，未来开发者基于适配接口，可能还将围绕灵动岛做出更为惊艳的动画设计。而安卓这边，由于生态能力以及系统底层的 UI 设计权限与硬件规格标准能力存在差异，很难用一套规范来适配所有机器。</p><p>这意味着安卓厂商未来想模仿，可能也模仿不了了——这本身就充满了讽刺意义。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
