<p><strong>在 2021 年 6 月推出 Windows 11 版本 21H2" 太阳谷 " 之后，微软现在正准备发布今年的操作系统大更新，它被称为 " 版本 22H2"，</strong>其中将包括新的开始菜单选项，更好的任务栏，改进的上下文菜单，带有标签的文件资源管理器，性能增强，错误修复等。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres1.myzaker.com/202204/6264bba38e9f096d976b7d28_1024.jpg" data-height="1080" data-width="1920" src="https://cors.zfour.workers.dev/?http://zkres1.myzaker.com/202204/6264bba38e9f096d976b7d28_1024.jpg" referrerpolicy="no-referrer"></div></div>正如你可能知道的那样，微软会在 Insider 社区内同时测试多个功能更新版本，现在已经在开发 23H2 版本。现在的状况是 Windows 11 的 22H2 版本的开发周期临近尾声，预计将在未来几个月内最终测试完成。与此同时，该公司已经开始了 23H2 版本的工作。<p></p><p>微软现在计划向之前选择 " 开发渠道 " 的 Windows Insiders 发布下一个功能更新的初始构建。用户可以在任何时候选择加入 ' 开发渠道 ' 计划，但如果用户不想接收不太稳定和完善的实验性构建版本，应该选择转回 Beta 渠道。</p><p>现在，Github 上已经出现了对下一个开发分支的预览版构建的引用。Windows 11 Build 22603（与目前的 Nickel 开发分支相比，数字上有很大的飞跃）已经在 Windows 终端中被发现，这表明微软正在为下一代预览版构建做准备。</p><p>当新版 Windows 11 倍发布到开发频道时，构建号会有所不同，但这是其中一个迹象，似乎暗示用户将很快开始测试明年更新的第一批新功能。</p><p>希望继续使用 Windows 11 22H2 版本的用户应该及时切换到 Beta 通道，这是微软为消费者级别的用户提供早期完全稳定更新时做最后的测试所用。而进入开发频道的 Windows 11 新版本接下来的更新会变得更频繁，最初主要包括错误修复，但新功能将在未来几周开始浮现。</p><div id="recommend_bottom"></div><div id="article_bottom"></div>
