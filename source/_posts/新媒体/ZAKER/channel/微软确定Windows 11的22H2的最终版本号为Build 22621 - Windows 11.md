<p><strong>Windows 的下一个主要版本已经完成：正如预期的那样，build 22621 就是 Windows 11 版本 22H2 的最终版本。</strong>当然，现在早已经没有以往压盘送厂的过程，已经没有真正意义上的 "RTM" 了，Windows 11 22H2 版在出现在新 PC 上并向公众发布之前和之后都会很快被更新版本取代，所谓正式版与否意义并不大。</p><p></p><div class="img_box" id="id_imagebox_0" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_0" data-original="http://zkres2.myzaker.com/202205/628ddb44b15ec05f360bea5c_1024.jpg" data-height="394" data-width="700" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202205/628ddb44b15ec05f360bea5c_1024.jpg" referrerpolicy="no-referrer"></div></div>该消息来自于 Michael Reinders，他指出微软已经通过其技术社区博客确认这是最终版本。事实上，尽管它只在 Windows 硬件认证页面上出现过一次，但微软首次在公开场合详细介绍了与这个 Windows 版本有关的一些非常具体的内容。该公司在那里写道："Windows 11 22H2 版经过认证的最低可接受的构建是 RTM（即 22621）"。<p></p><p>Windows 11 build 22621 大约两周前首次发布到 Windows Insider Program 的 Beta 频道。当时，微软指出，" 桌面右下角的构建水印在这个构建中不再出现， [ 但 ] 这并不意味着 [ 它 ] 已经完成，水印将在未来的构建中返回给内部人员。" 但显然，今天的消息确实意味着它已经完成了最终的开发工作。</p><p>另外，并非巧合的是，微软在那时发布了这个版本的 ISO 文件，而且还有一个 SDK。所以您可以通过 Windows Insider Preview 下载网站提前获取到 Windows 11 22H2 的最终版本，如果你想抢先一步安装，并愿意承担 bug（如有）带来的不便的话。</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" data-original="http://zkres2.myzaker.com/202205/628ddb44b15ec05f360bea5d_1024.jpg" data-height="636" data-width="700" src="https://cors.zfour.workers.dev/?http://zkres2.myzaker.com/202205/628ddb44b15ec05f360bea5d_1024.jpg" referrerpolicy="no-referrer"></div></div><p></p><div id="recommend_bottom"></div><div id="article_bottom"></div>
