是不是就理所当然地不会有侵犯版权的问题？<p></p><p>不然不然，据国外网友的吃瓜讨论，这当中至少存在以下两条争议。</p><p>咱们一条一条地撸。</p><p>GitHub 的就是微软的 ?</p><p>虽然 Copilot 是 GitHub 联合 OpenAI 发布的产品，但它却不是 GitHub 的一部分，反而却是 VS Code 编辑器的一个<strong>插件</strong>。</p><p>没有 VS Code，你根本使用不了 Copilot！</p><p>利用 GitHub 上开源代码训练出来的 Copilot，却在 VS Code 编辑器上帮用户自动生成代码，这岂不是 Copilot 在 GitHub 之外分发 GitHub 上的代码吗？</p><p>难怪有国外网友大呼：</p><p>这怎么可能不是微软寄生在 GitHub 上所有代码上，让 VS Code 变得更好呢？</p><p>有人会驳道，VS Code 不是微软推出的开发工具吗？而在微软 2018 年 6 月斥 75 亿美元买下 GitHub 后，GitHub 不就已经是微软的了？</p><p>自家人开发的 AI 程序员插件，怎么不能装在自家推出的代码编辑器上了？</p><p>一位国外网友表示，这完全取决于服务条款中的<strong>人称主语</strong>的界定。</p><p>在 GitHub 的服务条款里，GitHub 对 "GitHub"、"We"、"Us" 作出了明确界定。</p><p>其中，包括 GitHub 公司，以及它们的联属公司、董事、子公司、承包商、许可方、管理人员、代理和员工。</p><p></p><div class="img_box" id="id_imagebox_1" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_1" 开发团队吗？包括微软的所有人吗？</p><p></p><div class="img_box" id="id_imagebox_2" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_2" Copilot 合不合法，最大的争执点在于服务条款上对代码托管和许可使用范围的界定。</p><p>按知识产权律师 Kate Downing 的说法，不管你使用了什么样的<strong>许可证</strong>，你都同意了 GitHub 托管你的代码，并使用你的代码来提升它们的产品和服务。</p><p>但很明显，训练 Copilot 不在向托管代码的用户提供的服务之内，也没有用来帮助 GitHub 改进它们的代码托管服务。</p><p>在这种情况下，版权侵权的问题将在很大程度上取决于<strong>合理使用</strong>的概念。</p><p>如果 GitHub 能证明 Copilot 的出现对编程行业具有很大的变革性，进而证明用开源代码训练 Copilot 是一种合理使用，那么法院很有可能会支持 GitHub 的这一行为。</p><p>这种情况也有过先例。</p><p>谷歌从 2004 年开始对图书进行了大规模数字化，为研究人员提供了搜索书籍的服务，用户可以迅速搜索到书中自己想要的内容。</p><p>没有取代书本身的作用，也没有从版权人那里拿走任何东西。</p><p>谷歌图书让读者更容易接触到作品，也为图书作者开辟了更广阔的市场。</p><p></p><div class="img_box" id="id_imagebox_3" onclick><div 100% 生成代码的版权。</p><p></p><div class="img_box" id="id_imagebox_4" onclick><div class="content_img_div perview_img_div"><img class="lazy opacity_0 " id="img_4"
