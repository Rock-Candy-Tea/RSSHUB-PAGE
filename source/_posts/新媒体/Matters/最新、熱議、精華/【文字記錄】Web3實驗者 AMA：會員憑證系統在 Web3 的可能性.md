Discord <a href="https://discord.gg/matterslab" rel="noopener noreferrer" target="_blank">https://discord.gg/matterslab</a></blockquote><p>以下是Valley 22 x Matters AMA的文字紀錄。</p><h2><strong>Web3 實驗者：會員憑證系統在 Web3 的可能性 </strong></h2><ul><li>日期：26/04 （周二） </li><li>時間：21:30-22:30 PM（東八區） </li><li>地點：Matters Discord 「自由二台」 </li><li>分享：Andrew（Valley 22 founder）</li><li>主持：多比（Matters Discord MOD）</li><li>AMA 重溫：<a href="https://youtu.be/TsOOa-rjgbQ" rel="noopener noreferrer" target="_blank">https://youtu.be/TsOOa-rjgbQ</a></li><li>簡介：</li><li>在現今 NFT 市場，不少人關注它想傳達的概念、團隊的願景。項目賦能、社群共識會是未來 NFT 賽道的重要指標。 Karma Labs 針對Web2 的會員制度，希望在Web3發展新的會員制系統，並以Valley22酒吧作為試驗場地。在 Web3 的世界，會員制系統可以有著什麼變化和可能性？</li><li>Karma Labs ：以NFT作為消費者與商家間的橋樑，以此增強彼此互動，嘗試在Web3建造嶄新的消費體驗，消費者能根據喜好擁有專屬客製化權益，商家則針對會員給予獨特獎勵，Valley22為首個應用場景。</li><li>Valley 22：首家會員代幣應用酒吧，將發行404個會員代幣，存在SOL鏈上，持有代幣者能享有專屬權利。</li></ul><p>Dobby ：今天很高興請到 Valley22 以及 Karma 團隊的 founder 來這邊 AMA。我們經常看到很多大項目，有國際級團隊，都望麈莫及，但其實 blockchain 的應用很日常，隨著愈來愈多的人使用 ，以至小社群都在思考怎樣可以參與的時候，我們今天請到台灣一間酒吧的老板，同時是 karma 團隊的 founder，來跟我們談談小社群可以怎樣運用web3，從而發展自己的生意。</p><h3>介紹環節</h3><p>問：可以介紹一下項目嗎？為什麼會想要在 Web3 世界中做會員系統呢？</p><p>Andrew：Matters的朋友大家晚安，我是 Karma 及 Valley22 的 founder - Andrew，區塊鏈團隊名為 Karma，團隊想做的是 Web3 會員制系統，Valley22 是第一個實驗場域（位於台北的酒吧）。我們的目標，是打造出跨消費場景的會員系統，希望做到消費歷程累積，並將消費歷程數據進行量化，最後把數據創造出來的價值還給消費者。NFT 在這之中所起到的作用是「會員卡」，用 NFT 打造出會員系統，把消費者個人喜好，無論是喜歡吃什麼喝什麼。對我們而言，「消費」即是「需求」的展現，而我們希望藉由「消費」這一行為具象化。</p><p>問：先來談談持有 Valley 22 的權益、當中的賦能？有和沒有 pass 的最大差別是什麼呢？</p><p>Andrew：現階段比較好玩的會員權益 ，第一個是持有者每次來店都可以透過「搖鈴」請全場喝 shot（反應最熱烈的權益）；第二個是將「提案」性質引入會員系統，讓會員可以參與到店家的營運。會員可針對酒譜進行提案，經過社群投票後會在店裡上架，同時提案者可以得到該酒品的分潤（10%）；第三個是 VIP 之夜與各項活動，包括與酒類、Web3、區塊鏈相關的主題，同時還有「調酒師」的體驗活動。</p><p>現在我們正在對第二階段進行規劃，利用「裝備」的機制來達成前面所提到的「消費數據量化的過程」具象化到 NFT上。我們會根據消費者的消費行為進行 NFT 的空投，對應到客製化權益，例如喜歡喝 whisky 的人可以得到 whisky 的相關權益，將消費歷程紀錄下來，在完成某個目標或標準之後可得到裝備的空投。這套系統可以想像成在玩線上遊戲，第一階段的 NFT 是進入到這個遊戲的 LV1 角色，後續則是透過任務收集不同的裝備，換到這邊則是不同品項的權益。我們做的就是把線上的「Gaming」過程搬到線下。</p><p>問：我還不是太聽得懂，為何一定要在區塊鏈上做不可呢？</p><p>我們想做的是「跨消費場景的會員系統」，即消費歷程是可以跨店累積的，在討論 Web2 公司之時，常會說這些 Web2 的公司在數據外圍打造了護城河，不願意去分享得到的數據。而在 Web3 的角度，我們想要達成的是「數據的共享」。</p><p>以商家的角度來說，可以知道使用此系統的會員們各自的消費歷程與喜好；對消費者來說，好處就是<strong>在現今世界，在進行網頁的瀏覽時，通常會有 cookie 去做數據的紀錄，而這些紀錄價值又是由 Web2 公司所掌控。當商家們想要進行廣告的投放時，第一時間會找到各大 Web2 去做投放，但是實際的數據產生者（即消費者）並沒有獲得任何利益。我們希望做的就是將「數據產生的價值還給消費者」，藉由「主動」產生數據的過程獲得客製化權益。</strong></p><p>問：「將數據的權益還給消費者」，是你們 project 的主要目的，這麼理解對嗎？</p><p>Andrew：對的，像是前幾年歐盟提到的 GDPR（<em>General Data Protection Regulation）</em>，「數據的隱私權」在近幾年也備受關注，自己的 data 應該要自己決定是否被 cookie 收走，而數據的提供方並沒有得到任何應有的價值。而我們則是將數據的價值返還給消費者，使他們主動提供數據。</p><p>問：開始更明白為何要在區塊鏈上做了，但現階段來說 Web3 概念不算普及，而玩的人比較炒賣性質。開初 Andrew 是怎麼接觸到 Web3 的？ 比起做一個「炒賣」項目，為何你會更想做這一個酒吧的客人實驗NFT呢？</p><p>Andrew：會接觸 web3，最初是因為大學有修到智能合約的開發，到了研究所也做了區塊鏈相關的分散式系統研究，從而正式開啟了 web3 的旅程（也就是在市場當韭菜）。從去年年底開始，不同地區展開了 Alpha 社群的理念，藉由持有 NFT 加入 Discord 獲得一手資訊。</p><p>在了解到線上的玩法後，我也希望將「社群」的概念帶到線下，並思考線下的「社群」該如何呈現，最先浮現的便是與會員系統的結合。其實我們可以看到 Opensea 上，1 月開始在 Utility 方面的 NFT 是有崛起的趨勢，我也認為這是未來 NFT 的趨勢，所以才有了這項企劃。</p><p>問：你有看到未來趨勢與需求的眼光，在2 月時 GaryVee發行了在餐廳飲食的 NFT，從「線下」的角度來說，可以說你們兩者有著差不多的概念。所以你是在智能合約上有著一定的研究，同時也是 Valley22 的 founder 對吧？</p><p>Andrew：是的，因為剛好自己有間酒吧，所以會想去嘗試將線下場景與 Web3 做結合。</p><p>問：「累積消費」歷程這件事情蠻酷的，但是對於 Web3 世界的人們來說，更常是坐在電腦前，在電腦前比起去酒吧的時間可能還多，若是不方便常去 Valley 22 的持有者，可以怎樣用到呢？</p><p>Andrew：其實這也是我們遇到的瓶頸之一，因為我們的 NFT 是直接綁定到線下的消費行為，假設不喜歡喝酒或是無法到店面消費的話，NFT 本身是不會有價值的。</p><p><strong>在看了去年的 NFT 市場後，我認為之後的 NFT 可能會通往「實際的線下消費」。畢竟人不可能永遠活在電腦前面。在做這個項目後，也認識到了許多項目的成員，通過與他們的對談中，大家都一致認為實體的社交行為是必須的，交際就是人的本性。在 Web3 的世界中，「社群」的概念是尤為重要的，而「社群」不只是線上，線下的連結也是必要的。</strong></p><p>問：這個就是在 Web2 世界 promote Web3 的過程了，從線上回歸到線下，將 Web3 的應用帶回實體。剛剛也有提到這個 NFT 會是給客人使用的，這邊想問的是你們之前賣的與未來賣的，在你的判斷上，客群上有重疊性嗎？（指購買 NFT 者與原酒吧客戶的重疊性）</p><p>Andrew：對於新手來說，現階段的Web3、區塊鏈門檻還是蠻高的，無論是辦理錢包、交易所的註冊等，這些手續對不懂 Web3 的用戶來說門檻是很高的，所以與過往的客群重疊率是不高的，換句話說就是新的商機。在這方面可以理解成透過這套系統，為商家帶來了新的用戶群體，因為這是用 Web3 的玩法吸引到 Web3 的客人。</p><p>問：知道你們一個「 item 系統」，這個 item的概念是什麼？以及你設計了什麼內容，去在NFT中體現出社交的屬性？（Andrew 答公會及遊戲）</p><p>Andrew：在 item 的機制上線後，會讓人持有者們將 NFT 帶到店家與合約互動，將 PFP 依據部位拆分成 6 個獨立的 NFT，同時線下的消費可以收集到不同的裝備，消費會綁定到裝備上面，這些裝備都是可以做個別的更換。就如同線上遊戲 - 不同的怪物會有不同的裝備，使自己的 PFP 具象化消費歷程。</p><p>問：剛才聽到你說你們好像會設計不同的遊戲，提高社群的黏著度。可以說一下這部分嗎？</p><p>Andrew：本身我就是酒吧的 Owner，在針對消費者上，我們都會做 CRM 的系統。對於商家來說，怎樣去增加「消費者」、「使用者」、「會員」與「商家」的黏著度上，本來就會去做消費數據的分析與量化，我們就是根據這套系統去制定。</p><p>制定的標準則是根據過去的酒吧經營經驗，去分析消費行為並衡量標準。這邊再以線上遊戲為例，關卡會有難度的區分，我們的裝備也會有取得上的難易度差別與綁定的賦能。</p><p>問：多不多客人跟你說不懂玩 NFT？反而要求你推出普通的會員卡？你怎樣去跨越從web2帶人走進web3中間的這一條線？</p><p>Andrew：一開始在推行的時候，多數客人也不懂區塊鏈，前期推廣上相對來說還是有難度的，我們也花了許多時間來做基礎的教學。後來認識到了其他的項目，他們也很願意幫我們去做合作與社群內的推廣，吸引喜歡喝酒的 Web3 玩家近來。簡單來說還是需要出圈，就像是我們今天來到 Matters 進行 AMA，或是其他店家會到我們的店家進行線下的 AMA。</p><p>問：目前你評估客人有什麼反應？這 NFT 日後將會與其他店家展開合作嗎？</p><p>Andrew：會，因為我們的終極目標就是會員系統的殺手級應用，任何想要加入的店家都可以直接套用這套系統，與其他店家合作一直都是我們的目標。</p><p>問：如果我是愛喝 Whisky 的用戶，也收集到了相關的裝備。如果我移民且之後不在台灣了，將裝備拿到二級市場拍賣的話，下個人不就是直接擁有了我的消費歷程？</p><p>Andrew：這是兩個拆開來的東西，Mint 到的 NFT 是遊戲的入場券，賦能則是綁定在裝備 NFT 上。 就像是線上遊戲一樣，有些人喜歡慢慢完成任務收集裝備，有些人則是喜歡直接克金做大老。</p><p>若裝備拿到二級市場販賣，這些喜歡克金的可以不用花時間去累積消費歷程，直接享受權益。有趣的是，會員權益的部分我們想要將數據價值還給消費者，而消費者們可以享受到「Consume to earn」的部分，我們的會員權益是可以拿到回饋與 token，這些 token 可以在線下去做其他消費。</p><p>問：對於未來愈來愈多 NFT 的出現的現在，還有未來的願望有什麼是想做到的？以及有什麼想要推廣的？</p><p>Andrew：把 NFT 帶到線下實際場景做應用，現在就是大家都在定義 Web3 的時代，同時大家也可以找尋任何 NFT 真正該有的應用。</p><h3>社群提問</h3><p>問：想請問這個項目在設計時，跟一般現有會員卡的設計上思路會有什麼不同？ 持有NFT者也可以參與討論嗎（這裡指會員才有的獎勵制度）？</p><p>Andrew：目前初步是將架構勾勒出來，剩下的細節則是不斷跟社群溝通，因為會員系統本來就是根據消費者的需求去制定，站在商家的角度有時會忽略消費者的需求，所以我們一直都是保持開放的心態讓社群的成員提供意見並做出調整。</p><p>問：很好奇怎樣去跟一個剛走進店的新手去解釋錢包的問題？</p><p>Andrew：說法上大致有兩種：完全不懂區塊鏈的用戶與稍懂的。對稍懂的來說只需要說明錢包跟 Mint，剩下就是他們會開始自己玩這套系統；對不懂的來說其實門檻蠻高的，說法偏向是開一個網銀，裡面會存放一點東西。回到前面的問題，所以在客群重疊率上是很低的，因為這套系統與玩法無法引起他們的興趣。</p><p><strong>大家對於區塊鏈、Web3 的概念還停留在「一夜暴富」、「前進虛擬世界」上，我們則是反其道而行，即便是在虛擬世界的人們還是需要吃飯、消費，所以我們的切入點會聚焦在線下的應用。</strong></p>
