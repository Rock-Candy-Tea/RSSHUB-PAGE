<p>在 BidDipper 有一個設定頁面，說的全是 LikeCoin chain 的 <a href="https://likecoin.bigdipper.live/params" rel="noopener noreferrer" target="_blank">Params</a> 即參數，例如：</p><ul><li>驗證人被監禁為甚麼要扣本金？扣多少？</li><li>為甚麼不能一次過取回 12 個委託回報？</li><li>為甚麼通漲率是浮動的？</li></ul><p>其實很多東西都是有解的，只是去不去研究就是了。</p><figure class="image"><img src="https://assets.matters.news/embed/5930b809-cc96-476f-9981-653c22201c53.png" data-asset-id="5930b809-cc96-476f-9981-653c22201c53" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><hr><h2>Staking 委託</h2><p><strong>Bond Denom 綁定面值</strong></p><p>Denom 是 denomination 的簡稱，即是面值。 LikeCoin 委託以 nanolike 為單位（nanolike 為 1 LIKE 之後多 12 個位）。</p><p><strong>Unbonding Time 取回委託</strong></p><p>現時取回委託需時 21 天。</p><p><strong>Max Entries 最大條目數</strong></p><p>一次過解除委託、轉移委託、領取收益的最大條目數為 7（所以某 _ 一次過取回 12 個驗證人的委託回報是不成的，會出現 Error）。</p><p><strong>Historical Entries 歷史條目數</strong></p><p>可查看最後 10,000 條委託數據。</p><p><strong>Max Validators 最大驗證人數目限制</strong></p><p>50 名活躍驗證人。</p><h2>Slashing 削減</h2><p><strong>Downtime Jail Duration 停機時間</strong></p><p>600 SECOND(S)，驗證人節點於 600 秒即 10 分鐘沒有活動，就會被視為停機、jailed 被送入獄及不在 active set 活躍驗證人之列。在此期間，驗證人無法簽署區塊，其委託人將無法獲得委託獎勵。在 DowntimeJailDuration 期過後，驗證人可以發送 unjail 解除監禁交易以恢復節點操作。有關驗證人的各種狀態請先看 <a href="https://matters.news/@daisy/like-coin-%E9%A9%97%E8%AD%89%E4%BA%BA%E7%9A%84%E4%B8%8D%E5%90%8C%E7%8B%80%E6%85%8B-inactive-%E4%B9%9F%E5%88%86%E9%A1%9E%E5%9E%8B-bafyreicmslexuh3zsm5rihnoh7hfowt6xpwyyi2wx7zmy3ka7plgxxpuyy" rel="noopener noreferrer" target="_blank">這篇</a> 會比較好了解。</p><p><strong>Min Signed Per Window 最少簽署窗口</strong></p><p>5%，在每個窗口簽署區塊的最小比例。</p><p><strong>Signed Block Window 簽署區塊窗口</strong></p><p>10,000，即 10,000 個區塊。</p><p>如果驗證人離線時間過長，將被 SlashFractionDowntime 削減並暫時從 active set 活躍驗證人移除至少為 DowntimeJailDuration 的時間，即 10 分鐘。</p><p>究竟長時間離線即是有多長？將由 SignedBlocksWindow 及 MinSignedPerWindow 判定。由於 MinSignedPerWindow 為 5%，SignedBlocksWindow 為 10,000，驗證人必須在指定時間簽署至少 10,000 個區塊中的 5%（10,000 個區塊中的 500 個）方可合符活躍驗證人的規定。 系統會認為錯過 9,500 個連續塊的驗證人已違反活性。</p><h2>Slash Fraction Double Sign 雙重簽署削減分數</h2><p>5 / 100，假如驗證人於同一區塊高度簽署了兩次交易，即雙重簽署，5/100 即由委託人委託予驗證人的0.050000000000000000 ( 5％ ) 本金將會被永久銷毀（即「削減」）。此驗證人將會從 active set 中被「永久移除」。</p><p><strong>Slash Fraction Downtime 離線削減分數</strong></p><p>1 / 10,000，如果活躍驗證人離線時間過長，系統將通過 0.000100000000000000 ( 0.01% ) 的 SlashFractionDowntime 永久銷毀（即「削減」）該驗證人的總委託。 所有委託予違規驗證人的委託人將失去委託給該驗證人的所有 LIKE 的 0.01%。驗證人將被「監禁」，並從 acitve set 中被暫時移除。</p><h2>Minting 鑄造</h2><p><strong>Blocks Per Year 每年區塊</strong></p><p>每年產出 6,311,520 個區塊。</p><p><strong>Goal Bonded 委託目標</strong></p><p>67%</p><p><strong>Inflation Max 最高通漲率/增發</strong></p><p>20%</p><p><strong>Inflation Max 最低通漲率/增發</strong></p><p>7%</p><p><strong>Inflation Rate Change 每年通漲率更改</strong></p><p>不得多於 25%</p><p><strong>Mint Denom</strong></p><p>LikeCoin 鑄造以 nanolike 為單位。</p><p>委託目標、最高和最低通漲率、每年通漲率更改現時由 <a href="https://likecoin.bigdipper.live/proposals/7" rel="noopener noreferrer" target="_blank">議案 7</a> 界定。</p><p><em>鑄造機制旨在：</em></p><ul><li>容許由市場需求決定靈活調整通脹率針以保對特定的 bonded-stake ratio 委託比率。</li><li>讓市場流動性與委託供應之間的平衡。</li></ul><p>為了最大程度決定通貨膨脹/增發獎勵相當於適當的市場利率，LikeCoin chain 依照 Cosmos Hub 採用 moving change rate 移動變化率機制。移動變化率機制確保如果 % bonded 委託百分比高於或低於 goal %-bonded 委託目標百分比，通貨膨脹率將分別調整以進一步激勵或不激勵委託。將委託目標百分比設置為低於 100% 鼓勵整個網絡的人們保留一些沒有委託代幣，有助提供流動性。</p><p><em>它的邏輯為：</em></p><ul><li>如果通脹率低於委託目標百分比，通脹率將增加直到達到最大值。</li><li>如果維持委託目標百分比（在 LikeCoin chain 中為 67%），那麼通貨膨脹率將保持不變。</li><li>如果通脹率高於委託目標百分比，通脹率將下降，直到達到最小值。</li></ul><blockquote>簡單來說就是，如果委託量達不到目標的 67%，為了吸引人去委託，通漲率會增加，讓人們透過委託得到更多 LIKE，而如果委託量太多，就會降低通漲率，讓委託 LIKE 這件事變得不吸引，就沒有那麼多人去做委託從而達到市場平衡。</blockquote><blockquote>現時 LikeCoin chain 的委託率為 45.05% 達不到 Goal Bonded 委託目標 67%，所以現時通漲率就不是 7% 而是 1X% 了。</blockquote><h2>Distribution 分發</h2><p><strong>Base Proposer Reward 基本建議者獎勵</strong></p><p>1%，區塊建議者的基本獎勵獎金，以交易費用作比例。</p><p>所有來自 active set 的活躍驗證人將獲得分配區塊獎勵，而作為區塊建議者的驗證人將額外獲得 0.010000000000000000 ( 1% ) 的交易費用作為獎金。除了該區塊的建議者之外，必須包含至少三份之二活躍驗證人簽署方能成為有效的區塊並讓該驗證人獲得 baseproposerreward 基本建議者獎勵。</p><p><strong>Bonus Proposer Reward 額外建議者獎勵</strong></p><p>4%，建議區塊的驗證人可獲得的最大額外獎勵獎金，以交易費用作比例。</p><p>除有效區塊的建議者外，活躍驗證人均能分享產生區塊的獎勵。 如果該區塊建議者驗證人能囊括來自至少三份之二活躍驗證人預先簽署，則他們有資格獲得高達 4% ( 0.040000000000000000 ) bonusproposerreward ，即在 1% baseproposerreward 基本建議者獎勵以外的額外建議者獎勵。 驗證人收到的額外建議者獎勵取決於提議的區塊中包含多少預先提交簽名（除了必要的 2/3 之外）。</p><p><strong>Community Tax 社區稅</strong></p><p>2%，委託獎勵的比例分流到社群池中。</p><p>在 LikeCoin chain 上委託使參與者有權獲得通貨膨脹（又名「區塊」）獎勵和交易費用。 這些委託獎勵的一部分被轉移到 Community Pool 社群池中，用於成功通過的 Community Pool Spend Proposal 社區支出議案。 communitytax 是決定委託獎勵分流到社群池的比例的參數，目前是所有委託獎勵的 0.020000000000000000 ( 2% )。</p><p><strong>Withdraw Address Enabled 委託人是否可以設置單獨的地址用以接收委託獎勵</strong></p><p>TRUE，即是可以。</p><p>當 withdrawaddrenabled 設置為 true 時，委託人可以指定一個單獨的提款地址（帳戶）來接收委託獎勵。 當 withdrawaddrenabled 設置為 false 時，委託人不能再指定單獨的提款地址。這就是 BigDipper 的 Self Delegate Address 自我委託錢包地址。</p><p>所以：</p><ul><li>驗證人驗證交易，幫忙記帳的回報就是來自 inflation 通漲/增發的新 LIKE。</li><li>每年的通漲率就是以 Minting 鑄造的參數去界定。</li><li>委託人委託予驗證人，也能從驗證人獲得的通漲/增發中得到相應的新發行 LIKE 份額。</li><li>現時每個區塊生成的時間為 5-6 秒，而每一個區塊的產出，就會將增發出的 LIKE 由所有驗證人及持份者按委託比例分配。</li><li>所有驗證人均有責任驗證及決定是否確認新產出的區塊。若最少三份之二的活躍驗證人即 Voting Power 投票權同意，該區塊及所記錄的所有帳目便會被確認並產生新的區塊。</li><li>簽署確認記帳，增發的新發行 LIKE 其中 2% 會當作「稅項」抽起，撥入 Community Pool 社群基金 ，除下 98% 就是驗證人及持份者按委託比例獲分配的份額。每筆交易的交易費用的 98% 也會按委託比例回報給驗證人及持份者。</li><li>如果你委託的驗證人因上面 Slashing 所談的原因未能驗證區塊，除了不能獲得委託回報，你委託的「本金」也會被扣減。</li></ul><h2>Gov 治理</h2><p><strong>Min Deposit 最低保證金要求</strong></p><p>100,000 LIKE。原本的要求是 1,000,000 LIKE，<a href="https://likecoin.bigdipper.live/proposals/8" rel="noopener noreferrer" target="_blank">議案 8</a> 通過修改為 100,000 LIKE</p><p><strong>Max Deposit Period 存入保證金最長期限</strong></p><p>14 天</p><p><strong>Quorum 法定投票權</strong></p><p>40%，即參與議案投票的所有驗證人及持份者，共持有超過已委託投票權（即已委託 LIKE）總量的百份之四十或以上。</p><p><strong>Threshold 門檻</strong></p><p>50%，需要超過 50% 或以上投票讚成方達到議案通過門檻。</p><p><strong>Veto Threshold 強烈反對鬥檻</strong></p><p>33%，即三分之一。如有 33% 投強烈反對，議案不通過之餘保證金也會被沒收並撥入社群基金。</p><p><strong>Voting Period</strong></p><p>7 天，這是由 <a
