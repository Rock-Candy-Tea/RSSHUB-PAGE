src="https://assets.matters.news/embed/8efcd2b3-4909-443c-95d9-dbe154ae18b8.png" data-asset-id="8efcd2b3-4909-443c-95d9-dbe154ae18b8" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p><strong>Market Cap</strong> 市值</p><p>標示為美元。</p><p><strong>Inflation</strong> 通漲率/增發</p><p>LikeCoin chain 的通帳率為 7%，以這個通脹率新發行的 LIKE 將被用作驗證交易的回報，讓所有驗證人及持份者按委託比例分配。</p><p><strong>Supply</strong> 供應量</p><p>LIKE 現時的總供應量。</p><p><strong>Community Poo</strong>l 社群池</p><p>社群池現在有多少 LIKE。</p><p><strong>Validator </strong>驗證人</p><p>LikeCoin chain 是 Cosmos Hub 世系的區塊鏈，而 Cosmos Hub 建基於 Tendermint，依賴驗證人去保護網絡。 驗證人的角色是運行一個 full-node 並通過廣播以其私鑰加密簽署的交易參與共識。 驗證人在區塊鏈中提交新區塊並從他們的的工作獲取收入。驗證人必須通過對提案進行投票參與治理，並根據他們的總委託量去獲得賦權。</p><p>點上下的小三角可以依照驗證人的名字按照 A-Z 或倒轉排序。接下來看到小三角都可以為不同的內容依照大小需求而排序。</p><p><strong>Voting Power </strong>投票權</p><p>LikeCoin chain 是 Proof-Of-Stake ( PoS ) 區塊鏈，驗證人的權重取決於作為抵押的委託代幣的數量。代幣可以由驗證人直接自行委託，也可以由其他持有人委託予他們。</p><p>系統中的任何用戶都可以通過發送創建驗證人交易宣告他們成為驗證人的意圖，並成為驗證人候選人。</p><p>驗證人的權重，即 Voting Power 投票權決定了他們是否是為活躍驗證人。在 LikeCoin chain，只有擁有最多投票權的前 50 個驗證人才會成為活躍驗證人。</p><p>有多少錢包委託了 LikeCoin 給驗證人，驗證人就有多大的投票權。當然這也未必是實際的權限，畢竟現時委託人經已可以自行投票，就算驗證人投 No 委託人也可以投 Yes 去不認同驗證人的投票。</p><p>留意顯示的數字要除 1,000 才是真正已委託給這個驗證人的 LIKE 數量，而綠色的 % 就是驗證人佔整個 LikeCoin chain 委託量的百分比。</p><p><strong>Commission</strong> 佣金</p><p>在 <a href="https://matters.news/@daisy/%E9%97%9C%E6%96%BC-like-coin-%E9%A9%97%E8%AD%89%E4%BA%BA-liker-%E5%BF%85%E9%A0%88%E7%9F%A5%E9%81%93%E7%9A%84%E4%BA%8B-bafyreicul5czhurzo7u7habj6xrwpukdbj57brcgb74esgqbztnjmiix4y" rel="noopener noreferrer" target="_blank">這篇</a> 文章都有談過甚麼是高低佣金，而後來我發現 <a class="mention" href="https://matters.news/@catding" target="_blank" data-display-name="catding" data-user-name="catding" data-id="VXNlcjoyMTkzMQ">﻿<span>@catding</span>﻿</a> <a href="https://matters.news/@catding/%E5%A7%94%E8%A8%97like%E8%BA%BA%E8%91%97%E8%B3%BA-%E4%BD%A0%E8%81%BD%E9%81%8E%E9%80%9A%E8%84%B9%E7%8E%87%E8%88%87%E4%BD%A3%E9%87%91%E7%8E%87%E5%97%8E-bafyreif76gaugeydbjwybckqso2gajplvx5fwpnnbam46nrylb4gvltgwm" rel="noopener noreferrer" target="_blank">這篇</a> 就更加直白。</p><p><strong>Self ( Self Delegate ) </strong>自我委託</p><p>自我委託是驗證人把 LikeCoin 委託回給自己的意思。跟據 Kin <a href="https://ckxpress.com/on-likecoin-validator-commission/" rel="noopener noreferrer" target="_blank">這篇文章</a> 說：「自我委託比率（一般會考慮比率而不是絕對值）越高越好，因為那代表驗證人把收益重新投入而不是拿到市場套利」。</p><p>但當然驗證人也可以換個戶口去委託給自已就是了。</p><p><strong>Delegators</strong> 委託人</p><p>可以看到有多少個錢包委託給這個驗證人。</p><p><strong>Condition</strong> 節點情兄</p><figure class="image"><img src="https://assets.matters.news/embed/0080379c-511a-496c-9098-eee2a8749844.png" data-asset-id="0080379c-511a-496c-9098-eee2a8749844" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>綠色是健康、橙色要注意、紅色就是危險。一般而言去到紅色以後如果驗證人來不及理會，就很有機會變成白色 RIP。</p><p>點進去任何一位驗證人後：</p><figure class="image"><img referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p><strong>Operator Address </strong>驗證人節點地址</p><p>驗證人的節點地址是 cosmosvaloper1 開首，與我們平日的錢包地址不同。</p><p><strong>Self Delegate Address</strong> 自我委託錢包地址</p><p>驗證人自我委託和收到的回報及佣金就是放在這個錢包地址，可以說驗證人的收益就是放在這裡被大家看光光。與平常的錢包地址不同就是這個地址會顯示 Commission 佣金。然而如果你點到零佣金驗證人的話，他們當然不會有 Commission 這一欄。</p><figure class="image"><img referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p><strong>Last Seen</strong> 最後被看見</p><p>判別驗證人最後「生存」的日期。</p><p><strong>Voting Power</strong> 投票權</p><p>除了可以看到驗證人所佔的投票權百份比，以此圖為例：</p><figure class="image"><img src="https://assets.matters.news/embed/8e410046-1d20-41c3-9d7e-c8d6e6ac190d.png" data-asset-id="8e410046-1d20-41c3-9d7e-c8d6e6ac190d" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><ul><li><em>1.19% </em>是這個驗證人於整個 LikeCoin chain 所佔的投票權比例。</li><li><em>5,586,251,339 / 468,394,297,429 </em>中 5,586,251,339 是這個驗證人從委託人所獲得的委託數量，除 1,000 才是實際的 LIKE 數量。所以這位驗證人現時實際被委託的是 5,586,251.339 LIKE ，而 LikeCoin chain 的總委託量是 468,394,297.429 LIKE。</li><li><em>Block 1603,892</em> 所顯示的是最近一個被被驗證人簽署的區塊，可以是 50 個活躍驗證人中任何一名簽署的。</li><li><em>Self Delegated Tokens</em> 這兒顯示驗證人委託了多少代幣給自己，這個數字不用除 1,000。而這兒顯示驗證人委託了 1 LIKE 給自己。 (0%) 是指這個自我委託 LIKE 的數量佔所有委託給這個驗證人的 LIKE 數量的百份比。因為這個驗證人只自我委託了 1 LIKE 給自己，顯示出來的 % 太低就變成 0% 了。</li></ul><p><strong>Delegations, Redelagtions, Undelegations </strong>委託、轉移委託、取回委託</p><figure class="image"><img src="https://assets.matters.news/embed/e830df60-af5b-4d39-a6b3-615251725108.png" 有多少個錢包正在解除對這個驗證人委託。</p><p><strong>Transactions </strong>交易</p><p>關於這個驗證人的交易都會列在下面，都是委託、轉移委託、取回委託及領取回報。</p><h2>stake.like.co</h2><p>在 <a href="https://stake.like.co/" rel="noopener noreferrer" target="_blank">stake.like.co</a> 點選任何一個驗證人，可以看到：</p><figure class="image"><img src="https://assets.matters.news/embed/1c057443-3125-499c-a31c-45090d2f60b7.png" data-asset-id="1c057443-3125-499c-a31c-45090d2f60b7" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p><strong>Rewards</strong> 回報/收益率</p><p>驗證人回報率。這個我打算另外再寫文章去談，所以這邊先不說。</p><p><strong>Voting Power </strong>投票權</p><p>% 和 BigDipper 相同。</p><p><strong>Total Stake</strong> 總委託量</p><p>這邊的 LIKE 就不用除 1,000 了。</p><p><strong>Number of Stakers</strong> 委託人數</p><p>委託錢包數字。留意這個數字有時可以與 BigDipper 差距頗大。我會說 BigDipper 在這方面比較準確。</p><p><strong>Uptime </strong>在線率</p><p>驗證人時否時常在線，100.00% 即是表現很好。</p><p><strong>Validator Since</strong> 驗證人自何時開始</p><p>通常都會看到 Block #0，因為所有驗證人在設定伺服器時，都需要同步起始的區塊記錄。</p><p><strong>Current Commission Rate</strong> 現時佣金率</p><p>和 Big Dipper 相同。</p><p><strong>Max Commission Rate </strong>最大佣金</p><p>此驗證人可以收取的最大佣金率。這個參數是驗證人自訂的，在發送創建驗證人交易後不能更改。</p><p><strong>Max Daily Commission Change </strong>佣金最大變化率</p><p>驗證人佣金的最大每日增幅。 這個參數是驗證人自訂的，在發送創建驗證人交易後不能更改。</p><p><strong>Last Commission Change </strong>最後佣金修改</p><p>顯示驗證人在何時修改過佣金率。</p><h2>為甚麼 BigDipper 和 stake.like.co 的驗證人數目不同？</h2><p>抱歉我也不知道。截止現時 BigDipper 有 96 驗證人而 stake.like.co 有 92 驗證人。但我只知道活躍驗證人的清單是能夠對得上的，所以不用太擔心。</p><h2>為甚麼 BigDipper 和 stake.like.co 看到的驗證人名字不同？</h2><p>也許大家有留意到，在 BigDipper 查看提供保證金或投票名單時，有些人是會出現名字而不是錢包地址的。那是因為他們在 Demsos Network 空投時將 Keplr 錢包地址掛勾了 Desmos Profile 所致。因為 BigDipper 和 Desmos Network 都是 Forbole 旗下的產品，所以有這層關係是很正常的事。以往未有空投前，只有驗證人才能在提供保證金或投票時顯示出名字。</p><p>每個驗證人在設定節點時，依照 Cosmos Hub 的技術要求，都需要輸入 <em>Validator's name (moniker)</em>，即是正常我們看到驗證人的名字。</p><p>然而如果驗證人的 Self Delegate Address 因為領空投而連結了 Desmos Profile，驗證人的 moniker 就會被 Desmos Profile 蓋過，而如果兩者名字不相同，就會出現 Operator Address 驗證人節點地址相同，但在 stake.like.co 顯示 moniker 而在 BigDipper 顯示 Desmos Profile 的現像。</p><p>例如 <a class="mention" href="https://matters.news/@kenookamihoro" target="_blank" data-display-name="Horo" data-user-name="kenookamihoro" target="_blank">Yoitsu</a> 但在 BigDipper 就是 <a href="https://likecoin.bigdipper.live/validators/cosmosvaloper15f60vns6g04vururr8m4mrwkzyjc23uvmkjk9z" rel="noopener noreferrer"
