<h2>前言</h2><p>本文會說明我在 <a href="https://www.fxhash.xyz/" rel="noopener noreferrer" target="_blank">fxhash</a> 會如何透過 <a href="https://www.google.com/intl/zh-TW/chrome/dev/" rel="noopener noreferrer" target="_blank">Chrome 瀏覽器開發工具</a>來看一件作品，所謂的<strong>看</strong>，是指看藝術家用什麼手法，是用程式產生的還是載入圖片來產生，真的很神奇的作品我也會試著打開 JavaScript 看看怎麼寫的，厲害的作品通常也是非常難懂，另外有些藝術家會把程式碼做過編碼，變的不可讀，那就沒辦法看 coding 的寫作方式了。</p><p>對於 Tezos 生成藝術平台 fxhash 如果不太熟悉的話可以看這幾篇文章。</p><ul><li><a href="https://matters.news/@aquaponics_kana/276876-%E5%BE%9Efxhash%E9%96%8B%E5%A7%8B%E8%AA%8D%E8%AD%98%E7%94%9F%E6%88%90%E5%BC%8F%E8%97%9D%E8%A1%93-bafyreibypeulz6c6ity2yambq3qckrdkflnsljcru2blf5n7njuemir5ja" rel="noopener noreferrer" target="_blank">從fxhash開始認識生成式藝術 - 水耕香菜</a></li><li><a href="https://matters.news/@interactiont/287892-%E9%80%B2%E5%85%A5tezos-nft%E7%9A%84%E6%96%B0%E6%89%8B%E5%8C%85-%E5%BE%9E%E6%8C%91%E9%81%B8%E4%B8%80%E4%BB%B6fxhash%E7%9A%84%E4%BD%9C%E5%93%81%E9%96%8B%E5%A7%8B%E5%90%A7-bafyreihri77fhk5xghvrgptrdzr4766ukkidrjh5nzdn36tm4msrwg4nx4" rel="noopener noreferrer" target="_blank">進入tezos-nft的新手包-從挑選一件fxhash的作品開始吧</a></li></ul><h2>是圖片還是程式？</h2><p>有些作品在看到預覽圖時，第一個好奇的點是，這個作品是全部程式產生的嗎？也太細緻，太像照片，太逼真了吧！？當你看到一個感興趣的作品時，也許會有同樣的問題。</p><figure class="image"><img src="https://assets.matters.news/embed/b79ce934-ac05-48a7-8ac0-0a869156cc02.jpeg" data-asset-id="b79ce934-ac05-48a7-8ac0-0a869156cc02" referrerpolicy="no-referrer"><figcaption><span>Mediterranean #5 on fxhash https://www.fxhash.xyz/gentk/621626</span></figcaption></figure><p>以 <a href="https://www.fxhash.xyz/gentk/621626" rel="noopener noreferrer" target="_blank">Mediterranean</a> 為例，他是即時生成給你看的，所以基本上你可以假設他是程式即時生成的。</p><p>另外有些作品畫出很漂亮的造型，你預期應該是程式去畫出來的，但也有可能是載入了圖片去完成。大部分的情況下 fxhash 或作者自己會在作品介紹頁給一個 <strong>Image composition</strong> 的標籤，表示這個作品有透過圖片合成的方式去呈現部分效果。有時候載入圖片是很自然的作品要素，例如 <a href="https://www.fxhash.xyz/generative/slug/gtx" rel="noopener noreferrer" target="_blank">GAN ⫷glitch⫸ Ting Xu | 幹亭序</a>，就是用其中一個作者自己的書法創作圖檔加上演算法堆疊圖像處理後的效果。<br class="smart"></p><figure class="image"><img src="https://assets.matters.news/embed/88dc2222-41a7-4609-8512-919724282225.png" data-asset-id="88dc2222-41a7-4609-8512-919724282225" referrerpolicy="no-referrer"><figcaption><span>GAN ⫷glitch⫸ Ting Xu | 幹亭序</span></figcaption></figure><p>所以說 <strong>Image Composition</strong> 這個標籤並不代表一個判斷好壞標準，在創作中有沒有用圖片只是手法上的差異。<br class="smart"></p><h2>Chrome dev tool 開發工具</h2><p>要用開發工具檢視作品時，請在全螢幕模式下瀏覽，如果是在作品資料頁的話請按右邊的 Open（以 <a href="https://www.fxhash.xyz/generative/slug/playing.cccccards" rel="noopener noreferrer" target="_blank">playing.cccccards</a> 作品為例）</p><figure class="image"><img src="https://assets.matters.news/embed/22bae044-c481-4bc0-880a-8d0b4703235c.png" data-asset-id="22bae044-c481-4bc0-880a-8d0b4703235c" referrerpolicy="no-referrer"><figcaption><span>點擊 open 打開全螢幕模式</span></figcaption></figure><p>在全螢幕模式下，點滑鼠右鍵叫出選單，然後選 <strong>Inspect</strong> 。（鍵盤快速鍵 <strong>F12</strong> 或 Mac<strong> opt + cmd + I</strong> )。如果你用的是 Microsoft Edge 或是 Brave 瀏覽器，一樣可以使用這個瀏覽器內建的開發工具。</p><figure class="image"><img src="https://assets.matters.news/embed/ed29ba74-c4ab-482d-8cc7-6026de9f349f.png" data-asset-id="ed29ba74-c4ab-482d-8cc7-6026de9f349f" referrerpolicy="no-referrer"><figcaption><span>滑鼠右鍵打開選單</span></figcaption></figure><p>此時你已經打開了 Chrome 開發工具，請在工具列的頁籤 (tab) 切換到 <strong>Network</strong> (網路) 並且重新載入一次頁面 (Refresh/reload)，我們要來看看這個 NFT 作品會載入哪些檔案。</p><figure class="image"><img src="https://assets.matters.news/embed/9bd51091-4671-441f-944b-fe4a82d128eb.png" data-asset-id="9bd51091-4671-441f-944b-fe4a82d128eb" referrerpolicy="no-referrer"><figcaption><span>點 Network 然後重新載入頁面</span></figcaption></figure><p><strong>Network</strong> 的作用是可以看這個網頁載入了哪些不同類型的檔案，當你重新整理頁面之後，Chrome 開發工具會記錄下之後產生的每一個網路連線（也就是去網路上某處下載檔案）。</p><p>重新整理後，你的開發工具應該是保持開著的狀態，並且你會看到刷了一排的 svg 檔載入，以及其它很多看不太懂的資訊，真正對我們來說重要的是中間下面這個表格而已，其它地方可以先不用管。</p><figure class="image"><img src="https://assets.matters.news/embed/5ee702d1-7423-473e-afa7-fb89559e0b7b.png" data-asset-id="5ee702d1-7423-473e-afa7-fb89559e0b7b" referrerpolicy="no-referrer"><figcaption><span>列出網路連線記錄</span></figcaption></figure><p>你可以看到右邊有 scrollbar，可以捲到最上面看看，從打開作品的最一開始時，依序載入了什麼東西。表格從最上面開始是先載入了一個 Document 類型的檔案（也就是 HTML 網頁），然後是撲克牌上會用到的字型檔，以及圖案會用到的一些SVG向量圖檔。</p><figure class="image"><img src="https://assets.matters.news/embed/a1dc7f48-2de0-4d8f-9919-15956930e534.png" data-asset-id="a1dc7f48-2de0-4d8f-9919-15956930e534" referrerpolicy="no-referrer"><figcaption><span>各種檔案類型</span></figcaption></figure><p>如果是圖檔，可以點選進去預覽看看是什麼圖</p><figure class="image"><img src="https://assets.matters.news/embed/b9c35a03-fb33-41a7-9b67-1130d3aa2d52.png" data-asset-id="b9c35a03-fb33-41a7-9b67-1130d3aa2d52" referrerpolicy="no-referrer"><figcaption><span>選 Preview 預覽一般圖檔</span></figcaption></figure><p>有些 SVG 檔案無法預覽，但可以用右鍵選單另開新頁來做預覽</p><figure class="image"><img src="https://assets.matters.news/embed/1525aeea-cb9a-4827-8470-c11124d90047.png" data-asset-id="1525aeea-cb9a-4827-8470-c11124d90047" referrerpolicy="no-referrer"><figcaption><span>另開新頁預覽 SVG</span></figcaption></figure><p>以這個作品來說，算是 PFP （Profile Picture) 類型的 NFT，目的並不是為了以生成藝術的形式來呈現，所以像這樣事先畫好元件來隨機組合會是比較合理的作法。藉由這個方式，我們也可以檢視一些剛上架或是即將上架的作品，是用什麼樣的形式來產生，讓我們在面對作品時，有比較多的參考資訊可以做評估。</p><p>有時候藝術家使用的圖片媒材，跟實際作品呈現會有很大的差異。以阿亂的 <a href="https://www.fxhash.xyz/generative/slug/chaos-memory" rel="noopener noreferrer" target="_blank">chaos memory</a> 作品來說，他就利用了大量的灰階點陣圖來做處理，最後的效果非常的虛幻，但又很有生命感（有機），看起來非常震撼。豆泥有寫一篇很精彩好看的介紹（<a href="https://matters.news/@mashbean/289341-%E6%B7%B7%E6%B2%8C%E4%B8%89%E5%B9%95%E5%8A%87-%E8%A7%A3%E7%A2%BC%E9%98%BF%E4%BA%82%E7%9A%84%E5%8A%A0%E5%AF%86-%E5%8F%96%E6%A8%A3-%E6%BC%94%E7%AE%97%E8%AC%8E%E5%9C%98-bafyreif7beo3q6j3p7hf7caoaoebidbftk5snwxumnjqyrbblfk7vnppwu" rel="noopener noreferrer" target="_blank">混沌三幕劇 — 解碼阿亂的加密、取樣、演算謎團</a>）。<br class="smart"></p><figure class="image"><img src="https://assets.matters.news/embed/964e86b3-e4ea-4be3-a97c-322dfa4748ce.png" data-asset-id="964e86b3-e4ea-4be3-a97c-322dfa4748ce" referrerpolicy="no-referrer"><figcaption><span>chaos memory by 阿亂</span></figcaption></figure><p>對於想要學習生成藝術的創作者來說，Chrome 開發工具最大的好處是可以學習別人怎麼寫。最常見的 <a href="https://p5js.org/" rel="noopener noreferrer" target="_blank">p5.js</a> 平台會把主要程式預設放在一個 sketch.js 的檔案裡，我們就可以從這個檔案去研究別人怎麼處理細節跟筆刷。不過有些藝術家會希望把自己的作品加密，避免產生太多 copymint （山寨) 的麻煩。</p><figure class="image"><img src="https://assets.matters.news/embed/f3a79834-dc85-4d32-9715-8e91ca25af18.png" data-asset-id="f3a79834-dc85-4d32-9715-8e91ca25af18" referrerpolicy="no-referrer"><figcaption><span>用 Network 頁籤來檢視 sketch.js 檔案內容</span></figcaption></figure><h2>小技巧：Features 檢視</h2><p>有些作品會設定 features 來區分不同屬性跟稀有度，並且在開始 mint 之後，會在作品頁上顯示已經 mint 的作品各種 features 的統計結果。</p><figure class="image"><img src="https://assets.matters.news/embed/d66ca484-6626-464e-ba1b-932d9aace755.png" data-asset-id="d66ca484-6626-464e-ba1b-932d9aace755" referrerpolicy="no-referrer"><figcaption><span>(kinder)Garden, Monuments #3 的 features 統計</span></figcaption></figure><p>我們在開放 NFT mint 之前，也可以在全螢幕模式下去看作品有設定哪些屬性（可以變化的元素）。檢視的方式也是在開發工具裡，切換到 <strong>Console</strong> 的頁籤，你會看到有些作品直接會在 Console 這裡（有點像資訊區或除錯區的地方）顯示了 feature 的物件</p><figure class="image"><img src="https://assets.matters.news/embed/ea40818b-e292-4c65-b8e1-22968191fd87.png" data-asset-id="ea40818b-e292-4c65-b8e1-22968191fd87" referrerpolicy="no-referrer"><figcaption><span>Console 顯示 features 參數物件</span></figcaption></figure><p>如果點那個黑色小箭頭展開的話，就可以看到這個作品編號，被指定了什麼 feature 屬性。以上面那個國王頭為例，background 屬性是 <strong>reflection</strong>，鬍子是被指定了編號 18 的鬍子 (<strong>bear 18</strong>) 以此類推。</p><figure class="image"><img src="https://assets.matters.news/embed/91ce01cc-f18f-4368-a63f-07960402221f.png" data-asset-id="91ce01cc-f18f-4368-a63f-07960402221f" referrerpolicy="no-referrer"><figcaption><span>展開 feature 物件檢視</span></figcaption></figure><p>有些作品並不會主動在 Console 顯示 features 物件，如果想要看的話，可以直接在 Console 輸入變數名稱 <strong>window.$fxhashFeatures </strong>按 Enter ，還是能看到 feature 屬性的。</p><figure class="image"><img src="https://assets.matters.news/embed/4c649d48-7181-4c5c-952b-736bc3385d61.png" data-asset-id="4c649d48-7181-4c5c-952b-736bc3385d61" referrerpolicy="no-referrer"><figcaption><span>自己手動在 console 輸入 window.$fxhashFeatures</span></figcaption></figure><p>如果手動輸入了得到 undefined 的結果，表示這個作品沒有定義 features。<br class="smart"></p><figure class="image"><img src="https://assets.matters.news/embed/b5d84359-5ef2-420b-8d32-ab100102e98f.png" data-asset-id="b5d84359-5ef2-420b-8d32-ab100102e98f" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><h2><br></h2><h2>看生成藝術作品的變化</h2><p>我覺得生成藝術最好玩的地方是透過隨機性帶來不同作品的變化，不過有些作品只有提供限定的幾組變化 (variations) 給你預覽。我們還是可以從網址列去調整參數看看作品的不同變化。</p><figure class="image"><img src="https://assets.matters.news/embed/13029070-2f70-4887-b238-d0b5710aeabf.png" data-asset-id="13029070-2f70-4887-b238-d0b5710aeabf" referrerpolicy="no-referrer"><figcaption><span>只提供四組變化的作品預覽</span></figcaption></figure><p>先進入全螢幕模式的網頁，透過修改 <strong>fxhash</strong> 這個參數，來產生不同的結果。（這裡不用 Chrome 開發工具喔）</p><figure class="image"><img src="https://assets.matters.news/embed/a259133b-ce9c-4935-a293-840c5e3dbd33.png" data-asset-id="a259133b-ce9c-4935-a293-840c5e3dbd33" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><pre class="ql-syntax" spellcheck="false">fxhash=ooQWGcQhgNGY2p9DeT9iKGU5kQ7BBdYiCQzpMCgGQgyPGQ6Qv2T
</pre><p>一個正規的 fxhash 參數會是 oo 開頭然後一串亂碼（雜湊 hash），只要把這串亂碼的後面幾個字母用隨便打幾個字替換掉，像我隨便打 11111 替換最後大約10個字母，可以看到另外的玫瑰花造型結果。注意整個 <strong>fxhash = </strong>後面的字串還是要保持一定的長度才比較能看出變化，自己多試幾次就會知道了。</p><figure class="image"><img src="https://assets.matters.news/embed/bb7a623c-364c-4ebc-a4e4-331defd51d64.png" data-asset-id="bb7a623c-364c-4ebc-a4e4-331defd51d64" referrerpolicy="no-referrer"><figcaption><span>修改網址列上 fxhash 的參數</span></figcaption></figure><p>好了，以上就是用 Chrome 開發工具看 fxhash NFT 作品的一些方法，希望大家對 tezos 上目前很熱門的生成藝術平台 fxhash 有更多瞭解！</p><p><br></p><p><br class="smart"></p>
