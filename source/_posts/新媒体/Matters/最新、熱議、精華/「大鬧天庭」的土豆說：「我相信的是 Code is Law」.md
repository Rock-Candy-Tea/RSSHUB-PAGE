<p>自從 2020 年 <a class="mention" href="https://matters.news/@potato_pepper" target="_blank" data-display-name="土豆炒青椒" data-user-name="potato_pepper" data-id="VXNlcjozNzE3OQ">﻿<span>@土豆炒青椒</span>﻿</a> 在馬特市發出第一篇文以來，<a href="https://matters.news/@potato_pepper/223580" rel="noopener noreferrer" target="_blank">見證過社區裏許多次的論戰</a>，甚至吵架。直到 2022 年 1 月，他也成為炮轟論戰的中心人物，集結了社區中的理性與情緒性爭議。</p><p>半年多過去了，我才與土豆私下聯繫，試圖重新釐清當時所發生的事，還能帶給我們什麼啟示。社區吵架真的無可避免嗎？</p><h2>細說從頭</h2><p>首先說明當時的論戰背景，若你已熟悉這些資訊，可跳至下一段落。</p><p>LikeCoin 區塊鏈在議案 11 之後，支援活躍驗證人最高 50 名，這表示驗證人所持、加上受委託的 LikeCoin 數量，必須進入排名前 50 名，稱之為「活躍」驗證人，才會有驗證人收益。</p><p>為了鼓勵更多人來擔任驗證人，議案 20 成立了「社群基金委託委員會（Community Delegation Committee, 以下簡稱為 CDC）」，負責管理一筆社群池裏的公共基金，可以委託給對社群有貢獻的驗證人，並由七人小組來人工審核。</p><p>對於許多初始基金不多的新進驗證人來說，這份「社群委託」基金不但能增加收益，最重要的是能協助進入前 50 排名，才會有收益回報，得以支付節點營運成本。</p><p>問題在於，「對社群有貢獻」是主觀的質化標準，至截稿時「<a href="https://github.com/likecoin/docs/blob/d4607647c42bc392dc414ed731aa0e69d6cee527/guides/governance/what-is-a-validator/community-delegation.md" rel="noopener noreferrer" target="_blank">社群委託</a>」文件中，審批準則如下：</p><blockquote>社群基金審批委員會檢視（但不限於）：<br class="smart">1. 驗證人節點是不是時常在線。<br class="smart">2. 驗證人有沒有在 <a href="http://discord.gg/likecoin" rel="noopener noreferrer" target="_blank">Discord</a> 活躍於 LikeCoin 的發展並獲得足夠社群認可。<br class="smart">3. 驗證人有沒有在網站或其他渠道宣傳 LikeCoin 及與他們的委託人/持份者溝通。<br class="smart">4. 驗證人的佣金設定是不是依照社區共識的最低佣金進行調節，及是否合理。<br class="smart">5. 驗證人有沒有參與議案投票。<br class="smart">6. 驗證人有沒有依照他們在 #Validator Intro 所寫的為社群帶來好處<br class="smart">當驗證人真心投入公共事務，自自然然會被社區認可。驗證人獲得委員會五個同意 ( 5 Aye ) 或以上即可獲得社群委託或續期。<br class="smart"></blockquote><p>土豆向 CDC 申請兩次社群基金委託，皆遭駁回後，發表「<a href="https://matters.news/@potato_pepper/228575" rel="noopener noreferrer" target="_blank">炮轰 LikeCoin 社群委托及社群委托改革</a>」，對 CDC 的制度提出批判。這篇文指出 「CDC 成員身兼驗證人但投票未避嫌」的問題，以及可能作惡的制度缺陷，但因措辭激烈，繼而引發誹謗爭議。</p><p>該文發表四天後，土豆即提出議案 32 要求解散 CDC，並在 <a href="https://matters.news/@ckxpress/234855" rel="noopener noreferrer" target="_blank">LikeCoin 發起人高重建主動投入抵押資金</a>後，進入投票程序，最終以：</p><blockquote>Yes: 31.48%<br class="smart">No: 53.53%<br class="smart">Veto: 0.17%（強烈反對）<br class="smart">Abstain: 14.83%（棄權）<br class="smart"></blockquote><p>該議案遭否決，也是迄今唯一因「反對票高於贊成票」而被否決的議案。</p><h2>半年之後</h2><p>對土豆來說，當時發表了炮轟文，對他來說最沒預料到的，是因此結識了 Welike。</p><p>Welike 是一個團體，有經營<a href="https://welike.weshare.hk/" rel="noopener noreferrer" target="_blank">新聞網站</a>與推廣區塊鏈知識等，不過最為人知的應該是「LikeCoin 零佣金驗證人」，並因而成為「神秘的巨鯨」（意即挾有龐大資金者）。</p><p>在 LikeCoin 世界裡，你可以委託資金給驗證人，並分潤驗證人的收益，其中驗證人通常會收取部份比例作為佣金，畢竟伺服器資源需要成本。而零佣金就表示驗證人不收取佣金，因此可最大化委託人的收益，Welike 因此拿下極高份額的委託資金。</p><p>然而在 LikeCoin 世界裡，驗證人是「<a href="https://github.com/likecoin/docs/tree/d4607647c42bc392dc414ed731aa0e69d6cee527/guides/governance/what-is-a-validator" rel="noopener noreferrer" target="_blank">銀行兼立委</a>」，獲得委託資金愈高的驗證人，所代表的投票權也愈高，這就導致了巨鯨的中心化權力風險（儘管 LikeCoin 實行的是「流動式民主」，委託人可以繞過驗證人來直接投票）。</p><p>土豆表示，Welike 的觀點與他很一致，不喜歡 CDC 的社群基金申請與續約制度。我翻查 Welike 的投票紀錄，不僅在土豆發起解散 CDC 的議案 32 投下 Yes，在更早之前成立 CDC 的議案 20，甚至是「唯一沒有投下 Yes」的驗證人（投的是棄權 Abstain），立場非常鮮明。</p><p>為什麼土豆與 Welike 要反對 CDC 審查社群基金委託的制度？</p><h2>Code is Law（程式碼即規則）</h2><p>土豆認為，LikeCoin DAO 仰賴於「信任」CDC 七人小組，既有的鬆散制度很容易產生弊端，也確實發生過 CDC 成員「自己投給自己」而未迴避利益衝突的事實；他理想中的 DAO 是符合區塊鏈的「無需信任（trustless）」精神。</p><p>土豆：</p><blockquote>现行的 LikeCoin 很多方案都是建立在信任的基础之上。而 DAO 有一个特点，就是不需要信任就能执行。所以 LikeCoin DAO 并没有解决区块链无信任问题。<br class="smart"><br class="smart">传统交易都是靠大型机构资产担保，政府信誉，国家授权银行之类达成，而且银行股市什么的注册都是实名制。而区块链一个特征就是匿名，没有身份认证。所以我认为区块链时代需要依靠建立“无信任”背景制度解决匿名环境下安全交易，不用担心财产损失。<br class="smart"><br class="smart">还有就是，在财产受到损失的时候，一般都是靠之前的银行注册实名制追究刑事民事责任什么的。而且就算实名制也存在跑路、追不回逃款之类的。到了匿名制上，追查损失几乎就更不可能了。<br class="smart"></blockquote><p>這讓我想起了 Cosmos 生態系先前有過 <a href="https://matters.news/@ckxpress/269588" rel="noopener noreferrer" target="_blank">Juno 議案 16 通過沒收鯨魚資產</a>的問題，可說是一種多數暴政。除此之外，crypto 資產損失通常只會是個人操作失誤或遭駭客盜取。</p><p>「無信任」確實是過往區塊鏈革命的重要核心價值之一，其中最廣為人知的信念便是「code is law（程式碼即規則）」，一切依據公開的程式碼來自動運作，無需再額外信任他人。</p><p>土豆：</p><blockquote>我是比较倾向 code is law，就像传统的法律也是可以修改的一样，我们可以通过升级代码完善 DAO。类似于 Web2 码工升级产品一样，通过升级代码实现更多的 DAO 的应用。Web2 的产品有一个问题，就是对用户隐私的保护，所以 Web3 时代应该有新的应用可以解决 Web2 的诸如数据采集等问题。<br class="smart"><br class="smart">我不认为 code is law 完全自由经济模式会在 Web3 时代完全取代曾经的某机构制定规则之类的，类似于 Web2 时代出现的电商，SNS（Social Network Service，社群网络服务）为人们提供了新的购物和网络互动模式，但传统的实体商店，书本等仍然不能取代。所以，我希望 code is law 可以在 Web3 给用户提供新的应用体验，解决 Web2 的某些问题，比如网络身份、大型机构垄断之类的。<br class="smart"><br class="smart">另外，code is law 基本上也是从某种程度上把人力解决出来，让人不再去做重复的事（比如 CDC 评委每月人工审核），这和人工智能机器学习之类的理念也差不多。<br class="smart"><br class="smart">Code is law 也是人治，因为 code 不也是人写的嘛。我们为什么要写这样的代码，需要解决什么问题（类似 Web2，我们为什么要写代码开发这个产品一样）？这些都是需要人脑经过思考讨论后才有的嘛，只是我们把要完成的事交给代码去处理。如果我们有了更好的方案，我们通过修改代码实现升级，类似于法院通过修改宪法升级制度一样。这就是我对 Web3 的一点期望。<br class="smart"></blockquote><p>當土豆充滿理想性地，理性地暢談這些理念時，彷彿換了一個人似的。但在對談過程中，他仍不時會吐露出對 CDC 的不滿，就跟半年前炮火四射的情形一樣。</p><h2>Code is Law 能營運一個 DAO 嗎？</h2><p>回顧 2016 年 Ethereum 創始時期，領頭鋒的 The DAO 被駭客利用合約漏洞盜走了大筆資產，最後使得 Ethereum 社群決議「硬分叉（hard fork）」出來，來強行取回駭客資產，留下堅持「code is law」的 Ethereum Classic 反而難以為繼，三不五時被 51% 算力攻擊。</p><p>或許可以說，當時的 Ethereum 社群成員認為「貨幣體系必須為民服務」比「code is law」更重要。這一個歷史性的標誌事件，常讓我誤以為「code is law」已不被接納。迄今仍有不少 Web3 專案因合約漏洞而被駭走資產，但 Ethereum 早已不再處理這種事，或許只有當初的 The DAO 資產是「大到不能被盜」。</p><p>我前陣子在 <a href="https://matters.news/@denkeni/294152" rel="noopener noreferrer" target="_blank">Matters Lab Discord 聽了滿多外部 DAO 參與者</a>，來分享他們的治理方式，確實有一些 DAO 是相當「人治」的，要加入那種 DAO 並不容易，甚至需要獲得內部成員的「信任」。</p><p>我想，所謂組織治理通常都是人治、法治兼具，最終都是為了確保內部成員「互信合作」，組織才有運作的可能。由此觀點來看，LikeCoin DAO 的發展模式本就是以人治為主、法治為輔。那麼除了抱怨 CDC 申請過程的體驗不佳之外，更根源的問題應該是：Code is law 能營運一個 DAO 嗎？</p><p>土豆：</p><blockquote>需要时间检验。现在还在初期，大家都在尝试构想。CDC 现在做的也可以算一种尝试，而且还得到了一定认可，比如说投票（註：應指成立 CDC 的議案 20）被通过了。</blockquote><p>在更好的 DAO 典範出現之前，我們還是會循以往經驗。因為一個組織的初創成立，最重要就是靠人的互信，這要如何靠程式碼去定義？</p><p>即使我們可以明確去定義組織的目標（譬如做一個 App 想要達到的功能），寫成 DAO 的智慧合約來招募工程師，功能開發完成後，由智慧合約驗收通過就給錢，那恐怕就只是廉價外包形式了。譬如，程式碼品質要如何定義，寫進智慧合約裡？最終還是經常仰賴人來把關。</p><p>畢竟，我還不知道我們真能用「無需信任、冷冰冰的規則」來聚集人群，並有效地長久營運下去嗎？又譬如，一個社群可以仰賴獎勵制度，來長久經營下去嗎？但這卻似乎是近年 Web3 社群的常態。</p><h2>自由市場與國家補助</h2><p>另一項土豆所點出的問題，則是針對「社群基金委託」這項制度：</p><blockquote>我的观点是 CDC 搞的社群委托，把节点分为了国有节点和私营节点两个阵营。国有节点依靠政府扶持，节点佣金普遍比私有节点高。</blockquote><p>當 CDC 以「社群貢獻度」來審核社群基金委託時，自然能以這樣的評判標準，來吸引具有相同價值觀的節點驗證人，獲得更高的佣金。</p><p>但反之，這就使得節點驗證人，很可能過度仰賴社群基金委託才得以盈利。自此，「國家補助」可能過度干預了自由市場。</p><p>在 LikeCoin 文件「<a href="https://docs.like.co/v/zh/guides/governance/what-is-a-validator/community-delegation" rel="noopener noreferrer" target="_blank">社群委託</a>」中指出：「社群委託基金計劃旨在支持一些希望為 LikeCoin chain 治理作出貢獻的新加盟驗證人提供初創基金」，這確實類似於國家補助發展早期產業。此外，「續期」制度能持續檢視驗證人的社群貢獻度績效。</p><p>然而，作為「新加盟驗證人的初創基金」，社群委託不但沒有「退場」機制，反而新增了「高社群委託」可獲得 1.5 倍的委託基金；這就使得「社群委託基金」實質上成為「社群貢獻獎勵」。</p><p>假若驗證人們都在社群中一起活躍貢獻，這樣不是很好嗎？讓我們來觀察另一個 LikeCoin DAO 的現象。</p><h2>LikeCoin DAO 議案回顧</h2><p>回顧 LikeCoin DAO 已完成的前 52 個議案中（<a href="https://dao.like.co/proposals/past" rel="noopener noreferrer" target="_blank">參考出處</a>）：</p><ul><li>有 47 個進入投票程序</li><li>有 16 個以 100% Yes 通過，佔全部投票議案約 34%</li><li>有 38 個以超過 99% Yes 通過，佔全部投票議案約 81%</li><li>有 43 個以超過 80% Yes 通過，佔全部投票議案約 91%</li></ul><p>低於 80% Yes 的議案僅有 3 個（依 Yes 投票率排序）：</p><ul><li>【議案 20】75.76% Yes / 24.24% Abstain<br class="smart">Propose new set of rules and structure of the Community Delegation Committee</li><li>【議案 9】53.57% Yes / 46.43% No<br class="smart">Delegate the like in the ecological pool<br class="smart"></li><li>【議案 32】31.48% Yes / 53.53% No / 0.17% Veto / 14.83% Abstain<br class="smart">Proposal:Dismiss LikeCoin Community Delegation Committee</li></ul><p>目前 47 個投票議案中，僅有 2 案未通過：一案是土豆提的議案 32，另一案為議案 51 則是因投票人數（39.02%）未達法定門檻（40%）而未通過，但投票是 100% Yes。</p><p>如此高比例的 Yes 投票通過率，我認為有兩種可能性：<br class="smart"></p><p>一是這種「民主」效率太低，幾乎每個議案都經拍手才通過，等於沒有效率。加密貨幣投資基金 Dragonfly 共同創辦人 Haseeb Qureshi 甚至曾撰文認為「<a href="https://hackernoon.com/blockchains-should-not-be-democracies-14379e0e23ad" rel="noopener noreferrer" target="_blank">Blockchains should not be democracies</a>」:</p><blockquote>Businesses are not democracies, militaries are not democracies, nonprofits are not democracies, and open-source software projects are not democracies. There are good reasons for this.</blockquote><p>不過近期 LikeCoin DAO 透過議案 36 成立 Tech Subdao、議案 38 成立 Marketing Subdao，則近似於三權分立中的「行政」機構，並由「立法」議案來審核預算，是踏出合理的一步。</p><p>二是這個社群的同質性太高。正如 <a class="mention" href="https://matters.news/@leafwind" target="_blank" data-display-name="leafwind" data-user-name="leafwind" data-id="VXNlcjo1ODU">﻿<span>@leafwind</span>﻿</a> 曾在「<a href="https://matters.news/@leafwind/237828" rel="noopener noreferrer" target="_blank">議案 32 棄權原因</a>」一文中指出：</p><blockquote>畢竟最慘的狀態不是有人提出異議，而是完全沒有異議。</blockquote><p>土豆在提出議案 32 遭否決，但仍獲得 31.48% Yes 與少見的高比例 14.83% Abstain。接著很快又提出納入「公共節點」的議案 34，但未能再引發社群討論，最終無其他人投保證金，因押金不足未能進入投票程序。</p><p>土豆表示，那之後幾位前輩「都提示过我，时间宝贵，过多折腾这事没意思」。</p><p>自此，LikeCoin DAO 的「反對黨」勢力幾乎消失殆盡。那之後的 19 個投票議案中，更有高達 17 個獲得超過 99% Yes。</p><h2>如果當時⋯⋯</h2><p>然而，在推動議案 32 時期，土豆的言論用詞多以「有罪推定」的問題，並造成社群紛爭。當時在 LikeCoin Discord 討論中，leafwind 更<a href="https://discord.com/channels/763001015712350231/763001016358535203/936072881908109393"
