<p>Crescent 的 CRE 與 bCRE 真是非常有趣的東西，<a href="https://crescentnetwork.medium.com/spotlight-on-liquid-staking-537001825d6f" rel="noopener noreferrer" target="_blank">此文</a> 解釋 Liquid Staking 及 dPos 理念，想了解不妨看看下文介紹。</p><hr><p>區塊鏈的關鍵基礎之一是治理。 驗證人透過共識機制驗證網絡上發生的所有交易，委託讓用戶對網絡的變更進行投票。 許多區塊鏈網絡運用共識機制，用戶可以在其上委託他們的代幣以獲得委託回報，讓停滯的資產發揮作用，同時對治理提案擁有投票權。</p><p>於 DeFi 用戶角度，Staking 可能是一把雙刃劍。 雖然傳統的委託機制能產生一定的回報，但資產不再具有流動性，並且無法採取二級或三級投資動作以潛在地增加總回報。資產一直鎖定在委託模組中直到被取回委託，導致機會成本增加。</p><p>Crescent Network 利用委託權益證明 ( Delegated Proof of Stake, dPoS ) 共識機制，在該機制中，代表（驗證人）被選舉出來代表選民（委託人）投票及管理區塊鏈。 這些代表們會獲得委託獎勵，而獎勵亦會按照委託數量比例分配給各個選民。這種方法讓預先定義數量的驗證節點快速達成共識，終極提升網絡性能。</p><p>於委託權益證明 ( dPoS ) 機制之上，Crescent Network 透過實施流動委託 ( Liquid Staking ) 為委託資產增加了更多流動性。委託的資金仍處於託管狀態，但不會如傳統委託那樣被鎖定及無法使用。</p><p>流動委託透過二次投資機會以解決資產處於休眠狀態的問題，同時仍然賦予用戶相同的投票及委託權。 Crescent Network 的流動委託功能由列入白名單的流動委託驗證人組成，這些驗證人直接從流動委託模組接收相同的委託。 當用戶對他們的 CRE 進行流動委託時，委託的數量將平均分配在白名單流動委託驗證人的組合中，並由治理機制管理。</p><p>流動委託的主要好處是用戶資產的流動性超越了傳統委託。當用戶委託 CRE 時，他們會收到名為 bCRE( pegged staked representative tokens ) 的代幣，可以將它投資於其他用途以獲得額外獎勵。 bCRE 代表 CRE 委託的數量加上累積的委託獎勵。 通過持有 bCRE（及/或向 bCRE 配對池提供流動性），委託獎勵被添加到由所有流動委託 CRE 獎勵組成的流動委託基金中，就像傳統金融機構的「基金」一樣。</p><p>bCRE 代表了這種委託資產的「流動委託基金」中的擁有權比例。流動委託獎勵與原始委託金額成正比添加於其上，用戶「取回」他們原始委託數量加上流動委託的累計獎勵。作為委託的原生代幣，用戶只需持有 bCRE 或於 bCRE 池中提供流動性即可對議案進行投票，同時降低機會成本。</p><p>透過流動性委託，用戶可以自由地透過提供流動性進行挖礦或使用 bCRE 於其他程序之上，並可能根據投資偏好令回報增加，同時保持對投票權的完全控制。此外，在 Crescent Network 用戶不受取回委託期限（在 Cosmos 生態系統中通常為 14~21 天）的強制約束，因為除了取回流動委託（14 天）外，用戶更可以選擇將 bCRE 換成各種資產。</p><p>在 Crescent 當用戶決定取回委託他們的 CRE 時，他們可以通過 Crescent DEX 輕鬆地將 bCRE 換成有 bCRE 配對流動池的資產，或者取回流動委託，透過銷毀 bCRE 以取回委託的 CRE 及累積的獎勵。 由至到終，用戶可以透過使用流動委託體驗增強流動性及優資的代幣投資選擇。</p><hr><h2>流動性挖礦獎勵於 4 月 18
