href="https://forum.cosmos.network/t/proposal-last-call-include-cosmwasm-in-rho-upgrade/6243" rel="noopener noreferrer" target="_blank">#69 治理投票討論案</a>， 在 Rho 升級中包含 生態系的最原始核心價值就是一條鏈只做一件事情，什麼都做不但會降低安全性，也會讓效率變差，就與目前很多 L1 無異。</p><p>比較具有爭議的是 Jae Kwon 說投 Yes 的人將領不到他的專案（GNO Land）的空投，有人認為這是買票分裂社群，但他覺得這反而是 Cosmos 生態系的主權自由：任何人都可以透過 fork 新鏈來取得用戶。</p><p>我的觀點是，Cosmos Hub 的確承受了巨大的「價值捕捉」壓力，這點應該是多數人都同意的；但同時我也很懷疑讓 Cosmos Hub 去學 Juno 等鏈，把 CosmWasm（智能合約）納入，能否吸引到新的專案跟資金。</p><p>畢竟做一個市場的 follower，並沒有讓 Cosmos Hub 增加什麼獨特的優勢，反而把鏈的效率以及開發速度拖慢、還可能把智能合約的安全隱憂帶到 Cosmos Hub 上；反之，如同 Jae Kwon 說的，等跨鏈帳號與共享安全上線之後，Cosmos Hub 仍然可以做到一樣的效果，並且更安全，更能捕捉價值。</p><p>從技術的角度來看，或許堅持 Cosmos Hub 做自己，不要仿效其他鏈「為了智能合約而智能合約」是比較好的；只是市場永遠沒有絕對，有時候看起來很不合理的策略也會成功，反之，有時候看似最明智的策略，卻可能因為其他因素而失敗，我們能做的就是投下自己的票，然後靜觀其變。</p><hr><h2>Cosmos Hub 的「價值捕捉」期待</h2><p>我在《<a href="https://leafwind.tw/2022/01/21/cosmos-value-capture/#%E5%85%B1%E4%BA%AB%E5%AE%89%E5%85%A8%E6%80%A7%EF%BC%9A_2022_%E5%B9%B4_%24ATOM_%E6%9C%80%E5%BC%B7%E7%9A%84%E5%83%B9%E5%80%BC%E6%8D%95%E7%8D%B2" rel="noopener noreferrer" target="_blank">跟著 Cosmos 離開地球表面：開放能否捕獲價值？</a>》一文中曾經提到，共享安全是 Cosmos Hub 2022 年最強的價值捕捉，然而幾個月過去，多數人也都明顯感覺到 Cosmos Hub 社群的情緒是越來越浮躁，因此不管是持份者還是開發團隊，似乎都越來越焦慮。</p><p>為什麼大家會感到焦慮？我認為有兩個原因（或者說其實是同一個）。</p><h3>Hub 的安全性與穩定性沒有反應在價格</h3><p>原本預期 Cosmos Hub 在 Hub 這個角色會比其他鏈做得更好，所以許多鏈會有龐大的需求來使用 Cosmos Hub，但幾個月過去，Cosmos Hub 的使用率並沒有飛躍的增加，反倒是其他 Hub 部屬了自己的 Hub 之後，並沒有使用 Cosmos Hub，而現在 IBC 流量最高的 Hub 其實是 Osmosis。</p><p>單就技術來看，將一條鏈定為成單純的 Hub 是合理的，因為如果每條鏈都要做 Hub，到時候將會有 N^2 個連線，效率會變得極差，一定需要有特化的鏈來複雜做「跨鏈轉機」。</p><p>然而年初說的「Cosmos 生態會出現兩百條鏈」目前看來遠遠未達標，因此目前 N 還不夠大，沒有急迫效率的需求，並且更重要的是，用戶在出現重大安全性/意外之前，並不會在意安全性跟可用性：斷鏈了也無所謂，IBC token 消失了可能也不知道這是因為 Hub 服務差。</p><p>就算用戶知道這些技術細節，也體會到 Hub 的重要性，我們也無法強迫 Osmosis、Juno 以及其他小鏈去用 Cosmos Hub，每個鏈的團隊很可能會因為自身利益而把自己的鏈當做 Hub 使用，所以短期內 Cosmos Hub ($ATOM) 的重要性無法被看見。</p><h3>跨鏈帳戶的好處也沒有反應在價格</h3><p>如果我的理解沒錯，Cosmos Hub 的跨鏈帳戶，可以作為所有鏈的智能合約共享計算資源：簡單來說，其他鏈不需要佈署智能合約，也可以把 Cosmos Hub 當做「共享計算機」。</p><p>這不只是會有 gas 過路費等等的好處，還有智能合約的安全性跟計算、交易量都會跑在 Cosmos Hub 上，所以技術上來說，讓很多小鏈不需要自己做智能合約，可說是一個技術上很重要的更新。</p><p>但問題是，即使能夠將 Cosmos Hub 作為「共享計算機」，也仍然要有人開發智能合約。試想如果我是合約開發者，要選在哪一條鏈佈署？有投資的開發者通常自己的團隊也有合約鏈（Juno, Scrt 等等），就不可能放棄自己的合約鏈，甚至還會把 Cosmos Hub 視為競爭對手：「反正我都要寫智能合約了，我寧願在自己的鏈部屬 CosmWasm，也不願意去使用共享計算機」。</p><p>這可能也是為什麼在跨鏈帳戶上線之後，$ATOM 價格沒有任何起色的原因。</p><p>還是回到價值捕捉這個命題，如果我組了一個團隊，弄一條鏈遠比佈署一個合約在 Cosmos Hub 還要簡單吸錢，即便我們團隊人力只足夠開發一個小到不能再小的 dapp，都可以因為公鏈迷思而吸到遠超過對應這個價值的資金；反之，如果團隊沒有自己的鏈，要吸引投資人可能就會非常困難。</p><hr><h2>被低估的技術開發能量</h2><p>很多人可能會想說，會有這些現象都是因為開源的關係：因為 Cosmos Hub 開發出來的模組總是開放給別的團隊免費抄去用，因此無法捕捉價值。</p><p>但我認為這不是最根本的原因，最主要的原因是在 web3 的開發能量重要性被市場嚴重低估。</p><p>回到 web2 的世界，微軟開源、Google 開源、Amazon 開源，每間大型軟體公司都開源，但光是複製這些開源軟體通常一毛錢也賺不到。</p><p>因為一間公司得有相對應的商業團隊去拓展業務、要有技術團隊在開源軟體之上去開發應用，以及之後的維護、更新、除錯；更重要的是甚至還常常要替這些開源軟體 debug，如果不是有經驗的工程師，光是複製開源能做到的事情非常有限。</p><p>所以很多公司索性直接使用這些大型軟體公司的 SaaS/PaaS/IaaS 服務，用一個簡單的比喻來說，開源軟體就像是那些公司的試用版軟體，作為技術實力的火力展示，吸引潛在顧客去買他們的付費服務。</p><h3>抄免費軟體就能賺大錢嗎？</h3><p>反過來在 web3，這些很簡單的道理被輕易地無視，隨處可見一人開發團隊、用了多數預算跟時間在打廣告，不斷地畫大餅吹氣球，好一點的等到募資完之後消聲匿跡、更常見的則是人去樓空，而散戶也樂此不疲地追逐一個又一個這樣的專案。</p><p>忽略了技術開發的難度，也忽略了維持一個產品穩定進步所需要的成本，總是天真地覺得「那個很簡單吧？抄來用就好了！」。</p><p>是不是很熟悉？沒錯，這就跟許多人抱怨的慣老闆一樣，作為旁觀者會認為很愚蠢，但當自己是拿著錢投資，而不是真正動手做的那個人，就很容易低估了技術專業所需要付出的成本，於是乎，我們都成為了那個「慣老闆」。</p><p>從技術角度看，Cosmos Hub 團隊的確有整個生態中數一數二的開發實力，因此若要評估作為 Hub、作為跨鏈帳號、共享安全的穩定性與安全性，我想很難有團隊能出其右。</p><p>然而市場的現實卻使得這些優點無法被突顯出來，多數資金追逐的是論述、是行銷，而不是實力與真正做出來的產品。在傳統金融界尚且如此，在幣圈這種注意資源極其短暫、資金大量追逐話題 FOMO 的生態中又更加極端。</p><p>因此，Cosmos Hub 在走一條非常困難的路，這一路上會遭受到很多質疑，也會有很多競爭對手用行銷與論述優勢，不斷地打擊更像是技術優先的 Cosmos Hub。甚至像 #69 一樣，團隊內部以及社群都焦慮地想要改變路線。</p><p>究竟是不在意技術、追逐論述的人會發財，還是支持做出好產品、有實力的團隊的人會獲得長期回報？這也是我參與 Cosmos 生態最想觀察到的議題。你的想法又是如何呢？歡迎讓我知道。</p><hr><h2>社群廣告</h2><h3>驗證人節點</h3><p>我在 <a href="https://stake.like.co/validators/cosmosvaloper1g2dpslkge0wmhgpdegeg0wq549syz8tjrpnum3" rel="noopener noreferrer"
