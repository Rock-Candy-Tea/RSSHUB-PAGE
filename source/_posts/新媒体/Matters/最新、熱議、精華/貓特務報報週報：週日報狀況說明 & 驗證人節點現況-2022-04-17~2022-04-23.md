<h2>日報 & 週報狀況說明</h2><blockquote><strong>4月19日日報狀況</strong></blockquote><p>4月19日之日報因為無文章資料所以於 4月20日中午隱藏了，後來發現因為由 Matters API抓取文章資料時，依序抓的到文章資料原本是由新到舊排序，但從 20 日零點日報執行抓文章時，取得的文章一開始就是 4月16日，所以抓取文章的動作便不會再繼續，目前這個狀況還是存在，從 20 日的日報開始我改用另外一隻 API 取文章資料，目前每日日報己正常運作。</p><blockquote><strong>週報狀況</strong></blockquote><p>不過目前日報使用的 API 只有最新文章，因此不符合週報使用，因為無法抓到整週的文章資料，而我每日抓文章時會存一份檔案備用，因此本週週報會以文章存檔來做統計，拍手數<strong>只結算發文當日</strong>，又因為 4月19日 沒有抓取到資料，因此本週的週報中<strong>沒有4月19日</strong>的文章，還請見諒 ...</p><p>至於拍手數只有發文當日，我本來想說會不會有失真問題？要不要本週週報乾脆先停掉，後來想想，原來統計方式，如果星期日發的文章可以有一整週的時間累計，如果星期六才發的文章在週報上就只有當天累計的拍手數，似乎不管拍手數累計期間怎麼切，也不可能每篇文章都有一樣的累計時間，那麼 ... 就這樣試試看吧！看看這樣統計出來的排行，大家感覺如何吧 ... 比較抱歉的就是 19 號沒抓到資料就真的沒有了 ... 哭 ...</p><p>另外就是因為使用存檔資料，也就是說每天日報抓了文章資料之後，資料就是靜態的不會再更新，因此如果不是在發文當日在文章中新增的標籤，都不會被計入喔！</p><blockquote><strong>週報-馬特市市集最新十篇文章狀況</strong></blockquote><p>目前  Matters API 以關鍵字搜尋搜某標籤下文章的這個功能，回傳的資訊全部是空的，簡單來說就是 ... 抓不到資料抓不到資料抓不到資料，因此這一部目前為人工檢查更新文章，還好只需檢查兩個標籤，不過功能復原前就不放拍手數了。</p><blockquote><strong>週日報 - 社區活動部份</strong></blockquote><p>社區活動部份因為現在 Matters 的社區活動發起模式改變，目前的機制我無法直接得知那些新活動開跑〔之前直接找發起活動文即可〕，所以目前也不知道有那些新活動在執行中，而且新機制似乎也不只限於徵文活動，我也不確定發起人是否有建立專屬標籤給活動使用，因此日報與週報的社區活動部份，目前想讓發起人來主動來告知，如果發起人有建立專屬標籤給活動使用，可以提供活動標籤的連結給我，並讓我知道活動結束日，如果到2022/05/15 都沒有活動主辦人來留言，那這一個區塊我就會從日報與週報上移除。</p><blockquote><strong>總結</strong></blockquote><p>希望這些 API 能儘快回復正常可使用狀況，如果沒回復 .... 那我也沒辦法 ....就 .... 秉著能用就發報的原則囉！！！</p><hr><p>Matters 標籤總數：105806<br
