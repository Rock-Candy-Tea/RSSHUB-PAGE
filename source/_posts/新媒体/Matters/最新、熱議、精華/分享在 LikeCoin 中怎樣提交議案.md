<p>BusinessAsMission 作為驗證人有差不多一個月時間了，常常也想著如何貢獻社群。由於驗證人均可以提交議案，所以我們今次借此機會學習提交議案，及把經驗與大家分享。</p><p><br></p><p>我不打算在此講解為何需要提交議案，為何要投票，這是 DAO ( decentralized autonomous/democratic organization, 分散式自治機構 ) 的特點，就是任何持有 LikeCoin 的人都是持份者，每個持份者都能參與讚賞公民共和國的治理。</p><p>詳情可以參考：（<a href="https://docs.like.co/v/zh/guides/governance/liquid-democracy" rel="noopener noreferrer" target="_blank">https://docs.like.co/v/zh/guides/governance/liquid-democracy</a>）</p><figure class="image"><img src="https://assets.matters.news/embed/30aa9d87-0872-46ba-8549-76d971cd9b9f.png" data-asset-id="30aa9d87-0872-46ba-8549-76d971cd9b9f" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>怎樣投票（<a href="https://docs.like.co/v/zh/guides/governance/direct-vote" rel="noopener noreferrer" target="_blank">https://docs.like.co/v/zh/guides/governance/direct-vote</a>)</p><hr><h2><strong>為何我會提交議案？</strong></h2><p>事原我在2022年1月的社群治理會議 （可參考：<a href="https://docs.like.co/v/zh/guides/governance/community-call%EF%BC%89%E4%B8%AD%E5%BE%97%E7%9F%A5%E9%81%93Proposal" rel="noopener noreferrer" target="_blank">https://docs.like.co/v/zh/guides/governance/community-call）</a> 中得知道Proposal 25 的議案需要修改部份內容，所以需要發出一個新的議案讓所有LikeCoin 的持份者表決接受修改的內容，通過後才能執行議案。</p><figure class="image"><img src="https://assets.matters.news/embed/2b2fd242-8f8f-4d12-88d6-74a15d5e8d9b.png" data-asset-id="2b2fd242-8f8f-4d12-88d6-74a15d5e8d9b" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>在會議中我們可以實時討論，像這次我想參與某些項目或提出問題，很多資深的驗證人或 LikeCoin 持份者也十分樂意幫助。</p><figure class="image"><img src="https://assets.matters.news/embed/96144e2c-9e1a-4c2e-a20b-b2971fc2d984.png" data-asset-id="96144e2c-9e1a-4c2e-a20b-b2971fc2d984" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>就這樣我就去參與第一次的議案提交</p><hr><p><br></p><h2>有什麼類型的 proposal？</h2><p>首先要知道有什麼proposal 種類, 所以先找一些參考，在discord 社群中人求問是最直接的方法。</p><p>我得到以下的資料：<a href="https://t.me/decentralizehk/19888" rel="noopener noreferrer" target="_blank">https://t.me/decentralizehk/19888</a></p><p>（LikeCoin/Cosmos）的 proposals 有幾種：</p><p>1. parameter change proposal: 修改某項參數，比如鑄幣頻率、投票期、抵押所需等，議案一旦通過，代碼自動在下一個區塊執行，不經人手。議案 #2、3、5、7、8、18 等很多都是這種</p><p>2. community-pool-spend proposal: 議案指定了花某一數額的公帑作某用途，發到指定錢包（有時是 multisig），也是議案通過，下一區塊自動執行。事實上，community pool 沒有人有 key，唯一的使用方法就是透過提交議案，說服社群通過。最近的議案 #25、26、27、28 都是這種。</p><p>3. upgrade proposal: 議案指定某特定時間更新到某特定版本，也是自動執行，各驗證人自行手動更新節點。因為議案通過才會更新，是社群共識，因此不會產生 hardfork。LikeCoin 暫時沒有先例</p><p>4. signalling proposal: 社群共識一個意願，比如參與 ATOM hackathon、成立 Creators Fund Committee 並指定成員等等，如議案 19、20。這是唯一一類牽涉人手跟進的議案，一般就會在議案裡面共識執行者。</p><p>再以一個議案 #28 為例，是「decentralized procurement」，提案動用 community pool 裡面的 4500 鎂等值 LIKE 去招聘一家推廣公司負責維護 <a href="https://t.me/likecoin" rel="noopener noreferrer" target="_blank">@likecoin</a> Twitter 帳號，而該公司的人正在 Discord 回應社群的提問，比如是否打算投放廣告（ <a href="https://discord.com/channels/763001015712350231/796036074396844052/923424673050542100" rel="noopener noreferrer" target="_blank">https://discord.com/channels/763001015712350231/796036074396844052/923424673050542100</a> ）。比如你是個 coder，也可以自己提案，讓社群撥款給你，寫一個 LikeCoin 的功能和應用。</p><p>整體來說，任何人都可以提案，任何人都可以是執行者，不需得到任何人批准，但需要爭取整個社群支持提案，這就是 LikeCoin 的無大台，跟開源軟件類似，但再加上整個治理的機制，所有議案寫在鏈上，能自動執行的都自動執行。<br class="smart"></p><p><br></p><p>之前的議案可以在這裡找到 （<a href="https://likecoin.bigdipper.live/proposals" rel="noopener noreferrer" target="_blank">https://likecoin.bigdipper.live/proposals</a>）<br class="smart"></p><figure class="image"><img src="https://assets.matters.news/embed/b218845e-6df4-4471-8a58-34e967043735.png" data-asset-id="b218845e-6df4-4471-8a58-34e967043735" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>由於今次是去修改proposal 25 的內容，而不是提出一個新的airdrop 議案，所以用signalling proposal 去提出修改的動議。</p><hr><p><br></p><h2>實際是要怎樣做？</h2><p>好了，現在好像知道要做什麼了，那就預備一下要怎樣做：(我先列出我所要做的，下面再詳留講解我的做法)</p><pre class="ql-syntax" spellcheck="false">1.   嘗試在 testnet 上提交一個議案
a.   我有一個 testing node 是用 testnet 的，之前做驗證人前用來做測試用的  （這是我的選擇，其實可以在任何能行 liked command 的地方也可執行，只要其它的 parameters 填對就是了）
b.   了解要行什麼 commands 來 raise proposal
c.   預備好議案的內容 （把詳細的議案內容放到 IPFS 或 ISCN, 再把link放入 JSON file ）
d.   去領取一點 ekil (testnet 用的 like)
e.   提交議案到 testnet 並檢查一下 （https://testnet.likecoin.bigdipper.live/proposals）
2.   到 Discord 裡的 proposal-discussions 與大家討論一下
3.   到 mainnet 去提交議案
</pre><hr><p><br></p><h2>詳細的解說:</h2><p>1.    先要在Testnet 裡 raise a proposal 好讓大家去討論</p><p>a.   我在我的 Testing node 上開始預備 proposal 內容 （怎樣setup a node: <a href="https://docs.like.co/validator/likecoin-chain-node/setup-a-node" rel="noopener noreferrer" target="_blank">https://docs.like.co/validator/likecoin-chain-node/setup-a-node</a>）</p><p>通常做 validator 前也會setup 一個 testing node 在 testnet 作測試。</p><p>注意事項：</p><p>要用 LIKECOIN_CHAIN_ID="likecoin-public-testnet-3"</p><p>Testnet 會用 nanoekil，而不是 nanolike</p><p><br></p><p>b.   我在discord 找到 @nnkken 曾經分享過怎 raise a text proposal.</p><figure class="image"><img src="https://assets.matters.news/embed/cf951671-fe9c-4341-b9a9-cc3f62c75eba.jpeg" data-asset-id="cf951671-fe9c-4341-b9a9-cc3f62c75eba" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>但由於我要先 submit 在 testnet 作測試, 所以我更改了一部份，更改為</p><pre class="ql-syntax" spellcheck="false">docker-compose run --rm liked-command \
tx gov submit-proposal \
     --proposal=/host/text-proposal.json \
     --from validator \
     --node
https://node.testnet.like.co:443/rpc/ \
     --chain-id likecoin-public-testnet-3
</pre><p><br></p><p>解說：</p><p>--proposal 的 content 會用 json file 注入</p><p>--以 validator 身份來提議案</p><p>--node: 就是用那個 rpc server 的 endpoint (要有 port number), 所以現在要用 testnet 的rpc endpoint (請參考：<a href="https://github.com/likecoin/testnets/tree/master/likecoin-public-testnet-3" rel="noopener noreferrer" target="_blank">https://github.com/likecoin/testnets/tree/master/likecoin-public-testnet-3</a> )</p><p>--chain-id, 當然也是用 testnet 的 chain-id (請參考：<a href="https://github.com/likecoin/testnets/tree/master/likecoin-public-testnet-3" rel="noopener noreferrer" target="_blank">https://github.com/likecoin/testnets/tree/master/likecoin-public-testnet-3</a>)</p><figure class="image"><img src="https://assets.matters.news/embed/bd2355c0-f624-468f-b39b-725be93b711d.png" data-asset-id="bd2355c0-f624-468f-b39b-725be93b711d" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>注意，如要raise 另一種類的 proposal, 如 community-pool-spend, param-change, software-upgrade, 就要用該 command，如：</p><pre class="ql-syntax" spellcheck="false">docker-compose run --rm liked-command \
tx gov submit-proposal param-change some-proposal.json \
     --from private-key \
     --node
https://node.testnet.like.co:443/rpc/ \
     --chain-id likecoin-public-testnet-3
</pre><p><br></p><p>c. 先把詳細的議案內容放到 IPFS 或 ISCN。我支持likecoin, 當然是用 ISCN 註冊。</p><p>我的做法是把議案先轉為 PDF, 再到 <a href="https://app.like.co/" rel="noopener noreferrer" target="_blank">https://app.like.co/</a> 去把 PDF 註冊. 詳細教程請看(<a href="https://docs.like.co/general-guides/decentralized-publishing/app.like.co)" rel="noopener noreferrer" target="_blank">https://docs.like.co/general-guides/decentralized-publishing/app.like.co)</a></p><figure class="image"><img src="https://assets.matters.news/embed/7d8ab070-c5b6-4af7-9b0c-cbc345164d89.png" data-asset-id="7d8ab070-c5b6-4af7-9b0c-cbc345164d89" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>完成後我會得到下面的資料，如ISCN ID, Content Fingerprints, ifps link... 等</p><figure class="image"><img src="https://assets.matters.news/embed/9bc9c2ee-761d-4a9d-93c6-b613195e251e.png" data-asset-id="9bc9c2ee-761d-4a9d-93c6-b613195e251e" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>議案內容要放在 text-proposal.json 裡，例子如下。而text-proposal.json 要放在 docker-compose.yaml 同一層</p><p><br></p><p>如我想要把下面的文字放入 JSON 內的 dsecription</p><pre class="ql-syntax" spellcheck="false">This is a Test!
