referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>對於悠遊於網際網路的我們來說，鮮少人從未接觸過 VPN (Virtual Private Network) ，即使不是經常性使用，也有可能會因為玩線上遊戲的跨服需求、看串流平台的跨區需求而使用 VPN 來進行作業。</p><p>當然了，這項行之有年的技術面臨種種爭議與挑戰，例如其中最常見的即是 VPN 的 Server 端擷取用戶在透過 Tunnel 連線時所傳送、填寫的資料 (Server Log User Data) 。</p><p>在資訊安全意識抬頭、因應遠端工作而需更加確保隱密性的情況下，勢必將興起一種嶄新的技術 —</p><blockquote>一個承襲 VPN 原有的優勢，並加以某方面的新技能改良，一個更符合當代人們在使用網際網路的便利性以及安全性等等必要條件考量上之技術。</blockquote><p><br></p><hr><p><br></p><h2>什麼是 VPN ？</h2><p>在這篇文章中，主要仍以介紹新技術 DPN 為主，因此這邊僅會簡短呈現 VPN 的用途，關於技術導向的詳細資訊可以參考各大論壇上專家之筆。</p><p>VPN (Virtual Private Network) ，臺灣多譯為虛擬私人網路、虛擬專用網路，能夠允許使用者透過各種協定加密後的的網路連線在公共網路上進行專屬連線，這種將私人網路延伸到公共網路的技術配合各種加密協定後，能夠讓使用者較安全地在網際間瀏覽、存儲以及傳輸資料。</p><p>這項技術讓網路使用者即使在遠端，也能夠透過連接專屬供應商提供的 VPN 將公共網路轉化為專用網路，提升原有網路的安全性、管理效度。這邊提及的 “各種協定” 最為常見的是各種基於 Tunnel Protocol （通道、隧道協定）上衍生出的協定，佐以加密（VPN 也能夠透過加密程度進行等級分類）或是各種 “擬態” 方法來實現更高等級的匿名性（常見的有透過 Server 顯示與使用者原生存取點不同的網路協定地址，俗稱切換 IP 位置），讓僅有通過認證的使用者能夠存取、使用。</p><p>VPN 看似相當實用且強大，也能提供諸如金融加密級別的服務，但顯然人類的發展並不會止步於此。</p><p><br></p><h2>綜觀虛擬私人網路的應用及技術佈局，我們可以發現下列幾項缺陷：</h2><p><br></p><ul><li><strong>Server 端對於資訊的存取、販售</strong></li></ul><p>許多付費型的 VPN 公司（這裡不提及免費 VPN 的原因是，通常使用免費版 VPN 僅會拿來做跨區遊戲、串流平台影片欣賞等用途，較不會進行較敏感的資訊傳送）所提供的服務皆有強調加密以及資訊安全性，但 Server 本身很有可能就是讓人防範不及的敵人，例如：不肖廠商直接存取用戶資訊加以販賣或者是中國境內的 VPN 服務商不得不遵循政府要求，將用戶之資訊公開給政府機關。</p><p>實例： Facebook Research VPN 事件</p><p><br></p><ul><li><strong>不見得那麼安全</strong></li></ul><p>很多人相信付費 VPN 遷移 IP 的匿名性，但若沒有特殊設置，其實回朔地址是許多資安專家能夠做得到的事情，而且，目前 VPN 技術提供的 Server 為中心化管理，攔截並解密資訊的駭入技術也在加密日漸高深的情況下緊追在後。</p><p><br></p><ul><li><strong>在運作方式上較不智慧</strong></li></ul><p>在現有 VPN 技術（大多數）上，一但連上某地區提供的 Server ，並改定 IP 後，即可使用該地區的特定服務，同時，裝置也無法（大多數）同時瀏覽存取先前尚未使用 VPN 時的地區內容，例如翻牆入某國境內，即會被阻擋原本國外的網站存取。</p><p><br></p><h2>所以？一個更安全且聰明的 VPN ？</h2><p>筆者今年（2022）初發現到一家位於美國矽谷的科技公司，試圖以區塊鏈的特性、技術來改善原先存在已久的 VPN 之問題，他們提出的解方名為：</p><p><strong>DPN (Decentralization Virtual Private Network) ，也就是去中心化虛擬私人網路。</strong></p><p><br></p><p>這項技術相當有趣，筆者這邊簡略說明：</p><pre class="ql-syntax" spellcheck="false">有一個實體裝置（Hardware），接上你自己的 Router 或甚至直接連上你的電腦，即可以將你目前所使用的裝置網路變成一個節點（Node），而這個節點同時是別的 Deeper Network 硬體使用者的 Server 。
</pre><p><br></p><p>也就是說，透過使用 Deeper Network 所推出的實體裝置，能夠在全球各地創造出眾多節點，這些節點皆可以被另一個節點（使用者）當作 Tunnel 連結，如此一來，便實現一個去中心化（各個 Nodes）管理的虛擬私人網路。</p><p>Deeper Network 在實體裝置所帶來的優勢上下了不少工夫，例如最新推出的 Pico 標榜可以擋下廣告、企業級的資安防護、超級簡易的連結即可使用，甚至還可以利用其來進行挖礦，（例如可透過分享頻寬）來賺取專屬加密貨幣 $DPR 。</p><figure class="image"><img src="https://assets.matters.news/embed/5b1bd111-e3ff-4be3-acd5-0078f150c983.png" 的缺陷中，運作較不智慧這點也獲得了解決， Deeper 提出：單一節點可選擇多 Tunnel 運作。</p><figure class="image"><img src="https://assets.matters.news/embed/de7f87bd-91a2-4e28-a82d-25d45d292960.webp"
