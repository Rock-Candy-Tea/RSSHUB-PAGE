referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><hr><h2><strong>第四步 Configure capture</strong></h2><p>這裡的動作是要設定透過 fxhash 的伺服器來自動抓 preview 預覽圖檔</p><figure class="image"><img src="https://assets.matters.news/embed/50831593-5f73-4c8f-8cd4-05955dd18c06.png" data-asset-id="50831593-5f73-4c8f-8cd4-05955dd18c06" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p><strong>Trigger</strong> 是告訴伺服器端 (server side)，要在什麼時間點<strong>觸發螢幕截圖</strong>作為這個作品的預覽圖。因為伺服器端會實際執行一次你上傳的這包網頁跟程式，但伺服器端不知道每個作品會在什麼時間點把作品的圖像或程式跑完，或有些作品永遠都在動，所以需要創作者告訴伺服器在進入網頁經過一定的時間後進行截圖 (<strong>Fixed delay</strong>) ，或是主動呼叫一個 function fxpreview() 來觸發截圖的動作 (<strong>Programatic trigger using fxpreview() </strong>)。</p><figure class="image"><img src="https://assets.matters.news/embed/5bfa217a-b02f-44ec-9bf7-ed812c7e2759.png" data-asset-id="5bfa217a-b02f-44ec-9bf7-ed812c7e2759" referrerpolicy="no-referrer"><figcaption><span>從兩種 Trigger 的方式裡選擇</span></figcaption></figure><p><strong>Target</strong> 是設定要抓截圖的對象為何。有分 <strong>From <canvas> </strong>跟 <strong>Viewport capture</strong> 兩種。</p><figure class="image"><img src="https://assets.matters.news/embed/3096db54-977b-42fb-bf75-7f4c9a35257e.png" data-asset-id="3096db54-977b-42fb-bf75-7f4c9a35257e" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p><strong>From <Canvas></strong> 是透過指定 CSS 選擇器來抓取網頁中特定的 canvas 物件。我自己是用 p5.js 做的所以我選的是這個。那要怎麼知道 canvas 的 CSS 選擇器描述，以我自己為例，打開作品的 HTML 檔案執行，然後用 Chrome 瀏覽器開發工具來看物件結構，會看到有產生一個 canvas 標籤， id = "defultCanvas0"，所以我填入那個 Canvas CSS selector 的值就是 <strong>canvas#defaultCanvas0</strong>。</p><figure class="image"><img src="https://assets.matters.news/embed/ac92318f-35d2-425b-8805-10ca82aa77b5.png" data-asset-id="ac92318f-35d2-425b-8805-10ca82aa77b5" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>如果你的作品會產生很多 canvas 標籤或是其它非 canvas 的情況，那就需要用 <strong>Viewport capture</strong> 指定一個瀏覽器視窗大小，告訴伺服器端用這個尺寸的視窗來直接抓整個網頁的截圖。</p><figure class="image"><img src="https://assets.matters.news/embed/8bbe40a3-0e34-43c3-815f-430578e36b9c.png" data-asset-id="8bbe40a3-0e34-43c3-815f-430578e36b9c" referrerpolicy="no-referrer"><figcaption><span>指定視窗大小來截圖</span></figcaption></figure><p>接下來的 <strong>Extra settings</strong> 是決定是否用 GPU 來產生截圖。一般情況下是不需要打勾的，除非你發現你的截圖結果跟你預期的不一樣，那才需要勾這個選項，勾了這個選項的話截圖產生時間會滿很多，會花幾分鐘的時間。</p><figure class="image"><img src="https://assets.matters.news/embed/e6c73601-cbb2-4ddf-9ea2-bb322e3f6428.png" data-asset-id="e6c73601-cbb2-4ddf-9ea2-bb322e3f6428" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>到這邊都設定完了就按下<strong> test capture </strong>抓一次截圖看看。稍後一下，你應該會在右邊看到伺服器抓截圖下來的結果，看看是不是跟你預期的一樣，有需要就再調整參數然後再 test capture 一次。沒問題的話就按 <strong>next step</strong> 吧。</p><hr><h2><strong>第五步</strong> <strong>Verification</strong></h2><p>這邊要確認你的線上的網頁跟預覽截圖是否一樣，可以看到我的截圖裡，山的顏色有一點點不同，我自己是後來透過 GPU 截圖才能得到一致的截圖。</p><figure class="image"><img src="https://assets.matters.news/embed/123fe3e1-0c02-4d34-9a16-cc0c07167a95.png" data-asset-id="123fe3e1-0c02-4d34-9a16-cc0c07167a95" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>除了這兩個畫面要一樣之外，也要確認 open in new tab 之後全螢幕是否也是一樣的結果。</p><p>我自己曾經遇到的問題是，我用了一些 random 來產生物件位置，同時我的 canvas size 也會根據視窗的可用寬度自動調整，所以有時候在比較小的螢幕看時， canvas 尺寸是 600 x 800，當螢幕比較大時，可能變成 900 x 1200。再加上我 random 產生物件的單位是 pixel 所以只要螢幕尺寸不同，random 出來的結果就會不太一樣。</p><p>解決方法是把各種會計算到尺寸的地方用 0 到 1 的比例來表示，比如 原本可能是 10 pixel 寬度的物件，我可能就寫成 0.0166，這樣的話當螢幕寬度是 600 時，寬度是 10，螢幕寬度是 900 時寬度就會是 15，變成一個相對寬度的尺寸。這樣就能確保 random 出來的結果在不同螢幕與縮放下，會得到相同的結果。如果有使用 random / noise 的需要多注意一下。</p><p>確認兩者相同的話就可以去下一步。</p><hr><h2><strong>第六步 Distribution </strong></h2><p>要決定這個 GT 怎麼銷售。這裡的某些銷售設定可以在之後再次修改。逐一說明如下：</p><ul><li><strong>Number of editions </strong>決定作品會被銷售的版數，版數選了之後不能更改</li><li><strong>Pricing methods</strong> 銷售方式，分為固定價格 Fixed class="image"><img src="https://assets.matters.news/embed/376e77c2-03f8-4fd8-aca5-a9001a124d51.png" data-asset-id="376e77c2-03f8-4fd8-aca5-a9001a124d51" splits</strong> 銷售收入是否與其它人分，或是捐出一部份比例。點 donations 的話就有一些預設的機構可以選。</li></ul><figure class="image"><img src="https://assets.matters.news/embed/7cde9cab-b6c2-4afd-8253-0bfbdeff762c.png" Splits</strong> 如果你的作品有人買了之後 list 在 marketpalce 銷售賣出，這時候邁出價格會有一部份以佣金的方式回饋給作者，這裡就是設定回饋比例，以及回饋給除了作者之外的其它人。這裡如果選 royalties to the minter 就會分一定比例給第一個向你購買的那位藏家，這個作法也會間接提供動機購買後再次做銷售（也就是二級市場銷售）。</li></ul><figure class="image"><img src="https://assets.matters.news/embed/75b3c54b-b3b3-452e-bc6d-415e0b130111.png" data-asset-id="75b3c54b-b3b3-452e-bc6d-415e0b130111" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><ul><li><strong>Reserves</strong> 可以設定作者保留的件數，也就是白名單，可以從自己以前的作品擁有者匯入，也可以手動輸入錢包地址。這裡的功能我還沒用過，等我之後有機會使用的話再來介紹。XD</li></ul><figure class="image"><img src="https://assets.matters.news/embed/03bb5b73-ba20-4b04-8546-a7d81188047a.png" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><ul><li><strong>Enabled</strong> 這個沒有打勾的話，就表示作品上架後你需要手動進來打勾才會開賣。如果打勾的話就會照你設定的時間自動開放 mint 作品。</li></ul><figure class="image"><img src="https://assets.matters.news/embed/91d81656-8ccd-4786-82c9-61219979e141.png" data-asset-id="91d81656-8ccd-4786-82c9-61219979e141"
