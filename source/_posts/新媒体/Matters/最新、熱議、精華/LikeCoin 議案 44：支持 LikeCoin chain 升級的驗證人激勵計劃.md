referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>然後，在 Discord 開了這個 <a href="https://discord.com/channels/763001015712350231/958252126449053707" rel="noopener noreferrer" target="_blank">Incentive program to reward validators who upgrade to 1.2 in likecoin-public-testnet-4</a> 的討論串，寫了議案最初的版本去討論，接下來驗證人 Chun <a href="https://www.facebook.com/a.tech.guy/posts/7258811457522571" rel="noopener noreferrer" target="_blank">將 likecoin-public-testnet-4 整個 KO</a>，之後變成了 likecoin-public-testnet-5，再得到 William 及 rickmak 幫忙修正，並由俄國帥哥驗證人 Bulgakov 幫忙提案，於是出現了這個議案。</p><h2>議案內容</h2><p>這是一個社群池支出議案，涉及一個獎勵計劃，用於獎勵在測試網升級到 LikeCoin chain 2.0.0 的驗證人，以確保升級穩健及順利。</p><p>通過對詳案投贊成票，你同意將 100,000,000 LIKE 存入社群委託委員會的多重簽署錢包 cosmos1cd3rvuavswgf8p3t30vafpypg9n76evpuvmnxa，用於獎勵在 likecoin-public-testnet-5 上升級到 LikeCoin chain 2.0.0 版的驗證人，以提共為期 6 個月的委託。</p><h2>議案細節</h2><p>1. 將社群池中的 100,000,000 LIKE 存入社群委託委員會的多重簽署錢包 cosmos1cd3rvuavswgf8p3t30vafpypg9n76evpuvmnxa。</p><p>2. 是次獎勵計劃是「一次性」的，讓於 likecoin-public-testnet-5 上升級到 LikeCoin chain 2.0.0 版的驗證人之間共享 100,000,000 LIKE 委託資金，並且不會影響常規社群委託的資金及程序。</p><p>3. 於 4 月 13 日 23:59 GMT+8 / 15:59 UTC 之前在 likecoin-public-testnet-5 上升級到 LikeCoin chain 2.0.0 版本的驗證人（如果因為技術問題必須重新啟動測試網升級，日期將於更新區塊後延期 48 小時，具體日期將在 LikeCoin Discord #validator 頻道公佈）將根據以下條件獲得資金委託：</p><blockquote>A. 此特殊委託僅適用於在提出此議案時於主網上的 Active Set 活躍驗證人集的驗證者。快照將在議案提出時進行。</blockquote><blockquote>B. 測試網驗證人必須在測試網網站欄位公布他們的主網驗證人地址。</blockquote><blockquote>C. 以下驗證人將不能獲得這次特殊的一次性委託：<br class="smart"><strong>a. BigDipper 現時顯示為前 34％ 的投票權。<br class="smart">b. 佣金率低於5%。</strong></blockquote><blockquote>D. 基於以下條件，委託將被撤回：<br class="smart">a. 如果驗證者節點被 Jailed 監禁並且超過一週沒有回應社群成員的呼叫。<br class="smart">b. 正式升級發生後 72 小時內，驗證人未升級到 Cosmos-SDK 0.44。</blockquote><p>4. 是次 100,000,000 LIKE 的委託將於2022年10月31日撤回，亦也是本屆社群委託委員會任期結束的日子。</p><h2>議案解釋</h2><p>Snapshot 經已拍好，顯示如下：</p><figure class="image"><img src="https://assets.matters.news/embed/7f420d2a-61cb-469f-93c5-d3a878ca8b86.png" data-asset-id="7f420d2a-61cb-469f-93c5-d3a878ca8b86" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>至於 34% 是指這一堆被 BigDipper 顯示為灰色，指他們有機會減少網絡安全、令網絡停止的驗證人</p><figure class="image"><img src="https://assets.matters.news/embed/ce7bebd5-f7cb-49d7-bd9f-f5e0fee7020a.png" data-asset-id="ce7bebd5-f7cb-49d7-bd9f-f5e0fee7020a" referrerpolicy="no-referrer"><figcaption><span></span></figcaption></figure><p>至於佣金率低於 5% 的問題是甚麼，我以住的文章都簡介過，但既然有人翻了 kin 在 2 月 25 日就 <a href="https://matters.news/@daisy/desmos-network-%E8%AD%B0%E6%A1%88-14-%E5%B0%87%E6%9C%80%E4%BD%8E%E4%BD%A3%E9%87%91%E8%A8%AD%E7%BD%AE%E7%82%BA-5-bafyreihnbsntjdflcagcag6qtym3eyblraox5scurkgdfrahihehsm4lwu" rel="noopener noreferrer" target="_blank">Desmos 議案 14</a> 引用 <a href="https://twitter.com/ricmontagnin/status/1496834219762458631" rel="noopener noreferrer" target="_blank">Riccardo Montagnin @ricmontagnin 的 推文</a> 出來 <a href="https://twitter.com/ckxpress/status/1497083110277595138" rel="noopener noreferrer" target="_blank">延伸推文</a>，我也順道翻譯一下：</p><p>0/ 分享 LikeCoin 社群去年關於最低佣金進行了激烈的辯論。</p><p>1/ 現時，Likecoin chain 前 50 名驗證人，也是活躍驗證人總數當中有 7 名為 0%。 其中，第 1 名的投票權為 13.51%，超過第 2-4 名 的總和。</p><p>2/ 當社群進行討論時，排名最前的 0% 驗證人早已存在，但沒有如此戲劇性的領導地位。</p><p>3/ 社群，至少是那些願意發聲的人，要求最低佣金為 5%。 沒有科學理據說明為甚麼不是 4%、6% 或其他什麼，而更像是一個「神奇數字」。 這也是從其他鏈中的參考，這些鏈有 5% 的最小佣金值而且運行良好。</p><p>4/ 並未提出議案（去界定最低佣金），原因與 @ricmontagnin 提到代碼並未能配合相同。 雖然通過文本議案確實能提供了一個信號（給大家知道有最低佣金這回事），但我們認為當代碼到位時最低佣金會更有意義，可能是在 Cosmos SDK 升級之後。</p><p>5/ 但在社群等待代碼到位期間，0% 佣金的頂級驗證人大大擴大了領先優勢。 除了佣金本身，錢包 UI 的驗證人預設排序 ( default sorting )、新用戶教育等都是重要的部份。（<strong>所以大家有沒有發現 </strong><a href="https://dao.like.co/validators" rel="noopener noreferrer" target="_blank"><strong>https://dao.like.co/validators</strong></a><strong> 由原先的 Voting Power 投票權最多排先變成了每次進入是 random 隨機出現？</strong>）</p><p>6/ 一般來說，我支持最低佣金（不一定是 5%），理由與最低工資類似。 防止勞動者因競爭而設定過低的報酬。 這是一個勞動者做事而不會賠錢的底線，因為競爭可以被認為是不健康的。</p><p>7/ 出於類似的原因，@decentralizehk 為 @DesmosNetwork 議案 14 投贊成票。我們同意可以進行更深入的討論，但我們不是沒有考慮過就投票。我們把它理解為一個指引式議案，運營團隊有空間在合理的時間裡實施。</p><p>8/ 順便說一句，LikeCoin 正在開發正在為 Cosmos SDK 開發一個 dao 前端（現在正在做<a href="https://www.figma.com/file/SMaKDFUALEopfVeDHcBemr/DAO-likecoin?node-id=390%3A5014" rel="noopener noreferrer" target="_blank">改版 mock-up</a> 中，我提了一些 <a href="https://discord.com/channels/763001015712350231/960488209601667092/961259574344708186" rel="noopener noreferrer" target="_blank">意見</a>，有與趣的歡迎一起討論）。現時的治理缺少很多改進，例如議案的提出、解釋及討論。提案者可能想解釋，委託人可能想討論，但沒有 UI。也很難怪他們。</p><h2>個人意見</h2><ul><li>首先，這個議案是我弄的，規則是我設定（想認定我是邪惡軸心的人是否很開心？）</li><li>然而這個特殊委託是給驗證人的，而且也不存在「審批」，社群委託委員會只是幫忙去處理這畢錢的委託，只要是活躍證人肯預先進測試網就有會有這畢委託了，規則是顯而易見的，也不存在私相受授，在指定日期前技術上不能達標就沒有，就是這麼簡單。</li><li>現時社群池有 224,346,359.37
