<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我们在工作中，经常遇到问题就直接想怎么办，很容易陷入“线性思维”的泥潭，难以自拔。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>我们很少去想要解决的问题是什么，为什么要解决这个问题？</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>上面的情况，我想我们很多人都遇到过，概括起来说就是我们没有建立自己的思维方式，或<a href="https://www.niaogebiji.com/tag/%E6%80%9D%E7%BB%B4%E4%B9%A0%E6%83%AF" target="_blank">思维习惯</a>。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>为什么要建立思维方式？最重要的原因就是它可以提高我们的思维效率，提高思维的针对性和有效性。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>通过不断强化和练习，我们会掌握更多的思维模型，思维的水平和层次逐渐提高，最终促进我们认知水平的升级，让我们生活的幸福感和获得感更强。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>前段时间“高级<a href="https://www.niaogebiji.com/tag/%E4%BA%A7%E5%93%81%E7%BB%8F%E7%90%86" target="_blank">产品经理</a>与普通产品经理的差异”这个话题讨论得火热，联系到这段时间的工作感悟，通过与身边高P产品经理及产品总监的交流，总结出了“高效能产品经理的4个思维模型”。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>相信通过对这4个思维模型的介绍，能够帮助大家了解高级产品经理的思维方式，提升自身思维的水平与层次。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>尽早升级认知水平，获得工作的获得感与满足感。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-mpa-template="t"> </p>

<p> </p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="Rwc0CW5O">管理学大师彼得·格鲁克说过的：「If you can’t measure it，you can’t improve it.」比起感性的推测，数据无疑是很靠谱的东西。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="Rwc0CW5O"><span><strong>而对于产品经理来说，在现有的数据里面发现机会，去找到能够让新用户用的更好停的更久，推荐更多人的方法，实现产品价值，就是数据最大的作用。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="B60UKA7S">如果产品经理对数据抽取漫无目的，花了时间，但并不能产生指导性的产品设计思路，那就没有搞清楚产品的价值功能，目标涣散，导致把控不了价值数据。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="2zdImeTQ">因此精明的产品经理不会漫无目的追踪数据，他们一般基于精确的理解产品定位后，会很好的规划自己要分析的数据。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="2zdImeTQ"><span>以数字为依据的、用严谨逻辑来推演的思维方式。</span><span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="2zdImeTQ"><span></span><span>现代社会所有的真相大部分最终会汇总成</span><span>数据，并以数据形式呈现和存储。</span><span>然而，数据本身并不能表达任</span><span>何含义，只有数据与逻辑结合在一起时，我们才能发现和表达真知灼见。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="IxL7azKt"><strong><span>数据不可迷信，只是更加客观，帮助我们去更清楚的认识。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="IxL7azKt">因为产品经理不是靠创意和情感开发功能，这时数据就是一种度量的手段，能衡量很多问题。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-mpa-template="t"> </p>

</section>
</section>
</section>
<p> </p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">“表象”是每天都能见到的看起来纷繁无序的事件和各种信息。“洞见”是能链接所有相关表象的筋络，是表象背后的根本原因。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">作为产品经理，我们的职责就是在纷繁的表象中寻找并提炼洞见；</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">从用户数据、产品数据、市场数据中，我们每天可以接收很多信息：DAU下降、ROI降低、推广成本提升等等。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>而如何从表象信息中逐步挖掘，通过深入思考抽丝剥茧，找到导致现象发生的真实原因，就显得尤为重要。在此，我们基于数据，提出以下分析步骤：</strong></span></p>
<ul>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">寻找数字中的规律和趋势</p>
</li>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">寻找极端的数字及其含义</p>
</li>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">对比参照数据并分析差异</p>
</li>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">寻求其他相关信息</p>
</li>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">推演并提炼洞见</p>
</li>
</ul>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">同样的，在沟通层面上我们也需要掌握“洞见”而非“表象”。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">举个例子，与领导汇报时，应先说洞见，再叙述表象；先说结论，再说过程。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>因为领导更关注你的产品决策，而非你的主观表述。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">洞见是行为导向。一旦找到洞见，解决方案基本就近在眼前了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-mpa-template="t"> </p>

</section>
</section>
</section>
<p> </p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">该原则是高级产品经理最核心的思维模式，也是基本的能力要求。、</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">该原则要求每一次切分后：子分类相互独立无重叠，子分类加起来穷尽全部的可能。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>通常运用MECE都是从顶层的问题开始，逐层向下进行分解，实际运用中你只需要不停问自己两个问题：</strong></span></p>
<ul>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">我是不是把所有的可能因素都考虑到了，有没有遗漏的？如果有，再去找。</p>
</li>
<li>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这些因素之间有没有互相重叠的部分？如果有，进行去重。</p>
</li>
</ul>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">在产品经理的实战中，会经常运用到MECE原则，典型如网站或者APP的注册登录流程梳理，我们就需要把所有可能的因素都考虑进去，比如密码错了应该出现什么提示，账号不存在又该怎么办等等。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">在梳理产品流程的过程中，需要将产品的初始状态、常态、边界状态、错误状态都给考虑清楚。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">MECE还可以用来进行目标分解，原理一样，把一件事情按照某一维度尽可能去分解，直到在现有掌握的信息下无法分解为止，然后再去筛选路径，制定策略，完成目标。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">拿「增加外部引流」这个大目标来看，进行多级拆分后，如图：</p>

</section>
</section>
</section>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">在我们开会的时候，乍一听身边其他产品经理的工作情况报告，感觉几乎每个论证都显得有道理。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>但是，表面的理由并不是唯一支撑结论的想法，因为他们并没有考虑“隐含假设”。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">往往有些内在的、隐藏的看法对于从众说纷纭的论述中厘清真相也起着重要的作用。这些论证中讲述者认为理所当然、但没有明说出来的特定想法，就是假设。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/207862305761e18352dce007.25738286.jpeg" referrerpolicy="no-referrer"></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">要找出一段完整论述的所有假设是困难而且没有必要的。一个论证的主要结构由理由和结论两部分组成。因此，我们需要关注的只是影响到论证结构质量的那些假设。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>我们要学会在他人的论述过程中找到“隐含假设”。同时，也要学会在产品设计过程中“大胆假设”。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但假设，是有依据的猜测。在决策过程中根据已有的有限数据先提出问题动因或解法的假想，然后以该假想为标靶收集足够多的数据证明真伪。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">如何构建问题假设？比如设计一款产品前，我们要先假设：哪类用户确定谁是你的目标人群？他们的需求，目标和渴望是什么?</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">拿即刻app举例，作为一个用户，我希望能和朋友分享一些短小的想法但是又不是很占用我写作的时间。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-pid="UxaGcpXZ">如果有时候你发现你可以明确你的目标群体和他们所面临的问题，但是你提出的解决方案可能不是特别合适。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">有些假设不能证实，此时就应该立即调整重新假设再次收集数据进行验证，直到假想被支持成为洞见。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>“大胆假设，仔细求证。”胆大心细，也是高级产品经理必备的素质。</span></strong></p>
