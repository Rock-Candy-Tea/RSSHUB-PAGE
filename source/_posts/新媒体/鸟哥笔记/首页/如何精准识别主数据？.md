<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>彭友们好，我是老彭啊。最近有彭友跟我欲言又止，吞吞吐吐不知道要干啥。我明白，他是有问题了。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>绕了半天，他才问：怎么才能精准识别主数据呢？</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我一看这个问法，肯定是遇到难题了，可不能用主数据的定义、什么N大特性糊弄过去了。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/57159521361109e7cb8df27.25660889.jpeg" referrerpolicy="no-referrer"></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><span><strong><span>主数据</span></strong></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>DMBOK里的定义是这样婶儿的：</span></p>
<section>
<p>主数据是<span><strong>有关业务实体</strong></span>（如雇员、客户、产品、金融结构、资产和 位置等）的数据，这些实体为<span><strong>业务交易和分析</strong></span>提供了语境信息。实体是客观世界的对象（人、组织、地方或事物等）。实体被实体、实例以数据/记录的方式表示。</p>
</section>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>发现没有？主数据和实体有关系哟~~~所以主数据其实跟模型关系是很紧密的。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>其实理解主数据很简单，我之前就写过一篇文章专门解释这个东东：【戳这里查看：主数据又是啥东东？应该怎么建？】</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>简单来说，就是核心业务中，非数值的关键数据。这个理解不精准，但是容易理解。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>不过这哥们的问题，显然不是这篇文章能解决的，因为他肯定是在进行主数据识别的时候遇到模棱两可的内容，无法进行区分了。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/139373035761109e7ce16636.86128267.jpeg" referrerpolicy="no-referrer"></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><span><strong><strong><span>主数据识别</span></strong></strong></span><span></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>想要确认两个内容是否是主数据，就得从主数据的定义入手，从主数据特征入手。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>当然，也有一些偏门的手法可以辅助识别。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>石秀峰石老师在他的书《一本书讲透数据治理》里写了两个方法：</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>1、主数据特征识别法</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>2、业务影响和共享程度分析矩阵法</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>主数据</span><span><strong>特征识别法</strong></span><span>顾名思义，就是对着主数据的特征比划一下就行了：<br></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><img src="https://ngbjimg.xy599.com/154972987462c35846b83f67.20045160.png" referrerpolicy="no-referrer"></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我们看看这些数据是否有以上特征，如果都有，那么是。如果缺一两个，可以考虑考虑。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>一般可列为6个问题：</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>1、<strong>是否体现业务核心价值</strong>？这一点非常非常重要！（客户信息肯定是，但是配送地址所在省份就不是核心价值数据了）</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>2、<strong>是否是独立的实体</strong>？（商品是独立不可拆分的实体，但是临期商品则不是）</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>3、<strong>是否相对稳定</strong>？（之所以加上相对，就是某些主数据是会变的，比如客户信息）</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>4、<strong>是否会在其他系统共享</strong>？（如果只是单个系统使用，即便是核心价值的，一般也不会列为主数据）</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>5、<strong>是否具有唯一性</strong>？（如果这个数据不强制唯一，全局可能重复，那么可以踢出去了）</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>6、<strong>是否长期有效</strong>？（如果是短期使用，一般不作为主数据。但是这个长期短期和业务有关，比如互联网的订单和造船厂的订单时效性就不一样，前者半个月后大概率就无了，后者一般都持续好几年）</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>至于</span><span><strong>业务共享矩阵法</strong></span><span>，其实就是看这个数据的重要程度和共享程度：</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/131401162362c35846e1b867.48595142.png" referrerpolicy="no-referrer"></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>按照重要程度和共享程度一分，优先级别自然就出来了。至于那些又不重要又不共享的，自然就排除在外了。</span><span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/94848747261109e7d146030.69430921.jpeg" referrerpolicy="no-referrer"></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><span><strong><strong><span>区分难题</span></strong></strong></span><span></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>虽然已经有方法了，但是有些时候遇到不熟悉的业务，还是会蒙圈。一般容易搞混淆的是参考数据和主数据。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>因为参考数据有很多特征和主数据非常类似，比如也是长期有效、跨系统共享、也很重要（价值不一定高）、非常稳定、全局唯一等。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>一些大家熟知的还行，但是一旦跟业务挂钩，如果你不懂业务，几乎就没办法与主数据拆分开。 </span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>DMBOK里提出了二者管理重点的区别：</span></p>
<section>
<p>对于参考数据和主数据，管理的重点是不同的：</p>
<p><br>1）参考数据管理（Reference Data Management，RDM）。需要对<strong><span>定义的域值及其定义进行控制</span></strong>。参考数据管理的目标是确保组织能够访问每个概念的一整套准确且最新的值。</p>
<p><br>2）主数据管理（Master Data Management，MDM）。需要对主数据的值和标识符进行控制，以便能够<strong><span>跨系统地、一致地使用核心业务实体中最准确、最及时的数据</span></strong>。主数据管理的目标包括确保当前值的准确性和可用性，同时降低由那些不明确的标识符所引发的相关风险（那些 被识别为具有多个实例的实体和那些涉及多个实体的实例）。</p>
</section>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>至于如何区分二者，除了分清楚数据价值之外，还有一个比较取巧的方式：看表字段多寡和数据量。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>一般来说，参考数据的</span><strong><span>数据集通常会比交易数据集或主数据集小</span></strong><span>，复杂程度低，拥有的</span><strong><span>列和行也更少</span></strong><span>。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>所以如果你看到一个3列的数据表，无法区分这是主数据还是参考数据，那么盲猜一波参考数据准没错~~~</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>当然，更重要的区分，还是看其价值，就是参与到</span><strong><span>核心业务的程度和价值</span></strong><span>。以及该数据是否完成“</span><strong><span>识别和管理来自不同系统和流程的数据之间的关联关系</span></strong><span>”。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>参考数据会在核心流程里体现，比如配送地址所在省份，但是程度比较低，价值也不是特别大，丢失了甚至都没太大关系（可以通过其他数据推出来）。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>而且，该数据也无需识别和管理不停系统和流程的数据之间的关联关系。所以，列为参考数据是没问题的。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/108292177661109e7e433260.63098105.jpeg" referrerpolicy="no-referrer"></p>

<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>区分方式有很多种，常规方法有两个：</span></p>
