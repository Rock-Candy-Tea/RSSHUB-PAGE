<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>今天我们聊聊<a href="https://www.niaogebiji.com/tag/%E4%BA%A7%E5%93%81%E7%BB%8F%E7%90%86" target="_blank">产品经理</a>的成长路径，理想与现实的矛盾。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>下面这张图是几年前，起点学院之前发布的一张关于产品经理成长路径的片子，我认为写的非常准确合理。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>但我们必须明确一个现实：图中的产品经理成长路径，是理想情况。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong>而现实，永远与理想存在诸多矛盾。我们针对2022年产品经理岗位的发展现状，为大家梳理出三条矛盾点，共同探讨。</strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/74198021262b3f60ae30057.29293665.jpeg" title="产品经理的成长，理想与现实的三条矛盾" alt="产品经理的成长，理想与现实的三条矛盾" referrerpolicy="no-referrer"></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><span><strong><span>01、</span></strong></span><span><strong><span>理想是产品经理，现实是“实施”经理</span></strong></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>首先说个非常扎心的点：</span><span>你有没有想过这样一个问题，先不讨论产品经理的成长路径是否走上最优路径，你是否真的走在产品经理的成长路径上吗？</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">如果起点就不是产品经理岗位的角色的话，又何谈成长路径呢？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这里简单说两点：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>很多实施经理、售前工程师、项目交付经理对自己的工作理解不够清晰，将自己的角色定位成了产品经理岗位。但其实并没有对产品需求的独立决策权。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">随着公司运营成本的提高、领导经营压力的提升等环境因素的提升，产品需求的决策成本逐渐升高。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">很多中小型公司甚至大公司，都已经不敢将权责下放给普通产品经理，而是由产品总监提出需求，由下游产品经理代为执行。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>掌握不了独立提出产品需求的权力，对产品经理的成长是非常不利的影响。它将制衡你对产品的真实感觉，并让你对职业发展产生误导和迷茫。</span></strong></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><span><strong><span>02、</span></strong></span><span><strong><span>理想是用户需求，现实是竞品需求</span></strong></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">亚马逊创始人贝佐斯曾在一些公开场合半开玩笑的讲：不要管你的竞争对手在做什么，因为他们又不给你钱。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但这句话就像名人自传书中说的一样，宣扬成功人士在获得成功之后的感悟对大多数人来说其实价值不大。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">因为一切事件发生的前提是当前时间节点下天时、地利、人和的高效统一，从结果去推原因是最不成熟的思考方式。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>做产品同样如此，对用户需求的把握，需求调研工具的使用，从需求获取、分析、萃取、转化的成熟方法论，其实是贯穿产品经理成长生涯的关键，是产品经理产品设计能力成长背后的一条暗线。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">就像图中所示，从初级产品经理到产品总监，对用户需求的认知能力也在逐渐提升，从明确用户目标到用户需求引导的升级。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但大多数公司为了应对激烈的竞争态势，往往采取防御或跟随策略。我们经常可以看到A产品推出XX功能后，不到一个月B产品也推出了相似的功能。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>从公司层面的战略逐渐渗透影响到产品团队，大家不再关心如何满足用户的需求，而是如何“追上”竞品的脚步。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这对产品经理的成长来说极为不利。关注用户是为了更好的发掘用户未被满足的需求，从而找到新的可能性，不断的拓展产品和业务的边界，这才是最优路径。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但往往短视的公司，会讲所谓的追随竞品“最短”路径，定义为最优路径。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这种现象正在逐渐成为常态。</p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><span><strong><span>03、</span></strong></span><span><strong><span>理想是从0到1，现实是1.1到1.1.1</span></strong></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">我们可以在图中非常明确的看到，产品经理的成长路径，是与自身所负责的产品功能范围与业务领域息息相关的。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">产品新人通常只能负责产品功能的某个feature，逐渐成长到负责整个功能模块，甚至完整产品线。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但这里还隐含了另外两条逻辑：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>第一条：能够从负责某一个功能feature到负责整条产品线，往往是被push的状态。公司对你的培养，你个人经验的提升，都要求你逐渐扩大自己的产品决策管辖范围，为公司贡献更大的价值。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但请注意，这是理想情况。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">如果尝试换位思考一下你会发现这样一个问题：在市场环境如此挤压的情况下，为什么公司要花人力、时间成本去培养一个新人，并在存在一定风险的情况下，逐渐扩大大的影响力和决策权限？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">我直接招聘一个业务能力极强的人，去cover培养成本、时间、和开发风险，岂不是最优方案？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">产品新人的成长路线，就此被割裂了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong>第二条：从高级产品经理到产品总监的跨越，需要时运和机遇。在以往的大公司体系中，一些新孵化的创新项目，是一些高级PM锻炼自身成为产品掌舵人的关键。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这种项目往往需要候选人有较强的主观能动性，去承担面对未知的环境，未知的市场，有限的资源下如何协调好各方利益，将项目按照你的思路按时贯彻推动，产品如期上线。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">就像雷军说过的那句话：从0到1做一个产品很幸运，也很痛苦，因为它太耗费心力了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但这条通道同样正在被割裂。已经很少有公司能够有魄力去孵化新业务了，也很少有高级PM有魄力去承担这样的角色了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这也许会导致这样一个现象，靠谱的高级PM越来越多，但靠谱的产品总监只能在存量中寻找了。</p>
