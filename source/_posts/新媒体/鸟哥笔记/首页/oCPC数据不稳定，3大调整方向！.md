<h1 style="background-image: url('https://qnssl.niaogebiji.com//48aedb8880cab8c45637abc7493ecddd.png'); font-size: 20px; color: #212629; text-align: center; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 16px 20px; background-position: 0px 5px; background-repeat: no-repeat;" align="center"><span>搜索跑<a href="https://www.niaogebiji.com/tag/oCPC%E6%95%B0%E6%8D%AE" target="_blank">oCPC数据</a>不稳定怎么办？</span></h1>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>相信每一位使用oCPC功能的<a href="https://www.niaogebiji.com/tag/SEM" target="_blank">SEM</a>er都和我有同样的经历，每天到办公室后第一件事就是立刻打开百度后台，看<span>o</span>CPC投放包有没有数据异常的情况。<br></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>在我刚接触oCPC时，对oCPC的印象除了心惊肉跳的数据波动，就是不可避免的模型衰退，<span>o</span>CPC的效果如何好像变成了一道玄学题。面对领导和老板的责问，模型衰退也成了惯用的借口。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>想总结一些调整的方法论，用魔法打败魔法，却总有一个问题困扰我：<strong><strong>oCPC功能迭代速度很快，</strong></strong><strong><strong>总结之前版本</strong></strong><strong><strong>的方法论有没有意义？</strong></strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>在三年的oCPC投放后，我给自己的答案是 <strong><strong>有意义</strong></strong><strong><strong>。</strong></strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>无论功能如何变化，通过收集转化数据跑出稳定的数据模型从而大幅度提升广告主消费的本质是不会变的。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><span>在一阶没有被取消时，当时的同事们为了解决转化数低达不到二阶门槛的问题，对物料结构进行了大胆的测试，最终测试出的投放包计划构成最优解：</span><span><strong><span><strong>一套物料</strong><strong>+不同匹配模式的两个计划+标准CPC*1.2的出价。</strong></span></strong></span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>这套方案在百度的历次改版与我数次更换投放行业后，仍然经得起考验，可以快速起量。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>而且重要的不是具体的方法，而是方法背后的调整思路，掌握了思路自然不怕产品的变迁与赛道的更换。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>对于学习成功的oCPC投放包，最常见的优化方式就是新增物料，我的一位同事遇到波动，就从关键词到创意新增个遍，但是往往效果不明显。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>他从网上取经看到“新增物料数量最好别超过原有物料的10%<span>”</span>，从此奉为经典，认为自己之前的问题纯粹是因为新增的物料过多，但是如法炮制后仍然不见数据好转。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>其实问题在于，<strong><strong>o</strong></strong><strong><strong>CPC</strong></strong><strong><strong>投放包的调整没有必成的方法，需要对症下药。</strong></strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>对于度过学习期的oCPC投放包，常见的负面数据波动形式有：<strong><strong>展现量下降、点击率下降、转化率下降、</strong><strong>CPC突增或突降。</strong></strong><strong><strong> </strong></strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>四种波动经常组合出现，并伴随着另一个常见的问题：<strong><strong>搜索词紊乱</strong></strong>。接下来与大家分享如何解决这几类问题。</span><span><span></span></span></p>
<section>
<section>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; color: #212629; font-size: 18px; padding-left: 12px; margin: 25px 0px 20px; font-weight: 600; line-height: 30px; text-align: left;" aria-describedby="tooltip260448"><span>1、</span><strong style="font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, 'Open Sans', 'Helvetica Neue', sans-serif;">两步操作解决展现下降与CPC异常</strong></h2>
</section>
</section>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>对于模型展现量的突然降低，首先要检查的就是 <strong>搜索词结构</strong>，找到降低部分。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span></span><span>确</span><span>认是总体</span><span>降低</span><span>，还</span><span>是头部词降低</span><span>，还是集中在某些单元的下降。</span><span>找到降低</span><span>部分</span><span>对应的关键词后，第二步</span><span>要针对关</span><span>键词进行CPC与上方展</span><span>现胜出率的分析。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>CPC会直接影响关键词排名，从而影响展现量，在考虑展现变化时，需要及时关注对应的CPC变化。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>如果CPC没变或升高但是上方展现胜出率偏低，需要对对应单元的搜索词结构与创意相关物料进行检查，</span><span>确定搜索词与创意物料匹配度，手动筛选创意，暂停低点击率创意，新增创意或高级样式。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>而CPC与上访展现胜出率均偏低，则除了搜索词与创意外，需要更多的关注市场情况，</span><span>是否有竞对在争抢广告位</span><span>。</span><span>对于竞对争抢广告位导致的下降，可以选择的大幅度提升关键词的本地出价来促进模型提升CPC，加速消费。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>虽然建立OCPC投放包后，就处于模型自动出价模式，但是从操作经验与百度客服的反馈看，<strong>超过90%幅度的价格调整，可以显著的影响关键词的实际CPC</strong>。同理对oCPC包成本出现异常的升高时，也可以</span><span>对高成本的关键词出价进行大幅度的下调，来降低其CPC，抑制其消费，效果也好于直接暂停高成本词</span><span>。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>展现消费下降明显的单元还可以进行计划内复制， 把关键词出价*1.5或2，做一个物料相同的单元。也可以起到新增物料的正向效果，同时高的价格促进这部分关键词的消费，把量级拉回来。</span><span></span></p>
<section>
<section>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; color: #212629; font-size: 18px; padding-left: 12px; margin: 25px 0px 20px; font-weight: 600; line-height: 30px; text-align: left;" aria-describedby="tooltip260448">2、<strong style="font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, 'Open Sans', 'Helvetica Neue', sans-serif;">重新跑包应对率值下降</strong></h2>
</section>
</section>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>点击率与转化率的突然下降经常同步出现，对于此类问题同样需要最先关注搜索词结构，</span><span>从根</span><span>本上排除垃圾词或长尾词突增的情</span><span>况，才谈得到对模型的优化。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>当这两个率值同时下降，且搜索词结构没有明显变化时，最可能是模型本身不具备竞争力，已经在逐渐衰退，对于这类包建议重新跑包。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>重新跑包前要注意检查，</span><span>关键词本地出价是否已经调回正常出价、创意物料是否已经完善，最后要注意的是跑包的时间</span><span>。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>在oCPC包取消了核对功能之后，建包的时间就变得非常关键，一定要选择</span><span>在全天流量最充沛的时间段建包。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>在流量峰值时段建包能保证转化数据累计的速度，在建包初期就给到模型尽可能多的转化量。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>以艺考培训行业为例，刚接触时跑oCPC包都是在早上刚上班时，希望能够尽可能的把全天的数据都统计进去。但是艺考培训类更多的是学生自行咨询，而学生有时间看手机搜百度的时间则是午休、下午放学或晚上放学。</span><span>早早建立的投放包要么空耗没有转化，要么被其他的成型包挤压，基本没有办法顺利起量，而把跑包时间改为量级峰值的中午之后，效果立竿见影。</span></p>
<section>
<section>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; color: #212629; font-size: 18px; padding-left: 12px; margin: 25px 0px 20px; font-weight: 600; line-height: 30px; text-align: left;" aria-describedby="tooltip260448">3、<strong style="font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, Oxygen, Ubuntu, Cantarell, 'Open Sans', 'Helvetica Neue', sans-serif;"><strong>合理否词，看量级不看频率</strong></strong></h2>
</section>
</section>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>对于搜索词结构紊乱，最常见的动</span><span>作就是否词。对</span><span>于学习成功的投放包，大部分的操作建议是每周否一次词，但是对部分低质量模型而言，搜索词杂乱，有效转化少，一周一次根本不现实。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我的建议是 <strong>否词频率可以高，但是要减小影响的范围</strong>，</span><span>能单元否不做计划否，能精确否不做短语否，每次控制在否掉的部分垃圾词消费或者转化量在模型总量的20%以内。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>如果一个投放包20%以上的消费或转化都是无效词带来，优化模型的意义已经不大。那应该做的是立刻暂停计划，重新检查物料，把低质长尾词剔除。 </span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>对于多久调下调一次价格，每次下调的幅度是多少，多久加一次词，每次加多少词这种调整问题，相信各位的百度代理都已经给了大家答案，在此不再赘述。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>另外，oCPC模型要数据波动时，要 <strong>多做</strong><strong>操作回溯</strong>。及时恢复操作记录里的负向操作，也可以起到很好的优化效果。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>最后想与大家分享的是，我的投放包数据突然变差，要不要立刻调整？</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我的答案是 <strong>让子弹再飞一会儿，看看量级峰值时段的表现</strong>。如果在峰值时段没有明显的恢复，可以按照以上的操作步骤进行优化，或选择重新跑，一定要给oCPC自我调节的时间，不能和CPC模式一样追求瞬间出效果。</span></p>
