<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;" data-mpa-powered-by="yiban.io"><span>《微信背后的产品观》提到，<strong>一个优秀的<a href="https://www.niaogebiji.com/tag/%E4%BA%A7%E5%93%81%E7%BB%8F%E7%90%86" target="_blank">产品经理</a>不是工程师、也不是被培训出来，而是经过多个案例，只有在多个案例里失败过，才能通过经验的积累去做出好产品。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>世界知名杂志《纽约客》的著名撰稿人麦尔坎·葛拉威尔在其畅销书《异数》中提出过一个著名的“一万小时定律”说，或许能带给我们一些启示。该“定律”指出：“人们眼中的天才之所以卓越非凡，并非天资超人一等，而是付出了持续不断的努力。只要经过一万小时的锤炼，任何人都能从平凡变成超凡。”</span><span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>对于产品经理来说，这种经验的积累可以通过外在的收入表现出来。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>一个刚毕业的大学生，在一线城做产品经理，大部分的同学起步在8-10K左右，加上年终奖，还没有工作多长时间的产品新人就可以拿到一年15万左右</span><span>的平均薪资。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>相较于其他职业，产品经理的收入在早期是非常可观的。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>这个职位可能也是包容度最强的一个职业了，<strong>没有之一。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>因为新技术的变化快，所以在大学时期，是没有完全符合产品经理的最新课程内容和专业的。</span><span>因为产品经理始终属于IT行业，所以<span>从业者主要聚焦在计算机类专业的，同时</span>产品经理要求有医疗、金融、物流等专业知识，所以还是有不少其他专业的朋友加入。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>至少我带过的学生和身边的朋友，有心理学、土木工程、数学等专业成为了产品经理。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong><span>产品经理靠什么成长：野蛮自主。</span></strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>曾经我在问一个阿里的产品朋友，你们在大厂做产品经理怎么成长？</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>“野蛮成长，只能靠自己”</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>因为没有具体的专业，每家公司产品经理的评级标准也不一样，所以很多人成为产品经理靠的是野蛮成长，即使是大厂的产品经理们。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span><strong><span>野蛮成长的后果是注定会有各种失败</span></strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>比如我们重构PMTalk，就顺应了这样的结果。</span><span>2019年来PMTalk的用户DAU翻了10倍，但这个数据在2022就止步不前了。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>现在复盘看来，主要问题是：<br></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>做了多余的功能模块，用户不会用；也少了优质的内容，降低了用户留存时间；功能路径缺失，造成了用户体验不完善。</span></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><strong><span>1.没有调研用户规模</span></strong><span><br></span></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>确定一个功能模块什么时候该做什么时候不该做，早期犯的最错错误就是看竞品和同行热门产品，感觉那些用户喜欢的火的产品功能，就一定要自己也要加上。<br></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>比如我们做了每天体验1款APP，用户社群转化率高，就拍脑袋的做成了功能设计，但目标用户少得可怜，同时内容的生产门槛高，因此上线后内容产生乏力、目标用户群体小，没有带来明显的用户增长。</span></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><strong><span>2.只看竞品的热门功能</span></strong></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>做产品设计的时候，很多产品经理一定会去查看竞品。包括同行的竞品和不同行但是有相似功能的“竞品”</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>他们能够成为产品头部产品，是因为做了“某些功能”才导致的，这是很多初级产品经理最容易犯的错。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>不要看别人做了什么，而是看别人为什么不做。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我建议产品经理对待竞品，尤其是同行的竞品，相关的打发和产品设计方案一定有尝试过，所以不要轻易去抄袭别人的功能一模一样。</span></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><strong><span>3.不要重点考虑风口</span></strong></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我见过很多产品经理，都因为雷军这句话：”站在风口上，猪都会飞“。去一头脑的抓住风口去投入资源做产品，结果死得很惨。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>因为很多产品经理的产品其实<strong>”站都站不起来“</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>比如在国产手机供应链成熟的时候，雷军当时自己敢于拿出了1亿美元出来创业，为小米奠定了风口的基础。</span><span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>一个产品在用户不多的时候，建议先做一个端。稳定产品框架和打磨好用户体验，不要轻易去做多端。</span><span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>从需求调研到立项，再到产品设计和研发，一个产品要经过数个版本才能有一个足够稳定的MVP版本。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>2010年移动互联网爆发，大家看做APP就容易拿到投资，很多产品经理就伙同几个开发哥们，辞掉稳定的、有股权的工作去做产品创业了，<strong>结果亏得家破人亡的案例比比皆是。</strong></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>给用户提供一个有市场规模、可用、操作闭环、商业模式清晰的产品是做产品的首要条件。</span></strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>找到一个你商业模式下目标用户最常用的产品形态，专注打磨就可以了。比如对于文章内容浏览，始终PC上的操作更加符合用户消费习惯。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>站稳了一个产品后，再考虑风口才会飞得起来。</span></strong></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;"><strong><span>4.一个业务不亏，就加油做大他</span></strong></h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>很多产品经理都会因为ROi，变得很贪心。一旦当产品有收入后，就希望去扩展其他业务获得收入。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong><span>但是当前的收入瓶颈真的到了吗？</span></strong><span>很多产品经理并去挖掘。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>比如2021年二季度知乎平均月活跃用户数9430万，同比增长46.2%。平均月付费会员数为470万，同比增长121.1%；付费率由2021年一季度的4.7%提升至二季度的5%。可以看见知乎的ROI增加缓慢了，需要急需新的业务线来带来收入。</span><span></span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我们要去看到同行、自家产品的增长环比，是否达到瓶颈了。如果没有到底，则应该继续专注打磨他不断提升，减少了额外的资源浪费投入。</span></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><span>我相信，如果你今天看了这篇文章，你一定会经历以上某个错误，别怕，当做积累，相信你也能够成为优秀的产品经理。</span></p>
