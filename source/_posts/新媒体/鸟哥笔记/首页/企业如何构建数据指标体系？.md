<section style="box-sizing: border-box; min-height: 22px;">
<section style="box-sizing: border-box; min-height: 22px;">对于企业而言，实现<a href="https://www.niaogebiji.com/tag/%E8%90%A5%E9%94%80%E6%95%B0%E5%AD%97%E5%8C%96" target="_blank">营销数字化</a>转型的核心是数据，其中有两个要点：<span style="box-sizing: border-box; font-weight: bolder;">第一</span>，企业必须拥有足够量级的一方数据，<span style="box-sizing: border-box; font-weight: bolder;">第二</span>是数据要应用于业务侧，而非只是用于分析洞察，仅停留在数据表面，成为数据库。</section>
</section>
</section>
<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<section style="box-sizing: border-box; min-height: 22px;">
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">拥有一方数据的重要性毋庸置疑，我们不再佐证。谈谈第二点，企业在实践数据应用时，往往有两个层面的应用场景或者称为数据价值：</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">对内提升经营效率</span>：通过全渠道数据分析，辅助企业制定更为科学的经营决策，例如销售预测、商品分析、供应链赋能等细分场景；</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">对外提升营销效果</span>：构建用户标签画像以实现分层精准运营，有效提升营销运营的效果，增加粘性创造增长，例如会员权益、复购分析、成长路径分析等细分场景。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">由于数据可承载的应用场景十分庞大（参考下图），为此企业无论是在搭建CDP等平台还是做有关数据层面的建设时，一定要明确业务增长目标和用户需求场景，避免出现CDP是归IT负责还是业务负责等非此即彼的问题，即需要不同的部门用统一的视角看待业务和用户，才能把事情做好。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">这也就要求着企业在构建One-ID体系下，根据业务增长目标和用户运营场景，进行包括前后链路数据，业务系统与运营系统数据的<a href="https://www.niaogebiji.com/tag/%E6%95%B0%E6%8D%AE%E6%8C%87%E6%A0%87" target="_blank">数据指标</a>体系规划</span>。创略科技认为，数据<a href="https://www.niaogebiji.com/tag/%E6%8C%87%E6%A0%87%E4%BD%93%E7%B3%BB" target="_blank">指标体系</a>是实现数据驱动决策的“指南针”。</p>

<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">下面我们来详细解读“数据指标体系”：</p>

<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">

<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">先说结论。数据指标是一种度量，是基于业务目标度量业务过程中重要业务要素的一种手段。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">这里产生了三个核心关键点：业务目标、业务过程、业务要素，并包含了两大维度：业务及技术（数据）。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">业务目标</span>：明确业务目标是实现数据驱动的核心。往往业务目标是基于公司、业务、产品等多维度拟定，如提升GMV、盘活沉睡会员、用户日活量等依托于现阶段发展现状的核心价值。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">业务过程</span>：业务目标是最终企业希望达到的效果，为实现这一目标则会有一系列的业务流程与策略。以提升GMV为例，用户的常见购买流程多以浏览-注册-下单-支付-复购-裂变，为提升GMV企业则需要关注用户数、转化率以及客单价，为此需要梳理用户全渠道生命旅程，基于消费者核心运营链路（消费生命周期、会员运营周期、互动生命周期、产品消费周期等）展开策略设计。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">业务要素</span>：支撑业务过程的则为关键业务要素，如以会员运营周期为例，其业务要素即为未入会率、等级和贡献度变化、付费和VIP会员、会员激活率等。</p>
</section>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;"><img style="box-sizing: border-box; margin: 0px; padding: 0px; vertical-align: baseline; border: 1px solid #e5e5e5; max-width: 100%; height: auto !important;" src="https://v-static.36krcdn.com/ent/dianping_editor/fbc62863-cb90-4572-ba33-8166165c247b.png" alt="企业如何构建数据指标体系？" width="1080" height="608" referrerpolicy="no-referrer"></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">数据指标拥有多维度的特性，需要以多视角来看。举个例子，以官网点击率作为数据指标，但其还能延伸出工作人员官网点击率、某落地页跳转官网点击率、某个时段官网点击率等等维度。而这个多维度则由业务数据所决定的，为此数据指标需要业务与技术的双向协同。即以数据发生、业务运营为核心视角，将数据关键节点设计出来，满足后续的数据存储和归类需求，方便数据应用快速定位和业务技术统一使用口径。</p>
<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">当你懂得了什么是数据指标，那么为什么要做数据指标这件事就呼之欲出了。企业可以通过数据指标，从不一样的维度了解业务过程，用指标去衡量各个业务过程、业务要素带来的价值，从而加以干涉，引导更具效率的完成业务目标。所以，企业在构建数据标签体系时，一定要遵从四大标准：</p>

</section>
<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<section style="box-sizing: border-box; min-height: 22px;">
<section style="box-sizing: border-box; min-height: 22px;">
<section style="box-sizing: border-box; min-height: 22px;">
<section style="box-sizing: border-box; min-height: 22px;">在企业落地数据指标体系时，往往会遇到基于最佳实践和业务需求设计的指标无法落地；规划的数据指标体系配套的运营策略没有细化等问题，导致数据指标无法真正发挥效用。实际上，数据标签体系与数据质量、业务流程、企业制度等各个企业经营维度息息相关。在真正建设之前，企业需要严格自查优化。</section>
</section>
</section>
</section>
</section>
<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">数据质量和数据标准</span>：数据标签度量效果对数据样本数以及数据质量，有着强依赖性。为此，企业需要明确数据标准化梳理需求、新增维度需求等，并将数据进行科学治理确保数据价值及数据唯一性治理；</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">业务在线化建设</span>：由于部分企业尚未构建完整的在线化流程，使得执行和过程管理数据在线化缺失，并且目标管理大多依赖于手工管理，进度监控较难。企业需完善在线化能力，确保数据及流程记录在案，并利用数字化能力提高管理效率以及效果；</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">业务规则和制度优化</span>：多数企业拥有线上线下多渠道，但往往不同渠道对业务的理解，对权益的制定并不相同，这导致数据指标的衡量标准参差不齐、效果大打折扣。企业需要推动全渠道会员权益、业务理解统一化发展，并对类似于异常会员判定、监管机制等制度标准化完善，形成统一标准；</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;"><span style="box-sizing: border-box; font-weight: bolder;">数据应用体系缺失</span>：数据指标体系只是数据应用的一环，其中还包括标签体系构建、分层分群策略、数据模型应用等多个环节，所以企业仍同步完善基于标签体系的消费者运营体系、私域裂变链路等维度，确保可以环环相扣。</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">当然，不同业务要素、不同业务流程、不同业务目标下的前置条件并不相同，更需要因地制宜的筛检自查，在最大程度实现业务目标的同时，推动业务、技术改造，并完善业务流程与运营体系，以点到面驱动各个维度的转型升级。</p>
</section>

<h2 style="box-sizing: border-box; margin: 25px 0px 20px; font-size: 18px; background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; color: #212629; padding-left: 12px; font-weight: 600; line-height: 30px; text-align: left;"><span style="box-sizing: border-box; font-weight: bolder;">三、数据指标体系梳理流程如何？</span></h2>
</div>
<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">当我们已经充分理解数据标签体系的搭建流程，我们则需要考虑落地流程，以及各部门的协同情况。创略科技基于多年项目实操经验实现流程化梳理，帮助企业搭建基于业务场景及目标的数据指标体系：</p>
</section>
<section style="box-sizing: border-box; min-height: 22px; color: #333333; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', 'PingFang SC', 'Hiragino Sans GB', 'Microsoft YaHei', 'Helvetica Neue', Helvetica, Arial, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol'; font-size: 16px; background-color: #ffffff;">
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">1.<span style="box-sizing: border-box; font-weight: bolder;">数据需求发起</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">项目业务人员参与需求调研，并进行业务流程梳理和需求沟通</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">2.<span style="box-sizing: border-box; font-weight: bolder;">需求调研评估</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">数据分析师整理需求调研内容，编制《需求调研报告》</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">3.<span style="box-sizing: border-box; font-weight: bolder;">需求范围确认</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">项目业务人员针对《需求调研报告》的内容和评估意见，确定指标体系需求范围</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">4.<span style="box-sizing: border-box; font-weight: bolder;">指标体系方案规划</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">数据分析师就已确认需求，开展指标划设计工作，反馈相关业务体系指标规划方案</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">5.<span style="box-sizing: border-box; font-weight: bolder;">指标体系方案确认</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">数据分析师就指标体系方案，对项目业务人员进行内容串讲，项目业务人员确定方案内容</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">6.<span style="box-sizing: border-box; font-weight: bolder;">数据采集补充</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">数据分析师就指标体系规划方案，确定是否需要数据采集，若需要则选择埋点方式为埋点事件选择代码埋点、无埋点、服务端接入、活动采集等方式</p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">7.<span style="box-sizing: border-box; font-weight: bolder;">数据验证及效果评估</span></p>
<p style="box-sizing: border-box; margin: 0px 0px 20px; padding: 0px;">数据分析师根据指标体系方案，搭建对应的可视化看板，项目业务人员基于可视化数据，对数据指标体系进行效果评估并优化</p>
