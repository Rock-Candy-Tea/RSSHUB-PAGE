<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/58603551462b160f5bf16b2.62395160.jpeg" title="别让这3个逻辑陷阱，毁了你的深度思考力" alt="别让这3个逻辑陷阱，毁了你的深度思考力" referrerpolicy="no-referrer"></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">今天总结了3个在生活和工作中常见的<a href="https://www.niaogebiji.com/tag/%E9%80%BB%E8%BE%91%E9%99%B7%E9%98%B1" target="_blank">逻辑陷阱</a>，希望能够帮到各位的思维能力成长。文章比较长，但大家看完一定会有所收获。<br><br>下面进入正文：</p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;">1. 逆推谬误：先有结论，再有过程</h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">很早以前我在做咨询行业的时候，老板告诉我一条铁则：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">“过程比结论更重要。”</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">怎么去理解这句话？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">举个例子。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">比如现在你是一个电商营销经理，公司最近新推了一个口红出去，营销费用花了不少，打了很多广告，可用户增长一直做不上去，销售额也平平无奇。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这个时候你会怎么做？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">找原因，做分析，对吧。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">那最容易想到的原因有哪些呢？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">脑子里很快蹦出来几个选项，比如产品本身的卖点不突出不吸睛；或者竞争对手是金主爸爸，砸的营销费用是我们的N倍；也可能是消费者的品味发生了变化，不再喜欢这种风格的口红设计。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">以上这些，是最容易想到的3个原因。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">那为了证明这些假设的原因，接下来会去做什么呢？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">找support, 也就是我们常说的论据。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">产品本身卖点不突出，那就来看看具体有哪些卖点，再和同类的竞品对比一下，看看是不是足够吸睛；</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">竞争对手花的营销费用更多，那就来看看他们把钱都投在了哪些渠道，估算一下整体的媒介预算花费有多少；</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">消费者品味发生了变化，这个更简单，去扒拉一下产品主页的评论区留言，看看用户们都是怎么评价咱们产品的。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">到目前为止，这个思路似乎没什么问题，最后洋洋洒洒写出一个分析报告PPT，提给老板交差，大功告成。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但是大家有没有发现一个bug?</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">以上这套思考路径，是按照先有结论（假设3个销量不佳的可能原因），再去论证（找资料和数据）的逆推逻辑来操作的。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但这么做是正确的吗？真能找到问题的症结吗？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">并不能。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这是一个真实发生过的故事，所以我才拿出来举例，这件事最后真正的原因是：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong>电商平台的算法规则改了。</strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">因为以往的营销流量打法，并不适用于现在的新规则，所以才导致用户增长和销售额上不去。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">通俗来讲，就是过去你可以100块钱全部梭哈买流量，可现在你要把100块拆成20块、30块、50块分别去花，去操盘不同的版块内容，最后形成营销组合拳，这样才会有效果。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">而之前设想的3个原因，没有一个说到点子上。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这就是逆推型的逻辑谬误：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong>只有结论，再有过程。</strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">仅凭经验主义总结出几个可能的原因出来，导致后面所有的论证都在潜意识里向这些方向去靠拢，而忽略了意想不到的新可能。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这和警察探案很像。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">当警察带着主观意识去锁定一个犯罪嫌疑人的时候，在找证据的过程中，就会不自觉朝着能证明TA有罪的方向去寻找，而忽略了能证明TA无罪的信息。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">如果没有及时意识到这个思维谬误，最后很容易导致冤假错案的发生。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">在工作中也是如此。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">遇到问题了，我们往往会根据个人经验先提出假设，再去做论证。这个方法虽然能够帮助我们聚焦思考方向，但同时也屏蔽了其他更多的可能。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">有一个词专门描绘这种现象，叫：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong>懒惰的真相。</strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">在大众传播的语境里，人们更倾向于别人告诉他们真相，而不愿意自己动手去挖掘真相。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">因为从无到有找寻一个真相，投入的成本太高。可如果事先已经有一个假设的真相存在，我们只用去证明它的对错，反而更容易。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">面对简单的问题还好，一旦问题的复杂系数变高，这么做往往会离真正的真相越来越远。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">那么正确的做法是什么呢？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">很简单，把顺序倒过来。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">过去是先有结论，再去证明。现在我们先找证据，再推理结论。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">还是文章开头那个例子，销售额和用户增长上不去，这是问题。那么基于这个问题，我们不要急着提出假设，而是去搜寻事实。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">怎么搜寻？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><strong>三个字，找异常。</strong></p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">既然按照以往的经验操作，得出了完全不一样的结果，那过程中一定发生了异于平常的事情，你要把它们全部找出来。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">当时我们事后复盘就找出了很多异常，比如媒体预算投入的ROI显著降低，追踪消费者的行为发现路径明显拉长，该出现我们产品广告的地方却出现了别的内容。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">所有这些异常现象拼接在一起，推理出一个我们在过去怎么也不会想到的结论：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">平台的规则改了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">然而当我们带着这个结论去问电商平台的小二时，对方表示确实如此。并且一个月前就告知了我们此事，可当时谁也没有引起重视。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">虽然蒙受了损失，但大家吃一堑长一智，以后分析问题的时候，再也没有掉入这样的逻辑陷阱中去。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;"><img src="https://ngbjimg.xy599.com/161709594662b160f6004937.82873963.jpeg" title="别让这3个逻辑陷阱，毁了你的深度思考力" alt="别让这3个逻辑陷阱，毁了你的深度思考力" referrerpolicy="no-referrer"></p>
<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;">2. 断层谬误：我以为你知道</h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">大家一定玩过一个游戏：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">你画我猜。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">一个人在画板上画画，另一个人根据画板上的图案来猜对方画的是什么。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">听起来很简单对吧，但玩起来总让人啼笑皆非。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">比如有一次我和朋友玩，我抽中了一句诗：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">“锄禾日当午。”</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">于是我画了一个人弯着腰在锄地，头上再画个太阳，很好猜对吧？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但我那个二货朋友楞是猜了四五个都不对，急得我手舞足蹈跟他比划，而我又不能说话，最后时间到了只能含恨作罢。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">事后我吐槽我的朋友，你搞什么飞机，这都猜不出来。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">可他满脸无辜地望着我，表示这一幅图有很多种可能啊！</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">在这个例子中，其实我和我的朋友都没有错，错的是什么呢？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">信息认知的不对等。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">我拿到题目的时候，脑子里第一反应的画面是一个人在烈日下锄地，这没有任何错误。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">但我的朋友他不知道我拿的题目是一首诗，而且还是一句中文诗。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">所以当他看到一个人在锄地的画面，会想到很多种可能。比如梵高有一幅画也是一个人在太阳下锄地，他当时猜的就是这个。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这种情况，在《让创意更有粘性》这本书里面，有一个专门的概念来描述它：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">知识的诅咒。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">当你知道一件事情的时候，你就无法想象自己不知道它的时候是什么样子。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这里的知识，不一定是真正的“知识”，它也可以是信息，事实，情报等。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">而这就导致了我今天要说的第二个逻辑谬误，沟通断层。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">你以为对方都知道，可对方并不知道。你以为对方能对你感同身受，可对方与你的悲喜并不相通。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这种现象在日常工作中，简直太常见了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">比如你在工作的时候，突然被叫入参加一个会议。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">会上坐了七八个人，大家你一言我一语讨论地热火朝天，你完全不知道前因后果，不知道发生了什么事情，心里着急想着手头还有别的工作亟待完成。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">会议结束，你的老大突然跟你说，刚才大家说的你都记下来了吧。这个新项目需要你来协助，你先把接下来的工作内容规划整理一下。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">而你听完不知所措一脸懵逼地呆在那里，发出灵魂三问：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">“我是谁？我在哪儿？我要干嘛？”</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这就是知识的诅咒，你的老大和你处于信息层级的两个维度：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">他以为你对项目都知道，都理解。而你脑子里一片空白，连这个项目的前因后果都不晓得。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这时，你们的沟通就发生了断层。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">类似的情况还有很多。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">再比如，产品经理和程序员之间的沟通。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">产品经理是市场导向思维，关注的是用户体验，是商业转化，是deadline。而程序员是理工科思维，关注的是可实现性，是因果转化，是问题聚焦。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">一个是现实主义，一个是理想主义，两者的认知角度不同，就会导致日常沟通上的障碍发生。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">产品经理说明天这个功能一定要上线，用户非常喜欢。可程序员拒绝，说实现这个功能等于要把之前的产品框架全部大改，不可能这么快做好。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">于是矛盾就这么产生了。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">那如何解决这个问题？</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">很简单，在沟通之前，先弄清楚自己和对方处于信息认知的哪个阶段。这里有个NPA原则，大家可以记一下。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">所谓的NPA，分别指代三个单词：</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">Nothing/Part of it/All.</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">它们分别代表了信息沟通的三个不同阶段。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">Know nothing，是最初的阶段，对方对你要沟通的信息一无所知。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这个时候你就要从事情最初的起因背景开始讲起，按照事情发展的时间顺序，将重点信息摘录出来告诉对方。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">Know part of it，对方对你要讲的信息知晓一部分。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">这个阶段，在沟通时我们可以做一个简单的前情概要，一般不超过一分钟。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">简单回顾事件之前的重要信息，然后开始说对方不知道的信息，做好承上启下。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">比如：“上次不是说到......，现在有了新变化.....”, “之前有提到......事情后来有了发展......”</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">最后，Know it all, 对方对你要说什么完全了解。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">此刻你要做的就不是告知信息，而是要基于信息进行讨论。这时就不需要前情回顾，而是针对未来会发生的事情商量对策。</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">事先了解对方对信息的接受阶段，能够大大提高我们的沟通效率，避免“知识的诅咒”产生。</p>

<h2 style="background-image: url('https://qnssl.niaogebiji.com//6cdd60ea0045eb7a6ec44c54d29ed402.png'); font-size: 18px; color: #212629; text-align: left; font-weight: 600; line-height: 30px; margin: 25px 0px 20px; background-size: 8px 20px; background-position: 0px 5px; background-repeat: no-repeat; padding-left: 12px;">3. 滑坡谬误：粗暴的因果推理</h2>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">所谓滑坡谬误，就是使用一连串单线程的因果推论，来强行达到自己的目的。但在这个过程中夸大了每个因果链的关系强度，最终导致逻辑谬误的产生。<br><br>在滑坡谬误中有一个经典句式：<br><br>“如果发生A，就会发生B，如果发生B，就会发生C。所以为了避免C的发生，我们要阻止A.”<br><br>这里的A到B，B到C，就是一个个的“坡”，最后从A推论到C的整个过程就是一个滑坡。<br><br>比如，大家小时候一定听过长辈这样教训过：<br><br>“考不上好初中，你就考不上好高中。考不上好高中，你就上不了好大学。上不了好大学，就找不到好工作。所以考不上好初中，以后就找不到好工作。”<br><br>乍一看好像没毛病，但仔细思考就会发现问题。<br><br>考不上好高中，就一定上不了好大学吗？<br><br>上不了好大学，就一定找不到好工作吗？<br><br>再往深一点想，身边有没有人没考上好高中，但依然上了好大学的呢？<br><br>有没有人没上过好大学，但依靠自己的不懈努力最后也获得了好工作呢？<br><br>显然是有的，我就是其中之一。<br><br>那这一连串推论的逻辑漏洞在哪里？<br><br>答案是，把每一个环节的可能性当做了必然性。<br><br>考上了好高中，只是增加了考上好大学的可能性。而考上了好大学，也只是增加了找到好工作的概率，后面依然要靠你的专业技能、工作经验等综合实力。<br><br>用前面那个句式来解释，就是发生了A，也不一定就会发生C.<br><br>而相对应的，如果要证实这个滑坡确实存在，就要有充足证据证明每个“坡”都有合理且强烈的因果关联。<br><br>比如你若能够证明，考上好高中是考上好大学的唯一途径，考上好大学是找到好工作的唯一途径，那么这个滑坡推论就是成立的。<br><br>但很显然不能。<br><br>那说了这么多，这个理论对我们的日常生活和工作有什么作用呢？<br><br>有，培养我们的思辨能力，不被轻易带节奏。<br><br>当我们看到一个现象，发现一个问题，不要一下子去想根本原因，而是去找关联事实。<br><br>还是前面这个例子，如果“没有找到好工作”是一个问题，那么和这个问题所相关的事实有哪些？<br><br>比如，很多企业缩减招聘名额，同一个岗位竞争激烈，市场薪酬行情下降等等。<br><br>然后围绕这些事实，去做相应的拆解和对策分析。<br><br>同一个岗位竞争激烈，那么接下来可以有3个选择：<br><br>第一，降低自己的薪资要求，提升自己的性价比；<br><br>第二，给自己的能力做加码，面试的时候突出自己更丰富的项目案例经验，给面试官留下深刻印象，或者请公司的同事做内推；<br><br>第三，换个岗位应聘。<br><br>围绕每个事实层层向下做拆解，你就会找到相应的办法。而不是简单粗暴地把原因归结为：<br><br>当年没有考上好学校，所以现在没办法了。<br><br>说白了，打破滑坡谬误的关键就在于，不要只盯着一开始的因，和最后的果，而是把中间的每一个环节都拿出来，思考它们之间的必然性。<br><br>只要一个环节出现了问题，整个推导过程就不成立。<br><br>以上就是今天的内容，最后简单总结一下，3个常见的逻辑陷阱，请你一定要避免：<br><br>1. 逆推谬误：先有结论，再有过程；</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">2. 断层谬误：我以为你知道；</p>
<p style="background-image: none; font-size: 16px; color: #000000; text-align: inherit; font-weight: inherit; line-height: inherit; margin: 25px 0px 20px;">3. 滑坡谬误：粗暴的因果推理。<br><br>以上，感谢大家的时间 =）</p>
