style="color:rgb(37,37,37);">Chief Technology Officer「⾸席技术官」</span></p></li><li><p><span style="color:rgb(61,170,214);">COO：</span><span style="color:rgb(37,37,37);">Chief Operations style="color:rgb(61,170,214);">PM：</span><span style="color:rgb(37,37,37);">Product Manager「产品经理」或 Project Manager 「项⽬经理」</span></p></li><li><p><span style="color:rgb(61,170,214);">OP：</span><span style="color:rgb(37,37,37);">Operations 「技术运维」</span></p></li><li><p><span style="color:rgb(61,170,214);">HE：</span><span style="color:rgb(37,37,37);">Hardware Engineer「硬件⼯程师」</span></p></li><li><p><span style="color:rgb(61,170,214);">FE ：</span><span style="color:rgb(37,37,37);">Front End Engineer 「前端⼯程师」</span></p></li><li><p><span style="color:rgb(37,37,37);">Database Administrator 「数据库管理员」</span></p></li><li><p><span style="color:rgb(61,170,214);">QA：</span><span style="color:rgb(37,37,37);">QA Engineer 「测试⼯程师」</span></p></li></ul><p style="text-align:justify;"><span style="color:rgb(37,37,37);">2、⽂档术语</span></p><ul><li><p><span style="color:rgb(255,76,0);">MRD：</span><span style="color:rgb(37,37,37);">Market Requirements Document 市场需求⽂档，常⻅的为竞品分析，⼀般⽤于⽴项，基于⽬前市场数据及竞品等进行项目提出，⼀般用于提案。</span></p></li><li><p><span style="color:rgb(255,76,0);">PRD：</span><span style="color:rgb(37,37,37);">Product Requirement Document 产品需求文档，⼀般是说明实现的过程，较为详细。有些公司为了敏捷开发需要很多时候会直接在原型图上⾯通过注释⽅式进⾏更直观的展示。</span></p></li><li><p><span style="color:rgb(255,76,0);">PMD：</span><span style="color:rgb(37,37,37);">Program Managment Document 项目管理⽂档，⼀般包括项目进度、项目资源、责任⼈和项目输出物，常规通过visio进⾏⽢特图绘制管理。该⽂档⼀般贯穿整个项目全程，衡量项目进度。</span></p></li><li><p><span style="color:rgb(255,76,0);">BRD：</span><span style="color:rgb(37,37,37);">Business Requirement Document 商业需求文档，是基于商业目标或价值所描述的产品需求内容文档（报告）。其核⼼的用途就是用于产品在投⼊研发之前，由企业⾼层作为决策评估的重要依据。其内容涉及市场分析，销售策略，盈利预测等，通常是供决策层们讨论的演示文档，一般比较短小精炼，没有产品细节。</span></p></li><li><p><span style="color:rgb(255,76,0);">DRD：</span>Design Requirement Drawing 交互设计文档，⼀般用来承载交互说明，并交付给前端、测试以及开发⼯程师参考的⽂档。</p></li><li><p><span style="color:rgb(255,76,0);">FSD：</span>Functional Specifications Document 功能详细说明，定义产品功能需求的全部细节。FSD⼀般通过⼀张张的截屏和⼀条条功能点来定义产品规格。这是⼀份可以直接让⼯程师创建产品的文档。</p></li><li><p><span style="color:rgb(255,76,0);">PSD：</span>Product SpecificationsDocument 产品规格文档，PSD是⼀个较不流⾏的缩写，但是在有这样⼀个文档的机构中，它⼤体和上⾯描述的功能规格文档（FSD）相同。</p></li><li><p><span style="color:rgb(255,76,0);">SRS：</span>Software RequirementsSpecification 软件需求文档，软件需求文档（SRS）是另⼀较不流⾏的缩写，在创建SRS的机构中，它在内容和细节上和上⾯描述的PRD或FSD有些想像。</p></li></ul><p style="text-align:justify;"><span style="color:rgb(37,37,37);">3 运营术语</span></p><ul><li><p><span style="color:rgb(0,122,170);">AARRR模型：</span><span style="color:rgb(37,37,37);">Acquisition、Activation、Retention、Revenue、Refer这个五个单词的缩写，分别对应这⼀款移动应⽤⽣命周期中的5个重要环节。分别为：获取⽤户、提⾼活跃度、提⾼留存率、获取收⼊、⾃传播。</span></p></li><li><p><span style="color:rgb(0,122,170);">RFM⽤户模型</span><span style="color:rgb(37,37,37);">：R(Recency)表示客户最近⼀次购买的时间有多远，F(Frequency)表示客户在最近⼀段时间内购买的次数，M (Monetary)表示客户在最近⼀段时间内购买的⾦额。</span></p></li><li><p><span style="color:rgb(0,122,170);">⽤户增⻓的S型曲线、 J型曲线：</span><span style="color:rgb(37,37,37);">指⼤多数产品，在产品发展周期中，都会遵循S型曲线的增⻓⽅式；但有些产品，可能产⽣J型曲线。</span></p></li><li><p><span style="color:rgb(0,122,170);">⽤户⾦字塔模型：</span><span style="color:rgb(37,37,37);">第⼀级是社区的管理⼈员。第⼆级是⽤户管理⼯具。第三级是有价值⽤户，基本就是在社区⾥⾯⾜够活跃，并且给你的社区贡献有效价值的那些⽤户。第四级⼀般性⽤户。</span></p></li><li><p><span style="color:rgb(0,122,170);">OMTM：</span><span style="color:rgb(37,37,37);">One metric that matters北极星指标，指唯⼀重要的指标。</span></p></li></ul><p style="text-align:justify;"><span style="color:rgb(37,37,37);">4 交互术语</span></p><ul><li><p><span style="color:rgb(255,76,0);">UI：</span><span style="color:rgb(37,37,37);">User Interface ，⽤户界⾯，泛指⽤户的操作界⾯，UI设计主要指界⾯的样式，美观程度。</span></p></li><li><p><span style="color:rgb(255,76,0);">NUI：</span><span style="color:rgb(37,37,37);">Natural User Interface ⾃然⽤户界⾯，NUI则只需要⼈们以最⾃然的交流⽅式（如语⾔和⽂字）与机器互动，使⽤NUI的计算机不需要键盘或⿏标。</span></p></li><li><p><span style="color:rgb(255,76,0);">GUI：</span><span style="color:rgb(37,37,37);">Graphical User Interface 图形⽤户界⾯，指采⽤图形⽅式显示的计算机操作⽤户界⾯。</span></p></li><li><p><span style="color:rgb(255,76,0);">HUI：</span><span style="color:rgb(37,37,37);">Handset User Interface ⼿持设备⽤户界⾯</span></p></li><li><p><span style="color:rgb(255,76,0);">WUI：</span><span style="color:rgb(37,37,37);">Web User Interface ⽹⻚⻛格⽤户界⾯</span></p></li><li><p><span style="color:rgb(255,76,0);">UE or UX：</span><span style="color:rgb(37,37,37);">User Experience ⽤户体验，指以⽤户体验为中⼼的设计。</span></p></li><li><p><span style="color:rgb(255,76,0);">UED ：</span><span style="color:rgb(37,37,37);">user experience design ⽤户体验设计，指以⽤户为中⼼的⼀种设计⼿段，以⽤户需求为⽬标⽽进⾏的设计。</span></p></li><li><p><span style="color:rgb(255,76,0);">UCD：</span><span style="color:rgb(37,37,37);">User Centered Design 以⽤户为中⼼的设计，在设计过程中以⽤户体验为设计决策的中⼼， 强调⽤户优先的设计模式。</span></p></li><li><p><span style="color:rgb(255,76,0);">HMI：</span><span style="color:rgb(37,37,37);">Human Machine Interface  ⼈机界⾯，⼈机界⾯是系统和⽤户之间进⾏交互和信息交换的媒介， 它实现信息的内部形式与⼈类可以接受形式之间的转换。</span></p></li><li><p><span style="color:rgb(255,76,0);">CLI：</span><span style="color:rgb(37,37,37);">Command-line Interface 命令⾏界⾯，它通常不⽀持⿏标，⽤户通过键盘输⼊指令，计算机接收到指令后，予以执⾏。</span></p></li><li><p><span style="color:rgb(255,76,0);">UIMS：</span><span style="color:rgb(37,37,37);">User Interface Management System ⽤户界⾯管理系统</span></p></li><li><p><span style="color:rgb(255,76,0);">DSC：</span><span style="color:rgb(37,37,37);">Decision Support Center 决策⽀持中⼼，以决策⽀持⼩组为核⼼，为决策的全过程提供技术⽀持。</span></p></li><li><p><span style="color:rgb(255,76,0);">CHI：</span><span style="color:rgb(37,37,37);">computer-human interaction ⼈机交互。</span></p></li><li><p><span style="color:rgb(255,76,0);">HCI：</span><span style="color:rgb(37,37,37);">human computer interaction ⼈机交互。</span></p></li><li><p><span style="color:rgb(255,76,0);">OO：</span><span style="color:rgb(37,37,37);">Object Oriented ⾯向对象，是⼀种把⾯向对象的思想应⽤于软件开发过程中，指导开发活动的系统⽅法，是建⽴在 对象 概念基础上的⽅法学。</span></p></li><li><p><span style="color:rgb(255,76,0);">OOA：</span><span style="color:rgb(37,37,37);">Object Oriented Analysis ⾯向对象分析，是确定需求或者业务的⻆度，按照⾯向对象的思想来分析业务。</span></p></li><li><p><span style="color:rgb(255,76,0);">OOD：</span><span style="color:rgb(37,37,37);">Object Oriented Design ⾯向对象设计，是OO⽅法中⼀个中间过渡环节。其主要作⽤是对OOA分析的结果作进⼀步的规范化整理，以便能够被OOP直接接受。</span></p></li><li><p><span style="color:rgb(255,76,0);">OOP：</span><span style="color:rgb(37,37,37);">Object Oriented Programming ⾯向对象程序设计，OOP 的⼀条基本原则是计算机程序是由单个能够起到⼦程序作⽤的单元或对象组合⽽成。</span></p></li></ul><p style="text-align:justify;"><span style="color:rgb(37,37,37);">5 数据指标</span></p><ul><li><p><span style="color:rgb(0,122,170);">DAU：</span><span style="color:rgb(37,37,37);">Daily Active User  ⽇活跃⽤户数量，统计⼀⽇（统计⽇）之内，登录或使⽤了某个产品的⽤户数（去除重复登录的⽤户）。</span></p></li><li><p><span style="color:rgb(0,122,170);">WAU：</span><span style="color:rgb(37,37,37);">Weekly Active Users 七天内登陆过产品的⽤户数。统计⼀周之内（统计⽇）之内，登录或使⽤了某个产品的⽤户数（去除重复登录的⽤户）。</span></p></li><li><p><span style="color:rgb(0,122,170);">MAU：</span><span style="color:rgb(37,37,37);">Monthly Active User  ⽉活跃⽤户量，统计⼀⽉之内（统计⽇）之内，登录或使⽤了某个产品的⽤户数（去除重复登录的⽤户）。</span></p></li><li><p><span style="color:rgb(0,122,170);">DOU：</span><span style="color:rgb(37,37,37);">Day Old User ⽇⽼玩家⽤户，表示当天登录游戏的⽼玩家，指⾮当天新增的⽤户。</span></p></li><li><p><span style="color:rgb(0,122,170);">DNU：</span><span style="color:rgb(37,37,37);">Day New User⽇新增⽤户，表示当天的新增⽤户。</span></p></li><li><p><span style="color:rgb(0,122,170);">ACU：</span><span style="color:rgb(37,37,37);">Average concurrent users 平均同时在线⼈数。</span></p></li><li><p><span style="color:rgb(0,122,170);">PCU：</span><span style="color:rgb(37,37,37);">Peak  concurrent  users  最⾼同时在线⼈数。</span></p></li><li><p><span style="color:rgb(0,122,170);">UV：</span><span style="color:rgb(37,37,37);">Unique Visitor 唯⼀访问量，可以理解为⻚⾯被多少⼈看过。</span></p></li><li><p><span style="color:rgb(0,122,170);">PV：</span><span style="color:rgb(37,37,37);">Page View⻚⾯浏览量，可以理解为⻚⾯被⼈看过的总次数。⽤户每1次对⽹站中的每个⽹⻚访问均被记录1次。⽤户对同⼀⻚⾯的多次访问，访问量累计。</span></p></li><li><p><span style="color:rgb(0,122,170);">ARPU：</span><span style="color:rgb(37,37,37);">Average Revenue Per User 平均每活跃⽤户收益。</span></p></li><li><p><span style="color:rgb(0,122,170);">转化率：</span><span style="color:rgb(37,37,37);">⽤户点击⻚⾯后，产⽣被期望的特定⾏为的数量，占总量的⽐。</span></p></li><li><p><span style="color:rgb(0,122,170);">ARPPU：</span><span style="color:rgb(37,37,37);">Average Revenue Per Paying User 每付费⽤户平均收益。统计周期内，付费⽤户对产品产⽣的平均收⼊。</span></p></li><li><p><span style="color:rgb(0,122,170);">LTV：</span><span style="color:rgb(37,37,37);">Life Time Value LTV ⽣命周期价值，是产品从⽤户所有的互动中所得到的全部经济收益的总和。</span></p></li><li><p><span style="color:rgb(0,122,170);">CAC：</span><span style="color:rgb(37,37,37);">Customer Acquisition Cost ⽤户获取成本，获取⽤户所花费的成本。</span></p></li><li><p><span style="color:rgb(0,122,170);">KPI：</span><span style="color:rgb(37,37,37);">Key Performance Indicator 关键绩效指标，是企业绩效考核的⽅法之⼀。</span></p></li><li><p><span style="color:rgb(0,122,170);">ROI：</span><span style="color:rgb(37,37,37);">Return On Investment 投资回报率，投资回报率（ROI）=利润总额/投⼊成本总额×100%</span></p></li><li><p><span style="color:rgb(0,122,170);">CR转化率：</span><span style="color:rgb(37,37,37);">ConversionRate的缩写，是指访问某⼀⽹站访客中，转化的访客占全部访客的⽐例。</span></p></li><li><p><span style="color:rgb(0,122,170);">⼆跳率：</span><span style="color:rgb(37,37,37);">⽹站⻚⾯展开后，⽤户在⻚⾯上产⽣的⾸次点击被称为 ⼆跳 ，⼆跳的次数即为 ⼆跳量  ，⼆跳量与浏览量的⽐值称为⻚⾯的⼆跳率。</span></p></li><li><p><span style="color:rgb(0,122,170);">跳出率：</span><span style="color:rgb(37,37,37);">跳出率是指浏览了⼀个⻚⾯就离开的⽤户占⼀组⻚⾯或⼀个⻚⾯访问次数的百分⽐。</span></p></li><li><p><span style="color:rgb(0,122,170);">⼈均访问⻚⾯：</span><span
