职位这件事，贝索斯曾经是矛盾的。一方面，他十分钦佩同城老友比尔·盖茨，那种将大权逐渐释放给优秀手下的仙风道骨；另一方面，他又一度认为没有人比自己更适合指挥亚马逊这艘他本人一手打造的大船，担心接替者无法像自己一样，令投资者、分析师、政客和公众信服。</p><p>不过最终，贝索斯成功战胜了自己心中的迟疑。他终于发现，经过多年的沉淀，最合适的人选其实就在他身边。</p><p>新任亚马逊 CEO 是安迪·加西 <span class="text-remarks" label="备注">（Andy Jassy）</span>现任云计算业务 AWS 的 CEO，<strong>是一位24年亚马逊老兵，也是贝索斯的亲密战友和“影子顾问”。</strong>AWS 的成绩已无需盛赞。加西是云计算领域的最重要领袖之一，而云计算也已经成为这个时代最关键的商业引擎。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202107/06/083441634092.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="631" data-h="365" src="https://img.huxiucdn.com/article/content/202107/06/083441634092.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p label="图片备注" class="text-img-note">安迪·加西   图片来源：Twitter @ankithharathi<br></p><p>贝索斯之所以将交印的日子定在了7月5日，是因为这是1994年亚马逊正式注册公司同一天。“担任亚马逊 CEO 是一项很重，也很消磨人的责任。当你承担着这样的责任时，你将很难有精力做其它任何事情。”</p><p>卸任后，贝索斯终于可以有更多的精力，花在 Day 1 基金、贝索斯地球基金、《华盛顿邮报》，以及蓝色起源<span class="text-remarks" label="备注"> (Blue Origin) </span>公司和他热爱的航天事业上了。大约15天后，他将和弟弟一起乘坐自己的太空公司——蓝色起源制造的火箭一起去外太空环游。而这也是蓝色起源的第一次载人飞行。</p><h3 label="大标题" class="text-big-title">一、贝索斯和亚马逊的创立</h3><p>杰夫·贝索斯于1964年出生在新墨西哥州的阿尔伯克基，原名 Jeffery Preston Jorgensen。他的母亲于68年离婚再嫁，他跟了继父改姓Bezos。</p><p>贝索斯从小就展现出对科学和技术的兴趣，曾经自己组装了一套警报系统安装在房间。在佛罗里达州就读高中期间，贝索斯因成绩优秀曾获评毕业生代表 <span class="text-remarks" label="备注">（Valedictorian）</span>、《迈阿密先驱报》银骑士奖等。</p><p>他在自己的高中毕业生代表演讲中大胆宣称，未来有一天人类将会离开地球，前往太空，建立太空中的酒店和游乐园，以及可以容纳上百万人居住的环地球殖民地。在他的梦想中，人类最终将完全离开地球，开始在宇宙中生存繁衍的下一篇章。而我们留下的地球，将会成为一个巨大的“国家公园”。</p><p>尽管大学毕业后曾辗转多个行业和公司，并最终在电商行业功成名就，贝索斯一直没有忘记宇宙——这一少年时代的梦想。</p><p>当然，现实中的贝索斯更加现实。与其那么快就投入到殖民宇宙的理想当中，他更想要现在地球上扎下根来。这也是他从普林斯顿的物理系转投计算机系的原因之一。</p><p>本科毕业之后，贝索斯曾收到英特尔、贝尔实验室等知名公司的 Offer。不过，他还是选择了 Fitel——一家为投行开发通讯软件的创业公司，主管技术，做到了开发部门和客户服务部门的总监。</p><p>Fitel 后来黄了。他辗转去了现在德银旗下的 Banker's Trust，26岁做到了副总裁，到了当时刚创办没多久的量化对冲基金 D.E. Shaw，30岁时就升任高级副总裁，主盯当时新兴的互联网行业。</p><p>在这家公司，贝索斯遇到了自己的研究助理，后来的妻子麦肯齐·斯科特 <span class="text-remarks" label="备注">（MacKenzie Scott, 现已离婚）</span>。</p><p>Scott 在普林斯顿大学师从诺贝尔文学奖得主 Toni Morrison，也将写小说视为自己的人生方向。虽然被导师评价为最优秀的学生之一，但为了养家糊口，Scott 还是去了对冲基金公司打工。</p><p>通过妻子，贝索斯对图书行业有了更多的了解。而自己专精的互联网行业，让他也开始对创业这件事蠢蠢欲动。1997年，贝索斯偶然进入了一个网站。上面的数据令他惊呆：<strong>互联网的使用人数正在以每年2300%的速度爆发增长。</strong>第二天，他就坐上了一架前往洛杉矶的飞机，参加美国书商协会的年度大会，了解到了又一个令他欣喜的结果：<strong>主流书商在当时认定电子书会是未来趋势之一，并已经着手将旗下图书电子化备档。</strong></p><p>贝索斯的脑海中，已经有了一个未来的亚马逊的雏形。他的第一个正式创业项目将会专注于图书电子商务，从纸质书做起，并且时刻为了电子书的流行做好准备。</p><p>贝索斯选择了离职。1994年的美国国庆节，他和妻子飞到了得克萨斯州，从继父那里借走了一辆雪佛莱 SUV，直冲西雅图——虽然他也考虑过科罗拉多州、俄勒冈州等地，但他更希望自己的新公司坐落在西雅图，像软件时代的微软一样，在互联网时代获得巨大成功。</p><p>在这场横跨美国的短途旅行当中，因为贝索斯忙于制作商业计划书和沟通搬家、新公司注册等相关事务，麦肯齐主要开车。一开始他想用 Cadabra 这个著名的魔术咒语给公司命名，结果被律师取笑。后来换成了 Amazon，官方说法是世界最长河流的名字和未来的世界最大电商公司很匹配。其实还有一层原因，是当时的互联网各种目录都按照字母表排列，以 A 开头的公司更容易排在前面。</p><p>那一年7月5日，亚马逊公司正式注册。就靠图书电商这一项核心业务，亚马逊就在创办三年后的1997年5月成功上市。当时投资分析机构和媒体并未完全看清互联网的趋势，对亚马逊的营收能力和未来前景多有批判，但贝索斯坚称，互联网的增速极快，亚马逊将毫无疑问超越 Barnes & Noble 等传统书商。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202107/06/083445707949.jpg?imageView2/2/w/1000/format/jpg/interlace/1/q/85" data-w="204" data-h="212" src="https://img.huxiucdn.com/article/content/202107/06/083445707949.jpg?imageView2/2/w/1000/format/jpg/interlace/1/q/85" referrerpolicy="no-referrer"></p><p label="图片备注" class="text-img-note">最初的亚马逊Logo，上书：地球上最大的书店<br></p><p>上市之后的十年，亚马逊在前期过的并不好，一度濒临破产，而这很大程度是受到世纪之交的互联网泡沫的影响。即便如此，贝索斯仍然坚定地带领公司投身于在他看来属于互联网未来趋势的新业务，比如2002年推出的 Amazon Web Services。</p><p>AWS 的雏形是 Merchant.com, 这是亚马逊为其它想要转型电商的零售类公司开发的电商基础设施解决方案 <span class="text-remarks" label="备注">（E-commerce as a service）</span>。通过这项服务，零售公司开发和运营自营网店的成本降低，可以专注于提供产品和客户服务，让亚马逊去做所有的技术开发运维类“重活累活”。特别是服务器方面的运维，交给亚马逊的数据中心即可。</p><p>AWS 的早期 Logo 如下图：</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202107/06/083446990273.jpg?imageView2/2/w/1000/format/jpg/interlace/1/q/85" data-w="316" data-h="133" src="https://img.huxiucdn.com/article/content/202107/06/083446990273.jpg?imageView2/2/w/1000/format/jpg/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>左边的一个个小方块代表"Building Blocks"，意为 AWS 为商家提供便捷、模块化的技术服务。<strong>而我们熟悉的亚马逊 Logo 当中的“笑脸”元素，实际上并非笑脸，代表的其实是亚马逊成功“渡劫”互联网泡沫的低潮后，愈发优秀的业绩表现。</strong></p><p>今天的 AWS 已经成为云计算领域绝对的全球领导者，在整个亚马逊架构当中的角色格外重要。而 AWS 的强势发展，即将接任亚马逊 CEO 的安迪·加西功不可没。</p><h3 label="大标题" class="text-big-title">二、安迪·加西</h3><p>在 AWS 正式成为一块独立业务之前，亚马逊作为一家大科技公司已经遇到了大公司普遍存在的问题：招的人原来越多，能力越来越强，业务的发展却没有获得匹配的增长。</p><p>贝索斯要求加西去给出整改方案。加西发现亚马逊项目团队的大部分开发时间都花在了“造轮子”这件事上，为每一个项目开发需要的存储、计算和数据库，结果很多项目无法按期交付。</p><p><strong>在2003年，加西和一众高管到贝索斯家开会。会上他提议，亚马逊应该让技术基础架构标准化，节约开发时间。</strong>更何况，当时亚马逊做 Merchant.com 就是为了吸引非技术型公司来用自己的服务。如果亚马逊自己的团队都不用，又怎么能让客户信服呢？</p><p>贝索斯同意了。加西主张的这些东西，就成了后来的 AWS。</p><p>AWS 业务2002年正式成立的时候，是由1998年加入公司的 Colin Bryar 暂时领导。那段时间，亚马逊高层内部关于谁来负责这块新业务的讨论，也持续了很久。</p><p>贝索斯最初想让时任 CTO Allan Vermeulen 来领导 AWS，但 Vermeulen 属于那种“怪胎”型选手<span class="text-remarks" label="备注">（从他的领英资料也可以看得出来）</span>——技术厉害但不是很愿意直接操办业务，不愿意背负业绩压力。</p><p>事实上，AWS 团队的早期核心成员包括 Vermeulen、Bryar、Jeff Barr<span class="text-remarks" label="备注">（现任 AWS 首席布道师）</span>等都对背数字这件事天生不感冒。一家科技公司的成功需要这样的人，但只有真正的业务高手——像贝索斯，以及安迪·加西 <span class="text-remarks" label="备注">（Andy Jassy）</span>这样的人，才能真正率领公司。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202107/06/083447615471.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="305" data-h="202" src="https://img.huxiucdn.com/article/content/202107/06/083447615471.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p label="图片备注" class="text-img-note">Allan Vermeulen 的领英资料截图<br label="图片备注" class="text-img-note"></p><p>2003年夏天，加西终于站了出来。</p><p>作为这个新业务的早期员工，加西对 AWS 其实有很多系统化的想法。他认为互联网在未来一定会发展到一个高度先进和普及化的程度。届时，更多的人和机构也可以从互联网经济当中获益，即使他们对技术一窍不通，或者缺乏搭建像微软、亚马逊和谷歌等大公司的复杂技术架构的资源。而 AWS 应该为这些人提供标准化的基础架构服务。</p><p>在互联网经济发展的初期，人们希望赶上潮流，奈何软件开发的速度很慢，流程太长。缩短这个流程，降低软件开发和分发的门槛，将成为 AWS 收获增长的制胜法宝。加西的这套思路在亚马逊内部被称为“发明、发布、重新发明、重新发布和循环往复”<span class="text-remarks" label="备注">（Invent, Launch, Reinvent, Relaunch, Rinse, Repeat）</span>——<strong>和今天科技行业非常熟悉的“快速迭代”策略异曲同工。</strong></p><p>从某种角度来看，AWS 年度技术峰会 re:Invent 的命名，也是对当年重复造轮子这件事的一个“吐槽”。</p><p>加西和 AWS 早期核心团队一起构思了 AWS 服务的基础级核心部件应该由数据库、存储和计算这三大块构成。这些核心部件，就是后来人们熟悉的 AWS EC2<span class="text-remarks" label="备注">（虚拟机）</span>、S3<span class="text-remarks" label="备注">（云存储）</span>和 RDS<span class="text-remarks" label="备注">（分布式数据库）</span>。</p><p>在管理方面，加西组建了一支由57人组成的 AWS 早期团队，包括工程和商业背景的优秀人才，并且当时他们当中很多人都来自公司外部。就像“PayPal 黑帮”那样，AWS 的早期员工后来也创办了一些在企业级领域较为成功的公司，比如 Twilio、Tableau和Pacvue 等。</p><p>在美国科技行业的一些知名大公司的高层内部存在某种非常特殊的核心高管培养暨未来 CEO 训练制度。比如英特尔的 CEO 通常曾担任前任或前前任 CEO 的 TA <span class="text-remarks" label="备注">（Technical Advisor）</span>。<strong>而在亚马逊，我们从加西的身上看到一个类似的机制首次开花结果——这个过程花了27年。</strong></p><p>加西1997年加入公司，在执掌 AWS 之前曾经做过贝索斯的“办公室主任”。这不是一个纯粹的秘书性职务，它带有的意义更加丰富和远大。</p><p>正因为加西在 AWS 上的远见卓识，在相当长的一段时间里，甚至到转任 AWS 负责人之后，他都会陪同贝索斯出席每一次重要会议、做笔记和学习，并且参与一些公司重大事项的决策。</p><p><strong>加西成为了贝索斯的“影子”。这个只留给贝索斯最信赖员工的隐藏职位，加西是第一人。</strong></p><p><strong>加西成为了今天我们所理解的云计算理念的先行者，在过去的20年里带领 AWS 成长成为亚马逊旗下发展最快的业务。</strong></p><p>2010年，亚马逊电商业务全部迁移到 AWS 架构上。2012年，Netflix CEO 里德·哈斯廷斯宣布 Netflix 所有架构将会迁移到 AWS 上。同年，分析师预计 AWS 的年营收或将超过15亿美元。</p><p>2015年，亚马逊宣布 AWS 实现盈利。当年第一季度收入15.7亿美元，营业净利润2.65亿美元，贝索斯预测 AWS 的年收入在50亿美元左右。同年第三季度，AWS 的营业净利润就翻了一番，达到5.21亿美元。年底，亚马逊的销售额达到了96亿美元。Gartner 预测 AWS 的业务量是排在其后的14个云计算服务商总和的十倍。</p><p><strong>2016年第一季度，亚马逊首次宣布 AWS 的收入达到25.7亿美元，净利润率6亿美元，超越了北美电商零售业务，成为了整个亚马逊旗下利润率最高的业务板块。加西也从亚马逊高级副总裁的职位升任 AWS CEO。</strong></p><p>虽然之后 AWS 的增长率有所下降，但仍然连续多年保持了超过30%的业绩增速，并且直至今天仍然稳坐云计算市场份额第一的宝座。</p><p>虽然不如贝索斯知名，加西仍然毫无疑问有着科技圈大佬的身份。此前微软和Uber 曾经将加西作为接任 CEO 的人选。</p><p>今年2月，贝索斯发布致全体员工信，正式宣布了自己的退休计划和加西接棒 CEO
