href="https://mp.weixin.qq.com/s/GGYsURn-yCbCTNuQMlRxTg" target="_blank" rel="nofollow" style="text-decoration: none;"><span class="text-remarks">CSDN（ID：CSDNnews）</span></a><span class="text-remarks">，作者：郑丽媛，原文标题：《微软和谷歌又要“打”起来了！网友：太好了》，题图来自：pixabay</span></p><p>停战 6 年后，微软和谷歌又要“打”起来了。</p><p>作为在搜索引擎、浏览器等许多领域均有涉及的两大科技巨头，微软和谷歌多年来一直纷争不断：今天你起诉我侵犯专利，明天我公布你软件中的漏洞，实在是一对名副其实的“冤家”。</p><p>在斗了许多年后，2015 年这对“冤家”终于都觉得累了，双方达成协议全面和解，终止了近 20 起专利侵权诉讼。</p><p>不过，近日据知情人士透露，<strong>这份协议已于今年 4 月中旬到期，而微软和谷歌均已决定不再续签</strong>，这也就意味着他们都同意再次开始互相起诉的日子。</p><h3 label="大标题" class="text-big-title">一、十几年恩怨不断</h3><p>纵观微软和谷歌这十几年，不论是达成协议前还是达成协议后，双方恩怨始终不断。</p><ul class=" list-paddingleft-2" style="list-style-type: disc;"><li><p><strong>YouTube 事件</strong></p></li></ul><p>2013 年 5 月，微软更新了 Windows Phone 版的 YouTube 应用，但这次更新却引起了谷歌<span class="text-remarks" label="备注">（YouTube 的东家）</span>的极大不满：<strong>微软未经谷歌同意就发布的这版 YouTube 应用，支持用户下载视频，还屏蔽了广告。</strong></p><p>这让谷歌怒不可遏，要求微软立即下架并停止支持该应用：“你们禁止广告并允许视频下载，这是断送了内容创作者可持续的收入来源，而且对 YouTube 繁荣的内容生态圈也造成了严重伤害。”</p><p>随后微软更新了 WP 版的 YouTube 应用，不过更新之后的版本仅是不能下载视频，广告依旧还被屏蔽，但微软表示愿意和谷歌合作开发官方应用以解决问题，而谷歌也同意了这个邀请，彼此难得迎来了近 3 个月的和平期。</p><p><strong>但 3 个月后，战火再起</strong>：微软发布了符合谷歌要求的 WP 版 YouTube 应用，不仅有广告，还添加了上传视频等功能，但谷歌却以“微软没有升级浏览器以支持 YouTube 的全功能体验，并且重新发布了违反我们服务条款的 YouTube 应用”为理由，单方面禁用了 WP 版的 YouTube 应用。</p><p>微软这下忍不了了，特地写了一篇名为《Google 开放性的局限》的长文斥责谷歌无理禁用新 WP 版 YouTube 应用。</p><ul class=" list-paddingleft-2" style="list-style-type: disc;"><li><p><strong>找漏洞事件</strong></p></li></ul><p>要说微软和谷歌之间的矛盾，找漏洞绝对算得上是代表之一。</p><p>2010 年，一名谷歌工程师向微软报告了一个 Windows 漏洞，但他只给微软 5 天的时间去修补和发布补丁<span class="text-remarks" label="备注">（在安全界，一般发现这种 0 day 漏洞的公司或专家都会给软件开发商预留 60 天的反应期）</span>。结果可想而知，微软根本无法在 5 天之内完成修复，而这位工程师不仅公开了那个漏洞的有关信息，还生怕黑客们不知道要怎么用，例举了一组攻击代码手把手教程，把微软气得够呛。</p><p>到了 2015 年，谷歌又故伎重施。2014 年 9 月谷歌发现了 Win 8 中的一个安全漏洞并告知微软，而这次谷歌“宽容”地给予了微软 90 天的反应期。可到了 2015 年 1 月 2 日，微软依旧没有发布这个漏洞的相关补丁，谷歌就毫不顾忌地公开了漏洞信息，并与之前一样将利用漏洞入侵系统的方法公之于众。</p><p>微软安全中心资深总监 Chris Betz 为此发表了一篇长文博客《号召更好的漏洞协同披露》控诉谷歌：</p><blockquote class="js_blockquote_wrap" data-type="2" data-url data-author-name data-content-utf8-length="96" data-source-title><p>我们曾要求谷歌与我们合作，等到 1 月 13 日我们发布补丁时，再公布漏洞细节，以保护客户。但谷歌并未同意甚至公布漏洞，与其说它是不知变通，不如说是抱着一种幸灾乐祸的心态，让用户为这一切买单。</p></blockquote><p>这还没完，2016 年这种事又发生了一次。谷歌再次发现了微软 Windows 系统中的一个重大漏洞，但从告知到公开，谷歌只给了微软 10 天的时间。10 天期限一到，谷歌就在其安全博客中披露了该漏洞，理由是：已经有恶意软件在利用这个漏洞攻击 Chrome 浏览器了，因此谷歌只能尽快公布针对该漏洞的 Chrome 补丁。</p><p>但这个做法明显就不顾微软的“死活”了，因此微软对谷歌强烈谴责：“谷歌今天公布的信息很可能将用户置于危险的境地！”</p><p>不过俗话说得好，天道好轮回，苍天饶过谁，微软自然不愿单方面被谷歌一次次“虐”。2017 年微软也找到了谷歌的一个远程 Chrome 漏洞，又写了一篇博文大致讲述了 Chrome 中的远程代码执行问题，并批评谷歌仅在 Chrome 测试版本中修复漏洞后，便在 Github 上提供了修复程序的源代码，这就给了攻击者线索去攻击尚未修复漏洞的 Chrome 稳定版和公测版。</p><ul class=" list-paddingleft-2" style="list-style-type: disc;"><li><p><strong>浏览器 Battle</strong></p></li></ul><p>去年，基于 Chromium 的新版 Edge 正式上线，但当用户使用新的 Edge 浏览器尝试从 Chrome Web Store 添加扩展程序时，会弹出一条消息：“谷歌建议您切换到 Chrome 以便安全地使用扩展程序”。</p><p>此外，如果用微软的 Bing 搜索“ Chrome Web Store”，出现的第一条也并非是“Chrome Web Store”网站推荐，而是微软自家的 Microsoft Edge 浏览器下载推荐。</p><p>以上这三种矛盾只是微软和谷歌无数恩怨之中的一角，2015 年双方达成的互不侵犯协议只是有效减少了彼此间的诉讼官司，即使彼此间有合作，但矛盾也不可忽视。</p><h3 label="大标题" class="text-big-title">二、“互不侵犯”协议到期，不再续约</h3><p>据外媒 arsTECHNICA 报道，此次不再续签这个“互不侵犯”协议于微软而言大致有两个原因。</p><p>首先，微软从去年开始就在抱怨谷歌广告管理工具 Search Ads 360 对支持 Bing 的一些新广告格式方面拖拖拉拉，并就这个问题联系了英国和美国监管机构。此外，谷歌还通过自动拍卖的方式为其搜索引擎上的广告位更快地提供竞标细节，但 Bing 上却无法使用这种工具，导致广告商们发现相比 Bing，在谷歌上购买广告更方便容易。</p><p>微软总裁 Brad Smith 对此表示，<strong>由于谷歌不愿意与 Bing 合作，每年给公司带来数亿美元的广告收入损失。</strong></p><p>其次，虽然最近监管机构一直在以谷歌、苹果等大型科技公司为目标进行反垄断审查，但微软至今都相对安然无恙，甚至前一阵还成为了美国第二个市值破 2 万亿美元大关的上市公司。其中原因可能在于虽然微软近年来在云计算和游戏等方面取得不错成绩，但在这些领域微软都不占据绝对的主导地位，所以没有引起监管机构的注意。</p><p>因此，据知情人士透露，微软 CEO Satya Nadella 和微软总裁 Brad Smith 都急于向监管机构证明微软没有与谷歌一样做出那些被质疑的垄断行为，希望能与被审查的目标保持距离。而协议结束后，微软也就可以对谷歌提起各种诉讼，为谷歌的反垄断案提供更多证据。</p><p>与此同时，谷歌也对近来微软的攻击越来越失望，6 年的和平协议终于就此终止了。</p><h3 label="大标题" class="text-big-title">三、网友：太好了</h3><p>对于微软和谷歌可能即将重返 6 年前那般相互诉讼的日子，许多网友居然还挺“支持”：</p><p>网友 @Old_Fogie_Late_Bloomer：“这太好了，最近他们两个过于亲密的关系让我很不舒服。”</p><p>网友 @Happysin：“任何迫使监管机构密切关注这些巨头公司正在发生的事情的举动都是一件好事。”</p><p>网友 @jdvorak：“这个互不侵犯协议早在几年前就应该受到竞争主管部门的严格审查。人们只想知道他们在这种情况下为了减少竞争并对客户造成伤害达成了哪些交易。它让人想起硅谷公司遵守的关于相互挖角员工的竞业禁止协议，从而限制了员工的薪酬。”</p><p>也有人因此联想到这是否与微软发布 Windows 11 时提到可以在亚马逊 Appstore 下载安卓 App 的举动有关：</p><p>“这就是 Windows 11 要使用亚马逊应用商店的原因吗？”</p><p>“这很难不让人猜测到微软的想法：我跟谷歌的友谊结束了，现在亚马逊是我最好的朋友。”</p><p><span class="text-remarks"
