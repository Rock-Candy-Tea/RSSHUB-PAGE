<p><span class="text-remarks" label="备注">本文来自微信公众号：</span><a href="https://mp.weixin.qq.com/s/PqU2bJ3V0sv9g2MpL5vxLg" target="_blank" rel="nofollow" style="text-decoration: none;"><span class="text-remarks">新智元（ID：AI_era）</span></a><span class="text-remarks">，编辑：David、桃子，原文标题：《GitHub换帅，开源大佬Nat Friedman辞任CEO！发告别信感恩“这三年”》，头图来自：视觉中国</span></p><p>GitLab上市后估值165亿美元，GitHub急了！周三，微软宣布GitHub首席执行官Nat Friedman卸任，他的职位将由GitHub首席产品官Thomas Dohmke接任。</p><p>Friedman 在 GitHub 内部信中表示，“考虑到我们已经取得的所有成就，以及我在微软的5年多辉煌时光，我决定是时候回到我的创业根源了。”</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162304224896.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="1000" data-h="525" src="https://img.huxiucdn.com/article/content/202111/04/162304224896.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>“我将继续下一次冒险：支持、指导以及投资那些正在用技术创造未来并应对当今一些最大机遇的创始人和开发者。”<br></p><p>Dohmke 将于11月15日上任，并向即将升任微软开发部总裁Julia Liuson汇报工作。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162306247014.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="1000" data-h="195" src="https://img.huxiucdn.com/article/content/202111/04/162306247014.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><h3 label="大标题" class="text-big-title">开源社区名人，和微软渊源近20年</h3><p>实际上，<strong>在担任微软旗下的Github的CEO之前，Nat Friedman 早已经是开源社区的重要人物。</strong>他和微软的渊源最早也可以追溯到近20年前。</p><p>早在1996 年，Friedman 在MIT读大一时，就在 LinuxNet 上与著名墨西哥籍开源软件开发者 Miguel de Icaza 结识，并成为朋友。1999年，二人在共同创办了Helix Code，这家公司之后改名为Ximian。</p><p>2000年，微软发布.NET框架<span class="text-remarks" label="备注">（.NET Framework）</span>，受该框架启发，2003年，Miguel开启了Mono项目。这是一个免费开源项目，目标是创建一系列符合ECMA标准的.NET通用平台工具。</p><p>与微软的.NET不同，Mono项目不仅可以运行于Windows系统上，还可以运行于Linux、FreeBSD、Unix、OS X和Solaris，甚至一些游戏平台上。这算是和微软打上擂台了。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162309521268.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="531" data-h="605" src="https://img.huxiucdn.com/article/content/202111/04/162309521268.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>.NET框架是专有平台，Mono是通用平台，你封闭收费，我免费开源。</p><p>2003年，多平台网络操作系统服务商Novell收购了Ximian，Friedman和Miguel随之进入Novell。在2010年 1 月之前，Friedman一直担任开源业务的首席技术官和战略官。</p><p>2011年4月，Novell再次被收购。Friedman和Miguel在收购敲定后接受了Arstechnica采访，<strong>Friedman对此次收购表示遗憾。</strong>当时他在网友互动时表示，“Novell被收购了，他们在Linux上仍然很强，但说到在全栈开源策略上的机会则是另一回事了，太遗憾了”。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162310290991.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="983" data-h="158" src="https://img.huxiucdn.com/article/content/202111/04/162310290991.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>果然，被收购后，新公司开启大规模裁员，Miguel等百余名员工遭解雇。<br></p><p>随后，Friedman和Miguel共同创办了Xamarin公司，由Friedman担任CEO，继续Mono项目的开发工作，旨在提供微软.NET软件栈的免费实现。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162310849847.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="805" data-h="362" src="https://img.huxiucdn.com/article/content/202111/04/162310849847.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>同时，微软也在慢慢改变着对开源软件和开源社区的敌视态度。拥抱开源，既能获取大量潜在客户和开发者资源，又能收获不少口碑。开源绝非这家商业软件巨头的死敌。那么，<strong>化敌为友不好吗？</strong></p><p>2014年，微软宣布.NET框架开源，并可以跨多平台使用。Miguel加入.NET基金会，成为基金会理事之一。2016年， Xamarin被微软收购，Friedman出任为开发者服务的微软副总裁。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162312406028.jpg?imageView2/2/w/1000/format/jpg/interlace/1/q/85" data-w="1000" data-h="564" src="https://img.huxiucdn.com/article/content/202111/04/162312406028.jpg?imageView2/2/w/1000/format/jpg/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>据公开消息，在被微软收购时，Xamarin 已经成长为通用开源平台领域具有全球影响力的企业。在 120 个国家/地区拥有 15000 多个客户，其中包括 100 多家“世界500强”企业，有超过 130 万独立开发人员在使用Xamarin的产品。</p><p>三年前，在微软以75亿美元收购GitHub后，Friedman接下了CEO一职。<strong>在他任职期间，带领GitHub团队推出许多新的功能并增强了现有特性。</strong>就在6月份，Github和OpenAI联合发布了AI代码生成工具——Copilot，简直就是编程神器。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162312742726.gif?imageView2/2/w/1000/format/gif/interlace/1/q/85" data-w="984" data-h="512" src="https://img.huxiucdn.com/article/content/202111/04/162312742726.gif?imageView2/2/w/1000/format/gif/interlace/1/q/85" referrerpolicy="no-referrer"></p><h3 label="大标题" class="text-big-title">“卖身”的这3年，GitHub还好吗？<br></h3><p>2018 年，GitHub卖身微软，估值75亿美元，现在的它还好吗？根据GitHub 2020年度最新报告显示，北美、亚洲、以及欧洲地区用户活跃度最高，开发者数量超过5600万。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162323269692.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="1000" data-h="656" src="https://img.huxiucdn.com/article/content/202111/04/162323269692.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p label="图片备注" class="text-img-note">（图源：Linux 中国）<br></p><p>用户总量变化的同时，<strong>开发者的比重在所有用户的所占比重在下降，</strong>从 2016 年的 60% 逐年递减至 2020 年的 54%。相反，<strong>学生成为用户中的主力。</strong>教育相关的学生用户、教师用户的比重在不断的攀升，到 2020 年，教育相关用户的比例已达 23%，几乎达到了 Github 用户比重的四分之一。</p><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162325478384.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="938" data-h="552" src="https://img.huxiucdn.com/article/content/202111/04/162325478384.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>微软透露，目前有超过7300万的开发者在使用 GitHub，最初收购时收购时仅有2800万用户。</p><p>就在几周前 GitHub 的主要竞争对手 GitLab 在纳斯达克上市，<strong>上市后它的估值达到 165 亿美元。</strong>GitLab 在首次公开发行<span class="text-remarks" label="备注">（IPO）</span>的招股说明书中表示，“微软收购 GitHub 后，这是它的主要竞争对手”。目前，GitLab 估计自己拥有3000万注册用户。</p><p>2007年10月1日，GitHub最初由Chris Wanstrath、PJ Hyett和Tom Preston-Werner使用Ruby on Rails编写而成。网站并于2008年2月以beta版本开始上线，4月份正式上线。GitHub被收购后，曾经的首席执行官Chris Wanstrath任职微软技术院士。</p><p>在 GitHub 将被微软收购消息放出后，很多程序员在很多社交平台上表达出对微软未来操控 GitHub、盗取用户代码数据，GitHub 将无法像现在一样独立自主发展的担心。当时，微软首席执行官Satya Nadella表示，在这次收购中看到了三个明显的机会，并提到GitHub在收购后保持独立运营。</p><p><strong>此次换帅Thomas Dohmke后，GitHub依旧保持独立运营。</strong>Dohmke在博客中表示，未来将和大家一起携手同建GitHub未来。</p><h3 label="大标题" class="text-big-title">告别信<br></h3><p class="img-center-box"><img class="lazyImg" _src="https://img.huxiucdn.com/article/content/202111/04/162332224721.png?imageView2/2/w/1000/format/png/interlace/1/q/85" data-w="1000" data-h="658" src="https://img.huxiucdn.com/article/content/202111/04/162332224721.png?imageView2/2/w/1000/format/png/interlace/1/q/85" referrerpolicy="no-referrer"></p><p>当地时间11月3日，Friedman在Github社区发布了一封告别信，宣布辞去CEO一职，开始“继续人生的下一次冒险”。公开信主要内容如下：</p><p>我将继续我的下一次冒险，现任首席产品官Thomas Dohmke将成为 GitHub 的下一任 CEO。我将成为名誉主席，这实现了我拥有拉丁文头衔的毕生抱负。我衷心感谢每一位 Hubber 和每一位让 GitHub 成就今天的开发人员。</p><p>谢谢你，GitHub。</p><p>每天，全球有数以百万计的开发人员来到 GitHub，将他们的想法转化为代码。这些开发人员跨行业、跨时区，有着不同的技能水平，使用不同的工具集。只需按几下键盘，就可能诞生下一个大问题的解决方案。</p><p>在 GitHub，我们的愿景是成为所有开发人员的家，在过去的三年里，我也有幸将 GitHub 当做自己的家。</p><p>我一生的大部分时间都在构建开源项目，在 GitHub，我能够与充满激情、才华横溢的 Hubbers 团队以及出色的社区并肩工作。作为 GitHub 的 CEO，为这个全球开发者社区服务是我一生的荣幸。</p><p>我为我们共同取得的成就感到无比自豪。GitHub 已经从一个存储和共享代码资源库，变成了一个完整的开发者平台。</p><p>今天，我们可以自豪地说，超过 7300 万开发人员正在 GitHub 上构建软件的未来，我们的客户中涵盖了 84% 的全球财富100强企业，自收购以来，业务增长速度一年比一年快。</p><p>考虑到我们已经取得的成就，以及我过去在微软度过的五年多美好时光，我决定是时候回归我的创业根源了。</p><p>我认为，构建软件要能够构建未来。我喜欢与开发新工具和新项目、解决棘手问题、用代码创造奇迹的开发人员一起工作，并向他们学习。下一步我的目标是为创业者和开发人员提供支持、建议和投资。他们正在用技术创造未来，应对当今一些最大的机遇。</p><p>所以，我很高兴地宣布，自 11 月 15 日起，GitHub 的首席产品官 Thomas Dohmke (@ashtom) 将成为首席执行官，而我将改任名誉主席。</p><p>Thomas 为开发人员服务了数十年，在促成GitHub 收购交易的成功方面发挥了关键作用。他为保险行业、汽车、移动应用程序等行业构建软件。</p><p>在 GitHub，他领导了为开发人员免费提供私有存储库的工作，解决了数百个客户痛点。由他制定的路线图，确保了我们的客户通过一个可以在未来几年为提供支持的平台获得他们需要的功能。在他的领导下，我对 GitHub 的未来感到无比兴奋。</p><p>我对过去三年心怀感恩，感恩你们所有人。</p><p>我们构建软件的方式正处于一个转折点。开发本身正在向云迁移。AI 正在彻底改变我们的编码方式。我对 GitHub 团队和 Thomas迎接未来的每一个新挑战和机遇充满信心。感谢你们每天为工作带来的能量和创造力，让 GitHub 成为 GitHub。</p><p><span class="text-remarks" label="备注">参考资料</span></p><p><span class="text-remarks"
