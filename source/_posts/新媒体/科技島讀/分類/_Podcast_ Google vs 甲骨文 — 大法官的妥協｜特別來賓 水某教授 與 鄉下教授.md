<h2>本週 podcast</h2>
<p>纏訟十年的 Google 與甲骨文案終於塵埃落定。美國最高法院以 6：2 的多數決，認定 Google 抄襲 Java 的 API 為合理使用，因此不需負擔賠償。軟體社群也普遍額手稱慶。</p>
<p>然而判決本身卻讓法律人士大失所望。原本眾所期待大法官們能釐清軟體的著作權地位，但多數的大法官不是迴避問題，就是對關鍵的論點輕輕帶過。也顯示要引導與維護健全的軟體生態有多麼的不容易。本集解釋此一判決的重要性、判決的內容，以及對軟體業與著作權本身的影響。</p>
<p>特別感謝 podcast 節目《<a href="https://player.soundon.fm/p/d36d9b85-0640-4a80-8737-d61188fc55c7" data-wpel-link="external" target="_blank" rel="external noopener noreferrer">法學叫獸的 x 話群組</a>》的「水某教授」與「鄉下教授」加入對談。《法學叫獸的 x 話群組》由四位正牌法律系教授主講。他們用輕鬆有趣的方式討論法律、學術與輕鬆的話題。</p>

<h2>議題</h2>
<ul>
<li>軟體業者為何關注 Google vs. Oracle</li>
<li>美國最高法院的多數派與少數之爭</li>
<li>著作權保護「表達」不保護「概念」</li>
<li>為何軟體受到著作權保護</li>
<li>API 的功能性與執行碼不同</li>
<li>最高法院迴避的 API 是否受著作權保護的問題</li>
<li>合理使用（fair use）限制著作權人的權利範圍</li>
<li>谷阿莫「5 分鐘看完電影」為何不是合理使用</li>
<li>最高法院輕輕帶過 Google 的使用是否有轉化（transformative）效果</li>
<li>為何軟體人士叫好，法律人士失望</li>
<li>軟體開發者接下來可以抄襲 API 了嗎</li>
<li>企業未來要如何保護 API</li>
</ul>
