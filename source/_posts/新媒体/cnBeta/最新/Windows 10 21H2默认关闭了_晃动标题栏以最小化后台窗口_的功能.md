早在 Windows 7 时代，微软就为自家桌面操作系统引入了 Title Bar Window Shake 功能（又名 Aero Shake）。<strong>顾名思义，这项功能允许用户用鼠标指针拖住当前应用程序的标题栏，然后通过左右晃动多次的方式，将其它窗口都最小化（重复操作可恢复）。</strong>然而 Windows Latest 刚刚发现，微软已经在 Windows 10 21H2 编译版本中默认禁用了该选项。<br>
 <p><a href="https://static.cnbetacdn.com/article/2021/0424/0a30eefbfd0f905.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0424/0a30eefbfd0f905.jpg" referrerpolicy="no-referrer"></a></p><p>在打开了一大堆应用程序窗口的情况下，Aero Shake 能够瞬间还你一个清爽的桌面。但或许是考虑到了某些可能意外触发的情况，<a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 10 21H2 还是默认关闭了该选项。</p><p>今年早些时候，Windows Latest 已经报道过<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>计划在 Windows 10“Sun Valley”更新中砍掉 Aero Shake 功能。但最新测试版本表明，该公司还是听取了大家的合理反馈。</p><p>有需要的朋友，可移步至“设置”（Settings）应用，并将 Aero Shake（Title Bar Window Shake）功能重新打开。</p><p>熟悉注册表编辑器（regedit）的朋友，亦可直接用 CMD 命令行来召唤。不过后续涉及手动创建“DisallowShaking”的 DWORD 项目，并指定数值为“0”。</p><p><a href="https://static.cnbetacdn.com/article/2021/0424/1744b9d28e3ba67.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0424/1744b9d28e3ba67.jpg" referrerpolicy="no-referrer"></a></p><p>此外正如 Windows Latest 在 2 月份报道的那样，Windows 10“Sun Valley”更新似乎将包括新的的多任务控件。</p><p>比如用户可将应用程序窗口拖动到屏幕的各个角落，然后系统会自动排列和调整它们的大小（内部称作“基于快照的任务组”）。显然，这项功能对于触屏设备的用户也相当友好。</p><p>最后，我们被告知微软正在为 Windows 10“Sun Valley”开发一种新的触控和手势体验，预计可在下月分享更多细节。</p>
