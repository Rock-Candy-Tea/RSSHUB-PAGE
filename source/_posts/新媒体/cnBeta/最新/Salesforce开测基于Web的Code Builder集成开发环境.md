大约两年前（2020 年 6 月），Salesforce 发布了 Code Builder 。<strong>作为一个以 GitHub Codespaces 为模型的基于 Web 的集成开发环境（IDE），我们终于在近日迎来了它的 beta 测试版本。</strong>官方将其描述为“专为 Salesforce 而优化的开发环境”，可知在亚马逊 Web 服务（AWS）的加持下，Code Builder 允许开发者在组织内的浏览器中轻松启动。<br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0715/aa052c8190f6698.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/aa052c8190f6698.png" alt="1.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（来自：Salesforce <a href="https://developer.salesforce.com/blogs/2022/07/build-from-anywhere-with-salesforce-code-builder-beta" target="_self">Developer's Blog</a>）</p><p>除了代码完成、搜索、重构等功能，Code Builder 还附带了对 Salesforce 框架的支持，预装了工具、并且包括了 Salesforce 扩展。</p><p><a href="https://static.cnbetacdn.com/article/2022/0715/91ad4d554b766a2.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/91ad4d554b766a2.png" alt="2.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">访问 Salesforce 特定命令</p><p>Code Builder 的另一优势，就是能够与 Salesforce 开发的编程语言（包括 Apex 和 Lightning Web 组件）完美配合，为所有这些语言提供自动完成功能。</p><p><a href="https://static.cnbetacdn.com/article/2022/0715/18121f5da9e4fe7.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/18121f5da9e4fe7.png" alt="3.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">连接到多个 Salesforce 组织</p><p>Apex 常用于在 Salesforce 的 CRM 平台之上构建“软件即服务”（SaaS）类应用程序，而 Lightning Web 组件则可使用 HTML 和 JavaScript 构建自定义 Web 元素。</p><p><a href="https://static.cnbetacdn.com/article/2022/0715/ca9b1e34bb59b4d.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/ca9b1e34bb59b4d.png" alt="4.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">使用浏览器探索组织元数据</p><p>在该 IDE 的帮助下，开发者能够测试并部署对 Apex 类的更改和 Lightning Web 组件。或构建并运行 Salesforce 对象查询语言查询，以搜索组织的 Salesforce 数据和获取特定信息。</p><p><a href="https://static.cnbetacdn.com/article/2022/0715/3a088fa37b61f4e.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/3a088fa37b61f4e.png" alt="5.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">创建并部署 Apex 和 LWC</p><p>如大家所料，Code Builder 还内置了与 GitHub 等版本控制系统的集成。Salesforce 的首席开发倡导者 Mohith Shrivastava 在博客文章中提到：</p><blockquote><p>Code Builder 带有与 <a data-link="1" href="https://microsoft.pvxt.net/zaZYr" target="_blank">Visual Studio</a> Code 的 Salesforce Extensions 包中相同的扩展集，且 UI 观感类似于 VS Code 。</p><p>如此一来，VS Code 的老用户能够轻松上手并使用 Code Builder 工具，比如 Salesforce Extensions 扩展包。</p></blockquote><p><a href="https://static.cnbetacdn.com/article/2022/0715/4bf749782f551bb.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/4bf749782f551bb.png" alt="6.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">使用 SOQL Builder 编写和执行 SOQL 查询</p><p>需要注意的是，在 beta 测试阶段，Code Builder 有 20 小时的使用时长限制、且最长为 30 天。如需变更，必须将更改部署到组织、或提交到源码控制。</p><p><a href="https://static.cnbetacdn.com/article/2022/0715/87fc3a9be7fe157.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0715/87fc3a9be7fe157.png" alt="7.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">使用代码生成器终端运行 Salesforce CLI 命令</p><p>此外 Salesforce 也不承诺不会在没有预警的情况下移除 Code Builder 开发环境，并表示会在全面上市前彻底清理。</p>
