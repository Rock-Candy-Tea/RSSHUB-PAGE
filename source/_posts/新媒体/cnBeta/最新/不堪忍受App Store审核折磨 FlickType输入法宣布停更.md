曾多次公开批评山寨应用的 FlickType 创始人 Kosta Eleftheriou 于今日宣布：<strong>在经受了苹果 App Store 审核团队带来的长久折磨之后，他已决定停更 iPhone 端的 FlickType Watch Keyboard 输入法应用。</strong>上周，开发者提交了未添加任何新功能、仅包含错误修复和相关改进的一个更新，但还是遭到了苹果的拒绝。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0817/b1bc3be8dba3210.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p>苹果给出的理由是“如果不授予完全的访问权限，这款输入法就无法运行”。然而 Eleftheriou 表示，该问题早在 3 年前就已经得到了解决。</p><p>期间开发团队还希望以 TestFlight 测试版的形式保留输入法扩展，可惜同样遭到了苹果的拒绝。</p><p><img src="https://static.cnbetacdn.com/article/2021/0817/97722b58843b96d.png" alt="2.png" referrerpolicy="no-referrer"></p><p>作为 Apple Watch 上深受欢迎的一款输入法，FlickType 宣称其效率是标准键盘的三倍。</p><p>除了基于点击或滑动的输入方式，它还专为盲人和视力障碍人士提供了一款增强组件，具有大按键、高对比度、输入反馈、以及 VoiceOver 交互。</p><p><img src="https://static.cnbetacdn.com/article/2021/0817/217e22ae4dd2be5.png" alt="3.png" referrerpolicy="no-referrer"></p><p>Kosta Eleftheriou 曾于今年 3 月向苹果发起诉讼，理由是该公司未能让开发者摆脱山寨应用程序的困扰。</p><p>其声称该公司第三方输入法 API 相当可怕，自 2014 年以来，这个 API 长期遭遇缺陷、不一致、不断变化和体验破坏等问题。</p><p>最后，他希望有朝一日能够以纯正的方式，让 FlickType 输入法重返 <a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fiphone%2F" target="_blank">iPhone</a> 和 Apple Watch 平台。</p><p>同时呼吁参照相关立法，要求苹果公司允许第三方应用商店和侧载等实现方案。</p>
