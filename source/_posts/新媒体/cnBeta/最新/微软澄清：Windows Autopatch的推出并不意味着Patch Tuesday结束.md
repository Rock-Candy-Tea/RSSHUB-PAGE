<strong>针对部分 Windows 10/11 企业客户，微软在今年 4 月推出了自动更新服务 Windows Autopatch。</strong>微软发言人当时表示，该服务旨在让管理员更轻松地管理和部署 Windows 和 Microsoft 365 应用程序的更新，希望在传统 Patch Tuesday活动中剥离出针对企业的 Patch Tuesday。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0617/09a159f14184bc4.webp" alt="7vzh13g4.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">Windows Autopatch 目前处于公共预览阶段，将于 2022 年 7 月上旬向拥有 Windows Enterprise E3 或更高版本许可证的 Microsoft 客户普遍提供。管理员将能够继续使用他们当前的工具和流程来部署更新，或者他们可以让 Autopatch 服务为他们做这件事。</p><p style="text-align: left;">不过在本月的Patch Tuesday（6 月 14 日）活动后，几家主要的网络安全公司和著名的安全新闻出版物均表示这是企业用户收到的最后一个Patch Tuesday。媒体将其描述为“有史以来最后一个 Patch Tuesday”、“Patch Tuesday 的结束”和“Patch Tuesday 时代的结束”。</p><p style="text-align: left;">但这是不准确的。Windows Autopatch 的推出并不意味着将不再有 Patch Tuesday 更新。</p><p style="text-align: left;">微软告诉 SecurityWeek，该公司将在每月的第二个星期二继续发布安全更新。 Windows Autopatch 是针对某些企业客户的可选服务，但不会对其他所有人进行任何更改。</p><p style="text-align: left;">来自微软内部的高级消息人士也向 SecurityWeek 证实，有关 Patch Tuesday 结束的公开报道是错误的。甚至微软关于 Windows Autopatch 的公开文档也指出，新服务不会影响 Patch Tuesday。</p><p style="text-align: left;">造成混淆的部分原因还在于，对于使用 Windows Autopatch 的组织而言，将有一个默认的更新发布时间表，但将加快解决关键漏洞的补丁程序，以立即保护设备。</p>
