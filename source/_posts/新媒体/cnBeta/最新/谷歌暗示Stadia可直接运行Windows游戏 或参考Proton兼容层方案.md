<strong>当 Google 于三年前首次推出 Stadia 云游戏服务时，后端服务器硬件都是在 Linux 平台上运行的。</strong>这意味着开发者必须经过一番移植工作，才能让他们的游戏在 Stadia 上顺利跑起来 —— 即使 Google 同 Unreal / Unity 这两大游戏引擎开发商、甚至 Havok 这样的中间件公司展开了深入的合作。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/1027/25ce0ef3ef9ff00.jpg" referrerpolicy="no-referrer"></p><p>不过近日有消息称，Google 已经打造出了一套给力的解决方案，以支持 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 游戏在未经额外修改的情况下，于 Stadia 云平台上顺畅运行。</p><p>对此，许多人可能第一时间想到了基于模拟器的实现方案。但更有可能的情况是，该公司特地为 Windows 游戏构建了一个兼容层，以避免遇到潜在的性能问题。</p><p><a href="https://static.cnbetacdn.com/article/2021/1215/0defd4fefe49e75.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1215/0defd4fefe49e75.jpg" referrerpolicy="no-referrer"></a></p><p>作为参考，Valve 自行打造的 Proton 兼容层，就允许 Windows 游戏借助修改后的 Wine 版本在 Linux 平台上运行。</p><p>具体说来是，通过将<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>的 Direct3D 图形 API 映射到 Vulkan，有助于增强 Steam Deck 掌机的实际游戏体验。</p><p><a href="https://static.cnbetacdn.com/article/2022/0301/2704bd63f08274c.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0301/2704bd63f08274c.png" referrerpolicy="no-referrer"></a></p><p>当然，Google 可能不是唯一想要这么做的云游戏企业，比如亚马逊旗下的 Luna 游戏串流服务。</p><p>不同的是，其干脆直接采用了 Windows 后端，同时亚马逊也在长期努力聘请具有 Proton 合作经验的开发人员。</p><p>至于 Google Stadia 的更多改进，还请耐心等待定于 3 月 15 日（下周二）召开的谷歌游戏开发者峰会。</p>
