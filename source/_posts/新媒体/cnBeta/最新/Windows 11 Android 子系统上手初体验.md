随着微软放出 Windows 11 Android 应用的首个预览，Beta 通道的 Insider 测试者们已经能够抢先体验。<strong>为了带来第一手的讯息，TheVerge 老伙计 Tom Warren 也分别在两套平台上开展了体验。</strong>可知其中一套平台为 Surface Pro X，另一套则是基于酷睿 i9-11900K 的游戏台式机。<br>
 <p><a href="https://static.cnbetacdn.com/article/2021/1021/cdab3215a1c0256.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1021/cdab3215a1c0256.jpg" alt="0.jpg" referrerpolicy="no-referrer"></a></p><p>首先，应用的安装体验相当简便。通过与亚马逊合作，Microsoft Store 已经打通了第三方 Android 应用商店，以查找、安装、或更新相关 Android 应用。</p><p>与此同时，Windows 11 会在后台静默安装配套的 Android 子系统（简称 WSA），之后 Android 应用就能够在虚拟机中有效运行。</p><p>不过对于普通用户来说，你并不会发现它和原生 Windows 应用有何不同，依然可以将之固定到“开始”菜单或任务栏上、支持窗口化和多任务处理。</p><p style="text-align: center;"><iframe src="//tv.sohu.com/s/sohuplayer/iplay.html?bid=297126026&autoplay=false&disablePlaylist=true" width="640" height="480" frameborder="0"></iframe></p><p style="text-align: center;">Windows 11 Android Apps First Look（<a href="https://tv.sohu.com/v/dXMvODIyMjQwNTMvMjk3MTI2MDI2LnNodG1s.html" target="_self">via</a>）</p><p>尽管首个预览版本只能用到 50 款 Android App，但 Tom Warren 还是顺利体验了亚马逊 Kindle、BBS Sounds、以及一系列手游，以评估在 Windows 11 上模拟运行的效率。</p><p>结果发现，无论在基于 AMR 芯片组的 <a data-link="1" href="https://microsoft.pvxt.net/9W473" target="_blank">Surface</a> Pro X 上、还是在 x86 平台的酷睿 i9-11900K 游戏台式机上，Android 应用的使用都相当顺畅，且能够像 Word、Chrome、以及其它普通 Windows 桌面应用程序一行运行。</p><p><a href="https://static.cnbetacdn.com/article/2021/1021/4772d99e61aad11.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1021/4772d99e61aad11.jpg" alt="1.jpg" referrerpolicy="no-referrer"></a></p><p>即使在 Windows 11 中检索应用程序，WSA 应用也能够像常规 Windows 应用程序一样被索引到，更别提在“开始”菜单和任务栏上无缝集成了。</p><p>不过预览版本的 WSA 子系统也不是 100% 完美。比如尝试通过 Windows 11 的“Snap Layouts”功能，让四个 Android 游戏并排放置时，一切会突然变得相当迟钝，直到 CPU 占用率在几秒钟后恢复正常。</p><p><a href="https://static.cnbetacdn.com/article/2021/1021/1bf3d8d15a5e23f.jpeg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1021/1bf3d8d15a5e23f.jpeg" alt="2.jpeg" referrerpolicy="no-referrer"></a></p><p>另一方面，在 Surface Pro X 上运行 Android 游戏的体验也相当不友好，低帧率和卡顿时常态，但愿<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>能够尽快在后续版本中加以改进。</p><p>此外目前想要在 Windows 11 上获取 Android 应用的话，除了手动侧载、官方也仅提供了亚马逊 Appstore 这一个途径。</p><p><a href="https://static.cnbetacdn.com/article/2021/1021/8a38ea2428f2981.jpeg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1021/8a38ea2428f2981.jpeg" alt="3.jpeg" referrerpolicy="no-referrer"></a></p><p>Tom Warren 还注意到，在体验《最终幻想 15》游戏和某些 Android App 时，如果你尝试调节窗口大小，几乎逃不过重新加载的命运。</p><p>当然，这主要取决于 Android 应用开发者是如何构建他们的 App 的。毕竟起初大家只想到在<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://shouji.jd.com/" target="_blank">手机</a>屏幕上体验，WSA 这种“异端”只能期待着在未来某一天迎来优化。</p><p><a href="https://static.cnbetacdn.com/article/2021/1021/f128ecab4fd2d62.jpeg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1021/f128ecab4fd2d62.jpeg" alt="4.jpeg" referrerpolicy="no-referrer"></a></p><p>最后，在 Windows Android / Linux 子系统面世之前，许多玩家都选择了通过蓝叠（Bluestacks）来模拟运行 Android 游戏或应用。</p><p>但随着微软官方 WSA / WSL 解决方案的日渐成熟，一个“只有 Bluestacks 受伤的世界”或许就要达成了。</p><p>好消息是，作为一套面向未来的应对措施，当微软还苦恼于为 Windows 引入 WSL / WSA 的时候，Bluestacks 早就在酝酿将 Android 虚拟环境搬到浏览器中去运行了。</p>
