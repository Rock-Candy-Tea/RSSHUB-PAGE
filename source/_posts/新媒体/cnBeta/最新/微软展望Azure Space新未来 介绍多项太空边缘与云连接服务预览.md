微软最近放出了想要将 Azure 云服务打入太空的新公告，<strong>今天早些时候，该公司又披露了如何将太空与卫星连接作为其服务平台的一部分、以及如何实现计划目标的更多细节。</strong>首先，这家雷德蒙德软件巨头宣布了 Azure Orbital Cloud Access 的预览。作为一项全新的服务，其旨在将微软强大的云功能，带到客户所需的任何地方。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0915/84bc3f02516d0ff.jpg" alt="0.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：Azure <a href="https://azure.microsoft.com/en-us/blog/new-azure-space-products-enable-digital-resiliency-and-empower-the-industry/" target="_self">Blog</a>）</p><p><strong>其次，微软宣布了 Azure Orbital 地面站的全面启用。</strong></p><blockquote><p>可知自 2020 年 10 月推出 Azure Space 以来，该公司一直在谈论 Azure 轨道地面站。</p><p>今天，该公司正式携手 KSAT 在内的合作伙伴网络，向所有卫星运营商提供这项服务 —— 比如 Pixxel、Muon Space 和 Loft Orbital 。</p></blockquote><p><strong>第三，微软致力于推进卫星通信网络的数字化转型。</strong></p><blockquote><p>除了完全虚拟化的 iDirect 调制解调器的首次演示，微软还携手 SES 宣布了一项全新的联合卫星通信虚拟化计划。</p></blockquote><p><img src="https://static.cnbetacdn.com/article/2022/0915/917525c971775a5.png" alt="1.png" referrerpolicy="no-referrer"></p><p><a href="https://www.onmsft.com/news/aure-in-space" target="_self">OnMSFT</a> 指出，Azure Space 更新的空间连接计划的一部分，现还包括一项名为 Azure Orbital Cloud Access 的新预览，该服务旨在为更偏远的企业提供 IT 连接。</p><p>然后是由合作伙伴主导的 Azure Orbital Ground Station，致力于以零成本将客户数据交付到所选的 Azure 区域的倡议。</p><p><img src="https://static.cnbetacdn.com/article/2022/0915/e22a9599e5c581f.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p>通过搭载 SpaceX 的 Starlink 硬件，微软可让 Azure 边缘设备优先处理光纤、蜂窝和卫星网络并提供流量，同时能够与 Juniper Networks 的 SD-WAN 技术原生集成。</p><p>微软还希望通过 5G 提供数字弹性、并已开始与几名荒地消防员开展测试，以更好地评估 Azure Orbital Cloud Access 在更偏远地区的有效性。</p><p>同时与 Pegatron、SES 合作探索自然灾害发生时的情景，为新竹消防局提供 5G 与空间技术支撑。</p><p><img src="https://static.cnbetacdn.com/article/2022/0915/2b4d779984b350d.png" alt="3.png" referrerpolicy="no-referrer"></p><p>最后，虽然微软致力于通过解锁其云服务的连接性和数据，来实现空间民主化的终极目标。</p><p>但我们可以肯定，其在云领域的最大竞争对手，也在积极筹划结合来自 AWS / 地面站的航空航天与卫星解决方案。</p>
