随着 USB 功能的飞速发展，许多人在使用公共充电设施的时候，也会犹豫自己的设备是否会被窃取数据。<strong>好消息是，为了化解这方面的顾虑，苹果已在 macOS 13 Ventura 的首个 Beta 测试版本中引入了一项改进。</strong>具体说来是，当使用 USB-C 或雷电配件在 M1 / M2 MacBook 上建立通讯前，系统会明确向用户提请许可。<br>
  <p><img src="https://static.cnbetacdn.com/article/2022/0607/b3d0a7b40a5cc38.webp" referrerpolicy="no-referrer"></p><p>苹果在<a href="https://developer.apple.com/documentation/macos-release-notes/macos-13-release-notes" target="_self">发行说明</a>中写道：</p><blockquote><p>在配备 Apple Silicon 的便携式 Mac 计算机上，USB-C / 雷电<a data-link="1" href="https://microsoft.pvxt.net/Xx07X" target="_blank">配件</a>将获得用户的批准，才能实现与 macOS 的通讯。</p><p>不过需要指出的是，这项政策并不影响 M1 / M2 <a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fmac%2F" target="_blank">MacBook</a> 与 USB-C 电源适配器、外接显示器、或经授权的集线器之间的连接。</p><p>即使你选择了不允许传输数据，设备仍可正常地接受充电。</p></blockquote><p>如有需要，Apple Silicon MacBook 用户可在“系统设置 → 安全和隐私 → 安全”选项卡下变更默认的安全配置。</p><p>默认设置是始终询问，而经过批准的设备最多可连接到锁定的 Mac 三天。如果外设可信，怕麻烦的用户也可设置使用允许附件连接。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0607/1b9583cdd5afee1.webp" referrerpolicy="no-referrer"></p><p>虽然听起来只是为了阻挡可疑的 USB 外设，但我们近年确实听到过一些导致 Mac 损坏的案例（USB Killer 这种“大杀器”就有些缺德到冒烟了）。</p><p>庆幸的是，随着 macOS 13 Ventura 引入新的外设连接许可策略，Apple Silicon MacBook 用户将能够更加放心地使用、且苹果并未就此推出新的认证（坑钱）项目。</p><p>事实上，与 USB-IF 在 2019 年提出的“USB-C 身份验证项目”相比，苹果的解决方案要轻松且明智得多 —— 因为前者需要为每个 USB 设备提供加密证书，以验证身份并确认其能力。</p>
