<strong>在年度 re:Invent 大会上，亚马逊 AWS 宣布了两项针对汽车行业的新服务。首先是 AWS FleetWise，其能够帮助汽车制造商更轻松地收集和检索车队的传感器与要测数据。</strong>其次是 AWS Automotive，这是一项更广泛的、特定于行业的计划，旨在将一系列产品汇聚到一处，类似于 AWS 的其它行业解决方案（比如 AWS for Industrial）。<br>
 <p><a href="https://static.cnbetacdn.com/article/2021/1201/de92dcb51ab9a6f.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1201/de92dcb51ab9a6f.png" referrerpolicy="no-referrer"></a></p><p>虽然传感器数据收集这件事听起来并不复杂，但 FleetWise 承诺提供端到端的解决方案，辅以更大的灵活性。</p><blockquote><p>举个例子，厂商可能想要在汽车上巧妙地过滤数据，以减少传输到云端的数据量 —— 比如只从特定传感器中提取数据（发动机的运行温度等）。</p><p>当然，如有需要，FleetWise 仍可收集所有的传感器数据，决定权始终掌握在汽车制造商自己的手中。</p></blockquote><p>此外 AWS 汽车部门总经理 Mike Tzamaloukas 指出：</p><blockquote><p>在使用 FleetWise 之前，制造商必须先使用开源的车辆信号规范，以便对车辆机器传感器进行描述和适当的建模，目前已有一些汽车制造商采用。</p><p>然后，AWS 会为汽车制造商提供与 FleetWise 交互并收集数据的源代码，以供开发者获取、根据实际需要进行修改、并置于车辆网关的整体构建中。</p><p>如此一来，工程师们就可以轻松开展收集活动，以从车队中提取实时数据。</p></blockquote><p>Mike Tzamaloukas 解释称：</p><blockquote><p>数据收集活动的可能性是无穷无尽的，我们正在推动这项任务的极限，因为它不仅基于时间或单纯的事件，而是整个车队的智能数据过滤。</p><p>我们希望让汽车制造商具备这样的能力，即以更高的信噪比、提取不同汽车产生的 TB 级数据。</p><p>如果一切顺利，AWS FleetWise 将于 2022 年全面上市，并于 2024 年前后推出内置该功能的量产车型。</p><p>届时除了位于数据收集后端的汽车制造商，前端的车主们也可以借助 FleetWise，查看到更详尽的月度车辆健康报告。</p></blockquote><p>至于 AWS for Automotive，负责该计划的 Dean Phillips 认为，它将是彰显 AWS 行业能力的一种方式。</p><blockquote><p>我们对面向汽车的 AWS 应用感到十分激动，因其确实有助于向我们的客户阐明从亚马逊云服务中获得不同解决方案的能力。</p><p>比如汽车制造商能够按需选配针对不同领域的解决方案，从简短的自动驾驶移动服务到产品工程支持、供应链追踪、以及其它数字型客户参与解决方案。</p></blockquote>
