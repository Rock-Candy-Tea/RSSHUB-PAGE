微软今天发出提醒，表示 Windows Server 20H2 即将于 8 月 9 日停止支持。也就是说一旦到了最后期限，Windows Server 20H2 版将不再得到任何进一步的更新，因此可能影响操作系统的安全漏洞就会一直得不到修复。<br>
<p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0519/f68541a605f1456.webp" alt="4thpurom.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">也就是说，在 2022 年 8 月的补丁星期二活动之后，微软将不会再为该 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Server 带来最新的更新，届时该平台将被视为退役，不再获得任何其他改进。</p><p style="text-align: left;">微软表示：“Windows Server 20H2 将在2022年8月9日达到服务的终点。这也将是Windows Server半年度渠道（SAC）的退休。未来将不会 有Windows Server 的 SAC 版本，而且这个产品在2022年8月9日之后将不再收到安全更新”。</p><p style="text-align: left;">微软表示：“Windows Server 正在转移到长期服务渠道（LTSC）作为主要的发布渠道。使用 Windows Server SAC 的客户应该转移到 Azure Stack HCI。另外，客户可以使用 Windows Server 的长期服务渠道”。</p><p style="text-align: left;">此外微软还提醒 IE 浏览器将于 6 月 15 日退役，届时所有人都需要转到与Microsoft Edge捆绑的IE模式。</p><p style="text-align: left;">微软表示：</p><blockquote style="text-align: left;"><p style="text-align: left;">从2022年6月15日开始，Internet Explorer 11（IE11）桌面应用程序将在Windows 10半年度渠道和Windows 10 IoT半年度渠道中停止支持。我们鼓励客户转移到Microsoft Edge，它为传统和现代网站和应用程序提供支持。</p><p style="text-align: left;">对于依赖基于Internet Explorer的传统网站和应用程序的组织，将需要对网站进行配置，以便在Microsoft Edge中使用Internet Explorer（IE）模式打开。</p></blockquote>
