Fedora工程和指导委员会(FESCo)已经为下一个Fedora Linux发行版批准了一批新的功能。Fedora 37的更多变化得以确定下来，以便开发人员继续进行本周期的计划工作。最新一批为Fedora 37批准的项目包括：<br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0628/27498b7e8daf301.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0628/27498b7e8daf301.jpg" title alt="release_schedule.jpg" referrerpolicy="no-referrer"></a></p><p>- 所有由Fedora发货的Java JDK现在都将根据树内库和静态libstdc++库来构建。这样做的目的是为了更像其他JDK，并为可移植性提供适当的可转移镜像。</p><p>- LLVM 15作为领先的开源编译器堆栈的最新版本，将被打包给Fedora 37。</p><p>- 一个合适的Fedora KVM虚拟磁盘镜像将作为Fedora Server 37的一部分被运送。</p><p>- 在x86_64体系上，Fedora现在将在新安装中使用GPT分区，而不是BIOS系统的MBR/MSDOS。目前 Fedora 只在 EFI 系统上使用 GPT 格式。</p><p>- GRUB2 现在将被用于 BIOS boot.iso 启动，而不是 syslinux。上游的syslinux开发已经终止，发行版中将放弃它，所以转而使用GRUB2。</p><p>- 继续收紧加密政策 Fedora 37暂时没有改变任何默认值，但在Fedora 38和Fedora 39之前，会有高级警告，以获得额外的可见性和未来的指导。</p><p>- Fedora 37 Python 将为 Python shebang 宏的默认 shebangs 添加"-P"标志。</p><p>- Erlang 25 将在 Fedora 37 中提供。</p><p>- Perl 5.36 将被打包给 Fedora 37。</p><p>更多关于过去一周批准的Fedora 37变化的信息，请参见Fedora开发主题：</p><p><a href="https://lists.fedoraproject.org/archives/list/devel@lists.fedoraproject.org/thread/AEF4US5TEZK66YLG7UGGIGWW2HYP732E/" _src="https://lists.fedoraproject.org/archives/list/devel@lists.fedoraproject.org/thread/AEF4US5TEZK66YLG7UGGIGWW2HYP732E/" target="_blank">https://lists.fedoraproject.org/archives/list/<span class="__cf_email__" data-cfemail="54303122313814383d2720277a3231303b263524263b3e3137207a3b2633">[email protected]</span>/thread/AEF4US5TEZK66YLG7UGGIGWW2HYP732E/</a><br></p><p>Fedora 37 正在努力争取在十月底前发布。</p>
