过去几个月，Phoronix 一直有报道为 Linux 内核或 Mesa 等组件，准备英特尔 DG2 / Alchemist 图形代码支持的文章。<strong>期间，我们已知晓 Intel ARC 的工作状态和开发进度，但官方尚未就预期中的 Linux 硬件支持 / 版本要求发布过正式通告。</strong>早前有消息称，英特尔 ARC DG2 / Alchemist 的大部分图形支持已被撤销。<br>
<p><a href="https://static.cnbetacdn.com/article/2022/0331/c623749ea09be01.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0331/c623749ea09be01.jpg" referrerpolicy="no-referrer"></a></p><p>好消息是，随着一个补丁系列的到来（有待实际公开），Linux 平台仍有望迎来基于<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://intel.jd.com/" target="_blank">英特尔</a> ARC DG2 / Alchemist 图形产品的“计算支持”。</p><p>通过各种内核驱动程序上的变更，英特尔可向 Linux 用户空间传递 ARC DG2 / Alchemist 图形硬件的计算支持。</p><p>更确切地说，该系列补丁暴露了 Xe_HP 软件开发工具、DG2 的计算命令流（CCS）引擎，以及适当的 GPU 计算支持所需的各种解决方案。</p><p><img src="https://static.cnbetacdn.com/article/2022/0414/d3955b9a77d0616.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">邮件公告列表（来自：<a href="https://lists.freedesktop.org/archives/intel-gfx/2022-April/295655.html" target="_self">FreeDesktop.org</a>，via <a href="https://www.phoronix.com/scan.php?page=news_item&px=Intel-Preps-DG2-Compute-Patches" target="_self">Phoronix</a>）</p><p>需要指出的是，这段代码尚未正式合并到 Linux 内核主线。对于尚未正式发布的产品来说，我们对此并不感到意外。</p><p>如果一切顺利，我们有望于今夏发布的 Linux 内核中迎来这方面的支持（前提是已准备好即使发布 v5.19）。</p><p>若你手头已有一款 ARC A 系列独显，可能只需稍等片刻，即可为 Linux GPU 计算支持做好准备。</p>
