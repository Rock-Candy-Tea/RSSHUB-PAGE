<strong>随着 Canonical 今日正式开放了“Jammy”存档，意味着 Ubuntu 22.04“Jammy Jellyfish”长期支持版本（LTS）的开发项目也被正式摆上了议事日程。</strong>如果一切顺利，新 LTS 版本将于 2021 年 4 月 21 日到来。目前“Jammy”存档已经向开发者开放，且与 Debian 保持自动同步。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/1019/ce150e1ef804e9a.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">截图（来自：<a href="https://discourse.ubuntu.com/t/jammy-jellyfish-release-schedule/23906" target="_self">Canonical</a>）</p><p>与 Ubuntu 22.04 一同到来的，包括 Python 3.10 和 OpenSSL 3.0 。而在未来半年内，LTS 版本还计划引入 PHP 8.1、Ruby 3.0、GNOME 42.0、以及其它各种软件更新包。</p><p>值得庆幸的是，Canonical 在本次 LTS 周期中从 GNOME Shell 40 跳到了 42（而不是最近发布的 GNOME 41），意味着明年正式发布的 Ubuntu 22.04 LTS 会搭载最新的 GNOME 桌面组件。</p><p>同时 Mesa 22.0 也有望成为 Ubuntu 22.04 LTS 的一部分，可获得全新的开源图形驱动程序支持。内核方面，就算用不上最新的 Linux 5.17，至少也有 Linux 5.16 + 最新硬件支持 / 底层改进。</p><p>遗憾的是，按照传统，Ubuntu 22.04 LTS 很可能会坚持使用 GCC 11 作为代码编译器 —— 即便 GCC 12 即将于明年 4 月发布。所以在 XX.10 发布之前，别指望系统自带的 GCC 编译器会有大版本更新。</p><p>若目前正在大力开发中的新版 Ubuntu 桌面安装程序的状态足够好，那 Canonical 应该也会将它指定为 22.04 LTS 的默认安装程序，不过最终还得看它能否在未来 6 个月内走向成熟。</p>
