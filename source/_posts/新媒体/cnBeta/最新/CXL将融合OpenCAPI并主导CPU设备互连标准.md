定于本周召开的 2022 闪存峰会，似乎也成为了 I/O 互连标准的一个重要讨论场。<strong>由周一的 OpenCAPI 公告可知，其将与 CXL 组织实现更深度的融合。</strong>集成后，已被几乎所有主要制造商支持的 CXL，将继续主导 CPU 与设备之间的互连标准。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0802/0895805999d2397.png" alt="1.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：<a href="https://www.globenewswire.com/news-release/2022/08/01/2489825/0/en/CXL-Consortium-and-OpenCAPI-Consortium-Sign-Letter-of-Intent-to-Transfer-OpenCAPI-Specifications-to-CXL.html" target="_self">Press Release</a>）</p><p>据悉，比 CXL 诞生早几年的 OpenCAPI 是缓存一致 CPU 互连的早期标准之一，并得到了 <a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://amd-cpu.jd.com/" target="_blank">AMD</a>、Xilinx 和 IBM 等公司的支持。</p><p>OpenCAPI 基于 IBM 现成的相干加速器处理器接口（CAPI）而扩展，并选择了在行业联盟的护航下向其它公司开放。</p><p>然而过去六年里，OpenCAPI 的使用率并不高。相对最瞩目的，也就是 IBM 在自家 POWER9 处理器系列上的实现。</p><p><a href="https://static.cnbetacdn.com/article/2022/0802/0dcba711f0eecbf.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0802/0dcba711f0eecbf.jpg" alt="2.jpg" referrerpolicy="no-referrer"></a></p><p><a href="https://www.anandtech.com/show/17519/opencapi-to-fold-into-cxl" target="_self">AnandTech</a> 指出：与类似的 CPU 到设备互连标准一样，OpenCAPI 本质上是现有高速 I/O 标准之上的进一步应用扩展。</p><p>其添加了缓存一致性和更快 / 更低延迟的访问模式等内容，以便 CPU 和加速器能够更多地协同工作 —— 即便它们的物理分解很是紧密。</p><p>只是作为几项竞争标准之一，OpenCAPI 从未在业界引发多高的关注。诞生于 IBM 襁褓中的它，正好也处于该公司服务器份额持续下滑的时期。</p><p>除了身为最大客户的 IBM，甚至像 AMD 这样的新兴联盟成员，也最终抛弃了这项技术、转而使用自家的 Infinity Fabric 架构来实现 AMD 服务器 CPU / GPU 的连接。</p><p><a href="https://static.cnbetacdn.com/article/2022/0802/fb4fe2f5f932fe5.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0802/fb4fe2f5f932fe5.jpg" alt="3.jpg" referrerpolicy="no-referrer"></a></p><p>在没有大企业护航和推动发展的情况下，OpenCAPI 最终出于制造商和客户利益的考量，选择了与行业最期待的 CXL 统一互连标准进行整合。</p><p>根据拟议的条款，在必要各方签署完成后，OpenCAPI 联盟的所有标准与资产（比如鲜为人知的 OMI 开放内存标准），都将退出并被 Compute Express Link 所吸收。</p><p>话虽如此，接手后的 CXL 联盟显然不会继续开发 OpenCAPI 的遗留资产，只是意味着后者的任何可用技术、都有望集成到 CXL 的未来版本标准中，以加强整个生态系统的竞争活力。</p>
