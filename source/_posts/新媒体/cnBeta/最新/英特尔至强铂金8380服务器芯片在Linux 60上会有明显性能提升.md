<strong>国外科技媒体 Phoronix 的编辑 Michael Larabel 近日在窗口合并期即将结束的 Linux 6.0 Git 上，对英特尔至强铂金 8380 芯片进行了测试</strong>。此外文章中还提及，AMD 的霄龙（EPYC）芯片同样有非常不错的性能提升。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0815/4cc6be5f6645463.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0815/b93535b6bb9bcb8.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">Larabel 认为<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://intel.jd.com/" target="_blank">英特尔</a>在开源兼容性上的努力和关注已经取得了回报，尤其是在 Ice Lake 服务器 CPU 方面。他提到，预计会有更多的调度程序更改和内核改进，并且很可能会很好地转移到英特尔至强铂金 8380 处理器上。</p><p style="text-align: left;">本次测试使用的是英特尔至强铂金 8380 2P 服务器芯片。当前的测试环境是 Linux 5.19 稳定内核与 Linux 6.0 Git，于 2022 年 8 月 9 日完成。他使用 Ubuntu Mainline Kernel PPA 访问了两个内核，理由是“易于重现”。</p><p style="text-align: left;">Larabel 在测试的早期透露，Linux 6.0 中的 Intel Xeon Platinum 8380 处理器在发布时似乎已准备就绪。测试的亮点表明 PostgreSQL 在 Linux 6.0 下的性能得到了增强。</p><p style="text-align: left;">他还指出，数据库工作负载有所改善，Apache Spark、Apache Cassandra 和 ClickHouse 显示出从最新的 Linux Git 中获益，而 Apache 和 RocksDB 服务器工作负载自之前的 Linux 5.19 稳定内核以来有所增加。</p><p style="text-align: left;">在 Linux 6.0 中增加了 SVT 视频编码，Larabel 指出他在之前的测试中没有目睹这种情况发生。 LAMMPS 是一项 HPC 工作负载测试，在新的 Linux 中越来越好，并且还增加了几个多线程工作负载。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0815/98fcc2a9b770825.webp"
