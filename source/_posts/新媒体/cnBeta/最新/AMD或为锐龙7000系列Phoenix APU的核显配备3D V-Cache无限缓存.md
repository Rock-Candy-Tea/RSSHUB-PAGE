知名爆料人 @Greymon55 刚刚在 Twitter 上援引封装厂消息人士的话称：<strong>AMD 锐龙 7000 系列“Phoenix”APU 处理器不仅集成了 RDNA 3 核显、还配备了 R7-5800X3D 上率先为消费级桌面 CPU 引入的 3D V-Cache 缓存。</strong>得益于强大的 RDNA 3 iGPU，据说 Phoenix APU 的表现可与英伟达 GeForce RTX 3060 移动独显一较高下。<br>
<p><img src="https://static.cnbetacdn.com/article/2022/0517/a17d4b142a6247f.jpg" referrerpolicy="no-referrer"></p><p>此外在核心数量上，Phoenix APU 也将在 Zen 4 架构的加持下，向<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://intel.jd.com/" target="_blank">英特尔</a>竞品发起更直接的挑战 —— 不过 3D V-Cache 也有望成为该系列芯片的一招秘技。</p><p>参考 Zen 3 锐龙 R7-5800X3D 上的 96MB 垂直缓存，尽管仍有积热上的顾虑、3D V-Cache 还是在不少特定测试项目中具有独特的领先优势。（霄龙 Milan-X 服务器 CPU 也较 Milan 升级了 3D V-Cache）</p><p>不过在 <a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://amd-cpu.jd.com/" target="_blank">AMD</a> 的 Radeon RX 6000 系列独显产品线上，该公司则是采用了独特的“Infinity Cache”无限缓存。</p><p><img src="https://static.cnbetacdn.com/article/2022/0517/648948e8f4b1e8b.png" referrerpolicy="no-referrer"></p><p>目前尚不清楚锐龙 7000 系列 Phoenix APU 上的 3D V-Cache 缓存的确切用途，即它到底是作为 CPU 的末级缓存、还是像 Navi 2x GPU 那样给核显提供“Infinity Cache”加持。</p><p>与英特尔相比，两家公司的核显方案有相当大的不同之处，比如 Intel 芯片允许每个 IP 块都通过环形总线在缓存中寻址。</p><p>而 AMD Cezanne / Rembrandt 等 APU 上的 L3 高速缓存，仅作为 CPU 核心复合体（CCX）的一部分 —— 核显有自己的 LLC，并将 Infinity Fabric 互连总线作为所有 IP 块的通讯枢纽。</p><p><img src="https://static.cnbetacdn.com/article/2022/0517/b49f987486fee7f.webp" referrerpolicy="no-referrer"></p><p>展望未来，AMD APU 有望朝着统一 CCX 与 iGPU 末级缓存的方向去发展，但前提是缓存容量达到足以支撑这方面的功能（很可能通过 3D V-Cache 堆叠来实现）。</p><p>若真如此，RDNA 2 核显与 RTX 3060M 对标也将不在话下。作为参考，Radeon RX 6650M XT 笔记本独显采用了 Navi 23 GPU 芯片、具有 32MB 的无限缓存。</p><p>这意味着，AMD 确实可以通过一些巧妙的内存管理，为 Phoenix APU 提供 64MB 左右的 LLC 缓存。</p>
