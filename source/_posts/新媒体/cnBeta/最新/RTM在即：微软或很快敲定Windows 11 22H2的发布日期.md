<strong>随着 Windows 11 Build 22610 编译版本的发布，微软已在该系统的新预览中引入了大量 Bug 修复、改进、以及一些新设计的功能。</strong>鉴于 Build 22610 不会在桌面上带有“技术预览”的水印，我们推测这家雷德蒙德软件巨头或很快完成 Windows 11 22H2 的 RTM 开发工作。<br>
<p><a href="https://static.cnbetacdn.com/article/2022/0502/ba7ea4a739b9d16.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0502/ba7ea4a739b9d16.jpg" alt="1.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（截图 via <a href="https://www.windowslatest.com/2022/05/01/microsoft-closer-to-finalizing-windows-11-22h2-release-date/" target="_self">Windows Latest</a>）</p><p>微软曾在官方博客文章中指出，桌面右下角的水印缺失，并不一定意味着下一个功能更新的开发阶段已经结束，但至少意味着我们距其又近了一步。</p><p>这家雷德蒙德软件巨头将检查 Windows 11 22H2 的最终代码、并为下一个未命名更新做好准备，此外据说 Version 22H2 的发布时间或较传统的 10 / 11 月份更早一些。</p><p>目前微软已经向 Dev 和 Beta 通道推送了一个更新，其中包含了大量的改进。不过在 RTM 与正式推送期间，我们还有望见到更多重大更新与改进。</p><p><a href="https://static.cnbetacdn.com/article/2022/0502/0c2e6d20c19d570.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0502/0c2e6d20c19d570.jpg" alt="2.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">任务管理器将支持强调色</p><p>Windows Latest 指出，当前 Windows 11 22H2 预览属于 Nickel 开发分支，变化主要体现在诸多细节上。早前的主要变化已被冻结，但一些改进仍可能出现在即将到来的新版本中。</p><p>在截止 5 月 1 号的最新预览版本（Build 22610）中，微软已经修复了 Windows 11 的大量问题 —— 包括任务栏的不稳定行为、文件资源管理器的各种问题、以及一个奇怪的 Bug 。</p><p>尽管部分测试者可能遇到文件资源管理器（Explorer.exe）的反复崩溃，但这一系统工具也迎来了多彩的新外观（强调色 / accent colours），并且为其赋予了更一致的 Windows 11 体验。</p><p><a href="https://static.cnbetacdn.com/article/2022/0502/d868c35555ee77d.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0502/d868c35555ee77d.jpg" alt="3.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">文件资源管理器</p><p>新功能方面的变化并不太多，目前尚不清楚微软何时为文件资源管理器引入多选项卡支持，但至少该公司已在发行说明中提到一些改进（比如情境菜单的速度 / 新的侧边栏设计）。</p><p>Windows 11 的首次更新还待回了拖放支持，以及在“开始”菜单中固定更多图标的能力。不过在积极更新的同时，22H2 也在剔除 Windows 10 时代的一些功能（比如日历弹出按钮 / 取消组合任务栏图标的能力）。</p><p>综上所述，目前仅知 Windows 11 22H2 会带来较视觉风格 / Bug 修复更多的小改进。至于这个周年更新的确切发布日期，也许会在 8 月或更晚。</p>
