<strong>随着 FFmpeg 5.0 的正式到来，这个被全行业广泛依赖的开源、跨平台多媒体库，也正在努力提供长期支持（LTS）版本。</strong>官网写道，开发团队为这个姗姗来迟的新版本倾注了大量的心力，以移除旧版编解码 API、并用基于 N:M 的新 API 来代替。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0118/3980ea68842f2ff.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">截图（来自：<a href="https://ffmpeg.org/" target="_self">FFmpeg.org</a>）</p><p>此外 FFmpeg 5.0 移除了整个 libavresample 库，libswscale 迎来了基于 AVframe、更易于使用的新 API 。</p><p>Vulkan 代码得到了很大的改进，添加了许多新的过滤器（filter），包括 libplacebo 集成。</p><p>最后提供了对 DoVi 的支持，包括色调映射和 remuxing，并且更改了默认的 AAC 编码器设置以提升质量。</p><p><img src="https://static.cnbetacdn.com/article/2022/0118/fba3be6127162c6.png" referrerpolicy="no-referrer"></p><p>在周一的<a href="http://www.jbkempf.com/blog/post/2022/FFmpeg-5.0" target="_self">博客文章</a>中，VLC 首席开发者 Jean-Baptiste Kempf 详细介绍了 FFmpeg 5.0 的新里程碑。</p><p>首先，FFmpeg 5.0 有望开始转向基于时间的发布节奏，期待着每年都能迎来大版本更新。</p><p>其次，我们有望每隔一年迎来一个长期支持版本（LTS），而 FFmpeg 5.0 正好拉开了这一序幕。</p>
