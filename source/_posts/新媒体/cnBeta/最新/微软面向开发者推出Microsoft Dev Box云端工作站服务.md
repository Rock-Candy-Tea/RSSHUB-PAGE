<strong>在去年提供了 Windows 365 云 PC 服务之后，微软现又希望吸引广大开发者使用基于云的 Microsoft Dev Box 工作站环境。</strong>作为一项云服务，其依然构建在 Windows 365 的基础之上，意味着开发者能够轻松启动新的虚拟机环境、并及时投入相关工作。<br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0525/1be9c401f63cfa6.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0525/1be9c401f63cfa6.png" alt="1.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">Windows 365 集成的 Microsoft Dev Box（来自：MS <a href="https://techcommunity.microsoft.com/t5/azure-developer-community-blog/introducing-microsoft-dev-box/ba-p/3412063" target="_self">Dev Blog</a>）</p><p>这意味着 Microsoft Dev Box 可以在任何现代浏览器上运行，且支持任何集成开发环境（IDE）、软件开发工具包（SDK）、或其它在 Windows 之上运行的工具。</p><p>目前微软正在向各种可能从事具有软件冲突和依赖性的项目开发者们推销 Microsoft Dev Box，原本这项工作会让许多开发者和维护人员感到十分头疼。</p><p>除了桌面、移动、物联网和游戏开发者，Microsoft Dev Box 也适用于 Windows Linux 子系统（WSL）上的跨平台应用程序构建。</p><p><a href="https://static.cnbetacdn.com/article/2022/0525/d9f580d1157a925.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0525/d9f580d1157a925.png" alt="2.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">该服务允许开发者创建多个 Dev Boxes 环境</p><p>微软首席项目经理 Anthony Cangialosi 表示：借助 Microsoft Dev Box，开发团队能够使用开发人员构建和运行应用程序多虚的所有工具和依赖项，来创建和维护 Dev Box 映像。</p><p>在此基础上，开发团推能够容纳他们的应用程序源码、以及每夜构建的二进制文件，使得开发者能够立即着手运行和理解相关代码、而无需经历漫长的时间去等待运行环境的重新构建。</p><p>据悉，微软已经建立了一个门户，以便开发者创建和移除 Dev Box、甚至在多台机器上并发执行相关任务。此外开发者能够自由地维护老旧的应用程序、或为旧版本重建一个定制的运行环境。</p><p><a href="https://static.cnbetacdn.com/article/2022/0525/ae002939ecabab7.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0525/ae002939ecabab7.png" alt="3.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">内建安全性与管理特性</p><p>更棒的是，Dev Box 能够指定部署于微软的全球 Azure 云区域、辅以千兆连接带宽，甚至可在开发者当天注销时自动启用 / 停止（计划性休眠）。</p><p>此外由于 Microsoft Dev Box 被集成到了 Windows 365 之中，IT 管理员们也可使用 Intune 和 Microsoft Endpoint Manager 来管理 Dev Box 。</p><p>最后，该服务已于即日起开启封闭预览（Private Preview Mode）。如果一切顺利，微软将于“未来几个月内”提供公开预览体验。感兴趣的朋友，可到官网上提交注册（<a href="https://aka.ms/devbox-signup" target="_self">传送门</a>）。</p>
