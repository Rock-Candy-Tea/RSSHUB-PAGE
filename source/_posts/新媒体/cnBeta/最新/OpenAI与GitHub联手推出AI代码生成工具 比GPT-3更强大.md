比GPT-3更强大！昨日，微软与OpenAI共同推出了一款AI编程工具GitHub
Copilot，这款工具基于GitHub及其他网站的源代码，可根据上文提示为程序员自动编写下文代码！GitHub 的首席执行官 Nat
Friedman 介绍说，GitHub
Copilot是结对编程的虚拟版本。结对编程是一种常见的敏捷软件开发技术——即两个开发人员在同一个项目上并肩协作，轮流编写代码并检查合作伙伴的输出。<br>
 <section by="yiban.io"><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/c762f66662e889e.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/c762f66662e889e.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/c762f66662e889e.jpg" referrerpolicy="no-referrer"></a></section><p>“它可以在编辑器中直接获提供多行或整个函数的代码建议。另外，随着时间的推移和使用量的增加，模型会变得越来越复杂，代码质量也会越来高。”</p><p>对于程序员来讲，一款编程工具最重要的当然是支持哪些类型语言。GitHub在预告中说，Copilot 可以支持十几种语言，与 Python、JavaScript、TypeScript、Ruby 和 Go 等主流语言配合效果更佳。</p><p><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/a5fba1134b7d841.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/a5fba1134b7d841.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/a5fba1134b7d841.jpg" referrerpolicy="no-referrer"></a></p><p>目前GitHub官网已经晒出了 Copilot 的功能介绍，我们来了解一下。</p><section><section><section><strong>1</strong></section></section><section><section></section></section></section><p><strong>给力的结对编程伙伴</strong></p><p>GitHub表示，相比大多数代码助手，GitHub Copilot可以理解更大范围的上下文。因此，无论是文档字符串、注释、函数名称还是代码本身，GitHub Copilot 都可以基于提供的上下文（已输入的代码）合成新代码以进行匹配。</p><p>如果真有如此可靠，那么正如GitHub所说，这项功能可以让程序员丢掉很多的文档阅读工作，从而专注于编辑器。</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/9c66d569b2ae8fa.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/9c66d569b2ae8fa.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/9c66d569b2ae8fa.jpg" referrerpolicy="no-referrer"></a></section><p>GitHub Copilot还可以将注释转换为代码。只需要写一条评论，描述想要的逻辑，GitHub Copilot 就能自动“理解”并写出相应功能的代码。</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/d192ec42b3b6096.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/d192ec42b3b6096.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/d192ec42b3b6096.jpg" referrerpolicy="no-referrer"></a></section><p>GitHub Copilot 也能自动填充重复代码。GitHub表示，GitHub Copilot 非常适合快速生成样板和重复代码模式。对于需要编写大量相似但难以转换为循环语句代码的程序员而言，这项功能非常“香”。</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/83809da2c101acb.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/83809da2c101acb.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/83809da2c101acb.jpg" referrerpolicy="no-referrer"></a></section><p>包办测试也是GitHub Copilot值得注意的亮点，GitHub表示，测试是任何强大的软件工程项目的支柱，“导入单元测试包，就可以让 GitHub Copilot 自动编写与你的实现代码匹配的测试。”</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/7dc0e75611ef00e.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/7dc0e75611ef00e.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/7dc0e75611ef00e.jpg" referrerpolicy="no-referrer"></a></section><p>对于相同的上下文，GitHub Copilot可以给出多种解决方案，供程序员自主选择。</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/32400cf9bd8d84d.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/32400cf9bd8d84d.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/32400cf9bd8d84d.jpg" referrerpolicy="no-referrer"></a></section><p>看到这里，小伙伴们是不是跃跃欲试了呢？可惜，GitHub Copilot不是完全开放的，和GPT-3一样，只提供有限用户的试用体验。总之，先排队吧。</p><p><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/b64a7c059d645af.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/b64a7c059d645af.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/b64a7c059d645af.jpg" referrerpolicy="no-referrer"></a></p><p>项目地址：https://copilot.github.com/</p><section><section><section><strong>2</strong></section></section><section><section></section></section></section><p><strong>Copilot，专为编程而生</strong></p><p>Brockman介绍说，GitHub Copilot的核心模型Codex借助了GPT-3强大的生成能力。</p><p>我们知道，GPT-3是OpenAI推出的超大规模自然语言生成模型，早期训练GPT-3时，OpenAI无意教它如何帮助编写代码，它更像是一种通用语言模型，主要用于文本生成、语法修改等任务。</p><p>后来OpenAI向公众开放API，这个基于文本训练的模型，被网友们玩出了50多种花样，其中包括自动生成代码。OpenAI意识到了GPT-3在编写代码方面的潜力，并在其网站上提供了在线“代码生成”服务。</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/ecc2cb569fe1fef.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/ecc2cb569fe1fef.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/ecc2cb569fe1fef.jpg" referrerpolicy="no-referrer"></a></section><p>几十年来，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>的研究人员一直在尝试如何教计算机编写代码，但未取得显著成果。此次与OpenAI合作，微软将提供了强大 Azure 云计算能力和GitHub平台（2018年被微软收购）的大量语言编程代码。</p><p>关于这款产品的代码生成质量，Friedman说，GitHub员工已经做了大量尝试，能够确保GitHub Copilot生成安全、高质量的代码。他说：“我们已经在系统中建立了一些安全机制，这些机制在减少各个领域能够显著减少出错几率。”也就是说，GitHub Copilot并不能保证完全不出错，它仍然只是一个辅助生成代码工具。</p><section><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/857a5f573868703.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/857a5f573868703.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/857a5f573868703.jpg" referrerpolicy="no-referrer"></a></section><p>官网报道说，Copilot的底层技术属于微软和OpenAI共有，今年夏天OpenAI将发布Codex模型，供第三方开发者使用。微软也表示会发布该产品的一个版本，让企业通过培训了解自己的编程风格。但目前，微软只提供公共存储库中存储的代码的服务。</p><p>这不是微软第一次依靠OpenAI来提供智能软件。上个月，微软展示了powerapps Studio应用程序的更新服务，非技术人员可以使用它来编写应用程序——用户输入他们想要添加的元素单词，GPT-3可提供一些必要代码的选项。</p><section><section><section><strong>3</strong></section></section><section><section></section></section></section><p><strong>网友讨论</strong></p><p>GitHub Copilot的发布在Reddit上引发了热烈的讨论，不少网友看好编码自动化的产业趋势，网友@markbowick说道：</p><blockquote type="2" length="186"><section><section><p>毫无疑问，这将极大地提高大多数程序员的日常编程生产力，并且（我认为）这将是推动全球软件影响呈指数增长的最重要步骤之一。</p><p>值得注意的是，GPT-J（GPT-3 较小模型之一的开源实现）在大量 GitHub 和 StackExchange 查询存储库上进行了训练，并且在特定编程相关任务上的表现明显优于 OpenAI 的其它模型。在接下来的几个月里，我预期会看到性能更好的类似的（更大）模型。</p></section></section></blockquote><p><a target="_blank" href="https://static.cnbetacdn.com/article/2021/0630/9eabe6fa3bc0358.jpg"><img data-original="https://static.cnbetacdn.com/article/2021/0630/9eabe6fa3bc0358.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0630/9eabe6fa3bc0358.jpg" referrerpolicy="no-referrer"></a></p><p>也有人提出了质疑，网友@laprika0表示，GitHub Copilot的测试方法令人担忧，“让 GitHub Copilot 建议与你的实现代码匹配的测试，但人们并不应该以这种方式写测试。”不少网友表示同意，并补充道，“如果实现代码有错误，那么GitHub Copilot将把错误引入测试中，从而影响测试效果。”</p><p><img src="https://static.cnbetacdn.com/article/2021/0630/64c7dc058838aa5.jpg" referrerpolicy="no-referrer"></p><p>还有人则注意到关于代码所有权的问题，OpenAI发表了一项声明，表示用户使用GitHub Copilot写代码时，代码片段和建议结果将与 GitHub 和 OpenAI 共享，并用于诊断目的和改进建议。同时，为保护隐私，GitHub Copilot 不会使用用户的私有代码来为 GitHub Copilot 的其他用户推荐代码。</p><p>网友@touristtam对此声明感到困惑，“他们是否拥有在用户启用GitHub Copilot时编写的代码？”</p><p><a
