虽然 Windows 7 系统已停止主流支持，但针对那些付费企业用户在本月的补丁星期二活动日中，微软还是发布了月度更新汇总 <a href="https://www.catalog.update.microsoft.com/search.aspx?q=kb5003233" target="_blank">KB5003233</a>。该更新汇总更注重安全改进，主要有两个重大的改进。<br>
<p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0512/12b467d47d245dd.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0512/12b467d47d245dd.jpg" alt="dv267cg2.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">首先，KB5003233 解决了一个冲击 SMB 协议的问题，该问题在采用 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Server 2008 R2 Service Pack 1 的设备上导致蓝屏死亡错误。这个问题并不影响 Windows 7 工作站。</p><p style="text-align: left;">此外，该更新附带了几个 Windows 组件的安全更新，包括 Windows App Platform and Frameworks, Windows Silicon Platform 和 Microsoft Scripting Engine。</p><p style="text-align: left;">鉴于 Windows 7 不再为家庭用户提供支持，只有加入微软扩展安全更新（ESU）计划的电脑才能获得这个月度更新。因此，除非你的公司为安全更新付费，否则你将无法下载新的安全更新。微软表示，从其他来源下载更新的用户在尝试安装时将收到一个错误。</p><p style="text-align: left;">微软解释说：“安装此更新并重新启动设备后，你可能会收到‘配置Windows更新失败。正在恢复更改。不要关闭你的电脑’的错误，而且更新可能在更新历史中显示为失败。如果你已经购买了ESU密钥并遇到了这个问题，请确认你已经应用了所有的先决条件，并且你的密钥已经激活”。</p>
