去年，红帽（Red Hat）决定停止对 CentOS 8 的支持，并将重心转移到 CentOS Stream，然而 CentOS 仍拥有包括迪士尼、GoDaddy、RackSpace、丰田、Verizon 在内的一些大企业客户。<strong>有鉴于此，CentOS 创始人之一的 Greg Kurtzer 于 2020 年 12 月启动了 Rocky Linux 项目，并于七个月后发布了 Rocky Linux 8.4 的首个稳定版本。</strong><br><br>
<p><img src="https://static.cnbetacdn.com/article/2021/0703/c6f3d592a3ad975.png" referrerpolicy="no-referrer"></p><p>最新消息是，作为 Rocky Linux 的创始赞助商和服务合作伙伴，在此前筹集到 3300 万美元、以及携手 Google Cloud 为客户提供 Rocky Linux 支持后，Kurtzer 旗下 CIQ 公司又获得了由 Two Bear Capital 领投的 2600 万美元 A 轮融资。</p><p>Kurtzer 表示：Rocky Linux 已经得到了“大规模”的采用，且 OS 镜像的淡季月下载量达到了 25 万左右、高峰时期甚至达到了 75 万。更棒的是，即使作为一个年轻的项目，Rocky Linux 还是在两个月内吸引了上万名开发人员和贡献者。</p><blockquote><p>据悉，在 Rocky Linux 发布之前，CIQ 正在为企业高性能计算、HPC 中心、以及云端人工智能（AI）和机器学习（ML）应用开发一个新平台。</p><p>然而就在这个过程中，Red Hat 终止了 CentOS 支持 —— 即便在过去十年里，它一直是许多企业、云、虚拟机和容器应用的首选操作系统之一。</p><p>而后当 Red Hat 为 CentOS 划上句点之时，我们觉得自身背景更有利于重拾这件正确的事情。</p></blockquote><p>此外 Kurtzer 认为开源许可证将是企业软件领域的一个关键性差异化因素：“我们非常支持开源，乃至于自己发布的所有内容，通常也更青睐于 MIT 和 BSD 许可证”。</p><blockquote><p>即便许多其它公司将开源视作一个营销噱头，以获得更高的采用率。但他们往往牢牢把控着项目及其发展方向，某种程度上甚至将项目据为人质。</p><p>然而一个真正的开源社区，并不应该由一家公司来控制，这也是我们的坚定信念。如果一个组织想要开源某款特定的软件，那他们必须接受竞争对手或另一家企业最终取得更大的成功或收益。</p><p>只有这样，开源项目才能发挥出它最大的意义。假使你所在组织还没做好准备，还请先保持闭源。</p></blockquote><p>最后，Rocky Linux 的下一步大动作，是构建一款名叫 Peridot 的系统，以确保 Rocky Linux 能够搭上现代的 Linux 内核。</p><p>有趣的是，Kurtzer 补充道：大多数 Rocky Linux 贡献者更喜欢用 Google 创建的 Golang 变成语言来构建操作系统。</p><blockquote><p>该项目得到了主线稳定 Linux 内核维护者 Greg Kroah-Hartman 的支持，以满足社区对 Rocky Linux 在更现代、更优化的内核上运行的需求。</p><p>同时 Kroah-Hartman 正在领导 Rocky Linux 的内核特别兴趣小组（SIG），并致力于为 Rocky Linux 构建一个可选的增强型内核。</p></blockquote>
