<strong>Google Project Zero 团队的一项新研究表明，Linux 平台较微软 Windows 和苹果 macOS 要安全得多。</strong>当然，这并不意味着 Linux 拥有压倒性的优势，人们有在 Linux 中发现许多安全漏洞，但是得益于维护平台的开发者们的漏洞修复积极性，安全性短板也得到了及时的补足。<br>
<p><img src="https://static.cnbetacdn.com/article/2022/0219/2e905ab33319c72.png" referrerpolicy="no-referrer"></p><p>即使曝出了不少漏洞，也不意味着为千万级服务器和家用计算机提供支撑的 Linux 平台，在日常使用中不够安全。</p><p>由 Google <a href="https://googleprojectzero.blogspot.com/2022/02/a-walk-through-project-zero-metrics.html" target="_self">Project Zero</a> 团队分享的新研究可知，Linux 开发者在安全漏洞的修复方面，做得较竞争平台都要快。</p><p><img src="https://static.cnbetacdn.com/article/2022/0219/1a2ce8a555c94a8.png" alt="1.png" referrerpolicy="no-referrer"></p><p>回顾 2019 年 1 月 - 2021 年 12 月期间的已修复 Bug 报告，Project Zero 团队发现开源程序员平均只需 25 天来修复 Linux 问题。</p><p>与此同时，Linux 开发者一直在稳步减少修补安全漏洞所需的时间。早在 2019 年，这个数据就已经缩短至一个月。但现在，通常都可在两周内完成。</p><p><img src="https://static.cnbetacdn.com/article/2022/0219/b400ca6f73df1a3.png" alt="2.png" referrerpolicy="no-referrer"></p><p>作为对比，<a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fmusic%2F" target="_blank">苹果</a>平均用了大约 69 天、Google 为 44 天、Mozilla 大约为 46 天。</p><p>尴尬的是，作为当前最流行的家庭 / 办公室用操作系统，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a> <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 平台平均需要将近 3 个月的时间来修复安全漏洞。</p><p><img src="https://static.cnbetacdn.com/article/2022/0219/6dc1114a8c12f28.png" alt="3.png" referrerpolicy="no-referrer"></p><p>最后，这份报告还分析了开发者修复移动操作系统中的安全漏洞所花费的时间。</p><p>可知尽管面临更多安全威胁，Google Android 的补丁发布速度，还是落后于苹果 iOS 。</p><p><img src="https://static.cnbetacdn.com/article/2022/0219/43ecc1c885a6399.png" alt="4.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">三大浏览器平台的 Bug 修复对比</p><p>【背景资料】</p><p>作为 Google 内部安全研究团队，Project Zero 致力于找寻各大平台的安全漏洞，并且通常会给予 90 日的修复缓冲期。</p><p>庆幸的是，所有公司在 Bug 修复方面的表现都在变得越来越快、且越来越好。</p>
