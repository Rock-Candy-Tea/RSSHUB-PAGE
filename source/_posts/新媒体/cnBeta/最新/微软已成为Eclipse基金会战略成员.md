早在 2016 年，微软就以“解决方案成员”的身份，加入了 Eclipse 基金会。此举旨在与 Eclipse 社区开展更紧密的合作，并为所有开发团队提供一套出色的工具与服务。<strong>今天，微软再次拓展了其参与力度，宣布已成为 Eclipse 基金会的最新“战略成员”，以推进对 AISBL 的支持。</strong><br>
<p><img src="https://static.cnbetacdn.com/article/2021/0804/85f0ed844c2c626.png" alt="strategic-members.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">公告（来自：<a href="https://www.eclipse.org/membership/showMember.php?member_id=1196" target="_self">Eclipse</a> | <a href="https://cloudblogs.microsoft.com/opensource/2021/08/03/microsoft-expands-support-with-the-eclipse-foundation/" target="_self">Microsoft</a>）</p><p>与此同时，微软 Azure <a data-link="1" href="https://microsoft.pvxt.net/P0JMe" target="_blank">Office</a> 首席技术官、兼首席项目经理的 Stephen Walli，也将成为 Eclipse 基金会董事成员。</p><p>微软表示，Eclipse 基金会仍是 Java 生态系统的重要基石，而该公司也致力于为 Java 开发者提供相应支持，并维护这一生态系统的健康。</p><p>通过积极参与 Eclipse Adoptium（曾用名 AdoptOpenJDK）等项目，我们已经从微软身上看到了相关努力。</p><p>在成为 Eclipse 基金会战略成员之后，还有助于微软进一步秉承开源精神、来推进 Java 的现代化计划。</p><p><img src="https://static.cnbetacdn.com/article/2021/0804/38a9bee78101004.png" alt="Eclipse Strategic Member.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">Eclipse 基金会现有<a href="https://www.eclipse.org/membership/exploreMembership.php" target="_self">战略成员</a>一览</p><p>Eclipse 基金会还与 Java 社区的核心项目保持着密切联系，包括 Eclipse IDE、Jakarta EE（Java EE 的继任者）、以及托管的 MicroProfile 。</p><p>对于微软及其合作伙伴而言，Eclipse 基金会是继承 AdoptOpenJDK 这一使命的合理选项。</p><p>此外致力于“供应商中立”的多供应商计划，使得 Eclipse Adoptium 将继续成为基于 OpenJDK 源代码，完全兼容、且高质量 Java 运行时分发的领先供应商。</p><p>最后，Eclipse 基金会正在通过各个工作组来拓展其能力，且其中许多工作组都会微软及其合作伙伴有至关重要的帮助。</p><p>比如最近围绕 Eclipse 数据空间连接器和 Eclipse Tractus-X 的工作，就是 Eclipse 基金会在微软有兴趣参与的工作组中，开启的新工作尝试的最新案例。</p>
