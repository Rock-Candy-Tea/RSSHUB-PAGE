<strong>开源软件开发商 GitHub 近日表示，新写的 30% 代码都是在 AI 编程工具 Copilot 的帮助下完成的。</strong>Copilot 可以查看人类程序员编写的代码，并建议进一步的行数或替代代码，从而消除了编码中的一些重复性劳动。<br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/1028/156f99944381833.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1028/156f99944381833.jpg" alt="94zpe3ay.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">Copilot 建立在 OpenAI Codex 算法的基础上，该算法是在数 TB 的公开源代码上训练出来的，可以将人类语言翻译成编程语言。它作为程序员的一个更复杂的自动完成工具使用。</p><p style="text-align: left;">GitHub Next 的副总裁 Oege de Moor 表示：“我们从用户那里听到很多消息，使用 Copilot 后，他们的编程实践发生了变化。总的来说，他们能够在编程中变得更有成效”。</p><p style="text-align: left;">在今天的 GitHub Universe 会议上，GitHub 宣布 Copilot 将支持包括 Java 在内的所有主流编程语言。de Moor 表示：“这将有助于把这项技术带给更多的人。这也是 GitHub 努力让下一个 2 亿开发者使用编程的一部”。</p><p style="text-align: left;">De Moor 还指出，事实证明 Copilot 在社区的基础上很有吸引力：自 7 月推出以来，50% 的尝试过该产品的开发者一直在使用它。与 OpenAI 的大规模文本生成自然语言产品 GPT-3 不同，Copilot 在增强人类工作方面比创造自己的代码要有效得多。</p>
