Silverlight 是微软在 2007 年发布的一款用于开发 Web 应用程序的框架，但最近一次大版本更新已经可以追溯到差不多三年前。<strong>截至目前，旧框架仅在 IE 浏览器上受到支持。但不久后，Silverlight 即将迎来它的大限。</strong>正如 Treo Alhonen 在 Twitter 上指出，微软将于 2021 年 10 月 12 日终止对 Silverlight 的支持。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0914/e0dda06875f8ace.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：Microsoft <a href="https://support.microsoft.com/en-us/windows/silverlight-end-of-support-0a3be3c7-bead-e203-2dfd-74f0a64f1788" target="_self">官网</a>）</p><p>对于关注软件发展历史的朋友们来说，Silverlight 命途多舛的结局并不难以预料。比如作为 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Phone 的主要开发平台，Silverlight 4 就随着它一起被微软扫落历史的尘埃里了。</p><p>其实早在 2011 年，就有前微软高管宣布 Silverlight 已死，而后 Netflix 也于 2013 年从 Silverlight 过渡到了 HTML5 。</p><p>浏览器方面，Google Chrome 于 2015 年 9 月终止了支持，Mozilla Firefox 亦于 2017 年 3 月跟进。</p><p>就连微软自家（末代 IE11 之后）的 Microsoft Edge，也根本就不支持 Silverlight（除非开启 IE 兼容性模式）。</p><p><img src="https://static.cnbetacdn.com/article/2021/0914/66bfd67125088f6.png" referrerpolicy="no-referrer"></p><p>现在，微软已于官方支持页面通报了 Silverlight 将于 2021 年 10 月 12 日终止支持。在此之前，该公司将继续为使用 Silverlight 5 的客户提供付费技术支持。</p><p>涵盖以下平台 IE11 上运行的 Silverlight 应用程序：</p><blockquote><p>● Windows 10</p><p>● Windows Server 2016</p><p>● Windows 8.1</p><p>● Windows Server 2012 R2</p><p>● Windows Server 2012</p><p>● Windows Embedded 8.1 Industry</p><p>● Windows Embedded 8 Standard</p><p>● POSReady 7</p><p>● Thin PC 7</p></blockquote><p><img src="https://static.cnbetacdn.com/article/2021/0914/300408466e1039c.png" referrerpolicy="no-referrer"></p><p>微软强调其并不打算采取强制措施来终结 Silverlight 应用程序。更重要的是，在此框架上运行的软件，将不再获得任何安全或功能更新。</p><p>对于已经购买了 Silverlight 永久许可证的客户来说，仍可在缺少官方支持的情况下继续合法使用，但微软警告称不得再分发 Silverlight 的永久许可证。</p><p>在 2021 年 10 月 12 日之后，所有 Silverlight 客户都将无法获得延长支持 —— 即使你愿意为此付费。Silverlight 的门户网站仍可继续访问，但程序的安装链接将被撤下。</p><p>对于基于 Web 的应用程序，微软已推荐将 Blazor 作为替代方案。但若你正考虑迁移至无需依赖浏览器的桌面因公程序，则微软鼓励使用基于 Windows App SDK 的 WinUI 3 。</p>
