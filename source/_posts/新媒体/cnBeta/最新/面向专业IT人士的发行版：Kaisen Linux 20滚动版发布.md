 <p><a target="_blank" href="https://static.cnbetacdn.com/article/2021/1216/469d037ce775358.png"><img data-original="https://static.cnbetacdn.com/article/2021/1216/469d037ce775358.png" src="https://static.cnbetacdn.com/thumb/article/2021/1216/469d037ce775358.png" referrerpolicy="no-referrer"></a></p><p>同时，最新版本现在基于 Debian Bookworm (Debian 12)，这也是版本号更迭的重要原因。</p><p>更新内容摘要：</p><ul class=" list-paddingleft-2"><li><p>修正了许多错误，目前基于 Debian Bookworm (Debian 12)。</p></li><li><p>在系统菜单中使用了新的、更美观更现代的图标，并进行了简化。</p></li><li><p>使用 LXQT 取代 LXDE。</p></li><li><p>增加了 kaisen-interfaces-common 包。</p></li><li><p>删除了 ISO CONSOLE。</p></li><li><p>使用 XFCE 作为 SR ISO 的默认 GUI。</p></li><li><p>对 <a href="https://linuxstory.org/tag/netinst-iso/">NETINST ISO</a> 进行了大量更改。</p></li><li><p>改进了 <a href="https://linuxstory.org/tag/conky/">Conky</a> 主题在系统上已经安装的分区上的显示。</p></li><li><p>默认配置文件将会随着接口的不同而更新。</p></li><li><p>在基础架构级别重新设计了 <a href="https://linuxstory.org/tag/kaisen-linux/">Kaisen Linux</a> 的存储库。</p></li><li><p>增加和删除了一些工具。</p></li></ul><p>更新内容详情：</p><p>用 LXQT 取代 LXDE</p><p>相比之下，LXQT 更现代，并且与 <a href="https://linuxstory.org/tag/conky/">Conky</a> 等 Kaisen 的配置更兼容。仍然可以安装 LXDE 但是不能通过默认的 ISO 提供或用 <a href="https://linuxstory.org/tag/netinst-iso/">NETINST ISO</a> 安装。当未来 Kaisen Linux 升级到3.0时，kaisen-lxde 元包将不能再安装，并且会从库中删除。当然，会有至少1.5年的时间来过渡。</p><p>增加了 kaisen-interfaces-common 包</p><p>现在的kaisen-interfaces-common 包中安装了对所有借口和切换器接口通用的工具，同时修复了在机器上安装多个 GUI 时的 dpkg 错误。</p><p>删除了 ISO CONSOLE</p><p>删掉了 ISO CONSOLE，现在 ISO SR 将会在控制台模式下自动启动，并且可以用单一命令来按需启动 GUI。</p><p>对 NETINST ISO 进行了大量更改</p><p>NETINST ISO 中的软件安装菜单将支持法语、西班牙语、葡萄牙语、意大利语和德语，界面也会更新来适应不同包的更改和安装。</p><p>如果不是用 NETINST ISO 安装的 GUI，默认用户值将是 Bash，也不会安装 ZSH 配置文件。如果安装了 GUI 或 kaisen-skeleton 包之一，将会为所有用户自动复制默认配置文件，并把 ZSH 设置为默认值。</p><p>所有的默认接口配置文件都会更新，XFCE 也会有一个新菜单（Whisker 菜单），这是一个更现代、直观且易于定制的菜单。</p><p>kaisen-dhcp-tools、kaisen-dns-tools 和 kaisen-tls-tools 元软件包现在会由 kaisen-sysadmin-tools 软件包安装。</p><p>Apparmor 和默认配置文件已经添加到分发中。Apparmor 管理工具和配置文件都默认集成在实时安装中，并能够用于安装在 NETINST ISO 上。</p><p>把 HTTPS 协议作为 NETINST ISO 里下载数据包的默认协议。</p><p>所有 ISO 上对专业安装模式都被重新设计和简化了。移除了一些不与 Kaisen 兼容的加载模块。预设置了内核映像的默认选择和 sources.list 中的 Kaisen 存储库。但很可惜，专业安装模式还是会比经典安装模式更慢。</p><p>最新的 1.X 版本因为不能生成密钥，故不能通过持久实时模式（persistent live mode）启动。在 2.X 版本中博人包含了这个修复，并简化了生成持久可引导密钥的过程，让这个过程更简单安全。</p><p>一些软件已经重新打包以便更新并集成到 Kaisen Linux 中，比如 psensor、kubernetes、qbittorrent 等。</p><p>改进了 Conky 对系统上已安装分区的显示。更新了这个功能的 Python 代码，修复了相关的多个错误，比如当 Docker 被激活或使用 Timeshift 时/分区会消失。同时还支持命名名称中包含空格的分区。</p><p>工具的增改</p><p>新工具：</p><ul class=" list-paddingleft-2"><li><p>增加了 alsa 和 atmel 固件来改进对声音和 wifi 的支持。</p></li><li><p>添加了 etcdctl、mariadb 和 psql 工具。</p></li><li><p>添加了 eksctl、trivy、buildhah、helm、k6、k9s、minikube、podman。</p></li><li><p>添加了 virtualbox-ext-pack。</p></li><li><p>添加了 sysstat 包。</p></li></ul><p>移除的工具们：</p><ul class=" list-paddingleft-2"><li><p>kismet</p></li><li><p>fusesmb</p></li><li><p>lockdown</p></li><li><p>virtualbox-guest-dkms</p></li><li><p>logrotate</p></li></ul><p>删除的包：</p><ul class=" list-paddingleft-2"><li><p>kaisen-cli-tools（被老的 CONSOLE ISO 调用）</p></li><li><p>kaisen-zip-tools（集成在 kaisen-miscellaneous-tools 元包中了）</p></li><li><p>kaisen-mount（集成在 kaisen-filesystems 元包中）</p></li><li><p>kaisen-various-analysis（集成在 kaisen-filesystems 元包中）</p></li></ul><p>重新设计 Kaisen Linux 的存储库</p><p>在基础架构层次重新设计了 Kaisen Linux 存储库，创建了 kaisen-rolling-snapshot 存储库。这个存储库的目的是将 Debian 删除的任何软件包都重新注入 kaisen-rolling 库。这可以保证所有工具始终具有真正的稳定性和可用性。通过 NETINST 安装也将具有同样的优点。存储库的详细工作原理可以点击<a href="https://gitlab.com/kaisenlinux/reprepro/-/blob/master/conf-prod/README">此处</a>查看。</p><p>更新方法</p><p>整个发行版将可以通过 APT 更新，但因为 2.0 版本的改动较多，一些操作需要手动进行：</p><p>重新安装所有工具：</p><p>sudo kaisen-update && sudo apt install kaisen-update kaisen-btrfs-snapshot kaisen-sysops-tools</p><p>如果您不为分区使用 BTRFS 文件系统的话：</p><p>sudo kaisen-update && sudo apt install kaisen-update kaisen-sysops-tools</p><p>虽然不强制，但也建议使用 kaisen-update-skel 命令来将 /etc/skel 的内容放在用户的目录里。缺点是会失去个性化的桌面环境。</p><p>其他的具体更新方法可以点击<a href="https://kaisenlinux.org/blog/kaisenlinuxrolling2.0.php">官方文档</a>来查看。</p><p>出现问题可以在此处提供的<a href="https://forum.kaisenlinux.org/">论坛</a>上报告错误。</p><p>参考资料：</p><p><a href="https://kaisenlinux.org/blog/kaisenlinuxrolling2.0.php">Kaisen Linux Rolling 2.0 Release Notes</a></p>
