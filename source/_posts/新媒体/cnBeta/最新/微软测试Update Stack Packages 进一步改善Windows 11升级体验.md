<strong>微软宣布启用 Update Stack Packages 方式，进一步改善 Windows 设备的升级体验。</strong>服务栈负责安装操作系统更新的 Windows 部分，而微软会定期发布新的服务堆栈更新，从而改善 Windows Update 的升级。<br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/1015/bce010a62925670.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1015/bce010a62925670.jpg" alt="m5bb30xa.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">目前，这种新方法只适用于 Dev 渠道中少量用户。微软解释说：“目前，Update Stack Packages 仅限于一套非常小的、更新相关的系统文件，这些文件是独立于操作系统开发的。通过首先在 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Insiders 中测试这一过程，我们希望在未来扩大范围和发布频率”。</p><p style="text-align: left;">微软继续说道：“通过 Update Stack Packages，我们可以在 PC 采取每月或功能更新之前提供更新体验的改进。Update Stack Packages 将有助于确保你的个人电脑有最大的可能成功安装新的更新，并获得最佳和最少的破坏性体验”。</p><p style="text-align: left;">微软表示，Update Stack Packages 会像传统的累积更新一样出现在 Windows Update 列表中，至于该公司计划何时开始在全球范围内向生产设备推广，还没有时间表，但在这一点上，很明显它处理这种新方法相当谨慎。</p>
