<strong>在 Xbox Series S / X 游戏主机上率先引入的 DirectStorage API，将以开发者预览的形式、于今夏登陆 PC 平台。</strong>据说上周五的时候，微软已经向签订了保密协议的 PC 游戏开发者提供了首个预览套件。有趣的是，这家软件巨头已放弃“仅在 Windows 11 平台上可用”的限制。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0720/cd1e5a448a360e9.jpg" referrerpolicy="no-referrer"></p><p>项目经理 Hassan Uraizee 解释称：</p><blockquote><p>我们希望确保当游戏开发者采用新 API 时，能够接触到尽可能多的游戏玩家。</p><p>因此针对 DirectStorage SDK 构建的游戏，将与 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 10 1909 及更高版本的操作系统兼容，并与 DirectX 12 Agility SDK 保持一致。</p></blockquote><p>至于 DirectStorage API 的功能，主要有以下几点：</p><blockquote><p>● 新的 DirectStorage API 编程模型 -- 提供 DX12 风格的批量提交 / 完成调用模式，使应用程序无需每秒单独管理数千个 IO 请求 / 完成通知；</p><p>● GPU 解压缩 -- 为流媒体等场景提供超快的资源解压体验（稍后提供预览）；</p><p>● 存储堆栈优化 -- Windows 11 平台将迎来系统级存储堆栈升级，以释放 DirectStorage 的全部潜力。</p><p>此外即使在 Windows 10 平台上，受支持的游戏仍可受益于更有效地调用操作系统的存储堆栈。</p></blockquote><p>WCCFTech 指出，这意味着任何基于 DirectStorage 构建的游戏，都将受益于 Windows 10 1909 及更高版本上的新编程模型和 GPU 解压缩技术。</p><p>当然，在构建之初就已经充分考虑了 DirectStorage 特性的 Windows 11 操作系统，还是能够获得进一步的存储堆栈优化体验。</p><blockquote><p>该 API 的运行时（Runtime）和 GPU 解压缩技术，都通过 DirectStorage SDK 与游戏一起提供。</p><p>作为游戏开发者，你只需在引擎中单次部署，即可自动为游戏玩家提供适当的扩展。</p></blockquote><p>最后，尽管许多游戏都需要较长一段时间才能用上这个输入 / 输出 API，但《地铁》开发商 4A Games 认为，新功对整个 PC 游戏行业都具有相当积极的意义。</p>
