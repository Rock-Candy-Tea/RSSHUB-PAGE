<strong>随着微软正式发布 .NET 6，David Ramel 也于第一时间在 Visual Studio Magazine 上，介绍了这一开发框架的最新变化。</strong>从 2017 年的 .NET Core 2.0 开始，微软开始将之作为 .NET Framework 的开源、跨平台替代品。然后去年的时候，微软去掉了名称中的“Core”单词，于是现在就直接称作 .NET 5 和 .NET 6 。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/1109/b98cafb63fa973b.png" alt="0.png" referrerpolicy="no-referrer"></p><p>随着最近引入的机器学习（AI）与浏览器 WebAssembly 应用，.NET 平台现也能够用于 Web、移动、桌面、游戏、以及物联网（IoT）等项目。</p><p>其实早在一年前，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>就已经想要在 .NET 5 上实现大一统，但 COVID-19 大流行拖慢了这一进度，且开发团队一直被某些问题所困扰。</p><p>随着 .NET 6 的正式发布，微软仍落下了一些需要继续完善的功能组件，比如移动平台上的 Xamarin.Forms 演变 —— .NET 多平台应用程序用户界面（.NET MAUI）。</p><p>其原本是要与 iOS 和 Android 一起作为应用程序目标而添加的，但最终时间表还是被拖到了比 .NET 6 发布稍晚的时候。</p><p>对于桌面开发者来说，此前被称作 Project Reunion 的 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> App SDK，旨在减轻因出现两组不同的 API（Win 32 和 UWP）而导致的应用程序开发的复杂性。</p><p><a href="https://static.cnbetacdn.com/article/2021/1109/6f629d6dc552939.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1109/6f629d6dc552939.png" alt="1.PNG" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（图 via <a href="https://themesof.net/" target="_self">Themes of .NET</a>）</p><p>在一篇全文超过 16000 字的冗长<a href="https://devblogs.microsoft.com/dotnet/announcing-net-6/" target="_self">博客文章</a>中，<strong>.NET 项目经理 Richard Lander 提到了本次更新的如下亮点：</strong></p><blockquote><p>● 巨大的性能提升；</p><p>● 针对 C# 10 和 F# 6 编程语言的支持与改进；</p><p>● 原生支持 Apple Silicon（Arm64）平台；</p><p>● 借助 dotnet monitor 与 OpenTelemetry 改进云诊断；</p><p>● 引入新的动态配置文件引导优化（PGO）系统，旨在为运行时（Runtime）提供深度优化。</p></blockquote><p><a href="https://static.cnbetacdn.com/article/2021/1109/e7db89194146554.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1109/e7db89194146554.png" alt="2.PNG"
