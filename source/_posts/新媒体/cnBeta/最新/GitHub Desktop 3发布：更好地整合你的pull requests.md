<a href="https://github.blog/2022-04-26-github-desktop-3-0-brings-better-integration-for-your-pull-requests/" target="_blank"><strong>GitHub Desktop 3</strong></a><strong> 于今天正式发布，</strong>它是专门为帮助简化开发者的工作流程而设计的。在本次更新中增强了多项现有功能，包括压制、重新排序或修改提交，大幅改善 Git 操作。
本次版本更新的核心增强功能之一就是可以审查你 pull requests 的状态，这反过来帮助开发者确认代码是否确实准备好用于生产。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/ca276c37ff5c198.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">官方博文中写道：“只要点击带有 pull request 编号的徽章，就可以开始潜入检查、工作和步骤，以更好地了解和解决你可能遇到的任何问题。你仍然找不到检查失败的原因？别担心！你可以重新运行你的检查，以解决这个问题。你可以重新运行你的检查，给他们另一个成功的机会。用户现在还可以重新运行失败的或个别的 GitHub Action 检查”。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/400aaf4b0618e23.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/0aa751d61e522e3.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/5c9875272b099a0.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/231b2a00a4fcb2d.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/ca5d2d6811790ed.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/66a11ed22856257.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0428/30bd7ab81221c60.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">GitHub Desktop 3.0 还提供了对高信号通知的支持。从本质上讲，这将帮助用户筛除不断弹出的众多通知，相反，它将只提醒用户在用户所在的仓库中发生的重要通知。</p><p style="text-align: left;">开发人员也将能够在失败的拉取请求检查中抢先一步，因为GitHub Desktop 3会在提交失败时通知他们。它将用户重定向到一个新的对话框，详细说明发生了什么。作为回报，这使得开发人员更容易确定问题所在并加以纠正。</p>
