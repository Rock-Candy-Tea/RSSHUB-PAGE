Microsoft Edge 93 已进入稳定频道。新版本带来了全新的标签组功能，使其能够轻松地将浏览器标签组织成不同的组。此外优化了垂直标签功能（将所有打开的标签移动到网页浏览器左侧的窗格中），有了隐藏标题栏的新选项。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2021/0903/57a8ac89b8a9c9c.jpg" alt="QQ截图20210903080342.jpg" referrerpolicy="no-referrer"></p><p style="text-align: left;">本次更新还带来了在支持的视频上悬停时在画中画窗口中查看视频的功能，不过这个功能目前只在 macOS 上可用。微软 Edge 93 版还带来了各种安全更新和 IE 模式的小变化，你可以查看下面的完整功能更新列表。</p><p style="text-align: left;"><strong>● 初始偏好（Initial Preferences）</strong></p><p style="text-align: left;">微软 Edge 现在支持数量有限的初始首选项（此前为 Master Preferences）。IT 管理员可以在用户首次运行浏览器之前将这些设置部署为默认设置。使用初始偏好设置配置 Microsoft Edge，以便首次运行。</p><p style="text-align: left;"><strong>● IE 模式支持“no-merge”行为</strong></p><p style="text-align: left;">对于终端用户来说，当一个新的浏览器窗口从 IE 模式应用中启动时，它将处于一个单独的会话中，类似于 IE11 中的无合并行为。你需要调整你的网站列表，将需要防止会话共享的网站配置为“no-merge”。</p><p style="text-align: left;">在幕后，对于 Microsoft Edge 的每个窗口，在该窗口内第一次访问 IE 模式标签时，如果是指定的“no-merge”网站之一，该窗口将被锁定在与所有其他Microsoft Edge窗口不同的“no-merge”IE会话中，至少到该窗口中最后一个IE模式标签关闭为止</p><p style="text-align: left;">这是继以前的行为之后，用户可以启动IE的免合并，也可以通过其他机制启动 Microsoft Edge 的“no-merge”。<a href="https://docs.microsoft.com/en-us/deployedge/edge-ie-mode-faq#does-ie-mode-on-microsoft-edge-support-the--nomerge--option-that-was-supported-in-internet-explorer-11-" target="_blank">IE 模式的故障排除和常见问题</a> | <a href="https://docs.microsoft.com/en-us/deployedge/edge-ie-mode-faq#does-ie-mode-on-microsoft-edge-support-the--nomerge--option-that-was-supported-in-internet-explorer-11-" target="_blank">微软文档</a></p><p style="text-align: left;"><strong>● 禁止隐式登录的新策略</strong></p><p style="text-align: left;"><a href="https://docs.microsoft.com/en-us/deployedge/microsoft-edge-policies#implicitsigninenabled" data-linktype="absolute-path" target="_blank" rel="noopener">ImplicitSignInEnabled</a> 策略允许系统管理员在 Microsoft Edge 浏览器上禁用隐式登录。</p><p style="text-align: left;"><strong>● 绕过ClickOnce和DirectInvoke提示的策略</strong></p><p style="text-align: left;">微软更新了策略，以便绕过 ClickOnce 的提示和 DirectInvoke 的应用程序，用于指定的文件类型，来自指定的域。要做到这一点，你将需要。</p><p style="text-align: left;">→ 启用 <a href="https://docs.microsoft.com/en-us/deployedge/microsoft-edge-policies#clickonceenabled" data-linktype="absolute-path" target="_blank" rel="noopener">ClickOnceEnabled</a> 或者 <a href="https://docs.microsoft.com/en-us/deployedge/microsoft-edge-policies#directinvokeenabled" data-linktype="absolute-path" target="_blank" rel="noopener">DirectInvokeEnabled</a></p><p style="text-align: left;">→ 启用 <a href="https://docs.microsoft.com/en-us/deployedge/microsoft-edge-policies#autoopenfiletypes" data-linktype="absolute-path" target="_blank" rel="noopener">AutoOpenFileTypes</a> 策略并设置 ClickOnce 和 DirectInvoke 应被禁用的特定文件类型列表</p><p style="text-align: left;">→ 启用 <a href="https://docs.microsoft.com/en-us/deployedge/microsoft-edge-policies#autoopenallowedforurls" data-linktype="absolute-path" target="_blank" rel="noopener">AutoOpenAllowedForURLs</a> 策略，并设置 ClickOnce 和 DirectInvoke 应被禁用的特定域的列表。</p><p style="text-align: left;">注意：AutoOpenAllowedForURLs 是 AutoOpenFileTypes 的一个支持者策略。如果没有设置 AutoOpenAllowedForURLs，而设置了AutoOpenFileTypes，那么列出的文件类型将自动从所有 URL 打开。</p><p style="text-align: left;"><strong>● 标签组</strong></p><p style="text-align: left;">它提供了将标签归类为用户定义的组的能力，并帮助你更有效地在多个工作流中寻找、切换和管理标签。</p><p style="text-align: left;"><strong>● 垂直标签页隐藏标题栏</strong></p><p style="text-align: left;">在垂直标签页中，通过隐藏浏览器的标题栏，获得额外的几个像素的回报。现在你可以去 edge://settings/appearance 页面，在自定义工具栏部分选择在垂直标签模式下隐藏标题栏的选项。</p><p style="text-align: left;"><strong>● 悬停工具条上的视频画中画（PiP）</strong></p><p style="text-align: left;">当你把鼠标悬停在一个支持的视频上时，会出现一个工具栏，允许你在画中画窗口中查看该视频。请注意：目前这只适用于MacOS上的Microsoft Edge用户。</p><p style="text-align: left;"><strong>● 移除 TLS 中的 3DES</strong></p><p style="text-align: left;">对 TLS_RSA_WITH_3DES_EDE_CBC_SHA 密码套件的支持将被移除。这一变化发生在Chromium项目中，Microsoft Edge是基于该项目。欲了解更多信息，请浏览<a href="https://chromestatus.com/feature/6678134168485888" target="_blank">Chrome平台状态条目。</a></p><p style="text-align: left;">此外，在微软Edge 93版本中，<a href="https://docs.microsoft.com/en-us/deployedge/microsoft-edge-policies#tripledesenabled" data-linktype="absolute-path" target="_blank" rel="noopener">TripleDESEnabled</a> 策略将可用于支持需要保留与过时服务器的兼容性的场景。这个兼容性策略将变得过时，并在Microsoft Edge 95版中停止工作。确保你在那之前更新受影响的服务器。</p>
