面向那些希望更新更加自动化的企业，微软在今年 4 月推出了 Windows Autopatch 服务。不过自该服务上线以来，围绕着新闻稿中某些短语的使用，在媒体和社区中产生了很多的混乱。<br>
 <p style="text-align: left;">在 Windows Autopatch 服务上线之后，不少媒体解读为“补丁星期二”活动的结束。不过随后<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>进行了澄清，表示“补丁星期二”活动仍将继续，而对于那些喜欢自动更新部署的公司来说，Autopatch 基本上是一种免费的自动化方法。</p><p style="text-align: left;">今天早些时候，Microsoft 总监 Jeremy Chapman 还分享了一个有用的 PSA，通过流程图说明 Autopatch 服务将如何与 Microsoft 的 Endpoint Manager 和 Windows Update for Business 协同工作以部署自动更新。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0708/300a658605892ab.webp" alt="5qtxalqu.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">对于 Autopatch 服务如何部署 Windows Update，官方解释如下：</p><p style="text-align: left;">Windows Update</p><p style="text-align: left;">Windows Autopatch 进行 Windows 更新的来源是 General Availability Channel。可以独立设置质量和功能更新策略以满足您的需求。</p><p style="text-align: left;">管理员将能够通过 Endpoint Manager 中的 Windows Autopatch 消息中心查看已应用的更新，并将了解预期的更新。每月 Windows 安全和质量更新（也称为“B”版本）的熟悉节奏将继续，并将根据需要应用带外更新。</p><p style="text-align: left;">该服务目前处于公开预览阶段，预计将于本月推出。</p>
