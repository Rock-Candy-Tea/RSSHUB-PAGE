<strong>在今年的 Facebook Connect 大会期间，这家社交网络巨头展示了名叫 Presence 的新平台。</strong>以期为 Oculus Quest 虚拟现实（VR）头显的开发者们，提供一种打造混合现实（MR）体验的新方法。其中包括了一套面向开发者的人工智能（AI）工具包，旨在帮助打造更具沉浸感的环境，有助于让虚拟世界和物理现实相结合。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/1029/777f1b0627a75ec.jpg" referrerpolicy="no-referrer"></p><p>Presence Platform 包含了三款主要工具集，分别是 Insight SDK、Interaction SDK、以及 Voice SDK 。</p><blockquote><p>据悉，Insight 建立在今夏推出的 Passthrough API 之上，允许开发人员在 Oculus Quest 的 passthrough 视频源上叠加虚拟对象。</p><p>Facebook 还引入了一个名为 Spatial Anchors 的系统，以便开发者将虚拟对象‘锁定’到特定的物理区域（对应其在混合现实环境中的位置）。</p></blockquote><p>需要指出的是，尽管 Passthrough API 为 Oculus Quest 2 独占，但新平台却同时提供了对两代 Quest 头显的支持。</p><p>此外场景感知（Scene Understanding）填补了 Insight SDK 的最后一环，其允许开发者构建与玩家房间规模相匹配的 VR 体验。</p><p>Facebook 举例称，你可以将虚拟屏幕附加到用户家的墙上、或者让虚拟角色在地板上以假乱真地导航。</p><p><img src="https://static.cnbetacdn.com/article/2020/0917/a7a9c7d5fbc8847.jpg" referrerpolicy="no-referrer"></p><p>第二款工具集（Interaction SDK）允许开发者轻松集成基于控制器和手势的交互体验，且 Facebook 计明年开放 Unity 库，以便开发者整合现有的交互元素。</p><p>除了提供改进的手部跟踪体验（比如抓取、手戳、选择目标等），它还允许开发者创建自定义手势。</p><p>至于第三款工具集（Voice SDK Experimental），其由 Facebook 的 Wit.ai 自然语言平台提供支撑，能够为用户带来自由的虚拟体验、且允许开发者创建语音导航和搜索功能。</p><p>基于此，玩家能够获得语音提问的方式得到常见问题的解答。而“语音驱动”的游戏，更是可以让用户用自己的声音来玩耍（比如与化身角色交谈、或释放咒语）。</p><p>最后，最近有关“元宇宙”（Metaverse）的讨论有些密集，且 Facebook 也希望成为一家此类企业。</p><p>该公司称，Presence 将是让开发者将 VR 内容与现实世界融合到一起，从而让他们在元宇宙中获得联系的一个关键平台。</p>
