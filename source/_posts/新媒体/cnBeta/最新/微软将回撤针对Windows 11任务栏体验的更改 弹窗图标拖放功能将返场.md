随着“视窗即服务”（Windows as a Service）理念的不断发展和演变，微软会定期在预览版本中对新设计理念展开测试。<strong>比如早些时候，Windows 11 预览版中就禁用了一项关键的任务栏功能，使得测试者们无法欢快地在弹出窗口中拖放。</strong>由正在向用户推送的一项新更改可知，该公司的想法总是让人捉摸不透。<br>
<p style="text-align: center;"><a href="https://static.cnbetacdn.com/article/2022/0506/e24e462e28aba8f.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0506/e24e462e28aba8f.jpg" alt="1.jpg" referrerpolicy="no-referrer"></a></p><p><a href="https://www.windowslatest.com/2022/05/06/microsoft-backtracks-on-windows-11s-unwanted-taskbar-changes/" target="_self">Windows Latest</a> 指出：几周前，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>对任务栏进行了修改，是的系统托盘弹出窗口中的图标难以被重新排序（显示隐藏的图标）。</p><p>官方反馈中心中的一篇帖子证实，任务栏拖放功能已被移除，理由是为了优化平板电脑的使用体验。</p><blockquote><p>● 一位官方人员答道：随着在 Build 22563 更新中带来针对平板电脑的专项优化，我们不再支持于系统托盘 / 显示隐藏图标的弹出窗口之间拖动图标。</p><p>● 然而微软后来又澄清说，这项变动确实针对 Windows 11 的平板电脑任务栏使用体验而优化，但它也影响到了桌面端的用户体验。</p></blockquote><p>该公司态度的反复，其实并不难理解，毕竟平板和桌面生态有着天生的隔阂。庆幸的是，在被用户严厉批评之后，微软正在回撤其对桌面任务栏体验做出的这项更改。</p><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0506/cf7458c4c7eec90.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p>从 Windows 11 Build 22616 开始，任务栏系统托盘上的功能，将回归该操作系统的原始版本。如上图所示，新预览版本又可以在任务栏 / 系统托盘弹窗中自由拖动图标了。</p><p>此外基于广大用户的反馈，微软还回撤了针对平板电脑做出的 UI 改进，且不会随 Windows 11 22H2 大版本更新一起提供。</p><p>回顾这家软件巨头的历史，几乎每隔一段时间就会搞出惹恼用户的幺蛾子。而在影响 Windows 11 操作系统普及的诸多因素中，也少不了开发团队莫名其妙引入的某些更改或限制。</p><p>当然，就算有这次妥协，也不意味着微软就会放弃给平板电脑的桌面提供更多专属优化。但愿该公司别忘了当年在 Windows 8 操作系统上的市场失利。毕竟只有多听取用户的反馈，才不至于一次次重走这样的弯路。</p>
