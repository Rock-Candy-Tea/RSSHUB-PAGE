在早前的发布会上，微软证实 Windows 11 操作系统将在支持 Linux 子系统（WSL）的同时，也提供 Android 子系统（WSA）。<strong>后者兼容的 Android 应用程序，将可通过 Microsoft Store 进行获取。但实际下载过程，其实还得借助亚马逊的应用商店。</strong><br>
 <p><a href="https://static.cnbetacdn.com/article/2021/0717/6fa23d3dbe8ac1c.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0717/6fa23d3dbe8ac1c.jpg" referrerpolicy="no-referrer"></a></p><p>目前<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>正在将 Amazon Appstore 集成到已提供 Android 应用程序的 Microsoft Store 中。</p><p>在通过 Microsoft Store 检索 Android 应用之后，可点击产品列表、以打开里面的 Amazon Appstore，然后轻点几下完成应用的安装。</p><p>需要指出的是，6 月下旬，谷歌证实开发者将需要遵循新的 Android App Bundle 标准（简称 AAB），而不是 Android Application Package（即 APK）打包方式。</p><p>这意味着今后的谷歌官方 Play 商店将逐渐告别 APK 格式，尽管这一变化不会马上到来，但 AAB 或最终取代它、成为标准的 Android 应用发布格式。</p><p><a href="https://static.cnbetacdn.com/article/2021/0717/f5e7a27c8bba63c.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0717/f5e7a27c8bba63c.jpg" referrerpolicy="no-referrer"></a></p><p>好消息是，Amazon Appstore 也正在积极努力地支持新格式，以提供更小的下载安装包和按需下载等 AAB 新体验。</p><p>与 Play 商店不同，亚马逊并不强制要求遵循 App Bundle 。但若开发者希望在谷歌官方和第三方应用商店之间轻松分发相同的安装包，对 AAB 的支持也是相当讨喜的。</p><p>此外在开发者许可的情况下，Amazon Appstore 中提供的大多数应用程序，都可在 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 操作系统上自动运行。</p><p>最后，微软已确认了对 APK 安装包的侧载支持，且 Windows 11 也可能迎来对 AAB 的侧载支持，只是后者的实施过程可能会稍显复杂。</p>
