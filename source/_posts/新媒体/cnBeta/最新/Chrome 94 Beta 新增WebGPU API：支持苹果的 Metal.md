<strong>Chrome 94 Beta 本周发布，除了常规改进之外，本次更新还添加了对 WebGPU API 的支持。</strong>在 Chrome 94 正式版中将会替代 WebGL，并支持访问苹果的 Metal API。正如Google在一篇博客文章中所描述的 ，WebGPU 是一种新的、更高级的 Web 图形 API， 能够访问 GPU 硬件，从而为网站和 Web 应用程序中的渲染界面带来更好的性能。<br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0831/10fe8576d94a11e.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0831/10fe8576d94a11e.jpg" alt="jeqrth3n.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">和其他基于 Web 的图形加速 API，WebGPU API 的主要区别在于它基于设备的原生技术，例如<a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fmusic%2F" target="_blank">苹果</a>的 Metal、<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>的 Direct3D 或开放的 Vulkan 标准。这将使 Web 开发人员更容易创建具有更密集图形的 Web 应用程序和游戏。</p><p style="text-align: left;">WebGPU API 是 Web 的 WebGL 和 WebGL2 图形 API 的继承者。它提供了现代功能，例如 “GPU 计算”以及对 GPU 硬件的较低开销访问以及更好、更可预测的性能。这是对现有 WebGL 界面的改进，这些界面是为绘制图像而设计的，但只能付出很大努力才能重新用于其他类型的计算。</p><p style="text-align: left;">Metal 是 Apple 于 2014 年推出的 API， 它为 iOS、macOS 和 tvOS 应用程序提供对 GPU 硬件的低级访问。换句话说，应用程序可以在不使 CPU 过载的情况下访问 GPU， 这是 OpenGL 等旧 API 的限制之一。</p><p style="text-align: left;">正如 The Verge 所指出的，开发人员可能需要一些时间才能将新的 WebGPU API 应用到他们的 Web 项目中，因为它仍然被认为是一个实验性的功能。Google表示，预计在 2022 年初之前，所有 Chrome 用户都不会默认启用 WebGPU。</p>
