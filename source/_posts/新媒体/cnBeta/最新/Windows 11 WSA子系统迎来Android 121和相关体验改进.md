XDA-Developers 报道称：<strong>Windows 11 的 Insider 预览成员们，现已能够在 Windows Android 子系统（简称 WSA）中体验 Android 12.1（又称 Android 12L）。</strong>几周前，相关特性已在 Dev 通道中首次提供。其中一项重要的改进，就是支持高级网络（Advanced Networking）。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0603/f26a9f898b89801.webp" alt="0.webp" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：<a href="https://www.xda-developers.com/windows-insiders-try-android-12-1-wsa/" target="_self">XDA-Developers</a>）</p><p>首先，随着 2204.40000.19.0 版本的到来，<a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 Android 子系统上运行的 Android 应用程序，现已能够连接到与用户 PC 处于同一物理网络上的其它设备，比如扬声器或安防摄像头。</p><p>其次，使用摄像头的应用程序，现已能够根据摄像头的物理安置方向来正确显示，同时修复了黑边 / 失真等问题。此外改进了键鼠输入输入支持，辅以对 Windows 通知的更好集成。</p><p><a href="https://static.cnbetacdn.com/article/2022/0603/841dde62ebf563f.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0603/841dde62ebf563f.png" alt="1.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">资料图（来自：Microsoft <a href="https://docs.microsoft.com/en-us/windows/android/wsa/" target="_self">官网</a>）</p><p>除了 Android 12.1，本次 Windows 11 WSA 更新还包括了最新版本的 Chromium WebView 支持、以及全新的设置应用程序。</p><p>通过改进分组设置的 UI 设计，新设置应用能够更加轻松地找到用户所需的内容。并且允许强制不可调节应用的大小，或有助于在特定场景下测试 App 的行为方式。</p><p><a href="https://static.cnbetacdn.com/article/2022/0603/5b199980dd03b6e.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0603/5b199980dd03b6e.png" alt="2.png" referrerpolicy="no-referrer"></a></p><p>需要指出的是，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>默认关闭了适用于 WSA 子系统的诊断数据。如有需要，你可在设置应用中查看所有诊断数据。不过我们还是建议大家在设置中重新开启，以帮助微软改善未来的使用体验。</p><p>最后，想要获取适用于当前 Windows 平台的 Android 子系统的最新版本，还请移步至 Microsoft Store 资源库并手动检查更新。</p>
