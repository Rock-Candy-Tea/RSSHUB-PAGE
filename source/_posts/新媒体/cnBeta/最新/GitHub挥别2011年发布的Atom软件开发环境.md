<strong>在周三的一篇博文中，知名代码托管平台 GitHub 宣布了 Atom 软件开发环境已走到终点。</strong>作为 GitHub 在 2011 年推出的软件开发文本编辑器，该平台将于 2022 年 12 月 15 日归档 Atom 存储库和社区中剩余的所有其它资源。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0609/130f182a5748acf.webp" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：GitHub <a href="https://github.blog/2022-06-08-sunsetting-atom/" target="_self">Blog</a>）</p><p>据悉，作为 Electron 框架的基础，Atom 为包括<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a> <a data-link="1" href="https://microsoft.pvxt.net/zaZYr" target="_blank">Visual Studio</a> Code、Slack 和 GitHub Desktop 在内的数千款应用程序提供了支撑。</p><p>但是近年来，随着更多新工具的出现，GitHub 发现 Atom 社区的参与度有所下降。而在过去几个月里，除了简单的维护和安全更新，Atom 本身也没有看到任何重大的功能开发。</p><p><strong>GitHub 员工在题为《Sunsetting Atom》的文中写道：</strong></p><blockquote><p>当在 2011 年推出 Atom 时，我们想着为开发者提供一个可深度定制、同时方便而更多人构建软件的文本编辑器。</p><p>GitHub 始终将可靠、安全与性能铭记于心，但为了更好地服务于开发者社区，我们正在归档 Atom、以优先支持面向未来的软件开发技术。</p></blockquote><p><img src="https://static.cnbetacdn.com/article/2022/0609/9f43cd567978ba8.png" referrerpolicy="no-referrer"></p><p>GitHub 表示，未来它将重新专注于 Microsoft Visual Studio Code (VS Code) 和 GitHub Codespaces —— 后者也是该公司力推的云驱动型开发环境。</p><p>与此同时，GitHub 意识到仍有不少社区内开发者在使用 Atom，并承认需要耗费一定的时间和精力，才能完成向替代解决方案的迁移工作。</p><p>让人感到唏嘘的是，微软于 2015 年推出的 VS Code，起初也是对 Atom 的一个回应。</p><p>然后这家软件巨头于 2018 年收购了 GitHub，并将 Atom 与 VS Code 深度融合到了一起，期间 VS Code 的受欢迎程度一直在保持着增长。</p><p>Stack Overflow 的 2021 年度开发者调查结果表明，仅 13% 开发者将 Atom 当做主力开发环境，而 VS Code 却高达 71% 。</p><p>当然，GitHub 不会就此冻结 Atom 生态。完成了相关文档的文档工作之后，有需要的开发者们，仍可随时回看和检查他们的构建。</p><p>此外作为该项目的核心贡献者之一，Max Brunsfeld 正在牵头推动 Atom 的精神续作 Zed（定于本周进入 Alpha 封测）。</p>
