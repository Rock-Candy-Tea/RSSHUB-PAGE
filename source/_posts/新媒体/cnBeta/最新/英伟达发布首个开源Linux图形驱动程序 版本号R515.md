<strong>英伟达刚刚表示，从 R515 驱动程序开始，其已将 Linux GPU 内核模块发布为具有 GPL / MIT 双重许可证的开源版本。</strong>感兴趣的开发者们，可前往 GitHub 上的 NVIDIA Open GPU Kernel Modules 资源库，以获取这些内核模块的源码。<br>
<p><img src="https://static.cnbetacdn.com/article/2022/0512/5bae6cb905d6395.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p>该版本开源驱动旨在通过与操作系统更紧密地集成，以改善 Linux 平台上的 Nvidia GPU 使用体验、还有开发人员的调试和反馈等各项重要工作的开展。</p><p>对于 Linux 发行版的开发团队来说，开源模块极大地增强了代码的易用性。同时通过家简化 Nvidia GPU 驱动程序的开发，它还改进了终端用户的开箱即用体验。</p><p>在首批支持的发行版中，Canonical Ubuntu 和 SuSE Linux Enterprise Distributions 就已经将 Nvidia GPU 的开源驱动程序打包了进去。</p><p><a href="https://static.cnbetacdn.com/article/2022/0512/5a9db96e0fd4d41.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0512/5a9db96e0fd4d41.png" alt="2.png" referrerpolicy="no-referrer"></a></p><p>有需要的开发者，可借此追踪代码路径、查看内核事件角度与工作负载的交互，从而加速底层调试的过程。</p><p>此外对于企业软件开发者来说，他们现可将驱动程序无缝整合到为其项目专门配置的定制 Linux 内核中。</p><p>后续通过参考 Linux 最终用户社区的相关意见和评论，我们还有望进一步提升 Nvidia GPU 开源驱动程序的质量和安全性。</p><p><a href="https://static.cnbetacdn.com/article/2022/0512/e8f7387f61a8fea.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0512/e8f7387f61a8fea.png" alt="3.png" referrerpolicy="no-referrer"></a></p><p>之后每次发布新版开源 GPU 驱动程序的时候，英伟达将都会向 <a href="https://github.com/NVIDIA/open-gpu-kernel-modules" target="_self">GitHub</a> 代码托管平台上传源码快照。至于社区提交的补丁，也可在审核通过后被集成到未来版本的驱动程序中。</p><p>有意贡献的开发者们，还请参阅 Nvidia 的官方指南、驱动发布时间表 / 生命周期文档上的概述。至于首个开源的 GPU 内核模块（R515 版本），则是作为 <a href="https://developer.nvidia.com/cuda-downloads" target="_self">CUDA Toolkit 11.7</a> 套件的一部分而到来。</p><p>当然你也可以前往 Nvidia 官网，以获取“Beta”测试版本的开源图形驱动程序（<a href="https://www.nvidia.com/en-us/drivers/unix/" target="_self">传送门</a>）。</p>
