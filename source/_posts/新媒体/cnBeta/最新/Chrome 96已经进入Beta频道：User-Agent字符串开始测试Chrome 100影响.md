<strong>在 Chrome 95 稳定版正式发布之后，Google 将开发的重心转移到下个版本中。今天，Chrome 96 已经进入到 Beta 频道版本中。</strong>明年 Chrome 将迎来第 100 个版本，为确保平稳过渡，Chrome 96 Beta 版增加了一个新的实验 Flag，强制将主要版本已经宣传为第 100 个版本。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2021/1022/bbde0b690d53ba1.jpg" alt="dvdyq8am.jpg" referrerpolicy="no-referrer"></p><p style="text-align: left;">这个新的面向开发者的选项将把 Chrome 浏览器的 HTTP 用户代理字符串设置为 Chrome 100，以帮助开发者根据这个三位数的版本号测试他们的网站/网络应用。由于一些特别老的脚本可能只是检查两个主要数字，因此Google开发人员提前添加了这个选项，以帮助捕捉任何可能无法正确处理三位主要版本号的区域。</p><p style="text-align: left;">Chrome 96 Beta 的一个有趣的变化是通过一个允许自动/低/高值的“重要性”属性引入了优先提示。优先级提示是用来帮助设置正在加载的资源的重要性。</p><p style="text-align: left;">Chrome 96 Beta 版还为桌面带来了后向缓存、CSS的“:autofill”伪类、字体合成 CSS 属性、新的媒体查询“prefers-contrast”，以及各种 WebAssembly 改进。</p><p style="text-align: left;">关于 Chrome 96 Beta 版变化的更多细节，请参见 <a href="https://blog.chromium.org/2021/10/chrome-96-beta-conditional-focus.html" target="_blank">Chromium.org 博客</a>。关于 Chrome 96 新功能的更多细节，可以在 <a href="https://chromestatus.com/features#milestone%3D96" target="_blank">ChromeStatus.com</a> 上找到。Chrome 96.0 稳定版将于 11 月 16 日左右发布。</p>
