在上周发布了 Chrome 90 之后，谷歌周四又推出了定于下月到来的 Chrome 91 的新 Beta 测试版本。<strong>Phoronix 指出，Chrome 91 Beta 将默认启用 WebTransport 和 WebAssembly SIMD 这两项实验性功能。</strong>前者是一套全新的协议框架 / 标准，适合与远程服务器展开安全的多路传输通信。后者则是 WebSockets 的一个替代方案，并提供了类似 UDP 的数据报双向通信 API 。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0423/55b33c7d377e4eb.png" referrerpolicy="no-referrer"></p><p>WebTransport 现可参考 <a href="https://w3c.github.io/webtransport/" target="_self">GitHub</a> 上的 W3C 编辑草案来使用，能够简单总结为“向服务器发送数据”和“从服务器接收数据”这两个部分。</p><p>不过在 WebSockets 的基础上，WebTransport 还支持多路 / 单向流、无序传递、以及可靠 / 不可靠的传输。</p><p><img src="https://static.cnbetacdn.com/article/2021/0423/90c359d224c6c3c.png" alt="1_OChqO9y1iP2XddAGOLY8tA.png" referrerpolicy="no-referrer"></p><p>对 WebAssembly SIMD 的支持（<a href="https://github.com/WebAssembly/simd/tree/master/proposals/simd" target="_self">传送门</a>），也从早期试用转向了新的阶段。现能够在 WebAssembly 中以独立的方式，来调用系统的 SIMD 指令。</p><p>感兴趣的朋友，可移步至 Chromium <a href="https://blog.chromium.org/2021/04/chrome-91-handwriting-recognition-webxr.html" target="_self">官方博客</a>、或 ChromeStatus <a href="https://www.chromestatus.com/features#milestone%3D91" target="_self">状态页面</a>了解详情。</p>
