<strong>在 Fedora / 红帽 Anaconda 之后，openSUSE / SUSE 也宣布了基于 Web 的发行版安装器的开发计划、同时探索在现有的 YaST 之上构建基于 Web 的安装前端。</strong>参考红帽的最新 Web UI 实践，其希望 D-Installer 能够从 Anaconda 当前基于 GTK 的界面、转变为基于 Web 的界面。此外在 Cockpit 的加持下，新 Anaconda 安装程序的 Web UI，还可在本地或远程环境中工作。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0121/1a807c10706b5b9.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p>当然，无论是 Qt 还是 CLI 版本，（open）SuSE 的 D-Installer 开发计划，并不是要彻底取代现有的 YaST 前端安装程序、而是作为另一种补充。</p><p><a href="https://www.phoronix.com/scan.php?page=news_item&px=openSUSE-D-Installer" target="_self">Phoronix</a> 指出，D-Installer 将充分利用现有的 YaST 代码库、同时通过 D-Bus 与新服务连接。最终 D-Installer 有望带来更好的 UI / 可重用性体验，以及更好的集成等益处。</p><p><img src="https://static.cnbetacdn.com/article/2022/0121/9715a13187b815d.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：<a href="https://yast.opensuse.org/blog/2022-01-18/announcing-the-d-installer-project" target="_self">YaST.OpenSuSE.org</a>）<br></p><p>截至目前，开发团队已围绕这个基于 Web 的 YaST 安装程序开展了早期实验，但尚未给出一份实际投入使用的时间表。</p><p>不过 SuSE 开发者至少承认了其工作原理与 Fedora 相通，甚至有望采纳 Cockpit 方案。Cockpit 是一个相当不错的项目，他们已经为 Wicked 发布了一个模块，所以未来可能寻求一些方面的合作。</p>
