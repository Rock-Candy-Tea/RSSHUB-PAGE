<strong>在 Red Hat 工作了 15 年后，曾领导 PulseAudio 项目、并从根本上重塑现代 Linux 发行版 systemd 的 Lennart Poettering，现已正式加盟微软。</strong>周二的时候，有报道称 Lennart 已经告别了红帽，期间有各路传闻暗示这位 systemd 创建者跳槽到了雷德蒙德软件巨头。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0707/51756013a070c1e.png" alt="1.png" referrerpolicy="no-referrer"></p><p>起初 Phoronix 认为大家可能只是在无下限玩梗，但经过一天的跟进，事实证明这并不是一个玩笑。</p><p>Lennart Poettering 多年来投身的生态系统和他对 Linux / 开源世界的巨大贡献，已经得到了社区的公认。</p><p>而在加盟微软后，这位曾领导多个著名开源项目的开发者，仍将继续专注于 systemd 的开发。</p><p><img src="https://static.cnbetacdn.com/article/2022/0707/163fd725c84fe80.png" alt="2.png" referrerpolicy="no-referrer"></p><p>虽然在 Azure / WSL 普及之前的十多年，微软对 Linux 开源生态系统的态度一直相当冷淡。</p><p>但今年早些时候，另一位长期 Linux 内核开发人员 Christian Brauner 也投入了微软的怀抱。</p><p>他与 Lennart 都在柏林，五年里在 Canonical 从事过与 Linux 内核、LXC、systemd 等相关的开发工作。</p><p><img src="https://static.cnbetacdn.com/article/2022/0707/bddfda77b8273d0.jpg" alt="Linux TAG.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">微软在柏林 LinuxTag 2012 活动期间的大量展示（还赞助了啤酒排队和<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https%3A%2F%2Flist.jd.com%2Flist.html%3Fcat%3D737%2C752%2C761" target="_blank">咖啡</a>吧）</p><p>显然，随着 Linux 在 Azure 云平台上的广泛应用，<a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Linux 子系统（WSL）也被证明是相当成功的。</p><p>微软致力于让 Mesa
作为在 Direct3D 12 之上支持各种图形 / 计算 API 的一部分，确保 Linux 内核中良好的 Hyper-V
支持。</p><p>以及维护 CBL-Mariner 和 Azure Cloud Switch 等内部发行版，并继续吸引更多上游 Linux 开发人员 —— 尤其是开源生态系统社区里的大拿们。</p><p><img src="https://static.cnbetacdn.com/article/2022/0707/5186d583b3caa01.png" alt="3.png" referrerpolicy="no-referrer"></p><p>近年来，微软招聘网站一直充满了各种 Linux / 开源项目职位 —— 当前 Microsoft Careers 页面上就有 663 个。</p><p>而随着又一位经验丰富的关键上游开发者加盟（同时继续专注于 systemd 开发），该公司还有望在不久后带来更多惊喜。</p>
