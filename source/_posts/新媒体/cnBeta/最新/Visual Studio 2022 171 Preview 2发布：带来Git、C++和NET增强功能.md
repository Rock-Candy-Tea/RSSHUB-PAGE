微软刚刚放出了 Visual Studio 2022 17.1 的第二个预览版本，<strong>首先是能够更轻松地在并排视图中，比对当前 Git 和其它分支。其次是支持增强的 Detached Head，开发者能够在其中检查提交、导航到存储库中的旧点并运行。</strong>此外 Preview 2 现可相对更快地评估查询请求和团队更新，增强多存储库体验，且集成开发环境（IDE）可灵活地开展轻量级的分支管理操作。<br>
 <p style="text-align: center;"><a href="https://static.cnbetacdn.com/article/2022/0106/abc5322850952c6.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0106/abc5322850952c6.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（图 via <a href="https://www.neowin.net/news/visual-studio-2022-171-preview-2-now-live-with-git-c-and-net-enhancements/" target="_self">Neowin</a>）</p><p>与此同时，开发者现可在所有活动的存储库中自动创建新分支，而无需手动创建每一个。至于仍处于预览阶段的 Line-staging，也为开发者提供了代码块的暂存支持。</p><p>代码管理方面，显式保存操作的代码清理，现可利用自定义清理配置文件而实现自动化。值得一提的是，Visual Stuido 能够在窗口失去焦点时自动保存代码，而不会触发自动代码清理。</p><p>不过考虑到 VS 2022 17.1 目前仍处于 2 阶预览状态，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>还是希望大家能够积极提交与该主题相关的反馈。</p><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0106/d656058a420f2be.png" referrerpolicy="no-referrer"></p><p>.NET 也迎来了生产力方面的功能增强，开发者可将光标放到目标付好上，然后按 F12 功能键以快速导航至源文件。</p><p>通过新添加的 Stack Trace Explorer 窗口，开发者可轻松导航至相关代码，此外命名样式（Naming Styles）被编辑到了 EditorConfig UI，在 C++ 项目中查找所有引用的速度也提升了 22% 。</p><p>需要在 Visual Studio 中使用 C++ 的嵌入式开发者们，现也迎来能够与微控制器上的实时操作系统（RTOS）对象和外围寄存器进行交互的更多支持视图。</p><p>对于 macOS 开发者来说，微软正在简化利用 LLDB 开展调试的功能体验，且 IDE 会在 CMake 期间自动指定 macOS 默认配置。</p><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0106/0ae64a04a11ff19.png" alt="colored_tabs_s.png" referrerpolicy="no-referrer"></p><p>最后是颜色选项卡，Visual Studio 2022 已经运用了默认的配色方案，但现在用户可以自由选择不同的颜色，以便分辨不同的项目和文件扩展名。</p><p><strong>Visual Studio 2022 17.1 Preview 2 下载地址：</strong></p><blockquote><p><a href="https://visualstudio.microsoft.com/zh-hans/vs/preview/" _src="https://visualstudio.microsoft.com/zh-hans/vs/preview/" target="_blank">https://visualstudio.microsoft.com/zh-hans/vs/preview/</a></p></blockquote>
