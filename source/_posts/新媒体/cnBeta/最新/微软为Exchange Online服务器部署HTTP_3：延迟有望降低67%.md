从一个多月前正式发布的 Windows Server 2022 开始，<strong>Exchange Online 团队就一直与 Microsoft 365 团队展开密切合作，以期在 Outlook.office.com 的服务器端提供对 HTTP/3 的支持。</strong>由微软分享的最新消息可知，其部署进度已达 20% 。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/1013/52ac03d170ab76a.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p>这家软件巨头提供了所谓“最后一英里请求延迟指标”的一些详情。作为一个重要的性能指标，这意味着从总请求时间中（客户端与前门服务器之间），减去的后端处理所需时间。</p><p>对于 Web 端的 Outlook 来说，它也会对完整的用户体验产生较大的影响。好消息是，得益于 HTTP/3 的部署，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>为 Exchange Online 服务器带来了全面的性能改进。</p><p><a href="https://static.cnbetacdn.com/article/2021/1013/0fcdfebe6062695.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1013/0fcdfebe6062695.jpg" alt="2.jpg" referrerpolicy="no-referrer"></a></p><p>据悉，HTTP/3 于 2018 年被正式认定为超文本传输协议（HTTP）的一项重大更新，曾用名为 HTTP-over-QUIC 。</p><p>其特点是不使用传统 TCP / TLS 层，而是基于更快速的 UDP 互联网连接方案（简称 QUIC）。</p><p>QUIC 通过减少协议握手往返来提升访问性能，并在 TCP 丢失恢复的基础上实现了安全新方面的增强。</p><p>随着微软继续在所有前门服务器上落实 HTTP/3，广大 <a href="https://docs.microsoft.com/zh-cn/microsoft-365/enterprise/microsoft-365-network-connectivity-principles" target="_self">Microsoft 365</a> 客户也将获得不同的性能体验提升。</p>
