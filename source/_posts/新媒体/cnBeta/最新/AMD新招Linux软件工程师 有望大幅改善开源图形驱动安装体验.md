<strong>长期致力于快速更新开源图形驱动的 AMD，一直深受 Linux 社区的喜爱。</strong>但是对于那些倾向于在支持生命周期内使用旧版本软件包、或想要运行经过审查的企业 Linux 发行版的客户们来说，想要获得适用的 Linux 驱动程序堆栈，可能就有些痛苦了。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0712/781b0df1c6f62ac.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">截图（来自：<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://amd-cpu.jd.com/" target="_blank">AMD</a> <a href="https://jobs.amd.com/job/Markham-Linux-Build-Engineer-Onta/907325500/" target="_self">官网</a>）</p><p>好消息是，AMD 已经意识到了这方面的需求，并且正在招募一位“Linux Build Engineer”。</p><p>由本周列出的职务描述可知，AMD 希望前来应聘的开源 Linux GPU 驱动工程师具有丰富的经验。</p><p>尤其精通多个 Linux 发行版，以及 rpm、dpkg、apt、yum、dnf、zypper 等包管理器。</p><p>通过增强其构建、打包和部署能力，当前和未来的 AMD GPU 开源 Linux 驱动程序都将受益于此。</p><p><img src="https://static.cnbetacdn.com/article/2022/0712/8071e98c7731629.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">职位要求</p><p>鉴于 AMD 目前仅提供了面向一小部分 Linux 发行版的“Radeon Software for Linux”驱动程序，这则招聘启事显然预示着一个美好的开始。</p><p><a href="https://www.phoronix.com/scan.php?page=news_item&px=AMD-Linux-Build-Engineer-Needed" target="_self">Phoronix</a> 指出：除了少数几个支持 ROCm 开源计算堆栈的平台，此前想要为不受支持的 Linux 发行版提供打包，一直是项颇为艰巨的任务。</p><p>除了 Fedora 和 Debian 有取得一些进展，其它地方的进展几可忽略。</p>
