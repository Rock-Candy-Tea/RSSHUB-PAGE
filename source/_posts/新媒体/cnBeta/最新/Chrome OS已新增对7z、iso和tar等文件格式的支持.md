早在去年 7 月，9to5Google 就已报道 Google 正酝酿改善 Chrome OS 的文件管理体验。<strong>时隔将近一年，我们终于在 5 月更新的 Chrome OS 101 版本中看到了一些苗头。</strong>现在，你已能够在 Chrome OS 上顺利打开 7z 等压缩文件、安装非 Chrome 网上应用店渠道获取的 .crx 浏览器扩展程序、以及正确挂载 ISO 镜像文件。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0621/4cd353bbcc6b669.gif" alt="1.gif" referrerpolicy="no-referrer"></p><p>7z 属于 7-Zip 主导的一种压缩包格式，其以压缩率较标准 zip 格式更高而被大家所熟知。不过需要注意的是，Chrome OS 暂不支持打开受密码保护的 7z 格式文件。</p><p>ISO 则是常见的光盘镜像打包格式（兼容 CD / DVD / 蓝光等标准），不过随着光驱日渐失宠，对于 ISO 的解包支持也变得愈加重要。</p><p>为做测试，<a href="https://9to5google.com/2022/06/20/chromeos-archives-7z-iso-tar/" target="_self">9to5Google</a> 下载了最新版的 Ubuntu Linux 发行版 ISO 文件，证实了新版 Chrome OS 确实能够快速访问到光盘镜像里的文件。</p><p>不过 Chromium 团队指出，现阶段的兼容性依然不够完美，尤其是 <a href="https://en.wikipedia.org/wiki/Universal_Disk_Format" target="_self">UDF</a> 格式的镜像文件。</p><p><img src="https://static.cnbetacdn.com/article/2022/0621/524709b06f5edc2.webp" alt="2.webp" referrerpolicy="no-referrer"></p><p>最后，对于 Linux 爱好者们来说，Chrome OS 终于带来了对“tape archive”归档文件的支持。</p><p>作为一种可追溯到 1979 年的老格式，我们今日仍可在各种软件分发渠道看到 .tar 文件的身影 —— 通常还会进一步压缩为 .tar.gz 或 .tar.xz 之类的格式（遗憾的是 Chrome OS 目前尚不支持）。</p><p>按照计划，Google 还会在 8 月的 104 版本中，为 Chrome OS 带来对更多文件格式的支持。</p><p>目前已知的有 bzip2、gzip、lzma、lzw、xz 和 Zstandard，它们都是 Linux 平台上常用的打包格式。</p>
