去年 7 月，微软宣布了 <a href="https://www.microsoft.com/en-us/windows-365" target="_blank">Windows 365 Cloud PC</a>，这是一款虚拟化的云操作系统产品。<strong>该服务在接下来的一个月里开始正式上线</strong>，事实上，它是如此受欢迎，以至于微软不得不暂时暂停免费试用，以平衡其云基础设施的负载。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0317/fad31056bafd928.webp" alt="p7w3iuok.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">对于那些不了解的人来说，<a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 365 允许企业从托管在<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>云上的 Windows 系统。这是一个完整的 Windows 体验，为每个员工提供个性化的设置、应用程序和数据。你不需要任何特殊的硬件来运行它，只需要从 20 美元一直到 162 美元的多个定价层中进行选择--这可以让你得到一台最高拥有 8 个 vCPU、32GB 内存和 512GB 存储的云电脑。</p><p style="text-align: left;"><img src="https://static.cnbetacdn.com/article/2022/0317/0e411c29a8d7b12.webp" alt="xdz0py5m.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">现在，微软已经透露，它正在为 Windows 365 Cloud PC 提供嵌套虚拟化的支持。在推出时，这将支持 Linux 和 Windows Subsystem for Android（WSA）的预览。基本上，这意味着企业可以在 Hyper-V Windows 365 虚拟机内运行 Hyper-V，并在其中直接运行 Linux 和 WSA。</p><p style="text-align: left;">这种能力势必会使使用Windows 365云电脑的开发者受益，并希望虚拟出另一个操作系统进行测试和/或跨平台开发。然而，它将只对"某些许可证类型"启用，这可能是更昂贵的Windows 365企业级，但目前还不清楚。</p>
