面向 Beta 频道的 Windows Insider 项目成员，今天微软发布了 Windows 11 Build 22000.176 预览版更新。不过很多用户反馈在尝试升级新版本重启之后，出现了任务栏无响应导致设备无法正常工作的问题。<br>
 <p><iframe src="https://v.qq.com/txp/iframe/player.html?vid=l32722k14km" allowfullscreen="true" width="750" height="480" frameborder="0"></iframe></p><p>外媒 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Central 编辑 Zac Bowden 在推文中表示：“那么，有谁在更新到22000.176后遇到了这个错误？不知道如何修复它。也无法打开反馈中心来报告”。</p><p>外媒 NeoWin 的编辑在 <a data-link="1" href="https://microsoft.pvxt.net/9W473" target="_blank">Surface</a> Go 2 上也遇到了这个错误，令人沮丧的是，如果没有活动的任务栏或响应的 Windows Explorer，就无法访问 Windows GUI 的其他部分。</p><p>Reddit 用户 u/kesu1 开了一个讨论这个问题的帖子，他表示：“只是基于我迄今为止的发现。文件资源管理器可以工作，通过在Program Files文件夹中找到它来启动任何应用程序，等待修复，这样你就不必降级或做任何大幅度的改变。你还可以进入控制面板，剪切/复制/粘贴任何东西。因此，没有数据是无法访问/丢失的，只是在这次更新中失去了大量系统功能”。</p>
