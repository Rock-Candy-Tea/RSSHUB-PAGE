GitHub Copilot最近又出了个新功能。这一次，它不再是帮你写代码了，而是充当一个“翻译官”：用“人话”告诉你，这段代码到底写了啥。瞧，只需要选中一段代码，然后点一下旁边的“Ask Copilot”，就会快速得到代码的“人话版”解释：<br>
 <p class="textAlignCenter"><a target="_blank" href="https://static.cnbetacdn.com/article/2022/0130/6811b9ed0ca82a7.gif"><img data-original="https://static.cnbetacdn.com/article/2022/0130/6811b9ed0ca82a7.gif" src="https://static.cnbetacdn.com/thumb/article/2022/0130/6811b9ed0ca82a7.gif" referrerpolicy="no-referrer"></a></p><p>示例中的代码片段是这样的：</p><p>if len(l)=0: return Nonelse: max_int=l[0] for i in l if i>max_int max_int=i return max_int</p><p>Copilot 对此进行“翻译”后，输出的内容是：</p><p>定义一个函数 max_int ()，它将一个列表作为输入。</p><p>设置一个变量 max_int=l [0]，它包含列表中的第一个元素。</p><p>设置一个 for 循环，迭代列表中的每个元素。</p><p>如果元素大于 max_int，便让 max_int 等于 I。</p><p>返回 max_int。</p><p>这便是 GitHub Copilot 的最新功能 ——Explain Code。之前 Copilot 是基于人类的注释生成代码，现在好了，它也学会了反向输出。</p><p>网友们的反应也是可想而知了：</p><p class="textAlignCenter"><img src="https://x0.ifengimg.com/ucms/2022_06/290554A48DC10C48AC6F82ADE1F67688DED7C864_size2593_w695_h451.gif" alt="网友反应" referrerpolicy="no-referrer"></p><p>但在“新奇”之后，这件事也引发了不小的讨论。</p><p>AI 给代码做注释，有必要吗？</p><p>不仅是 Copilot，“AI 给代码做注释”这事最近在 Reddit 也比较火。</p><p class="textAlignCenter"><a target="_blank" href="https://static.cnbetacdn.com/article/2022/0130/d57ccce288ca365.png"><img data-original="https://static.cnbetacdn.com/article/2022/0130/d57ccce288ca365.png" src="https://static.cnbetacdn.com/thumb/article/2022/0130/d57ccce288ca365.png" referrerpolicy="no-referrer"></a></p><p>有位贴主上传了他用 AI 技术，给代码做注释的 demo。不同于 Copilot 的 Explain Code，这个 AI 不会非常详尽地去描述每行代码运行的过程。而是概括性地去讲“这段代码是干嘛的”。</p><p>以下面这个代码片段为例：</p><p class="textAlignCenter"><a target="_blank" href="https://static.cnbetacdn.com/article/2022/0130/251a350d48a5a4a.png"><img data-original="https://static.cnbetacdn.com/article/2022/0130/251a350d48a5a4a.png" src="https://static.cnbetacdn.com/thumb/article/2022/0130/251a350d48a5a4a.png" referrerpolicy="no-referrer"></a></p><p>在 AI“解读”过后，它就会告诉你：</p><p>这段代码是用来从 GitHub 的 Repo 中收集数据。会返回一个数据的矩阵。</p><p>再如下面这个代码片段：</p><p class="textAlignCenter"><a target="_blank" href="https://static.cnbetacdn.com/article/2022/0130/65f2982c5105b34.png"><img data-original="https://static.cnbetacdn.com/article/2022/0130/65f2982c5105b34.png" src="https://static.cnbetacdn.com/thumb/article/2022/0130/65f2982c5105b34.png" referrerpolicy="no-referrer"></a></p><p>AI 给出的注释是：</p><p>函数运行的是梯度下降算法。</p><p>而后它还会对函数中的变量依次做解释。看似不错的效果，但依旧还是引来了网友们激烈的讨论。</p><p>“乐观派”网友认为：</p><p>辅助写注释和写测试比写代码的帮助大更多，而犯错可能更少。</p><p class="textAlignCenter"><a target="_blank" href="https://static.cnbetacdn.com/article/2022/0130/a17c7b7d5117697.png"><img data-original="https://static.cnbetacdn.com/article/2022/0130/a17c7b7d5117697.png" src="https://static.cnbetacdn.com/thumb/article/2022/0130/a17c7b7d5117697.png" referrerpolicy="no-referrer"></a></p><p>但有人质疑这种 AI 的能力：</p><p>如果我函数写的特别乱、变量这名字乱糟糟的，它还能好使吗？</p><p data-original="https://static.cnbetacdn.com/article/2022/0130/aef2187225d8659.png" src="https://static.cnbetacdn.com/thumb/article/2022/0130/aef2187225d8659.png" referrerpolicy="no-referrer"></a></p><p>也有人认为，诸如此类的 AI“没有什么用处”：</p><p class="textAlignCenter"><a target="_blank" href="https://static.cnbetacdn.com/article/2022/0130/17dd5051ee257e3.png"><img
