面向 Dev 频道的 Windows Insider 项目成员，今天微软推出了 Windows 10 Build 21343 预览版（隶属于 RS_PRERELEASE 分支）更新。本次更新主要为桌面、文档、下载和照片等几个文件夹提供了全新的 Fluent 图标。<br>

Notepad。现在，微软正采取下一步措施，对文件资源管理器中使用的多个图标进行升级。</p><p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0325/821f048c66501eb.jpg" target="_blank"><img left;"><strong>注意：升级新版本之后，任何固定在快速访问上的自定义文件夹将消失。您可以在更新到此构建之前将它们记录下来，或者暂停更新，直到微软得到修复。</strong></p><p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0325/0038891d5b5968d.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0325/0038891d5b5968d.jpg" src="https://static.cnbetacdn.com/thumb/article/2021/0325/23c4d90e77d64d2.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">为了让文件在微软产品中的显示实现更大的一致性，微软对文件夹图标的方向和默认文件类型图标进行了一些改变。值得注意的是，桌面、文档、下载和图片等顶层用户文件夹采用了新的设计，应该可以让人更容易地将它们一目了然地区分开来。没错，回收站的图标也更新了!</p><p style="text-align: left;">正如你可以想象的那样，Windows 10中有很多图标，随着时间的推移，更多的图标将继续得到更新。</p></blockquote><p style="text-align: left;"><strong>改进 Windows Sandbox 和 Microsoft Defender Application Guard (MDAG)</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">Windows Sandbox 和 Microsoft Defender Application Guard 都利用了 Windows 10 内置的特殊容器技术。此前，容器运行时与主机相匹配。从 Build 21343 开始，微软引入了针对容器场景设计和优化的新运行时。它是轻量级的，并允许 Windows Sandbox 和 Microsoft Defender Application Guard 启动更快。</p><p style="text-align: left;">微软预计这一变化不会影响 Windows Sandbox 内部的应用程序兼容性。但是，这是一个重大的变化，用户可能会遇到一些行为上的差异。</p><p style="text-align: left;">此外，Windows Sandbox现在包含了新的基于 Chromium 的 Microsoft Edge 浏览器（从Build 21313 开始）。以下是关于如何开始使用 Windows Sandbox 的说明。</p></blockquote><p style="text-align: left;"><strong>改进和优化</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">● 正将“开始”中的“Windows 管理工具”（Windows Administrative Tools）文件夹更名为“Windows 工具”（Windows Tools）。微软正在努力更好地组织Windows 10中所有的管理和系统工具。</p><p style="text-align: left;">● [新闻和兴趣]更新推出：继上一次更新语言和市场后，本周微软还向国内用户开放！微软将继续向 Windows Insiders 推出新闻和兴趣，所以目前还不是每个人都可以在开发频道中使用。</p><p style="text-align: left;">● 现在向 Dev 频道中所有使用简体中文 IME 的 Windows Insiders 推出新的 IME 候选窗口设计。</p><p style="text-align: left;">● 正在更新触摸键盘中的“获取帮助”链接，现在显示“了解更多”。</p><p style="text-align: left;">● 更新了文件资源管理器在重命名文件时，现在支持使用 CTRL +左/右箭头在文件名中的单词之间移动光标，以及 CTRL + Delete 和 CTRL + Backspace 来一次删除单词，就像 Windows 中的其他地方一样。</p><p style="text-align: left;">● 对 Windows 中与网络相关的界面进行了一些更新，使显示的符号使用最近在 Dev 频道中添加的更新系统图标。</p><p style="text-align: left;">● 根据用户反馈，如果 Shared Experiences 页面确定了你的账户连接问题，现在会直接将通知发送到 Action Center，而不是需要驳回的重复通知 toasts。</p></blockquote><p style="text-align: left;"><strong>修复：</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">● 修复了一个问题，即某些 NVMe 驱动器的设备会遇到磁盘重置或 WHEA_UNCORRECTABLE_ERROR 错误检查。</p><p style="text-align: left;">● 修复了一个问题，其中一些设备收到DPC_WATCHDOG_ERROR错误检查。</p><p style="text-align: left;">● 修复了一个问题，即一些使用Realtek网络适配器的设备在运行1.0.0.4版本的驱动程序时，会出现网络连接间歇性丢失的情况。</p><p style="text-align: left;">● [<strong>新闻和兴趣</strong>]修复了新闻和兴趣按钮文本在某些情况下使用错误的高对比度颜色的问题。</p><p style="text-align: left;">● [<strong>新闻和兴趣</strong>]修复了一个问题，即在没有互联网访问的情况下登录Windows时，新闻和兴趣可能无法使用，但在线时又会返回。</p><p style="text-align: left;">● [<strong>新闻和兴趣</strong>] 进行了多项修复，以帮助提高 explorer.exe 的性能和可靠性。</p><p style="text-align: left;">● 修复了一个导致explorer.exe以事件ID 1002崩溃的问题。</p><p style="text-align: left;">● 修复了与回收站交互时的内存泄漏问题。</p><p style="text-align: left;">● 修复了最近的 Dev Channel 版本中与索引器有关的死锁，这可能导致升级后首次启动时无法启动开始菜单或其他应用程序。</p><p style="text-align: left;">● 修复了一个问题，在一些高刷新率的显示器上，游戏只能以60Hz的速度运行。这个问题也可能导致在可变刷新率显示器情况下的撕裂。</p><p style="text-align: left;">● 修复了一个潜在的问题，导致一些应用程序在安装过程中崩溃，并可能是最近的其他活动。</p><p style="text-align: left;">● 最近修复了一个潜在问题，导致一些应用程序意外地显示 "您必须在新设置生效前重新启动计算机 "的消息。</p><p style="text-align: left;">● 修复了在最近的Dev Channel构建中，当显示器被设置为纵向时，导致辅助显示器上的文本模糊的问题。</p><p style="text-align: left;">● 修复了在最近的构建中影响WIN+Shift+左/右键的可靠性的问题。</p><p style="text-align: left;">● 修复了一个问题，在文件资源管理器的属性对话框中，大容量驱动器的大小信息可能会被截断。</p><p style="text-align: left;">● 修复了一个可能导致 "设置 "顶部的标题在某些语言中的文本被截断的问题。</p><p style="text-align: left;">● 修复了即 "设置 "标题中的用户配置文件图片在调整窗口大小时可能会闪烁的问题。</p><p style="text-align: left;">● 修复了一个问题，在改变音频端点后，声音设置中的音量控制可能会停止工作。</p><p style="text-align: left;">● 修复了最近在网络状态设置页面中缺少属性和数据使用选项的问题。</p><p style="text-align: left;">● 修复了一个问题，如果您搜索 "高级触摸板手势配置 "并点击结果，它将只启动设置，而不是特定的设置页面。</p><p style="text-align: left;">● 修复了一个问题，导致一些Insiders通过双击任务栏中的Windows更新图标启动 "设置 "后崩溃。</p><p style="text-align: left;">● 修复了一个阻止 Azure Data Studio 在 ARM64 上更新到较新版本的问题。</p><p style="text-align: left;">● 修复了导致 Ngen.exe 无法在 ARM64 上预编译 .NET Framework 二进制文件的问题。</p><p style="text-align: left;">● 修复了一个可能导致某些触摸键盘的子键被切断的问题。</p><p style="text-align: left;">● 修复了一个问题，即在使用小型触摸键盘布局时，向上轻弹触摸键盘顶行不会插入相应的数字，这与其他布局不同。</p><p style="text-align: left;">● 修复了在浅色主题和深色主题切换后，可能导致IME候选窗口显示问题的问题。</p><p style="text-align: left;">● 修复了输入一串很长的文字后，日语IME会失效的问题。</p><p style="text-align: left;">● 修复了在Excel中用日语IME打字时无法使用SHIFT+Space的问题。</p><p style="text-align: left;">● 修复了在假名输入模式下输入以"? "开头的句子时，无法输入日语IME的问题。</p></blockquote><p style="text-align: left;"><strong>已知问题</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">●  [<strong>游戏</strong>]在此构建中存在一个问题，某些游戏可能会崩溃，在不同设备上玩游戏时不会同步保存数据，或者在安装新游戏时不会保存。我们正在进行修复，但如果你认为这可能会影响你，为了避免任何潜在的数据损坏，我们建议你暂停更新，直到我们发布带有修复功能的新构建。</p><p style="text-align: left;">●  [<strong>文件资源管理器</strong>]安装此构建后，文件资源管理器中钉入快速访问的所有文件夹将消失。请确保在升级前记下被钉住的内容，以便您可以再次找到这些文件夹。</p><p style="text-align: left;">●  [<strong>文件资源管理器</strong>]部分内部人员可能会遇到Quick Access的整个文件夹部分丢失的情况。如果遇到这种情况，删除这个字符串%appdata%/Microsoft/Windows/Recent/AutomaticDestinations/f01b4d95cf55d32a.automaticDestinations-ms，就会恢复该部分，但是不会恢复成为未钉的项目。我们正在努力解决这个问题，感谢您的耐心等待。</p><p style="text-align: left;">●  我们正在研究当试图安装一个新的构建时，更新过程长时间挂起的报告。</p><p style="text-align: left;">● 我们正在努力为现有的钉钉网站启用新的任务栏体验。在此期间，您可以从任务栏中取消钉住网站，将其从edge://apps页面中移除，然后重新钉住网站。</p><p style="text-align: left;">●  [<strong>新闻和兴趣</strong>] 按ESC键取消飞出中打开的上下文菜单，反而会取消整个飞出。</p><p style="text-align: left;">●  [<strong>新闻和兴趣</strong>] 有时新闻和兴趣的弹窗不能用笔删除。</p><p style="text-align: left;">●  [<strong>ARM64</strong>] 在<a data-link="1" href="https://microsoft.pvxt.net/9W473" target="_blank">Surface</a> Pro X上安装了高通Adreno图形驱动程序预览版的用户可能会遇到显示屏亮度降低的问题。此问题已通过更新版本的预览版图形驱动修复，网址为：https://aka.ms/x64previewdriverprox。如果您遇到此问题，请查看反馈意见集了解更多信息。</p><p style="text-align: left;">● 登录屏幕上的网络飞出在此构建中无法打开，这将阻止您在登录前连接到新的网络。如果你的账号处于需要网络才能登录的状态，你可以通过插入以太网、让设备进入之前配置的Wi-Fi网络范围，或者先用其他可用账号登录来解决这个问题。</p><p style="text-align: left;">● 在运行此构建的ARM64电脑上，手写输入面板将无法用于笔输入。</p><p style="text-align: left;">● 在此构建中存在一个问题，可能会导致一些内部用户在启动搜索时崩溃。微软正在努力解决这个问题。</p></blockquote>
