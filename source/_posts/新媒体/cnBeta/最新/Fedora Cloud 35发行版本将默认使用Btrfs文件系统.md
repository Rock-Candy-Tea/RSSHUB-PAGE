自 Fedora 33 以来，Fedora Workstation 一直默认为 Btrfs 文件系统，而 Fedora 的其他版本则默认采用 EXT4 文件系统。不过在即将到来的 Fedora Cloud 35 中，这个云端的分支发行版本也将迁移到 Btrfs 文件系统。<br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0526/a593d40182bfd74.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0526/a593d40182bfd74.jpg" alt="i97c5lh8.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">计划在今年晚些时候发布的 Fedora Cloud 35 默认使用 Btrfs，以便更容易地使用高级文件系统功能。Fedora Cloud 的这项调整已经得到了 Fedora 开发者、Facebook 和亚马逊等利益相关者的支持。</p><p style="text-align: left;">该变更提案写道：“Fedora Cloud Edition 镜像将转而使用 Btrfs。Cloud Edition 的配置将与桌面版的设置相匹配，因为这已经在多个 Fedora Linux 版本的生产部署中得到了很好的测试。这包括在 Fedora Linux 33 中引入的用于桌面版本的相同的子卷布局，以及在Fedora Linux 34中引入的透明的Zstd压缩”。</p><p style="text-align: left;">在Fedora Cloud使用Btrfs的好处中，包括与DNF软件包管理器处理相结合的写时拷贝增强功能、透明的文件系统压缩、重新链接和快照支持、更大的数据完整性、改进的系统响应速度、在线收缩和增长支持，以及以更容易的方式支持更复杂的存储配置。</p>
