在昨日的补丁星期二活动日中，微软面向 Windows 11 设备发布了累积更新 KB5015814，向更多用户开放 Windows Search Highlights 功能之外，还修复了诸多 BUG 并进行了诸多改进。<strong>不过用户反馈安装该更新中出现了诸如“0x8000ffff”、“0x8007007e”和“0x80073701”错误。</strong><br>
 <p>一位受影响的用户在 Feedback Hub 上留言道：“我在安装这个更新时出现了 0x8000ffff 错误，我只能重启重新进入系统。不过重启后我在更新列表中没有看到 KB5014668，我也不确定是否已经安装”。</p><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0714/bfda8b860dc23e3.webp" alt="t5078nef.webp" referrerpolicy="no-referrer"></p><p style="text-align: center;">图片来自于 <a href="https://manima.de/2022/03/migrating-ccm-agents-to-a-new-site-0x8000ffff-and-smsslp/">manima.de</a></p><p>另一位用户留言道：“这个更新存在一些错误，导致我设备循环重启。所幸的是 Windows 能够进行自我修复。推荐不要安装这个更新”。</p><p>在 Reddit 社区，有用户反馈安装该更新之后开始菜单存在性能问题，但似乎这个问题并不普遍。目前<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>官方并未提供任何解决方案，所以你可以暂停更新以防止安装失败。部分用户也反馈通过 Microsoft Update Catalog 手动安装该更新可以解决这些问题。</p><p style="text-align: left;">本次更新修复了重定向 PowerShell 命令输出的问题，以便脚本日志不包含命令的任何输出。今天的更新还带来了与 KB5014668（2022 年 6 月 23 日发布）相同的改进和功能。提醒一下，该更新的推出也首次让非 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Insider 项目成员测试 Search Highlights。</p><p><img src="https://static.cnbetacdn.com/article/2022/0713/5fc117c68647d78.webp" alt="nz1z1gx8.webp" data-original="https://static.cnbetacdn.com/article/2022/0713/5fc117c68647d78.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">此更新中有两个已知问题，在过去的两个补丁星期二版本中一直存在。安装此更新后，某些
.NET Framework 3.5 应用可能会出现问题或无法打开。您可以通过在 Windows 功能中重新启用 .NET Framework
3.5 和 Windows Communication Foundation 来修复它。第二个已知问题影响 Edge 中的 IE
模式选项卡无响应。这可以使用已知问题回滚来修复。</p>
