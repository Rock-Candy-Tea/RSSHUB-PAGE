随着微软开始向 Beta 测试通道的 Insider 测试者们推送 Windows 11 Build 22622.436 编译版本，<strong>可知 Windows Terminal 也终于成为了 Windows 11 操作系统中的默认终端。</strong><br>
 <p><a href="https://static.cnbetacdn.com/article/2021/0717/d3a531b96cff521.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0717/d3a531b96cff521.jpg" referrerpolicy="no-referrer"></a></p><p>微软在 Windows 11 最新预览版的<a href="https://blogs.windows.com/windows-insider/2022/07/19/announcing-windows-11-insider-preview-build-22621-436-and-22622-436/" target="_self">发行说明</a>中写道：</p><blockquote><p>Windows Terminal 现已是 Windows 11 中的默认终端，意味着所有命令行应用程序（包括 CMD 命令提示符和 PowerShell）都将在 Windows Terminal 中自动打开。</p><p>感兴趣的朋友，可移步至‘设置 → 隐私和安全 → 开发者设置’（Settings > Privacy & security > For developers）找到该选项。</p><p>不过需要注意的是，该功能需要搭配 1.15 或更高版本的 Windows Terminal 终端。</p></blockquote><p><a href="https://static.cnbetacdn.com/article/2022/0721/3e186bb9b699ffa.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0721/3e186bb9b699ffa.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（via <a href="https://news.softpedia.com/news/windows-terminal-becomes-the-default-terminal-in-windows-11-535794.shtml" target="_self">Softpedia</a>）</p><p>与此同时，Windows 11 Build 22622.436 带来了其它一系列修复：</p><blockquote><p>● 首先是设备从睡眠状态恢复时，可能遇到黑屏故障。</p><p>● 其次是开启移动热点后，可能导致网络连接断开的问题。</p><p>● 以及安装 Build 22621.290 / 22622.290 失败，显示 0x800f081f 报错的 bug 。</p></blockquote><p>如果一切顺利，上述 Beta 更新将很快推向生产设备。</p><p>最后，鉴于微软已经选择了 RTM 版本，预计面向非 Insider 测试者的下一个主要“功能更新”会在未来几个月内到来。</p>
