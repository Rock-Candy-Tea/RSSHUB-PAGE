尽管距离 Windows 11 正式版的发布日期已经相当邻近，微软 6 月官宣的 Windows Subsystem for Android 功能却迟迟没能向测试用户交付。<strong>不过从现已上线 Microsoft Store 应用商店的介绍页面来看，该公司还是揭示了 Android 子系统的一些有趣细节。</strong><br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0903/9f8266dadfe15e4.png" referrerpolicy="no-referrer"></p><p>老伙计 WalkingCat（@_h0x0d_）在 Twitter 上分享了这个专门的链接，不过目前似乎仍处于未正式启用的状态。</p><p>即使点击后可以下载，这个 WSA 软件也只会在 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 上安装和打开时，蹦出一个空白的窗口。若是 Windows 10 平台，它甚至无法执行安装。</p><p><a href="https://static.cnbetacdn.com/article/2021/0903/2d5c03a99ff2cb0.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0903/2d5c03a99ff2cb0.png" referrerpolicy="no-referrer"></a></p><p>好消息是，商品详情页至少包含了一些有趣的细节。一方面，其已将系统要求降低至 Windows 10 Build 22000.0，与我们当前在 Beta 测试通道上见到的开发分支相同。</p><p>另一方面，Microsoft Store 还将 <a data-link="1" href="https://microsoft.pvxt.net/e4yLO" target="_blank">Xbox</a> One 列为受支持的平台，这点是<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>此前从未提到过的。</p><p>受支持的架构包括了 ARM64 和 x64，所需最低内存（RAM）容量为 8GB 。至于推荐配置，除了将内存要求提升到 16GB，其余并没有差别。</p><p>有趣的是，Microsoft Store 页面警告称，用户不应该截取 WSA 的屏幕界面或泄露其它内容，暗示目前仍处于仅限于微软员工的内部测试阶段。</p><p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0628/7e644f0451e1e74.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0628/7e644f0451e1e74.jpg" referrerpolicy="no-referrer"></a></p><p>最后，尽管微软尚未披露有关 Android 应用程序支持的更多细节，但该公司确实有在与亚马逊展开 Android 应用商店方面的合作。</p><p>更新后，用户将能够下载并安装 TikTok、Pinterest 和 Kindle 等 Android 应用程序。</p><p>此外这些 App 可以像 Windows 应用程序那样，将之固定到任务栏、与其它程序对齐、以及调整窗口大小等。</p>
