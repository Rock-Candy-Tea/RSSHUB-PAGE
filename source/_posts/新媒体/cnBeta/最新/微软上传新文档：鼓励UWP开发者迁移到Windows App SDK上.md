自 2015 年 Windows 10 系统推出以来，微软就一直鼓励 UWP 开发者将他们的应用迁移到新的 Windows App SDK 上。<strong>近日微软上传的<a href="https://docs.microsoft.com/en-us/windows/apps/windows-app-sdk/migrate-to-windows-app-sdk/overall-migration-strategy" target="_blank">新支持文档</a>中，为开发者提供了更贴心的指导。</strong><br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/1019/245c86333d97bf9.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/1019/245c86333d97bf9.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">微软写道：“<a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> App SDK 提供了一套广泛的 Windows API，其实现与操作系统脱钩，并通过 NuGet 包发布给开发者。Universal Windows Platform 的开发者，你可以通过将你的应用程序迁移到 Windows App SDK，来充分利用你现有的技能组合和源代码”。知名的 Windows 开发者 Rafael Rivera 指出，UWP SDK 只会得到“错误、可靠性和安全性的修复”。</p><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2021/1019/2ac8663a21dda4d.jpg" referrerpolicy="no-referrer"></p>
