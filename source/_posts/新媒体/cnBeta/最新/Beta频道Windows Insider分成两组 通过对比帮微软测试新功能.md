面向 Beta 频道的 Windows Insider 项目成员，微软今天发布了适用于 Windows 11 的累积更新 KB5014959，用户安装后分别升至 Build 22621.290 和 Build 22622.290。关于本次分割更新，微软官方表示：“和 2019 年所做的那样，自今天开始，Beta 频道获得的更新分为两个组”。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0706/dfb8a3441604aff.webp" alt="8vn8tmj9.webp" referrerpolicy="no-referrer"></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0706/028eadcb6ef2ce4.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;"><strong>微软表示</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">● Beta 频道的部分内部成员将会收到 Build 22622.xxx更新，今后功能上的更新和启用均通过启用包（<a href="https://support.microsoft.com/en-us/topic/kb5003791-update-to-windows-10-version-21h2-by-using-an-enablement-package-8bc077be-18d7-4aac-81ce-6f6dad2cd384" target="_blank">enablement package</a>）的方式进行，就像现在为 Windows 10 发布的更新方式。</p><p style="text-align: left;">● Beta 频道的另一部分成员会收到 Build 22621.xxx 更新，这些更新的新功能将默认为关闭。</p></blockquote><p style="text-align: left;"><strong>此外，微软发表了一篇冗长的博文，解释了在 Beta 渠道中进行分割版本的理由。</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">这种方法将帮助我们验证我们发布默认关闭功能的更新的能力。这也将使我们能够更深入地了解启用新功能的影响。通过比较这两个组中的Insiders的反馈和使用数据，我们将看到一个功能是否会导致可靠性问题。</p><p style="text-align: left;">例如基于这种洞察力，它将帮助我们做出决定，是否通过增加新功能的推出来让更多的 Insiders 使用这些功能，或者在某些情况下，禁用一个功能来解决 bug。我们的目标是利用对比数据，确保在 Beta 渠道中与Windows Insiders 一起尝试新功能和体验时，获得尽可能好的体验。</p><p style="text-align: left;">Beta 频道中的大多数内部人员将自动获得该更新（Build 22622.xxx）。需要注意的是，并不是所有的功能都会在此次更新中立即开启，因为我们计划在向更多的内测者推送这些功能之前，先推出这些功能并监测反馈，看看它们是如何登陆的。</p><p style="text-align: left;">我们意识到，在Beta频道的内部人员将希望选择他们得到的更新。处于默认关闭新功能（Build 22621.xxx）组的用户可以检查更新，并选择安装将有功能推出的更新（Build 22622.xxx）。</p><p style="text-align: left;">作为过渡到使用启用包的一部分组，一些之前有功能推出的 Insider 可能会看到这些功能消失。如果你遇到这种情况，请安装启用包，它们应该很快就会回来。我们意识到这并不理想，我们正在学习如何最好地调整我们未来的推出，以尽量减少对预览体验的干扰。</p><p style="text-align: left;">这种方法仅用于 Beta 渠道，并不表明最终功能推出的任何变化或计划。启用包人为地增加了更新的构建号，新功能被推出并打开，以使其更容易与默认关闭功能的更新设备区分开来。</p><p style="text-align: left;">作为提醒，我们在 Windows Insiders 中尝试的功能将在它们准备好的时候发货，而不是我们在Beta通道中尝试的每一个功能都将发货给普通客户。</p></blockquote><p style="text-align: left;">简单来说，Beta 频道在安装 KB5014959 之后可以划分为两个组，两个组的最新版本分别为 Windows 11 Insider Preview Build 22621.290 和 Build 22622.290。简单来说</p><blockquote style="text-align: left;"><p style="text-align: left;">● Build 22622.xxx = 可以获得新功能</p><p style="text-align: left;">● Build 22621.xxx = 默认关闭新功能。</p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22622.290 主要引入的新功能</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>● Suggested Actions</strong></p><p style="text-align: left;">Windows Insiders 可以在此版本中尝试一项新功能--Suggested Actions，通过内嵌的建议操作能更快捷地完成 Windows 11 中的日常任务。当你复制一个日期、时间或电话号码时，Windows 会建议你采取相关的行动，如创建日历事件或用你喜欢的应用程序打电话。这项功能目前只对美国、加拿大和墨西哥的 Windows Insider 开放。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0706/d86fce5313be58d.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">在复制一个电话号码时，Windows 会弹出一个内嵌的用户界面，建议使用 Teams 或其他提供点击拨号功能的已安装应用程序拨打该电话号码的方法。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0706/4eedae6963a518f.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">在复制一个未来的日期和/或时间时，Windows 会弹出一个内嵌的轻度 UI，建议使用支持的日历应用程序创建事件。在用户选择偏好后，该应用将被启动，并提供相应的日历事件创建页面，并自动填充日期和/或时间。</p><p style="text-align: left;"><strong>● 在设置中引入了 OneDrive 容量警告和订阅管理</strong></p><p style="text-align: left;">今年 3 月，微软增强了 Windows 11 设置中的 Microsoft 365 订阅管理体验，并在设置中的账户中增加了查看您的 Microsoft 365 订阅的支付方式的功能。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0706/acb6ca790d1c435.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">从今天的版本开始，微软已经开始在“设置”中的“账户”页面启用 OneDrive Standalone 100GB 订阅，与 Microsoft 365 订阅类似。这将允许你在 Windows 11 中查看你的常规账单、支付方式和 OneDrive 存储使用情况。此外，如果你接近或超过你的 OneDrive 存储限制，你将在同一页面被告知。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0706/396b29b1d764f4c.webp" referrerpolicy="no-referrer"></p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22622.290 独有修复内容</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>[文件管理器]</strong></p><p style="text-align: left;">● 在文件资源管理器的主体中点击一个文件夹，现在将在一个新的标签中打开它。</p><p style="text-align: left;">● 当按下Tab或F6时，标签行现在应该包括在键盘焦点循环中。一旦焦点在标签行中，你可以使用左或右方向键来浏览它们。</p><p style="text-align: left;">● 修正了一个问题，即如果你在文件资源管理器中重新排列了标签，使用CTRL + Tab时，标签顺序会出现错误。</p><p style="text-align: left;">● 对启用"在标题栏显示完整路径"时做了一些调整，以确保文件夹名称始终可见。</p><p style="text-align: left;">● 修正了一个缩放问题，该问题可能导致标签页意外变大。</p><p style="text-align: left;">● 右键点击一个标签，然后点击文件资源管理器中的其他地方，现在应该能更可靠地取消上下文菜单。</p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22621.290 和 Build 22622.290 所包含的修复</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">●<strong> 新功能! </strong>微软添加了服务器信息块（SMB）重定向器（RDR）特定的公共文件系统控制（FSCTL）代码FSCTL_LMR_QUERY_INFO。</p><p style="text-align: left;">● <strong>新功能! </strong>微软在Windows客户端和服务器轻量级目录访问协议（LDAP）实现中增加了对传输层安全（TLS）1.3的支持。</p><p style="text-align: left;">● 更新了Remove-Item cmdlet，以正确地与微软OneDrive文件夹互动。</p><p style="text-align: left;">● 启用了InternetExplorerModeEnableSavePageAs组策略。欲了解更多信息，请参阅Microsoft Edge浏览器策略文档。</p><p style="text-align: left;">● 修复了一个问题，当你使用普通文件对话框打开相机时，会导致拍照按钮消失。</p><p style="text-align: left;">● 修复了一个问题，该问题可防止设备在已经安装了扩展驱动程序而没有基本驱动程序的情况下收到来自Windows Update的同一扩展驱动程序的提议。</p><p style="text-align: left;">● 修复了一个问题，该问题导致为活动目录创建从媒体安装（IFM）失败，出现错误"2101 JET_errCallbackFailed"。</p><p style="text-align: left;">● 修复了当活动目录轻量级目录服务（LDS）为userProxy对象重置密码时发生的问题。密码重置失败，出现错误，如："0000000005: SvcErr: DSID-03380C23，问题5003（WILL_NOT_PERFORM），数据0"。</p><p style="text-align: left;">● 启用了InternetExplorerModeEnableSavePageAs组策略。欲了解更多信息，请参阅Microsoft Edge浏览器策略文档。</p><p style="text-align: left;">● 修复了当活动目录轻量级目录服务（AD LDS）为userProxy对象重置密码时发生的问题。当你试图重置别人的密码，并且你使用简单的绑定进行验证时，密码重置失败。错误是这样的："0000000005: SvcErr: DSID-03380C23，问题5003（WILL_NOT_PERFORM），数据0"。</p><p style="text-align: left;">● 修复了一个阻止Microsoft Edge在Windows沙盒中可用的问题。</p></blockquote><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0706/39e57c2c37bf26a.webp" referrerpolicy="no-referrer"></p>
