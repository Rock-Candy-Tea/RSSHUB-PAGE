为进一步减少渐进式网络应用（PWA）和本地原生应用之间的区别，谷歌正努力让 PWA 应用能够像原生应用一样，<strong>注册成为某些特定扩展名的文件处理程序。这就意味着，未来当你双击某个类型文件的时候将直接在 PWA 应用中打开。</strong><br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0404/d760636e459ecd7.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0404/d760636e459ecd7.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0404/82a03131fdd83de.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0404/82a03131fdd83de.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2021/0404/60c068919b97b43.jpg" referrerpolicy="no-referrer"></p><p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0404/739e1d752fdd24f.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0404/739e1d752fdd24f.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0404/53c46f51be31936.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0404/53c46f51be31936.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">援引 Techtsp 报道，谷歌工程师 Darwin Huang 在一份设计文件中写道：“这个项目的目标是实现一个API，允许Chrome渐进式网络应用(PWA)在主机操作系统的文件系统中像本地应用一样‘处理’（读取）文件。这与增加网络应用和本地应用之间透明度的更大目标相吻合，根据渐进式网络应用(PWA)模式，使两者的用户体验更加一致”。</p><p style="text-align: left;">网络应用将能够在其 manifest 中声明他们支持的 MIME 类型。用户将能够允许或阻止文件类型的注册，每次当一个应用程序想要写入它已经打开的文件时，都会询问用户是否允许。该功能已经可以在 Chrome 或 Edge 的 File Handling API 标志下进行测试，应该可以在所有平台上使用。</p>
