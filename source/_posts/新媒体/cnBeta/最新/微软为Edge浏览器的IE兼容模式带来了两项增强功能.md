3 月中旬，微软预告了 Internet Explorer 浏览器即将于 2022 年 6 月 16 日退役。<strong>但是在最后的三个月时间里，该公司仍会为 Microsoft Edge 的“IE 兼容模式”带来持续的改进。</strong>基于用户的直接反馈，两项增强特性将可进一步提升该浏览器的生产力，以帮助有需要的组织机构平稳过渡。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0318/394b9904c50bde0.png" referrerpolicy="no-referrer"></p><p><a href="https://mspoweruser.com/microsoft-edge-for-ie-mode-offers-two-new-enhancements/" target="_self">MSPU</a> 指出：借助 Microsoft Edge for IE 的兼容模式，用户将可访问专为 IE 打造的传统网站和应用程序，辅以更加现代的浏览体验。</p><p>在最新的改进中，包括了恢复 IE COM 对象调用、以及双向 cookie 共享的实现。前者已经被优先考虑、且正在广泛部署，因为它是许多应用程序和工作流功能的重要组成部分。</p><blockquote><p>在 Internet Explorer 彻底退役前，广大用户还可结合 IE / Microsoft Edge 来互补。但在彻底剔除了独立的 IE 浏览器之后，许多业务流程都会经历中断。</p><p>鉴于许多开发者都提出了这方面的问题，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>最终还是决定为 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 10 / 11 操作系统引入并部署 IE 模式的增强功能。那样在 IE 11 桌面应用程序清退之后，相关工作仍可继续开展。</p></blockquote><p>其次，对于正处于转型过程中的组织机构而言，Edge for IE 的兼容模式，将可实现现代 / 旧版引擎的双向 cookie 共享。</p><p>即使在 IE 停用后，Microsoft Edge 中的 IE 模式仍可扩展旧应用程序的使用。在单个浏览器中访问新旧站点的能力，将有助于广大客户顺利推进现代化转型。</p><blockquote><p>此前，IE 模式下的 cookie 只允许从使用现代引擎的站点、到旧引擎站点的共享会话，而无法反过来进行。</p><p>庆幸的是，在意识到需要在新旧站点的工作流程中共享正确数据的重要性之后，站点和引擎之间共享凭据的 SSO 方案已正式解锁。</p></blockquote><p>感兴趣的朋友，现可通过手动设置 cookie 来启用。此外这项改进已合并到 Microsoft 365 管理中心的云站点列表管理体验中。</p><p>如此一来，用户便可识别其托管的特定站点，以便在必要时允许 cookie 共享。</p><div class="article-relation"><p><strong>相关文章:</strong></p><p><a href="https://www.cnbeta.com/articles/tech/1248453.htm" target="_blank">微软再次昭告天下：Internet Explorer浏览器将于6月15日终止支持</a></p></div>
