随着 Windows 11 的发布，微软对任务栏实施了重大改造。<strong>受早前 Windows 10X 操作系统开发项目的启发，Windows 11 默认采取了居中的任务栏图标排布，同时将“开始”按钮保持在左起第一位。</strong>对于习惯了旧版 Windows 操作系统的用户来说，多少还是对这些变化感到有些困惑，因而需要一段时间去适应。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0325/43ff4c208038788.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：Microsoft | <a href="https://blogs.windows.com/windows-insider/2022/03/23/announcing-windows-11-insider-preview-build-22581/" target="_self">Official Changelog</a>）</p><p>好消息是，自 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 发布以来，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>一直在不停地进一步完善任务栏体验。其中最大的一项变化，就是重新引入了对拖放操作的支持。</p><p>作为一个在首发时就该存在的缺失特性，Dev 和 Beta 通道的 Insider 测试者们，终于在本周早些时候的 Windows 11 Build 22581 预览版本中看到了它的身影。</p><p><strong>此外该版本还其它修复：</strong></p><blockquote><p>● 修复了一个在英语之外的语言环境中，使用错误窗口标题字体的任务栏预览问题。</p><p>● 任务栏拖放功能，现已能够与自动隐藏任务栏功能一起使用。</p><p>● 将某些内容拖动到任务栏时，被拖动项目上的消息将带有圆角效果（若不支持，则显示 X）。</p><p>● 优化任务栏，缓解滑动以折叠平板优化任务栏时，可能意外调用小部件板的问题。</p></blockquote><p><a href="https://static.cnbetacdn.com/article/2022/0325/03777224aa3edc5.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0325/03777224aa3edc5.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">折叠 / 舒展状态下的平板优化任务栏</p><p>需要指出的是，目前上述改进都仅限于 Windows Insider 测试环境。但其中一部分内容，可能随着 Windows 质量更新的发布，而被添加到正式版本中。</p>
