用于创建可启动媒体介质的流行第三方应用程序 <strong>Rufus 刚刚迎来了 3.20 BETA 版本更新，主要变化是引入了创建本地账户、并从当前已安装的 Windows 操作系统平台上复制区域设置的选项。</strong>不过需要注意的是，创建账户时必须使用与当前相同的用户名、且密码为空。在首次重启后，Windows 才会提示修改密码。<br>
 <p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0728/bc836728586eca8.png" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：<a href="https://github.com/pbatard/rufus/releases/tag/v3.20_BETA" target="_self">GitHub</a>）</p><p>在变更日志中，pbatard 还提到了 Rufus 3.20 BETA 版本的其它变化：</p><blockquote><p>● 为 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 10 启用合适的 Windows 用户体验选项</p><p>● 在会话之间记住并保持 Windows User Experience 不变</p><p>● 添加自动本地帐户创建和区域选项复制</p><p>● 为指向 /isolinux/ (Knoppix) 的 syslinux 符号链接的 ISO 镜像提供解决方案</p><p>● 尽力为 TPM / SB / RAM 绕过提供脱机插入的注册表项</p><p>● 移除毫无用处的虚假存储旁路</p><p>● 提升 BIOS 兼容性以减少‘仅限 UEFI 引导’报错</p><p>● 修复 Windows To Go 出现两次 Windows 用户体验对话框的问题</p><p>● 修复 Windows 用户体验选项不适用于 ARM64 的问题</p><p>● 修复除非选择了 TPM / SB / RAM 绕过，否则无法应用 Microsoft Account 绕过的问题。</p></blockquote><p style="text-align: center;"><img src="https://static.cnbetacdn.com/article/2022/0728/6ebcbd0c8f01b7a.png" alt="rufus_en.png" referrerpolicy="no-referrer"></p><p><a href="https://www.neowin.net/news/the-latest-rufus-update-lets-you-copy-regional-settings-and-create-a-local-windows-account/" target="_self">Neowin</a> 指出，Rufus 最近成为了哪些想要使用 Windows 11 操作系统，但又不想被各种繁杂的规则所限制的用户们的首选实用程序。</p><p>通过 Rufus，用户可轻松制作绕过“可信平台模块”（TPM）与安全启动（Secure Boot）要求的 Windows 11 安装媒介。</p><p>与此同时，3.19 稳定版也引入了一个急需的选项，以在初始设置期间绕过对<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>账户（Microsoft Account）的要求。</p><p><strong>下载地址：</strong></p><blockquote><p>● Rufus 3.20 <a href="https://github.com/pbatard/rufus/releases/download/v3.20_BETA/rufus-3.20_BETA.exe" target="_self">BETA</a></p><p>● Rufus 3.19 <a href="https://rufus.ie/en/" target="_self">稳定版</a></p></blockquote>
