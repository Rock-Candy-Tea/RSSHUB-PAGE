网页浏览器中的标签页大幅优化了我们的浏览体验，能够让你轻松地管理多个网络会话。但是在 Windows 系统的资源管理器中，并不支持标签化操作。不过在未来的 Windows 11 系统版本中，我们将会获得这样的体验。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0325/fe4ed140025d438.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">资源管理器中的标签化体验如何呢？假设你在文件资源管理器窗口中打开了一个目录，你想要从另一个目录或文件夹中轻松复制一个文件。那么你所需要做的就是在另一个标签中打开第二个文件夹，并轻松地在目录之间移动项目或比较信息。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0325/67c3c034506be71.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">正如你在下面的截图中所看到的，你可以简单地点击带有“加号”图标的新按钮，在文件资源管理器中打开标签，就像你在<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a> Edge 或 Google Chrome 中打开一个新标签一样。然后，你可以在主窗口中作为标签启动更多的资源管理器窗口。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0325/5f633cf118efc3c.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">标签式文件资源管理器的早期版本首次出现在 Build 22572 中，这是 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 22H2 版本的开发版本，但它在更新的 Build 22579 中被删除。随后，它在 Windows 11 的一个小更新（Build 22579.100）中重新出现，但现在该功能已从本周的最新更新（Build 22581）中删除。</p><p style="text-align: left;">Build 22581 供 Dev 频道用户使用，该构建中包含的所有功能事实上都与 Sun Valley 2 绑定。与 Dev 频道即将包括23H2版本的构建不同，测试频道将有来自NI（Nickel）开发分支的构建，这是 Sun Valley 2 开发的分支，即22H2。</p>
