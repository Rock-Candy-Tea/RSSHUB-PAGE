<strong>本周二，谷歌推送了 Chrome 浏览器的 101 稳定版更新，并且带来了诸多改进。</strong>首先是提供了联合凭据管理 API（FedCM）的早期试用，据说该应用程序接口旨在保护身份隐私的情况下继续使用相关用例、而无需借助扩展点追踪（比如第三方 cookie）。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0427/58ca317a329ad72.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">视频封面（来自：Google <a href="https://developer.chrome.com/blog/new-in-devtools-101/" target="_self">Chrome Developers</a>）</p><p>不过目前 FedCM 仅在 Android 移动平台上的 Chrome 101 稳定版上启用，桌面版还得等到下一次 Chrome 102 版本。</p><p><img src="https://static.cnbetacdn.com/article/2022/0427/4ce8c5f60c23630.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p><strong>其它变化还有：</strong></p><blockquote><p>● 优先级提示（Priority Hints）可用于指示 Web 浏览器资源的相对重要性，反之也能够在加载不同资源时更好地调节优先级。</p><p>● 字体面板（font-palette）的 CSS 属性，现允许开发者从颜色字体里选择调色板。</p><p>● 支持 hwb() 的 CSS 函数，用于为色相黑 / 白度（HWB）值指定 sRGB 颜色。</p><p>● 支持 WebRTC 流的 MediaCapabilities API 。</p><p>● 支持安全支付确认用的 API v3 。</p><p>● 通用串行总线设备（USBDevice）现可通过 forget() 方法，来自愿撤销相关权限。</p><p>● 随着 WebSQL 数据库标准被抛弃十年，Chrome 101 终于剔除了对第三方上下文 WebSQL 的支持。</p></blockquote><p style="text-align: center;"><iframe src="//tv.sohu.com/s/sohuplayer/iplay.html?bid=344115705&autoplay=false&disablePlaylist=true" width="640" height="480" frameborder="0"></iframe></p><p style="text-align: center;">Chrome 101 - What’s New in DevTools（<a href="https://tv.sohu.com/v/dXMvODIyMjQwNTMvMzQ0MTE1NzA1LnNodG1s.html" target="_self">via</a>）</p><p>此外 <a href="https://chromestatus.com/features#milestone%3D101" target="_self">ChromeStatus</a> 网站上也概述了 Chrome 101 的相关变动，且 <a href="https://chromereleases.googleblog.com/2022/04/stable-channel-update-for-desktop_26.html" target="_self">Chrome Release</a> 博客提到了面向各大主流平台的 Chrome v101 稳定版更新的安全修复程序。</p><p>与此同时，Chrome 102 正在开发一种新的句柄捕获（capture handle）机制，以允许应用程序将某些信息公开给正在捕获它的相关应用程序（比如在两者之间协作）。</p><p>最后还有改进的文件处理集成、HTTP 到 HTTPS 重定向的动态域名解析（DNS）记录、支持“inert”属性，以及其它诸多更改。</p>
