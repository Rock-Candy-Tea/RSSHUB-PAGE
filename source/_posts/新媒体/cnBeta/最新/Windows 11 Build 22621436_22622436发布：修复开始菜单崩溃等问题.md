数周前，微软将 Windows Insider 项目的 Beta 频道分割成为两组，以更好地测试 Windows 11
系统更新。今天，微软发布了 Windows 11 Build 22621.436 和 Build 22622.436
两个版本更新，修复了开始菜单崩溃和黑屏在内的诸多问题。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0720/7337fdb8d4fa553.webp" alt="fuujt1mb.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">在官方博文中写道</p><blockquote style="text-align: left;"><p style="text-align: left;">今天我们面向 Beta 频道发布了 Windows 11 Build 22621.436 和 Build 22622.436（KB5015888） 两个更新。我们还将会在明天为 Beta 频道用户启动新一轮的 Bug Bash 活动，细节如下：</p><p style="text-align: left;"><strong>● Build 22622.436 = 获得新功能</strong></p><p style="text-align: left;"><strong>● Build 22621.436 = 默认不启用新功能</strong></p><p style="text-align: left;">注：Build 22621.xxxx 组别在默认情况下会禁用新功能，但用户可以手动检查更新。而 Build 22622.xxxx 组别会获得新的功能。</p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22622.436 新功能</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>改善 Nearby Sharing</strong></p><p style="text-align: left;">使用内置的 Windows 分享窗口分享来自桌面、File Explorer、Photos、Snipping Tool、<a data-link="1" href="https://microsoft.pvxt.net/e4yLO" target="_blank">Xbox</a> 等其他应用的本地文件之时，Nearby Sharing 功能可以发现更多的设备，可以使用 UDP（网络需要设置为私有）和蓝牙来发现附近的设备。这意味着你可以发现和分享包括桌面 PC 在内的更多设备。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0720/037eacfe040fdb3.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;"><strong>分享本地文件至 OneDrive</strong></p><p style="text-align: left;">使用内置的 Windows 分享窗口分享来自桌面、File Explorer、Photos、Snipping Tool、Xbox 等其他应用的本地文件之时，你能够选择 OneDrive，那么该文件就会直接上传到 OneDrive。在文件管理器中的整个分享上传过程并不需要额外的内容切换或者打开 OneDrive 应用。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0720/f8b2409ea0ed39d.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">需要注意的是，该应用目前仅限于已登录 Microsoft 账号设备。用户需要在分享窗口切换到<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>账户，在未来将会支持通过 AAD 方式登陆。</p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22622.436 修复和改进</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>[Windows Terminal]</strong></p><p style="text-align: left;">Windows Terminal 现在成为 Windows 11 系统的默认终端。这意味着所有命令行的应用都会自动在 Windows Terminal（例如 Command Prompt 和 PowerShell）中打开。这项设置可以在Settings > Privacy & security > For developers 中进行修改。</p><p style="text-align: left;"><strong>[输入]</strong></p><p style="text-align: left;">在 Emoji 面板（Win键+.键组合）你可以举报认为不合适的动画 GIF。</p></blockquote><p style="text-align: left;">Windows 11 Build 22622.436 修复的 BUG</p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>[文件资源管理器］</strong></p><p style="text-align: left;">● 修复了一个问题，即如果你在文件资源管理器中重新安排了标签，那么在使用CTRL + Tab时，标签顺序将是错误的。</p><p style="text-align: left;">● 修复了在拖动标签时explorer.exe崩溃的问题。</p><p style="text-align: left;">● 如果启用了"显示所有文件夹"，文件资源管理器的导航窗格中的分隔符应该不再显示。有了这个改变，它也应该能解决在其他一些文件夹选取器中意外显示分隔线的问题。</p><p style="text-align: left;">● 在一个新的标签页中打开一个压缩的文件夹，应该不再有一个空白的标签页名称。</p><p style="text-align: left;">● 可移动驱动器不应该再意外地在导航窗格中单独显示在一个部分中，这使得该部分与"本机"和"网络"分开。</p><p style="text-align: left;">● 修复了一个问题：当使用Aquatic或Desert对比主题时，添加新标签的按钮不清晰可见。</p><p style="text-align: left;">● 在当前标签和命令栏之间不再显示一条模糊的线。</p><p style="text-align: left;">● 更新了悬停在标签上时的工具提示，建议使用CTRL + W来关闭标签（而不是CTRL + F4，后者不起作用）。</p><p style="text-align: left;">● 如果焦点在标签行上，CTRL + W将不再意外地关闭两个标签，而不是只关闭焦点中的标签。</p><p style="text-align: left;">● 修复了一个问题，即标签行可能意外地垂直扩展，覆盖了命令栏的内容。</p><p style="text-align: left;"><strong>[开始]</strong></p><p style="text-align: left;">● 修复了Build 22622.160版本中影响一些内测者的启动崩溃问题。</p></blockquote><p style="text-align: left;"><strong>Build 22621.436 和 Build 22622.436 共同修复的 BUG</strong></p><blockquote style="text-align: left;"><p style="text-align: left;">● 修复了一个阻止故障排除器打开的问题。</p><p style="text-align: left;">● 修复了一个导致智能应用控制阻止目录签名文件的问题。</p><p style="text-align: left;">● 修复了一个可能使您使用相机应用拍摄的照片高度失真的问题。这个问题在某些低光照条件下使用某些相机时发生。</p><p style="text-align: left;">● 修复了一个问题，当你使用<a data-link="1" href="https://microsoft.pvxt.net/zaZYr" target="_blank">Visual Studio</a> 2022 17.2或更高版本调试驱动程序时，会抛出一个异常。</p><p style="text-align: left;">● 修复了一个导致Windows配置文件服务零星失败的问题。该故障可能在登录时发生。错误信息是："gpsvc服务登录失败。拒绝访问"。</p><p style="text-align: left;">● 修复了一个问题，它使虚拟化的App-V <a data-link="1" href="https://microsoft.pvxt.net/P0JMe" target="_blank">Office</a>应用程序无法打开或导致它们停止响应。</p><p style="text-align: left;"><strong>[通用]</strong></p><p style="text-align: left;">● 修复了一个问题，导致Beta通道中的少数内部人员在Explorer.exe和其他Windows UI组件中出现周期性崩溃，使屏幕看起来像在闪烁。请注意，这次修复应该可以阻止更多的Insiders受到新的影响，但是如果你已经受到了影响，你需要在PowerShell中运行以下命令来解决。Add-AppxPackage -Register -Path C:\Windows\SystemApps\Microsoft.UI.Xaml.CBS_8wekyb3d8bbwe\AppxManifest.xml -DisableDevelopmentMode -ForceApplicationShutdown</p><p style="text-align: left;">● 修复了一个问题，该问题导致一些在之前的Beta通道飞行的Insiders在从睡眠中醒来后，他们的电脑会卡在一个黑屏上。</p><p style="text-align: left;">● 修复了一个问题，该问题在打开移动热点后会导致失去网络连接。</p><p style="text-align: left;">● 修复了一个导致Build 22621.290 / 22622.290安装失败的问题，对于一些Insiders，错误为0x800f081f。</p></blockquote>
