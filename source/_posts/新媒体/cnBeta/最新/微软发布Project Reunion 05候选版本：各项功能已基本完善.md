Build 2020 开发者大会上，微软宣布了 Project Reunion。该项目旨在减少 Windows API 和 Universal Windows Platforms 之间的碎片化，从而开发者更容易为所有 Windows 设备创造应用。在推出了不少预览版更迭之后，<a href="https://blogs.windows.com/windowsdeveloper/2021/03/29/announcing-project-reunion-0-5/" target="_blank">今天微软推出了 0.5 版本更新</a>，各项功能已基本完善，这是首个生产就绪（production ready）版本，也就是稳定版的候选版本。<br>
 <p style="text-align:center"><a href="https://static.cnbetacdn.com/article/2021/0330/c2910166790f71b.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0330/c2910166790f71b.jpg" alt="nr5bribn.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: left;">微软表示，该版本的最终目标就是让桌面应用更容易创建。该版本最低支持 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 10 Version 1809 功能更新，能够将 Project Reunion 与 .NET 5、WinUI 3 和 WebView2 应用一起使用。这些功能将作为 Project Reunion 框架包的一部分出货。</p><p style="text-align: left;">如果你对该项目感兴趣，可以阅读这个<a href="https://docs.microsoft.com/en-us/windows/apps/project-reunion/get-started-with-project-reunion" target="_blank">开发者文档</a>或者<a href="https://github.com/microsoft/ProjectReunion/discussions/657" target="_blank">前往 Github</a>。需要注意的是，Project Reunion 0.5 支持打包的应用程序，它使用 <a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://msi-pc.jd.com/" target="_blank">MSI</a>X 作为部署方法。这也意味着，这个版本是第一个适合生产型应用的版本。它也是向前兼容的。</p><p style="text-align: left;">微软表示，还有更多的内容正在进行中。Project Reunion 0.8 应该会在未来几个月内推出，而 Project Reunion 1.0 将在今年晚些时候推出。</p>
