Tom's Hardware 报道称，AMD 正探索为 300 系芯片组添加对锐龙 5000 系列 CPU 的支持。<strong>公司副总裁兼客户渠道业务总经理 David McAfee 在接受采访时称，这绝对是他们正在努力解决的问题之一：</strong>“我们没有彻底忘记这件可让社区受益的事情，并且正在努力搞清楚将如何去实现”。<br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0108/a1afa45a9fc62e5.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0108/a1afa45a9fc62e5.png" alt="1.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">截图（来自：<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://amd-cpu.jd.com/" target="_blank">AMD</a> <a href="https://www.amd.com/en/products/chipsets-am4" target="_self">官网</a>）</p><p>此前围绕 AMD AM4 平台的一大争议，就是尽管该公司承诺了长期支持该平台，但新老芯片组主板却未能实现对锐龙 CPU 的同等支持。</p><p>作为 AMD 有史以来最长寿的消费级平台之一，AM4 接口的处理器型号也变得繁多，导致主流 16MB 的主板 BIOS 空间（SPI ROM）变得有些捉襟见肘。</p><p>如果不将 BIOS 容量升级到 32MB，厂商将被迫放弃图形用户界面（GUI），或剔除部分 CPU 的支持代码。</p><p><img src="https://static.cnbetacdn.com/article/2022/0108/2150bf7bb02c3d0.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p>尽管 David McAfee 的最新表述算不上是一个承诺，但此前不乏“妖板”厂商突破 AMD 的某些限制 —— 比如许多供应商为入门级的 A320 芯片组，添加了对 AMD 锐龙 5000 系列 CPU 的支持。</p><p>虽然出于主板供电与扩展性能的考虑，中高端的 B350 / X370 主板可能更加合适。但对于那些早期选用 AM4 平台的老用户，能否获得对锐龙 5000 CPU 的支持，还得看 AMD 与主板厂商的是否有心推动。</p><p>David McAfee 补充道：“老实说，这个话题已在 AMD 内部引发大量关注和讨论。我这么说并不是在开玩笑，但就今天一天，我就三次谈论了相关话题，且不包含与媒体的沟通”。</p><p><a href="https://static.cnbetacdn.com/article/2022/0108/3a9f93e330a5857.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0108/3a9f93e330a5857.jpg" alt="3.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（via <a href="https://www.tomshardware.com/news/amd-exploring-ryzen-5000-support-on-300-series" target="_self">Tom's Hardware</a>）</p><p>据悉，AMD 工程团队正在商讨他们能够为此做些什么，以及如何为想要升级至锐龙 5000 系列 CPU 的 300 芯片组主板用户带来相适应的体验。</p><p>当然，在具体实施过程中，技术攻关可能较预期更加复杂。毕竟 AMD 旗舰款锐龙 R9-5850X 处理器拥有 16 核 / 32 线程，而当初为 8 核 CPU 设计的主板供电等配套组件，或在单独验证期间面临相当大的挑战。</p>
