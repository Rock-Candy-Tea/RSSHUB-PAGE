自 Linux Kernel 4.8 版本开始，Linux 团队就开始整合名为 eXpress Data Path (XDP)
的高性能网络项目。该项目已经得到了包括亚马逊、Google、英特尔在内的诸多大型科技公司的支持，允许应用程序在网络中实现低延迟和高吞吐量。<strong>现在，微软也加入该行列,还推出了基于
XDP 的 XDP for Windows 开源项目。</strong><br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0525/7eb745c671908f0.webp" alt="ii8k259p.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;"><a href="https://techcommunity.microsoft.com/t5/networking-blog/open-source-xdp-for-windows/ba-p/3421922" target="_blank">该公司称这是其对XDP社区的第一个贡献</a>，并表示倾向于使网络数据路径真正跨平台。目前，XDP for <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 包含一个网络驱动程序和一个占位的用户模式 API。微软强调，该项目仍在积极开发中，但应该已经准备好进行测试和原型设计。该公司还在与其他公司合作，并在MsQuic协议中增加对XDP的支持。</p><p style="text-align: left;">微软指出，任何人都可以克隆 XDP for Windows 资源库并开始构建，因为该项目旨在加速多种情况下的网络性能。该公司正在增加更多的功能，如校验和卸载以及与eBPF for Windows的整合，但与此同时，你可以在GitHub上查看XDP for Windows。</p>
