<strong>红帽一直努力为其 Anaconda 操作系统安装程序和 Fedora 37 版本开发基于 Web 的新 UI，辅以一个预览选项。</strong>此前 RHEL 和 Fedora 长期使用基于 GTK 的安装图形界面，但最近几个月一直在积极酝酿转变，且该 UI 能够整合为红帽的 Cockpit 远程项目的一部分 / 无头安装。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0809/b566393f128d7af.webp" alt="1.webp" referrerpolicy="no-referrer"></p><p style="text-align: center;">（via <a href="https://www.phoronix.com/news/Anaconda-Web-UI-FESCo-F37" target="_self">Phoronix</a>）</p><p>虽然新 UI 仍在开发中，但他们正按部就班地 Fedora 37 带来基于 Web 的新安装 UI 预览，以在正式发布前积累足够多的测试反馈。</p><p>据悉，Anaconda Web UI 将提供基于众所周知、且广泛使用的 PatternFly 工具包打造，并由经过验证的 Cockpit 工具为更现代的响应式 UI 提供支撑。</p><blockquote><p>屏幕布局基于最新的用户体验（UX）设计指南，辅以可用性测试和大量模型工作。</p><p>另一方面，得益于选择了更现代和更主流的 UI 技术和强大的 Cockpit 测试工具（丰富的单元测试 / 像素测试框架），开发人员的体验也得到了极大的改善。</p></blockquote><p><img src="https://static.cnbetacdn.com/article/2022/0809/82500ed630b7c81.webp" alt="2.webp" referrerpolicy="no-referrer"></p><p>Web UI 的无状态属性，允许几乎实时的编码风格 / UI 开发。对于 Anaconda 团队、插件开发人员、以及任何感兴趣的贡献者来说，都可更轻松地在 Anaconda Web UI 上工作。</p><p>此外远程 Web UI 的访问，也有望较当前缓慢低效、基于 VNC 的远程 GUI 安装方案带来更好的使用体验 —— 无需在受限硬件上本地渲染远程驱动的 GUI 安装（精简镜像）。</p><p>最新消息称， Fedora 工程指导委员会（<a href="https://pagure.io/fesco/issue/2839#comment-809730" target="_self">FESCo</a>）已批准了 Fedora 37 的这项新特性。如果一切顺利，其有望于 10 月底随 Fedora 37 一起正式到来。</p><p>当然，红帽并不是唯一追求基于现代 Web UI 的操作系统安装体验的开发商 —— 比如 SUSE 正在努力开发的 <a href="https://github.com/yast/d-installer/pull/223" target="_self">D-Installer</a> 。</p>
