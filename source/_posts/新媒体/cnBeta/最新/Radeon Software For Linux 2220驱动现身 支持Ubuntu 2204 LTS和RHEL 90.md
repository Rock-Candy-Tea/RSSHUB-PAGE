对于使用消费者型号的Radeon GPU的Linux用户来说，驱动的更新频率与Windows相比相差甚远，"Radeon Software
for Linux"打包的驱动版本自3月底以来一直停留在22.10系列上。而对于AMD Radeon
PRO专业/工作站图形卡，官方"Radeon Pro Software for Enterprise on
Linux"驱动程序仍然是去年12月的21.Q4版。<br>
 <p>不过现在，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://amd-cpu.jd.com/" target="_blank">AMD</a>似乎最终准备发布"22.20"打包的驱动程序，作为这个打包的AMD Linux驱动程序栈的下一个功能版本，供那些不只是依赖上游内核和Mesa的用户使用。</p><p><img src="https://static.cnbetacdn.com/article/2022/0629/f39d6e0b4180c93.jpg" title alt="image.jpg" referrerpolicy="no-referrer"><a href="https://static.cnbetacdn.com/article/2022/0629/827a96bdac88cab.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0629/827a96bdac88cab.jpg" title alt="1.jpg" referrerpolicy="no-referrer"></a></p><p>Radeon Software for Linux打包的驱动版本主要针对运行企业级Linux发行版（如SUSE、Ubuntu和RHEL/CentOS）的企业/专业用户。除非需要OpenCL/ROCm或有工作负载，专有的Vulkan驱动程序表现得更好，否则玩家和其他运行尖端Linux发行版的人通常最好只运行那些上游Linux内核/Mesa组件以获得最佳支持。</p><p>距离上一次打包的Radeon Linux驱动更新已经有一段时间了，现在22.20已经上传到了AMD软件包档案。尽管AMD官方还没有宣布22.20驱动程序，但它在今天出现在repo.radeon.com，并且可以获取软件包的二进制安装程序。</p><p>由于AMD.com还没有发表公告，正式的发布说明还没有公布以详细说明这个新的打包的Linux驱动版本中的所有变化。但最值得注意的是，当通过Radeon资源库时，现在有一个Jammy的条目引起了桌面用户的注意，这意味着Ubuntu 22.04 LTS现在将被支持。Ubuntu 22.04 LTS早在4月就发布了，但AMD在长达两个月的时间里都没有给出去动，现在随着我们即将进入7月，看起来他们将准备好他们的官方Ubuntu 22.04支持。</p><p>同样，新的22.20软件包也是RHEL方面第一个支持Red Hat Enterprise Linux 9.0的软件包。RHEL 9.0在5月达到普遍可用装填。在SUSE方面，最近发布的SUSE Enterprise Linux 15 SP4现在也被支持。</p><p>因此，至少企业级Linux发行版都将被这个即将发布的22.20系列打包的Linux驱动所支持，现在只是在等待AMD.com上的正式发布，22.20软件包可以在正式发布前从<a href="http://repo.radeon.com/" target="_blank">repo.radeon.com</a>下载。</p>
