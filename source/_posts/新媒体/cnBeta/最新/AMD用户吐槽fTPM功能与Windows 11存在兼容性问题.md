在推广 Windows 11 操作系统的同时，微软也提出了 TPM 2.0 这项硬性要求。不过除了存在争议的旧处理器的支持，大多数新 PC 都已经具备了这项安全特定。<strong>尴尬的是，在 AMD 平台上，该公司选用了基于系统固件的 fTPM 方案，而不是配备专用的可信平台芯片。</strong>且在某些情况下，启用 fTPM 或导致性能方面的问题。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0126/88fbf4b17531ab1.jpg" alt="1.jpg" referrerpolicy="no-referrer"></p><p>在<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://intel.jd.com/" target="_blank">英特尔</a>平台上，可信平台模块（TPM）实施起来还是相对轻松的。然而在 <a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://amd-cpu.jd.com/" target="_blank">AMD</a> 平台上启用了 fTPM 之后，不少人汇报其遇到了音频故障、或掉帧卡顿等问题。</p><p>显然，此事与 AMD fTPM 方案和 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 之间的兼容性有关。启用 fTPM 或导致设备性能降低，以及在 Windows 10 / 11 操作系统上玩游戏时遭遇卡顿。</p><p>在<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>反馈中心、Reddit 等社交媒体、以及各大科技新闻网站的评论区，已有许多 AMD 用户留言称遇到了同样的问题。</p><p><a href="https://static.cnbetacdn.com/article/2022/0126/355ef71f4e5856f.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0126/355ef71f4e5856f.jpg" alt="2.jpg" referrerpolicy="no-referrer"></a></p><p>比如一位 Reddit 网友指出，其使用的是锐龙 R5-1600AF 处理器，主要问题是电脑会出现随机的卡顿和爆音。</p><p>另一位 Reddit 网友使用的是锐龙 R9-3900X 处理器 + <a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://msi-pc.jd.com/" target="_blank">微星</a> MEG X570 ACE 主板，意外的是同样在启用 fTPM 后遇到了随机问题。</p><p>如果不是播放音乐，可能到现在也不知道自己已经躺枪 —— 即便 Windows 11 专业版操作系统、主板 BIOS 和驱动程序都已是最新。</p><p><img src="https://static.cnbetacdn.com/article/2022/0126/9457e1836704e48.jpg" alt="3.jpg" referrerpolicy="no-referrer"></p><p>庆幸的是，该问题并未影响所有 PC，且似乎更多地出现在从 Windows 10 升级到 Windows 11（而不是全新安装）的系统中。</p><p>此外问题也没严重要让机器彻底变砖的地步，顶多在随机情况下持续个几秒钟的时间。</p><p>如果实在难以忍受，不妨尝试在 UEFI BIOS 设置中临时禁用 fTPM 功能（Valorant 等游戏将无法运行），直到微软和 AMD 给出更正式的解决方案。</p>
