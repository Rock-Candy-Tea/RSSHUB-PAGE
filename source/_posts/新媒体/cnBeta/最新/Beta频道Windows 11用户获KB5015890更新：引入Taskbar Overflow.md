几周前，微软将 Beta 频道的 Windows Insider 项目成员分成 2 组，以便于更好的交错测试新功能。今天，微软面向该频道发布了更新，用户在升级之后版本号为 Build 22621.440 和 Build 22622.440。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0729/cff68b5c4357e39.webp" alt="l2mc2dbv.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">公司在博文中写道：</p><blockquote style="text-align: left;"><p style="text-align: left;">Windows Insider 们，大家好。今天我们面向 Beta 频道用户发布了 Windows 11 Insider Preview Build 22621.440 和 Build 22622.440 (KB5015890) 。</p><p style="text-align: left;"><strong>Build 22622.440 = 已部署新功能</strong></p><p style="text-align: left;"><strong>Build 22621.440 = 新功能默认情况下未启用</strong></p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22622.440 新内容和改进</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>Taskbar Overflow</strong></p><p style="text-align: left;">微软希望通过 Taskbar Overflow，来重新设计 Windows 11 的使用体验。该任务栏体验的设计初衷是，在任务栏空间受限的情况下，让你的应用切换和打开更具生产力。</p><p style="text-align: left;">在任务栏应用数量超出长度之后，这些应用图标就会自动折叠。在这种状态下，任务栏会提供一个访问折叠应用图标的“...”按钮，允许用户查看被折叠的应用图标。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0721/74b53c7a5e8fa83.webp" alt="s0vr7ufq.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">Taskbar Overflow 的折叠菜单同样会包含目前用户熟悉的任务栏操作：包括固定应用、jump list、和扩展 UI 等等。当你点击外部区域或者导航到某个应用的时候，这个折叠菜单就会自动隐藏。</p></blockquote><p style="text-align: left;"><strong>以下是 Windows 11 Build 22621.440 和 Build 22622.440 都具备的新功能</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>在任务栏上展示 Dynamic Widgets 内容</strong></p><p style="text-align: left;">微软曾尝试做些调整，在你的任务栏上引入更多的 Dynamic Widgets 内容。除了关于天气 Widgets 的动态内容之外，用户即将可以看到体育赛事、金融以及突发新闻、警报 Widget 的动态更新</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0729/6a11cfe5a280023.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">通过动态信息，能够让你更轻松地知道当前发生的重要大事，并让你追踪突发新闻。</p><p style="text-align: left;">如果用户点击任务栏上的这个 Widget 板，那么用户可以更快速、更全面地了解更多内容。但是如果你并未和这些动态内容进行交互，那么将会重新返回显示天气信息。</p><p style="text-align: left;">微软正积极推广这项功能，因此目前并非所有 Beta 频道用户都可以使用该功能。</p></blockquote><p style="text-align: left;"><strong>Windows 11 Build 22622.440 提供了如下改进：</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>[常规]</strong></p><p style="text-align: left;">● 微软开始再次推出更新后的“打开方式”对话框，以符合 Windows 11 的设计原则。更新后的对话框支持浅色和深色主题。我们还通过一键更新默认应用程序来简化体验。自 3 月份恢复此更改以来，我们已根据 Insider 反馈对性能进行了改进。</p><p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0729/d0003bd2f4feaae.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;"><strong>[输入]</strong></p><p style="text-align: left;">● 微软更新了美国英语 (EN-US) 手写模型，使其更快、更准确。使用更新的手写面板进行墨迹转换，让我们知道您的想法。</p><p style="text-align: left;"><strong>[设置]</strong></p><p style="text-align: left;">● 设置现在支持管理以前仅从控制面板支持的应用程序。这包括卸载具有相互依赖关系的应用程序（例如，在 Steam 上运行的 Steam 和游戏应用程序）、修复和修改 Win32 应用程序。</p></blockquote><p style="text-align: left;"><strong>以下是 Windows 11 Build 22622.440 中的错误修复：</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>[常规]</strong></p><p style="text-align: left;">● 修复了导致某些 Insiders 在 Build 22622.436 中对接和取消对接监视器时遇到 explorer.exe 崩溃的问题。</p><p style="text-align: left;"><strong>[文件管理器]</strong></p><p style="text-align: left;">● 在文件资源管理器中使用选项卡时，做了一些修复内存泄漏的工作。</p><p style="text-align: left;">● 修复了当焦点在标签中移动时讲述人没有读出标签标题的问题。</p><p style="text-align: left;">● 修复了当焦点在标签中移动时讲述人没有读出标签标题的问题。</p><p style="text-align: left;">● 当使用文本缩放并打开许多选项卡时，添加新选项卡按钮不应与标题栏中的最小化按钮重叠。</p><p style="text-align: left;"><strong>[<strong>Suggested Actions</strong>]</strong></p><p style="text-align: left;">● 修复了如果启用了建议操作，则会导致某些应用程序在复制操作后冻结的问题。</p><p style="text-align: left;">● 修复了影响建议操作可靠性的高命中崩溃。</p></blockquote><p style="text-align: left;"><strong>已知的问题</strong></p><blockquote style="text-align: left;"><p style="text-align: left;"><strong>[常规]</strong></p><p style="text-align: left;">● [新] 最近的 Beta 频道更新中存在导致照片应用程序崩溃的问题，并且很快将通过商店的照片应用程序更新推出修复程序。</p><p style="text-align: left;">● [新] 我们正在调查有关 SQL Server Management Studio 无法为某些内部人员启动的报告。</p><p style="text-align: left;"><strong>[文件管理器]</strong></p><p style="text-align: left;">● [新] 文件资源管理器标题栏的左半部分可能无法通过鼠标或触摸拖动。</p><p style="text-align: left;">● [新] 我们正在努力解决以下报告：在使用暗模式（例如，从命令行）时以某些方式启动文件资源管理器会在亮模式下意外显示文件资源管理器的主体。</p><p style="text-align: left;">● 文件资源管理器选项卡中的向上箭头未对齐。这将在未来的更新中修复。</p></blockquote>
