过去一段时间，不少 AMD 锐龙 CPU 用户吐槽 Radeon 图形驱动程序，称其会扰乱系统固件中的某些 CPU 特定设置。<strong>在媒体的广泛走访与调查后，AMD 现终于意识到了这个问题的存在。</strong>遗憾的是，该公司暂未披露任何可能导致 Windows 11 设备用户躺枪的问题细节，也没有给出预估的 bug 修复日期。<br>

referrerpolicy="no-referrer"></a></p><p style="text-align: center;">资料图（via <a href="https://www.computerbase.de/forum/threads/adrenalin-22-3-1-mischt-sich-ins-bios-ein-und-verursacht-crashes.2077733/#post-26741015" target="_self">ComputerBase</a> / der-Kalli）</p><p>AMD 在致 <a href="https://www.tomshardware.com/news/amd-confirms-its-gpu-drivers-are-overclocking-cpus-without-asking" target="_self">Tom's Hardware</a> 的一份声明中称：</p><blockquote><p>我们得知 AMD 软件套件中的一个问题，正在扰乱某些用户的 AMD 处理器设置。</p><p>我们正在就此事展开调查，并将尽快分享更多信息。</p></blockquote><p><a href="https://static.cnbetacdn.com/article/2022/0401/e4b51a628cee5b2.png" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0401/e4b51a628cee5b2.png" alt="2.png" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（via <a href="https://www.reddit.com/r/Amd/comments/ruuqdv/amd_you_need_to_completely_remove_your_buggy_cpu/" target="_self">Reddit</a> / gaojibao）</p><p>据悉，AMD 在去年的 21.9.1 版 Radeon GPU 驱动程序中，正式引入了对 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 操作系统的支持。</p><p>与此同时，该公司为 Radeon Software 添加了一项名为“Auto Overclock”的超频功能，以便用户能够在一处轻松调节 CPU / GPU 的超频设置。</p><p><a href="https://static.cnbetacdn.com/article/2022/0401/6d2e2939eab3881.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0401/6d2e2939eab3881.jpg" alt="3.jpg" referrerpolicy="no-referrer"></a></p><p>尴尬的是，即使用户没有实施任何超频配置，该功能也会在用户不知情的情况下，默认覆盖设备的固件设置 —— 结果导致许多人在网络上吐槽他们的 PBO 设置被扰乱。</p><p><img src="https://static.cnbetacdn.com/article/2022/0406/93be75844004469.png" alt="4.png" referrerpolicy="no-referrer"></p><p>其实早在 7 个月前，就有一位名叫 TalesofWin 的 Reddit 网友，在 r/AMD 子版块上发起过这一讨论。</p><p>至于 PBO，可知作为 AMD 锐龙 CPU 的一项自动超频功能，Precision Boost Overdrive 会智能分析 VRM 电流、SoC 功率和温度，以确定有多少余量可以提升处理器的时钟频率。</p><div class="article-relation"><p><strong>相关文章:</strong></p><p><a
