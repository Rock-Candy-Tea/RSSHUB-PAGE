<strong>2022 年会成为 Linux 桌面元年吗？答案可能依然是否定的，但我们越来越接近了。</strong>今年早些时候，TechRepublic 认为虽然 2021 年不是 Linux 桌面元年，但不可否认的是 Linux 在企业领域持续主导地位、而 Linux 在桌面上也缓慢（和微妙）增长。而在几乎所有的领域（除了智能手机领域），Linux 都取得了一些重大进展。<br>
 <p><iframe src="//player.bilibili.com/player.html?bvid=BV1gr4y1v7Lp&page=1" width="750" height="480" frameborder="0"></iframe></p><p>所谓的 Linux 桌面元年，是指将来自由开源的操作系统将达到一个阶段，普通用户可以在他们的电脑上安装和使用它而不会遇到问题。Linus Tech Tips 的 Linus Sebastian 最近做了一个实验，他在自己的家用电脑上安装了一个月的 Linux，看看他是否不仅可以用它来完成日常任务，还可以用来玩游戏和播放流媒体。</p><p>最终他得出结论（在刚刚发布的视频中），今年不会是 Linux 桌面元年，虽然做日常工作还算可以，但 Linux 上的游戏状况（尽管 Valve 的目标很高），非常糟糕（shit-show）。</p><p>Sebastian 所做的实验似乎表明，虽然有些游戏使用 Proton（Valve 的 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 兼容层）确实运行良好，但也有同样多的游戏运行有问题。其中一些问题可能会破坏游戏。比如游戏在运行，但它的多人游戏功能却完全不工作。有些游戏由于依赖易反作弊等服务而根本无法运行......</p><p>在他的视频中，Linus 提到阻碍“Linux 桌面元年”的主要问题是碎片化。他所说的碎片化是指可用的发行版的范围，以及每个发行版都有（潜在的）不同版本的库、驱动和软件，使幕后操作....。Flatpak 和 Snap 以及 AppImage 在解决这个碎片化问题方面正在取得进展，但这些也还不够完美。Flatpak 的作用是确保该软件所需的预期版本的库与它一起安装，并且独立于发行版可能提供的现有库...</p><p>因此，尽管Linus确信2022年还不是Linux桌面的一年，而且碎片化是最大的问题。我想也许，只是也许，我们更接近于解决这些问题，也许更接近于一些人可能意识到的 Linux 桌面元年。</p>
