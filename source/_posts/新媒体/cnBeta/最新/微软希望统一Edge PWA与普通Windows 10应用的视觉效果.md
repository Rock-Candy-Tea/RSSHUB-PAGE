长期以来，微软一直试图通过“渐进式 Web 应用”（PWA）来弥补 Windows 10 操作系统在应用程序方面的空白，并使之在视觉效果上更接近于原生应用程序的体验。<strong>有趣的是，在近日披露的一份技术文档中，我们就见到了微软的下一步规划。</strong>techTSP 指出，Edge PWA 应用将抛弃标题栏、并将该区域的控制权完全交到开发者手上。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0329/778d01590c97a61.jpg" referrerpolicy="no-referrer"></p><p>Microsoft Edge 团队的 Amanda Baker 表示：“这项特性使得 Web 应用开发者们能够更好地利用屏幕空间，允许他们将之设计得更接近于操作系统上的原生应用程序”。</p><blockquote><p>显然，随着设备便携性的提升，传统标题栏会占用屏幕上较大的显示区域，结果就是减少了用户可与之交互的实际可用空间。</p><p>此外对于有视觉障碍的用户来说，如果增加了屏幕上所有内容的缩放级别，实际体验也会是一个更加麻烦的问题。</p></blockquote><p>好消息是，在改进之后，开发者无需将标题栏这一大块区域留白，而是将应用程序的标题栏融入标准的浏览器字体中，同时用自定义 Logo、搜索框、或导航组件来填充该区域。</p><p><img src="https://static.cnbetacdn.com/article/2021/0329/72fdb828febf599.png" alt="Desktop-PWA-windows-control-overlay.png" referrerpolicy="no-referrer"></p><p>与此同时，<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>打算与谷歌展开深入合作，以引入新的窗口布局控制（<a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> Control Overlay）功能，为开发人员提供针对该区域的更多控制权。</p><p><strong>以下是 Windows Control Overlay 的目标：</strong></p><blockquote><p>● 允许 Web 开发者访问渐进式 Web 应用的标题栏区域；</p><p>● 应用程序窗口必须具有包括最小化、关闭和拖动窗口之类的基础控件选项；</p><p>● Edge 浏览器可遵从主机系统的窗口控件设计；</p><p>● 带来尽可能接近于原生应用的视觉效果和使用体验。</p></blockquote><p>感兴趣的朋友，可在 Chrom://flags 标记中尝试开启这项特性，或到 <a href="https://github.com/WICG/window-controls-overlay/blob/main/explainer.md#examples-of-desktop-apps-customizing-the-title-bar-area" target="_self">GitHub</a> 上查看有关该项目的更多详情。</p>
