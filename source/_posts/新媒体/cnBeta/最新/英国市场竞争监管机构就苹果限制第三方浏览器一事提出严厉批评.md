最近，大型科技公司一直被反垄断机构给密切关照。比如英国竞争与市场管理局（CMA）负责人 Andrea Coscelli 就在一篇新闻稿中指出 —— <strong>以苹果和谷歌为代表的科技巨头，正牢牢钳制着广大智能机用户的日常体验，该机构很担心这会导致数以百万计的英国民众权益受损。</strong><br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0226/c7cf637a9e6acc8.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0226/c7cf637a9e6acc8.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">资料图（来自：Apple 官网）</p><p>在一众反竞争行为中，就包括 iOS / <a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fipad%2F" target="_blank">iPad</a>OS 上的默认 Web 浏览器。长期以来，业界一直在猛烈批评的这一做法，声称该政策强硬阻碍了非 WebKit 内核的浏览器登陆苹果移动设备平台。</p><blockquote><p>由该公司在 App Store 应用审查指南中的第 2.5.6 条款可知，苹果有强制第三方软件开发者使用其主导的 WebKit 框架 / JavaScript 引擎。</p><p>围绕这一强制性要求引发的激烈争议持续已久，一些开发者和监管机构认为这么做会扼杀 iOS / iPadOS 平台上的积极创新。</p></blockquote><p>另一方面，苹果方便却辩称是出于安全和隐私的考虑，以防被占据市场主导地位的 Google Chromium 内核给染指。</p><p><img src="https://static.cnbetacdn.com/article/2022/0226/ae6423ab44454a9.png" referrerpolicy="no-referrer"></p><p>一些人认为，Chromium 占据的市场主导地位，正在导致浏览器文化变得更加单一、扼杀了竞争对手的网络技术发展。</p><p>比如苹果布道者兼 Safari 倡导者 Jen Simmons 就在一篇推文中，提到了维护 WebKit 限制的重要性。</p><blockquote><p>由 StatCounter 分享的统计数据可知，Safari 目前仅占据 9.84% 的桌面浏览器市场份额，而 Google Chrome
拥有压倒性的 65.38%，更别提其它套着 Chromium 内核马甲的第三方浏览器（包括 Microsoft Edge、Brave 和
Opera）。</p><p>至于移动平台，Safari 的市场地位仅次于 Google Chrome，且被认为较桌面平台更加安全。作为 <a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fiphone%2F" target="_blank">iPhone</a> / iPad 上的默认浏览器，Safari 在移动设备上的市场份额为 26.71，而 Chrome 在 Android / iOS 上占 62.06% 的主导地位。</p><p>此外其它基于 Chromium 内核的移动浏览器（例如 Microsoft Edge）也吃下了一定的份额。</p></blockquote><p>若苹果撤回强制在 iOS / iPadOS 上使用 WebKit 框架的要求，Chrome / Edge 等移动浏览器背后的开发者，显然更倾向于像桌面浏览器那样投奔 Chromium 内核，结果也会阻碍竞争对手的技术发展。</p><p><img src="https://static.cnbetacdn.com/article/2022/0226/33591789537047b.jpg" referrerpolicy="no-referrer"></p><p>参考 CMA 的移动生态系统中期市场<a href="https://www.gov.uk/government/publications/mobile-ecosystems-market-study-interim-report/interim-report" target="_self">报告</a>，可知苹果采用了以下声辩理由：</p><blockquote><p>苹果告诉我们，只允许在 iOS 平台上使用 WebKit，主要是出于隐私安全方面的顾虑，尤其在许多现代网站运行着来自未知开发者的代码的大环境下。</p><p>受 WebKit 限制，苹果得以为所有 iPhone 用户快速、有效地解决相关安全问题（假设仅限一款浏览器引擎）。</p><p>此外在苹果看来，WebKit 提供了较 Blink 和 Gecko 更高的安全防护级别，且单一引擎策略使之能够为自家移动设备带来全面的安全与隐私改进，从而带来更好的用户体验和防止碎片化。</p></blockquote><p>然而在其他人看来，苹果此举已经翻盖到了 iOS 平台上的浏览器竞争。比如曾因 App Store 高额分成政策与苹果闹得不可开交的 Epic Games 首席执行官 Tim Sweeney 就表示，WebKit 限制是反竞争且不包容的。</p><p><img src="https://static.cnbetacdn.com/article/2022/0226/059bd0da4b5c74d.png" referrerpolicy="no-referrer"></p><p>相关争议显然引起了不少监管机构的注意，英国 CMA 也就这一限制提出了严厉的批评：</p><blockquote><p>我们发现，通过强制要求 iOS 设备上的所有浏览器使用该公司的 WebKit 浏览器引擎，苹果已牢牢把持住该平台上所有移动浏览器的质量和功能界限，且限制了竞争对手将其与 Safari 区分开来的潜力。</p><p>举个例子，第三方浏览器无法在 WebKit 框架的束缚下提供更快的页面加载速度、无法呈现不受 WebKit 支持的视频格式，且苹果不会向竞品提供与 Safari 相等的 API 访问权限。</p><p>总体而言，这意味着 Safari 无法在 iOS 平台上面临来自其它浏览器的有效竞争。</p></blockquote>
