<strong>周三的时候，AMD GPUOpen 官网宣布了面向游戏开发者的一款新工具，旨在利用光线追踪技术来帮助组织场景中模型的几何形状。</strong>可知作为 Radeon 开发者工具套件的一部分，目前 1.0 版本的 Radeon Raytracing Analyzer（以下简称 RRA）光追分析器已提供 Windows 和 Linux 平台的可下载安装包。<br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0729/d429f1dbec3288d.jpg" alt="2.jpg" referrerpolicy="no-referrer"></p><p style="text-align: center;">（来自：<a href="https://www.techpowerup.com/297265/amd-introduces-radeon-raytracing-analyzer-1-0" target="_self">GPUOpen.org</a>）</p><p>近年来，行业开始有更多开发者接纳从光栅化、向基于光追的几何图形渲染转型。</p><p>与此同时，他们也迫切需要一款工具来指出性能问题、辅以开发过程中的各种变通方法。</p><p><img src="https://static.cnbetacdn.com/article/2022/0729/d7a5acccfe12c71.webp" alt="1.webp" referrerpolicy="no-referrer"></p><p>好消息是，在 RRA 的帮助下，AMD 能够帮助广大 Radeon 开发者搞定如下问题：</p><blockquote><p>比如使用了多少加速结构和显存、实现 BVH 的复杂度，以及 BLAS 轴中的几何形状是否足够对齐等。</p></blockquote><p>AMD 解释称，RRA 是 Radeon Software 驱动工程师长期努力下结出的一份果实，旨在为面向开发者的驱动技术引入光追支持。</p><p><img src="https://static.cnbetacdn.com/article/2022/0729/c2e2e363746c7ad.jpg" alt="0.jpg" referrerpolicy="no-referrer"></p><p>换言之，在将应用程序切换到开发人员模式下运行时 —— 通过 RRA 附带的 Radeon Developer Panel 面案 —— 驱动程序便可通过点击按钮，记录场景中的所有加速结构。</p><p>然后 Radeon Raytracing Analyzer 光追分析工具可以加载并询问驱动生成的数据，并以易于理解的方式去呈现。</p><p><strong>下载地址：</strong></p><blockquote><p>● <a href="https://gpuopen.com/rdts-windows/" target="_self">Windows</a> | <a href="https://gpuopen.com/rdts-linux/" target="_self">Linux</a></p><p>● GitHub <a href="https://github.com/GPUOpen-Tools/radeon_raytracing_analyzer" target="_self">项目主页</a> | 阅读 <a href="https://radeon-raytracing-analyzer.readthedocs.io/en/latest/" target="_self">RAA 文档</a></p></blockquote>
