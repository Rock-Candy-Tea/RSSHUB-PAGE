即将于今年秋季发布的 Fedora 37 虽然会保留基于 GTK 的 Anaconda 安装器，提供默认的安装体验，<strong>但也会提供下一代基于网页的 Anaconda 安装器，带来全新的界面和安装体验。</strong>在今年早些时候，Red Hat 宣布正在为 RHEL/Fedora 的 Anaconda 安装器开发全新基于网页的 UI。<br>
 <p style="text-align:center"><img src="https://static.cnbetacdn.com/article/2022/0718/29e89e5aea0ab74.webp" alt="v4uho0wq.webp" referrerpolicy="no-referrer"></p><p style="text-align: left;">全新的 UI 更加现代化，未来将会彻底替代现有基于 GTK 的 Anaconda 安装器。这个网页 UI 将会在 Red Hat 的 Cockpit 项目基础上创建，使用 PatternFly 开源系统进行设计。</p><p style="text-align: left;">Fedora 37 计划在公开预览镜像中提供代远程和本地访问的全新 Anaconda Web UI。关于全新的 Web UI，Fedora 团队表示</p><p style="text-align: left;">● 界面布局基于最新的 UXC 设计指导守则，目前已经在大量设备上进行了广泛测试</p><p style="text-align: left;">● 由于选择了更现代化、更主流的 UI 技术以及更强大的 Cockpit 测试工具，在开发体验上有明显的改进。WebUI3 的无状态属性允许 live-coding 风格的 UI 开发。全新的 Web UI 不仅更有利于 Anaconda 团队，而且也更适用于扩展开发者和任何感兴趣的贡献者。</p><p style="text-align: left;">● 相比较缓慢、效率低的 VNC 远程 GUI 安装，Remote Web UI 访问可以提供更优秀的体验。由于不再需要本地渲染远程驱动 GUI 安装，这样在设备上安装最小化镜像成为可能。</p><p style="text-align: left;">Fedora 的这项调整还需要通过 Fedora Engineering and Steering Committee (FESCo) 的批准才可以生效，不过鉴于这项调整的需求很高，而且 Red Hat 工程团队已经为该功能开发预览镜像了，因此极大可能会批准通过，出现在 Fedora 37 中。</p>
