由于相对封闭的生态策略，苹果多年来一直面临多项反竞争行为指控。<strong>最新消息是，一群开发者发起了名为“开放网络倡导”的项目</strong> —— 以敦促苹果在自家移动平台上向第三方浏览器引擎敞开怀抱，并且允许开发者访问与 iOS 版 Safari 浏览器相同的功能。<br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0302/abe523529cb4bc6.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2022/0302/abe523529cb4bc6.jpg" alt="Safari.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">资料图（来自：Apple 官网）</p><p>据悉，与 macOS 桌面操作系统不同的是，<a data-link="1" href="https://apple.pvxt.net/c/1251234/435400/7639?u=https%3A%2F%2Fwww.apple.com%2Fcn%2Fmusic%2F" target="_blank">苹果</a>自家 iOS 移动操作系统选用了 WebKit 引擎来呈现全局网络内容，意味着其它 Web 浏览器都只是套上了一层 Safari 的马甲在运行。</p><blockquote><p>Lawson 解释称，Open Web Advocacy 希望能够说服苹果，以允许在 iOS 上运行其它浏览器引擎，那样它才会成为一个对现代网络内容开发更友好的平台。</p><p>遗憾的是，截至目前，无论是 iOS 版本的 Google Chrome、Mozilla Firefox、还是 Microsoft Edge，其底层都还基于 Safari / WebKit 来实现。</p></blockquote><p>由于缺乏竞争，该平台的颓势正变得愈发明显。更糟糕的是，第三方开发者甚至无法访问到 Safari 的某些独享功能，比如全屏和 Apple Pay 支付集成。</p><p>受此影响，iOS 上的 Web 应用程序也无法全屏运行，更别提允许第三方浏览器将 Web Apps 添加到 iOS 的主屏上，且 WebKit 中缺乏 Web NFC 和其它 API 。</p><p>好消息是，上周有报道称，英国竞争与市场管理局（CMA）已经知悉了这方面的担忧，并希望说服苹果变更其政策。</p><p>此外 9to5Mac 指出，Safari 已经落后于竞争对手有段时间，用户正迫切希望切换至其它 Web 浏览器。但是对于苹果来说，或许更担心这么做会损害自家 App Store 的服务营收。</p><div class="article-relation"><p><strong>相关文章:</strong></p><p><a href="https://www.cnbeta.com/articles/tech/1240925.htm" target="_blank">英国市场竞争监管机构就苹果限制第三方浏览器一事提出严厉批评</a></p></div>
