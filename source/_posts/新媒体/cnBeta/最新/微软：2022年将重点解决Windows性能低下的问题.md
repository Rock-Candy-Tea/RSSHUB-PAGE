<strong>WinUI 是 Windows 系统中的原生 UI 平台，但在某些配置下会出现缓慢和迟钝的的情况。</strong>Windows 11 也将 WinUI 用于其核心 UI 组件，虽然它们看起来不错，但性能迟缓是现代 UI 元素造成的一个不希望看到的副作用。<br>
 <p><a href="https://static.cnbetacdn.com/article/2021/1115/3decf5ea1879b01.webp" target="_blank"><img src="https://static.cnbetacdn.com/article/2021/1115/3decf5ea1879b01.webp" referrerpolicy="no-referrer"></a></p><p>在 Feedback Hub 和社交媒体平台上，用户抱怨在 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 中使用 WinUI 或 XAML UI 元素时的性能问题。例如，Windows 文件管理器已经更新了一个新的命令栏和一个基于 WinUI 的更现代的界面。</p><p>不幸的是，文件资源管理器的性能低下和错误的体验是WinUI的一个不受欢迎的副作用。在反馈中心的帖子中，一些用户指出，他们的文件资源管理器会出现随机崩溃/卡死，他们无法理解系统的基本组件如何会出现问题。</p><p>一位用户解释说：“在使用 OneDrive 同步的文件夹时，这一点最明显，而且还是非常滞后的。使用 F11 选项来全屏 Windows，在我的安装上使体验要快得多，但这不是一个有效的解决方法，以确保Windows如此关键的核心组件的快速性能”。</p><p>根据几份报告，整个体验似乎很草率。<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>官员说，该公司正在努力改善开发渠道的性能，在较新的构建中推出修复措施。例如，最近修复了一个命令栏做不必要的计算的错误。</p><p>此外，看起来微软的UI团队也计划在2022年专注于性能。在一个Reddit 帖子中，当用户抱怨在Windows中使用WinUI或XAML元素时体验迟缓时，该软件制造商承诺进行与性能有关的修复。</p>
