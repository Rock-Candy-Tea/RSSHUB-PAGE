Google Cloud 刚刚发起了名为“Data Cloud Alliance”的新联盟，并且吸引了埃森哲（Accenture）、Confluent、Databricks、Dataiku、德勤（Deloitte）、Elastic、Fivetran、MongoDB、Neo4j、Redis 和 Starburst 等知名企业加盟。<strong>可知数据云联盟的使命是“让数据在不同的业务系统、平台和环境中更加可移植和可访问”，确保数据访问永远不会成为向数字化转型的一个障碍类型。</strong><br>
 <p><img src="https://static.cnbetacdn.com/article/2022/0406/9dfe13ac59a62c4.png" alt="1.png" referrerpolicy="no-referrer"></p><p>其实早在 2020 年，Snowflake 就已经提交了“Data Cloud Alliance”的商标注册申请。但由于该公司不属于这个新联盟，该注册已被放弃。</p><p>据悉，数据云联盟致力于共同提供 API 和集成支持，通过允许平台之间的数据可移植性和可访问性，让成员客户的日常工作流程能够更加轻松通畅 —— 无论这些平台是在本地使用，还是构建于私有云、公共云、或混合云。</p><p>此外 Google 在今日的<a href="https://cloud.google.com/solutions/data-cloud-alliance" target="_self">公告</a>中指出，联盟成员将共同创建“全新通用的行业数据模型、流程与平台集成”，以提升数据的可移植性、并降低与数据治理和全球合规性相关的复杂问题。</p><p>Databricks 表示，其很高兴能够与 Google 携手促进基于 Databrick 的 Delta Lake 等开放标准的数据共享。</p><p><img src="https://static.cnbetacdn.com/article/2022/0406/c967f174411a689.png" alt="2.png" referrerpolicy="no-referrer"></p><p>不过需要指出的是，该联盟的大多数成员都没有直接的竞争关系，而是提供相辅相成的服务。</p><p>此前许多公司都有与 Google 合作的经验，比如 Confluent、Elastic、MongoDB、Neo4j 和 Redis Labs 就尝试过将它们的服务与谷歌云平台集成。</p><p>最后，Google Cloud 数据库、数据分析和商业智能副总裁兼总经理 Gerrit Kazmaier 表示：</p><blockquote><p>数据是所有数字化转型的共同基础，通过推动开放数据标准、访问，以及当今最流行的数据平台和应用程序之间的集成，相信我们可以显著加速业务转型、以及弥合数据的价值鸿沟。</p></blockquote>
