<strong>谷歌已经预告了将在 Pixel 6 系列智能机中采用自研 Tensor 芯片组，如果 XDA 的爆料准确，那传闻中这枚 SoC 的 CPU 配置可能会相当奇怪。</strong>此前，谷歌一直在卖力宣传 Tensor SoC 的 AI 性能，而没有披露 CPU / GPU 的任何基础规格。Rick Osterloh 也只是向 TheVerge 表示：“虽然大家喜欢极具竞争力的标准事物，但人工智能领域却与之截然不同”。<br>
 <p><img src="https://static.cnbetacdn.com/article/2021/0916/d44e16ec600bb0c.png" referrerpolicy="no-referrer"></p><p>XDA 在一份早期报告中指出，Pixel 6 或采用公版 ARM Mali-G78 GPU 设计（类似<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://samsung.jd.com/" target="_blank">三星</a> Exynos 2100）。此外路透社报道称，Google 将采纳三星的 5G 调制解调器方案。</p><p>至于传说中的“谷歌自研芯片”的 CPU 部分，则长期处于不为人知的状态。好消息是，在今日一份报告中，XDA 已从 Geekbench 基准测试数据库和消息人士那里拿到了靠谱的信息。</p><p>据说 Tensor SoC 上的 CPU 配置为两个主频 2.802 GHz 的 Cortex-X 高性能核心 + 两个主频 2.253GHz 的 Cortex-A76 普通核心 + 四个 Cortex-A55 节能核心的组合。</p><p><img src="https://static.cnbetacdn.com/article/2021/0916/ca7f487bbc7ff85.jpg" referrerpolicy="no-referrer"></p><p>对于某些消费者来说，或许会对新旧强弱核心的混搭设计感到有些奇怪。但 ARM 推行已久的 big.LITTLE 架构设计方案，已经在实际应用中表明了它的合理性。</p><p>常见的 ARM 芯片设计，可能包括四个性能核心（比如 Cortex-A78）和四个节能核心（比如 Cortex-A55）。不过去年，ARM 又为客户带来了更强大的 Cortex-X1 内核选项。</p><p>正因如此，我们才在今年发布的旗舰智能机上看到了许多“三 CPU 集群”的设计，比如高通骁龙 888 SoC 就采用了一个 Cortex-X1 + 三个 Cortex-A78 + 四个 Cortex-A55 内核的（三星 Exynos 2100 SoC 也类似）。</p><p><a href="https://static.cnbetacdn.com/article/2021/0916/0477ea404ec0cb9.jpg" target="_blank"><img src="https://static.cnbetacdn.com/thumb/article/2021/0916/0477ea404ec0cb9.jpg" referrerpolicy="no-referrer"></a></p><p style="text-align: center;">（图自：<a href="https://www.xda-developers.com/google-pixel-6-same-gpu-exynos-galaxy-s21/" target="_self">XDA-Developers</a>）</p><p>正因如此，我们才在看到谷歌 Tensor SoC 将采用两个 Cortex-X1 高性能核心的时候感到如此诧异。若真如此，谷歌自研芯片或迎来远超高通 / 三星竞品的理论性能表现。</p><p>与此同时，我们也不得不留意两个较旧的 Cortex-A76 核心，因为它们很可能在某些基准测试项目中严重拖后腿。至于真相究竟如何，仍有待时间去检验。</p>
