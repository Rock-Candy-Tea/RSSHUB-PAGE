昨日，本站曾报道 Windows 11 系统在新增微软账号之后，可能会出现阻止用户登录的情况。微软承认该问题是由可选更新 KB5016691（于 8 月 25 日发布）导致的，不过这个问题已迅速得到解决，并提供了一些细节。<br>
 <p><a href="https://static.cnbetacdn.com/article/2022/0905/f0aaf9776d7f7d9.webp" target="_blank"><img src="https://static.cnbetacdn.com/article/2022/0905/f0aaf9776d7f7d9.webp" referrerpolicy="no-referrer"></a></p><p>微软表示：“在 Windows 中安装 KB5016691 并添加新的 Microsoft 帐户用户后，您可能在第一次重新启动或退出后的短时间内无法登录。该问题仅影响新添加的 Microsoft 帐户用户且仅适用于首次登录”。</p><p>根据<a data-link="1" href="https://c.duomai.com/track.php?site_id=242986&euid=&t=https://www.microsoftstore.com.cn/" target="_blank">微软</a>的说法，用户在第一次重启或注销后会被短暂锁定。该问题仅在使用
Microsoft 帐户的设备上出现一次，并且影响时间比较短。Active Directory 域用户和 Azure Active
Directory 帐户不受这个问题影响。此外，它仅发生在运行 <a data-link="1" href="https://microsoft.pvxt.net/x9Vg1" target="_blank">Windows</a> 11 21H2 客户端版本的系统上。</p><p>为了最终结束它，微软宣布将发布已知问题回滚，这应该会改回早先推出的有缺陷的 Windows 修复程序。不会超过 24 小时，这意味着受影响的消费者和非托管业务设备的问题应该在周末之前得到解决。微软还建议重新启动你的机器，以帮助解决现在正在推动的解决方案。</p>
