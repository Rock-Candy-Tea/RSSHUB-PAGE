<p data-vmark="f0fe"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 4 日消息，微软今日向 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Win11</a> Dev 预览版用户推送了 Build 22494 版本更新，带来任务栏一键静音麦克风，并包含多项修复功能。</p><h2 data-vmark="aace">新增：</h2><p data-vmark="36b9"><span class="accentTextColor">在任务栏中可一键静音麦克风</span>。从 Microsoft Teams 开始，用户会发现在通话中，会自动将麦克风图标添加到任务栏。用户可以查看通话音频状态、正在访问麦克风的应用程序，并随时快速将麦克风静音和取消静音。</p><p data-vmark="588a"><img src="https://img.ithome.com/newsuploadfiles/2021/11/af63536d-db7c-4692-8afe-e9880ef1009b.png" w="840" h="280" title="微软 Win11 Dev 预览版 Build 22494 发布，任务栏可一键静音麦克风" width="840" height="273" referrerpolicy="no-referrer"></p><p data-vmark="6f0e">当用户加入会议时，用户会立即在任务栏中看到以下图标。该图标将整个通话过程中出现，因此无论在屏幕上打开了多少个窗口，它都可以随时访问。</p><p data-vmark="cad3">IT之家了解到，该功能首先推广到 Microsoft Teams for Work 或 Schools 的 Windows Insiders 用户群体，因此一开始并不是所有人都能够使用该功能。</p><h2 data-vmark="346d">改进：</h2><ul class=" list-paddingleft-2"><li><p data-vmark="01e1">正在尝试使用某些 Windows 预览体验成员在 Alt + Tab 和任务视图中显示快照组（snap groups），就像用户将鼠标悬停在任务栏上打开的应用程序一样。并非所有内部人员都可以使用此功能。微软计划监控反馈并查看反馈情况，然后再将其推送给所有用户。</p></li><li><p data-vmark="a023">在“设置”>“应用程序”>“默认应用程序”下搜索文件类型或链接类型，将显示包含用户当前查询的选项下拉列表，而无需按回车键。</p></li><li><p data-vmark="ad11">用户可以通过以下 URI 直接在“设置”>“应用”>“安装的应用”下启动已安装的应用设置页面：ms-settings:installed-apps。</p></li><li><p data-vmark="b80a">调整了“设置”>“应用”>“安装的应用”下排序方式的名称，使其更清晰，并新增了从小到大排序的选项。</p></li></ul><h2 data-vmark="e9ec">修复：</h2><p data-vmark="f16f"><strong>任务栏：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="2ffe">将鼠标滑过任务栏角落的音量、电池、网络或其他图标后，工具提示不再出现在任务栏上的随机位置。</p></li><li><p data-vmark="36ae">解决了导致任务栏角落中某些图标意外重复的潜在问题。</p></li></ul><p data-vmark="18ee"><strong>文件资源管理器：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="e6fe">解决了上下文菜单在用户滚动时崩溃的问题。</p></li><li><p data-vmark="de21">修复了在屏幕的某些区域，上下文菜单子菜单会绘制在上下文菜单的顶部而不是旁边的问题（比如“新建”）。</p></li><li><p data-vmark="6ebb">在具有混合 DPI 的多个显示器的系统上，上下文菜单图标应该能够清晰显示。</p></li><li><p data-vmark="28fb">解决了可能导致在上下文菜单中选择打开方式时，在某些情况下意外打开文件而不是实际打开“打开方式”对话框的问题。</p></li><li><p data-vmark="c4ae">上个版本重命名桌面文件问题已在此版本修复。</p></li><li><p data-vmark="b53e">对命令栏底层逻辑进行了另一项调整，以帮助提高文件资源管理器中命令操作的性能。</p></li></ul><p data-vmark="0209"><strong>搜索：</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="e971">修复了最近索引器数据库变得过于碎片化，导致索引器意外消耗大量内存和 CPU 时间过长的问题。这对于拥有大型 Outlook 邮箱的人来说尤其明显。</p></li></ul><p data-vmark="91b3"><strong>输入：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="c858">缓解了在尝试按住 Shift 或 Ctrl 键拖动某些内容时导致某些应用程序挂起的问题。</p></li><li><p data-vmark="1744">修复了在用户尝试从登录屏幕重置 PIN 时，在点击文本字段时，平板电脑上不显示触摸键盘的问题。</p></li><li><p data-vmark="ee75">提高了笔菜单的可靠性。</p></li></ul><p data-vmark="5ea0"><strong>窗口：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="f875">修复了一些与使用窗口功能（截图、Alt + Tab 和桌面）相关的 explorer.exe 崩溃问题。</p></li><li><p data-vmark="b220">如果用户在具有多台显示器的系统上打开任务视图，则两台显示器上的背景现在都应该显示亚克力效果。</p></li><li><p data-vmark="ec18">解决了任务视图和 Alt + Tab 中窗口缩略图的几个 UI 问题，特别是如果应用程序窗口太小，关闭按钮可能会被切断。</p></li></ul><p data-vmark="095a"><strong>设置：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="0af7">解决了面部识别 (Windows Hello) 在某些情况下可能会在登录设置中意外变灰的问题，直到关闭和打开设置。</p></li><li><p data-vmark="5725">修复了存储感知功能未清理 C:\Windows\SystemTemp 的问题。</p></li><li><p data-vmark="197c">如果未授予位置访问权限，标准用户（又名非管理员）现在应该能够在“设置”中更改时区，而不是下拉列表变为空白。</p></li></ul><p data-vmark="fe1e"><strong>其他：</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="f50b">修复了导致 Windows 更新、恢复和开发人员的链接显示在主 Windows 更新设置页面下的问题。</p></li><li><p data-vmark="b03c">修复了在 HDR 模式下，图像在 Adobe Photoshop、Adobe Lightroom 和 Adobe Lightroom Classic 中具有黄色色调的错误。</p></li><li><p data-vmark="187d">缓解了与 DHCP 相关的问题，该问题在某些 Insiders 的最新版本中导致屏幕关闭时意外耗电。</p></li><li><p data-vmark="48ed">解决了一些服务主机问题：WinHTTP Web 代理自动发现服务意外使用大量 CPU 的问题。</p></li><li><p data-vmark="4499">修复了可能导致某些设备在退出睡眠模式时出现黑屏的问题（不显示锁定屏幕）。</p></li><li><p data-vmark="7828">解决了导致一些使用 ARM64 PC 的用户在最近几个 Dev 版本中遇到 Microsoft Teams 崩溃增加的潜在问题。</p></li><li><p data-vmark="fbe2">通过点击文件资源管理器上下文菜单中的显示更多选项或任务管理器中的菜单选项，增加了所选项目的填充。</p></li><li><p data-vmark="587f">WSL：修复了通过‘\\wsl.localhost’或‘\\wsl$’访问 Linux 发行版时出现的错误 0x8007010b。</p></li></ul><h2 data-vmark="0c67">已知问题：</h2><p data-vmark="1e7e"><strong>常规：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="8147">使用最新的 ISO 从版本 22000.xxx 或更早版本更新到更新的开发人员渠道的用户可能会收到错误警告；</p></li><li><p data-vmark="2099">屏幕和睡眠超时时间可能会缩短，正在调查该问题对能源消耗的潜在影响。</p></li></ul><p data-vmark="7b91"><strong>开始菜单：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="ddbc">在某些情况下，使用“从开始搜索”或任务栏时可能无法输入文本。如果遇到此问题，请按键盘上的 WIN+R 以启动“运行”对话框，然后将其关闭。</p></li></ul><p data-vmark="dde8"><strong>任务栏：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="b48f">切换输入法时，任务栏有时会闪烁。</p></li><li><p data-vmark="17fa">正在调查此版本中的一个问题，即任务栏中的时钟可能会卡住且无法更新，尤其是在通过远程桌面访问 PC 时。</p></li></ul><p data-vmark="ba5b"><strong>输入：</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="a33b">剪贴板历史记录是空白的。</p></li></ul><p data-vmark="2069"><strong>搜索：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="b5a4">点击任务栏上的搜索图标后，搜索面板可能无法打开。如果发生这种情况，请重新启动“Windows 资源管理器”进程，然后再次打开搜索面板。</p></li></ul><p data-vmark="263a"><strong>快速设置：</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="da1f">正在调查来自内部人士的报告，即音量和亮度滑块在快速设置中没有正确显示。</p></li></ul>
