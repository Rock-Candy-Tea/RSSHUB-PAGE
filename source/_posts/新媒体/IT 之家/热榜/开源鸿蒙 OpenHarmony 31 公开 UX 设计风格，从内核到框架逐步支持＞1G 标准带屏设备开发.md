            <p data-vmark="87ce"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT 之家</a> 4 月 28 日消息，近期，OpenAtom OpenHarmony 首次对外解读了 OpenHarmony 3.1 Release 技术特性。数据显示，截至目前，OpenHarmony 吸引了 40 多家主仓代码贡献单位，汇聚 160 万 + 社区用户，建立 44 个特别兴趣小组，全球下载次数高达 6300 万，5.5 万 + 次代码提交。</p><p data-vmark="8809" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/1/9dd1686e-7834-4cb8-b781-4f5284f6f190.png" alt="开源鸿蒙" title="开源鸿蒙 OpenHarmony 3.1 公开 UX 设计风格，从内核到框架逐步支持＞1G 标准带屏设备开发" referrerpolicy="no-referrer"></p><p data-vmark="244c">据介绍，通过 4 个关键版本迭代，OpenHarmony 系统能力持续升级，<span class="accentTextColor">OpenHarmony 3.1 Release 版本已具备支撑复杂带屏标准系统设备和应用开发的基础能力</span>。</p><p data-vmark="5166">技术日主论坛首次对外解析了 OpenHarmony 3.1 Release 技术特性。相较于旧版本，3.1 Release 不仅强化了内核，构建了业务自适应优化内核调度机制，为流畅性能奠定基础；还增强了驱动开发模型，开放丰富 HDI 接口，为系统提供更多硬件访问能力。同时，图形、多媒体、分布式硬件、分布式数据管理、分布式安全等能力全面提升。“OpenHarmony 3.1 Release 从内核到框架，逐步支持复杂标准带屏设备开发。”</p><p data-vmark="c945">下面是 OpenHarmony 3.1 公开的 UX 设计风格，包括启动器、系统交互、应用风格、更圆润造型、多个窗口界面等等。</p><p data-vmark="5aac" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/4/ce4d0213-894d-4650-b64e-d48af1ad8e5f.jpg" w="750" h="2226" title="开源鸿蒙 OpenHarmony 3.1 公开 UX 设计风格，从内核到框架逐步支持＞1G 标准带屏设备开发" width="750" height="2226" referrerpolicy="no-referrer"></p><p style="text-align: left;" data-vmark="7141">下面是视频：</p><p style="text-align: center;" data-vmark="45b7"><iframe class="ithome_video bilibili" src="https://player.bilibili.com/player.html?bvid=bv17r4y1n722&page=1" frameborder="no" width="100%" allowfullscreen="true"> </iframe></p><p data-vmark="b48b">除系统能力提升外，社区还推出了配套开发工具。API Version 8 版本开放 11000+JS API 接口，DevEco Studio 3.0 Beta3 构建了 OpenHarmony 应用开发环境，DevEco Device Tool 3.0 Release 则是一款一站式的 OpenHarmony 硬件开发工具。</p><p data-vmark="338a">目前，已有 11 款主流芯片进入 OpenHarmony 社区主干，到 2022 年末适配芯片预计将增至 35 款；同时截至 4 月，44 款产品已获得 OpenHarmony 生态产品兼容性证书。</p>
