<p data-vmark="a4be"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 10 月 15 日消息，今天，微软面向 Dev 开发频道发布了 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> Insider Preview Build 22478。该版本包括新的 Fluent 流畅表情符号，<a href="https://www.ithome.com/0/580/610.htm" target="_blank">由微软设计团队设计的特殊桌面壁纸</a>，以庆祝 Windows Insider 计划 7 周年，而且微软已经开始测试更新堆栈包，该包在主要的操作系统更新之外提供更新改进，如新的构建版本。</p><p data-vmark="9c0a"><img src="https://img.ithome.com/newsuploadfiles/2021/10/b6bb9361-d7c6-47a7-8af1-41ecbc3bf31e.jpg@s_2,w_820,h_460" w="1440" h="808" title="微软 Win11 Dev 预览版 22478 发布：全新 Fluent 流畅表情符号，大量变化和修复（附更新内容大全）" srcset="https://img.ithome.com/newsuploadfiles/2021/10/b6bb9361-d7c6-47a7-8af1-41ecbc3bf31e.jpg 2x" width="1440" height="460" referrerpolicy="no-referrer"></p><p data-vmark="7766"><strong>下面是 Windows 11 Build 22478 中新内容</strong></p><h2 data-vmark="c987">新的 Fluent 表情符号</h2><p data-vmark="5736">早在 7 月，微软设计团队就宣布了刷新的和新的 Fluent 流畅设计的 Emoji 表情符号，将在所有的 Microsoft 365 中推出--包括 Windows。微软很高兴能让 Windows Insiders“先睹为快”，看看新的表情符号设计在 Windows 11 中的样子，从今天的 Dev 频道构建开始。</p><p data-vmark="ca09"><img src="https://img.ithome.com/newsuploadfiles/2021/10/623c2340-3440-41cd-b6ac-4c2f63cfd9d4.png" w="1160" h="668" title="微软 Win11 Dev 预览版 22478 发布：全新 Fluent 流畅表情符号，大量变化和修复（附更新内容大全）" width="1160" height="472" referrerpolicy="no-referrer"></p><p data-vmark="be1a">表情符号已经成为我们交流方式的一部分。通过提供幽默、情感和乐趣来扩展我们的表达。你会注意到微软重新设计了所有的表情符号，并为粉丝们引入了一些怀旧的东西，将标准的回形针表情符号变成了心爱的 Clippy! 作为这次推出的一部分，微软增加了对 Unicode 表情符号的支持，直至 Emoji 13.1，其中包括新的表情符号，如云中的脸、着火的心、螺旋眼的空间以及更多的表情符号</p><p data-vmark="6a56">要查看新的表情符号，只需打开表情符号面板（WIN + .）。</p><p data-vmark="bf3f">IT之家获悉，在对 Windows Insiders 进行预览后，微软计划通过未来的服务更新将这些新的表情符号带给 Windows 11 上的所有用户。</p><h2 data-vmark="4fe6">变化和改进</h2><ul class=" list-paddingleft-2"><li><p data-vmark="e7d6">更新后的韩国 IME 输入法版本，从 Build 22454 开始推出，现在可以在 Dev 频道中向所有用户提供。</p></li><li><p data-vmark="90d9">当你的笔记本电脑关闭时，现在可以在连接的外部显示器上使用面部识别（Windows Hello）登录到你的电脑，前提是有支持 Windows Hello 的摄像头。</p></li><li><p data-vmark="ac6b">微软对底层索引器平台做了改变，以帮助提高整体可靠性和数据库大小，这应该有助于减少系统的磁盘空间使用 —— 特别是在用户有非常大的 Outlook 邮箱的情况下。</p></li><li><p data-vmark="f8d7">在任务栏的音量图标上滚动你的鼠标滚轮，现在将改变当前的音量水平。</p></li><li><p data-vmark="6f26">在“设置”中添加新的语言时，对话框做了一些小的调整，使其看起来更现代一些，包括圆角处理。</p></li></ul><h2 data-vmark="44a1">修复内容</h2><p data-vmark="b508"><strong>[开始] 菜单</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="db90">如果你在“开始”菜单中打开用户配置文件弹出后按下 ESC，键盘焦点将不再丢失。</p></li><li><p data-vmark="3c3e">更新了将 UWP 应用中的内容固定到“开始”菜单的提示文本，现在显示“你想将这个固定到开始吗”。</p></li></ul><p data-vmark="f2f4"><strong>[任务栏]</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="9f51">解决了在最近的版本中，通知中心会卡在一个状态，停止启动的问题。</p></li><li><p data-vmark="89b5">当使用阿拉伯语和希伯来语时，将鼠标悬停在任务栏的任务视图图标上时，用于添加新桌面的加号（+）图标现在将正确显示。</p></li><li><p data-vmark="eacd">在从任务栏调出一个应用的跳转列表后，右键点击应用名称现在应该显示正确的“固定在开始”图标（一个钉子图标）。</p></li><li><p data-vmark="2047">同时左键和右键点击任务栏不会再让 explorer.exe 出现崩溃。</p></li></ul><p data-vmark="844d"><strong>[文件资源管理器]</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="9dbd">修复了在导航到文件夹时，命令栏会进行不必要的计算，导致性能意外下降。这也被认为是使用拖动选择文件，以及使用方向键在某些文件夹中导航时的出现性能问题的根本原因。</p></li><li><p data-vmark="6eb8">当多个文件资源管理器窗口打开时，“显示隐藏的项目”选项不会再出现不同步的情况。</p></li></ul><p data-vmark="4381"><strong>[小工具]</strong></p><p data-vmark="75c2">以下两个问题已经在微软 Edge 95 版本中得到解决，目前为微软 Edge 内测人士提供服务，因此微软将它们从已知问题列表中删除。</p><ul class=" list-paddingleft-2"><li><p data-vmark="b8b2">小工具面板可能显示为空。</p></li><li><p data-vmark="d072">小工具在外部显示器上可能会以错误的尺寸显示。</p></li></ul><p data-vmark="177c"><strong>[搜索]</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="52cc">当悬停在任务栏的搜索图标上时，如果管理窗口处于焦点位置，最近的搜索弹出不应再在调用时立即消除。</p></li></ul><p data-vmark="319f"><strong>[输入]</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="b6bc">当使用任务栏设置为自动隐藏的平板电脑模式时，将焦点设置为文件资源管理器中的搜索框，现在不会使触摸键盘调用并立即消除。</p></li><li><p data-vmark="c20a">修复了当使用手写板书写中文时，手写板不能插入文本的问题。</p></li><li><p data-vmark="0e85">缓解了 ctfmon 中可能出现的内存泄漏问题，该问题会随着时间的推移导致意外的资源使用。</p></li><li><p data-vmark="7adb">解决了少数用户的导致 TextInputHost.exe 在对任何文本字段设置焦点时崩溃的问题。</p></li></ul><p data-vmark="f166"><strong>[设置]</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="7f14">修复了在更新单选按钮时导致“设置”崩溃的问题，例如在“设置”中的“声音”页面打开时连接额外的音频端点。</p></li><li><p data-vmark="2c56">改进了加载个性化设置中显示的主题的性能。</p></li><li><p data-vmark="dba1">如果非管理员用户在“语言设置”中改变了他们的显示语言，微软将不再在那里显示可点击的退出按钮，因为它将弹出一个 UAC 对话框，会导致混淆。</p></li><li><p data-vmark="4047">在“设置”中的“系统”页面顶部的 Windows Update 条目在德语等语言中不会再与其他选项（显示时）错位。</p></li><li><p data-vmark="15a1">讲述人用户设置中的盲文选项现在应该与盲文输入/输出模式的实际状态保持同步。</p></li><li><p data-vmark="5b5d">进行了一项修改，以帮助解决网络和共享中心的“媒体流选项”在升级后会意外地恢复为默认设置的问题。</p></li><li><p data-vmark="beee">改进快速设置启动可靠性。这一变化有助于解决导致窗口不能完全呈现的问题（只是一个薄的矩形）。</p></li><li><p data-vmark="937b">解决了如果点击快速设置中的媒体控制区，在某些情况下它不会将显示的应用程序带到前台的问题。</p></li><li><p data-vmark="ab57">修复了快速设置中的项目颜色可能被卡住，不能反映该状态的正确颜色的问题，特别是在使用对比主题时。</p></li><li><p data-vmark="eaae">将管理音频端点的快速设置页面从 "音量" 更新为 "声音输出"。</p></li><li><p data-vmark="b489">当列出多个设备时，在快速设置中打开管理音频端点的页面不会再导致音频播放短暂中断。</p></li></ul><p data-vmark="c043"><strong>[窗口]</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="0ecf">缓解与快速使用 ALT + Tab 有关的问题，该问题可能会使 explorer.exe 崩溃。</p></li><li><p data-vmark="5951">按住 ALT 并在按下 ALT + Tab 后按下 F4 将不再使 explorer.exe 崩溃。</p></li><li><p data-vmark="76d6">当使用阿拉伯语或希伯来语显示语言时，将一个应用程序窗口拖到不同的桌面上，现在应该显示缩略图向正确的方向移动。</p></li></ul><p data-vmark="2f68"><strong>[其他]</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="3909">修复了在某些情况下会导致 LP.cab 文件被意外删除，这可能会导致你的显示语言改变的问题，直到在设置中重新添加所需的显示语言。</p></li><li><p data-vmark="1f96">解决了导致一些用户看到错误“KERNEL_SECURITY_CHECK_ERROR”的错误检查问题。</p></li><li><p data-vmark="8b62">缓解了导致 SysMain 服务在最近的构建版本中使用意外电量消耗的问题。</p></li><li><p data-vmark="fe02">如果 BCD 设置中缺少显示顺序，不会再导致升级失败。</p></li><li><p data-vmark="67ef">修复了导致某些应用程序中的字幕以错误的偏移量绘制的问题。</p></li><li><p data-vmark="42c4">缓解了导致与 audiosrv.dll 有关的崩溃问题。</p></li><li><p data-vmark="3e46">如果使用阿拉伯语或希伯来语显示语言通过 OOBE 开箱设置体验，音量和辅助功能按钮现在会显示在屏幕的正确一侧。</p></li></ul><p data-vmark="a701">注意：这里提到的来自活跃开发分支的 Insider Preview 构建中的一些修复可能会进入 10 月 5 日开始普遍提供的 Windows 11 正式版服务更新。</p><h2 data-vmark="81f0">已知问题</h2><p data-vmark="2a0d"><strong>[一般]</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="dfc5">用户使用最新的 Dev 频道预览版 ISO 镜像，从 Build 22000.xxx 或更早的版本更新到较新的 Dev 频道版本，可能会收到以下警告信息。“你试图安装的构建版本是 Flight 签名的。要继续安装，请启用 Flight 签名。如果你收到这条消息，请按下启用按钮，重新启动电脑，并重新尝试更新。”</p></li><li><p data-vmark="a11c">一些用户可能会遇到他们的屏幕和睡眠超时被减少的情况。微软正在调查缩短屏幕和睡眠超时可能对能源消耗产生的潜在影响。</p></li></ul><p data-vmark="a816"><strong>[开始] 菜单</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="b8a8">在某些情况下，当从“开始”或任务栏使用搜索时，你可能无法输入文本。如果你遇到这个问题，请按键盘上的 WIN + R 来启动运行对话框，然后关闭它。</p></li></ul><p data-vmark="ac70"><strong>[任务栏]</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="9318">在切换输入法时，任务栏有时会出现闪动。</p></li><li><p data-vmark="30bc">微软正在努力解决导致工具提示在悬停在任务栏角落后，出现在一个意外位置的问题。</p></li></ul><p data-vmark="0fe0"><strong>[搜索]</strong></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="bba9">点击任务栏上的搜索图标后，搜索面板可能无法打开。如果发生这种情况，请重新启动“Windows Explorer”进程，并再次打开搜索面板。</p></li><li><p data-vmark="690e">搜索面板可能显示为黑色，在搜索框下面不显示任何内容。</p></li></ul><p data-vmark="aed9"><strong>[快速设置]</strong></p><ul class=" list-paddingleft-2"><li><p data-vmark="259c">微软正在调查来自内测人士的报告，即音量和亮度滑块在快速设置中不能正常显示。</p></li></ul>
