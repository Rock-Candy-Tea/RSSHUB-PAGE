<p data-vmark="97ea"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 8 月 28 日消息，微软 Windows 11 作为全新一代操作系统，对于设备要求包括 TPM 2.0、安全启动、基于虚拟化的安全（VBS）等，导致部分设备无法升级。</p><p data-vmark="0c9f">而根据最新发现的文件，微软早在 Windows 10 中就开始为 TPM 和安全启动等铺路。</p><p data-vmark="4f6f">据推特爆料者 Xeno 在 <span class="accentTextColor">Windows 10 Build 21327</span> 版本中发现的信息，appraiserres.dll 文件中出现了关于 TPM、安全启动的描述。</p><p data-vmark="12e6" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/8/50954ae1-839e-4685-ac1f-4c7a9c3da98f.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzExMi5wbmc=,g_3,x_11,y_11,a_0,t_100" w="729" h="725" title="微软 Win10 中发现 TPM、安全启动要求文件，用于为 Win11 铺路" width="729" height="725" referrerpolicy="no-referrer"></p><p data-vmark="ac80">Windows 10 Build 21327 属于微软 Windows 开发的 Cobalt Semester，与 Windows 11 属于同一 Semester，因此可以看作是为 Windows 11 实装铺路。</p><p data-vmark="0ee1">IT之家了解到，微软 Windows 10 22H2 即将面向所有用户推送，与此同时，微软在一篇用于帮助商业客户和公司部署 Windows 10 的博客文章明确指出，<span class="accentTextColor">22H2 版的 Windows 硬件兼容性计划 (WHCP) 没有变化</span>，甚至与 2004 版本完全相同。所以大家也不用担心会在新版本中被限制。</p><p data-vmark="8b62" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/7/9cc62945-f125-4bcf-b9f2-836114291ecd.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzE4MS5wbmc=,g_9,x_18,y_18,a_0,t_100" w="1176" h="753" title="微软 Win10 中发现 TPM、安全启动要求文件，用于为 Win11 铺路" width="1176" height="525" referrerpolicy="no-referrer"></p><p data-vmark="2331">微软对 Windows 10 的支持将持续到 2025 年，不过目前开发重心已经转移到了 Windows 11 中。</p><p data-vmark="727b">Windows Central  的消息来源还称，<a href="https://www.ithome.com/0/629/675.htm" target="_blank"><span class="accentTextColor">微软现在打算每三年发布一次 Windows 的主要版本</span></a>，目前计划在 2024 年发布下一个版本（暂称 Windows 12），比 2021 年发布 Windows 11 晚三年。</p><p data-vmark="5528">从 Windows 11 22H2（Sun Valley 2）开始，微软正在启动一项新的“<span class="accentTextColor">Moments</span>”工程，旨在让公司能够在全年的关键点推出新的功能和体验，而不是主要的操作系统版本。消息称微软打算从 2023 年开始，<span class="accentTextColor">每隔几个月向市场上的 Windows 版本发布新功能，每年最多四次</span>。</p>
