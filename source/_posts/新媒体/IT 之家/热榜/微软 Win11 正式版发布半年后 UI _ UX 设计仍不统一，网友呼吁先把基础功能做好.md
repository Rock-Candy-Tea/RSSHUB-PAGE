<p data-vmark="40bf"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 4 月 18 日消息，据 Neowin 报道，在 2021 年审查 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 时，该网站给它打了 6.5/10 的分数，同时表示“简化 UI 并不是糟糕的主意，但半生不熟的方式并没有带来真正意义上的用户体验”。遗憾的是，六个月后它仍然没有改变。</p><p data-vmark="2dfb" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/4/7cdd8ae6-119c-4f73-96ab-e0cef3009655.jpg" w="760" h="428" title="微软 Win11 正式版发布半年后 UI / UX 设计仍不统一，网友呼吁先把基础功能做好" width="760" height="428" referrerpolicy="no-referrer"></p><p data-vmark="c5b0">正如我们在最近的 Insider 版本中看到的那样，微软似乎有意在添加新功能，但同时忽略了现有版本中的 UI 不一致和缺乏基本功能的情况。</p><p data-vmark="15aa">截图显示中的所有应用程序都是微软开发的内置应用，但我们看到 UI 和 UX 始终不一致，就好像没有高级领导来监督操作系统的设计一样。甚至图标的样式也不同，有些边框较粗，有些则更简约。就好像每个开发团队都在自己的孤岛中工作，只专注于他们认为看起来漂亮的东西，而不去考虑 Windows 11 的整体美感如何。</p><p data-vmark="9e42" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/4/a5c13445-d4df-4986-9f78-7ff7daba7abc.jpg" w="680" h="421" title="微软 Win11 正式版发布半年后 UI / UX 设计仍不统一，网友呼吁先把基础功能做好" width="680" height="421" referrerpolicy="no-referrer"></p><p data-vmark="afb9">事实上，这个问题不仅限于图标。我们发现，<a class="s_tag" href="https://win10.ithome.com/" target="_blank">Windows 10</a> 的旧菜单遍布 Windows 11 整个操作系统，菜单选项基本上已经从以前的 Windows 版本中提升和转移，但根本没有考虑整体 UX。</p><p data-vmark="95ef">所有这些都表明 Windows 11 明显缺乏设计标准化，这对于被数百万人使用的操作系统来说并不是好的感官体验。</p><p data-vmark="5c24">Windows 11 还存在缺少基本功能的问题。微软仍然以荒谬的理由拒绝让人们移动 Windows 11 任务栏，但却开发了记事本可转动的设置按钮齿轮。</p><p data-vmark="59f2">Windows 11 开发团队很庞大，由多个子团队组成，不同的人从事不同的工作。但这种类型的优先排序会导质量参差不齐。</p><p data-vmark="08a7" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/3/6a7761e6-c051-4883-a873-b9649ae82a5b.gif" w="600" h="398" title="微软 Win11 正式版发布半年后 UI / UX 设计仍不统一，网友呼吁先把基础功能做好" width="600" height="398" referrerpolicy="no-referrer"></p><p data-vmark="b7f8">那么什么是优先的基础性功能？微软反馈中心那里有大量的项目有成千上万的赞成票，理想情况下，微软应该在开发方面以更高的优先级跟踪这些项目。然而，这些请求中的许多已经被搁置了几个月，都没有来自微软的更新，说明微软有意忽视了这些反馈。</p><p data-vmark="6a39">Windows 11 显然缺乏集中监督，应该负责管理优先级、指导缺失功能的开发和标准化 UI 设计。</p><p data-vmark="7690">现有方式的问题在于，每个团队将继续以孤立的方式在自己的关注领域进行开发，虽然这可能会带来大量新功能，但它可能不是用户实际需要的东西。因为 UX 的不一致，Windows 11 不同组件的集合，可能导致该操作系统很难具有凝聚力表现。</p><p data-vmark="e386">微软是时候考虑做出改变了。</p>
