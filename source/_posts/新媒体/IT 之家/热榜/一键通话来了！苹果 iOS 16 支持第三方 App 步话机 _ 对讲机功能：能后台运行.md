<p data-vmark="50e2"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 6 月 10 日消息，苹果发布了 iOS 16、iPadOS 16 系统，并且已经发布了首个开发者预览版 Beta。</p><p data-vmark="12d6">iOS 16 带来了新功能，例如重新设计的带有小组件的锁屏、扩展的专注模式、iMessage 改进、无密码等。此外，iOS 16 附带了多个新 API，其中一个为第三方 App 启用步话机 / 对讲机功能。</p><p data-vmark="b319" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/6/e31cd38e-50b0-46c5-aa43-be8d2e0d7e75.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzkyLnBuZw==,g_7,x_9,y_9,a_0,t_100" w="600" h="587" title="一键通话来了！苹果 iOS 16 支持第三方 App 步话机 / 对讲机功能：能后台运行" width="600" height="587" referrerpolicy="no-referrer"></p><p data-vmark="614b">新的 API 被称为“Push to Talk”，基本上允许开发人员在他们的 iOS 应用程序中添加对讲机。在 WWDC22 的一次会议上，苹果展示了新 API 的工作原理以及开发人员如何实现它。</p><p data-vmark="12fc">iOS 已经允许音频应用在后台运行，但到目前为止，这仅限于常规语音消息甚至音频通话等内容。使用 Push to Talk API，即使在后台，也可以实时发送和播放音频消息。</p><blockquote><p data-vmark="001c">“我们帮助你将大声而清晰地对讲机通信带入应用程序 ，了解如何将突出的系统 UI 添加到 Push to Talk 应用程序，通过点击按钮实现快速通信。我们将介绍 Push To Talk 框架，并展示如何配置你的应用程序以传输和接收音频 —— 甚至从后台。”</p></blockquote><p data-vmark="017b"><span class="accentTextColor">当有人向你发送消息时，屏幕顶部会显示对讲机通信，iOS 会显示应用程序图标和与你交谈的人的姓名。</span>听到消息后，可以选择使用音频实时回复或离开对话。Push to Talk 适用于个人对话甚至团体。</p><h3 data-vmark="f770">苹果的 walkie-talkie 呢？</h3><p data-vmark="8c0b">Apple Watch 用户可能熟悉苹果的 Walkie-Talkie 应用程序，该应用程序自 watchOS 5 起默认预装。虽然 iOS 16 中的新 API 以类似的方式工作，但它们是不同的东西。</p><p data-vmark="e0dc">苹果的 Walkie-Talkie 应用程序使用 FaceTime 音频基础设施实时发送和接收消息。至于新的 API，它需要开发者实现自己的后端来传递和接收音频消息。API 启用的唯一功能是对讲机对话的通用接口，它甚至可以在后台运行。</p><p data-vmark="749e">当然，有了这个新的 API，苹果将来可以将自己的 Walkie-Talkie 应用程序带到 iPhone 和 iPad 上。但就目前而言，此功能依赖于第三方应用程序。</p>
