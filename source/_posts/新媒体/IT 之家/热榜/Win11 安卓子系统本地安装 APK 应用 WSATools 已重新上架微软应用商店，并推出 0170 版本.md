<p data-vmark="d25a"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 12 月 4 日消息，WSATools 是一款为 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> <a class="s_tag" href="https://android.ithome.com/" target="_blank">安卓</a>子系统提供本地 APK 安装的第三方程序，在被微软从应用商店下架后，<span class="accentTextColor">目前已再次公开上架</span>。此前，该应用可通过私链下载。</p><p data-vmark="2639"><img src="https://img.ithome.com/newsuploadfiles/2021/12/6c9a51c7-9c11-4b5b-bc7a-35df68b6176e.png" w="1080" h="748" title="Win11 安卓子系统本地安装 APK 应用 WSATools 已重新上架微软应用商店，并推出 0.1.70 版本" width="1080" height="568" referrerpolicy="no-referrer"></p><p data-vmark="76eb">12 月 2 日，WSATools 开发者 Simone Franco 通过 Twitter 表示，WSATools 已推出 0.1.70 版本，并在微软应用商店公开上架。新版本改进了 ADB 安装，还修复了部分加载屏幕卡住的问题。</p><p data-vmark="fcb0"><img src="https://img.ithome.com/newsuploadfiles/2021/12/2453149c-31c6-447e-86bf-c12d7d42e2e9.png" w="874" h="1165" title="Win11 安卓子系统本地安装 APK 应用 WSATools 已重新上架微软应用商店，并推出 0.1.70 版本" width="874" height="1093" referrerpolicy="no-referrer"></p><p data-vmark="92f1">据 Simone Franco 此前透露，WSATools 被下架的原因包括“包含 WSA（Windows 安卓子系统）字母的名称问题”。微软称这会造成混淆，且开发者应当在应用页面明确说明基于 WSA 的特点。</p><p data-vmark="b8be"><a href="https://www.microsoft.com/zh-cn/p/app/9n4p75dxl6fg" target="_blank">WSATools 应用</a>介绍显示：“易于 Windows Subsystem for <a class="s_tag" href="https://android.ithome.com/" target="_blank">Android</a> 使用的 apk 安装程序。WSATools 是一个简单的 apk 安装程序，适用于 Windows 11 的 Android 子系统，设计直观，让用户立刻拥有家的感觉。”</p><p data-vmark="31d6"><img src="https://img.ithome.com/newsuploadfiles/2021/12/4bc087c9-49fc-4506-8f27-194522a1fe78.png" w="1080" h="762" title="Win11 安卓子系统本地安装 APK 应用 WSATools 已重新上架微软应用商店，并推出 0.1.70 版本" width="1080" height="579" referrerpolicy="no-referrer"></p><p data-vmark="5b0a">IT之家了解到，Windows 11 安卓子系统 (Windows Subsystem for Android) 包括 Linux 内核和基于 Android 开源项目（AOSP）版本 11 的 Android 操作系统。该子系统在 Hyper-V 虚拟机中运行，就像 Linux 子系统一样，可以将 AOSP 环境中 App 的运行时和 API 映射到 Windows 图形层、内存缓冲区、输入模式、物理和虚拟设备以及传感器，可以在英特尔、AMD、高通的 CPU 上运行。</p>
