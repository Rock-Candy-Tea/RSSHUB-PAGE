<p data-vmark="295e"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 10 月 16 日消息，今天，微软发布了 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> Build 22000.282（KB5006746）预览版，面向 Insider 计划的 Beta 和 Release Preview 频道测试者推出，最终修复了 AMD 之前承认的 CPU 性能下降问题。</p><p data-vmark="1c53">Windows 11 正在逐步推出，很多用户已经安装了新的操作系统，但有些用户发现性能大幅下降。根据媒体报道和 AMD 的说法，Windows 11 中的一个错误 Bug 可能会大幅降低兼容的 AMD 处理器性能。</p><p data-vmark="631c">事实上，用户在 AMD 硬件上运行 <a class="s_tag" href="https://win10.ithome.com/" target="_blank">Windows 10</a> 时，也可能会遇到常规应用程序的问题。如果你在 AMD CPU 上安装了 Windows 11，并注意到性能问题或应用程序/游戏的问题，微软现在正通过 Windows Update 推出修复程序。</p><p data-vmark="1556"><img src="https://img.ithome.com/newsuploadfiles/2021/10/ca17b964-1152-4a30-b59e-2b3fac11ba08.png" w="1015" h="765" title="微软 Win11 Build 22000.282 Beta/Release 预览版发布：修复 AMD CPU 性能下降问题（附更新内容大全）" width="1015" height="618" referrerpolicy="no-referrer"></p><p data-vmark="8089">Windows 11 Build 22000.282 现在可供 Insider 计划的测试者使用，它最终解决了 L3 缓存的兼容性问题，这影响了一些应用程序的性能。事实上，AMD 声称，如果用户安装 Windows 11，L3 缓存延迟可能增加近三倍。</p><p data-vmark="c8ec">因此，应用程序的性能受到 3-5% 的影响，而一些游戏的性能下降高达 15%，特别是用于电子竞技的游戏。显然，这个错误 Bug 足够严重，微软立即实施了修复措施。</p><p data-vmark="c8ec"><img src="https://img.ithome.com/newsuploadfiles/2021/10/5e4da9a1-9270-412b-8b32-f86f0e8d4137.jpg@s_2,w_820,h_583" w="1019" h="724" alt="Build 22000.282 for Windows 11" title="微软 Win11 Build 22000.282 Beta/Release 预览版发布：修复 AMD CPU 性能下降问题（附更新内容大全）" srcset="https://img.ithome.com/newsuploadfiles/2021/10/5e4da9a1-9270-412b-8b32-f86f0e8d4137.jpg 2x" width="1019" height="583" referrerpolicy="no-referrer"></p><p data-vmark="be68">安装 Windows 11 Build 22000.282 后，如果你在 AMD CPU 上加载应用程序或游戏，理论上，Windows 将不会降低应用程序的性能。</p><p data-vmark="8c85">不过，本次更新还在测试中，Windows 11 正式版频道的用户可能要到本月底才能得到这个修复，并且是作为可选更新补丁。</p><p data-vmark="44ce">IT之家获悉，这一修复补丁也将出现在 2021 年 11 月的“星期二更新补丁”中。在此之前，你可以降级到 Windows 10，暂时不玩游戏或者干脆等待 10 月晚些时候发布的可选更新。</p><p data-vmark="f808">除了性能改进外，微软还修复了一个问题，即从 Windows 10 升级时，新的“开始”菜单无法在 Windows 11 上出现。还修复了一个资源竞争 Bug，即 Windows 启动时可能导致蓝屏或黑屏死机的错误。</p><p data-vmark="1603">此外，微软还修复了在注册表中使用非 ASCII 字符，导致
