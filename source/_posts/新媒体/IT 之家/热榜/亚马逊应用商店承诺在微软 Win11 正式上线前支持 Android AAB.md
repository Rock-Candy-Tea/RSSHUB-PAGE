<p><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 7 月 17 日消息 据外媒 Windows Latest 报道，微软最近证实，即将推出的 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 将能运行 <a class="s_tag" href="https://android.ithome.com/" target="_blank">Android</a> 应用。这些 Android 应用将通过 Microsoft Store 提供，但实际下载过程将通过亚马逊应用商店（Amazon Appstore）进行。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/6/0bdd29be-4721-4e1d-ab26-c6352765a36c.jpg@s_2,w_820,h_479" title="亚马逊应用商店承诺在微软 Win11 正式上线前支持 Android AAB" referrerpolicy="no-referrer"></p><p>微软正在将亚马逊应用商店集成到微软应用商店中。用户可以通过微软商店轻松搜索 Android 应用，打开微软商店中的亚马逊应用商店，然后点击几下就可以安装应用。</p><p>IT之家了解到，6 月下旬，谷歌确认将要求应用开发者使用新的 Android App Bundle (AAB) 标准，而不是 Android Application Package (APK)。据谷歌称，此政策适用于 Play 商店中的新应用，开发者将被要求为新应用放弃当前的 APK 文件格式。</p><p>从技术上讲，Android APK 应用不会停止使用，因为 Android 将继续支持现有格式，但在谷歌开始对 Play 商店实施 AAB 后，开发人员可能会停止以 APK 格式发布应用。这也意味着，这种新格式最终会取代 APK 成为标准的发布格式。</p><p>不过，这对 Windows 11 来说并不是问题。亚马逊已经确认，<span class="accentTextColor">其应用商店将支持 Android App Bundle (AAB)</span>。亚马逊称“其应用商店正在积极支持新格式。”</p><p>亚马逊应用商店中的大多数应用在获得开发者许可后都可以在 Windows 11 上自动运行。</p>
