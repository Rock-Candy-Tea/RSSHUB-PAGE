<p data-vmark="34d0"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 13 日消息，据IT之家网友 <a contenteditable="false" class="mention-user-link" href="https://m.ithome.com/html/app/open.html?url=ithome%3A%2F%2Fuserpage%3Fid%3D1640835" data-appurl="ithome://userpage?id=1640835" target="_blank">@阿里郎ya</a> 反馈，小米路由官方在小米社区表示，<span class="accentTextColor">小米路由器 AX9000 开发者版本支持 Docker 功能</span>，可从小米路由器官网下载 1.0.140 版本或以上的 AX9000 开发者固件开启体验。</p><p data-vmark="19f5"><img src="https://img.ithome.com/newsuploadfiles/2021/4/1ff3cb7b-5929-462f-8eb2-d86b9b834e92.jpg@s_2,w_1440,h_960" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" referrerpolicy="no-referrer"></p><p data-vmark="9853">安装好 Docker 环境及管理工具后，用户可通过管理工具创建并使用容器、镜像，并可调用第三方管理工具 Portainer 的绝大部分功能。</p><p data-vmark="eec1">其中，Docker 镜像是一个特殊的文件系统，封装了运行时需要的库、资源、应用等。docker 镜像采用分层存储结构。构建时一层一层地进行，每一层构建好后不再变化，后一层以前一层为基础进行构建，最终组成多层文件系统。</p><p data-vmark="6320">据介绍，Docker 容器本质上则是一个进程，运行于自己独立的命名空间中，所有的数据都存储在容器中。容器销毁后，这些数据也随之消失。多个容器互相之间是隔离的，一个容器所做的任何变更都只影响容器自己。</p><p data-vmark="fc3c"><strong>以下为小米路由官方对 AX9000 开发者版本 Docker 功能的介绍：</strong></p><h2 data-vmark="94b6">一、操作指导</h2><p data-vmark="450a"><strong>第一步：</strong>登录<a href="http://www1.miwifi.com/miwifi_download.html" target="_blank">小米路由器官网</a>下载小米路由器 AX9000 最新的开发者版本，并升级路由器至开发者版本 (如已升级，忽略此步骤）；</p><p data-vmark="645d"><strong>第二步：</strong>进入开发者版本管理后台（默认为 192.168.31.1 或 miwifi.com)，在高级设置中找到 DOCKER 功能；</p><p data-vmark="64ff"><img src="https://img.ithome.com/newsuploadfiles/2021/11/4d09b6c4-cb61-49cb-a3e7-23d77b5229ae.png" w="750" h="490" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="750" height="490" referrerpolicy="no-referrer"></p><p data-vmark="28c5"><strong>第三步：</strong>插入 ext4 格式且剩余存储空间大于 32G 的 USB 存储设备到小米路由器 AX9000 USB 口，单击“安装 Docker”按钮，安装 Docker 套件直至提示安装成功；</p><p data-vmark="bebe"><img src="https://img.ithome.com/newsuploadfiles/2021/11/0b9b7207-3a7f-4876-9568-ab77ed4ffe33.png" w="1440" h="725" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="413" referrerpolicy="no-referrer"></p><p data-vmark="63e5"><img src="https://img.ithome.com/newsuploadfiles/2021/11/ffed3638-60ad-4005-b441-3ed867b06108.png" w="1440" h="643" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="366" referrerpolicy="no-referrer"></p><p data-vmark="6ab1"><strong>第四步：</strong>Docker 安装成功后，页面将自动刷新，打开运行 Docker 拉钮，并且安装第三方管理，直至提示安装成功；</p><p data-vmark="0e57"><img src="https://img.ithome.com/newsuploadfiles/2021/11/aad9a490-c6d4-4136-a365-5519b9840264.png" w="1440" h="707" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="403" referrerpolicy="no-referrer"></p><p data-vmark="2c30"><strong>第五步：</strong>此时即可单击“管理 Docker 按钮”，体验 Docker 功能。</p><p data-vmark="fc27"><img src="https://img.ithome.com/newsuploadfiles/2021/11/6c447305-db1f-43c7-9517-8f1eb72f4e6d.png" w="1440" h="729" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="415" referrerpolicy="no-referrer"></p><h2 data-vmark="1f0e">二.小米路由器 AX9000 上的 Docker 可以实现哪些功能？</h2><p data-vmark="cf67">安装好 Docker 环境及管理工具后，你可以通过管理工具创建并使用容器、镜像，并可调用第三方管理工具 Portainer 的绝大部分功能。</p><p data-vmark="19c8">1. 第三方工具 Portainer 管理功能</p><p data-vmark="b739">第三方工具 Portainer 安装后可以新建镜像和容器并对他们做管理。</p><p data-vmark="8ee5">用户在小米路由器 AX9000 首次使用 Portainer 时，默认用户名 admin，密码 admin。更多信息可参考 <a href="https://www.portainer.io/" target="_blank">Portainer 官网</a>。</p><p data-vmark="f58f">2. Docker 镜像</p><p data-vmark="31db">Docker 镜像是一个特殊的文件系统，这个文件系统封装了运行时需要的库、资源、应用等。docker 镜像采用分层存储结构。构建时一层一层地进行，每一层构建好后不再变化，后一层以前一层为基础进行构建，最终组成多层文件系统。</p><p data-vmark="0776">3. Docker 容器</p><p data-vmark="30b0">Docker 容器本质上是一个进程，运行于自己独立的命名空间中，所有的数据都存储在容器中。容器销毁后，这些数据也随之消失。多个容器互相之间是隔离的，一个容器所做的任何变更都只影响容器自己。</p><p data-vmark="afa9">注意事项：</p><p data-vmark="d41f">（1）使用过程中，如遇到右上角所示的提示弹窗，等提示消失再输入用户名密码（2 秒会自动消失）。</p><p data-vmark="af2a"><img src="https://img.ithome.com/newsuploadfiles/2021/11/cfc1fa8a-5e85-4544-ab06-43a0885117e2.png" w="1432" h="590" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1432" height="338" referrerpolicy="no-referrer"></p><p data-vmark="5ac5">（2) 首次登录时选择第一个选项，本地管理容器。</p><p data-vmark="9a09"><img src="https://img.ithome.com/newsuploadfiles/2021/11/7df2d18e-97e6-4151-aefd-28d99106e0e9.png" w="802" h="405" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="802" height="405" referrerpolicy="no-referrer"></p><p data-vmark="01dc">功能示例（创建 Maria 数据库容器）</p><p data-vmark="5f10">第三方管理工具中，进入 Containers 页，单击 add container 按钮。</p><p data-vmark="e824"><img src="https://img.ithome.com/newsuploadfiles/2021/11/933aaba0-e305-4177-8769-0a58ff73505d.png" w="1440" h="506" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="288" src="https://img.ithome.com/newsuploadfiles/2021/11/f74db1e7-7d63-4eb1-8646-07ccb8b61081.png" w="1440" h="650" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="370" referrerpolicy="no-referrer"></p><p data-vmark="74e3">例如搜索关键词”mariadb“，搜索到 Mariadb 官方镜像。</p><p data-vmark="005a"><img referrerpolicy="no-referrer"></p><p data-vmark="8044">复制 mariadb 官方镜像名到镜像输入框。</p><p data-vmark="d11a"><img src="https://img.ithome.com/newsuploadfiles/2021/11/7542d9d3-d77f-4255-8f5d-c9b2298b0049.png" w="1440" h="650" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="370" referrerpolicy="no-referrer"></p><p data-vmark="8139">如果是再次创建的情况，按默认提示点选填充了 latest 后缀的镜像。</p><p data-vmark="53f1">单击”Deploy the container" 部署镜像在容器中。</p><p data-vmark="be48"><img src="https://img.ithome.com/newsuploadfiles/2021/11/cee76a67-acd5-466e-929e-5cd1ecc8133c.png" w="1044" h="704" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1044" height="553" referrerpolicy="no-referrer"></p><p data-vmark="63ae">安装完毕后，查看是否已创建好 MariaDatabase 容器，如下图所示为创建容器成功。</p><p data-vmark="b5b2"><img src="https://img.ithome.com/newsuploadfiles/2021/11/45832803-a656-4f8a-b5d1-68e638673ef6.png" w="1440" h="304" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1440" height="173" referrerpolicy="no-referrer"></p><h2 data-vmark="d868">三、使用 Docker 的注意事项</h2><p data-vmark="a3dc">1. Docker 启动过程对端口转发功能的影响</p><p data-vmark="5276">Docker 启用时的 30 秒内，由于 Docker 生成的虚拟容器中的防火墙会和系统预设防火墙管理会有差异，所以可能导致 Docker 生成的容器中的防火墙丢失。所以建议这段时间内不对路由系统防火墙做反复改动，或者先关闭 Docker。</p><p data-vmark="108b"><img src="https://img.ithome.com/newsuploadfiles/2021/11/7d01c70f-864a-4594-b617-c498ab5d2754.png" w="1258" h="654" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="1258" height="426" referrerpolicy="no-referrer"></p><p data-vmark="7c2e">2. Docker 运行中不建议更换局域网配置</p><p data-vmark="a6e8">由于做局域网设置的改动可能会引发路由系统重启，建议配置路由器局域网前先关闭 Docker 功能，否则出现未知错误。</p><p data-vmark="8538"><img src="https://img.ithome.com/newsuploadfiles/2021/11/2e912685-ac8e-405a-9562-6838da00ee96.png" w="541" h="656" title="小米路由器 AX9000 开发者 1.0.140 版本或以上已支持 Docker 功能" width="541" height="656" referrerpolicy="no-referrer"></p><p data-vmark="e545">3. Docker 创建容器上限</p><p data-vmark="5b76">现阶段建议创建容器数量不要超过 3 个，如果超过 3 个，可能出现未知错误。</p><p data-vmark="1139">4. 插拔硬盘注意事项 Docker 使用过程中，如果需要拔出硬盘，请先到存储状态页点击“安全移除硬盘”。</p>
