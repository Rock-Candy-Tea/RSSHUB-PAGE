<p data-vmark="ebb2"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 10 月 13 日消息，今天微软官方宣布了一项 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 任务管理器关于 Edge <a class="s_tag" href="https://qiyu.ruanmei.com/" target="_blank">浏览器</a>进程的显示方式改进。微软表示，Edge 性能至关重要，这使它成为追求的核心。微软在继续改进 Microsoft Edge 的过程中始终牢记性能为先，但也在努力帮助用户了解 Microsoft Edge 是如何利用你设备上的资源来提供快速、高效的网络浏览的。</p><p data-vmark="5a6b"><img src="https://img.ithome.com/newsuploadfiles/2021/10/960650b8-d7c7-42ce-a418-49b2ecfe962d.jpg@s_2,w_820,h_547" w="1200" h="800" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" srcset="https://img.ithome.com/newsuploadfiles/2021/10/960650b8-d7c7-42ce-a418-49b2ecfe962d.jpg 2x" width="1200" height="547" referrerpolicy="no-referrer"></p><p data-vmark="3236">假如，有一天你正在浏览互联网，突然你的电脑开始变慢，风扇转起，电脑变热。你不确定是什么导致了这个问题，所以你打开 Windows 任务管理器，试图诊断这个问题。当你打开 Windows 任务管理器时，你很难理解是什么在占用浏览器资源，所以你只能随机关闭标签，希望它能解决性能问题。</p><p data-vmark="9a7c">微软希望大家在使用 Edge 时为用户改善这种体验。考虑到这一点，微软已经对 Microsoft Edge 在 Windows 任务管理器中的表现方式进行了改进。这些变化现在可以提供给自行托管 Windows 11 的内测人员，并且已经在 Edge 94 稳定版中实现。微软进行这些改进的目的是让用户更好地了解资源去向，并在遇到性能问题时帮助做出明智的决定。</p><h2 data-vmark="f5e8">Windows 任务管理器对 Microsoft Edge 的改进</h2><p data-vmark="6218">目前，当你打开 Windows 任务管理器并在进程标签上展开 Microsoft Edge 时，你会看到一个名为 Microsoft Edge 的进程列表。比如展开“Microsoft Edge”条目，显示有 11 个进程都是“Microsoft Edge”。</p><p data-vmark="0560"><img src="https://img.ithome.com/newsuploadfiles/2021/10/cfc1d756-0d3c-40b6-aca8-2e13d9a68f1a.png" w="624" h="395" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="624" height="395" referrerpolicy="no-referrer"></p><p data-vmark="6896">IT之家获悉，随着 Windows 任务管理器的最新改进，你现在将看到 Microsoft Edge 进程的详细视图。如果你不熟悉 Microsoft Edge 的多进程架构，你可以<a href="https://blogs.windows.com/msedgedev/2020/09/30/microsoft-edge-multi-process-architecture/" target="_blank">在这里</a>找到更容易理解的解释。在这篇博文中，你将了解为什么 Microsoft Edge 有多个进程在运行，每个进程的目的是什么，以及多进程架构的好处。</p><p data-vmark="2629">下面是微软 Edge 在 Windows 任务管理器中的表现，有了最新的改进。显示 Microsoft Edge 的任务管理器扩展到列出各种进程和子进程，每个进程都有独特的名称和图标，如“GPU 进程”、“扩展”等。</p><p data-vmark="7236"><img src="https://img.ithome.com/newsuploadfiles/2021/10/992fa286-80ea-4ffb-8200-89bd4b529428.png" w="1024" h="721" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="1024" height="577" referrerpolicy="no-referrer"></p><h2 data-vmark="8ab8">微软公布的 Edge 进程显示方式的改进内容</h2><ul class=" list-paddingleft-2"><li><p data-vmark="b60d">对于浏览器进程、GPU 进程和 Crashpad 进程，你将看到带有描述性名称和图标的进程类型（如“Browser”、“GPU Process”或“Crashpad”。</p></li><li><p data-vmark="d975">对于组件程序、插件和扩展进程，你将看到进程类型和服务、插件或扩展的名称。</p></li><li><p data-vmark="32d4">对于渲染器进程：</p></li></ul><p data-vmark="252f">对于标签，你将看到“tab”一词，以及网站名称和图标（注意：对于 InPrivate 标签，网站名称将被删除，图标将更新为默认的标签图标）。</p><p data-vmark="2d06">对于子框架（通常用于广告），你将看到“subframe”这个词，后面是 subframe 的 URL。</p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="7e82">所有专门的 workers 和服务 workers 将被列出。</p></li></ul><p data-vmark="4455">在 Windows 任务管理器中 Microsoft Edge 下的行并不总是一个单独的进程。有些进程包含多个项目，在 Windows 任务管理器中会用多行表示。</p><p data-vmark="fb14">有多种方法可以区分哪些项目在自己的进程中，哪些项目共享一个进程。</p><p data-vmark="9349"><strong>识别一个进程中的单个项目</strong></p><p data-vmark="634f">对应于单个进程的项目，其图标旁边没有箭头。</p><p data-vmark="2c1b"><img src="https://img.ithome.com/newsuploadfiles/2021/10/1da404d2-fdd2-499f-b176-8311e54727d5.png" w="182" h="21" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="182" height="21" referrerpolicy="no-referrer"></p><p data-vmark="e2f7">当点击时，只有一行被突出显示。</p><p data-vmark="203a"><img src="https://img.ithome.com/newsuploadfiles/2021/10/2562af2c-ba5c-42fd-954e-0845b66a05f6.png" w="157" h="20" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="157" height="20" referrerpolicy="no-referrer"></p><p data-vmark="1bbb">每一行都会有自己的资源使用情况。</p><p data-vmark="ec52"><img src="https://img.ithome.com/newsuploadfiles/2021/10/6cd9641e-e947-4c99-a6db-d3147b47c089.png" w="252" h="79" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="252" height="79" referrerpolicy="no-referrer"></p><p data-vmark="af44">当点击“结束任务”时，单一的高亮行将被关闭。</p><p data-vmark="7b6c"><strong>识别共享一个进程的多个项目</strong></p><p data-vmark="b03e">图标旁边的一个箭头表示该项目与其他项目共享一个进程。在标题的末尾，括号里还会有一个数字，表示有多少其他项目共享该进程。箭头可以被点击来展开该进程。</p><p data-vmark="d241">当进程被折叠时，箭头指向右边，你将只看到进程中的一个项目。</p><p data-vmark="a3d2"><img src="https://img.ithome.com/newsuploadfiles/2021/10/240ebbdf-f060-40b3-8975-dd2e725aee2c.png" w="201" h="17" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="201" height="17" referrerpolicy="no-referrer"></p><p data-vmark="ab56">当进程被展开时，箭头指向下方，你将看到与该项目共享一个进程的所有项目（一旦点击箭头就会显示的项目，其旁边不会有图标）。</p><p data-vmark="dda5"><img src="https://img.ithome.com/newsuploadfiles/2021/10/d415db35-c94d-4edf-89ac-66432ca69f72.png" w="326" h="51" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="326" height="51" referrerpolicy="no-referrer"></p><p data-vmark="0534">当点击并且进程被折叠时，只有一行被突出显示。</p><p data-vmark="6326"><img src="https://img.ithome.com/newsuploadfiles/2021/10/ffb02513-0560-475f-98b3-0da6c55b8d9a.png" w="203" h="18" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="203" height="18" referrerpolicy="no-referrer"></p><p data-vmark="0f7b">当点击并且进程被展开时，所有共享一个进程的项目被高亮显示。</p><p data-vmark="b616"><img src="https://img.ithome.com/newsuploadfiles/2021/10/a77f0633-ed91-4305-b9c0-de70d0bfbfda.png" w="324" h="51" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="324" height="51" referrerpolicy="no-referrer"></p><p data-vmark="31c1">由于多个项目共享一个进程，旁边有箭头的项目将向你显示该进程的总资源使用量。当展开时，其余行的资源使用量将保持空白。</p><p data-vmark="f880"><img src="https://img.ithome.com/newsuploadfiles/2021/10/55d288ba-5068-4ff3-88f0-96ccf6431208.png" w="309" h="57" title="微软 Win11 任务管理器重磅改进 Edge 浏览器进程显示方式：资源占用一目了然" width="309" height="57" referrerpolicy="no-referrer"></p><p data-vmark="5df1">当点击“结束任务”时，一个进程被展开时，所有突出显示的行将被关闭。你只能关闭进程，而不是 Windows 任务管理器中的单个项目。</p><p data-vmark="a351"><strong>结束任务</strong></p><p data-vmark="201a">当结束任务时，根据你要结束的进程，你会看到不同的行为。下面是可能看到的几个例子。</p><ul class=" list-paddingleft-2"><li><p data-vmark="8b10">浏览器进程：与该 Microsoft Edge 实例相关的所有进程将被关闭。</p></li><li><p data-vmark="8f9e">GPU 进程，组件进程：进程将关闭并重新启动。当进程关闭时，你可能会看到你的 Microsoft Edge 窗口有短暂的变化。例如，当 GPU 进程关闭时，你的 Microsoft Edge 窗口将短暂变黑，一旦进程重新启动，将恢复正常；当音频组件程序进程关闭时，如果你正在听音乐，音频将短暂停止，一旦进程重新启动，将恢复正常。</p></li><li><p data-vmark="c094">渲染器进程：进程将关闭。如果该进程包含标签，该进程中的所有标签将被替换成一个错误页面，说“这个页面有问题”。如果该进程包含子框架，可见的子框架将被一个崩溃的标签图标所取代。对于不可见的子框架，你可能不会看到有什么不同，但页面可能会受到某种影响。</p></li><li><p data-vmark="2f30">扩展和插件进程：在你的屏幕右下角会出现一个气球，说扩展或插件已经崩溃，并会问你是否愿意重新加载扩展或插件。</p></li><li><p data-vmark="3475">Crashpad 进程：进程将关闭。Microsoft Edge 中的崩溃可能会通过其他方式报告。</p></li><li><p data-vmark="02b1">专门 workers/服务 workers：标签的功能可能会中断。</p></li></ul><h2 data-vmark="369b">对浏览器任务管理器的改进</h2><p data-vmark="dc87">如果你熟悉 Microsoft Edge 中的任务管理器，可以在 Microsoft Edge 中按 Shift+Esc 打开，或者到浏览器的顶端选择设置和更多（...）> 更多工具 > 浏览器任务管理器，你会看到对 Windows 任务管理器的改进看起来很相似。微软希望在 Windows 任务管理器中提供与浏览器任务管理器相同的细节，以帮助你更好地了解资源去向，并在你遇到性能问题时帮助你做出明智的决定。</p><p data-vmark="4c0a">需要注意的一点是，Windows 任务管理器显示的内存值与浏览器任务管理器不同。</p>
