<p data-vmark="e775">GitHub 上突然有人上传了一个 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Win11</a> 最新漏洞的利用办法，几天之内暴涨 1300 多星。</p><p data-vmark="8c0b"><img src="https://img.ithome.com/newsuploadfiles/2021/11/a7041b81-facb-43f7-8caf-9ae0847507ac.png" w="812" h="220" title="微软 Win11 高危漏洞被公开，只因悬赏奖金打骨折：3 秒获得系统管理员权限" width="812" height="220" referrerpolicy="no-referrer"></p><p data-vmark="fa14">▲ 学名“概念性验证攻击程序”</p><p data-vmark="8c57">话说，发现漏洞不是可以报告给微软领取高额赏金来着，他怎么不要了？</p><p data-vmark="6aca">按这位黑客自己的说法“现在微软的赏金已成了垃圾”。</p><p data-vmark="71dc"><span class="accentTextColor">微软漏洞发现赏金现已大幅缩水</span>，曾有白帽黑客抱怨本来该获得 1 万美元的漏洞，最后只拿到 1 千美元，直接缩水 90%。</p><p data-vmark="9afa"><img src="https://img.ithome.com/newsuploadfiles/2021/11/854f7bd5-3492-4ad5-a185-6bd2206425a1.png" w="1080" h="353" title="微软 Win11 高危漏洞被公开，只因悬赏奖金打骨折：3 秒获得系统管理员权限" width="1080" height="268" referrerpolicy="no-referrer"></p><p data-vmark="11d2">这次的这位黑客 Naceri 也很失望，索性剩下那点钱也不要了，直接公开算了。</p><p data-vmark="f9b3">通过这个漏洞，<span class="accentTextColor">恶意程序能在几秒内获得管理员权限</span>，能在你电脑上为所欲为的那种。</p><p data-vmark="6064"><img src="https://img.ithome.com/newsuploadfiles/2021/11/da62330c-25c0-454b-83f1-8b96a205e3a0.gif" w="900" h="520" title="微软 Win11 高危漏洞被公开，只因悬赏奖金打骨折：3 秒获得系统管理员权限" width="900" height="474" referrerpolicy="no-referrer"></p><p data-vmark="2b2c">漏洞出在 Windows Installer Service 上，就是用.msi 文件安装和卸载软件时用到的服务。</p><p data-vmark="9ff2">Naceri 在 GitHub 页面上说漏洞会影响到最新的 Win11 和服务器版 Windows Server 2022。</p><p data-vmark="5c7c">不过安全技术网站 Bleeping Computer 测试发现，<span class="accentTextColor">现在最普及的 <a class="s_tag" href="https://win10.ithome.com/" target="_blank">Win10</a> 也逃不过</span>。</p><p data-vmark="f9fc">现在，思科安全情报团队 Talos 已侦测到了利用这个漏洞的恶意程序。</p><p data-vmark="0913"><img src="https://img.ithome.com/newsuploadfiles/2021/11/6b7cb6d0-ac7c-4545-9639-24613a044f63.png" w="1080" h="745" title="微软 Win11 高危漏洞被公开，只因悬赏奖金打骨折：3 秒获得系统管理员权限" width="1080" height="566" referrerpolicy="no-referrer"></p><h2 data-vmark="7c15">还有人 7 个月没收到钱</h2><p data-vmark="b9df">微软漏洞悬赏缩水这件事非常打击白帽黑客们的积极性。</p><p data-vmark="a594">另一位发现了 Hyper-V 虚拟机漏洞的老选手就在推特上直呼新规定“不公平！”</p><p data-vmark="a881">按照微软悬赏计划公开的说法，<span class="accentTextColor">此类漏洞赏金上限可达 25 万美元，结果他只拿到了 5000 美元</span>。</p><p data-vmark="755d"><img src="https://img.ithome.com/newsuploadfiles/2021/11/edebe044-3e96-4818-afc8-120435474d37.png" w="1080" h="872" title="微软 Win11 高危漏洞被公开，只因悬赏奖金打骨折：3 秒获得系统管理员权限" width="1080" height="662" referrerpolicy="no-referrer"></p><p data-vmark="b20e">白帽黑客因微软抠门愤而公开漏洞这事也不是第一次发生。</p><p data-vmark="c5ba">去年 9 月，一位长期从事漏洞挖掘的研究者 Lykkegaard 发现了能在 System32 目录添加任意文件的方法，而且一旦写入就无法再删除或修改。</p><p data-vmark="8739">相当严重的一个 Bug，他选择直接给公开了，因为当时微软还拖欠他之前的赏金长达 7 个月。</p><p data-vmark="67aa">Lykkegaard 找到这个漏洞用了 30 个小时，按照缩水后的规则只能拿到 2 千美元。</p><p data-vmark="a1b2">他一算这时薪才 66 美元，关键还不一定能拿得到，实在不值得。</p><p data-vmark="3c4a">公开漏洞是一把双刃剑，虽然可能被人恶意利用，但也能让更多第三方技术高手参与修复。</p><p data-vmark="0276">不过这一次的漏洞却不是那么好修复的。</p><p data-vmark="e1ff">其实这次与 Windows Installer 相关的漏洞，微软已经发布过一次补丁。</p><p data-vmark="33ac">结果这个补丁非但没能完全解决问题，还引发了更复杂的漏洞。</p><p data-vmark="2fc9"><img src="https://img.ithome.com/newsuploadfiles/2021/11/e918c46e-8eac-45c6-b8e6-44830559c8ea.gif" w="460" h="258" title="微软 Win11 高危漏洞被公开，只因悬赏奖金打骨折：3 秒获得系统管理员权限" width="460" height="258" referrerpolicy="no-referrer"></p><p data-vmark="20df">白帽黑客 Naceri 这次公开的实际就是绕过上一个安全补丁的办法，而且他警告再次尝试修复可能带来额外的问题。</p><blockquote><p data-vmark="3b40">不建议第三方尝试修补二进制文件，可能会破坏 Windows Installer。</p></blockquote><p data-vmark="fbc9">所幸的是，第三方社区 0patch 还是在几天之后成功制作并发布了补丁，</p><p data-vmark="e0e4">如果你担心遇到攻击，可以到通过 0patch
