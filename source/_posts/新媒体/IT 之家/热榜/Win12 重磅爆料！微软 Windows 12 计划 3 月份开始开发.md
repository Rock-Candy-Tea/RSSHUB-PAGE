            <p data-vmark="26ad"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 2 月 22 日消息，据 MSPoweruser 报道，<a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 虽然刚刚开始步入正轨，但最新爆料称微软已经在开启下一个计划，Windows 12 的开发将在 3 月份开始。</p><p data-vmark="8ca7" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/2/572d14cc-68de-41d9-adbe-bbb6e1697816.jpg@s_2,w_820,h_461" w="1280" h="720" title="Win12 重磅爆料！微软 Windows 12 计划 3 月份开始开发" srcset="https://img.ithome.com/newsuploadfiles/2022/2/572d14cc-68de-41d9-adbe-bbb6e1697816.jpg 2x" width="1280" height="461" referrerpolicy="no-referrer"></p><p data-vmark="f4d3">德国科技网站 Deskmodder.de 称，根据内部消息，微软将在 3 月开始开发 Windows 12 系统，尽管 Windows 11 正式版才在去年 10 月份发布。</p><p data-vmark="9aaf">正如 Tom's Hardware 所指出的，这一报告似乎得到了 Swift On Security 的证实，他们在推特上声称，“根据微软的消息来源，Windows 12 已经在开发中了，它将需要两个 TPM。”虽然这很快激起了人们的猜测，但 Swift On Security 不久后透露，该推文“只是开了个玩笑”，但 Deskmodder.de 仍然坚称 Windows 12 即将开发。</p><p data-vmark="60c3" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/2/6506e757-dfc3-43d4-8cb5-d68dfd8ce688.png" w="602" h="541" title="Win12 重磅爆料！微软 Windows 12 计划 3 月份开始开发" width="602" height="541" referrerpolicy="no-referrer"></p><p data-vmark="9470">德国网站 WindowsUnited 表示，微软几乎要从头开始构建 Windows 12，没有过多借鉴 <a class="s_tag" href="https://win10.ithome.com/" target="_blank">Windows 10</a>。其想法是建立一个现代的、轻量级的、高效的操作系统，就像当初的 Windows 10X 那样。这可能意味着商店应用程序将被优先考虑，而经典的桌面应用程序将被虚拟化运行，就像 Windows 10X 计划的那样。</p><p data-vmark="635e">即使 Windows 12 很快进入开发阶段，我们也可能要等待相当长的一段时间，才会迎来微软的官方宣布，毕竟一款系统的开发需要多年的时间。</p><p data-vmark="29d7">Windows 10 在 2015 年发布时被称作是 Windows 操作系统的最后一个版本。大家以为会进入较长的稳定期，每两年交替进行一次大更新。然而，Windows 11 的宣布和推出再次唤醒了 Windows 系统的“咆哮”发展。</p><p data-vmark="ee18">Windows 11 在某种程度上是十分必要的，因为一切需要根据时代来适应。一些最重要的预示性变化包括改进的任务调度器，随着英特尔混合架构 Alder Lake 处理器成为主流，这将使 PC 更加高效。此外，系统的安全性也得到重大强调。微软觉得有足够的信心牺牲许多可使用的老一代 CPU（英特尔 Kaby Lake 或更早，AMD Ryzen 1000 或更早）。</p><p data-vmark="12d5">到目前为止，Windows 11 似乎还没有明显获取 PC 用户的芳心。然而，Windows 还有传说中的好版本 / 坏版本周期。基于此，以及也许来自系统和组件制造商促进硬件升级的压力，微软可能会重新回到常规的主要 Windows 版本更新轨道上。</p>
