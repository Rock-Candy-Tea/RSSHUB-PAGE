<p><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 7 月 26 日消息 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Win11</a> 正式版将于今年晚些时候推出，其中包含重新设计的开始菜单和通知中心，以及应用商店。</p><p>据微软官方称，Win11 应用商店将允许几乎任何框架构建的应用上架，包括 <span class="accentTextColor">Win32、.NET、Java、UWP 和 Web</span> 应用程序。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/6/75dc41ff-0f52-499a-ae70-16dc836d6ef5.png" title="微软：Win11 商店可以下载 Win32 应用，但无法提供更新服务" referrerpolicy="no-referrer"></p><p>对于 Win32 应用程序（传统的 .exe 程序），微软将允许开发人员在 Win11 应用商店中提交未打包的 Win32 应用程序，且可以使用自定义内容交付网络进行托管和更新，但似乎有一个问题。</p><p>根据微软应用商店的<a href="https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE4OG2b" target="_blank">官方文档</a>，用户将<span class="accentTextColor">无法从 Win11 商店接收 Win32 应用程序的更新</span>，需要开发人员自行设计更新机制。</p><p>此外，微软还允许开发人员在 Win11 商店中<span class="accentTextColor">发布一个简单的安装包</span>，用户下载之后另行安装，就像微软自己上架的 Edge <a class="s_tag" href="https://qiyu.ruanmei.com/" target="_blank">浏览器</a>一样。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/7/5c22b1d2-ecd5-4f39-812c-ebe0a347bfe3.png" title="微软：Win11 商店可以下载 Win32 应用，但无法提供更新服务" referrerpolicy="no-referrer"></p><p><img src="https://img.ithome.com/newsuploadfiles/2021/7/9c554423-10c5-44f4-897e-84218825a886.png" title="微软：Win11 商店可以下载 Win32 应用，但无法提供更新服务" referrerpolicy="no-referrer"></p><p>IT之家了解到，在推特中，微软官方人员 Rudy Hyun 证实了这一文档说明。为此，Win11 应用商店中将会<span class="accentTextColor">明确标明无法通过商店更新的应用</span>。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/7/0cd50489-9573-4d1c-a61c-ebb57797a946.png" w="735" h="412" title="微软：Win11 商店可以下载 Win32 应用，但无法提供更新服务" width="735" height="412" referrerpolicy="no-referrer"></p><p>比如微软官方的 Edge 浏览器，就标有“<span class="accentTextColor">由 Microsoft Corporation II 提供和更新</span>”，大家在下载的时候可以进行甄别。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/7/6c7c6cb4-d690-4c23-8c74-eff2f97055ab.png" w="1008" h="320" title="微软：Win11 商店可以下载 Win32 应用，但无法提供更新服务" width="1008" height="260" referrerpolicy="no-referrer"></p>
