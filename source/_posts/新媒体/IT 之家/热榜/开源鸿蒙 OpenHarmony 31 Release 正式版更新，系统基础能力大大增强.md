            <p data-vmark="a5e8"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 3 月 31 日消息，开放原子开源基金会现发布了 OpenHarmony 3.1 Release 版本，开源<a class="s_tag" href="https://hmos.ithome.com/" target="_blank">鸿蒙</a> SDK 也更新到了 Ohos_sdk 3.1 Release  （API Version 8 ）版本，基础特性、版本软件和工具配套关系有所升级。</p><h2 data-vmark="faa3">版本概述</h2><p data-vmark="95f0">当前版本在 OpenHarmony 3.1 Beta 的基础上，更新支持以下能力：</p><h3 data-vmark="f459">标准系统基础能力增强</h3><p data-vmark="2961">本地基础音视频播放能力、视频硬编解码、相机基础预览、拍摄能力。</p><p data-vmark="0515">RenderService 新渲染框架、2D / 3D 绘制能力、新的动画和显示效果框架。</p><p data-vmark="8d7f">窗口管理新框架，提供更加灵活的窗口框架能力，支持全屏、分屏、窗口化形态，支持跨窗口拖拽能力。</p><p data-vmark="fb87">display 管理能力，支持分屏，并增强亮灭屏管理能力；支持窗口动画和效果。</p><p data-vmark="b87e">鼠标、键盘、触摸板的基本功能支持，常见传感器加速度、陀螺仪、霍尔、马达振动等的基本能力支持。</p><p data-vmark="5131">语言区域选择、新增 / 增强国际化特性、系统资源、rawfile 资源。</p><p data-vmark="d5f9">时间时区同步的支持管理能力、新增对剪贴板、锁屏服务、静态壁纸、下载服务管理能力的支持。</p><p data-vmark="ad00">系统服务状态监控功能、新增跨设备 oncall 能力（跨设备启动 FA 能力）、长时、短时及延迟任务调度能力。</p><p data-vmark="24f1">内存管理功能增强、电源管理基本功能支持、增进程调度功能增强等特性。</p><p data-vmark="69c6">本地账号、域账号与本地账号绑定功能，分布式账号 ID 派生和状态管理功能，本地应用权限管理、分布式权限的管理能力。</p><p data-vmark="1bd8">Wi-Fi STA、AP、P2P 相关基本能力以及 JS API、新增蓝牙 BR、SPP、BLE 相关的能力以及 JS API、新增位置服务子系统，提供位置服务框架能力。</p><h3 data-vmark="e76b">标准系统分布式能力增强</h3><p data-vmark="9eab">分布式软总线网络切换组网、P2P 连接、流传输、蓝牙等能力。</p><p data-vmark="a9df">支持硬件互助 ，资源共享，支持镜像和扩展投屏等。</p><p data-vmark="5741">设备上下线和 PIN 码认证等增强功能，存储管理、应用沙箱隔离、公共数据沙箱隔离等能力，支持分布式数据库，分布式数据对象，支持本地数据库访问和跨应用数据库访问等能力。</p><h3 data-vmark="1ff3">标准系统应用程序框架能力增强</h3><p data-vmark="9d64">ArkUI 自定义绘制能力和 Lottie 动画能力、键盘、鼠标交互操作能力。</p><p data-vmark="96df">声明式 Web 组件、XComponent 组件能力。</p><p data-vmark="c2b1">卡片能力，提供卡片添加、删除、刷新等基础能力。</p><p data-vmark="8748">多用户能力，提供适应多端的基础 JS 工具链及运行时能力，对多 HAP 安装、隐式查询、多用户、权限管理的基本功能支持，支持分布式通知，通知模板功能。</p><h3 data-vmark="237f">标准系统应用能力增强</h3><p data-vmark="92e9">系统应用构建，包含系统桌面、SystemUI、系统设置、相机、图库、通话、联系人、信息、备忘录、文件选择器、输入法等应用支持。</p><h2 data-vmark="06b3">源码获取</h2><p data-vmark="89d1">IT之家了解到，开发者现可通过 repo + ssh Foundation）孵化及运营的开源项目，目标是面向全场景、全连接、全智能时代，搭建一个智能终端设备操作系统的框架和平台，促进万物互联产业的繁荣发展。</p><p data-vmark="15be">OpenHarmony 支持如下几种系统类型：</p><h3 data-vmark="27cc">轻量系统（mini system）</h3><p data-vmark="147b">面向 MCU data-vmark="99c5">面向应用处理器例如 Arm Cortex-A 的设备，支持的设备最小内存为 1MiB，可以提供更高的安全能力、标准的图形框架、视频编解码的多媒体能力。可支撑的产品如智能家居领域的 IP Camera、电子猫眼、路由器以及智慧出行域的行车记录仪等。</p><h3 data-vmark="9eff">标准系统（standard system）</h3><p data-vmark="2960">面向应用处理器例如 Arm Cortex-A 的设备，支持的设备最小内存为 128MiB，可以提供增强的交互能力、3D GPU 以及硬件合成能力、更多控件以及动效更丰富的图形能力、完整的应用框架。可支撑的产品如高端的冰箱显示屏。</p><blockquote><p data-vmark="9f7b">OpenHarmony 主库组织地址：<span
