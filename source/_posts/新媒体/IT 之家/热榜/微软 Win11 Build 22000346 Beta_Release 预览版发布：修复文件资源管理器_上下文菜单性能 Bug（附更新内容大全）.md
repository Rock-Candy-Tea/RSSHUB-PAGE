<p data-vmark="23ab"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 13 日消息，据 Windows 博客发布，微软正在为 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 进行另一次累积更新，这次更新解决了新操作系统中的几个错误 Bug。在 Beta 和 Release Preview 预览版频道中，微软正在测试 Windows 11 Build 22000.346（KB5007262），对文件资源管理器、新通知中心等进行修复。</p><p data-vmark="51b0"><img src="https://img.ithome.com/newsuploadfiles/2021/11/51c5bea6-6d6f-4859-ab84-a9b9e912977b.jpg" w="768" h="431" title="微软 Win11 Build 22000.346 Beta/Release 预览版发布：修复文件资源管理器/上下文菜单性能 Bug（附更新内容大全）" width="768" height="431" referrerpolicy="no-referrer"></p><p data-vmark="a901">正如我们在以前的报告中指出的那样，Windows 11 文件资源管理器在一些设备上存在着 Bug。例如，当打开上下文菜单时，文件资源管理器中的一个 Bug 会减慢性能。同样，一些用户报告说，当你在资源管理器窗口内，甚至在桌面上右键点击一个文件或文件夹时，上下文菜单反应会变得滞后。</p><p data-vmark="0d9e">在 Windows 11 中，微软已经改变了上下文菜单的设计，以解决长期以来的性能问题，并优先考虑常用的选项，如复制、剪切、重命名。不幸的是，此举导致了各种与上下文菜单相关的问题，而且似乎也影响了整个操作系统的可靠性。</p><p data-vmark="8bf1">值得庆幸的是，上下文菜单问题将在未来的版本中得到修复，微软也正在为文件资源管理器进行其他错误 Bug 修复。</p><p data-vmark="851d">Windows 11 Build 22000.346，现在面向 Beta/Release Preview 频道发布，最终修复了一个影响文件资源管理器和桌面上下文菜单中依赖上下文菜单的快捷标签应用的 Bug。当你选择使用单次点击来打开一个项目时，这个问题就会发生。</p><p data-vmark="ea5c">IT之家获悉，另一个导致文件资源管理器在关闭窗口后停止工作的错误 Bug 已经被修复。</p><p data-vmark="2775">除了文件资源管理器，Windows 11 Build 22000.346 还包含操作系统其他区域的错误 Bug 修复，包括设置应用中的 Windows Update 历史页面。</p><p data-vmark="5831">对于任务栏，只有一个错误 Bug 的修复，即当你悬停在任务栏的图标上时，会导致闪烁的问题。</p><p data-vmark="72b5"><strong>表情符号和通知中心的改进</strong></p><p data-vmark="7d0e">此外，微软现在引入了对 Emoji 13.1 的支持和更新的表情符号字典。</p><p data-vmark="b0a8">另一个错误 Bug 已经得到修复，即未读通知出现不正确或数字没有出现在通知区的圆心的问题。</p><p data-vmark="4198">这些错误修复将在 11 月晚些时候或 12 月初在生产频道发布。</p><h2 data-vmark="edb1">下面是 Windows 11 Build 22000.346 更新日志</h2><ul class="
