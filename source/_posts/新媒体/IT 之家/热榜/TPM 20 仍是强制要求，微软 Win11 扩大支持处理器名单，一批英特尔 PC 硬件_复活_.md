<p><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 8 月 28 日消息 微软今天更新了 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 的兼容性处理器要求，使得更多的处理器或 PC 硬件纳入到微软 Windows 11 的支持列表中。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/8/8760ff53-d845-4682-9166-6b3423c7daeb.png" w="1213" h="1104" title="TPM 2.0 仍是强制要求，微软 Win11 扩大支持处理器名单，一批英特尔 PC 硬件“复活”" width="1213" height="746" referrerpolicy="no-referrer"></p><p>微软表示，今年 6 月，我们听到了你们关于如何设定 Windows 11 最低系统要求的问题，并分享了更多关于指导设定这些要求的既定原则的信息。作为一个团队，我们承诺通过 Windows Insider 测试和与 OEM 厂商一起探索是否有运行在英特尔第七代和 AMD Zen 1 处理器上的设备符合这些原则的情况。</p><p>根据测试结果，微软正在对兼容处理器列表进行少量补充，但除此之外，将维持最初设定的最低系统要求。IT之家获悉，微软的结论是，<strong>所选择的兼容 64 位处理器、4GB 内存、64GB 存储空间、UEFI secure boot、图形要求和 TPM 2.0 是正确的最低系统要求，以实现我们建立的原则，为你提供最佳支持</strong>。</p><p>微软表示，发现了一组在英特尔第七代处理器上运行时符合这些原则的 PC 型号，但最初并没有将其纳入最低系统要求。根据这些发现，微软扩大了兼容的 64 位处理器的名单，包括以下内容：</p><ul class=" list-paddingleft-2"><li><p>Intel® Core™ X-series, Xeon® W-series</p></li><li><p>Intel® Core™ 7820HQ（只选择那些基于声明式、集成化、硬件支持应用（DCH）设计原则的现代驱动设备，包括 Surface Studio 2)</p></li></ul><p>微软添加到支持名单中的处理器具体型号有：</p><ul class="ai-word-checked list-paddingleft-2"><li><p>英特尔酷睿 i5-7640X</p></li><li><p>英特尔酷睿 i7-7740X</p></li><li><p>英特尔酷睿 i7-7800X</p></li><li><p>英特尔酷睿 i7-7820HQ</p></li><li><p>英特尔酷睿 i7-7820X</p></li><li><p>英特尔酷睿 i7-7900X</p></li><li><p>英特尔酷睿 i7-7920X</p></li><li><p>英特尔酷睿 i9-7940X</p></li><li><p>英特尔酷睿 i9-7960X</p></li><li><p>英特尔酷睿 i9-7980XE</p></li></ul><p>微软表示，他们将很快增加更多的英特尔至强 W 系列处理器。</p><p>对于运行英特尔酷睿 i7-7820HQ 处理器的设备，微软表示它们必须符合 DCH 驱动标准，否则将无法更新到 Windows 11。这与 Surface Studio 2 使用的处理器相同，现在可能会得到更新，但许多不遵循该标准的设备仍将被排除在外。</p><p>在与 AMD 合作仔细分析了 AMD Zen 第一代处理器后，微软和 AMD 表示<span class="accentTextColor">受 AMD 支持的 CPU 列表中不会添加任何内容</span>，也就意味着 AMD Ryzen 1000 系列将继续不被支持。微软将在未来几周内更新 PC 健康检查应用程序，以便在该工具普遍发布之前，用新增加的英特尔 CPU 识别正确的系统。此外，随着今年晚些时候 Windows 11 的全面上市，微软将有更多关于 IT 专业人员可以用来了解其组织的硬件资格的工具和报告的分享。</p><p><strong>扩展阅读：</strong></p><p>《<a href="https://www.ithome.com/0/570/399.htm" target="_blank">纯净安装，微软 Win11 Build 22000.132 预览版 ISO 官方镜像下载</a>》</p><p>《<a
