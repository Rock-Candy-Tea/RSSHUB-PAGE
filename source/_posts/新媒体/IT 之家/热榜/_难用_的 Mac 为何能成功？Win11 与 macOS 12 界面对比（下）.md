界面对比（下）" referrerpolicy="no-referrer"></p><p>遗憾的是，不是所有产品都有量化的参数，比如本文即将讨论的<span class="font-color-red">图形用户界面（GUI）</span>。然而，难以量化不代表无法比较。如果我们不愿讨论 GUI 的优劣，那人机交互就会止步不前，用户忍受着不合理的设计却<span class="font-color-red">不自知</span>。好在有关人类认知的研究切实地总结出一些有关界面设计的<span class="font-color-red">客观规律</span>，这便是本文讨论的重点。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_001353_655.jpeg?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xdy9xdzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>从上篇测评的热评来看，多数家友与我同时使用 PC、Macbook 后获得的感想一致：在系统交互方面，<span class="font-color-red">macOS 要更胜 Windows 一筹</span>，即使是 Win11 也不例外。苹果电脑的销量也从侧面支持着这一结论。macOS 为何能在 Windows 的垄断中不断发展？<span class="font-color-red">优秀的 GUI 功不可没。</span></p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_001419_176.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><h2>一、菲茨定律 Fitts's Law</h2><blockquote><p>用户将鼠标移向目标所需的<span class="accentTextColor">时间</span>，与目标的距离成<span class="accentTextColor">正比</span>，与目标区域的面积成<span class="accentTextColor">反比</span>。</p></blockquote><p>作为图形界面设计的基石之一，<span class="font-color-red">菲茨法则</span>指导了 Windows 与 macOS 的桌面构成，拿来开场再合适不过。该定律由美国心理学家 Paul Fitts 于 1954 年提出，是人类行为领域最成功的数学模型之一。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_001646_897.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>这个复杂的数学公式描述了一件可被称为常识的事：</p><ul class=" list-paddingleft-2"><li><p><span class="font-color-red">近处</span>的按钮比远处的按钮容易操作；</p></li><li><p><span class="font-color-red">较大</span>的按钮比较小的按钮容易操作。</p></li></ul><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005432_850.jpeg?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>想满足菲茨定律看似简单，但在实际场景中，按钮既不能<span class="font-color-red">太大</span>而占用页面空间，也不能<span class="font-color-red">随意改变位置</span>而扰乱用户习惯。这就需要设计者加入一些奇思妙想。举个例子：macOS Dock 栏中的图标会在鼠标悬停时<span class="font-color-red">放大</span>，因此比 Windows 任务栏中的图标<span class="font-color-red">更容易点按</span>。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_002600_270.gif" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 菲茨法则<span class="font-color-red">推论一</span> - 按钮的尺寸与间距要充足</p><p>除了按钮的大小，鼠标与按钮的<span class="font-color-red">距离</span>也能借由合理的设计尽可能缩短。比如：鼠标右键的弹出式菜单中，macOS 会在剪贴板中有数据时，将 “粘贴” 按钮<span class="font-color-red">置于顶部</span>。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005456_125.jpeg?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 菲茨法则<span class="font-color-red">推论二</span> - 按钮要设置在鼠标附近</p><p>对于这些细枝末节的设计，有人可能会说：“你就是在<span class="font-color-red">尬吹</span>！我使用时都没注意到过。” 但这绝非我的一家之言，Win11 便是践行者之一。全新的弹出式菜单提升了<span class="font-color-red">行高</span>，且在列表的首尾加入了<span class="font-color-red">快捷按钮</span>（复制、粘贴等）。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005513_290.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 菲茨法则在 Win11 中的应用<span class="font-color-red">之一</span></p><p>此外，全新的开始菜单将 “推荐的项目” 置于<span class="font-color-red">面板下方</span>，而用户点击开始菜单时，鼠标必然在<span class="font-color-red">任务栏处</span>，因此这一改动让鼠标距离常用的软件<span class="font-color-red">更近</span>。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005539_416.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 菲茨法则在 Win11 中的应用<span class="font-color-red">之二</span></p><p>除了按钮的大小与距离，菲茨法则还证明了一件有趣而重要的事。我们知道，将鼠标移向屏幕边缘时，屏幕中的鼠标指针将<span class="font-color-red">停留</span>在边缘处，但现实中的鼠标设备仍能<span class="font-color-red">不断向前</span>。这说明了两件事：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005600_301.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><ul class=" list-paddingleft-2"><li><p>屏幕边缘的交互面积可视为<span class="font-color-red">向外侧无限延伸</span>，故将鼠标移动到边缘的耗时极短；</p></li><li><p>屏幕角落更为特殊，它不仅有无限的面积，还有唯一的位置，是整块屏幕中<span class="font-color-red">最容易触及</span>的位置。</p></li></ul><p>这一推论决定了 macOS 与 Windows 都将优先级最高的 Dock 栏、任务栏固定于<span class="font-color-red">屏幕的边缘</span>，而非以悬浮窗的形式呈现。macOS 对屏幕边角的利用更是到达了极致。</p><p>进入 macOS 桌面，映入眼帘的除了底部的 Dock 栏，还有<span class="font-color-red">顶部的菜单栏</span>：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_002824_618.png" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>作为 macOS GUI 的常驻元素之一，菜单栏的<span class="font-color-red">最左侧</span>提供了 “关机、重启” 等系统指令，<span class="font-color-red">最右侧</span>提供了控制中心、通知中心两大实用功能，而<span class="font-color-red">中间</span>则是当前活跃 App 的菜单栏。</p><p>类比 Windows，它并未设计全局菜单栏，经典 Win32 程序会将其置于<span class="font-color-red">窗口顶部</span>。换言之，Win11 并未利用屏幕的顶边。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_002848_223.jpeg?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_3" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 用户将鼠标定位到窗口顶部的耗时<span class="font-color-red">大于</span>屏幕顶部</p><p>此外，macOS 拥有一项名为<span class="font-color-red">"触发角"</span>的功能<span class="font-color-orangered"></span>。当把鼠标移动到<span class="font-color-red">屏幕角落</span>时，macOS 会执行一项特定操作，比如 “显示桌面”“打开启动台” 等。用户可在设置中自行选择。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_004330_530.gif" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ macOS 将屏幕角落设计为快捷按钮</p><p>类比 Windows，任务栏的最右侧、屏幕的右下角有 “显示桌面” 按钮。Win11 中，由于开始菜单被移动到任务栏中央，屏幕的另外三角<span class="font-color-red">被闲置</span>。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_002942_785.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ Windows 忽视了屏幕的角落</p><p>回到 macOS Dock 栏，它若只是一个 App 快捷启动器，未免<span class="font-color-red">浪费了</span>屏幕底部的宝贵空间。它由三部分组成：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003005_240.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>除了用户自行置于 Dock 栏的 App 外，正在运行的 App 也会显示在 Dock 栏中，这与 Windows 的任务栏<span class="font-color-red">相同</span>。不同的是，最近使用的 App 会<span class="font-color-red">保留</span>在 Dock 栏中，而其最右侧能固定文稿、文件夹等<span class="font-color-red">非 App 的对象</span>。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_004641_584.gif" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 用户可拖拽<span class="font-color-red">文字</span>于 “邮件” 图标上以快速创建邮件</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_004403_364.gif" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ Dock 栏的<span class="font-color-red">文件夹</span>可直接展开，且支持文件拖拽</p><p>令人遗憾的是，Win11 的任务栏惨遭<span class="font-color-red">史诗级削弱</span>。先不说它不支持展开文件夹等功能，拖拽程序到任务栏以创建快捷方式这种基本操作也不行了。希望这只是微软尚未修复的 Bug。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003105_650.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 想固定程序，只能从菜单中选择</p><h2>二、希克定律 Hick’s Law</h2><blockquote><p>界面中的选项<span class="accentTextColor">越多</span>，用户的决策时间<span class="accentTextColor">越长</span>。</p></blockquote><p>聊完菲茨法则，相信各位对 GUI 的设计思路有了大致了解。我们再来看看另一个界面设计的基本原则：<span class="font-color-red">希克定律</span>。1952 年，来自英国、美国的心理学家 William Edmund Hick 和 Ray Hyman 通过统计刺激数量与<span class="font-color-red">人类反应时间</span>的关系，建立了以下数学模型：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005654_404.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>根据这项研究，其实任何人都有<span class="font-color-red">选择困难症</span>。即使你心中有明确的目标，过多的选项会分散大脑的注意力，进而让一个简单的操作<span class="font-color-red">耗时更长</span>。各位家友不妨看看下面两种页面，你喜欢哪一个？</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003212_960.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>操作系统中，最容易变乱的地方莫过于<span class="font-color-red">弹出式菜单</span>。先不论其中的系统内置功能你平时用不用得到，<span class="font-color-red">随着 App 的安装</span>，越来越多的无用功能被塞到菜单里。macOS 和 Win11 的方案很简单：把多余的选项放进<span class="font-color-red">二级菜单</span>。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003255_279.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>改变菜单布局并不难，减少<span class="font-color-red">软件</span>中的按钮就不容易了。简约不代表简陋。削减软件的功能也许能满足希克定律，但会让用户不得不花时间<span class="font-color-red">重新学习</span>新的使用逻辑。macOS 在这方面有<span class="font-color-red">先天优势</span>。以<span class="font-color-red">“访达”</span> 为例，这个苹果版资源管理器的界面十分简约：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003319_629.jpeg?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><ul class=" list-paddingleft-2"><li><p>窗口顶部的<span class="font-color-red">工具栏</span>提供了有限的常用功能</p></li><li><p>屏幕顶部的<span class="font-color-red">菜单栏</span>则提供了完整功能</p></li></ul><p>相比 Win10 的 Ribbon 功能区，Win11 资源管理器的顶栏也如 macOS 般仅提供一些<span class="font-color-red">常用功能</span>，可谓一目了然。但 Win11 并没有常驻菜单栏，砍掉 Ribbon 的同时，微软也<span class="font-color-red">砍掉了某些功能</span>，比如 “卸载程序”“设备管理” 等。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005718_852.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ “我可以不用，但你不能没有”</p><p>此外，macOS 的 “访达” 支持<span class="font-color-red">自定义工具栏</span>，毕竟常用功能对每个人而言都是不同的。希望 Win11 也能跟进这一特性。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003432_98.jpeg?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzEwMC5wbmc=,g_9" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 如果按钮太多，macOS 会根据窗口宽度自动隐藏一部分</p><h2>三、完形法则 Law of Prägnanz</h2><blockquote><p>人眼能从<span class="accentTextColor">复杂</span>的图像中分辨<span class="accentTextColor">简单</span>的图形，而简单的图形会比复杂的图像给人<span class="accentTextColor">更深的印象</span>。</p></blockquote><p>简约的设计之所以重要，不仅因为过多的选项会拖累用户的思考，<span class="font-color-red">App 窗口的形态本身</span>也决定着用户的注意力。1910 年，以匈牙利心理学家 Max Wertheimer 为代表的研究者组建了 “格式塔学派”，意在解释人类是如何处理<span class="font-color-red">视觉信息</span>的。举个例子：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003459_872.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>仔细观察 A～D 四幅图，你不难发现：三角形、圆柱形、球形与蛇形。这些图形<span class="font-color-red">并未出现</span>在图片中，而是大脑从视觉信息中提取出的轮廓让你产生了<span class="font-color-red">错觉</span>。</p><p>换言之，人类天生具备将眼前的事物归纳为<span class="font-color-red">基本形状</span>的能力，因此越是简约、规则的图形越易于记忆、理解。下面这两张地图，各位家友喜欢哪一张？</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003514_774.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 完形法则的内容有很多，此为其中之一</p><p>应用程序的<span class="font-color-red">窗口化</span>可以说是现代 GUI 的立身之本。典型的窗口一般有以下几个元素：</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_003537_214.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p><span class="font-color-red">标题栏</span>位于窗口顶部，<span class="font-color-red">菜单栏</span>位于标题栏下方，<span class="font-color-red">滚动条</span>位于窗口边缘，<span class="font-color-red">状态栏</span>位于窗口底部，它们的中央便是<span class="font-color-red">用户界面</span>。</p><p>虽说每个窗口元素都有各自的用处，但用户真正关注的是窗口中央的<span class="font-color-red">主界面</span>。周围的元素越多，用户的视觉体验就<span class="font-color-red">越割裂</span>。遵循完形法则，优秀的 GUI 设计应当将各个元素融合、简化。macOS 便是如此。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_004802_742.gif" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><ul class=" list-paddingleft-2"><li><p>标题栏与工具栏<span class="font-color-red">合并</span>为一行，菜单栏独立于窗口，置于屏幕顶部</p></li><li><p>顶栏与主界面间<span class="font-color-red">无分割线</span>，页面滚动后呈现遮盖效果</p></li></ul><p>以上设计提升了 macOS 窗口的<span class="font-color-red">整体感</span>，每个元素都成为了主界面的一部分。这不仅能提高屏幕的显示效率，也让用户更易抓住窗口的焦点。这种<span class="font-color-red">简化窗口</span>的设计趋势同样能在 Win11 中看到。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_004825_591.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 新版的 Edge 与 Safari 浏览器均添加了<span class="font-color-red">隐去标题栏</span>的显示模式</p><p>不过，Win11 的设计师们虽有简约的审美意识，但落实时<span class="font-color-red">不如</span> macOS 彻底。无论是新版的资源管理器，还是前卫的 UWP，它们的窗口乍看过去也是一体的，但标题栏、工具栏仍<span class="font-color-red">分别</span>占用着空间。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_004849_354.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 按钮间有不可交互的较大留白</p><p>此外，Win11 的设置中虽提供了 “隐藏滚动条” 的选项，但该设置仅在 UWP 中有效。这导致在页面无需滚动的场景下，滚动条依然<span class="font-color-red">占据</span>着窗口边缘。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005748_906.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 常驻的滚动条破坏了窗口的对称性</p><h2>四、雅各布法则 Jakob’s Law</h2><blockquote><p>用户使用某个程序的习惯必然受<span class="accentTextColor">其他程序</span>影响。</p></blockquote><p>聊完以上 macOS 与 Windows 的设计差异，有人可能会说：“你比来比去就只有桌面、文件夹这几个场景，有谁整天盯着桌面看？” 的确，优秀的 GUI <span class="font-color-red">单靠</span> Dock 栏、任务栏、菜单栏是支撑不起来的，<span class="font-color-red">应用程序</span>的表现才是关键。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005011_776.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>来自丹麦的人机交互学家 Jakob Nielsen 曾指出：优质的用户体验源于界面的<span class="font-color-red">一致性</span>。理想情况下，所有软件都共享一套 UI，于是用户只需<span class="font-color-red">学习一次</span>就能上手所有软件。而作为平台的提供者，苹果、微软显然得先<span class="font-color-red">统一</span>自己的设计语言。macOS 在这方面的表现令人满意。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005031_398.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 新应用代表：访达、Keynote、音乐</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005049_432.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 旧应用代表：系统报告、磁盘工具、字体册</p><p>不难看出，透明的<span class="font-color-red">边栏</span>和融合的<span class="font-color-red">顶栏</span>已是 macOS 应用的标配。菲茨法则、希克定律、完形法则融入了<span class="font-color-red">整个平台</span>，而不仅仅是个门面。</p><p>至于 Win11，我姑且乐观地认为细节处的不一致在未来会逐步修复，但微软<span class="font-color-red">随意改变设计语言</span>的做法令人生厌。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005818_363.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 三种不同的工具栏共存</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005836_607.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 三种不同的标题栏透明材质共存</p><p>至于带着自己的设计语言登陆 macOS 的第三方应用，macOS 的<span class="font-color-red">全局菜单栏</span>成为了 App 间的共通点。无论是使用 Ribbon 的 Office 全家桶，还是使用下拉列表的 Chrome，它们至少在菜单栏处与 macOS 原生应用一致。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005857_254.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 苹果强制开发者适配全局菜单栏</p><h2>五、美学 - 易用效应 Aesthetic-Usability Effect</h2><blockquote><p>用户总是将<span class="accentTextColor">美观</span>的设计视为<span class="accentTextColor">更实用</span>的设计。</p></blockquote><p>作为本文的压轴，请各位家友思考一件事：倘若有个用户，它从不使用微软、苹果的第一方软件，而全部采用跨平台的<span class="font-color-red">第三方软件</span>，那 macOS 与 Windows 对它而言<span class="font-color-red">有区别吗</span>？</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_005947_157.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>跨平台的软件通常会采用<span class="font-color-red">相似的界面</span>以减少开发成本和用户学习成本，此时操作系统的类别似乎无关紧要。不过在 1995 年，两名日本研究员 Masaaki Kurosu 和 Kaori Kashimura 在研究 ATM 机的界面时发现，对于功能完全相同的两个程序，用户会认为 <span class="font-color-red">UI 美观</span>的那个功能更强大。同样的事也在 macOS、Windows 间上演。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_010003_372.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>▲ 考虑到腾讯的程序员偏爱苹果，我们再看看友商微软的 ‍<span class="font-color-red">Outlook</span>。Win11 中，Outlook 终于用上了 Fluent Design，但云母材质并未实装，纯白的底色略显单调。令人费解的是，Outlook for macOS 先行一步用上了满血的 Fluent UI。不愧是‍<span class="font-color-red">苹果最佳开发者</span>！</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_010046_238.jpeg" title="“难用”的 Mac 为何能成功？Win11 与 macOS 12 界面对比（下）" referrerpolicy="no-referrer"></p><p>再看看专业软件 <span class="font-color-red">Prism 8</span>。也许是 macOS 的原生 API 很好适配，如此一个功能大于美观的软件在 macOS 端使用了毛玻璃、云母等视觉效果，而在 Win11 中几乎没有装饰性元素。值得一提的是，弹出式菜单的实时模糊效果在 macOS 中也是<span class="font-color-red">全局</span>的，你能点出的菜单都是相同的样式。</p><h2>结语</h2><p>不知有多少家友看到了最后？一口气阅读这么多内容，<span class="font-color-red">真是辛苦各位了</span>。综合来看，macOS 与 Windows 到底谁更好，我无法评价；不过 macOS 始终会是<span class="font-color-red">更漂亮</span>的那一个吧！功能与审美并重，易用与好用兼备，这才是消费者想要的操作系统。</p><p><img src="https://img.ithome.com/mpimg/content/56182154/2021/8/20210802_010132_618.gif" title="“难用”的 Mac 为何能成功？Win11 与 macOS
