<p data-vmark="86ab"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 21 日消息，半个月前，微软悄悄从 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows </a>应用商店中删除了 WSATools 工具 —— 这是为 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> <a class="s_tag" href="https://android.ithome.com/" target="_blank">安卓</a>子系统（WSA）提供 APK 应用本地安装的程序，但微软本身没有提供任何明确的原因。</p><p data-vmark="cc5b"><img src="https://img.ithome.com/newsuploadfiles/2021/11/f23e2d3c-a59b-46ee-906a-1cc612b609f4.png" w="808" h="555" title="Win11 安卓子系统本地安装 APK 应用回归应用商店，微软为下架一事向开发者致歉" width="808" height="555" referrerpolicy="no-referrer"></p><p data-vmark="05f1">经过一些争议后，微软似乎已经努力与 WSATools 开发者 @Franco 解决了这个问题。外媒发现目前该应用已经重新上架了 Windows 官方的微软商店。此外，据开发者所说，微软之前还特意写了一封邮件向他道歉，并说明了下架的原因。</p><p data-vmark="a456"><img src="https://img.ithome.com/newsuploadfiles/2021/11/8a208a2d-c5e7-47a3-896b-a9a2afb23a8f.png" w="600" h="554" title="Win11 安卓子系统本地安装 APK 应用回归应用商店，微软为下架一事向开发者致歉" width="600" height="554" referrerpolicy="no-referrer"></p><p data-vmark="7575">IT之家了解到，Windows 11 安卓子系统 (Windows Subsystem for <a class="s_tag" href="https://android.ithome.com/" target="_blank">Android</a>) 包括 Linux 内核和基于 Android 开源项目（AOSP）版本 11 的 Android 操作系统。该子系统在 Hyper-V 虚拟机中运行，就像 Linux 子系统一样，可以将 AOSP 环境中 App 的运行时和 API 映射到 Windows 图形层、内存缓冲区、输入模式、物理和虚拟设备以及传感器，可以在英特尔、AMD、高通的 CPU 上运行。</p><p data-vmark="8b34"><img src="https://img.ithome.com/newsuploadfiles/2021/11/a180b592-3467-4bcf-b6d6-ea67b7824227.png" w="1440" h="494"
