<p data-vmark="5727"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 7 日消息，微软和英特尔近日<a href="https://devblogs.microsoft.com/commandline/oneapi-l0-openvino-and-opencl-coming-to-the-windows-subsystem-for-linux-for-intel-gpus/" target="_blank">宣布</a>，<span class="accentTextColor">Windows Linux 子系统（WSL2）已支持英特尔的图形硬件加速 oneAPI Level Zero 和 Inlel OpenCL</span>。</p><p data-vmark="a9c9"><img src="https://img.ithome.com/newsuploadfiles/2021/11/a889e888-71e5-437e-bccc-cdc86e47194d.png" w="1440" h="423" alt="092541_VQT1_5430600.png" title="微软 Windows Linux 子系统（WSL2）已支持英特尔图形硬件加速" width="1440" height="241" referrerpolicy="no-referrer"></p><p data-vmark="7d31">正如英伟达将 CUDA 及其加速 GPU 支持引入 WSL2 一样，英特尔的硬件现在也可以通过 WSL2 来进行专业的计算工作。IT之家了解到，使用最新的更新和驱动程序后，就可以接近原生 Linux 系统的性能水平。</p><p data-vmark="8c52"><img src="https://img.ithome.com/newsuploadfiles/2021/11/0b4cd7c6-5554-453c-8979-9ee9d4e5aaa6.jpg@s_2,w_820,h_461" w="1440" h="810" alt="Integrated GPU on the same die as your CPU" title="微软 Windows Linux 子系统（WSL2）已支持英特尔图形硬件加速" srcset="https://img.ithome.com/newsuploadfiles/2021/11/0b4cd7c6-5554-453c-8979-9ee9d4e5aaa6.jpg 2x" width="1440" height="461" referrerpolicy="no-referrer"></p><p data-vmark="9293">值得注意的是，微软让英特尔的开源 Compute-Runtime 在微软的开源但不是主线内核驱动程序上工作。这是参考 DirectX“dxgkrnl”驱动程序，该驱动程序去年发布，但被主流化的前景渺茫，很大程度上是由于缺乏开源的客户端。现在有了 Intel 的 Compute-Runtime 代码，微软有可能尝试将他们的 DXGKRNL 驱动程序合并到主线。</p><p data-vmark="e3e3"><strong>详细的使用步骤：</strong><a href="https://www.intel.com/content/www/us/en/artificial-intelligence/harness-the-power-of-intel-igpu-on-your-machine.html" target="_blank">点此查看</a></p>
