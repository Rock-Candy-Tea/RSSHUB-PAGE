<p data-vmark="4756"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 4 月 13 日消息，Windows 更新和防病毒解决方案偶尔会导致一些应用程序被误杀，也有可能会产生潜在兼容性问题，进而导致崩溃。</p><p data-vmark="4717">虽然微软最近的 Windows 更新比较稳定可靠，很少有重大问题，但外媒发现微软在 2022 年 4 月发布的 <a class="s_tag" href="https://win10.ithome.com/" target="_blank">Windows 10</a> 和 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 更新可能会导致 Chrome, Edge 和 Firefox 崩溃报错，如“应用程序无法正确启动 (0xc0000022)。单击确认关闭”。目前还不知道微软是否意识到这个新问题。</p><p data-vmark="b6ad">根据各大论坛和微软自己的反馈中心的用户报告，有部分人在尝试启动 Chrome.exe、msedge.exe 和 Firefox.exe 时，会出现 0xc0000022 报错。</p><p data-vmark="19ab">“在安装 KB5012599 后，我就出现了 0xc0000022 错误启动，所以 Edge, Firefox 和 Chrome 不再工作。幸运的是，Vivaldi 和 Brave 依然正常运行，所以我仍然可以上网，而且也能证实卸载 KB5012599 即可解决这个问题。”</p><p data-vmark="c4d3"><img src="https://img.ithome.com/newsuploadfiles/2022/4/04fe9a1f-f2ac-434c-a198-b0cd22260d8c.jpg" w="698" h="297" alt="0xc0000022 error" title="安装 Win11/10 四月更新后，谷歌 Chrome、微软 Edge 浏览器发生 0xc0000022 崩溃" width="698" height="297" referrerpolicy="no-referrer"></p><h2 data-vmark="c247">0 xc0000022 错误</h2><p data-vmark="77ce">另一位用户表示，“安装最近的累积更新 KB5012592 后，所有搜索引擎（Edge、Firefox 等）的反应都是错误代码 0xc0000022”。</p><p data-vmark="96e3">那么，该如何在 Windows 10 和 Windows 11 中修复 0xc0000022 问题呢？很简单，卸载更新。</p><p data-vmark="a821">报告显示，这个问题是由于最新的 Windows 10 和 Windows 11 累积更新引起的。然而，我们的调查表明，Windows 10/11 更新和 ESET 等不太知名的杀毒软件之间的兼容性问题可能导致应用程序崩溃。</p><p data-vmark="bb76">此外，这个问题似乎已经影响到了一小部分运行 Windows 10/11 和 ESET 等应用程序的人。该安全公司发表回应称，这些报告已经传递给团队，他们已经部署了一个潜在的修复方案，但修复可能仍然需要用户的手动操作。</p><p data-vmark="ec0c">这家安全公司在一份新文档中表示，目前有部分用户在更新 Windows 后出现了“浏览器无法安全启动”或“应用程序无法正常启动”的现象。</p><p data-vmark="1071">“对于给我们的客户带来的任何不便，我们深表歉意，”该公司补充并强调了 0xc0000022 的两个变通办法:</p><ul class=" list-paddingleft-2"><li><p data-vmark="1872">从设置启用 ESET LiveGrid。</p></li><li><p data-vmark="62fb">禁用安全浏览器</p></li></ul><p data-vmark="d044">总的来说， 这不失为一个还不错的简单方法，对于大部分 Windows 10 和 Windows 11 用户来说操作起来并不难。因此，如果你已经安装了 4 月更新并无法使用浏览器，那么这个小窍门应该可以帮到你。</p><p data-vmark="3c70">目前还不清楚这个 bug 是否影响到其他反病毒解决方案。不过，如果你的电脑上出现了上述错误信息而且很难通过上面的方法解决，那么你也可以卸载累积更新或禁用浏览器的安全扩展。</p>
