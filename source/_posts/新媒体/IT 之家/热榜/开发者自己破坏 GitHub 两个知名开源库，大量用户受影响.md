<p data-vmark="7ba2"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 1 月 10 日消息，根据外媒 theverge 报道，知名开源工具 Faker.js 的作者 Marak 近日故意破坏了 GitHub 上的这个开源库，<span class="accentTextColor">已经将项目所有代码清空，并在 commit 中留下“endgame”字样</span>，还在 README 中写道“What really happened with Aaron Swartz?”。</p><p data-vmark="364c"><img src="https://img.ithome.com/newsuploadfiles/2022/1/1d85dead-1218-4832-9482-40beda90d03a.png" w="1313" h="791" title="开发者自己破坏 GitHub 两个知名开源库，大量用户受影响" width="1313" height="494" referrerpolicy="no-referrer"></p><p data-vmark="96d9">Faker.js 是一个十分流行的 Node.js 工具库，用于开发调试，可以在<a class="s_tag" href="https://qiyu.ruanmei.com/" target="_blank">浏览器</a>和 node.js 中生成大量的虚假数据进行模拟测试。目前该项目每周下载量接近 250 万。Marak 投入了大量精力来开发这个项目，许多商业公司深度使用这一项目，然而开发者从中获得的收益十分少。Marak 曾在 2020 年 11 月发文，要求使用 Faker.js 的公司向他支付费用，否则他将不再提供维护工作。</p><p data-vmark="4fa6">他开发的另一个库 colors.js 也受到了影响，但是目前状态正常，没有被清空。这个库可以为 javascript 控制台添加颜色，每周下载量可达 2240 万。</p><p data-vmark="9605"><img src="https://img.ithome.com/newsuploadfiles/2022/1/ba075de9-4654-4857-8cd2-ab4369e644ac.png" w="1316" h="1174" title="开发者自己破坏 GitHub 两个知名开源库，大量用户受影响" width="1316" height="732" referrerpolicy="no-referrer"></p><p data-vmark="2d6f">海外论坛 reddit 上的一个帖子中有人写到，Marak 清空 Faker.js 仓库的原因是个人遇到了经济困难，并且开源项目被别人滥用。虽然代码看似被清空，但是有人发现真正的 Faker.js 仓库被他隐藏了起来，设置为私有。目前，<span class="accentTextColor">Faker.js 上一个版本依旧可以下载</span>，版本号为 6.6.6（在西方 666 是代表恶魔的邪恶数字）。</p><p data-vmark="8370"><img src="https://img.ithome.com/newsuploadfiles/2022/1/7644d675-d895-4b8f-a9cb-381aebd6770b.png" w="978" h="1272" title="开发者自己破坏 GitHub 两个知名开源库，大量用户受影响" width="978" height="1067" referrerpolicy="no-referrer"></p><p data-vmark="a0d0">早在 2020 年 10 月，Marak 的公寓失火，他本人因此失去了所有家当，接近身无分文。为此，他不得不在网上公布他的 PayPal 账号，请求网友们给他捐助。</p><p data-vmark="3ef2">IT之家获悉，Marak 在最新的推文中表示，NPM 已经恢复到之前版本的 faker.js 包。目前他已经暂停 GitHub 账号。</p><p data-vmark="6fa6"><img src="https://img.ithome.com/newsuploadfiles/2022/1/74111a00-417a-4b2c-a31d-06b2819806f1.png" w="648" h="725" title="开发者自己破坏 GitHub 两个知名开源库，大量用户受影响" width="648" height="725" referrerpolicy="no-referrer"></p>
