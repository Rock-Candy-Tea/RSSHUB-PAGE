            <p data-vmark="20ee"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 6 月 9 日消息，微软 GitHub 宣布，将停用 Atom，并将于 2022 年 12 月 15 日归档该组织下的所有项目。</p><p data-vmark="3c87" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/6/4798ca56-564d-4393-8417-ed67ecbbd5e4.png" w="1158" h="953" title="一个时代结束！微软 GitHub 宣布代码编辑器 Atom 将停用，可使用 Visual Studio Code" width="1158" height="675" referrerpolicy="no-referrer"></p><p data-vmark="e9f6">下面是 GitHub 博客内容：</p><blockquote><p data-vmark="3aa9">“当我们在 2011 年推出 Atom 时，我们着手为开发人员提供可深度定制但也易于使用的文本编辑器，让更多人能够构建软件。虽然发展软件创建者社区的目标仍然存在，但我们决定让 Atom 退役，以进一步履行我们通过微软 Visual Studio Code 和 GitHub Codespaces 将快速可靠的软件开发带到云端的承诺。</p><p data-vmark="0187">今天，我们宣布将停用 Atom，并将于 2022 年 12 月 15 日归档该组织下的所有项目。</p><p data-vmark="4e61"><strong>为什么这样做？</strong></p><p data-vmark="b875">在过去的几年里，Atom 没有重大的功能开发，尽管我们在此期间进行了维护和安全更新，以确保我们成为项目和产品的好管家。多年来，随着新的基于云的工具出现和发展，Atom 社区的参与度显著下降。因此，我们决定停用 Atom，以便我们可以专注于使用 GitHub Codespaces 增强云中的开发人员体验。</p><p data-vmark="742b">这是一个艰难的告别。值得一提的是，<span class="accentTextColor">Atom 是 Electron 框架基础，它为创建数千个应用程序铺平了道路，包括微软 Visual Studio Code、Slack 和 GitHub Desktop。</span>但是，可靠性、安全性和性能是 GitHub 的核心，为了最好地服务于开发者社区，我们将 Atom 归档以优先考虑支持软件开发未来的技术。</p><p data-vmark="7393"><strong>接下来发生什么？</strong></p><p data-vmark="94f1">我们认识到社区仍在使用 Atom，并希望承认迁移到替代解决方案需要时间和精力。我们致力于帮助用户和贡献者规划他们的迁移。</p><p data-vmark="ba18">今天，我们宣布六个月后的退役日期。</p><p data-vmark="40f5">在接下来的六个月中，我们将继续在产品和 atom.io 上通知用户 Atom 退役 。</p><p data-vmark="f248">2022 年 12 月 15 日，我们将归档 atom / atom 存储库以及 Atom 组织中剩余的所有其他存储库。</p><p data-vmark="1265"><strong>感谢</strong></p><p data-vmark="944b">GitHub 和我们的社区从提交问题、创建扩展、修复错误 Bug 和在 Atom 上构建新功能的人中受益匪浅。Atom 在许多开发人员的旅程中扮演着不可或缺的角色，我们期待着共同构建和塑造软件开发的下一个篇章。”</p></blockquote>
