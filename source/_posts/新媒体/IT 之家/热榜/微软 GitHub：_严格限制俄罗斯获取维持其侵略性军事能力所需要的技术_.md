            <p data-vmark="7163"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 3 月 4 日消息，近期，微软旗下 GitHub 官方表达了针对俄罗斯乌克兰战争的态度。</p><p style="text-align: center;" data-vmark="1ef4"><img src="https://img.ithome.com/newsuploadfiles/2022/3/34d35fc0-99f8-4a66-b74a-07cb0174b3f4.png" w="737" h="657" title="微软 GitHub：“严格限制俄罗斯获取维持其侵略性军事能力所需要的技术”" width="737" height="657" referrerpolicy="no-referrer"></p><p data-vmark="1ef1">GitHub 官方博客称，作为开源之家以及所有开发者的家园，我们认真对待我们的角色，保护开放合作和相互联系社区中的信息自由流动。这包括保持平台的开放性和对所有开发者的可用性，无论他们居住在哪里。GitHub 副总裁 Thomas Dohmke 分享了 GitHub 作为一个公司、一个平台和一个全球 Hubbers 社区如何应对的最新情况。</p><p data-vmark="c0c0">GitHub 副总裁 Thomas Dohmke 称，GitHub 是沟通、人道主义工作和组织变革的强大工具，也是难以置信的技术进步。我们真正相信开源的力量，所以将继续努力工作，以保持平台对全世界所有开发者的可用性和安全性。在努力确保所有国家的开发者都能使用 GitHub 的同时，我们也在继续确保所有人都能获得免费的开源服务，包括俄罗斯的开发者。</p><p data-vmark="ae25">Thomas Dohmke 称，我们还致力于提供强大的安全功能，以防止 GitHub 用户及其账户受到损害，敦促开发者设置 2FA，最好是使用 WebAuthn，以保护他们的账户。</p><p data-vmark="b046">Thomas Dohmke 称，同时，我们正在采取行动支持平台，并遵守可能在这场战争中需要的许多政府强制规定。我们的法律团队彻底审查这些任务，正在遵守出口管制和贸易法规，因为它们在不断发展。这包括实施严格的新出口管制，旨在严格限制俄罗斯获得其维持侵略性军事能力所需的技术和其他物品。此外，我们处理的任何政府突检通知都会公开发布，因为我们相信透明度对良好的治理至关重要。</p><p data-vmark="26e7">Thomas Dohmke 称，社区和产品运营团队正在监控论坛中的对话，并做出评估，督促遵守社区论坛和公共反馈的行为准则。我们将锁定违反这些行为准则的讨论。</p>
