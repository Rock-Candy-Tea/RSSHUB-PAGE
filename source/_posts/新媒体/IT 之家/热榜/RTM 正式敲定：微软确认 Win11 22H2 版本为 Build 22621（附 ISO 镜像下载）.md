<p data-vmark="3e81"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 5 月 25 日消息，Windows 11 第一个大版本更新现已敲定。@Michael Reinders 发现了一份微软刚发布的<a href="https://techcommunity.microsoft.com/t5/windows-hardware-certification/windows-hardware-compatibility-program-for-windows-11-version/ba-p/3423905?utm_source=dlvr.it&utm_medium=twitter" target="_blank">官方文件</a>，硬件认证页面中提到 Windows 11 22H2 版本的 RTM 构建即为 22621（最低版本号）。</p><p data-vmark="48b1">随着 Windows 11 的更新，22H2 版本将会在今年秋季正式发布，然后慢慢出现在所有新机和已安装 Win11 的用户 PC 上。此外，微软还公开了一些非常具体的细节，这似乎是微软首次公开承认 RTM 细节。</p><p data-vmark="434e">该公司写道：</p><blockquote><p data-vmark="bc59">Windows 11, version 22H2 certification build</p><p data-vmark="1948">Minimum acceptable build for Windows 11, version 22H2 certification is RTM build(i.e. 22621).</p></blockquote><p data-vmark="4cc6" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/5/dceccf29-38fd-4447-bd61-9ffc0182a320.png" w="1440" h="997" title="RTM 正式敲定：微软确认 Win11 22H2 版本为 Build 22621（附 ISO 镜像下载）" width="1440" height="568" referrerpolicy="no-referrer"></p><p data-vmark="446c">从微软的话术来看，Windows 11 中似乎不再存在某个特定的版本作为 RTM，只要满足最低版本号为 Build 22621 的系统都可以作为最终版本交付。Windows 11 版本 22H2 后续也将会在新 PC 上或面向公众发布之前以及之后不断进行更新。</p><blockquote><p data-vmark="5036">科普：Windows 操作系统在零售商店正式上架前需要一段时间来压片，包装和配销。所以程序代码必须在正式发行前一段时间就要完成，这个完成的程序代码叫做 Final.Code。</p><p data-vmark="2fd3">RTM 即 Release to Manufacturing 或者 Release to Marketing，即交付给 OEM 和市场的版本。</p></blockquote><p data-vmark="3c23">大约两周前，微软首次面向 Beta 频道推送了 Windows 11 build 22621 版本 。当时，微软指出“桌面右下角的版本水印已经不再存在，但水印后续将会在新版本中重新出现”，这显然它确实意味着它已经完成了 22H2 版本的选择，而且还有爆料称微软可能会在今天向 OEM 交付 RTM 版本。</p><p data-vmark="c935">现在来看，微软确实选定了 22H2 版本，但微软并非公开进行。</p><p data-vmark="0e85">IT之家了解到，Windows 11 Build 22621 早在 5 月 11 日就发布到了 Beta 频道，以及一个 SDK。此外，微软那时还提供了 ISO
