list-paddingleft-2"><li><p>修复了一个问题，如果用户在任务栏上拖动应用图标来重新排列，会在用户释放图标时启动或最小化应用。</p></li><li><p>使用长按并触摸任务栏中的应用图标来打开跳转列表现在应该可以工作了。</p></li><li><p>右键单击任务栏中的“开始”图标后，单击其他位置现在应该能更可靠地关闭菜单。</p></li><li><p>Shift + 右键单击任务栏中的应用图标现在将像以前一样调出窗口菜单，而不是跳转列表。</p></li><li><p>解决了鼠标悬停在任务栏预览上时移动缓慢的问题。</p></li><li><p>修复了使用多个桌面时出现的问题，其中任务栏中的应用图标可能会显示多个窗口打开，而该桌面上并非如此。</p></li><li><p>使用阿姆哈拉语 IME 时，用户不会再在任务栏中的 IME 图标旁边看到意外的 X。</p></li><li><p>如果用户单击任务栏上的输入指示器，会突出显示快速设置的问题已得到修复。</p></li><li><p>当用户将鼠标悬停在任务视图上时，使用 Esc 关闭桌面后，桌面的预览弹出窗口将不再弹出。</p></li><li><p>修复了以下问题：将鼠标悬停在任务栏中的任务视图图标上后 explorer.exe 在“设置”中启用日历时，在弹出窗口中看不到日历文本的情况。</p></li><li><p>解决了一个可能意外使任务栏背景透明的问题。</p></li><li><p>右键单击任务栏中的焦点辅助图标，现在应显示上下文菜单。</p></li><li><p>之前版本中任务栏角落的图标被压在任务栏顶部的问题已得到解决。</p></li><li><p>任务栏中正在使用的位置图标工具提示应该不再是空白的。</p></li></ul><p><strong>设置</strong></p><ul class=" PIN 后，“我忘记了 PIN”链接在登录设置中意外丢失的问题。</p></li><li><p>解决了“设置”中“应用程序和功能”下的“移动”选项无法可靠工作的问题。</p></li><li><p>解决了一个问题，即设置中的部分颜色在深色和浅色模式之间切换后没有更新，从而留下不可读的文本。</p></li><li><p>提高在明暗模式之间切换时设置的性能。</p></li><li><p>解决了一个问题，即当窗口尺寸较小时，“设置”中“主题”页面的部分元素最终会挤在一起。</p></li><li><p>解决了任务栏设置下的笔菜单切换与功能的实际状态不同步的问题。</p></li><li><p>对辅助功能设置中的“在此时间后关闭通知”所做的更改现在应该保持不变。</p></li><li><p>修复了用户在任务栏设置中启用的部分图标被错误标记为 Windows 资源管理器问题。</p></li><li><p>快速设置中的连接文本已更新为投射。</p></li></ul><p><strong>文件管理器</strong></p><ul class="ai-word-checked class=" class="ai-word-checked list-paddingleft-2"><li><p>将 Outlook 客户端与 Microsoft 帐户一起使用时，日历和待办事项更新应该更快地同步到小部件。</p></li><li><p>解决了一个问题，如果用户从小部件设置中快速添加多个小部件，可能会导致部分小部件不可见。</p></li><li><p>修复了一个错误，即小部件可能都卡在加载状态（窗口中的空白方块）。</p></li><li><p>交通小部件现在应该遵循 Windows 模式（亮或暗）。</p></li><li><p>体育小部件的标题不应再与小部件的内容不匹配。</p></li></ul><p><strong>其他</strong></p><ul class=" list-paddingleft-2"><li><p>解决了在用户释放键后，有时 ALT + Tab Tab，然后才能使用箭头键浏览对齐布局。</p></li><li><p>解决了在反复用触摸键弹出窗口后，亚克力区域可能会留在屏幕上的问题。</p></li><li><p>微软做了一些工作，来减轻通过触摸移动捕捉窗口时意外的闪光。</p></li><li><p>当“在标题栏和窗口边框上显示强调色”被关闭时，微软进行了更改，以帮助窗口边框具有更多对比度。</p></li></ul>
