<p data-vmark="44b0">一提到“网管”，很多人就会想到网吧。网吧里，那个专门负责维护电脑的人，就叫做“网管”。</p><p data-vmark="c4a3"><img src="https://img.ithome.com/newsuploadfiles/2021/12/f86f9bad-9750-43f1-b6e5-571d27c6d329.jpg" w="800" h="533" alt="网吧的“网管”" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="800" height="533" referrerpolicy="no-referrer"></p><p data-vmark="dd62">▲ 网吧的“网管”</p><p data-vmark="a3b8">也有人会想到公司里的系统管理员，也就是那个每当你的电脑或网络出现问题，都会出来救火的人。他们，有时候也会被称为“网管”。</p><p data-vmark="12ab"><img src="https://img.ithome.com/newsuploadfiles/2021/12/696ecf3f-887b-4bc9-960b-c5b6fb22aa61.jpg" w="500" h="304" alt="系统管理员" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="500" height="304" referrerpolicy="no-referrer"></p><p data-vmark="4e0f">在通信行业，“网管”是一个常用词，通信人几乎每天都会用到。那么，通信行业里的“网管”，和前面所说的“网管”，是同一个意思吗？</p><p data-vmark="938c">当然不是！</p><p data-vmark="17ee">通信行业里的“网管”，和网吧或企业的“网管”，完全是两回事。</p><p data-vmark="de30">通信里的“网管”，虽然全称也叫“网络管理”，但实际上，<span class="accentTextColor">特指通信网络中用于管理设备的软件、硬件及平台（主要是软件）</span>。</p><p data-vmark="c8cf">而通信里的“网管工程师”，一般是指开发网管软硬件的研发、测试工程师。</p><p data-vmark="531e">真正使用网管软件，进行设备调测、维护的工程师，叫做技术支持工程师，或者运维工程师。</p><p data-vmark="2e3a">网管工程师，是通信行业最容易被人误解的职位。</p><p data-vmark="d7f1">公司 HR 小姐姐去招聘的时候，最怕的就是招聘“网管工程师”。每次都要费老半天的劲，和对方解释，这个网管不是那个打杂、拉网线、装系统的网管。</p><p data-vmark="e1d3">做网管的同事更悲催。每次亲朋好友问：“你主要是做什么工作的呀？”如果直接回答：“网管”，气氛瞬间凝固，场面一度尴尬到无法呼吸。</p><p data-vmark="8150"><img src="https://img.ithome.com/newsuploadfiles/2021/12/1d176025-f644-484c-b4cb-1d42d2aea691.jpg" w="640" h="640" alt title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="640" height="640" referrerpolicy="no-referrer"></p><p data-vmark="5382">后来，同事们干脆放弃了“网管”这个词，直接回答“软件开发工程师”。</p><p data-vmark="2b2c">接下来说说网管的具体知识。</p><p data-vmark="bb5d">网管的诞生时间很早。六七十年代电子交换机出现的时候，网管也同步出现了。</p><p data-vmark="439e">网管到底管些啥呢？</p><p data-vmark="e2dc">简单来说，网管系统主要的功能包括五大块，分别是<span class="accentTextColor">配置管理、故障管理、性能管理、安全管理、帐户管理</span>。</p><p data-vmark="3e17">说白了，就是配置设备的数据，修改参数，查看告警，查看性能指标，之类的。</p><p data-vmark="49a1">再通俗一点说，网管就是你和设备之间的交互界面。你要控制设备，就离不开网管。</p><p data-vmark="b6fd"><img src="https://img.ithome.com/newsuploadfiles/2021/12/a8024996-b05d-4ace-88ff-40208c957a3b.png" w="1080" h="350" alt="工程师——网管——设备" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="1080" height="266" referrerpolicy="no-referrer"></p><p data-vmark="f0b2">最近十几年，网管的变化很大。</p><p data-vmark="bf12">以前，很多网管都是安装在特殊的客户端设备上，通过专有线缆，连接通信设备。</p><p data-vmark="0519">后来，专有的客户端设备逐渐被普通 PC 所取代，专有线缆也逐渐改成了普通的 TCP / IP 以太网。一对一，也变成了一对多（一个服务端，N 个客户端）。</p><p data-vmark="0d59"><img src="https://img.ithome.com/newsuploadfiles/2021/12/6c70cd50-98fd-4b16-b295-ef4ddcad5b0b.png" w="1080" h="466" alt="一个服务端，N 个客户端" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="1080" height="354" referrerpolicy="no-referrer"></p><p data-vmark="fed4">网管的软件平台，也逐渐变化。</p><p data-vmark="5e0b">以前，很多设备都是 GUI（图形用户界面）模式，需要找一台台式机，安装专门的客户端软件。然后启动软件，进行登录，从而打开网管软件界面。</p><p data-vmark="c4e6"><img src="https://img.ithome.com/newsuploadfiles/2021/12/64961586-8d6c-4a86-a88d-ed188699a9b9.png" w="688" h="449" alt="GUI 界面" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="688" height="449" referrerpolicy="no-referrer"></p><p data-vmark="4198">▲ GUI 界面</p><p data-vmark="a94c">现在，WEB 模式更为流行。任何一台台式机，只要网络和服务端相连，都可以通过 WEB <a class="s_tag" href="https://qiyu.ruanmei.com/" target="_blank">浏览器</a>，就像登录网站一样，打开网管界面。</p><p data-vmark="6e8f"><img src="https://img.ithome.com/newsuploadfiles/2021/12/4939545b-6c7f-4c6f-8dc8-296c0ccfd448.png" w="504" h="301" alt="WEB 界面" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="504" height="301" referrerpolicy="no-referrer"></p><p data-vmark="db45">▲ WEB 界面</p><p data-vmark="86d1">基于 C / S 架构（Client / Server，客户端 / 服务器）的网管，越来越多地向 B / S 架构（Browser / Server，浏览器 / 服务器）演进。</p><p data-vmark="5dff">值得一提的是，<span class="accentTextColor">网管还有一种模式，叫做 MML</span>，也就是咱们通信人戏称的“慢慢来”（三个字的声母）。</p><p data-vmark="ff2c">MML 其实就是命令行模式，和 DOS、LINUX 一样，敲命令，然后与设备进行交互。这个比较适合批量导入脚本。</p><p data-vmark="4057">网管有几个比较常见的单词，分别是 EMS、NMS、UNMS。</p><p data-vmark="22ed">EMS 不是快递的那个 EMS，而是 Element Management System，网元管理系统。</p><p data-vmark="a599">EMS 一般只管本地网元。现在更多见的是 NMS，Network Management System，网络管理系统。</p><p data-vmark="3980">其实比较好理解，以前是只管理单个网元。后来，网管软件变得越来越强大，变成了一个平台，可以接入管理很多的网元，就变成了 NMS（网络管理系统）。</p><p data-vmark="b356">UNMS 呢，就是统一网管系统（Unified NMS），也叫综合网管系统。</p><p data-vmark="a23a">说白了，UNMS 就是以一种操作界面和网管系统，实现对 N 种不同设备的全面管理，也就是通杀。</p><p data-vmark="26a0">网管还有两个比较常见的概念，分别是<span class="accentTextColor">北向和南向</span>。</p><p data-vmark="0583">正所谓“上北下南，左西右东”，南向、北向、东西向，其实广泛用于网络拓扑结构中：<span class="accentTextColor">北向，就是上一级；南向，就是下一级；东西向，就是平级（不分东向、西向）</span>。</p><p data-vmark="70a3"><img src="https://img.ithome.com/newsuploadfiles/2021/12/4fafe98c-f7c9-4140-b6dc-b85b879ee802.png" w="817" h="719" alt="网络拓扑结构" title="通信人每天挂在嘴边的“网管”，和网吧的“网管”有什么区别" width="817" height="719" referrerpolicy="no-referrer"></p><p data-vmark="a6fd">所谓北向接口，就是开放给上一级的接口。南向接口，就是开放给下一级的接口。</p><p data-vmark="09cb">网管平台也存在的层层管理，国家级（集团层面）的网管平台，往下就是省级（省公司）的网管平台，再往下，就是市级。以此类推。</p><p data-vmark="3db5">好了，相信大家都搞明白到底什么是“网管”了。希望这篇文章能帮“网管”正名，也希望网管同事们能早日洗白。</p><p data-vmark="8a31">谢谢大家！</p>
