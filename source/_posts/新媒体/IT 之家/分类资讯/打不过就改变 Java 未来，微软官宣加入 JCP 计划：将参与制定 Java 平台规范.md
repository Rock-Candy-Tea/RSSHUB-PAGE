<p data-vmark="d893"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 8 日消息，微软近日在<a href="https://devblogs.microsoft.com/java/microsoft-deepens-its-investments-in-java/" target="_blank">博客</a>宣布，将加大对 Java 的投资力度，<span class="accentTextColor">已签署 Java 规范参与协议，称为 JSPA</span>，正式加入 JCP（Java Community Process）计划。</p><p data-vmark="23c1"><img src="https://img.ithome.com/newsuploadfiles/2021/11/00fab751-82f4-47e1-9978-70dfa112d61e.png" w="954" h="380" title="打不过就改变 Java 未来，微软官宣加入 JCP 计划：将参与制定 Java 平台规范" width="954" height="327" referrerpolicy="no-referrer"></p><p data-vmark="6d4a">甲骨文 JCP 计划主席兼主任 Heather VanCura 表示：“在过去的 23 年里，JCP 计划与国际 Java 开发人员社区合作指导了 Java 平台的规范。JCP 计划欢迎企业、开源、个人和 Java 用户参与。我们很高兴欢迎微软加入 JCP 计划，它继续代表充满活力的 Java 生态系统，我们期待看到微软的贡献。”</p><p data-vmark="fae5">IT之家了解到，<span class="accentTextColor">JCP 是为 Java 技术开发标准技术规范的机制</span>，支持对提交 Java API 建议感兴趣的个人开发人员、客户和供应商之间的协作。</p><p data-vmark="f21a">微软表示，自 2021 年 4 月宣布 Microsoft Build of OpenJDK 以来，微软内部的 Java 使用已经超越了 Minecraft，有超过 50 万个 JVM 在生产中运行数百个内部系统。除了大量的内部 Java 使用外，还有许多客户和开发人员在 Microsoft Azure 和 GitHub 上编码和运行 Java。<span class="accentTextColor">加入 JCP 是微软在帮助塑造 Java 平台的未来方面向前迈出的重要而自然的一步</span>。</p><p data-vmark="775e">此外，2021 年 9 月，甲骨文公司提出了商业 Oracle JDK 二进制文件的两年制 LTS 节奏，微软表示将跟随甲骨文的脚步，<span class="accentTextColor">Microsoft Build of OpenJDK 也将保持两年制 LTS 节奏</span>。</p><p data-vmark="b991">在工具方面，微软的 Visual Studio Code、GitHub Copilot 和 GitHub Codespaces 也都已经支持了 Java。</p><p data-vmark="5172"><img src="https://img.ithome.com/newsuploadfiles/2021/11/10265985-3885-434b-9a7d-97b743e0f1df.gif" w="952" h="938" title="打不过就改变 Java 未来，微软官宣加入 JCP 计划：将参与制定 Java 平台规范" width="952" height="808" referrerpolicy="no-referrer"></p>
