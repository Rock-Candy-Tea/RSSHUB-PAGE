<p data-vmark="da30"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 9 日消息，微软今日推出了 Visual Studio 2022 的正式版，与此同时，为 Visual Studio 2022 for Mac 带来了预览版 3，正式支持了 .NET 6。</p><p data-vmark="7bb1">微软表示，Visual Studio 2022 for Mac 将在未来的预览版中<span class="accentTextColor">很快原生支持苹果 M1 芯片</span>。</p><p data-vmark="4459"><img src="https://img.ithome.com/newsuploadfiles/2021/11/87b5f7fa-4ea1-40d3-a620-3fdd9b900b4c.png" w="1022" h="370" title="微软 Visual Studio 2022 for Mac 预览版 3 发布，将原生支持苹果 M1 芯片" width="1022" height="297" referrerpolicy="no-referrer"></p><h2 data-vmark="a18b">为 Visual Studio 创建本机 macOS 体验</h2><p data-vmark="2b73">Visual Studio 2022 for Mac 的目标是为 Mac 量身打造现代 .NET IDE，提供在 Visual Studio 中相同的高效体验。</p><ul class=" list-paddingleft-2"><li><p data-vmark="904f">Visual Studio 2022 for Mac 正在重写用户界面，以提供舒适的 Mac 用户体验并为辅助技术提供更好的支持。</p></li></ul><p data-vmark="8280"><img src="https://img.ithome.com/newsuploadfiles/2021/11/1f2cbbe3-ee6b-4f75-ad22-068bb0686a59.png" w="1440" h="1016" title="微软 Visual Studio 2022 for Mac 预览版 3 发布，将原生支持苹果 M1 芯片" width="1440" height="579" referrerpolicy="no-referrer"></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="48cd">提高 IDE 的速度和稳定性：随着迁移到本机和迁移到 .NET CLR，微软也在提高 IDE 性能、解决长期存在的 UI 错误并减少崩溃。</p></li><li><p data-vmark="ea66">通过在 Visual Studio IDE 之间更加一致的新 Git 更改窗口改善日常 Git 用户体验。</p></li></ul><h2 data-vmark="6f18">使用 .NET 6 GA 版本开发应用程序</h2><p data-vmark="ee07"><span class="accentTextColor">Visual Studio 2022 for Mac Preview 3 支持 Mac 上最新的 .NET 6 版本</span><span class="accentTextColor">，包括对 C# 10 语言功能的支持</span>。开发者现在可以使用它为 Web、云以及基于终端的控制台应用程序构建应用程序。</p><p data-vmark="ea2c"><img src="https://img.ithome.com/newsuploadfiles/2021/11/1099ba7c-151f-4c85-a1ab-d2684f9cbc02.png" w="448" h="168" title="微软 Visual Studio 2022 for Mac 预览版 3 发布，将原生支持苹果 M1 芯片" width="448" height="168" referrerpolicy="no-referrer"></p><ul class=" list-paddingleft-2"><li><p data-vmark="dcf1">新的编辑器功能：子词导航和新的重构</p></li></ul><p data-vmark="dfeb">此版本中编辑器的一个很棒的新功能是子词导航（subword navigation），以字符串“HelloWorld”为例，当开发者使用 Ctrl+Alt + 向左箭头和 Ctrl+Alt + 向右箭头快捷键时，光标将在子词“Hello”和“World”之间移动。</p><p data-vmark="7e42"><img src="https://img.ithome.com/newsuploadfiles/2021/11/d7b496b7-a923-463d-b3f0-cf8aefc4e5db.gif" w="331" h="121" title="微软 Visual Studio 2022 for Mac 预览版 3 发布，将原生支持苹果 M1 芯片" width="331" height="121" referrerpolicy="no-referrer"></p><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="5fb8">新的代码修复和重构：代码修复和重构是编译器通过灯泡和螺丝刀图标提供的代码建议。要触发快速操作和重构菜单，请按 Option+Enter。以下是预览版 3 提供的新建议：</p></li></ul><blockquote><p data-vmark="1bef">转换为文件范围的命名空间；</p><p data-vmark="bb91">比类型检查更喜欢空检查；</p><p data-vmark="1870">简化 LINQ 表达式。</p></blockquote><p data-vmark="da66"><img src="https://img.ithome.com/newsuploadfiles/2021/11/a8d1d779-5bb3-423c-8c82-6912a8fe88c8.png" w="1024" h="619" title="微软 Visual Studio 2022 for Mac 预览版 3 发布，将原生支持苹果 M1 芯片" width="1024" height="496" referrerpolicy="no-referrer"></p><h2 data-vmark="a725">继续转向原生 macOS UI</h2><p data-vmark="3a9d">此版本继续将 IDE 迁移到完全原生的 macOS UI，<span class="accentTextColor">其中 75% 的 UI 已经迁移</span>。此版本侧重于移动 Git 管理分支对话框、首选项 > NuGet 屏幕以及一组代码格式首选项和项目选项。</p><p data-vmark="cb91"><img src="https://img.ithome.com/newsuploadfiles/2021/11/50c825e9-18c2-4a9a-989b-5914ca459bc4.png" w="1024" h="668" title="微软 Visual Studio 2022 for Mac 预览版 3 发布，将原生支持苹果 M1 芯片" width="1024" height="535" referrerpolicy="no-referrer"></p><h2 data-vmark="222e">即将推出：通过 .NET CLR 在苹果 M1 处理器上本地运行</h2><p data-vmark="2936"><span class="accentTextColor">对苹果 M1 ARM 处理器的本机支持会在即将发布的预览版本中实现</span>。为了启用此支持，微软将使 IDE 在 .NET 6 CLR（公共语言运行时）上运行。这是一个重大的架构变化，为所有使用英特尔和 M1 处理器的开发者带来了改进。在最初的实验室测试版中，大型解决方案的加载时间是 v8.10 版本的一半。</p><p data-vmark="c6ad">IT之家了解到，<span class="accentTextColor">微软计划在 2022 年上半年发布 Visual Studio 2022 for Mac v17.0 的 GA 版本</span>。最终发布日期将用户反馈决定。这是 IDE UI 和核心架构的重大更新，微软将在最终日期临近时分享一个发布候选 (RC) 版本。</p><p data-vmark="81cf">与此同时，用户可以继续使用 Visual Studio 2019 for Mac v8.10 来构建应用程序，包括 .NET 5、Xamarin、Unity 或 .NET Core 3.1。</p><p data-vmark="eba9"><strong>Visual Studio 2022 for Mac 预览版 3：</strong><a href="https://aka.ms/installer/preview" target="_blank">点此下载</a></p>
