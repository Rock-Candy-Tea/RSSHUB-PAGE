<p data-vmark="6480"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 12 月 12 日消息，随着 Visual Studio 2022 的发布，微软正在更改支持从 Visual Studio 安装程序安装 Python 运行时的方式。</p><p data-vmark="a628">根据微软官博的介绍，<span class="accentTextColor">Visual Studio 的生命周期和 Python 的生命周期不一致</span>，这就导致 Visual Studio 有时捆绑了旧版的 Python。</p><p data-vmark="0223">比如，Visual Studio 2019 的目标是 Python 3.7，而 Visual Studio 2022 的目标是 Python 3.9。每年 Python 都会发布新版本，并且二进制安装程序在发布后的 1.5 年内可用于 Windows。这意味着 Python 3.9 的最后一个二进制安装程序更新将在 2022 年 3 月左右发布，之后将不再获得支持。</p><p data-vmark="ccc3">展望未来，最新版本的 Visual Studio 将支持已被广泛采用的最新版本的 Python。在未来的某个时候，<span class="accentTextColor">Visual Studio 2022 将增加对 Python 3.10 和 3.11 的支持</span>。但是，Visual Studio 的旧版本不会支持新版本的 Python。</p><p data-vmark="8837">IT之家了解到，微软对于 Visual Studio 旧版本的做法是，将该 Python 版本标记为“不受支持”，但不会删除这些版本，也不会添加更新的版本。</p>
