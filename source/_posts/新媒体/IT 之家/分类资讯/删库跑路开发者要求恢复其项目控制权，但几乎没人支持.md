<p data-vmark="97ac"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 1 月 25 日消息，此前IT之家报道了知名开源工具 Faker.js 的作者 <a href="https://www.ithome.com/0/597/603.htm" target="_blank">Marak 故意破坏了 GitHub 上的这个开源库</a>，将项目所有代码清空，并在 commit 中留下“endgame”字样。这件事对开源社区产生了较大的影响，后来他的 GitHub 账号还被官方封禁，引发大量争议。后来这个被毁掉的 Faker.js 已经转交由社区维护，项目团队共有 8 名工程师，并且创建了一个官网 fakerjs.dev。</p><p data-vmark="af56"><img src="https://img.ithome.com/newsuploadfiles/2022/1/c1a1578e-908d-42ca-adae-2ee8caabb391.png" w="743" h="585" title="删库跑路开发者要求恢复其项目控制权，但几乎没人支持" width="743" height="585" referrerpolicy="no-referrer"></p><p data-vmark="c180">而近日，这位删库跑路的开发者 Marak 在 Twitter 上发文，希望 NPM 恢复他对其项目的控制权。他在 Twitter 上 @GitHub ，并询问怎么回事，自从 GitHub 取消自己向 NPM 发布和修复 colors.js 项目中的 Infinity Zalgo 错误的权限后，已经过去十天了。他说 GitHub 从来没有回复他的邮件，并称自己有 100 多个包需要维护。</p><p data-vmark="6a18">“每个人都会不时犯下编程错误。没有人是完美的。”他说。</p><p data-vmark="b0b7"><img src="https://img.ithome.com/newsuploadfiles/2022/1/7abdb7a7-e5ad-48f5-a521-514932bd6a57.png" w="743" h="415" title="删库跑路开发者要求恢复其项目控制权，但几乎没人支持" width="743" height="415" referrerpolicy="no-referrer"></p><p data-vmark="44c1">不过，Marak 的诉求并没有得到网友们的支持。有网友直接表示，“在你秀出那种操作之后，你真觉得有人会再让你为这个开源项目做贡献吗？去别处寻求帮助吧，别在这误导他人了，这只会让你变得更糟。”还有网友则反驳称：“朋友，这不是‘编程错误’，你知道的。这是在破坏公共资源。没有人强迫你做出贡献，现在更没有多少人希望你继续‘做贡献’了。”</p>
