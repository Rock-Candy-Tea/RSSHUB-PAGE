<p data-vmark="f240"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 1 月 19 日消息，知名开源工具 Faker.js 的作者 Marak 近日<a href="https://www.ithome.com/0/597/603.htm" target="_blank">故意破坏了 GitHub 上的开源库</a>，将项目所有代码清空，并在 commit 中留下“endgame”字样。</p><p data-vmark="b501">关于这位开发者为什么“删库跑路”，大家可以<a href="https://www.ithome.com/0/597/859.htm" target="_blank">点此查看</a>之前的文章进行了解。</p><p data-vmark="e06f">目前，<span class="accentTextColor">这个被毁掉的 Faker.js 已经转交由社区维护</span>，项目团队共有 8 名工程师，并且创建了一个官网 <a href="https://fakerjs.dev/" target="_blank">fakerjs.dev</a>。</p><p data-vmark="1458"><img src="https://img.ithome.com/newsuploadfiles/2022/1/7deb3643-a0d6-49a3-abac-a9357ab4c464.png" w="433" h="285" title="开发者删库跑路后，Faker.js 成为社区维护的项目：由 8 名工程师管理，还制定了路线图" width="433" height="285" referrerpolicy="no-referrer"></p><p data-vmark="f570">IT之家了解到，Faker.js 是一个十分流行的 Node.js 工具库，用于开发调试，可以在<a class="s_tag" href="https://qiyu.ruanmei.com/" target="_blank">浏览器</a>和 node.js 中生成大量的虚假数据进行模拟测试。目前该项目每周下载量接近 250 万。</p><p data-vmark="31b4"><img src="https://img.ithome.com/newsuploadfiles/2022/1/466f6f3f-4263-42a0-9343-e7597e1b22e9.png" w="725" h="509" title="开发者删库跑路后，Faker.js 成为社区维护的项目：由 8 名工程师管理，还制定了路线图" width="725" height="509" referrerpolicy="no-referrer"></p><p data-vmark="64be">官网文档显示，新的团队接手 Faker.js 以来，已经完成了不少工作：</p><ul class=" list-paddingleft-2"><li><p data-vmark="70ca">在 npm 上的 @faker-js / faker 发布了所有以前版本的 Faker。</p></li><li><p data-vmark="aeab">发布了第 6 个 Alpha 版。</p></li><li><p data-vmark="4c1e">几乎完成了向 TypeScript 的迁移，因此 definitelyTyped 不再需要维护其外部 @types / faker 包。</p></li><li><p data-vmark="1498">创建了一个公共 Twitter 帐户，用于与社区进行交流。</p></li><li><p data-vmark="73a5">发布了第一个官方 <a href="https://fakerjs.dev/" target="_blank">Faker 文档网站</a>。</p></li><li><p data-vmark="89b7">清理了 Prettier、CI、Netlify Deploy Previews 和 GitHub Actions 等工具。</p></li><li><p data-vmark="fc1d">完成了大量的问题分类和 PR 审查。</p></li><li><p data-vmark="35b3">与 Open Collective 取得了联系，并讨论了该项目的过渡计划。</p></li></ul><p data-vmark="fdb2"><img src="https://img.ithome.com/newsuploadfiles/2022/1/eb3e2808-7863-4792-a8c2-92c91c7251e6.png" w="1312" h="1178" alt="64 pull requests were opened. 34 were merged. 44 issues were opened. 25 were closed." title="开发者删库跑路后，Faker.js 成为社区维护的项目：由 8 名工程师管理，还制定了路线图" width="1312" height="736" referrerpolicy="no-referrer"></p><p data-vmark="165e">新团队打算扩展 Faker，不断开发它，让它变得更好，<span class="accentTextColor">将在发布 6.x 后制定路线图</span>，并在下周合并所有 TypeScript Pull Requests。</p><h2 data-vmark="aee8">路线图：</h2><ul class="ai-word-checked list-paddingleft-2"><li><p data-vmark="dab6"> 支持 ESM</p></li><li><p data-vmark="73e7">Browserify => Rollup/Vite</p></li><li><p data-vmark="185d">改进测试基础设施</p></li><li><p data-vmark="0c0a">类型生成文档</p></li><li><p data-vmark="2717">与 Faker 生态系统的现有维护者互动</p></li><li><p data-vmark="d1cb">文档提供交互式 Playground</p></li><li><p data-vmark="4424">兼容 Node 18</p></li></ul>
