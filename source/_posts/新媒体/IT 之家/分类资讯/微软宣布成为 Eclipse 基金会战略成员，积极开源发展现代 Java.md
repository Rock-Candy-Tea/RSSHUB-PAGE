<p><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 8 月 4 日消息 外媒 MSPoweruser 报道，早在 2016 年，微软就作为解决方案成员加入了 Eclipse 基金会，与 Eclipse 社区进行更紧密的合作，为所有开发团队提供一套优秀的工具和服务。今天，微软宣布，它正在推进对 Eclipse 基金会 AISBL 的支持，将其参与范围扩大到战略成员。IT之家获悉，另外，Azure 首席技术官办公室的首席项目经理 Stephen Walli 将加入基金会的董事会。</p><p><img src="https://img.ithome.com/newsuploadfiles/2021/8/4d42eef6-1e66-4ac4-a784-003bf963e844.png" w="901" h="532" title="微软宣布成为 Eclipse 基金会战略成员，积极开源发展现代 Java" width="901" height="484" referrerpolicy="no-referrer"></p><p>以下是微软为什么要扩大其对 Eclipse 基金会的支持的原因：</p><ul class=" list-paddingleft-2"><li><p>Eclipse 基金会仍然是 Java 生态系统的一个重要基石。微软致力于 Java 开发者和 Java 生态系统的健康发展，积极参加 Eclipse Adoptium（以前的 AdoptOpenJDK）和其他项目。作为战略成员扩大我们对基金会的参与，将有助于本着开源精神推进现代 Java 计划。</p></li><li><p>Eclipse 基金会还与 Java 社区的核心部分有着密切的联系，Eclipse IDE、Jakarta EE（Java EE 的继任者）和 MicroProfile 项目都是由该基金会主办的。对于微软及其合作伙伴来说，Eclipse 基金会是 AdoptOpenJDK 的合理选择，以继续履行这一使命。作为一个厂商中立的多厂商倡议，Eclipse Adoptium 继续成为基于 OpenJDK 源代码的完全兼容、高质量的 Java 运行时分发的领先供应商。</p></li><li><p>Eclipse 基金会正在通过工作组扩大其作用，这些工作组中有许多对微软及其合作伙伴很重要。最近围绕 Eclipse Dataspace Connector 和 Eclipse Tractus-X 的工作是 Eclipse 基金会在工作组中开始的新工作的例子，微软有兴趣参与其中。</p></li></ul>
