<p data-vmark="e793"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 11 月 16 日消息，此前据 MSPoweruser 报道，几年来，微软一直在争取迫使 Windows 用户使用 Edge <a class="s_tag" href="https://qiyu.ruanmei.com/" target="_blank">浏览器</a>，经常地迫使链接在浏览器中使用 microsoft-edge://协议打开。</p><p data-vmark="df5d">例如，这个协议被用于内置的 Windows 搜索和“新闻和兴趣”小组件中，而大多数用户希望这些来源的链接能在他们的默认浏览器中打开。</p><p data-vmark="c562"><img src="https://img.ithome.com/newsuploadfiles/2021/11/bac851c7-9e8c-42b3-941d-39c2232e035d.jpg@s_2,w_820,h_586" title="Win11 封禁 EdgeDeflector 等修改默认浏览器链接，微软回应：部分体验是为了端到端设计的" referrerpolicy="no-referrer"></p><p data-vmark="70ed">用户此前可以通过使用像 Edge Deflector 这样的工具软件进行反击，对 microsoft-edge:// 协议进行修改，将链接引导到任意浏览器。</p><p data-vmark="4375">但现在 Edge Deflector 开发者报告说，<span class="accentTextColor">微软已经使 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 上协议无法被劫持</span>，这意味着 Edge Deflector 应用程序不再有效。</p><p data-vmark="feba">据 The Verge 报道，微软发言人已对此情况作出回应：“<a class="s_tag" href="https://win10.ithome.com/" target="_blank">Windows 10</a> 和 Windows 11 中提供了某些<span class="accentTextColor">端到端的客户体验</span>，比如任务栏的搜索体验就是这样一种端到端体验，这种体验不是为了重定向而设计的。<span class="accentTextColor">当我们意识到不正确的重定向时，会发布修复程序来修复它</span>。”</p><p data-vmark="a697"><img src="https://img.ithome.com/newsuploadfiles/2021/11/75386630-9ed3-4b0c-b985-c94462a825c9.jpg@s_2,w_820,h_554" w="1200" h="810" title="Win11 封禁 EdgeDeflector 等修改默认浏览器链接，微软回应：部分体验是为了端到端设计的" srcset="https://img.ithome.com/newsuploadfiles/2021/11/75386630-9ed3-4b0c-b985-c94462a825c9.jpg 2x" width="1200" height="554" referrerpolicy="no-referrer"></p><p data-vmark="f8a2">IT之家了解到，这个“修复”是即将到来的 Windows 11 更新的一部分，该更新将于周五晚些时候为 Beta 和 Release Preview 用户提供。</p>
