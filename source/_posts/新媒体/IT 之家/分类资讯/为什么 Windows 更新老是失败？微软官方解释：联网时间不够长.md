<p data-vmark="e6ec"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 1 月 29 日消息，微软近日宣布，它将强制把兼容的 <a class="s_tag" href="https://win10.ithome.com/" target="_blank">Windows 10</a> 设备更新到 Windows 10 版本 21H2，以训练自己的基于机器学习 (ML) 的 Windows 更新模型。去年 11 月，微软还加快了 <a class="s_tag" href="https://win11.ithome.com/" target="_blank">Windows 11</a> 的推出，因为它对所有“积极”的反馈感到非常满意。</p><p data-vmark="6b65">尽管微软已不断优化 Windows 更新机制，但有部分用户仍然发现，<span class="accentTextColor">自己的 Windows 设备似乎无法正确更新</span>。近日，微软对这些用户的疑问进行了<a href="https://techcommunity.microsoft.com/t5/windows-it-pro-blog/achieve-better-patch-compliance-with-update-connectivity-data/ba-p/3073356" target="_blank">解答</a>。</p><p data-vmark="9356">微软表示，Windows 更新需要计算机连接到 Windows Update 服务一段时间才能成功交付，将其称之为 Update Connectivity 数据。官方建议，<span class="accentTextColor">设备至少需要连续连接 2 个小时的互联网（注意：连续连接），在更新发布后，还需要累计 6 个小时的互联网连接</span>，才能够得到可靠的 Windows 更新。</p><p data-vmark="a5ee">IT之家了解到，用户可以在 Microsoft Intune 上的 Devices > Monitor 下找到 Update Connectivity 数据，然后选择功能更新失败或 Windows 加速更新失败，如下图所示，就可以看到更新失败的原因。</p><p data-vmark="5b79"><img src="https://img.ithome.com/newsuploadfiles/2022/1/62c3d94b-d207-4b81-a55b-fe527f904b6c.jpg@s_2,w_820,h_294" w="1429" h="512" alt="Update failed devices on Microsoft Intune" title="为什么 Windows 更新老是失败？微软官方解释：联网时间不够长" srcset="https://img.ithome.com/newsuploadfiles/2022/1/62c3d94b-d207-4b81-a55b-fe527f904b6c.jpg 2x" width="1429" height="294" referrerpolicy="no-referrer"></p><p data-vmark="f869">微软鼓励更新失败的设备保持联网状态，以便 Windows 更新顺利进行。</p>
