<p data-vmark="ee28"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 7 月 27 日消息，谷歌工程师 Chandler Carruth 此前在 CppNorth 大会上宣布 <a href="https://www.ithome.com/0/630/631.htm" target="_blank">Carbon 编程语言</a>，该语言已在 GitHub 开源，并称其是 C++ 的实验性继任者。</p><p data-vmark="b89a" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/7/2f3a725b-4f14-400d-beb6-2f79dba09ef1.png" w="1440" h="1080" title="C++ 之父评谷歌推出替代品 Carbon：语言太新且规范不足，无法提供有意义的技术评论" width="1440" height="615" referrerpolicy="no-referrer"></p><p data-vmark="c322">近日，外媒 devclass 询问了 C++ 之父 Bjarne Stroustrup 对谷歌 Carbon 有什么看法，得到的回应是：</p><blockquote><p data-vmark="ebc7">总有新的语言试图成为 C++ 的继任者。我欢迎对编程语言和编程风格进行实验，但我不想引发争议。批评已建立的语言很容易，因为我们知道它们的问题，但通常很难在不产生语言规则、库和管理方面的全新问题的情况下提供替代方案。Carbon 太新了，而且规范不足，以至于我无法真正做出有意义的技术评论。</p></blockquote><p data-vmark="9850">IT之家此前报道，据 Carbon 的 GitHub 仓库页面，Carbon 从根本上来说是一种后继语言方法，而不是逐步发展 C++ 的尝试。它是围绕与 C++ 的互操作性，以及现有 C++ 代码库和开发者的大规模采用和迁移而设计的，谷歌将 Carbon 类比为 <span class="accentTextColor">TypeScript、Kotlin</span>，后两者是 JavaScript 和 Java 的增强。</p><p data-vmark="e53e">谷歌工程师 Carruth 并不认为 Carbon 是 C++ 的竞争对手。他在 C++ North 被问及 Carbon 将如何与 C++ 未来改进版本竞争时表示：“<span class="accentTextColor">我们的目标不是竞争，而是探索用 C++ 做不到的事情</span>。”</p><p data-vmark="f5de">此外，在被问及为什么 Carbon 没有从一开始就构建内存安全性时，Carruth 解释到：“不幸的是，C++ 非常不安全，所以我们从一个相当低的标准开始。<span class="accentTextColor">该项目将迁移优先于安全性，但其目的是在以后提高内存安全性</span>。从长远来看，我们希望至少与 Go 或 Swift 一样安全。”Carruth 接着谈到了 Rust，称 Rust 来自另一端，也就是优先于内存安全性，但其可能永远无法与 C++ 实现足够的互操作性以实现平滑迁移。</p>
