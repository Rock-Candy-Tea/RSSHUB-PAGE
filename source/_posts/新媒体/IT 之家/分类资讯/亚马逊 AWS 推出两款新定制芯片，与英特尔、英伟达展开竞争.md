<p data-vmark="6d40">12 月 1 日消息，美国当地时间周二，<span class="accentTextColor">亚马逊旗下云计算部门 AWS 推出了两款新的定制计算芯片</span>，旨在帮助其客户降低使用英特尔和英伟达芯片的成本。</p><p data-vmark="79b9">AWS 将其计算能力出租给客户，如今是世界上最大的云计算服务提供商，也是数据中心芯片的最大买家之一，<span class="accentTextColor">2020 年销售额达 453.7 亿美元</span>。自从 2015 年收购了名为 Annapurna Labs 的初创公司以来，AWS 始终在致力于开发自己的定制芯片。</p><p data-vmark="b55a"><img src="https://img.ithome.com/newsuploadfiles/2021/12/f1efbb91-cfe9-4f18-8807-3e81bc42fac7.jpg" w="660" h="374" title="亚马逊 AWS 推出两款新定制芯片，与英特尔、英伟达展开竞争" width="660" height="374" referrerpolicy="no-referrer"></p><p data-vmark="cc18">周二，AWS 发布了第三代基于 Arm 架构的 Graviton 芯片，旨在与英特尔和 AMD 中央处理器竞争。Graviton3 比其前身快 25%，<span class="accentTextColor">浮点性能提高 2 倍，机器学习工作负载的速度提高 3 倍</span>。AWS 还承诺，<span class="accentTextColor">新芯片将减少 60% 的电力消耗</span>。</p><p data-vmark="1efd">亚马逊弹性计算云副总裁戴夫・布朗（Dave Brown）表示，该公司预计 Graviton3 的性价比将大大超过英特尔芯片。</p><p data-vmark="cc08">AWS 云计算战略师杰夫・巴尔（Jeff Barr）指出，这款芯片对于各种各样的工作负载都非常有用，包括计算密集型的 HPC、批处理、电子设计自动化 (EDA)、媒体编码、科学建模、广告服务、分布式分析和基于 CPU 的机器学习推理。</p><p data-vmark="c353"><img src="https://img.ithome.com/newsuploadfiles/2021/12/76894515-c6aa-4327-8a70-5f8abdc0930d.jpg" w="660" h="359" title="亚马逊 AWS 推出两款新定制芯片，与英特尔、英伟达展开竞争" width="660" height="359" referrerpolicy="no-referrer"></p><p data-vmark="bfe4">AWS 还表示，名为 Tradium 的新型芯片将很快向客户推出，该芯片旨在训练机器学习计算机模型，将与英伟达的同类芯片展开竞争。AWS 预计 Tradium 将可帮助训练机器学习模型，可被用于图像识别、自然语言处理、欺诈检测和预测等，成本比英伟达旗舰芯片低 40%。</p><p data-vmark="d3ba">不过，AWS 仍与英特尔、AMD 和英伟达密切合作。举例来说，它正在与英伟达合作，将其 Graviton 处理器搭配使用，为 <a class="s_tag" href="https://android.ithome.com/" target="_blank">Android</a> 游戏开发商提供将其游戏加载到设备上的方式。布朗说，AWS 希望通过提供额外的芯片选择来保持在云计算市场的竞争力。</p><p data-vmark="416f">布朗表示：“我们已经在性能方面发起了挑战。我相信，在未来几年里，你会看到所有这些公司（如英特尔、AMD）等，在性价比方面都会有更好的表现。这是他们必须做的事情，这样才能让我们的客户满意。”</p><p data-vmark="e97d">市场研究公司 Gartner 副总裁兼分析师拉杰・巴拉 (Raj Bala) 称，从长远来看，芯片公司应该认真对待来自 AWS 的竞争。</p><p data-vmark="23f4">目前，许多云计算客户希望使用英特尔和英伟达的芯片，因为几十年来软件都是在它们上面运行的。巴拉说，只有能够处理重写自己软件复杂性的早期采用者才可能尝试新的 AWS 芯片。</p><p data-vmark="8805">但 15 年前 AWS 上线时，情况也是如此，当时使用它的是规模更小技术客户。该公司最终扩展为主流公司，现在正朝着思科等传统公司那样的规模发展。</p>
