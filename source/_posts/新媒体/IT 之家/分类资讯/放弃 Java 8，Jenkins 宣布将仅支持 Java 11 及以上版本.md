            <p data-vmark="dfac"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 7 月 2 日消息，开源 Devops 工具 Jenkins 在<a href="https://www.jenkins.io/blog/2022/06/28/require-java-11/" target="_blank">官方博客平台</a>宣布，从 6 月 28 日发布的 Jenkins 2.357 和将于 9 月发布的 LTS 版本开始，<span class="accentTextColor">Jenkins 需要 Java 11 才能使用，将放弃 Java 8</span>。</p><p style="text-align: center;" data-vmark="58c9"><img src="https://img.ithome.com/newsuploadfiles/2022/7/4bc67b85-8e81-4013-97e0-729d0b633069.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzIwNy5wbmc=,g_3,x_21,y_21,a_0,t_100" w="1343" h="622" title="放弃 Java 8，Jenkins 宣布将仅支持 Java 11 及以上版本" width="1343" height="380" referrerpolicy="no-referrer"></p><p data-vmark="1521">Jenkins 是一款流行的 Java 平台持续集成工具，自 2005 年成立以来，Jenkins（当时的 Hudson）项目经历了多次 Java 迁移，而此次迁移到 Java 11 符合此前的历史数据规律，同时将是改变最大的一次迁移。</p><p style="text-align: center;" data-vmark="0000"><img src="https://img.ithome.com/newsuploadfiles/2022/7/0a8cca8d-2ab2-4cf9-bb96-ad772f86a057.png" w="795" h="563" title="放弃 Java 8，Jenkins 宣布将仅支持 Java 11 及以上版本" width="795" height="563" referrerpolicy="no-referrer"></p><p data-vmark="2ce1">IT之家了解到，Jenkins 项目虽然在可预见的未来可能会保留 Java 8 支持，但官方称出于几个原因，这样做是不谨慎的。首先，Jenkins 项目使用的许多关键第三方库（例如 Jetty、JGit、Spring Framework 和 Spring Security）<span class="accentTextColor">开始需要更新版本的 Java</span>，而停留在 Java 8 上会使 Jenkins 项目面临最终无法使用的风险。</p><p data-vmark="48b8">此外，<span class="accentTextColor">近年来 Java 平台进行了显著的运行时改进</span>。例如，LinkedIn 在迁移到 Java 11 时看到了性能明显改进，而 Adoptium 在迁移到 Java 11 时看到了内存使用改进。Jenkins 官方还建议开发者直接使用 Java 17，官方称 Java 17 是比 Java 11 更可靠的选择。</p>
