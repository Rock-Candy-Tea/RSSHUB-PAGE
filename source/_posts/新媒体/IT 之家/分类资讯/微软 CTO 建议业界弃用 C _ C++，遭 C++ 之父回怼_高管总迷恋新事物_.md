<p data-vmark="2bcb"><a class="s_tag" href="https://www.ithome.com/" target="_blank">IT之家</a> 9 月 22 日消息，9 月 20 日，微软 Azure 首席技术官（CTO）Mark Russinovich 在推特上称，<span class="accentTextColor">建议业界弃用 C / C++ 编程语言</span>。</p><p data-vmark="fc97">Mark Russinovich 表示，是时候停止使用 C / C++ 创建任何新项目了，如果需要 non-GC 语言的场景，<span class="accentTextColor">建议使用 Rust 替代</span>。为了安全和可靠，业界应该宣布 C / C++ 已被弃用。有大量的 C / C++ 将被维护和发展数十年（或更长时间），对于新工具，他会偏向于使用 Rust。</p><p data-vmark="aa90" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/9/0c1cd174-fb6b-4bd9-8819-25bfb79a1077.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzExMi5wbmc=,g_3,x_11,y_11,a_0,t_100" w="730" h="341" title="微软 CTO 建议业界弃用 C / C++，遭 C++ 之父回怼“高管总迷恋新事物”" width="730" height="341" referrerpolicy="no-referrer"></p><p data-vmark="3c2b" style="text-align: center;"><img src="https://img.ithome.com/newsuploadfiles/2022/9/3a4f4c9d-db5c-4bbe-8c13-ab04e1c855ee.png?x-bce-process=image/watermark,image_aW1nL3dhdGVybWFyay9xYy9xYzExMy5wbmc=,g_9,x_11,y_11,a_0,t_100" w="735" h="185" title="微软 CTO 建议业界弃用 C / C++，遭 C++ 之父回怼“高管总迷恋新事物”" width="735" height="185" referrerpolicy="no-referrer"></p><p data-vmark="6775">Rust 被许多人认为是 C++ 的后继者，并连续七年成为 StackOverflow 年度调查中最受欢迎的编程语言。此外，Rust 也有望并入 Linux 6.1 中，也因此备受期待。</p><p data-vmark="e337">近年来，苹果、微软、谷歌、亚马逊、Meta 等科技巨头都在使用 Rust，<span class="accentTextColor">而且自 2019 年以来，微软一直在谈论抛弃 C / C++ 并探索 Rust</span>，还在开发自己的面向云的内存安全编程语言，称为 Project Verona。因此，Russinovich 呼吁弃用 C / C++ 并非没有先例。</p><p data-vmark="f7d8">据微软称，<span class="accentTextColor">自 2006 年以来，它修补的 CVE 中约有 70% 是由于内存安全问题造成的</span>。消除这些错误将明显提高软件安全性，同时降低漏洞修复的成本。</p><p data-vmark="caee">IT之家了解到，外媒 The Register 向微软询问，是否 Russinovich 的建议在全公司范围内被采纳，<span class="accentTextColor">微软官方拒绝置评</span>。</p><p data-vmark="0de3">不过，C++ 之父 Bjarne Stroustrup 回应了该观点，他表示：</p><blockquote><p data-vmark="5d53">人们 —— 尤其是高管 —— 迷恋那些承诺可以让他们的生活更轻松的新事物，这并不少见。此外，支持新事物远比解决旧工具和知名工具的已知问题更令人兴奋。</p><p data-vmark="82dc">不过，新语言通常需要多年时间和重大努力才能在其广泛的应用领域与成熟语言相匹配。爱好者很少看到这一点，并且他们的评论往往是片面的。</p><p data-vmark="98c8">在许多情况下，安全性显然至关重要，因此多年来我一直致力于提高 C++ 的安全性。</p></blockquote><p data-vmark="3b20">Stroustrup 表示，现在可以在 ISO C++ 中实现有保证的完美类型和内存安全，而包括 Rust 在内的每一种“安全”语言都存在允许不安全代码的漏洞。替换它们或者只是让它们安全是一项艰巨的任务，必须逐步做到这一点，否则大量不安全的 C 和老 C++ 代码将存在永远。</p>
